Graal 的 Substrate VM 是一个支持将 Java 程序编译为独立可执行文件的框架。自其发布以来，我们一直对这种做法相较于标准字节码加 JVM 部署工具链的潜在优势深感兴趣。首先，对于命令行工具（CLI）等启动速度至关重要的场景，原生镜像（native image）具有显著的性能优势。其次，Substrate VM 有助于降低长期运行应用的整体内存占用。最后，Substrate VM 与我们的软件发布和部署理念高度契合：相比于配置可变的长期运行应用服务器，我们更倾向于频繁部署和重新部署小型且不可变的镜像。

在探索过程中，我们开发了两个小型库：

  • 一个 Gradle 插件，利用 Substrate VM 的 native-image 工具链构建独立可执行文件。
  • 一个 Java 注解处理器，能够根据 Java 注解生成 Substrate VM 的反射调用配置文件。（要在提前编译（AOT）中使用反射，Graal 要求显式的反射调用，或者提供一个配置文件来记录可能用于反射调用的类、构造函数、方法和字段。）

该 Gradle 插件可以轻松地从 Gradle 项目生成原生镜像，例如：

// Main.java
package com.palantir.test;

public final class Main {
    public static final void main(String[] args) {
        System.out.println("Hello, Graal!");
    }
}

// build.gradle
apply plugin: 'com.palantir.graal'
graal {
    mainClass 'com.palantir.test.Main'
    outputName 'hello-graal'
    graalVersion '1.0.0-rc8'
}

nativeImage Gradle 任务会在 build/graal/hello-graal 目录下构建可执行文件。

这项技术尚处于起步阶段，我们在该领域还有许多悬而未决的问题，例如 Graal 和 Substrate VM 在稳定性、性能特征以及性能分析与调试工作流方面的表现。我们诚邀您通过试用我们的 Gradle 工具或分享您的经验与心得，与我们一起踏上这段探索之旅。GitHub 的 issue 和 pull request 是进行讨论和改进的最佳场所，期待您的反馈！
