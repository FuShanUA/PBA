Palantir 与 SpecterOps 的合作

在 Palantir，我们致力于开发能够解决现实问题的软件。我们的软件曾用于阻止恐怖袭击、研发新药、在全球金融市场中获取优势、打击儿童贩卖等。针对这些关键用例，我们必须防止产品被滥用，确保客户数据安全，并最终提高对手的攻击成本——无论他们在何处活动。

Palantir 的信息安全计划有三个驱动所有工作的核心目标：

  • 让 Palantir 更安全。
  • 让我们的客户更安全。
  • 让世界更安全。

我们在实现这些目标方面取得实质性进展的能力，与我们对企图加害我们的对手的了解和认知直接相关。正是这种对深入了解对手及模拟能力的追求，促使我们与 SpecterOps 建立了紧密且持久的合作关系。作为红队行动、攻防研究及工具开发领域的领导者，SpecterOps 在我们的环境中部署时展现了极强的对抗能力。随着我们的防御控制、告警和能力日益成熟，SpecterOps 也不断识别出新的攻击手法、规避技术和绕过方式。

随着时间推移，Palantir 信息安全团队与 SpecterOps 建立了紧密的关系，并对他们带来的研究、开源项目及持续创新产生了深深的相互敬意。我们非常高兴地宣布，与 SpecterOps 正式建立合作伙伴关系，以助力 Palantir 的信息安全计划。尤为重要的是，这一合作关系代表着双方对深度协作的共同承诺，最终将推进 Palantir 信息安全团队的核心目标。该承诺的部分细节如下。

致力于让 Palantir 更安全

自初步合作以来，SpecterOps 在加速和提升 Palantir 的信息安全态势方面发挥了关键作用。双方的合作最初侧重于针对我们的企业和云基础设施的对抗性红队演练。随着红队演练愈发高级，我们被迫提升安全控制和检测能力。我们的网络防御者迅速采用了 SpecterOps 首创的防御工具和技术（例如 PowerForensics、BloodHound、《活动目录研究》等）。反过来，红队也投入研发更隐蔽、更有效的技术和工具（例如 Empire 的开发、向 C# 工具的过渡等）。通过这种创新、研究和测试的持续循环，我们的安全计划得到了极大发展。

我们坚信，与 SpecterOps 持续进行的对抗性评估不仅会让 Palantir 更加安全，还将实质性地有助于保护我们的客户和更广泛的社区。

致力于让我们的客户更安全

除了 Palantir 的企业网络外，我们的信息安全团队还负责保护 Palantir 的产品、客户数据以及各种托管环境。SpecterOps 通过参与长期的演红队演练，对这些工作起到了至关重要的作用。最显著的是，红队操作员被定期部署到模拟客户环境中，以评估安全控制和检测能力。在许多情况下，这是在评估在受限执行环境中持久化或突破隔离的能力，或者在隔离网络和基础设施之间进行横向移动的能力。

值得注意的是，SpecterOps 开发的所有技术都与更广泛的社区进行了共享。在这些模拟演练中发现、开创或完善的每一项技术都没有保密，这使得网络防御者和产品工程师能够不断提升自己，从而更好地保护我们的客户。这种对透明度的执着不仅使我们能够构建强大的安全控制，最终还通过出版物和演讲回馈了开源社区（见下文）。

致力于让世界更安全

从我们最初的开源项目（《用于网络防御的 Windows 事件转发》、《企业级 osquery 部署》、《告警与检测策略框架》、《基于 HashiStack 的企业安全》）开始，我们的团队就将致力于让世界更安全作为核心使命的一部分。我们的对手不仅仅针对 Palantir，如果我们能让他们在攻击其他组织时更加困难，我们就应该去做。在内部，我们秉持“开源优先”的理念，每个项目启动时都会先问一个问题：“为什么我们不能把这个开源？”我们希望并打算提高我们对社区贡献的频率、深度和价值——尤其是通过与 SpecterOps 的联合合作。

示例

作为我们如何合作以及 SpecterOps 如何迫使我们团队创新的例子，我将分享最近两次对抗性评估中的见解。在这两次行动中，我们的网络防御都遭到了破坏和攻击。结果，我们的网络防御者不得不开发新的安全控制和告警/检测策略来对抗红队操作员。最终，这促成了安全态势的改善、风险的降低以及知识的积累，我们计划在《告警与检测策略框架》之外尽快将这些知识开源。

macOS 攻击技术

当我们最初委托 SpecterOps 对我们庞大的 macOS 工作站集群进行红队演练时，公开可用的 macOS 后渗透工具寥寥无几。他们没有依赖相对不完整的工具集，而是彻底重构了 PowerShell Empire，使其也能支持基于 Python 的 macOS 智能体。借助他们全新的 macOS 工具，我们不仅获得了现有攻击类别的变种（以此加强当前检测），他们还开发并提供了我们未曾预料到的新技术。我们从 SpecterOps 在 macOS 环境中投入的精力中获益匪浅，我们也很自豪能够成为他们的孵化器，帮助他们进一步拓展本已丰富的技术手段。此后，SpecterOps 也扩展了他们在 macOS 威胁狩猎方面的专业知识，我们期待能从中受益。

Windows 攻击技术

除了 macOS 工作站，Palantir 还维护着一个由 Windows 服务器和工作站组成的大型活动目录域。在该环境中，最初我们的对手从传统的 Windows PE 文件格式转向采用基于 PowerShell 的技术，例如 Empire 和 Cobalt Strike。在早期版本的 PowerShell 中，日志记录和遥测技术并不成熟，对手能够以隐蔽的方式执行任意有效载荷。这一切在 2015 年 6 月发生了改变，微软发布了《Powershell ❤ 蓝队》的博文并发布了 PowerShell v5。此次更新包含强大的安全控制和遥测源（例如，PowerShell 脚本块日志记录、转录日志记录、AMSI 等），使得基于 PowerShell 的攻击技术对攻击者来说充满风险。我们立即在全网络部署了 PowerShell v5，移除了现有的 v2 版本，并设置了版本回退告警。

SpecterOps 的操作员起初受挫，因为他们的工具严重依赖 PowerShell，而 v5 的变化使得执行操作变得异常嘈杂。然而，这种挫折并未持续太久，他们迅速更换了工具，并开发了非托管 PowerShell，这是一种在任意进程中实例化 PowerShell 执行的技术。此外，SpecterOps 开始使用反射、本地组策略编辑以及其他技术来完全破坏和篡改 PowerShell 日志记录。这最终促成了利用系统访问控制列表识别工作站失陷的研究与实施，以及识别异常 PowerShell 宿主进程的告警/检测策略。

最终，SpecterOps 发现在我们的环境中，PowerShell 环境对其操作极具敌意，于是他们作为 GhostPack 项目的一部分开发并实施了基于 C# 的工具。目睹 SpecterOps 为响应我们实施更新的安全控制而使其攻击技术多样化和创新，让我们受益匪浅，这也印证了一点：检测工程永远是一个过程，而非终点。

最后，我们也逐渐欣赏到 SpecterOps 研究人员开发并阐述的安全研究。例如，他们在破坏 Windows Defender 应用控制（即 Device Guard）和代码签名方面的工作，促使我们更正式地思考在 diverse 环境中什么才被视为可信。建立信任基准也有助于开发出比供应商提供的更具体、更激进的检测手段。
