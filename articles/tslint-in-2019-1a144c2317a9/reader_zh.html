# TSLint in 2019 Palantir 是 TSLint 的创建者和主要维护者，后者是 TypeScript 编程语言的标准 linter。鉴于 TypeScript 社区正致力于在 TypeScript 和 JavaScript 语言间实现统一的开发者体验，我们也致力于支持 TSLint 和 ESLint 的融合；在这篇博文中，我们将阐述相关工作的原因与具体做法。 ## TSLint 与 ESLint 现状 目前，TSLint 实际上已成为 TypeScript 代码仓库及 TypeScript 自身实现的标准 linter。TSLint 生态系统包含一套核心规则集，以及由社区维护的自定义规则和配置包。 与此同时，ESLint 是标准的 JavaScript linter。与 TSLint 类似，它也由一套核心规则集加上社区维护的定制化扩展组成。ESLint 支持许多 TSLint 所缺失的功能，例如条件 lint 配置和自动缩进。反之，ESLint 规则（至少目前）无法受益于 TypeScript 语言服务提供的静态分析和类型推断，因此无法捕获 TSLint 语义规则所涵盖的某类 bug 和代码异味。 ## TypeScript + ESLint TypeScript 团队的战略方向是实现“让每个桌面、每个家庭、每位 JS 开发者都能用上类型”。换言之，其方向是利用类型和静态分析等 TypeScript 特性，逐步丰富 JavaScript 开发者体验，直至 TypeScript 和 JavaScript 的开发者体验或许最终走向融合。

显然，linting 是 TypeScript/JavaScript 开发者体验的核心组成部分，因此 Palantir 的 TSLint 团队与 TypeScript 核心团队在雷德蒙德会面，探讨了 TypeScript/JavaScript 的融合对 linting 意味着什么。TypeScript 社区打算从 JavaScript 开发者当前所在的位置去迎合他们的需求，而 ESLint 正是 JavaScript linting 的首选工具。为了避免 TypeScript 领域的 linting 工具走向分裂，我们计划弃用 TSLint，转而将精力集中于改进 ESLint 对 TypeScript 的支持。我们相信这是正确的前进方向，原因既有战略考量，也有务实因素： - **开发者易用性：** JavaScript 开发者迁移至 TypeScript 的障碍之一，是从 ESLint 迁移到 TSLint 的不小成本。允许开发者从现有的 ESLint 配置起步，并逐步加入特定于 TypeScript 的静态分析，能够降低这一门槛。 - **凝聚社区：** ESLint 和 TSLint 的核心目标一致：提供卓越的代码 linting 体验，具备强大的核心规则集和丰富的插件点。既然 TypeScript 解析已能在 ESLint 中使用，我们认为最好的做法是让社区实现标准化，而不是通过相互竞争的工具来维持碎片化。 - **更高性能的分析基础设施：** ESLint API 允许更高效地实现某些类别的检查。尽管重构 TSLint 的 API 并非不可行，但利用 ESLint 的架构并将开发资源投入到其他地方似乎是更明智的做法。 - **投资假设：** 在 Palantir 内部，TSLint 编码了我们在 TypeScript 语言方面的专业技能；正因如此，其功能集和架构已经成熟并进入稳定状态。因此，我们很难再证明投入所需级别的精力来跟进社区贡献和需求是合理的。 ## 后续计划 Palantir 将通过提供一系列功能和插件贡献，支持 TSLint 社区向 ESLint 的平滑过渡（在此感谢 James Henry 及其他贡献者推动了这项工作！），例如： - **支持并编写文档以指导在 TypeScript 中编写 ESLint 规则：** 请参阅此 typescript-eslint 议题线。 - **typescript-eslint 中的测试基础设施：** ESLint 内置的规则测试器难以使用，且测试用例语法可读性差。我们希望将类似于 TSLint 的测试基础设施引入该项目，以确保 TSLint 规则开发体验不出现倒退。 - **基于类型检查器的语义规则：** 移植并添加使用 TypeScript 语言服务的新规则。 一旦我们认为 ESLint 在功能上已与 TSLint 持平，我们将弃用 TSLint 并帮助用户迁移至 ESLint；在此之前，我们的主要任务包括： - **持续的 TSLint 支持：** TSLint 中最重要的维护任务是确保其与新编译器版本及新功能的兼容性。 - **TSLint → ESLint 兼容包：** 一旦 ESLint 的静态分析检查与 TSLint 不相上下，我们将发布一个 eslint-config-palantir 包，作为我们 TSLint 规则集的直接替代 ESLint 方案。 过去几年里，我们很高兴看到 TypeScript 和 TSLint 的采用率不断增长，我们也非常兴奋能为 TypeScript 在 Web 开发生态系统中更具影响力的未来贡献力量！如果您有任何疑问或顾虑，或者愿意提供帮助，请在这篇文章下留言或通过 GitHub Issues 与我们联系。 ## 作者 Adi D.、John W.、Robert F.、Stephanie Y.
