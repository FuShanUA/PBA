  • Open-Source Flexibility

Open-Source Flexibility

Rubix App Platform is not a custom or proprietary Linux or Kubernetes distribution.

Instead, it uses open-source or commercial off-the-shelf (COTS) Linux and Kubernetes distributions, configured to meet stringent security and compliance standards.

Linux & Kubernetes Synergy

To ensure optimal performance and security, Rubix App Platform requires both the Linux OS and Kubernetes distribution to be used together.

More Features

Immutable Node Assumption

Rubix App Platform operates under the assumption that nodes are immutable. Patches and updates to the kernel, OS, and Kubernetes node are applied by updating the host image and restarting the host. This approach ensures that all nodes remain consistent and secure, minimizing the risk of configuration drift and vulnerabilities.

Application and Service Availability

Rubix uses Kubernetes PodDisruptionBudgets (PDBs) to manage application availability. The RCP enforces limited node drainage delays, forcibly evicting blocking pods after a set time to ensure updates and patches are applied without major impact on service availability.
