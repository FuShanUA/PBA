How It Works

Pipeline Builder is designed around speed and simplicity.

Spend less time worrying about wrangling code, maintaining libraries, running builds, and managing repository complexity — and focus on the projects that matter.

  • Add Data
  • Transform Data
  • Collaborate with Branches
  • Merge Changes
  • Deploy

Add Data

  • To get started with Pipeline Builder, simply select datasets in Palantir Foundry.

  • Foundry makes it easy to ingest data from any source system — including structured, unstructured, IoT, and geospatial source systems.

  • Alternatively, you can also drag-and-drop raw files into Foundry for quick one-time uploads.

Transform Data

  • Once datasets are added, you can now transform them without using code.

  • With Pipeline Builder’s powerful set of point-and-click functions, you can parse raw XML, JSON, PDF and other formats into structured tables and combine datasets through joins and unions.

  • You can also apply a vast and growing set of functions to data, including: clean, transform, derive, pivot, encrypt, aggregate, and filter. Builder will process images, calculate geospatial distances, perform finance operations, and much more. 

Collaborate with Branches

  • If you are contributing to an existing pipeline, you can create a branch to work safely in isolation, without affecting the main version of the pipeline. 

  • Users can collaborate across multiple branches and even make branches of branches. This flexibility in branching facilitates rapid iteration in parallel, development of new functionality, bug fixes, and other pipeline improvements.  

Merge Changes

  • When you complete work on a branch, you can propose merging the changes back into the pipeline (or branch) that you branched from. Side-by-side comparisons show exactly what has changed. 

  • Pipeline Builder will check for any merge conflicts — including those caused by other users editing the same transforms as you — and will prompt you to resolve them.

  • At all times, you can access a complete history of previous versions, edits, and merges — allowing you to collaborate at scale and iterate with confidence.

Deploy

  • Pipeline Builder conducts immediate checks across the pipeline to ensure your proposed logic produces all of the required columns — preventing errors that lead to costly downtime. 

  • Once validated, the merge completes and the branch is deployed to produce updated pipeline outputs.

  • Pipeline Builder can deploy to a variety of outputs — including batch, streaming, Palantir Gotham, time series, and modeling. 

  • Builder guides you through the process. For example, if you choose a time series target, Builder will guide you through the required inputs, such as the individual ticks and metadata for each series. These rails enable you to focus on the logic rather than the implementation details. 

Engineered for Speed, Scale, & Stability

Independent Pipeline Logic

Pipeline Builder can connect to different logic execution engines, including Spark, Flink, and more.

Full Version Control

Versioning and branching allow for experimentation and help you deploy with confidence.

Reusable Logic

Pipeline Builder lets you easily extract and reuse pipeline logic for different pipelines, helping save budget and time spent creating new pipelines.

Type-Safe Functions

Functions are strongly typed and can flag errors immediately rather than at build time, preventing breaks that lead to expensive fixes and costly downtime.

Abstract Implementation Details

Users focus on describing their end-to-end pipeline and desired outputs. Builds, syncs, and other orchestrations are handled behind-the-scenes.

Strict Output Targets

If the expected output target is not met, deployment is prevented to avoid unintentional downstream breaks.

Built-In Security

Pipeline Builder incorporates granular security checks and Git-style change management — all without sacrificing speed or ease-of-use.

Interoperability

Avoid lock-in with robust export-to-code capabilities, and an architecture that was designed from scratch to be runtime-agnostic.
