Integration Types

Data Integrations

Sync data to and from other systems, with logic, models, and human-driven decisions in between.

Foundry is designed to integrate and operationalize data from across your organization. Our customers have integrated data from a wide range of sources, including commonly used databases and warehousing technologies, as well as hundreds of bespoke systems containing invaluable records of day-to-day business operations.

Use Cases:

  • Bulk data loading and import

  • Bulk data export (reverse ETL)*

  • External pipeline integrations*

*Not available for all systems

Get started with Data Integration →

Application Integrations

Enhance apps built in Foundry with external integrations.

Build data-rich applications that support realtime interactions with any operational system, app, or service. From a single application hosted in Foundry, developers can quickly build secure and scalable integrations with bespoke systems as well as widely used SaaS providers: from Twilio and Slack for realtime messaging, Stripe for payments, SAP and Salesforce for CRM, through to JIRA and PagerDuty for operations.

Use Cases:

  • Integrate from Workshop / Slate to external applications

  • Enhance native functionality of Foundry by enabling third-party integrations

Get started with Application Integration →

Streaming Integrations

Stream data into Foundry with low latency and high throughput to support real-time decision-making.

Foundry Streaming allows customers with streaming data sources and infrastructure to take advantage of that data alongside non-streaming data and models built and deployed in Foundry.

Use Cases:

  • Pull data from a streaming source or message queue

  • Push data from streaming sources directly to Foundry

  • Outbound streaming [Coming Soon]

Get started with Streaming data in Foundry →

Data Science Integrations

Leverage existing model investments and integrate models into operational decision-making processes using Foundry.

Externally developed models may be imported into Foundry or consumed via integration with an external API, making outputs available to end users, pipelines, and operational applications.

Use Cases:

  • Develop models in-platform via open-source tools such as scikit-learn, TensorFlow, and-OR-tools, or via custom libraries

  • Import models from an external environment (such as notebooks, third-party data science products, or container registries)

  • Consume models via externally-hosted APIs against data hosted in Foundry

Get started with Data Science Integrations →

Analytics Integrations

Connect to Foundry from third-party tools to view and interact with your data.

Browse third-party connectors to learn more about which systems offer out-of-the-box connectivity with Foundry using a dedicated connector.

Use Cases:

  • Integrate curated data from Foundry into external reporting and business intelligence applications

  • Use third-party analytics tooling securely on data stored in Foundry
