Modular To The Core

Foundry's modularity enables it to integrate with existing infrastructure in many ways.

Four potential integration models:

  • Foundry Extends Data Platforms
  • Foundry Fuels Data Platforms
  • Full Stack Foundry
  • Adaptive Integration

Foundry Extends Data Platforms

Operationalize Existing Assets

Foundry is designed to extend existing sources of truth (e.g., data lakes & warehouses), fusing data and models for use in operations.

Each facet of Foundry’s architecture is purpose-built to uphold existing data governance protocols, and ensure that existing sources are never fractured.

↳ Only the data required for operational work is synchronized into the platform

↳ All data flows are bidirectional; data produced in Foundry can be automatically synced back to enterprise data platforms

↳ Data usage is tracked throughout the platform, enabling administrators to set policies on how much data is retained and under what circumstances

↳ Role-, classification- , and purpose-based access control paradigms are granularly enforced

Foundry Fuels Data Platforms

Create a Connected Foundation

While Foundry can rapidly integrate models and data from data platforms into an operational ontology, it can also fuel existing data platforms.

Full Stack Foundry

End-to-End Enterprise Operating System

While Foundry is ready and able to integrate with all enterprise data and modeling systems, there are cases where the enterprise hasn’t (yet) made those software investments.

In those circumstances, Foundry can be deployed standalone - providing all necessary capabilities for data connection, data integration, and model building.

↳ Data Integration: Foundry's data integration capabilities codify 15+ years of data integration experience to operationalize data at scale and pace. This includes 200+ out-of-the-box connectors, and integrated data quality, data versioning, and change management frameworks

↳ Model Integration: Foundry's model development environments are designed with data science standards in mind, including runtimes bundled into the development experience (PySpark, R), the ability to leverage popular package managers, and a firm commitment to industry standard model formats

Foundry’s data and model integration suites are designed to prevent lock-in, and assume that myriad tools will be interoperating with Foundry as the organization grows.

Granular Governance

Foundry's robust data governance controls help ensure seamless compliance, auditability and security enterprise-wide.

  • Foundry provides rich, granular frameworks for role-, classification-, and purpose-based access controls

  • Foundry’s Lineage System tracks all usage of synchronized data

  • Data produced in Foundry can be automatically written back to existing systems

  • Foundry’s Retention System can enforce limits on “data in scope”

  • Native integration of enterprise SSO and authorization models

  • Foundry's security infrastructure is designed to enable secure data sharing - both within organizations and with chosen partners
