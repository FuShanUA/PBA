Rubix Technical Requirements

Specs for Rubix Compliance

Architecture and Software Design Requirements

Packaging and Deployment in Kubernetes with Helm Charts

Applications must be deployable into Kubernetes and must be packaged as one or more Helm charts.

Publishing Helm Charts to Accessible Container Registries

Helm charts that need to be deployed must be published to a container registry that is accessible to the runtime environments at install/upgrade time.

Ensuring Application High Availability and Fault Tolerance

* Applications must be highly available and able to tolerate loss of at least one replica in steady state without catastrophic failures. Alternatively, must not require strong uptime guarantees to function

Operating Without Backend SSH or kubectl Access

Operators may not have backend SSH or kubectl exec access, and Applications are expected to function without SSH or kubectl commands.

Linux amd64 Architecture Requirement

Applications must run on Linux amd64 architecture.

Supported Cloud Services and Self-Hosting Requirements

Cloud-rubix supports services such as Blob storage (S3), Network File System (EFS), and Managed Postgres/MySQL (RDS). Any additional required cloud services must be self-hosted within the Kubernetes cluster.

Rubix Deployability

Supported Cloud Providers and Security Levels

The supported cloud providers include AWS (Govcloud, SECRET, TOP SECRET, and FENCES), Azure (Govcloud and SECRET), as well as GCP and Oracle.

On-Prem Deployment Capability

Rubix is deployable in on-prem datacenters.

Edge Tactical Footprint Deployment

The edge tactical footprint is deployable with a minimum configuration of a 1-node cluster, with exact resource requirements detailed in the table below.

Kubernetes Requirements

Explicit RBAC Permissions for Helm Chart Applications

Applications do not require Admin (*/*) Kubernetes RBAC permissions; instead, the specific set of RBAC permissions required must be explicitly listed in each Helm chart.

Limitations on Cluster-Scoped RBAC Permissions

Cluster scoped RBAC is unsupported except for CustomResourceDefinitions.

Enforced Restricted Kubernetes Pod Security

Containers are required to run with the Restricted Kubernetes Pod Security Standard

Namespace Restrictions for Application Deployment

Applications must run in a statically defined and externally provided set of Kubernetes Namespaces.

Restrictions on Direct Kubelet and Host API Usage

Applications cannot use the Kubelet API directly or any other host specific API.

Resource Requirements

Tactical Edge (single host)

14 CPU | 42 GB Memory | Minimum 2x 1 TB disks

Deployable across a variety of tactical hardware.

Tactical Edge (multi-host)

16 CPU | 58 GB Memory | Minimum 6x 1 TB disks

Default 3-node tactical edge clusters are highly available and best for mobile footprints.

On-Premises

60 CPU | 160 GB Memory | Minimum 10x 1 TB disks

Larger on-premises environments for persistent high-scale workloads.
