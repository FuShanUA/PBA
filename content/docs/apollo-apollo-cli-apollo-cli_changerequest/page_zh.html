apollo-cli changerequest

与 Apollo 变更请求进行交互

标志（Flag）用途

从父命令继承的标志（Flags）

标志（Flag）用途
--apollo-client-id用于生成 Bearer Token 的客户端 ID
--apollo-client-secret用于生成 Bearer Token 的客户端密钥
--apollo-token用于经过身份验证的端点的 Bearer Token
--apollo-token-provider指定用于经身份验证的 Apollo 端点调用的 Bearer Token 的提供方式。有效值为 "auto"、"static"、"service-user" 或 "sso"（默认值为 "auto"）。若指定 "auto"，则根据配置自动选择模式：若设置了 "apollo-token"，则采用 "static" 模式；若同时设置了 "apollo-client-id" 和 "apollo-client-secret"，则采用 "service-user" 模式；否则采用 "sso" 模式（即交互式浏览器登录）。若同时设置了 Bearer Token 以及（客户端 ID 或客户端密钥），则报错。若指定 "static"，则使用 "apollo-token" 提供的 Token；若指定 "service-user"，则使用 "apollo-client-id" 和 "apollo-client-secret" 向 Multipass 请求生成 Token；若指定 "sso"，则通过 Multipass 执行交互式基于浏览器的单点登录（SSO）（支持 YubiKey 等硬件密钥）；生成的 Token 将按配置文件缓存，并在后台静默刷新。当某条命令首次需要 Token 且当前无有效缓存 Token 时，将自动触发登录流程。
--apollo-urlApollo 的基础 URL，用于推导 API 端点
--debug启用调试级别日志记录
-e, --environment用于环境范围命令的环境 ID（含后缀）
--http-timeout所有 Apollo 请求的超时时间（单位：分钟）
-k, --insecure-skip-verify跳过服务器证书验证
-o, --output输出格式（json、yaml、pretty）
--profile使用配置文件中的特定 profile
--quiet不向 stderr 输出日志信息
--space-id用于某些空间范围命令的空间 ID

另请参阅

  • apollo-cli changerequest approve：批准变更请求（异步执行）

  • apollo-cli changerequest reject：拒绝变更请求
  • apollo-cli changerequest reopen：重新打开已被取消或拒绝的变更请求
