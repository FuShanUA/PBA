Tableau OAuth 设置

Tableau 支持通过 OAuth 向 Foundry 进行身份验证。这意味着用户无需手动输入令牌，而是可以通过网页浏览器提示登录 Foundry。有关 Tableau 中 OAuth 的概述，请参阅 Tableau 的《OAuth 连接 ↗》文章。

启用此 OAuth 集成需要 Foundry 管理员权限。此外，如果需要为 Tableau Server 启用 OAuth 集成，还需要 Tableau 管理员权限，并且必须重启 Tableau Server。

第一部分：为 Tableau Desktop 启用 OAuth 客户端

  1. 按照《说明》访问控制面板中的第三方应用程序配置页面。
  2. 在应用程序列表中找到Tableau Desktop；从操作菜单中选择启用设置。
  3. 使用开关切换启用该应用程序。

当 Tableau 用户使用 OAuth 向 Foundry 进行身份验证时，将强制执行单个用户的权限。如果您使用了《项目访问》或《标记限制》面板来配置第三方应用程序的限制，这些限制将在用户个人权限的基础上叠加生效。

此时，Tableau Desktop 用户可以按照《说明》使用 OAuth 向 Foundry 进行身份验证。

第二部分：为 Tableau Server 配置 OAuth 客户端

按照以下说明为发布到 Tableau Server 的报表启用 OAuth 身份验证。

步骤 1：为 Tableau Server 注册第三方应用程序

在与上述相同的第三方应用程序页面上，选择新建应用程序以创建新的第三方应用程序：

  1. 在详情步骤中，将应用程序命名为 <ORGANIZATION> Tableau Server，并替换为您自己的组织名称。
  2. 在客户端类型步骤中，选择机密客户端。
  3. 在授权许可类型步骤中，选择授权码许可，并将重定向 URL 设置为 https://<YOUR_SERVER>/auth/add_oauth_token，其中 <YOUR_SERVER> 是您的 Tableau Server 主机名。

创建该应用程序并妥善保管客户端 ID 和密钥。

步骤 2：配置 Tableau Server

在服务器上运行以下命令，并替换为上一步中的客户端 ID、密钥和重定向 URL：

tsm configuration set -k oauth.config.clients -v "[{\"oauth.config.id\":\"FoundryJdbc\", \"oauth.config.client_id\":\"<YOUR_CLIENT_ID>\", \"oauth.config.client_secret\":\"<YOUR_CLIENT_SECRET>\", \"oauth.config.redirect_uri\":\"https://<YOUR_TABLEAU_SERVER>/auth/add_oauth_token\"}]" --force-keys

步骤 3：重启 Tableau Server

运行以下命令重启 Tableau Server：

tsm pending-changes apply

用法

Tableau Desktop

在 Tableau Desktop 中，用户现在可以按照《说明》通过 Foundry OAuth 身份验证选项进行身份验证。

Tableau Server

发布到 Tableau Server 时，可以将报表配置为在打开报表时提示查看者通过 OAuth 进行身份验证。这样，实时查询将使用查看者的权限运行。

要进行此配置，请在 Tableau Desktop 中使用 Foundry OAuth 身份验证选项开发报表。准备发布时，为身份验证选择提示选项。当用户在 Tableau Server 上查看报表时，实时连接将使用查看者的凭据刷新。
