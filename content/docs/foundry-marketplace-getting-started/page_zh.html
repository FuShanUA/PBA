入门

在本教程中，您将使用一个虚构的数据源安装一个警报收件箱应用程序，该数据源包含汽车零件问题，我们已将其作为独立的市场产品提供。

首先，导航到您的Foundry实例上的Marketplace，并搜索Marketplace Getting Started Tutorial - Datasource，如下所示。如果您无法找到此产品，请联系您的Palantir代表。

安装您的数据源

一旦您找到教程数据源，请点击右上角的安装。如果您的组织中有人已经创建了一个您可访问的安装，您将看到打开和重新安装按钮。在这种情况下，选择重新安装以继续进行安装。

在安装时：

  • 选择一个 项目（项目） 进行安装。命名空间是从项目的所在位置推导出来的。
  • 由于该产品仅包含一个数据集，因此不需要 本体论（本体论）。
  • 此产品没有任何输入，因此您可以跳过所有其他步骤，并在左侧面板的底部选择 安装。

开始安装后，您将进入安装作业页面。一旦安装完成，请在右上角选择 查看安装。从这里，在新标签页中打开您的 Car Part Issues - Source 数据集。返回到您的数据源安装标签页。

安装您的应用程序

我们将使用我们的Car Part Issues - Source数据集作为输入来开始安装目标应用程序。使用标题中的产品搜索栏，搜索Marketplace Getting Started Tutorial - Alert Inbox。该产品也属于参考资源商店。与数据源产品类似，点击右上角的安装以开始安装，如果您的组织中有人已经创建了安装，请选择重新安装按钮以继续安装。

常规

  • 选择一个要安装到的 项目。命名空间是从项目的所在位置推导出来的。
  • 选择一个要安装到的 本体论。如果您没有看到想要的本体论，请联系您的平台管理员，这意味着您没有编辑权限。

输入

与您第一次安装不同，您需要映射一个输入以创建警报收件箱应用程序。这将是您刚刚安装的Car Part Issues - Source，其中包含将填充警报收件箱的虚拟问题数据。要映射此输入：

在您打开的Car Part Issues - Source选项卡中，通过点击左上角的文件，然后点击复制完整路径来复制文件路径，如下所示。

现在，您需要将您的Car Part Issues - Source进行映射，以满足警报收件箱产品的唯一必需输入。为此，请返回到您的安装并点击选择，然后将您复制的文件路径粘贴到搜索栏中并选择您的数据集。

如果所有列都已正确映射，您将看到如下蓝色对勾标记。

内容

这一步确认将要安装的内容，本例中包括一个 Workshop 应用程序（Workshop 应用），一个 对象类型（对象类型），以及三个 操作类型（操作类型）。

打开 前缀本体论实体（Prefix ontology entities） 以自定义对象类型和操作的名称。我们建议您在本教程中使用您的名称。

新版本

您可以跳过此步骤进行本教程。有关自动升级、发布渠道和维护窗口的更多信息，请访问 安装文档。

审核

在完成输入映射并满意于安装位置和名称自定义后，选择 安装。

安装任务

开始安装后，您将进入安装任务页面。安装完成后，在右上角选择 查看安装。

打开您的应用程序

从您的安装中，在新标签页中打开您的 Marketplace Tutorial - Alert Inbox 应用程序。在您的 Car Part Issue 对象索引之前，您的应用程序将不可用。

你现在有一个警报收件箱工作区应用程序，可帮助你对Car Part Issues - Source问题进行分类！只要存在所需的输入列，您就可以使用任何问题源重新安装此应用程序，以用于新的用例。
