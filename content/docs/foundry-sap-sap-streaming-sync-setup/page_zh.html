SAP SLT

借助 SAP SLT 连接器，您可以通过 SAP Landscape Transformation (SLT) 复制服务器将 Foundry 连接至 SAP 系统。系统会基于数据库触发器，将数据从源 SAP ERP 系统复制到 SLT 系统中相应的操作增量队列 (ODQ) 内。SAP SLT 连接器支持 Foundry 与以下对象进行交互：

  • 从源 ERP 系统复制而来的 SLT 复制服务器表
  • SLT 复制服务器系统内部的本地表

若要使用 SAP SLT 数据源，必须在 SAP SLT 复制服务器上安装《Palantir Foundry Connector 2.0 for SAP Applications》附加组件。如需了解更多关于通过 SLT 进行连接的信息，请参阅相关文档。

设置

  1. 打开数据连接应用，并在屏幕右上角选择 + 新建源。
  2. 从可用的连接器类型中选择 SAP SLT。
  3. 选择在 Foundry worker 或 agent worker 上运行源功能。
  4. 按照后续的配置提示，结合下文各节中的信息继续完成连接器的设置。

如需了解更多关于在 Foundry 中设置连接器的信息，请参阅相关文档。

远程上下文

SAP SLT 连接器需要提供一个 Remote context 值，该值是 SLT 复制服务器与源 ERP 系统之间 RFC 连接的唯一标识符。如需了解更多配置信息，请参阅《SLT 配置指南》以及关于SLT 连接的文档。

身份验证

SAP SLT 连接器支持以下身份验证方法：

身份验证方法说明
基本认证 (Basic Auth)提供安装连接器时创建的技术用户的用户名和密码。
身份验证令牌提供用于身份验证的令牌。
自定义身份验证标头提供自定义身份验证标头。
无身份验证如果已在 agent 机器上通过证书完成身份验证设置，则使用此选项。

网络与连接

您必须正确配置出站策略（如果使用 Foundry worker）或您的 agent 网络（如果使用 agent worker），以确保 SAP SLT 系统能够与 Foundry 正常通信。

许多 SAP 系统使用自定义签名证书，这可能导致首次配置连接时出现 SSL 握手异常。请确保从系统中获取正确的自定义证书，并将其添加到数据源中（如果使用 Foundry worker）或直接添加到 agent 中（如果使用 agent worker）。

SAP 数据加速器

若要通过《SAP 数据加速器》（SAP 为通过 SAP Cloud Connector 暴露的系统提供的托管连接服务）访问该 SAP 系统，请将数据源的 主机 和 端口 设置为您的 Cloud Connector 虚拟主机和端口。然后，分配由 Palantir 提供的客户端证书，允许出站访问 SAP 数据加速器的控制面和数据面，并添加带有控制面 URL 的 SAP Data Accelerator 属性。此路径需要使用 Foundry worker。

从 SAP SLT 同步数据

使用数据探索视图为 SAP SLT 表创建同步。在数据源概览页面中，选择批量同步旁边的 + 新建，然后从下拉菜单中选择所需的对象类型：

  • SLT 复制服务器：来自源系统的数据通过 SLT 复制服务器，利用源 ERP 系统上的数据库触发器进行复制。
  • 本地表：直接从 SLT 复制服务器系统内部的表中读取数据。

同步参数

对于批量同步，以下设置显示在高级设置下。部分设置也可以在数据源级别进行配置。

设置说明
最大文件大小每个输出 Parquet 文件的最大大小。默认为每个文件 50,000 行。
清理 Avro 字段名对字段名进行清理，使其符合 Foundry 流所使用的 Avro ↗ 架构规则。如果数据集将用于流处理管道，则此设置是必需的。
忽略意外值启用后，解析失败的日期或数字值将写入为 null，并在同步结束时记录解析异常的摘要。
将日期转换为字符串将日期字段作为字符串摄取。当 SAP 日期字段包含无法解析但具有特殊含义且需要在下游处理的值时，此功能非常有用。
页面大小从 SAP 检索数据时每页返回的行数。默认为 50,000。最小为 5,000。
并行分页线程用于生成页面数据的 SAP 工作进程数。仅适用于传统 ODP 同步；不适用于 OData 同步。
插件工作线程用于检索页面数据的 Data Connection 智能体线程数。
远程上下文到源系统的 RFC 连接标识符。如果设置，初始加载将绕过 SLT 复制服务器队列并改为通过 RFC 消费数据。不适用于 OData 同步。
序列化引擎用于数据传输的序列化方法。
重试与超时重试次数、重试延迟和请求超时时间。
资源检查提取期间的内存和 CPU 检查。禁用可能会给 SAP 系统带来过重负载。
调试设置跟踪日志和调试日志。调试日志会在 SAP 中启动后台进程——请谨慎使用。

对于增量批量同步，每次同步最大行数设置限制了每次运行返回的大致行数。如果间歇性问题（如网络故障）中断了长时间运行的同步，可使用此设置将大表的初始同步拆分为一系列更小、更具弹性的运行。

流式同步

SAP SLT 连接器支持用于实时数据复制的流式同步。要创建流式同步，请从源概览页面中流式同步旁边的+ 新建进行选择。

强制快照

强制快照开关控制流是否在轮询增量更新之前先读取全表。在以下情况下启用此设置：

  • 源 ERP 系统上的数据库触发器已重置或损坏。
  • SLT 复制服务器中的指针已丢失。

在全表流式传输到 Foundry 后，请禁用此设置并保存，以便后续流式传输仅轮询更新。

API 版本

根据《SAP Note 3255746 ↗》（版本 11，2026 年 4 月），在本地和私有云环境中，禁止非 SAP 应用程序通过 RFC (ODP-RFC) 使用 ODP 数据复制 API。2026 年 6 月发布的 SAP 安全补丁在技术上阻止了未经授权的 ODP-RFC 调用。请在应用该补丁之前，将仍使用传统 ODP API 的同步迁移到 OData API。

对于附加组件版本 2.34 (SP34) 或更高版本，SAP SLT 同步可以使用 OData API。这基于 SAP 推荐的《通过 OData 进行基于 ODP 的数据提取 ↗》。SAP 不再允许使用传统 ODP API。

使用 OData API 需要：

  • 在 SLT 系统上安装 PALODATA 附加组件。
  • SAP NetWeaver 版本 7.50 SP09 或更高版本。了解有关先决条件和配置的更多信息。
API说明
OData每个表仅允许一个活动订阅。如果某个表已存在同步，则针对同一表的任何新同步都将失败，除非先释放现有订阅。不包括远程上下文或并行分页。
ODP（传统）允许每个表有多个订阅，从而使不同的同步能够独立摄取同一个表。包括远程上下文和并行分页。SAP 不再允许使用。

重置队列注册

使用 OData API 时，为表配置的第一个同步会占据活动订阅。如果需要用不同的配置替换该同步，请启用重置队列注册以释放现有订阅并将其分配给当前同步。

启用重置队列注册后，每次执行同步时都会触发数据集的全量重新加载。在订阅重新分配后应禁用此设置，以避免重复的全量加载。
