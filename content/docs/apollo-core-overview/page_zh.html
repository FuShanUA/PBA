技术概览

Apollo平台上存在两种类型的环境：中心（Hubs）和边缘（Spokes）。

  • 中心环境接收关于边缘环境的信息和遥测数据，并发出计划（Plans）以进行更改。中心可以管理多个边缘环境，也可以设置为自我管理。
  • 每个Spoke环境都运行一个Spoke Control Plane，它会将信息和遥测数据报告回管理的Hub，并执行由Hub发出的计划。

Hubs

每个Hub包含一个编排引擎，用于协调系统。编排引擎向在Hub管理的每个Spoke环境中运行的代理程序发布计划。编排引擎提出的特定计划取决于多个关键因素。

首先，编排引擎必须知道产品目录中可用的可能的产品版本。产品目录还包含有关版本的其他有用信息，包括该版本属于哪个版本通道、该版本是否因错误而被召回、对其他产品的依赖关系以及支持的数据库模式。版本可以通过版本提升管道自动添加到特定的版本通道，或由版本通道贡献者手动添加。

第二，编排引擎会使用环境的设置以及环境中任何由Apollo管理的软件或基础设施的设置，这些设置包括已安装的产品、发布通道订阅以及其他由环境编辑器定义的约束和设置。对环境设置的更改需要通过变更管理系统并获得适当的批准。

从产品目录中获取的信息和环境设置共同构成了编排引擎运行的规则。环境或由Apollo管理的软件或基础设施没有单一的目标状态；例如，Apollo管理的软件可能被定义为一个产品和一个发布通道，而不是特定的版本。

来自中心可观测性的具体环境的当前状态提供了最后的输入信息。编排引擎只能提出不违反任何约束条件的更改。

Spokes

代理 是 Spoke Control Plane 的一部分，负责将观察到的状态报告回中心 (已报告状态、探测、遥测与日志)。代理还会从由编排引擎发布的计划中轮询中心，并执行这些计划，同时报告计划的状态。
