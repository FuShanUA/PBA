循环布局

本页讨论循环布局。有关Workshop嵌入功能的概述，请参阅 嵌入概述页面。

循环布局允许您遍历一个对象集或数组，为集合中的每个对象或用作输入的数组中的每个条目显示一个嵌入式模块。

循环布局中的每个嵌入式模块都独立于其他嵌入式模块实例，并且有自己的变量作用域和布局状态。与对象表或对象列表等其他对象集显示小部件相比，循环布局对如何显示对象集提供了更多的控制。虽然其他对象集显示小部件具有固定的一组功能，但循环布局允许使用Workshop中所有可用的功能组合来显示对象集中的每个对象或数组中的每个条目。

以下是一个循环布局的示例，用于为三个对象集（每列一个）中的每个对象创建交互式票证卡片，以创建看板风格的应用程序。每张票证卡片都是一个嵌入式模块实例，该实例使用提供给列的循环布局的对象集中的对象进行填充。在此嵌入式模块中，可以完全控制布局，构建者可以配置要显示的内容以及每个对象可用的操作，比其他对象集显示小部件提供的灵活性更高。

配置

循环 布局配置允许构建者选择一个对象集或数组，为集合中的每个对象或数组中的每个条目嵌入一个模块，并提供排序和分页样式控制。

遍历对象集

要遍历的对象集

如果选择了对象集选项，则在循环布局中做的第一个配置是“要遍历的对象集”变量输入。此对象集将在循环布局中被遍历，该集合中的每个对象将用于显示在“模块选择”步骤中配置的子嵌入模块的一个实例。

排序

可以对要遍历的对象集应用属性排序，以确定对象在循环布局中显示的顺序。

通过在 由函数支持的对象集变量 的对象集定义中返回一个具有所需排序顺序的静态对象集，可以实现自定义排序。

对于不是静态对象集的单一对象类型对象集，将在任何用户配置的排序之后应用主键排序，以确保对象的一致顺序。如果主键不可排序，这可能导致循环布局无法显示。

遍历数组

循环布局支持遍历各种数组类型，包括字符串、布尔值、数字、日期、时间戳和 结构体 数组。

要遍历的数组

如果选择了数组选项，第一个配置是array to loop through变量输入。循环布局会遍历数组中的每个条目，并且每个条目都会以在模块选择步骤中配置的嵌入式模块实例的形式显示。模块按照数组中条目的位置进行排序。对数组进行插入、重新排序和删除条目将反映在循环布局中。

分页样式

  • 限制：这种分页样式将仅显示一个页面，该页面最多显示前X个对象或数组条目，其中X是“最大显示项数”配置中的值。
  • 分页：这种分页样式将显示大小为X的对象或数组条目页面，其中X是“每页最大项数”配置中的值。循环布局目前仅限于浏览前一万个项目。

显示

  • 列表：此显示选项会以列表形式显示对象或数组条目。没有其他配置选项。
  • 网格：此显示选项会以网格形式显示对象或数组条目。此外，可以配置最大列数和最小卡片宽度（以像素为单位）。

无论选择哪种选项，都可以通过在选定的子模块中对顶层部分配置绝对高度或最大高度来实现每行的一致高度。

模块选择

循环布局模块选择与嵌入式模块小部件模块选择类似，需要选择一个子模块。构建者可以通过指南针资源选择器选择现有模块，或者使用创建新模块选项创建一个新的嵌入式模块。

如果循环布局配置为遍历对象集，则所选模块必须具有模块接口对象变量；如果配置为遍历数组，则必须具有数组类型的变量。

使用“创建新”选项创建的嵌入式模块将包含一个基本模块，该模块具有与数组类型或对象集匹配的预设模块接口变量，以及用于显示每个条目或项目的控件。配置为使用对象集的循环部分将包含一个 对象集标题控件，而配置为使用数组的循环部分将包含一个 Markdown 控件 以显示数组条目的值。当使用结构体数组时，结构体类型的模块接口变量将包含一个 变量转换，在 Markdown 控件中渲染数组中每个结构体条目的字段。

接口变量

在此配置中，构建者必须从子模块中指定将映射到 object set to loop through 变量的对象或 array to loop through 变量的条目，具体取决于循环布局配置为遍历哪一个。有关更多信息，请参阅 配置所选模块的模块接口。

如果在子模块中更改了此变量值，例如通过“设置变量值”事件，可能会出现意外行为。

接口配置

除了上述的 模块接口变量输入 之外，循环布局变量映射方式与其他所有模块接口变量的 嵌入式模块接口配置 相同。

不同于object set to loop through变量，这些额外的变量可以在子模块模板中被修改。每个变量对于每个循环实例来说都是相同的变量引用，允许在循环实例和父模块之间共享变量状态。

布局设置

内边距

这是其他Workshop布局提供的相同内边距配置。配置的内边距将应用于循环布局中嵌入模块的每个实例。

内部边框样式

如果对循环布局应用了内边距，可以选择一种边框样式，该样式将应用于循环布局中嵌入模块的每个实例。

共享变量

在循环布局中的 接口配置 允许在循环布局嵌入模块之间共享变量值。这可能有用，例如共享所选对象的状态。在这种用例中，每个嵌入模块都有能力设置共享 selected object 变量的值，然后可以用于父模块。

限制

循环布局分页限制

目前循环布局仅限于浏览前一万件对象；此限制未来可能会发生变化。

动态值

任何每个模块实例的动态变量值都必须从传入的对象中派生。传入的对象是区分每个模块实例的唯一方式。在循环布局中，可以将其他变量传递给每个模块实例，这些变量将在实例之间 共享。
