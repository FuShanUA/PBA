循环布局

本页面讨论循环布局。有关 Workshop 嵌入式功能的概述，请参阅《嵌入式模块概览》页面。

循环布局允许您对对象集或数组进行遍历，为作为输入的对象集中的每个对象或数组中的每个条目显示一个嵌入式模块。

循环布局中的每个嵌入式模块独立运行，与其他嵌入式模块实例互不干扰，且拥有各自的变量作用域和布局状态。相比于对象表格或对象列表等其他对象集显示组件，循环布局在控制对象集的显示方式上提供了更强的灵活性。其他对象集显示组件通常附带固定的功能集合，而循环布局则允许将 Workshop 中可用的任意功能组合，应用于对象集中每个对象或数组中每个条目的显示。

以下示例展示了如何使用循环布局为三个对象集（每列一个）中的每个对象创建交互式工单卡片，从而搭建一个看板风格的应用。每张工单卡片都是一个嵌入式模块实例，填充了提供给该列循环布局的对象集中的对象。在这个嵌入式模块中，构建者可以完全掌控布局，精确配置要显示的内容以及每个对象可执行的操作，其灵活性远超其他对象集显示组件。

配置

循环布局配置允许构建者选择对象集或数组、为集合中的每个对象或数组中的每个条目选择要嵌入的模块，并配置排序和分页样式控件。

遍历对象集

要遍历的对象集

如果选择了对象集选项，在循环布局中首先要配置的是“要遍历的对象集”变量输入。该对象集将在循环布局中进行遍历，集合中的每个对象将用于显示在“模块选择”步骤中配置的子嵌入式模块实例。

排序

可以对正在遍历的对象集应用属性排序，以确定对象在循环布局中的显示顺序。

通过在《基于函数的对象集变量》的对象集定义中返回具有所需排序顺序的静态对象集，可以实现自定义排序。

对于非静态对象集的单一对象类型集，系统会在用户配置的排序规则之后应用主键排序，以确保对象排序的一致性。如果主键不可排序，可能会导致循环布局无法正常显示。

遍历数组

循环布局支持遍历多种数组类型，包括字符串、布尔值、数字、日期、时间戳以及《结构体》数组。

要遍历的数组

如果选择了数组选项，首先要配置的是 要遍历的数组 变量输入。循环布局会遍历数组中的每个条目，每个条目都会显示为在 模块选择 步骤中配置的嵌入式模块实例。模块的顺序由条目在数组中的位置决定。在数组中插入、重新排序或删除条目的操作都会反映在循环布局中。

分页样式

  • 限制：此分页样式仅显示单个页面，最多展示前 X 个对象或数组条目，其中 X 是在“《最多显示的项目数》”配置中设定的值。
  • 分页：此分页样式将分页展示对象或数组条目，每页大小为 X，其中 X 是在“《每页最多显示的项目数》”配置中设定的值。循环布局目前仅支持对前一万个对象进行分页浏览。

显示

  • 列表：此显示选项将以列表形式展示对象或数组条目。无其他额外配置选项。
  • 网格：此显示选项将以网格形式展示对象或数组条目。此外，还可以配置最大列数和以像素为单位的最小卡片宽度。

无论选择哪种选项，只需在所选子模块的顶层部分配置绝对高度或最大高度，即可实现每一行高度的一致性。

模块选择

循环布局的模块选择机制与嵌入式模块微件的模块选择类似，都需要选择一个子模块。构建者可以通过 Compass 资源选择器选择现有模块，也可以使用新建选项创建一个新的嵌入式模块。

为循环布局选择的模块，如果配置为遍历对象集，则必须包含模块接口对象集变量；如果配置为遍历数组，则必须包含类型为数组的变量。

使用“新建”选项创建的嵌入式模块将包含一个基础模块，该模块带有一个预设的模块接口变量（匹配数组类型或对象集）以及一个用于展示每个条目或项目的微件。配置为使用对象集的循环部分将包含一个《对象集标题微件》，而配置为使用数组的循环部分将包含一个《Markdown 微件》来展示数组条目的值。当使用结构体数组时，结构体类型的模块接口变量将包含一个变量转换，用于在 Markdown 微件中渲染数组中每个结构体条目的字段。

接口变量

在此配置中，构建者必须从子模块中指定模块接口变量，该变量将映射至“要循环遍历的对象集”变量中的对象，或是“要循环遍历的数组”变量中的条目，具体取决于循环布局配置为遍历哪种数据结构。有关更多信息，请参阅《为所选模块配置模块接口》。

如果此变量值在子模块中被更改（例如通过“《设置变量值》”事件），可能会导致意外行为。

接口配置

除了上述模块接口变量输入之外，对于所有其他模块接口变量，循环布局的变量映射机制与《嵌入式模块接口配置》的机制完全一致。

与“要循环遍历的对象集”变量不同，这些附加变量可以在子模块模板中进行更改。每个变量在各个循环实例中将引用同一个变量，从而允许在所有循环实例与父模块之间共享变量状态。

布局设置

内边距

此内边距配置与其他 Workshop 布局中提供的配置相同。所配置的内边距将应用于循环布局中嵌入式模块的每个实例。

内部边框样式

如果为循环布局应用了内边距，则可以选择一种边框样式，该样式将应用于循环布局中嵌入式模块的每个实例。

共享变量

循环布局中的接口配置允许在循环布局的嵌入式模块间共享变量值。这在共享诸如所选对象状态等信息时非常实用。在此场景下，每个嵌入式模块均能通过事件设定共享的 selected object 变量值，随后该值便可在父模块中使用。

局限性

循环布局分页限制

循环布局目前仅支持对前一万个对象进行分页操作；该限制未来可能会有所调整。

动态值

任何针对模块实例动态变化的变量值，都必须基于传入的对象派生而来。传入的对象是区分各个模块实例的唯一途径。可以向循环布局中的每个模块实例传入额外变量，这些变量将在各实例间共享。
