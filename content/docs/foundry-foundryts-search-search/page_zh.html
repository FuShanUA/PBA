Foundry搜索.Search

类 foundryts.search.Search

使用本体论对时间序列执行搜索查询的接口。

该接口允许使用本体中的属性来搜索时间序列集合。可以通过细粒度过滤器来精炼搜索，以确定哪些对象包含在搜索中。

当对时间序列的程序访问是动态的，并且基于本体的属性值时，这非常有用。考虑下面的例子：

您的本体包含一个 股票 对象类型，其属性为 [ticker, sector, exchange, price]。 要获取“科技”行业所有股票的价格，您可以使用 Search.series() 并使用以下查询：

Copied!
1>>> fts = FoundryTS()
2>>> tech_stocks_price = fts.search.series(
3...     query=(ontology("sector") == "Technology"),
4...     object_types=["stock"],
5...     property_type_id="price"
6... )
7>>> tech_stocks_price.to_pandas()

类似地，您可以在特定的交易所中搜索股票，或使用您的对象类型的其他相关属性。

搜索方法可通过FoundryTS.search单例属性访问，如上例所示。

注意

搜索只能查询时间序列引用，而不能查询时间序列本身中的值（例如搜索满足谓词条件的系列或点）。有关在时间序列内搜索数据，请参见dsl()和time_series_search()。

抽象 系列(query, max_results=10000, **kwargs)

使用传递的查询来搜索时间序列，并返回一个包含所有结果的NodeCollection。

搜索语法通过使用ontology()，利用==运算符对本体论中的属性进行等值检查。此外，您还可以使用object_types将搜索限定为本体论中的特定对象类型。建议这样做以避免在整个本体论上进行搜索。

  • 参数：
    • query （任意）– 由上述运算符和操作数组成的搜索查询。
    • max_results （整数，可选）– 返回的系列定义结果的最大数量，范围在1到10,000之间（默认为10,000）。
    • **kwargs – 用于额外搜索行为的行为选项和标志。
  • 关键字参数：
    • object_types （列表[字符串]，可选）– 具有时间序列属性的对象，用于在查询中搜索（默认是在所有时间序列对象上进行搜索）。
    • property_type_id （字符串，可选）– 在对象中返回的时间序列属性，如果对象包含多个时间序列属性（TSP），并且没有设置默认TSP，则必须提供此参数。 ↗ 可以在平台中为具有TSP的任何对象定义默认TSP。
    • experimental_enable_complex_properties （布尔值， 可选) – 是否将时间序列属性（TSPs）发送到时间序列后端。当使用非原始的时间依赖属性，如 ↗ 模板 或者由多个时间序列同步支持的时间序列属性（此类时间序列会以 ↗ 限定系列ID 的形式写入本体论）时，此选项是必需的。 （默认值为 False）。
  • 返回值： 一个 NodeCollection，包含所有与搜索查询匹配的时间序列引用，作为 FunctionNode。
  • 返回类型： NodeCollection
注意

确保您在查询中使用的属性（通过 ontology()）是可搜索的，否则搜索将会失败。时间序列属性不可被搜索。

当使用非原始的时间依赖属性，如模板或多同步属性时，experimental_enable_complex_properties 标志是必需的。
