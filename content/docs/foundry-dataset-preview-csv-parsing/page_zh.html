CSV 解析

Foundry 支持 CSV 数据集。这类数据集包含 CSV 格式的文件。

CSV 格式可以使用不同的分隔符、引用字符和转义字符。为了对此进行管理，您可以定义控制 CSV 文件解析方式的参数。这些参数存储在数据集的 schema 中。Foundry 会利用推断机制为特定数据集推荐一组合理的参数，但用户应对结果进行验证，并在必要时做出调整。

在 Foundry 中解析

Foundry 的 CSV 数据集通常会在其 schema 中将 TextDataFrameReader 定义为 dataFrameReaderClass。该读取器支持一组自定义参数，能够有效处理杂乱数据。在执行阶段，这部分工作会委托给《Spark CSV DataFrameReader ↗》处理，以获得最佳的性能和可靠性。

配置

在 Foundry 中，您可以进入《数据集预览》应用程序，切换至 Details 标签页并选择 Schema，以查看任意数据集的 schema。有关 schema 的更多详情，请参阅《数据集》文档。

您可以在 Edit Schema 用户界面中操作 CSV schema，在查看《数据集预览》的预览标签页时即可访问该界面。该界面有助于可视化可用选项及其对输出数据集的影响。遇到 CSV 格式严重错乱的情况，您可能需要手动修改 schema 才能获得预期的输出结果。

TextDataFrameReader 选项

若要在 schema 中手动配置 TextDataFrameReader 选项，您可以进入《数据集预览》中 Details 标签页下的 schema 页面，然后选择 Edit。在 schema 底部，您会看到如下部分：

recordDelimiter用于将 CSV 文件拆分为多条记录的行尾符号。以换行符结尾的字符串否，默认为 \n（换行符） CSV_PARSER, MULTILINE_CSV_PARSER

Spark CSV 选项

如果您已熟悉 Spark CSV DataFrameReader，可以在 customMetadata 中将 dataFrameReaderClass 配置为 DataSourceDataFrameReader，并将 format 配置为 csv。

有关支持的选项列表，请参阅《Spark CSV DataFrameReader 文档》。您可以按如下方式以键值对形式添加配置：

Copied!
1  "dataFrameReaderClass": "com.palantir.foundry.spark.input.DataSourceDataFrameReader",
2  "customMetadata": {
3    "format": "csv",
4    "options": {
5      "unescapedQuoteHandling": "STOP_AT_DELIMITER",
6      "multiline": true,
7      ...
8    }
9  }

请注意，上面列出的模式选项仅适用于基于 CSV 文件构建的数据集。
