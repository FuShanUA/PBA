摘要

摘要板显示了您在路径当前位置的表格中的行数和列数。

如果您完全没有过滤数据，那么这是您起始集中的行数。如果您已经应用了过滤器（例如，通过添加直方图并选择某些条形），那么这是过滤后的剩余行数。

筛选

筛选板的目的是对您的数据集应用可自定义的筛选条件。虽然您也可以在其他板（分布、直方图）中应用筛选条件，但筛选板允许在一个地方构建涉及多个变量的更复杂的筛选条件。

在筛选板中使用列表相当于 SQL 中的 WHERE IN (x,y,z) 子句。Contour 可以处理筛选板中数千项的列表。然而，大型列表会增加浏览器的负担，过大的列表可能会导致浏览器崩溃。在这种情况下，应将列表作为单独的集合导入 Contour，并使用关联或集合数学板来实现筛选器。 了解如何使用关联或集合数学板。

配置

点击 添加筛选，选择要筛选的列，然后从下拉菜单中选择一个筛选类型。根据您选择的列，Contour 将选择适当的筛选类别（例如，对于数值列，选择数字）。

提示

在一些文本过滤器中，你可以使用通配符：* 可以被多个字符替换，而 ? 可以被一个字符替换。

在“匹配”（正则表达式）文本过滤器中，你可以直接输入你的正则表达式（不需要引号或字符串指示符）。

要添加另一个过滤器，请再次点击添加过滤器。你可以选择匹配所有过滤器或任意过滤器。要删除一个过滤器，请点击该过滤器旁边的垃圾桶按钮。 点击保存以应用你的过滤器。

文本过滤器详细信息

文本过滤器目前提供以下选项：

  • 包含： 这将返回包含任何搜索词的行。你的搜索词只能包含文本。例如，“hello”这个术语会匹配包含“hihellohi”的行。
  • 包含（带通配符）： 这将返回包含您搜索词中的任何一个的任何行。您的搜索词可以包含?以表示单个字符通配符，或*表示多个字符通配符。例如，h?l*o 该项将匹配“hi hello hi”或“hi halqqqqqo hi”。
  • is: 这将返回与您的任意搜索词相等的任何行。您的搜索词应只包含文本。例如，hello 的一项将匹配“hello”，但不会匹配“hi hello hi”。
  • is (带通配符): 这将返回与您的任意搜索词相等的任何行。您的搜索词可以包含?表示单个字符通配符，或*表示多个字符通配符。例如，h?l*o 该项将匹配“hello”或“halqqqqqo”。
  • matches: 这将返回与任何术语匹配的任何行，其中术语是一个正则表达式。此选项使用 Java Pattern ↗ 来评估正则表达式。

表达式

除了直方图和图表等可视化工具外，Contour 还提供了一个表达式板，允许您使用 Contour 的丰富表达式语言从您的数据中推导出新列，执行复杂过滤，或执行复杂聚合。

  • 在使用表达式编辑器时，点击?图标以快速查阅表达式语言。
  • 当您输入时，下拉列表中会出现建议的函数。点击或使用 Enter 键选择您需要的函数。

列名区分大小写。此外，当选择一列时，您可以使用双引号或不使用双引号来编写列名。例如，year("birthdate_col") 等于 year(birthdate_col)。为了保持一致性，本文档中的列名都用双引号表示。

表

表格板以表格形式显示了您的数据集的快照。请注意，仅显示数据集中的前 limit（默认值：1,000）行。此限制是为了防止浏览器性能问题，通常不可配置。

表格板对于快速检查您的数据是否符合您的预期非常有用。您可以与表格进行交互：拖放列以重新排序它们，或从每列的下拉菜单中选择。对表格的这些格式更改不会改变底层数据（如果您仅查看部分列，所有列仍然存在于底层数据中）。

要一次移动多列，请按住 Shift 键的同时选择列。 您也可以使用 配置 面板一次性修改多列。

条件格式

你可以通过点击列下拉菜单，为表格板添加条件格式。

然后，使用对话框为某一列添加规则。经过条件格式化的单元格将显示所选颜色的文本和背景。不支持对日期列应用规则。

表格板与表格面板

你可以在路径中的任何位置添加表格板，以快速预览该时刻的数据，或者你可以从路径视图切换到表格面板。

表格面板以表格（而不是表格板）为重点，因此你可以看到在添加每个表格板时数据如何变化。这在编写表达式时特别有帮助。

你可以通过点击右上角的表格来切换到表格面板。再次点击该按钮或点击隐藏表格返回到路径视图。

表格面板不支持条件格式化。

转换数据

您可以使用以下面板对某一列的数据进行转换：

混淆

  • 哈希单元格值（例如，隐藏敏感数据，如姓名）。该列中的每个值都会使用 SHA-1 ↗ 哈希函数替换为该值的哈希表示。

SHA-1 哈希可以被解密，并不被认为是完全安全的。因此，不应将其用于数据合规性目的。

  • 遮蔽值中的一些字符（例如，遮蔽电话号码中除最后两位数字以外的所有数字）。
  • K-匿名化 作为一种隐私技术，旨在设置一个阈值（k）以应用于数据集，确保至少有 k 个具有相同敏感信息集的实例，从而降低重新识别的风险（即使没有个人可识别信息）。此过程通过“抑制”可能有助于数据重新识别的特定字段来完成。

根据您的使用场景，合适的k值由上下文决定。组织通常会根据分析的上下文和重新识别的统计风险，制定自己的k值政策。一些示例政策包括 国家教育统计中心 ↗ 和 美国卫生与公众服务部 ↗。至少，k值应始终大于1且小于数据集中的总行数。

使用 k-匿名化函数，仪表板会要求指定要进行 k-匿名化的列、k 值目标、抑制策略，以及如何处理在抑制后不满足 k 值的行。

  1. 列（Columns）: 表示“准标识符”或可以与外部数据关联以唯一识别个人的属性。
  2. k 值（k-value）: 表示阈值 k，在此阈值下，至少有 k 个实例具有相同的一组敏感信息。
  3. 策略（Strategies）: 表示数据应如何被抑制以及按什么顺序进行。您可以设置给定的操作顺序以达到指示的 k 值。对于列出的每一列，可以选择不同的策略来应用于数据以满足 k 值：
    • 分桶（Bucket）: 用范围替换整数；仅当选择数值类型列时可用。
    • 掩码（Mask）: 用 * 替换最后 n 个字符。
    • 替换（Replace）: 用字符串替换整个值。默认行为建议使用 *** 作为替换值，但也可以用用户提供的值替换。
    • 具有 抑制列（Suppress Column） 的列 已勾选的标志将适用于所有值，无论是否满足k值。这种行为在诸如对年龄进行分桶等情况下特别相关，因为对于所有值拥有统一的分桶策略很有用。
  4. 未满足k值的行：如果某些行未满足k值阈值且无法被抑制以达到大于k的计数，则有以下选项可用：
    • 保留：保留这些行以防止数据丢失。请注意，如果您保留这些行，数据集将不会被k匿名化；这通常是审查k匿名化结果的一个有用步骤。
    • 删除：删除所有未满足k值的行。选择删除时，请务必计算混淆前后的行数，以了解被删除的行数。
    • 遮蔽：使用***对表中的所有值进行遮蔽。如果希望保留相同的行数，此选项特别相关。

分组选择

在配置按...分组列（例如 x 轴）和按...分段列时，您可以选择如何对数据点进行分组。只有数值、日期或时间列可以进行分组。例如，如果您创建一个柱状图并选择一个日期列作为 x 轴的按...分组列，并选择分组类型为年，则生成的图表将为每一年显示一个柱子。可用的分组类型如下所示。

数值列的分组类型：

  • 精确值： 数据未进行分桶，显示精确值。
  • 最佳: 分桶数量等于底层数据范围中点数的平方根。数据范围是该列最大值和最小值之间的差值。
  • 最细粒度: 图表使用可以在 结果限制 内容纳的最大分桶数量。在可能的情况下使用精确值。
  • 自定义: 可以手动选择分桶数量。注意，分桶数量不能超过 结果限制。

日期和时间列的分桶类型：

  • 精确时间: 数据未进行分桶，显示精确值。
  • 四舍五入: 数据根据所选内容分桶到最近的 秒、分钟、小时、天、周、月 或 年。例如，如果按 年 进行分桶， ，一个日期为2018年6月15日的数据点被归入2018年桶。
  • 序数：数据被归入序数日期。例如，如果选择星期几，数据将被归入七个桶，每个桶对应一周中的每一天。

如果桶选择不符合结果限制，将应用最细粒度的选项以确保数据不被丢弃。有关更多信息，请阅读结果限制。

结果限制

Contour 限制它在浏览器中显示的数据点数量。实际上，Contour 不能显示超过屏幕像素数的数据点。为了生成准确的图表并且不丢失任何数据，图表板会将图表配置重新分组到尽可能细粒度的分组选择，以符合结果限制。

结果限制由你的 Palantir 管理员设置，默认为 1000 个点。对于数值、日期或时间列，将发生重新分组。

为了说明重新分组，请考虑以下示例：

  • 在一个包含出生日期的数据集上创建了一个图表板。
  • 该板被配置为柱状图，x 轴设置为出生日期列。例如，用来统计具有相同出生日期的人数。
  • 出生日期列精确到秒，因此选择了 Second（秒）分组类型。
  • 在这个数据集中，每秒的唯一出生日期数量超过了结果限制。
  • 因此，在计算过程中，图表板会根据小时而不是秒对数据进行自动分桶，因为小时是适合此特定数据集结果限制的最细粒度分桶大小。

关于采样的重要说明

为防止前端和后端性能变慢，计算的行数受到限制。大多数环境中的限制是1,000行，并且通常不可配置。

假设如上图所示，您有PERIOD和PRACTICE的透视表行聚合，以及POSTCODE的列聚合。对于每种组合，您希望获取行计数和列NAME的最大值。如果您的环境限制是默认值1,000，则只会计算1,000行完整的数据。每一行都保证是完整的，但可能有些行并不存在。

在对透视表中的某一列进行排序时，排序是在预览中执行的，而不是在整个数据集上执行。要对整个数据集进行排序，请使用排序板。有关更多信息，请参阅 排序。

为了与透视数据的全部内容进行交互，请在板上使用 切换到透视数据 选项，这会将您的轮廓分析转换为透视表板下所有板的完全计算后的透视数据。或者，您可以通过在透视表上游进一步筛选数据来尝试避免单元格限制。

您可以在现有数据集的列上执行二进制操作，以创建新的派生列，或者将字符串类型的列解析为数字或日期格式的列。

SQL 等效项

派生列相当于在 SQL 或 Spark 中使用运算符。例如，以下内容用于派生 人均收入 列：

Copied!
1SELECT
2    [Household Members],
3    [Marital Status],
4    [Income Column] / [Household Members] AS [Income per person]
5FROM [Table Name]

SQL 等效语句

通过列编辑器板删除列相当于在 SQL 中选择列名。例如，给定一个包含 5 列（A-E）的表，以下操作会删除列 D 和 E：

Copied!
1SELECT columnA, columnB, columnC
2FROM <tableName>

多列编辑器

多列编辑器板允许您重新排序、重命名和删除数据中的列，并移除重复行。后续的板将使用您选择保留的列集合。

面板的左侧显示所有列，而右侧显示保留列。在保留列部分，您可以选择重命名或重新排序保留的列，或者使用批量重命名功能。

SQL 对应项

重新排序、重命名和删除列等同于在 SQL 中选择列名。例如，给定一个包含 5 列 A-E 的表，以下代码会删除列 D 和 E，并将 A 重命名为 A_1：

Copied!
1SELECT columnA as columnA_1, columnB, columnC
2FROM <tableName>

排序板让您对数据集中的所有数据进行排序。请注意，这种排序仅限于分析，并不会保存到已保存的数据集中。任何下游聚合（例如连接或删除重复行）都可能导致排序丢失，因此建议在排序前先进行此类聚合。

计算

计算板允许您以卡片或列表的形式在数据上显示多个聚合计算。可用的聚合指标包括：唯一计数，最小值，最大值，总和，平均值，中位数，标准差 方差。

卡片格式对于水平或垂直方向和指标大小有额外的格式选项。

最后，每个计算都可以根据一组指定规则（条件）进行条件格式化。这意味着字体颜色和背景颜色可以根据条件是否满足而改变。
