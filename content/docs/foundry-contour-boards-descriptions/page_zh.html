摘要

摘要板显示了您在路径当前位置的表格中的行数和列数。

如果您完全没有过滤数据，那么这是您初始集合中的行数。如果您已经应用了过滤器（例如，通过添加直方图并选择某些条形），那么这是过滤后的剩余行数。

筛选

筛选板的目的是对您的数据集应用可自定义的筛选条件。虽然您也可以在其他板（分布、直方图）中应用筛选，但筛选板允许在一个地方构建涉及多个变量的更复杂的筛选条件。

在筛选板中使用列表相当于 SQL 中的 WHERE IN (x,y,z) 子句。Contour 可以处理筛选板中数千项的列表。然而，大型列表会增加浏览器的负担，过大的列表可能会导致浏览器崩溃。在这种情况下，应将列表作为单独的集合导入 Contour，并使用关联或集合数学板来实现筛选器。了解如何使用关联或集合数学板。

配置

点击添加筛选，选择要筛选的列，然后从下拉菜单中选择一种筛选类型。根据您选择的列，Contour 将选择合适的筛选类别（例如，数值列选择数字类型）。

提示

在一些文本过滤器中，你可以使用通配符：* 可以被多个字符替换，而 ? 可以被一个字符替换。

在“匹配”（正则表达式）文本过滤器中，你可以直接输入你的正则表达式（不需要引号或字符串指示符）。

要添加另一个过滤器，请再次点击添加过滤器。你可以选择匹配所有过滤器或任意过滤器。要删除一个过滤器，请点击该过滤器旁边的垃圾桶按钮。 点击保存以应用你的过滤器。

文本过滤器详细信息

当前文本过滤器提供了以下选项：

  • 包含： 这将返回包含任何搜索词的行。你的搜索词只能包含文本。例如，“hello”这个术语会匹配包含“hihellohi”的行。
  • 包含（带通配符）： 这将返回包含您搜索词中的任何一个的任何行。您的搜索词可以包含?以表示单个字符通配符，或*表示多个字符通配符。例如，h?l*o这样的术语将匹配“hi hello hi”或“hi halqqqqqo hi”。
  • is: 这将返回与您的任意一个搜索词相等的任何行。您的搜索词应只包含文本。例如，hello这样的术语将匹配“hello”，但不会匹配“hi hello hi”。
  • is（带通配符）: 这将返回与您的任意一个搜索词相等的任何行。您的搜索词可以包含?表示单个字符通配符，或*表示多个字符通配符。例如，h?l*o这样的术语将匹配“hello”或“halqqqqqo”。
  • matches: 这将返回与任何术语匹配的任何行，其中术语是一个正则表达式。此选项使用 Java Pattern ↗ 来评估正则表达式。

表达式

除了直方图和图表等可视化工具外，Contour 还提供了一个表达式板，允许您使用 Contour 的丰富表达式语言从您的数据中推导出新列，执行复杂过滤，或执行复杂聚合。

  • 在使用表达式编辑器时，点击?图标以快速查阅表达式语言。
  • 当您输入时，下拉列表中会出现建议的函数。点击或使用 Enter 键选择您需要的函数。

列名区分大小写。此外，选择列时，您可以使用带引号或不带引号的列名。例如，year("birthdate_col") 等同于 year(birthdate_col)。为了保持一致性，本文档中的列名都使用引号。

表

表格板以表格形式显示了您的数据集的快照。请注意，仅显示数据集中前limit（默认值：1,000）行。此限制是为了防止浏览器性能问题，通常不可配置。

表格板对于快速检查您的数据是否符合预期非常有用。您可以与表格进行交互：通过拖放列来重新排序它们，或从每列的下拉菜单中选择。对表格所做的这些格式更改不会改变底层数据（如果您只查看部分列，所有列仍然存在于底层数据中）。

要一次移动多列，请按住 Shift 键的同时选择列。 您也可以使用 配置 面板一次修改多列。

条件格式化

你可以通过点击列下拉菜单为表格板添加条件格式。

然后，使用对话框为某一列添加规则。经过条件格式化的单元格会以所选颜色的文本和背景显示。不支持对日期列应用规则。

表格板与表格面板

你可以在路径中的任何位置添加表格板，以快速预览该时刻的数据，或者你可以从路径视图切换到表格面板。

表格面板以表格（而不是表格板）为焦点，因此你可以看到在添加每个表格板时数据是如何变化的。这在编写表达式时尤其有用。

你可以通过点击右上角的表格来切换到表格面板。再次点击该按钮或点击隐藏表格返回到路径视图。

表格面板不支持条件格式化。

    转换数据

    您可以使用以下面板对列中的数据进行转换：

    混淆

    • 哈希单元格值（例如，隐藏敏感数据，如姓名）。该列中的每个值都会使用 SHA-1 ↗ 哈希函数替换为该值的哈希表示形式。

    SHA-1 哈希可以被解密，并不被认为是完全安全的。因此，不应将其用于数据合规性目的。

    • 遮蔽值中的某些字符（例如，将电话号码的最后两位以外的所有数字遮蔽）。
    • K-匿名化 作为一种隐私技术，旨在设置一个阈值（k）以应用于数据集，确保至少有 k 个具有相同敏感信息集的实例，从而降低重新识别的风险（即使没有个人可识别信息）。此过程通过“抑制”可能有助于数据重新识别的特定字段来完成。

    根据您的使用场景，合适的k值由上下文决定。组织通常会根据分析的上下文和重新识别的统计风险，制定自己的k值政策。一些示例政策包括 国家教育统计中心 ↗ 和 美国卫生与公众服务部 ↗。至少，k值应始终大于1且小于数据集中的总行数。

    使用 k-匿名化函数，仪表板会要求选择要进行 k-匿名化的列、k 值目标、抑制策略，以及如何处理在抑制后不满足 k 值的行。

    1. 列（Columns）: 表示“准标识符（quasi-identifiers）”或可以与外部数据关联以唯一识别个体的属性（Attribute）。
    2. k 值（k-value）: 表示阈值（threshold）k，在此阈值下，至少有 k 个实例具有相同的一组敏感信息（sensitive information）。
    3. 策略（Strategies）: 表示数据应该如何被抑制以及抑制的顺序。您可以设置给定的操作顺序以达到指示的 k 值。对于列出的每一列，您可以选择不同的策略来应用于数据以满足 k 值：
      • 分桶（Bucket）: 用一个范围（rangee）替换整数；仅在选择数值类型（numeric type）列时可用。
      • 掩码（Mask）: 用一个 * 替换最后 n 个字符。
      • 替换（Replace）: 用字符串替换整个值。默认行为建议使用 *** 作为替换值，但也可以用用户提供的值替换。
      • 具有 抑制列（Suppress Column） 的列 标记已选中将对所有值应用策略，无论是否满足k值。这种行为在诸如按年龄分桶等情况下特别相关，因为对于所有值拥有一致的分桶策略很有用。
    4. 在抑制后不满足k值的行：如果某些行不满足k值阈值且无法被抑制以达到大于k的计数，则有以下选项可用：
      • 保留：保留这些行，以免数据丢失。请注意，如果您保留这些行，数据集将不会被k匿名化；这通常是审查k匿名化结果的一个有用步骤。
      • 删除：删除所有不满足k值的行。选择删除时，请务必在混淆前后计算行数，以了解被删除的行数。
      • 遮蔽：用***遮蔽表中的所有值。如果希望保留相同数量的行，此选项尤其相关。

    列编辑器看板允许您轻松地从数据集中删除列并派生新列。后续看板将使用您选择保留的列集合。

    您可以在现有数据集的列上执行二进制操作，以创建新的派生列，或者将字符串列解析为数字或日期格式的列。

3    [Marital Status],

    移除重复行

    您可以使用列编辑器板中的移除重复行选项来移除重复行。

    SQL 等效语句

    通过列编辑器板删除列等同于在 SQL 中选择列名。例如，给定一个包含 5 列（A-E）的表，以下代码将删除列 D 和 E：

    Copied!
    1SELECT columnA, columnB, columnC
2FROM <tableName>

    多列编辑器

    多列编辑器板允许您对数据中的列进行重新排序、重命名和删除，以及移除重复行。后续的板将使用您选择保留的列集合。

    板的左侧显示所有列，而右侧显示保留列。在保留列部分，您可以选择重命名或重新排序保留的列，或者使用批量重命名功能。

    SQL 对应项

    重新排序、重命名和删除列等同于在 SQL 中选择列名。例如，给定一个有 5 列 A-E 的表，以下代码会删除列 D 和 E，并将 A 重命名为 A_1：

    Copied!
    1SELECT columnA as columnA_1, columnB, columnC
2FROM <tableName>

    宏板允许您将之前创建的宏应用到您的路径。

    排序

    排序板让您对数据集中的所有数据进行排序。请注意，这种排序仅限于分析，并不会保存到已保存的数据集中。任何下游聚合（例如连接或删除重复行）都可能导致排序丢失，因此建议在排序前先执行这些聚合操作。

    计算

    计算板允许您以卡片或列表的形式显示数据上的多个聚合计算。可用的聚合指标包括： 唯一计数，最小值，最大值，总和，平均值，中位数，标准差 ，以及方差。

    计算板可以格式化为卡片或列表。

    卡片格式为横向或纵向方向和指标大小提供了额外的格式选项。

    最后，每个计算都可以根据一组指定规则（条件）进行条件格式。这意味着字体颜色和背景颜色可以根据是否满足条件而变化。

    解旋转

    展开板允许您通过将某些列转换为行来重塑您的数据。您选择的列将被重新格式化为两个新列：一个标题列（包含原始列名）和一个值列（包含原始数据值）。
