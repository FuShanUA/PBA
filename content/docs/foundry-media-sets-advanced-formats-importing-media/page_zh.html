您可以通过多种方式将媒体导入到媒体集，包括：

  • 直接上传
  • 连接到外部源系统
  • 转换
  • 使用操作
  • OSDK

路径去重

如果一个文件被上传到媒体集，并且其路径（通常是文件名，如example.png）与媒体集中已有的项目相同，则新项目将覆盖现有项目。

去重行为

您必须确保路径唯一以避免覆盖。在上传覆盖现有媒体项之前不会出现警告或确认。

通过直接 媒体引用 来引用该媒体项时，被覆盖的媒体项仍然可用。也可以通过设置 deduplicate_by_path=False 在转换中查看被覆盖的媒体项或列出它们。

媒体项版本历史

当多个媒体项以相同路径上传时，仅显示媒体集中的最新项。但是，您可以在特定路径下查看上传的完整版本历史记录，以查看所有被覆盖的媒体项。

要查看媒体项路径的版本历史：

  1. 在媒体集中选择一个媒体项。
  2. 在元数据面板中，选择版本历史。

版本历史会按时间顺序显示所有已上传到该路径的媒体项，最新的上传内容列在最前面。每个条目显示媒体项的上传时间，并允许您预览内容。

被覆盖的媒体项不会从媒体集永久删除，仍可能在构建中处理。为了管理存储并确保仅处理最新项，建议配置 覆盖项的保留策略。

直接上传

要通过直接上传将媒体文件导入媒体集，请将文件拖放到新的媒体集中。文件必须与创建媒体集时指定的预期文件类型匹配，才能上传到媒体集。

  1. 首先，在项目中选择 新建，然后从搜索栏中选择 媒体集，如下所示。

  1. 接下来，为新媒体集选择所需的媒体文件类型，然后选择 创建媒体集。

  1. Finally, drag and drop the media files into your new media set.

  • 创建媒体集后，您可以通过将媒体拖放到空的媒体集上，或者通过选择从您的计算机中选择提示来上传媒体。

    数据连接

    通过数据连接与外部源进行同步，可以导入媒体集。详细的步骤说明可以在媒体集同步文档中找到。

    要创建新的媒体集 同步，请导航到所需 源 的 概览 选项卡。

    创建同步后，在媒体集视图中触发构建，以便媒体出现在您的媒体集中。

    您也可以通过 选择一个源 选项将现有源连接到新媒体集。

    Virtual media sets

    For supported source types, media sets can optionally be configured to read directly from the external source system so no data is copied into Foundry's backing store. These are called virtual media sets.

    External transforms

    For sources with REST APIs, you can import media to a media set through external transforms.
