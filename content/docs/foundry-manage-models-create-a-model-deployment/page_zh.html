创建直接模型部署

直接模型部署是实时托管的端点，可将模型立即连接到 Workshop 和 Slate 等用户应用程序。可以通过 模型函数 使用 TypeScript 查询直接模型部署，也可以通过 REST API 调用从外部系统进行查询。

以下各节将介绍如何创建、配置和发布直接模型部署，并说明在开始之前需要审查的一些调试步骤和功能注意事项。

1. 创建直接模型部署

要创建并启动直接模型部署，请导航至模型。在模型页面顶部的 实时推理 下，选择 启动部署。运行后，可以通过选择 运行 交互式地测试部署。

2. 配置直接模型部署

要配置直接模型部署的资源，请在 实时推理 面板的右上角选择 配置 按钮。直接模型部署可以配置为从零开始扩容。当部署达到 75% 的容量时，系统将创建一个额外的副本，直到达到运行时扩缩容配置中指定的最大副本数。这也允许部署在没有实时请求 30 分钟后自动缩容。

3. 为部署发布函数

您可以为模型发布函数，从而能够在 Workshop、Vertex 和其他最终用户应用程序中使用模型进行实时推理。

要发布函数，请在模型工件侧边栏中选择 + 图标并提供函数名称。每个分支只能注册一个函数。此操作会创建一个与您的模型具有相同输入和输出 API 的包装函数，该函数可以从函数仓库导入并调用，以添加自定义业务逻辑。

有关函数行为、版本升级和配置选项的详细信息，请参阅《模型函数开发者指南》。

自动升级

模型的每个分支可以创建一个直接模型部署。当新的模型版本发布到该分支时，直接模型部署将自动升级到新的端点，期间不会停机。如果您不希望进行自动升级，请考虑使用建模目标实时部署，并查看下文介绍的实时部署与直接部署之间的差异。如果为该部署创建了函数，系统将自动创建一个新版本。

自动水平扩缩容

直接模型部署由计算模块提供支持，因此支持在用户指定的最小和最大副本范围之间进行自动水平扩缩容，如上文所述。

配置计划覆盖

您可以为一周中的特定日期和时间安排最小和最大副本配置的覆盖计划。这在您预期会有可预测的变更时非常有用，例如工作时间内的流量较高或周末的负载减少。

要配置计划覆盖：

  1. 导航至直接模型部署的 实时部署 选项卡，并选择 启动部署 右侧的设置图标以编辑运行时配置。



  2. 在 扩缩容 部分，切换开启 启用计划覆盖，以呈现一个配置面板，供您安排覆盖计划。



  3. 为您的覆盖配置以下设置：
    • 覆盖最小副本数： 计划周期内的最小副本数量。
    • 覆盖最大副本数： 计划周期内的最大副本数量。
    • 生效日期： 应用覆盖设置的星期几。
    • 时间范围： 覆盖设置的起止时间及其时区。

在配置的时间段之外，将应用默认副本配置。目前，您只能计划一个覆盖设置。

模型 API 类型安全

直接模型部署会对所有推理请求强制执行类型安全检查，以确保模型 API 类型与输入类型相匹配。所有输入类型均需遵循类型安全要求，特别是以下几种类型：

  • 数值： 如果模型的 API 定义为 int 类型，而向模型传递了 3.6，则会截去 0.6，输入值将变为 3。
  • 日期和时间戳： 直接模型部署会在将数据提供给 predict() 方法之前，对日期和时间戳类型进行转换。时间戳字段现在需要使用 ISO 8601 格式的字符串。
  • 强制的 API 结构： 直接模型部署会明确要求提供模型 API 中标记为必填的字段。

模型类型安全与实时建模部署不同，后者目前不支持类型转换。

调试直接模型部署

要查看直接模型部署的调试信息和日志，请选择模型页面顶部的 部署健康状况 标签页。在此处，您可以找到部署的运行构建、副本健康状况信息、日志以及有关每个副本状态的指标。

您还可以在 运行调试 卡片下查看模型推理的调用堆栈。这有助于您了解每个 Python 函数的执行时长以及可以在何处进行性能优化。

注意： 对于容器模型，或者在推理期间抛出错误的情况下，不会显示调用堆栈。
