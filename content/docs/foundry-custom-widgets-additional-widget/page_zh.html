向小部件集添加额外的小部件

小部件集允许您从单一代码仓库发布多个小部件，以便于代码共享和性能优化。此页面解释了如何向小部件集添加额外的小部件，并假设项目结构从我们提供的使用 React、Vite 和 @osdk/widget.vite-plugin ↗ 的模板开始。

一个小型部件集最多可以包含 50 个小部件。

以下示例项目结构适用于包含两个小部件的小部件集：

repo/
├── src/
│   ├── first.tsx
│   ├── first.config.ts
│   ├── second.tsx
│   ├── second.config.ts
│   └── ...
│── first.html
│── second.html
│── vite.config.ts
└─ ...

import { FoundryWidget } from "@osdk/widget.client-react";
import { createRoot } from "react-dom/client";
import Second from "./second.config.js";

const root = document.getElementById("root")!;

createRoot(root).render(
  <FoundryWidget config={Second}>
    {/* Render something for the widget! */}
  </FoundryWidget>,
);

import foundryWidgetPlugin from "@osdk/widget.vite-plugin";
import { defineConfig } from "vite";

export default defineConfig({
  plugins: [foundryWidgetPlugin()],
  build: {
    rollupOptions: {
      input: ["./first.html", "./second.html"],
    },
  },
});

在小部件集的生产构建中创建的 .palantir/widgets.config.json 清单文件现在应包含有关您额外小部件的信息。

发布额外的小部件

按照我们 文档 中的说明操作 发布 widget 集的新版本。您的附加 widget 在发布后将可供使用。请注意，在继续开发之前，您必须先发布该附加 widget 的第一个版本。
