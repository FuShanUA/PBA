数据集预览常见问题解答

以下是关于数据集预览的一些常见问题。

有关一般信息，请参阅《数据集预览概述》。

  • 由于引号不匹配导致 CSV 解析失败
  • 由于部分底层 CSV 包含更多列导致数据集解析失败
  • 在 Microsoft Excel 中打开下载的数据集时显示异常的原因是什么？

由于引号不匹配导致 CSV 解析失败

上传的 CSV 文件中，如果带引号的字段内包含嵌套的双引号和换行符（\n），模式推断将会失败，导致无法使用模式编辑器创建有效模式。

如需排查此问题，请执行以下步骤：

  1. 将该 CSV 文件上传至数据集，然后选择编辑模式。
  2. 设置列和引号属性。
  3. 忽略模式验证错误，在保存并验证旁边的下拉菜单中选择不验证直接保存。这将创建一个包含正确列定义的模式。
  4. 在详细信息选项卡中，以编辑模式打开该模式。
  5. 将 dataFrameReaderClass 修改为 com.palantir.foundry.spark.input.DataSourceDataFrameReader。
  6. 将以下内容添加到 "customMetadata" 对象中：
  "customMetadata": {
    "format": "csv",
    "options": {
      "header": true,
      "multiLine": true
    }
  }

由于部分底层 CSV 包含更多列导致数据集解析失败

如果数据集由多个 CSV 文件组成（例如通过数据连接的 APPEND 事务生成），且其中部分 CSV 包含更多列，模式推断将会失败。一种处理方式是忽略参差不齐的行（例如缺失某些列的行）。为此，请选择编辑模式，展开解析选项区域，然后勾选忽略参差不齐的行。

然而，如果希望保留这些参差不齐的行并为数据集指定一个标准化模式，则适用本节内容。如果您的数据符合下文前提条件部分列出的情况，那么按照排查步骤操作后，将生成一个由用户定义的标准模式数据集，其中参差不齐的行在缺失的列上会自动填充 null 值。

解析失败的错误表现：

您可能会遇到类似以下的错误信息：“无法加载预览：解析输入的 CSV 数据时遇到错误。请确保所有数据格式正确。”

在选择编辑模式然后选择保存并验证之后，您也可能会遇到以下错误信息：“您的数据集有 x 行验证失败。”

前提条件：

  1. 您需要能够定义目标模式，即数据集应包含的所有列名和类型。

  2. 您的模式需强制执行严格的列顺序。例如，如果希望数据集包含并显示 {a, b, c} 列，底层 CSV 可以具有如下的列结构：

  • {a}
  • {a, b}

但不能具有如下的列结构：

  • {b, a}
  1. 如果底层 CSV 包含第 (n+1) 列，则它必须包含第 n 列之前的所有列。例如，如果希望数据集包含并显示 {a, b, c, d} 列，底层 CSV 可以具有如下的列结构：
  • {a}
  • {a, b}
  • {a, b, c}

但不能具有如下的列结构：

  • {b, c, d}
  • {a, c, d}
  • {a, b, d}

以下示例说明上述排查步骤适用且有效的场景：

CSV 文件通过 APPEND 事务定期添加到数据集中。某天新增了一个列，并成为 CSV 中的最后一列。在数据集中，之前所有已追加 CSV 中的行都应该包含这个新列，其字段值自动填充为 null，而不是被视为参差不齐的行。

这些排查步骤无法复制原始 Parquet 数据集可用的 mergeSchema 选项功能（该数据集使用 com.palantir.foundry.spark.input.ParquetDataFrameReader 作为 dataFrameReaderClass 进行解析）。要在原始 CSV 数据集上实现类似功能，需要通过用户编写的转换来完成。

如需排查此问题，请执行以下步骤：

  1. 在详细信息选项卡中，打开模式选项卡，然后选择编辑
  2. 修改 fieldSchemaList，确保其包含数据集应包含的所有列。例如，如果数据集应包含 {a, b, c} 列且均为整数类型，fieldSchemaList 可能如下所示：
  "fieldSchemaList": [
    {
      "type": "INTEGER",
      "name": "a",
      "nullable": null,
      "userDefinedTypeClass": null,
      "customMetadata": {},
      "arraySubtype": null,
      "precision": null,
      "scale": null,
      "mapKeyType": null,
      "mapValueType": null,
      "subSchemas": null
    },
    {
      "type": "INTEGER",
      "name": "b",
      "nullable": null,
      "userDefinedTypeClass": null,
      "customMetadata": {},
      "arraySubtype": null,
      "precision": null,
      "scale": null,
      "mapKeyType": null,
      "mapValueType": null,
      "subSchemas": null
    },
    {
      "type": "INTEGER",
      "name": "c",
      "nullable": null,
      "userDefinedTypeClass": null,
      "customMetadata": {},
      "arraySubtype": null,
      "precision": null,
      "scale": null,
      "mapKeyType": null,
      "mapValueType": null,
      "subSchemas": null
    }
  ],
  1. 修改 "dataFrameReaderClass" 及其嵌套的 customMetadata 对象，使模式 JSON 的末尾如下所示：
"dataFrameReaderClass": "com.palantir.foundry.spark.input.DataSourceDataFrameReader",
  "customMetadata": {
    "format": "csv",
    "options": {
      "multiLine": true,
      "header": true,
      "mode": "PERMISSIVE"
    }
  }
}
  1. 选择保存。

为什么下载的数据集在 Microsoft Excel 中打开后看起来不正常？

从平台导出数据集时，部分文件在打开后可能会出现内容挤压在一起的情况。在某些地区已观察到此问题，其原因在于 Excel 使用的默认分隔符。要解决此问题，需在导出设置中更改分隔符模式：

  1. 在 Excel 中打开该文件。
  2. 在功能区中选择数据选项卡。
  3. 在数据工具组中选择分列选项。
  4. 在文本分列向导窗口中，选择分隔符号选项，然后点击下一步。
  5. 在分隔符号部分，选择您要使用的分隔符（例如逗号、分号、制表符）。
  6. 选择下一步，然后选择完成以完成操作。
