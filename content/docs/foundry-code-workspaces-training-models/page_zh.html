训练并与模型交互

与模型交互仅在 JupyterLab® 代码工作区 中支持。

代码工作区允许您训练和发布模型，以及导入它们以与之前发布的模型进行交互。

代码工作区要求您为每个 Foundry 模型资产 创建一个 模型别名（model alias）。模型别名作为对特定模型的引用，使您可以与模型进行交互，以发布新版本或下载保存的模型文件。

在 模型（Models） 选项卡中注册模型时，代码工作区会在工作区的 /home/user/repo/.foundry 文件夹下的隐藏文件中创建所选模型别名与 Foundry 模型唯一标识符之间的映射关系。

要了解如何从 Jupyter® 笔记本发布模型，请查看 模型集成文档。

将模型添加到工作区

所有模型都可以从模型侧边栏面板创建或导入。

要创建一个新模型，打开模型侧边栏面板并选择添加 > 创建新模型。这将引导您创建一个新模型、注册别名，并提供发布模型的代码片段。这还将生成一个骨架 模型适配器 Python 文件，该文件必须实现以将模型发布回 Foundry。

要导入现有模型，打开模型侧边栏面板并选择添加 > 导入现有模型。这将引导您导入模型并注册别名，并提供下载已保存模型状态文件的代码片段。

导入的模型会从给定的模型版本中加载序列化的模型状态。Code Workspaces 提供了不同的控件，用于在导入模型时选择使用哪个模型版本，允许您选择当前工作区分支上的最新版本、提供的分支上的最新版本或指定的模型版本。

可用代码片段

模型侧边栏面板提供了可复制和粘贴到笔记本中的代码片段，用于常见的工作流。所有代码片段都会指导您安装 palantir_models，这是Palantir提供的用于开发和与模型交互的库。

侧边栏中的每个模型卡片都会创建特定于该模型的独特代码片段。选择模型卡片将展开卡片并显示一个选择器，以选择显示特定任务的代码片段。

发布模型版本

要发布一个模型版本，您首先需要 开发一个模型适配器。对于在代码工作区内部创建的模型，会为您创建一个骨架适配器，以便在以模型别名命名的 Python 文件中实现。模型卡片将提供有关 开发您的模型适配器 和 将您的模型重新发布到 Foundry 的详细说明。

下面的代码片段适用于别名为 linear_regression_model 的模型。假设已在 linear_regression_model_adapter.py 中编写了模型适配器，模型发布代码应如下所示：

Copied!
1from palantir_models.code_workspaces import ModelOutput
2
3# Model adapter api has been defined in linear_regression_model_adapter.py
4from linear_regression_model_adapter import LinearRegressionModelAdapter
5
6# sklearn_model is a model trained in another cell
7linear_regression_model_adapter = LinearRegressionModelAdapter(sklearn_model)
8
9model_output = ModelOutput("linear_regression_model")
10model_output.publish(linear_regression_model_adapter)

下载模型

要下载之前发布的模型版本，请将代码片段选择器更改为 下载模型 . 这将提供对解析模型版本的控制，这将控制哪个模型版本用于下载模型文件。

  • 当前分支上的最新版本： 所选模型版本将是当前工作区分支上的最新版本。
  • 指定分支上的最新版本： 所选模型版本将是所选分支上的最新版本。
  • 特定模型版本： 所选模型版本将被固定到给定的模型版本RID。

代码片段将自动更新以支持所选模型版本解析控制。请查看下面的 ModelInput 类文档以了解更多信息。

为推理部署模型

模型可以在代码工作区中进行推理部署。这些部署是按模型版本共享的，这意味着一个特定的模型版本最多只能有一个相关的部署，并且这些部署与现有的 建模目标实时部署 和 模型部署 完全隔离。部署可以从代码中完全配置，并具有默认的扩展策略，在 20 分钟无使用后将关闭部署。

要加载特定模型版本的部署，请使用 选择代码片段 下拉菜单，并将选项更改为 用于推理的部署。这将加载部署模型的代码片段，并显示与 下载文件 选项相同的选取器以选择模型版本。这些代码片段将引导您完成模型的部署和对其执行推理的过程。

要在代码工作区内部监控部署，请展开您正在使用的模型的模型卡片并选择 部署。 . 在“部署”选项卡中，将显示已部署模型版本的列表。选择一个模型版本将在右侧展开一个新的面板，其中包含有关部署的信息，例如使用的资源、日志等。

请参阅下面的 ModelInput 类文档以了解更多信息。

开发模型适配器

模型适配器用于指导Foundry如何在平台上保存、加载和执行模型。发布模型时，必须使用模型适配器包装模型。

在代码工作区中，模型适配器必须定义在一个单独的 .py 文件中。当向工作区添加新模型时，会根据提供的别名名称生成一个文件。例如，如果别名是 my_model，则会生成一个名为 my_model_adapter.py 的文件，并带有名为 MyModelAdapter 的骨架模型适配器类。

在模型面板中提供的代码片段是基于别名、文件名和类名匹配的假设生成的。如果文件被移动或适配器类被重命名，您可能需要手动编辑生成的片段。

了解有关开发模型适配器的更多信息。

模型输出类

从ModelOutput导入的palantir_models.code_workspaces类为您的Foundry模型提供了一个可写的引用。要发布您的模型，您必须将模型包装在模型适配器中，并使用publish方法传递给引用。

调用publish方法会将模型保存为新的模型版本。如果model_output.publish方法调用是任何Jupyter®笔记本单元格中的最后一行，则会显示一个预览窗口，确认模型已成功发布。

模型输入类

从ModelInput导入的palantir_models.code_workspaces类提供了一个轻量级的可读引用，用于您的Foundry模型，允许读取已发布模型的序列化状态，并将模型本地部署以进行推理。下面的代码显示了如何获取一个别名为my_model的模型的可读引用，并获取test分支上的最新模型版本：

Copied!
1from palantir_models.code_workspaces import ModelInput
2my_model_input = ModelInput("my_alias", branch="test")

下载模型

ModelInput允许下载在模型发布时存储的序列化模型权重。

Copied!
1from palantir_models.code_workspaces import ModelInput
2# get the latest model version
3my_model_input = ModelInput("my_alias")
4
5# download the files
6my_model_input.download_files()

这会返回一个临时目录的路径，文件将被下载到该目录。

初始化模型适配器

对于涉及模型微调/重新发布的流程，通常很有用的是能够下载模型文件并将其加载到适配器中。如果工作区有用于发布模型的适配器的本地副本，或者该适配器来自共享适配器包并且您已将它作为依赖项添加到工作区中，那么您可以加载适配器并像在其他地方一样使用适配器。

Copied!
1# in this example, the adapter is in a local python file `my_adapter_file.py`
2from my_adapter_file import MyAdapter
3from palantir_models.code_workspaces import ModelInput
4
5my_model_input = ModelInput("my_alias", model_version="ri.models.main.model-version...")
6
7# initialize the adapter
8initialized_adapter = my_model_input.initialize_adapter(MyAdapter)
9
10# the initialized adapter can now be republished using `ModelOutput`, or you can access the python objects stored inside my adapter
11# for example, if there is a variable `pipeline` that holds a huggingface pipeline. Note that this code varies depending on your
12# adapter definition.
13pipeline = initialized_adapter.pipeline

部署用于推理的模型

模型也可以在代码工作区中部署用于推理。这些部署是按模型版本共享的，因此使用相同模型版本的多个并发会话会发送请求到同一个部署。

要与模型部署进行交互，必须为模型获取一个部署客户端。

Copied!
1from palantir_models.code_workspaces import ModelInput
2
3my_model_input = ModelInput("my_alias")
4
5deployment_client = my_model_input.deploy(cpu=2, memory="8G", max_replicas=1)

部署客户端有许多可用于控制部署的方法。

Copied!
1# wait until the model is ready
2client.wait_for_readiness()
3
4# scale the deployment
5client.scale(cpu=4)
6
7# run inference
8client.predict(...)
9
10# manually shutdown (deployments will auto-shutdown after 20 minutes of no use)
11client.disable()

推理输入大小限制

单个推理请求的大小有40 MB 的限制，因为这种方法旨在涉及其他模型的快速迭代，而不是用于生产规模的推理。实际上，这通常对应于大约 100,000 行的平均数据集，具体取决于行的大小。

如果您需要在更大的数据集上运行您的模型，可以启用 分块，这会将大数据集分解为可管理的块，并一次发送一个块到模型中。您可以通过将 client.use_chunking 方法设置为 True 来启用分块，如下所示：

Copied!
1client.use_chunking = True
2client.predict(...)

请注意，目前分块仅支持一个数据集作为输入，但您可以包含任意数量的其他参数。如果您的数据集超出这些限制或分块方法不足以满足您的工作流，请考虑使用代码仓库和转换来处理更大规模的数据处理。

转换

Jupyter® 笔记本如果可以生成模型输出，也可以在从代码工作区创建的转换中使用。请按照以下步骤为模型输出注册一个转换：

  1. 在模型面板中，找到您希望从转换中输出的模型卡片，然后在菜单中选择作为转换输出发布。

  1. 在创建转换面板中，您可以配置要执行的文件以及转换所需的输入和输出。在下面的示例中，训练数据用作输入，测试数据集和模型用作输出。

  1. 一旦输入和输出配置完成，选择同步并运行 在配置页面底部保存您当前的代码更改并创建转换。转换将开始运行，并且作业的状态将在模型面板中显示，同时关联的数据集和模型也会显示出来。

然后您可以选择查看构建以查看正在运行的转换作业。如果您对笔记本代码进行了更改并希望重新构建模型，请在模型卡片中选择重新构建以重新运行构建。通过转换创建的模型和数据集可以像平台其他地方创建的转换一样进行调度。了解有关调度转换的更多信息。
