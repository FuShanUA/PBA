训练模型并与之交互

仅在 JupyterLab® 代码工作区 中支持与模型进行交互。

代码工作区（Code Workspaces）支持训练和发布模型，也可以导入模型并与已发布的模型进行交互。

代码工作区要求您为工作区中引用的每个 Foundry 模型资产 创建一个 模型别名。模型别名充当对特定模型的引用，让您能够通过操作模型来发布新版本或下载已保存的模型文件。

在 模型 标签页中注册模型时，代码工作区会在工作区 /home/user/repo/.foundry 文件夹下的一个隐藏文件中，建立所选模型别名与 Foundry 模型唯一标识符之间的映射关系。

要了解如何从 Jupyter® 笔记本发布模型，请查阅 《模型集成文档》。

向工作区添加模型

所有模型的创建或导入工作，均可在 模型 侧边栏面板中完成。

要创建新模型，请打开 模型 侧边栏面板，选择 添加 > 创建新模型。系统将引导您完成新模型的创建和别名注册，并提供用于发布该模型的代码片段。同时，系统还会生成一个基础的 模型适配器 Python 文件，您必须实现该文件才能将模型重新发布回 Foundry。

要导入现有模型，请打开 模型 侧边栏面板，选择 添加 > 导入现有模型。系统将引导您完成模型导入和别名注册，并提供用于下载已保存模型状态文件的代码片段。

导入的模型会从指定的模型版本中加载序列化的模型状态。代码工作区针对导入模型时使用哪个版本提供了不同的控制选项，允许您选择当前工作区分支上的最新版本、指定分支上的最新版本，或某个特定的模型版本。

可用的代码片段

模型 侧边栏面板提供了一些代码片段，方便您将其复制并粘贴到笔记本中用于常见工作流。所有代码片段都会提示您安装 palantir_models，这是 Palantir 提供的用于开发和交互模型的库。

侧边栏中的每个模型卡片都会生成特定于该模型的专属代码片段。选择模型卡片即可将其展开，并显示一个选择器，供您选择查看特定任务的代码片段。

发布模型版本

要发布模型版本，您必须先 开发模型适配器。对于在代码工作区内创建的模型，系统会生成一个基础适配器，供您在一个以该模型别名命名的 Python 文件中进行实现。模型卡片将提供有关 开发模型适配器 和 将模型重新发布回 Foundry 的详细说明。

下方代码片段适用于别名为 linear_regression_model 的模型。假设已在 linear_regression_model_adapter.py 中编写好模型适配器，模型发布代码应如下所示：



已复制！
1from palantir_models.code_workspaces import ModelOutput
2
3# 模型适配器 API 已在 linear_regression_model_adapter.py 中定义
4from linear_regression_model_adapter import LinearRegressionModelAdapter
5
6# sklearn_model 是在另一个单元格中训练的模型
7linear_regression_model_adapter = LinearRegressionModelAdapter(sklearn_model)
8
9model_output = ModelOutput("linear_regression_model")
10model_output.publish(linear_regression_model_adapter)

下载模型

要下载之前发布的模型版本，请将代码片段选择器切换为下载模型。系统将提供用于解析模型版本的控制选项，这些选项将决定下载模型文件时使用哪个模型版本。

  • 当前分支上的最新版本：选定的模型版本将是当前工作空间分支上的最新版本。
  • 指定分支上的最新版本：选定的模型版本将是指定分支上的最新版本。
  • 特定模型版本：选定的模型版本将固定为给定的模型版本 RID。

代码片段会自动更新以适配所选的模型版本解析控件。请查阅下方的 ModelInput 类文档以了解更多信息。

部署模型进行推理

模型可以在代码工作空间中部署以进行推理。这些部署是按模型版本共享的，这意味着一个给定的模型版本最多只能有一个关联的部署，且这些部署与现有的 建模目标实时部署和 模型部署完全独立。部署可以通过代码进行完全配置，并具有默认的扩缩容策略：在 20 分钟无使用后自动关闭部署。

要加载给定模型版本的部署，请使用选择代码片段下拉菜单，并将选项更改为部署以进行推理。系统将加载用于部署模型的代码片段，并显示与下载文件选项相同的用于选择模型版本的选择器。这些代码片段将引导您完成部署模型并对其进行推理的过程。

要在 Code Workspaces 中监控部署，请展开您正在使用的模型卡片，并选择 部署。在部署选项卡中，将显示已部署模型版本的列表。选择某个模型版本会展开右侧的一个新面板，其中包含有关该部署的信息，例如使用的资源、日志等。

请参阅下方的 ModelInput 类文档以了解更多信息。

开发模型适配器

模型适配器用于指示 Foundry 如何在整个平台中保存、加载和执行模型。在发布模型时，您必须使用模型适配器包装该模型。

在 Code Workspaces 中，模型适配器必须定义在单独的 .py 文件中。向工作区添加新模型时，系统会根据给定的别名生成一个文件。例如，如果别名为 my_model，则会生成一个名为 my_model_adapter.py 的文件，其中包含名为 MyModelAdapter 的模型适配器类骨架。

模型面板中提供的代码片段是在假设别名、文件名和类名相匹配的前提下生成的。如果移动了该文件或重命名了适配器类，您可能需要手动编辑生成的代码片段。

了解有关开发模型适配器的更多信息。

模型输出类

从 palantir_models.code_workspaces 导入的 ModelOutput 类提供了对 Foundry 模型的可写引用。要发布模型，您必须将模型包装在模型适配器中，并使用 publish 方法将其传递给该引用。

调用 publish 方法会将模型保存为新的模型版本。如果 model_output.publish 方法调用是任何 Jupyter® 笔记本单元格中的最后一行，则会显示一个预览窗口，确认模型已成功发布。

模型输入类

从 palantir_models.code_workspaces 导入的 ModelInput 类提供了对 Foundry 模型的轻量级可读引用，允许读取已发布模型的序列化状态，并在本地部署模型进行推理。以下代码展示了获取别名为 my_model 的模型可读引用的示例，该示例获取了 test 分支上的最新模型版本：

Copied!
1from palantir_models.code_workspaces import ModelInput
2my_model_input = ModelInput("my_alias", branch="test")

下载模型

ModelInput 允许下载在模型发布时存储的已序列化模型权重。

2# 获取最新模型版本

4
5# 下载文件
6my_model_input.download_files()

该操作会返回文件下载到的临时目录路径。

初始化模型适配器

对于涉及模型微调或重新发布的工作流，支持下载模型文件并将其加载到适配器中往往大有益处。如果工作区中已存在用于发布模型的本地适配器副本，或者该适配器来自共享适配器包且已作为依赖项添加到工作区中，您就可以像在其他任何地方一样加载并使用该适配器。



已复制！
1# 在此示例中，适配器位于本地 python 文件 `my_adapter_file.py` 中
2from my_adapter_file import MyAdapter
3from palantir_models.code_workspaces import ModelInput
4
5my_model_input = ModelInput("my_alias", model_version="ri.models.main.model-version...")
6
7# 初始化适配器
8initialized_adapter = my_model_input.initialize_adapter(MyAdapter)
9
10# 现在可以使用 `ModelOutput` 重新发布已初始化的适配器，或者您可以访问存储在我的适配器内部的 python 对象
11# 例如，如果有一个变量 `pipeline` 持有 huggingface pipeline。请注意，此代码会根据您的
12# 适配器定义而有所不同。
13pipeline = initialized_adapter.pipeline

部署模型用于推理

还可以在代码工作区内部署模型进行推理。这些部署是按模型版本共享的，因此使用相同模型版本的多个并发会话会将请求发送到同一部署。

要与模型部署进行交互，您必须获取该模型的部署客户端。



Copied!
1from palantir_models.code_workspaces import ModelInput
2
3my_model_input = ModelInput("my_alias")
4
5deployment_client = my_model_input.deploy(cpu=2, memory="8G", max_replicas=1)

部署客户端提供了多种方法来控制部署过程。



已复制！
1# 等待模型就绪
2client.wait_for_readiness()
3
4# 扩缩容部署
5client.scale(cpu=4)
6
7# 运行推理
8client.predict(...)
9
10# 手动关闭（部署在无使用 20 分钟后会自动关闭）
11client.disable()

推理输入大小限制

单次推理请求的大小限制为 40 MB，因为这种方式旨在实现涉及其他模型的快速迭代，而非用于生产规模的推理。在实际应用中，这通常对应于大约 100,000 行的平均数据集，具体取决于行的大小。

如果需要在更大的数据集上运行模型，可以启用分块功能。该功能会将大型数据集分解为易于管理的小块，并每次向模型发送一个块。可以通过将 client.use_chunking 方法设置为 True 来启用分块，如下所示：



已复制！
1client.use_chunking = True
2client.predict(...)

请注意，分块目前仅支持单个数据集作为输入，但您可以包含任意数量的其他参数。如果您的数据集超出这些限制，或者分块方法无法满足您的工作流需求，请考虑使用代码仓库和转换来处理更大规模的数据。

转换

生成模型输出的 Jupyter® 笔记本也可用于在代码工作空间中创建的转换。请按照以下步骤为模型输出注册转换：

  1. 在模型面板中，找到要从转换输出的模型对应的模型卡片，然后在菜单中选择发布为转换输出。

  1. 在创建转换面板中，您可以配置要执行的文件以及转换所需的输入和输出。在下面的示例中，训练数据用作输入，测试数据集和模型用作输出。

  1. 配置好输入和输出后，选择配置页面底部的同步并运行以保存当前代码更改并创建转换。转换将开始运行，作业状态将在模型面板中与关联的数据集和模型一起显示。

然后，您可以选择查看构建以查看正在运行的转换作业。如果您更改了笔记本代码并希望重新构建模型，请在模型卡片中选择重新构建以重新运行构建。通过转换创建的模型和数据集可以像平台其他地方创建的转换一样进行调度。了解更多关于调度转换的信息。
