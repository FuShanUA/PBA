在 Foundry 上托管 OSDK 应用

Developer Console 中的网站托管功能为使用 OSDK 构建纯前端应用的开发者提供了直接在 Foundry 上托管这些应用的选项，从而无需额外的托管基础设施。

与 GitHub Pages 类似，该网站托管功能仅支持托管静态资源，不支持运行服务器。这意味着您可以托管：

  • HTML、CSS 和 JavaScript 文件
  • 完全在浏览器中运行的单页应用（React、Vue、Angular 等）
  • 图像、字体和其他静态资源

您无法使用此功能运行诸如 Node.js 后端、Python 服务器或服务端渲染等服务器端代码。对于任何服务端功能，您的应用必须通过 OSDK 或其他外部服务向 Foundry 发起 API 调用。

网站托管仅适用于配置为面向客户端的应用。如果您的应用同时配置为后端服务，网站托管选项将不会显示，因为这种组合会创建专用于服务器端应用的机密客户端。

每个托管的网站都可以通过您 Foundry 注册域名的子域名或您拥有的自定义域名进行访问。默认情况下，您将选择一个子域名，并且您的应用将通过 <YOUR-APPLICATION-SUBDOMAIN>.[YOUR-ENROLLMENT].palantirfoundry.com 提供服务。或者，您也可以将应用托管在自定义域名上，例如 [your-organization].com。详情请参阅在自定义域名上托管网站。

如果您的 Foundry 注册域名不以 .palantirfoundry.com 结尾，请联系 Palantir 支持团队以协助设置网站托管，因为这需要进行额外的协调。

准备您的应用

以下部分介绍了在 Foundry 上托管 Developer Console 应用所需的步骤。

单页应用渲染

如果您的应用中未包含自定义 404 页面，Foundry 将假定这是一个单页应用 ↗，并将指向该子域名下任意路径的请求路由至 index.html。

更新重定向 URL

作为身份验证流程的一部分，您需要更新重定向 URL，使其包含您的托管域名并在末尾加上 /auth/callback。如果您使用的是注册子域名，该 URL 应为 <YOUR-APPLICATION-SUBDOMAIN>.[YOUR-ENROLLMENT].palantirfoundry.com/auth/callback。如果您使用的是自定义域名，该 URL 应为 <YOUR-CUSTOM-DOMAIN>/auth/callback。 您还必须在 Developer Console 中将相同的重定向 URL 添加到您的应用中。详情请参阅创建新的 OSDK。

准备资源

将包含网站文件生产构建版本的目录内容进行压缩。请勿包含目录本身。在常见的 Web 框架中，该目录通常是 dist/。

如果在压缩文件中包含了任何目录，这些目录将成为您网站路径的一部分。

设置域名

在 Developer Console 的应用页面中，从左侧菜单中选择网站托管。

在子域名上托管网站

若要在您注册域名的子域名上托管网站，请按照以下步骤操作。

  1. 为您的应用选择子域名；可以是应用名称或您选择的任何其他名称。然后，选择申请应用域名。在以下示例中，我们选择了 my-first-hosted-app.example.palantirfoundry.com：



  2. 请求您所在环境的信息安全官审批，或者如果您具备相应权限，可通过选择查看申请自行批准。环境管理员可以在 Control Panel 中管理环境权限。



  3. 请求获批后，刷新页面。此时应显示 **Domain ready**，表明该域名已准备就绪。此过程可能需要几分钟完成。



在自定义域名上托管您的网站

您可以不使用环境子域名，而是将应用程序托管在自己拥有的自定义域名上，例如 [your-organization].com。当您希望通过品牌化或面向公众的域名访问应用程序时，此功能非常实用。

并非所有环境都提供自定义域名托管功能。如果该选项未在您的 Developer Console 应用程序中显示，请联系您的 Palantir 代表寻求帮助。

前提条件

在为应用程序申请自定义域名之前，请确保已在 Control Panel 中创建了涵盖该域名的证书。如果该域名没有对应的证书，审批请求将无法成功。有关创建证书的说明，请参阅《配置域名和证书》。

申请自定义域名

在 Developer Console 的应用程序中，从左侧菜单中选择 **Website hosting**。

  1. 选择 **Request to host on a custom domain**。该选项显示在环境子域名注册表单下方。



  2. 在弹出的对话框中，输入您的自定义域名（例如 [your-organization].com）、请求标题和可选的描述，然后选择 **Request**。



  3. 这将创建一个审批任务，必须由您所在环境的 **Information Security Officer** 批准。您可以在成功通知中选择 **View**，以导航到 Control Panel 中的审批请求。



  4. 请求获批后，自定义域名即与您的应用程序关联。您可能需要刷新页面才能看到更新后的状态。请注意，您用于登录 Foundry 的域名将与您配置的自定义域名关联。换言之，网络入口和身份验证提供程序配置将继承自该域名。



您还必须更新自定义域名的 DNS 设置，使其指向您的 Foundry 环境。有关更新 DNS 记录的指导，请参阅《配置域名和证书》。

高级配置

你可以在 网站托管 页面的 高级 选项卡中找到额外的配置选项。

内容安全策略

默认情况下，你的应用将使用受限的 内容安全策略 (CSP) ↗ 提供服务，该策略仅允许从你的子域加载资源。如有需要，你可以为应用内的特定交互配置额外的 CSP 规则，这些规则将与默认策略合并。但请注意，进行这些更改可能会增加你的应用遭受跨站脚本 (XSS) 和数据注入攻击的风险。

路由匹配规则

Foundry 支持在带有或不带有扩展名和尾部斜杠的路由上提供 HTML 页面。

给定以下网站文件布局：

├── file.html
├── folder
│   └── index.html
├── both.html
└── both
    └── index.html

Foundry 在以下路由上提供这些 HTML 页面：

路由文件
/file/file.html
/file//file.html
/file.html/file.html
/folder/folder/index.html
/folder//folder/index.html
/folder/index.html/folder/index.html
/both/both.html
/both.html/both.html
/both//both/index.html
/both/index.html/both/index.html

Foundry 不会重定向到首选路由格式，例如强制使用尾部斜杠或从路由中移除扩展名。

自定义 404 页面

你可以在网站的根目录添加一个 404.html 页面，作为路由未匹配时的自定义错误页面。这将禁用为未匹配路由提供根 index.html 页面的默认行为，如 单页应用 (SPA) 渲染 中所述。
