工具搜索

Palantir MCP 通过工具搜索支持按需工具发现。当启用工具搜索时，服务器在启动时会暴露一个单独的工具 search_tools，而不是完整的工具目录。您的代理仅在搜索工具时激活额外的工具，因此加载到您的上下文中的工具定义更少，这可以减少令牌使用量。

工具搜索的工作原理

默认情况下，所有工具在启动时都会加载，并在整个会话中保持可用。当启用工具搜索时，Palantir MCP 启动时只会暴露 search_tools 工具。从那里：

  1. 当您提示代理时，代理会使用您任务的简短描述调用 search_tools。
  2. search_tools 在本地对工具名称、类别和精选关键词进行排名，无需额外的模型调用或网络往返。然后它会激活匹配度最高的工具并返回它们的描述。
  3. 代理直接调用已激活的工具。

这可以保持您的上下文较小。而不是一开始就加载数十个工具定义，代理只会加载与任务相关的少数几个。

激活的工具会保留在您的工具列表中，并在会话的其余时间保持在您的上下文中。重新连接到服务器会将您的工具列表重置为仅包含search_tools。

客户端要求：工具搜索依赖于您的MCP客户端在新工具被激活时刷新其工具列表，使用tools/list_changed通知。不进行动态刷新的客户端将无法看到新激活的工具。因此，Continue 不使用工具搜索。

启用工具搜索

工具搜索是可选的。默认情况下，所有工具在启动时都会加载。要启用工具搜索，请执行以下操作之一：

  • 将--tool-search参数传递给服务器，或者
  • 设置环境变量PALANTIR_MCP_TOOL_SEARCH=true。

这两种选项是等效的；请根据您的配置选择使用其中一种。

Claude Code

当使用 claude mcp add 添加 Palantir MCP 时，传递 --tool-search 参数。该参数必须位于 -- 分隔符之后，作为 palantir-mcp 的参数，而不是 claude mcp add 的参数。



已复制！
1claude mcp 添加 palantir-mcp \
2    --作用域 项目 \
3    -e FOUNDRY_TOKEN=<令牌> \
4    -- npx "-y" "palantir-mcp" "--tool-search" "--foundry-api-url" "https://<您的 foundry 主机名>.palantirfoundry.com"

或者设置环境变量：



已复制！
1claude mcp add palantir-mcp \
2    --scope project \
3    -e FOUNDRY_TOKEN=<token> \
4    -e PALANTIR_MCP_TOOL_SEARCH=true \
5    -- npx "-y" "palantir-mcp" "--foundry-api-url" "https://<your-foundry-hostname>.palantirfoundry.com"

Agent Flows (pro-code agents)

In a pro-code agent, add the --tool-search argument to the server configuration:



Copied!
1export const PALANTIR_MCP_CONFIGURATION: McpStdioServerConfig = {
2  type: "stdio",
3  command: "palantir-mcp",
4  args: ["start", "--tool-search", "code-workspaces"],
5  env: {
6    FOUNDRY_PROXY_URL: `https://${process.env.FOUNDRY_PROXY_URL!}/`,
7    FOUNDRY_SERVICE_DISCOVERY_V2: process.env.FOUNDRY_SERVICE_DISCOVERY_V2!,
8    FOUNDRY_EXTERNAL_HOST: "https://<your-foundry-hostname>.palantirfoundry.com",
9    FOUNDRY_PROXY_TOKEN: process.env.FOUNDRY_PROXY_TOKEN!,
10    FOUNDRY_SCOPED_TOKEN: process.env.FOUNDRY_SCOPED_TOKEN!,
11    MAESTRO_REPO_RID: process.env.MAESTRO_REPO_RID!,
12    NODE_EXTRA_CA_CERTS: process.env.NODE_EXTRA_CA_CERTS!,
13  },
14};

Alternatively, add PALANTIR_MCP_TOOL_SEARCH: "true" to the env block instead of passing the --tool-search argument.
