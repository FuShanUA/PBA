异常检测

资源管理的异常检测功能仅对与Palantir拥有< strong >基于使用量的活跃合同的客户可用。

异常检测在资源管理中主动通知用户异常的资源使用模式。异常检测器由以下部分组成：

  • 范围：要监控的实体。有关更多详细信息，请参阅下面的 范围 部分。
  • 策略：检测异常的方法。每种策略都有独特的配置。
  • 异常：被选定策略视为异常的事件。
  • 订阅者 当检测到异常时，将通知的组/用户。

用户创建异常检测器；每个异常检测器依赖于一个范围和策略来检测异常并通知其订阅者。

由于测量使用情况的延迟，异常可能在发生后才被检测到（例如，配置一个当使用量超过移动平均值的70%时检测异常的策略，可能会在75%时检测到异常）。在某些罕见情况下，这种延迟可能长达26小时。

作用域

异常检测器目前设计为监控 所有使用情况。将来，它们还将能够处理 使用账户。

策略

异常检测器目前支持 2 种不同的策略：

  • 偏离移动平均值： 当使用量超过或低于移动平均值时通知订阅者；此策略可以突出显示使用量的临时增加和减少。
  • 零使用：当在一段时间内使用量几乎为零时，会通知订阅者；此策略可能表明出现重大问题，例如数据管道故障。

权限

要使用异常检测功能，需要以下角色之一：

  • 注册管理员: 查看、创建、编辑和删除异常检测器。
  • 资源管理管理员: 查看、创建、编辑和删除异常检测器。
  • 资源管理查看员: 查看异常检测器。

通过 注册权限 页面 在控制面板中授予角色。

查看所有异常检测器

在资源管理中，选择左侧边栏中的 异常检测器。这将显示您注册中所有可用的异常检测器列表。选择一个异常检测器以查看其异常和详细信息。定期观察有助于了解异常检测的频率以及检测器是否提供有用信号。

Create an anomaly detector

To create an anomaly detector, select the New button while viewing all anomaly detectors. Select a scope, configure a strategy, and specify the subscribers. Then, select Create detector.

Delete an anomaly detector

Deleting an anomaly detector also deletes its anomalies and unsubscribes all subscribers. This action cannot be undone.

While viewing a single anomaly detector, select the actions menu at the top right, and select Delete. When the warning dialog appears, select Delete detector.
