快速入门

提示

以下说明将引导您完成一次简单的 Python 数据转换。如果您刚开始接触数据转换，建议先查阅《Pipeline Builder》或《Code Repositories》的批量管道教程。

本教程将详细介绍如何使用 Python 转换，将记录近期陨石发现情况的电子表格转换为可用于分析的数据集。

关于数据集

本教程使用来自 NASA 开放数据门户 ↗ 的数据。您可以使用以下示例数据集，在自己的代码仓库中同步操作：

下载 meteorite_landings

该数据集包含了在地球上发现的陨石数据。请注意，为了便于操作，数据已经过清理。

数据集涵盖了每颗陨石的名称、质量、分类及其他识别信息，同时还包含发现年份和发现地点的坐标。在上传至 Foundry 之前，先打开 CSV 文件查看数据是一个良好的习惯。

设置 Python 代码仓库

首先创建一个 Python 代码仓库。

  1. 导航至某个项目，然后选择 + 新建 > 代码仓库。
  2. 在 仓库类型 部分，选择 管道。
  3. 选择 Python 作为 语言。
  4. 选择 初始化仓库。

可选：使用本地 Python 仓库

或者，您也可以按照以下步骤将本地 Python 仓库复制到 Code Repositories 中：

  1. 按照上述说明创建一个新的 Python 代码仓库。

  2. 在本地仓库中，移除原先的 Git origin（例如，如果您是从 GitHub 克隆的）：git remote remove origin

  3. 添加您的代码仓库的 Git 远程 URL：git remote add origin <repository_url>

    您可以在 GitHub 界面的右上角找到代码仓库的 URL。选择绿色的 克隆 按钮，然后复制 Git 远程 URL。运行 git remote -v 返回代码仓库 URL 以进行确认。

  4. 将 Code Repositories 中当前的 master 分支（或您选择的任何其他分支）合并到您的本地分支：git merge master

    如果出现关于 refusing to merge unrelated histories 的错误，请运行命令：git merge master --allow-unrelated-histories。这将清除与您之前远程 GitHub 仓库关联的当前 Git 历史记录。

    此合并操作会将必要的文件引入本地仓库，这些文件是在 Code Repositories 中提交和更改所必需的。

  5. 创建一个新分支并为其命名（例如 testbranch）：git checkout -b testbranch。

  6. 进行更改并将其提交到您的分支。

  7. 执行 git push，确认新分支已显示在 Code Repositories 界面中。验证检查是否成功。

详细了解 Code Repositories 中的《本地开发》。

添加到数据转换

使用 Python 转换，您可以在单个 Python 文件中创建多个输出数据集。

假设您想进一步筛选，仅保留相对于其陨石类型而言体积特别大的陨石。为此，您需要：

  1. 找出每种陨石类型的平均质量，以及
  2. 将每颗陨石的质量与其对应陨石类型的平均质量进行比较。

首先，在 meteor_analysis.py 中添加一个数据转换，用于计算每种陨石类型的平均质量。此转换将您的 meteor_landings 数据集作为输入，并创建 meteorite_stats 作为输出：

10def stats(output, meteorite_landings):
11    df = (
12        meteorite_landings.polars()
13        .groupby("class")
14        .agg(pl.col("mass").mean().alias("avg_mass_per_class"))

接下来，创建一个数据转换逻辑，将每颗陨石的质量与其所属陨石类型的平均质量进行对比。该转换所需的信息分散在本教程中创建的 `meteorite_landings` 和 `meteorite_stats` 两张表里。你必须关联这两个数据集，并对关联后的数据集进行过滤，以找出质量高于平均水平 的陨石，如下所示：

2from transforms.api import transform, Input, Output
3
4

6    output=Output("/Users/jsmith/meteorite_enriched"),
7    meteorite_landings=Input("/Users/jsmith/meteorite_landings"),

9)
10def enriched(output, meteorite_landings, meteorite_stats):
11    df_landings = meteorite_landings.polars()
12    df_stats = meteorite_stats.polars()
13
14    enriched = df_landings.join(df_stats, on="class", how="inner")
15    enriched = enriched.with_columns(
16        (pl.col("mass") > pl.col("avg_mass_per_class")).alias("greater_mass")
17    )
18    result = enriched.filter(pl.col("greater_mass"))
19    output.write_table(result)

将数据转换应用于多个输入

到目前为止，你已经创建了一个包含质量高于平均值的所有类型陨石的数据集。下一步是为每种陨石类型创建独立的数据集。在 Python 转换中，你可以使用 for 循环将相同的数据转换逻辑应用于每种类型的陨石。有关对不同输入应用相同数据转换的更多信息，请参阅《转换生成》部分。

在 src/myproject/datasets 目录下创建一个新文件，并命名为 meteor_class.py。请注意，你完全可以继续在 meteor_analysis.py 文件中编写数据转换代码，但本教程使用新文件是为了将数据转换逻辑隔离开来。

要为每种陨石类型创建独立的数据集，需按类别对 meteorite_enriched 数据集进行过滤。然后，定义一个 transform_generator 函数，将相同的数据转换逻辑应用于你要分析的每种陨石类型：

Copied!
1import polars as pl
2from transforms.api import transform, Input, Output
3
4
5def transform_generator(sources):
6    transforms = []
7    for source in sources:
8        @transform.using(
9            output=Output(f'/Users/jsmith/meteorite_{source}'),
10            my_input=Input('/Users/jsmith/meteorite_enriched')
11        )
12        def filter_by_source(output, my_input, source=source):
13            df = my_input.polars()
14            result = df.filter(pl.col("class") == source)
15            output.write_table(result)
16        transforms.append(filter_by_source)
17    return transforms
18
19TRANSFORMS = transform_generator(["L6", "H5", "H4"])

提示

对于在 meteor_analysis.py 文件中创建的初始数据转换，你无需进行任何额外配置即可将转换添加到项目的流水线中。这是因为默认的 Python 项目结构采用了自动注册机制，能够自动发现 datasets 文件夹内的所有转换对象。

若要利用自动注册机制将此最终转换操作添加到项目流水线中，你必须将生成的转换对象以列表形式添加到某个变量中。在上文的示例中，我们使用了 TRANSFORMS 变量。有关自动注册和转换生成器的更多详细信息，请参阅 Python 转换文档中有关 转换生成 的章节。
