提示

以下说明将引导您完成一个简单的Python数据转换。如果您刚开始接触数据转换，建议先通过 Pipeline Builder（管道构建器） 或 代码仓库 的批处理管道教程。

本教程将展示如何使用Python转换来将最近的陨石发现表格转换为可用于分析的数据集。

关于数据集

本教程使用了来自 NASA开放数据门户 ↗ 的数据。您可以使用此示例数据集在自己的代码仓库中进行操作：

Download meteorite_landings

这个数据集包含有关在地球上发现的陨石的数据。请注意，数据已经过清理，以便于使用。

该数据集包括每块陨石的名称、质量、分类和其他标识信息，以及发现年份和发现地点的坐标。建议在将数据上传到Foundry之前，先打开CSV文件以查看数据。

设置一个Python代码仓库

通过创建一个Python代码仓库开始吧。

  1. 导航到一个项目，然后选择+ 新建 > 代码仓库。
  2. 在仓库类型部分，选择管道。
  3. 将语言设置为Python。
  4. 选择初始化仓库。

可选：使用本地Python仓库

或者，您可以按照以下步骤将本地Python仓库复制到代码仓库中：

  1. 创建一个新的Python代码仓库，如上所述。

  2. 在您的本地仓库中，删除您之前的Git远程（例如，如果您从GitHub克隆了它）： git remote remove origin

  3. 添加您的代码仓库的Git远程URL： git remote add origin <repository_url>

    您可以在GitHub界面的右上角找到您的代码仓库URL。选择绿色的克隆按钮，然后复制Git远程URL。通过运行 git remote -v 来确认此URL。

  4. 将代码仓库中的当前 master 分支（或您选择的另一个分支）合并到您的本地分支中： git merge master

    如果出现关于 refusing to merge unrelated histories 的错误，请运行命令： git merge master --allow-unrelated-histories。这将删除与您之前远程GitHub仓库相关联的当前Git历史记录。

    此次合并会将必要的文件带入您的本地仓库，以便在代码仓库中进行提交和更改。

  5. 创建一个新分支并将其命名为（例如）：testbranch。

  6. 进行您的更改并将它们提交到您的分支。

  7. 执行 git push，并确认新分支出现在代码仓库界面中。验证检查是否成功。

了解有关 本地开发 的更多信息。

添加到您的数据转换

使用 Python 转换，您可以在一个 Python 文件中创建多个输出数据集。

假设您想进一步筛选出特定类型的陨石中特别大的那些。为此，您需要：

  1. 为每种陨石类型找到平均质量，并
  2. 将每颗陨石的质量与其陨石类型的平均质量进行比较。

首先，在 meteor_analysis.py 中添加一个数据转换，以找出每种陨石类型的平均质量。这个转换将您的 meteor_landings 数据集作为输入，并生成 meteorite_stats 作为其输出：

Copied!

2from transforms.api import transform, Input, Output
3
4
5@transform.using(
6    # Output dataset name must be unique
7    output=Output("/Users/jsmith/meteorite_stats"),
8    meteorite_landings=Input("/Users/jsmith/meteorite_landings"),
9)
10def stats(output, meteorite_landings):
11    df = (
12        meteorite_landings.polars()
13        .groupby("class")
14        .agg(pl.col("mass").mean().alias("avg_mass_per_class"))

接下来，创建一个数据转换，将每颗陨石的质量与该类型陨石的平均质量进行比较。此转换所需的信息分布在您在本教程中创建的meteorite_landings和meteorite_stats表中。您必须将这两个数据集进行连接，并过滤结果数据集以找到质量大于平均值的陨石，如下所示：

Copied!

2from transforms.api import transform, Input, Output
3
4
5@transform.using(
6    output=Output("/Users/jsmith/meteorite_enriched"),
7    meteorite_landings=Input("/Users/jsmith/meteorite_landings"),
8    meteorite_stats=Input("/Users/jsmith/meteorite_stats"),
9)
10def enriched(output, meteorite_landings, meteorite_stats):
11    df_landings = meteorite_landings.polars()
12    df_stats = meteorite_stats.polars()
13
14    enriched = df_landings.join(df_stats, on="class", how="inner")
15    enriched = enriched.with_columns(
16        (pl.col("mass") > pl.col("avg_mass_per_class")).alias("greater_mass")
17    )
18    result = enriched.filter(pl.col("greater_mass"))
19    output.write_table(result)

将您的数据转换应用于多个输入

到目前为止，您已经创建了一个包含所有类型且质量高于平均水平的陨石的数据集。下一步是为每种类型的陨石创建单独的数据集。使用 Python 转换，您可以使用 for 循环将相同的数据转换应用于每种类型的陨石。有关对不同输入应用相同数据转换的更多信息，请参阅 转换生成 部分。

在 src/myproject/datasets 中创建一个新文件，并将其命名为 meteor_class.py。请注意，您可以在 meteor_analysis.py 文件中继续编写数据转换代码，但本教程使用一个新文件来分离数据转换逻辑。

为了为每种类型的陨石创建单独的数据集，按类别对 meteorite_enriched 数据集进行过滤。然后，定义一个 transform_generator 函数，该函数将相同的数据转换逻辑应用于您想要分析的每种类型的陨石：

Copied!
1import polars as pl
2from transforms.api import transform, Input, Output
3
4
5def transform_generator(sources):
6    transforms = []
7    for source in sources:
8        @transform.using(
9            output=Output(f'/Users/jsmith/meteorite_{source}'),
10            my_input=Input('/Users/jsmith/meteorite_enriched')
11        )
12        def filter_by_source(output, my_input, source=source):
13            df = my_input.polars()
14            result = df.filter(pl.col("class") == source)
15            output.write_table(result)
16        transforms.append(filter_by_source)
17    return transforms
18
19TRANSFORMS = transform_generator(["L6", "H5", "H4"])

提示

对于在meteor_analysis.py文件中创建的初始数据转换，您无需进行任何其他配置即可将转换添加到项目的管道中。这是因为默认的 Python 项目结构使用自动注册来发现您datasets文件夹中的所有转换对象。

要使用自动注册将此最终转换添加到项目的管道中，您必须将生成的转换添加到一个变量中作为列表。在上面的例子中，我们使用了变量TRANSFORMS。有关自动注册和转换生成器的更多信息，请参阅 Python 转换文档中的 转换生成 部分。
