工具

工具是外部功能或 API，大语言模型（LLM）可利用其执行特定操作，或获取超出其自身能力范围的信息。在允许 LLM 决定控制流和构造输入方面，工具尤其有用。

工具类型

共有以下六种可用的工具类型：

  • 操作（Action）： 赋予聊天机器人执行本体编辑的能力。可配置为自动运行，或在用户确认后运行。
  • 对象查询（Object query）： 该工具用于指定 LLM 可访问的对象类型。可添加多种对象类型并指定可访问的属性，从而提高查询的 Token 效率。对象查询工具支持对已配置对象进行过滤、聚合、检查和链接遍历。
  • 函数（Function）： 允许 LLM 调用任意 Foundry 函数，包括已发布的 AIP Logic 函数。系统会自动使用最新版本的函数，也可指定已发布的版本以实现更精细的控制。
  • 更新应用变量（Update application variable）： 该工具用于更新在“应用状态”（Application state）选项卡中配置的应用变量值。
  • 命令（Command）： 这些工具使聊天机器人能够使用一个或多个命令，触发其他 Palantir 应用中的操作。
  • 请求澄清（Request clarification）： 该工具允许聊天机器人暂停执行，并向用户请求澄清。
  • （旧版）本体语义搜索： 该工具可使用向量属性来检索相关的本体上下文。此工具为旧版工具，不包含引用或输入/输出变量，也不会将生成的对象集返回给 LLM。建议改用“本体上下文”（Ontology context）。

工具模式

通过工具模式设置，可控制将配置好的工具提供给 LLM 的方式，以及 LLM 调用这些工具的方式。可用的工具模式设置如下：

  • 提示式工具调用： 该模式将指令插入提示词中以提供工具，并允许 LLM 使用这些工具。处于该工具模式下的聊天机器人每次只能调用单个工具，因此在回答需要多次工具调用的复杂查询时，耗时可能更长。所有工具类型和所有可用模型均支持此模式。
  • 原生工具调用： 该模式利用受支持模型的内置能力来提供工具，并允许 LLM 直接调用这些工具。由于具备更高的 Token 效率，且该模式下的聊天机器人能够并行调用多个工具，因此比提示式工具调用具有更快的速度和更好的性能。目前，此模式仅可用于 Palantir 提供的部分模型，且仅支持以下工具类型：操作、对象查询、函数和更新应用变量。若需使用原生工具调用模式不支持的模型或工具类型，请改用提示式工具调用模式。
