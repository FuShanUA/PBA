故障排除

本页面包含在使用自定义小部件时可能遇到的问题的提示。

页面崩溃

自定义小部件在 iframe 内运行，如果消耗过多内存或阻塞浏览器线程，可能会崩溃。发生这种情况时，浏览器可能会变得无响应或显示已终止状态。使用 Chrome DevTools 中的 性能 选项卡来记录浏览器执行并查找性能异常。

请查看以下常见的反模式，以减轻在使用自定义小部件时出现浏览器页面崩溃的情况。

避免无限循环和重新渲染

无限或实际上无限的循环和重新渲染是常见的错误类型，可能导致页面崩溃。

在 React ↗ 中，要注意可能导致无限重新渲染的循环。检查以下内容：

  • useEffect 钩子，这些钩子更新了相同钩子的依赖项，导致它们再次运行。
  • setTimeout、setInterval、requestAnimationFrame 循环从未被清除。
  • 事件处理程序，例如 window.addEventListener，这些处理程序导致的更新会再次触发相同的处理程序。
  • 渲染代码，对于 每个 项目在 每次 渲染时执行昂贵的操作。

function InfiniteRender({ objects }: { objects: MyObject[] }) {
    const [data, setData] = useState<MyObject[]>([]);

    // BUG: updating data which is also a dependency
    useEffect(() => {
        setData([...objects]);
    }, [objects, data]);

    return <MyComponent />;
}

// BUG: effectively infinite loop if bounds of myValues are unknown and unexpectedly large

    doSomething(currentValue);
    currentValue += 1;
}

避免将所有数据加载到浏览器中

当本体论对象类型包含大量对象时，不要将所有对象加载到浏览器中进行客户端处理。这可能会耗尽浏览器资源，并在网络上传输大量数据，这对小部件用户来说可能很慢。

let pageToken: string | undefined;
const allObjects = [];

do {
    const page = await client(MyObject).fetchPage({
        $pageSize: 1_000,
        $pageToken: pageToken,
    });

    allObjects.push(...page.data);
    pageToken = page.nextPageToken;
} while (pageToken != null);

相反：

  • 向用户显示单页数据，并仅在需要时加载其他页面。
  • 在查询中应用过滤和排序，而不是在客户端处理中。
  • 当需要汇总统计信息（如总数）时，使用聚合功能。
  • 仅请求所需的属性，对于具有许多属性的对象类型，请使用 $select。
  • 通过 .withProperties 使用 派生属性，以根据对象上其他属性或链接的值进行计算。
