模型适配器 API

模型适配器的 api() 方法用于指定预期的输入和输出，以便执行该模型适配器的推理逻辑。输入和输出需分别指定。

在运行时，系统会使用指定的输入来调用模型适配器的 predict() 方法。

api() 实现示例

以下示例展示了一个指定了一个名为 input_dataframe 的输入和一个名为 output_dataframe 的输出的 API。输入和输出对象均被指定为 Pandas 数据框，其中输入数据框包含一个名为 input_feature 的 float 类型列，而输出数据框包含两列：名为 input_feature 的 float 类型列，以及名为 output_feature 的 int 类型列。

API 类型

模型适配器 API 的输入和输出类型可以通过以下类来指定，具体定义如下：

  • pm.Pandas，用于 Pandas Dataframes (*)
  • pm.Spark，用于 Spark Dataframes (*)
  • pm.Parameter，用于常量单值参数
  • pm.FileSystem，用于 Foundry Dataset 文件系统访问
  • pm.MediaReference，用于媒体引用
  • pm.Object，用于本体对象
  • pm.ObjectSet，用于本体对象集

(*) 在将模型发布为函数时，请查阅下文描述的关于使用这些类型的要求。

指定表格列

对于 Pandas 或 Spark 的输入和输出，列可以通过指定列名的 strings 列表来定义，或者采用包含两个对象的 tuples 列表，格式为 (<name>, <type>)。其中 <name> 是代表列名的字符串，<type> 是代表该列数据类型的 Python 类型。如果列定义仅提供字符串，其类型将默认为 Any。

列类型

表格列支持以下数据类型：

  • str
  • int
  • float
  • bool
  • list (*)
  • dict(*)
  • datetime.date
  • datetime.time
  • datetime.datetime
  • typing.Any (*)
  • MediaReference [Beta]

与实时推理不同，列类型在批处理推理中通常不作强制校验，其主要作用是为该模型适配器的使用者提供文档说明。有关 API 强制校验规范的更多详情，请参阅模型适配器参考文档。

(*) 在将模型发布为函数时，请查阅下文描述的关于使用这些类型的要求。

参数类型

对于 Parameter 输入和输出，支持以下数据类型：

  • str
  • int
  • float
  • bool
  • list(*)
  • dict(*)
  • typing.Any(*)
  • pm.NDArray(shape: list[int], dtype: numpy.typing.DTypeLike)

参数类型会进行强制校验。任何输入到 model.transform() 的参数如果不匹配指定类型，都将引发运行时错误。

(*) 在将模型发布为函数时，请查阅下文描述的关于使用这些类型的要求。

NDArray 类型

模型 API 通过 pm.NDArray(shape, dtype) 参数类型定义支持 NumPy ndarray ↗ (N 维数组) 类型。必须为 ndarray 输入同时指定 shape 和 dtype。

支持的 dtype

支持以下 NumPy dtype：

  • np.str_
  • np.bool_
  • np.int8
  • np.int16
  • np.int32
  • np.int64
  • np.float16
  • np.float32
  • np.float64

形状规范

形状通过 list[int] 指定，其中每个元素代表一个维度。可以在任意维度中使用 -1，表示在该特定维度上忽略形状检查。

API 定义要求


目标批量部署的要求


在 Modeling Objectives 应用程序中直接设置批量部署和自动模型评估，仅适用于具有单个表格数据集输入的模型。如果您的模型适配器需要多个输入，可以在Python 转换中设置批量推理。


目标评估的要求


Modeling Objectives 中的评估需要一个表格输入和一个表格输出。两者都应包含标签列，以便能够通过将预测结果与评估集中的真实值进行比较来计算指标。必须牢记以下规则，以确保您的模型适配器与评估兼容：


    

  1. 您的 API 必须仅包含一个表格输入和输出。
    2. 

  2. 任何标签或真实值列都不应包含在输入或输出表的列列表中。

      

    • 由于在实时推理期间会强制执行声明的列类型，因此在不存在标签列的情况下，这将阻碍模型用于实时推理。
      • 

    

    3. 

  3. 

predict 方法不应丢弃标签或真实值列。

      

    • 如果丢弃了这些列，基于 predict 方法的推理输出运行的评估逻辑将无法获取用于比对的标签。
      • 

    

    4. 



以回归任务的适配器为例：



```python
import palantir_models as pm
from palantir_models.serializers import DillSerializer

class LinearRegressionModelAdapter(pm.ModelAdapter):

    @pm.auto_serialize(
        model=DillSerializer()
    )
    def __init__(self, model):
        self.model = model

    @classmethod
    def api(cls):
        input_columns = [
            ('median_income', float),
            ('housing_median_age', float),
            ('total_rooms', float),

            # 不要在 API 中显式包含目标变量，
            # 因为如果找不到标签，适配器逻辑在实时推理场景下会抛出错误。
            # 当模型应用于训练集或评估集以外的任何数据时，必然会出现这种情况。

            # ('house_price', float) # 不应包含此项
        ]
        output_columns = columns + [('predicted_house_price', float)]
        return {'df_in': pm.Pandas(input_columns)}, \
               {'df_out': pm.Pandas(output_columns)}

    def predict(self, df_in):
        cols_to_keep = ['median_income', 'housing_median_age', 'total_rooms']
        # 如果不这样做，下一行代码
        # 会隐式丢弃 'house_price' 列，
        # 该列代表“标签”或目标变量，
        # 这将导致此适配器无法用于
        # 建模目标中的评估功能。
        if 'house_price' in df_in.columns:
            cols_to_keep.append('house_price')
        df_in['prediction'] = self.model.predict(
            df_in[cols_to_keep]
        )
        return df_in
```


直接函数发布及函数内调用模型的要求


若要更改已发布函数的模型 API，需同步更新该函数的调用方，具体请参阅《模型函数指南》。


要使模型支持发布为函数，需满足以下条件：


    

  • 模型 API 的输入只能包含参数、表格、对象 或 ObjectSet 类型。
    • 

  • API 中不得包含 Typing.Any 类型。
    • 

  • 对于表格输入，API 必须明确指定所需的输入或输出列。严禁仅使用 pm.Pandas() 或 pm.Spark()，否则其会被隐式解析为 TypeScript 的 Array<any>。
    • 

  • 任何集合类型（如 list 或 dict）必须指明元素类型（例如使用 list[str] 或 dict[str, str]）。若未指定，元素类型默认按 any 处理。
