模型适配器 API

模型适配器的 api() 方法指定了执行此模型适配器推理逻辑所需的输入和输出。输入和输出是分别指定的。

在运行时，模型适配器的 predict() 方法将使用指定的输入进行调用。

示例 api() 实现

以下示例显示了一个 API，该 API 指定一个输入，名为 input_dataframe，以及一个输出，名为 output_dataframe。输入和输出对象均被指定为 Pandas 数据框，其中输入数据框有一个名为 input_feature 的 float 类型列，输出数据框有两个列：(1) 一个名为 input_feature 的 float 类型列，(2) 一个名为 output_feature 的 int 类型列。

Copied!
1import palantir_models as pm
2
3
4class ExampleModelAdapter(pm.ModelAdapter):
5    ...
6
7    @classmethod
8    def api(cls):
9        inputs = {
10            "input_dataframe": pm.Pandas(columns=[("input_feature", float)])
11        }
12        outputs = {
13            "output_dataframe": pm.Pandas(columns=[("input_feature", float), ("prediction", float)])
14        }
15        return inputs, outputs
16
17    ...

API 定义也可以扩展以支持任意类型的多个输入或输出：

API 类型

模型适配器 API 的输入和输出类型可以使用以下类进行指定，详细定义如下：

  • pm.Pandas, 用于 Pandas 数据框（*）
  • pm.Spark, 用于 Spark 数据框（*）
  • pm.Parameter, 用于常量、单值参数
  • pm.FileSystem, 用于 Foundry 数据集文件系统访问
  • pm.MediaReference, 用于媒体引用
  • pm.Object, 用于本体论对象（Ontology Object）
  • pm.ObjectSet, 用于本体论对象集合（Ontology Object Set）

指定表格列

对于 Pandas 或 Spark 的输入和输出，列可以指定为一个 strings 列表，指定列名，或者是一个由两个对象组成的 tuples 列表，格式为 (<name>, <type>)，其中 <name> 是表示列名的字符串，<type> 是表示列中数据类型的 Python 类型。如果为列定义提供字符串，则其类型将默认为 Any。

列类型

以下类型支持表格列：

  • str
  • int
  • float
  • bool
  • list (*)
  • dict(*)
  • datetime.date
  • datetime.time
  • datetime.datetime
  • typing.Any (*)
  • MediaReference [Beta]

列类型通常在批量推理中 不 被强制执行，与实时推理不同，它们主要是作为此模型适配器的消费者文档。有关 API 强制执行规范的更多细节，请参考模型适配器 参考。

(*) 请查看以下 使用这些类型时的要求，当 将模型作为函数发布 时。

参数类型

对于 Parameter 的输入和输出，支持以下类型：

  • str
  • int
  • float
  • bool
  • list(*)
  • dict(*)
  • typing.Any(*)
  • pm.NDArray（形状：list[int]，数据类型：numpy.typing.DTypeLike）

参数类型是强制的，任何输入到 model.transform() 中与指定类型不匹配的参数都会抛出运行时错误。

(*) 请查看以下 使用这些类型时的要求，当 将模型作为函数发布 时。

NDArray 类型

NumPy ndarray ↗ (N-dimensional array) 类型通过 pm.NDArray(shape, dtype) 参数类型定义在 Model API 中得到支持。您必须为 ndarray 输入指定 shape 和 dtype。

Supported dtypes

以下 NumPy dtypes 是被支持的：

  • np.str_
  • np.bool_
  • np.int8
  • np.int16
  • np.int32
  • np.int64
  • np.float16
  • np.float32
  • np.float64

Shape specification

shape 是通过一个 list[int] 来指定，其中每个元素代表一个维度。您可以使用 -1 在任何维度中表示应忽略该特定维度的形状检查。

Example usage

Copied!
1import numpy as np
2import palantir_models as pm
3
4
5class NumpyModelAdapter(pm.ModelAdapter):
6    ...
7
8    @classmethod
9    def api(cls):
10        inputs = {
11            "ndarray_in": pm.Parameter(
12                type=pm.NDArray([1, 1, 2], np.int8),
13                default=np.array([[[1, -5]]], dtype=np.int8)
14            )
15        }
16        outputs = {
17            "ndarray_out": pm.Parameter(type=pm.NDArray([1, 1, 2], np.int8))
18        }
19        return inputs, outputs
20
21    def predict(self, ndarray_in):
22        # Call your model on the ndarray
23        return self.model.process(ndarray_in)

在函数中使用带有NDArray输入的模型时，您必须始终将输入作为JSON兼容的嵌套列表提供。请注意，如果输出是np.ndarray，输出也将作为嵌套列表返回：

此示例使用Pandas数据框作为输入和输出，以及一个参数。

  • 对于表格输出，API中指定的列列表只能包含所需的列。在运行时，未在API中指定但传递到数据框的列将在预测方法的输入中可用，并可以作为输出返回。在上面的示例中，传递具有列 input_feature 和 extra_column 的数据框将返回具有列 input_feature、extra_column 和 prediction 的数据框。
  • 一些模型要求仅传递训练期间使用的列到其预测方法。因此，我们建议仅提取特征列以传递给模型。
  • 一些模型需要保留列的顺序。当通过REST API请求以JSON对象传递输入时，列的顺序不一定被保留。因此，我们建议在将它们传递给模型的推理方法之前重新排序列。

API定义要求

目标批量部署要求

在Modeling Objectives应用程序中直接设置 批量部署 和 自动模型评估 仅与具有单个表格数据集输入的模型兼容。如果您的模型适配器需要多个输入，您可以在 Python转换中设置批量推理。

目标评估要求

建模目标中的评估需要一个表格输入和一个表格输出。两者都应包含标签列，以便根据评估集中的预测与真实值进行比较来计算指标。为了确保您的模型适配器与评估兼容，必须记住以下规则：

  1. 您的API只能包含一个表格输入和一个表格输出。
  2. 任何标签或真实值列都不应包含在输入或输出表的列列表中。
    • 由于声明的列类型在 实时推理 期间被强制执行，这将防止在标签列不存在的情况下使用模型进行实时推理。
  3. predict 方法不应删除标签或真实值列。
    • 如果这些列被删除，评估逻辑（在 predict 方法的推理输出上运行）将无法获得标签用于比较。

例如，考虑以下回归任务的适配器：

Copied!
1import palantir_models as pm
2from palantir_models.serializers import DillSerializer
3
4class LinearRegressionModelAdapter(pm.ModelAdapter):
5
6    @pm.auto_serialize(
7        model=DillSerializer()
8    )
9    def __init__(self, model):
10        self.model = model
11
12    @classmethod
13    def api(cls):
14        input_columns = [
15            ('median_income', float),
16            ('housing_median_age', float),
17            ('total_rooms', float),
18
19            # Do not include the target variable explicitly in the API
20            # since the adapter logic will throw an error in the live
21            # inference case if the label is not found.
22            # That will necessarily happen when the model is applied
23            # to anything other than training or evaluation sets.
24
25            # ('house_price', float) # should not be included
26        ]
27        output_columns = columns + [('predicted_house_price', float)]
28        return {'df_in': pm.Pandas(input_columns)}, \
29               {'df_out': pm.Pandas(output_columns)}
30
31    def predict(self, df_in):
32        cols_to_keep = ['median_income', 'housing_median_age', 'total_rooms']
33        # If we didn't do this, the next line
34        # would implicitly drop the 'house_price' column,
35        # which represents the 'label' or target variable,
36        # making this adapter unsuitable for use with Evaluation
37        # in Modeling Objectives.
38        if 'house_price' in df_in.columns:
39            cols_to_keep.append('house_price')
40        df_in['prediction'] = self.model.predict(
41            df_in[cols_to_keep]
42        )
43        return df_in

直接函数发布和函数中模型使用的条件

更改已发布函数的模型API将需要更新该函数的使用者，如 模型函数指南 所述。

为了让模型 支持作为函数发布，以下条件必须满足：

  • 模型API只能包含 参数（Parameter）、表格（Tabular）、对象（Object） 或 对象集（ObjectSet） 输入。
  • API不能包含 Typing.Any 类型。
  • 对于表格输入，API必须指定所需的输入或输出列。仅使用 pm.Pandas() 或 pm.Spark() 是不允许的，因为这会被隐式解释为 TypeScript Array<any>。
  • 任何集合类型，如 list 或 dict，必须指定元素类型（例如，使用 list[str] 或 dict[str, str]）。否则，元素类型将被解释为 any 类型。
