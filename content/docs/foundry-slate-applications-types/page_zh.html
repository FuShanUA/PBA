应用类型

Slate 支持两种类型的应用：集成应用和公开应用。

  • 集成应用
  • 公开应用
    • 限制
    • 权限

集成应用

在大多数情况下，您会希望创建集成应用。集成 Slate 应用可以利用 Foundry 生态系统来创建微件、使用函数、配置事件和动作等。集成应用会发布给组织内的 Foundry 用户，并可根据用户权限进行查看或编辑。

详细了解如何创建集成应用。

公开应用

Slate 支持“公开”应用，即没有 Foundry 账户的个人也可以使用的 Slate 应用。公开应用允许没有 Foundry 账户的用户提交信息、上传数据或上传文件到 Palantir Foundry 中，但需遵循验证逻辑和其他安全措施。

来自公开应用上传的数据可以直接流入数据管道，无需额外建立与外部系统的数据连接。开发者可以构建并发布公开应用，为没有 Foundry 账户的合作伙伴、供应商或员工提供高度定制化的工作流。

借助 Slate，您无需配置服务器、DNS 或身份验证即可开发公开应用，同时还能充分利用 Slate 提供的微件、自定义 CSS 覆盖、函数和事件等工具集。

详细了解如何创建公开应用。

限制

公开 Slate 应用无法读取应用本身以外的数据和资源。该应用无法访问对象、数据集、动作或文件（如存储的图像或视频）。因此，需要访问平台其他元素的微件和组件在公开 Slate 应用中不可用。Slate 应用所需的所有数据和资源都必须存储在 Slate 应用本身中。图像、字体和其他资源可以通过 base64 编码存储在 Slate 应用中。

示例 1：通过 base64 编码的图像可以使用 HTML 中的 image 标签添加。source 标签不再是 URL，而是需要配置为显示编码后的图像，如下所示：

<image src="data:image/png;base64,<<base64_code>>">

示例 2：base64 编码的字体可以添加到 Slate 的“样式”面板中。虽然网络字体可以直接使用，但自定义字体需要在应用内声明，如下所示。添加字体后，即可通过字体族名称在应用的任何位置使用它。

@font-face {
    font-family: '<<font_name>>';
    src: url(data:application/font-woff2;charset=utf-8;base64,<<base64_code>>) format('woff2'),
    }

权限

除了现有的文件和项目权限外，公开 Slate 应用还需要一组额外的权限才能创建和编辑文件。要创建或编辑公开应用，用户需要访问管理公开 Slate 应用工作流，该权限可通过控制面板中安全与治理部分下的组织权限设置进行授予。

发布后，非 Foundry 用户可以通过公开链接以只读方式查看已发布的 Slate 应用。

该应用与 Foundry 本身托管在同一网络中，因此用户需要具备对 Foundry 实例的访问权限才能查看公开应用。只能通过受限网络访问的 Foundry 实例无法将应用发布到网络上。

由于公开应用默认不会被搜索引擎索引，因此需要向非 Foundry 用户提供链接才能访问该应用。
