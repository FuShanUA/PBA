评估函数和本体论编辑

当运行评估套件时，每个测试用例的逻辑函数会被执行。对于涉及本体论编辑（如创建、编辑或删除对象）的函数，每个测试用例会在一个本体论模拟中执行。这确保了在测试和评估期间实际的本体论不会被更改。

用于本体论编辑的自定义评估函数

对于导致本体论编辑的AIP逻辑函数，用户必须使用TypeScript配置自定义评估函数或使用中间参数。自定义评估函数必须返回布尔类型或数值类型。一个评估函数也可以通过返回一个struct来返回多个指标，该struct由布尔类型或数值类型组成。

在评估应用程序中添加评估函数时，您将被提示在代码仓库中编写一个函数或选择一个已发布的函数。通过下面的指南，探索各种本体论（Ontology）编辑，并学习如何有效地使用TypeScript函数进行评估。

创建的对象

在涉及创建对象的本体论（Ontology）编辑情况下，创建的对象只存在于模拟的本体论（Ontology）中。因此，这些创建的对象不能直接配置为测试用例参数。相反，使用可识别的属性搜索它并检查其属性。

14    }

16    return matches[0].classification === expectedClassification;

编辑的对象

对于对象编辑输出类型，编辑后的对象已经存在于实际的本体论（Ontology）中。在模拟的本体论（Ontology）中，您可以直接将其传入函数并检查其属性，如下所示：

1@Function()
2public checkTicketClassification(
3    ticket: SupportTicket,

7}

已删除的对象

在涉及删除对象的本体论（Ontology）编辑中，该对象应在模拟的本体论（Ontology）中被删除，因此不能传递到评估函数中。为了验证该对象确实已被删除，请传递该对象的一个可识别属性，并进行搜索以确保没有结果。

1@Function()
2public async checkTicketWasDeleted(ticketId: string): Promise<boolean> {
3    const count = await Objects.search().supportTicket()
4        .filter(ticket => ticket.ticketId.exactMatch(ticketId))
5        .count();
6    return count === 0;
