时间

时间部件类别包含以下部件：

  • 日历
  • 日期选择器
  • 日期范围选择器
  • 时间线

日历部件

下表提供了日历部件可用属性的使用详情。

属性

属性描述类型必填修改方
areEventsAllDay布尔值数组，用于确定对应的事件是否为全天事件。全天事件将显示在视图的“全天”区域，并且只会在完全包含于开始和结束时间内的日期上显示。此外，选中全天事件时，其开始和结束时间将反映调整后的时间边界。例如：从“1月1日 03:00”到“1月3日 03:00”的全天事件将渲染为“1月1日 00:00”到“1月3日 00:00”。如果留空，事件默认为全天 = false。json否直接编辑
dateRangeEnabled将日期范围添加为对应当前视图的日历标题。根据视图的不同，这将是当前显示的日、周或月。string是直接编辑
defaultDate页面加载时日历默认的日期。如果设置为空字符串，视图将默认为“今天”，从而使范围与页面加载时的当前日期相匹配。string是用户交互
eventEnds每个事件的结束时间。json是直接编辑
eventNames每个事件上显示的名称。json是直接编辑
eventSeries标签（字符串）数组，用于将事件分组为系列以便设置样式。如果留空，事件将使用默认样式显示。json否直接编辑
eventStarts每个事件的开始时间。json是直接编辑
pagingEnabled添加按钮，允许按当前显示的间隔向前或向后移动，以及返回当前日期的按钮。boolean是直接编辑
pagingYearEnabled添加额外的按钮，允许将当前查看的时间范围向前或向后移动整整一年。boolean是直接编辑
selectedIndex日历上所选事件的索引。如果未选择任何事件，则为“null”。number是用户交互
viewDefault应用程序加载时将渲染的视图类型（日、周、月）。此视图中显示的日期由“defaultDate”的值确定。默认为“月”视图。string是用户交互
viewType控制启用“日”和“周”按钮时可访问的视图。当类型设置为“List”时，事件将按提供给组件的顺序显示为项目列表，而不是按其开始时间的顺序。当类型设置为“Time”时，事件将显示为跨越事件发生时间的块。显示类型对“月”视图没有影响。string是直接编辑
viewSwitchingEnabled启用/禁用日/周/月视图之间切换的按钮。默认情况下，这些按钮处于启用状态。boolean是直接编辑

日期选择器组件

下表详细说明了日期选择器组件可用属性的具体用法。表格之后附有若干示例。

属性

属性描述类型必填修改方式
disabled指定用户能否与该组件交互。默认值为 false。取值通常为返回布尔值的模板表达式（例如 “{{w1.on}}”）。boolean是直接编辑
displayFormat在选择器输入框中显示所选内容时使用的格式。必须是有效的 moment.js 格式 ↗。string否直接编辑
maxDate可选的最大日期，以字符串形式表示。若留空，则使用默认最大日期（本年 12 月 31 日）。日期必须采用 ISO 8601 标准 ↗ 格式化。string否直接编辑
minDate可选的最小日期，以字符串形式表示。若留空，则使用默认最小日期（20 年前的 1 月 1 日）。日期必须采用 ISO 8601 标准 ↗ 格式化。string否直接编辑
placeholder未选择日期时用作占位符的文本。string是直接编辑
timeEnabled指定是否在日期选择器中显示时间选择器。boolean是直接编辑
timePickerPrecision确定时间选择器的精度。number是直接编辑
value当前选择的内容，以字符串形式表示。string是用户交互
valueFormat“value” 字段的输出字符串格式。必须是有效的 moment.js 格式 ↗。若未提供或设为 null，将使用 ISO 8601 格式。string否直接编辑

示例

日期选择器



已复制！
1{
2  "disabled": false,
3  "displayFormat": "MM/DD/YYYY",
4  "timeEnabled": false,
5  "value": "2014-09-10T05:00:00.000Z"
6}

启用时间功能的日期选择器



已复制！
1{
2  "disabled": false,
3  "timeEnabled": true,
4  "value": "2014-09-10 13:00",
5  "valueFormat": "YYYY-MM-DD HH:mm"
6}

默认值


```json
{
  "disabled": false,
  "timeEnabled": false,
  "value": "",
  "valueFormat": "YYYY-MM-DD"
}
```

日期范围选择器组件


下表提供了日期范围选择器组件可用属性的使用详情。表格后附有若干示例。




属性


























































































属性描述类型必填修改方式
disabled指定用户能否与该组件交互。默认为 false。其值通常为返回布尔值的模板表达式（例如 “{{w1.on}}”）。boolean是直接编辑
displayFormat日期显示格式。必须为合法的 moment.js 格式 ↗。string否直接编辑
fromDateValue当前“开始”日期的选择值，以字符串形式表示。日期必须采用 ISO 8601 标准 ↗进行格式化。string是用户交互
maxDate用户在日期选择器中可选择的最大日期，以字符串形式表示。如留空，则使用默认最大日期（本年 12 月 31 日）。日期必须采用 ISO 8601 标准 ↗进行格式化。string否直接编辑
minDate用户在日期选择器中可选择的最小日期，以字符串形式表示。如留空，则使用默认最小日期（过去 20 年中的 1 月 1 日）。日期必须采用 ISO 8601 标准 ↗进行格式化。string否直接编辑
shortcutLabels显示快捷方式时所展示的快捷范围名称，例如“最近 3 天”或“最近 3 个月”。若未提供标签，则使用默认的快捷范围与标签。string[]是直接编辑
shortcutRanges各快捷方式对应的日期范围。每个范围必须采用 ISO 8601 标准 ↗格式化：[开始日期字符串, 结束日期字符串]，例如 [‘2016-05-01’, ‘2016-05-10’] 或 [‘2016-05-01 16:00’, ‘2016-05-01 18:00’]string[][]否直接编辑
shortcutsEnabled决定是否显示日期范围快捷方式（例如最近 3 天、最近 3 个月等）。boolean否直接编辑
toDateValue当前“结束”日期的选择值，以字符串形式表示。日期必须采用 ISO 8601 标准 ↗进行格式化。string是用户交互
timeEnabled指定是否在日期选择器旁显示时间选择器。boolean是直接编辑
timePickerPrecision决定时间选择器的精度。number是直接编辑

时间轴组件


以下表格提供了时间轴组件可用属性的详细用法。表格之后附有若干示例。


属性










































属性描述类型必填修改方式
cssClasses类名数组，每个类名将分别添加到时间轴上对应的事件中。string[]否直接编辑
dates日期数组，表示时间轴上要显示的事件。将按提供的顺序进行显示。any[]是直接编辑
details辅助详细信息，显示在对应事件标题的下方。string[]否直接编辑
titles时间轴上显示的事件名称/标题。string[]是直接编辑


示例


时间轴



```json
{
  "cssClasses": [
    "green",
    "red",
    "green"
  ],
  "dates": "{{query1.dates}}",
  "details": [
    "第一个事件非常精彩",
    null,
    "第三个事件还不错"
  ],
  "titles": "{{query1.titles}}"
}
```


默认值


已复制！
1{
2  "cssClasses": [],
3  "dates": [],
4  "details": [],
5  "titles": []
6}
