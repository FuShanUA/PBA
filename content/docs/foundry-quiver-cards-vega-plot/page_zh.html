Vega 图表

Quiver 允许用户使用 Vega 或 Vega-Lite 库创建完全可定制且可交互的可视化效果。查阅《Vega ↗》和《Vega-Lite ↗》文档以获取灵感和示例。

什么是 Vega？

Vega 和 Vega-Lite 允许你以简洁的 JSON 规范形式创建、保存和共享交互式可视化设计，该规范描述了可视化的外观和行为。Vega 支持多种可视化设计，例如：

  • 《数据加载 ↗》与《转换 ↗》
  • 《比例尺 ↗》
  • 《地图投影 ↗》
  • 《坐标轴 ↗》
  • 《图例 ↗》
  • 《图形标记 ↗》，如矩形、线条、绘图符号等。

Vega-Lite 是在 Vega 基础上构建的一种高级语言，为编写常见可视化提供了更简洁、便捷的方式。我们建议从 Vega-Lite 入手，因为相比 Vega，Vega-Lite 语法更易于阅读、编写和调试。若 Vega-Lite 的选项无法满足需求，请考虑改用 Vega。

配置 Vega 图表

要配置 Vega 图表，请将鼠标悬停在目标卡片上，选择可视化，并在下拉菜单中导航至（或搜索）Vega 图表。选择Vega 图表以打开配置菜单。

在编辑器菜单中，选择配置 Vega 图表或 Vega-Lite 图表。如下图所示，选择模板下拉菜单提供了常见可视化的模板，包括气泡图、箱线图、旭日图等。

选择 Vega 绘图模板后，系统会提示你通过映射到转换表数据源的列，来填写所选绘图的相关参数。

填写完模板参数后，选择 Apply 以生成你的 Vega 绘图。请注意：

  • 对于留空的字段，将使用参数名作为默认值。
  • 已填写的模板会覆盖编辑器中的任何当前值。

配置数据输入

Vega 绘图接受以下数据输入：

  • 转换表：任何转换表都可以作为 Vega 绘图的输入。从对象集添加 Vega 绘图时，系统会自动将一个支持转换表添加到 Analysis Contents 面板中（但不会添加到画布中）。
  • 数组：数组会被解析为一个包含名为 ARRAY_VALUES 单列的转换表。

使用分析中的变量

你还可以在 Vega 图表中引用值类型（例如数值指标卡片）。例如，若要使用数值指标卡片在特定值处设置阈值线，可通过该指标卡片的全局 ID 来引用该值。

"encoding": {
    "y": { "datum": $C }
}
...

预览解析后的 Vega 规范

通过查看解析后的 Vega 规范，你可以排查 Vega 图表配置中的问题。选择 Preview 即可查看解析后的 Vega。

请注意，解析后的 Vega 规范加载速度可能较慢，因为其中会包含所引用数据输入中的所有数据点。

在其他应用程序中嵌入 Vega 图表

Vega 编辑器包含两个实用设置，可用于通过 Quiver 仪表板在 Workshop 中嵌入 Vega 图表：

  • 自动缩放图表： 启用“自动缩放图表”后，你的 Vega 图表将根据提供的数据自动调整以适应卡片尺寸。
  • 默认样式： 向 Vega 规范中注入一些自动尺寸调整和样式配置，使图表外观与 Quiver 卡片样式保持一致。若禁用此选项，Vega 可视化效果将与 Vega 公共网站上的显示效果一致。

使用 AIP 配置 Vega 图表

AIP 能够根据自然语言描述的预期图表或图表修改需求，生成 Vega 图表配置。你只需描述想要创建或修改的图表，AIP 便会根据提示给出建议，从而简化容易出错的 Vega 图表 JSON 配置过程。

描述预期的 Vega 图表

要使用 AIP，请在 Vega 绘图卡片的右上角选择 AIP Configure。然后，输入提示词，选择 Configure，并查看 AIP 生成的建议。要采纳建议，请选择 Apply。若希望 AIP 提供替代建议，请编辑提示词，然后选择 Reconfigure。

在以下示例中，用户向 AIP 输入了提示词：“在散点图上展示 ph 与咖啡因的关系，并在 x 轴 13 和 y 轴 6 的位置绘制一条红线”。AIP 根据该提示词，建议了一个绘图配置，其中包含了关注的两项数值属性（咖啡因和 pH 值）。

要采纳 AIP 提议的更新，请选择 Apply。Vega 绘图将基于更新后的 Vega 规范渲染可视化图表。

在现有绘图基础上进行构建

AIP 能够识别并区分提示词中包含的抽象概念，并利用这些信息帮助生成或修改 Vega 绘图。例如，AIP 可以引用先前生成的绘图特征，让你能够“在现有绘图基础上进行构建”。

在下图中，用户输入了提示词：“图中的两条线将点划分为 4 个象限。为每个象限填充不同的颜色。”AIP 识别出由垂直线和水平线界定的区域即为象限，即使当前的绘图配置中并不包含任何关于象限的定义或配置。

AIP 准确地生成了符合用户提示词的预期绘图，该绘图是在初始绘图的基础上构建的。

使用 AIP Generate 创建 Vega 绘图

使用 AIP 生成的 Quiver 分析图表可以包含 Vega 图表。若要指示 AIP 生成 Vega 图表，而不是原生的 Quiver 可视化图表（例如折线图或散点图），请在提示词中明确指出 AIP 应返回“Vega 图表”。例如，准确的提示词可以是：“使用 vega 图表在散点图上显示咖啡因与 pH 值的关系。”

注意：AIP 功能的可用性可能会发生变化，且不同客户之间可能存在差异。

Vega-Lite 选择

配置 Vega-Lite 图表的选择

Vega-Lite 选择是用于构建交互式可视化的一项强大且高度可定制的功能。Vega-Lite 库内置了对选择功能的支持，并且可以将 Quiver 中的 Vega-Lite 图表配置为将选择结果作为转换表输出。用户可以利用选择数据对下游卡片进行参数化，构建下钻工作流，并继续进行分析。Vega-Lite 选择的行为与对象集图表的选择有所不同，需要进行额外的设置步骤。

Vega-Lite 允许用户通过两种类型的选择与图表进行交互：

  • 点选择：选择单个点，或按住 Shift 键选择画布上的多个点。
  • 区间选择：拖动以选择画布上有界的矩形区域。

并非所有 Vega-Lite 图表类型都支持选择。有关更多详细信息，请参阅下方的常见问题解答部分。

要将图表选择数据作为转换表输出，请执行以下步骤：

  1. 打开 Vega 图表卡片的配置面板，并滚动至选择选项。然后，开启将点选择输出为转换表和/或将区间选择输出为转换表。

  2. 在 Vega 图表的 JSON 编辑器中，向 params 字段添加选择参数。请查阅 Vega 图表文档以了解有关选择参数的更多信息。

    1. 对于点选择，参数必须命名为 quiverDefaultClick；对于区间选择，参数必须命名为 quiverDefaultBrush。

    2. 将 type 属性设置为 point（用于 quiverDefaultClick）或 interval（用于 quiverDefaultBrush）。

    3. 在 encodings 属性中指定一个或多个编码 ↗。这些是您希望选择的字段，例如 x、y 或 color。编码决定了如何选择值以及输出什么信息。

  3. 选择完成后，会出现包含选择数据的卡片页脚。选择 Output Selection 以将选择数据输出为转换表。或者，使用 Vega 图表的下一步操作菜单，选择 Convert > New Transform Table。

    • 点选择输出为编码字段和值的表格，其中每列对应一个字段，每行代表一个选定的点。



    1. 区间选择：如果字段是连续的，则输出为区间边界的范围（最小值，最大值）；如果字段是离散的，则输出为值数组。



与对象集图表不同，在选择作为下钻操作输出过滤后对象集的情况下，Vega 图表无法根据当前选择自动过滤输入数据。相反，Vega 图表将输出给定编码的选定值，这些输出可用于手动构造输入表上的过滤器。有关更多信息，请查阅构造下钻工作流部分。

来自 Vega-Lite 图表的选择数据可用于构建下钻工作流。在该工作流中，图表选择操作将作为筛选条件，用户可基于上游选择的数据子集继续分析。以下步骤描述了如何构建下钻工作流。

  1. 按照上述步骤设置选择参数，并将当前选择结果输出为转换表。



  2. 选择 Pop out column as array 将属性转换为数组。





  1. 复制原始转换表（或复制派生出转换表的根对象集），并使用所选值作为筛选参数对其进行过滤。



4. 筛选后的表格将根据上游图表的选择动态更新。



常见问题与限制

是否可以为任何类型的 Vega-Lite 图表配置选择功能？

并非所有 Vega-Lite 图表类型都支持选择功能。虽然 Vega 文档未提供受支持图表类型的完整列表，以下是一个非详尽的清单：

  • 支持选择：条形图、散点图、热力网格、折线图、饼图（仅支持点选择）、瀑布图（仅支持点选择）、地理坐标投影、GeoJSON 地图（仅支持点选择）
  • 不支持选择：箱线图、置信区间图

我可以在单个图表规范中定义多个选择参数吗？

每个图表仅限使用一个点选择参数和一个区间选择参数将数据输出到 Quiver。这是 Quiver 设置的限制；虽然 Vega 允许定义多个参数，只要它们具有唯一的名称即可，但只有命名为 quiverDefaultClick 和 quiverDefaultBrush 的参数才会作为 Quiver 转换表输出。

我无法选择轴的子集，即使将其添加到了选择参数的编码列表中也是如此。造成该问题的原因是什么？

请检查该轴上的值是否在 Vega 规范内进行了聚合。Vega 不支持选择由 Vega 聚合的字段。例如，如果 y 编码字段包含如下代码片段所示的内联聚合，用户将无法选择 y 轴的子集。

"encodings": {
  "y": {
    "field": $C.field_y,
    "aggregate": "avg"
  },
  ...
}

要启用对 y 轴的选择功能，请先在 Quiver 中聚合数据（例如使用透视表），然后再将其传入 Vega。

为什么在 spec 编译通过的情况下，我尝试进行选择时却收到 Invalid data selection 错误？

请检查以下几点：

  • 确保所有选择参数都包含非空的 encodings 字段。每个参数必须指定一个或多个编码，以便输出选择信息。
  • 如果这是区间选择，请检查选择参数中定义的编码是否包含非轴字段。Vega 不支持对非轴编码字段（如 color、shape 等）进行区间选择。从 encodings 数组中移除这些字段即可解决该错误。
