使用 Ontology Manager 为对象添加时间序列属性

本指南参考了有关在时间序列对象类型上设置时间序列属性的文档。您可以根据需要为对象类型添加任意数量的时间序列属性，前提是每个对象始终会关联一个时间序列集合。请查阅我们的文档，以了解应该选择时间序列对象类型还是传感器对象类型。

您必须对 Route 和 Airport 对象类型分别重复以下步骤。在本指南结束时，Carrier、Route 和 Airport 对象类型都将各自拥有三个时间序列属性：Daily Count of Flights、Daily Average Arrival Delay 和 Daily Average Departure Delay。

  1. 在 Ontology Manager 中导航至 Carrier 对象类型，并选择 Capabilities 选项卡。
  2. 在 Time series property 部分中，选择 + Add。

  1. 选择现有的 Daily Count of Flights 属性作为时间序列属性，然后选择 Set as default time series property，使其自动显示在 Quiver 中。

  1. 选择您在 Pipeline Builder 中创建的时间序列同步。在我们的示例中，它名为 [Example] Time Series Sync | Event Pipeline。

  1. 对 Daily Average Arrival Delay 和 Daily Average Departure Delay 时间序列属性重复此过程。

既然已经为对象类型添加了时间序列属性，我们就可以在操作上下文中使用这些时间序列属性了。请继续阅读文档，了解如何在 Workshop 和 Quiver 中使用对象的时间序列属性。
