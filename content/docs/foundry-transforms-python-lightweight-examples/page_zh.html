高级单节点计算选项

以下集成的基本原理是，您可以以多种格式访问 表格型 Foundry 数据集，例如 pandas DataFrame（数据框）, Arrow Table（表）, Polars DataFrame（数据框）, 甚至作为原始的 Parquet 或 CSV 文件。这在 转换概览 中也有说明。当将内存中的表保存到 Foundry 时，我们可以使用它们被读取时的任何格式。

与 Apache DataFusion 一起工作

有时，直接将原始的底层数据集文件传递给我们的计算引擎，而不是进行数据反序列化步骤会更容易。我们可以获取磁盘上文件的路径，这些文件通过调用 my_input.path() 按需下载。在将原始文件写回 Foundry 时，我们需要记住两个限制条件：

  • 只能通过此 API 在 Foundry 数据集中存储 Parquet 文件。
  • 文件必须放置在 my_output.path_for_write_table 值所指向的文件夹中。

当同时满足这两个条件时，我们可以调用 write_table 并传入该文件夹的路径，如下所示：

Copied!
1my_output.write_table(my_output.path_for_write_table)

要看到实际效果，请考虑以下示例，演示如何在平台上使用 DataFusion ↗。

Copied!
1import datafusion
2from datafusion import lit
3from datafusion.functions import col, starts_with
4from transforms.api import transform, Output, Input
5
6
7@transform.using(my_input=Input('my-input'), my_output=Output('my-output'))
8def my_datafusion_transform(my_input, my_output):
9    ctx = datafusion.SessionContext()
10    table = ctx.read_parquet(my_input.path())
11    my_output.write_table(
12        table
13        .filter(starts_with(col("name"), lit("John")))
14        .to_arrow_table()
15    )

使用 cuDF

您也可以通过使用 pandas.DataFrame 实现集成。以下代码片段显示了一个使用 cuDF ↗ 的实例

在轻量级转换中。这将基本上在可能的情况下在GPU上以高度并行化的方式运行pandas代码。

Copied!
1@transform.using(
2    my_input=Input('my-input'),
3    my_output=Output('my-output')
4).with_resources(
5    cpu_cores=4,
6    memory_gb=32,
7    gpu_type='NVIDIA_T4'
8)
9def my_cudf_transform(my_input, my_output):
10    import cudf  # Only import CUDF at runtime, not during CI
11    df = cudf.from_pandas(my_input.pandas()[['name']])
12    filtered_df = df[df['name'].str.startswith('John')]
13    sorted_df = filtered_df.sort_values(by='name')
14    my_output.write_table(sorted_df)

上述代码片段假定您的Foundry注册已配备NVIDIA T4 GPU，并且通过< a href="/docs/foundry/resource-management/resource-queues/#use-gpus">资源队列可供您的项目使用。

使用cuDF的GPU加速Polars

您的Foundry注册必须配备支持的NVIDIA GPU，并且必须通过< a href="/docs/foundry/resource-management/resource-queues/#use-gpus">资源队列供您的项目使用，才能使用此功能。有关可用GPU类型的列表，请参见< a href="/docs/foundry/transforms-python/configure-resources/#gpu-provisioning">GPU分配。

您可以使用< a href="https://docs.rapids.ai/api/cudf/stable/cudf_polars/">cudf-polars ↗来在GPU上加速< a href="/docs/foundry/transforms-python/polars-lazy/">Polars延迟API查询 ，用于 Polars 的 GPU 支持的执行引擎。通过将 pl.GPUEngine() 传递给 collect() 调用，您可以仅进行少量代码更改即可在 GPU 上运行现有的 Polars 懒惰 API 查询。有关支持的操作和性能特征的详细信息，请参阅 Polars GPU 支持文档 ↗。

目前GPU加速的集合仅支持 collect()，这会将结果完全材料化到内存中。流式操作如 sink_parquet() 尚未被支持。请确保您的输出数据适合可用的GPU和系统内存。

使用 Polars GPU 引擎编写

使用标准的 Polars 延迟 API 构建你的查询，然后将 engine="gpu" 传递给 write_table()，以使用 GPU 收集并写入结果：

Copied!
1import polars as pl
2from transforms.api import transform, Output, Input
3
4@transform.using(
5    output=Output("/Users/jsmith/output"),
6    my_input=Input("/Users/jsmith/input"),
7).with_resources(
8    gpu_type="NVIDIA_T4",
9    memory_gb=32,
10)
11def compute(output, my_input):
12    lf = my_input.polars(lazy=True)
13
14    result = (
15        lf.with_columns(
16            (pl.col("a") * pl.col("b")).alias("product"),
17            (pl.col("a").pow(2) + pl.col("b").pow(2)).sqrt().alias("norm"),
18        )
19        .group_by("category")
20        .agg(
21            pl.col("product").sum().alias("total_product"),
22            pl.col("norm").mean().alias("mean_norm"),
23            pl.len().alias("n"),
24        )
25        .sort("total_product", descending=True)
26    )
27
28    output.write_table(result, engine="gpu")

GPU 引擎仅支持 Polars 操作的一个子集。有关完整列表，请参阅 Polars GPU 支持文档 ↗。默认情况下，不支持的查询会静默回退到 CPU 引擎。要改报错，请使用 raise_on_fail=True 手动收集，并将生成的 DataFrame 传递给 write_table()：

Copied!
1df = lf.collect(engine=pl.GPUEngine(raise_on_fail=True))
2output.write_table(df)
