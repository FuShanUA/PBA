高级单节点计算选项

以下集成的基本原理是，你可以通过多种格式访问 Foundry 表格数据集，例如 pandas DataFrame、Arrow Table、Polars DataFrame，甚至可以直接作为原始 Parquet 或 CSV 文件访问。转换概述中也展示了这一点。在将内存中的表格保存回 Foundry 时，我们可以使用读取时支持的任何格式进行传递。

使用 Apache DataFusion

有时，省去数据反序列化步骤，直接将底层的原始数据集文件传递给计算引擎会更加简便。我们可以通过调用 my_input.path() 来获取文件在磁盘上的路径，系统会按需下载这些文件。在将原始文件写回 Foundry 时，需要注意以下两个限制：

  • 通过此 API 只能将 Parquet 文件存储在 Foundry 数据集中。
  • 文件必须放置在 my_output.path_for_write_table 值所对应的文件夹中。

当满足这两个条件时，我们就可以使用该文件夹路径调用 write_table，如下所示：

Copied!
1my_output.write_table(my_output.path_for_write_table)

要查看其实际效果，请参考以下示例，该示例演示了如何在平台中使用 DataFusion ↗。


```python
import datafusion
from datafusion import lit
from datafusion.functions import col, starts_with
from transforms.api import transform, Output, Input


@transform.using(my_input=Input('my-input'), my_output=Output('my-output'))
def my_datafusion_transform(my_input, my_output):
    ctx = datafusion.SessionContext()
    table = ctx.read_parquet(my_input.path())
    my_output.write_table(
        table
        .filter(starts_with(col("name"), lit("John")))
        .to_arrow_table()
    )
```

使用 cuDF 实现 GPU 加速的 Polars


要使用此功能，您的 Foundry 环境必须配备受支持的 NVIDIA GPU，并且可以通过资源队列将其提供给您的项目使用。有关可用 GPU 类型的列表，请参阅GPU 预配。


您可以使用 cudf-polars ↗（Polars 的 GPU 支持执行引擎）在 GPU 上加速 Polars 惰性 API 查询。这样，您只需将 pl.GPUEngine() 传递给 collect() 调用，即可通过极少的代码改动，在 GPU 上运行现有的 Polars 惰性 API 查询。有关支持的操作和性能特征的详细信息，请参阅 《Polars GPU 支持文档》 ↗。


目前，GPU 加速的收集操作仅支持 collect()，该操作会将结果完全物化到内存中。尚不支持诸如 sink_parquet() 之类的流式操作。请确保您的输出数据能够容纳在可用的 GPU 和系统内存中。

使用 Polars GPU 引擎进行写入


使用标准的 Polars 惰性 API 构建查询，然后向 write_table() 传入 engine="gpu"，即可使用 GPU 收集并写入结果：



```python
import polars as pl
from transforms.api import transform, Output, Input

@transform.using(
    output=Output("/Users/jsmith/output"),
    my_input=Input("/Users/jsmith/input"),
).with_resources(
    gpu_type="NVIDIA_T4",
    memory_gb=32,
)
def compute(output, my_input):
    lf = my_input.polars(lazy=True)

    result = (
        lf.with_columns(
            (pl.col("a") * pl.col("b")).alias("product"),
            (pl.col("a").pow(2) + pl.col("b").pow(2)).sqrt().alias("norm"),
        )
        .group_by("category")
        .agg(
            pl.col("product").sum().alias("total_product"),
            pl.col("norm").mean().alias("mean_norm"),
            pl.len().alias("n"),
        )
        .sort("total_product", descending=True)
    )

    output.write_table(result, engine="gpu")
```


GPU 引擎仅支持 Polars 操作的子集。完整列表请参阅《Polars GPU 支持文档 ↗》。默认情况下，不支持的查询会静默回退到 CPU 引擎。若希望改为抛出错误，请使用 raise_on_fail=True 手动执行 collect 操作，并将所得 DataFrame 传递给 write_table()：
