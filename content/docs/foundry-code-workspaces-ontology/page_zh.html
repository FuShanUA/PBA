与本体论交互

受限视图支持的本体论实体和查询函数目前在代码工作区中尚未得到支持。

代码工作区支持使用JupyterLab®和RStudio®中的对象、关联和操作类型与本体论进行交互。

创建一个新的本体论SDK

要在代码工作区中创建一个新的本体论SDK，首先导航到本体论侧边栏。每个代码工作区可以有一个SDK，并且可以创建每个SDK的多个版本。从本体论侧边栏中选择创建新的SDK以打开SDK配置表单。

新的本体论 SDK 需要一个包名称和一个相关的本体论。包名称用于确定您的 SDK 在代码中如何被访问：包名称为 example 的 SDK 版本将在库名称 example_sdk 下可导入。SDK 包名称只能包含字母、数字和连字符，并且不能以连字符结尾。一旦发布，SDK 包名称就不能更改。

配置完您的 SDK 后，选择 保存选择 以继续。

选择数据实体

本体论 侧边栏包含两个选项卡：数据实体 和 SDK 生成。

  • 数据实体 选项卡用于从您的本体论中导入对象和操作类型。
  • SDK 生成 标签用于检查和安装SDK版本。

要选择数据实体，请导航到数据实体标签。选择添加，然后选择所需的数据实体类型以打开资源选择器对话框，该对话框允许浏览并选择数据实体。选择数据实体后，可以使用侧边栏底部的保存按钮保存您的选择。选择保存后，将生成一个新的SDK版本，并可访问所选的数据实体。

当选择影响对象类型的行动类型或具有对象类型的参数时，这些对象类型也将自动添加到选择中。如果任何所选对象类型有尚未导入到您的代码工作区项目范围内的支撑数据源，您还将被提示确认这些数据源的自动导入。

更改所选本体论

要从其他本体论导入本体论实体，请使用 数据实体 选项卡中的 本体论选择器。选择不同的本体论后，选择 保存 以应用更改并生成新版本。更改所选的本体论将重置 SDK 的数据作用域，清除所有选定的对象、操作或关联类型。

发布新的本体论 SDK 版本

配置完成后，可以通过 本体论 侧边栏底部的 生成新版本 按钮发布新的本体论 SDK 版本。已发布的和待发布的版本可以通过 SDK 生成 选项卡进行检查。

选择 生成新版本 后，新创建的版本将在 SDK生成选项卡 中显示。SDK版本在状态显示为“已发布”之前不可用。

发布失败的常见原因包括：

  • 使用了未导入依赖对象类型的动作类型。
  • 使用了没有支撑数据源的对象类型，而这些数据源不在项目范围内。

在代码中使用本体论 SDK 版本

成功发布的版本可以从您的编辑器中导入并在代码中使用。首先，点击 本体论 的 SDK生成 选项卡中您要使用的版本的卡片。 侧边栏。这将展开一个卡片，其中包含针对您编辑器语言的多个代码片段。点击某个片段会将其添加到剪贴板中，以便可以将其粘贴到您的编辑器中。

可见的代码片段会根据您编辑器的语言而有所不同。所有支持的语言都有“安装包”片段，用于安装 SDK 的版本，以及“初始化客户端”片段，用于导入 SDK 并初始化一个 Foundry 客户端以与本体论进行交互。

基本使用示例

任何导入的对象或链接类型都可以在 数据实体 选项卡中选择，以查看该特定资源的使用示例。以下示例是更长、更通用的示例，展示了在代码工作区中端到端的OSDK使用。

Python OSDK 使用示例

以下示例演示了如何使用 Python 与 OSDK 进行交互。此示例假定您 SDK 最近成功发布的版本包含对象类型 "Aircraft（飞机）" 和操作类型 "ActionMode（操作模式）"。此示例中的 my_sdk_package 应替换为您的 SDK 的包名称。

Copied!
1AircraftObject = client.ontology.objects.Aircraft
2
3# Retrieve an object and view properties
4my_aircraft = AircraftObject.get(1)
5my_aircraft.date_of_manufacture
6
7# Iterate over objects
8for aircraft in AircraftObject.take(2):
9    print(aircraft.current_location)
10
11# Aggregations
12AircraftObject.aggregate({"min_id": Aircraft.id.min(),
13                          "max_id": Aircraft.id.max(),
14                          "aircraft_count": Aircraft.count()}).compute()
15
16# Filter
17my_a330s = AircraftObject.where((Aircraft.type == "A330") | (Aircraft.id == 160)).take(1)
18
19# Actions: Validate and apply
20import datetime
21from my_sdk_package.types import ActionMode
22
23action_validation = client.ontology.actions.change_manufacture_date({"mode": ActionMode.VALIDATION_ONLY},
24                                                                     aircraft=1,
25                                                                     date_of_manufacture="2020-05-01")
26if action_validation.result == "VALID":
27    client.ontology.actions.change_manufacture_date(aircraft=1, date_of_manufacture="2023-05-26")

时间序列属性

要访问 时间序列属性，请按照前面的步骤创建、发布并导入一个本体论SDK。然后通过foundryTS库访问时间序列属性并对其进行操作。此示例展示了如何从一组City对象中检索Denver对象，并将其电力价格作为时间序列进行访问：

Copied!
1from my_sdk_package import FoundryClient
2from foundryts import TimeSeries
3
4client = FoundryClient()
5city_objects = client.ontology.objects.City
6
7denver = city_objects.get("Denver")  # This is the object's primary key
8denver_electricity_price_tsp = TimeSeries.from_osdk(denver.electricity_price)
9
10# Convert the series into a foundryts node
11node = denver_electricity_price_tsp.node()
12
13# Convert the series into a pandas dataframe
14df = denver_electricity_price_tsp.to_pandas()

R OSDK使用示例

Code Workspaces通过reticulate ↗包在R中支持OSDK，这允许您从R调用Python。

此示例演示了如何将Python OSDK版本导入到R中并与其对象类型交互。此示例假定您的SDK包含一个名为"Aircraft"的对象类型。术语my_sdk_package应替换为您的SDK的包名称。

Copied!
1library(reticulate)
2
3osdk <- import("my_sdk_package")
4client <- osdk$FoundryClient()
5
6aircraft_object = client$ontology$objects$Aircraft
7
8# Retrieve 1 object
9aircraft_object$take(1L)
10
11# Retrieve object by ID
12aircraft_object$get("1")

由于R和Python有不同的默认数值类型，当Python API期望整数时，必须在R中使用L后缀。 在官方文档中了解更多有关 reticulate 的信息。 ↗
