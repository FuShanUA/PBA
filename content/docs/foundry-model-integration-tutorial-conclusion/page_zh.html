后续步骤

下一步是将本示例工作流转换为你所在组织的真实工作流。

这通常包括：

  1. 将来自多种数据源的数据集成到 Foundry 中，创建一个可用于训练和测试不同模型的 features_and_labels 数据集。
  2. 尝试不同的模型架构、参数和特征，以使模型获得最佳性能。
  3. 通过批处理部署、实时部署或 Python 转换，将模型的预测结果集成到 Foundry 本体 中，以便在运营应用中使用。
  4. 在你的建模目标中创建发布前检查，以确保模型在发布前获得审批。
  5. 创建“回写”操作，将用户操作捕获为新的数据集，并利用这些数据对模型进行持续再训练和改进。
  6. 创建模型推理历史记录，以改进和迭代你的模型，从而实现更准确的性能和更优的使用效果。
