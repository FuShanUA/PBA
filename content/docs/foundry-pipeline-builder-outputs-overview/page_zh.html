输出

管道输出是管道构建器中管道的最终结果；输出可以是数据集、虚拟表，或本体组件（如对象类型、对象链接或时间序列）。

输出有助于描述您的管道，从添加数据到创建转换逻辑。首次部署管道构建后即会生成输出，并允许您在其他 Foundry 应用程序中扩展工作流和项目功能。

要添加输出，请在图表右侧的输出面板中选择添加管道输出。

这将跳转至输出类型选择界面。

请查看以下各节以获取有关每种输出类型的详细信息。

数据集

数据集是 Foundry 中的基本元素。管道构建器中的工作流从添加数据集开始，继而对数据集进行转换，并可以以数据集作为输出结束。当您想要构建生成干净、已转换数据的管道时，请将数据集添加为输出。您可以将最终数据集作为在本体管理器中构建本体的基础。

详细了解如何添加数据集输出。

媒体集

媒体集是具有通用模式（例如格式相同）的媒体文件集合。媒体集支持多种形式的非结构化数据，包括可视媒体、PDF 文档、音频等。当您想要构建生成干净、已转换媒体的管道时，请将媒体集添加为输出。

详细了解如何添加媒体集输出。

地理时间序列同步

地理时间序列可用于随时间推移跟踪实体的地理位置，并由不同的观测结果组成。每次观测都包含一个标识符、时间戳、位置和其他用户指定的属性。您可以在管道构建器的管道中添加地理时间序列同步输出，从而将 Foundry 中的批处理或流式地理时间数据提供给下游应用程序（如 Map 应用程序）使用。Foundry 中的地理时间观测结果是在管道构建器中写入的。

详细了解如何在管道构建器中添加地理时间序列同步。

为了在管道构建器中使用地理时间序列同步输出，您的部署实例必须启用地理时间序列功能。如需了解启用此功能的更多问题，请联系您的 Palantir 代表。

虚拟表 [Beta]

虚拟表充当指向 Foundry 外部源系统中表的指针，允许您在 Foundry 中使用该数据而无需将其接入。与数据集类似，您可以将虚拟表添加到管道中，继续进行转换，并以虚拟表或数据集作为输出结束。当您希望使用外部系统进行存储时，请将虚拟表作为输出。管道可以包含数据集和虚拟表的混合；例如，您可以从数据集开始并将输出写入外部，或者从虚拟表开始并将输出写入 Foundry。

详细了解如何添加虚拟表输出。

本体输出

将本体元素作为输出添加到管道中，可以引导您的工作流从原始数据集过渡到已清理和结构化的数据，这些数据在您的全局本体中定义了新的对象类型元素。通过管道构建器，您可以在一个工作流界面中添加和编辑对象类型、链接类型和时间序列，而无需导航回本体管理器。

详细了解如何添加本体输出。

对象类型

对象类型是现实世界实体或事件的模式定义。在工作流中添加对象类型输出，有助于将数据转换为已定义的元素，进而用于在 Foundry 中构建应用程序，包括 Workshop 模块或 Slate 应用程序。

对象链接

链接类型定义了两个对象类型之间关系的模式。链接则指代两个对象之间该关系的单个实例。如果在管道中添加两个对象类型输出，就可以通过创建链接来定义它们的关系，并将其添加到全局本体中。利用对象链接，可以在 Workshop 和 Slate 中构建功能完善的应用程序。

时间序列同步

时间序列数据指由一组或多组时间戳与数值对组成的数据；这些数据对用于衡量随时间变化的量。这可以包括按年统计的销售额、按日统计的总航班量、按小时统计的产量，或是分辨率达到亚秒级的高频温度读数。在管道中添加时间序列同步输出，可以对支撑时间序列属性的数据进行索引。

详细了解时间序列的设置与使用。
