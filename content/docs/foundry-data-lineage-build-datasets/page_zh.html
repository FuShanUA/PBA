构建数据集

你可以利用数据血缘关系图查看管道中哪些数据集已过期，随后使用构建助手直接从数据血缘关系中启动构建。

从数据血缘关系触发的构建，始终应用于图中配置的分支（包括回退分支）。

以下是几种常见的构建工作流：

  • 构建所有上游数据集
  • 所选数据集间的所有转换
  • 仅构建所选数据集

构建所有上游数据集

该策略会构建所选数据集及其全部上游数据集，确保所选数据集完全更新至最新状态。

默认情况下，该操作仅构建已过期的上游数据集，但你也可以选择强制重新构建已更新至最新状态的数据集。强制重新构建在构建时间和资源消耗方面可能成本较高。

  1. 将数据集添加至图中，或打开已保存的快照。
  2. 选择需要构建的数据集。
  3. 在构建助手中，选择所有上游数据集，然后点击下一步。

点击下一步暂不会触发任何构建操作。你只会看到待构建数据集的预览。

  1. 若需强制重新构建已更新至最新状态的数据集，请在该数据集上点击强制构建。
  2. 检查待构建的数据集列表后，点击运行构建以触发构建。

如果你决定不构建所有已过期的上游数据集，必须在当前构建预览界面点击取消，然后更改所选节点。你无法在构建预览界面更改选择。

所选数据集间的所有转换

该策略允许你将构建绑定到管道的特定子集。该策略的常见应用场景是：当新的原始数据定期进入管道，且你希望更新特定数据集以反映这些新数据，但不想构建所有已过期的上游数据集时。此时，你可以利用数据血缘关系确定还需要构建哪些其他数据集，以使目标数据集更新至最新状态。

  1. 将最终需要构建的数据集添加至图中。
  2. 将所有原始数据集（或任何上游数据集）添加至图中。
  3. 选择所有节点。
  4. 在构建助手中，选择所选数据集间的所有转换策略，然后点击下一步。

点击下一步暂不会触发任何构建操作。你只会看到基于所选节点的待构建数据集预览。现在你可以确切地看到更新目标数据集需要构建哪些内容。你可能不希望构建所有数据集——比如某个非常大的派生数据集可能只需每天构建一次——因此可以点击列表底部的全部添加至图。

所选数据集

该策略允许你挑选需要构建的单个数据集。如果数据集之间存在依赖关系，构建将按正确的顺序执行，确保下游数据集在其上游数据集构建完成后再进行构建。

若要更改正在构建的数据集，必须在当前构建预览界面点击取消，更改所选节点后再进入新的预览。你无法在构建预览界面更改构建选择。

检查最终待构建的数据集列表后，点击运行构建以触发构建。
