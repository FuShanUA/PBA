窗口函数

《PostgreSQL 官方文档 ↗》对窗口函数的定义如下：

窗口函数会在与当前行存在某种关联的一组表行上执行计算。这与聚合函数所能执行的计算类型相似。但与常规聚合函数不同的是，使用窗口函数不会导致行被分组为单行输出——各行会保持其独立的标识。在底层处理中，窗口函数能够访问的数据不仅限于查询结果中的当前行。

本文档将介绍一些在 Contour 表达式中可能用到的窗口函数的语法。有关窗口函数的更多背景信息，请参阅以下补充资料：

  • 《窗口函数（PostgreSQL 官方文档） ↗》
  • 《SQL 窗口函数（Mode Analytics 教程） ↗》
  • 《SQL 窗口函数简介（Apache Drill） ↗》
  • 《分析函数（Oracle 官方文档） ↗》

基本语法

从最基本的形式来看，窗口函数可以拆解为：

<函数> OVER <某个窗口>

这里的函数是指受支持的聚合函数之一，而窗口则是指表中的部分行集合。

你可以使用 () 来省略窗口设定——这会将函数应用于表中的所有行。

以下示例会在每一行中添加一个条目，其值为 date 列中的最大值。

MAX("date") OVER ()

PARTITION BY

你也可以在窗口定义之前添加一个可选的 PARTITION BY 子句。PARTITION BY 会根据给定列中的值对窗口内的行进行分组。随后，聚合函数会分别应用于每个分区。

例如，在一个包含人员记录的表中，以下表达式会分别计算男性和女性的总数，并将该性别的计数值添加到对应的每一行中：

COUNT("person_id") OVER (PARTITION BY "gender")

ORDER BY

如果你确实定义了窗口的表达式，就必须指定窗口的边界以及对表中行进行排序的方式。这个子表达式可以简写为：

<排序方式> ROWS BETWEEN <起始位置> AND <结束位置>

这里的“排序方式”是指：（1）依据哪一列进行排序；（2）按升序还是降序排列。

指定窗口边界（即“起始位置”和“结束位置”）有以下几种方式：

  • UNBOUNDED PRECEDING：从表的起始处到当前行。
  • n PRECEDING（例如 2 PRECEDING）：从当前行之前的第 n 行到当前行。
  • CURRENT ROW
  • n FOLLOWING（例如 5 FOLLOWING）：从当前行到当前行之后的第 n 行。
  • UNBOUNDED FOLLOWING：从当前行到表的末尾。

因此，假设有一个包含销售记录的表，你可以使用以下语句来计算过去 5 笔销售的平均销售额：

AVG("sale_value") OVER (ORDER BY "date_of_sale" ASC 
    ROWS BETWEEN 4 PRECEDING AND CURRENT ROW)

这里的“过去 5 笔销售”就是窗口。OVER 之后的子表达式会按日期对表进行排序，随后针对每一行，计算其前面 4 行以及当前行的平均值。

综合运用

以下这个综合示例将上述所有语法结合在一起。 该表达式会显示按产品类别分组的、截至当前销售的累计销售数量。

COUNT("sale_id") OVER (PARTITION BY "product_category" 
    ORDER BY "date_of_sale" ASC 
    ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW)

请注意，条目现在按分区排序——也就是说，表首先进行分区，然后在每个分区内对行进行排序。

更多示例

假设你有一张记录已购商品的表。你可以使用以下窗口函数派生出一个新列，用于计算 purchases 的累计运行总额：

SUM("item_cost") OVER (ORDER BY "purchase_date" ASC 
    RANGE BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW )

若要计算按类别分组的累计运行总额，可以添加分区：

SUM(“item_cost”) OVER (PARTITION BY “category” 
    ORDER BY “purchase_date” ASC 
    RANGE BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW)

这将会按“category”列对行进行分区，在每个类别内按购买日期对行进行排序，并计算该类别下所有商品成本的累计运行总额。

非确定性

警告

在窗口函数中使用 ROW_NUMBER、FIRST、LAST、ARRAY_AGG 或 ARRAY_AGG_DISTINCT 时，需注意非确定性问题。假设我们按列 A 进行分区并按列 B 进行排序。如果列 A 的值相同，且存在多行列 B 的值也相同的行，那么这些窗口函数的返回结果可能是不确定的——即对于相同的输入数据和逻辑，可能产生不同的结果。
