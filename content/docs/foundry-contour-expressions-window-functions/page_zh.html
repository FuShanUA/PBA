窗口函数

PostgreSQL 文档 ↗ 将窗口函数定义如下：

一个 窗口函数 在与当前行有关的一组表行上执行计算。这类似于可以使用聚合函数完成的计算类型。但与常规的聚合函数不同，使用窗口函数不会导致行被分组为单个输出行——这些行保留其单独的身份。在后台，窗口函数能够访问查询结果的不仅仅是当前行。

本文档解释了您可能想在 Contour 表达式 中使用的某些窗口函数的语法。如需更多关于窗口函数的背景信息，请参阅以下其他资源：

  • 窗口函数（PostgreSQL 文档） ↗
  • SQL 窗口函数（Mode Analytics 教程） ↗
  • SQL 窗口函数简介（Apache Drill） ↗
  • 分析函数（Oracle 文档） ↗

基本语法

最基本的情况下，窗口函数可以分解为：

<function> OVER <some window>

其中 function 是 支持的聚合函数 中的一种，window 是表中的一行子集。

你可以通过使用 () 来省略 window - 这会将函数应用于表中的 所有 行。

以下示例将在每一行中添加一个条目，该条目的值是 date 列中的最大值。

MAX("date") OVER ()

PARTITION BY

你可以选择在窗口定义之前添加一个可选的PARTITION BY子句。PARTITION BY根据给定列中的值对窗口内的行进行分组。然后，将聚合函数分别应用于每个分区。

例如，在包含人员记录的表中，以下表达式计算男性和女性的总数，并为该行中的性别值添加一个计数：

COUNT("person_id") OVER (PARTITION BY "gender")

ORDER BY

对于你定义窗口的表达式，你还必须指定窗口的范围以及如何对表中的行进行排序。这个子表达式可以简化为：

<how to sort table> ROWS BETWEEN <start location> AND <end location>

其中，“如何对表进行排序”包括（1）按哪一列排序，以及（2）是按升序还是降序排序。

指定窗口范围（“起始位置”和“结束位置”）有以下几种可能性：

  • UNBOUNDED PRECEDING: 从表的开始到当前行。
  • n PRECEDING（例如 2 PRECEDING）：从当前行之前的n行到当前行。
  • CURRENT ROW
  • n FOLLOWING（例如 5 FOLLOWING）：从当前行到当前行之后的n行。
  • UNBOUNDED FOLLOWING: 从当前行到表的末尾。

下面是一个包含上述每种情况的示例表：

（来源：blog.jooq.org ↗）

因此，给定一个包含销售记录的表，你可以使用以下内容来查找最近5次销售的平均值：

AVG("sale_value") OVER (ORDER BY "date_of_sale" ASC 
    ROWS BETWEEN 4 PRECEDING AND CURRENT ROW)

“最近5次销售”是窗口。在OVER之后的子表达式按日期对表进行排序，然后对于每一行，计算前4行和当前行的平均值。

综合起来

以下这个复杂的示例结合了上述所有提到的语法。 此表达式显示了按产品类别分组的累计销售数量，直到当前销售。

COUNT("sale_id") OVER (PARTITION BY "product_category" 
    ORDER BY "date_of_sale" ASC 
    ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW)

请注意，现在条目是按照分区排序的——换句话说，首先对表进行分区，然后在每个分区内部对行进行排序。

更多示例

假设你有一张记录你购买物品的表格。你可以使用以下窗口函数来推导出一个新列，表示购买的累计总数：

SUM("item_cost") OVER (ORDER BY "purchase_date" ASC 
    RANGE BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW )

要按类别计算累计总数，你可以添加分区：

SUM(“item_cost”) OVER (PARTITION BY “category” 
    ORDER BY “purchase_date” ASC 
    RANGE BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW)

这将根据“类别”列对行进行分区，按购买日期对每个类别的行进行排序，并计算该类别中所有物品成本的累计总数。

要计算按陨石类别的总（非运行）总质量：

Copied!
1SUM("mass") OVER (PARTITION BY "class")

这个聚合本身可能没有用，但如果我们扩展它，就可以计算出一个更有趣的统计信息——这块陨石占该类别的总质量的百分比是多少？

Copied!
1"mass" / (SUM("mass") OVER (PARTITION BY "class")) * 100

Copied!
1COUNT("id") OVER (PARTITION BY "class")

非确定性

警告

在窗口函数中使用 ROW_NUMBER、FIRST、LAST、ARRAY_AGG 或 ARRAY_AGG_DISTINCT 时，要注意非确定性。想象一下，我们按列 A 进行分组，并按列 B 排序。如果对于列 A 的相同值，有多个行具有列 B 的相同值，这些窗口函数的结果可能是非确定性的——在相同的输入数据和逻辑下，可能会产生不同的结果。
