媒体集同步

本页介绍如何通过数据连接设置媒体集源并同步至 Foundry。

以下源类型支持媒体同步：

  • Amazon S3
  • OneLake 与 Azure Blob 文件系统 (ABFS)

支持媒体同步的源列表正在不断扩充。如果尚不支持你所需的基于文件的源，你可以将文件接入至数据集中，并通过 Python 转换 将其转换为媒体集。例如，若要将 SharePoint Online 中的文件接入至媒体集，可使用 SharePoint Online 连接器 将文件同步至 Foundry 数据集，随后创建 Python 转换以读取这些文件并写入媒体集。

设置媒体集源与同步

  1. 通过 + 新建源 进入 源 页面以查找受支持的源。然后搜索 媒体同步 以查找所有受支持的源。

  1. 确保你拥有导入任何必要网络策略的权限，然后按照下方相应的说明设置受支持的源：
  • Amazon S3
  • OneLake 与 Azure Blob 文件系统 (ABFS)
  1. 在源的 概览 页面找到 媒体集同步 部分以创建媒体集同步。

  1. 选择所需的媒体文件类型以设置媒体集同步。参见 受支持的媒体集架构。

  1. 为媒体同步接入创建所需的构建调度。你可以在初始配置后编辑调度。

  1. 选择源中的相关子文件夹。如果你的媒体文件位于根路径下，则无需添加子文件夹配置。

  1. 设置同步筛选器。可用的同步筛选器包括 排除已同步的文件、路径匹配、文件大小限制 以及 忽略不匹配架构的项。

  1. 选择初始配置后，选择 保存媒体集同步。

  2. 选择 运行 以触发首次同步并查看你的媒体同步。

设置好媒体集同步后，了解如何通过 Pipeline Builder 中的转换 利用你的媒体集。
