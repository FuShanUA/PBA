定义建模目标 API

已弃用功能

以下文档介绍了在“建模目标”中将模型绑定至对象库的操作，该做法目前已不再适用于大多数场景。建议改用将模型直接部署为函数的方式，以便在各类下游应用中调用模型，包括 Workshop、Slate、Actions 以及更多应用。

为了明确用户或应用程序应如何在 Foundry 中使用您的建模目标，您可以为该目标创建一个 API。

要为新目标定义输入和输出 API，请选择建模目标详情并编辑目标 API部分。系统将跳转至定义建模目标 API页面，如下图所示：

在左侧标记为定义初始目标 API的面板（如上图所示）中，您可以选择以下方法之一来辅助开始填充建模目标 API（您也可以直接在右侧手动定义输入和输出）：

  • 若选择对象类型，请参阅关于映射对象属性的文档。
  • 若您已有心仪的模型并希望将其提交至此目标，可以通过选择模型，并按照下方的步骤从该模型的 API 入手。目前，此工作流仅支持基于对象库的模型。
  • 若希望目标 API 的输入或输出基于任意现有 Foundry 数据集的列，选择数据集即可便捷地基于这些列创建属性，具体说明请参阅此处文档。

手动定义 API 输入和输出

要手动定义 API，您可以直接指定输入和输出属性，随后选择对象类型以将属性映射至对象库。

点击“添加输入”或“添加输出”按钮，并指定属性名称、类型以及可选的描述。类型应为以下选项之一：

  • boolean
  • datetime
  • double
  • long
  • integer
  • string

若属性类型预期未知，则应使用 any 类型。

定义输入和输出属性后，您可以根据需要选择将其映射至对象属性，具体说明详见下一节。

该 API 规定了提交至建模目标的任何模型的输入与输出格式。输入将以单个数据帧的形式提供，且该数据帧的列名应与上文“输入”部分中给出的名称相匹配。同理，建模目标要求模型输出的数据帧列名须与“输出”中指定的列名相匹配。

将目标 API 映射至对象库

通过将目标 API 中的各个字段映射（亦称绑定）至对象库中的对象属性，您可以为这些字段提供额外的元数据。若建模目标需使用对象库中的数据，或用于驱动基于对象库的运营类应用程序，强烈建议进行属性映射。

例如，在 Workshop 应用程序中使用场景时，要求目标 API 必须映射至对象库。这是因为场景功能会利用这些输入映射，在执行模型时获取相应的输入数据，并利用输出映射确定需要覆盖的对象库属性。

映射对输入和输出属性的意义如下所示：

  • 对于输入属性，这意味着该属性的数据应从所映射的对象类型和属性中提取。
  • 对于输出属性，这意味着该属性的模型输出必须与所映射的对象类型属性兼容，且可写入该属性。

运营类应用程序可利用对象库映射来获取模型的输入数据，随后使用模型的输出覆盖对象库属性。因此，若您添加了对象库映射，务必清楚应用程序可能会如何使用您的目标。

要进行对象属性映射，你可以在左侧选择“对象类型”，或者在右上角切换“映射至本体”开关。随后，选择你想要映射的“根”对象类型。可供映射的对象属性将包含该对象自身的属性，以及通过多对一关系与所选对象关联的任何对象的属性。

选择对象类型后，该对象类型的所有属性都将可用于映射。要映射某个属性，请点击该属性旁边的 In 或 Out 按钮：

也可以通过属性左侧的拖拽手柄，将属性类型拖动至 Inputs 或 Outputs 区域。

也可以一次性添加该对象类型的所有属性，方法是通过对象图标左侧的拖拽手柄拖动整个对象卡片：

要映射来自关联对象类型的属性，请滚动到左侧边栏底部。如果存在关联对象，将会看到 添加关联对象类型 按钮。点击该按钮并选择一个关联对象类型。

该关联对象的属性现在即可用于映射：

默认情况下，映射后的属性将与其所映射到的对象属性类型保持相同的名称、类型和描述。如有需要，可以更改名称和描述，以使其更符合目标的需求。

完成映射后，点击 Save 按钮，即可将输入和输出 API 映射保存至目标。该 API 现将记录提交至此目标的模型所需的输入和输出规范。

随时点击 Save 都会保存目标 API 属性，并返回至主建模目标页面。

从数据集列添加属性

从左侧的下拉菜单中选择 Dataset，即可开始从现有的 Foundry 数据集中向你的目标 API 添加属性。

使用 Select Dataset 按钮浏览文件目录并找到你的数据集。

如果要在非默认分支的其他分支上使用数据集，请使用分支选择器菜单。

选择数据集后，你将在侧边栏中看到该数据集的所有列。

使用 In 或 Out 按钮可将某一列添加到目标 API 的输入或输出中。也可以通过列名左侧的拖拽手柄拖入属性。通过数据集名称左侧的拖拽手柄拖动整个数据集卡片，即可一次性添加数据集中的所有属性。

如果列的基础属性类型是建模目标 API 支持的类型，则会将其列出。否则，将显示为“any”。

要将你当前使用的数据集保存为目标的关联文件，请选择 Add as relevant file to objective。你可以随时在侧边栏中选择 Change，以切换到其他数据集进行操作。你也可以通过选择 Map to ontology 并按照映射对象属性的说明，将属性映射至本体。

随时选择 Save 都会保存目标 API 属性，并返回至主目标页面。

从模型添加属性

从左侧的下拉菜单中选择模型，开始从现有模型向目标 API 添加属性。

使用选择模型按钮浏览文件目录并选择模型。

选择模型后，侧边栏会列出该模型所有的输入和输出属性。

使用In或Out按钮，向目标 API 的输入或输出中添加列。也可以通过拖拽列名左侧的拖拽手柄来添加属性。通过拖拽模型名称左侧的拖拽手柄移动整个模型卡片，还可以一次性添加模型的所有属性。

要将当前使用的模型保存为目标的相关文件，请选择添加为目标的相关文件。随时可以通过在侧边栏选择更改来切换其他模型。还可以选择映射到本体，并按照映射对象属性的说明，将属性映射到本体。

随时点击保存即可保存目标 API 属性，并返回主目标页面。
