日志权限

服务和跟踪日志可能包含工作流所访问的任何数据源的敏感内容，包括语言模型提示和完成内容、对象属性值以及用户提供的输入。平台不会从源执行器的资源或执行过程中访问的任何数据中传播标记；只有管理员在启用日志访问时显式应用的标记才会对查看者生效。因此，启用日志访问会带来安全风险，并应根据工作流可能接触到的数据的最大敏感性进行审查。

所需角色

下表列出了AIP可观测性中各种操作所需的权限。

功能所需角色
查看 指标对资源的 查看 权限¹
查看 运行历史对资源的 编辑 权限¹
查看 追踪 和 服务日志²对资源的 编辑 权限¹ + 日志访问已启用³ + 访问所有标记
搜索日志²对资源的 编辑 权限¹ + 日志访问已启用³ + 访问所有标记
配置日志访问信息安全官员 或 注册管理员 角色
删除日志信息安全官员 或 注册管理员 角色

¹每个功能都由一个 Foundry 操作支撑：指标使用 foundry-telemetry-service:read-metrics（由 查看者 角色授予），运行历史和日志使用 foundry-telemetry-service:view-execution-history（由 编辑者 角色授予）。您可以通过使用 自定义角色 为其他角色授予这些操作。

²用户只需拥有 foundry-telemetry-service:view-execution-history 操作权限，即可访问过去24小时内自己执行的日志，而无需考虑日志访问设置。此例外情况不适用于 CBAC 堆栈，在这些堆栈中，必须启用日志访问才能查看自己执行的日志。

³日志访问启用：管理员必须直接在源执行器资源上启用日志访问（作为资源覆盖），或在源执行器的项目上启用（如果资源已被移动，则包括关联项目）。有关完整详细信息，请参见下方的日志访问要求，以及配置日志记录以了解如何启用它。

日志访问要求

要查看某个资源的运行历史，您必须对该资源具有编辑权限。

要查看您未调用的执行的跟踪和服务日志，您必须满足以下所有三个要求。日志访问概览对话框中描述了以下内容，在下面的审查日志访问要求中可以看到每个要求以及您目前是否符合这些要求。

  • 角色：您必须对源执行器具有编辑权限。源执行器是调用链中的第一个可执行资源，可以是函数、操作、自动化、AIP逻辑、AIP代理或模型实时部署。
  • 日志访问策略： 必须为源执行器启用日志访问权限，可以在其 项目（以及其 归属项目 —— 资源最初发送遥测数据的项目 —— 如果资源已被移动）上启用，或者通过 资源级覆盖 来启用。由 信息安全官 或 注册管理员 启用此功能。有关如何启用日志访问，请参阅 配置日志记录。
  • 标记： 您必须拥有应用于日志的每一项标记。请参见下面的 日志内容上的标记。

操作是一个特殊情况。由 旧版本体论权限 管理的操作目前还不能在项目级别进行管理，因此需要使用 资源级覆盖 来为其启用日志访问权限，而不是项目设置，直到该操作被 迁移到基于项目的权限。有关详细信息，请参阅 配置日志记录。

源执行器日志访问状态

在 运行历史 和 日志搜索 面板中，工作流谱系显示了源执行器日志访问的状态标签。该标签反映了您当前的访问级别：

  • 完整日志访问权限：项目上的日志访问权限已启用，或通过资源级别覆盖启用，且您具备所需的标记。来自该资源的所有执行日志均可查看。
  • 用户ID受保护的日志访问权限：根据您的角色，您可以查看过去24小时内自己执行的日志。即使日志访问权限未被启用（CBAC堆栈除外），此权限仍然适用。
  • 无日志访问权限：日志访问权限未被启用，且您没有其他途径访问日志。

当项目上的日志访问权限已启用且标记权限满足时，源执行器日志访问权限将显示为已启用。所有来自已启用项目的执行日志都将可见。

否则，仅显示您过去24小时内的执行日志（CBAC堆栈除外）。

在 基于分类的访问控制（CBAC）堆栈 上，您对自己执行日志的24小时访问权限被禁用。在用户可以查看追踪和业务日志或搜索日志之前，必须在源执行器的项目中启用日志访问权限，即使对于他们自己的执行也是如此。

打开日志访问概览

要查看您需要读取日志的内容，请从以下任一位置打开 日志访问概览 对话框：

  • 从工作流谱系图中，选择资源的节点并选择 查看日志访问。
  • 从 运行历史 或 日志搜索 面板中，在右上角选择 查看日志访问。

查看概览至少需要对资源及其项目具有 Viewer 角色。

如果你拥有 信息安全官 或 注册管理员 角色，并且可以管理资源的日志访问权限，那么 运行历史 和 日志搜索 面板将显示 编辑权限，而不是 查看日志访问。该菜单添加了 配置日志访问 和 删除日志历史，这两项内容详见 配置日志记录。

审查日志访问要求

日志访问概览 对话框列出了查看者必须满足的三个要求才能查看日志。

  • 角色： 显示您在资源上的当前角色。如果您的角色不授予日志访问权限，对话框会建议一个权限最小的角色，该角色可授予日志访问权限。
  • 日志访问策略： 显示包含此资源的项目的日志访问是否已启用（如果资源已被移动，则包括关联项目），是否通过资源级覆盖启用，或者未启用。如果您具有 信息安全官员 或 注册管理员 角色，您可以选择 编辑 来配置该策略（参见 配置日志记录）。
  • 日志访问标记： 显示应用于日志的标记，以及如果缺少任何标记，缺少的数量。

日志内容上的标记

保护日志内容的标记由管理员在启用日志访问时显式配置。这些是当用户尝试查看日志时唯一强制执行的标记。

标记不是从源执行器的资源、其输入或执行过程中访问的任何数据中派生的。管理员负责选择能够反映工作流可能涉及的数据的最大敏感性的标记，因为平台无法预先确定工作流可能访问的所有数据源。未设置标记的日志对满足上述角色和日志访问要求的每个用户都是可见的。
