注册第三方应用程序

用户应使用 开发者控制台 来注册新的应用程序配置。控制面板视图仅在用户未启用开发者控制台时适用。

在将第三方应用程序连接到 Foundry 之前，必须先在 Foundry 平台上进行注册。初始注册过程会为第三方应用程序创建名称、客户端 ID 和客户端密钥；有关授权工作流中使用的客户端 ID 和客户端密钥的更多信息，请参阅 OAuth.com 文档 ↗。随后，需要为第三方应用程序配置用于授权过程的重定向 URL，以及用于在平台内展示该应用程序的名称、描述和图标。

注册

  1. 要开始注册新应用程序的过程，请导航至控制面板中的第三方应用程序选项卡，然后点击新应用程序。

  1. 这将打开注册新应用程序向导。接下来的四个步骤依次为：详细信息、客户端类型、授权许可类型和摘要。

  1. 在详细信息步骤中，为您的应用程序提供名称、描述（可选）和徽标（可选）。
  2. 在客户端类型步骤中，为您的应用程序指定客户端类型。客户端类型指的是关于客户端应用程序能否安全存储密钥的 OAuth2 标准。客户端类型有两个选项：
    • 机密客户端 ↗：适用于能够安全持有其凭证的客户端；例如，在安全服务器上实现且对客户端凭证访问受限的客户端。此客户端类型支持授权码许可和客户端凭证许可两种授权选项。
    • 公共客户端 ↗：适用于无法安全持有其凭证的客户端；例如，授权客户端直接在浏览器中运行的基于浏览器的应用程序。此客户端类型支持带 PKCE 的授权码许可，这意味着必须使用 code_verifier 和 code_challenge 参数。不支持 客户端凭证许可。

      有关这些客户端类型的更多信息，请参阅有关编写 OAuth2 客户端的文档。
警告

原生或单页应用程序（如移动应用）会分发给用户进行部署。因此，应用程序的二进制文件是可获取的，可以通过反编译提取客户端密钥。随后，该客户端密钥可能在攻击中被用于冒充已授权用户。代码交换证明密钥 (PKCE) ↗ 可用于防范此类攻击。

  1. 在授权许可类型步骤中，您将看到上一步选择的客户端类型所支持的许可类型。如果您选择启用授权码许可，系统会要求您至少指定一个重定向 URL。

    • 在授权过程中，OAuth2 使用浏览器重定向将用户从授权提供方（此处为 Foundry）发送回用户尝试授权的客户端（此处为第三方应用程序）。因此，指定重定向 URL 有助于在第三方应用程序请求访问 Foundry 资源时提供额外的安全性。
    • 请注意，稍后可以在管理应用程序界面中更新重定向 URL。

    如果您选择启用客户端凭证许可（仅机密客户端可用），系统将为该应用程序创建一个服务用户。可以为该服务用户授予访问 Foundry 资源的权限，以代表应用程序处理请求。

  2. 在摘要步骤中，系统将显示所提供的所有信息的概览，并指出任何仍需补充的遗漏部分。填写完必填字段后，您可以点击屏幕右下角的注册应用程序。

  3. 提交后，系统将向您展示新创建的客户端 ID 和密钥（如适用）。

警告

如果使用机密客户端，您必须在此时复制客户端密钥。离开此页面后，该密钥将不再显示。如果您丢失了对客户端密钥的访问权限，将需要轮换密钥。
