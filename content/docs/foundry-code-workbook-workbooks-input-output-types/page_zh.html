输入类型与转换

Code Workbook 允许您指定转换将接收的每个输入类型。您可以通过转换名称下方的下拉菜单来设置输入类型：

如果请求的输入类型与实际传入的输入类型不匹配，Code Workbook 会执行必要的转换。

例如：如果 transform_A 的输出是 pandas dataframe，而其派生转换 transform_B 手动将输入设置为 Spark dataframe，Code Workbook 会将 pandas dataframe 转换为请求的 Spark dataframe。如果 transform_C 的输出是 pandas dataframe，而其派生转换 transform_D 请求其输入为 R data.frame，Code Workbook 会先将 pandas dataframe 转回 Spark dataframe，然后再转换为 R data.frame。

输入与输出限制

输出类型

一个 Code Workbook 转换可以拥有任意数量的输入，每个输入都可以配置不同的输入类型。但是，一个转换只能有一个输出。目前不支持多输出。

Code Workbook 根据编程语言的不同，仅允许其转换中使用特定类型的输出：

语言允许的输出类型
PythonSpark dataframe, Pandas dataframe, FoundryObject, None
RR data.frame, FoundryObject, NULL
SQLSpark dataframe

如果转换返回的类型不是上述列出的类型，Code Workbook 将返回不支持的类型错误。

关于 None 和 NULL 输出类型的说明

Code Workbook 允许您运行返回值为 None/NULL 的转换，但下游转换不会接受 None/NULL 作为输入。

Python 输入类型

默认情况下，Python 转换会将输入作为 Spark dataframes 接收。您可以通过转换名称下方的下拉菜单设置输入类型，Code Workbook 会负责将上游输出转换为所需的输入类型：

Python 转换支持以下输入类型：

输入类型描述
Spark dataframe（默认）类为 pyspark.sql.dataframe.DataFrame 的 pySpark dataframe
Pandas dataframe类为 pandas.core.frame.DataFrame 的 Pandas dataframe
Python 转换输入（仅在输入“另存为数据集”时可用）一个允许对输入进行直接文件访问的 FileSystem 对象。参见《访问非结构化文件》文档。
对象一个允许您执行建模工作流的 Foundry 模型对象。参见《模型》文档。

R 输入类型

默认情况下，R 转换会将输入作为 R data.frames 接收。您可以通过转换名称下方的下拉菜单设置输入类型，Code Workbook 会将上游输出转换为所需的输入类型：

R 转换支持以下输入类型：

输入类型描述
Spark dataframe类为 SparkDataFrame 的 SparkR dataframe
R data.frame（默认）类为 data.frame 的 R dataframe
R 转换输入（仅在输入“另存为数据集”时可用）一个允许对输入进行直接文件访问的 FileSystem 对象。参见《访问非结构化文件》文档。
对象一个允许您执行建模工作流的 Foundry 模型对象。参见《模型》文档。

SQL 输入与输出类型

SQL 转换中唯一支持的输入和输出类型是 Spark dataframes。该 dataframe 可以在 SQL 中作为表进行读取。
