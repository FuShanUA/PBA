建立直接连接

此文档已过时，仅作历史参考保留。有关连接设置的最新指南，请参阅我们关于《设置源》的文档。

直接连接依赖于 Foundry 的容器基础设施，该基础设施仅在 Foundry 的托管 SaaS 平台中提供。因此，您的环境中可能无法使用基于云的直接连接。

如果您的数据源可通过互联网访问（例如 REST API、SFTP 服务器或 Azure 存储帐户），您可以配置直接连接，从而无需《设置智能体》。使用直接连接具有以下优势：

  • 无需预配、配置和管理智能体及其主机
  • 避免将互联网至 Foundry 的流量路由通过您的网络
  • 提供出色的正常运行时间和性能，因为基于云的同步不依赖于智能体软件包或其主机

如果您希望配置基于云的直接连接，请按照以下步骤操作：

  1. 为您的环境配置网络出口策略。
  2. 预配连接数据源所需的凭据。
  3. 在数据连接中创建源。

配置网络策略

您必须在您的环境中拥有信息安全官角色才能配置网络出口。如果您没有配置出口的权限，请联系您的 Palantir 代表寻求帮助。

信息安全官角色可以在控制面板的环境权限部分找到。管理员需要拥有环境管理员角色才能看到此部分。

要配置网络策略，请使用工作区侧边栏中的其他工作区链接导航至控制面板。在控制面板中，选择侧边栏中的网络出口。如果您看不到此选项，请联系您的 Palantir 代表完成以下步骤。

通过选择添加网络策略来添加网络策略。添加描述和连接详细信息，类似于您在联系 Palantir 时提供的详细信息：

  • 如果您通过 HTTP(S) 连接，请添加数据源的 DNS 主机名
  • 如果需要使用非 HTTP 协议，请添加 CIDR 地址和端口

保留默认的可选策略类型选择，然后选择添加网络策略。

预配凭据

在大多数情况下，Foundry 需要经过授权的凭据（例如用户名和密码）才能访问源。最佳做法是使用专门为 Foundry 创建的服务帐户。

请遵循您组织内部关于建立服务帐户的任何指南和流程，为源预配服务帐户。在继续下一步之前，请记录好凭据。

在数据连接中创建源

完成上述步骤后，您可以继续在数据连接中创建源：

  • 登录后，使用侧边栏导航至数据连接。
  • 选择源选项卡。
  • 在右上角选择新建源。
  • 选择与您的数据源对应的源类型。
  • 选择直接连接，然后在右下角选择继续。

将源保存在项目中

接下来，命名您的源并选择一个项目来放置它。我们通常建议为每个源创建一个新项目，因为这提供了对源自该源的数据集进行权限设置的最简洁方式。有关更多信息，请参阅《源权限最佳实践》。Foundry 中有关如何构建端到端数据管道的完整指南可在《推荐的项目结构文档》中找到。

在右下角选择创建源并继续。

选择您的网络策略

在下一页中，点击 使用现有策略 并搜索策略名称，选择您先前配置的网络策略。

配置源并添加驱动程序

添加有关如何连接到您的源的详细信息。这些详细信息取决于您所使用的源类型，通常包含连接 URL、云提供商区域等基本凭证。

JDBC 源可能需要添加并选择连接到您的源所需的驱动程序。尽管 Foundry 内置了许多开箱即用的驱动程序，您可能仍需上传并选择一个驱动程序才能继续。

添加凭证

添加您先前配置的凭证，以允许直接连接连接到您的数据。

保存并继续

在右下角选择 保存 以完成直接连接的设置。源完全设置好后，您可以继续设置同步，将数据引入 Foundry。
