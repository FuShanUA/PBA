发现和使用 Python 库

代码仓库（Code Repositories）允许您导入和使用公共库以及 Foundry 生成的库。以下信息仅适用于 Python 库。

发现 Python 库

要搜索 Python 库，请点击代码仓库环境左侧面板上的包选项卡。使用搜索框搜索可用的库。

点击库名称以显示库详细信息和 添加库 选项，该选项会将库的推荐版本添加到您的分支中。

一旦您添加了库，代码助手依赖项将被刷新，并且您将能够从库中可用的包导入模块。

添加特殊依赖项

添加Python库有时需要将必要的工件仓库添加到您的代码仓库并引用在Project中。在这种情况下，会弹出一个对话框，要求您确认此操作。

依赖项对话框将突出显示您没有访问权限的任何必需仓库。您需要访问这些仓库才能使用该库。

Code Repositories 将尝试为您的库找到可用的支撑仓库并自动使用它们。在需要特定支撑仓库的情况下，您可以直接在仓库的 工件设置 页面中进行设置。

固定特定库版本

如果您需要特定版本，可以 固定 通过点击设置按钮并从列表中选择所需的版本来完成。这使您可以设置不同的版本以运行您的转换和测试。

请注意，固定特定库的版本后，它将被添加到您的meta.yaml文件中，但不会通过任务运行器进行安装。验证检查是否成功通过，然后重新启动代码助手以将更改应用到您的meta.yaml文件。然后，您就可以在您的代码仓库中使用所选版本的库了。

警告

固定特定版本时请格外小心。固定版本可能会阻止您的代码获得重要的更新。请始终确保审查并更新您的依赖项。

查看库的变更

添加和删除代码仓库中的 Python 库的操作与其他代码更改类似。您应该提交更改并将其合并到受保护的分支中。在拉取请求中，库更改将显示为 meta.yaml 文件中的更改。

有时，当您创建拉取请求时，您会看到类型为 .lock 的文件出现合并冲突。这是当您在分支上工作时发生更新时发生的。在这种情况下，请接受所有更改并继续您的拉取请求。

发布到共享通道的库 [已弃用]

警告

这是一个已弃用的功能，在您的环境中可能不可用。

如果你的库已发布到共享频道（参见 发布一个Python库），你必须编辑Python子项目的 build.gradle。你将要修改的是 隐藏 文件，因此在继续之前，请确保选择“显示隐藏文件”。你的Python转换项目需要能够访问你已发布共享库的频道。在你的Python转换子项目文件夹中的 build.gradle 文件中，添加以下内容：

Copied!
1 transformsPython {
2     sharedChannels  "libs"
3}

你必须编辑位于你的Python子项目文件夹中的 build.gradle 文件，而不是仓库根目录下的那个文件。确保将内容添加到文件末尾，这样 apply plugin: 就会在处理它之前被正确执行。

构件仓库设置

通常在使用Python库时不需要访问仓库设置。当您添加Python包时，所需的工件仓库将自动添加。您应避免直接在设置选项卡中编辑引用的Python仓库列表。

当使用共享库时，会将相关仓库的引用添加到项目中。您可以通过访问仓库设置选项卡中的“工件”部分来查看引用仓库的列表。

如果您手动添加了共享库依赖项，而不是通过工件包搜索添加的，那么您需要通过工件设置将库仓库添加到消耗仓库的支撑仓库中。请注意，这样做是不推荐的，因为包搜索会为您执行此操作。

任务运行器

任务运行器仅在较新的代码仓库中可用。您可以通过显示隐藏文件并检查 templateConfig.json 文件来查看您的代码仓库版本。

  • 如果您的代码仓库有一个 parentTemplateId，其值为 transforms，请确保 parentTemplateVersion 的版本为 8.220.0 或更高，并且 Python 子模板的版本为 1.484.0 或更高。
  • 如果您的代码仓库有一个 parentTemplateId，其值为 python-library，请确保 parentTemplateVersion 的版本为 1.497.0 或更高。

您可以 升级您的代码仓库 以启用此功能。

当任务运行器被启用时，在左侧面板的库选项卡中添加一个包将不再提供一个新的代码助手工作区，而是会在当前环境中安装所请求的包，并返回底层进程的所有日志。

如果安装成功，锁定文件将更新为新的环境。如果安装失败，任务运行器底部面板将显示错误消息，可用于进一步查找问题。

任务运行器只会更新 run 环境，并且目前不支持安装仅用于测试的依赖项。

meta.yaml 文件

警告

您应避免编辑 meta.yaml 文件，因为这容易出错。相反，建议通过库搜索界面添加库。

要在一个代码仓库中使用的 Python 库必须包含在 conda_recipe/meta.yaml 文件中。当通过仓库库搜索界面添加库时，会自动完成此操作。

Copied!
1requirements:
2...
3run:
4    - python
5    ...
6    - {LIBRARY NAME} # Replace this with the name of your shared library.

添加库后，点击 meta.yaml 文件顶部的“刷新依赖项”。这将确保 Code Assist 与新依赖项同步，使您能够从可用包中导入模块。

Conda 解决 Python 包

Conda 是一个开源的、语言无关的包和环境管理器，在 Code Repositories 中用于解析包依赖关系，并将一组包安装到独立环境中。有关更多信息，请参阅 官方 Conda 文档 ↗ 或 环境创建简介。

Conda 锁定文件

当在代码仓库中运行检查时，我们会解析 meta.yaml 文件中列出的包的 Conda 环境，并生成带有 .lock 扩展名的隐藏 Conda 锁文件；这些锁文件保存了 Conda 环境。此预解析的环境使后续检查在您提交代码时更快。

在以下情况下，我们会重新解析 Conda 环境并编写新的锁文件：

  • meta.yaml 文件中的包列表发生了变化。
  • 仓库升级到了更新的模板版本。
  • 在您的锁文件中发现了被撤回的包。
  • 隐藏的 Conda 锁文件已被删除或编辑。

当我们重新解析环境并编写新的锁文件时，初始提交哈希为 SUPERSEDED，然后是另一个提交哈希，该哈希会重新写入锁文件。当需要重新解析环境时，检查可能需要更长时间运行。

下载平台外发布的 Python 库

可以从 Foundry 之外下载在 Palantir Foundry 中发布的 库：

  1. 获取用户令牌。在 Foundry 中，导航到您的用户账户设置并选择 令牌。选择 创建令牌，输入令牌的名称和描述，并选择 生成。复制您的令牌。
警告

不要与其他应用程序或用户共享您的令牌；恶意人员可能使用它来冒充您。

  1. 找到您的 Python 库的 <identifier>。导航到感兴趣 Python 库的代码仓库。浏览器 URL 现在应该类似于：https://<my-foundry-url>/workspace/data-integration/code/repos/<identifier>/contents/refs%2Fheads%2F<branch>。在 URL 中，找到 <identifier>，它看起来像 ri.stemma.main.repository.XXXXXXXX-XXXX-XXXX-XXXX-XXXXXXXXXXXX。
  2. 获取该库的包索引 . 在本地计算机上打开终端并运行以下命令： curl -H "Authorization: Bearer $TOKEN" https://$STACK_URL/artifacts/api/repositories/$IDENTIFIER/contents/release/conda/$PLATFORM/repodata.json 其中 $TOKEN 是您的用户令牌，$STACK_URL 与上一步中的 <stack-url> 相同，$IDENTIFIER 是上一步中的 <identifier>，$PLATFORM 是代码仓库平台（例如 noarch, linux-64）。上述命令的输出是库内容的索引（即它包含的包）。每个包的形式为 <name>-<version>-<build-string>.tar.bz2。
  3. 下载特定包。您可以通过在终端中运行以下命令从 Python 库中下载特定包： curl -H "Authorization: Bearer $TOKEN" https://$STACK_URL/artifacts/api/repositories/$IDENTIFIER/contents/release/conda/$PLATFORM/$PACKAGE 其中 $PLATFORM 是包平台（例如 noarch, linux-64），$PACKAGE 的形式为 <name>-<version>-<build-string>.tar.bz2。

在任务运行器中运行任务

您可以通过手动指定命令，从任务运行器中手动触发包安装。

例如，要安装 pandas，请在底部面板中选择 任务运行器 选项卡，然后输入以下命令：

install --packageSpecs=pandas

Task Runner also supports the following commands:

  • uninstall: Uninstalls packages from the run environment.
    • Note: This command will also uninstall any package that depends on the specified package. This command is only available on newer repositories. If your repository has a parentTemplateId that is transforms, ensure that the Python child template is on 1.525.0 or higher. Similarly, if your repository has a parentTemplateId that is python-library, ensure that parentTemplateVersion is on 1.525.0 or higher.
    • Usage: uninstall --packageSpecs=<eg.numpy> for Conda packages.
  • formatCode: Formats the files in the repository. Uses Black to format the code and pyproject.toml for custom formatter configuration. See the Black formatter configuration file documentation ↗ for more information.
  • whoneeds: Shows the tree of packages that require the installation of a package in the current run environment.
    • Note: This command is only available on newer repositories. If your repository has a parentTemplateId that is transforms, ensure that the Python child template is on 1.522.0 or higher. Similarly, if your repository has a parentTemplateId that is python-library, ensure that parentTemplateVersion is on 1.522.0 or higher.
    • Usage: whoneeds --packageSpec=<eg.transforms>
  • tasks: Lists all the available tasks to run.
