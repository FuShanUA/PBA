使用 TypeScript OSDK 订阅对象集合的更改

GA

订阅功能作为 OSDK v2.1 的一部分发布；要访问此功能，请打开 Developer Console 并生成你的 SDK 的 2.1.x（或更高版本）。

在使用订阅功能之前，请确保通过以下命令在项目中安装了匹配的 2.1.x 版本的 @osdk/client 包，或更新你的 package.json。

Copied!
1npm install @osdk/client@^2.1.0

本页面提供有关如何使用 TypeScript OSDK 接收关于您的本体论中对象更改的实时更新的文档。

概述

订阅功能允许您在指定对象集中的对象发生更改时接收更新。这包括新增和删除对象，以及对象集中对象属性的更改。

以下各节中的示例使用了一个Country对象类型，其包名为@subscribe-osdk-example/sdk。请分别将示例包名和对象类型替换为您创建的包和您选择的对象类型。最后，将对population的引用替换为您的对象类型中的一个属性。

请注意，使用.pivotTo构建的对象集不支持订阅。此外，每次只能有一个订阅处于活动状态。例如，如果您正在构建一个OSDK应用程序并使用TSv2来订阅对象集更改，两个React组件不能同时在不同的订阅上监听同一对象集的更改。

.subscribe

要订阅一个对象集，将 .subscribe 链接到 OSDK 对象集。.subscribe 函数接受多种函数，这些函数将在订阅生命周期的不同事件中被触发。这些函数可以包含您希望针对给定事件执行的任何逻辑。

subscribe 函数返回一个包含用于结束订阅的函数的对象。

Copied!
1import { Country } from "@subscribe-osdk-example/sdk";
2
3const subscription = client(Country).subscribe(
4    {
5        onChange: (update) => {},
6        onSuccessfulSubscription: () => {},
7        onOutOfDate: () => {},
8        onError: () => {},
9    },
10    { properties: [] },
11);
12
13subscription.unsubscribe();

此外，还可以过滤所订阅的对象集。这将缩小您将收到更新的对象集合。

Copied!
1client(Country)
2    .where({ population: { $gte: "1000000" } })
3    .subscribe({});

使用 .pivotTo 构建的对象集不支持订阅。

onChange

当指定对象集中的对象被添加、更改或删除时，会触发 onChange 处理程序。

处理程序函数提供了一个包含两个属性的对象：

  • 对象（Object）: 一个 Osdk.Instance 对象和一个描述所提供对象状态的枚举。这与从对象集上的 .fetchPage 方法返回的对象类型相同。
  • 状态： 一个描述所提供对象状态的枚举。
    • ADDED_OR_UPDATED: 所提供的对象被添加到对象集中，或者所提供的对象上的某个属性发生了变化。
    • DELETED: 所提供的对象被删除。

下面是 onChange 处理程序使用的示例：

Copied!
1const populations: { [key: string]: number } = {};
2
3client(Country).subscribe({
4    onChange: (update) => {
5        if (update.state === "ADDED_OR_UPDATED") {
6            populations[update.object.$primaryKey] =
7                update.object.population ??
8                populations[update.object.$primaryKey];
9        } else if (update.state === "DELETED") {
10            delete populations[update.object.$primaryKey];
11        }
12    },
13});

属性可为空性

提供给处理程序的 Osdk.Instance 对象上的属性代表该对象的最新值。对象上的主键属性和 $primaryKey 字段始终是定义好的。

Osdk.Instance 对象上的任何属性都可能是未定义的。这并不一定表示该属性被更新为未定义，而是表示该属性根本没有被返回。

如果自上次生成SDK以来，某个对象类型中添加了更多属性，则即使提供的值上的属性没有更改，也可能会收到更新。这表明您的SDK未意识到的某个属性已发生更改。

地理时间序列更新

如果为具有地理时间序列属性的对象类型请求订阅，当该序列有新值时将触发更新。此值将在返回的对象的lastFetchedValue属性上可用。

onSuccessfulSubscription

当OSDK建立成功订阅时会执行此处理程序，并开始接收更新。在触发此处理程序之前接收到的更新可能不会反映在对其他处理程序的调用中。

onOutOfDate

当无法为对象集的更新提供最新信息时，将执行此处理程序。这表示您应通过调用 .fetchPage 来重新加载整个对象集，以确保接收最新的信息。

onError

当订阅过程中发生错误时，将执行此处理程序，表示订阅已被关闭。

指定属性

该 .subscribe 方法允许指定从更新中返回的对象上接收哪些属性。这可以通过减少通过网络连接发送的信息量来提高性能。要指定这些属性，请在方法的第二个参数中传入您想要的属性数组。

Copied!
1client(Country).subscribe({}, { properties: ["population"] });
