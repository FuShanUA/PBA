在Apollo中导出和导入数据的指南

本策略概述了您可以从Apollo Hub导出并导入到其他Apollo Hubs的数据类型。

无法导出或导入的资源

Hub 设置和配置

Apollo Hub 的设置对其安全、合规性和运营健康至关重要。您无法从 Apollo 中的 设置与配置 页面导出配置的字段，并将其导入到另一个 Hub。

Hub 设置必须通过 Apollo 服务默认设置，或由管理员在目标 Hub 初始化时设置。这将确保目标 Hub 管理员可以查看和理解哪些设置不会被导入过程更改。

权限

您无法导出或导入资源的角色授权、资源所有权的更改，或团队或组成员的更改。目标中心的权限必须从该中心进行设置。

可以导出或导入的资源

以下项目将不带设置或权限导出。导入 Bundle 后，您需要从目标中心配置这些缺失的设置。

Apollo 资源

您可以导出和导入普通用户经常创建、修改或交互的资源。这包括新的产品发布、产品等。

Kubernetes 命名空间

您可以在一个环境中导出和导入由 Apollo 管理的 K8s 命名空间。请注意，如果您导出一个启用了特权功能的 K8s 命名空间，它将作为普通非特权 K8s 命名空间导入。导入后，您可以打开一个变更请求，将导入的 K8s 命名空间再次标记为特权。这确保了任何特权 K8s 命名空间在运行它们的 Hub 中已经获得批准，即使它们是从另一个 Hub 导入的。

产品发布元数据中的发布渠道

发布渠道本身并不是可以单独导出的资源，但它们可以作为产品发布元数据的一部分进行导出和导入。下文将讨论在产品发布上下文中导出和导入发布渠道的情况，以及特定产品发布中元数据包含的发布渠道会如何处理。

发布通道导出

每个发布通道都可以配置为可导出或不可导出，作为产品发布元数据的一部分。该配置是针对每个发布通道的，并可以在 中心设置 > 发布通道 下找到。一个不可导出的发布通道仍然可以作为导出管道配置中的资源选择器使用，但导出的产品发布将 不包含 该发布通道。

发布通道导入

在导入产品发布时，Apollo 将仅自动导入以下发布通道： DEV、RELEASE_CANDIDATE、RELEASE、STABLE、STABLE_2。对于其他所有发布通道，您必须在导入产品发布之前在目标中心手动创建发布通道。此外，在产品发布被提升到发布通道之前，必须通过任何 发布通道要求。
