工作流血缘

以前称为Workflow Builder的应用程序现在称为Workflow Lineage。

Workflow Lineage 提供了一个交互式工作区，用于理解并管理应用程序及其底层流程。

使用 Workflow Lineage，您可以执行以下操作：

  • 探索工作流以查看对象、操作、函数、大语言模型和应用程序的详细信息，包括：
    • API 名称
    • 输入
    • 本体论编辑
    • 提交标准
    • 代码片段
  • 对于对象中的特定列，查看下游所有用法，包括依赖的操作和 Workshop 应用程序。
  • 使用颜色图例轻松查看所有操作、应用视图等中过时的函数、资源或本体论权限。
  • 批量选择操作，将这些操作同时更新到特定版本。

Workflow Lineage 特别适合以下用户：

  • 创建、调试或维护工作流的应用程序构建者。在对工作流进行更改或扩展时，溯源图、更深入的属性和工作区小部件/变量溯源以及升级工具都非常有帮助。
  • 希望从现有工作流中学习以回答“此工作流中使用了本体论的哪些部分，以及如何使用？”这一问题的新建者。
  • 正在展示现有工作流的用户：该图提供了所用本体资源的概览，当与他人分享工作流时，可以作为有用的演示和教学工具。

要启用工作流溯源，请联系您的平台管理员以在控制面板中 修改应用程序访问权限。

何时使用工作流溯源

工作流溯源的目的是帮助理解、管理和调试工作流。工作流通常涉及多个本体资源，并且常常流入一个应用程序。因此，工作流溯源与 管道构建器 和 数据溯源 是互补的：

  • 管道构建器：将原始数据源的数据整合到本体论中。
  • 工作流血缘关系：对基于本体论构建的工作流进行管理。
  • 数据血缘关系：从源数据到本体论，再到工作流的端到端数据流动视图。

例如，如果您需要了解在工作流血缘关系中某个对象类型的数据调度信息，您可以右键点击并打开该对象类型以在数据血缘关系中探索调度信息。

了解如何开始使用工作流血缘关系。

访问控制

您可以在控制面板中控制组织中的哪些用户可以访问工作流血缘关系。

在 应用设置 选项卡的 应用访问 中，导航到 本体论 部分。在 工作流血缘关系 条目右侧选择 管理。

选择 管理 以打开一个窗口，您可以在此配置注册的首选项。
