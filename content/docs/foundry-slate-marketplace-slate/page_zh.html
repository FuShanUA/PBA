将 Slate 应用程序添加到 Marketplace 产品中

使用 Foundry DevOps，您可以将 Slate 应用程序包含在 Marketplace 产品中，供其他用户安装和复用。Marketplace 支持单页和多页 Slate 应用程序，包括不加载数据的静态应用程序。在打包和安装过程中，所有页面、路由和页面结构都会予以保留。了解如何创建您的第一个产品。

将 Slate 应用程序添加到产品中

若要将 Slate 应用程序添加到产品中，请先创建产品，然后添加输出。选择添加文件选项，从 Compass 文件系统中导航至该 Slate 应用程序并将其添加到您的产品中。

使用 Marketplace 参数变量打包 Slate 应用程序

您可以将本地变量标记为 Marketplace 参数，允许用户在产品安装过程中自定义其值。这有助于配置单次安装的设置和默认值，而无需在安装后修改 Slate 应用程序代码。

要将变量标记为 Marketplace 参数：

  1. 在您的 Slate 应用程序中打开变量面板。
  2. 选择您想要使其可配置的本地变量。
  3. 选择该变量的菜单，启用 Marketplace 参数选项并选择一种类型（字符串或布尔值）。

被标记为 Marketplace 参数的变量将在变量面板中显示视觉指示器。当用户安装产品时，可以在安装过程中为这些参数提供自定义值。

只有主文档页面上的本地变量才能被标记为 Marketplace 参数。目前不支持将多页应用程序中其他页面上的共享变量和变量作为 Marketplace 参数。

打包使用代码沙盒组件的 Slate 应用程序

如果 Slate 应用程序在代码沙盒组件上依赖外部库，可以通过 Marketplace 进行打包。

若要在安装时生成库文件的副本，请确保在创建产品时，将库文件包含在内容选项卡下的文件选项卡中。用户安装产品时，其项目中将获得这些文件的副本。随后，代码沙盒组件将引用他们的库文件副本。

Slate 还支持通过 CDN 链接提供库（例如，https://unpkg.com/browse/chart.js@2.7.1/）。CDN 链接保持不变，因此安装后的 Slate 应用程序将具有相同的 CDN 链接。这可能意味着用户需要配置其 CSP 以允许这些 CDN 链接。

有关在代码沙盒组件中使用库的更多信息，请参阅代码沙盒组件文档。

打包使用 Ontology SDK 的 Slate 应用程序

使用 Ontology SDK (OSDK) 的 Slate 应用程序可以通过 Marketplace 进行打包。Foundry DevOps 会在打包过程中自动处理 OSDK Ontology 依赖项，因此无需手动配置依赖关系。

OSDK 打包工作原理

发布使用 OSDK 的 Slate 应用程序时，预构建的 OSDK 包会自动包含在 Marketplace 包中。这意味着在安装过程中，该包将被直接复制，而不是从头重新生成，从而显著减少安装时间。

重新打包现有应用程序以实现更快的安装

在预构建包优化功能推出之前打包的 Slate 应用程序，仍会在安装期间重新生成 OSDK 包。若要利用更快的安装速度，请发布现有 Marketplace 产品的新版本。新版本将自动包含预构建的 OSDK 包。

要重新打包您的应用程序：

  1. 在 Foundry DevOps 中打开产品。
  2. 创建产品的新版本。
  3. 发布新版本。

Slate 应用程序本身无需进行任何更改。预构建的代码包会在发布时自动包含。

OSDK 要求

所有本体（Ontology）资源必须使用与源应用程序相同的 API 名称。可以通过不加前缀进行安装，或将输入映射至具有匹配 API 名称的现有本体资源来实现这一点。

如果 API 名称不匹配，Slate 函数代码将与目标环境中的本体实体不匹配。安装后通过手动更新 Slate 函数代码，可以解决此类不匹配问题。

在以下情况下，安装时可能会发生 API 名称冲突：

  • 目标环境中存在具有相同 API 名称的本体实体。例如，两个环境都有一个名为 flights 的对象类型。
  • 对象的 API 名称与其显示名称不匹配。例如，如果显示名称为 flights 的对象类型其 API 名称为 flightsXYZ，则在安装时会为其分配默认的 API 名称 flights。这种情况下，Slate 函数代码将不再与 API 名称匹配，并且如果已存在 API 名称为 flights 的对象类型，则可能会发生命名冲突。

这适用于所有本体实体，包括对象类型、链接类型、操作类型和函数。

为防止 API 名称冲突：

  • 使用具有描述性的显示名称，确保生成的 API 名称不会与现有名称发生冲突。
  • 检查目标环境中的现有对象类型。
  • 如果在安装期间 API 名称发生更改，需准备好更新 Slate 函数代码。

仅支持函数的最新版本。将 OSDK 与 Marketplace 配合使用时，不支持版本化函数。如果应用程序引用了旧版本的函数，请将其更新至最新版本并重新打包应用程序。

不支持的功能

使用 Marketplace 打包 Slate 应用程序时，不支持以下功能：

  • 不支持自定义数据源（包括 Postgres 查询）。API Gateway 是唯一支持的数据源。有关更多信息，请参阅《API Gateway 查询》。
  • 不支持全局样式表。
  • 不支持 Foundry Form 和 Time Series 组件。
