Syncing data to Iceberg tables from Data Connection

Beta

Syncs into Iceberg tables are in the beta phase of development.

You can use Data Connection to sync data from supported external sources directly into Foundry Iceberg tables.

Supported sources

Currently supported source types are:

  • Custom JDBC
  • Microsoft SQL Server
  • Oracle
  • PostgreSQL
  • Snowflake

Iceberg syncs are only supported on the Foundry worker runtime. When ingesting data from a private network, configuring agent proxy egress policies is recommended.

写入模式

Foundry 支持对这些源类型进行非增量和增量的 批量同步，将数据写入 Iceberg。您可以从源概览页面创建新的 Iceberg 同步。

在配置您的同步时，可以选择以下写入模式选项：

  • "覆盖" 写入模式： 用新数据覆盖 Iceberg 表中的所有行，生成一个 overwrite 的 Iceberg 快照。
  • "追加" 写入模式： 将新行追加到 Iceberg 表中，同时保留所有现有数据，生成一个 append 的 Iceberg 快照。
  • "更新" 写入模式： 更新 Iceberg 表中与主键标识列匹配的现有行，并为之前未见过的标识符插入新行。这会生成一个 overwrite 的 Iceberg 快照，仅覆盖受影响的数据文件。

还可以使用基于通配符的自定义 SQL 语句支持更复杂的增量摄入。有关更多详细信息，请参阅 增量同步文档。

Schema evolution on Iceberg syncs is not supported. This means that re-running a sync after modifying the schema of the source table will fail. Consider creating a new sync on the updated table schema in these circumstances.
