入门

在本指南中，我们将创建一个用于更改工单优先级的简单操作类型。

我们将配置提交条件，以确保优先级为 P0、P1 或 P2，并且工单状态为 Open。

前提条件

在本指南中，我们将使用一个 Demo Ticket 对象类型，它有四个属性：

  • Ticket ID
  • Title
  • Priority
  • Status

我们还有两个演示对象可用：

Ticket IDTitleStatusPriority
PDS-123Demo Ticket OneOpenP2
PDS-124Demo Ticket TwoClosedP1

如需，您可以将它们重新创建到您的本体论中，但这不是必需的。

请注意，为了让用户能够执行在操作类型配置中定义的操作，需要额外的配置。如果使用对象存储V2，则用户必须通过切换开关启用编辑功能。如果使用对象存储V1（Phonograph），则必须创建一个回写数据集。请注意，对象存储V1正处于< a href="/docs/foundry/platform-overview/development-life-cycle/">计划中的淘汰阶段；迁移到对象存储V2。

创建一个新的操作类型

我们首先为更改工单优先级创建一个新的操作类型。在本体论管理器中，选择左侧边栏上的操作类型，然后在视图的右上角选择新建操作类型。

创建向导允许您配置操作类型的最重要功能。为您的操作类型输入一个显示名称。接下来，选择更改对象(s)选项，并将其设置为修改。从以下下拉菜单中选择Demo Ticket对象类型，并通过选择添加属性来添加Priority属性。最后，在右下角选择创建。

现在您可以查看操作类型的完整详细视图。您还可以进行其他调整，例如在概述选项卡中添加一个描述，或在规则选项卡中添加其他要修改的属性。

编辑参数

选择表单选项卡以查看参数的概览。Ticket和Priority参数已经根据规则创建好了。

选择 Priority 参数以限制其可取的值。将约束条件从 用户输入 更改为 多选。这将允许您选择此参数可选的值。添加 P0、P1 和 P2 作为选项。如果您现在对一个对象执行操作，可以将工单的优先级更改为 P0、P1 或 P2。现在您将添加提交标准，以限制只能更改开放工单的优先级。

添加提交标准

在侧边栏的 安全与提交标准 选项卡中打开提交标准部分。在 执行 部分中选择 条件 来创建一个新条件。使用 参数 条件模板，在 Ticket 对象参数的 Status 属性上设置一个条件。使用 is 运算符，您可以将工单状态与特定值 Open 进行精确字符串比较。

添加一个失败消息，以便用户可以看到操作失败的原因。您的操作定义现在已经完成，您可以配置它以便在对象资源管理器中的对象视图旁边显示。

将操作添加到对象视图

转到演示工单一并编辑其对象视图。在顶部添加一个新的小部件，并选择操作小部件。在侧边栏中，选择添加项目。从本体论管理器复制操作的RID并粘贴到操作RID字段中。将标签命名为“更改工单优先级”。

默认情况下，操作表单会将每个参数作为操作表单中的字段显示，包括Ticket参数。此外，一个操作并不知道它应该将当前对象填入Ticket参数中。我们将配置操作表单以隐藏票证字段（这样用户无法更改其他票证的状态），并将其值设置为当前对象。 在默认值下，选择添加项。输入Ticket参数的参数ID——在本教程中，我们将其设置为ticket。将值类型更改为环境变量，并选择当前对象。最后，将显示选项更改为隐藏。

解决冲突的用户编辑（操作）和数据源更新

在Foundry本体论中的对象实例可以通过输入数据源和用户编辑/操作进行创建和修改。当一个对象实例（即具有特定主键值的行或对象）同时从输入数据源和用户编辑中接收到数据时，这些接收的值必须通过冲突解决策略透明地进行解决。

有两种冲突解决策略：

  • 策略1：应用用户编辑（默认）
  • 策略2：应用最新值（可能在您的注册中不可用）
