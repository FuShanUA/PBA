手动提升

在某些情况下，贡献者和受限贡献者需要手动将一个 发布版本添加到发布通道中。例如，需要通过手动质量保证测试验证的发布版本无法自动提升。在需要紧急修复的情况下，也可以作为应急机制以加快发布版本的提升。

在本节中，我们将考虑已经发布到Apollo的一个产品的版本14.3.8。

前提条件

将发布版本发布到Apollo

在手动提升发布版本之前，您需要先将该发布版本发布到Apollo。

了解更多关于 将发布版本发布到Apollo 的信息。

确保标签要求得到满足

Apollo 除了自动提升外，还对手动提升强制执行 标签要求。如果一个 Release 不满足 Release Channel 的标签要求，则不能手动将其提升到 Release Channel。

了解更多关于如何 为 Release 添加标签 的信息。

配置一个 Release 提升管道（可选）

在手动提升一个 Release 之前，我们建议您设置一个 Release 提升管道。

进入手动提升工作流

要手动提升一个 Release，您可以从 Product Release 概览中的 操作 下拉菜单中选择 将 release 提升到通道 选项。

你也可以将鼠标悬停在 发行通道节点 和 发行推广管道图 上，然后选择 手动推广到此通道。

进入手动推广工作流的第三种方式是导航到产品概览页面的 发行 选项卡。在发行列表中，点击发行行右侧的快进图标。

定义手动推广策略

一旦进入手动推广页面，你可以查看你的配置好的推广管道，并定义要手动推广到所选发行的发行通道。

复制现有发布

这种推广策略允许您将一个发布添加到所有具有不同发布的发布通道中。选择此选项，然后使用要跟随的目标发布下拉菜单选择要复制的发布。

例如，我们可以使用此选项将版本14.3.8推广到所有具有版本14.3.7的发布通道。

快速直达通道

您可以使用此推广策略，将发布版本推广到推广管道中的特定发布通道。这在需要紧急将发布版本添加到管道末尾的发布通道时非常有用。选择此选项后，使用目标发布通道选择器来定义您希望快速推广的发布通道。

在我们的示例中，我们可以使用此选项将版本14.3.8推广到STABLE_3发布通道。所有突出显示的发布通道都将被推广。

所有发布通道

使用此推广策略，将发布版本推广到推广管道中所有尚未包含所选发布版本的发布通道。

在我们的示例中，我们将版本 14.3.8 提升到管道中的每个发布通道。

自定义通道

此提升策略允许您选择任何发布通道以手动提升，只要满足标签要求。您也可以选择不在配置的提升管道中的发布通道以及在管道中的发布通道。选择此选项后，在目标发布通道搜索栏中搜索一个或多个您想要提升的发布通道。

在本示例中，我们希望将版本 14.3.8 添加到 PRE_STABLE（位于提升管道中），以及添加到 ANOTHER_CHANNEL 和 PILOT（不位于管道中）。

审查手动升级

在配置了升级策略后，选择 下一步。下一步是确认发布的标签要求是否符合目标发布通道，输入手动升级的原因，并查看将受到手动升级影响的 实体 数量。

Apollo 将考虑升级运行较旧版本的实体。例如，当我们手动升级版本为 14.3.8 的发布时，只有当所有约束都满足时，运行低于 14.3.8 版本的实体才能进行升级。

您还可以查看所选发布渠道的标签要求摘要，这些要求可能缺失或不匹配。

当您完成时，选择 提升发布。

在提升管道中，您可以悬停在所选目标发布渠道之前的转换节点上以查看手动提升的状态。

恢复提升评估

如果有在管道中未被手动提升的发布通道，当所有约束条件都满足时，Apollo 将进行 提升评估。

在下面的示例中，手动提升完成后，如果版本 14.3.8 满足 STABLE_7 的标签要求、健康标准和维护窗口，Apollo 将对其是否可以提升到 STABLE_7 进行评估。
