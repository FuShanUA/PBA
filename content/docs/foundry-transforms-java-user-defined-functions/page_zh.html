用户定义的函数（UDF）

Pipeline Builder 支持 Java 和 Python 用户定义的函数（UDF）。 了解更多关于在 Pipeline Builder 中使用 Python UDF 的信息。

如果您无法通过 Pipeline Builder 中现有的转换选项操作数据，想要整合外部 Java 或 Python 库，或者有复杂的逻辑希望在多个管道中重复使用，您可以创建自己的用户定义函数（UDF）。 用户定义的函数允许您在 Pipeline Builder 或代码仓库中运行自定义的任意 Java 或 Python 代码，并且可以进行版本控制和升级。

用户定义的函数仅应在必要时使用，我们建议尽可能使用 Pipeline Builder 或 Java 转换仓库 中的现有转换。

要使用用户定义函数（UDF），注册必须在Palantir的云托管基础设施上进行。目前，Pipeline Builder仅支持行映射和扁平映射UDF，但将来会添加对其他类型的支援。用户定义函数是一项高级功能。请仔细阅读以下文档以了解在您的管道中使用用户定义函数的影响。

创建一个用户定义函数（UDF）

要创建一个用户定义函数仓库，请先导航到您希望保存该仓库的项目。

然后，选择 新建 并选择 代码仓库。在 Foundry UDF 定义 选项中，选择 Map UDF 定义/实现 以通过 UDF 模板启动您的仓库。最后，选择 初始化仓库。

您可以选择在自己的本地环境中进行本地开发（推荐）或直接在 代码仓库 中进行。

设置本地开发环境（推荐）

我们建议在本地创建UDF。按照以下步骤设置您的本地环境。

  1. 要克隆您的代码仓库，请在屏幕右上角选择本地工作，然后复制URL。

  1. 打开命令行界面（CLI），运行git clone <repo url>。

  2. 使用以下命令在您的开发环境中打开项目：

  • IntelliJ IDEA: ./gradlew idea open
  • Eclipse IDE: ./gradlew eclipse open

关键CLI命令：

  • 在编辑UDF定义或Java实现文件后重新运行代码生成文件：
    • IntelliJ IDEA: ./gradlew idea
    • Eclipse IDE: ./gradlew eclipse。
  • 创建一个lockfile以发布到Pipeline Builder：
    • ./gradlew generateEddieLockfile
  • 运行测试：
    • 仅运行testExamples: ./gradlew test --tests examples.ExamplesTest.testExamples
    • 运行所有测试: ./gradlew test

定义用户自定义函数

UDF 是在一份文件中定义的，该文件指定了其名称、输入模式、输出模式和参数类型。将根据此文件生成 Java 类，并且文件名在您发布后将在 Pipeline Builder 中可见。

要创建一个新的 UDF，请在 src/main/resources/udfs/definitions/ 下添加一个名为 <YourUdfName>.yml 的文件，并包含以下字段：

  • name（名称）：用于代码生成以创建描述其输入、输出和参数的类的 UDF 名称。此名称也将用于引用此 UDF 的部署仓库中的 deployment.yml 文件，并在 Pipeline Builder 中作为转换可见。
  • customTypes（可选）：用于定义可以在 inputSchema 和 outputSchema 定义中重复使用的自定义类型的块。有关详细信息，请参阅下面的 自定义类型 部分。
  • inputSchema（模式）：输入行必须遵循的模式，以便针对 UDF 运行。如果运行时可以选择进入此模式，则接受列数多于此模式的数据集。代码生成将根据此模式创建强类型的输入对象。参数将在 Pipeline Builder 中可见。请参阅 模式定义部分 下面详细说明。
  • outputSchema：此UDF输出行的模式。代码生成将根据此模式创建强类型输出对象。有关详细信息，请参阅下面的模式定义部分。
  • arguments：对UDF参数的描述。这些参数将在Pipeline Builder或相应的部署仓库中由用户在构建时指定。代码生成将根据此定义创建一个强类型参数对象。

在你的仓库模板下的src/main/resources/udfs/definitions/Multiply.yml中提供了一个示例定义。有关参数的更多信息，请参阅下面的UDF定义部分。

你可以通过在src/main/resources/udfs/definitions中添加另一个定义文件，在此仓库中定义多个UDF。

实现行映射用户自定义函数

行映射UDF接收一行作为输入，然后对每个输入行输出一行。行映射是默认的UDF类型。

要实现您的UDF创建和转换逻辑，请在src/main/resources/java/myproject下创建两个新的Java类文件，包含以下内容：

  • YourUdfNameMapUdfFactoryImpl.java：负责创建UDF的实例并为执行提供任何参数。
    • 此类应实现生成的YourUdfNameMapUdfFactory接口。
    • 任何UDF参数均可通过args.config()参数获得。
    • create方法将在运行时被调用一次以实例化UDF。
  • YourUdfNameMapUdfImpl.java：实现从输入对象到输出对象的实际转换逻辑。
    • 此类应实现生成的YourUdfNameMapUdf接口。
    • 此逻辑将针对通过转换的每一行被调用一次。
    • 输入和输出类的属性和配置参数可通过同名的方法访问，但采用驼峰式命名。例如，argument_one将具有匹配的argumentOne()方法。

仓库模板包含下面列出的Java类的示例实现。您可以使用这些示例来构建您自己的实现。在编辑后重新运行代码生成以确保没有问题。

  • src/main/java/myproject/MultiplyMapUdfImpl.java

实现一个扁平映射用户定义函数

扁平映射UDF（用户定义函数）将一行作为输入，并且每个输入可以输出零行、一行或多行。

要实现您的扁平映射UDF创建和转换逻辑，请在 src/main/resources/java/myproject 下创建两个新的Java类文件，使用下面的示例。

YourUdfNameFlatMapUdfFactoryImpl.java：负责创建UDF的实例并提供执行参数。

  • 这个类应该实现生成的 YourUdfNameFlatMapUdfFactory 接口。
  • 任何UDF参数都可以通过 args.config() 参数获得。
  • create 方法将在运行时被调用一次以实例化UDF。

YourUdfNameFlatMapUdfImpl.java：实现从输入对象到输出对象的实际转换逻辑。

  • 这个类应该实现生成的YourUdfNameFlatMapUdf接口。
  • 此逻辑将针对通过转换的每一行调用一次。
  • 输入和输出类属性及配置参数可以通过同名的方法访问，但方法名使用驼峰式命名。例如，argument_one 将具有匹配的 argumentOne() 方法。

示例：DuplicateRowsFlatMapUdfImpl.java

Copied!
1package myproject;
2
3import com.palantir.foundry.duplicaterows.config.DuplicateRowsConfiguration;
4import com.palantir.foundry.duplicaterows.DuplicateRowsFlatMapUdf;
5import com.palantir.foundry.duplicaterows.input.InputRow;
6import com.palantir.foundry.duplicaterows.output.OutputRow;
7import com.palantir.foundry.udf.api.flatmap.Collector;
8/**
9 * Implementation for the "DuplicateRows" UDF logic.
10 */
11public final class DuplicateRowsFlatMapUdfImpl implements DuplicateRowsFlatMapUdf {
12
13    private final DuplicateRowsConfiguration config;
14
15    public DuplicateRowsFlatMapUdfImpl(DuplicateRowsConfiguration config) {
16        this.config = config;
17    }
18
19    @Override
20    public void flatMap(Context ctx, InputRow input, Collector<OutputRow> out) throws Exception {
21        OutputRow outputRow = OutputRow.create(ctx.getRowBuilderFactory())
22                .key(input.key())
23                .value(input.value());
24        // Duplicate all input rows by collecting twice
25        out.collect(outputRow);
26        out.collect(outputRow);
27    }
28}

为行映射用户定义函数编写示例

示例提供了关于给定用户定义函数的信息，并可以作为单元测试框架使用。这些示例在将UDF导入到管道时出现在Pipeline Builder中，使理解UDF功能更加容易。示例框架仅支持行映射用户定义函数，对于扁平映射用户定义函数，我们建议编写自己的JUnit测试，这些测试不会在Builder转换文档中显示。

定义一个示例测试用例是可选的，但强烈建议您这样做，因为它将在Pipeline Builder中显示，位于转换文档下，这将使用户更容易理解UDF的功能。如果您不想包含示例，请删除文件夹 src/test 并继续到下一节。

  • src/test/java/examples/registry/MultiplyExamples.java

要为UDF定义新示例，请在 examples.registry 下创建一个实现 Examples 接口的类。请参考 MultiplyExamples.java 了解接口期望。

示例实现中的 name() 方法必须返回与被测试的UDF相同的名称。否则，这些示例将无法正确注册或发布到Pipeline Builder界面。

实际示例应在 examples() 方法中定义。以下是默认 MultiplyExamples 类中定义的 examples() 方法的实现：

Copied!
1@Override
2public List<UdfExample<SettableMultiplicand, Product, MultiplyConfiguration>> examples() {
3    return List.of(UdfExample.<SettableMultiplicand, Product, MultiplyConfiguration>builder()
4            // A unique ID for this example
5            .id(ExampleId.of("baseCase"))
6            // Optional: A description of what this example represents
7            .description("Multiply values by 2.")
8            // Whether this example should have PROMINENT or DEFAULT visibility in Pipeline Builder
9            .visibility(ExampleVisibility.DEFAULT)
10            // The case that this example illustrates (base case, null case, edge case)
11            .category(ExampleCategory.BASE)
12            // The arguments to pass to the UDF that this example illustrates
13            .configuration(MultiplyConfiguration.builder().multiplier(2.0d).build())
14            // The example input rows to the UDF
15            .input(
16                    SettableMultiplicand.create(ctx().getRowBuilderFactory())
17                            .key("key")
18                            .value(1.5d),
19                    SettableMultiplicand.create(ctx().getRowBuilderFactory())
20                            .key("key")
21                            .value(3.0d))
22            // The example output rows (what the input rows should look like after the UDF has been called on them)
23            .output(
24                    Product.create(ctx().getRowBuilderFactory()).key("key").value(3.0d),
25                    Product.create(ctx().getRowBuilderFactory()).key("key").value(6.0d))
26            .build());
27}

部署用户定义函数

您可以通过Pipeline Builder（推荐）或标准UDF部署仓库部署UDF。

发布到Pipeline Builder（推荐）

目前，仅支持在Rubix注册中发布行映射和扁平映射的UDF到Pipeline Builder。其他类型的UDF支持将随后添加。

按照以下步骤将您的UDF发布到Pipeline Builder：

  1. 从底部面板打开任务运行器。然后，选择Java 标签和回车 generateEddieLockfile 以生成一个包含唯一标识符的文件，Pipeline Builder 将使用该文件注册仓库中的每个 UDF。

generateEddieLockfile 命令将根据 UDF 名称返回一个随机的唯一标识符。如果您在发布后重命名 UDF，则代码将作为新的 UDF 发布，而不是覆盖之前的 UDF。同样，如果您删除并重新生成 lockfile，UDF 将获得新的唯一标识符，并作为新的 UDF 注册。所有版本的 UDF，包括旧名称、新名称和新 lockfile，都会出现在 Pipeline Builder 中导入 UDF 时出现的弹出窗口中。

  1. 提交 lockfile 和您的更改。
  2. 用版本号标记您的发布（例如 0.0.1）。
  3. 验证检查是否通过。如果检查失败，UDF 将不会发布到 Pipeline Builder。

如果不存在锁文件，UDF 将不会注册到 Pipeline Builder 中以导入到管道中。要生成一个锁文件，请运行 generateEddieLockfile。如果您在代码仓库中工作，则不需要锁文件。

  1. 一旦检查通过，导航到当前或新的 Pipeline Builder 管道。
  2. 转到 可重用项 > 用户定义的函数 > 导入 UDF。
  3. 从列表中选择您的 UDF，然后选择 Add。您的 UDF 应该出现在 Pipeline Builder 转换选择器中，并可以像其他任何转换一样在您的管道中使用。

在您的 UDF 的第一个版本之后，要部署新更改或修复，包括对定义 YML 的编辑，请重复上面的实现和发布步骤。然后，在您的 Pipeline Builder 管道中：

  1. 转到 可重用项 > 用户定义的函数。
  2. 为您的 UDF 选择 编辑版本，并选择更新后的版本。

发布到部署仓库

按照以下步骤将您的UDF发布到代码仓库：

  1. 选择提交您的更改并打版本标签，以设置发布的版本。
  2. 在标签检查通过后，使用Foundry UDF Definitions > Foundry Streaming UDF Deployment模板创建另一个仓库用于流处理，或使用Foundry UDF Definitions > Foundry Batch UDF Deployment模板用于批量处理。
  3. 按照模板中的说明部署UDF。

在UDF的第一个版本之后，如果您需要部署新更改或修复程序，包括对定义YAML的编辑，请重复上述的实现和发布步骤。然后，更新部署仓库的build.gradle依赖项中引用的标签版本。

8            nullable: false
9            type:
10              type: double
11              double: {}

自定义类型可以在模式中如下引用：

Copied!
1inputSchema:
2  name: "Input"
3  fields:
4    - name: "inputStruct"
5      nullable: false
6      type:
7        type: custom
8        custom: customStruct # The name of the custom type defined above
9outputSchema:
10  name: "Output"
11  fields:
12    - name: "outputStruct"
13      nullable: false
14      type:
15        type: custom
16        custom: customStruct # The name of the custom type defined above

当如上所述在UDF模式中引用自定义结构体时，代码生成不会为InputStruct和OutputStruct创建单独的类。相反，输入和输出对象都引用被引用的自定义类型的相同代码生成类（例如下面的CustomStruct）：

Copied!
1public final Output map(FoundryRowMapUdf.Context ctx, Input input) {
2    // A no-op UDF that passes the input struct field to the output struct field
3    Output output = Output.create(ctx.getRowBuilderFactory());
4    CustomStruct inputStruct = input.inputStruct(); // Returns a `CustomStruct`
5    output.outputStruct(inputStruct); // Expects a `CustomStruct` arg
6    return output;
7}

自定义结构体是使用自定义类型的主要优势；当它们在结构上相同时，不需要额外的工作来将输入代码生成类型转换为输出代码生成类型。

故障排除

本节描述了用户定义函数实现和部署中的一些常见问题，并提供了调试步骤。

代码生成未运行

通常，这是由无法解析的UDF定义YAML引起的。

按照以下步骤进行解决：

  1. 检查代码辅助功能中是否有任何可识别的错误，在红色 Code Assist task failed 弹出窗口中查看。
  2. 确认所有字段都已提供，并且没有拼写错误。您可以对照初始示例定义进行核对。
  3. 如果问题仍然存在，请通过将鼠标悬停在代码编辑器底部的 Code Assist running 消息上并选择 刷新 来刷新代码辅助功能。
  4. 将仓库本地下载，并执行 ./gradlew idea 或 ./gradlew eclipse 命令以查看控制台输出的错误信息。

我可以注销一个UDF吗？

一旦发布，UDF 无法从管道构建器中取消发布。如果您删除锁文件条目并在您的代码仓库中重新运行 generateEddieLockfile 命令，您的 UDF 将获得一个新的 ID，这会导致它在 UDF 部分的管道构建器转换列表中显示两次。

本地环境：不支持的类文件主版本号

如果 ./gradlew idea open 或 ./gradlew eclipse open 出现错误 Could not open proj generic class cache for build file <build.gradle> ... Unsupported class file major version，您可能正在使用与 Gradle 不兼容的 Java 版本。

请按照以下步骤进行解决：

  • 通过运行 ./gradlew --version 检查您的 Java 和 Gradle 版本。
  • 核对您的 JVM 版本是否与 Gradle 文档中列出的 Gradle 版本兼容 Gradle 文档 ↗。
  • 如果您的 JVM 版本不兼容，并且您尚未拥有其他兼容的本地版本，请 下载一个兼容的 Java 版本 ↗。
  • 将您的 JAVA_HOME 设置为 export JAVA_HOME=<jdk install directory>/Contents/Home/。

你现在应该能够成功运行打开命令。

在任务运行器中无法运行 generateEddieLockfile 任务

如果你收到 Failed to start Java server 错误，或者无法在任务运行器中运行该命令，请验证以下内容：

  1. 你的代码仓库模板版本为 0.517.0 或更高。
    • 要检查这一点，请导航到文件编辑器，然后选择 显示设置 > 显示隐藏文件和文件夹。 templateConfig.json 文件将出现在文件列表中。
    • 如果代码仓库不是兼容版本，请从代码仓库视图的最右侧选择 ... > 升级 来生成一个包含最新模板的拉取请求。批准拉取请求后，一旦请求被合并并检查通过，请重新构建你的工作区。

  1. 你的工作区已连接。
    • 如果您的工作区未连接，请重新构建您的工作区。
  2. 您在任务运行器中选择了Java选项卡，而不是Eclipse Java。
