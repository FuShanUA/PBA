发出命令

Apollo 只有在满足所有 实体 的 约束条件（如维护窗口和产品依赖）时，才会对它们执行操作。然而，在稳定性事件等紧急情况下，您可以覆盖 Apollo 和/或 Apollo 计划优先级队列施加的约束，尽快执行计划。这是通过 命令 实现的。

当您发出命令时，Apollo 会在该实体上没有其他活动的计划或命令时立即执行请求的操作。如果需要立即执行此操作，您可以为该实体创建一个抑制窗口以中止当前的计划，并且可以按照 本节稍后所述 中的方法中止当前的命令。

执行命令后，Apollo 将继续对实体评估潜在的计划。需要考虑可以与命令一起使用的其他操作，例如 召回一个发布 和创建一个抑制窗口。如果没有这些操作，使用命令所做的更改可能会被撤销。例如，如果您在没有召回当前发布或创建实体级别的抑制窗口的情况下执行降级实体的命令，Apollo 可能会在满足所有相关约束条件时重新发出升级该实体的计划。

发出命令

实体或环境操作员 可以通过导航到实体的主页并从 操作 下拉菜单中选择 执行命令... 来发出命令。

Apollo 支持三种类型的命令： 安装、强制配置 和 强制配置并版本化。选择一个命令将打开一个对话框，您可以在其中提供发出该命令的理由。一旦命令被发出，Apollo 将在所有 必要条件 满足后尽快执行它。您可以在 实体 页面的 命令 选项卡下监控命令的进度，并查看之前执行过的所有命令。

每个实体只能有一个挂起或正在执行的命令。在上一个命令终止之前，您不能为该实体发出新命令。

命令类型

安装

此命令适用于显示 安装待处理 状态的实体。它将在环境中安装所选的 目标版本 的产品。

卸载

该命令适用于显示待卸载状态的已安装实体。它将从环境中卸载实体。

当您尝试卸载一个实体但卸载被约束所阻止时，请使用此命令。发出命令后，Apollo 将覆盖这些约束并强制卸载实体。

强制配置

此命令类型会根据当前运行的版本，将最新的配置覆盖应用到实体。

强制配置和版本

该命令将根据当前运行的版本，将最新的配置覆盖应用到实体，并更新其版本。

此命令类型会将实体的当前版本更改为所选的目标版本，并应用适用于该版本的最新配置覆盖。

Helm-Chart：资源重启

此命令会对 Helm 图表实体中由 Deployment 或 StatefulSet 控制器管理的所有 Kubernetes ReplicaSets 和 Pods 执行重启操作。此命令类似于 kubectl rollout restart，但通过 Apollo 执行。一旦执行，Kubernetes 将使用定义的滚动更新策略或默认的滚动更新（如果未指定）对控制器管理的所有 Pod 进行更新。

要重启资源：

  1. 导航到你的 Helm 图表实体的 Kubernetes 资源视图。
  2. 选择所需的 Deployment 或 StatefulSet 资源，或者点击资源右上角的按钮。
  3. 选择可用选项中的重启操作。
  4. Apollo 将创建并推荐一个新计划以重新启动资源。
  5. 一旦计划执行，可在活动选项卡中监控进度，并在资源视图中观察新 Pod 的创建。

Secrets

当为支持密钥的实体发出命令时，任何待处理的密钥操作都会包含在命令中并在命令规范中可见。这意味着，例如，如果您发出安装产品新版本的命令，并且该实体有待处理的密钥操作，那么该命令也将包括密钥操作。

命令执行

一旦发出命令，其状态将变为“待处理”，这意味着 Apollo 正在等待满足必要的条件后再执行该命令。通常情况下，命令会短暂地处于“待处理”状态，并会在尽可能早的时机被推荐给代理程序。然而，如果 Apollo 正在进行与该命令冲突的其他工作，则该命令可能无法运行。例如，Apollo 永远不会在一个正处于活动计划目标的实体上运行命令。

如果您发出的命令被阻止，可以通过悬停在状态中的信息图标上来查看原因。如果您需要紧急执行该命令，可以在相关实体上创建抑制窗口以取消任何冲突的计划。

命令约束和优先级

Apollo所做的大部分工作，例如升级和配置更改，都受到一些约束的限制。例如，Apollo只会在维护窗口期间对实体执行升级，并且只会升级处于非错误状态的服务。命令在两个方面具有独特性：

  • 只要命令不与其他活动计划冲突，Apollo将始终尝试执行它。命令不受适用于计划的任何约束的限制。即使实体有活动的抑制窗口，或者实体不在维护窗口中，Apollo也会执行命令。Apollo还会在不健康的实体和运行已召回版本的实体上执行命令。
  • Apollo会优先处理命令而不是计划。如果一个实体既有待处理的命令，又有待处理的升级，Apollo将阻止升级计划并执行该命令。一旦命令完成，计划将被执行。

探索已发出和执行的命令

从实体页面，您可以访问命令选项卡，查看过去六个月对此实体执行的所有命令。

对于每个命令，将显示以下内容：

  • 命令的状态。选择该命令将提供额外信息，例如命令失败的原因。
  • 执行的命令类型。对于“强制配置和版本”命令，还将显示所选的目标版本。
  • 谁发出了该命令以及发出时间。您可以在问号上悬停以查看该命令发出者提供的理由。

取消和中止命令

任何已发出或正在执行的实体命令都会显示在命令选项卡的顶部。

如果你已经发出一个命令，但不再希望它执行，或者它已经开始执行但你想要停止它，你可以取消或中止该命令。如果你在命令开始执行前取消它，该命令将永远不会被推荐给代理。如果你在命令执行过程中中止它，代理将停止执行该命令。Apollo无法保证在你中止一个命令后实体会处于健康状态。然而，Apollo会自动尝试启动任何仍然处于停止状态的节点。

取消一个计划可能不会立即生效，因为Apollo在没有代理同意的情况下无法终止一个计划。Apollo会通知代理该计划应该被中止，但在代理关闭或无响应的情况下，这可能需要很长时间甚至永远不会发生。
