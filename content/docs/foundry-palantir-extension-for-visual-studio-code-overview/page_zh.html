Palantir 适用于 Visual Studio Code 的扩展

VS Code 工作区和 Palantir 适用于 Visual Studio Code 的扩展与 Microsoft 不存在附属关系，也未获得其认可。

Palantir 适用于 Visual Studio Code 的扩展旨在将《代码仓库》中的许多功能集成到 VS Code 中。该扩展目前的功能主要集中在 Python 转换上。

访问扩展

Palantir 适用于 VS Code 的扩展既可以作为 Palantir 平台中 VS Code 工作区的一部分使用，也可以作为本地版本 VS Code 的可下载扩展使用。

如需了解如何在这些选项之间做出选择，请查阅我们关于《本地开发与 VS Code 工作区之间的选择》的文档。

从 VS Code 工作区访问

VS Code 工作区默认提供 Palantir 扩展。请查阅《VS Code 工作区文档》以了解更多信息。

本地访问

Palantir 扩展也可以在本地开发中使用，并且是 Python 转换本地开发的推荐方法。

即使您的注册账户中已提供该扩展，您也可能没有在本地使用该扩展所需的权限。必须由平台管理员启用本地扩展使用权限，他们可以通过《控制面板设置》授予用户访问权限。

为了在本地进行预览，用户必须启用上述控制面板设置，并且对预览输入具有 downloader 权限，即 s3-proxy:datasets-read 和 api:datasets-read。这些是允许用户在本地下载整个数据集的特权操作，应谨慎授予。在无法授予用户此权限的情况下，仍可通过《代码工作区》使用 VS Code 开发者体验，这能保证严格的数据控制。

1. 下载 Palantir 扩展

  1. 导航到 VS Code 工作区，然后选择屏幕右上角 VS Code 按钮旁边的设置图标。
  2. 从下拉菜单中选择 本地 VS Code，以更改默认按钮语言。
  3. 选择更新后的 本地 VS Code 按钮，并按照弹出窗口中的说明下载扩展。

注意：只有在之前从未安装过该扩展的情况下，才需要下载它。

Palantir 适用于 Visual Studio Code 的扩展尚未在 Visual Studio Code Marketplace 中提供。目前只能从 Palantir 平台下载该扩展。

2. 安装扩展

执行以下任一操作来安装扩展：

  • 将 VSIX 文件拖放到 VS Code 的 扩展 侧边栏中，或者
  • 使用命令 Extensions: Install from VSIX...

3. 打开 Palantir 代码仓库

安装 Palantir 扩展后，您应该会在 VS Code 侧边栏上看到 Palantir 徽标。在侧边栏中，您可以执行以下任一操作：

  • 打开您已经克隆的 Palantir 仓库，或者
  • 选择 克隆仓库 来克隆新的 Palantir 仓库。为此，请打开一个 Palantir 仓库并复制 git 远程 URL 链接。
