访问非结构化文件

除了对具有定义的表格模式的Foundry数据集进行操作外，Code Workbook还支持访问数据集中的非结构化文件。这可以用于分析和转换非结构化数据，例如图像和其他类型的媒体、半结构化格式如XML或JSON、压缩格式如GZ或ZIP文件，或者R数据格式如RDA和RDS。

Python中的非结构化文件

读取文件

你可以通过将上游数据集读取为Python transform input来在Python转换中读取文件。此API公开了一个FileSystem对象，该对象允许根据Foundry数据集内文件的路径进行文件访问，从而抽象底层存储。 了解有关FileSystem的更多信息。。其他信息，包括分支和RID（如< a href="/docs/foundry/api-reference/transforms-python-library/api-transforminput/#transforms.api.TransformInput">转换输入文档中详细说明的）也会被公开。

使用输入辅助栏更改输入类型，或在输入选项卡中进行更改。

只能将导入的数据集和持久化数据集作为Python转换输入读取。未保存为数据集的转换不能作为Python转换输入读取。

没有模式的数据集应自动作为转换输入读取。

示例：读取ZIP文件中的CSV文件

例如，以下代码将读取 ZIP 文件内的 CSV 文件，并将 CSV 内容作为数据框返回。

Copied!
1import tempfile
2import zipfile
3import shutil
4import io
5from pyspark.sql import Row
6
7# datasetOfZippedFiles is a dataset with a single zipped file that contains 3 CSVs with the same schema: ["id", name"].
8def sample(datasetOfZippedFiles):
9    df = datasetOfZippedFiles
10    fs = df.filesystem() # This is the FileSystem object.
11    MyRow = Row("id", "name")
12    def process_file(file_status):
13        with fs.open(file_status.path, 'rb') as f:
14            with tempfile.NamedTemporaryFile() as tmp:
15                shutil.copyfileobj(f, tmp)
16                tmp.flush()
17                with zipfile.ZipFile(tmp) as archive:
18                    for filename in archive.namelist():
19                        with archive.open(filename) as f2:
20                            br = io.BufferedReader(f2)
21                            tw = io.TextIOWrapper(br)
22                            tw.readline() # Skip the first line of each CSV
23                            for line in tw:
24                                yield MyRow(*line.split(","))
25    rdd = fs.files().rdd
26    rdd = rdd.flatMap(process_file)
27    df = rdd.toDF()
28    return df

写入文件

可以将数据写入输出 FileSystem。这在写入非表格数据格式（包括图像、PDF、文本文件等）时非常有用。

调用 Transforms.get_output() 来实例化一个 TransformOutput。 了解有关 TransformOutput API 的更多信息。

您只能在保存为数据集的节点中使用 TransformOutput 写入文件。您不能在控制台中使用 TransformOutput 写入文件。

一旦您实例化了一个 TransformOutput 并通过调用 filesystem() 或其他方法使用了它，返回除 TransformOutput 对象以外的任何内容都将被忽略。

示例：写入文本文件或数据集

以下代码是写入文本文件的示例：

Copied!
1def write_text_file(): 
2    output = Transforms.get_output()
3    output_fs = output.filesystem()
4    
5    with output_fs.open('my text file.txt', 'w') as f: 
6        f.write("Hello world")
7        f.close()
8

以下代码是写入数据集并指定分区和输出格式的示例。

Copied!
1def write_dataset(input_dataset): 
2    output = Transforms.get_output()
3    output.write_dataframe(input_dataset, partition_cols = ["colA", "colB"], output_format = 'csv')

R 中的非结构化文件

读取文件

您可以通过在 R 转换中读取上游数据集来读取文件，将其作为 R transform input。 TransformInput 对象被暴露出来，允许根据 Foundry 数据集内的文件路径进行文件访问。 了解有关 FileSystem API 的更多信息。

使用输入帮助栏更改输入类型，或在输入选项卡中更改。

只有导入的数据集和持久化的数据集可以作为R转换输入读取。未保存为数据集的转换不能作为R转换输入读取。

默认情况下，没有模式的数据集应设置为R转换输入。

示例：加载一个RDS

使用以下代码加载一个位于导入数据集中的RDS文件。RDS包含一个R数据框。

Copied!
1RDS_reader <- function(RDS_dataset) {
2    fs <- RDS_dataset$fileSystem()
3    
4    ## The name of the file is test_loading_RDS.rds
5    
6    path <- fs$get_path("test_loading_RDS.rds", 'r')
7    rds <- readRDS(path)
8    return(rds)
9}

1result <- function(zip_file_with_csvs) {
2    fs <- zip_file_with_csvs$fileSystem()
3
4    ## Get the remote path (name) of the zipfile
5    zipfile_name <- fs$ls()[[1]]$path
6
7    ## Get the local path of the zipfile
8    path <- fs$get_path(zipfile_name, 'r')
9
10    # List the zipped files 
11    zipped_files <- as.list(unzip(path, list = TRUE)$Name)
12
13    # For every element on the list, return a dataframe
14    list_of_data_frames <- lapply(zipped_files, function(x){read.csv(unz(path, x), header = TRUE, sep = ",")})
15
16    # Bind all of the dataframes together
17    rbind_df <- do.call(rbind,list_of_data_frames)
18
19    return(rbind_df)
20
21}

写入文件

可以将数据写入输出的FileSystem。这在写入非表格数据格式（包括图像、PDF、文本文件等）时非常有用。

调用 new.output() 来实例化一个TransformOutput。 了解有关 FileSystem API 的更多信息。

您只能在保存为数据集的节点中使用TransformOutput写入文件。您不能在控制台中使用TransformOutput写入文件。

使用以下代码将 R 数据框保存为 RDS 文件。
