管理群组

在侧边栏的 平台设置 部分，选择 群组。选择一个群组，即可在仪表盘视图中查看其详细信息。

您可以查看所选群组的多种信息：

  • 群组名称：建议不要更改群组名称，但拥有管理权限的用户可以执行此操作（参见下文的 重命名群组）。
  • 群组描述：用于描述群组，对组织中所有能够发现该群组的用户可见。
  • 群组 ID：群组的永久唯一标识符。
  • 群组类型：群组的类型；外部、内部或 基于规则。内部域的群组成员资格在 Foundry 中进行管理。
  • 域：身份验证来源，分为外部或内部。对于外部群组，域用于标识管理该群组的提供者。
  • 组织：定义了哪些组织的成员可以看到该群组及其描述。
  • 成员：属于该群组用户的成员。这些成员可以是单个用户，也可以是其他群组。
  • 群组权限：定义了有权管理群组各项事务的用户。管理权限分为两种类型：
    • 管理权限：用户可以授予管理群组各项事务的权限、管理群组成员以及编辑群组的元数据。
    • 管理成员资格：用户可以管理群组的成员，包括成员资格的过期属性。
  • 属性：以键值对格式存储的群组信息，通常供其他 Foundry 服务使用。

您只能查看那些您在群组所属组织上拥有 查看群组成员资格 权限的群组。该权限可以在 设置 > 平台设置 > 组织 中授予，方法是选择目标组织，然后在 组织权限 下选择 管理。系统将显示用户和群组列表以及一个搜索框；对于已添加的用户和群组，使用下拉框启用 查看群组成员资格 选项即可。

成员资格过期

如果您对某个群组拥有 管理成员资格 权限，您可以强制要求加入该群组的新成员资格是临时的。这可以通过配置以下群组属性来实现：

  • 最迟过期时间：所有新成员资格的过期日期必须早于该日期。
  • 最大期限：所有新成员资格必须在规定的期限内过期。

可以同时设置上述一个或两个属性。如果两者同时设置，则允许的最迟过期时间将以两者中限制更严格的属性为准。

此外，如果您拥有 管理成员资格 权限，您可以将设置了成员资格过期属性的临时成员添加到群组中。您可以将这些临时成员添加到未设置 最迟过期时间 或 最大期限 属性的群组中。

对于设有 最迟过期时间 或 最大期限 属性的群组，任何由此产生成员资格请求的 访问请求 都将受到最大过期时间的限制。

成员资格过期通知

当临时群组成员资格过期时，系统会向受影响的用户发送撤销通知。此外，拥有临时成员资格的用户将在其成员资格过期前七天收到提醒通知。但是，如果用户被添加到群组时设置的过期时间不足七天，他们则不会收到提醒通知。

如果用户不希望收到这些通知，可以在平台侧边栏的 账户 > 设置 > 通知 中配置平台通知设置。

自定义审批访问请求表单

如果您可以管理某个在项目中已定义角色的群组权限，那么您可以通过使用以下结构向群组添加属性，来为该项目配置自定义的 访问请求 表单：

  • 属性键：access-request:'group-category-name'
  • 属性值：'intended group value'

一旦您将这些属性添加到群组的 属性 中，它们会自动显示在 请求访问 弹出窗口中，用户可以在访问请求表单中将它们作为选项进行选择。

例如，添加以下群组属性将生成下方的自定义请求流程：

  • 键：access-request: Team
  • 值：blue

对于包含多个群组的项目，自定义访问请求表单尤其实用。它能提供精简直观的访问申请流程，从而优化管理员和用户的体验。

在访问请求表单中排除群组

默认情况下，在项目中拥有角色的所有群组都会显示在访问请求表单中。如果需要隐藏某些群组，可以在项目设置中将它们排除。

要配置需从项目的访问请求流程中排除的群组，请转至 Access panel > Settings > Project access request。

如果您是项目的 数据主人，请选择 Manage 按钮，然后选择要从请求流程中隐藏的群组。如果您不是该项目的 数据主人，则会看到一个 View 按钮，通过该按钮可以查看哪些群组已被排除，但无法进行任何更改。

排除群组属于项目级别的独立设置，具备较强的灵活性。一个群组可以在大多数项目中被排除，但可以包含在特定的项目中。用户可以在这些特定项目中申请访问该群组。如果获得批准，他们将获得该群组拥有角色的所有项目的访问权限。

此功能有助于防止用户误操作申请加入管理员等群组，同时仍允许通过指定的项目获取必要的访问权限。

项目访问

选择 Details 选项卡旁边的 Project access，以查看所选群组的项目访问详情。

Project access 视图允许群组管理员查看群组有权访问的所有项目，以及授予该群组的项目角色。在决定从群组中添加或移除用户时，此视图尤为实用，因为您可以直观看到访问权限将发生怎样的变化。

Show inherited permissions 开关默认处于 开启 状态，它将遍历所有嵌套群组，以查明该群组有权访问哪些项目。如果将此开关 关闭，列表将仅显示直接应用了该群组的项目。

重命名群组

拥有 Manage membership 权限的用户可以重命名群组。当用户重命名群组时，系统会自动执行以下操作：

  • 当前群组将被重命名，群组 ID 保持不变。
  • 系统将使用原始名称创建一个新群组，以支持可能依赖于原始群组名称的应用程序。
  • 原始群组将成为新重命名群组的成员。

设置联系详情

您可以为群组指定联系详情，使其作为联络点。拥有manage permissions（管理权限）的用户可以管理权限。用户可以设定通过 Foundry Issues 还是特定电子邮件联系该群组。

若要在 Project 资源侧边栏中将某个群组设为 Project 联络点，为该群组设置联系详情会非常有用。Project 数据主人可在 Project 概览的 Project point of contact（Project 联络点）部分下选择 Add（添加）来设定 Project 联络点。

应用群组权限

从 Group details（群组详情）仪表板访问 Group Permissions（群组权限）视图。拥有 Manage 权限的用户可通过此部分向群组授予访问权限，而非向单个用户授权，从而让权限管理更透明、更易审计。

为 Project 分配权限时，授予群组权限尤其有用，因为管理员可通过上述 Project 访问标签页查看某群组拥有访问权限的 Project。我们建议在 Project 设置中至少配置三个群组，分别对应一个默认角色：Viewer（查看者）、Editor（编辑者）和 Owner（数据主人）。您应将 Project 默认角色设为 Discoverer（发现者）。

受限视图群组名称策略

创建以Group name（群组名称）作为策略条件之一的受限视图时，您需指定群组所属的域，以便其能正确匹配群组名称。您可在 Platform Settings > Groups（平台设置 > 群组）界面查看群组域，并在受限视图规则编辑器底部更改域名称。

域

用户域

管理员通常在控制面板中设置外部域提供程序（如 SSO、SAML 域或 ADFS）。必要时，Foundry 的平台设置自带身份提供程序的内部实现。在外部身份验证系统不适用的情况下，可使用此内部身份提供程序。

群组外部域

外部域是直接从外部系统（如 ADFS 这类身份提供程序）派生的群组。平台设置配置定义了分配身份提供程序的域。

外部域在 Foundry 中无法修改，处于只读状态。只能在外部系统中执行的操作包括重命名、将用户添加到群组、修改属性以及创建新群组。外部域群组非常适用于 Foundry 中的大多数授权与身份验证相关功能，包括：

  • 分配自主控制与强制控制，以及
  • 启用对平台的二元访问（例如允许或拒绝属于特定群组的一组用户）。

由于外部域群组处于只读状态，用户无法在 Foundry 中请求加入外部域群组。但是，在控制面板中，您可以配置用户尝试请求访问外部域群组时收到的消息。导航至 Control Panel > Authentication > Your SSO > SAML > Manage > Attribute mapping > External group management（控制面板 > 身份验证 > 您的 SSO > SAML > 管理 > 属性映射 > 外部群组管理），为外部域设置自定义消息和 URL。仅与该外部域同属一个域的用户才会看到此自定义消息和 URL。以下示例展示了管理员如何为其内部 Jira 实例添加消息和链接。

所有外部域组都必须分配一个组织。如果未分配组织，无论用户属于哪个组织，该组都将对所有用户可见。

组内部域

在 Foundry 中创建的组会被分配到内部域。

可以在 Foundry 内部修改内部域组。在组界面中可执行的操作包括重命名、向组中添加用户、修改属性以及创建新组。内部域组非常适合用于授予 Foundry 层级功能的访问权限，包括将服务用户账户捆绑到内部域组中，以实现允许列表或拒绝列表的目的（例如，将服务用户账户排除在用户账户过期规则之外）。

如果正在使用外部域组，则务必避免将用户直接分配到内部域组。相反，应将添加或移除用户所在的外部域组嵌套在内部域组中。
