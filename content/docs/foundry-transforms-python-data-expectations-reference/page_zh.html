使用 any 运算符时，每一行都会针对所有的列期望进行单独检查。 例如，要验证列 c1 的值应大于 10 或小于 0：

2

4    E.col("c1").lt(0),

大于或小于


```python
from transforms import expectations as E

E.col('c').gt(number|string)
E.col('c').gte(number|string)
E.col('c').lt(number|string)
E.col('c').lte(number|string)
```


例如：

此期望会忽略空值（即空值会自动通过校验）。若要检查空值，请使用 E.col('col').non_null()。


列比较



```python
from transforms import expectations as E

E.col('c1').equals_col('c2')  # 列 c1 的值等于列 c2 的值
E.col('c1').not_equals_col('c2')  # 列 c1 的值不等于列 c2 的值
E.col('c1').gt_col('c2')  # 列 c1 的值大于列 c2 的值
E.col('c1').gte_col('c2') # c1 的值大于或等于 c2 的值
E.col('c1').lt_col('c2')  # 列 c1 的值小于列 c2 的值
E.col('c1').lte_col('c2') # c1 的值小于或等于 c2 的值
```

时间戳期望


时间戳期望仅适用于 Timestamp 类型的列。目前不支持 Date 类型的列。


静态时间戳比较




将时间戳列中的值与静态时间戳进行比较。静态时间戳可以以 ISO8601 格式字符串提供，也可以以 Python datetime 对象提供。
所有时间戳必须包含时区信息，以避免产生歧义。


警告
严禁将静态时间戳期望与从 `datetime.now()` 获取的时间戳配合使用。虽然这最初可能会产生看似正确的结果，但此行为不受支持，并且可能会在毫无警告的情况下导致错误的结果。此外，如果将静态时间戳期望与从 datetime.now() 获取的时间戳配合使用，Foundry 中的数据健康度和其他消息将无法引用正确的时间戳。请改用相对时间戳比较期望。




已复制！
1from transforms import expectations as E
2
3E.col("timestamp").is_after("2020-12-14T11:32:23+0000")
4E.col("timestamp").is_before(datetime(2017, 11, 28, 23, 55, 59, 342380))
5E.col("timestamp").is_on_or_after("2020-12-14T11:32:23+0000")
6E.col("timestamp").is_on_or_before("2020-12-14T11:32:23+0000")



列时间戳比对


将一个时间戳列中的值与另一个时间戳列中的值进行比对。可以提供可选的偏移量（以秒为单位的整数），该偏移量将加到另一列的值上。


比对逻辑如下：


first_column ($OPERATOR) second_column + offset_in_seconds





已复制！
1from transforms import expectations as E
2
3E.col("timestamp").is_after_col("second_timestamp")
4E.col("timestamp").is_on_or_after_col("second_timestamp")
5# 操作符接受可选的 offset_in_second 参数
6# 检查 `second_timestamp` 是否比 `timestamp` 晚不到一小时
7E.col("timestamp").is_before_col("second_timestamp", 3600)
8# 检查 `second_timestamp` 是否比 `timestamp` 早两个多小时
9E.col("timestamp").is_on_or_before_col("second_timestamp", -7200)



相对时间戳比较


将时间戳列的值与检查运行时（例如构建发生时）的时间加上用户指定的偏移量进行对比。偏移量可以提供整数秒数，也可以提供 timedelta Python 对象。


相对时间戳比较精度
我们期望相对时间戳比较的精度在几分钟以内。这是由于检查实例化或运行的时间存在不精确性。检查运行后，所使用的确切时间戳将显示在 Expectations 检查结果中。


主要提供两种方法：timestamp_offset_from_current_time 和 timestamp_offset_to_current_time。我们提供这两种不同的方法，是为了帮助以自然的方式对相对时间偏移进行推理。因此，我们仅支持正的时间偏移量作为参数。如果需要使用负偏移量，请考虑改用另一种方法。


timestamp_offset_from_current_time


此方法适用于未来的相对时间，即 timestamp - now() 为正值的情况。随后会将该值与提供的偏移量进行比较。所有常规比较运算符均可用于比较。




已复制！
1from datetime import timedelta
2from transforms import expectations as E
3
4# 时间戳值早于未来 1 小时
5A = E.col("timestamp").timestamp_offset_from_current_time().lt(3600)
6
7# 时间戳值晚于未来 2 小时
8B = E.col("timestamp").timestamp_offset_from_current_time().gt(timedelta(hours=2))



timestamp_offset_to_current_time


该方法适用于过去的相对时间，此时 now() - timestamp 为正值。随后会将该值与提供的偏移量进行比较。所有常规比较运算符均可用于此比较。




已复制！
1from datetime import timedelta
2from transforms import expectations as E
3
4# 时间戳值早于当前时间不到 90 分钟
5C = E.col("timestamp").timestamp_offset_to_current_time().lt(5400)
6
7# 时间戳值早于当前时间超过 2 小时
8D = E.col("timestamp").timestamp_offset_to_current_time().gt(timedelta(hours=2))



示例：预期结果


假设检查在 1 月 1 日下午 4 点执行，针对上述检查，不同时间戳值对应的预期结果如下。


   < ------- 过去 ---------------------  现在  -------------------- 未来 ------>
   |    1pm   |    2pm   |    3pm   |    4pm   |    5pm   |    6pm   |    7pm   |
---+----------+----------+----------+----------+----------+----------+----------+
 A |   PASS   |   PASS   |   PASS   |   PASS   |   FAIL   |   FAIL   |   FAIL   |
---+----------+----------+----------+----------+----------+----------+----------+
 B |   FAIL   |   FAIL   |   FAIL   |   FAIL   |   FAIL   |   FAIL*  |   PASS   |
---+----------+----------+----------+----------+----------+----------+----------+
 C |   FAIL   |   FAIL   |   PASS   |   PASS   |   PASS   |   PASS   |   PASS   |
---+----------+----------+----------+----------+----------+----------+----------+
 D |   PASS   |   FAIL*  |   FAIL   |   FAIL   |   FAIL   |   FAIL   |   FAIL   |
---+----------+----------+----------+----------+----------+----------+----------+



检查 B 和 D 中的比较属于严格比较。若要进行非严格比较，请使用 ge 和 le。

条件式


条件式期望接受三个期望并进行验证：


    

  1. 满足 when 期望的数据行也必须满足 then 期望
    2. 

  2. 不满足 when 期望的数据行必须满足 otherwise 期望
    3. 



已复制！
1from transforms import expectations as E
2
3E.when(
4    when_exp,
5    then_exp
6).otherwise(
7    otherwise_exp
8)




例如，当 "myCol" 大于 0 时，"myOtherCol" 必须为 ["a"]，否则 "myOtherCol" 必须为 ["b"]。



```python
from transforms import expectations as E

E.when(
    E.col("myCol").gt(0),
    E.col("myOtherCol").is_in("a")
).otherwise(
    E.col("myOtherCol").is_in("b")
)
```


恒真 / 恒假期望
使用 E.true() 和 E.false() 为条件期望分支的 otherwise 设置简单的默认值。


外部值期望 [实验性]


实验性
外部值期望目前处于实验性开发阶段，您的实例中可能尚未提供此功能。在活跃开发期间，相关功能可能会发生变化。


外部值期望用于验证不同数据集之间的数据关联关系。此类期望涉及连接操作，且评估成本可能极高，请谨慎使用。


引用完整性


该期望用于验证目标数据集中指定列的所有值，是否均存在于外部数据集的指定列中。空值将被忽略。


需匹配的外部列通过数据集引用进行限定，该引用使用另一个数据集的名称创建：E.dataset_ref('other_dataset_name').col('f_col')。


外部数据集必须作为转换操作的输入项（不能仅传入 RID 或路径），且 E.dataset_ref 中的引用应为分配给该数据集的变量名。


列引用的使用方式与 is_in() 类似：




Copied!
1E.col('pk').is_in_foreign_col(E.dataset_ref('other_dataset').col('fk'))
