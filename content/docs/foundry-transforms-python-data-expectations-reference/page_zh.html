数据期望参考

以下是所有可用数据期望的分类列表。

操作符

Copied!
1from transforms import expectations as E
2
3E.true()           # Always passes
4E.false()          # Always fails
5E.all(e1,e2,...)   # Passes when all sub-expectations pass
6E.any(e1,e2,...)   # Passes when any of the sub-expectations pass
7E.negate(e1)       # Passes when the sub-expectation fails

例如：

Copied!
1from transforms import expectations as E
2
3E.all(
4    E.col('a').gt(0),
5    E.col('a').lt(100),
6    E.any(
7        E.col('b').gt(100),
8        E.col('b').lt(0)
9    )
10)

列期望

列期望以 E.col('column_name') 开始。

当使用 any 操作符时，每一行都会单独检查是否符合任何列期望。 例如，验证列 c1 的值应该大于 10 或小于 0：

Copied!
1from transforms import expectations as E
2
3E.any(
4    E.col("c1").lt(0),
5    E.col("c1").gt(10)
6)

大于或小于

Copied!
1from transforms import expectations as E
2
3E.col('c').gt(number|string)
4E.col('c').gte(number|string)
5E.col('c').lt(number|string)
6E.col('c').lte(number|string)

例如：

Copied!
1from transforms import expectations as E
2
3E.col(‘age').lt(120)

此期望忽略空值（意味着空值将自动通过）。要检查空值，请使用 E.col('col').non_null()。

列比较

Copied!
1from transforms import expectations as E
2
3E.col('c1').equals_col('c2')  # Column c1 value is equal to column c2 value
4E.col('c1').not_equals_col('c2')  # Column c1 value is not equal to column c2 value
5E.col('c1').gt_col('c2')  # Column c1 value is greater than column c2 value
6E.col('c1').gte_col('c2') # c1 value is greater than or equal c2 value
7E.col('c1').lt_col('c2')  # Column c1 value is lower than column c2 value
8E.col('c1').lte_col('c2') # c1 value is lower than or equal c2 value

属性比较

Copied!
1from transforms import expectations as E
2
3E.col('c').null_percentage().lt(value)
4E.col('c').null_count().gt(value)
5E.col('c').distinct_count().equals(value)
6E.col('c').approx_distinct_count().equals(value) # Faster version of distinct_count which guarantees a relative standard deviation of the error of max 5%
7E.col('c').sum().gt(value) # Only works for numeric columns
8E.col('c').standard_deviation_sample().gt(value) # Only works for numeric columns
9E.col('c').standard_deviation_population().gt(value) # Only works for numeric columns

例如：

Copied!
1from transforms import expectations as E
2
3E.col("myCol").null_percentage().lt(0.1)    # myCol is less than 10% null values
4E.col("myCol").null_count().gt(100)         # myCol has more than 100 null values
5E.col("myCol").distinct_count().equals(5)   # myCol has 5 distinct values (since version 0.11.0)
6E.col('myCol').approx_distinct_count().equals(5).   # myCol has approximately 5 distinct values
7E.col('myCol').sum().equals(5)              # myCol values sum to 5
8E.col('myCol').standard_deviation_sample().gt(5) # myCol has sample standard deviation greater than 5
9E.col('myCol').standard_deviation_population().gt(5) # myCol has population standard deviation greater than 5

空值

包含于

此期望验证列值是否在批准的值列表中。 对于数组列，请参见 包含于（数组）。

Copied!
1from transforms import expectations as E
2
3E.col('c').is_in(a, b, ...)    # Column value is in given list

除非您将 None 添加到允许的值中，否则此期望会在空值上失败。

正则表达式部分匹配，类似于 pyspark.sql.functions.rlike。

3E.col('c').rlike(regex expression)    # Column value matches a regex expression (partial match)

例如：

Copied!
1from transforms import expectations as E

此期望忽略空值（即空值将自动通过）。要检查空值，请使用 E.col('col').non_null()。

1from transforms import expectations as E

存在

Copied!

2
3E.col('c').exists()  # Column 'c' exists in the output dataframe

此期望检查输出数据框中是否包含指定名称的列。 该列可以是任何类型。

时间戳期望

时间戳期望仅适用于类型为 Timestamp 的列。Date 列目前不被支持。

静态时间戳比较

将时间戳列中的值与静态时间戳进行比较。静态时间戳可以以ISO8601格式的字符串或Python datetime对象提供。 所有时间戳都必须包含时区信息，以避免歧义。

警告

请勿使用静态时间戳期望与从 `datetime.now()` 得到的时间戳一起使用。虽然这可能最初看起来会得到正确结果，但此行为不受支持，可能导致没有警告的错误结果。此外，如果您使用从 datetime.now() 得到的时间戳进行静态时间戳期望，则 Data Health 和 Foundry 中的消息将不会引用正确的时间戳。相反，请使用 相对时间戳比较期望。

Copied!
1from transforms import expectations as E
2
3E.col("timestamp").is_after("2020-12-14T11:32:23+0000")
4E.col("timestamp").is_before(datetime(2017, 11, 28, 23, 55, 59, 342380))
5E.col("timestamp").is_on_or_after("2020-12-14T11:32:23+0000")
6E.col("timestamp").is_on_or_before("2020-12-14T11:32:23+0000")

列时间戳比较

将一个时间戳列中的值与其他时间戳列中的值进行比较。可以提供一个可选偏移量（整数秒数），并将该偏移量添加到其他列的值中。

比较将是：

first_column ($OPERATOR) second_column + offset_in_seconds

Copied!
1from transforms import expectations as E
2
3E.col("timestamp").is_after_col("second_timestamp")
4E.col("timestamp").is_on_or_after_col("second_timestamp")
5# Operators accept an optional offset_in_second argument
6# Check `second_timestamp` is less than an hour after `timestamp`
7E.col("timestamp").is_before_col("second_timestamp", 3600)
8# Check `second_timestamp` is more than 2 hours before `timestamp`
9E.col("timestamp").is_on_or_before_col("second_timestamp", -7200)

相对时间戳比较

Relative timestamp comparison

Compare the values of a timestamp column against the time at which the check is run (such as when the build happens) plus a user-specified offset. The offset can be provided as an integer number of seconds, or as a timedelta Python object.

Relative timestamp comparison precision

We expect the relative timestamp comparison to be precise up to a few minutes. This is due to the imprecision for the time at which the check is instantiated or run. The exact timestamps used will be available after the check runs and presented in the Expectations check result.

Two main methods are provided: timestamp_offset_from_current_time and timestamp_offset_to_current_time. We provide two different methods to help with reasonning against relative time offsets in a natural way. Therefore, we only support positive time offsets as arguments. If you need to use a negative offset, consider using the other method instead.

timestamp_offset_from_current_time

This method is intended for use relative times in the future, where timestamp - now() is a positive value. This value will then be compared to the provided offset. All regular comparison operators are available for comparison.

Copied!
1from datetime import timedelta
2from transforms import expectations as E
3
4# Timestamp values are less than 1 hour in the future
5A = E.col("timestamp").timestamp_offset_from_current_time().lt(3600)
6
7# Timestamp values are more than 2 hours in the future
8B = E.col("timestamp").timestamp_offset_from_current_time().gt(timedelta(hours=2))

timestamp_offset_to_current_time

This method is intended for use with relative times in the past, where now() - timestamp is a positive value. This value will then be compared to the provided offset. All regular comparison operators are available for comparison.

Copied!
1from datetime import timedelta
2from transforms import expectations as E
3
4# Timestamp values are less than 90 minutes in the past
5C = E.col("timestamp").timestamp_offset_to_current_time().lt(5400)
6
7# Timestamp values are more than 2 hours in the past
8D = E.col("timestamp").timestamp_offset_to_current_time().gt(timedelta(hours=2))

Example: Expected results

Assuming the check is running at 4pm on January 1st, here are the results we expect for the checks above for different values of timestamp.

   < ------- PAST ---------------------  NOW  -------------------- FUTURE ------>
   |    1pm   |    2pm   |    3pm   |    4pm   |    5pm   |    6pm   |    7pm   |
---+----------+----------+----------+----------+----------+----------+----------+
 A |   PASS   |   PASS   |   PASS   |   PASS   |   FAIL   |   FAIL   |   FAIL   |
---+----------+----------+----------+----------+----------+----------+----------+
 B |   FAIL   |   FAIL   |   FAIL   |   FAIL   |   FAIL   |   FAIL*  |   PASS   |
---+----------+----------+----------+----------+----------+----------+----------+
 C |   FAIL   |   FAIL   |   PASS   |   PASS   |   PASS   |   PASS   |   PASS   |
---+----------+----------+----------+----------+----------+----------+----------+
 D |   PASS   |   FAIL*  |   FAIL   |   FAIL   |   FAIL   |   FAIL   |   FAIL   |
---+----------+----------+----------+----------+----------+----------+----------+

The comparisons in checks B and D are strict comparison. Use ge and le for non-strict comparison.

Group and property timestamp comparison

Most timestamp comparisons are also available on derived properties of regular dataframes or grouped dataframes.

Copied!
1from datetime import timedelta
2from transforms import expectations as E
3
4# Check that the highest timestamp is after a given static date
5E.col("timestamp").max_value().is_after("2020-12-14T12:23:50+0000")
6
7# Check that the oldest timestamp is less than 1 day in the past
8E.col("timestamp").min_value().timestamp_offset_to_current_time().lt(timedelta(days=1))
9
10# Check that the last date in each category is more than 2 month in the future
11E.group_by("category")
12    .col("timestamp")
13    .max_value()
14    .timestamp_offset_from_current_time()
15    .gt(timedelta(months=2))

Array expectations

Not all expectations work for array-type columns. Array-type columns can only use the specific expectations described below.

Array is in

The is_in expectation also works on columns of array type. For arrays, this expectation tests that arrays only contain the values specified in the is_in clause.

Copied!
1from transforms import expectations as E
2
3E.col('array_col').is_in('a', 'b', 'c') # Any array in the 'array_col' can only contain values 'a', 'b' or 'c'.

Array contains

The array_contains expectation allows you to check that each row of the array column contains a specific value.

Copied!
1from transforms import expectations as E
2
3E.col('array_col').array_contains('a') # All rows must contain value 'a' in 'array_col'.

Array size

The size expectation allows you to check that each row of the array has a specific size.

Copied!
1from transforms import expectations as E
2
3E.col('array_col').size().gt(1) # 'array_col' must have length greater than 1.
4E.col('array_col').size().equals(2) # 'array_col' must have length equal to 2.

Group-by expectations

Group-by expectations begin with E.group_by('column_1', 'column_2', ...). Group-by expectations allow setting expectations on a combination of columns.

Is unique

Copied!
1from transforms import expectations as E
2
3E.group_by('col1', 'col2').is_unique()   # When combined together, the values of the combined columns are unique within the dataset

Row count

Row count expectation tests the row count for each group. If the group_by is empty, this tests against the row count of the entire dataset.

Copied!
1from transforms import expectations as E
2
3E.group_by('col1', 'col2').count().gt(100)    # For each group by 'col1', 'col2', the row count must be greater than 100
4E.group_by().count().lt(100)    # The row count of the dataset is less than 100.
5E.count().equals(0)    # Shorthand for an empty group_by. Assert row count of the dataset is equal to 0.

Group-by property expectations

All property comparison expectations can also be used as grouped expectations.

Copied!
1from transforms import expectations as E
2
3E.group_by('col1').col('value_col').distinct_count().equals(3) # For each group by 'col1', the distinct count of 'value_col' must be equal to 3.
4E.group_by('col1').col('value_col').null_percentage().lt(0.5) # For each group by 'col1', the null percentage of 'value_col' must be less than 50%.

Primary key

Primary key expectations take one or more column names and verify:

  1. Each column has no null values
  2. The combination of columns is unique
Copied!
1from transforms import expectations as E
2
3E.primary_key('c1')             # Column `c1` is unique and not null.
4E.primary_key('c1', 'c2',...)   # Columns {'c1', 'c2',...} are each not null and together are unique
ExpectationDescriptionExample
E.primary_key('c1')Column c1 is unique and not nullE.primary_key('object_id')
E.primary_key('c1', 'c2',...)Columns {'c1', 'c2',...} are each not null and together are uniqueE.primary_key('time', 'event')

For example:

Copied!
1from transforms import expectations as E
2
3E.primary_key('time', 'event')

Schema expectations

All schema expectations start with E.schema().

Use the appropriate data type checks based on the compute engine used for your transform. For the default single-node (lightweight) transforms, use checks with Polars data types ↗. For distributed (Spark) transforms, use checks with PySpark SQL types ↗.

Copied!
1from transforms import expectations as E
2
3E.schema().contains({'col_name':type})  # Dataset columns must contain the listed columns.
4E.schema().equals({'col_name':type})   # Dataset columns match exactly the listed columns (no additions)
5E.schema().is_subset_of({'col_name':type})   # Dataset columns must be a subset of the listed columns.
6                                             # All columns in the dataset must be defined in the check.
7                                             # Columns can be defined in the check without being present in the dataset.

For example:

Copied!
1from transforms import expectations as E
2import polars as pl
3
4E.schema().contains(
5    {
6        'name': pl.String(),
7        'int_list': pl.List(pl.Int32)
8    }
9)
Copied!
1from transforms import expectations as E
2from pyspark.sql import types as T
3
4E.schema().contains(
5    {
6        'id': T.IntegerType(),
7        'name': T.StringType()
8    }
9)

Conditional

Conditional expectations take three expectations and verify:

  1. Rows passing the when-expectation also pass the then-expectation
  2. Rows failing the when-expectation pass the otherwise-expectations
Copied!
1from transforms import expectations as E
2
3E.when(
4    when_exp,
5    then_exp
6).otherwise(
7    otherwise_exp
8)

For example, when "myCol" is greater than 0, then "myOtherCol" must be in ["a"], otherwise "myOtherCol" must be in ["b"].

Copied!
1from transforms import expectations as E
2
3E.when(
4    E.col("myCol").gt(0),
5    E.col("myOtherCol").is_in("a")
6).otherwise(
7    E.col("myOtherCol").is_in("b")
8)
Always true / always false expectations

Use E.true() and E.false() to set simple defaults for the otherwise branch of the conditional expectation.

Foreign value expectations [Experimental]

Experimental

Foreign value expectations are in the experimental phase of development and may not be available on your enrollment. Functionality may change during active development.

Foreign value expectations verify relationships between data in different datasets. These expectations involve joins and can be very expensive to evaluate, so use cautiously.

Referential integrity

This expectation verifies that all values in the selected column of an expected dataset are present in a specified column of a foreign dataset. Null values are ignored.

The foreign column to match against is qualified by a dataset reference created using the name of the other dataset: E.dataset_ref('other_dataset_name').col('f_col').

The foreign datataset must be an input to your transform (you cannot simply pass in a RID or a path), and the reference in E.dataset_ref should be the name of the variable to which it is assigned.

Using the column reference is similar to the usage of is_in():

Copied!
1E.col('pk').is_in_foreign_col(E.dataset_ref('other_dataset').col('fk'))

Cross-dataset row count comparisons

Cross-dataset row count comparisons can be used to compare the number of rows in one dataset with the number of rows in another dataset.

For example, we can check that an output row count is the same as the row count of an input dataset:

Copied!
1E.count().equals(E.dataset_ref('input_dataset_name').count())

The dataset to compare against is qualified with a dataset reference created using the name of the other dataset.

You can use the following operators for dataset row count comparisons:

Copied!
1from transforms import expectations as E
2
3E.count().equals(E.dataset_ref('input_dataset_name').count()) # Equal to
4E.count().lt(E.dataset_ref('input_dataset_name').count()) # Less than
5E.count().lte(E.dataset_ref('input_dataset_name').count()) # Less than or equal to
6E.count().gte(E.dataset_ref('input_dataset_name').count()) # Greater than or equal to
7E.count().gt(E.dataset_ref('input_dataset_name').count()) # Greater than
