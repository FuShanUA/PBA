工作簿

在 Code Workbook 中，您主要交互的资源是工作簿。工作簿用于从 Foundry 导入数据集，并对这些输入数据集进行转换，以实现以下目的：

  • 清洗和连接从外部源导入的原始数据，从而为其他用户生成经过治理的数据集。
  • 分析已处理的数据源，获取有价值的洞察。
  • 训练并应用模型以进行预测分析。
  • 创建参数化可视化图表以在报告中展示。

图谱

工作簿界面的核心组件是图谱。图谱展示了数据在工作簿中流经各个逻辑步骤并最终生成输出的全过程。

输入数据集从 Foundry 的其他位置导入，作为工作簿中的源数据。除输入数据集外，图谱中的每个节点都代表一个转换。转换是一段逻辑代码，能够返回输出数据框或模型，并可能渲染出可视化图表。转换可以保存为派生数据集。将转换保存为派生数据集后，运行该转换会自动将结果写入 Foundry，从而使其能在 Code Workbook 外部使用并方便地与他人共享。

Code Workbook 提供了三种类型的转换：

  • 代码转换允许您编写代码来处理输入并返回结果。
  • 模板转换为转换代码提供了基于表单的界面，简化了代码的复用与共享。
  • 手动输入转换允许用户直接向节点中输入数据。

导航图谱

点击图谱中的任意转换，即可在界面底部打开逻辑面板。在此面板中，您可以查看并编辑转换逻辑，同时查看转换输出。在 Windows 系统上使用 Ctrl+点击，或在 macOS 系统上使用 Cmd+点击，即可在图谱中选择多个转换。

为方便浏览大型工作簿，您可以使用图谱左下角的缩放选项，其中包括缩放以适应按钮，点击后可自动调整图谱比例，将所有转换纳入视野。

与转换交互

与转换交互的主要方式是通过上下文菜单。您可以点击图谱中每个转换右上角的省略号 (...)，或者在图谱或侧边栏中右键点击该转换来打开此菜单。

该菜单提供了一系列实用操作，包括添加新的下游转换、运行与此数据集关联的转换，或从工作簿中删除该转换。对于已保存为数据集的转换，操作选项允许您访问 Foundry 中与该数据集关联的常规操作集合。

若要选择多个转换（在 Windows 系统上使用 Ctrl+点击，或在 macOS 系统上使用 Cmd+点击），您可以在图谱任意位置右键点击，为所有选中的转换打开上下文菜单。此操作有助于添加具有多个输入的新转换，或同时运行多个转换。

将鼠标悬停在工作簿中的数据集名称上，会弹出一个工具提示，显示该数据集在 Foundry 中的完整路径。

路径

您可以在工作簿顶部切换至路径视图。路径视图是一种替代模式，支持在工作簿内进行线性开发。该视图非常适合对单一数据集进行深入分析并执行顺序转换步骤的工作流。

路径可以从导入到工作簿的数据集开始，或者从工作簿中创建的转换开始。在路径视图中，使用左侧边栏可在不同路径之间进行导航。

在路径视图中创建的所有转换也会保存至图中。点击路径节点上的 在图中打开转换 图标（），即可从路径节点导航至图。

在图中，你可以通过观察节点之间连线上的注释，查看哪些节点属于同一路径。将鼠标悬停在注释上即可打开其源路径。

你也可以点击节点上的路径图标，将其在源路径中打开。

全屏编辑器

你可能需要专注于某个转换，并在全屏模式下与其进行交互。在图或路径视图中，点击给定节点上的展开按钮（）即可在全屏编辑器中查看。

在全屏模式下，你可以使用停靠操作或拖放功能重新排列标签页，从而同时查看两个标签页。

要退出全屏编辑器并返回图或路径视图，请使用 Esc 键或选择折叠按钮（）。

面板

代码工作簿有Contents、Global Code和Console三个界面面板，无论在图、路径视图还是全屏编辑器中，始终可以随时使用它们。

目录

在界面左侧，点击 Contents 栏即可打开工作簿目录面板。

在图模式下，Contents 面板会按顺序列出图中的所有转换，以便汇总整个工作簿。默认情况下，转换列表按拓扑顺序排序：输入数据集位于顶部，最下游的转换位于底部。列表中的每个转换旁边都有一个缩放按钮，点击后可将图居中显示在该转换上。你也可以使用空格键缩放并聚焦于选定的转换。点击 Edit Saved Datasets 可更改哪些转换会被保存为数据集。

在路径视图中，Contents 面板会显示工作簿中所有转换的微型图，方便在各节点间快速导航。

全局代码

利用工作簿界面右侧的 Global Code 面板，你可以定义代码（如变量或函数），这些代码将在整个工作簿中该语言的所有代码转换里可用。例如，你可以使用全局代码来定义将在多个转换中使用的常量，或者定义需要反复使用的辅助函数。了解有关如何使用全局代码的更多信息。

控制台

在工作簿右侧，控制台提供了一个 REPL（读取-求值-打印循环），支持对图中的任何数据集进行快速的临时分析。了解有关如何在工作簿各语言中使用控制台的更多信息。

在控制台下方，可使用变量面板为控制台中的特定转换设置输入类型。

如果控制台命令返回一个数据框，你可以使用 + Add to graph 按钮将该控制台命令转换为节点。这样你可以先在控制台中测试逻辑，然后再将其提升为可重复执行的转换。

你可以使用快捷键在 Windows 上按 Ctrl+Shift+Enter，或在 macOS 上按 Cmd+Shift+Enter，将代码从转换直接发送至控制台。
