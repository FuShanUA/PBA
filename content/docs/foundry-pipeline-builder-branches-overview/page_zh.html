分支

使用分支的版本控制在Foundry中广泛使用，是Pipeline Builder（管道构建器）的关键部分。版本控制对于维护健康的管道工作流、支持安全协作以及确保主生产分支保持受保护状态至关重要。

Pipeline Builder将每个独特的管道工作流版本称为一个管道 分支（类似于Git中的分支），其中一个分支作为 主分支。

分支 是管道的一个副本，用户可以在不保存回主管道的情况下进行迭代。Pipeline Builder中的分支类似于Git仓库中的代码分支；用户在其自己的分支中工作以进行编辑和测试更改，而不会对管道产生负面影响的风险。每个管道工作流都从一个 主分支 开始 分支，用户可以在需要协作时从主分支创建额外的分支。一旦用户对其分支中的更改满意，就可以提议将该分支合并到主分支。

全局和Pipeline Builder分支

创建分支时，您可以选择一个Pipeline Builder分支或一个全局分支。全局分支允许您在一个分支上对多个应用程序进行更改，而Pipeline Builder分支则限于您正在处理的管道。请注意，在Pipeline Builder中创建的对象类型在分支上的Ontology Manager中不可修改。

使用代码仓库分支

您可以创建一个与代码仓库分支同名的Pipeline Builder分支，以便Pipeline Builder转换可以从该匹配的分支中读取输入数据集。这使您可以在共享的分支名称上同时对Pipeline Builder的管道和代码仓库的转换进行迭代。您还可以在Pipeline Builder中配置回退分支，以控制当输入数据集未在当前分支上构建时使用哪个分支，其工作方式与在代码仓库中设置作者回退相同。

管理分支

要管理分支，请导航到顶部工具栏并选择管理分支下的设置。

活跃分支

在 活跃分支 选项卡中，可以查看所有当前活跃的分支，或者选择将一个活跃分支归档。归档的分支不会显示在管道图中的分支下拉菜单中，也不能被编辑或使用，除非恢复。要恢复归档的分支，请在 活跃分支 选项卡中选择 查看归档分支。找到要恢复的分支，然后选择右侧的 恢复分支 图标。

分支保护

在此选项卡中，启用 需要提案... 以防止用户对指定分支或分支进行直接更改，从而保护一个或多个分支。此选项要求用户在将其合并到任何受保护的分支之前，必须对另一个分支进行更改。

选择 需要至少一个批准... 以增加另一层保护，在合并到主分支之前需要额外的用户对所提议更改的批准。有效的批准者是具有 Edit 权限的用户，他们没有对所提议的更改做出贡献。在 关于分支保护的文档 中了解更多有关多个受保护分支的信息。

在 常规设置 下，您可以选择：

  • 要求提案更新受保护的分支
  • 在合并到受保护的分支之前要求至少一个批准

提案模板

在此标签页中添加或查看提案模板。在可用的文本框中用Markdown编写新提案，或在 预览 标签页中预览您的文本。如果添加了模板，它将包含在管道中所有新提案的正文中。
