分支

包含分支功能的版本控制在整个 Foundry 中得到了广泛应用，也是流水线构建器的核心组成部分。版本控制对于维护健康的流水线工作流、支持安全协作以及确保主生产分支免受破坏至关重要。

流水线构建器将流水线工作流的每个独立版本称为一个流水线分支（类似于 Git 中的分支），其中一个分支作为主分支。

分支是流水线的一个副本，用户可以在其上进行迭代，而无需将更改保存回主流水线。流水线构建器中的分支类似于 Git 仓库中的代码分支；用户在自己的分支内进行编辑和测试更改，不会对主流水线造成负面影响。每个流水线工作流最初都有一个主分支，用户在需要协作时可以从主分支创建额外的分支。当用户对各自分支中的更改感到满意后，可以提议将该分支合并到主分支。

全局分支与流水线构建器分支

创建分支时，您可以选择创建流水线构建器分支或全局分支。全局分支允许您在单个分支上对多个应用进行更改，而流水线构建器分支的作用域仅限于您正在操作的流水线。需要注意的是，在流水线构建器中创建的对象类型无法在 Ontology Manager 的分支上进行修改。

使用代码仓库分支

您可以创建一个与代码仓库分支同名的流水线构建器分支，以便让您的流水线构建器转换逻辑从该匹配分支读取输入数据集。这使您能够在一个共享的分支名下，同时迭代您的流水线构建器流水线和代码仓库转换逻辑。您还可以在流水线构建器中配置后备分支，以控制在当前分支未构建输入数据集时使用哪个分支，其功能与在代码仓库中设置创作后备分支的方式相同。

管理分支

要管理分支，请前往顶部工具栏，在设置下选择管理分支。

活跃分支

在活跃分支选项卡中，您可以查看当前所有处于活跃状态的分支，或选择将某个活跃分支归档。已归档的分支将不会显示在流水线图的下拉菜单中，除非将其恢复，否则无法编辑或使用。要恢复已归档的分支，请在活跃分支选项卡中选择查看已归档分支。找到要恢复的分支，然后选择右侧的恢复分支图标。

分支保护

在此选项卡中，启用要求提案...可以通过阻止用户直接更改指定分支来保护一个或多个分支。此选项要求用户必须先在单独的分支上进行更改，然后才能将其合并到任何受保护的分支中。

选择要求至少一个批准...可增加一层保护，在提议的更改合并到主分支之前，需要额外一名用户予以批准。有效的批准者是对该流水线拥有编辑权限且未参与提议更改的用户。请在分支保护文档中了解有关多个受保护分支的更多信息。

在常规设置下，您可以选择：

  • 要求通过提案来更新受保护分支
  • 在合并到受保护分支前要求至少一个批准

提案模板

在此选项卡中添加或查看提案模板。可在可用文本框内使用 Markdown 编写新提案，或在预览选项卡中预览文本。若添加模板，该模板将自动包含至流水线中所有新提案的正文中。
