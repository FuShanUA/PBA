状态保存

状态保存是 Workshop 的一项强大功能，模块使用者能够借此保存当前的工作状态，日后既可以重新载入该状态，也能将其共享给其他用户。

借助状态保存，在 Workshop 中构建复杂且耗时的流程将更加轻松，同时也促进了用户间的协作。状态保存的典型应用场景包括：

  • 筛选 Status 属性为“未解决”、Location 属性为“苏黎世”的告警对象，随后保存该状态以备日后查阅。
  • 选中某个特定的目标对象，随后保存当前视图并分享给同事。
  • 在使用 Workshop 原生组件（如文本输入组件或日期输入组件）构建的表单中填写了部分内容，通过保存状态以便后续继续完成数据录入。

保存状态时，Workshop 会留存两部分信息：(1) 已启用状态保存功能的变量的当前取值（即“状态”）；(2) 用户正在查看的当前页面（此为可选操作）。在 Workshop 的编辑模式下，模块开发者可以决定哪些变量需参与状态保存，并配置其他状态保存选项。而在 Workshop 的查看模式下，模块使用者可以根据流程需要，自行保存、打开和分享状态。

如何启用状态保存

在 Workshop 的编辑模式下，开发者可通过以下三个步骤来启用状态保存功能，详情如下：

  1. 打开模块设置面板，开启 Enable State Saving 开关。
  2. 为需要保存状态的变量启用状态保存功能。
  3. （可选）配置该模块已保存状态的允许存放位置和快捷方式。

变量值通过其外部 ID 存储在已保存的状态中。因此，在配置状态保存后修改变量的外部 ID，可能会导致之前配置的状态无法成功重新加载。

修改变量的外部 ID，使得模块配置可以在随时间推移发生变化的同时，依然支持状态保存。例如，如果某个模块最初配置为对象下拉组件（允许用户选择单个对象），随后被替换为对象选择组件（允许用户选择多个对象），只要这些组件输出的对象集使用相同的外部 ID，状态保存就会继续生效。

配置可选设置

在 设置 面板的 状态保存 部分，您可以配置相关设置，以便在已保存的状态中保留用户当前所在的页面。您还可以为该模块已保存的状态设定允许的保存位置和文件夹快捷方式。通过文件夹快捷方式，可以更轻松地确保该模块的所有可共享状态都保存到同一位置。

配置选项

对于状态保存，核心配置选项如下：

  • 启用状态保存： 此开关用于控制是否为当前模块启用状态保存。如果开启，模块使用者可以保存已启用变量的状态以及模块内的当前页面。如果关闭，模块使用者将看不到任何状态保存选项。
  • 显示
    • 状态显示名称： 此字段决定在此模块中向使用者引用单个已保存状态的方式，旨在使屏幕上的语言适应特定应用程序的需求。默认设置为 模块状态。如果设置为 收件箱，模块使用者将在屏幕上看到对 已保存的收件箱 的引用，而不是 已保存的模块状态。
    • 状态显示名称（复数）： 此字段决定在此模块中向使用者引用多个已保存状态的方式，旨在使屏幕上的语言适应特定应用程序的需求。默认设置为 模块状态。如果设置为 收件箱，模块使用者将在屏幕上看到对 已保存的收件箱 的引用，而不是 已保存的模块状态。
  • 数据
    • 状态保存变量： 保存状态将保留模块内任何已启用状态保存的变量的当前值。
    • 页面： 如果启用，保存状态将保留模块内当前活动的页面。
  • 位置
    • 添加快捷方式： 此选项允许构建者配置指向特定项目文件夹的快捷方式，以便将此模块的状态保存到这些文件夹中。可以选择为每个文件夹快捷方式配置自定义显示名称。
    • 用户主文件夹： 如果启用，状态可以保存到用户的私有主文件夹中。
    • 任何 Compass 位置： 如果启用，状态可以保存到模块使用者选择的任何项目文件夹中。

支持的变量类型和组件

通过状态保存，您可以为以下 Workshop 变量类型保留值：

  • 由布尔值、日期、数值、字符串或时间戳值组成的数组
  • 布尔值
  • 日期
  • 对象集
  • 对象集筛选器
  • 数值
  • 字符串
  • 时间戳

输出上述所列变量类型之一的组件也支持状态保存。部分受支持的组件包括：

  • 复选框
  • 日期输入
  • 日期时间选择器
  • 筛选列表
  • 数值输入
  • 对象下拉列表
  • 对象列表
  • 对象表格
  • 字符串下拉列表

限制

不继承被嵌入模块的状态保存配置

嵌入 Workshop 模块不会继承被嵌入模块的状态保存配置。若要将被嵌入模块的变量值保存到状态中，请将所需变量添加到子模块的模块接口中，并在被嵌入模块配置中从父模块传入已保存的状态变量。

状态保存仅对拥有平台访问权限的用户开放

只有为最终用户启用平台访问权限后，该用户才能使用状态保存功能。未获得平台访问权限的用户无法访问状态保存组件。了解更多关于《平台访问限制》的信息。

需保持模块标题可见

要让模块使用者在查看模式下使用状态保存选项，必须确保模块标题可见。如果隐藏了模块标题，即使已为该模块正确配置了状态保存功能，界面中也不会显示状态保留下拉菜单。
