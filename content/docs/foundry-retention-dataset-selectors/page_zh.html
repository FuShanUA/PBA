数据集选择器

每个数据集选择器都可以配置为对符合选择器中标准的数据集进行Select或Exclude操作。例如，Select派生的数据集将缩小过滤器范围，只包括派生的数据集。Exclude文件夹/palantir/finance中的数据集将通过不包含给定文件夹中的数据集来缩小过滤器范围。

一些数据集选择器还包括第二个参数，例如要包含或排除在策略中的文件夹列表或工作器类型。

以下列表描述了在配置保留策略时可用于保留应用程序的数据集选择器。

在以下数据集中

根据其给定的RID选择所有数据集。请注意，即使您重命名了数据集，数据集的RID也不会改变。

了解更多关于识别数据集RID的信息。

需要 1 个参数： list of datasets（由它们的 RID 保存的数据集列表）。

示例

Select 以下数据集：<list of datasets>

以下文件夹中的数据集

选择给定文件夹或通过其给定 RID 标识的 Projects 中的所有数据集。这些文件夹或 Projects 中未来创建的任何数据集也将受此策略约束。

需要 1 个参数： list of folders（由它们的 RID 保存的文件夹或 Projects 列表）。

示例

Select 以下文件夹中的数据集：<list of folders>

派生数据集

如果满足以下条件，数据集被定义为派生数据集：

  • 数据集的主分支包含一个 JobSpec（作业规范）。
  • 数据集有非零数量的数据集输入。

不满足这些条件的数据集，包括原始数据集、从外部源导入的数据集以及从未在主分支上构建过的数据集，将不会被此选择器选中。

接受一个可选的 worker type 列表作为参数： worker type 列表是 JobSpec 中的 workerType 字段中指定的工作机类型（例如，图像下方的 transforms 和 phonograph2-writeback）。如果此字段为空，此选择器将影响所有派生数据集。

示例

Select 使用以下工作器类型生成的数据集: transforms, phonograph2-writeback

在垃圾桶中

选择位于垃圾桶中的数据集。

不接受任何参数。

示例

Select 垃圾桶中的数据集

在特定文件夹中应用具有豁免的广泛策略

防止数据集被保留

您可以通过使用排除模式和在以下数据集中选择器，来防止特定数据集受到给定的保留策略的影响。这会通过它们的RID来排除这些数据集。

请注意，此排除仅适用于您正在配置的策略。在同一 空间 中运行的其他保留策略——包括推荐（平台管理）策略和任何其他自定义策略——仍可能对被排除的数据集进行标记。为了完全保护一个数据集，需要审查所有可能适用于它的策略。
