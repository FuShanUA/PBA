使用项目引用

项目引用提供了一种机制，允许拥有较高权限的用户（通常是数据集或流水线的数据主人）允许其在其他项目中发现和使用他们的数据。中间项目可用作权限边界或发现中心，在其中拥有流水线某一部分的用户可以解密、筛选或以其他方式分发可共享版本的数据集，并将其提供给下游消费者。

项目引用通过在数据集从项目导出或导入到项目时进行明确确认，为在项目之间移动数据增加了一层更严格的审查。

操作说明

  1. 代码编辑器会提示用户，通过转换生成的输出必须位于代码库的项目范围内。例如，如果您的代码库位于名为“数据清洗项目”的项目中，则代码库的输出只能位于“数据清洗项目”中，而不能位于任何其他项目中。
  2. 当存在来自项目范围之外的输入数据集时，代码编辑器会标记该数据集必须作为项目引用添加。通过点击灯泡图标，可以将该输入数据集添加为项目引用。
  3. 也可以从项目与文件中添加对项目外部数据集的引用。
  4. 如果代码库的语言包已过时，则在更新之前无法强制执行项目范围。代码库会标记需要更新这些内容，点击出现的更新按钮即可解决此问题。
  5. 您只能将项目范围之外的数据集作为项目引用添加为输入数据集，而不能作为输出数据集。尝试输出到项目范围之外的数据集将触发 AccessOutsideProjectDenied 错误。

项目引用与权限

要引用资源，您必须对该资源拥有 compass:import-resource-from 权限（通常从查看者角色扩展而来），并且对目标项目拥有 compass:import-resource-to 权限（通常从编辑者角色扩展而来）。可以使用自定义角色来定制这些权限。

用户需要拥有对代码库中引用的数据集相关的所有标记的访问权限，才能修改该代码库中的代码。
