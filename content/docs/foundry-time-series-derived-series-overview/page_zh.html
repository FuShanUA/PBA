派生系列

派生系列允许用户保存并复制对本体论（Ontology）中时间序列应用的计算和转换。通过将这些数据作为Palantir资源保存，派生系列可以被共享并保存回本体论作为时间序列属性。一旦进入本体论，派生系列的行为就像任何其他时间序列属性一样，但会实时进行计算，从而无需管理或存储派生数据，也不需要在平台上重复这些计算。

从 派生系列 选项卡中探索、创建和管理派生系列，该选项卡位于 时间序列目录 中。

派生系列类型

派生系列可以是模板化的或单个的。

模板化派生系列

模板化派生系列基于一个根对象类型进行模板化，因此必须在一个单一的根对象上运行。模板化派生系列功能强大，因为它们允许您快速在给定类型的全部对象之间复制逻辑。例如，如果许多假设的 Machine 对象具有温度传感器，那么模板化派生系列将使您能够快速在 Machines 上创建一个滚动平均温度传感器。

单个派生系列

单个派生系列不用于模板化，并且可以在多个对象上运行。单个派生系列允许您创建不受所有输入都来自一个对象限制的逻辑。

要求

以下部分解释了在创建派生系列时必须遵循的要求。

时间序列对象类型

时间序列本体论（Ontology）是创建派生系列的先决条件。派生系列是在 时间序列对象类型 上创建并存储的，这些对象类型可以是 根对象类型上的时间序列属性 或者 传感器对象。有关更多信息，请参阅 时间序列本体论文档。

逻辑要求

派生系列逻辑管理由 Quiver 提供支持。Quiver 的大多数时间序列操作都适用于派生系列；请查看 支持的操作列表，以了解派生系列逻辑的相关内容。

模板派生系列的逻辑

模板派生系列的逻辑必须包含 一个根对象。根对象上的时间序列属性以及与根对象关联的传感器对象上的时间序列属性可以用于该逻辑。有关根对象和传感器对象类型之间区别的更多信息，请参阅 时间序列对象类型。

可以通过 时间序列属性 卡片从根对象访问时间序列属性和关联的传感器对象。

在此示例中，Temperature 上的 Machine 1 时间序列属性以及 Inlet 和 Outlet pressure 上与 Machine 1 关联的传感器都可以访问。

单个派生系列

单个派生系列逻辑可以包含任意数量的传感器或根对象，只要它们都存在于同一个本体论中。

权限要求

要保存一个派生系列，需要在绑定的对象类型上具有< a href="/docs/foundry/object-permissioning/ontology-permissions-legacy/#create-new-resources-with-ontology-roles">对象类型编辑权限（对象类型编辑权限）。

自动保存要求

与传感器对象类型绑定的派生系列可以通过操作类型< a href="/docs/foundry/time-series/derived-series-create/#6-configure-ontology-saving-options">自动保存（自动保存）到本体论。派生系列的自动保存会安排本体论操作来根据本体论中请求的内容管理相关传感器对象。派生系列也可以< a href="/docs/foundry/time-series/manual-ontology-saving/">手动保存到本体论（手动保存到本体论）。

用于自动本体论保存的传感器对象类型要求

  1. 用于自动保存到本体论的传感器对象类型的主键必须是String类型。
  2. 传感器对象类型必须使用 对象存储V2 存储；这是操作写入时间序列属性所必需的。
  3. 传感器对象类型必须启用编辑功能。
  4. 对于模板化的衍生系列，根对象类型和传感器对象类型之间必须有一个 一对多基数关联，其中根对象类型位于“一”端。

用于自动本体论保存的操作类型要求

如果未满足以下任何要求，则无法选择操作类型以进行自动本体论写入：

  1. 每个操作类型必须有一个规则。
  2. 操作类型参数不能使用限制可提供值的约束。同样，操作类型参数不能使用导致值约束的覆盖。
  3. 操作类型不应包含未使用的参数。如果一个参数未被使用，则不能将其配置为必需。
  4. 未提供值的属性被视为未映射。对于未映射的传感器对象属性的参数必须配置为“非必需”。对于模板派生系列，不属于根对象类型的传感器对象类型上的外键属性始终是未映射的。
  5. 操作类型的提交标准不能使用基于参数的条件。

您必须创建三个独立的操作类型：创建、修改和删除。这些操作类型的规则如下所示。

Create object 操作类型

规则： 每个属性类型使用相同类型的参数，并编辑对象类型的全部属性。如果您使用本体论管理器来配置操作类型，您必须从左侧的表单选项卡中手动为您的主键创建一个字符串参数。

Modify object 操作类型

规则：与Create object 操作类型类似，Modify object 操作类型应使用与相关属性类型相同的参数，并且应编辑除了主键之外的所有属性。

Delete object 操作类型

规则： 配置 Delete object 操作（操作）类型以删除传感器对象（对象）类型。无需进一步的属性或参数配置。

权限（权限）要求

要使用自动本体论（本体论）保存，您必须满足 操作（操作）类型提交条件。

此外，您必须能够查看根对象类型和传感器对象类型的对象。当使用受限制视图支持的对象类型时，请注意自动保存的使用，因为派生系列的本体论状态是根据用户可以查看的对象计算的。对于模板化的派生系列，用户应能够访问整个根对象范围以及相关的传感器对象。同样，单个派生系列只能由能够查看相关单个传感器对象的用户进行更新。
