Transforms

Transforms 是一段逻辑，接收一个或多个输入并返回单个输出。输入和输出可以是 Foundry 数据集或模型。

Code Workbook 目前支持三种类型的 Transforms：

  • 代码 Transforms 允许用户使用所选语言编写任意逻辑。
  • 模板 Transforms 将代码封装在简单的、基于表单的界面背后，方便在不同工作簿和用户之间复用。
  • 手动输入 Transforms 允许用户将数据直接输入到节点中。

添加 Transforms

在 Code Workbook 中，有多种方式可以添加新的 Transforms。

使用图表左上角的新建 Transform 按钮，可以选择要添加的 Transform 类型。例如，选择 Python 代码来添加一个新的 Python 代码节点，或选择模板来打开模板浏览器。

或者，将鼠标悬停在图表中现有的 Transform 上，点击出现的蓝色 + 按钮以打开选择下拉菜单。新创建的 Transform 将作为所选 Transform 的子节点。

编辑 Transforms

编辑名称

Code Workbook 中的 Transforms 可以保存为数据集。阅读更多关于数据持久化的内容。

未保存为数据集的 Transforms 会通过一个特定于工作簿的别名进行标识，你可以在代码中引用该 Transform。对于输入数据集以及保存为数据集的 Transforms，Code Workbook 会为每个数据集显示两个不同的名称：底层 Foundry 数据集的名称，以及一个特定于工作簿的别名。

使用别名有以下几点优势：

  • 你可以采用直观符合工作簿逻辑的方式来引用数据集。例如，如果在对应不同更新频率的不同文件夹中有两个名为“Titanic”的数据集，你可以将其中一个别名为 titanic_historical，另一个别名为 titanic_daily_feed。
  • 你可以随意编辑数据集别名，而不会影响 Foundry 中其他位置使用相同数据集的用户的原始数据集名称。
  • 你可以在某个分支上更改数据集的别名，并跨分支合并此更改。

要编辑数据集名称或其别名，请点击逻辑面板中的名称，输入新名称，并在编辑完成后按回车键。如果你更改了数据集名称或别名，系统会提示你是否需要自动更新另一个名称。

添加输入

点击任意 Transform 会打开逻辑面板。要添加输入，请点击顶部输入栏的 + 按钮，然后点击所需的输入即可。

要添加多个输入，请使用 + 旁边的按钮。

代码

代码 Transforms 会显示为一个代码编辑器，其中预置了当前工作簿所使用语言的基本结构。有关每种语言编写代码的具体细节，请参阅语言文档。

全局代码

全局代码面板允许你定义在工作簿中每个代码 Transform 执行之前运行的代码。全局代码面板可用于定义变量和函数，这些变量和函数将在工作簿的所有 Transforms 中可用。

模板

对于模板 Transforms，逻辑面板会显示一个用于编辑模板参数的表单。

  • 点击数据集参数后，点击图表中的任意数据集即可将其设置为该参数的值。
  • 列参数取决于指定的输入数据集。指定数据集后，您可以从下拉列表中选择列名。
  • 对于整数和字符串等值参数，您可以输入自定义值。

手动录入

点击图表上的手动录入按钮，即可添加手动录入节点。

在手动录入选项卡中输入数据。目前支持的列类型包括双精度浮点数、整数、布尔值和字符串。最多支持 500 行。

您可以通过复制和粘贴，轻松从其他来源导入数据。

运行转换

在底部面板选择运行，即可执行选定的转换。如需运行所有下游转换，请点击箭头图标并选择运行所有下游转换。

运行转换需要工作簿的编辑权限，同时还需要访问工作簿所用数据集涉及的所有限定标记。

系统还提供了多个快捷键，方便您快速运行代码。输入 ? 即可查看快捷键列表。

批量运行多个转换有两种方式：

  • 按住 Ctrl + 单击或 Cmd + 单击，在图表中选中多个数据集。然后在图表任意位置右键单击，并从菜单中选择运行 N 个数据集。
  • 点击顶部栏的设置图标 ()，然后选择运行所有转换。

转换运行期间，所有打印输出都会以流式传输到日志选项卡。这样一来，您可以在转换过程中打印输出，并在转换仍在运行时查看结果，这对于调试或评估逻辑执行时间非常有用。

数据保存到数据集后，代码中应用的任何数据排序都可能无法保留。换言之，如果您运行了一个对数据进行排序的转换，那么在以数据集形式查看该数据时，其排列顺序可能已发生改变，但转换中的其他操作均已生效。

选项卡

在图表或全屏编辑器中操作时，可以使用节点底部的选项卡来导航转换的输入和输出。

输入

使用“输入”选项卡可以查看转换的输入项、添加额外输入或更改输入类型。

警告

与其他基于 Spark 的应用程序一样，Code Workbook 在读取数据集时默认不会保留行顺序。如果您的分析需要特定的行顺序，请在代码中对数据进行排序。

预览

在代码工作簿中运行转换时，系统会计算 50 行的预览数据，以便查看结果的大致形态。在数据集完成写入 Foundry 之前，此预览会显示在预览选项卡中。数据集持久化到 Foundry 后，预览选项卡将更新为完整结果。

可视化

根据转换中使用的语言，你可以使用绘图库返回可视化结果，这些结果将传播到代码工作簿用户界面。有关如何使用每种语言执行此操作的详细信息，请参阅《语言文档》。

生成的可视化结果默认显示在图表中，也可在底部的“可视化”选项卡中查看。

通过上下文菜单，你可以在图表模式下切换表格和图表视图。

在可视化选项卡中，你可以将图表下载为图片或将其添加到报告中。这些操作也可以通过上下文菜单执行。

日志

查看运行转换时的打印语句（标准输出）、警告和错误（标准错误）。如果存在标准错误，系统会自动选中日志选项卡，并出现一个小的提示框通知你有警告/错误输出。

描述

更新数据集的描述。请注意，更新描述会跨所有分支进行更新——该描述也将显示在“打开数据集”视图中。

你可以使用 Markdown 来排版描述，并在图表中通过右键（操作）菜单启用“显示描述”来查看它们。要显示工作簿中所有节点的描述，请点击标题栏中的工具齿轮图标，然后选择“显示所有描述”。

模型

返回模型的转换会自动作为模型写入 Foundry，从而允许使用 Foundry ML 接口。有关如何使用每种语言处理模型的详细信息，请参阅《语言文档》。

非确定性转换

某些转换具有非确定性——即使输入数据和转换逻辑相同，它们也会返回不同的结果。一些可能导致非确定性转换的函数包括调用当前时间戳、行号、单调递增 ID，以及在包含重复值的列上进行排序后的转换。

在 Code Workbook 中，运行转换操作会启动两个作业：一个作业用于计算 50 行预览数据，另一个作业则对完整数据集执行转换，并将结果写入 Foundry。如果存在非确定性转换，你在 50 行预览中看到的结果可能与 Foundry 数据集中的结果不一致。

在转换过程中运行非确定性代码时，数据框（dataframe）不一定会被物化。例如，假设有一个非确定性转换“转换 A”，它是“转换 B”和“转换 C”的父节点。如果你在 Code Workbook 中分别运行每个转换，每个数据集将按顺序写入 Foundry，从而确保“转换 B”和“转换 C”之间的结果保持一致。然而，如果你在 A 上选择“运行所有下游节点”，由于运行 B 和 C 时 A 不一定被物化，你可能会看到冲突的结果。从数据血缘关系中运行构建也可以避免这种情况，因为每个数据集都将被物化并按顺序计算，将其转换为 Pandas 数据框也能达到同样的效果。

导致非确定性转换的函数示例包括 current_date()、current_timestamp() 和 row_count()，不过其他函数也可能导致非确定性。有关 Spark 计算转换时可能导致意外行为的更多信息，请参阅《Apache Spark 文档 ↗》。
