Pipeline Builder 中的单元测试

通过单元测试来提升 Pipeline Builder 中数据管道的可靠性。这些测试在调试、检测破坏性变更方面是极具价值的工具，最终能够确保构建出更高质量的数据管道。

什么是单元测试？

与代码中的单元测试类似，Pipeline Builder 中的单元测试用于验证数据管道逻辑在给定预定义输入时能否产生预期输出。单元测试包含以下部分：

  • 测试输入
  • 转换节点
  • 预期输出

测试输入和预期输出通过手动录入的表来创建，但您也可以通过复制粘贴来加快创建速度。您可以在主 Pipeline Builder 工作区中选择需要进行测试的转换节点。有关创建单元测试的更多信息，请参见下文。

创建单元测试

  1. 在主工作区的右侧面板中，选择 Unit tests 图标。



  2. 在屏幕中央选择 Create new test，或在右上角选择 New test。这将在工作区顶部打开一个对话框，提示您选择相关节点。



  3. 选择所有相关节点后，点击 Start。



    随后将进入单元测试配置窗口。

    • 黄色节点对应先前选定的转换节点。
    • 绿色节点对应测试输入。
    • 蓝色节点对应测试输出。

    对于每个单元测试，您都必须填写输入和输出数据。



  4. 双击节点即可填写输入数据或预期输出数据。这将跳转至以下页面：



    在左侧选择：

    • Reuse schema: 将输出模式设置为与所连接表的模式相匹配。
    • From dataset: 使用现有数据集的模式。
    • Add column: 手动录入数据模式。

    设置好模式后，在中间的表格中填写数据行，然后依次点击 Apply 和 Back to graph。



  5. 对所有输入和输出数据集重复此步骤。

完成后，您可以在右侧面板中看到手动录入的数据，其中详细展示了每个表的行数和列数。

运行单元测试

对于每个单元测试，您都可以在右上角选择 Run test。

测试运行完成后，您可以在下方查看测试结果。如需查看具体的表结果，请选择 View test result。

这会在屏幕底部展开一个视图，展示预期输出和实际输出。

完成单元测试的编辑和查看后，您可以点击右上角的 Close unit test。

删除单元测试

要删除单元测试，请将其选中，并点击右上角的三个点以打开选项菜单。选择删除测试用例。

编辑现有单元测试

点击单元测试图标，查看管道中的单元测试列表。点击铅笔图标即可编辑选中的单元测试。

要更改单元测试中选中的测试转换，请使用重新选择按钮。系统将跳转回选择页面。

如果作为测试转换已包含在单元测试中的节点之间插入新节点，这些新增节点会自动出现在现有的单元测试中。

要修改任何测试输入或预期输出，可以直接双击图视图中的节点，或点击右侧面板上的铅笔图标。

完成单元测试编辑后，点击右上角的关闭单元测试即可返回主图。

提案中的单元测试

对单元测试的任何更改也会显示在提案页面的左侧面板中，位于单元测试标签页下。

在提案页面上，您会看到单元测试成功部分。Pipeline Builder 会在合并提案前检查单元测试是否通过。

流处理中的单元测试

对于流式单元测试，测试输入数据的每一行都需要一个额外的 ordering 长整型字段。

ordering 列是流式单元测试必需的元数据列，用于控制数据行发出的全局顺序，但不会影响测试数据的实际内容或模式。在单次测试的所有测试数据源中，每一行的 ordering 值必须是唯一的长整型数值，数据源将按照 ordering 值从低到高的顺序依次发出数据行。

为了从流式转换中获得确定且符合预期的输出，排序至关重要，特别是对于连接和联合操作。
