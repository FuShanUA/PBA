隐藏代码仓库

Code Workbook 通过使用分支提供轻量级的版本控制。此外，每个工作簿背后都有一个特殊的隐藏代码仓库。该仓库不仅作为代码工作簿中所写代码的安全备份，还会暴露工作簿上所有代码变更的历史记录。

要访问工作簿的隐藏代码仓库，请点击 Code Workbook 界面右上角的齿轮图标菜单，然后选择打开隐藏代码仓库。

特殊属性

工作簿的隐藏仓库始终具有以下属性：

  • 仓库为只读： 仓库内容只能查看，绝不能在仓库中直接更新。更新仓库的唯一方法是在工作簿中进行代码更改。
  • 仓库默认隐藏： 只能通过 Code Workbook 界面发现它。
  • 仓库存储三个独立的文件： pipeline.R、pipeline.py 和 pipeline.sql 文件各自包含相应语言转换后的工作簿所有代码。
  • 仓库保存完整的变更历史： 在隐藏仓库的分支选项卡下，可以查看每个分支的工作簿代码变更完整历史。
  • 仓库包含一个隐藏的 workbook.yml 文件： 该文件存储有关工作簿的基本元数据。

在工作簿分支上进行的每次代码更改，都会自动向隐藏代码仓库中的相应分支创建一个新的提交。

## 恢复丢失的代码

通过定期保存工作簿的备份，隐藏代码仓库是恢复丢失或误删代码的推荐方式。要查看特定工作簿分支的历史记录，请打开隐藏仓库顶部的“分支”选项卡，选择目标分支（如 `master`），然后选择你想要检查代码变更的提交记录。随后，你可以将代码复制并粘贴回工作簿中。
