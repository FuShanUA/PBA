创建或检索对象集

平台选项卡下的对象集面板支持以下操作：

  • 无需任何额外的服务配置，即可创建带或不带过滤条件的对象集。
  • 通过模板点号记法（如 {{s_object_set1.data.property1}}）直接访问对象集中的对象。
  • 使用 {{s_object_set1.objectRids}} 访问对象 RID。

共有两种模式：

  • 构建器模式：支持通过图形用户界面 (GUI) 构建对象集。
  • RID模式：支持通过引用 对象集 RID 或 版本化对象集 RID 来构建对象集。

对象集过滤器构建器模式

若要创建对象集 s_object_set1，请选择基础对象类型，随后可根据需要添加带有 AND/OR 逻辑的过滤条件。过滤字段会自动适配对象的属性数据类型。例如，数值型属性将采用数值范围作为过滤条件。你还可以在每个过滤层级中嵌套独立的 AND/OR 过滤器，从而将对象限定在特定的结果范围内。

你也可以在过滤字段中使用 Slate 的 Handlebars 语法，从而基于动态变化的输入对对象集进行参数化处理。

过滤字段还支持通过多词过滤器和通配符过滤器进行模板化处理：

  • 若要使用多词过滤器进行模板化，可以直接传入多个词（例如 ["value1", "value2"]）。
  • 或者，你可以传入一个 Handlebars 模板，直接引用某个输出多个词的函数。
    • 例如，如果 {{f_my_filters}} 返回 ["value1", "value2"]，则可以直接在对象集编辑器中使用 {{f_my_filters}}。

对象集结果将以表格形式返回至 data 字段下，同时所有对象 RID 会包含在 objectRids 字段中。

对象集结果限制

对象集过滤器还允许你为返回的对象数量设定上限。该机制在概念上类似于 SQL 的 LIMIT 子句，有助于提升测试与迭代的效率。

若限制值小于结果总数，可使用 getNextPage 和 getPreviousPage 事件对结果进行分页浏览。这在返回大量结果时能够显著提升性能表现。

聚合

在 平台 选项卡的对象集编辑器中，可对对象集执行简单的聚合计算。若要创建聚合，请在 输出选项 下开启 聚合 开关，并选择 添加聚合。聚合操作将基于上方定义的对象集结果执行。

聚合操作支持以下聚合类型之一：计数、平均值、求和、最大值、最小值或基数。在执行计数以外的聚合计算时，必须选定一个数值型属性。

通过选择 添加聚合，即可基于同一对象集计算多个聚合指标。启用聚合功能后，预览界面将仅展示聚合结果。不过，在函数、微件及 Slate 的其他模块中通过 Handlebars 引用该对象集时，细粒度的对象数据依然可正常访问。

多维度聚合

若要对对象集数据开展多维度聚合（例如引入“分组”或分段机制），需编写《函数》，在为最终生成的数据桶计算指标前，对数据进行分组与分段处理。

这些函数将返回《聚合类型》，Slate 会将其解析为平行列表数据结构，以供配置图表及其他 Slate 微件使用。

《关于如何调用 Foundry 函数的文档》详细介绍了源自 Slate 的此类交互机制。

对象集排序

生成的对象集可按任意参数进行升序或降序排列。排序支持配置多个属性，且排序属性的生效顺序为自上而下。所设定的数量限制仅在排序操作完成后生效。

对象集 RID 模式

在对象集 RID 模式下，Slate 的对象集面板能够接收 对象集 RID 以及 带版本的对象集 RID，并将其解析还原为 Slate 的数据数组格式。

这意味着，除了使用 Slate 的参数化对象集构建器外，您还可以灵活地传递 RID。
