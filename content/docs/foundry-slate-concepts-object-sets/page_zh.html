创建或检索对象集合

平台选项卡下的“对象集合”面板允许您：

  • 创建一个对象集合，可以带过滤器也可以不带过滤器，无需任何额外服务配置。
  • 通过模板点表示法（例如 {{s_object_set1.data.property1}}）直接访问对象集合中的对象。
  • 使用 {{s_object_set1.objectRids}} 访问对象的 RID。

有两种模式：

  • Builder 模式，允许您通过 GUI（图形用户界面）构建对象集合，以及
  • RID 模式，允许您通过引用 object set RID 或 versioned object set RID 构建对象集合。

对象集合过滤器构建模式

要创建一个对象集 s_object_set1，请选择基础对象类型，然后可选择性地添加带有AND/OR逻辑的过滤器。过滤器的字段将考虑对象的属性数据类型。例如，数值属性将使用数字范围作为过滤器。您还可以在每个过滤器级别中进一步嵌套单独的AND/OR过滤器，以限制对象到特定结果。

您也可以在过滤器字段中使用Slate的handlebars，以便根据动态变化的输入对对象集进行参数化。

过滤器字段还支持使用多词过滤器和通配符过滤器进行模板化：

  • 要使用多词过滤器进行模板化，可以直接传入多个词（例如，["value1", "value2"]）。
  • 或者，可以传入一个Handlebars模板，并直接引用返回多个词的函数。
    • 例如，如果 {{f_my_filters}} 返回 ["value1", "value2"]，则可以在对象集编辑器中直接使用 {{f_my_filters}}。

对象集结果以表格格式在字段 data 中返回，并且还包含字段 objectRids 中的所有对象 RID（资源标识符）。

对象集限制结果

对象集过滤器还可以让您设置返回的对象数量的上限。这在概念上类似于 SQL 的 LIMIT 子句，并且可以更快地进行测试和迭代。

如果限制小于总结果数，可以使用 getNextPage 和 getPreviousPage 事件来分页浏览结果。当返回大量结果时，这可以提高性能。

聚合

可以在 平台 选项卡中的对象集编辑器中使用简单的对象集聚合。要创建一个聚合，请打开 聚合 在 输出选项 下选择 添加聚合。该聚合基于上述定义的结果对象集。

聚合可以使用以下聚合类型之一：计数、平均值、总和、最大值、最小值或基数。在计算除计数以外的聚合时，必须选择一个数值属性。

您可以通过选择 添加聚合 来基于一个对象集计算多个聚合。使用聚合时，预览仅显示聚合结果。然而，当通过 handlebars 在 Functions（函数）、Widgets（小部件）和其他 Slate 部分引用对象集时，这些细粒度的对象仍然可用。

多维聚合

为了对多个维度（如包括“按...分组”或细分）的对象集数据进行聚合，需要编写一个 函数 来对数据进行分组和细分，然后对生成的桶计算指标。

这些函数返回 聚合类型 那Slate会解析成用于配置图表和其他Slate小部件的并行列表数据结构。

有关如何调用Foundry函数的文档中提供了更多关于这些与Slate交互的详细信息。

对象集排序

结果对象集可以按任意参数进行升序或降序排列。可以配置多个属性用于排序，排序属性从上到下依次应用。限制设置只有在排序之后才会生效。

对象集RID模式

在对象集RID模式下，Slate的对象集面板能够接受object set RID以及一个versioned object set RID，并将其还原为Slate的数据数组格式。

这意味着您将可以灵活地传递RID，除了使用Slate的参数化对象集构建器之外。
