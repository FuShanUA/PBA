对象监视器 [停用]

停用

对象监视器处于 停用 阶段，并将在未来某个日期被弃用。目前仍提供完整支持。我们建议将您的工作流程迁移到 Automate。Automate 是一个完全向后兼容的产品，为平台中的所有业务自动化提供单一入口。

对象监视器 应用程序允许终端用户和应用构建者查看 Foundry 本体论中数据何时发生变化。当发生更改时，对象监视器可以在满足指定条件时自动发送通知或提交操作。对象监视器运行在您的数据之上，旨在帮助用户跟踪特定的搜索和对象。对象监视器也作为应用构建者的工具，用于在 Foundry 中构建的应用程序中包含监控和警报功能。

对象监控是 Foundry 对象层的一项功能，可用于各种工作流，包括：

  • 关注的搜索: 用户可以配置对象监视器，以便在保存的对象探索有新结果时或在搜索的所有结果中满足聚合条件时收到通知。
  • 自动通知: 工作流构建者和自助数据消费者可以配置对象监视器，以响应数据更改发送通知。通知可以通过以下方式发送：
    • 在 Foundry 通知中心的平台内弹窗
    • 电子邮件
    • 短信（使用 webhooks 发送到第三方服务，例如 Twilio ↗）
    • 即时消息（使用 webhooks 发送到第三方服务，例如 Slack ↗ 或 Microsoft Teams ↗）
    • 定制或专有的消息系统
  • 工作流自动化: 可以使用对象监视器对符合特定条件的对象数据自动执行操作。使用对象监视器可以自动完成的一些任务包括：
    • 检查数据异常，并将这些对象自动传递到具有修复逻辑的操作中。
    • 监控建议或潜在操作，并在预设事件和时间条件满足时自动应用它们。此类操作可能包括通过 webhooks 向外部系统发出 API 调用，直接在外部系统中应用更改。

访问对象监视器

要访问对象监视器应用程序，请点击浏览器左侧导航侧边栏中的名称或图标。 概览 页面将显示您最近的监视活动列表，以及总监视器数量、已订阅或静音的监视器数量，以及出现错误或即将过期的监视器数量。

监视器 页面显示您可用的所有监视器列表。可以通过活动状态、通知和操作设置、创建者、过期日期、监视器类型或条件状态来筛选此列表。点击一个监视器以打开监视器概览面板并查看历史活动、订阅者和其他详细信息。

通过 创建一个新的对象监视器 来了解更多有关对象监视的信息。

对象监视器旨在监视您的数据内容。如果您正在寻找用于数据连接和管道构建的健康状况监控，请查阅 健康检查 文档。
