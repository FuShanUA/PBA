模板

代码模板将代码逻辑隐藏在简单的表单界面背后，使不同技术背景的用户能够协同工作。用户选择的值会代入代码模板中，随后即可像工作簿中的其他转换操作一样运行。

凡是通过代码能实现的转换操作，包括 Matplotlib 或 Plotly 可视化，均可借助代码模板完成。若想实际上手体验模板，请参阅《模板教程》。

搜索与发现

所有可用模板均可在模板浏览器中找到。要打开模板浏览器，请点击屏幕左上方的+ 新建转换并选择模板，或者将鼠标悬停在现有转换上点击+按钮，随后选择模板。

模板浏览器会展示最近使用的模板，并支持根据关键词、模板作者、标签等进行搜索。

使用代码模板

在图表视图中选中某个代码模板后，界面底部会向用户展示一个简单的表单输入框，而非代码编辑器。模板代码中的参数会渲染为待用户填写的输入项。

输入类型

  • 数据集：数据集字段的值必须是当前工作簿中的有效数据集。用户点击数据集输入框时，系统会提示从图表中选择一个输入数据集。数据集输入还可配置为始终映射到同一个输入数据集；例如，某个代码模板可能始终依赖于特定的本体数据集。更多信息请参阅下文的创建和编辑模板章节。
  • 列：系统会提示用户从某个输入数据集中选择一列或多列。列参数可配置为选择单列或多列。
  • 文本：文本输入框，允许用户输入自由文本值。
  • 数字：必须填入数值的输入框。
  • 选择：允许用户从预定义的候选值列表中进行选择。
  • 多选：允许用户从预定义的候选值列表中选择任意数量的值。
  • 布尔值：一个开关，供用户在 True 和 False 之间切换值。
  • 列表：允许用户输入任意数量的值，这些值将作为值列表传递给底层代码。

填写完输入字段后，即可运行模板转换，其方式与代码转换相同：选择运行按钮，或在图表上打开上下文菜单后选择运行。

使用切换视图可预览模板生成的底层代码。

创建和编辑模板

任何代码转换都能转化为代码模板，以便在无需重写代码的情况下重复应用相同的转换操作。要创建代码模板或编辑现有模板，请在代码编辑器顶部选择创建/编辑模板选项。

创建代码模板的过程，是将代码转换中的某些值指定为参数，并添加元数据以定义允许用户输入的类型。选择创建模板后，系统会进入全屏编辑器的分屏视图。屏幕左侧的代码编辑器可供您选择要参数化的代码片段；屏幕右侧的模板编辑器则用于配置各个参数的元数据。

默认情况下，新建模板时会将输入项转换为数据集类型的参数。您可以编辑这些参数，为其提供更具描述性的名称和说明。此外，模板本身也应设定标题和详细的描述。

要创建新参数，请在模板编辑器或代码编辑器中选择 添加新参数 选项。系统会提示您高亮显示一段文本，该段文本将替换为参数的输入值。若需添加同一参数的多个实例，可在模板编辑器中点击代码实例列表旁边的 添加 按钮来完成。

添加参数后，必须选择参数类型：数据集、列或变量。若选择 变量，还需为该变量选择一种参数类型：文本、数字、单选、多选、布尔值或列表。列输入可选择性配置为创建多列列表，数据集则可配置为默认使用特定的输入数据集。建议为所有参数添加描述，并为所有变量类型的参数提供默认值。

在首次创建模板的工作簿中，可用的全局函数将自动附加到模板代码的内容中，以便在其他工作簿中也能使用。模板无法访问其应用工作簿中的全局代码。

系统会保存模板的版本历史记录，对模板的新编辑始终会保存为该模板的新版本。对模板的编辑不会自动更新该模板的实例；若实例使用的是过时版本的模板，每个实例都会提示更新至最新版本。

最佳实践

  • 尽可能提供默认值。
  • 为模板和模板参数编写详细的描述。
  • 建立一套组织机制，方便用户为其工作流找到最相关的模板。这通常意味着需要创建一个专门的公共文件夹，存放高质量且经过审核的模板。可按工作流将模板分组至不同文件夹，并配置默认的模板文件夹。
