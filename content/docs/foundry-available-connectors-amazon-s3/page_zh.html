Amazon S3

将 Foundry 连接至 AWS S3，以在 S3 和 Foundry 之间读取和同步数据。

支持的功能

功能状态
探索🟢 正式可用
批量导入🟢 正式可用
增量同步🟢 受支持的文件格式正式可用
媒体集🟢 正式可用
虚拟表🟢 正式可用
文件导出🟢 正式可用

设置

  1. 打开《数据连接》应用程序，并在屏幕右上角选择 + 新建源。
  2. 从可用的连接器类型中选择 S3。
  3. 按照后续的配置提示，结合下方各章节的信息，继续完成连接器的设置。

详细了解如何在 Foundry 中设置连接器。

连接详情

选项是否必填描述
URL是S3 存储桶的 URL。数据连接支持 s3a 协议。需以斜杠结尾。详见 AWS 的 官方文档 ↗。
例如：s3://bucket-name/
Endpoint是用于访问 S3 的端点。
例如：s3.amazonaws.com 或 s3.us-east-1.amazonaws.com
Region否配置 AWS 服务时使用的 AWS 区域。使用 STS 角色时必填。注意：若同时提供区域以及包含该区域的 S3 端点，可能会导致连接失败。
例如：us-east-1 或 eu-central-1
Network connectivity是第 1 步：Foundry 出站策略
将 Foundry 出站策略附加到存储桶，允许 Foundry 向 S3 发起出站连接。数据连接应用会根据提供的连接详情推荐合适的出站策略。
例如：bucket-name.s3.us-east-1.amazonaws.com (端口 443)

第 2 步：AWS 存储桶策略
此外，需将相关的 Foundry IP 和/或存储桶详情加入 S3 的访问白名单。可以在控制面板应用的 网络出站 中找到 Foundry IP 详情。有关如何在 S3 中配置存储桶策略的更多详情，请参阅 AWS 官方文档 ↗。

注意：若要访问与 Foundry 订阅位于同一区域的 S3 存储桶，需进行额外配置。请阅读网络出站文档了解相关要求。
Client certificates & private key否源系统可能会也可能不要求提供客户端证书和私钥以保障连接安全。
Server certificates否源系统可能会也可能不要求提供服务器证书以保障连接安全。
Credentials是选项 1：访问密钥和 secret
提供用于连接 S3 的 Access Key ID 和 Secret。
可在 AWS 账户中为 Foundry 创建新的 IAM 用户，并授予该 IAM 用户访问 S3 存储桶的权限，以此生成凭证。

选项 2：OpenID Connect (OIDC)
按照显示的源系统配置说明设置 OIDC。有关 OpenID Connect 的详情，请参阅 AWS 官方文档 ↗；有关 OIDC 如何与 Foundry 配合使用的详情，请参阅我们的文档。

有关创建 AWS IAM 用户的更多详情，请参阅 AWS 官方文档 ↗。请查阅我们关于 S3 权限的文档，了解 Foundry 期望该用户具备哪些 AWS 权限。
STS role否S3 连接器在连接 S3 时，可选择承担安全令牌服务 (STS) 角色 ↗。详见STS 角色配置。
Connection timeout否初始建立连接时放弃并超时前的等待时间（毫秒）。
默认值：50000
Socket timeout否在已建立且开启的连接上传输数据时，连接超时并关闭前的等待时间（毫秒）。
默认值：50000
Max connections否允许开启的 HTTP 连接最大数量。
默认值：50
Max error retries否针对可重试的失败请求（例如：服务返回 5xx 错误响应）的最大重试次数。
默认值：3
Client KMS key否用于通过 AWS SDK 执行客户端数据加密的 KMS 密钥名称或别名。在 PCloud 中的智能体上使用此选项需更改代理配置。
Client KMS region否KMS 客户端使用的 AWS 区域。仅在提供了 AWS KMS 密钥时才相关。
Match subfolder exactly否可选择将子文件夹下指定的路径精确匹配为 S3 中的子文件夹。若设为 false，则 s3://bucket-name/foo/bar/ 和 s3://bucket-name/foo/bar_baz/ 都会与子文件夹设置 foo/bar/ 匹配。
Proxy configurations是 - 仅限基于智能体的连接配置 S3 的代理设置。
注意：如果满足以下条件则必填： Foundry 订阅托管在 AWS 中； 连接的 S3 存储桶所在的 AWS 区域与 Foundry 订阅不同； 通过数据连接智能体进行连接。详见S3 代理配置。
Enable path style access否使用路径式访问 URL（例如 https://s3.region-code.amazonaws.com/bucket-name/key-name），而非虚拟主机式访问 URL（例如 https://bucket-name.s3.region-code.amazonaws.com/key-name）。详见 AWS 官方文档 ↗。
Catalog否为存储在此 S3 存储桶中的表配置数据目录。详见虚拟表。

S3 所需的读取和同步权限

交互式探索 S3 存储桶需具备以下 AWS 权限：



Copied!
1{
2    "Action": ["s3:ListBucket"],
3    "Resource": ["arn:aws:s3:::path/to/bucket"],
4    "Effect": "Allow",
5}

针对从 S3 进行的批量同步、虚拟表和媒体同步，需要以下 AWS 权限：

Copied!
1{
2    "Action": ["s3:GetObject"],
3    "Resource": ["arn:aws:s3:::path/to/bucket/*"],
4    "Effect": "Allow",
5}

有关如何在 S3 中配置存储桶策略的更多详细信息，请参阅《Amazon S3 策略和权限官方 AWS 文档》。

S3 代理配置（基于智能体的连接）

使用数据连接智能体连接到 S3 时，可通过以下两种方式定义代理设置：

  • 源配置：按照下表所述，直接在源配置中定义各项代理设置。
  • 智能体的系统属性：作为备选方案，您可以在智能体的系统属性中配置代理设置。为此，请在智能体的高级配置设置中包含适当的 JVM 参数（例如，-Dhttps.proxyHost=example.proxy.com）。
参数是否必填默认值描述
host是HTTP 代理主机（无协议）。
port是HTTP 代理端口。
protocol否HTTPS要使用的协议。可以是 HTTPS 或 HTTP。
nonProxyHosts否不应使用代理的主机名（或通配符域名）列表。例如：`*.s3-external-1.amazonaws.com
credentials否如果您的代理需要基本 HTTP 身份验证（由 HTTP 407 响应 ↗ 提示），请包含此块。
credentials.username否HTTP 代理的明文用户名。
credentials.password否HTTP 代理的加密密码。

STS 角色配置

STS 角色配置允许您利用 AWS 安全令牌服务 ↗ 在读取 S3 时担任某个角色。

参数是否必填默认值描述
roleArn是STS 角色 ARN 名称。
roleSessionName是担任此角色时要使用的会话名称。
roleSessionDuration否3600 秒会话持续时间。
externalId否担任角色时要使用的外部 ID。

云身份配置

云身份验证允许 Foundry 访问您 AWS 实例中的资源。云身份在控制面板的 注册 层级进行配置和管理。了解如何 配置云身份。

使用云身份验证时，角色 ARN 将显示在凭据部分。选择 云身份 凭据选项后，您还必须配置以下内容：

  1. 在目标 Amazon AWS 账户中配置身份和访问管理 (IAM) 角色。
  2. 授予 IAM 角色访问您希望连接的 S3 存储桶的权限。您通常可以通过 存储桶策略 ↗ 来执行此操作。
  3. 在 S3 源配置详细信息中，将该 IAM 角色添加到 安全令牌服务 (STS) 角色 ↗ 配置下。访问 S3 时，Foundry 中的云身份 IAM 角色将尝试担任 AWS 账户 IAM 角色 ↗。
  4. 配置相应的信任策略 ↗，以允许云身份 IAM 角色担任目标 AWS 账户 IAM 角色。

虚拟表

本部分提供了有关从 S3 源使用 虚拟表 的更多详细信息。本部分不适用于同步到 Foundry 数据集的场景。

下表重点列出了 S3 支持的虚拟表功能。

功能状态
批量注册🔴 不可用
自动注册🔴 不可用
表输入🟢 正式可用：Avro ↗、Delta ↗、Iceberg ↗、Parquet ↗ 在 代码仓库、管道构建器 中
表输出🟢 正式可用：Avro ↗、Delta ↗、Iceberg ↗、Parquet ↗ 在 代码仓库、管道构建器 中
增量管道🟢 对于 Delta 表正式可用：仅限 APPEND（详细信息）
🟢 对于 Iceberg 表正式可用：仅限 APPEND（详细信息）
🔴 对于 Parquet 表不可用
计算下推🔴 不可用

有关存储在 S3 中的表可用作输入或输出的受支持 Foundry 工作流的详细信息，请查阅 虚拟表文档。

源配置要求

注册 虚拟表 时，请注意以下源配置要求：

  • 您必须使用 Foundry worker 源。虚拟表不支持使用 智能体 worker 连接。
  • 确保建立双向连接和允许列表，如 连接详细信息 中的 网络连接 部分所述。
  • 如果在代码仓库中使用虚拟表，请参阅 《虚拟表文档》 了解所需额外源配置的详细信息。
  • 如果您的存储桶包含 .，则必须启用路径样式访问并设置适当的出口策略。

有关更多详细信息，请参阅上方的 连接详细信息 部分。

Delta

要为以虚拟表为底座的管道启用增量支持，请确保在源 Delta 表上启用 变更数据流 ↗。支持 Python 转换 中的 current 和 added 读取模式。_change_type、_commit_version 和 _commit_timestamp 列将在 Python 转换中可用。

Iceberg

加载由 Apache Iceberg 表支持的虚拟表需要 Iceberg 数据目录。有关 Iceberg 数据目录的更多信息，请参阅《Apache Iceberg 文档》。在数据源上注册的所有 Iceberg 表必须使用同一个 Iceberg 数据目录。

默认情况下，系统将使用 S3 中的 Iceberg 元数据文件创建表。注册表时必须提供用于指示这些元数据文件位置的 warehousePath。

当表存储在 S3 中时，可将 AWS Glue 用作 Iceberg 数据目录。有关此集成的更多信息，请参阅《AWS Glue 文档》。在数据源上配置的凭证必须具有访问 AWS Glue Data Catalog 的权限。可以在数据源的“连接详情”选项卡中配置 AWS Glue。在此数据源上注册的所有 Iceberg 表将自动使用 AWS Glue 作为数据目录。注册表时应使用 database_name.table_name 命名模式。

在 Databricks 平台 中使用 Delta Universal Format (UniForm) 时，可将 Unity Catalog 用作 Iceberg 数据目录。有关此集成的更多信息，请参阅《Databricks 文档》。与 AWS Glue 一样，可以在数据源的“连接详情”选项卡中配置该数据目录。您需要提供端点和个人访问令牌才能连接到 Unity Catalog。注册表时应使用 catalog_name.schema_name.table_name 命名模式。

增量支持依赖于 Iceberg 的增量读取，目前仅支持仅追加模式。Python Transforms 中的 current 和 added 读取模式均受支持。

Parquet

使用 Parquet 的虚拟表依赖于模式推断。最多将使用 100 个文件来确定模式。

导出数据至 S3

要导出至 S3，首先需为您的 S3 连接器启用导出。然后，创建新导出。

S3 所需的导出权限

将数据导出至 S3 需要以下 AWS 权限：

Copied!
1{
2    "Action": ["s3:PutObject"],
3    "Resource": ["arn:aws:s3:::path/to/bucket/*"],
4    "Effect": "Allow",
5}

有关如何在 S3 中配置存储桶策略的更多详细信息，请参阅《Amazon S3 官方关于策略和权限的文档》。

导出配置选项

选项是否必需默认值描述
Path Prefix否无用于导出文件的路径前缀。导出文件的完整路径计算方式为 s3://<bucket-name>/<path-in-source-config>/<path-prefix>/<exported-file>
Canned ACL否无使用预定义 ACL 之一，设置附加到上传文件的 AWS 访问控制列表 (ACL)。有关各 ACL 的说明，请参阅 AWS 文档 ↗。

导出任务（遗留）

导出任务是一项遗留功能，不建议用于新实施的项目。所有新的导出操作均应使用当前的推荐导出工作流。本文档面向仍在使用遗留导出任务的用户提供。

S3 导出任务配置

若要通过导出任务将文件从 Foundry 数据集导出至 S3 存储桶，源类型必须为 s3-direct。任务类型为 export-s3-task。

基础配置

Copied!
1type: export-s3-task
2directoryPath: /some/directory

高级配置选项

选项是否必需默认值描述
directoryPath是无文件将导出到的 S3 路径
retriesPerFile否0每个文件导出的重试次数
setBucketPolicy否true是否对导出文件应用存储桶策略
bucketPolicy否无要应用的特定存储桶策略（例如 BucketOwnerFullControl）。请参阅 AWS CannedAccessControlList ↗
incrementalType否snapshot设为 incremental 以进行增量导出
useMultipart否false启用并行导出
threads否1并行导出的对象数量
rewritePaths否无用于路径重写的正则表达式映射

配置示例

带重试的增量导出：

Copied!
1type: export-s3-task
2directoryPath: /exports/data
3incrementalType: incremental
4retriesPerFile: 2

带自定义存储桶策略的并行导出：


```yaml
type: export-s3-task
directoryPath: /exports/data
useMultipart: false
threads: 4
bucketPolicy: BucketOwnerFullControl
setBucketPolicy: true
```


路径重写以移除 Spark 前缀：


```yaml
type: export-s3-task
directoryPath: /exports/data
rewritePaths:
  "^spark/(.*)": "$1"
```


在代码中使用 S3 数据源


你可以从 Python 转换代码仓库连接到 S3 存储桶，以导出数据集，或使用 外部转换处理文件。


将文件写入 S3 存储桶


下面的示例演示了如何使用转换生成器将一组数据集 RID 导出到 S3 存储桶：



Copied!
1# This example shows how to export a list of datasets to an S3 bucket using your AWS session credentials.
2# For each exported dataset, an output log containing the dataset's RID, export timestamp, number of exported
3# files, and S3 destination path will be generated.
4
5from pyspark.sql import functions as F, types as T
6from transforms.api import Input, Output, transform, Transform
7from transforms.external.systems import external_systems, Source
8from typing import List
9from datetime import datetime
10import boto3
11import logging
12
13log = logging.getLogger(__name__)
14
15S3_BUCKET_REGION = "<region>"
16S3_BUCKET_NAME = "<name>"
17S3_ENDPOINT_URL = "<endpoint_url>"
18S3_DEST_PATH = "<s3_dest_path>"
19
20S3_SOURCE_RID = "<s3_source_rid>"
21
22LOG_OUTPUT_PATH = "<log_output_path>"
23
24DATASET_RID = "<dataset_rid>"
25
26@external_systems(
27    s3_source=Source(S3_SOURCE_RID)
28)
29@transform(
30    export_log=Output(f"{LOG_OUTPUT_PATH}/export_{DATASET_RID}_log"),
31    input_dataset=Input(f"{DATASET_RID}")
32)
33def compute_function(ctx, input_dataset, s3_source, export_log):
34    # 1. Retrieve credentials for the S3 bucket
35    refreshable_creds = s3_source.get_session_credentials()
36    creds = refreshable_creds.get()
37
38    # 2. Construct S3 key for the current dataset
39    s3_dataset_key = f"{S3_DEST_PATH}/{DATASET_RID}/"
40
41    # 3. Set up the boto3 client
42    # NOTE: For large exports that may exceed credential TTL (~1 hour),
43    # consider refreshing credentials periodically or implementing
44    # boto3's RefreshableCredentials pattern
45    s3_client = boto3.client(
46        "s3",
47        aws_access_key_id=creds.access_key_id,
48        aws_secret_access_key=creds.secret_access_key,
49        region_name=S3_BUCKET_REGION,
50        endpoint_url=S3_ENDPOINT_URL,
51    )
52
53    # Helper function to write dataset files to S3
54    def write_file(row):
55        try:
56            with input_dataset.filesystem().open(row.path, "rb") as in_f:
57                s3_key = s3_dataset_key + row.path
58                s3_client.upload_fileobj(in_f, S3_BUCKET_NAME, s3_key)
59
60        except Exception as e:
61            log.error(f"Exception when uploading {DATASET_RID} to S3: {e}")
62            raise
63
64    # 4. Write dataset files to S3
65    result = input_dataset.filesystem().files()
66    result.foreach(write_file)
67
68    # 5. Define output export log schema
69    schema = T.StructType([
70        T.StructField("dataset_rid", T.StringType(), False),
71        T.StructField("export_timestamp", T.TimestampType(), False),
72        T.StructField("files_uploaded", T.IntegerType(), False),
73        T.StructField("s3_dest_path", T.StringType(), False)
74    ])
75    file_count = result.count()
76    output_df = ctx.spark_session.createDataFrame([(DATASET_RID, datetime.now(), file_count, s3_dataset_key)], schema)
77    export_log.write_dataframe(output_df)
