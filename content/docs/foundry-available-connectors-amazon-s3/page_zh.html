Amazon S3

将Foundry连接到AWS S3，以便在S3和Foundry之间读取和同步数据。

支持的功能

功能状态
探索🟢 通常可用
批量导入🟢 通常可用
增量🟢 对支持的文件格式通常可用
媒体集🟢 通常可用
虚拟表🟢 通常可用
文件导出🟢 通常可用

设置

  1. 打开 数据连接 应用程序，并在屏幕右上角选择 + 新源。
  2. 从可用的连接器类型中选择 S3。
  3. 按照后续配置提示继续使用以下部分中的信息设置您的连接器。

了解更多关于 在Foundry中设置连接器 的信息。

S3 的必需读取和同步权限

3    "Resource": ["arn:aws:s3:::path/to/bucket"],
4    "Effect": "Allow",
5}

以下 AWS 权限是用于从 S3 进行批量同步、虚拟表和媒体同步所必需的：

Copied!

2    "Action": ["s3:GetObject"],
3    "Resource": ["arn:aws:s3:::path/to/bucket/*"],
4    "Effect": "Allow",
5}

有关如何在 S3 中配置存储桶策略的更多信息，请参阅 AWS 官方文档关于 Amazon S3 中的策略和权限 ↗。

云身份配置

云身份认证允许Foundry访问您AWS实例中的资源。云身份在控制台的 注册 级别进行配置和管理。了解如何 配置云身份。

使用云身份认证时，角色ARN将在凭据部分显示。选择Cloud identity凭据选项后，还必须配置以下内容：

  1. 在目标Amazon AWS账户中配置一个身份和访问管理（IAM）角色。
  2. 授予IAM角色对您希望连接的S3存储桶的访问权限。通常可以通过 存储桶策略 ↗ 来实现。
  3. 在S3源配置详细信息中，将IAM角色添加到 安全令牌服务（STS）角色 ↗ 配置下。当访问S3时，Foundry中的云身份IAM角色会尝试假定 AWS账户IAM角色 ↗。
  4. 配置相应的信任策略 ↗ 允许云身份IAM角色假定目标AWS账户的IAM角色。

虚拟表

源配置要求

在注册 虚拟表 时，请记住以下源配置要求：

  • 您必须使用 Foundry 工作人员 源。虚拟表不支持使用 代理工作人员 连接。
  • 确保按照 网络连接 部分在 连接详细信息。
  • 如果在代码仓库中使用虚拟表，请参考 虚拟表文档 了解所需的额外源配置详情。
  • 如果您的存储桶包含 .，您必须启用路径样式访问并设置适当的出站策略。

有关更多详细信息，请参见上面的 连接详细信息 部分。

Delta

为了在由虚拟表支持的管道中启用增量支持，请确保在源 Delta 表上启用了 Change Data Feed ↗。在 Python 转换 中的 current 和 added 读取模式是受支持的。_change_type、_commit_version 和 _commit_timestamp 列将在 Python 转换中可用。

Iceberg

一个 Iceberg 目录是加载由 Apache Iceberg 表支持的虚拟表所必需的。有关 Iceberg 目录的更多信息，请参阅 Apache Iceberg 文档 ↗。在源上注册的所有 Iceberg 表必须使用相同的 Iceberg 目录。

默认情况下，将使用 S3 中的 Iceberg 元数据文件创建表。在注册表时，必须提供一个 warehousePath 以指示这些元数据文件的位置。

AWS Glue ↗ 可以作为 Iceberg 目录，当表存储在 S3 中时。有关此集成的更多信息，请参阅 AWS Glue 文档 ↗。配置在源上的凭据必须能够访问您的 AWS Glue 数据目录。AWS Glue 可以在源的 连接详细信息 选项卡中进行配置。在此源上注册的所有 Iceberg 表将自动使用 AWS Glue 作为目录。应使用 database_name.table_name 命名模式注册表。

Unity Catalog ↗ 可以在 Databricks 中使用 Delta Universal Format（UniForm）时用作 Iceberg 目录。有关此集成的更多信息，请参阅 Databricks 文档 ↗。与 AWS Glue 类似，目录可以在源的 连接详细信息 选项卡中进行配置。您需要提供端点和一个 个人访问令牌 来连接 Unity 目录。表应使用 catalog_name.schema_name.table_name 命名模式注册。

增量支持依赖于 Iceberg 增量读取 ↗，并且目前仅支持追加。在 Python 转换 中，current 和 added 读取模式是受支持的。

Parquet

使用 Parquet 的虚拟表依赖于模式推断。最多将使用 100 个文件来确定模式。

导出数据到 S3

要导出到 S3，请首先 启用导出 用于您的 S3 连接器。然后，创建一个新的导出。

S3 的所需导出权限

要将数据导出到 S3，需要以下 AWS 权限：

Copied!
1{
2    "Action": ["s3:PutObject"],
3    "Resource": ["arn:aws:s3:::path/to/bucket/*"],
4    "Effect": "Allow",
5}

有关如何在 S3 中配置存储桶策略的更多信息，请参阅 AWS 上关于 Amazon S3 中的策略和权限的官方文档 ↗。

导出配置选项

选项是否必需？默认值描述
Path Prefix否N/A应为导出文件使用的路径前缀。导出文件的完整路径计算为 s3://<bucket-name>/<path-in-source-config>/<path-prefix>/<exported-file>
Canned ACL否N/A 设置附加到上传文件的 AWS 访问控制列表 (ACL)，使用其中一种预定义的 ACL。有关每个 ACL 的说明，请参阅 AWS 文档 ↗。

导出任务（旧版）

导出任务 是一个旧版功能，不推荐用于新实现。所有新的导出都应使用当前的 推荐的导出工作流程。此文档是为仍在使用旧版导出任务的用户提供的。

S3 导出任务配置

要使用导出任务将文件从 Foundry 数据集导出到 S3 存储桶，源必须是 s3-direct 类型。任务类型是 export-s3-task。

基本配置

Copied!
1type: export-s3-task
2directoryPath: /some/directory

高级配置选项

选项必需默认值描述
directoryPath是无文件将被导出的S3路径
retriesPerFile否0每个文件导出的重试次数
setBucketPolicy否true是否对导出的文件应用存储桶策略
bucketPolicy否无要应用的特定存储桶策略（例如，BucketOwnerFullControl）。参见 AWS CannedAccessControlList ↗
incrementalType否snapshot使用 incremental 进行增量导出
useMultipart否false 启用并行导出
threads否1并行导出的对象数量
rewritePaths否N/A路径重写用的正则表达式模式映射

示例配置

带重试的增量导出：

Copied!
1type: export-s3-task
2directoryPath: /exports/data
3incrementalType: incremental
4retriesPerFile: 2

带自定义存储桶策略的并行导出：

Copied!
1type: export-s3-task
2directoryPath: /exports/data
3useMultipart: false
4threads: 4
5bucketPolicy: BucketOwnerFullControl
6setBucketPolicy: true

移除Spark前缀的路径重写：

Copied!
1type: export-s3-task
2directoryPath: /exports/data
3rewritePaths:
4  "^spark/(.*)": "$1"

在代码中使用S3源

您可以从 Python转换代码仓库 连接到S3存储桶，以导出数据集或使用 外部转换 处理文件。

将文件写入S3存储桶

将文件写入 S3 存储桶

下面的示例演示了如何使用转换生成器将一组数据集 RID 写入 S3 存储桶：

Copied!
1# This example shows how to export a list of datasets to an S3 bucket using your AWS session credentials.
2# For each exported dataset, an output log containing the dataset's RID, export timestamp, number of exported
3# files, and S3 destination path will be generated.
4
5from pyspark.sql import functions as F, types as T
6from transforms.api import Input, Output, transform, Transform
7from transforms.external.systems import external_systems, Source
8from typing import List
9from datetime import datetime
10import boto3
11import logging
12
13log = logging.getLogger(__name__)
14
15S3_BUCKET_REGION = "<region>"
16S3_BUCKET_NAME = "<name>"
17S3_ENDPOINT_URL = "<endpoint_url>"
18S3_DEST_PATH = "<s3_dest_path>"
19
20S3_SOURCE_RID = "<s3_source_rid>"
21
22LOG_OUTPUT_PATH = "<log_output_path>"
23
24DATASET_RID = "<dataset_rid>"
25
26@external_systems(
27    s3_source=Source(S3_SOURCE_RID)
28)
29@transform(
30    export_log=Output(f"{LOG_OUTPUT_PATH}/export_{DATASET_RID}_log"),
31    input_dataset=Input(f"{DATASET_RID}")
32)
33def compute_function(ctx, input_dataset, s3_source, export_log):
34    # 1. Retrieve credentials for the S3 bucket
35    refreshable_creds = s3_source.get_session_credentials()
36    creds = refreshable_creds.get()
37
38    # 2. Construct S3 key for the current dataset
39    s3_dataset_key = f"{S3_DEST_PATH}/{DATASET_RID}/"
40
41    # 3. Set up the boto3 client
42    # NOTE: For large exports that may exceed credential TTL (~1 hour),
43    # consider refreshing credentials periodically or implementing
44    # boto3's RefreshableCredentials pattern
45    s3_client = boto3.client(
46        "s3",
47        aws_access_key_id=creds.access_key_id,
48        aws_secret_access_key=creds.secret_access_key,
49        region_name=S3_BUCKET_REGION,
50        endpoint_url=S3_ENDPOINT_URL,
51    )
52
53    # Helper function to write dataset files to S3
54    def write_file(row):
55        try:
56            with input_dataset.filesystem().open(row.path, "rb") as in_f:
57                s3_key = s3_dataset_key + row.path
58                s3_client.upload_fileobj(in_f, S3_BUCKET_NAME, s3_key)
59
60        except Exception as e:
61            log.error(f"Exception when uploading {DATASET_RID} to S3: {e}")
62            raise
63
64    # 4. Write dataset files to S3
65    result = input_dataset.filesystem().files()
66    result.foreach(write_file)
67
68    # 5. Define output export log schema
69    schema = T.StructType([
70        T.StructField("dataset_rid", T.StringType(), False),
71        T.StructField("export_timestamp", T.TimestampType(), False),
72        T.StructField("files_uploaded", T.IntegerType(), False),
73        T.StructField("s3_dest_path", T.StringType(), False)
74    ])
75    file_count = result.count()
76    output_df = ctx.spark_session.createDataFrame([(DATASET_RID, datetime.now(), file_count, s3_dataset_key)], schema)
77    export_log.write_dataframe(output_df)
