高级代码仓库设置

大多数代码仓库设置均可在设置选项卡中找到。部分附加配置值需通过位于仓库根目录下的 repoSettings.json 文件进行设定。若该文件尚不存在，您可以在仓库根目录下创建一个名为 repoSettings.json 的文件。

自定义标签名校验

为了确保仓库内所有新建标签在代码仓库默认限制的基础之上遵循特定的命名规则，您可以配置自定义正则表达式，并在用户输入不符合该正则时显示错误提示信息。例如：

Copied!
1"tagNameValidation": {
2    "regex": "^(0|[1-9]\\d*)\\.(0|[1-9]\\d*)\\.(0|[1-9]\\d*)(-rc\\d+)?$",
3    "errorMessage": "Tag name must have the format x.x.x or x.x.x-rcx"
4}

如果为 tagNameValidation 设置了值，您必须同时配置 regex 和 errorMessage。

拉取请求描述模板

为了在创建新的拉取请求时维持最佳实践，您可以配置描述模板，以便在创建拉取请求时自动预填 描述 字段：

Copied!
1"prDescriptionTemplate": "First line of the description template\nAnd another line with instructions"

新建转换的输出数据集路径

默认情况下，新建转换文件的模板会直接将文件路径作为初始的输出数据集路径。例如，对于 Python 转换，路径为 /path/to/your/repository/transforms-python/src/myproject/datasets/name_of_file。您可以通过配置 outputPathPrefix，将其修改为更便捷的位置，例如项目中的特定文件夹：



已复制！
1"outputPathPrefix": "/My/Custom/Prefix"

采用上述设置后，输出数据集的路径将改为 /My/Custom/Prefix/name_of_file。

拉取请求校验规则

通过在 repoSettings.json 文件中添加 prValidation 条目，即可对代码库中的拉取请求强制执行相应规则。创建拉取请求时，系统会根据基础分支上的 repoSettings.json 文件对其进行校验。

拉取请求校验规则包含一个正则表达式、一个需与该表达式进行匹配的拉取请求字段列表，以及一条在字段不符合规则时显示的错误消息。

示例：repoSettings.json



已复制！
1{ 
2    "tagNameValidation": {
3        "regex": "^(0|[1-9]\\d*)\\.(0|[1-9]\\d*)\\.(0|[1-9]\\d*)(-rc\\d+)?$",
4        "errorMessage": "标签名格式必须为 x.x.x 或 x.x.x-rcx"
5    },
6    "prDescriptionTemplate": "描述模板的第一行\n带有说明的另一行",
7    "outputPathPrefix": "/My/Custom/Prefix",
8    "prValidation": [
9        {
10            "regex": "TEST-\\d{3}",
11            "fields": ["title", "branchName"],
12            "errorMessage": "该字段必须包含有效的 TEST-XXX 工单号，例如 'TEST-123'。"
13        },
14        {
15            "regex": "This is the description",
16            "fields": ["description"],
17            "errorMessage": "请在描述中的某处输入 'This is the description'。"
18        }
19    ]
20}
