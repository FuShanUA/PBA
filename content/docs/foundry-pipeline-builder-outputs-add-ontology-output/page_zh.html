添加本体输出

你可以选择添加本体输出，引导管线集成生成干净、结构化的数据，从而定义全局本体中的新数据要素。添加输出引导有助于节省计算检查时间，并为端到端构建工作流提供参考。

管线构建器目前支持从批处理和流式数据集创建本体对象和链接。但是，在管线构建器中，无法为基于流式数据集的对象启用编辑功能或配置多对多链接。

添加对象类型输出

添加对象类型输出有两种方式。你可以通过管线输出面板创建一个空白对象，或者在图上创建一个连接到转换节点的已填充对象。

通过管线输出面板

打开位于管线构建器右侧的管线输出面板，然后选择对象类型 > 添加，即可创建一个空白对象模板，你可以在其中编辑对象名称、图标和属性。

请注意，为了方便使用，复数名称和对象类型 ID 将根据名称自动填充。

你可以在管线构建器中定义新对象，但无法修改或删除现有对象，除非这些对象是在管线构建器中创建的。首次部署后，对象类型 ID 将无法修改。

现在，你可以通过选择手动输入数据，或者将现有数据集或转换的输出拖动到该节点的输入端，来为对象填充数据。

通过图

选择要用于创建对象的转换节点，然后选择 添加输出 > 新对象类型。

如果您的接入层包含多个本体，在创建对象时需要选择该对象所属的本体。否则，系统将显示错误消息 必须指定本体才能发布本体类型输出。要设置本体，请在底部面板或对象上选择 设置本体，如下所示。

在对象类型上启用编辑

如果对象是在管道构建器中创建的，且由批处理数据集提供数据支持，您可以在管道构建器或本体管理器中启用对象类型的编辑功能。

基于流数据的对象类型目前不支持编辑。要启用基于流数据的对象类型的编辑功能，请使用直接数据源功能。

要启用编辑功能，请将鼠标悬停在右侧面板的对象类型上，然后选择 编辑。切换 允许编辑此类型的对象。保存并部署您的管道以使更改生效。

启用直接数据源 [Beta]

Beta

直接数据源目前处于Beta 开发阶段，您的接入层可能尚未提供此功能。在活跃开发期间，相关功能可能会发生变化。

直接数据源支持向本体进行低延迟、低吞吐量的写入操作，包括对由流数据支持的对象类型进行编辑。

要启用直接数据源：

    - 创建一个带有对象类型输出的流式管道。您应该会看到使用直接数据源支持此对象类型和允许编辑此类型的对象选项。请启用这两项设置。 - 根据需要配置对象目标，然后进行部署。部署完成后，该对象类型将出现在[本体管理器]中，其对象可在[Insight]和[对象浏览器]中查看。

    了解更多关于部署后监控和调试直接数据源的信息。。

    如果您脱离了由直接数据源支持的对象目标，图中脱离后的节点将显示为流数据集。然而，该节点代表的是支持该对象类型的直接数据源，而不是流式数据集。与流数据集不同，直接数据源不直接存储数据；数据存储在本体中。脱离后，您仍然可以在本体管理器或 Insight 中导航至该对象类型。

    已知限制

    作为正在积极开发中的测试版功能，直接数据源目前存在以下限制：

    • 重放会清空对象视图： 重放流式管道会移除现有的对象视图，以便立即提供新视图。由于切换是瞬间完成的，新视图尚未完全填充，因此对象视图可能会暂时不完整或为空。完整的视图会随着时间推移逐渐填充。用于解决此限制的重放功能正在开发中，本文档将相应更新。
    • 初始部署超时： 在初始部署时，或者在使用重置输出进行重放时，流式作业可能在提交前失败，这是由于部署期间执行的初始设置可能导致超时。
    • 不支持分支： 直接数据源不能在分支上使用。
    • 回填可能较慢：在重放后回填数据可能较慢。相关改进功能正在开发中。

    有关补救步骤和其他常见问题，请参阅《直连数据源常见问题解答》。

    添加链接类型输出

    添加对象类型输出后，即可定义链接类型输出。与对象输出类似，可以在面板或图形上创建这些输出。

    使用管道输出面板

    在管道输出面板中，选择链接类型 > 添加。这将打开一个表单，您可以在其中定义链接类型的左右两侧。

    在管道构建器中定义的链接只能使用在同一管道内创建的对象。而在本体管理器应用程序内部，您可以在管道构建器创建的对象和本体管理器创建的对象之间建立链接。
    首次部署后，链接类型 ID 或源和目标对象类型将无法再修改。

    图例

    链接类型 ID：这是自动生成的 ID，用于唯一表示该链接。

    选择对象类型输出：从下拉菜单中选择一种对象类型。如果管道中没有任何对象输出，您需要先配置对象输出类型。

    基数：选择链接类型两侧的基数。支持两种基数类型：一对多和多对多。目前不支持一对一链接类型。在以下示例中，假设存在通过基数相互关联的两种对象类型：Aircraft（飞行器）对象类型和 Flight（航班）对象类型。

    • 一对多基数：表示一架 Aircraft 可以关联多个 Flights。
    • 多对多基数：表示一架 Aircraft 可以关联多个 Flights，且一个 Flight 可以关联多架 Aircraft。
      • 请注意，选择多对多基数时，系统会提示点击并拖动来自数据集或转换的输出，将其用作连接表。该表应包含第一个对象类型（本例中为 Aircraft）与第二个对象类型（本例中为 Flight）之间所有链接组合。

    主键/列：选择用于创建链接左侧的属性。

    • 在一对多基数链接类型中，主键由对象类型决定。
    • 在多对多基数链接类型中，选择连接表中映射到各个链接对象类型主键的列。

    外键/列：选择用于创建链接右侧的属性。

    • 在一对多基数链接类型中，一个对象类型的外键属性必须引用另一个对象类型的主键属性。
    • 在多对多基数链接类型中，选择连接表中映射到各个链接对象类型主键的列。

    名称：为链接类型的每一侧填写显示名称。链接类型的一侧代表指向该对象类型的链接。Aircraft 对象类型的显示名称描述了从 Flight 到 Aircraft 的链接。在本示例中，由于一个 Flight 对应一个分配的 Aircraft，您可以选择显示名称“Assigned Aircraft”。

    复数显示名称：对于基数为多的链接类型侧，系统还会提示您填写复数显示名称，以便用户应用程序在显示链接对象时能够展示正确的名称。在我们的示例中，Flight 对象类型的复数显示名称将描述从 Aircraft 到 Flight 的链接。由于一架 Aircraft 有多个预定的 Flights，我们可以将复数显示名称设为“Scheduled Flights”。

    解决本体验证错误

    用户对管道所做的修改可能会导致本体出现验证错误。以下是一些常见的错误示例：

    • 在管道构建器和本体管理器中对同一对象类型或链接类型进行了冲突修改。
    • 修改了被管道外部链接类型引用的对象类型主键。
    • 对对象类型的结构进行了破坏性修改，需要进行结构迁移。

    所有错误都将显示在右侧面板的部署此管道区域下。在错误标签页中，系统会提供提示信息，说明部署前需要进行何种类型的处理。

    必须先解决这些错误，然后才能部署管道。

    如果解决按钮显示箭头，点击该按钮将跳转至本体管理器以处理验证错误。否则，您可以直接在管道构建器中解决这些错误。

    在 Ontology Manager 中解决合并冲突

    按照 Ontology Manager 中的提示应用合并冲突解决方案。如果您的更改影响到潜在的编辑，您可以决定是放弃这些编辑，还是使您的编辑与对象的最新版本兼容。否则，无需额外配置即可解决合并冲突，您可以直接选择应用。

    应用更改后，本体（Ontology）将更新为最新状态，您可以选择返回数据管道。

    解决冲突后，返回数据管道时您偶尔会遇到 Ontology type outputs are out of date with the ontology 错误。这是因为数据管道尚未同步通过上述冲突解决工作流对本体输出所做的更改。

    选择解决应可修复此问题，并允许您部署数据管道。

    在数据管道构建器中解决冲突

    部分本体验证错误无需切换至 Ontology Manager 即可处理。在以下示例中，需要进行模式迁移，这通常发生在数据管道构建器中的对象类型模式与本业主分支上的最新模式不一致的情况下。选择解决即可直接在数据管道构建器中选定迁移策略。

    针对受影响的属性，您可以选择是否放弃属性编辑，或者迁移您的编辑以使其与该属性的最新版本兼容。

    选定最合适的处理方式后，请选择应用迁移。

    将对象类型输出引回 Pipeline Builder

    按照以下步骤，您可以将管道中由数据集输出的对象类型引回 Pipeline Builder：

    在右侧面板中，选择管道输出旁边的齿轮图标。

    选择替换为对象。这将打开将数据集替换为本体类型输出对话框。

    选择您想要在管道中转换回相应对象的数据集输出，然后选择导入所选输出。

    保存更改并部署您的管道。

    在 Pipeline Builder 中放弃本体类型输出 [已弃用]

    在 Pipeline Builder 中放弃本体类型输出会将这些本体类型的管理权移交给 Ontology Manager。这意味着，后续对这些本体类型的所有修改只能在 Ontology Manager 中进行。当您需要 Pipeline Builder 不支持的对象类型配置（例如共享属性类型或运行时派生属性）时，此功能非常有用。要放弃本体类型，请选择您想要放弃的对象类型。任何引用所选对象类型的链接类型都将自动转移到 Ontology Manager。

    只有在 main 分支最新保存的版本成功部署后，才能将本体类型输出替换为数据集输出。

    选择放弃所选对象和链接类型。

    在此流水线中，*对象类型*输出节点将转换为支持 Ontology Manager 中对象类型的*数据集*输出节点。在右侧面板中，选择输出旁边的图标，即可导航至 Ontology Manager 中的对象类型。

    在取消选定对象的数据权属后，部署您的流水线。

    底层数据源中的列名必须与为给定对象类型属性配置的底层列名完全匹配。列名不匹配将导致数据丢失或索引作业失败。例如，底层列为 routeId 的属性，要求底层数据源中必须存在名为 routeId 的列。

    配置细粒度安全权限

    您可以通过使用受限视图策略或对象安全策略，为对象配置细粒度安全权限：

    • 受限视图（RVs）支持行级安全。
    • 对象安全策略同时支持行级和列级安全。

    与 Ontology Manager 中的对象安全策略相比，流水线构建器中的对象安全策略目前仅支持部分功能。支持的功能详情请参见下文。

    要设置细粒度安全策略，请转到输出对象并选择配置细粒度安全。

    在弹出的窗口中选择受限视图策略或对象安全策略。

    对象安全策略

    对象安全策略选项允许您使用行级策略、属性安全策略、标记移除管理和分类更改，来配置单元级别的细粒度安全。

    默认情况下，对象安全策略会应用于所有属性。当属性安全策略包含某个属性时，用户必须同时通过对象安全策略和属性安全策略，才能查看该属性值。

    要添加行级策略，请选择细粒度安全旁边的加号 (+) 图标。

    在策略编辑器中，您可以创建规则，通过比较属性值、用户属性或常量值来定义您的安全条件。

    请注意，要基于标记或分类用户属性创建策略，请确保值类型已设置为标记或分类。

    要强制实施正确的类型，请在转换节点中使用逻辑类型转换表达式。

    此外，标记和分类必须包装在数组中，以确保与本体兼容。将单个标记或分类直接映射到对象属性会导致错误。

    标记必须使用关联的标记 ID。详细了解标记 ID 的预期格式。

    配置好细粒度安全策略后，选择返回（左下角）回到主配置窗口。如果无需进一步更改，请选择应用以保存您的策略。

    您可以前往强制控制移除任何继承的标记，或前往分类调整分类。

    Pipeline Builder 目前不支持调整手动输入表格的分类。

    添加属性安全策略的方式与此类似。在属性安全策略部分选择添加。这将创建一条新的属性安全策略。要配置属性安全策略，请选择要编辑的策略上的箭头。

    在策略名称下，您可以对属性安全策略进行重命名。在受影响属性下，选择要应用此策略的一个或多个列，然后按照与上述相同的方式配置细粒度安全、强制控制或分类。

    配置完属性安全策略后，选择返回（左下角）回到主配置窗口。使用应用保存您的策略。

    受限视图策略

    要配置用户可以在输出中查看哪些行，您可以创建受限视图策略并将其应用于您的对象。该策略由受限视图支持，该视图是在您的更改合并并部署到 main 分支后构建的。

    您可以直接在受限视图策略窗口中配置策略。此处适用的标记和分类规则与上文所述相同。了解更多关于受限视图策略和最佳实践的信息。您还可以选择将受限视图策略转换为对象策略，以实现更高的可配置性。

    完成创建细粒度安全策略后，请保存您的管道并提交更改。底层的安全策略只有在您成功部署到 main 分支后才会创建并生效。
