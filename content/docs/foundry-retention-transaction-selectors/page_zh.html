事务选择器

以下列表描述了在配置保留策略时可用于保留应用程序的事务选择器。

this format 中的文本表示可以在每个策略中定义的参数。

仅在分支中

删除仅在给定分支中出现的事务。

接受 1 个参数： branch

示例： "仅在分支 main"

设置 "仅在分支 main" 不会删除前两个 SNAPSHOT 事务，因为它们也是 SNAPSHOT 事务在功能分支上的根事务。

不在分支中

删除所有事务，除了给定分支中的事务。

需要 1 个参数： branch

示例： "不在分支 main"

事务数量

事务数量选择器允许您定义要保留的事务。它不是用来指示将被删除的事务。

只保留任何分支上最_number of transaction to retain_最近的数据包含事务。如果以下陈述为真，则事务被定义为包含数据：

  • 事务已提交。
  • 事务不是 DELETE 事务。

所有中止的事务不包含数据，将会被删除。由于此选择器无法区分 SNAPSHOT、APPEND 或 UPDATE 事务，我们建议在增量管道中使用 viewCount 选择器。

需要一个参数： number of transactions to retain

示例："事务数量 2"

此选择器确保每个分支上至少有 2 SNAPSHOT 事务可用。功能分支可能有 3 个事务；最旧的事务也是分支 main 上的第 2 个事务，因此不会被删除。

查看次数

查看次数选择器允许您定义要保留的视图中的事务。它不是用来指示将被删除的视图中的事务。

查看次数选择器只保留最后 number of views to retain 数据集视图 中的事务 . 由于视图仅由已提交的事务组成，任何中止的事务也会被删除。例如，numViewsToRetain: 1 表示删除最新视图之前的所有事务（即所有在最新 SNAPSHOT 事务之前的事务）以及所有中止的事务。

接受 1 个参数： number of views to retain

示例： "视图计数 1"

超过一定持续时间

此选择器用于选择超过给定持续时间的事务。

接受 1 个参数： duration

已被投影（高级）

对于具有 投影 的数据集，此选择器会选择已传播到所有投影的事务。

当前视图中没有文件（高级）

这个选择器选择所有不在最新视图中的事务（以及所有当前在视图中的事务，其中所有文件都已被较新事务中的文件取代）。这对于具有许多事务的最新视图的数据集很有用，并且只能与Allow deletion from latest view标志一起使用。

仅存在于比（高级）更早的视图中

选择仅存在于给定持续时间之前的视图中的事务。视图年龄定义为视图中最新事务的关闭时间和现在之间的时间。如果一个视图有打开的事务，则该视图中的任何事务都不会被删除。

需要一个参数： duration

由（高级）衍生

选择由运行构建生成的交易。这些是通过运行构建生成的交易。通过手动上传数据或通过数据连接创建的交易不被视为衍生。
