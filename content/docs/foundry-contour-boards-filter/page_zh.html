筛选数据

Contour 的许多可视化看板不仅可以用来可视化数据，还能用于筛选数据。本文将介绍如何使用直方图、筛选器和图表看板来实现这一操作。高级用户若需更强大的筛选选项，可参阅 Contour 的《表达式语言》。本页截图均使用开源航空数据。

直方图看板

直方图是最简单的筛选方式之一，推荐在直观探索不熟悉的数据集时使用。

创建直方图后，直接点击对应的柱状条即可进行筛选：

在直方图中选中柱状条后，其底部会显示保留 `column_name` 为 `value` 的行。这表明当前工作数据集已筛选为仅包含该特定列中具有该值的行。（如果只想保留不包含该值的行，可通过下拉选择器将保留更改为移除。）

在已明确筛选目标的情况下，筛选器看板能发挥最佳效果。该看板极具灵活性，允许您精确输入所需的筛选条件。

从操作功能区中选择筛选，即可添加筛选器看板：

多数据筛选器

  • 若要筛选同时满足多个条件的数据，请使用匹配全部 (AND MATCHING)。
  • 若要筛选满足多个条件但无需同时满足或在同一数据行内满足的数据，请使用匹配任一 (OR MATCHING)。

调整筛选器

当有人针对特定筛选器（例如 carrier_code = DL）创建新分析时，其他人可以轻松地通过将筛选器更改为符合其自身场景的条件（例如 carrier_code = UA）来复现该分析，或者直接移除该筛选器以执行全局分析。

在 Foundry 中，分析操作默认会应用于整列数据，以方便对大型数据集进行分析。若您只想对较小的行子集运行分析（类似于在 Excel 中选择特定的单元格区域），请先筛选出目标行，再应用相关操作。

字符串过滤器中的通配符

某些字符串过滤器类型支持使用 ? 和 * 作为通配符。? 代表任意单个字符，而 * 代表任意数量的字符序列（该数量可以为 0）。

支持使用通配符的过滤器如下：

  • contains
  • contains (with wildcards)
  • is (with wildcards)

尽管命名方式不太直观，但 contains 和 contains (with wildcards) 实际上都支持通配符。这两种过滤器的区别在于：contains 允许使用两个正斜杠（\\）对通配符进行转义，而 contains (with wildcards) 不支持通配符转义。例如，值为 apples\\*pears 的 contains 过滤器会匹配字符串 apples*pears，但不会匹配字符串 apples_pears。反之，如果使用 contains (with wildcards) 过滤器，则无法构造出匹配 apples*pears 但不匹配 apples_pears 的条件，因为 contains (with wildcards) 过滤器始终将 * 视为通配符。

由于 contains 和 contains (with wildcards) 之间的这种行为差异可能不太明显，依赖此行为的 Contour 分析过滤器逻辑往往难以理解和维护。若需表达复杂的字符串匹配条件，建议使用 matches（正则表达式）过滤器，而不是使用带通配符转义的 contains 过滤器。
