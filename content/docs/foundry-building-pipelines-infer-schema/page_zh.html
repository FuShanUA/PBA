为 CSV 或 JSON 文件推断模式

在 Foundry 中，如果数据集具备模式，处理起来会最为简便。Foundry 允许你通过在数据集中选择 Apply a schema 按钮来手动为包含 CSV 或 JSON 文件的数据集添加模式。Apply a schema 按钮会根据部分数据自动推断出模式。应用模式后，在数据集视图中选择 Edit schema 即可修改列类型，或应用其他解析选项来丢弃不规则行、更改编码，亦或添加额外列（如文件路径、行的字节偏移量、导入时间戳或行号）。

如果数据发生变动，基于初始数据集文件静态应用的模式可能会过时。因此，让 Spark 在半结构化数据的转换管道第一步中动态推断模式，往往大有裨益。

请注意，在每次构建管道时动态推断模式会产生性能开销，因此应谨慎使用该技术（例如，仅在实际模式可能发生变化时使用）。

以下是 CSV 和 JSON 输入的示例。

Parquet 是 Transforms 的默认输出文件格式，它不允许自动推断出的模式中包含某些特殊字符。因此，建议你像下方示例中那样使用 sanitize_schema_for_parquet 来防止潜在问题。

除了动态模式推断外，像下方示例中那样使用 SparkSession.read 读取全部或部分数据集文件还有许多其他用例。如果你的用例实际上并不需要动态模式推断行为，则应将其禁用：既可以将 inferSchema 设为 False（这会导致所有列被解析为字符串），也可以忽略该选项并显式传递模式。禁用自动模式推断将显著提升性能和一致性，尤其是在增量管道中，因为不同增量批次间模式推断结果不一致可能会引发问题。

8def read_csv(ctx, raw, output):
9    filesystem = raw.filesystem()
10    hadoop_path = filesystem.hadoop_path
11    files = [f"{hadoop_path}/{f.path}" for f in filesystem.ls()]
12    df = (
13        ctx
14        .spark_session
15        .read
16        .option("encoding", "UTF-8")  # UTF-8 is the default
17        .option("header", True)
18        .option("inferSchema", True)
19        .csv(files)
20    )
21    output.write_dataframe(sanitize_schema_for_parquet(df))
