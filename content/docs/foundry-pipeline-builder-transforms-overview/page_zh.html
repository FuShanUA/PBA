转换

Pipeline Builder 为在 Foundry 中转换数据提供了一个灵活、强大且易于使用的界面。对于非编程人员和经验丰富的软件开发者而言，使用现有工具（例如 Spark 或 SQL）编写数据转换往往颇具挑战且容易出错。此外，现有工具通常绑定于特定的执行引擎，并要求使用代码库来表达数据转换。

Pipeline Builder 采用通用模型来描述数据转换。该后端充当用于编写转换的工具与实际执行转换之间的中间层。

Pipeline Builder 的底层架构旨在支持各类输出结果——数据集、本体对象、流、时间序列以及导出至外部系统。你可以针对数据集、对象类型、链接类型运行批处理管道，也可以运行对应流式数据集的流式管道。

在 Pipeline Builder 中使用转换

在 Pipeline Builder 中，你可以使用两种类型的数据转换：表达式和转换。表达式将表中的列作为输入并输出单列（例如 Split string），而转换将整个表作为输入并返回整个表（例如 Pivot 或 Filter）。

我们将表达式和转换归拢在同一配置界面中。例如，你可以在 Cast 和 Concatenate strings 等表达式旁边找到 Drop columns 转换。如此一来，你便可在同一路径中结合使用表达式与转换，并在单一配置表单内将表达式嵌入转换之中，正如以下将 Length 表达式插入 Filter 转换所示。

其他用于数据结构化的转换，即 Join 和 Union，拥有独立的配置面板，并在 Pipeline Builder 界面中标有专属图标。

为简便起见，我们通常将所有类型的数据转换统称为转换。

Join

联接（Join）用于合并两个至少包含一个匹配列的数据集。根据你配置的联接类型，联接输出可以合并匹配行并排除非匹配行。

Union

合并（Union）用于合并两个数据集以包含所有行。

合并转换要求所有输入具有相同的模式。如果输入模式未能全部匹配，合并操作将显示错误信息并列出缺失的列。

用户定义函数

如果无法通过现有的转换选项处理数据，或者有需要在各管道间复用的复杂逻辑，你可以创建用户定义函数（UDF）。用户定义函数允许你在 Pipeline Builder 中运行可进行版本控制和升级的自定义代码。

注意：我们建议使用 Python 函数以获得最佳体验。如果需要访问特定的 Java 库，也支持使用 Java UDF。

仅在必要时才应使用用户定义函数。建议在可能的情况下，使用 Pipeline Builder 中经过优化的 转换面板。
