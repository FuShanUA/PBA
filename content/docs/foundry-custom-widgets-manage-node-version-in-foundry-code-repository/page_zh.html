在 Foundry 代码仓库中管理 Node.js 版本

Foundry 代码仓库支持 Node Version Manager (nvm) ↗ 来管理项目和 CI 中使用的 Node.js 版本。通过在仓库根目录添加一个 .nvmrc 文件，您可以指定项目使用的 Node.js 版本。

如果您不使用 nvm，CI 将使用 .jemma/settings.sh 中指定的 Node.js 版本，而 Foundry 开发环境将使用版本 18.20.8。

添加一个 .nvmrc 文件

要为您的代码仓库设置 Node.js 版本，请在项目根目录下创建一个名为 .nvmrc 的文件。此文件应仅包含您想要使用的 Node.js 版本号，前面加上 v，例如：

v24.13.0

您可以通过在代码仓库内的终端运行 nvm ls-remote 来查看可用的 Node.js 版本。

使用指定版本

在添加了 .nvmrc 文件后，在终端中运行以下命令以切换到文件中定义的版本：

Copied!
1
nvm install

配置 CI 使用 .nvmrc 版本

为了确保您的 CI 构建使用您 .nvmrc 文件中指定的相同 Node.js 版本，请更新您仓库中的 .jemma/settings.sh 文件。

将现有的 NODE_INSTALLATION_VERSION 导出替换为以下行，该行直接从您的 .nvmrc 文件中读取版本：

Copied!
1
export NODE_INSTALLATION_VERSION=$(sed 's/^v//' "$(dirname "${BASH_SOURCE[0]}")/../.nvmrc")

通过此更改，您只需更新 .nvmrc 文件即可更改 Foundry 开发和 CI 环境中的 Node.js 版本。
