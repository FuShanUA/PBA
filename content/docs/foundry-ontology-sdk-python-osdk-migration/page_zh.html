Python OSDK 升级指南（1.x 到 2.x）

本指南旨在帮助您将使用 Python OSDK 1.x 的应用程序迁移到 Python OSDK 2.x。以下各节解释了两个版本之间的差异，并突出了语法和结构的相关更改。

Python OSDK 文档也可以在平台内的 Developer Console 中找到，位于 /workspace/developer-console/ 处。使用版本选择器来选择 2.x 以获取有关 Python OSDK 2.x 的文档。

为什么从 Python OSDK 1.x 升级到 2.x？

有四个主要原因可以升级从 Python OSDK 1.x 到 2.x：

  • 用于 Python Platform SDK 和 OSDK 的共享客户端
  • 查询、链接、聚合和附件上传的改进返回类型
    • 查询、关联、聚合和附件上传的返回类型得到改进
  • 能够编辑您的客户端配置
  • 测试功能

用于 Python 平台 SDK 和 OSDK 的共享客户端

现在，使用 OSDK 和平台 SDK 进行开发不再需要用户管理两个独立的客户端。现在可以通过一个客户端访问这两个 SDK。

查询、关联、聚合和附件上传的返回类型得到改进

  • 查询可以返回对象、对象集合、附件、二维聚合、三维聚合和映射类型。
    • 在 Python OSDK V1 中，返回类型为对象、对象集合和附件的函数查询将返回这些类型的对象标识符。例如，如果一个查询返回一个对象，在 V1 中，该查询实际上会返回对象的主键，而不是对象的实例。现在，查询会返回该类型的实例。
    • Python OSDK V1 不支持二维聚合、三维聚合或映射类型。这些现在在 OSDK V2 中已得到支持。
  • 关联返回对象集合或 Optional[Objects]
    • 一对多/多对多类型的关联现在返回对象集合，而一对一/多对一类型的关联返回 Optional[Objects]。
    • 对于一对一/多对一类型的关联，用户不再需要使用 .get 来检索关联的对象。关联本身将成为对象。
    • 对于一对多/多对多类型的关联，所有可用于对象集合的方法（如 .page、.where 或 .group_by）现在也适用于这些关联。在 Python OSDK V1 中，只有 .iterate 和 .get 方法可用。
  • 聚合返回一个 AggregateObjectsResponse
    • 在 Python OSDK V1 中，聚合响应的类型为 dict[str, Any]，其中键包括排除项、准确性、数据等字段。在 Python OSDK V2 中，聚合现在返回一个 AggregateObjectsResponse 的实例，其字段与 V1 的字典类型相同，但具有更好的类型安全性。Python OSDK V2 中的 AggregateObjectsResponse 具有一个 .to_dict 方法，将返回与 V1 相同的响应。
    • 在 Python OSDK V1 中执行“按范围分组”操作时，需要输入一个字典列表，每个字典的键为 "start_value" 和 "end_value"，每个字典的值为范围的起始和结束值。在 V2 中，输入类型现在是一个元组集合，元组的第一个元素被推断为 "start_value"，第二个元素为 "end_value"。
  • 附件上传返回一个 Attachment
    • 在 Python OSDK V1 中，client.ontology.attachments.upload(...) 返回一个 AttachmentMetadata. 在 V2 中，上传附件返回一个 Attachment。

编辑客户端配置的能力

在 Python OSDK V1 中，发送请求时没有简单的方法来编辑客户端配置。例如，如果用户想要编辑其请求的 user-agent，他们必须访问客户端上的保留 ._session 方法才能做到这一点。为了避免这个问题，Python OSDK V2 提供了一个 Config 类，用户可以导入并将其传递给他们的客户端。

测试功能

Beta特性可能会发生变化，其语法、稳定性和存在性均无法保证。Python OSDK V2的稳定版本包含Beta特性，使您可以尝试并提供反馈。要使用Beta特性，您必须从AllowBetaFeatures导入foundry_sdk_runtime，并在其上下文中编写代码，或者用@AllowBetaFeatures()装饰您的代码。在使用Foundry Python函数时，@function装饰器包含一个beta布尔值。

4    <your code>

如何从 Python OSDK 1.x 升级到 2.x

您可以通过在应用程序的SDK 版本选项卡中的生成新版本选项（通过该选项可以访问）使用 Python OSDK 2.x 生成器为现有的 Developer Console 应用程序生成新的 SDK 版本。

现有应用程序也必须具有使用新生成的 SDK 所需的依赖项。在 Developer Console 中创建的新应用程序默认会使用 Python OSDK 2.x 生成器生成 SDK。您可以通过遵循开始开发选项卡中的说明快速入门。

从 Python OSDK 1.x 到 2.x 的语法转换和返回类型变化

本节包含简单的示例，说明如何在 Python OSDK 1.x 和 2.x 之间进行映射。

筛选

  • 在引用被筛选的属性时，必须在.object_type表达式中包含where。
  • .is_member_of 已重命名为 .in_。
  • .starts_with 已重命名为 .contains_all_terms_in_order_prefix_last_term。

使用 Python OSDK 1.x 进行筛选（旧版）

Copied!
1client.ontology.object.where(Object.id.is_member_of([1, 2, 3])).take(1)

使用 Python OSDK 2.x 进行筛选

Copied!
1client.ontology.object.where(Object.object_type.id.in_([1, 2, 3])).take(1)

按组

在 OSDK V1 中，当按范围分组时，接受的输入是一个字典列表，每个字典的键为 start_value 和 end_value，值为每个范围的起始和结束。在 V2 中，新的输入类型是一组元组，其中元组的第一个元素是 start_value，第二个元素是每个范围的 end_value。

6    )
7)

使用 Python OSDK 2.x 按范围分组

Copied!
1from ontology_sdk.ontology.objects import Object
2
3client.ontology.objects.Object.group_by(
