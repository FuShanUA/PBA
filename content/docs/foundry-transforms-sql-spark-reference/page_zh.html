Spark SQL 参考指南

本部分介绍了编写 Spark SQL 数据转换与其他类型 SQL 查询之间的一些关键差异，并提供了可用的 Spark SQL 函数列表。

提示

在查阅相关资源时请注意，Spark SQL 实际上是基于 HiveQL 方言的。你可以在网上找到关于 HiveQL 的更多信息。

入门指南

基本查询格式

每个 SQL 数据转换查询都必须创建一个表。SQL 查询的一般格式如下：

Copied!
1CREATE TABLE _____ AS SELECT _____

不要在语句末尾添加分号。包含分号将导致错误。

注释语法

你可以像这样在 SQL 代码中添加注释：

已复制！
1-- 你可以使用此语法来创建注释
2-- 
3
4/* 你也可以使用此语法
5   来创建注释 */

引用数据集

要引用数据集，请提供用反引号包围的数据集路径：



已复制！
1CREATE TABLE `/path/to/target/dataset` AS
2SELECT * FROM `/path/to/source/dataset`
3
4-- 备用语法
5
6CREATE TABLE `/path/to/target/dataset` AS (
7    SELECT * FROM `/path/to/source/dataset`
8)

请注意，数据集名称区分大小写。

引用列

若要引用数据集中的特定列，请提供列名：

Copied!
1SELECT Name FROM `/path/to/source/dataset`

请注意，列名区分大小写。

派生列与别名

派生列是对数据集列调用函数后生成的结果。您必须为任何派生列定义别名：



已复制！
1SELECT Sum(Val) AS Total FROM `/path/to/source/dataset`

以下查询将导致错误：



已复制！
1SELECT Sum(Val) FROM `/path/to/source/dataset`

请注意，别名称是区分大小写的。

SQL 子句中的派生列

在 WHERE 和 GROUP BY 等 SQL 子句中，无法使用别名。因此，在 WHERE 或 GROUP BY 子句中，必须引用实际的函数和数据集列：



已复制！
1SELECT Lower(Name) AS LowercaseName
2FROM `/path/to/source/dataset` WHERE Lower(Name) = "sara"
3
4SELECT Lower(Name) AS LowercaseName, Sum(Val) AS Total
5FROM `/path/to/source/dataset` GROUP BY Lower(Name)

你可以在以下 SQL 子句中使用别名：ORDER BY 和 HAVING。因此，以下任何查询均可生效：



已复制！
1-- 以下两个查询均有效
2
3SELECT Lower(Name) AS LowercaseName, Sum(Val) AS Total
4FROM `/path/to/source/dataset` GROUP BY Lower(Name) ORDER BY Total
5
6SELECT Lower(Name) AS LowercaseName, Sum(Val) AS Total
7FROM `/path/to/source/dataset` GROUP BY Lower(Name) ORDER BY Sum(Val)
8
9-- 以下两个查询均有效
10
11SELECT Lower(Name) AS LowercaseName, Sum(Val) AS Total
12FROM `/path/to/source/dataset` GROUP BY Lower(Name) HAVING Total > 100
13
14SELECT Lower(Name) AS LowercaseName, Sum(Val) AS Total
15FROM `/path/to/source/dataset` GROUP BY Lower(Name) HAVING Sum(Val) > 100

类型转换

你可以对表达式进行强制类型转换，将其从一种类型转换为另一种类型。类型转换的语法如下：

Copied!
1CAST(expr AS <TYPE>)

请注意，expr 表示需要转换的表达式，<TYPE> 表示要将表达式转换成的目标类型。如果 CAST(expr` `AS` `<TYPE>) 转换失败，将返回 null。<TYPE> 的可用值包括：

  • boolean
  • tinyint
  • smallint
  • int
  • bigint
  • float
  • double
  • decimal
  • date
  • timestamp
  • binary
  • string

日期格式化

你可能需要重新格式化日期值，或者将字符串转换为日期格式。date 的格式为 yyyy-MM-dd，且不包含时间部分。你可以使用 CAST 函数并结合可用的日期时间函数将字符串转换为日期。以下是一些简单的示例：



已复制！
1-- 将格式为 'yyyy-MM-dd' 的字符串转换为日期
2CAST('2016-07-30' AS DATE)
3
4-- 将格式为 'yyyy-MM-dd' 的字符串转换为时间戳（时间戳将基于所提供日期的午夜）
5CAST('2016-07-30' AS TIMESTAMP)
6
7-- 将格式为 'ddMMyyyy' 的日期字符串转换为日期
8TO_DATE(CAST(UNIX_TIMESTAMP('07302016', 'MMddyyyy') AS TIMESTAMP))
9
10-- 将格式为 'ddMMyyyy HH:mm:ss' 的时间戳字符串转换为仅包含日期的格式
11CAST('2016-07-30 11:29:27' AS DATE)
12
13-- 从 ISO 8601 时间戳中仅提取日期
14TO_DATE('2016-07-30T11:29:27.000+00:00')
15
16-- 从格式为 'yyyy-MM-dd HH:mm:ss' 的时间戳中仅提取日期
17TO_DATE('2016-07-30 11:29:27')

请注意，如果字符串未格式化为日期/时间戳，则无法将其转换为日期。因此，诸如 CAST('20160730' AS DATE) 之类的操作将返回 null。

WINDOW


描述：
根据指定时间戳的列生成滚动或滑动时间窗口。若未提供 slideDuration，则生成滚动时间窗口；若提供了 slideDuration，则将数据行分桶至一个或多个时间窗口中，从而生成滑动时间窗口。


窗口起始边界为闭区间，窗口结束边界为开区间，因此 12:05 会落入窗口 [12:05, 12:10) 内，但不会落入 [12:00, 12:05) 内。窗口支持微秒级精度，但不支持以月为单位的窗口。


除非提供了 startTime，否则窗口均从 1970-01-01 00:00:00 UTC 起开始计算。
该函数接受以下参数：


    

  • timeColumn：用作按时间划分窗口的时间戳列。该时间列必须为 TimestampType 类型。
    • 

  • windowDuration：用于指定窗口宽度的字符串，例如 10 minutes、1 second。需要注意的是，此持续时间是固定的时间长度，不会随日历时间而变化。
    • 

  • slideDuration：用于指定窗口滑动间隔的字符串，例如 1 minute。每经过一个 slideDuration 就会生成一个新窗口。该值必须小于或等于 windowDuration。同样地，此持续时间也是绝对值，不会随日历而变化。
    • 

  • 

startTime: 相对于 1970-01-01 00:00:00 UTC 的偏移量，用于设置窗口间隔的起始时间。例如，若要让每小时滚动窗口在过整点 15 分钟后开始（例如 12:15-13:15，13:15-14:15…），请将 startTime 设为 15 分钟。
    • 



函数签名：


Copied!
1WINDOW(Column timeColumn, String windowDuration)WINDOW(Column timeColumn, String windowDuration, String slideDuration)WINDOW(Column timeColumn, String windowDuration, String slideDuration, String startTime)

CRC32计算二进制列的循环冗余校验值，并将该值作为 bigint 返回。
GREATEST返回值列表中的最大值，跳过空值。
HASH计算给定列的哈希码，并将结果作为 int 列返回。
HEX计算给定列的十六进制值。
ISNAN当且仅当该列为 NaN 时返回 true。
LEAST返回值列表中的最小值，跳过空值。
MD5计算二进制列的 MD5 摘要，并将该值作为 32 字符的十六进制字符串返回。
NANVL如果 col1 不是 NaN 则返回 col1，如果 col1 是 NaN 则返回 col2。
NOT布尔表达式的反转。
SHA1计算二进制列的 SHA-1 摘要，并将该值作为 40 字符的十六进制字符串返回。
SHA2计算二进制列的 SHA-2 系列哈希函数，并将该值作为十六进制字符串返回。
WHEN评估一系列条件，并返回多个可能的结果表达式之一。


CRC32


描述：
计算二进制列的循环冗余校验值 (CRC32)，并将该值作为 bigint 返回。


签名：


Copied!
1CRC32(Column e)


GREATEST




描述：
返回输入值列表中的最大值，计算过程中会跳过空值。该函数使用“>”运算符进行值比较。此函数至少需要 2 个参数。仅当所有参数均为空值时，函数才会返回空值。


签名：


Copied!
1GREATEST(Column... exprs)


HASH


描述：
计算给定列的哈希码，并将结果作为整型列返回。


签名：




已复制！
1HASH(Column... cols)


HEX


描述：
计算指定列的十六进制值。如果 column 为 int 或 binary 类型，则以十六进制格式的字符串形式返回该数字。如果 column 为 string 类型，则将每个字符转换为其对应的十六进制表示形式，并返回生成的字符串。


签名：




已复制！
1HEX(Column column)


ISNAN


描述：
当且仅当该列为 NaN 时返回 true。


函数签名：


已复制！
1ISNAN(Column e)




LEAST


描述：
返回值列表中的最小值，跳过空值。使用“<”运算符比较值。该函数至少接收 2 个参数。当且仅当所有参数均为空时，返回空值。


签名：


Copied!
1LEAST(Column... exprs)


MD5


描述：
计算二进制列的 MD5 摘要，并以 32 字符的十六进制字符串形式返回该值。


签名：




已复制！
1MD5(Column e)


NANVL


描述：
如果 col1 不是 NaN，则返回 col1；如果 col1 是 NaN，则返回 col2。


函数签名：


已复制！
1NANVL(Column col1, Column col2)



NOT


描述：
对布尔表达式进行取反操作（即 NOT）。


签名：


已复制！
1NOT(Column e)


SHA1


描述：
计算二进制列的 SHA-1 摘要，并以 40 字符的十六进制字符串形式返回结果值。


签名：




已复制！
1SHA1(Column e)


SHA2


描述：
计算二进制列的 SHA-2 系列哈希函数，并以十六进制字符串的形式返回结果值。


签名：




已复制！
1SHA2(Column e, int numBits)


WHEN


描述：
评估一系列条件，并返回多个可能的结果表达式之一。


签名：



WHEN(Column condition, Object value)
