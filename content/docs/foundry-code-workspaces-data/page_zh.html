与数据交互

Jupyter® 和 RStudio® 代码工作区支持读取、分析、转换数据，并将其写回至 Foundry 数据集、Iceberg 表和虚拟表。此外，还支持读取和分析受限视图与时序属性。

安全

代码工作区会遵循应用于数据集的 Foundry 标记，工作区将继承加载到其中的所有数据集的标记。这被称为工作区血缘关系。这意味着，要访问某个工作区，您必须同时具备对该工作区内包含的所有数据集及其他输入所需的权限。如果失去对工作区中任一输入的访问权限，您将失去对整个工作区的访问权限。

资源考量

Jupyter® 和 RStudio® 工作区专为交互式分析工作流和应用开发设计。它们在单台虚拟机上运行，无法在分布式 Spark 环境中处理数据集。因此，您处理的数据必须适应机器的资源配置，默认最大为 8 个 CPU 和 64 GB 内存。在加载大型数据集之前，使用过滤器或 SQL 减少所需的数据量，有助于避免超出这些限制。过滤数据集文件是另一种将使用量控制在工作区限制内的策略。

要开发大规模数据管道，请使用基于 Spark 的 Python 转换或管道构建器。

数据集分支

默认情况下，代码工作区会从与工作区相同的分支加载数据集数据，否则将回退到 master 分支。例如，当前位于 my-branch 分支的代码工作区会尝试读取数据集的 my-branch 版本，如果数据集上不存在 my-branch，则回退到 master 分支。

要为特定导入的数据集固定数据集分支，请在左侧边栏的数据选项卡中单击该数据集旁边的•••选项图标。然后，选择固定读取的数据集分支，并为代码工作区内的使用选择所需的数据集分支。

读取数据

对于每个导入到工作区的 Foundry 数据集或受限视图，代码工作区要求您选择一个别名。该别名充当引用句柄，允许您在代码中读取数据集或受限视图并写入数据集。在数据选项卡中注册数据集时，代码工作区会将所选的数据集别名与 Foundry 资源的唯一标识符进行映射，该映射保存在工作区 /home/user/repo/.foundry 文件夹下的一个隐藏文件中。

代码工作区允许您加载表格型数据集、非表格型数据集、Iceberg 表、虚拟表和受限视图。您可以通过数据选项卡中的添加 > 读取数据按钮向工作区添加新的数据源。

选择要添加到工作区的数据集后，您必须为该资源定义一个别名，作为代码中唯一的标识符和引用。为数据集选择读取策略（例如 pandas DataFrame、Polars LazyFrame 或原始文件访问）将生成一段代码片段，将数据集作为指定的输入类型加载。

  • 您可以随时修改此读取策略并生成新的代码片段，即使数据集已经注册完毕。
  • 默认情况下，代码工作区会建议使用与数据集本身名称相同的数据集别名。

选择完成将结束数据集注册流程，您可以通过运行代码片段或复制到剪贴板选项在代码中使用这些数据。

本页提供了关于表格型数据集、非表格型数据集和受限视图的示例。

使用 SQL 查询表格型数据集

在 Python 中，你可以使用 SQL 查询任何表格型数据集、支持 SQL 查询的 Iceberg 表或虚拟表。

使用以下方法，每条 SELECT Spark SQL 语句只能查询一个数据集。查询结果将以 PyArrow 表的形式返回，你可以将其转换为其他格式，例如 pandas DataFrame。

在 Jupyter® 中：

受限视图

仅 Jupyter® 工作空间支持查询受限视图。

你可以使用 Python，通过《上述 SQL 方法》来查询受限视图。具体操作是将受限视图导入工作空间，并像处理其他表格数据集一样通过其别名进行查询。

请注意，将受限视图导入代码工作区需启用受限输出模式，以强制执行更高水平的数据安全。启用受限输出模式后，您可以对受限视图中的数据进行分析，但无法发布使用该数据的工作区输出。

首次向工作区添加受限视图时，系统会提示您安装特定的 Python 依赖项。随后，您可以定义别名（与数据集操作相同）并将受限视图添加到工作区。您必须启用受限输出模式并重启工作区，才能查询该受限视图。

受限视图的策略决定了用户可以查询的数据。例如，如果您将受限视图导入工作区，而另一名用户打开了该工作区，他们看到的数据可能与您不同。如果在受限视图加载到工作区期间您的访问权限发生变更，您将失去对该工作区的访问权限，需要重新启动工作区。

筛选数据集文件

代码工作区支持您下载支撑任何数据集的文件，无论是表格型（即具有架构）还是非表格型均可。您可以选择下载部分文件，方式包括按名称选择，或者对文件元数据（路径、字节大小、事务 RID 和更新时间）应用筛选逻辑。

在 Jupyter® 中：



已复制！

2
3# 下载数据集中的所有文件

5local_file = downloaded_files["file.pdf"]
6

8local_file = Dataset.get("my_alias").files().get("file.pdf").download()
9
10# 下载数据集中小于 1MB 的所有 PDF 文件
11downloaded_files = my_dataset.files()\
12    .filter(lambda f: f.path.endswith(".pdf") and f.size_bytes < 1024 * 1024)
13    .download()

筛选表格型数据集

Code Workspaces 支持在将数据集加载到内存之前对其进行筛选。这样可以减少导入工作空间的数据框所占用的内存，让你能够专注于与分析相关的数据子集。Code Workspaces 提供了灵活的操作方式，支持对列、行或两者同时进行筛选。

何时对表格型数据集使用筛选

为了确保工作空间的使用量不超出可用资源限制，我们提出以下建议：

  • 若未经压缩的数据量在工作空间的内存容量范围内，可以直接加载未加筛选的数据集，并在内存中执行数据转换，以实现最高效率。

  • 若未经压缩的数据量超出了工作空间的内存容量，可以使用列筛选和行筛选，仅将部分数据加载至内存。此类下推筛选会在数据加载至工作空间之前执行，从而有效降低导入数据的内存占用。筛选操作的执行速度取决于数据的规模与分区方式。此外，只要最终生成的结果数据不超过计算机内存限制，你也可以使用结合 Python 的 SQL 来查询任意规模的表格型数据集。

    • 若此方案无法满足你的业务需求，还可以采用基于 Spark 的应用程序处理数据，生成体量较小的数据集，随后在 Code Workspaces 中加以使用。这种方法通常能够加速交互式工作流，尤其适用于频繁调用同一数据子集或跨多个工作空间使用的场景，因为数据集中的 Parquet 文件仅需下载一次，而每次将数据加载至工作空间时则会重复执行筛选操作。

  • 面对更为复杂的数据加载需求，你可以通过非表格型数据集语法下载数据集对应的文件，并使用原生的 Python 或 R 包来处理文件内容。

下方列出了 Jupyter® 代码工作区支持的更多行过滤语法示例：


```python
# 仅保留等于特定值的行
.where(Column.get("column_name") == value)

# 仅保留不等于特定值的行
.where(Column.get("column_name") != value)

# 使用 ~ 运算符进行不等于判断
.where(~(Column.get("column_name") == value))

# 仅保留值可与另一个值进行比较的行
.where(Column.get("column_name") > value)
.where(Column.get("column_name") >= value)
.where(Column.get("column_name") < value)
.where(Column.get("column_name") <= value)

# OR / AND 运算符
.where((Column.get("column_name") == value1) | (Column.get("column_name") == value2))
.where((Column.get("column_name1") == value1) & (Column.get("column_name2") == value2))

# 仅保留值不为空的行
.where(~Column.get("column_name").isnull())

# 仅保留值属于给定列表的行
.where(Column.get("column_name").isin([value1, value2, value3]))

# 仅保留日期在两个包含边界值之间的行
.where(Column.get("date_column_name").between('lower_bound_incl', 'upper_bound_incl'))

# 仅保留前 N 行，其中 N 为数字。此操作会在其他过滤器之前应用
.limit(N)

# 选择列的子集
.select("column_name1", "column_name2", "column_name3")
```

事务类型


在以交互方式回写数据时，默认情况下，每次 SDK 函数调用都对应一个事务：


    

  • 回写表格型数据时（Python 中的 output_dataset_tabular.write_table(df_variable) 或 R 中的 datasets.write_table(df_variable, "output_dataset_tabular")），将创建一个 SNAPSHOT 事务。
    • 

  • 回写文件时（Python 中的 output_dataset_non_tabular.upload_directory(path_to_file_variable) 或 R 中的 datasets.upload_files(path_to_file_variable, "output_dataset_non_tabular")），将创建一个 UPDATE 事务。
    • 



脚本注册为转换任务后，交互式调用将开始写入以 code-workspace-sandbox/ 为前缀的分支，而转换任务运行时会更新当前分支。此时，即使有多个 SDK 函数调用，整个脚本执行过程也只会创建一个事务：


    

  • 默认情况下，事务类型为 SNAPSHOT。
    • 

  • 如果已配置增量设置，则事务类型为 APPEND。
    •
