与数据交互

Jupyter® 和 RStudio® 代码工作区允许您读取、分析、转换并写回 Foundry 数据集，Iceberg 表 和 虚拟表。它们还允许您读取和分析 受限视图 和 时间序列属性。

安全

代码工作区尊重应用于数据集的 Foundry 标记，并且工作区将继承所有加载到其中的数据集的标记。这被称为 工作区谱系。这意味着要访问一个工作区，您还必须拥有工作区中包含的所有数据集和其他输入的相应权限。如果您失去了工作区中任何一个输入的访问权限，您将失去对整个工作区的访问权限。

资源考虑因素

Jupyter® 和 RStudio® 工作区设计用于交互式分析工作流和应用开发。它们运行在单个虚拟机上，并不使用分布式 Spark 环境处理数据集。因此，您所处理的数据必须适合您的机器资源，默认最大为 8 个 CPU 和 64 GB 内存。在加载大型数据集之前，使用 过滤器 或 SQL 来减少所需数据量，可以帮助您避免超出这些限制。过滤数据集文件 是另一种保持在工作区限制内的策略。

要开发大规模数据管道，请使用 Python 转换（Spark） 或 管道构建器。

数据集分支

默认情况下，代码工作区将从与工作区本身相同的分支加载数据集的数据，并在其他情况下回退到master分支。例如，当前位于分支my-branch的代码工作区将尝试读取数据集的my-branch版本，如果数据集上不存在my-branch，则会回退到master分支。

要为特定导入的数据集固定分支，请在左侧边栏的数据选项卡中选择数据集旁边的选项•••图标。然后，选择固定数据集分支以供读取，并选择要在代码工作区中使用的所需数据集分支。

读取数据

代码工作区要求您选择一个别名 对于您导入到代码工作区中的每个Foundry数据集或受限制的视图。别名作为引用，使您可以在代码中从数据集或受限制的视图读取并写入到数据集中。在注册数据集时，在Data（数据）选项卡中，代码工作区会在工作区下/home/user/repo/.foundry文件夹中的隐藏文件中创建所选数据集别名与Foundry资源的唯一标识符之间的映射。

代码工作区允许您加载表格数据集、非表格数据集、Iceberg表、虚拟表和受限制的视图。您可以通过在Data（数据）选项卡中的Add > Read data（添加 > 读取数据）按钮将新的数据源添加到您的工作区。

选择要添加到工作区的数据集后，您必须为该资源定义一个别名，作为代码中的唯一标识符和引用。为数据集选择读取策略（如pandas DataFrame、Polars LazyFrame或原始文件访问）会生成一个代码片段，用于将数据集作为指定输入类型加载。

  • 您可以随时修改此读取策略并生成新的代码片段，即使在注册数据集之后也可以这样做。
  • 默认情况下，代码工作区将建议一个与数据集同名的数据集别名。

选择完成将完成数据集注册过程，并允许您使用运行代码片段或复制到剪贴板选项在您的代码中使用数据。

此页面提供了以下内容的示例：表格数据集、非表格数据集以及受限视图。

表格数据集

以下代码片段是为一个Cats表格数据集和一个数据集别名kittens生成的。请注意，Cats在代码片段中任何地方都没有被引用；代码工作区会根据您选择的别名隐式地进行注册。

在 Jupyter® 中：

Copied!
1from foundry.transforms import Dataset
2
3kittens = Dataset.get("kittens")\
4  .read_table(format="arrow")\
5  .to_pandas()

有关 Jupyter® 工作区中使用的 Dataset 类的完整方法和属性列表，请参阅 foundry.transforms.Dataset API 参考。

请注意，上面显示的 read_table 方法支持以下参数：

  • arrow（推荐）：将数据集转换为 Apache Arrow 表，可以在其上执行高效的过滤。然后可以使用 .to_pandas() 将此表转换为 pandas 数据框。
  • pandas 或 dataframe：将数据集转换为 pandas 数据框。
  • polars：将数据集转换为 Polars 数据框。然后可以使用 .to_pandas() 将其转换为 pandas 数据框。
  • lazy-polars：Polars 数据框的惰性变体。 过滤 无法在惰性 polars 上执行。
  • path：输出数据集存储的本地路径。

要使某种格式成为 read_table 操作的一部分，相应的包 pyarrow、pandas 和 polars 必须存在于环境中。这些包在默认的代码工作区环境中会自动包含，但可能需要手动添加到您的自定义环境中。

在 RStudio® 中：

Copied!
1kittens <- datasets.read_table("kittens")

上述语法加载数据集并自动将数据收集到R数据框中。

如果数据超过工作区的内存容量，您可以应用下推过滤器，仅使用以下语法加载行或列的子集：

2

4  # (optional) apply other transformations before collecting

通过 SQL 查询表格数据集

您可以通过 SQL 在 Python 中查询任何表格数据集、Iceberg 表 或 虚拟表支持 SQL 查询。

以下代码片段是为一个 Ducks 表格数据集和数据集别名 ducklings 生成的。请注意，在代码片段中没有引用 Ducks；Code Workspaces 会根据您选择的别名隐式注册它。

您只能使用以下方法在每个 SELECT Spark SQL 语句中查询一个数据集。查询结果返回为一个 PyArrow 表 ↗，然后您可以将其转换为其他格式，例如 pandas 数据框。

  • 请注意，默认的SQL查询限制是一百万行。如果您需要一次查询超过一百万行，可以并行化单独的查询并合并其结果。确保您的数据可以适合在工作区的内存中。

在 Jupyter® 中：

安装 containers-sql 和 foundry-platform-sdk 包后，您可以使用 FoundrySdkSqlExecutor 来编写 Spark SQL：

Copied!
1from containers_sql import FoundrySdkSqlExecutor
2sql = FoundrySdkSqlExecutor()
3query = sql.execute("SELECT * from `ducklings`")
4df = query.fetch_results() # optionally specify timeout arg: query.fetch_results(timeout_in_seconds=60)

您还可以通过首先在笔记本顶部运行以下内容来使用 魔法命令 ↗：

Copied!
1import containers_sql
2%reload_ext containers_sql.foundry_sdk.magics

然后，使用 %sql 魔法命令来运行单行查询，或使用 %%sql 来运行多行查询。使用 %%sql -o df 会将您的 SQL 查询结果赋值给 Python 对象 df，作为 PyArrow 表：

Copied!
1ducklings_df = %sql SELECT hatch_date, bill_color, * from `ducklings`
2ducklings_pandas_df = ducklings_df.to_pandas()
Copied!
1%%sql -o ducklings_df
2SELECT
3    hatch_date, bill_color, *
4FROM
5    `ducklings`
6LIMIT 10;

在 RStudio® 中：

首先安装 containers-sql、foundry-platform-sdk、pyarrow 和 reticulate。然后使用 reticulate 包与上述 Python SQL 执行器一起查询数据集：

Copied!
1library(reticulate)
2
3FoundrySdkSqlExecutor <- import("containers_sql")$FoundrySdkSqlExecutor
4sql <- FoundrySdkSqlExecutor()
5query <- sql$execute("SELECT * from `ducklings`")
6results_df <- query$fetch_results()
7
8# For smaller data, creating an intermediate R object is acceptable.
9r_list <- py_to_r(results_df$to_pydict())
10tibble <- as_tibble(r_list)
11
12# For larger data, you may want to write your data to disk
13# to avoid exceeding your container's memory limits.
14library(arrow)
15pa <- import("pyarrow")
16pa$feather$write_feather(results_df, "tmp.arrow")
17tibble <- read_feather("tmp.arrow") |> as_tibble()

非表格数据集

以下代码片段是为一个 Dogs 非表格数据集生成的，该数据集的别名是 puppies。请注意，Dogs 在代码片段中任何地方都没有被引用；代码工作区会自动将其注册到您选择的别名下。与读取表格数据集不同，这种方法使您可以通过一个 puppies_files 变量访问数据集中的文件，而不是将值插入到数据框中。

5with open(puppies_files["puppy_names.txt"], "rb") as f:
6    puppy_names_text = f.read().decode("utf-8")

在 RStudio® 中：

1puppies_files <- datasets.list_files("puppies")
2puppies_local_files <- datasets.download_files("puppies", puppies_files)
3puppy_names_text <- readLines(raw_files_local_files[["puppy_names.txt"]], warn = FALSE)

受限制的视图

仅在 Jupyter® 工作区中支持查询受限制的视图。

您可以使用 上述 SQL 方法 通过 Python 查询受限制的视图。为此，将受限制的视图导入您的工作区，并像查询其他表格数据集一样通过其别名进行查询。

请注意，将受限视图导入到您的代码工作区中需要启用 受限输出模式 以强制执行更高层次的数据安全。启用受限输出模式后，您可以在受限视图中进行数据分析，但不能从使用该数据的工作区发布输出。

第一次将受限视图添加到您的工作区时，会提示您安装特定的 Python 依赖项。然后您可以定义别名（如同数据集一样），并将受限视图添加到您的工作区中。您必须启用受限输出模式并重新启动工作区，才能查询受限视图。

受限视图的 策略 决定了用户可以查询哪些数据。例如，如果您将受限视图导入到工作区中，而另一个用户打开该工作区，他们看到的数据可能与您不同。如果在受限视图加载到工作区时您的访问权限发生变化，您将失去对该工作区的访问权限，并需要重新启动工作区。

筛选数据集文件

代码工作区（Code Workspaces）使您可以下载任何数据集的文件，无论这些文件是结构化（即它们具有模式）还是非结构化的。可以选择要下载的文件子集，通过名称或对文件元数据（路径、字节大小、事务RID和更新时间）应用过滤逻辑。

在 Jupyter® 中：

Copied!
1from foundry.transforms import Dataset
2
3# Download all files in the dataset
4downloaded_files = Dataset.get("my_alias").files().download()
5local_file = downloaded_files["file.pdf"]
6
7# Download a single file
8local_file = Dataset.get("my_alias").files().get("file.pdf").download()
9
10# Download all PDF files of less than 1MB in the dataset
11downloaded_files = my_dataset.files()\
12    .filter(lambda f: f.path.endswith(".pdf") and f.size_bytes < 1024 * 1024)
13    .download()

当下载多个文件时，应使用过滤语法而不是通过名称单独下载文件以利用并行下载。

在 RStudio® 中：

Copied!
1# Download all files in the dataset
2all_files <- datasets.list_files("my_alias")
3downloaded_files <- datasets.download_files("my_alias", all_files)
4local_file <- downloaded_files$`file.pdf`
5
6# Download files by name
7downloaded_files <- datasets.download_files("my_alias", c("file1.pdf", "file2.pdf"))
8
9# Download all PDF files of less than 1MB in the dataset
10all_files <- datasets.list_files("my_alias")
11pdf_files <- all_files[sapply(all_files, function(f) f.endswith(".pdf") && f$sizeBytes < 1024*1024)]
12downloaded_files <- datasets.download_files("my_alias", pdf_files)

在两种情况下，downloaded_files 将是从当前数据集视图中定义的文件名（可能包含斜杠）到文件被下载到本地路径的映射。请注意，此本地路径可能会更改，因此建议依赖于映射键。

筛选表格数据集

代码工作区允许您在将数据集加载到内存之前应用过滤器。这可以减少导入到工作区的数据帧的内存消耗，使您能够专注于与您的分析相关的数据子集。代码工作区提供了灵活的选择，可以处理选定的列、行或两者。

何时对表格数据集使用过滤器

为确保您不超出工作区的< a href="#resource-considerations">可用资源，我们建议如下：

  • 当未压缩的数据适合工作区的内存时，您可以无需过滤器加载数据集，并在内存中应用转换以获得最大效率。

  • 当未压缩的数据超过工作区的内存时，您可以使用< a href="#column-filters">列和< a href="#row-filters">行 筛选器用于将数据子集加载到内存中。这些下推筛选器在数据加载到工作区之前应用，从而减少导入数据的内存占用。这些筛选操作的速度取决于数据的规模和分区情况。您还可以使用 带有 Python 的 SQL 来查询任意大小的表格数据集，只要结果数据符合您机器的内存限制即可。

    • 如果此方法不适合您的使用场景，您也可以使用 基于 Spark 的应用程序 处理您的数据并生成一个较小的数据集，您可以在代码工作区中使用该数据集。这通常可以加快依赖频繁使用相同数据子集或跨多个工作区使用的交互式工作流的速度，因为数据集的 Parquet 文件只需下载一次，而每次数据加载到工作区时都会进行筛选。

  • 对于更复杂的数据加载需求，您可以使用 非表格数据集语法 下载支持数据集的文件，并使用原生 Python 或 R 包来处理文件内容。

在 Jupyter® 中的行过滤语法

以下语法可用于在 Jupyter® 中对数据集进行行级别的过滤。

您可以使用以下语法从 dogs 数据集中仅加载棕色的狗：

Copied!
1from foundry.transforms import Dataset, Column
2
3# Only load dogs of color "Brown"
4brown_dogs = Dataset.get("dogs")\
5  .where(Column.get("color") == "Brown")\
6  .read_table(format="pandas")

Copied!

2golden_dogs = Dataset.get("dogs")\
3  .where(Column.get("color") == "Brown")\
4  .where(Column.get("breed") == "Labrador")\
5  .read_table(format="pandas")

在RStudio®中的行筛选语法

以下语法可用于在RStudio®中对数据集进行行级别的筛选。

RStudio筛选是通过使用dplyr库实现的，并实现了标准方法filter、select和head。这些筛选器是< strong >下推的，这意味着它们在数据加载到工作区的内存之前就被应用了。

您可以使用以下语法从dogs数据集中仅加载棕色的狗：

Copied!
1library(dplyr) # should be imported by default in .Rprofile
2
3# Only load dogs of color "Brown"
4brown_dogs <- Dataset.get("dogs") %>%
5  foundry::filter(color == "Brown") %>%
6  collect()

请注意在数据加载到工作区之前使用foundry::filter进行预筛选。从技术上讲，foundry::前缀不是必需的，但我们建议使用它以避免与环境中其他类似名称的filter函数发生潜在冲突。这些filter语句可以串联起来，一次应用多个条件，使用%>%库中的dplyr运算符。该库应在您的RStudio工作区的.Rprofile文件中默认导入。

Copied!
1library(dplyr) # should be imported by default in .Rprofile
2
3# Only load dogs of color "Brown" and of breed "Labrador"
4brown_labradors <- Dataset.get("dogs") %>%
5  foundry::filter(color == "Brown") %>%
6  foundry::filter(breed == "Labrador") %>%
7  collect()

列和行过滤器一起使用

列过滤器和行过滤器可以一起使用，以加载一个同时包含其部分列和部分行的数据集。使用前面提到的 dogs 数据集：

写入数据

要安排您的数据转换，查看数据血缘关系或进行增量写入。您可以将代码转换为 Jupyter® 或 RStudio 转换。

您可以通过以下步骤使用 Code Workspaces 与 Foundry 数据集进行交互式编写。

  1. 通过打开 数据 选项卡并选择 保存到数据集 选项，可以在导入数据集右侧找到。
  2. 选择输出数据集的名称以及保存数据集的位置。
  3. 选择保存。
  4. 一个新的数据集将会出现在数据标签中。默认情况下，将选中保存到数据集选项，对于输出数据集应保持此状态。
  5. 您还将被提示指定一个数据集别名，该别名将成为工作区内的输出数据集名称，类似于导入数据时别名的工作方式。
  • 对于表格型输出数据集，您还将被提示指定将填充数据集的dataframe变量。
  • 对于非表格型数据集，您需要指定本地文件或文件夹路径以上传到数据集。
  1. 一旦设置数据集类型、数据集别名和dataframe变量，选择复制并注册数据集以在工作区中注册数据集，这也将把代码片段保存到您的剪贴板中。
  2. 粘贴代码片段到你的工作区中，并根据需要替换变量，然后执行代码以写入输出数据集。

事务类型

在进行交互式写回时，默认情况下每个SDK函数调用将对应一个事务：

  • 当写回表格数据时（Python中的output_dataset_tabular.write_table(df_variable)或R中的datasets.write_table(df_variable, "output_dataset_tabular")），将创建一个SNAPSHOT事务。
  • 当写回文件时（Python中的output_dataset_non_tabular.upload_directory(path_to_file_variable)或R中的datasets.upload_files(path_to_file_variable, "output_dataset_non_tabular")），将创建一个UPDATE事务。

一旦脚本被注册为转换，交互式调用将开始写入以code-workspace-sandbox/为前缀的分支，而当前分支将在转换运行时更新。在这种情况下，即使有多个SDK函数调用，也将为整个脚本执行创建一个事务：

  • 默认情况下，事务类型为SNAPSHOT。
  • 如果配置了增量设置，事务类型为APPEND。
