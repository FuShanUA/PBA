Code Workspaces 的计算资源使用情况

通过 Code Workspaces，用户可以安全地连接到现有的内部系统，并在 Foundry 的访问控制和数据权限管理下，基于数据构建分析、模型、应用程序或完整的工作流。

Code Workspaces 会启动专属的 Foundry 计算模块来执行任务，在此过程中会消耗 Foundry 计算秒数。

定价

Code Workspaces 的使用量按 Foundry 计算秒数进行追踪（参见使用类型）。只要工作区处于启动或活跃状态，就会持续计量计算秒数。决定所耗计算秒数的因素如下：

  • 虚拟 CPU（vCPU）的数量。
  • RAM 的吉比字节（GiB）数。
  • 正在使用的工作区。
  • GPU 的数量和类型。

VS Code 使用定价

默认情况下，新建的 VS Code 工作区会配置 0.75 个 CPU 和 6 GB 内存。此默认配置文件免费，不适用基于使用量的定价 (UBP)。您可以在创建工作区时通过高级配置部分调整这些资源，或者稍后使用设置侧边栏中的计算资源滑块进行调整。如果您增加了工作区的内存、CPU 或添加了 GPU，这些升级部分将适用标准的 UBP 定价。

查看工作区资源利用率

系统提供 CPU、内存和磁盘使用情况的实时工作区资源利用率指标。要查看这些指标，请展开设置侧边栏中的工作区利用率部分。

衡量 Foundry 计算量

打开 Code Workspace 时，Foundry 会为该工作区启动一个专属计算会话。在会话可用之前，其状态为 STARTING，随后变为 INITIALIZING，然后进入 RUNNING 状态。当用户手动停止会话，或者在超过自动关闭超时时间后仍未检测到用户交互时，该会话将转为 STOPPING 状态，随后变为 IDLE。Foundry 计算秒数仅在会话处于 INITIALIZING 或 RUNNING 状态时才会被消耗。

工作区所有可能的状态如下所示。Code Workspace 消耗计算资源的状态以右箭头 (→) 标示。

  • STARTING：已请求新会话
  • → INITIALIZING：会话可用且正在进行配置
  • → RUNNING：会话可用且可正常使用
  • STOPPING：会话正在停止
  • IDLE：会话已停止
  • FAILED：发生错误
  • TERMINATING：会话正在被永久删除
  • NOT STARTED：此工作区无可用会话

Code Workspaces 衡量计算秒数的方式与平台中其他弹性计算资源一致。请查阅通用的 Foundry 计算文档，了解具体的计量方式说明。

排查 Code Workspaces 的 Foundry 计算使用量

Foundry Code Workspaces 会将所有关联的计算资源归因于文件系统中的工作区资源。您随时可以在资源管理应用程序中查看所有会话的使用情况。

了解代码工作区中 Foundry 计算用量的驱动因素

代码工作区会话的计算用量与会话可用的专用计算资源以及会话时长成正比。

要管理会话的硬件规格，请前往设置 > 计算资源。您可以从多种不同的会话规格中进行选择。请参考下方截图，了解如何选择规格以及如何（可选）为工作区分配 GPU。包含该工作区的项目内的资源队列决定了可用的 GPU。进一步了解《在项目中使用 GPU》和《利用 GPU 进行模型训练》。

长时间运行的会话比短时间运行的会话消耗更多的计算资源。请务必调整自动关闭时间，使其与您的工作习惯保持一致，以免造成不必要的计算资源消耗。完成工作后，您应手动停止会话。
