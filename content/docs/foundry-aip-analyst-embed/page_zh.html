嵌入AIP分析员

AIP分析员可以通过iframe嵌入到Workshop或OSDK应用程序中。它支持多种URL参数，以允许进行定制化设置。接受多个值的参数使用逗号分隔列表（例如，objectTypeGroupRids=group1,group2）。

如需在Workshop模块内部进行更深入的集成，请参阅 AIP分析员Workshop小部件。

控制AIP分析员可访问的数据

  • ontologyRid: 设置AIP分析员可以探索的本体论（Ontology），例如 ri.ontology.main...。
  • objectTypeGroupRids: 将AIP分析员限制为仅搜索特定对象类型组。
  • compassProjectRids: 将AIP分析员限制为仅搜索特定Compass项目或文件夹内的资源。
  • hideManualContextMenu: 当设置为 true 时，防止用户手动添加其他数据源。

/workspace/aip-analyst?ontologyRid=ri.ontology.main.abc123&objectTypeGroupRids=group1,group2&hideManualContextMenu=true

预加载会话

预填充用户的第一个消息，或预加载 上下文，以便用户开始时已有相关资源：

  • workshopRids: 从您的 Workshop 模块中加载对象类型、关联和函数，例如 ri.workshop.main.module...。
  • objectSetRids: 加载保存的对象集，例如 ri.object-set.main...。
  • datasetRids: 加载特定的数据集。
  • objectTypeIds: 加载单独的对象类型。
  • objectRids: 加载单独的对象。
  • functionRids: 加载单独的函数。
  • actionTypeRids: 加载单独的操作类型。
  • notepadRids: 加载便签文档。
  • initialMessage: 在 AIP 分析师加载时预填充第一个用户消息。
  • autoStart: 当 true 和 initialMessage 设置时，AIP 分析师在页面加载后立即发送初始消息并开始分析。可用于嵌入“提问并回答”的体验，以响应宿主应用程序中的上下文。

/workspace/aip-analyst?workshopRids=ri.workshop.main.module.xyz789&initialMessage=Summarize+sales+by+region&autoStart=true

查看选项

  • embedded: 当设置为 true 时，隐藏工作区侧边栏以获得更简洁的外观。
  • hideSettingsMenu: 当设置为 true 时，隐藏设置菜单。
  • theme: 设置颜色主题（light 或 dark）。
  • modelRid: 为分析设置特定模型。
