使用图表可视化对象

Quiver 支持多种卡片用于可视化对象数据。这些卡片接收一个对象集作为输入，并返回图表或可视化效果。在某些情况下，图表可用作过滤器，下钻查看选定的对象。在其他情况下，图表中的聚合数据（也称为分类图表）可作为后续图表的输入。

用于简单对象集可视化的常见图表包括：

  • 条形图
  • 折线图
  • 饼图
  • 分类散点图
  • 数值散点图
  • 透视表
  • 热力网格
  • 瀑布图

添加并配置图表

将鼠标悬停在对象集卡片上时，可以在“下一步操作”菜单的可视化区域找到对象集图表。每种图表根据其类型会有略微不同的配置选项。通常，你可以使用图表上的编辑器面板执行以下操作：

  • 在数据选项卡中配置数据。
    • 更改输入的对象集。
    • 定义聚合：按属性分组、x 轴/y 轴属性、聚合指标等。
      • 支持的聚合指标：最小值、最大值、求和、平均值、计数、去重计数、百分位数、标准差和方差。
      • 对于由对象存储 V1 (Phonograph)支持的对象类型，不支持百分位数、标准差和方差指标。
    • 定义分段（可选）。
    • 定义排序顺序。
  • 在显示选项卡中配置显示效果。
    • 更改图表的方向，例如更改条形图中条形的方向。
    • 覆盖分段和指标的颜色与标签。
    • 配置坐标轴标题、格式和位置。
    • 选择是否显示标签。
    • 配置图例：是否显示图例及其位置。

精度限制

以下对象集图表聚合存在较小的精度限制：

聚合精度限制
去重计数你可以选择计算近似结果或精确结果。对于由对象存储 V1 (Phonograph)支持的对象类型，不支持精确结果，并且当结果计数超过 10,000 时可能会降低性能。
百分位数该指标的结果始终是近似的。满足相对误差或绝对误差；默认情况下，最大相对误差为 0.1%，绝对误差为 0.001%。
标准差由于浮点运算误差累积，该指标的结果始终是近似的。在均值非常高而标准差非常低的情况下，可能会产生不准确的结果。
方差由于浮点运算误差累积，该指标的结果始终是近似的。在均值非常高而标准差非常低的情况下，可能会产生不准确的结果。

请注意，这些限制不适用于基于转换表或物化构建的图表。

使用图表选择进行过滤

在 Quiver 中，大多数以对象集作为输入的图表都是交互式的，通过下钻可以快速直观地探索每个图表背后的底层数据。在 Quiver 中执行下钻最直接的方法是，在图表或可视化效果上选择数据，然后在卡片页脚中选择下钻选项。这将创建一个选择对象集，即由绘图区域选择定义的新对象集。要在图表中选择多个类别，可以单击并拖动以选择一个范围，或者在选择类别时按住 Cmd (macOS) 或 Ctrl (Windows) 键。

在下面的示例中，我们选择 garage 类别，然后选择 下钻 来创建一个 选择对象集。可以看到该对象集包含 218,785 个对象，与柱状条的长度相匹配。接着我们返回图表，使用多选功能选择 building 类别。筛选器随之更新，包含所有类型为 building OR garage 的建筑对象。最后，我们点击并拖动鼠标，框选一连串由于太小而无法单独选中的类别。

交叉筛选

在某些场景下，你可能希望组合多个图表的选择操作，使其不仅能筛选下游对象集，还能对其他已订阅的图表进行筛选。这为用户（尤其是仪表盘的使用者）提供了更加横向且交互性更强的体验。

这些工作流可以通过交叉筛选卡片进行配置。

使用叠加图表组合多个图表

对于某些图表可视化形式，你可能希望将两个图表叠加在一起显示。例如，在同一图表中同时展示柱状图和散点图，或者在同一图表中展示不同对象类型的数据。这可以通过叠加图表来实现。在分类图表卡片的下一步操作菜单中，选择 可视化 > 叠加图表 即可添加叠加图表。叠加图表将继承输入图表所使用的所有显示选项。

使用图表公式

Quiver 原生公式可作为绘制更复杂聚合的有效工具。在许多分类图表的 数据 标签页下的 指标 配置中可以找到该功能。选择 切换为公式指标 即可开始编写公式。你可以借此指定需要计算的多个聚合，然后通过公式将它们组合起来。该公式将应用于系列中的每个数据分段。

在下面的示例中，我们按建筑类型创建了一个柱状图。接着，我们使用公式指标计算每个类别的“范围”聚合，即用最高屋顶高度（$M0）减去最低屋顶高度（$M1）。

你还可以在带有分类公式图的分类图表中使用公式。该卡片支持使用公式将来自数值型数据、未分段的分类图表（二维）以及已分段的分类图表（三维）的数据进行组合。在公式中，可以使用全局标识符表示法来引用其他卡片（例如 $A）。

在公式中组合数据时，请遵循以下参考规则：

  • 数值型数据将应用于所有数据分段。
  • 未分段数据（二维）只有在分组依据值相同的情况下，才会与其他未分段数据进行组合。
  • 未分段数据（二维）会与所有分组依据值相同的已分段数据（三维）的分段进行组合。
  • 已分段数据（三维）只有在分组依据值和分段依据值均相同的情况下，才会与其他已分段数据进行组合。

在下面的示例中，我们首先创建所有建筑物的计数聚合（$S）。然后创建按建筑类型分组的计数柱状图（$T）。最后，我们使用分类公式图，将柱状图的计数值除以总计数（$T / $S）。最终生成的柱状图会将每个类别的计数显示为占总数的百分比（而非绝对计数）。

在分类图表中使用高级聚合

在某些情况下，你可能遇到需要更复杂聚合的分类图表可视化需求，而这些需求是上述图表绘制方法所不支持的。针对这些情况，Quiver 提供了多种高级聚合选项，例如：

  • 在关联对象类型的属性之间进行图表绘制。
  • 派生新的预聚合属性，然后对其进行聚合。
  • 执行原生不支持的聚合操作。

代码函数分类图

代码函数分类图支持使用代码函数来创建分类图。配置代码函数分类图需要编写一个返回 TwoDimensionalAggregation 或 ThreeDimensionalAggregation 的函数。该函数应与函数支撑的 Workshop 图表层所需的函数相同。

转换表

转换表是 Quiver 中用于派生新属性和连接关联对象的强大工具。转换表中的任何列都可以通过来自转换表的分类图卡片进行可视化。

物化

物化是 Quiver 中另一种可扩展的数据转换方法。特别是，表达式卡片可以利用强大的表达式语言，大规模派生新的预聚合列。连接物化卡片可用于在对象之间执行左连接、内连接或右连接，以支持跨链接的图表绘制。物化中的任何列都可以通过来自物化的分类图卡片进行可视化。

Vega 图表

如果你想要 Foundry 原生支持之外的自定义可视化，Quiver 也支持使用 Vega ↗ 和 Vega-Lite ↗ 通过 vega 绘图卡片来创建可视化。通过此卡片，你可以编写 JSON 规范来定义自定义图表，同时使用全局标识符（例如 $A 和 $A.columnname）引用 Quiver 转换表、数组和标量值。
