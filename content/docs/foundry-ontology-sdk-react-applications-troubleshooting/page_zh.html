故障排查

本页面提供了在开发 OSDK React 应用程序时排查错误的相关提示。如果你遇到其他问题，或者通过本指南未能解决你的问题，请向 Palantir 支持团队提交问题。

工作区错误

如果你在 Palantir 平台内部进行开发，将使用运行在 Palantir Code Workspaces 基础设施上的 VS Code 工作区。你可以在以下文档中找到更多排查信息：

  • VS Code 工作区故障排查
  • Code Workspaces 故障排查

npm 故障排查步骤

如果你在运行 npm 命令时遇到问题，请尝试以下排查步骤：

  • 删除锁定文件（package-lock.json）和依赖文件夹（node_modules/），然后重新运行失败的命令。
  • 暂停并恢复工作区。

NPM MODULE_NOT_FOUND 错误：添加新依赖

代码仓库要求你必须显式声明依赖项。

默认情况下，我们会添加以下 npm 仓库：

  • 常用 OSDK 依赖：foundry-sdk-asset-bundle、osdk-templates-bundle
  • npmjs.com ↗ 镜像：external-npm-npmjs
  • 你的 OSDK：SDK Artifacts Repository - <rid>

如果你尝试添加的包不存在于任何底层仓库中，npm install <package> 命令可能会失败。例如，如果你要安装的包是私有的，你必须确保它作为仓库的 Artifact 存在。请查阅我们的Artifacts 文档以获取更多信息。

错误示例：

当在 VS Code 工作区中无法访问 npmjs.com 时，npm 将会报出 code E401 Incorrect or missing password 错误。
