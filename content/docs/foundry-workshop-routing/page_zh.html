路由

Workshop 路由功能支持将模块的特定状态或视图写入 URL，方便用户通过分享链接将这些视图共享给他人。

启用路由

若要为 Workshop 模块启用路由，请在左侧 Settings 面板的 Pages 区域中，开启 Enable routing 选项。

Pages 区域会显示路由所用页面的概览，并提供一个附加面板，用于展示当前正在使用的变量，如下图所示。

页面路由

启用路由后，当前页面的 ID 将写入 URL。对于未定义页面 ID 的页面，URL 中不会写入页面 ID；页面加载时，用户将返回至模块的默认页面。

变量路由

你还可以利用路由功能保存并共享为模块界面配置的变量值。启用路由后，配置了路由 URL 更新行为的变量，其当前值将直接作为 URL 查询参数写入。

若要将模块界面变量包含在 URL 中，请使用以下配置选项之一。

  • 被可见组件或布局使用时写入 URL： 仅在以下条件均满足时，URL 才会包含该变量的值：
    • 该值不是变量的默认值
    • 该变量被当前视图中显示的组件或布局所使用。
  • 始终写入 URL： 只要该值不是变量的默认值，URL 就会始终包含该变量的值。
  • 从不写入 URL： 若禁用路由开关，URL 将绝不包含该变量的值。

如果查询参数的键与模块界面变量的外部 ID 相匹配，则无论配置了何种 URL 包含行为，该查询参数的值都将作为该变量的初始值。

限制

URL 中不支持的变量类型

以下变量类型无法在 URL 中使用：

  • 对象集筛选变量
  • 对象集变量 仅限于通过 RID 指定的单个对象

但是，你可以通过其他路由变量间接定义上述变量类型。例如，你可以创建一个用于路由的字符串变量，并使用该字符串变量来定义对象集筛选变量的 default value。

不从嵌入式模块继承路由配置

嵌入 Workshop 模块 不会继承该嵌入式模块的路由配置。若要在嵌入式模块中使用路由变量值，请将所需变量添加到子模块的 模块界面 中，并在嵌入式模块配置中从父模块传入路由变量。
