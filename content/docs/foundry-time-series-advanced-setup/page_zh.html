高级设置

我们建议按照《时间序列设置》页面的说明，使用《管道构建器》来设置您的时间序列管道。这样做会自动应用下文所述的转换优化。

在进行高级设置配置之前，请联系您的 Palantir 代表。

如果您需要 Pipeline Builder 尚未提供的底层转换控制或高级功能，本页将介绍如何使用《代码仓库》进行数据转换，从而手动设置您的时间序列管道。

要使用代码仓库设置时间序列，您必须完成以下操作：

  1. 创建并优化时间序列输入数据。
  2. 设置时间序列同步。
  3. 创建时间序列对象类型支持数据集。
  4. 设置时间序列对象类型。

1. 创建并优化时间序列输入数据

时间序列同步可以建立在数据集或流之上。这种选择通常取决于吞吐量和延迟的考量。请查阅《流处理与批处理对比》以获取更详细的分析。

在这两种情况下，当您手动设置管道时，必须明确生成一个包含已格式化时间序列数据的时间序列数据集或流；这是创建时间序列同步所必需的。数据集或流必须包含《术语表》中指定的 Series ID、Value 和 Timestamp 列，以便在时间序列同步中映射这些值。

数据集时间序列

当存在实时数据时，时间序列数据集通常配置为增量构建。增量构建使您能够节省计算成本，并大幅缩短从摄取原始数据到读取最新数据之间的延迟。

有关增量时间序列构建优势的更多信息，请参阅《常见问题文档》。

同一个序列 ID 下的所有值都应包含在同一个数据集中。由于值是通过其序列 ID 获取的，因此单个时间序列数据集可以包含多个序列 ID 的所有值。例如：

+------------------------+---------------------+---------+
| series_id              | timestamp           | value   |
+------------------------+---------------------+---------+
| Machine123_temperature | 01/01/2023 12:00:00 | 100     |
| Machine123_temperature | 01/01/2023 12:01:00 | 99      |
| Machine123_temperature | 01/01/2023 12:02:00 | 101     |
| Machine463_temperature | 01/01/2023 12:00:00 | 105     |
| Machine123_pressure    | 01/01/2023 12:00:00 | 3       |
| ...                    | ...                 | ...     |
+------------------------+---------------------+---------+

执行此重分区和排序操作，可以优化数据集，使其在作为时间序列使用时具备更佳性能。至少，您的数据集也应格式化为 Soho（如图所示），以便在尚未建立投影时，新数据能够被索引至时间序列数据库。您还应根据以下准则，将 repartitionByRange() ↗ 写入的分区数配置为适合您流水线的合适数量：

  • 尽量减少写入的分区数量。
  • 分区大小应大于 128 MB。
  • 通常情况下，分区所含行数应少于 50 亿行。

可写入分区数量的下限，取决于分区是否小到足以容纳于单个 执行器 中，同时又要保证分区数量足够多，使作业能够充分并行化，从而满足您期望的流水线延迟。写入更多分区会导致单个分区变小并加快作业速度，但其效率不如使用较大分区。

时间序列同步中的视图优化

在同步大量数据时，我们建议您按小时、按天或按周等离散时间间隔，将数据分割为较小的数据集。随后，您可以将这些分区整合为一个统一的 视图，并同步至时间序列。此方法有助于确保下游在 FoundryTS 库或 Quiver 中请求时间序列数据时获得良好的性能。最终，这些较小的数据集会通过统一视图，合并为单一的时间序列同步流。

请注意，由于视图包含了所有已分割的底层数据集，因此允许序列 ID 分布或交错于各个底层数据集中。

您可以在时间序列同步的 高级设置 中配置视图优化的使用方式。

请按照以下步骤配置视图优化：

  1. 建立包含底层序列数据集的视图。
  2. 将该视图作为您正在设置的时间序列同步的输入源。
  3. 在时间序列设置对话框的 高级设置 部分，开启 索引视图数据集输入 选项。

流式时间序列

为了优化流式时间序列的性能，我们建议采取以下步骤：

  • 使用 Series ID 列对流进行键控；这确保平台在索引时间序列数据时读取的分区数量最少。有关流式键的更多信息，请参阅《流式键控文档》。
  • 设置更多的分区，对于具有较高吞吐量要求的流式时间序列同步，我们建议至少配置 3 个分区。

请确保仅使用 Series ID 列对流进行键控，而不要使用任何其他列。这可以确保时间序列索引获得最佳的排序保证。

流式管道

我们建议使用流式管道来设置流式时间序列同步。您可以通过将时间序列同步设为管道的输出来实现这一点。

  • 使用 Series ID 列对流进行键控：创建一个键转换，且仅使用 Series ID 列。
  • 设置分区数量：在 构建设置 > 高级配置 > 输出分区数量 下自定义流式配置。

流式同步

配置流式时间序列输入的另一种方法是设置流式同步。

  • 使用 Series ID 列对流进行键控：在 配置 > 键 下，选择 Series ID 列。
  • 设置分区数量：在 吞吐量 下，选择 非常高，并根据吞吐量要求选择一个值。

如果您的流已经接收了记录，更改键控或分区将需要重置流。

2. 设置时间序列同步

如果您正在遵循设置助手的指引，请选择 转到构建器。

有关更多详细信息，请参阅《使用时间序列数据目录创建时间序列同步》文档。

创建时间序列同步后，请返回 Ontology Manager 并将此时间序列同步作为数据源添加到时间序列属性中。如果您的同步包含所有的序列 ID，则可以为新的时间序列属性选择同一个同步，而无需创建新的同步。

务必在 Ontology Manager 中选择 保存 以保存您的更改。如果您创建了一个新的对象类型，并且这是您第一次保存更改，则需要等待初始索引完成，然后才能分析 TSP。在 Ontology Manager 中转到该对象类型的 数据源 选项卡，可以检查索引状态。

3. 创建时间序列对象类型底层数据集

您可以通过首选的方法生成时间序列对象类型底层数据集，并且它应符合《术语表》中指定的模式。

要自动生成时间序列对象类型底层数据集，您可以在与时间序列数据集相同的转换中生成它，从中获取不同的序列 ID 集合，并从中提取/映射元数据。在增量管道中，您可以使用“合并并追加”模式来实现这一点。

4. 设置时间序列对象类型

请按照《标准流程》，在您的时间序列对象类型底层数据集上创建对象类型。也可以在《数据集预览》中选择 所有操作 > 创建对象类型，直接从数据集中生成对象类型。创建对象类型时，通过指定哪些属性应为《时间序列属性》来为其配置时间序列。
