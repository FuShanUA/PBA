高级设置

我们建议使用 Pipeline Builder（管道构建器） 来设置时间序列管道，如 时间序列设置 页面所述。这样做将自动应用以下描述的转换优化。

在进行高级设置配置之前，请联系您的 Palantir 代表。

如果您需要低级转换控制或 Pipeline Builder 尚未提供的高级功能，本页面将说明如何使用 代码仓库 手动设置时间序列管道，用于数据转换。

要使用代码仓库设置时间序列，您必须完成以下步骤：

  1. 创建并优化时间序列输入数据。
  2. 设置时间序列同步。
  3. 创建时间序列对象类型支撑数据集。
  4. 设置时间序列对象类型。

1. 创建并优化时间序列输入数据

时间序列同步可以基于数据集或流构建。这种选择通常取决于吞吐量和延迟的考虑因素。请查阅流式处理与批量处理的比较以获得更详细的分析。

在两种情况下，当您手动设置管道时，必须显式生成包含已格式化的时间序列数据的时间序列数据集或流；这是创建时间序列同步所必需的。数据集或流必须包含Series ID、Value和Timestamp列，如术语表中所述，以便这些值可以在时间序列同步中进行映射。

数据集时间序列

时间序列数据集通常在有实时数据时配置为< a href="/docs/foundry/transforms-python/incremental-overview/">增量式构建。增量构建可以节省计算成本，并实现从原始数据摄入到可读取最新数据的延迟大大缩短。

有关增量时间序列构建优势的更多信息，请参阅< a href="/docs/foundry/time-series/faqs/#why-is-my-time-series-taking-a-long-time-to-load">FAQ文档。

系列ID的所有值应包含在同一数据集中。由于值是通过其系列ID获取的，一个时间序列数据集可以包含多个系列ID的所有值。例如：

+------------------------+---------------------+---------+
| series_id              | timestamp           | value   |
+------------------------+---------------------+---------+
| Machine123_temperature | 01/01/2023 12:00:00 | 100     |
| Machine123_temperature | 01/01/2023 12:01:00 | 99      |
| Machine123_temperature | 01/01/2023 12:02:00 | 101     |
| Machine463_temperature | 01/01/2023 12:00:00 | 105     |
| Machine123_pressure    | 01/01/2023 12:00:00 | 3       |
| ...                    | ...                 | ...     |
+------------------------+---------------------+---------+

在代码中生成时间序列数据集时，格式化数据集后再写入，如下所示：

14public final class TimeSeriesWriter {
15    @Compute
16    public void writePartitioned(
17            @Input("/path/to/input/dataset") FoundryInput inputDataset,
18            @Output("/path/to/output/dataset") FoundryOutput outputDataset) {
19        Dataset<Row> inputDataframe = inputDataset.asDataFrame().read();
20
21        Dataset<Row> outputDataframe = inputDataframe
22            .repartitionByRange(inputDataframe.col('seriesId'))
23            .sortWithinPartitions('seriesId', 'timestamp');
24
25        outputDataset.getDataFrameWriter(outputDataframe)
26            .setFormatSettings(DatasetFormatSettings.builder()
27                .format('soho')
28                .build())
29            .write();
30    }
31}

运行此重新分区和排序操作将优化您的数据集，以便作为时间序列进行高效使用。至少，您的数据集还应格式化为 Soho（如所示），以便在新数据 未被投影 时将其索引到时间序列数据库中。您还应根据以下指导，配置由 repartitionByRange() ↗ 写入的分区数，以适合您的管道：

  • 尽可能少地写入分区。
  • 分区应大于 128 MB。
  • 通常，分区应少于 50 亿行。

您可以写入的最少分区数的限制是基于编写足够小的分区，使其能够放在 执行器 上，但要有足够的分区，使您的作业可以充分并行化，以满足您所需的管道延迟。写入更多分区会导致更小的分区和更快的任务，但不会像更大的分区那样优化。

对时间序列同步的视图优化

在同步大量数据时，我们建议使用离散的时间间隔（如每小时、每天或每周）将数据分成较小的数据集。然后可以将这些分区合并为一个统一的 视图，该视图可以同步到时间序列。这种方法有助于确保在 FoundryTS 库或 Quiver 中下游请求时间序列数据时保持良好的性能。最终，较小的数据集通过统一的视图合并为一个时间序列同步。

请注意，由于视图包含所有分段的支撑数据集，因此允许系列 ID 在支撑数据集中分布或错开。

您可以在时间序列同步 高级设置 中配置视图优化的使用。

按照以下步骤配置视图优化：

  1. 设置包含支撑系列数据集的视图。
  2. 使用该视图作为你正在设置的时间序列同步的输入。
  3. 在时间序列设置对话框的高级设置部分中切换索引视图数据集输入选项。

流式时间序列

为了优化流式时间序列性能，我们建议采取以下步骤：

  • 使用系列ID（Series ID）列对你的流进行键控；这将确保平台在索引时间序列数据时从最少数量的分区读取。有关流式键的更多信息，请参阅 流式键文档。
  • 设置更多 分区（Partitions），我们建议为具有更高吞吐量要求的流式时间序列同步配置至少 3 个分区。

确保你的流只通过 系列ID（Series ID） 进行键控 列，而不是由任何其他列。这确保了时间序列索引的最佳排序保证。

流式管道

我们建议使用 流式管道 来设置流式时间序列同步。

  • 使用 Series ID 列对流进行键控： 使用 仅 Series ID 列创建一个 按键转换。
  • 设置分区数量： 在 构建设置 > 高级配置 > 输出分区数量 下自定义 流式配置文件。

流式同步

配置流式时间序列输入的另一种方法是 设置流式同步。

  • 使用Series ID列作为流的键：在配置 > 键下，选择Series ID列。
  • 设置分区数量：在吞吐量下，选择非常高，并根据吞吐量需求选择一个值。

如果您的流已经摄入了记录，更改键或分区将需要 重置流。

2. 设置时间序列同步

如果您正在使用设置助手，请选择进入构建器。

有关更多详细信息，请参阅 使用时间序列目录创建时间序列同步 文档。

创建时间序列同步后，返回到本体论管理器，并将此时间序列同步作为时间序列属性的数据源。如果您的同步包含所有系列ID，则可以选择使用相同的同步来创建新的时间序列属性，而不是创建一个新的同步。

请确保在本体论管理器中选择保存以保存您的更改。如果您创建了一个新的对象类型，并且这是您第一次保存更改，那么在您可以分析TSP之前，需要等待初始索引完成。通过导航到本体论管理器中该对象类型的数据源选项卡来检查索引状态。

3. 创建时间序列对象类型支撑数据集

你可以通过你偏好的方法生成时间序列对象类型支撑数据集，并且它应该符合 术语表 中指定的模式。

要自动生成时间序列对象类型支撑数据集，你可以在生成时间序列数据集时使用相同的转换，在其中可以获取系列ID的唯一集合，并从中提取/映射元数据。在增量管道中，你可以使用 合并和追加 模式来实现此目的。

4. 设置时间序列对象类型

按照 标准流程 在你的时间序列对象类型支撑数据集上创建一个对象类型。也可以直接从数据集中生成对象类型，在 数据集预览 中选择 所有操作 > 创建对象类型。创建对象类型时，通过指定哪些属性应为 时间序列属性。
