使用服务用户初始化新的 OSDK TypeScript 应用程序

如《权限类型》部分所述，OSDK 可以基于服务用户的权限而非终端用户的权限来查询数据。以下操作指南展示了如何使用 Next.js© ↗（外部链接）通过 OSDK 和服务用户获取数据。

在使用机密客户端开发服务或应用程序时，系统会随 Developer Console 应用程序一起创建一个服务用户。如果打算使用属于非默认组织的本体来创建应用程序，必须完成相关步骤以《共享并启用该应用程序》。

1. 使用 Developer Console 创建 OSDK 包

在 Foundry 实例中《导航至 Developer Console》，然后选择 + 新建应用程序。

如果没有显示 + 新建应用程序 按钮，可能是权限不足。请查阅《权限文档》了解更多信息。

按照创建向导中的步骤操作，并填写以下详细信息：

  • 在 应用程序类型 页面，选择《后端服务》。

  • 在 权限 页面，选择《应用程序权限》。

Developer Console 会根据应用程序名称为该应用程序创建一个服务用户。在上面的示例中，生成的服务用户名为 Ontology SDK application using service user。除了满足任何操作类型的《提交条件》外，还必须向该服务用户授予《读取对象类型数据》所需的权限，以便在下一步中选择相应的对象类型。

  • 在 资源 页面，选择 是，生成本体 SDK。

  • 选择要使用的本体。

请注意，本体 SDK 生成后，其关联的本体将无法更改。

  • 选择你希望 OSDK 包含的对象类型和操作类型。在本练习中，请选择你可用的任意对象类型。

检查并确认你输入的信息，然后选择创建应用程序以查看新应用程序的客户端密钥。请务必复制并妥善保存该密钥，因为这是其唯一一次可见的机会。

如果丢失了客户端密钥，你可以在权限与 OAuth页面上轮换并获取新密钥。请注意，这将导致使用该服务用户和密钥的现有应用程序失效。

最后，选择生成首个版本以使用你新创建的 OSDK。

2. 安装生成的 SDK 包

OSDK 生成完成后，你将看到一组安装步骤，指导你将生成的 SDK 安装到你的代码项目中。

3. 在代码项目中使用 OSDK

在本演练中，我们使用 Next.js ↗。Next.js 支持在服务器端渲染代码，这是我们的服务用户示例所必需的。要初始化一个新的 Next.js 项目，请参阅 Next.js 文档 ↗。

创建客户端与 OAuth

服务用户身份验证通过机密 OAuth 客户端完成，允许您使用客户端密钥访问本体，无需进行用户身份验证。

创建名为 client.ts 的文件并输入以下代码：



已复制！
1import { createConfidentialOauthClient } from "@osdk/oauth";
2import { createClient } from "@osdk/client";
3
4export const auth = createConfidentialOauthClient(
5    process.env.CLIENT_ID!,
6    process.env.CLIENT_SECRET!,
7    process.env.STACK_URL!,
8);
9
10export const client = createClient(
11    process.env.STACK_URL!,
12    <ONTOLOGY-RID>,
13    auth
14)

创建一个包含相同变量的 .env 文件。请勿将此文件提交至代码仓库。

已复制！
1CLIENT_ID=<你的客户端 ID>
2CLIENT_SECRET=<你的客户端密钥>
3STACK_URL=<你的本体服务器域名> # 例如，https://myfoundrystack.com

访问本体

以下代码使用了带有 @serverside-osdk-example/sdk 包名的 Country 对象类型。请将示例包名和对象类型分别替换为你所创建的包名和所选的对象类型。最后，将 {country.countryName} 替换为你的对象类型中的属性。

将 page.tsx 中的代码替换为以下内容：



已复制！
1import { Country } from "@serverside-osdk-example/sdk";
2import { client, auth } from "./client";
3import { Osdk } from "@osdk/client";
4
5async function getCountries(): Promise<Osdk.Instance<Country>[]> {
6    // 处理身份验证
7    await auth.signIn();
8    // 需要授予服务用户对本体的读取权限
9    try {
10        const resp = await client(Country).fetchPage();
11        return resp.data;
12    } catch (err) {
13        console.log(err);
14    }
15    console.log("未找到任何国家");
16    return [];
17}
18
19export default async function Home() {
20    const countries: Osdk.Instance<Country>[] = await getCountries();
21    return (
22        <main>
23            <div>
24                {countries.map((country: Osdk.Instance<Country>) => (
25                    <span key={country.$primaryKey}>{country.countryName}</span>
26                ))}
27            </div>
28        </main>
29    );
30}
