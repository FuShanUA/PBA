使用服务用户引导一个新的OSDK TypeScript应用程序

如在权限类型部分所述，OSDK可以基于服务用户的权限而不是最终用户的权限来查询数据。以下操作指南展示了如何使用Next.js© ↗（外部链接）通过OSDK和一个服务用户来获取数据。

在开发使用保密客户端的服务或应用程序时，将与您的开发者控制台应用程序一同创建一个服务用户。如果您计划使用属于与默认组织不同的组织的本体论（Ontology）来创建应用程序，您必须完成步骤以共享并启用该应用程序。

1. 使用开发者控制台创建一个OSDK包

导航到开发者控制台，然后选择 + 新增应用。

如果 + 新增应用 按钮没有出现，您可能没有正确的权限。请查看 权限文档 获取更多信息。

按照创建向导的步骤，并添加以下详细信息：

  • 在 应用类型 页面上，选择 后端服务。

  • 在 权限 页面上，选择 应用的权限。

开发人员控制台将根据应用程序名称为此应用程序创建一个服务用户。在上面的示例中，生成的服务用户的名称是Ontology SDK 应用程序使用服务用户（Ontology SDK application using service user）。除了任何操作类型的提交标准之外，您还必须授予此服务用户所需的权限以读取您下一步将选择的对象类型的数据。

  • 在资源页面上，选择是，生成一个Ontology SDK。

  • 选择要使用的Ontology。

请注意，一旦生成了Ontology SDK，与之关联的Ontology将无法更改。

  • 选择您希望OSDK包包含的对象类型和操作类型。对于此练习，请选择您可用的任何对象类型。

查看并确认您输入的信息，然后选择创建应用以查看新应用的客户端密钥。请复制并将密钥安全存储，因为这是唯一可见的时间。

如果您丢失了客户端密钥，可以在权限与OAuth页面上旋转并获取一个新的密钥。请注意，这将破坏使用此服务用户和密钥的现有应用程序。

最后，选择生成第一个版本以使用您新创建的OSDK。

2. 安装生成的SDK包

一旦OSDK生成完成，您将看到一组安装步骤，指导您在代码项目中安装生成的SDK。

3. 在您的代码项目中使用OSDK

在此操作指南中，我们使用 Next.js© ↗。Next.js 支持服务器端渲染，这是我们的服务用户示例所需的。要引导一个新的 Next.js 项目，请按照 Next.js© 文档 ↗ 。

客户端和OAuth创建

服务用户认证是通过一个保密的OAuth客户端进行的，该客户端允许您使用客户端密钥访问本体论（Ontology），而无需要求用户认证。

创建一个名为 client.ts 的文件，并输入以下代码：

Copied!
1import { createConfidentialOauthClient } from "@osdk/oauth";
2import { createClient } from "@osdk/client";
3
4export const auth = createConfidentialOauthClient(
5    process.env.CLIENT_ID!,
6    process.env.CLIENT_SECRET!,
7    process.env.STACK_URL!,
8);
9
10export const client = createClient(
11    process.env.STACK_URL!,
12    <ONTOLOGY-RID>,
13    auth
14)

使用相同的变量创建一个 .env 文件。 不要将此文件提交到您的代码仓库中。

Copied!
1CLIENT_ID=<YOUR CLIENT ID>
2CLIENT_SECRET=<YOUR CLIENT SECRET>
3STACK_URL=<YOUR ONTOLOGY SERVER DOMAIN NAME> # for example, https://myfoundrystack.com

访问本体论

以下代码使用了一个 Country 对象（对象）类型，其包名（包名称）为 @serverside-osdk-example/sdk。请分别用您创建的包名和选择的对象类型替换示例包名和对象类型。最后，用您对象类型中的一个属性（属性）替换 {country.countryName}。

将 page.tsx 中的代码替换为以下内容：

Copied!
1import { Country } from "@serverside-osdk-example/sdk";
2import { client, auth } from "./client";
3import { Osdk } from "@osdk/client";
4
5async function getCountries(): Promise<Osdk.Instance<Country>[]> {
6    // Handle authentication
7    await auth.signIn();
8    // You need to give the service user read access to the ontology
9    try {
10        const resp = await client(Country).fetchPage();
11        return resp.data;
12    } catch (err) {
13        console.log(err);
14    }
15    console.log("No countries found");
16    return [];
17}
18
19export default async function Home() {
20    const countries: Osdk.Instance<Country>[] = await getCountries();
21    return (
22        <main>
23            <div>
24                {countries.map((country: Osdk.Instance<Country>) => (
25                    <span key={country.$primaryKey}>{country.countryName}</span>
26                ))}
27            </div>
28        </main>
29    );
30}
