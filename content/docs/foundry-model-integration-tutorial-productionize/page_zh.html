4. 教程 - 将模型投入生产

在继续本教程之前，您应该已经完成了 建模项目设置 和 模型训练，并可选地完成 模型评估 教程。此时，您应该在您的建模目标中至少有一个模型。

在本教程的这一步中，您将使用您的机器学习模型，并为其设置生产环境的使用方式。

  1. 在建模目标中发布模型。
  2. 为批量处理创建批量部署。
  3. 为您的模型创建交互式实时端点。

4.1 在建模目标中发布模型

现在我们有了一个模型，可以将其部署到生产环境中。根据该模型的预期用途，我们可能会以几种方式之一进行生产化。模型可以从建模目标中进行部署，建模目标提供功能齐全的发布和模型管理，包括文档、评估以及对多个支撑模型提交进行比较的支持。

从建模目标进行部署的方式如下：

  • 批量部署：为数据集生成预测结果，适用于一次性计算一组记录的预测结果。
  • 实时部署：将模型托管在 REST 端点之后，适用于从外部系统使用模型，或需要与模型进行实时交互的情况。
  • 在 Python 转换中：适用于转换中的快速迭代。

另外，偏爱更轻量级设置的用户可以将模型导入转换中以进行批量推理，如在代码仓库中运行批量推理的教程中所述，或者通过直接部署将其托管在 REST 端点之后，而无需创建建模目标。

了解有关 Modeling Objectives 中的实时部署与直接部署之间的区别 的更多信息。

在 Modeling Objective 中什么是发布？

批量和实时部署由模型发布支持。如果您在 Modeling Objective 中发布了新模型，该 Modeling Objective 中的部署将自动升级以使用新发布的模型。

这允许模型消费者使用模型而不必担心其具体实现。数据科学家可以专注于改进模型，而应用开发人员则专注于构建有用的应用程序。

发布管理

发布和部署分为暂存环境和生产环境。模型开发者可以在正式发布到生产环境之前，将模型发布到暂存环境。这会升级暂存部署，而不会改变生产部署。然后数据科学家或应用构建者可以在模型正式上线前，在暂存环境中测试新模型。

所有发布以及发布模型的人员都会被记录下来，以确保团队能够跟踪在何时使用了哪个模型。这有助于回答GDPR或欧盟人工智能法案等法规要求的问题。

操作：导航到您的建模目标的首页，并滚动到发布部分。选择发布到暂存，然后选择箭头图标将模型发布到生产环境。您需要为发布指定一个版本号，例如“1.0”。

4.2 如何为批量处理创建批量部署

批量部署将从输入数据集创建一个Foundry转换。批量部署的输出是一个数据集，该数据集是使用模型对输入数据集进行推理（生成预测）的结果。

操作：点击创建部署并配置生产环境的批量部署。给部署命名 Batch deployment 并添加描述 Production batch deployment。选择你之前创建的 housing_inference_data 数据集作为输入数据集，并在你的 data 文件夹中创建一个名为 house_price_predictions 的新输出数据集。选择 创建部署 以保存配置。

现在，我们可以安排批量部署，这样每当有新模型发布时，我们的输出数据集就会自动更新 - 确保我们始终使用最佳预测。让我们在输出数据集上添加一个计划，以便当它接收到新的逻辑时，输出数据集会重新构建。

操作：在部署表中选择您的批量部署。选择名为 house_price_predictions 的输出数据集。在 All actions 下，选择 管理计划。在 house_price_predictions 数据集上，选择 创建新计划 -> 当多个时间或事件条件满足时，当 house_price_predictions 数据集接收到 New logic 时。最后，选择 保存 以保存计划。

最后，由于我们尚未更新 house_price_predictions 数据集，可以运行此操作以生成预测。构建完成后，您可以打开 house_price_predictions 数据集，在 prediction 列中查看新生成的预测。

操作：选择 立即运行 在计划视图中构建 house_price_predictions 数据集。构建完成后，可以通过右键单击 house_price_predictions 数据集并选择 打开 来查看生成的预测。

4.3 如何为您的模型创建一个交互式实时端点

实时部署是一个可查询的端点，它在 REST 端点后托管生产模型。当您想要与模型进行交互式操作时，实时部署很有用，并且可以从以下位置查询：

  • 模型上的函数：模型上的函数 允许直接从 Slate 或 Workshop 应用程序在 Foundry 中查询模型。
  • 实时外部系统： 例如一个需要根据用户行为生成实时预测的网站。
  • CURL: 用于对您的模型进行本地测试。

在此示例中，您可能希望构建一个交互式仪表板，允许用户输入人口普查区域的详细信息，并查看这如何影响中位数房价。

操作: 从目标主页，点击创建部署并配置生产环境的实时部署。为部署命名 Live deployment 并添加描述 Production live deployment。 点击创建部署以保存配置。

实时部署可能需要几分钟才能启动。一旦初始化完成，您可以设置实时操作应用程序以连接到模型。

操作: 选择您的实时部署。当部署升级完成后，打开 查询 选项卡。粘贴以下示例并选择 运行 要测试模型。

警告

实时部署由持续运行的服务器支撑。一旦启动，Foundry不会自动终止暂存或生产部署。实时部署可能会产生较高的费用，因此请确保在完成教程后禁用或删除此实时部署。请注意，直接部署 可以配置为缩减到零，并在需要时启动服务器，尽管首次调用会因服务器启动而需要更长的响应时间。

您可以通过实时部署视图中的操作下拉菜单禁用或删除实时部署。

1from transforms.api import transform, Input
2from palantir_models.transforms import ModelOutput
3from main.model_adapters.adapter import SklearnRegressionAdapter
4
5
6def derive_housing_age_per_income(X):
7    X['housing_age_per_income'] = X['housing_median_age'] / X['median_income']
8    return X
9
10
11def train_model(training_df):
12    from sklearn.ensemble import RandomForestRegressor
13    from sklearn.impute import SimpleImputer
14    from sklearn.pipeline import Pipeline
15    from sklearn.preprocessing import StandardScaler, FunctionTransformer
16
17    numeric_features = ['median_income', 'housing_median_age', 'total_rooms']
18
19    numeric_transformer = Pipeline(
20        steps=[
21            ("rooms_per_person_transformer", FunctionTransformer(derive_housing_age_per_income, validate=False)),
22            ("imputer", SimpleImputer(strategy="median")),
23            ("scaler", StandardScaler())
24        ]
25    )
26
27    model = Pipeline(
28        steps=[
29            ("preprocessor", numeric_transformer),
30            ("regressor", RandomForestRegressor())
31        ]
32    )
33    X_train = training_df[numeric_features]
34    y_train = training_df['median_house_value']
35    model.fit(X_train, y_train)
36
37    return model
38
39@transform.using(
