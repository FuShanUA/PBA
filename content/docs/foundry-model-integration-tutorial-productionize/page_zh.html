4. 教程 - 模型生产化

在继续本教程之前，您应已完成《建模项目设置》和《模型训练》教程，并可选择完成《模型评估》教程。此时，您的建模目标中应至少包含一个模型。

在本教程的这一步骤中，您将获取您的机器学习模型，并为该模型设置生产环境用途。

  1. 在建模目标中发布模型。
  2. 为批处理创建批量部署。
  3. 为您的模型创建交互式在线端点。

4.1 在建模目标中发布模型

既然我们已经有了一个模型，就可以将其部署到生产环境中了。根据该模型的预期用途，我们可以通过几种方式将其生产化。可以从建模目标中部署模型，这提供了功能完善的发布和模型管理能力，包括文档记录、评估，以及支持对比来自多个支撑模型的提交结果。

从建模目标进行部署可以通过以下方式实现：

  • 在批量部署中：为数据集生成预测，推荐用于一次性计算一组记录的预测结果。
  • 在线部署中：在 REST 端点后托管模型，推荐用于从外部系统使用该模型，或者需要与模型进行实时交互的场景。
  • 在 Python 转换中：适用于在转换中进行快速迭代。

另外，倾向于更轻量化设置的用户，可以按照《在代码仓库中运行批量推理的教程》所述，在转换中导入模型进行批量推理；或者使用直接部署，在无需创建建模目标的情况下，将其部署在 REST 端点之后。

了解更多关于《建模目标在线部署与直接部署之间的差异》的信息。

什么是建模目标中的发布？

批处理与实时部署均由模型发布提供支持。在某个建模目标中发布新模型后，该建模目标下的部署任务会自动升级并应用最新发布的模型。

这使得模型使用者无需关注具体的实现细节。数据科学家可以专注优化模型本身，而应用开发者则能聚焦于开发实用的应用程序。

发布管理

发布与部署分为预发布（staging）和生产（production）两类环境。模型开发者可以先在预发布环境中发布模型，再推送到生产环境。此操作会升级预发布环境的部署任务，但不影响生产环境的既有部署。随后，数据科学家或应用构建人员可在模型正式投产后前，先在预发布环境对其进行测试。

系统会记录所有发布记录以及发布模型的具体人员信息，确保团队能准确追踪何时使用了哪个模型。这有助于解答 GDPR 或《欧盟人工智能法案》等监管合规要求相关的问题。

操作步骤：前往建模目标的主页并滚动至发布区域。选择发布到预发布环境，随后点击箭头图标将模型发布至生产环境。你需要为本次发布指定一个版本号，例如“1.0”。

4.2 如何为批处理创建批处理部署

批处理部署将基于输入数据集创建一个 Foundry 转换任务。批处理部署的输出是一个数据集，该数据集是使用模型对输入数据集运行推理（生成预测）的结果。

操作步骤：点击“创建部署”并配置生产环境的批处理部署。为部署指定名称 Batch deployment，描述为 Production batch deployment。选择先前创建的 housing_inference_data 数据集作为输入数据集，并在 data 文件夹中新建名为 house_price_predictions 的输出数据集。选择创建部署以保存配置。

现在，我们可以为批量部署设置计划任务，这样每当发布新模型时，输出数据集就会自动更新——确保我们始终获得最佳预测结果。让我们为输出数据集添加一个计划任务，使其在接收到新逻辑时自动重新构建。

操作：在部署表中选择您的批量部署。选择名为 house_price_predictions 的输出数据集。在 所有操作 下，选择 管理计划任务。在 house_price_predictions 数据集上，选择 创建新计划任务 -> 当满足多个时间或事件条件时，即当 house_price_predictions 数据集接收到 新逻辑 时。最后，选择 保存 以保存该计划任务。

最后，由于我们尚未更新 house_price_predictions 数据集，我们可以运行此操作来生成预测。构建完成后，您可以打开 house_price_predictions 数据集，在 prediction 列中查看最新生成的预测结果。

操作：在计划任务视图中选择 立即运行 以构建 house_price_predictions 数据集。构建完成后，您可以右键点击 house_price_predictions 数据集并选择 打开，查看生成的预测结果。

4.3 如何为模型创建交互式实时端点

实时部署是一个可查询的端点，它将生产模型托管在 REST 端点之后。当您希望与模型进行交互式访问时，实时部署非常有用，可以从以下位置进行查询：

  • 模型函数： 模型函数支持在 Foundry 中直接通过 Slate 或 Workshop 应用程序查询模型。
  • 实时外部系统： 例如需要根据用户行为生成实时预测的网站。
  • CURL： 用于在本地测试您的模型。

在此示例中，您可能希望构建一个交互式仪表盘，允许用户输入某个普查区的详细信息，并查看其对房屋中位数价格的影响。

操作： 在目标主页中，点击“创建部署”并配置生产环境的实时部署。将部署命名为 实时部署，描述填写为 生产环境实时部署。点击“创建部署”保存配置。

实时部署启动可能需要几分钟时间。初始化完成后，您可以设置实时操作应用程序以连接到该模型。

操作： 选择您的实时部署。部署升级完成后，打开 查询 选项卡。粘贴以下示例并选择 运行 以测试模型。

警告
实时部署由持续运行的服务器提供支持。一旦启动，Foundry 将不会自动终止预发布或生产环境的部署。实时部署可能会产生较高的费用，因此在完成本教程后，请务必禁用或删除该实时部署。需要注意的是，直接部署可以配置为缩容至零，并根据需要启动服务器，不过在服务器启动期间，首批调用的响应时间会较长。
你可以通过实时部署视图中的操作下拉菜单来禁用或删除实时部署。
