模式和功能

AI FDE 使用模式和功能来完成任务，并提供一种简便的方式来管理代理的上下文。模式 是当前的主要任务，例如数据集成或本体论编辑，而 功能 是可以在不同模式中使用的细粒度能力。

模式

模式告诉代理您正在处理哪种类型的任务。您可以手动选择一个模式，或者只需在输入字段中输入您的任务，让代理为您选择模式。当任务发展时，代理还可以在任务进行中更改模式。模式通过加载正确的文档，为代理提供相关工具的访问权限，并调整其解决问题的方式，从而聚焦代理。为代理提供正确的上下文有助于确保它们不会分心或使用错误的工具；编写 Python 转换的代理不需要治理或 React 应用程序工具，因此只提供相关的文档和工具。

AI FDE 模式包括以下内容：

  • 数据集成：构建或修改数据管道（Python 转换或 Pipeline 构建器）。
  • 数据连接：创建、管理和调试 数据连接 源、出站策略和其他功能。
  • 本体论编辑：创建或更新构成您本体论的对象、关联和操作。
  • 函数编辑：在 Logic、TypeScript 或 Python 中编写 Foundry 函数。
  • 探索：只读调查；在进行更改之前了解您的平台中存在什么。
  • 治理：审计权限、访问控制、标记和数据保护。
  • 机器学习： 训练、评估、部署和调优机器学习模型。涵盖分类、回归、时间序列预测和自定义预测建模。
  • OSDK React： 构建连接到Foundry数据的React应用程序或自定义小部件。
  • 平台问答： 关于Foundry工作原理的一般性问题。

某些模式允许您优化其配置。代理会根据这些选择来确定要阅读哪些文档以及调用哪些工具。

模式配置选项
数据集成Python转换或Pipeline Builder
函数编辑语言选择
机器学习Model Studio（无代码）或专业代码开发，以及首选的代码编辑环境

功能

功能是代理可以在任何模式下使用的单个能力。虽然模式决定了任务的大致上下文，但功能更具体。每个功能对应一个或多个代理可以调用的具体工具。功能分为代理功能和领域功能。

代理功能 是代理自我管理及通信的方式。这包括以下内容：

  • 更改模式： 当任务需要时，代理可以在任务进行中切换到不同的模式，而无需您手动切换。
  • 请求澄清： 在继续之前，如果代理需要更多信息，它可以向您提问（多项选择或自由文本）。
  • 生成计划： 在采取行动之前，代理会起草一个计划供您审阅。这对于模糊或涉及多个步骤的任务很有用。
  • 加载文档： 代理可以根据需要查找 Foundry 文档。
  • 管理上下文/管理功能： 代理可以整理自己的工作内存，并根据任务的发展调整哪些功能处于激活状态。

领域功能 是代理在 Foundry 中可以执行的实际操作，包括但不限于以下内容：

  • 文件系统： 创建文件夹、浏览资源并移动内容。
  • 便签本： 读取、创建和更新 便签本 文档。
  • 解决方案设计： 创建和编辑解决方案设计图。
  • 执行操作： 对本体论对象执行操作。

功能可以启用或禁用。如需，代理还可以在任务进行中开启或关闭功能，这由 管理功能 实现。
