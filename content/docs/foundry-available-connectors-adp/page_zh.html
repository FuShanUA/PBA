ADP

ADP 连接器是 Palantir 提供的用于 ADP 的驱动程序。

要创建新的 ADP 源，请遵循《Palantir 提供的驱动程序的标准设置流程》，然后使用以下章节进行 ADP 特定的配置和网络设置。有关完整的属性参考，请参阅《官方 ADP 驱动程序文档 ↗》。

配置

以下属性为必填或推荐项。

属性是否必填？描述默认值
SSLClientCert ↗必填指定由 ADP 签发的 TLS (SSL) 客户端证书，您的应用程序将出示该证书进行身份验证。—
TestConnectionEndpoint ↗必填指定提供程序用于测试与 ADP 连接的 API 端点。workers
UseUAT ↗必填指定提供程序是否连接到 ADP 用户验收测试 (UAT) 环境而非生产环境。FALSE
IncludeCustomFields ↗推荐一个布尔值，指示是否在列列表中包含自定义字段。TRUE
InitiateOAuth ↗推荐指定获取或刷新 OAuth 访问令牌的流程，以便在经过身份验证和授权的用户工作时维持其访问权限。GETANDREFRESH
OAuthClientId ↗推荐指定分配给您的自定义 OAuth 应用程序的客户端 ID（也称为使用者密钥）。在身份验证期间，需要此 ID 向 OAuth 授权服务器标识该应用程序。—
OAuthClientSecret ↗推荐指定分配给您的自定义 OAuth 应用程序的客户端密钥。此机密值用于向 OAuth 授权服务器验证应用程序。（仅适用于自定义 OAuth 应用程序。）—
SSLClientCertPassword ↗推荐指定访问 TLS/SSL 客户端证书存储所需的密码。如果所选的证书存储类型需要密码才能访问，请使用此属性。—
SSLClientCertType ↗推荐包含 TLS/SSL 客户端证书的密钥存储类型。PFXBLOB

网络连接

下表列出了源为成功运行而必须能够访问的域。

对于每个域，请添加相应的出站策略。如果源托管在本地并且无法从 Foundry 直接访问，请改用智能体代理出站策略；智能体主机本身也必须能够访问所列出的域。详情请参阅《将智能体用作代理》。

域是否必需
api.adp.com如果 UseUAT=FALSE
accounts.adp.com如果 UseUAT=FALSE
uat-api.adp.com如果 UseUAT=TRUE
uat-accounts.adp.com如果 UseUAT=TRUE
