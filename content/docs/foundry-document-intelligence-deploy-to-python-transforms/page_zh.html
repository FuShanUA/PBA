将提取策略部署到Python转换

在AIP文档智能中验证您的提取策略后，可以将其作为Python转换进行部署，以在媒体集中的所有文档的所有页面上运行批量提取。已部署的模板会生成与AIP文档智能中相应配置相同的结果。

使用已部署的模板

一旦在代码仓库中创建了模板：

  1. 在 src/myproject/document_extraction/my_extraction.py 文件中的 @transform.using 装饰器中指定您的输出数据集。
  2. 触发构建。

该模板使用轻量级转换以实现最佳性能。旧版本使用基于Spark的转换，由于Spark的开销，速度较慢。如果您尚未迁移，请建议您迁移到轻量级转换。

此模板不支持预览模式。使用预览时会出现错误，但实际构建将正常工作。

增量处理

默认情况下，文档提取转换是非增量的，这意味着每次运行时都会处理所有文档。您可以通过取消注释 @incremental(...) 装饰器行来配置转换以进行增量运行。对于增量转换，当新的文档被添加到输入媒体集时，重新运行转换将只处理新文档并将结果追加到输出数据集。

自定义提示

对于生成式AI配置，模板继承了您在AIP文档智能中指定的提示。您可以在 src/myproject/document_extraction/prompts.py 中查看提示。

我们不建议直接在模板中编辑提示，因为这会导致文档智能结果与批量作业结果之间出现差异。相反，请在文档智能中进行提示调整，然后验证结果，再重新部署以创建新模板。

转换输入类型可自定义的提示
VisionLLMDocumentsExtractorInput仅用户提示（系统提示是固定的）
VisionLLMLayoutDocumentsExtractorInput（布局感知提取）仅系统提示（用户提示是固定的）

For layout-aware extraction configurations, the user prompt must remain fixed because it contains a special JSON schema that preserves layout structure information. Modifying this prompt will significantly reduce extraction success rates.

Custom image preprocessing

For documents that require image transformations before extraction, such as rotated text content, you should:

  1. Create a separate transform pipeline to apply the image transformations.
  2. Save the processed results to a new media set.
  3. Use Document Intelligence on the processed media set for extraction.

Run on a subset of media items

For layout-aware generative AI configurations, the vision LLM must generate valid JSON adhering to a specific schema. If the response is invalid JSON or does not follow the schema, the extraction fails with an ERROR_RESPONSE_JSON_PARSING error.

In practice, approximately 5% of extractions may fail with top-tier models. For failed rows, you still get valid layoutInfo with extraction results from the layout model only.

To re-run extraction on failed rows:

  1. Use the filter_on_media_items argument with a list of media item IDs to process only specific items.
  2. Remove the @incremental decorator so these rows are re-processed instead of being detected as finished.

Improve runtime performance

The THREAD_NUMBER parameter controls concurrent threads, where each thread extracts data from one document page at a time. Higher values result in faster job completion.

SettingValueNotes
Default20Conservative setting suitable for most environments
Maximum tested300Achievable in development environments with abundant Vision LLM capacity

Setting the THREAD_NUMBER value too high in capacity-restricted environments will cause rate limit errors. The retry loop then consumes significant capacity, affecting other jobs using the same model. You should monitor usage when adjusting this parameter.

Find logs

To view build logs, select Telemetry on the build details page. To filter for document extraction logs, filter the message column for values starting with aip_workflows.

Row-level vs. document-level chunking

The extraction output contains one row per page. By default, DocumentChunker.create_chunks_per_document combines all pages from the same document into a single Markdown string before chunking.

To chunk each row independently without combining pages, use DocumentChunker.create_chunks_per_row instead:



Copied!
1chunking_result = chunker.create_chunks_per_row(
2    extraction_df,
3    chunk_mode="markdown",       # "recursive" for plain text, "markdown" for markdown text
4    content_column="extractionResult",
5    id_column="media_item_rid",  # used as prefix for chunk_id
6    chunk_size=8192,
7    chunk_overlap=0,
8    thread_number=20,
9    strip_markdown=False,        # set True to remove ```markdown and ``` wrappers before chunking
10)

Create embeddings without chunking

Chunking is recommended before creating embeddings because embedding models have context limits. To skip chunking while preserving the pipeline structure, set chunk_size to a very large value in create_chunks_per_row, such as sys.maxsize.

Template examples

The following examples show the transform code generated for each extraction configuration. These are for reference only. You should create the transform using the deploy tool in Document Intelligence instead of manually writing transform code for document extraction.

Traditional extraction: Raw text

Extracts text by reading document metadata. Only available for electronically generated PDFs.



Copied!
1import polars as pl
2from concurrent.futures import ThreadPoolExecutor
3from transforms.api import Output, incremental, transform
4from transforms.mediasets import MediaSetInput
5from transforms.mediasets.utils._constants import MEDIA_ITEM_RID, MEDIA_REFERENCE, PATH
6
7
8THREAD_NUMBER = 20
9
10
11# @incremental(v2_semantics=True)  # uncomment this line if incremental is needed
12@transform.using(
13    output=Output("ri.foundry.main.dataset.abc"),
14    media_input=MediaSetInput("ri.mio.main.media-set.abc"),
15)
16def extract(media_input, output):
17    """
18    Extracts content from pdf documents with raw text extraction
19    """
20    media_refs = pl.from_pandas(
21        media_input.list_media_items_by_path_with_media_reference().pandas(),
22        schema_overrides={MEDIA_ITEM_RID: pl.String, MEDIA_REFERENCE: pl.String, PATH: pl.String},
23    )
24
25    def process_batch(batch_df: pl.DataFrame) -> pl.DataFrame:
26        def create_page_tasks(row):
27            media_item_rid = row[MEDIA_ITEM_RID]
28            metadata = media_input.get_media_item_metadata(media_item_rid).document
29            if metadata is None:
30                raise ValueError(f"Media item {media_item_rid} is not a document")
31            if metadata.pages is None:
32                raise ValueError(f"Media item {media_item_rid} has no page count")
33            return [(row, page_num) for page_num in range(metadata.pages)]
34
35        def process_single_page(task):
36            row, page_num = task
37            media_item_rid = row[MEDIA_ITEM_RID]
38            media_reference = row[MEDIA_REFERENCE]
39            extraction_result = media_input.transform_document_to_text_raw(
40                media_item_rid, page_num
41            ).read().decode("utf-8")
42            return {
43                "media_item_rid": media_item_rid,
44                "media_reference": media_reference,
45                "page_num": page_num,
46                "extraction_result": extraction_result
47            }
48
49        all_tasks = []
50        for row in batch_df.iter_rows(named=True):
51            all_tasks.extend(create_page_tasks(row))
52
53        with ThreadPoolExecutor(max_workers=THREAD_NUMBER) as executor:
54            results = list(executor.map(process_single_page, all_tasks))
55
56        return pl.DataFrame(results)
57
58    extracted_data = media_refs.lazy().map_batches(
59        process_batch,
60        schema={
61            "media_item_rid": pl.String,
62            "media_reference": pl.String,
63            "page_num": pl.Int64,
64            "extraction_result": pl.String,
65        },
66        streamable=True,
67    )
68
69    output.write_dataframe(extracted_data)

Chunk extracted text and generate embeddings

If embedding is not needed, remove the embedder from the transform decorator and the embedding_result line.



Copied!
1from transforms.api import Input, Output, incremental, transform
2from aip_workflows.document_intelligence.transforms import DocumentChunker, DocumentEmbedderInput
3
4THREAD_NUMBER = 20
5
6
7# @incremental(v2_semantics=True, snapshot_inputs=["embedder"])  # uncomment this line if incremental is needed
8@transform.using(
9    extraction_input=Input("ri.foundry.main.dataset.abc"),  # typically the output dataset from the extraction transform
10    output=Output("ri.foundry.main.dataset.xyz"),
11    embedder=DocumentEmbedderInput("ri.language-model-service..language-model.text-embedding-3-large"),
12)
13def chunk_and_embed(extraction_input, output, embedder):
14    extraction_df = extraction_input.polars(lazy=True)
15    chunker = DocumentChunker()
16    chunking_result = chunker.create_chunks_per_document(
17        extraction_df,
18        chunk_mode="markdown",  # "recursive" for raw text, "markdown" for markdown text
19        content_column="extractionResult",  # content column name
20        id_column="media_item_rid",  # id of the document, used to combine content (e.g. from different pages) of the single document
21        page_column="page_num",  # page number column name
22        chunk_size=8192,
23        chunk_overlap=0,
24        thread_number=THREAD_NUMBER,
25        strip_markdown=True,  # when True, removes ```markdown prefix and ``` suffix from the content before chunking
26    )
27    embedding_result = embedder.create_embeddings(
28        chunking_result,
29        content_column="chunk_content",
30        thread_number=THREAD_NUMBER,
31    )
32    output.write_dataframe(embedding_result)
