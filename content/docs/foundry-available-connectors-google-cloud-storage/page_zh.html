谷歌云存储

将Foundry连接到谷歌云存储，以便在Foundry数据集和存储桶之间同步文件。

支持的功能

功能状态
探索🟢 通常可用
批量导入🟢 通常可用
增量🟢 通常可用
虚拟表🟢 通常可用
导出任务🟡 逐渐淘汰
文件导出🟢 通常可用

数据模型

数据模型

连接器可以将任何类型的文件传输到Foundry数据集。文件格式得到保留，在传输过程中或之后不会应用任何模式。对输出数据集应用任何必要的模式，或者编写下游转换以访问数据。

性能和限制

可传输文件的大小没有限制。然而，网络问题可能导致大规模传输失败。特别是运行时间超过两天的直接云同步将被中断。为了避免网络问题，我们建议使用较小的文件大小，并限制每次同步执行时摄入的文件数量。同步可以 安排 为频繁运行。

设置

  1. 打开 数据连接 应用并选择屏幕右上角的+ 新数据源。
  2. 从可用的连接器类型中选择Google Cloud Storage。
  3. 按照后续配置提示继续使用以下部分中的信息设置您的连接器。

了解有关在Foundry中< a href="/docs/foundry/data-connection/set-up-source/">设置连接器的更多信息。

您必须拥有一个 Google Cloud IAM服务账户 ↗，以便进行Google Cloud Storage身份验证和设置。

认证

访问存储桶所需的以下角色：

  • Storage Object Viewer: 读取数据。
  • Storage Object Creator: 将数据导出到Google Cloud Storage。
  • Storage Object Admin: 在将文件导入Foundry后删除Google Cloud Storage中的文件以及使用UPDATE事务和覆盖文件的增量数据集导出时所需。

了解更多有关所需角色的信息，请参阅 Google Cloud 关于访问控制的文档 ↗。

选择以下可用的身份验证方法之一：

  • GCP 实例： 请参阅 Google Cloud 文档 ↗，了解如何设置基于实例的身份验证。

    • 请注意，GCP 实例身份验证仅适用于通过在 GCP 中适当配置的实例上运行的代理程序运行的连接器。

  • JSON 凭据： 请参阅 Google Cloud 文档 ↗，了解如何创建和下载 JSON 服务账户密钥文件。

  • PKCS8 身份验证： 需要从 JSON 服务账户密钥文件中输入特定的凭据信息。请参阅 Google Cloud 文档 ↗，了解如何创建密钥文件。

  • 工作负载身份联合（OIDC）： 按照显示的源系统配置说明设置 OIDC。有关工作负载身份联合的详细信息，请参阅 Google Cloud 文档 ↗，以及 我们的文档，了解 OIDC 如何与 Foundry 配合使用。

网络

Google Cloud Storage 连接器需要对以下域名的 443 端口进行网络访问：

  • storage.googleapis.com
  • oauth2.googleapis.com（仅在使用 JSON 凭据或 PKCS8 认证时需要）
  • sts.googleapis.com（仅在使用 Workload Identity Federation 时需要）
  • iamcredentials.googleapis.com（仅在使用 Workload Identity Federation 时需要）

配置选项

Google Cloud Storage 连接器有以下配置选项：

选项是否必需？描述
Project Id是包含 Cloud Storage 存储桶的项目的 ID。
Bucket name是要读取和写入数据的存储桶的名称。

虚拟表

本节提供了有关使用来自Google Cloud Storage源的虚拟表的更多信息。当同步到Foundry数据集时，本节不适用。

下表突出了Google Cloud Storage支持的虚拟表功能。

功能状态
批量注册 🔴 不可用
自动注册🔴 不可用
表输入🟢 通常可用： Avro ↗, Delta ↗, Iceberg ↗, Parquet ↗ 在 代码仓库，管道构建器 中
表输出🟢 通常可用： Avro ↗, Delta ↗, Iceberg ↗, Parquet ↗ 在 代码仓库，管道构建器 中
增量管道🟢 对 Delta 表通常可用： APPEND 仅（详情）
🟢 对 Iceberg 表通常可用： APPEND 仅（详情）
🔴 对 Parquet 表不可用
计算下推🔴 不可用

有关存储在 Google Cloud Storage 中的表如何作为输入或输出使用的支持 Foundry 工作流程的详细信息，请参阅 虚拟表文档。

源配置要求

使用 虚拟表 时，请记住以下源配置要求：

  • 您必须使用 Foundry worker 源。虚拟表不支持使用 agent worker 连接。
  • 确保已按照 本文件的网络部分 中所述建立双向连接和允许列表。
  • 如果在代码仓库中使用虚拟表，请参考 虚拟表文档 以获取额外源配置的详细信息。
  • 在设置源凭据时，您必须使用 JSON credentials、PKCS8 auth 或 Workload Identity Federation (OIDC) 中的一种。使用虚拟表时，其他凭据选项不受支持。

Delta

为了启用由虚拟表支撑的管道的增量支持，请确保 更改数据流 ↗ 在源 Delta 表中已启用。 Python 转换 中的 current 和 added 读取模式受到支持。 _change_type、_commit_version 和 _commit_timestamp 列将在 Python 转换中可用。

Iceberg

加载由 Apache Iceberg 表支撑的虚拟表需要一个 Iceberg 目录。有关 Iceberg 目录的更多信息，请参见 Apache Iceberg 文档 ↗。在源上注册的所有 Iceberg 表必须使用相同的 Iceberg 目录。

表将使用 GCS 中的 Iceberg 元数据文件进行创建。在注册表时，必须提供一个 warehousePath 来指示这些元数据文件的位置。

增量支持依赖于 Iceberg 增量读取 ↗，目前仅支持追加操作。 Python 转换 中的 current 和 added 读取模式受到支持。

Parquet

使用 Parquet 的虚拟表依赖于模式推断。最多将使用 100 个文件来确定模式。

导出数据到 Google Cloud Storage

该连接器可以将 Foundry 数据集中的文件复制到 Google Cloud Storage 存储桶的任何位置。

要开始导出数据，您必须配置一个导出任务。导航到包含您想要导出的 Google Cloud Storage 连接器的项目文件夹。右键点击连接器名称，然后选择 Create Data Connection Task。

在数据连接视图的左侧面板中：

  1. 确认 Source 名称与您要使用的连接器匹配。
  2. 添加一个名为 Input 的 inputDataset。 输入数据集 是正在导出的 Foundry 数据集。
  3. 添加一个名为 Output 的 outputDataset。 输出数据集 用于运行、计划和监控任务。
  4. 最后，在文本字段中添加一个 YAML 块来定义任务配置。

在左侧面板中显示的连接器和输入数据集的标签不反映 YAML 中定义的名称。

创建导出任务 YAML 时请使用以下选项：

选项是否必需？描述
directoryPath是文件将被写入的云存储目录。
excludePaths否正则表达式列表；与这些表达式匹配的文件名不会被导出。
uploadConfirmation否当值为 exportedFiles 时，输出数据集将包含已导出的文件列表。
retriesPerFile否如果遇到网络故障，请增加此数值以允许导出任务在失败前重试上传到云存储。
createTransactionFolders否 启用后，数据将写入指定的 directoryPath 内的子文件夹。每个子文件夹基于在 Foundry 中事务提交的时间，并且每个导出的事务都有唯一的名称。
threads否设置用于并行上传文件的线程数。增加数量以使用更多资源。确保在代理上运行的导出任务在代理上有足够的资源来处理增加的并行化。
incrementalType否对于逐步构建的数据集，设置为 incremental 以仅导出自上次导出以来发生的事务。

示例任务配置：

Copied!
1type: export-google-cloud-storage
2directoryPath: directory/to/export/to
3excludePaths:
4  -  ^_.*
5  - ^spark/_.*
6uploadConfirmation: exportedFiles
7incrementalType: incremental
8retriesPerFile: 0
9createTransactionFolders: true
10threads: 0

配置导出任务后，在右上角选择 保存。
