Google Cloud Storage

Connect Foundry to Google Cloud Storage to sync files between Foundry datasets and storage buckets.

Data model

The connector can transfer files of any type into Foundry datasets. File formats are preserved, and no schemas are applied during or after the transfer. Apply any necessary schema to the output dataset, or write a downstream transformation to access the data.

Performance and limitations

There is no limit to the size of transferable files. However, network issues can result in failures of large-scale transfers. In particular, direct cloud syncs that take more than two days to run will be interrupted. To avoid network issues, we recommend using smaller file sizes and limiting the number of files that are ingested in every execution of the sync. Syncs can be scheduled to run frequently.

Setup

  1. Open the Data Connection application and select + New Source in the upper right corner of the screen.
  2. Select Google Cloud Storage from the available connector types.
  3. Follow the additional configuration prompts to continue the set up of your connector using the information in the sections below.

Learn more about setting up a connector in Foundry.

You must have a Google Cloud IAM service account ↗ to proceed with Google Cloud Storage authentication and set up.

Authentication

The following roles are required on the bucket being accessed:

  • Storage Object Viewer: Read data.
  • Storage Object Creator: Export data to Google Cloud Storage.
  • Storage Object Admin: Required for deleting files from Google Cloud Storage after importing them into Foundry, and also for exports with incremental datasets that use UPDATE transactions and overwrite files.

Learn more about required roles in the Google Cloud documentation on access control ↗.

Choose from one of the available authentication methods:

  • GCP instance: Refer to the Google Cloud documentation ↗ for information on how to set up instance-based authentication.

    • Note that GCP instance authentication only works for connectors operating through agents that run on appropriately configured instances in GCP.

  • JSON credentials: Refer to the Google Cloud documentation ↗ for information on how to create and download a JSON service account key file.

  • PKCS8 auth: Requires entering specific credential information from the JSON service account key file. Refer to the Google Cloud documentation ↗ for information on creating the key file.

  • Workload Identity Federation (OIDC): Follow the displayed source system configuration instructions to set up OIDC. Refer to the Google Cloud Documentation ↗ for details on Workload Identity Federation and our documentation for details on how OIDC works with Foundry.

Networking

The Google Cloud Storage connector requires network access to the following domains on port 443:

  • storage.googleapis.com
  • oauth2.googleapis.com (only required when using JSON credentials or PKCS8 auth)
  • sts.googleapis.com (only required when using Workload Identity Federation)
  • iamcredentials.googleapis.com (only required when using Workload Identity Federation)

虚拟表

本节提供了关于从 Google Cloud Storage 源使用虚拟表的更多细节。在同步至 Foundry 数据集时，本节内容不适用。

下表重点列出了 Google Cloud Storage 支持的虚拟表功能。

功能状态
批量注册🔴 不可用
自动注册🔴 不可用
表输入🟢 正式发布：在《代码仓库》和《管道构建器》中支持 Avro ↗、Delta ↗、Iceberg ↗、Parquet ↗
表输出🟢 正式发布：在《代码仓库》和《管道构建器》中支持 Avro ↗、Delta ↗、Iceberg ↗、Parquet ↗
增量管道🟢 针对 Delta 表正式发布：仅支持 APPEND（详情）

有关存储在 Google Cloud Storage 中的表作为输入或输出时所支持的 Foundry 工作流详情，请查阅《虚拟表文档》。

源配置要求

使用虚拟表时，请注意以下源配置要求：

  • 必须使用 Foundry worker 源。虚拟表不支持使用智能体 worker 连接。
  • 确保已按照本文档的《网络配置部分》建立双向连通性并完成白名单放行。
  • 如果在代码仓库中使用虚拟表，请查阅《虚拟表文档》以了解所需的额外源配置详情。
  • 设置源凭据时，必须使用 JSON 凭据、PKCS8 认证 或 工作负载身份联合 (OIDC) 中的一种。使用虚拟表时不支持其他凭据选项。

Delta

若要为基于虚拟表的管道启用增量支持，请确保在源 Delta 表上启用了《变更数据流 ↗》。支持在 Python 转换中使用 current 和 added 读取模式。_change_type、_commit_version 和 _commit_timestamp 列将在 Python 转换中可用。

Iceberg

加载由 Apache Iceberg 表支持的虚拟表需要 Iceberg 目录。要了解有关 Iceberg 目录的更多信息，请参阅《Apache Iceberg 文档 ↗》。在同一个源上注册的所有 Iceberg 表必须使用相同的 Iceberg 目录。

表将使用 GCS 中的 Iceberg 元数据文件进行创建。注册表时必须提供一个 warehousePath，用于指示这些元数据文件的位置。

增量支持依赖于 Iceberg 的《增量读取 ↗》，目前仅支持追加模式。支持在 Python 转换中使用 current 和 added 读取模式。

Parquet

使用 Parquet 的虚拟表依赖模式推断。最多将使用 100 个文件来确定模式。

导出数据至 Google Cloud Storage

连接器可以将文件从 Foundry 数据集复制到 Google Cloud Storage 存储桶中的任何位置。

若要开始导出数据，您必须配置一个导出任务。导航至包含您要导出到的 Google Cloud Storage 连接器的项目文件夹，右键单击连接器名称，然后选择 创建数据连接任务。

在数据连接视图的左侧面板中：

  1. 验证 源 名称与您要使用的连接器是否匹配。
  2. 添加一个名为 inputDataset 的 输入。输入数据集 即为正在导出的 Foundry 数据集。
  3. 添加一个名为 outputDataset 的 输出。输出数据集 用于运行、调度和监控任务。
  4. 最后，在文本框中添加一个 YAML 代码块以定义任务配置。

左侧面板中显示的连接器和输入数据集标签并不反映在 YAML 中定义的名称。

创建导出任务 YAML 时，请使用以下选项：

选项是否必填？描述
directoryPath是云存储中用于写入文件的目录。
excludePaths否正则表达式列表；文件名匹配这些表达式的文件将不会被导出。
uploadConfirmation否当值为 exportedFiles 时，输出数据集将包含已导出文件的列表。
retriesPerFile否遇到网络故障时，增大此数值可允许导出任务在整体失败前，多次重试向云存储上传文件。
createTransactionFolders否启用后，数据将写入指定的 directoryPath 内的子文件夹中。每个子文件夹的名称取决于事务在 Foundry 中提交的时间，且每个导出的事务对应唯一的文件夹名称。
threads否设置用于并行上传文件的线程数。增大该数值会消耗更多资源。需确保在智能体上运行的导出任务拥有充足的资源，以应对并行度的提升。
incrementalType否对于增量构建的数据集，将其设置为 incremental 可仅导出上次导出之后发生的事务。

任务配置示例：

已复制！
1type: export-google-cloud-storage
2directoryPath: directory/to/export/to
3excludePaths:
4  -  ^_.*
5  - ^spark/_.*
6uploadConfirmation: exportedFiles
7incrementalType: incremental
8retriesPerFile: 0
9createTransactionFolders: true
10threads: 0

配置完导出任务后，请在右上角选择保存。
