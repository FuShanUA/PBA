在模块之间配置导航

Carbon 工作区的优势之一是能够执行跨越多个模块的流程。这为用户提供了统一的工作流程，而不是一组孤立且不相连的应用程序。

在进行他们的工作流程时，用户不应该以特定的产品、模块或接口来思考。相反，用户应该能够无缝地完成他们的工作。Carbon 提供了一个导航框架，使对象或一组对象可以作为参数从一个模块传递到另一个模块。

Carbon 中导航简介

作为 Carbon 的构建者，您可以定义 输入（input） 和 输出（output） 对于每个模块以及当前可操作的对象或对象集合。可操作的对象或对象集合可以是列表上的选择、函数的结果、给定类型的全部对象，或者任何其他可以使用本体论原语编码的对象集合。在接收端，另一个模块可以指定（通过适当的约束）其参数中哪一个可以接受上述的输出。

在模块之间导航

如果用户决定从一个模块导航到另一个模块（通常是通过动作菜单中的按钮或其他从前端组件触发的动作），Carbon会在新标签页中打开接收模块，使用户能够在不丢失原始模块中保存的状态的情况下逐步完成工作流。

这有助于通过使一个模块的结果更容易分支到另一个模块的多个实例中，从而更轻松地构建复杂的流程。例如，Object Explorer 列表中的结果可以独立作为 Object View 标签页打开并进行检查，而不会丢失对原始结果列表的跟踪。

多个导航步骤

模块之间的导航操作数量没有限制。这意味着可以通过导航框架促进任意数量的工作流步骤，因为每个后续模块的输出都可以成为另一个模块的输入。同一个模块甚至可以在工作流中多次出现，如下面所示，每次可能有不同的输入。

多个导航步骤示例

在下面的流程图中，我们想使用Carbon工作区中可用的工具，找到一个特定的单通道飞机，以开始进行Quiver分析。

首先，我们搜索关键字aircraft；导航框架会将我们带到Search模块，并将aircraft作为输入。

我们从搜索结果中选择Aircraft object type。导航框架会将我们带到Object Explorer模块，并将Aircraft object type作为输入。

现在，我们想分析Single Aisle Aircrafts Quiver模块中的所有对象结果。导航框架会用Aircraft类型的185个对象作为输入，打开Single Aisle Aircrafts Quiver模块。

从该模块中，我们选择Q-AGM飞机进行进一步检查。导航框架会用Q-AGM Aircraft作为输入，打开Object View模块。

此时，我们决定需要对这个对象进行进一步分析。例如，我们可能想检查对象的关联或检查相关的时间序列数据。为了实现这一点，再次使用导航框架来打开 单通道飞机 Quiver 模块，这次以 Q-AGM Aircraft 对象作为输入。

Carbon 以外的导航

需要注意的是，尽管导航框架最初是为Carbon中的操作用例设计的，但该导航框架也用于对象领域的独立前端应用程序。在Carbon之外的这些应用程序中工作时，导航操作会在新浏览器标签中打开接收应用程序（与在Carbon标签中打开相反），但所有优势仍然适用。因此，最终用户和构建者无需区分在Carbon内部和外部的用例。

模块发现或在Carbon工作区中使用集成模块

为了在Carbon中创建高质量的操作界面，构建者必须能够指定哪些模块可以用于导航，以根据最终用户的操作流程定制工作区。

这些可用的模块称为可发现模块，相应的导航操作可在在...中打开中访问 一些模块中显示的菜单，例如对象资源管理器或对象视图（请注意，只有满足约束条件的操作才会被显示）。其他模块如工作区允许构建者在用户交互后指定要触发的具体导航操作，例如点击按钮。您可以在下面的 与 Carbon 模块导航框架的集成 部分找到有关每个模块类型如何处理导航操作的详细信息和示例。

模块发现行为 - 也就是在 在...中打开 菜单中出现的选项 - 取决于用户是在 Carbon 中工作还是在 Carbon 工作区之外。这在下面的文档中进行了描述 .

在 Carbon 配置编辑器中配置可发现的模块

  1. 在 Carbon 编辑器（通过 /workspace/carbon/edit 访问）中，导航到 常规 选项卡，并将模块添加到 可发现的模块 列表中。
  2. 选择 添加一个元素按钮，以打开提示输入模块类型的弹出窗口。
  3. 在模块类型下拉菜单中选择所需的模块类型，然后选择打开指南针对话框以选择您希望使其可发现的具体模块。

Carbon和Carbon外的模块发现行为

模块发现行为 - 即，在打开于菜单中出现的选项 - 会根据用户是在Carbon中工作还是在Carbon界面之外而有所不同。

当在Carbon工作区中工作时，打开于菜单将仅显示当前所选工作区配置的可发现模块。这样，每个工作区都可以作为一个空间，具有针对明确的业务流程集合的定制导航操作。

当在Carbon之外工作时， “在...中打开”按钮将显示用户有权访问的所有推广工作区中的所有可发现模块。这使用户在不使用特定Carbon工作区时，可以充分利用Foundry的功能。所有可发现模块的集合是通过以下方式由UNION操作确定的：

  1. 首先，考虑用户有权访问的所有组织，包括用户的主要组织以及用户有访客权限的所有组织。
  2. 对于每个组织，考虑该组织的所有< a href="/docs/foundry/carbon/workspaces-overview/#promoted-workspaces">推广工作区，并合并这些工作区的所有可发现模块。
  3. 将模块过滤为可以应用于当前显示“在...中打开”菜单的模块的输出的模块。

以下示例说明了在Carbon内部和外部进行模块发现之间的区别。

  • Zayna的主要组织是Primary Org，她还有Guest Org的访客权限。

  • Zayna 是来自 Primary Org 的两个不同 Carbon 工作区的成员：

    • 索赔工作区，其中配置了两个可发现模块（索赔警报应用程序和索赔调查员应用程序）。
    • 精算工作区，其中配置了一个可发现模块（索赔队列应用程序）。

  • Zayna 也是来自 Guest Org 的一个 Carbon 工作区的成员，即航班工作区。

    • 航班工作区配置了一个可发现模块（航班跟踪器）。

由于这种配置，Zayna 在在...中打开按钮中将根据她所在的位置看到不同的模块集合：

  • 在索赔工作区，在...中打开按钮将显示索赔警报应用程序和索赔调查员应用程序。
  • 在精算师工作区中，在...中打开按钮将显示索赔群体应用程序。
  • 在航班工作区中，在...中打开按钮将显示航班跟踪器。
  • 在Carbon之外，在...中打开按钮将显示四个模块：索赔警报应用程序、索赔调查员应用程序、索赔群体应用程序和航班跟踪器。

当Carbon未被用户直接访问时设置导航

在某些情况下，可能没有操作用户或使用场景用于Carbon工作区。但是，您可能仍希望配置在...中打开操作以在独立对象应用程序之间导航，如前所述在 模块发现或在Carbon工作区中使用集成模块。

与Carbon模块的导航框架集成

本节包含有关每种类型的Carbon模块如何与导航框架集成的信息。您将找到有关在每种模块类型上下文中被视为导航操作的输出的内容，以及Carbon参数如何成为导航操作的输入。

对于没有关联的Foundry资源（对象视图、对象浏览器、搜索）的模块，输入和输出是预定义的，不能更改或配置。例如，对象视图将始终接受任何单个对象作为其输入，而对象浏览器的输出设置为用户探索的当前结果。我们称这样的模块为内置；内置模块不需要通过显式配置使其可发现。

由构建者创建的模块（例如Workshop、Slate、Quiver或Map）有更复杂且更强大的方式来指定输入和输出，通常通过在该模块上下文中定义的变量或参数。我们称这样的模块为动态，它们必须通过显式配置使其可发现。

对象视图（内置）

对象视图：通过模块发现输入

对象视图模块将其输入支持为任何对象类型的单个对象。在大多数与单个选定对象相关的上下文或操作菜单中，默认存在一个导航操作以打开对象视图模块。无法禁用此导航操作，也无法在工作区配置编辑器中将对象视图模块配置为可发现。

对象视图：通过UI元素输入

在动态模块中，可以创建触发导航操作进入对象视图的UI元素：

  • 在Workshop中，您可以定义类型为打开对象视图的事件。
  • 在 Slate 中，通过常规 HTML 链接导航到独立的 Object View 应用程序时，Carbon 会拦截该操作，并改用包含链接中对象 RID 的新 Object View 模块打开。有关更多详细信息，请参阅 Slate 模块文档。

对象视图：输出

当前在对象视图中显示的对象是模块的 输出。

对象浏览器

对象浏览器：通过模块发现输入

对象资源管理器模块支持作为其输入的单个对象集（版本化或非版本化）。在大多数与单个选定对象集相关的上下文或操作菜单中，或者存在指向对象集的链接时，都有一个导航操作用于打开对象资源管理器模块（有关对象属性作为指向对象集的链接的示例，请参见下图）。无法禁用此操作，也不能在工作区配置编辑器中将对象资源管理器模块配置为可发现的。

也可以通过多种方式从搜索模块导航到对象资源管理器模块，例如：

  • 打开已保存的探索,
  • 打开已保存的对象列表,
  • 对对象类型启动探索，并且
  • 打开到模块的快捷方式。

对象资源管理器：通过UI元素输入

在 动态 模块中，可以创建UI元素，以触发导航操作进入对象探索：

  • 在Workshop中，您可以定义类型为 打开对象资源管理器 的事件。
  • 在Slate中，通过常规HTML链接导航到独立的对象资源管理器应用程序时，Carbon会拦截该导航，并改为打开一个新的对象资源管理器模块，其中包含链接中提供的对象集RID。有关更多详细信息，请参阅 Slate模块文档。

对象资源管理器：输出

对象资源管理器中当前选定的对象集是模块的输出。

搜索（内置）

搜索模块是打开各种本体论（Ontology）工件的入口，提供通过关键字过滤对象类型及其属性，并浏览资源列表的功能，例如探索、列表和模块。

搜索：通过模块发现输入

搜索模块只能通过工作区中的模块快捷方式直接访问，或者通过Carbon首页模块的搜索栏（见下文）。

搜索：通过用户界面元素输入

Carbon首页模块可以配置为显示搜索栏。在搜索栏中输入的查询将成为用于打开搜索模块的参数值。

搜索：输出

当用户在搜索模块中对某个资源进行操作时，Carbon 中会打开一个对应模块的新标签页。除了模块工件（此时仅打开所选模块），对象浏览器将接收到输入的对象集：

  • 对于对象类型，将基于该类型打开一个新的探索。
  • 对于探索，将基于支持该探索的版本化对象集打开一个新的探索。
  • 对于列表，将基于列表中包含的对象打开一个新的探索。
  • 对于比较，将基于比较视图以比较模式打开一个新的探索。

工作坊

工作区：通过模块发现输入

任何具有至少一个对象集合类型模块接口变量的工作区模块都可以配置为可发现，如 在“配置模块发现，工作区模块”部分中所述。

工作区：通过UI元素输入

在动态模块中可以创建触发导航操作进入工作区模块的UI元素：

  • 在工作区中，您可以定义一个< strong >打开工作区类型的事件。
  • 在Slate中，通过常规HTML链接导航到独立的工作区应用程序时，Carbon会拦截该请求，并打开一个新的工作区模块，其中包含链接中的对象集合RID。有关更多详细信息，请参阅 Slate文档。

工作区：模块接口

工作区变量可以通过在变量的设置面板中设置外部ID来添加到模块接口，以实现对工作区模块的参数化。模块接口变量的值可以通过URL传递给独立的工作区应用程序，或通过参数传递给Carbon工作区模块。

通过URL

要从Carbon模块内部通过URL将参数传递到工作区应用程序，必须在URL中的参数前添加param.variable.。

示例： workspace/carbon/ri.carbon.main.workspace.../ri.workshop.main.module...?param.variable.flight_id=1000,1001&param.variable.route=100,200

通过参数

当作为Carbon参数使用时（例如在顶部栏模块快捷方式中），参数的名称必须是变量外部ID前缀加上字符串variable.（包括点号）。

限制

目前，Carbon导航不支持以下类型的模块接口变量：

  • 值为空对象集的对象集变量
  • 值为NaN（非数字）的数字变量
  • 时间序列集变量
  • 地理形状和地理点变量
  • 结构体变量
  • 场景变量

工作区：事件

在Workshop模块中，可能通过上述段落中所述的变量概念同时对多个对象集进行操作。可以使用< strong >打开应用程序< /strong >类型的< a href="/docs/foundry/workshop/concepts-events/">Workshop事件< /a >，将任何此类对象集作为输入传递到许多其他模块中。

目前支持以下类型的模块：

  • Workshop
  • 对象视图
  • 对象探索器
  • 便签本（只读）
  • Vertex探索

一旦选择了要打开的应用程序或模块类型（并且对于< em >动态< /em >模块的情况，通过资源选择器选择了特定模块），就可以选择一个特定变量作为输入。

当在Workshop模块中触发配置的事件时，将使用所选变量的当前值构造一个导航操作，并打开一个新的Carbon标签页。

Slate：输出

Slate模块以特定方式与导航框架集成。每当在Slate文档中发生导航（例如，点击链接），Carbon会拦截此类导航并尝试进行解释。如果导航的目标地址可以被识别为独立的前端应用程序，并且该应用程序具有相应的Carbon模块，则会在新的Carbon选项卡中打开该模块。例如：

在 Slate 模块中的嵌套 iframe 中进行导航

Slate 具备在文档中嵌入 iframe 的能力。通过此类 iframe 内部的链接进行的任何导航将属于以下类别之一：

  • 导航发生在 iframe 内部： iframe 将更改其自身的源 URL。例如，iframe 可以被视为一个子视图，它会保持 iframe 内的内容变化。
  • 导航发生在 iframe 外部： 当点击 iframe 中的链接时，将打开一个新的 Carbon 标签页。例如，当 iframe 中的链接指向一个对象时，点击该链接将在新标签页中打开该对象。

在后一种情况下，对于在描述的机制中提到的... output 部分用于 Slate 模块 生效，嵌套的 iframe 中的链接必须将最外层的 Slate iframe（Carbon 显示顶级 Slate 模块的那个）指定为其 target。Carbon 会将该 iframe 的名称设置为 carbon-navigation-target。下面有一个链接的 HTML 代码片段示例：

<a
  href="/workspace/hubble/objects/ri.phonograph2-objects.main.object.4202d614-bb6e-471d-80d2-2cf9c735caf3"
  target="carbon-navigation-target"
>
  Link to an object with target="carbon-navigation-target"
</a>

这种方法在任何深度的 iframe 嵌套中都能正常工作。例如，你可以在第二个 iframe 内部嵌入第三个 iframe，而该第二个 iframe 又嵌入到 Carbon 创建的顶级 Slate 模块 iframe 中。

Quiver

在 Quiver 艺术品中，只有 Quiver 仪表板可以作为 Carbon 模块使用。请参阅 在 Quiver 中创建和嵌入仪表板的文档，以了解仪表板与分析之间的区别，以及如何创建仪表板。

一个 Quiver 仪表板可以将对象或对象集作为输入。

在两种情况下，对象或对象集必须是特定类型。这是为了确保每当一个参数（无论是对象还是对象集）被管道传输到仪表板时，所有定义的转换都能正常工作（例如，相同的对象属性和本体论关联集将始终被考虑）。

因此，仪表板所基于的对象类型会转化为对可能输入的约束。有关约束如何影响导向Quiver模块的导航操作的可用性，请参见模块发现或在Carbon工作区中使用集成模块。

例如，假设Single Aisle Aircrafts仪表板的输入是一个具有Aircraft对象类型的对象集。

对于导航操作推断出的约束是，输入的对象集只能包含Aircraft对象。在Carbon工作区中，Single Aisle Aircrafts模块被添加到配置中的可发现模块部分。
