时间序列属性用例

对象上的时间序列属性能够支持强大的分析工作流。本文档将详细介绍在 Pipeline Builder 中编写流水线的各个步骤、在 Ontology Manager 中设置对象，以及创建 Quiver 仪表板和 Workshop 模块的完整过程。我们将以航空本体为例，展示 Foundry 中时间序列属性的强大功能。

该航空本体包含 Flight、Carrier、Route、Airport 和 Flight Sensor 等示例对象类型。Flight 对象通过这些对象上的 flight_id 外键与 Aircraft、Flight Sensor、Route、Airport 和 Carrier 对象建立关联。

该航空本体源自一个使用开源数据的参考本体，您的部署环境中可能无法直接获取这些数据。无论您的部署环境中能否获取该数据，基于此示例本体构建的案例都将作为参考，帮助您利用时间序列属性创建自己的流水线、对象类型和 Workshop 模块。

按照本指南操作，您将生成一个 Workshop 模块，通过该模块可以调用并查看基于飞行数据的 Carrier、Route 和 Airport 对象上的时间序列属性。

以下指南将引导您完成创建和支持该 Workshop 模块的各个步骤：

  1. 使用 Pipeline Builder 从“Flight”对象集生成时间序列属性
  2. 将时间序列属性添加到航空对象
  3. 在 Workshop 和 Quiver 中使用航空对象的时间序列属性
