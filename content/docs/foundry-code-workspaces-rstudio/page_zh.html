RStudio® Workbench

要使用 RStudio® Workbench，您的平台管理员必须首先在控制面板中 配置其操作许可证。如果您是平台管理员且无法在控制面板中配置许可证，请联系 Palantir 支持。

代码工作区（Code Workspaces）使您可以在 Foundry 中使用 RStudio® Workbench ↗。代码工作区中的 RStudio® Workbench 支持：

  • 使用 Palantir R SDK ↗ 读取和写入表格数据集（dataset（数据集））。
  • 从非结构化数据集（dataset（数据集））下载或上传文件。
  • 从 CRAN、Posit™ 库管理器（library manager（库管理器））和 Bioconductor 导入 R 库（library（库））。
  • 发布 Shiny® 应用程序（application（应用程序））。
  • 发布 R 转换（transform（转换））。

代码工作区（Code Workspaces）目前支持 Shiny® ↗ 用于 R 应用程序（application（应用程序））。用户可以直接在代码工作区（Code Workspaces）中使用 Foundry 的 版本控制（version control（版本控制））、分支（branching（分支）） 和 数据保护与治理（data protection and governance（数据保护与治理）） 创建应用程序（application（应用程序））。 数据治理功能已内置。

代码工作区应用程序支持分支。如果您 创建一个新的工作区分支，发布新的应用程序或同步更改将在该分支上发布应用程序的新版本。这允许您在将应用程序暴露给用户之前进行预览。要发布到主分支，只需将您的分支合并到主分支中。

Shiny®

通过代码工作区，您可以从 RStudio® 用户界面直接创建并预览 Shiny® 应用程序。

默认情况下，RStudio® 在主目录（/home/user）中生成 Shiny® 应用程序，但 Foundry 要求应用程序位于 git 仓库 /home/user/repo 中，以便在会话之间持久化并发布。

在代码工作区中，选择 发布应用程序 ，选择您在文件和项目中的位置，以用于您的新应用程序，并输入相对于代码仓库根目录app.R的新Shiny® /home/user/repo文件应自动创建的目录名称。此字段可以留空，以便在代码仓库的根目录下创建应用程序。默认情况下，应用程序设置将与RStudio®工作区设置匹配，配置高级设置是可选的。

已发布的Shiny应用程序会受到30秒的超时限制，这意味着Shiny服务器必须在执行您的Shiny应用程序文件后的30秒内启动。否则，您的Shiny应用程序将无法启动。

选择发布并同步以注册您的新Shiny®应用程序并将代码同步到支撑代码仓库。CI检查和发布完成后，您可以选择应用程序面板中的链接以查看已发布的应用程序。

要针对Shiny®应用程序进行本地开发：

  1. 在RStudio®中打开app.R。
  2. 选择运行应用 . 一个应用程序的预览将出现在一个新窗口中。

配置 Shiny® 应用程序以自动重新连接

默认情况下，如果 WebSocket 连接关闭，例如网络中断超过 15 秒，Shiny 会将其置灰。为了确保在恢复网络连接时您的应用程序可以恢复，我们建议更新 server 函数以设置 session$allowReconnect("force")：

Copied!
1server <- function(input, output, session) {
2  # Handle input changes
3  session$allowReconnect("force")
4}

这将导致前端在网络可用时触发新的 WebSocket，并发送 UI 中最后设置的输入信息。

R 转换

R 转换目前不支持对外部服务进行 API 调用，即使在 RStudio 代码工作区中已添加了网络策略。

在代码工作区中编写的 R 脚本可以注册为代码工作区中输出数据集的转换。

请按照以下步骤为输出数据集注册一个转换：

  1. 在数据面板中选择数据集旁边的蓝色构建图标。
  1. 按照界面提示配置您的转换。代码工作区会推断出使用的 R 脚本以及添加的输入以生成输出数据集。
  1. 选择同步并运行以保存转换配置，并触发对输出数据集的转换和 JobSpecs 的发布进行 CI 检查。

一旦CI检查完成，您的转换就准备好进行构建了。您可以使用其他Foundry 数据集成工具来管理您的转换以及它所属的 数据管道。

该转换不会自动使用代码工作区的环境。您应该在脚本文件的顶部安装运行时所需的所有库。安装库的代码片段可以在 库 面板中找到。 例如，要在您的转换中使用 dplyr 和 tidyr，请将此代码片段添加到文件的顶部：

Copied!
1renv::install(c("dplyr", "tidyr"))

Markdown 报告

在RStudio®工作区中编写的R Markdown或Quarto文件可以作为报告与其他Palantir平台用户共享。报告是一个数据集，它使用包含通过将Markdown文件用Quarto ↗渲染生成的HTML的转换来构建。

由于报告是使用转换构建的，因此可以配置它们根据Markdown文件的最新更改自动更新，使用计划。这确保了您的分析和可视化内容是最新的，并且使用可重复、可自定义的工作流程，与Palantir平台的其余部分集成。

从RStudio®工作区发布报告

可以按照以下步骤从RStudio®工作区发布Markdown报告：

  1. 创建你希望用来生成报告的Markdown文件（.Rmd 或 .qmd）。此Markdown文件可以从已添加到你的代码工作区的Foundry数据集读取，方法是使用 数据侧边栏。
  2. 打开 应用程序 侧边栏并选择 报告 选项卡。
  3. 选择 添加 以打开报告配置对话框。
  4. 提供一个名称和位置；这将决定包含该报告的数据集将被保存的位置。
  5. 选择用于构建报告的Markdown文件。如果你的Markdown文件从Foundry数据集读取，请将这些数据集作为报告输入添加。选择 自动配置输入和输出 以自动解析你的Markdown文件，并用检测到的数据集填充表单。
  6. 选择 创建。这将在指定位置创建一个数据集。然后将运行构建报告的转换操作。请注意，这可能需要几分钟才能完成。
  7. 报告构建完成后，在 应用程序 侧边栏中选择报告名称，即可在新标签页中打开报告。此链接可以与其它用户共享。

要编辑或删除现有报告，请点击报告卡片右侧的...。这将打开一个菜单，提供编辑或删除报告的选项，以及复制报告链接的选项。

查看以前的报告版本

报告支持完整的版本历史记录。要查看报告的先前版本，请在应用程序侧边栏中展开报告卡片，或者在新标签页中查看报告时使用页面标题中的版本选择器。

选择报告版本将更新用于共享报告的链接，使您可以共享特定的历史版本报告，即使基础Markdown被修改，该版本也不会更新。

报告的最新版本将默认被选中，因此您的报告读者会自动收到最新的更改。

在RStudio®中管理的conda环境

通过使用管理环境，可以在RStudio®工作区、Shiny应用程序、转换和模型输出中一致地恢复RStudio®的conda环境。管理环境在代码工作区工作流中保持一致，无需每次重新启动工作区、启动Shiny应用程序或运行RStudio®转换时都重新运行相同的renv::install命令。

此功能现在通过Maestro（一个Foundry库管理工具）包括CRAN库管理，该工具可无缝集成到conda和CRAN中，以高效处理库安装并维护管理环境。新安装的CRAN库将被记录在您的环境文件中，并在RStudio®工作区、Shiny应用程序、转换和模型输出中一致地恢复，从而提高环境的可重复性。请参阅下面的开始使用部分以启用此功能。

要将库安装命令转换为Maestro，请参考以下说明，适用于 conda 或 CRAN。通过管理环境，您工作环境中的已安装库将通过以下文件进行跟踪，从而允许在不同工作流程中恢复到原始状态：

  1. /home/user/repo/.envs/maestro/meta.yaml：称为环境 清单，它包含您环境中所需的 请求 库和版本。
  2. /home/user/repo/.envs/maestro/hawk.lock：称为环境 锁定文件，它包含您环境中解析的 解决 库和版本，这些版本是从清单中指定的约束条件中求解出来的。hawk.lock 文件取代了 conda.lock，后者已被弃用。

当您同步 conda 环境中的任何新代码时，上述文件也会被同步。当您开始下一个会话时，恢复的代码将包括 maestro 文件，并且您的环境将被重新创建。

我们不建议修改 hawk.lock 锁定文件。相反，请依靠安装命令来代表您更新文件。除非存在 git 冲突，否则也不需要修改 meta.yaml 文件；当库被交互式安装时，它会自动更新。如果您确实修改了这些文件，可能会恢复过时的 conda 环境或导致环境构建失败。

您的托管环境安装在您的 /home/user/envs/default 文件夹中。此环境将在以下任何事件发生前被恢复：

  • RStudio® 工作区启动时
  • 从您的工作区生成的 Shiny 应用程序启动时
  • 从您的工作区生成的 RStudio® 转换运行时
  • 渲染 Markdown 报告 时

开始使用

1. 启用托管环境

要在 RStudio® 工作区中启用托管环境，请从左侧侧边栏导航到您的工作区设置，然后在 高级功能 下切换 托管 Conda 环境。它可能默认已开启。

然后，重新启动工作区：

要在 RStudio® 工作区中通过 Maestro 启用 CRAN 库管理，请使用左侧侧边栏导航到您的工作区设置，然后在 实验性功能 下切换 托管 CRAN 环境。此选项可能默认已启用。

然后，重新启动工作区。

2. 更新您的托管环境

按照以下步骤更新您的托管环境并管理新库：

使用 Maestro 管理 Conda 库（推荐）

Install R libraries with conda instead of CRAN to significantly reduce workspace startup time.

  • Install using the Libraries panel:

    • For standard libraries: Search for the library name with an r- prefix (for example, r-ggplot2 not ggplot2) and install.
    • For Bioconductor libraries: Search for the lowercase library name with a bioconductor- prefix (for example, bioconductor-genomeinfodb) and install.

  • Install using the terminal: To install the latest version of a library, run the following command in the terminal:

    Copied!
    1maestro env conda install <library_name>

    To install a specific version, run the following command:

    Copied!
    1maestro env conda install <library_name>==<library_version>

  • Uninstall using the terminal: To uninstall a library, run the following command in the terminal:

    Copied!
    1maestro env uninstall --conda-dependencies <library_names>
Manage CRAN libraries with Maestro

  • Install using the Libraries panel:

    • For standard libraries: Search for the library name (for example, ggplot2) and install.
    • For Bioconductor libraries: Search for the library name with initial capitalization format (for example, GenomeInfoDb) and install.

  • Install using the terminal: To install the latest version of a library, run the following command in the terminal:

    Copied!
    1maestro env cran install <library_name>

    To install a specific version, run the following command:

    Copied!
    1maestro env cran install <library_name>@<library_version>

  • Uninstall using the terminal: To uninstall a library, run the following command in the terminal:

    Copied!
    1maestro env uninstall --cran-dependencies <library_names>

When installing a library through the Libraries tab, the install command is executed directly in the RStudio® console rather than executed within your script or application code. Every time an install command is run, the meta.yaml manifest will be updated with your requested library, and the hawk.lock file will reflect the libraries installed in the current environment.

3. View currently installed libraries in your managed environment

Your installed conda libraries are visible in the Libraries tab in the left sidebar:

You can also choose to view the meta.yaml and hawk.lock files in the RStudio® interface.

4. Synchronize your managed environment with changes to the hawk.lock file

After checking out a new branch or pulling new git changes, the contents of your hawk.lock file may change.

To synchronize your managed environment, execute the following command in your workspace terminal:

Copied!
1maestro env install

maestro is the command line interface that updates managed environments. The above command re-resolves the lock file from the manifest and installs the environment into the /home/user/envs/default folder.

5. Resolve git conflicts to your environment files

To resolve git conflicts in your managed environment, first delete the hawk.lock file. Navigate to the meta.yaml file to resolve any conflicts, then merge your branch.

After merging, run the maestro env install command in your terminal to regenerate the hawk.lock file and reinstall your managed environment.

6. Upgrade your managed environment

To upgrade your managed environment to a more recent version, first remove the hawk.lock file. Then, delete your /home/user/envs/default folder (rm -rf /home/user/envs/default). Finally, run the maestro env install command in your terminal to regenerate your hawk.lock file. Your environment will be re-installed with upgraded library versions.
