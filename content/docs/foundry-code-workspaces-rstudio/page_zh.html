RStudio® Workbench

要使用 RStudio® Workbench，您的平台管理员必须先在控制面板中配置其运行许可证。如果您是平台管理员且无法在控制面板中配置许可证，请联系 Palantir 支持团队。

代码工作区使您能够在 Foundry 中使用 RStudio® Workbench ↗。代码工作区中的 RStudio® Workbench 支持以下功能：

  • 使用 Palantir R SDK ↗ 读取和写入表格数据集。
  • 从非结构化数据集中下载或上传文件。
  • 从 CRAN、Posit™ 库管理器和 Bioconductor 导入 R 库。
  • 发布 Shiny® 应用程序。
  • 发布 R 转换。

代码工作区目前针对 R 应用程序支持 Shiny® ↗。用户可以直接在代码工作区中创建应用程序，并内置 Foundry 的版本控制、分支管理和数据治理功能。

代码工作区应用程序支持分支管理。如果您创建了新的工作区分支，发布新应用程序或同步更改将会在该分支上发布应用程序的新版本。这允许您在将应用程序暴露给用户之前进行预览。要在主分支上发布，只需将您的分支合并到主分支即可。

Shiny®

通过代码工作区，您可以创建 Shiny® 应用程序并直接从 RStudio® 用户界面中进行预览。

默认情况下，RStudio® 会在主文件夹（/home/user）中生成 Shiny® 应用程序，但 Foundry 要求应用程序必须位于 git 仓库 /home/user/repo 中，以便在不同会话之间持久化并发布。

在代码工作区中，选择发布应用程序，在文件和项目中为新应用程序选择一个位置，并输入应该自动创建新 Shiny® app.R 文件的目录名称（相对于仓库根目录 /home/user/repo）。此字段可以留空，以便在仓库根目录下创建应用程序。默认情况下，应用程序设置将与 RStudio® 工作区设置相匹配，配置高级设置是可选的。

已发布的 Shiny 应用程序有 30 秒的超时限制，这意味着 Shiny 服务器必须在执行 Shiny 应用程序文件后 30 秒内启动。否则，您的 Shiny 应用程序将无法启动。

选择发布并同步以注册您的新 Shiny® 应用程序，并将代码同步到底层代码仓库。完成 CI 检查和发布后，您可以在“应用程序”面板中选择链接来查看已发布的应用程序。

要在本地针对 Shiny® 应用程序进行开发：

  1. 在 RStudio® 中打开 app.R。
  2. 选择运行应用程序。应用程序的预览将出现在新窗口中。

## R 转换

R 转换目前不支持对外部服务发起 API 调用，即使已在 RStudio 代码工作空间的网络策略中进行了相应配置。

在代码工作空间中编写的 R 脚本，可以注册为代码工作空间中输出数据集的转换操作。

请按照以下步骤为输出数据集注册转换：

- 在**数据**面板中，选择数据集旁边的蓝色**构建**图标。



- 按照界面提示配置转换。代码工作空间会自动推断所使用的 R 脚本以及为生成输出数据集而添加的输入项。



- 选择**同步并运行**以保存转换配置，并触发 CI 检查，从而在输出数据集上发布该转换和 JobSpecs。

CI 检查完成后，您的转换即可进行构建。随后，您可以利用其他 Foundry 数据集成工具来管理该转换及其所属的数据管道。


转换不会自动使用代码工作区的环境。您应该在脚本文件的顶部安装运行时所需的所有库。安装库的代码片段可以在库面板中找到。
例如，要在转换中使用 dplyr 和 tidyr，请将此代码片段添加到文件顶部：
Copied!
1renv::install(c("dplyr", "tidyr"))

Markdown 报告


在 RStudio® 工作区中编写的 R Markdown 或 Quarto 文件，可以作为报告与 Palantir 平台的其他用户共享。报告是基于 Markdown 文件构建的数据集，该构建过程使用了通过 Quarto ↗ 渲染 Markdown 文件生成的 HTML 转换。


报告通过转换构建，因此可以通过调度配置为根据 Markdown 文件的最新更改自动更新。这确保您的分析和可视化保持最新状态，并通过可重现、可定制的工作流与 Palantir 平台的其余部分集成。




从 RStudio® 工作区发布报告


可以通过以下步骤从 RStudio® 工作区发布 Markdown 报告：


    

  1. 创建您希望用于生成报告的 Markdown 文件（.Rmd 或 .qmd）。此 Markdown 文件可以读取已通过数据侧边栏添加到您工作区的 Foundry 数据集。
    2. 

  2. 打开应用程序侧边栏并选择报告选项卡。
    3. 

  3. 选择添加以打开报告配置对话框。
    4. 

  4. 提供名称和位置；这将决定包含报告的数据集的保存位置。
    5. 

  5. 选择要用于构建报告的 Markdown 文件。如果您的 Markdown 读取自 Foundry 数据集，请将这些数据集添加为报告输入。选择自动配置输入和输出，系统将自动解析您的 Markdown 文件并使用检测到的任何数据集填充表单。
    6. 

  6. 选择创建。这将在指定位置创建一个数据集。随后，构建报告的转换将运行。请注意，此过程可能需要几分钟才能完成。
    7. 

  7. 报告构建完成后，在应用程序侧边栏中选择报告名称，即可在新选项卡中打开报告。此链接可以与其他用户分享。
    8. 



要编辑或删除现有报告，请选择报告卡片右侧的 ...。这将打开一个菜单，其中包含编辑或删除报告的选项，以及复制报告链接的选项。


查看以前的报告版本


报告支持完整的版本历史记录。要查看报告的以前版本，请在应用程序侧边栏中展开报告卡片，或者在新选项卡中查看报告时使用页面头部的版本选择器。




选择某个报告版本会更新用于分享该报告的链接，方便你分享特定的历史版本；即使底层的 Markdown 内容后续被修改，该历史版本也不会随之更新。




系统会默认选中报告的最新版本，因此报告读者会自动获取最新的变更内容。


RStudio® 中的托管 conda 环境


借助托管环境，RStudio® conda 环境可以在 RStudio® 工作区、Shiny 应用、数据转换和模型输出之间保持一致地还原。托管环境在代码工作区（Code Workspace）的工作流中保持一致，省去了每次重启工作区、启动 Shiny 应用或运行 RStudio® 数据转换时重新执行 renv::install 命令的麻烦。


该功能现已支持通过 Maestro 进行 CRAN 库管理。Maestro 是一款 Foundry 库管理工具，可与 conda 和 CRAN 无缝集成，能够高效处理库的安装并维护托管环境。新安装的 CRAN 库将被记录在环境文件中，并在 RStudio® 工作区、Shiny 应用、数据转换和模型输出之间保持一致地还原，从而提升环境的可复现性。请参阅下方的入门部分以启用该功能。


若要将库安装命令转换为 Maestro，请参考下方关于 conda 或 CRAN 的说明。在托管环境中，工作环境里已安装的库会通过以下文件进行跟踪，使得环境能够在不同工作流之间还原至初始状态：


    

  1. /home/user/repo/.envs/maestro/meta.yaml：即环境清单，其中包含你的环境所请求的库及版本集合。
    2. 

  2. /home/user/repo/.envs/maestro/hawk.lock：即环境锁定文件，其中包含根据清单中指定的约束条件求解出的已解析库及版本集合。hawk.lock 文件取代了已弃用的 conda.lock。
    3. 



当你在 conda 环境中同步任何新代码时，上述文件也会同步更新。当你启动下一次会话时，还原的代码将包含 maestro 文件，且你的环境将被重新构建。


不建议修改 hawk.lock 锁定文件，请依靠安装命令来代为更新这些文件。除非发生 git 冲突，否则你也不需要修改 meta.yaml 文件；在交互式安装库时它会自动更新。如果你确实修改了这些文件，可能会导致还原出的 conda 环境过时，或导致环境构建失败。


你的托管环境安装在 /home/user/envs/default 文件夹中。该环境会在以下任一事件发生前进行还原：


    

  • RStudio® 工作区启动
    • 

  • 由工作区生成的 Shiny 应用启动
    • 

  • 运行由工作区生成的 RStudio® 数据转换
    • 

  • Markdown 报告渲染
    •

接着，重启工作区：

通过库选项卡安装库时，安装命令会直接在 RStudio® 控制台中执行，而不是在您的脚本或应用程序代码中执行。每次运行安装命令时，meta.yaml 清单文件都会更新您请求的库，而 hawk.lock 文件则会反映当前环境中已安装的库。


3. 查看受管环境中当前已安装的库


已安装的 conda 库显示在左侧边栏的库选项卡中：



您也可以选择在 RStudio® 界面中查看 meta.yaml 和 hawk.lock 文件。


4. 根据 hawk.lock 文件的变更同步受管环境


在检出新分支或拉取新的 git 变更后，hawk.lock 文件的内容可能会发生变化。


要同步受管环境，请在工作区终端中执行以下命令：




已复制！
1maestro env install


maestro 是用于更新受管环境的命令行界面。上述命令会根据清单文件重新解析锁定文件，并将该环境安装到 /home/user/envs/default 文件夹中。


5. 解决环境文件中的 Git 冲突


要解决受管环境中的 Git 冲突，请先删除 hawk.lock 文件。转至 meta.yaml 文件解决所有冲突，然后合并您的分支。


合并后，在终端中运行 maestro env install 命令，以重新生成 hawk.lock 文件并重新安装受管环境。


6. 升级受管环境


要将受管环境升级到更高版本，请先移除 hawk.lock 文件。接着，删除 /home/user/envs/default 文件夹（rm -rf /home/user/envs/default）。最后，在终端中运行 maestro env install 命令以重新生成 hawk.lock 文件。系统将使用升级后的库版本重新安装环境。
