防止实体的Product Release升级

在某些情况下，您可能希望“冻结”一个或多个实体的升级，或者更好地控制它升级到哪个Release。例如，当测试一个Release Candidate时，您不希望Apollo自动将实体从Release Candidate升级。

要防止自动升级：

  1. 创建一个新的自定义Release Channel，该Channel不属于Product的推广管道。
  2. 配置实体以遵循新的Release Channel，这将阻止进一步的升级，直到新的Release被手动添加到新的Release Channel中。

您也可以通过< a href="/docs/apollo/managing-entities/user-issued-commands/">发出命令来安装特定的Release，并创建一个< a href="/docs/apollo/managing-environments/suppression-windows-and-cancelling-plans/">抑制窗口，以防止Apollo更改实体的运行版本，从而防止不必要的升级。然而，这种方法有三个缺点：

  1. 执行命令时，Apollo会忽略< a href="/docs/apollo/core/plans-and-constraints/#product-dependencies-constraint">产品依赖性。因此，只有在您确定不会导致任何问题时才应使用此方法。
  2. 抑制窗口可能会过期或被意外删除，从而允许Apollo升级实体。
  3. 抑制窗口还将阻止对实体的其他更改。例如，新的配置覆盖不会被应用。

创建自定义发布渠道

您可以通过导航到设置与配置页面的发布渠道选项卡并选择创建发布渠道来创建一个新的发布渠道，例如FREEZE_VERSIONS_FOR_DEMO。您也可以使用现有的自定义发布渠道。

配置实体以遵循新的发布渠道

对于您希望防止发布升级的每个实体，您可以编辑实体设置以跟踪您创建的专用发布渠道。导航到环境页面的设置选项卡，然后选择手动编辑设置。

选择 编辑 以开始编辑 Environment 设置。然后，对于每个相关的 Entity，将 release-channel 字段的值更改为您创建的 Release Channel。在此示例中，您需要将 release-channel: FREEZE_VERSIONS_FOR_DEMO 添加到 Entity 的配置覆盖中。

完成之后，选择 审查 以查看您的更改。然后选择 提交 以提交一个 更改请求 来更新 Environment 设置。

当您准备好让 Entity 恢复自动升级时，可以 编辑 Environment 设置 以将 Entity 订阅的 Release Channel 更改回来。

在订阅自定义 Release Channel 期间，Apollo 将继续为该 Entity 发布 Plans，并将其升级到添加到自定义 Release Channel 的其他 Releases。这可以通过 手动 或通过自动化推广来完成。Apollo 也将按照常规遵守召回操作。

如果实体正在运行一个已召回的版本，并且在订阅的版本通道中没有满足撤回策略的版本，则该实体将继续停留在已召回的版本上，直到将合适的版本添加到版本通道中。
