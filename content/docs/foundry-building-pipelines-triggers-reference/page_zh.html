触发类型参考

Foundry 支持 AND 和 OR 触发器的任意嵌套，从而可以创建特定的自定义触发器。

时间触发器

时间触发器是在指定时间点满足的触发器。

时间触发器只在指定的时间点满足。一旦指定的时间点过去，时间触发器就不再满足。

Cron 表达式

时间触发器使用 Cron 表达式 ↗ 和时区来定义。

提示

计划编辑器提供了一个易于使用的界面，用于定义简单的定时触发器，而无需编写cron表达式。

更复杂的定时触发器可能需要编写自定义的cron表达式。

计划编辑器使用标准的Unix cron表达式格式，包含五个字段：

+--------- Minute
| +------- Hour
| | +----- Day of Month
| | | +--- Month
| | | | +- Day of Week
| | | | |
* * * * *

Data Health 使用 Quartz cron 格式 ↗（六参数），而不是 Unix cron 格式 ↗（五参数）。Quartz 增加了一个额外的秒字段，并将其添加到标准的 Unix cron 格式前面。

计划编辑器允许每个字段的以下值：

字段允许的值允许的特殊字符注释
分钟0-59* - / ,
小时0-23* - / ,
每月日期 1-31* - / , L
月份1-12 或 JAN-DEC* - / ,
星期几0-6 或 SUN-SAT* - / , L #7 也是星期日

特殊字符的含义如下所述：

  • 星号（*）：指定所有值。

    示例：

    分钟含义
    *每分钟

  • 短横线（-）：指定范围的值。

    示例：

    分钟含义
    10-2010 到 20 分钟之间，包括 10 和 20

  • 斜杠（/）：指定步进范围的值。

    步进范围可以使用一个值或一个范围来定义：

    • 当与值一起使用时，范围是从指定的值到该字段的最大值
    • 当与范围一起使用时，范围是所指定的范围

    示例：

    分钟含义
    25/10从25开始的每10分钟（25、35、45和55）
    25-45/10在25到45之间的每10分钟，包括25和45（25、35和45）

  • 逗号（,）： 指定一个列表，包含值、范围和/或步进范围。

    示例：

    分钟含义
    10,20,3010、20和30分钟
    10,20-3010分钟以及20到30之间的所有分钟，包括20和30
    10,25-45/1010分钟以及25到45之间的每10分钟，包括25和45

  • L（L）： 指定最后 值。

    L 的含义取决于其使用方式：

    • 在“月份中的日期”字段中使用时，它表示该月的最后一天
    • 在“星期中的日期”字段中单独使用时，它表示该周的最后一天（星期六）
    • 在“星期中的日期”字段中与一个值一起使用时，它表示具有指定星期几的该月的最后一天

    示例：

    月份中的日期含义
    L该月的最后一天
    星期中的日期含义
    L该周的最后一天（星期六）
    2L该月的最后一个星期二

  • 哈希（#）: 指定该月的第n个星期几。

    示例：

    每周的星期含义
    2#4每月的第三个星期二

当cron表达式与当前日期和时间匹配时，时间触发器将被满足。

如果“月份中的第几天”和“周中的第几天”字段都不是 *，则只要其中一个与当前日期和时间匹配，触发器就会被满足。

示例：

Cron 表达式 含义
30 9 * * 1周一上午9:30
30 17 * 2 1二月周一下午5:30
0 9-17 10 * *每月10日从上午9点到下午5点每小时一次
0 9-17/2 10 * *每月10日从上午9点到下午5点每两小时一次
0 9,17 10 * *每月10日的上午9点和下午5点
0/5 9-17 15 3 *3月15日早上9:00到下午5:55，每五分钟一次
0/5 9,17 15 3 *3月15日早上9:00到下午9:55以及下午5:00到下午5:55，每五分钟一次
0 9 L * *每月最后一天早上9:00
0 9 L 2 *2月最后一天早上9:00
0 9 * * L星期六早上9:00
0 9 * * 2L每月最后一个星期二早上9:00

时间更改

所有时间触发器都使用指定时区的墙上时间进行评估。每当出现满足 cron 表达式的墙上时间时，时间触发器就会被满足。时间变化的处理方式如下：

  • 如果时间向前移动，则在时间变化期间满足的时间触发器将不会被满足
  • 如果时间向后移动，则在时间变化期间满足的时间触发器将被满足两次

示例：

  • 如果时间从凌晨1:00向前移动到凌晨2:00，那么在凌晨1:30满足的时间触发器将不会被满足，因为凌晨1:30从未发生过
  • 如果时间从凌晨2:00向后移动到凌晨1:00，那么在凌晨1:30满足的时间触发器将被满足两次，因为凌晨1:30发生了两次

事件触发

事件触发器是当指定事件发生时被满足的触发器。

事件触发器在事件发生后仍保持满足状态，直到整个触发器被满足并且计划运行。

事件类型

当前调度编辑器支持以下事件类型：

  • 新逻辑：当计算数据集的逻辑被更新时发生。
  • 数据更新：当提交一个更新数据集的事务时发生。
  • 作业成功：当数据集上的作业完成时发生，无论是否提交了事务。
  • 计划成功运行：当计划构建成功完成时发生。

复合触发器

复合触发器是通过使用 AND 触发器和 OR 触发器组合多个组件触发器创建的。

  • 一个 AND 触发器创建一个其组件触发器的合取的触发器。
  • 一个 OR 触发器创建一个其组件触发器的析取的触发器。

示例：

在以下示例中，T1、T2 是时间触发器，而 E1、E2 是事件触发器。

触发器含义
AND(T1, E1)如果 E1 已经发生，则在 T1 时满足
OR(T1, E1)在 T1 时或当 E1 发生时满足
AND(T1, T2) [1]在同时满足 T1 和 T2 的时间点上满足
OR(T1, T2)在 T1 或 T2 上满足
AND(E1, E2)当 E1 和 E2 都已经发生时满足
OR(E1, E2)当 E1 或 E2 已经发生时满足
AND(T1, OR(E1, E2))如果 E1 或 E2 已经发生，则在 T1 时满足
OR(T1, AND(E1, E2))在 T1 满足，或当 E1 和 E2 均发生时

[1] 创建一个具有多个时间触发器的 AND 触发器时，只有当所有时间触发器同时发生时才会被满足。例如，一个具有每日触发器和每小时触发器的 AND 触发器每天只会被满足一次。相反，应使用更具体的单个时间触发器。在前面的例子中，AND 触发器应替换为仅每日触发器。
