本体论 SQL 函数

测试版

本体论 SQL 函数目前处于 测试版 开发阶段，某些功能在您的环境中可能不可用。在积极开发过程中，功能可能会发生变化。请联系 Palantir 支持以请求访问测试版功能。

本体论 SQL 函数允许您使用 SQL 在 本体论 对象上定义可重用的参数化查询。一旦发布，本体论 SQL 函数可以在 Foundry 中任何接受本体论函数的地方使用，包括 Workshop、Actions、Automate 和本体论 SDK（OSDK）。

本体论 SQL 函数是只读的：您不能在本体论 SQL 函数内部修改本体论。

测试版的本体论 SQL 函数目前不支持高并发工作负载，当资源可用性较低时可能需要排队，或者在负载下可能失败。有关其他查询和数据限制，请参阅 本体论 SQL 的限制和约束。

使用 SQL 定义您的函数

您可以在 SQL Studio 中编写本体论 SQL 函数。您必须处于 对象模式 才能创建函数。

函数支持参数，这些参数通过 变量 在 SQL 语法 中声明。在下面的示例中，animalParam 是一个在调用函数时传递的参数。当在编辑器中测试查询时，将使用默认值 'dog'。

参数只能包含本体论 SQL 语法支持的数据类型的字面值（例如字符串、数字或布尔值）。参数不能包含 SQL 片段或表达式，并且只能用于 SQL 语法期望字面值的位置，比如 WHERE 子句比较的右侧。

3SELECT *
4FROM `ri.ontology.main.object-type...`
5WHERE animaltype = @animalParam;

发布您的函数

当您对函数满意后，保存您的工作表并选择发布按钮。在您可以发布函数之前，您的工作表必须保存到一个项目中。

在发布函数对话框中，为您的函数提供一个名称和API 名称，就像在Foundry中的其他任何函数一样。

使用您的函数

本体论 SQL 函数返回一个结构列表，其字段与所选列匹配，结果中每行对应一个结构。当只选择一列时，函数将返回该列类型的列表。例如，SELECT animal, animalType, age FROM ... 返回 List<Struct<animal: STRING, animalType: STRING, age: INTEGER>>，而 SELECT animal FROM ... 返回 List<STRING>。
