环境配置

环境配置的推荐用途

环境配置（Environment Config）是一个有用的工具，用于定义需要传播到环境中多个实体的环境级别的属性。环境配置属性的良好候选包括那些与环境本身相关的属性（如云供应商或区域）以及需要在广泛范围内的实体中引用的环境属性（如DNS名称）。如果某个属性仅对环境中的少数实体有用，请考虑改用 配置覆盖。

更新环境配置

环境配置可以从 配置 中进行更新 在环境主页上找到的选项卡。它必须包含有效的YAML配置，任何键都不能包含连字符、句点或空格。例如，这是一个有效的配置：

Copied!
1domainName: my-domain-name.com
2numConnectionsAllowed: 5
3enableSchedulingConstraints: true
4availabilityZones:
5    - availability-zone-1
6    - availability-zone-2
7namespaces:
8    defaultNamespace:
9        name: myNamespace

与其他设置一样，对环境配置的修改通过 变更请求 进行，并且受为该环境配置的审批策略的约束。这会创建一个强大的配置更改历史记录，生产环境的清晰审计跟踪，以及验证的机会。一旦对环境配置的更改获得批准，Apollo 将为每个在环境中使用环境配置的实体生成一个计划来更新配置。

使用环境配置

您可以使用 Helm 的内置 Values 对象 ↗ 在您的 Helm 图表模板中引用环境配置：

Copied!
1{{ .Values.apollo.environment.<configKey> }}

您应该将 <configKey> 替换为您的环境配置 YAML 文件中的键。

例如，如果您在环境配置中有一个条目 myKey: myValue，您可以使用以下语法来引用此条目：

Copied!
1{{ .Values.apollo.environment.myKey }}

当Helm渲染模板时，这将被渲染为 myValue。

您还可以在实体的 配置覆盖 中使用 preprocess 指令来引用环境配置：

Copied!
1{{ preprocess .Values.apollo.environment.<configKey> }}

您应该用您的环境配置YAML文件中的键来替换 <configKey>。

例如，如果您在环境配置中有一个条目 myKey: myValue，您可以使用以下语法在实体配置覆盖中引用此条目：

Copied!
1{{ preprocess .Values.apollo.environment.myKey }}

当Helm渲染 Values 对象时，这将被渲染为 myValue。
