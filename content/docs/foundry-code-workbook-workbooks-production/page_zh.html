投入生产环境

对于基于代码的分析和管道原型，我们推荐使用 Code Workbook 或 Code Workspaces。对于稳健的生产管道，以及需要额外治理和审查层、海量数据或优化性能的工作流，我们推荐使用 Pipeline Builder 和 Code Repositories。

有关在何处编写管道的更多信息，请查阅我们的《Code Workbook、Code Workspaces 与 Code Repositories 对比》。

在 Workbook 中得出了满意的产出结果后，应当巩固逻辑和输出数据集，使其更加可靠。在 Code Workbook 中执行此操作非常简单。接下来我们将介绍几个简单的步骤，帮助您的工作变得稳健并准备好投入生产环境：

  • 保护分支
  • 使用批量构建
  • 数据健康检查
  • 组织输出
  • 组织图
  • 导出至 Code Repository

保护分支

要在 Workbook 中保护分支，您必须拥有该 Workbook 的数据主人权限。

分支保护功能可以锁定 Workbook 中的分支，防止任何人直接编辑该分支中的逻辑。相反，逻辑更改必须先在其他分支中创建，然后才能合并到受保护的分支中。通常，用户会保护 Workbook 中的 master 分支，但您也可以保护任何其他分支。

保存后，您的分支将被保护并设为只读。

使用批量构建

您可能需要定期刷新在 Code Workbook 中创建的输出数据集，这可以基于输入数据集的更新，也可以基于时间节奏。您可以通过在这些输出数据集上调度周期性构建来实现。

  • 要在单个输出数据集上调度周期性构建，请打开数据集操作并点击 Manage Schedules。
  • 要在多个输出数据集上调度周期性构建，请点击工作簿顶部的齿轮图标并选择 Explore Data Lineage。此时，您将看到工作簿中的所有输入数据集和已保存的输出数据集。

这两种操作都会将您引导至数据血缘关系应用。点击右侧窗格中的日历图标以打开 Manage Schedules 界面。按照提示设置周期性计划。在下图中，展示了每天构建 dataset_1 和 dataset_2 的计划。

了解有关在数据血缘关系中创建计划的更多信息。

请注意，批量构建不会更新未保存为数据集的转换（例如未持久化的转换）。具体来说，考虑未持久化的转换 A 是持久化转换 B 的父级这一情况。如果使用批量构建来构建转换 B，转换 B 将使用来自转换 A 的最新逻辑，以及来自上游输入数据集的最新数据。但是，工作簿中为转换 A 显示的预览，以及在转换 A 中创建的任何可视化内容，都不会由此批量构建进行更新。

数据健康检查

对于输出数据集的另一个最佳实践是添加 Data Health checks。打开您的输出数据集并点击 Health 选项卡以访问数据健康页面。设置健康检查后，如果数据集构建失败、变得陈旧或不符合您指定的其他要求，您将能够收到通知。

整理输出

在项目中将工作簿作为一部分使用时，我们建议在项目中创建以下文件夹结构：

  • /data
  • /workbooks
  • /templates

设置好此文件夹结构后，您可以轻松地从主文件夹中共享新的工作簿：

  1. 将工作簿移动到 /workbooks 目录。
  2. 如果工作簿中创建了任何模板，请将其移动到 /templates 目录。
  3. 要移动数据集，请点击工作簿左上角的“输出”下拉菜单，然后点击“更改文件夹…”以选择用于存放新输出数据集的文件夹。选择项目中的 /data 文件夹。默认情况下，工作簿中派生的所有数据集都将移动到您选定的新文件夹中。

整理图谱

您可能需要对图谱进行管理和组织，以便其他用户能够轻松理解转换流程。代码工作簿提供了两种组织功能：节点着色和自动布局。

您可以使用节点着色功能在图谱中对节点进行视觉分组。点击右上角的 颜色 按钮可创建新的颜色组，选中节点并使用颜色组上的 + 按钮即可将节点添加到该颜色组中。在工作簿内容助手中，您还可以按颜色组对数据集列表进行排序。

点击右上角的布局按钮，还可以对图谱的各个部分进行自动布局。默认情况下，自动布局会排列整个图谱，但您也可以选择特定节点并点击自动布局，仅对这些节点进行排列。

导出至代码仓库

如果在代码工作簿中完成了管道原型设计，并希望将代码迁移至代码仓库，可以使用 导出至代码仓库 助手。了解有关导出至代码仓库的更多信息。

您可能出于多种原因希望将代码迁移至代码仓库：

  • 代码仓库提供完整的 Git 支持，允许用户查看并恢复到之前的提交。
  • 代码仓库支持增量转换和多输出转换。
  • 代码仓库支持取消标记工作流，并可通过分支设置控制何时合并拉取请求。
