使用Apollo CLI入门

Apollo CLI（apollo-cli）是一个通用的命令行工具，用于在Apollo Hub上执行操作，例如上传新产品或查看由Apollo管理的环境中安装的内容。本指南将介绍下载和设置Apollo CLI的过程。有关更多详细信息，请参阅 Apollo CLI文档。

apollo-cli支持的主要操作如下：

  • product-release：将新产品发布作为分发包或Helm Chart上传，以便在环境中安装。
  • entity：列出给定环境中所有已安装的产品。
  • environment：列出由Apollo管理的所有环境的元数据。
  • product：列出Apollo Hub中当前可用的所有产品（通过Apache Maven坐标）。

初始设置

下载 CLI

在 Apollo 的左侧边栏中导航至设置。在 设置与配置 页面的左侧选择 下载 以下载 Apollo CLI。

将出现一个弹出窗口，根据您的操作系统提供 CLI 的下载选项。

选择该选项将下载 CLI 的编译二进制文件。下载完成后，将文件移动到您希望托管 CLI 的位置，并更改权限以便执行 CLI。

要使用 apollo-cli，请调用可执行文件并为其提供所需的命令和任何必需的标志。运行 apollo-cli --help 将显示可用命令的列表以及它们的使用说明。可以将 --help 或 -h 标志添加到任何命令中，以获取该特定命令及其标志的文档。例如，apollo-cli --help 表示 environment 是一个可用的命令，运行 apollo-cli environment --help 将显示 environment 命令的使用说明。然后，您可以运行诸如 apollo-cli environment list 等命令。

本指南中的所有命令都假设 apollo-cli 可执行文件位于当前工作目录中。我们建议将 apollo-cli 可执行文件移动到您的 $PATH 中的文件夹，以便可以从任何工作目录运行 CLI。有关更多详细信息，请参阅您 shell 的文档。

(可选) 设置 CLI 命令补全功能

apollo-cli 支持大多数常见 shell（如 bash、fish、powershell、zsh 等）的命令补全。当安装了命令补全支持后，您可以使用键盘上的 Tab 键来对子命令和标志进行自动补全。

运行针对您特定 shell 的 ./apollo-cli completion [shell] --help 命令，以获取如何为您的 shell 安装补全功能的说明。例如，./apollo-cli completion zsh --help 会打印出如何为 zsh shell 启用补全功能的说明。

认证

要使用 Apollo CLI，您需要提供以下内容之一：

  • Apollo Bearer Token
  • 服务账户的客户端 ID 和客户端密钥

当您运行 apollo-cli configure 时，系统会提示您选择 Apollo token provider 的身份验证方法。您可以输入 static 来使用 Bearer Token，或者输入 service-user 来使用服务账户。然后，您将被提示输入 Bearer Token 或者客户端 ID 和客户端密钥。如果您使用的是 Bearer Token，则不应输入客户端 ID 和客户端密钥。同样地，如果您使用的是服务账户，则不应输入 Bearer Token。

您不能同时使用两种身份验证方法。您可以通过重新运行 apollo-cli configure 来切换身份验证方法。您必须将之前的配置值替换为空字段。

创建一个 Bearer Token

您可以通过导航到 https://<Hub>.palantircloud.com/multipass/app 来生成一个 Bearer Token，其中 <Hub> 是您的 Apollo Hub 的唯一名称。

在侧边栏菜单的用户设置部分中，选择令牌。然后在右上角选择+ 创建令牌，并为该令牌输入名称、简要描述和过期日期。

将 Bearer 令牌作为名为 APOLLO_TOKEN 的密钥添加到您的 CI 管道中，以便该值可在 CI 脚本中使用。

设置 Apollo Hub 配置

在您首次与 Apollo Hub 进行交互之前，请运行 ./apollo-cli configure 以设置与您的 Hub 交互所需的配置。当您运行以下命令时，将被提示输入每个变量的值。

configure 命令会在您的本地 ~/.palantir/ 目录中创建一个 apollo-cli.config 文件。您可以在此处检查文件是否存在。也可以留空或不指定某些值。

当 Apollo CLI 运行需要在配置中指定值的命令时，如果配置文件存在，它将尝试从配置文件中读取这些值。但是，这些值也可以作为环境变量或命令行标志来指定。

示例1：使用CLI将Redis Helm Chart发布到Apollo

在这个示例中，我们将Artifact Hub上的开源 Redis Helm Chart ↗ 作为产品安装到Apollo中。

从概念上讲，将产品发布到Apollo需要将产品元数据和其清单添加到产品目录中。Apollo CLI 的 publish 命令会接收一个标准的Helm Chart，并创建一个包含Apollo特定元数据和产品清单的新工件。然后，它会将这两个组件发布到产品目录中。

每个发布到Apollo的Helm Chart产品都必须指定一个Helm仓库URL、图表名称和图表版本。当Apollo部署一个Helm Chart产品时，Apollo会在指定仓库中查找指定名称和版本的Chart。这意味着即使使用CLI发布了Helm Chart产品，除非该Chart的镜像存在于Helm Chart仓库中（这需要单独完成），否则Apollo将无法部署相应的Chart。

通过CLI将Helm Chart作为产品添加，可以通过直接指定从Helm仓库定位Helm Chart所需的信息，或者通过指定本地存在的Helm Chart文件来完成。如果您的机器可以访问该Chart的Helm仓库，那么基于Helm仓库信息发布产品通常更为简便。

选项1：从Helm仓库发布Helm Chart

以下 apollo-cli 命令从 Bitnami 的 Helm 仓库 https://charts.bitnami.com/bitnami 获取版本 16.9.11 的 Redis Helm Chart，然后使用标识符 com.bitnami:redis:16.9.11 将该产品发布到 Apollo：

已复制！
1
2
3
$ ./apollo-cli product-release helm-chart init --name redis --version 16.9.11 --repository-url "https://charts.bitnami.com/bitnami" --maven-coordinate "com.bitnami:redis:16.9.11" --output-dir "./chart"

$ ./apollo-cli product-release create --manifest "./chart/manifest.yml"

有关更多选项，请参阅 product-release helm-chart init 的完整文档。

从本地 Helm Chart 文件发布 Helm Chart

选项 2：从本地 Helm Chart 文件发布 Helm Chart

CLI 还支持将本地 Helm Chart 文件直接发布到 Apollo。这在您的机器无法与托管 Chart 的 Helm 仓库通信时非常有用。但是请注意，当 Apollo 部署 Helm Chart 时，它总是通过在指定的 Helm 仓库中查找该 chart 来进行部署，使用的是 Helm Chart 元数据中定义的名称和版本。这意味着，即使您使用 CLI 发布了本地 Helm Chart 文件，它也必须在 Helm Chart 仓库中可用，Apollo 才能部署它。

以下步骤演示了如何使用 apollo-cli 发布本地下载的 Redis Helm Chart 版本 16.9.11。

如果你在 Artifact Hub Bitnami Redis 页面 ↗ 上选择 安装，你可以下载一个 .tgz 格式的 Helm Chart（链接显示为“您也可以使用此链接直接下载该包的内容”）。下载 TGZ 文件（应命名为 redis-16.9.11.tgz），并将其放在与 apollo-cli 相同的目录中。

文件下载完成后，可以使用以下命令通过 CLI 将其发布到 Apollo：



已复制！
1
2
3
$ ./apollo-cli product-release helm-chart init --chart-path redis-16.9.11.tgz --repository-url "https://charts.bitnami.com/bitnami" --maven-coordinate "com.bitnami:redis:16.9.11" --output-dir "./chart"

$ ./apollo-cli product-release create --manifest "./chart/manifest.yml"

因为指定了--chart-path标志，不需要提供Helm图表--name或--version：这些值将从Helm图表文件中的元数据中解析出来。但是，Helm的--repository-url标志仍然是必需的，因为Apollo总是从Helm仓库获取Helm图表来部署（此URL不是CLI上传操作的一部分，但将在Apollo部署产品时使用）。

安装已发布的Redis产品

上述发布操作的结果是，Helm图表产品com.bitnami:redis:16.9.11已被发布到Apollo。如果您也创建了一个环境，那么现在可以将Redis安装到该环境中。

导航到环境页面的设置选项卡，并选择安装Helm图表。然后可以从选择产品下拉菜单中选择redis。

从那里，您可以配置并部署 Helm 图表。有关此过程的更多信息，请参阅 使用 Apollo 管理实体。

当你创建一个 Apollo 产品 tarball 时，它应该如下所示：

asset-1.2.3  # <产品名称>-<产品版本>
├── asset  # 要包含的任意文件的目录

在你的 manifest.yml 中，product-type 应该是 asset.v1。

以下 apollo-cli 命令创建了一个 Asset Product，该产品将 my-product 目录打包，并使用标识符 com.my-org:my-product:0.0.2 将产品发布到 Apollo：

$ ./apollo-cli product-release asset publish --maven-coordinate "com.my-org:my-product:0.0.2" --asset-path my-product

您可以通过添加 --images 标志来发布包含图片的 Asset Product：

$ ./apollo-cli product-release asset publish --maven-coordinate "com.my-org:my-product:0.0.2" --asset-path my-product --images docker.io/bitnami/postgresql:16.0.0-debian-11-r10
将产品发布 com.my-org:my-product:0.0.2 发布到 Apollo ... 完成

可以通过运行 ./apollo-cli product-release asset publish --help 查看完整的标志详情，但以下是一个概述：

  • --maven-coordinate: 用户选择的 Apache Maven 坐标，该坐标将在 Apollo 中作为产品发布的地址。可以是任何有效的 Maven 坐标格式，如 group:product-name:version。
  • --asset-path：要发布到Apollo的资产文件名或目录。
  • --images：Apollo在推荐容器化实体升级前会检查可用性的容器镜像URI。
