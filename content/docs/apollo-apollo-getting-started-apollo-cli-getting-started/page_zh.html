使用Apollo CLI入门

Apollo CLI（apollo-cli）是一个通用的命令行工具，用于在Apollo Hub上执行操作，例如上传新的Product或查看由Apollo管理的Environment中已安装的内容。本指南将介绍下载和设置Apollo CLI。有关更详细的信息，请参阅Apollo CLI文档。

apollo-cli支持的主要操作包括：

  • product-release: 将新的Product Release作为distribution或Helm Chart上传，以便在Environment中安装。
  • entity: 列出给定Environment中所有已安装的Products。
  • environment: 列出由Apollo管理的所有Environments的元数据。
  • product: 按Apache Maven坐标列出Apollo Hub中当前可用的所有Products。

初始设置

下载 CLI

导航到 Apollo 左侧的 设置（Settings）。在 设置与配置（Settings & Configuration） 页面左侧选择 下载（Download） 以下载 Apollo CLI。

将出现一个弹出窗口，可以根据您的操作系统下载 CLI。

选择该选项将下载 CLI 的编译二进制文件。下载完成后，将文件移动到您希望托管 CLI 的位置，并更改权限以便您可以执行 CLI。

Copied!
1
$ chmod +x apollo-cli

要使用 apollo-cli，请调用可执行文件并提供所需的命令和任何必要的标志。运行 apollo-cli --help 将显示可用命令的列表以及它们的使用说明。可以将 --help 或 -h 标志添加到任何命令中，以获取该特定命令及其标志的说明。例如，apollo-cli --help 表示 environment 是一个可用的命令，运行 apollo-cli environment --help 将显示 environment 命令的使用方法。然后，您可以运行诸如 apollo-cli environment list 等命令。

本指南中的所有命令都假定 apollo-cli 可执行文件位于当前工作目录中。我们建议将 apollo-cli 可执行文件移动到您 $PATH 上的一个文件夹中，这样 CLI 就可以从任何工作目录运行。有关更多详细信息，请参阅您的 shell 文档。

(可选) 设置 CLI 命令补全

apollo-cli 为大多数常见的 shell（如 bash、fish、powershell、zsh 等）支持命令补全。当安装了命令补全支持后，您可以通过键盘上的 Tab 键对子命令和标志进行自动补全。

运行 ./apollo-cli completion [shell] --help 命令以获取特定 shell 的安装说明。例如，./apollo-cli completion zsh --help 会打印出如何为 zsh shell 启用补全的说明。

认证

要使用 Apollo CLI，您需要提供以下内容之一：

  • Apollo Bearer Token（承载令牌）
  • 服务账户的客户端 ID 和客户端密钥

当您运行 apollo-cli configure 时，会在提示输入 Apollo token provider 时选择一种认证方式。您可以输入 static 来使用 Bearer Token 或输入 service-user 来使用服务账户。然后会提示您输入 Bearer Token 或客户端 ID 和客户端密钥。如果您使用的是 Bearer Token，则不应输入客户端 ID 和客户端密钥。同样地，如果您使用的是服务账户，就不应输入 Bearer Token。

您不能同时使用两种认证方式。您可以通过重新运行 apollo-cli configure 来切换认证方式。您必须将之前的配置值替换为空字段。

创建一个 Bearer Token

您可以通过导航到 https://<Hub>.palantircloud.com/multipass/app 来生成一个 Bearer Token，其中 <Hub> 是您的 Apollo Hub 的唯一名称。

在侧边栏菜单的 用户设置（User Settings） 部分中，选择 令牌（Tokens）。然后选择 + 在右上角创建令牌，并输入令牌的名称、简要描述和过期日期。

将 Bearer 令牌作为名为 APOLLO_TOKEN 的密钥添加到您的 CI 管道中，以便在 CI 脚本中可用。

设置 Apollo Hub 配置

在您第一次与 Apollo Hub 交互之前，请运行 ./apollo-cli configure 来设置与您的 Hub 交互所需的配置。当您运行以下命令时，会提示您输入每个变量的值。

configure 命令会在您的本地 ~/.palantir/ 目录中创建一个 apollo-cli.config 文件。您可以检查该目录以确认文件存在。也可以将某些值留空或不指定。

当Apollo CLI运行需要在配置中指定值的命令时，如果存在配置文件，它将尝试从配置文件中读取这些值。但是，这些值也可以作为环境变量或命令行标志指定。

示例1：使用CLI将Redis Helm图表发布到Apollo

在这个示例中，我们将安装一个开源的 Artifact Hub 上的 Redis Helm Chart ↗ 作为 Apollo 中的产品。

从概念上讲，将产品发布到 Apollo 需要将产品元数据及其清单添加到产品目录中。Apollo CLI publish 命令会使用标准的 Helm Chart 并创建一个包含 Apollo 特定元数据和产品清单的新制品。然后，它会将这两个组件发布到产品目录。

所有发布到 Apollo 的 Helm Chart 产品都必须指定 Helm 仓库 URL、图表名称和图表版本。当 Apollo 部署 Helm Chart 产品时，Apollo 会在指定仓库中查找具有指定名称和版本的 Chart。这意味着即使使用 CLI 发布了 Helm Chart 产品，除非该 Chart 的镜像存在于 Helm Chart 仓库中（这需要单独完成），否则 Apollo 将无法部署相应的 Chart。

通过 CLI 将 Helm Chart 作为产品进行添加，可以通过直接从 Helm 仓库指定定位 Helm Chart 所需的信息，或者通过指定本地存在的 Helm Chart 文件来完成。如果你的机器可以访问该 Chart 的 Helm 仓库，那么基于 Helm 仓库信息发布产品通常更为简便。

选项 1：从 Helm 仓库发布 Helm Chart

以下 apollo-cli 命令从 Bitnami 的 Helm 仓库 https://charts.bitnami.com/bitnami 获取 Redis Helm Chart 的版本 16.9.11，然后使用标识符 com.bitnami:redis:16.9.11 将产品发布到 Apollo：

Copied!
1
2
3
$ ./apollo-cli product-release helm-chart init --name redis --version 16.9.11 --repository-url "https://charts.bitnami.com/bitnami" --maven-coordinate "com.bitnami:redis:16.9.11" --output-dir "./chart"

$ ./apollo-cli product-release create --manifest "./chart/manifest.yml"

有关更多选项，请参阅 product-release helm-chart init 的完整文档。

选项 2：从本地 Helm Chart 文件发布 Helm Chart

CLI 还支持将本地 Helm Chart 文件直接发布到 Apollo。这在你的机器无法与托管 Chart 的 Helm 仓库通信时可能会很有用。但是请注意，当 Apollo 部署 Helm Chart 时，它总是通过在指定的 Helm 仓库中查找该 chart 来进行，使用的是 Helm Chart 元数据中定义的名称和版本。这意味着，即使你使用 CLI 发布本地 Helm Chart 文件，也必须在 Helm Chart 仓库中可用，Apollo 才能部署它。

以下步骤演示了如何使用 apollo-cli 来发布本地下载的 Redis Helm Chart 的 16.9.11 版本。

如果你在 Artifact Hub Bitnami Redis 页面 ↗ 上选择 安装，你可以下载一个 .tgz 的 Helm Chart（链接显示“您也可以使用此链接直接下载此包的内容”）。下载 TGZ（它应该被命名为 redis-16.9.11.tgz），并将其放在与 apollo-cli 相同的目录中。

一旦文件已下载，您可以使用以下命令通过CLI将其发布到Apollo：

Copied!
1
2
3
$ ./apollo-cli product-release helm-chart init --chart-path redis-16.9.11.tgz --repository-url "https://charts.bitnami.com/bitnami" --maven-coordinate "com.bitnami:redis:16.9.11" --output-dir "./chart"

$ ./apollo-cli product-release create --manifest "./chart/manifest.yml"

由于指定了 --chart-path 标志，不需要提供 helm-chart --name 或 --version：这些值将从Helm Chart文件中的元数据中解析出来。但是，仍然需要 helm --repository-url 标志，因为Apollo始终从Helm Repository获取Helm Chart以进行部署（此URL不是CLI上传操作的一部分，但将在Apollo部署产品时使用）。

安装已发布的Redis产品

上述任何发布操作的结果是，Helm Chart产品 com.bitnami:redis:16.9.11 已被发布到Apollo。如果您还创建了 环境，那么现在可以将Redis安装到环境中。

导航到 设置 选项卡，并选择 安装helm-charts . 然后您可以从选择产品下拉菜单中选择 redis。

从那里，您可以配置并部署 Helm 图表。有关此过程的更多信息，请参阅 使用 Apollo 管理实体。

示例 2：使用 CLI 将资产发布到 Apollo

在本例中，我们将一些自定义文件作为 资产（Asset） 发布到 Apollo。

假设我们有一个名为 my-product 的目录，其中包含以下文件：

当您创建一个 Apollo 产品 tarball 时，它应该如下所示：

asset-1.2.3  # <product-name>-<product-version>
├── asset  # directory of arbitrary files to include
│   └── ...
└── deployment
    └── manifest.yml

在您的 manifest.yml 中，product-type 应为 asset.v1。

以下 apollo-cli 命令将创建一个捆绑 my-product 目录的资产产品，然后使用标识符 com.my-org:my-product:0.0.2 将产品发布到 Apollo：

Copied!
1
2
$ ./apollo-cli product-release asset publish --maven-coordinate "com.my-org:my-product:0.0.2" --asset-path my-product
Publishing product release com.my-org:my-product:0.0.2 into Apollo ... done

您可以使用 --images 标志发布带有图片的资产产品：

Copied!
1
2
$ ./apollo-cli product-release asset publish --maven-coordinate "com.my-org:my-product:0.0.2" --asset-path my-product --images docker.io/bitnami/postgresql:16.0.0-debian-11-r10
Publishing product release com.my-org:my-product:0.0.2 into Apollo ... done

可通过运行 ./apollo-cli product-release asset publish --help 查看完整的标志详情，但以下为概述：

  • --maven-coordinate：用户选择的 Apache Maven 坐标，该坐标将作为 Apollo 中产品发布（Product Release）的地址。可以是任何格式为 group:product-name:version 的有效 Maven 坐标。
  • --asset-path：要发布到 Apollo 的资产文件名或目录。
  • --images：Apollo 在推荐容器化实体升级前会检查可用性的容器镜像 URI。
