Microsoft Exchange

Microsoft Exchange 连接器是 Palantir 提供的用于 Microsoft Exchange 的驱动程序。

要创建新的 Microsoft Exchange 源，请按照 Palantir 提供的驱动程序的标准设置流程操作，然后使用以下章节进行 Microsoft Exchange 特定的配置和网络设置。有关完整的属性参考，请参阅 官方 Microsoft Exchange 驱动程序文档 ↗。

配置

以下属性为必填或建议填写项。

属性是否必填？描述默认值
DefaultGroups ↗必填确定访问 Microsoft Exchange 中组级资源时的默认组上下文。AllGroups
DefaultUser ↗必填确定访问 Microsoft Exchange 中用户级资源时的默认用户上下文。CurrentUser
Server ↗必填指定要连接的 Microsoft Exchange 服务器 URL。https://outlook.office365.com/EWS/Exchange.asmx
AuthScheme ↗建议指定用于连接 Microsoft Exchange 服务器的身份验证方案。AzureServicePrincipal
AzureTenant ↗建议标识用于访问数据的 Microsoft Exchange 租户。接受租户的域名（例如 contoso.onmicrosoft.com）或其目录（租户）ID。—
GroupId ↗建议指定要访问其数据的 Microsoft Exchange 组的 ID。—
InitiateOAuth ↗建议指定获取或刷新 OAuth 访问令牌的流程，在经过身份验证和授权的用户工作时保持用户的访问权限。GETANDREFRESH
OAuthClientId ↗建议指定分配给自定义 OAuth 应用程序的客户端 ID（也称为使用者密钥）。在身份验证期间，需要此 ID 向 OAuth 授权服务器标识应用程序。—
OAuthClientSecret ↗建议指定分配给自定义 OAuth 应用程序的客户端密钥。此机密值用于向 OAuth 授权服务器验证应用程序。（仅限自定义 OAuth 应用程序。）—
Platform ↗建议指定建立连接时要目标的 Microsoft Exchange 平台。Exchange_Online
Schema ↗建议指定连接到 Microsoft Exchange 平台时要使用的架构。EWS
UserId ↗建议指定要访问其数据的 Microsoft Exchange 用户的 ID。—

网络连接

下表列出了为成功运行源而需要能够访问的域。

对于每个域，添加相应的出站策略。如果源托管在本地并且无法从 Foundry 直接访问，请改用智能体代理出站策略；智能体主机本身也必须能够访问列出的域。有关详细信息，请参阅使用智能体作为代理。

域是否需要
<Server>始终需要。对于 Exchange Online，使用 Server=' https://outlook.office365.com/EWS/Exchange.asmx'
outlook.office365.com如果 Platform=Exchange_Online 且 Schema=EWS
graph.microsoft.com如果 Platform=Exchange_Online 且 Schema=MSGraph
login.microsoftonline.com如果 Platform=Exchange_Online（默认）且 AuthScheme=AzureAD, AzureServicePrincipal 或 AzureServicePrincipalCert
<KerberosKDC>:88如果 AuthScheme=Negotiate
<KerberosServiceKDC>:88如果 AuthScheme=Negotiate 且 Kerberos 拓扑使用多个域
