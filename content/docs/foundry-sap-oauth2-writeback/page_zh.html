基于 OAuth 2.0 的用户级 SAP 回写

本页包含有关在 SAP 中设置 OAuth 2.0 服务器以及在 Foundry 中设置 OAuth 2.0 客户端的操作说明。

在 SAP 中设置 OAuth 2.0 服务器

前提条件

  • 面向 SAP 应用程序的 Palantir Foundry Connector 2.0（“连接器”）SP21 或更高版本
  • SAP 中的 Foundry 技术用户应为 SYSTEM 用户
  • 应将 /PALANTIR/OAUTH_CLIENT 分配给 Foundry 技术用户以及任何希望从 Foundry 向 SAP 执行回写操作的最终用户
  • 应将 /PALANTIR/CONTENT_FUNCTION_ALL 分配给最终用户
  • 需激活 /sap/public/bc 节点下的所有服务（用于 OAuth 2.0 配置）
    • /sap/bc/sec/oauth2*
    • /default_host/sap/bc/webdynpro/sap/oauth2_authority
  • SAP Gateway 处于活动状态
  • SAP_BASIS（技术组件版本，适用于 SAP NetWeaver 和 SAP S/4HANA 系统）7.4 SP09 或更高版本（支持 OAuth 2.0 和 OData）

参考文档

  • 《Note 1688545 - AS ABAP 中的 OAuth 2.0 服务器故障排除》 ↗（需登录 SAP）
  • 《帮助：AS ABAP 的 OAuth 2.0 服务器》 ↗

OAuth 2.0 配置

  1. 运行 SOAUTH2 事务码。
  2. 选择 创建...。
  3. 输入 Foundry 技术用户的用户名作为 OAuth 2.0 客户端 ID。
  4. 选择 下一步 >。
  5. 输入 Foundry 技术用户的用户名作为 用户 ID。
  6. 确保已勾选 客户端用户 ID 和密码 以及 SSL 客户端证书。
  7. 选择 下一步 >。
  8. 将 重定向 URI 设置为 https://<FOUNDRY_DOMAIN>/workspace/oauth2-clients/callback。
  9. 选择 下一步 >。
  10. 添加 作用域分配，其中 OAuth 2.0 作用域 ID 为 /PALANTIR/SRV_0001，描述例如为 Palantir Foundry writeback using SAP functions。
  11. 选择 下一步 >，然后选择 完成。

OData 配置

  1. 在 SAP 的 维护服务 页面中，按照服务层级结构找到 opu > odata > palantir。
  2. 右键点击 palantir 并选择 激活服务。
  3. 出现提示时，选择 是。
  4. 在 创建/更改服务 选项卡中，选择 交互式选项 下的 GUI 配置。
  5. 添加名为 ~CHECK_CSRF_TOKEN、值为 0（零）的参数。
  6. 按照此处说明禁用 CSRF_TOKEN 验证：https://help.sap.com/doc/saphelp\_hba/1.0/de-DE/e6/cae27d5e8d4996add4067280c8714e/content.htm
  7. 运行 /IWFND/MAINT_SERVICE 事务码。
  8. 在 系统别名 下选择 添加系统别名。
  9. 添加具有以下值的系统别名：
    • 服务文档标识符： /PALANTIR/SRV_0001
    • 用户角色： 留空
    • 主机名： 留空
    • SAP 系统别名： LOCAL
    • 元数据默认值： 未勾选
    • 默认系统： 已勾选
    • 技术服务名称： /PALANTIR/SRV
    • 外部服务名称： ODATA_SRV
    • 版本： 1
    • 用户名： 留空

在 Foundry 中设置 OAuth 2.0 客户端

此过程遵循《配置出站应用程序》中概述的通用方法，但已针对 SAP 系统进行了专门调整。

出站应用程序设置

  1. 导航至控制面板，并在 组织设置 下选择 出站应用程序。
  2. 选择 新建应用程序。
  3. 提供 应用程序名称，例如您的 SAP 系统名称。
  4. 设置 批准提示 以描述授权操作，例如 Allow Foundry to act on your behalf in SAP?。
  5. 在 OAuth 2.0 服务器连接 下，选择 Foundry Worker 选项。
  6. 将 授权页面 URL 设置为 SAP OAuth 2.0 授权端点：
https://<SAP_DOMAIN>/sap/bc/sec/oauth2/authorize
  1. 将 令牌端点 URL 设置为 SAP OAuth 2.0 令牌端点：
https://<SAP_DOMAIN>/sap/bc/sec/oauth2/token
  1. 在 出站流量策略 下，选择一种代理出站流量策略，该策略通过安装在您的网络中且具备 SAP 系统访问权限的数据连接代理路由流量。

  1. 在 OAuth 2.0 设置 下，将 客户端 ID 设置为 SAP OAuth 2.0 服务器配置中的客户端 ID。在 作用域 下，添加 /PALANTIR/SRV_0001。
  2. 保存出站应用程序。
  3. 在创建 SAP webhook 时，现在即可使用此出站应用程序。

有关出站应用程序配置选项的更多详情，请参阅《配置出站应用程序》。

（旧版）基于 Webhook 的 OAuth 2.0 配置

旧版

下文所述的基于 Webhook 的 OAuth 2.0 配置已进入旧版开发阶段，预计不再进行额外开发。请改用上方的标准出站应用程序设置。有关旧版方案的更多信息，请参阅《（旧版）基于自定义 Webhook 的 OAuth 2.0 握手》。

以前，为 SAP 配置 OAuth 2.0 需要创建一个带有 Webhook 的专用 REST API 源，以手动处理令牌和刷新流程。如果现有的 SAP 集成使用了此方法，它将继续运行。但是，对于新配置，请使用上方的标准出站应用程序设置。

源连接设置

警告

确保 SAP 源 URL 使用 HTTPS，否则在使用 OAuth 流程时 Webhook 将失败。

  1. 创建一个新的 REST API 源。

  1. 使用用于 SAP 源的基础域 URL 和端口配置该源。
  2. 选择 基本 身份验证，并添加用于连接到 SAP 的用户名和密码。

  1. 保存该源。

OAuth 2.0 授权流程 Webhook 设置

作为函数发布的 Webhook 不兼容

作为函数发布的 Webhook 不能用于基于旧版自定义 Webhook 的 OAuth 2.0 出站应用程序中。在创建令牌和刷新 Webhook 时，请确保已禁用函数发布。

  1. 在新的 REST API 源的概览页面上，选择 创建 Webhook。

  2. 为 Webhook 命名（例如 SAP OAuth2 authorization code flow webhook）。确保 函数配置 已关闭。

  3. 进入 请求配置 步骤。

  4. 在 调用 下，选择 POST 作为请求类型，并输入 sap/bc/sec/oauth2/token 作为路径。

  5. 在 查询参数 下，如果使用的客户端不是默认客户端，则可能需要设置 sap-client。

  1. 向下滚动到 输入参数 并添加以下三个参数（均为字符串类型）：
  • redirect_uri
  • client_id
  • authorization_code

  1. 向上滚动回 调用，然后选择 请求体 选项卡。
  2. 选择 表单 URL 编码 并添加以下四个条目：
  • grant_type → authorization_code
  • redirect_uri → 映射到 redirect_uri 输入参数（有关如何执行此操作，请参见下文）
  • client_id → 映射到 client_id 输入参数
  • code → 映射到 authorization_code 输入参数
  1. 要映射输入参数，请在字段中输入 @，然后选择 输入参数。找到相关参数，将其选中，然后在下方选择 添加。

  1. 进入 响应 步骤。
  2. 创建以下五个 输出参数。所有参数均应为字符串类型，并且应按键从响应中提取。
  • access_token
  • token_type
  • expires_in
  • refresh_token
  • scope

  1. 选择 创建 Webhook 并继续 以保存 Webhook。

OAuth 2.0 刷新流程 Webhook 设置

  1. 从 REST API 源创建一个新的 Webhook。
  2. 为 Webhook 指定一个独特的名称（例如 SAP OAuth2 refresh flow webhook）。确保 函数配置 已关闭。
  3. 再次将请求方法设置为 POST，并使用相同的路径（sap/bc/sec/oauth2/token）。
  4. 与之前的 Webhook 一样，如有需要，可将 sap-client 设置为 查询参数。
  5. 在 Headers 标签页中，添加以下请求头：
  • Content-Type → application/x-www-form-urlencoded

  1. 设置以下两个 输入参数（均为字符串类型）：
  • client_id
  • refresh_token
  1. 然后在 Body 标签页下，添加以下三项内容：
  • grant_type → refresh_token
  • client_id → 映射到 client_id 输入参数
  • refresh_token → 映射到 refresh_token 输入参数
  1. 创建与授权码流程 Webhook 完全相同的五个 输出参数。所有参数均应为字符串类型，并从响应中按键提取。
  • access_token
  • token_type
  • expires_in
  • refresh_token
  • scope
  1. 选择 Create webhook and continue 以保存该 Webhook。

传统出站应用设置

  1. 导航至 Foundry Control Panel 并选择 Outbound applications。
  2. 为应用命名，然后按照《(传统) 基于 Custom Webhook 的 OAuth 2.0 握手》中概述的步骤操作。
  3. 应将之前创建的两个 Webhook 分别用作 Token webhook 和 Refresh token webhook。
  4. Authorization page URL 应采用以下格式：
https://<SAP_DOMAIN>/sap/bc/sec/oauth2/authorize
  1. 在 OAuth 2.0 settings 下，将 Client ID 设置为 SAP OAuth 2.0 服务器配置中的客户端 ID。在 Scopes 下，添加 /PALANTIR/SRV_0001。
  2. 保存该出站应用。
  3. 现在，在创建 SAP Webhook 时即可使用此出站应用。
