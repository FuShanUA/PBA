注意事项：流水线构建器与代码仓库

Foundry 提供了两款用于编写和管理数据流水线的产品：流水线构建器和代码仓库。这两款工具相辅相成，协同工作，能够满足各种流水线需求。以下指南旨在帮助您确定哪款工具最适合您的业务场景，以及如何将它们结合使用。

流水线构建器

流水线构建器是 Foundry 的核心应用，用于快速、灵活且可扩展地交付数据流水线，同时保障稳健性与安全。借助该工具，终端用户和数据工程师可以在图形化与表单环境中协同工作，完成数据集成、构建业务逻辑转换，并为生产流水线定义严格的发布流程。用户无需编写代码即可构建能够提供实时反馈的流水线。此外，流水线构建器具备健康检查机制，确保只有完全合规的数据才能部署到生产环境。了解有关流水线构建器的更多信息。

代码仓库

代码仓库提供了一个基于网页的集成开发环境（IDE），方便用户在 Foundry 中编写可用于生产环境的代码并开展协作。该应用为与底层 Git 仓库交互提供了便捷的用户体验。了解有关代码仓库的更多信息。

最佳实践

我们建议在流水线构建器中进行流水线设计。这样做将：

  • 通过直观的点击式界面，实现不同用户群体间的协同工作。
  • 利用流水线构建器内置的规范约束进行安全、高效的数据转换和流水线管理，保障流水线健康。

如果用户需要流水线构建器尚不具备的特定代码逻辑，则应使用代码仓库创建相应的处理阶段，并将其添加到主流水线中。这类特定场景包括：

  • 调用 API
  • 使用自定义库
  • 添加基于代码的逻辑概念

由于流水线构建器和代码仓库均使用 Foundry 数据集作为输入和输出，因此在代码仓库中构建的流水线输入，可以添加到流水线构建器主流线的前面、后面或中间。无论使用哪款应用创建流水线，都可以在数据血缘关系中为整条流水线配置调度和健康检查。了解有关数据血缘关系的更多信息。
