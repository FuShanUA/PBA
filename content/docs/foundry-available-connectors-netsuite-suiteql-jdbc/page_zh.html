Oracle NetSuite SuiteQL

使用 SuiteQL 框架将 Foundry 连接到 Oracle NetSuite，并开始将数据从 NetSuite ERP 同步到 Foundry。

支持的功能

功能状态
数据探索🟢 正式可用
批量同步🟢 正式可用
增量同步🟢 正式可用

设置

  1. 打开《数据连接》应用程序，在屏幕右上角选择 + 新建源。
  2. 从可用的连接器类型中选择 NetSuite SuiteQL。
  3. 按照后续的配置提示，使用下方各节中的信息继续设置连接器。

详细了解在 Foundry 中《设置连接器》的相关信息。

身份验证

NetSuite SuiteQL 源使用《基于令牌的身份验证 (TBA)》。

您的账户必须启用基于令牌的身份验证功能。有关启用 TBA 的信息，请参阅《NetSuite 文档》。

在 NetSuite 中配置用户角色和权限

NetSuite 中的访问控制通过为用户分配角色来配置；每个角色是一组权限的集合，用于定义用户可以执行哪些任务以及可以访问哪些数据。我们建议为将连接到 Foundry 的用户进行如下配置：

  1. 创建一个具有适当权限的专用角色。

    1. 从 NetSuite 的工具栏中，选择 设置 > 用户/角色 > 管理角色 > 新建，并为该角色指定一个明确的名称。我们建议使用 foundry-role。
      • 您可以选择勾选 仅限 Web 服务 配置复选框。
    2. 导航到角色页面底部，选择 权限 > 设置，为该角色添加系统级权限。您需要添加的最低要求权限包括：
      • 使用访问令牌登录
      • REST Web 服务
      • 自定义记录类型
      • 自定义字段
      • 添加所需权限后，请记得 保存。
    3. 导航到角色页面底部，选择 权限 > 报表，为该角色添加报表权限。选择：
      • SuiteAnalytics 工作簿，
      • 请记得 添加 权限后再 保存。
    4. 导航到角色页面底部，选择 权限 > 列表，为您希望在 Foundry 中查询的表添加表权限。选择所需的表，选择 添加，然后 保存。

  2. 将新角色分配给用户。

    1. 从 NetSuite 的工具栏中，选择 设置 > 用户/角色 > 管理用户。选择您要用于连接 Foundry 的用户，然后选择 编辑。
    2. 导航到 访问 选项卡，确保勾选了 授予访问权限 复选框。
    3. 在 角色 选项卡中，从下拉列表中选择新创建的角色 (foundry-role)，选择 添加，然后 保存。

要验证您是否添加了正确的权限，请以分配了新角色的用户身份登录，并检查您是否可以查看所有预期数据。

在 NetSuite 中配置集成和访问令牌

NetSuite 中的集成记录用于管理与外部系统的连接。我们建议使用以下配置来连接 Foundry：

  1. 使用 TBA 创建新的集成记录（详见《更多详情》）。
    1. 从 NetSuite 的工具栏中，选择 设置 > 集成 > 管理集成 > 新建，并为该集成指定一个明确的名称。我们建议使用 foundry-integration。
    2. 确保 状态 为 已启用，并且只勾选 基于令牌的身份验证。所有其他复选框均应取消勾选。
    3. 保存 后，请记下 CLIENT ID 和 CLIENT SECRET。配置 Foundry 时将需要用到它们。

CLIENT ID 和 CLIENT SECRET 仅在您首次保存集成记录时显示。您必须重置它们才能获取新的值，这将使之前的值失效。

  1. 创建并分配 TBA 令牌。
    1. 从 NetSuite 的工具栏中，选择 设置 > 用户/角色 > 访问令牌 > 新建。
      • 如果无法为其他用户管理令牌，请前往 NetSuite 主页左下角的 设置 面板，选择 管理访问令牌。
    2. 选择新建的应用程序（本例中为 foundry-integration）、已分配新角色（foundry-role）的用户以及该新建角色。
      • 如果无法为其他用户管理令牌，系统会默认选中您的用户。请确保您的用户已分配该新建角色（foundry-role）。
    3. 点击保存后，请记下 TOKEN ID 和 TOKEN SECRET。配置 Foundry 时需要用到这些信息。

TOKEN ID 和 TOKEN SECRET 仅在首次保存令牌时显示。如需获取新的 TOKEN ID 和 TOKEN SECRET，您必须创建新令牌。

网络配置

NetSuite SuiteQL 连接器需要访问网络才能连接到目标 NetSuite 实例。

Foundry worker 连接

如果连接在 Foundry worker 上运行，您需要将以下出站策略添加到源中：

  • <ACCOUNT_ID>.suitetalk.api.netsuite.com 的 443 端口。连接 NetSuite 时，可以在 URL 中找到您的账户 ID。

如果此出站策略不存在，您可以申请出站策略；否则，您可以添加出站策略。

创建同步

可以探索 NetSuite SuiteQL 源来发现表并创建新同步。 您也可以在源的概览页面上手动创建新同步。

注意事项

SuiteQL 表的 Schema 定义

原生 SuiteQL 表的 schema 硬编码在驱动程序中，但您的账户或角色可能没有查看所有列的权限；默认情况下，系统会提取表的前 1000 行数据，用于确定当前角色可查看的确切列。

可以通过修改 RowScanDepth 连接属性来更改此行为。具体而言，将 RowScanDepth 设置为 0 将允许您查看所有列，但可能导致权限异常。

如需详细了解原生 SuiteQL 表的 schema 定义，请查阅 《NetSuite JDBC 驱动程序关于 SuiteQL 的文档》 ↗。

要更改 RowScanDepth 的值，请前往源的连接详情页面，然后依次进入更多选项和JDBC 属性。

在JDBC 属性中，您可以添加 RowScanDepth 并设置所需的值。
