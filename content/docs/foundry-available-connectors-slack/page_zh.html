数据模型

CData 驱动程序将 Slack 数据建模为关系数据库中的表列表，您可以使用标准 SQL 语句进行查询。有关可用表及其内容的信息，请查阅 CData 关于《Slack 表》的文档。

设置

  1. 打开数据连接应用程序，并在屏幕右上角选择 + 新建源。
  2. 从可用的连接器类型中选择 Slack。
  3. 按照后续的配置提示，使用以下各节中的信息继续设置您的连接器。

了解有关在 Foundry 中《设置连接器》的更多信息。

身份验证

Slack 身份验证通过 Bearer Token 执行。您可以使用与 Slack 工作区中已安装的 Slack 应用关联的 Bot 用户 Token 来提供此令牌。

使用 Bot Token 可以创建持久且稳定的数据源，而不会直接绑定到某个用户个体。要创建 Slack 应用并获取其 Bot Token，请按照 Slack 文档中的步骤操作：

  • 《在目标工作区中创建 Slack 应用》。
  • 《为应用配置所需权限范围》。
    • 有关所需权限范围的详细信息，请参阅权限。
  • 《将应用安装到目标工作区》。
    • 应用安装可能需要 Slack 工作区管理员的批准。为应用添加权限范围后也需要重新安装。
    • 安装完成后，请按照 Slack 文档中的步骤找到并复制已安装应用的 Bot Token，将其粘贴到数据连接中 Slack 源的凭据设置中（参见配置选项）。

权限

要接入数据表，必须在源配置的凭据设置中为配置 Bearer Token 的 Slack 应用授予特定的 Slack 权限范围。

有关支持所有 CData 驱动程序功能所需的权限范围完整列表，请参阅《CData 文档》。当 Slack 连接器仅用于数据接入功能时，不需要写入权限范围。有关权限范围的更多详细信息，请查阅《Slack 文档》。

如果只需接入特定的数据表或这些表中特定类型的 Slack 频道数据，用于数据接入的 Slack 应用所需的权限范围可以相应缩减。

示例：接入与 Slack 消息和频道类型相关的数据表所需的权限范围包括以下内容：

表名权限范围总是必需的吗？描述
Usersusers:read是加载用户。
Channelschannels:read, groups:read, im:read, mpim:read否仅在分别加载公开、私有、im（直接消息）或 mpim（群组直接消息）频道时需要。加载所有频道类型时需要全部权限（未指定频道筛选器时的默认设置）。
Messages 或 MessageReplieschannels:history, groups:history, im:history, mpim:history + Channels 的要求否仅在分别从公开、私有、im 或 mpim 频道加载消息/回复时需要。从所有频道类型加载时需要全部权限（未指定频道筛选器时的默认设置）。另请参阅下面关于频道成员资格要求的附加说明。

要使用 Slack 连接器通过 Slack 应用的 bot token 从指定 Slack 频道摄取消息和消息回复，该 Slack 应用必须先添加到该 Slack 频道中，并且需具备上述详述的对应频道类型的读取权限范围。 在 Slack 中，用户可以手动将 Slack 应用添加到频道，也可以使用应用的 bot user token，通过 Slack Web API ↗ 以编程方式加入公共频道。

网络连接

Slack 连接器需要通过网络访问 slack.com 的 443 端口。如果您的源端运行在 Foundry 中，请确保存在指向该地址的流出策略。

对于运行在智能体上的源端，智能体主机必须具备访问该域名的网络权限。

配置 Slack 同步

Slack 连接器支持高级同步配置，用于确定将哪些数据以及多少数据引入 Foundry。

探索可用的同步并将其添加到连接器后，导航至编辑同步。在左侧的同步面板中，找到要配置的同步并选择右侧的>。

选项是否必需？说明
Table是选择应将 Slack 中的哪个表复制到 Foundry 中。有关支持的表，请参见数据模型。
Column selection否选择要同步到 Foundry 中的列子集。有关更多信息，请查看下方的列选择部分。
Row filters否启用并添加筛选器，以根据列值移除不符合条件的行。有关更多信息，请查看下方的行筛选器部分。
Limit否向底层查询添加 SQL limit 子句。此设置可用于限制单次运行中同步到 Foundry 的行数。
Incremental否启用以小批量形式将数据同步到 Foundry。有关更多信息，请查看下方的增量部分。

增量

通常，同步会导入目标表中的所有匹配行，而不管同步之间数据是否发生了更改。相比之下，增量同步会保留有关最近一次同步的状态，并且仅从目标摄取新的匹配行。

在从 Slack 摄取大型表时，可以使用增量同步。要使用增量同步，表必须包含一个严格单调递增的列。此外，所读取的表必须包含具有以下数据类型之一的列：

  • DATE
  • TIMESTAMP

数值列类型：

  • int
  • TINYINT
  • SMALLINT
  • INTEGER
  • long
  • BIGINT
  • NUMERIC
  • NUMBER
  • DECIMAL
  • DEC

字符串列类型：

  • string
  • VARCHAR
  • CHAR
  • NVARCHAR
  • NCHAR
  • LONGNVARCHAR
  • LONGVARCHAR

增量同步需要以下配置：

选项是否必需？说明
Column是选择用于增量摄取的列。如果表不包含任何受支持的列类型，下拉菜单将为空。
Initial value是开始同步数据时所依据的值。

同步配置示例：

  • Messages 表包含配置的 bearer token 有权访问的每个频道的所有非线程回复消息。
  • 该表具有一个 CreatedTime 时间戳列，用于表示消息创建的时间戳。
  • 可以将同步配置为仅摄取新消息，方法是将 CreatedTime 列用作 APPEND 事务同步的增量列，并指定一个起始时间戳值作为开始读取消息的起点。
  • 运行初始同步时，将摄取在起始时间戳及之后创建的所有消息。
  • 在下一次同步时，系统将仅接收创建时间在上一次同步获取的最新 CreatedTime 时间戳及之后的记录。
  • 利用类似的增量同步配置，可以通过 CreatedTime 时间戳列，同步来自 MessageReplies 表的线程消息回复。

列选择

选择需要同步至 Foundry 的列。如果未选择任何列，系统将把所有列同步至 Foundry。

所有列名必须以下划线或英文字母开头。列名只能包含下划线、英文字母或数字。

以下示例为有效的列名：

  • _colum_name
  • a_column_name_123

以下示例为无效的列名：

  • 1ColumnName；以数字开头。
  • a-column-name*；包含连字符和星号。
  • å-column-name；包含非英文字符。

行过滤

可以添加过滤器以减少同步的数据行数，或者单独筛选需要同步至 Foundry 的行。要配置行过滤器，必须确保表的预览操作成功执行。在配置行过滤器之前，建议使用界面底部的 Preview of inbound data 部分刷新预览。可以随时刷新预览，以验证所添加过滤器的行为是否符合预期。

过滤器配置由一组条件构成，每一行数据都必须满足这些条件才能被同步至 Foundry。系统会评估每一组条件：若 all 条件均满足，或 at-least one 条件满足，即视为匹配。条件组支持嵌套，从而更精细地控制导入内容，提供极高的灵活性。这相当于标准 SQL 中的 AND 和 OR 语句。

条件本身是对 Slack 表某列中的值与在界面中输入的值进行的简单比较。例如，若设置条件为 CreatedAt 等于 12/04/2021，系统将仅导入 CreatedAt 列的值为 12/04/2021 的行。列的数据类型决定了其所支持的对比操作。

比较操作

过滤器支持以下操作：

  • EQUAL TO
  • GREATER THAN
  • LESS THAN
  • GREATER THAN OR EQUAL TO
  • LESS THAN OR EQUAL TO
  • NOT EQUAL TO
  • IS NULL
  • IS NOT NULL

这些比较操作的行为与标准 SQL 中的对应操作一致。

字符串类型的列还支持 Like 和 Not like 操作符，它们采用标准 SQL-92 语法。
