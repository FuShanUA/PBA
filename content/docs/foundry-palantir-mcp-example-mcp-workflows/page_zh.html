示例MCP工作流

此页面包含MCP工作流的示例，这些是AI代理用于实现目标的一系列关联操作。MCP工作流可以涉及工具使用以及与代码仓库等资源的连接。

代码仓库上下文发现

Palantir MCP能够为您的代理循环提供本体论和代码片段上下文。在提供上下文后，代理将能够执行以下操作：

  • 浏览您的代码库并使用内部Palantir库。
  • 浏览远程Foundry项目并调查项目中的资源。

示例查询：

  • "获取我的Foundry项目的代码/API上下文"
  • "使用Palantir MCP为该项目提供上下文"
  • "告诉我关于我的代码仓库的信息"

Palantir MCP 目前为 React OSDK、Python transform 和 Typescript function 仓库提供上下文。

数据集搜索和创建

Palantir MCP 能够查看数据集列表，对它们运行 SQL 查询，并使用虚拟数据创建数据集。

请注意，Palantir MCP 不允许覆盖现有的数据集，只能创建新的数据集。这可以保护您的现有数据。

示例查询：

  • 探索：“我有哪些 Foundry 数据集的访问权限？”
  • 调查：“显示我 /Path/to/dataset/Flight_routes 中延迟最严重的航班。”
  • 调查：“这个数据集是干净的吗？ri.foundry.main.dataset.033384ec-73de-41c1-bebe-45178cfc468b”
  • 创建：“用虚拟数据为我创建一个 Foundry 数据集。它应该包含列...，并且至少有 50 行。”
  • 创建：“给我创建一个Foundry数据集，包含示例数据。使用脚本生成10,000行数据。”

本体论探索

Palantir MCP 提供了用于搜索本体论的工具。这些工具允许代理在您的本体论中搜索对象类型、操作类型和函数。

示例查询：

  • "帮我找到一个对象类型 {description}。"
  • "帮我查找或创建一个对象类型 {description}。"
  • "有哪些函数与此对象交互？"
  • "帮我找到一些函数 {description}。"

本体论修改与SDK生成

Palantir MCP 可以创建提案，修改本体论，并为开发人员控制台 OSDK 重新生成，以便在代码中立即使用。

请注意，对本体论的所有修改都必须经过 提案 审查流程。这意味着在 MCP 对本体论进行任何持久性更改之前，需要人工审查和批准。

此流程由与本体论相关的查询触发，例如以下内容：

  • "创建一个对象类型 {description}"
  • "创建一个关联类型 {description}"
  • "创建一个对象类型 {description} 并将其与 {description} 关联"
  • "在 {object 1} 和 {object 2} 之间创建一对一的关联"
  • "在 {object 1} 和 {object 2} 之间创建多对多的关联"
