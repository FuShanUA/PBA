2b.4 如何从 Code Workspaces 发布模型


既然我们已经创建好了模型，就可以将其发布到 Foundry 中，以便与生产环境的应用集成。发布模型前，我们需要在 Foundry 中创建模型资源用于保存该模型，随后将其封装在模型适配器中，这样 Foundry 才能知道如何与你的模型进行交互。


操作：在 Models 标签页中，选择 Add model > Create new model 并将其命名为 linear_regression_model。你可以将该模型保存到之前创建的 models 文件夹中，然后选择 Create 以创建该资源。




创建好模型资源后，Foundry 会自动为你新建一个 Python 文件，供你实现模型适配器。模型适配器为 Foundry 中的所有模型提供了标准接口。该标准接口确保所有模型都能直接用于生产环境应用，因为 Foundry 会负责处理加载模型及其 Python 依赖项、暴露其 API 以及与你的模型进行交互所需的基础设施工作。


Code Workspaces 中的模型适配器必须定义在独立的 Python（.py）文件中，并导入到笔记本内。


创建模型适配器时，你需要实现以下四个函数：


    

  1. 模型保存与加载：为了复用你的模型，你必须定义模型的保存和加载方式。Palantir 提供了多种默认方法进行序列化（保存），在更复杂的场景下，你可以实现自定义序列化逻辑。
    2. 

  2. 

api：定义模型的 API，并告知 Foundry 您的模型需要何种类型的输入数据。
    3. 

  3. predict：由 Foundry 调用，用于向您的模型提供数据。您可以在此处将输入数据传递给模型并生成推理（预测）。
    4. 

```python
import palantir_models as pm
from palantir_models.serializers import DillSerializer


class LinearRegressionModelAdapter(pm.ModelAdapter):

    @pm.auto_serialize(
        model=DillSerializer()
    )
    def __init__(self, model):
        self.model = model

    @classmethod
    def api(cls):
        columns = [
            ('median_income', float),
            ('housing_median_age', float),
            ('total_rooms', float),
        ]
        return {"df_in": pm.Pandas(columns)}, \
               {"df_out": pm.Pandas(columns + [('prediction', float)])}

    def predict(self, df_in):
        df_in['prediction'] = self.model.predict(
            df_in[['median_income', 'housing_median_age', 'total_rooms']]
        )
        return df_in
```
