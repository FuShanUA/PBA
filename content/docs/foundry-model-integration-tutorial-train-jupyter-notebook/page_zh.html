2b. 教程：在 Jupyter® 笔记本中训练模型

在开始本教程的这一步之前，您应该已经完成了 建模项目设置。在本教程中，您可以选择在 Jupyter® 笔记本或 代码仓库 中训练模型。Jupyter® 笔记本适用于快速和迭代的模型开发，而代码仓库则适用于生产级的数据和模型管道。

在本教程的这一部分，我们将使用 代码工作区 在 Jupyter® 笔记本中训练一个模型。此步骤将涵盖：

  1. 为模型训练创建 Jupyter® 代码工作区
  2. 对特征数据进行分割以用于测试和训练
  3. 在代码工作区中训练模型
  4. 从代码工作区发布模型
  5. 查看模型并将其提交到建模目标

2b.1 如何为模型训练创建一个笔记本

Foundry中的Code Workspaces应用程序是一个基于网络的开发环境，它为数据分析和模型开发提供第三方IDE。您可以在Foundry内的Jupyter®笔记本中直接发布模型，这些模型可用于下游应用。

Code Workspaces通过在使用工作区时持续提供可用的计算资源，来提供交互式开发环境。Code Workspaces使您能够配置Python环境、转换数据、绘制图表并训练模型，而无需等待计算资源或打包Python环境。

操作：在本教程上一步中创建的code文件夹中，选择+ 新建 > Jupyter代码工作区。您的代码工作区应与您正在训练的模型相关命名。在此情况下，将仓库命名为median_house_price_model_notebook. 选择继续以使用默认的计算资源和仓库配置，然后使用创建来创建并启动工作区。

2b.3 如何在代码工作区中训练模型

Foundry中的模型由两个组件组成：

  • 模型工件：在模型训练作业中生成的模型文件。
  • 模型适配器：一个Python类，描述Foundry应如何与模型工件交互以执行推理。

模型依赖项

模型训练几乎总是需要添加包含模型训练、序列化、推理或评估逻辑的Python依赖项。Foundry通过conda和PyPI（pip）支持添加依赖项规范。这些依赖项规范会创建一个Python环境，可用于训练模型。

2b.4 如何从代码工作区发布模型

现在我们已经创建了一个模型，可以将其发布到Foundry，以便与我们的生产应用程序集成。要发布模型，我们需要在Foundry中创建一个模型资源，将模型保存到该资源中，然后使用模型适配器包装模型，这样Foundry就知道如何与您的模型进行交互。

操作：在模型选项卡中，选择添加模型 > 创建新模型，并将其命名为linear_regression_model. 您可以将模型保存到之前创建的models文件夹中，然后选择创建以创建资源。

您已经创建了模型资源后，Foundry会自动为您创建一个新的Python文件，用于实现模型适配器。 模型适配器 为Foundry中的所有模型提供标准接口。标准接口确保所有模型可以立即在生产应用程序中使用，因为Foundry将处理加载模型、其Python依赖项、公开其API以及与您的模型进行交互的基础设施。

Code Workspaces中的模型适配器必须定义在一个单独的Python（.py）文件中，并导入到笔记本中。

要创建模型适配器，您需要实现四个函数：

  1. 模型保存和加载：为了重复使用您的模型，您必须定义如何保存和加载您的模型。Palantir提供了许多默认的序列化方法（保存），在更复杂的情况下，您可以实现自定义序列化逻辑。
  2. api：定义您模型的API，并告诉Foundry您的模型需要哪种类型的数据输入。
  3. predict：由Foundry调用以向您的模型提供数据。这是您可以将输入数据传递给模型并生成推理（预测）的地方。
Copied!
1import palantir_models as pm
2from palantir_models.serializers import DillSerializer
3
4
5class LinearRegressionModelAdapter(pm.ModelAdapter):
6
7    @pm.auto_serialize(
8        model=DillSerializer()
9    )
10    def __init__(self, model):
11        self.model = model
12
13    @classmethod
14    def api(cls):
15        columns = [
16            ('median_income', float),
17            ('housing_median_age', float),
18            ('total_rooms', float),
19        ]
20        return {"df_in": pm.Pandas(columns)}, \
21               {"df_out": pm.Pandas(columns + [('prediction', float)])}
22
23    def predict(self, df_in):
24        df_in['prediction'] = self.model.predict(
25            df_in[['median_income', 'housing_median_age', 'total_rooms']]
26        )
27        return df_in
