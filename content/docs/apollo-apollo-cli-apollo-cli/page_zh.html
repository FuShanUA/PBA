apollo-cli

用于与 Apollo 交互的命令行工具（CLI）

Apollo CLI 提供了一种灵活的方式与 Apollo 进行交互。它可用于手动发布新的产品版本（Product Releases），也可作为用户界面（UI）的替代方案，用于检索 Apollo 的元数据（例如当前由 Apollo 托管的环境（Environments）和产品（Products））。

该 CLI 支持 Linux、macOS（Darwin）和 Windows 系统。

安装

如需安装 Apollo CLI，请参阅Apollo CLI 入门指南中提供的说明。

标志（Flags）

标志用途
--apollo-client-id用于生成 Bearer Token 的客户端 ID
--apollo-client-secret用于生成 Bearer Token 的客户端密钥
--apollo-token用于经身份验证端点的 Bearer Token
--apollo-token-provider指定如何提供用于经身份验证的 Apollo 端点调用的 Bearer Token。有效值为 "auto"、"static"、"service-user" 或 "sso"（默认为 "auto"）。若指定 "auto"，则根据配置自动选择模式：若设置了 "apollo-token"，则使用 "static"；若同时设置了 "apollo-client-id" 和 "apollo-client-secret"，则使用 "service-user"；否则使用 "sso"（即交互式浏览器登录）。若同时设置了 Bearer Token 及（客户端 ID 或客户端密钥）中的任一者，则会报错。若指定 "static"，则使用 "apollo-token" 提供的 Token；若指定 "service-user"，则使用 "apollo-client-id" 和 "apollo-client-secret" 向 Multipass 请求生成 Token；若指定 "sso"，则通过 Multipass 执行交互式基于浏览器的单点登录（SSO）（支持 YubiKey 等硬件密钥）；生成的 Token 将按配置文件缓存，并在后台静默刷新。当某条命令首次需要 Token 且当前无有效缓存 Token 时，将自动触发登录流程。
--apollo-url用于推导 API 端点的 Apollo 基础 URL
--debug启用调试级别日志记录
-e, --environment用于环境作用域（environment-scoped）命令的环境 ID（含后缀）
-h, --helpapollo-cli 的帮助信息
--http-timeout所有 apollo 请求的超时时间（单位：分钟）
-k, --insecure-skip-verify跳过服务器证书验证
-o, --output输出格式（json、yaml、pretty）
--profile从配置文件中使用特定配置文件（profile）
--quiet不向 stderr 输出日志信息
--space-id用于某些空间作用域（space-scoped）命令的空间 ID
