构建转换

Palantir 平台针对 Visual Studio Code 的扩展插件允许您直接从编辑器中执行构建。在完成对转换逻辑的预览和调试后，构建是在完整生产数据集上运行代码前的最后一道工序。构建任务在 Palantir 平台上执行，您无需离开 VS Code 环境即可监控进度、查看日志并实施管理。

构建与预览

  • 使用预览：在开发和测试阶段，利用样本数据或特定数据子集验证转换逻辑。
  • 使用构建：准备在完整生产数据集上运行转换，并将变更提交至数据管道。

如何发起构建

您可以通过以下三种方式，在本地 VS Code 环境或平台内置的 VS Code 工作区中启动构建：

方式一：命令面板

从命令面板中选择 Palantir: Build on Foundry 选项。

方式二：Palantir 侧边栏

在 Palantir 侧边栏中选择 Build 图标。

方式三：构建面板

打开 Build 面板，选择 Build 选项。

了解构建流程

对于处于打开状态的 Palantir 代码库，您可以对其中的任意文件发起构建。

在构建文件前，您必须将所有本地变更推送至远程代码库。

本地分支与远程分支同步后，构建流程将执行必要的检查步骤，并在 Palantir 平台上对完整数据集运行构建。

导航至 Builds 面板，查看构建状态、调取日志并监控输出数据集。
