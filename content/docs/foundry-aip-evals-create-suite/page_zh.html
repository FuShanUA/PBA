创建评估套件

评估套件是一组用于基准测试目标函数性能的测试用例和评估函数。运行评估套件将针对每个测试用例执行目标函数，并使用与套件相关联的评估方法。

您可以为以下目标函数创建评估套件：AIP Logic 函数、AIP Chatbot 函数和代码编写的函数：

AIP Logic 函数： 要在 AIP Logic 中开始使用评估套件，请参考 Logic 函数的评估套件。

AIP Chatbot 函数： 要测试一个 AIP Chatbot 函数，您需要先将聊天机器人发布为一个函数。然后，您可以从左侧边栏的 评估 选项卡中创建评估套件。由于边栏的相似性，您可以参考 Logic 函数的评估套件 部分以获取更多信息。

代码编写的函数：您可以直接从 已发布 选项卡中创建和打开针对在代码仓库中编写的函数的评估套件，通过导航到 代码仓库 > 代码 > 函数 > 已发布。

附加目标函数

您可以添加额外的目标函数，使用相同的评估套件测试多个函数。这允许您同时对不同的已发布的Foundry函数或Logic函数运行评估。这对于比较不同实现的性能或在相同测试用例上评估相似函数非常有用。

要添加目标函数，请在AIP Evals中打开您的评估套件，然后选择 添加目标函数。每个目标函数可以有不同的输入/输出签名，并且评估者会根据目标进行配置。

当您配置了多个目标时，可以在 运行您的评估套件 时选择要包含的目标。

添加测试用例

您可以通过手动定义单个测试用例、使用一个对象集来生成多个测试用例，或在同一套件中结合这两种方法来为评估套件创建测试用例。这种灵活性使您能够在利用现有对象集的同时，根据需要添加特定的手动测试用例。

您可以通过选择 编辑测试用例参数 来编辑评估套件的列。然后，您可以添加、删除或重新排序测试用例列及其相应的类型。

手动测试用例

要手动定义一个测试用例，请在评估套件视图的左下角选择添加测试用例。为每个测试用例命名，然后在相应的列中定义输入及其预期值。您可以选择测试用例名称旁边的紫色AIP星标图标以生成建议的名称。

对象集测试用例

你也可以从对象集中添加测试用例，每个测试用例将由所选对象集中的一个对象表示。要添加基于对象集的测试用例，请选择添加对象集，然后在对象集选择对话框中选择要使用的对象集和对象属性。

基于对象集的测试用例特别适用于使用真实数据进行大规模测试，确保你的函数能够在你实际数据的代表性样本上正常运行。你可以将多个对象集添加到同一个评估套件中，并与手动测试用例结合，以创建全面的测试覆盖率。

可以通过选择编辑对象集配置图标，在配置对话框中编辑支撑对象集。也可以通过对象集的表头列单独编辑各个属性。

对象集可以配置为向评估套件的列提供不同类型的数据。包括以下内容：

  • 支撑对象集的对象
  • 支撑对象集的对象属性
  • 与支撑对象集关联的对象或对象集合
  • 关联对象或关联对象集合的属性（仅在对象存储v2中可用）
  • 应用于支撑对象集每一行的静态值

评估器

评估器是一种用于将测试函数的输出与预期输出进行比较的方法。评估器可能返回简单的 true/false 结果，也可以生成数值，例如语义距离。没有评估器的评估套件可用于在多种场景下执行函数并手动审查每个输出。然而，评估器使大规模测量和客观化运行结果成为可能，因为它们可以产生可比较的性能指标。

AIP Evals 提供了一些内置的评估器，将在下一节中描述。您也可以定义自定义的评估函数，以根据特定标准衡量性能。

添加评估器

要添加评估器，请在测试用例表格的顶部选择+ 添加。这将打开一个选择面板，您可以在其中从内置评估器、市场部署的评估器或自定义评估器列表中进行选择。

一旦您选择了评估器并选择+ 添加，该评估器将会被添加到您的测试用例表格中。然后您可以配置评估器，将函数输出映射到实际值列，并将预期值映射到测试用例表格中的预期值列。

在配置评估器时，您可以为每个指标定义一个目标。对于布尔指标，指定该指标应为 true 还是 false。对于数值指标，设置优化方向；指定最大化（数值越高越好）或最小化（数值越低越好）目标。您还可以定义一个阈值：

  • 对于最大化目标，设置一个最低阈值（例如，“得分必须至少为10”）。
  • 为最小化目标，设置最大阈值（例如，“分数必须不超过 0.05”）。

在测试用例迭代中，如果度量值满足配置的目标，则视为通过。对于布尔型度量值，这表示该度量值是 true 还是 false。对于数值型度量值，这表示该度量值是否高于或低于阈值（如果已定义的话）。如果所有度量值都满足配置的目标，则整个测试用例迭代视为通过。如果一个测试用例包含多个迭代，那么只有当所有迭代都通过时，该测试用例才视为通过。

内置评估器

内置评估函数的示例包括：

  • 精确布尔匹配： 检查实际布尔值是否与预期布尔值完全相等。
  • 精确布尔数组匹配： 检查两个布尔数组是否包含相同的元素。默认情况下比较是顺序相关的，但这是可配置的。
  • 精确字符串匹配： 检查实际字符串是否与预期字符串完全匹配。匹配区分大小写，并默认包括空格，但这是可配置的。
  • 精确字符串数组匹配： 检查两个字符串数组是否包含相同的元素。比较是顺序相关的、区分大小写的，并默认包括空格，但这是可配置的。
  • 正则表达式匹配： 检查实际字符串是否与预期的正则表达式匹配。
  • Levenshtein 距离： 一种用于测量两个序列之间差异的字符串度量方法。计算将一个单词转换为另一个单词所需的最少单字符编辑次数（插入、删除或替换）。
  • 字符串长度： 检查实际字符串的长度是否在预期范围内。
  • 关键字检查器： 检查实际文本中是否包含特定的关键字。
  • 精确对象匹配： 检查实际对象是否与预期对象完全相等。
  • 对象集包含： 检查实际对象是否与目标对象集中的某个对象完全相等。
  • 对象集大小范围： 检查提供的对象集大小是否在预期范围内。
  • 整数范围： 检查实际值是否在预期值的范围内。仅支持整数。
  • 精确数值匹配： 检查两个数字是否完全相等。支持整型、长整型、浮点型、双精度型和短整型。
  • 精确数值数组匹配： 检查两个数值数组是否包含相同的元素。支持整型、长整型、浮点型、双精度型和短整型。默认情况下比较是按顺序进行的，但也可以进行配置。
  • 浮点数范围： 检查实际值是否在预期值的范围内。所有数值类型都支持作为参数。
  • 时间范围： 检查实际值是否在预期值的范围内。仅支持 Date 和 Timestamp 值。
  • 通用精确匹配： 检查实际值是否与预期值完全相同。数值类型在比较时会进行强制转换（例如，整数可以与相同值的双精度数匹配），日期和时间戳类型也是如此。对象和对象集通过引用进行比较，结构体和映射通过无序的键值对进行比较，列表通过有序元素进行比较，模型通过标识符和参数进行比较。如果您数据类型有特定的评估器，请使用它，因为它们提供更细粒度的比较选项和更好的类型安全性。
  • LLM作为评判者: 使用LLM来评估用户定义的条件是否适用于给定的值。当条件满足时返回true，否则返回false。实际值不能是引用类型（对象定位器、对象RID、对象集或模型）。需要一个清晰、可验证的断言作为条件参数，并且需要可用的模型来进行评估。

市场部署的评估函数

选择市场部署的函数将打开一个设置向导，引导您完成安装过程。以下是市场函数的一个示例：

  • ROUGE分数: ROUGE（用于摘要评估的召回导向替代）评分是一组用于评估机器生成文本质量的指标，特别是在摘要和翻译等任务中。较高的ROUGE分数表示与参考文本更接近，表明机器生成内容的表现更好。

自定义评估函数

自定义评估函数允许您选择之前发布的函数。这些可以是 对象上的函数，这些函数是在代码仓库中编写的，或者是其他 AIP Logic 函数。自定义评估函数必须至少返回一个布尔类型或数值类型作为指标。它们也可以返回字符串值，在调试视图中作为调试输出用于诊断目的。一个评估函数还可以通过返回由布尔类型或数值类型组成的 struct 来返回多个指标。
