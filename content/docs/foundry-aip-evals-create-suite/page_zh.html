创建评估套件

评估套件是一组测试用例和评估函数，用于基准测试目标函数的性能。运行评估套件将针对每个测试用例执行目标函数，并使用与套件相关的评估方法。

您可以为以下目标函数创建评估套件：AIP Logic 函数、AIP Chatbot 函数和代码编写的函数：

AIP Logic 函数：要开始在 AIP Logic 中使用评估套件，请参考 Logic 函数的评估套件。

AIP Chatbot 函数：要测试一个 AIP Chatbot 函数，您需要先将聊天机器人发布为一个函数。然后，您可以从左侧边栏的 评估 选项卡中创建评估套件。如需更多信息，您可以按照 用于 Logic 函数的评估套件 部分，由于侧边栏的相似性。

代码编写的函数：您可以直接从 已发布 选项卡中创建和打开用于代码仓库中编写的函数的评估套件，通过导航至 代码仓库 > 代码 > 函数 > 已发布。

附加目标函数

您可以添加额外的目标函数，以便使用同一个评估套件测试多个函数。这使您可以同时对不同的已发布 Foundry 函数或 Logic 函数运行评估。这对于在不同实现之间进行性能比较或在相同测试案例上评估相似函数非常有用。

要添加目标函数，请在 AIP Evals 中打开您的评估套件并选择 添加目标函数 每个目标函数可以有不同的输入/输出签名，评估器是根据目标进行配置的。

当您配置了多个目标时，您可以选择在运行您的评估套件时包含哪些目标。

添加测试用例

您可以通过手动定义单个测试用例、使用对象集生成多个测试用例，或在同一套件中结合这两种方法来为评估套件创建测试用例。这种灵活性使您能够利用现有的对象集，同时根据需要添加特定的手动测试用例。

您可以通过选择编辑测试用例参数来编辑评估套件列。然后，您可以添加、删除或重新排序测试用例列及其相应的类型。

手动测试用例

要手动定义一个测试用例，请在评估套件视图的左下角选择 添加测试用例。为每个测试用例命名，然后在相应的列中定义输入及其预期值。您可以选择测试用例名称旁边的紫色 AIP 星标图标以生成建议的名称。

对象集测试用例

您也可以从一个对象集中添加测试用例，其中每个测试用例将由所选对象集中的一个对象表示。要添加对象集测试用例，请选择 添加对象集，并在对象集选择对话框中选择对象集和您希望在对象集中使用的对象属性。

基于真实数据的大规模测试特别有用，确保您的函数在实际数据的代表性样本上都能正常工作。您可以将多个对象集添加到同一个评估套件中，并与手动测试用例结合，以创建全面的测试覆盖范围。

可以通过选择 编辑对象集配置 图标，在配置对话框中编辑支撑对象集。也可以通过对象集的表头列单独编辑属性。

对象集可以配置为向评估套件列提供不同类型的数据。这些包括以下内容：

  • 支撑对象集的对象
  • 支撑对象集的对象属性
  • 与支撑对象集关联的对象或对象集
  • 关联对象或关联对象集的属性（仅在对象存储v2中可用）
  • 应用于支撑对象集每一行的静态值

评估器

评估器是一种用于将测试函数的输出与预期输出进行比较的方法。评估器可能返回一个简单的 true / false 结果，也可以生成数值，例如语义距离。没有评估器的评估套件可用于在多种场景下执行函数并手动审查每个输出。然而，评估器使大规模测量和客观化运行结果成为可能，因为它们生成可比较的性能指标。

AIP Evals 提供了一些内置的评估器，将在下一部分中进行描述。您还可以定义自定义的评估函数，以根据特定标准衡量性能。

添加评估器

要添加评估器，请在测试用例表的顶部选择+ 添加。这将打开一个选择面板，您可以在其中从内置评估器、市场部署的评估器或自定义评估器列表中进行选择。

一旦您选择了评估器并选择+ 添加，评估器将被添加到您的测试用例表中。然后，您可以通过将函数输出映射到实际值列，并将预期值映射到预期值列来配置评估器。

在配置评估器时，您可以为每个指标定义一个目标。对于布尔型指标，请指定该指标应为 true 或 false。对于数值型指标，设置优化方向；指定最大化（值越大越好）或最小化（值越小越好）的目标。您还可以定义一个阈值：

  • 对于最大化目标，设置最小阈值（例如，“分数必须至少为10”）。
  • 对于最小化目标，设置最大阈值（例如，“分数必须最多为0.05”）。

在测试用例迭代中，如果指标满足配置的目标，则视为通过。对于布尔型指标，这取决于该指标是 true 还是 false。对于数值型指标，这取决于该指标是否高于或低于阈值（如果已定义）。整个测试用例迭代被视为通过，如果所有指标都满足配置的目标。如果多个迭代的测试用例的所有迭代都通过，则该测试用例被视为通过。

内置评估器

内置评估函数的示例包括：

  • 精确布尔匹配（Exact boolean match）: 检查实际布尔值是否与预期布尔值完全相等。
  • 精确布尔数组匹配（Exact Boolean array match）: 检查两个布尔数组是否包含相同元素。默认情况下比较是按顺序的，但可以进行配置。
  • 精确字符串匹配（Exact string match）: 检查实际字符串是否与预期字符串完全匹配。默认情况下匹配区分大小写并包含空格，但可以进行配置。
  • 精确字符串数组匹配（Exact string array match）: 检查两个字符串数组是否包含相同元素。默认情况下比较是按顺序、区分大小写并包含空格的，但可以进行配置。
  • 正则匹配（Regex match）: 检查实际字符串是否与预期的正则表达式匹配。
  • Levenshtein 距离（Levenshtein distance）: 一种用于测量两个序列之间差异的字符串度量方法。计算将一个单词转换为另一个单词所需的最少单字符编辑次数（插入、删除或替换）。
  • 字符串长度（String length）: 检查实际字符串的长度是否在预期范围内。
  • 关键字检查器： 检查实际文本中是否包含特定关键字。
  • 对象精确匹配： 检查实际对象是否与预期对象完全相等。
  • 对象集合包含： 检查实际对象是否与目标对象集合中的某个对象完全相等。
  • 对象集合大小范围： 检查提供的对象集合的大小是否在预期范围内。
  • 整数范围： 检查实际值是否在预期值范围内。仅支持整数。
  • 精确数值匹配： 检查两个数字是否完全相等。支持整数、长整型、浮点型、双精度型和短整型。
  • 精确数值数组匹配： 检查两个数值数组是否包含相同的元素。支持整数、长整型、浮点型、双精度型和短整型。默认情况下比较是顺序相关的，但可以进行配置。
  • 浮点型范围： 检查实际值是否在预期值范围内。所有数值类型都作为参数支持。
  • 时间范围： 检查实际值是否在预期值范围内。仅支持 Date 和 Timestamp 值。
  • 通用精确匹配：检查实际值是否与预期值完全相同。数值类型在比较时会进行强制转换（例如，整数可以与相同值的双精度浮点数匹配），日期和时间戳类型也是如此。对象和对象集合通过引用进行比较，结构和映射通过无序的键值对进行比较，列表通过有序元素进行比较，模型则通过标识符和参数进行比较。如果您的数据类型存在特定类型的评估器，请使用它，因为它们提供了更细粒度的比较选项和更好的类型安全性。
  • 大语言模型作为评判者：使用大语言模型来评估用户定义的条件是否适用于给定的值。当条件满足时返回 true，否则返回 false。实际值可以是除引用类型（对象定位器、对象RID、对象集合或模型）以外的任何类型。需要一个以清晰可验证的断言形式编写的条件参数以及一个可用于评估的可用模型。

    • 市场部署评估函数

    选择一个市场部署的函数将打开一个设置向导，引导您完成安装过程。下面是市场函数的一个示例：

    • ROUGE 分数： ROUGE（用于摘要评估的召回导向研究者）评分是一组用于评估机器生成文本质量的指标，特别是在摘要和翻译等任务中。较高的 ROUGE 分数表示与参考文本更接近，表明机器生成内容的表现更好。

    自定义评估函数

    自定义评估函数允许您选择之前发布的函数。这些可以是 对象上的函数 编写在代码仓库或其他AIP逻辑函数中。自定义评估函数必须返回至少一个布尔或数值类型作为指标。它们也可以返回字符串值，这些值在调试视图中作为调试输出用于诊断目的。一个评估函数还可以通过返回一个struct来返回多个指标，该struct由布尔或数值类型组成。
