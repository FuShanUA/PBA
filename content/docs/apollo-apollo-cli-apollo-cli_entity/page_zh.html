从父命令继承的标志

标志用途
--apollo-client-id用于生成Bearer Token的客户端ID
--apollo-client-secret用于生成Bearer Token的客户端密钥
--apollo-token用于认证端点的Bearer Token
--apollo-token-provider指定用于认证Apollo端点调用的Bearer Token的提供方式。有效值为 "auto"、"static"、"service-user" 或 "sso"（默认为 "auto"）。如果指定 "auto"，则根据配置选择模式：如果设置了 "apollo-token"，则使用 "static"；如果同时设置了 "apollo-client-id" 和 "apollo-client-secret"，则使用 "service-user"；否则使用 "sso"（交互式浏览器登录）。如果同时设置了Bearer Token和（客户端ID或客户端密钥），将报错。如果指定 "static"，则使用 "apollo-token" 提供的令牌。如果指定 "service-user"，则使用 "apollo-client-id" 和 "apollo-client-secret" 从Multipass生成令牌。如果指定 "sso"，则使用针对Multipass的交互式基于浏览器的SSO登录（支持硬件密钥如YubiKey）；生成的令牌将按配置文件缓存并静默刷新。当首次需要令牌且没有有效的缓存令牌时，将自动触发登录流程。
--apollo-urlApollo的基本URL，用于推导API端点
--debug启用调试级别日志记录
-e, --environment用于环境作用域命令的环境ID（包括后缀）
--http-timeout所有apollo请求的超时时间（分钟）
-k, --insecure-skip-verify跳过服务器证书验证
-o, --output输出格式（json,yaml,pretty）
--profile从配置文件中使用特定的配置文件
--quiet不将日志输出到标准错误
--space-id用于某些空间作用域命令的空间ID
