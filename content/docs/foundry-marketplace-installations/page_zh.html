Foundry Marketplace 中的安装

启动产品安装后，您可以监控安装任务。下方的截图展示了 Marketplace 的安装任务视图。

在右上角选择 查看安装，即可查看已完成的安装。在此处，您可以导航至已安装的资源并开始使用。右侧面板中提供了资源所保存的项目或文件夹位置的链接。

下方的截图展示了 Marketplace 中已完成的产品安装。

安装设置

安装完成后，您可以在 设置 面板中为该安装配置多项选项。

  • 发布渠道： 您可以设置安装所要追踪的发布渠道。带有该发布渠道标记的新版本将显示为可供升级的新版本。更多信息请参见升级。
  • 自动升级： 您可以为新产品版本启用自动升级；更多信息请参见自动升级。
  • 维护窗口： 此设置允许您控制升级的时机；例如，您可以设置在升级可用时立即执行，或者安排在特定时间段内执行（即维护窗口）。
  • 锁定或解锁安装： 锁定安装可防止对下游内容的编辑。若要确保安全升级，请保持安装处于锁定状态；您可以解锁安装，以便对已安装的内容进行分支操作。安装的默认锁定/解锁行为将取决于产品构建者指定的安装模式（生产模式 或 引导模式）。

下方的截图展示了 Marketplace 安装视图的 设置 面板。在此处，您可以设置自动升级配置，并对安装进行锁定或解锁以允许编辑已安装的内容。

已知限制

对于某些资源类型，解锁安装可能仍无法编辑已安装的资源。请注意，代码仓库 必须与源代码一起打包，才能在安装中进行编辑。

升级

自动升级 [Beta]

Beta

Marketplace 中的自动升级功能目前处于 开发生命周期 的 beta 阶段，您的租户可能尚未提供该功能。在活跃开发期间，功能可能会发生变化。

生产模式 和 引导模式 产品默认禁用自动升级。安装 生产模式 产品时，系统会提示您在 新版本 安装步骤中选择发布渠道和维护窗口。您随时可以在 安装设置 中更改这些设置。自动升级设置包括：

  • 您希望接收自动升级的 维护窗口。如果希望升级一可用就立即执行，您可以选择“始终开放”。请注意，升级会导致已安装资源停机，因此建议添加维护窗口以避免停机。
  • 您的安装应追踪的 发布渠道。在您配置的任何维护窗口内，只要升级不需要手动操作，您的安装就会自动升级到标记为该发布渠道的版本。

发布渠道是分层级的，而非互斥关系。具体取决于所追踪的渠道：

  • 发布： 安装将接收标记为 发布、测试 或 稳定 的版本。
  • 测试： 安装将接收标记为 测试 和 稳定 的版本。
  • 稳定： 安装将接收标记为 稳定 的版本。

如果新产品版本包含必须映射的新输入，升级仍需要手动操作。遇到这种情况时，系统将引导您完成与 手动升级 相同的手动配置工作流。

手动升级

若未启用自动升级，当有新版本可用时，您的安装页面顶部会显示一条包含新版本信息的横幅。您可以选择查看可能需要手动配置的变更内容，如下所示。

选择查看变更以了解新版本的更多信息。如果您对已安装的内容进行过任何编辑，这些编辑将被新版本中的内容覆盖。下图展示了一次示例升级的变更列表。

如果新产品版本需要手动配置（例如，需要映射新的输入项），您必须先完成这些步骤才能进行升级。升级表单将引导您完成所需的手动步骤，如下图所示。

降级

若要降级到先前版本或升级到产品的特定版本，请先点击安装页面右上角的省略号。接着，选择升级或降级至版本，如下图所示。

选择此选项会打开一个对话框，您可以在其中选择所需的确切版本。选择完毕后，点击创建草稿以启动升级或降级流程。

删除安装

若要删除安装及其所有资源，请先点击安装页面右上角的省略号 (...)。接着，选择永久删除安装，如下图所示。

选择永久删除安装后，系统会预览如果继续操作将被永久删除的资源。

接下来，在确认文本框中输入 delete installation 并选择删除以启动卸载流程。

警告：删除安装是不可逆的。卸载将永久删除所有安装版本中的资源以及安装本身。如果安装项目或文件夹仅包含属于选定删除安装的资源，卸载还将永久删除该安装项目或文件夹。否则，将跳对该项目或文件夹的删除。

如果卸载成功，您将被重定向至安装页面。如果卸载失败，您将收到错误消息，如下所示：

错误消息将列出未能删除的资源及失败原因，以及成功删除的资源。您可以选择先解决失败资源的问题再重试卸载，也可以勾选确认失败的内容将保留的复选框，然后选择强制删除按钮。这将忽略失败的资源并直接删除安装。只要该项目或文件夹不包含不属于该安装的其他资源，安装项目或文件夹也将被一并删除。
