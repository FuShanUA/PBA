Pilot

Beta

Pilot is in the beta phase of development and may not be available on your enrollment. Functionality may change during active development. To enable Pilot, contact your platform administrator to modify application access in Control Panel.

Pilot is an AI-powered application builder in Foundry that creates applications from natural language prompts. You can describe the application you want to build, and Pilot automatically generates the ontology entities, design specification, frontend code, and seed data. Applications built with Pilot use the OSDK and React, and are deployed through Developer Console as production-grade hosted applications.

Key capabilities

Pilot manages the full application lifecycle with specialized AI agents that can perform the following operations:

  • 本体论生成: Pilot 分析您的提示并创建应用程序所需的对象类型、操作类型、链接类型、属性和关系。
  • 设计规范: 一个设计代理会生成涵盖配色方案、排版、布局和交互模式的设计规范。
  • 前端生成: 一个应用构建代理使用 OSDK 钩子实现 React 应用，并在 Pilot 工作区中提供实时预览。
  • 种子数据生成: Pilot 在隔离的容器中生成示例数据，因此 AI 代理在构建过程中永远不会访问真实的企业数据。
  • 引导式部署: 逐步部署面板将引导您完成本体论提升、开发者控制台配置、CI 检查和发布。
  • 多种 AI 模型: 根据您的注册配置，可以选择 Claude、GPT 或 Gemini 模型系列。
  • 上下文附件: 提供现有的本体论实体、文档或图像，以指导 Pilot 如何构建您的应用程序。

How Pilot works

Pilot 从提示到生产遵循端到端的工作流程。

  1. 提示驱动构建：描述您想要构建的应用程序。使用您的描述，Pilot 会创建一个隔离的容器，并生成构建您的应用程序所需的本体论、设计、代码和种子数据。
  2. 本体论对齐： Pilot 生成或重用对象类型和操作。这些本体论实体通过 全局分支 和提案提升到主本体论中。
  3. 开发者控制台配置： Pilot 会创建一个与您的项目相关的开发者控制台应用程序，请求子域名，并设置应用程序限制以包含所需的本体论实体。
  4. 持续集成和部署： Pilot 会生成部署文件和 CI 配置。CI 检查必须通过后，您才能标记并发布版本。
  5. 托管生产应用程序： 应用程序在注册的子域名上提供。用户通过 OAuth 授权，可以立即使用真实数据的应用程序。

前提条件

确保您满足以下条件以使用 Pilot：

  • You are on a Foundry enrollment with Pilot enabled.
  • You have existing ontology entities or a project with ontology write permissions.
  • You have the Editor role or higher on the project, enabling deployment.
