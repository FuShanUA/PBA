配置的对象视图概述

配置的对象视图是可自定义、可重用的对象数据表示。您可以通过 Workshop 模块 为对象类型构建完整的 完整 和 面板 对象视图，从而在平台上以灵活的方式表示对象。

默认情况下，Foundry 会为所有对象类型创建一个 标准对象视图。当您创建一个配置的对象视图时，它将成为用户的默认视图，尽管他们可以通过与对象视图一起提供的切换按钮切换回标准对象视图。此外，用户可以悬停在使用自己自定义标题栏的 Palantir 应用程序（如 Gaia 或 Vertex）中显示的对象视图的省略号抽屉图标上，以在标准视图和配置视图之间切换。

目前 Workshop 尚未提供在标准和配置对象视图之间切换的功能。

默认配置

为每种对象类型自动创建默认配置的对象视图。默认的完整对象视图包含一组突出属性，或者如果没有突出属性，则包含所有非隐藏属性，以及该对象的链接列表。默认面板包含相同的属性列表。默认视图会动态更新以反映对对象类型的更改，例如新属性或属性重命名，但一旦对象视图被编辑，它将成为用户管理的视图，后续的所有更新都必须手动进行。

权限

编辑对象类型对象视图所需的权限取决于该对象类型是否使用 本体论角色：

  • 如果对象类型不使用本体论角色，用户必须在 控制面板 中拥有 对象视图管理员 应用程序权限，并且在该对象类型的所有输入数据源中拥有 编辑者 角色。
  • Ontology Editor 角色 on the object type.

除非您通过遗留配置选项手动将 Workshop 模块从 Object View 选项卡转换为独立模块，否则 Workshop 模块的权限将由对象类型管理。这确保了模块和对象类型之间的权限保持一致，因此具有编辑或查看对象类型权限的用户也将能够编辑或查看 Object View 中的所有模块。

Edit configured Object Views

有多种方式可以访问已配置的 Object View 配置。

可以在 对象视图 选项卡中预览对象类型的对象视图。在标题栏中，您可以选择并固定一个默认显示的对象以进行预览。您还可以预览完整的和面板形式，并测试对象视图在浅色和深色模式下的显示效果。可以通过标题栏右侧的 编辑 选项访问配置的对象视图。

在对象资源管理器中，当查看一个对象时，可以通过选择 更多 > 高级 > 编辑对象视图 来访问该对象类型的配置对象视图。

当在应用程序内查看一个面板对象视图时，可以通过将鼠标悬停在下拉菜单省略号上并选择 编辑 来访问配置。只有具有编辑对象视图权限的用户才会看到下拉菜单。

这些编辑入口将引导至配置的“对象视图”编辑器，在此可以管理“对象视图”选项卡，并可使用“工作区”模块的所有标准功能来编辑内容。发布后，编辑内容将应用于该对象类型的全部对象。有关更多编辑信息，请参阅 完整的对象视图配置 和 面板对象视图配置。
