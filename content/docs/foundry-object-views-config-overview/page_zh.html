配置对象视图概述

配置对象视图是可自定义、可重用的对象数据表示。你可以通过 Workshop 模块 为对象类型构建完整的 完整视图 和 面板视图，从而在平台上灵活地表示对象。

默认情况下，Foundry 会为所有对象类型创建一个 标准对象视图。当你创建一个配置的对象视图时，它将成为用户的默认视图，但他们可以通过与对象视图一起打包的切换按钮返回到标准对象视图。此外，用户还可以在 Palantir 应用程序中悬停在对象视图中的省略号抽屉图标上，这些应用程序使用自己的自定义标题栏，如 Gaia 或 Vertex，以在标准视图和配置视图之间切换。

在Workshop中，切换标准和配置的Object Views的功能尚未可用。

默认配置

为每种对象类型会自动创建默认配置的对象视图。默认完整对象视图包含一组显著属性，或者如果没有显著属性，则包含所有非隐藏属性，并包含该对象的所有关联。默认面板包含相同的一组属性。默认视图将动态更新以反映对对象类型所做的更改，例如新增属性或重命名属性，但一旦对象视图被编辑，它就会变为用户管理，并且所有后续的更新都必须手动进行。

权限

编辑对象类型的对象视图所需的权限取决于该对象类型是否使用本体论角色：

  • 如果对象类型不使用本体论角色，则用户必须在控制面板中拥有Object View Admin应用权限，同时在该对象类型的所有输入数据源上拥有Editor角色。
  • 如果对象类型使用本体论角色，则用户只需在该对象类型上拥有Ontology Editor角色。

除非您通过遗留配置选项将对象视图选项卡的Workshop模块手动转换为独立模块，否则Workshop模块的权限将由对象类型管理。这确保了模块和对象类型之间的权限保持一致，因此具有编辑或查看对象类型权限的用户也将能够编辑或查看对象视图内的所有模块。

Edit configured Object Views

有多种方式可以访问已配置的对象视图。

对象类型的对象视图可以在本体论管理器（Ontology Manager）的对象视图选项卡中预览。在标题栏中，您可以选择并固定一个默认显示对象以进行预览。您还可以预览完整版和面板版的形式，并测试对象视图在浅色和深色模式下的显示效果。可以通过标题栏右侧的编辑选项来访问已配置的对象视图的编辑功能。

在对象资源管理器（Object Explorer）中，当查看一个对象时，可以通过选择更多 > 高级 > 编辑对象视图来访问该对象类型的已配置对象视图。

当在一个应用中查看面板对象视图时，可以通过将鼠标悬停在下拉菜单省略号上并选择编辑来访问配置。只有具有编辑对象视图权限的用户才会看到该下拉菜单。

这些编辑入口将引导至配置好的对象视图编辑器，在这里可以管理对象视图选项卡，并可以使用工作区模块的所有标准功能来编辑内容。发布后，编辑内容将应用于该对象类型的全部对象。有关更多编辑信息，请参阅 完整对象视图配置 和 面板对象视图配置。
