示例：通知合同数据主人审查合同

在此示例中，我们希望在合同状态变更为 需要审查 时通知合同数据主人。我们将使用 合同 对象类型，这是专门为本示例创建的自定义对象类型。

条件

我们在自动化创建向导中，首先选择 对象添加到集合 条件。由于我们希望在合同状态变更为 需要审查 时收到通知，因此可以按下方所示，在选定的对象集合上添加针对 合同状态 的过滤器。当对象进入该过滤后的对象集合时，此自动化操作便会触发，无论是由于新创建的对象带有 需要审查 状态，还是现有对象的状态变更为 需要审查。

效果

接下来，我们选择 通知 作为效果。

为确保合同数据主人为每个需要审查的合同分别收到一封邮件，请选择 为添加的每个合同发送一条通知 选项。为了向相应的合同数据主人动态发送邮件，请使用 基于对象属性 的收件人功能，并选择 合同数据主人 属性作为用户属性。合同数据主人 是 合同 对象上的一个属性，包含关联数据主人的 Foundry 用户 ID 数组（类似于 1234a567-8bc9-12ab-3456-7ca89b1c234a）。请注意，所有收件人必须至少拥有对该自动化的 查看者 权限，否则将无法收到通知。

接下来，我们需要定义通知内容。为了给收件人配置更为复杂的通知，我们将使用《函数生成的通知内容》功能。函数生成的通知在构建通知内容方面提供了更高的灵活性，并且支持使用 HTML 内容（<html>）。

为了支持函数生成的通知，我们在代码仓库中创建一个《函数》，该函数接收收件人和合同对象作为输入，并返回一条通知。


```typescript
import { Function, Notification, User, ShortNotification, EmailNotificationContent } from "@foundry/functions-api";
import { _automateExampleContract  } from "@foundry/ontology-api";

@Function()
public createContractStatusChangeNotification(user: User, contract: _automateExampleContract): Notification | undefined {

    const shortNotification = ShortNotification.builder()
        .heading("合同变更")
        .content(`合同 "${contract.title}" 的状态已变更为 ${contract.contractStatus}`)
        .addObjectLink("查看合同", contract)
        .build();

    // 定义邮件正文。邮件正文可以包含无头 HTML，例如数据表格
    // 注意，我们可以在内容中访问用户和合同的属性
    const emailBody = `Hello, ${user.firstName}!
    合同 "${contract.title}" 的状态已变更为 "${contract.contractStatus}"。

    查看合同详情。查看更多客户信息 <a href="${contract.customerUrl}">这里</a>。
    `;

    const emailNotificationContent = EmailNotificationContent.builder()
        .subject(`合同变更 - ${contract.customerName}`)
        .body(emailBody)
        .addObjectLink("查看合同", contract)
        .build();

    return Notification.builder()
        .shortNotification(shortNotification)
        .emailNotificationContent(emailNotificationContent)
        .build();
}
```

发布函数后，我们可以在自动化创建向导中选择该函数，并将其连接到效果输入端。对于 user 属性，我们选择通知的 Recipient（接收者）。对于 contract，我们选择 New Contract added（已添加新合同），该项由我们的对象集条件作为条件效果输入暴露出来。


设置


为防止合同数据主人收到有关此自动化的状态通知，我们可以在自动化创建向导的 Settings（设置）选项卡中，将我们的自动化管理员组添加到 Automation administrators（自动化管理员）设置中。




摘要


为了完成流程，我们为自动化命名，选择保存位置，将过期日期调整为“永不过期”，然后保存该自动化。
