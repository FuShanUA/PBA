使用 Pipeline Builder 创建增量管道

在本教程中，我们将使用 Pipeline Builder 创建一个简单的增量管道，并输出单个数据集。

下文使用的数据集均为假设示例，旨在说明增量计算的适用场景。

对于增量管道，您可以选择强制输出进行增量计算，并在无法以增量方式运行时使构建失败。

第一部分：问题陈述

假设我们有一个 flights 输入数据集，每周会追加新数据。我们希望筛选出从 JFK 机场起飞的航班，然后仅将这些航班追加到输出数据集 filtered_flights 中。

假设 flights 数据集共有 2000 万行，但每周只新增 100 万行。通过增量计算，管道只需处理 flights 中最新未处理的事务，而无需像快照计算那样处理所有行。

如果管道定期运行，增量处理可以大幅缩减每次运行的数据规模，从而节省时间和资源。

接下来，我们来看看如何设置增量管道。

第二部分：验证增量需求

只有满足本节所述的全部条件，管道才会以增量计算方式运行。例如，输入数据必须通过不修改现有文件的 APPEND 或 UPDATE 事务进行更新。否则，将输入标记为增量模式将不起作用。

首先，请检查是否满足所有增量约束条件：

  • 输入数据集 flights 正在通过 APPEND 事务或不修改现有文件的 UPDATE 事务进行更新。
  • 从 flights 计算 filtered_flights 的逻辑，不需要在后续构建中修改 filtered_flights 中已写入的任何数据。
    • 如果您希望更改管道逻辑（例如，同时纳入从 LGA 机场起飞的航班），可以更新管道。如果要将该逻辑应用于已处理的航班数据，您可能需要对管道进行重算。
  • 如果管道包含窗口函数、聚合或透视操作，请确保这些操作仅针对当前事务执行。

如需查看完整的注意事项列表，请参阅 Pipeline Builder 中关于增量计算的重要限制说明。

第三部分：创建管道

现在，我们可以初始化一个新管道（有关分步操作指南，请参阅《在 Pipeline Builder 中创建批处理管道》）。假设我们已经将 flights 导入为输入数据集。

首先，使用数据集下方的按钮将输入数据集标记为增量。您会看到右上角出现一个蓝色徽章，表示已完成更改。

接下来，添加一个转换操作，将 flights 筛选为仅包含从 JFK 机场起飞的航班。请注意，数据集输入右侧带有提示为增量输入的图标。下游转换将显示此图标，表明它们正在进行增量处理。

在图表中，下游节点将带有与输入相同的蓝色徽章。

最后，添加输出数据集 filtered_flights。

第四部分：部署输出数据集

现在您可以部署管道了。

部署时重算

有时可能需要对之前的输入事务进行重新处理（例如，逻辑发生变更，导致先前版本的输出数据已过时）。在这种情况下，你可以选择 部署时重放，将全部输入数据重新流经管道逻辑。重放完成后，随着新的追加事务添加至输入端，管道应继续执行增量计算。

部署时重放将在输出数据集上生成一个 SNAPSHOT 事务。
