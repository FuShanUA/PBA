入门指南

本指南将向您展示如何通过 Report Builder 对 Foundry 进行身份验证、选择数据集，并开始构建您的第一个报告。

在 Report Builder 中将 Foundry 添加为数据源

  • 在 Report Builder 中，点击“报告数据”窗格中的工具栏。点击“新建”，然后点击“数据源”。系统将打开“数据源属性”对话框。
  • 在“名称”文本框中，输入任意便于识别的名称，例如 FoundrySqlServer。点击“使用嵌入在我的报告中的连接”选项。
  • 点击“选择连接类型”下拉菜单并选择 ODBC，此时您的窗口应如下所示：

构建连接字符串

接下来，您需要将以下基础连接字符串粘贴到 Report Builder 的“连接字符串”文本框中，并按照下文说明替换 '' 和 ''，从而为您的 Foundry 连接创建连接字符串：

  • 基础连接字符串： DSN=FoundrySql;BaseUrl=<URL>;PWD=<Token>
  • URL：将您的 Foundry 连接 URL 设为“Base URL”。该 URL 即为您平时用于访问 Foundry 的链接。设置时，需登录 Foundry，复制其 URL，删除 https:// 前缀以及 .com 之后的所有内容，然后用该路径替换 ''。
  • 令牌：请按照《生成令牌》指南中的说明，在 Foundry 内生成私有身份验证令牌。获取令牌后，将其粘贴到基础连接字符串的 '' 部分。
  • 至此，您的 Foundry 数据源应已成功连接。您可以尝试点击“测试连接”按钮。如果此时出现错误，请确认您已按《安装说明》完成相关操作。

您的凭据现已保存在 Report Builder 中，只要其保持有效便会一直生效。在令牌过期之前，系统不会再次提示您输入令牌。届时，您可以按照上述说明重新生成新令牌。

点击“确定”并进入下一步。

连接 Foundry 并查询数据集

  • 再次使用左侧的“报告数据”窗格，点击“新建”，然后点击“添加数据集”。
  • 系统会提示您输入数据集名称。点击“使用嵌入在我的报告中的数据集”选项，然后从下拉菜单中选择 FoundrySqlServer。
  • 若要开始使用特定数据集，请将该数据集的文件路径或 RID 复制到“查询”文本框中。您可以在 Foundry 中导航至目标数据集的“关于”页面，点击“查看更多”，然后复制“RID”值或“位置”来获取这些信息。（参见《指南：在 Foundry 中识别数据集的 RID 或文件路径》。）
  • 编写您的 SQL 查询语句，并像往常一样继续构建报告。
