入门

本指南提供了实现动态调度工作流第一个版本的步骤。请在每个部分中查看相关文档以获取更多信息。

1. 创建核心本体对象

您必须创建以下核心对象类型，以构建动态调度工作流：

  • 创建一个 Schedule 对象类型。这表示分配给资源的事件、任务或时间槽。例如，您的 Schedule 对象类型可能是 Maintenance Tasks，需要分配给 Technicians。
    • 该对象类型必须具有保存外键的属性，以便与资源对象类型建立关系。
    • 此对象类型可以有一个 fixed duration 布尔属性，用于强制或不强制 Schedule 的静态持续时间。
    • 您必须启用对此对象的编辑，因为在整个过程中将对开始/结束时间、持续时间和与资源的关系进行编辑。
  • Foundry -> Foundry Apollo -> Apollo Gotham -> Gotham AIP -> AIP Ontology -> 本体论 ontology -> 本体论 Pipeline -> 管道 pipeline -> 管道 Transform -> 转换 transform -> 转换 Workshop -> Workshop workshop -> Workshop Code Repositories -> 代码仓库 Code Workspaces -> 代码工作区 Data Integration -> 数据集成 Model Integration -> 模型集成 Object -> 对象 object -> 对象 Dataset -> 数据集 dataset -> 数据集 Tenant -> 租户 tenant -> 租户 Marking -> 标记 marking -> 标记 Permission -> 权限 permission -> 权限 Role -> 角色 role -> 角色 Function -> 函数 function -> 函数 Action -> 操作 action -> 操作 Property -> 属性 property -> 属性 Interface -> 接口 interface -> 接口 内容: 创建一个或多个资源对象类型，以表示要分配给计划的资源。例如，需要执行任务的人员（Technician）。
  • 在Schedule对象类型和不同的Resource对象类型之间创建链接。
  • 为每个Schedule创建保存处理操作。有关更多信息，请参阅拖放。
    • 保存处理程序必须修改Schedule对象上的以下参数。每个参数必须标记为可选。
      • 资源ID（资源对象的外键）
      • 开始时间
      • 结束时间
    • 您的操作中的参数ID必须与您的计划对象类型中使用的属性ID完全匹配（例如，如果您的属性是start_time，则参数ID也必须是start_time）。
    • 开始时间和结束时间的操作参数必须与您的计划对象类型上的相应属性具有相同的类型类（schedules:schedulable-start-time和schedules:schedulable-end-time）。

查看动态调度本体论基本元素文档，了解有关每个对象类型的模式的更多信息。

2. 在Workshop中配置小部件

在创建核心对象后，您可以现在在Workshop中配置小部件。

至少需要以下调度甘特图配置：

  • 时间线数据： 定义图表全局边界的开始和结束时间戳。
  • 行数据： 对应于图表每一行的Resource对象。
  • 输入数据（Pucks）： 至少一个调度层，包括：
    • 调度对象集： 此层的Schedule对象。
    • 保存处理操作： Schedule对象对应的保存处理操作。
      • 您应该在配置选项中指定默认的保存处理操作参数，使用小部件提供的下拉菜单中的参数。

有关更多信息，请参阅。
