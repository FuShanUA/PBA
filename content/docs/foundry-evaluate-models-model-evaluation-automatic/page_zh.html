自动评估模型

随着建模项目走向成熟、规模化并投入实际运营，系统性地评估和对比当前与新增模型提交物变得至关重要。应使用维护良好的代表性数据与明确的指标，对模型提交物进行一致的评估。

通过配置建模目标，可以自动为提交至该目标的所有模型生成推理与指标流水线，从而支持在软件层面落地系统化的测试与评估（T&E）方案。

配置模型评估

要启用模型自动评估功能，首先需要在建模目标中配置如何评估提交至该目标的模型。

启用流水线管理

在建模目标中配置模型自动评估的第一步是启用推理与指标生成。在“建模目标”主页上，选择配置评估面板；如果已经配置过模型评估，可以在评估面板中选择编辑评估配置。

在评估面板配置视图中，可以决定是自动生成推理和指标流水线，还是仅生成推理流水线。根据选择，系统将为所有现有模型提交物以及后续提交至该建模目标的任何新模型生成推理和指标数据集。

配置自动模型推理或评估

接下来，可以决定在提交新模型时是否自动构建推理数据集或推理与指标数据集。自动构建推理和指标数据集，能确保对建模项目中所有候选模型进行一致的评估。

推理和指标流水线仅在模型提交时构建。对于现有的模型提交物，需要通过评估面板上的构建评估按钮手动发起构建。

点击保存以保存推理和指标生成设置。

添加评估数据集

评估数据集是指在建模目标内部用于评估模型的 Foundry 数据集。如果建模目标配置为自动生成推理流水线，系统会针对模型提交物与评估数据集的每一种组合，分别生成一个推理数据集。每个评估数据集都应具有相关性并得到精心维护；它可以包含经过筛选的验证集或测试集、生产观测数据、用户反馈实例、关键测试用例，或假设场景的表征数据。评估数据集应包含模型运行推理所需的字段或文件。

评估数据集的规模、更新频率和权限可以各不相同；将这些数据集分开管理，有助于更好地控制指标计算的结果频率与权限。

权限

建模目标内部严格遵循权限控制。用户若未获得相应的访问权限，将无法查看模型、评估数据集、评估库或指标。

要配置评估数据集，请选择添加评估数据集。弹窗将支持配置以下内容：

  • 评估数据集。
  • 为该评估数据集生成推理数据集的 Foundry 文件夹。
  • 为该评估数据集生成指标数据集的 Foundry 文件夹。

Foundry 文件夹可以是新建的，也可以是现有的，且不要求各评估数据集必须使用独立的文件夹。通常建议为每个建模目标指定一个输出文件夹，但您可以根据具体使用场景灵活配置。

评估数据集和目标本身必须与推理及指标的目标文件夹位于同一个 Foundry 项目中，或者应将它们作为引用添加到该 Foundry 项目中。

建模目标中的模型自动评估功能仅兼容具有单一表格数据集输入的模型。

配置评估库

如果已将建模目标配置为生成指标和推理流水线，下一步就是配置评估库。评估库是 Foundry 中已发布的 Python 包，用于生成模型评估器。Foundry 为二分类和回归提供了默认的模型评估器，同时支持构建自定义模型评估器。评估库用于衡量模型性能、模型公平性、模型鲁棒性及其他指标。

配置完成后，评估库将生成一个数据集，其中包含每个已配置推理数据集对应的指标集。

要配置评估库，请点击选择评估库，选中一个评估库，然后配置该模型评估器所需的字段。对于列输入类型，建模目标会建议所有评估数据集中可用的列。此外，如果在下拉菜单中未看到建议的列，或者知道该列将由模型转换生成，你还可以通过添加预期模型输出来表示模型提交预期要生成的列。

最常见的预期模型输出类型包括预测输出（通常命名为 prediction）、概率输出（通常命名为 probability_1）以及置信度分数。

配置评估子集

配置自动模型评估的可选步骤之一，是定义评估子集，以便评估库针对其生成指标。评估子集是评估数据集中数据的一个子集，系统将单独为其生成指标。可以通过评估仪表板单独分析评估子集的指标。

评估子集有助于了解模型在特定输入数据组上的表现，因此可用于提升模型在潜在受保护类别中的可解释性、可说明性和公平性。评估子集可以基于评估数据集中的任意列生成，因此不要求必须基于模型转换的输入或输出生成，例如模型特征或模型预测。

系统始终会针对“总体”子集中的每个评估数据集的全部数据（每一行）生成指标。对于进一步子集上的自动指标生成，配置评估子集是可选项。

要配置评估子集，请点击添加评估子集，并选择要创建子集所基于的评估数据集列或预期模型输出。

分类分桶

如果从评估子集中选择的字段类型为 string，在构建评估流水线时，评估库将为每个唯一的字符串值生成一个唯一的子集。

数值分桶

如果从评估子集中选择的字段为数值类型，可以选择用于生成子集的数值分桶策略。分桶方式可以是固定宽度，也可以通过特定的范围截断点来定义。两种分桶策略创建的分桶均按以下方式定义：

  • 包含下限
  • 上限不包含在内

对于固定宽度的分桶，必须提供起始值和分桶宽度。系统将针对所选字段的整个范围，按正负两个方向，为每个分桶宽度生成一个唯一的子集。

范围截断点将生成介于你指定的所有值之间的子集。如果未覆盖所选字段的整个范围，系统可能会生成两个额外的分桶：一个从列最小值到最低截断点，另一个从最高截断点到列最大值。

警告

每一个唯一的子集都会与每一个评估数据集和库进行匹配计算。因此，生成大量子集可能会显著增加模型评估的构建时间。

多字段子集

你可以生成代表多个字段组合的子集。在多个字段上点击 添加额外字段，选择多列或预期的模型输出，将其组合为单个子集。这将为字段间的每个分桶组合创建一个子集。

可以为每个子集字段单独定义定量分桶策略。

查看评估子集预览

配置评估子集时，页面右侧会显示评估数据集的预览。你配置的每个评估数据集都会提供此预览功能，可用于判断当前的评估配置将生成多少个评估子集。

保存评估配置

点击页面右上角的 保存配置，即可保存评估配置并返回评估仪表板。
