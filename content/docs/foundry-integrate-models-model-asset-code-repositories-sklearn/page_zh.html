在代码仓库中使用 scikit-learn 训练二分类模型

以下文档提供了一个示例，说明如何使用开源 UCI ML 乳腺癌威斯康星（诊断）数据集 ↗ 在 代码仓库 应用程序中通过 Model Training Template 训练 scikit-learn 二分类模型。

如需了解以下步骤的详细操作指南，包括如何编写模型适配器和为模型训练编写 Python 转换，参考我们关于 如何在代码仓库中训练模型 的文档。

1. 编写一个模型适配器

首先，在代码仓库中使用 模型适配器 和 Model Training Template。

下面的示例逻辑假设如下：

  • 该模型适配器使用 scikit-learn model 初始化。
  • 提供给该模型的数据是表格形式的。
  • 该模型的输出是表格形式，包含columns、prediction、probability_0和probability_1的所有列，其中，
    • prediction 是 0 或 1，0 表示未检测到癌症，1 表示检测到癌症。
    • probability_0 是未检测到癌症的概率。
    • probability_1 是检测到癌症的概率。
  • 以下依赖项已添加到仓库中：foundry-transforms-lib-python、pandas 1.5.3、scikit-learn 1.3.2 和 dill 0.3.7
Copied!
1import palantir_models as pm
2from palantir_models.serializers import *
3
4
5class SklearnClassificationAdapter(pm.ModelAdapter):
6
7    @pm.auto_serialize(
8        model=DillSerializer()
9    )
10    def __init__(self, model):
11        self.model = model
12
13    @classmethod
14    def api(cls):
15        columns = [
16            'mean_radius', 'mean_texture', 'mean_perimeter', 'mean_area',
17            'mean_smoothness', 'mean_compactness', 'mean_concavity',
18            'mean_concave_points', 'mean_symmetry', 'mean_fractal_dimension',
19            'radius_error', 'texture_error', 'perimeter_error', 'area_error',
20            'smoothness_error', 'compactness_error', 'concavity_error',
21            'concave_points_error', 'symmetry_error', 'fractal_dimension_error',
22            'worst_radius', 'worst_texture', 'worst_perimeter', 'worst_area',
23            'worst_smoothness', 'worst_compactness', 'worst_concavity',
24            'worst_concave_points', 'worst_symmetry', 'worst_fractal_dimension'
25        ]
26        inputs = {"df_in": pm.Pandas(columns=columns)}
27        outputs = {"df_out": pm.Pandas(columns= columns + [
28                                            ("prediction", int),
29                                            ("probability_0", float),
30                                            ("probability_1", float)
31                                        ])}
32        return inputs, outputs
33
34    def predict(self, df_in):
35        X = df_in.copy()
36
37        predictions = self.model.predict(X)
38        probabilities = self.model.predict_proba(X)
39
40        df_in['prediction'] = predictions
41        for idx, label in enumerate(self.model.classes_):
42            df_in[f"probability_{label}"] = probabilities[:, idx]
43
44        return df_in

2. 编写一个 Python 转换来训练模型

在 model_training/model_training.py 中的同一仓库中，编写模型训练逻辑。

此示例使用了 scikit-learn 库中提供的开源 UCI ML 乳腺癌威斯康星（诊断）数据集 ↗。

Copied!
1from transforms.api import transform
2from palantir_models.transforms import ModelOutput
3from main.model_adapters.adapter import SklearnClassificationAdapter
4from sklearn.datasets import load_breast_cancer
5from sklearn.compose import make_column_transformer
6from sklearn.impute import SimpleImputer
7from sklearn.pipeline import Pipeline
8from sklearn.preprocessing import StandardScaler
9from sklearn.ensemble import RandomForestClassifier
10
11
12@transform.using(
13    model_output=ModelOutput("/path/to/model_asset"),
14)
15def compute(model_output):
16    X_train, y_train = load_breast_cancer(as_frame=True, return_X_y=True)
17    X_train.columns = X_train.columns.str.replace(' ', '_')
18    columns = X_train.columns
19
20    numeric_transformer = Pipeline(
21        steps=[
22            ("imputer", SimpleImputer(strategy="median")),
23            ("scaler", StandardScaler())
24        ]
25    )
26
27    preprocessor = make_column_transformer(
28        (numeric_transformer, columns),
29        remainder="passthrough"
30    )
31
32    model = Pipeline(
33        steps=[
34            ("preprocessor", preprocessor),
35            ("classifier", RandomForestClassifier(n_estimators=50, max_depth=3))
36        ]
37    )
38    model.fit(X_train, y_train)
39
40    foundry_model = SklearnClassificationAdapter(model)
41    model_output.publish(model_adapter=foundry_model)

3. 使用模型

在Python转换中进行推理

您可以在Python转换中使用您的模型进行推理。例如，一旦您的模型已经训练完成，将下面的推理逻辑复制到model_training/run_inference.py文件中并选择构建。

如果要在未定义该模型的转换仓库中运行模型，请在use_sidecar = True中设置ModelInput。这将在运行时自动导入模型适配器及其依赖项，并在单独的环境中运行以防止依赖项冲突。有关更多详细信息，请查看ModelInput类参考。此功能从palantir_models版本0.2010.0开始适用于非Spark转换（使用@lightweight或@transform.using装饰器）。

如果use_sidecar未设置为True，则必须将模型适配器及其依赖项导入到当前代码仓库中或在其内部定义。

Copied!
1from transforms.api import transform, Output, LightweightOutput
2from palantir_models.transforms import ModelInput
3from palantir_models import ModelAdapter
4from sklearn.datasets import load_breast_cancer
5
6
7@transform.using(
8    model=ModelInput(
9        "ri.models.main.model.cfc11519-28be-4f3e-9176-9afe91ecf3e1",
10        # use_sidecar=True is recommended for models defined outside the current transform repository
11        ),
12    inference_output=Output("ri.foundry.main.dataset.5dd9907f-79bc-4ae9-a106-1fa87ff021c3"),
13)
14def compute(model: ModelAdapter, inference_output: LightweightOutput):
15    X, y = load_breast_cancer(as_frame=True, return_X_y=True)
16    X.columns = X.columns.str.replace(' ', '_')
17
18    inference_results = model.transform(X)
19    inference_output.write_pandas(inference_results.df_out)

在建模目标中进行实时推理

一个 Palantir 模型可以提交到建模目标以执行以下操作：

  • 自动模型评估
  • 批量管道
  • 实时托管

在 将此模型提交到建模目标 之后，您可以创建一个发布版本以用于实时推理的模型托管。一旦部署准备就绪，您就可以进行实时推理并将此模型连接到一个操作应用程序。
