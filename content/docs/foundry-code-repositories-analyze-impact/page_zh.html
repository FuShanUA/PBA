分析变更影响

代码编辑可能会导致数据集内容、权限和结构发生意外变化。建议保护生产分支，并在合并前要求对提议的变更进行审查；这些选项可以在分支设置中找到。随后，拉取请求页面将提供多种方式，用于分析代码变更对受影响数据集的影响。

过期数据集

评估变更影响需要使用最新代码变更构建受影响的数据集。头部分支（开发环境）和基础分支（目标环境）都需要进行此构建：

  • 在头部分支（开发环境）上构建，以验证代码能否正常构建、输出是否符合预期，以及是否满足所有数据期望。
  • 在基础分支（目标环境）上构建，以便将输出与目标环境的最新版本进行比较。

受影响的数据集过期时，拉取请求页面会发出警告。点击配置并构建，可以查看过期数据集并在头部分支和基础分支上构建它们。

警告

拉取请求页面的过期警告仅涵盖特定代码仓库内的受影响数据集。对于仓库外的过期父数据集或未提交的变更，拉取请求页面不会发出警告。

影响分析

影响分析选项卡提供了受拉取请求影响的数据集信息。默认情况下，它仅显示直接受影响的数据集，不包含可能受波及的派生数据集。可以通过点击添加数据集到分析来检查对其他表的影响。

审查受影响的数据集需要数据访问权限。无权访问的数据集将在界面中标记为不可访问。

仓库确定受影响数据集列表的方式因所使用的语言而异。Python 仓库使用转换逻辑级版本控制（TLLV）来生成该列表。在 Java 转换中，如果数据集的源文件被拉取请求更改，则认为该数据集受到了直接影响。

影响分析选项卡显示以下信息：

  • 代码 - 查看源文件的变更（不包含可能被引用或使用的其他文件）。
  • 模式 - 查看数据集列的变更。
  • 安全 - 查看应用于输出数据集的标记变更。
  • 期望 - 查看头部分支上的数据期望。
  • 废弃数据集 - 被拉取请求更新的废弃数据集将以褪色形式显示。

查看对派生数据集的影响

点击添加数据集到分析，将分析拉取请求对派生数据集的影响。所选数据集与受影响数据集之间的所有中间数据集也将被添加进来。

添加的数据集不得过期，才能显示影响信息。点击配置并构建，查看更新后的过期数据集列表并触发构建。

管道审查

管道审查选项卡提供了受拉取请求影响的数据集的血缘关系视图。

选中某个数据集节点后，将显示生成该数据集的转换代码文件的变更，以及该代码文件生成的每个数据集的模式变更。按照数据流的顺序浏览受影响的数据集，有助于理解不同文件和数据集的变更是如何对应的。

在审查拉取请求中的变更时，可以逐个批准或拒绝每个文件，以便跟踪进度。对于转换源文件，此审查状态将作为指示器显示在管道图中相应的输出数据集节点上。
