2c. 教程：在代码仓库中训练模型

在开始本教程的此步骤之前，您应该已完成 建模项目设置。在本教程中，您可以选择在 Jupyter® 笔记本 或代码仓库中训练模型。Jupyter® 笔记本适用于快速和迭代的模型开发，而代码仓库则推荐用于生产级的数据和模型管道。

在本教程的这一部分，我们将在 代码仓库 中训练一个模型。此步骤将涵盖：

  1. 为模型训练创建代码仓库
  2. 对特征数据进行测试和训练拆分
  3. 在代码仓库中编写模型训练逻辑
  4. 运行批量推理
  5. 查看模型并将其提交到建模目标

2c.1 如何为模型训练创建代码仓库

Foundry 中的 Code Repositories 应用程序是一个基于网络的开发环境，用于编写生产级别的数据和机器学习管道。Foundry 提供了一个名为 Model Training 的模板仓库，用于机器学习。

操作： 在本教程上一步中创建的 code 文件夹中，选择 + New > 代码仓库。你的代码仓库应根据你正在训练的模型进行命名。在此情况下，将仓库命名为 "median_house_price_model_repo"。选择 模型训练 模板，然后选择 初始化。

模型训练模板包含一个示例结构，我们将根据本教程进行调整。你可以展开左侧的文件以查看示例项目。

2c.2 如何为测试和训练拆分特征数据

在监督式机器学习项目中，第一步是将我们的标记特征数据拆分为用于训练和测试的单独数据集。最终，我们将想要创建性能指标（我们模型在新数据上的表现估计），以便决定该模型是否足够好以用于生产环境，并且可以与其他利益相关者沟通对该模型结果的信任程度。我们必须使用单独的数据进行此验证，以帮助确保性能指标能代表我们在现实世界中看到的情况。

因此，我们将编写一个 Python 转换，它会获取我们的标记特征数据，并将其拆分为我们的两个训练和测试数据集。

2c.3 如何在代码仓库中编写模型训练逻辑

Foundry 中的模型由两个组件组成，模型工件（在模型训练任务中生成的模型文件）和模型适配器（一个 Python 类，描述 Foundry 应如何与模型工件交互以执行推理）。

模型训练模板包含两个模块，model_training 用于训练任务，model_adapters 用于模型适配器。

模型依赖

模型训练几乎总是需要添加包含模型训练、序列化、推理或评估逻辑的Python依赖项。Foundry支持通过conda添加依赖项规范。这些依赖项规范用于为执行模型训练任务创建一个解析后的Python环境。

在Foundry中，这些解析后的依赖项会与您的模型自动打包，以确保您的模型自动拥有执行推理（生成预测）所需的所有逻辑。在此示例中，我们将使用pandas和scikit-learn来生成我们的模型，并使用dill保存我们的模型。

操作：在左侧栏中，选择库，并为scikit-learn = 1.2.0、pandas = 1.5.2和dill = 0.3.7添加依赖项。然后选择提交以创建一个解析后的Python环境。

模型适配器逻辑

模型适配器 为Foundry中的所有模型提供了一个标准接口。标准接口确保所有模型可以立即用于生产应用程序，因为Foundry将处理加载模型、其Python依赖项、公开其API以及与您的模型进行交互的基础设施。

为此，您必须创建一个 ModelAdapter 类的实例，以作为此通信层。

有4个函数需要实现：

  1. 模型保存和加载：为了重新使用您的模型，您需要定义如何保存和加载您的模型。Palantir提供了许多 默认的序列化方法（保存），在更复杂的情况下，您可以 实现自定义的序列化 逻辑。
  2. api： 定义您的模型的API，并告诉Foundry您的模型需要哪种类型的数据输入。
  3. predict： 由Foundry调用，以向您的模型提供数据。这是您可以将输入数据传递给模型并生成推理（预测）的地方。
Copied!
1import palantir_models as pm
2
3
4class SklearnRegressionAdapter(pm.ModelAdapter):
5
6    @pm.auto_serialize
7    def __init__(self, model):
8        self.model = model
9
10    @classmethod
11    def api(cls):
12        columns = [
13            ('median_income', float),
14            ('housing_median_age', float),
15            ('total_rooms', float),
16        ]
17        return {"df_in": pm.Pandas(columns)}, \
18               {"df_out": pm.Pandas(columns + [('prediction', float)])}
19
20    def predict(self, df_in):
21        df_in['prediction'] = self.model.predict(
22            df_in[['median_income', 'housing_median_age', 'total_rooms']]
23        )
24        return df_in

操作 打开 model_adapters/adapter.py 文件并将上述逻辑粘贴到该文件中。

模型训练逻辑

现在我们的依赖项已经设置好，并且我们已经编写了一个模型适配器，我们可以在Foundry中训练一个模型。

Copied!
1from transforms.api import transform, Input
2from palantir_models.transforms import ModelOutput
3from main.model_adapters.adapter import SklearnRegressionAdapter
4
5
6def train_model(training_df):
7    from sklearn.impute import SimpleImputer
8    from sklearn.linear_model import LinearRegression
9    from sklearn.pipeline import Pipeline
10    from sklearn.preprocessing import StandardScaler
11
12    numeric_features = ['median_income', 'housing_median_age', 'total_rooms']
13    numeric_transformer = Pipeline(
14        steps=[
15            ("imputer", SimpleImputer(strategy="median")),
16            ("scaler", StandardScaler())
17        ]
18    )
19
20    model = Pipeline(
21        steps=[
22            ("preprocessor", numeric_transformer),
23            ("classifier", LinearRegression())
24        ]
25    )
26    X_train = training_df[numeric_features]
27    y_train = training_df['median_house_value']
28    model.fit(X_train, y_train)
29
30    return model
31
32@transform.using(
33    training_data_input=Input("<YOUR_PROJECT_PATH>/data/housing_training_data"),
34    model_output=ModelOutput("<YOUR_PROJECT_PATH>/models/linear_regression_model"),
35)
36def compute(training_data_input, model_output):
37    training_df = training_data_input.pandas()
38    model = train_model(training_df)
39
40    # Wrap the trained model in a ModelAdapter
41    foundry_model = SklearnRegressionAdapter(model)
42
43    # Publish and write the trained model to Foundry
44    model_output.publish(
45        model_adapter=foundry_model
46    )

2c.4 如何进行批量推理

在代码仓库中

一旦您的模型训练逻辑运行完成，您就可以直接在代码仓库中生成预测（也称为推断）。

Copied!
1from transforms.api import transform, Input, Output, LightweightInput, LightweightOutput
2from palantir_models.transforms import ModelInput
3from palantir_models import ModelAdapter
4
5@transform.using(
6    testing_data_input=Input("<YOUR_PROJECT_PATH>/data/housing_test_data"),
7    model_input=ModelInput("<YOUR_PROJECT_PATH>/models/linear_regression_model"),
8    predictions_output=Output("<YOUR_PROJECT_PATH>/data/housing_testing_data_inferences")
9)
10def compute(
11        testing_data_input: LightweightInput,
12        model_input: ModelAdapter,
13        predictions_output: LightweightOutput
14    ):
15    inference_outputs = model_input.transform(testing_data_input)
16    predictions_output.write_pandas(inference_outputs.df_out)

如果在未定义模型的转换仓库中运行模型，请在use_sidecar = True中设置ModelInput。这将自动导入模型适配器及其依赖项，同时在单独的环境中运行以防止依赖冲突。有关更多详细信息，请查看ModelInput类参考。

如果use_sidecar未设置为True，则必须将模型适配器及其依赖项导入或定义在当前代码仓库中。

操作：在您的仓库中打开model_training/run_inference.py文件，并将上述代码复制到仓库中。更新路径以正确指向您之前创建的模型资产和测试数据集。选择构建 在左上角运行代码。

一旦构建完成，您可以在构建输出面板中查看生成的预测结果。

2c.6 如何查看模型并将其提交到建模目标

在您的模型构建完成后，可以通过在linear_regression_model文件中选择model_training/model_training.py，或者导航到我们之前创建的文件夹结构中的模型来打开该模型。

模型视图显示了模型的训练来源、用于生成此模型的训练数据集、模型API以及该模型发布的模型适配器。重要的是，您可以将许多不同的版本发布到同一个模型；这些模型版本可在左侧侧边栏的下拉菜单中找到。

由于模型版本与训练期间使用的特定模型适配器相关联，因此您需要重新发布并构建模型训练过程以对模型适配器逻辑进行任何更改。

现在我们有了一个模型，可以将其提交到我们的建模目标中进行管理、评估并发布到生产应用程序中。

操作：在代码中选择linear_regression_model以导航到你创建的模型资产，选择提交到建模目标，并将该模型提交到你在本教程第1步中创建的建模目标。你将被要求提供一个提交名称和提交所有者。这是用于在建模目标内部唯一跟踪模型的元数据。将模型命名为linear_regression_model，并把你自己作为提交所有者。
