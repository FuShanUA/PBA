2c. 教程：在代码仓库中训练模型

在开始本步教程之前，您应已完成《建模项目设置》。在本教程中，您可以选择在 Jupyter® 笔记本中或在代码仓库中训练模型。Jupyter® 笔记本适合快速迭代的模型开发，而代码仓库则适用于生产级的数据与模型流水线。

在本步教程中，我们将在《代码仓库》中训练模型。本步骤将涵盖以下内容：

  1. 为模型训练创建代码仓库

  5. 查看模型并将其提交至建模目标

2c.1 如何为模型训练创建代码仓库

Foundry 中的代码仓库应用是一个基于 Web 的开发环境，用于编写生产级的数据与机器学习流水线。Foundry 为机器学习提供了一个模板化仓库，名为 模型训练 模板。

操作：在您于上一步教程中创建的 code 文件夹中，选择 + 新建 > 代码仓库。代码仓库的命名应与您正在训练的模型相关。此处将该仓库命名为“median_house_price_model_repo”。选择 模型训练 模板，然后点击 初始化。

模型训练模板包含一个示例结构，我们将针对本教程对其进行调整。您可以展开左侧的文件来查看示例项目。

2c.2 如何划分用于测试与训练的特征数据

监督式机器学习项目的首要步骤，是将带标签的特征数据集拆分为独立的训练集和测试集。后续我们需要构建性能指标（即评估模型在新数据上表现情况的估计值），以此来判断该模型是否达到投入生产环境的标准，并向其他利益相关方说明对该模型结果的信任程度。验证过程必须使用独立的数据，以确保性能指标能真实反映实际应用中的表现。

为此，我们将编写一个 Python 转换脚本，将带标签的特征数据拆分为训练集和测试集这两个数据集。

2c.3 如何在代码仓库中编写模型训练逻辑

Foundry 中的模型包含两个组件：模型工件（模型训练作业中生成的模型文件）和模型适配器（一个 Python 类，描述 Foundry 应如何与模型工件交互以执行推理）。

模型训练模板由两个模块组成：用于训练作业的 model_training 和用于模型适配器的 model_adapters。

模型依赖

模型训练几乎总是需要添加包含模型训练、序列化、推理或评估逻辑的 Python 依赖项。Foundry 支持通过 conda 添加依赖规范。这些依赖规范用于创建解析后的 Python 环境，以执行模型训练作业。

在 Foundry 中，这些解析后的依赖项会自动与你的模型打包在一起，确保模型自动具备执行推理（生成预测）所需的所有逻辑。在本例中，我们将使用 pandas 和 scikit-learn 生成模型，并使用 dill 保存模型。

操作： 在左侧边栏中，选择 Libraries 并添加 scikit-learn = 1.2.0、pandas = 1.5.2 和 dill = 0.3.7 的依赖项。然后选择 Commit 创建解析后的 Python 环境。

模型适配器逻辑

模型适配器为 Foundry 中的所有模型提供标准接口。该标准接口确保所有模型均可直接用于生产应用，因为 Foundry 会负责处理加载模型的基础设施、其 Python 依赖项、暴露 API 以及与您的模型进行交互。

为实现此功能，您必须创建一个 ModelAdapter 类的实例，以此充当通信层。

需要实现以下 4 个函数：

  1. 模型保存与加载：为了重用您的模型，您需要定义模型的保存和加载方式。Palantir 提供了多种默认方法进行序列化（保存），在更复杂的场景中，您可以实现自定义序列化逻辑。
  2. api：定义模型的 API，并告知 Foundry 您的模型需要何种类型的输入数据。
  3. predict：由 Foundry 调用，向您的模型提供数据。您可以在此处将输入数据传递给模型并生成推断（预测）。
```python import palantir_models as pm class SklearnRegressionAdapter(pm.ModelAdapter): @pm.auto_serialize def __init__(self, model): self.model = model @classmethod def api(cls): columns = [ ('median_income', float), ('housing_median_age', float), ('total_rooms', float), ] return {"df_in": pm.Pandas(columns)}, \ {"df_out": pm.Pandas(columns + [('prediction', float)])} def predict(self, df_in): df_in['prediction'] = self.model.predict( df_in[['median_income', 'housing_median_age', 'total_rooms']] ) return df_in ```

操作 打开 model_adapters/adapter.py 文件，并将上述逻辑粘贴到该文件中。

6def train_model(training_df):
7    from sklearn.impute import SimpleImputer
8    from sklearn.linear_model import LinearRegression
9    from sklearn.pipeline import Pipeline
10    from sklearn.preprocessing import StandardScaler
11
12    numeric_features = ['median_income', 'housing_median_age', 'total_rooms']
13    numeric_transformer = Pipeline(
14        steps=[
15            ("imputer", SimpleImputer(strategy="median")),
16            ("scaler", StandardScaler())
17        ]
18    )
19
20    model = Pipeline(
21        steps=[
22            ("preprocessor", numeric_transformer),
23            ("classifier", LinearRegression())
24        ]
25    )
26    X_train = training_df[numeric_features]
27    y_train = training_df['median_house_value']
28    model.fit(X_train, y_train)
29
30    return model

1from transforms.api import transform, Input, Output, LightweightInput, LightweightOutput

5@transform.using(
6    testing_data_input=Input("<YOUR_PROJECT_PATH>/data/housing_test_data"),

10def compute(

15    inference_outputs = model_input.transform(testing_data_input)
16    predictions_output.write_pandas(inference_outputs.df_out)

要在未定义模型的转换代码库中运行该模型，请在 ModelInput 中设置 use_sidecar = True。这会自动导入模型适配器及其依赖项，同时在独立环境中运行它们，以防发生依赖冲突。查阅 《ModelInput 类参考文档》 了解更多细节。

如果未将 use_sidecar 设置为 True，则必须将模型适配器及其依赖项导入或定义在当前代码库中。

2c.6 如何查看模型并将其提交至建模目标

模型构建完成后，您可以通过在 model_training/model_training.py 文件中选择 linear_regression_model 来打开模型，也可以通过先前创建的文件夹结构导航至该模型。

模型视图包含模型的训练来源、用于生成该模型的训练数据集、模型 API 以及发布该模型时使用的模型适配器。需要注意的是，你可以向同一个模型发布多个不同的版本；这些模型版本可以在左侧边栏的下拉菜单中查看。

由于模型版本与训练时使用的特定模型适配器相关联，若要更改模型适配器的逻辑，你需要重新构建模型训练流程并重新发布模型。

既然已经有了模型，我们可以将其提交到建模目标中，以便进行管理、评估，并最终发布到业务应用中。

操作：在代码中选择 linear_regression_model，导航至你创建的模型资产，选择 提交到建模目标 (Submit to a Modeling Objective)，然后将该模型提交到你在本教程第一步中创建的建模目标。系统会要求你提供提交名称和提交数据主人。这是用于在建模目标中唯一标识和跟踪模型的元数据。将该模型命名为 linear_regression_model，并将你自己标记为提交数据主人。
