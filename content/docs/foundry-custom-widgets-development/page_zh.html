开发

什么是开发模式？

开发模式是一种开发功能，允许您直接在Foundry应用程序中使用非生产环境代码预览小部件。当开发模式处于激活状态时，您的开发服务器（在本地或Foundry中运行）会覆盖已发布的部件资源，使您无需发布新版本即可查看您的代码和配置更改。

开发模式的关键特征：

  • 个人的： 仅影响您的用户帐户，其他用户仍然看到已发布的版本。
  • 临时的： 24小时后自动过期。
  • 实时更新： 来自您的开发服务器的代码更改立即出现在小部件中。
  • 上下文感知： 在自定义小部件的游乐场和Workshop等主机应用程序中都能正常工作。

设置项目

现在无需发布小部件即可对其进行开发。您可以在自定义小部件游乐场和代码工作区中预览未发布的的小部件。要在Workshop中使用全新小部件，只需发布一次小部件，使其在小部件设置面板中可选择。然后开发模式会将您的实时更改与已发布的的小部件进行预览。

如果您选择使用Foundry代码仓库来存储您的源代码，请首先按照VS Code工作区或代码仓库应用程序中的“本地开发”说明来克隆仓库到您的本地机器。

开发模式环境

一旦您的开发服务器正在运行，您可以在下面概述的不同环境中预览您的小部件。

自定义小部件游乐场

自定义小部件游乐场是一个专门用于测试单个小部件的开发环境，提供：

  • 隔离测试：在不同尺寸和配置中预览小部件。
  • 参数控制：交互式控件，用于测试您定义的小部件参数。
  • 事件监控：显示来自Foundry的小部件事件和参数更新的消息日志。
  • 开发模式集成：当开发模式激活时，游乐场会显示您的开发代码。

Workshop 集成

在开发 Workshop 小部件时，开发模式不仅限于游乐场，还扩展到实际的 Workshop 应用程序。这使您可以：

  • 在小部件将被使用的完整上下文中预览小部件更改
  • 与其他 Workshop 组件进行交互测试
  • 在真实的应用程序场景中验证小部件的行为

VS Code 代码工作区集成

开发服务器可以直接在 VS Code 代码工作区的终端中运行。当工作区启动时会自动完成此操作，但也可以使用以下命令手动在终端中执行：

开发模式控制和状态

通过自定义小部件游乐场界面可以控制开发模式。这些控件表示当前状态，并允许您根据需要启用、禁用或暂停开发模式。

禁用状态（默认）

开发模式默认是禁用的。您可以通过游乐场控件来启用它，但除非您的开发服务器正在运行并为当前小部件提供了覆盖设置，否则它将保持非活动状态。

启用状态

当处于激活状态时，开发模式会显示来自您的开发服务器的内容，而不是发布的组件版本。此状态确认您的本地更改正在被预览。

启用（非活动）状态

该状态表示开发模式已启用，但当前小部件没有开发服务器覆盖，因此仍然显示发布版本。这通常发生在您的开发服务器未运行或不包含您正在查看的小部件时。

暂停状态

暂停状态临时禁用开发模式，同时在 Workshop 等主机应用程序中保持控件可见，使需要时可以轻松恢复开发。

添加和修改参数和事件

小部件的参数和事件通常定义在main.config.ts文件中。这是类型安全的，因此可以轻松地添加或修改参数和事件而不会出错。有关支持的参数类型和事件的更多信息，请参阅 参数和事件。

预览参数和事件更改

使用 @osdk/widget.vite-plugin 版本 3.34.0 或更高版本，开发模式可以在不发布新版本的情况下预览参数和事件的更改。更新后的参数和事件会出现在自定义小部件的沙盒环境中、代码工作区以及 Workshop 中。

参数和事件更改不会像代码更改一样热重载。在编辑配置文件后，重新应用开发模式以预览更改。当在本地开发时，请再次打开终端中打印的设置链接。当在代码工作区中开发时，请刷新预览面板。

不支持的功能

自定义小部件运行时不支持某些用于持久化数据的浏览器API，例如：

  • Web Storage API ↗（localStorage, sessionStorage）
  • IndexedDB API ↗

要在页面上的多个小部件之间共享状态，请使用通过宿主应用程序配置的参数（例如，Workshop中的变量）。要持久化状态，您可以使用Workshop变量的已保存状态，或把数据写入本体论。

非本体论API在自定义小部件运行时也不受支持。

自定义小部件运行时的内容安全策略（CSP）无法配置，并且按设计是限制性的；不允许任何外部请求。您可以使用 Foundry 资源来包装外部请求，例如函数和 webhook。
