模型

在 Foundry 中，模型是一种用于推理的制品，包含机器学习、预测、优化、物理模型或业务规则。在特定用例中，模型将数据相关知识进行编码，从而生成预测并赋能决策。

在 Palantir 内部开发或集成至其中的模型提供以下功能：

  • 完整的版本历史记录、细粒度的模型权限控制、自动依赖管理、模型血缘关系以及 API 管理
  • 通过《模型部署》实现实时推理的无代码托管
  • 通过《管道构建器训练模型节点》在数据管道中实现无代码批量推理
  • 通过《建模目标》应用进行模型管理、评估与部署
  • 与 Foundry 本体绑定，允许通过《Foundry 应用》、《模型函数》和《场景》基础设施实现操作化。

架构

Palantir 中的模型资源包含两个相关但独立的组件：

  1. 模型制品：保存已训练模型的模型权重或容器。
  2. 《模型适配器》：描述平台如何与模型制品交互以加载、初始化并执行模型推理的逻辑。

适配器作为 Python 库的一部分发布，以实现与存储的模型制品之间的通信。它使平台能够加载、初始化并对任何类型的模型运行推理。适配器设计灵活，可用于包装 Foundry 支持的各种模型类型：

  • 在 Foundry 中通过《代码仓库》、《Jupyter 代码工作区》或《模型工作室》训练的模型
  • 手动上传的《模型文件或检查点》
  • 《作为容器上传》的模型
