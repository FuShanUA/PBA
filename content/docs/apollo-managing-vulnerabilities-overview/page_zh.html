管理漏洞

这是高级Apollo功能， 默认情况下未启用。请联系您的Palantir代表以了解更多信息或在您的Apollo Hub上启用该功能。

Apollo通过检测产品中的漏洞并自动召回有漏洞的版本来保护您的环境安全。Apollo支持Trivy ↗进行漏洞扫描和ClamAV ↗进行病毒扫描。Apollo的风险管理流程让您全面掌握您的安全扫描情况，并帮助您根据扫描结果简化操作，同时促进开发人员与安全团队之间的沟通，明确需要修复的内容及时间，以及可能的例外情况。

在您创建一个发布版本后，Apollo 将会自动对新发布版本的 产品清单 中声明的每个容器进行漏洞扫描。如果您使用的是 apollo-cli product-release create 命令，Apollo 会将您的 chart 默认中声明的容器镜像添加到产品清单中。漏洞扫描器要求所有容器定位符都必须包含完整的容器注册表 URL 和版本标签。例如，如果您想从 Docker Hub 扫描 Postgres，则您的产品清单应包含 docker.io/library/postgres:16。

Apollo 会自动扫描：

  • 当创建新发布版本时。
  • 所有最新发布版本以及安装在实体上的发布版本。这些发布版本默认每天都会被扫描，但您可以配置扫描频率。

Apollo 支持多种与安全相关的操作流程：

  • 漏洞扫描
    • 查看漏洞和病毒
    • 解决镜像扫描失败问题
    • 基于漏洞的自动召回
    • 手动运行漏洞扫描
  • 漏洞抑制
    • 创建漏洞抑制
    • 删除漏洞抑制
  • 向发布评估添加安全信息
    • 根据正在运行的扫描来限制发布
    • 根据扫描结果进行门控发布
  • 将实体升级到CVE召回的版本

风险评分

Apollo 使用以下信息来评估漏洞：

  • CVSS 评分 ↗: 严重程度的定性度量。有五种可能的严重程度：无、低、中、高和关键。请注意，Apollo 使用 CVSS v3.x 标准。
  • EPSS 评分 ↗: 对漏洞在接下来的 30 天内被利用的概率的估计。
  • 已知利用 ↗: 由美国政府维护的目录，记录了在现实世界中已被实际利用的漏洞。

入门

  1. 将以下信息发送给您的 Palantir 代表：

    • 您的容器注册表的域名
    • 您的容器注册表凭证（只读）以便 Apollo 可以拉取容器
    • 您是否希望同时进行病毒扫描和漏洞扫描

  2. 从主 Apollo 侧边栏导航到 设置与配置 页面，并设置您所需的 权限，用于 漏洞抑制 和 漏洞 SLA 设置。

  3. 如果您的注册表位于防火墙之后，您需要将 Apollo 出站 IP 加入白名单。
