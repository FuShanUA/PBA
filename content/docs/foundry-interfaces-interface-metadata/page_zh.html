元数据参考

接口在本体论中通过以下元数据表示：

  • RID： Palantir 中每个资源的自动生成的唯一标识符。平台上的错误消息中将引用接口的 RID。
  • 图标： 用于作为视觉标识的图片和颜色，当用户查看此接口时会在应用程序中显示。接口的图标周围有虚线，以在视觉上与对象类型图标区分开来。例如，被虚线包围的建筑物图标可能用于表示 Facility 接口。
  • 显示名称： 在用户应用程序中访问此接口时显示的名称。例如，Facility 接口的显示名称可能是“Facility”。
  • 描述： 关于接口的解释性文本，任何用户都可以在用户应用程序中阅读。例如，Facility 接口的描述可能是“一个抽象的对象类型接口，用于表示航空公司设施”。
  • API 名称： 在代码中以编程方式引用接口时使用的名称。例如，Facility 接口的 API 名称可能是 facility。
  • 状态： 用于向用户和本体论构建者表明该接口在开发过程中所处的阶段。它可以是 active、experimental、example 或 deprecated。默认情况下，Facility 接口的状态为 experimental。了解更多关于 状态 的信息。
  • 可搜索： 一个布尔值，用于指定该接口是否可以被搜索。可搜索的接口允许用户一次性加载或搜索该接口的所有对象。可搜索的接口最多支持 50 种实现的对象类型，而非可搜索的接口则最多支持 1,000 种。默认情况下，Facility 接口是可搜索的。
