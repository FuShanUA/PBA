附件字段

Foundry Forms 不再是 Foundry 上数据录入或写回工作流的推荐方法。相反，应使用 Foundry 本体论 构建用户输入工作流，将相关数据结构表示为对象类型，并配置与操作 的写回交互。请参阅 Forms 概述 文档了解更多内容。

表单的 附件 字段允许受访者上传附件。

附件字段支持以下配置选项：

  • 允许多次上传： 如果启用此选项，则允许受访者向该字段上传多个附件，而不是仅一个附件。

  • 目标文件夹：

    • 此选项指定附件应上传到哪个文件夹。所有条目中此字段的附件都将存储在此文件夹中。

  • 上传模式：此字段有两种选项：

    • 默认模式：所有附件将在目标文件夹中作为独立文件。

    • 子文件夹模式：表单成功提交后，将在指定的目标文件夹内自动创建一个新文件夹。所有附件将作为独立文件存储在该子文件夹中。配置此模式需要两个额外字段：

      • 子文件夹名称：此字段的值将用作子文件夹的名称。

      • 子文件夹RID：此字段的值将自动填充为子文件夹RID。

当重新提交条目时，附件将上传到此目标子文件夹。如果在重新提交时缺少子文件夹的RID，将创建一个新子文件夹。但是，新子文件夹的名称和RID不会自动保存到此字段中。

要使用子文件夹模式，您必须在表单中创建一个额外的文本字段。它将是一个空字段，将在上方选择为子文件夹RID。为了避免受访者混淆，请将此字段设为只读或隐藏。

Destination folder 和 Upload mode 不会受到 更新配置转换 的影响。
