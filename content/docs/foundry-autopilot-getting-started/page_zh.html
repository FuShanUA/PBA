入门

要开始使用，请从以下任一入口点创建一个Autopilot工作区：

  • 在Autopilot中直接操作：从应用门户打开Autopilot并选择对象类型（可选地选择相关自动化以导入），或直接选择自动化。
  • 从 自动化 中：在自动化概述页面上，在操作菜单中选择在Autopilot中打开。

设置您的Autopilot工作区

Autopilot会根据您选择的起始对象类型或自动化自动生成一个初始工作区。工作区顶部的 警报 会引导您完成配置完整系统视图的步骤。

工作区配置

  1. 查看并添加自动化: 当你打开你的工作台时，Autopilot 会提醒你有关与你的工作流程相关的自动化任务，这些任务尚未包含在你的工作台中。按照提示添加相关自动化以获得完整的系统视图。
  2. 自定义状态: Autopilot 会根据工作台中的自动化任务推断并自动生成状态（你的看板上的列）。你也可以添加并手动定义自己的状态。
    • 在 状态 侧边栏中，拖动状态以重新组织看板。



  3. 启用 项目范围模式 以获得完整可见性: 为了最全面地查看你的自动化系统，为你的自动化任务启用项目范围模式。此设置允许所有工作台用户查看自动化执行事件，并发现自动化任务之间的依赖关系。

项目范围模式会扩展权限，使自动化关系对项目中的所有用户可见。Autopilot 可能无法为由其他用户拥有的用户范围自动化推断关系。

  1. 为对象类型启用编辑历史记录： 为您的对象类型启用编辑历史记录，以跟踪更改并在 Autopilot 中查看详细的对象历史记录。
  2. 确保自动化至少运行过一次： Autopilot 依赖于历史执行来发现自动化之间的关系；如果自动化从未运行或自动化已安排运行，Autopilot 无法推断流程中的依赖自动化。
  3. 为每个资源启用跟踪日志： 跟踪日志会显示每次自动化运行的详细执行情况。请联系您的注册管理员以启用日志。

如需更详细的日志，请考虑在您的 TypeScript 函数中使用 console 或在您的 Python 函数中使用 import logging。

  1. 配置对象卡片属性和对象预览： 定义哪些对象属性出现在看板卡片上，设置属性的条件格式，并在本体论管理器中自定义对象视图。当可用时，Autopilot 将在侧边栏中渲染对象的面板视图。有关与本体论管理器配合使用的更多信息，请参阅 Autopilot 集成文档。
