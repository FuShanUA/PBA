运行评估套件

可以从不同的位置运行评估套件，包括 AIP Logic 的 AIP Evals 侧边栏和 AIP Evals 应用程序。您可以选择运行完整的评估套件，或仅执行单个测试用例。后者对于调试和快速函数迭代很有帮助。

完整评估套件运行

在 AIP Logic 中，导航到 AIP Evals 侧边栏面板并选择 运行评估套件。如果您有未保存的更改，将显示 保存并运行，以确保在运行评估套件之前保存更改。

另外，您也可以从 AIP Evals 应用程序中运行评估套件。要打开应用程序，请在 AIP Logic 侧边栏中选择 查看，或者从文件系统中打开评估套件。在 Evals 应用程序中，您可以通过选择右上角的 运行评估套件 来运行评估套件。

运行配置

在运行评估套件时，有多种配置选项可供选择。要访问运行配置选项，请点击“运行评估套件”旁边的齿轮图标。这将打开一个包含以下选项的对话框：

要测试的功能

评估套件可以针对在 AIP Logic 中编写的函数以及在 代码仓库 中编写的函数进行运行。根据该函数的来源，您可以针对该函数的不同版本：

  • AIP Logic 函数： 最后保存的（默认）和已发布的版本。
  • 非 AIP Logic 函数： 已发布的版本。
测试多个功能

如果你的评估套件配置了多个目标函数，你可以从AIP Evals中同时对多个函数运行评估。选择 测试多个函数 以切换到多目标模式，然后选择要包含在运行中的目标。

在多目标模式下无法进行实验配置。

输入映射

需要提供输入映射，将评估套件列中的值映射到被评估函数所需的输入。您将能够选择与预期函数输入类型匹配的套件列。

通常，评估套件列名和函数输入会匹配，但这不是必须的。

执行模式

在运行评估套件时，您可以在两种执行模式之间进行选择： 用户范围执行 和 项目范围执行。用户范围执行是默认模式，使用启动运行的用户的权限来执行评估套件。

用户范围执行：

  • 套件将使用用户权限执行。
  • 结果仅对你可见，并将在24小时后被删除。
  • 结果将 不会 被保存到 结果数据集 中。

项目范围执行 [Beta]：

Beta

项目范围执行处于 Beta 开发阶段，可能在您的注册中不可用。在积极开发期间，功能可能会发生变化。如果您遇到此执行模式的问题，请尝试改用用户范围执行来运行您的评估套件。

  • 评估套件以项目范围执行。这意味着在函数或评估器执行期间使用的所有资源都需要导入到同一项目中。
  • 运行结果将对所有具有项目访问权限的人可见。
  • 结果将被永久保存。
  • 如果已配置，结果将被写入到 结果数据集 中。

迭代次数

您可以指定每个测试用例应运行的次数。由于大语言模型（LLM）的非确定性特性，我们建议至少对基于LLM的功能运行测试用例三次。每次迭代的结果将被聚合以提供对该功能性能的全面概述。

像评分器这样的数值评估器的高方差可能表明测试用例和评估器没有意义，需要进一步优化。

测试并行化

默认情况下，同时执行十个测试用例。您可以调整并行测试用例执行的数量以优化您的评估套件运行的性能。当遇到速率限制时，减少该数量可能会有所帮助。

运行元数据

除了自动捕获的运行元数据，如使用的分支、版本或模型之外，您还可以为评估套件的运行添加自定义元数据。 这些元数据以键值对的形式提供，可用于区分评估套件运行历史中的不同运行。

查看结果

在评估套件运行完成后，您可以通过选择最新结果部分中的卡片来查看结果。这将打开结果视图，在这里您可以查看评估套件运行的聚合指标以及每个单独测试用例的结果。每个指标的通过或失败状态是根据您的配置目标（布尔型或数值型，方向和阈值）显示的。当鼠标悬停在一个测试用例上时，您可以在测试用例结果的右下角看到调试按钮。选择它将在新标签页中打开该测试用例的调试视图。调试视图将显示对该测试用例执行的逻辑函数和评估器的相关步骤。

此外，结果视图还提供了通过选择点击比较另一个运行来比较不同运行的能力。这将打开另一个运行，与当前运行并排显示，使您能够比较两个运行的结果。默认情况下，查看差异切换按钮将被启用，导致两个运行之间的输出差异会被高亮显示。

单个测试用例执行

在运行单个测试用例时，根据您设定的目标（布尔值或数值、方向和阈值），每个指标的passed或failed状态也会显示出来。

AIP Logic中的AIP Evals侧边栏提供了一种快速运行单个测试用例的方法。这在构建您的Logic函数或在完整评估套件运行后对失败的测试用例进行迭代时特别有用。

要运行单个测试用例，请在侧边栏中选择测试用例名称旁边的播放图标。这将立即执行测试用例，并打开调试器侧边栏面板。在此处，您将能够跟踪对测试用例结果运行的 Logic 函数和评估器的执行情况。此外，测试用例将在侧边栏中被标记为已执行。

执行后，侧边栏和结果面板将根据您配置的目标显示每个指标是否通过或失败。
