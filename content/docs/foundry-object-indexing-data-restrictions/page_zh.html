数据限制

对象存储 V2 (OSv2) 强制执行数据限制，以确保输入到本体论的数据质量，提供更确定的行为，并提高平台上的可读性。这些限制在索引过程中进行验证。对于由批量数据源支持的对象类型，违反这些限制会导致索引作业失败。对于由流数据源支持的对象类型，违反这些限制的记录将被丢弃。

主键和唯一性

OSv2 为数据源强制执行唯一的对象主键。如果在单个事务中有重复的主键，索引将会失败并抛出错误。如果在事务之间有重复的主键，则以后的事务中的版本将被使用。

OSv2 防止某些数据类型用作主键，以鼓励本体论建模的最佳实践。以下类型 不能 作为主键使用： 用作主键：

  • 地理点位
  • 地理形状
  • 数组
  • 时间序列属性
  • 实数类型（十进制、双精度、浮点）

属性类型限制

  • OSv2 在每次同步时强制要求数据源模式和对象类型模式之间的数据类型一致性。对于某个属性不兼容的数据类型会导致构建失败。
  • 当更改现有属性的基础类型（例如，从Double到Integer）时，该属性的所有现有值必须严格与目标类型兼容。如果任何数据条目包含与新类型不兼容的值（如将类型更改为整数时出现的小数，或货币符号），迁移将失败并出现类似A property could not be cast to the new type的错误。如果存在不兼容的值，模式迁移不会继续进行，并且迁移过程无法自动清理或转换这些值。
  • OSv2 不允许NaN或±infinity作为属性值。
  • 空字符串在 OSv2 中不允许；在 OSv1 中，空字符串会被静默转换为 null。
  • Lat, Long 应该是一个用逗号分隔的字符串，没有括号，例如 -29.123, 150.982。
  • OSv2 不允许具有嵌套数组的属性。
  • OSv2 不允许具有数组数据类型的属性在数组内包含 null 元素。
  • OSv2 在受限视图数据源的细粒度权限策略中支持 Not 条件，其中否定字段是一个集合并且有 非空 约束。这可以通过在本体论管理器中将相关属性标记为 必需 来配置。
  • OSv2 对地理点属性有更严格的验证。

属性大小限制

OSv2 对单个属性施加大小限制，以确保可靠的索引性能和稳定性。这些限制解决了处理大属性值时可能出现的序列化约束和内存压力问题。

属性类型最大大小
字符串属性（String）12 MB
数组属性（Array）100,000 个元素

超过这些限制的属性会导致索引作业失败。这些限制最初适用于由批处理数据源支持的对象类型；将来会对具有流数据源的对象类型应用类似的限制。

大型数据建议

  • 大型字符串属性：如果需要存储超过12 MB的数据，请改用 媒体引用属性（media reference property）。媒体引用允许您将大文件或二进制内容与对象关联，而不会影响索引性能。
  • 大型数组：如果需要建模会超出100,000 个元素的关系，请考虑改用 链接类型（link types） 而不是数组属性。链接提供了一种更可扩展且可查询的方式来表示对象之间的关系。
