导出环境

您可以使用导出管道来导出Apollo环境。在某些配置中，您可能希望在一个不同的Apollo Hub中定义环境，而不是该环境所连接的Hub。这在环境连接到一个不为所有用户所访问的Hub时很有用，但这些用户仍需要管理环境的配置。在本指南中，环境指的是环境及其所有内容，包括实体、配置和设置。

在某个Apollo Hub中定义并连接到另一个Hub的环境称为远程环境。与连接环境相比，它们的功能子集有限，其中代理程序报告到Hub。您可以在环境所连接的目标Hub上访问完整功能集。

了解有关源Hub和目标Hub的更多信息。

导出一个环境

创建远程环境

首先，您需要创建一个远程环境。从您的源中心（Source Hub）进入 环境创建对话框，选择高级设置，并将您的环境设置为远程。

从源中心（Source Hub）中，您可以像连接环境一样编辑远程环境的配置。但是，您将无法查看环境中代理程序向Apollo报告的信息，如计划和健康状态。您只能在与该环境连接的中心上查看这些信息。

您可以以与连接环境相同的方式在远程环境中安装产品，可以通过 安装模块 或 安装单个实体。在添加了您希望用于远程环境的相关实体和配置后，现在可以将其导出。

创建导出管道

接下来，您可以使用 选择环境 数据选择规则 创建一个新的 导出管道 来导出远程环境。

管理已导入的环境

在环境导入时的更改请求

当一个包包含环境更改时，Apollo会创建一个更改请求以在应用更改之前进行审核。一些更改，例如对网络安全规则的修改，需要手动审批。您可以从导入历史记录中查看每个包的审批状态。

了解更多关于环境导入时的更改请求信息。

配置权限

在导入环境时，初始状态下不会为任何团队授予该环境的任何权限。 全局环境管理员可以从目标中心为该环境授予团队权限。有关更多信息，请参阅 环境和实体的角色。

编辑远程环境的配置

从目标中心，远程环境的配置为只读。默认情况下，配置更改（编辑环境设置或配置，或实体的配置覆盖）必须在源中心进行，并作为包导入。如果环境配置或实体配置过于敏感，无法在源中心显示，您可以 更改配置的可编辑性 以便在目标中心进行编辑。

连接和操作环境

除了某些配置是只读的之外，导入的环境的行为与其他已连接的环境相同。您可以 生成一个清单 来连接并操作它。

高级：管理远程环境的配置

本节将介绍编辑和管理远程环境和实体的环境配置和实体配置覆盖的工作流程。您可以使用这些工作流程确保敏感属性不会从源中心传输到目标中心。相反，这些配置值将保留在目标中心的私有网络中。

初始状态

在导入远程环境后，您只能从源中心编辑和管理该环境及其实体的配置。通过导出和导入一个包，配置更改将传递到目标中心。在目标中心，配置选项卡将显示配置编辑被锁定的信息。

从源中心到目标中心更改配置可编辑性

源中心控制配置的可编辑性，这意味着哪些中心可以对远程环境的配置进行编辑。要将特定环境配置或实体配置覆盖的配置编辑从源中心切换到目标中心，请导航到源中心中该环境或实体的配置选项卡。然后选择更改配置可编辑性。选择在远程中心进行编辑选项，然后选择应用更改。

为配置可编辑性构建并传输一个包

一旦您在目标中心启用了某个配置的可编辑性，Apollo 将会立即禁用从源中心对该配置的编辑。在构建并导入一个包以将此更改传达给目标中心之前，您无法在源中心或目标中心的 Apollo 中心中编辑该配置。

在目标中心编辑配置

一旦Bundle被导入目标中心，Apollo将在目标中心启用配置编辑功能，并锁定源中心的文件。

从目标中心更改配置可编辑性到源中心

我们不建议从目标中心将配置的可编辑性改回源中心。从目标中心对配置文件进行的更改在从源中心构建并传输一个Bundle后将会丢失。

要安全地从目标中心将配置的可编辑性改回源中心，您可以：

  1. 确保在此过程中不会构建任何Bundle。这可能需要您暂停任何自动的Bundle构建。
  2. 在更改配置可编辑性对话框中，将配置可编辑性从目标端点更改为源端点。
  3. 更新源端点对此配置的表示，以匹配目标端点的表示。
  4. 恢复捆绑包的创建，并将包含配置可编辑性元数据的下一个捆绑包传输到目标端点。

高级：链接多个端点

一些复杂的端点设置可能需要在端点 A 中创建一个环境，经过端点 B 并连接到第三个端点 C。上述工作流程在这种情况下将不起作用，因为从端点 A 导出的环境将在端点 B 中作为已连接的环境导入，而无法导出到端点 C。如果端点 A 生成的捆绑包不允许直接导入到端点 C，而总是需要先经过端点 B，则可能需要这种设置。

在這種情況下，Hub A 的導出管道必須指定 Environment 必須作為 遠程 被導入到 Hub B 中。

然後可以從 Hub B 导出 Environment，其中 Hub B 的导出管道应指定该 Environment 将作为 本地 导入以在 Hub C 中创建一个连接的 Environment。
