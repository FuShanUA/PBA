设置智能体

智能体是一个可下载的程序，安装在您的组织网络内，并通过 Foundry 的 数据连接界面进行管理。智能体能够连接组织网络内的不同数据源。它们用于启用智能体代理出站策略，为私有托管的数据源提供网络连接；并启用智能体工作线程连接，以便从这些数据源读取数据，并使用受限访问令牌将其安全摄取至 Foundry。

本指南将引导您完成创建智能体所需的步骤。首先，请完成以下操作：

  1. 登录 Palantir 后，通过左侧侧边栏进入数据连接。
  2. 选择智能体选项卡。
  3. 点击右上角的新建智能体。

如果看不到创建新智能体的选项，您可能没有所需的权限角色。详细了解如何在控制面板中管理智能体创建工作流。

智能体运行起来后，若要将数据源连接至 Foundry，您必须获取源系统的凭证，以便智能体能够安全地读取数据。根据您组织的网络设置，您可能还需要配置网络设置，以允许智能体访问源系统。

请查看以下章节，开始设置您的智能体：

设置

创建智能体主机

为了让智能体程序成功运行，必须将其托管在合适的环境中（理想情况下是使用 Linux 作为操作系统的环境）。

Foundry 智能体最常用的托管方法是在云环境中配置 Linux 虚拟机（VM）。例如，您可以在 AWS、Azure 或 GCP 中配置 Linux VM，也可以将智能体托管在组织自有的 Linux 服务器上。需要注意的是，虽然可以在 Windows 上托管 Foundry 智能体，但 Palantir 不建议这样做，只有在无法在 Linux 环境中托管时才应考虑此方案。

用于运行智能体的主机应专用于运行单个 Foundry 智能体，不得与任何其他服务或进程共存。 不支持在同一台机器上运行多个 Foundry 智能体。

找到合适的智能体托管位置后，下一步是确保主机满足 Foundry 智能体正常运行所需的硬件和操作系统要求。这些要求包括：

  • 64 位 Linux或其他 Linux 操作系统（推荐 RHEL 8 或更高版本、Ubuntu 22.04 或更高版本，或同等版本）

    • 智能体运行在专为 Linux/x86-64 编译的自带 JDK 上。如有必要（例如在 AWS Graviton 或其他基于 ARM 的 CPU 上运行时），可以通过修改 service/bin/launcher-static.yml 中的 javaHome 值，在独立的 JDK 上运行智能体。

      我们通常不建议在独立的 JDK 上运行智能体，且未来可能不再提供对此方式的支持。

  • 4 个 CPU核心

  • 16 GB内存

  • 500GB可用磁盘空间挂载于 /opt（推荐使用 SSD）

推荐的限制如下：

  • 核心转储文件大小：硬限制和软限制均为 0
  • 打开文件数：硬限制和软限制均为 262144
  • 运行进程数：硬限制和软限制均为 65536
  • 栈大小：硬限制和软限制均为 32768 (KB)
  • 最大锁定内存：硬限制和软限制均为“无限制”

配置智能体网络访问

智能体仅向 Foundry 发起出站连接。要成功建立网络访问，您必须允许以下操作：

  • 智能体主机的出站流量
  • Foundry 的入站流量

允许出站和入站流量后，通过在智能体主机上执行以下命令，验证主机能否与 Foundry 虚拟私有云（VPC）通信：


```bash
curl -s https:///magritte-coordinator/api/ping > /dev/null && echo pass || echo fail
```

如果一切运行正常，您将看到输出 `pass`；输出 `fail` 则表示与 Foundry VPC 的连通性测试未完成。

#### 智能体主机的出站流量

智能体需要网络出站权限才能通过互联网访问 Foundry VPC。如果您的网络默认不允许出站流量，可能需要进行特定配置，以允许从智能体（及/或其主机）到 Foundry 实例的出站连接，例如开放防火墙或为出站流量配置代理。

您可以从数据连接的智能体设置工作流中的“服务器设置”选项卡复制 Foundry 的域名和端口，以便正确配置出站网络访问。



#### Foundry 中的入站流量

Foundry 必须允许来自您服务器 IP 的入站流量。您可以通过《控制面板中的网络入站页面》管理入站规则。在已批准的入站规则之外，您的 Foundry 域名将无法从外部访问。



### 使用防火墙保护智能体主机

我们强烈建议在智能体主机上配置防火墙，以监控网络流量并将其限制在绝对必要的目标地址。请务必确保智能体主机仍能与 Foundry 通信。可用的防火墙和监控选项取决于您运行智能体的操作系统以及您组织的安全最佳实践。

### 设置自动重启

如果未设置自动重启，只要智能体崩溃或智能体主机重启，就会出现服务中断。

要在智能体管理器崩溃时为其设置自动重启，请以具有创建定时任务权限的用户身份，在虚拟机或机器终端上从智能体管理器的服务目录运行命令 `${AGENT_MANAGER_DIR}/service/bin/auto_restart.sh`。

如果需要停止自动重启（例如在升级智能体管理器时），可以通过运行 `${AGENT_MANAGER_DIR}/service/bin/auto_restart.sh clear` 来实现。

### 将智能体资源保存到项目中

接下来，您必须为新智能体命名，并选择一个项目将其保存其中。在 Foundry 中，智能体被视为一种资源，将其保存到项目中可实现高度可配置的权限管理。

我们建议创建一个新项目来存储您的智能体。

Foundry 中的权限是一个涉及面广的主题。如果想了解更多信息，您可以参考以下资源：

- 《管理智能体权限的建议》
- 《关于如何端到端设置数据管道权限的指南》
- 《关于智能体权限如何与项目和角色交互的完整参考》


下载并安装智能体


为智能体配置好硬件后，下一步是从 Foundry 下载智能体软件并将其安装到主机上。请按照平台内指南中的步骤，在主机上下载安装包、解压并启动智能体。


如需配置代理，详情请参阅《代理配置文档》。


智能体成功启动后，请按照步骤配置自动升级，以确保智能体保持最新状态。


后续步骤


创建、安装并启动智能体后，请前往数据连接中的智能体页面，在此配置并监控智能体的权限、健康状态和连通性。


智能体设置完成后，您可以继续设置数据源，将智能体与您组织的数据源建立连接。
