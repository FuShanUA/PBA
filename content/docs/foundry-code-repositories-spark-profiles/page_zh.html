Spark profiles

仓库设置用于指定要在仓库中使用的 Spark profiles。在仓库设置中配置好 Spark profile 后，即可在代码中使用该 profile。

在应用特定 profile 之前，必须先将其导入到项目中。Profile 分为以下两种类型：

  • 无限制 Profile：所有用户均可导入到仓库中的 profile。
  • 受限 Profile：仅管理员可导入的 profile。

在仓库中已启用的 Spark profile，可在 Settings 标签页下的 Spark 区域中的 Enabled profiles 中查看。

导入 Spark Profiles

若要在转换作业中使用 Spark profile，必须先将该 profile 导入到包含该作业的项目中，否则在尝试发布转换时 Checks 将会失败。

可以通过代码仓库编辑器中的 Spark 配置标签页，浏览并将 Spark profile 导入到项目中。

要将 profile 导入项目，请转到 Settings 标签页并选择 Spark。点击 Add profiles 在下拉菜单中查找所需的 profile。将鼠标悬停在所需的 profile 上：

  • 如果是无限制 profile，可以点击 Import 将其导入到项目中。
  • 如果是受限 profile，其旁边可能会显示一个锁状图标。

默认情况下，任何资源管理管理员均具备导入受限 Spark profile 的必要权限。此外，可能还存在一个名为 spark-profile-admins 的用户组，同样拥有相应的权限。普通用户需请求管理员将 profile 导入其项目后方能使用。

已启用的 Spark Profiles

项目已启用的所有 profile 均可在项目摘要侧边栏的 References 面板中查看。导入到仓库后，profile 会自动作为整个项目的引用添加，并成为该项目中所有仓库的“已启用 profile”。
