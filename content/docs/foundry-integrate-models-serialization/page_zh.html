模型序列化

为了在工作流之间重用模型，Palantir 需要能够反序列化存储的权重。由于序列化过程可能因模型类型而异，因此模型适配器的作者应详细说明如何进行此操作。请注意，对于容器模型，序列化通常是容器生命周期的一部分，或者对于外部模型，其权重由外部托管，这并不是需要考虑的问题。

自动序列化

为了简化平台内模型权重的序列化和反序列化，Palantir 提供了一些默认的序列化方法用于保存和加载模型：

  • Dill（序列化）
  • Cloudpickle（云打包）
  • JSON（JavaScript 对象表示法）
  • YAML（YAML Ain't Markup Language）
  • Hugging Face（Hugging Face）
  • PyTorch（PyTorch）
  • Tensorflow
  • XGBoost
  • Spark MLlib

如何使用默认的序列化程序

可以通过在 模型适配器（model adapter） 上使用 __init__ 方法进行注释，从而使用默认的序列化程序。默认情况下，Palantir 将自动使用 Dill 序列化程序保存和加载每个 __init__ 方法的输入。通过为 @auto_serialize 注释提供参数，您可以指定用于 __init__ 方法的每个参数的序列化方法。以这种方式提供序列化参数时，您的 __init__ 方法中的每个参数都必须在您的 @auto_serialize 定义中具有对应的参数。

Python 依赖项

请注意，为了避免可能的冲突，palantir_models.serializers 包不包含对以下序列化框架的依赖项，除了 dill。当使用 palantir_models.serializers 包中的其他序列化器时，您仍必须添加对相关序列化包的依赖。

使用 auto_serialization 的示例模型定义

下面是一个有效的 Python 转换（Transform），它训练并发布一个模型。由于 AutoSerializationAdapter 使用了没有参数的 @auto_serialize 装饰器来序列化模型，因此 model 方法的 __init__ 参数是通过 Dill 进行序列化的。

请注意，当使用 palantir_models 的版本低于 0.1536.0 时，所有序列化器都需要作为 @auto_serialize 装饰器的参数进行定义。

Copied!
1from transforms.api import transform
2from palantir_models.transforms import ModelOutput
3
4from sklearn.linear_model import LinearRegression
5import numpy as np
6
7
8@transform(
9    model_output=ModelOutput("/Foundry/path/to/model_asset"),
10)
11def compute(model_output):
12    x_values = [i for i in range(100)]
13    y_values = [2 * i for i in x_values]
14
15    X = np.array(x_values, dtype=np.float32).reshape(-1, 1)
16    y = np.array(y_values, dtype=np.float32).reshape(-1, 1)
17
18    model = LinearRegression()
19    model.fit(X, y)
20
21    model_output.publish(
22        model_adapter=AutoSerializationAdapter(
23            model,
24            {'prediction_column': 'prediction'}
25        )
26    )
27
28import palantir_models as pm
29
30class AutoSerializationAdapter(pm.ModelAdapter):
31
32    @pm.auto_serialize
33    def __init__(self, model):
34        self.model = model
35
36    @classmethod
37    def api(cls):
38        inputs = {"df_in": pm.Pandas()}
39        outputs = {"df_out": pm.Pandas()}
40        return inputs, outputs
41
42    def predict(self, df_in):
43        df_in['prediction'] = self.model.predict(df_in)
44        return df_in

使用多个自动序列化的示例模型定义

使用多个 auto_serialization 的示例模型定义

以下的模型适配器为它的 __init__ 方法的每个输入定义了不同的序列化方法。

Copied!
1import palantir_models as pm
2from palantir_models.serializers import DillSerializer, JsonSerializer
3
4class AutoSerializationAdapter(pm.ModelAdapter):
5
6    @pm.auto_serialize(
7        model=DillSerializer(),
8        config=JsonSerializer()
9    )
10    def __init__(self, model, config={}):
11        self.model = model
12        self.prediction_column = config['prediction_column'] if 'prediction_column' in config else 'prediction'
13
14    @classmethod
15    def api(cls):
16        inputs = {"df_in": pm.Pandas()}
17        outputs = {"df_out": pm.Pandas()}
18        return inputs, outputs
19
20    def predict(self, df_in):
21        df_in[self.prediction_column] = self.model.predict(df_in)
22        return df_in

默认序列化程序

Dill

通过调用 dill.dump 并使用 dill.load 将对象保存和加载到磁盘，palantir_models.serializers.DillSerializer 可以使用 dill ↗ 对 Python 对象进行序列化。

DillSerializer 类可用于序列化许多 Python 对象，包括 scikit-learn 和 statsmodels。

Cloudpickle

通过调用 cloudpickle.dump 和 cloudpickle.load 将对象保存和加载到磁盘，palantir_models.serializers.CloudPickleSerializer 可以使用 Cloudpickle ↗ 对 Python 对象进行序列化。

CloudPickleSerializer 类可用于序列化许多 Python 对象，包括 scikit-learn 和 statsmodels。

JSON

palantir_models.serializers.JsonSerializer 将通过在您的 Python 字典上调用 yaml.safe_dump 和 json.safe_load 来将 Python 字典序列化为 JSON。

YAML

palantir_models.serializers.YamlSerializer 将通过在您的 Python 字典上调用 yaml.safe_dump 和 yaml.safe_load 来将 Python 字典序列化为 YAML。

Hugging Face

palantir_models.serializers 库目前为 Hugging Face 模型 ↗ 提供了三种默认的序列化器。 : HfPipelineSerializer, HfAutoTokenizerSerializer, 和 HfAutoModelSerializer. 三个 Hugging Face 序列化器都需要将 transformers 库作为依赖项添加到你的 Python 环境中。

HfPipelineSerializer

palantir_models.serializers.HfPipelineSerializer 将使用 save_pretrained 对 transformers.pipeline ↗ 对象进行序列化，并在加载时重新实例化你的管道对象。

HfPipelineSerializer 有一个必需的字符串参数，表示管道的 任务 ↗。任何其他 kwargs 都将用于加载管道。

Copied!
1import palantir_models as pm
2from palantir_models.serializers import HfPipelineSerializer
3from transformers import pipeline
4import pandas as pd
5
6class HFNerAdapter(pm.ModelAdapter):
7
8    @pm.auto_serialize(
9        pipeline=HfPipelineSerializer("ner"),
10    )
11    def __init__(self, pipeline):
12        self.pipeline = pipeline
13
14    @classmethod
15    def api(cls):
16        inputs = {"df_in" : pm.Pandas([("text", str)])}
17        outputs = {"df_out" : pm.Pandas([("text", str), ("generation", str)])}
18        return inputs, outputs
19
20    def predict(self, df_in: pd.DataFrame):
21        result = self.pipeline(df_in["text"].tolist())
22        df_in["generation"] = result
23        return df_in

HfAutoModelSerializer 和 HfAutoTokenizerSerializer

palantir_models.serializers.HfAutoModelSerializer 和 palantir_models.serializers.HfAutoTokenizerSerializer 将使用 save_pretrained 和 from_pretrained 对 transformers.AutoModel 和 transformers.AutoTokenizer 模型进行序列化。

注意，对于某些模型和分词器，建议使用特定的模型或分词器类，而不是通用的类；在这些情况下，可以将特定类作为第一个 model_class 参数传递给序列化器。

Copied!
1import palantir_models as pm
2from palantir_models.serializers import HfAutoModelSerializer, HfAutoTokenizerSerializer
3from transformers import AutoModelForSeq2SeqLM
4import pandas as pd
5import torch
6
7
8class HFTextGenerationAdapter(pm.ModelAdapter):
9
10    @pm.auto_serialize(
11        model=HfAutoModelSerializer(AutoModelForSeq2SeqLM),
12        tokenizer=HfAutoTokenizerSerializer()
13    )
14    def __init__(self, model, tokenizer):
15        self.model = model
16        self.tokenizer = tokenizer
17
18    @classmethod
19    def api(cls):
20        inputs = {"df_in" : pm.Pandas([("text", str)])}
21        outputs = {"df_out" : pm.Pandas([("text", str), ("generation", str)])}
22        return inputs, outputs
23
24    def predict(self, df_in: pd.DataFrame):
25        input_ids = self.tokenizer(
26            df_in["text"].tolist(),
27            return_tensors="pt",
28            padding=True
29        ).input_ids
30
31        outputs = self.model.generate(input_ids)
32        result = self.tokenizer.batch_decode(outputs, skip_special_tokens=True)
33
34        df_in["generation"] = result
35        return df_in

PyTorch

palantir_models.serializers.PytorchStateSerializer 将使用 torch.nn.Module 和 torch.save 序列化一个 torch.nn.Module ↗，以便将您的对象保存和加载到磁盘。

TensorFlow

palantir_models.serializers.TensorflowKerasSerializer 将使用 tensorflow.keras.Model 和 obj.save 序列化一个 tensorflow.keras.Model ↗，以便将您的模型保存和加载到磁盘。当您的模型被反序列化时，Foundry 将调用 obj.compile()。

XGBoost

palantir_models.serializers.XGBoostSerializer 将使用 save_model 和 load_model 序列化 xgboost.sklearn.XGBModel ↗ 以将您的模型保存和加载到磁盘。XGBModel 类是许多 XGBoost 模型的基类，包括 xgboost.XGBClassifier、xgboost.XGBRegressor 和 xgboost.XGBRanker。

Spark MLlib

palantir_models.serializers.SparkMLAutoSerializer 将使用 pyspark.ml.PipelineModel.write().overwrite().save() 和 pyspark.ml.PipelineModel.read().load() 方法序列化 pyspark.ml.pipeline.Pipeline ↗ 对象。鉴于任何 Spark MLlib 模型都可以包装在 Pipeline 对象中，此序列器旨在作为 Spark 的通用序列器。
