模型的序列化

为了在不同工作流中复用模型，Palantir 需要能够反序列化已存储的权重。由于序列化过程可能因模型类型而异，模型适配器的作者应详细说明具体的执行方式。需要注意的是，对于容器模型（其序列化通常是容器生命周期的一部分）或外部模型（其权重托管在外部），此问题无需考虑。

自动序列化

为了简化平台内模型权重的序列化与反序列化操作，Palantir 提供了一系列默认的序列化方法，用于保存和加载模型：

  • Dill
  • Cloudpickle
  • JSON
  • YAML
  • Hugging Face
  • PyTorch
  • Tensorflow
  • XGBoost
  • Spark MLlib

如何使用默认序列化器

可以通过在模型适配器的 __init__ 方法上添加 @auto_serialize 注解来使用默认序列化器。默认情况下，Palantir 会使用 Dill 序列化器自动保存和加载 __init__ 方法中的每一个输入参数。通过向 @auto_serialize 注解传递参数，你可以为 __init__ 方法中的各个参数指定所使用的序列化方法。以这种方式提供序列化器参数时，__init__ 方法中的每个参数都必须在 @auto_serialize 定义中具有对应的参数。

Python 依赖项

请注意，为避免潜在冲突，palantir_models.serializers 包不包含对以下序列化框架的依赖（dill 除外）。在使用 palantir_models.serializers 包中的其他序列化器时，你仍需添加对相关序列化包的依赖。

使用 auto_serialization 的模型定义示例

下面是一个有效的 Python 转换，用于训练并发布模型。由于 AutoSerializationAdapter 在使用 @auto_serialize 装饰器进行序列化时未传入任何参数，因此 __init__ 方法的 model 参数会通过 Dill 进行序列化。

请注意，如果使用的 palantir_models 版本低于 0.1536.0，所有序列化器都必须作为参数定义在 @auto_serialize 装饰器中。

2from palantir_models.transforms import ModelOutput
3
4from sklearn.linear_model import LinearRegression
5import numpy as np
6
7
8@transform(
9    model_output=ModelOutput("/Foundry/path/to/model_asset"),
10)
11def compute(model_output):
12    x_values = [i for i in range(100)]
13    y_values = [2 * i for i in x_values]
14
15    X = np.array(x_values, dtype=np.float32).reshape(-1, 1)
16    y = np.array(y_values, dtype=np.float32).reshape(-1, 1)
17
18    model = LinearRegression()
19    model.fit(X, y)
20
21    model_output.publish(
22        model_adapter=AutoSerializationAdapter(
23            model,
24            {'prediction_column': 'prediction'}
25        )
26    )
27
28import palantir_models as pm
29
30class AutoSerializationAdapter(pm.ModelAdapter):
31
32    @pm.auto_serialize
33    def __init__(self, model):
34        self.model = model
35
36    @classmethod
37    def api(cls):
38        inputs = {"df_in": pm.Pandas()}
39        outputs = {"df_out": pm.Pandas()}
40        return inputs, outputs
41
42    def predict(self, df_in):
43        df_in['prediction'] = self.model.predict(df_in)
44        return df_in

1import palantir_models as pm

3
4class AutoSerializationAdapter(pm.ModelAdapter):

17        outputs = {"df_out": pm.Pandas()}
18        return inputs, outputs
19
20    def predict(self, df_in):
21        df_in[self.prediction_column] = self.model.predict(df_in)
22        return df_in

默认序列化器

Dill

palantir_models.serializers.DillSerializer 会调用 dill.dump 和 dill.load，通过 dill ↗ 序列化 Python 对象，将其保存至磁盘并从中加载。

DillSerializer 类可用于序列化多种 Python 对象，包括 scikit-learn 和 statsmodels。

Cloudpickle

palantir_models.serializers.CloudPickleSerializer 会调用 cloudpickle.dump 和 cloudpickle.load，通过 Cloudpickle ↗ 序列化 Python 对象，将其保存至磁盘并从中加载。

CloudPickleSerializer 类可用于序列化多种 Python 对象，包括 scikit-learn 和 statsmodels。

JSON

palantir_models.serializers.JsonSerializer 会对 Python 字典调用 yaml.safe_dump 和 json.safe_load，将其序列化为 JSON 格式。

YAML

palantir_models.serializers.YamlSerializer 会对 Python 字典调用 yaml.safe_dump 和 yaml.safe_load，通过 JSON 序列化该字典。

Hugging Face

palantir_models.serializers 库目前为 Hugging Face 模型 ↗ 提供了三个默认的序列化器：HfPipelineSerializer、HfAutoTokenizerSerializer 和 HfAutoModelSerializer。所有这三个 Hugging Face 序列化器都要求将 transformers 库作为依赖项添加到您的 Python 环境中。

HfPipelineSerializer

palantir_models.serializers.HfPipelineSerializer 会使用 save_pretrained 序列化 transformers.pipeline ↗ 对象，并在加载时重新实例化您的 pipeline 对象。

HfPipelineSerializer 具有一个必需的字符串参数，用于表示流水线的任务 ↗。任何额外的 kwargs 将用于加载流水线。

1import palantir_models as pm
2from palantir_models.serializers import HfPipelineSerializer
3from transformers import pipeline
4import pandas as pd
5
6class HFNerAdapter(pm.ModelAdapter):
7
8    @pm.auto_serialize(
9        pipeline=HfPipelineSerializer("ner"),
10    )
11    def __init__(self, pipeline):
12        self.pipeline = pipeline
13
14    @classmethod
15    def api(cls):
16        inputs = {"df_in" : pm.Pandas([("text", str)])}
17        outputs = {"df_out" : pm.Pandas([("text", str), ("generation", str)])}
18        return inputs, outputs
19
20    def predict(self, df_in: pd.DataFrame):
21        result = self.pipeline(df_in["text"].tolist())
22        df_in["generation"] = result
23        return df_in

HfAutoModelSerializer 与 HfAutoTokenizerSerializer

palantir_models.serializers.HfAutoModelSerializer 和 palantir_models.serializers.HfAutoTokenizerSerializer 将通过 save_pretrained 和 from_pretrained 方法，对 transformers.AutoModel 和 transformers.AutoTokenizer 模型进行序列化。

请注意，对于某些模型和分词器，建议使用特定的模型或分词器类，而非通用类；在这种情况下，可以将特定的类作为首个 model_class 参数传递给序列化器。



已复制！
1import palantir_models as pm
2from palantir_models.serializers import HfAutoModelSerializer, HfAutoTokenizerSerializer
3from transformers import AutoModelForSeq2SeqLM
4import pandas as pd
5import torch
6
7
8class HFTextGenerationAdapter(pm.ModelAdapter):
9
10    @pm.auto_serialize(
11        model=HfAutoModelSerializer(AutoModelForSeq2SeqLM),
12        tokenizer=HfAutoTokenizerSerializer()
13    )
14    def __init__(self, model, tokenizer):
15        self.model = model
16        self.tokenizer = tokenizer
17
18    @classmethod
19    def api(cls):
20        inputs = {"df_in" : pm.Pandas([("text", str)])}
21        outputs = {"df_out" : pm.Pandas([("text", str), ("generation", str)])}
22        return inputs, outputs
23
24    def predict(self, df_in: pd.DataFrame):
25        input_ids = self.tokenizer(
26            df_in["text"].tolist(),
27            return_tensors="pt",
28            padding=True
29        ).input_ids
30
31        outputs = self.model.generate(input_ids)
32        result = self.tokenizer.batch_decode(outputs, skip_special_tokens=True)
33
34        df_in["generation"] = result
35        return df_in

PyTorch

palantir_models.serializers.PytorchStateSerializer 将通过 torch.save 和 torch.load 对 torch.nn.Module ↗ 进行序列化，以便将您的对象保存至磁盘并从中加载。

TensorFlow

palantir_models.serializers.TensorflowKerasSerializer 将通过 obj.save 和 tensorflow.keras.models.load_model 对 tensorflow.keras.Model ↗ 进行序列化，以便将您的模型保存至磁盘并从中加载。在反序列化您的模型时，Foundry 将调用 obj.compile()。

XGBoost

palantir_models.serializers.XGBoostSerializer 将通过 save_model 和 load_model 对 xgboost.sklearn.XGBModel ↗ 进行序列化，以便将您的模型保存至磁盘并从中加载。XGBModel 类是诸多 XGBoost 模型的基类，包括 xgboost.XGBClassifier、xgboost.XGBRegressor 和 xgboost.XGBRanker。

Spark MLlib

palantir_models.serializers.SparkMLAutoSerializer 将使用 pyspark.ml.PipelineModel.write().overwrite().save() 和 pyspark.ml.PipelineModel.read().load() 方法对 pyspark.ml.pipeline.Pipeline ↗ 对象进行序列化。鉴于任何 Spark MLlib 模型均可封装在 Pipeline 对象中，此序列化器旨在作为 Spark 的通用序列化器。

编写您自己的 AutoSerializer

自定义序列化

若默认序列化器无法满足需求，用户可自行实现 load() 和 save() 方法。有关更多细节与示例，请参阅 API 参考文档。
