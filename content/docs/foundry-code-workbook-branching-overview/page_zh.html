分支

版本控制工具在软件开发中应用广泛，旨在帮助开发者团队实现最高效的协同工作。版本控制结合良好的开发实践，使得工程师能够快速构建功能，同时确保所提交的变更具备较高的质量。

Code Workbook 中的分支提供了专为数据转换量身定制的版本控制体验，支持团队在同一工作簿中同时处理逻辑与数据。

分支

默认情况下，创建工作簿时会生成一个与 Foundry 全局默认分支同名的分支。通常，该分支被命名为 master。你可以通过分支菜单浏览并选择分支、创建新分支以及修改分支设置。

创建分支

默认情况下，Code Workbook 允许最多创建 100 个分支。若在工作簿引入此限制前已拥有超过 100 个分支，现有分支将予以保留；但在工作簿中的分支总数减至少于 100 个之前，你将无法创建新分支。

在分支菜单中输入新分支的名称，随后点击创建分支或直接按回车键即可完成新分支的创建。系统将以当前分支作为父分支，在工作簿中生成一个新分支。

创建分支时，Code Workbook 会记录每个数据集在分支创建时刻的状态。你在新分支上运行的任何转换操作，都会基于该存储状态来加载数据。这意味着，父分支上的后续变更不会导致你新建的分支出错。

在分支上运行转换时，Code Workbook 会在关联的 Foundry 数据集上创建相应分支，从而将逻辑变更产生的结果与其他分支隔离。写入 Foundry 的操作完成后，点击输出面板中的打开数据集，即可查看当前分支下的结果数据集。

删除分支

若要删除分支，请先切换至该分支，然后点击分支菜单右上角的垃圾桶图标。若删除的分支仍存在基于其创建的子分支，这些子分支将被重新挂载到上一级父分支。例如，假设你拥有三个分支 master -> develop -> feature，删除 develop 后，feature 的父分支将变更为 master。

合并分支

在分支上完成工作后，你可以将其合并回父分支，以整合你的工作成果。了解更多关于合并的详情。

分支设置

若要编辑分支设置，请先切换至目标分支，然后点击分支菜单右上角的齿轮图标。在大多数工作簿中，你只需对 master 分支进行设置修改。

分支设置可用于自定义分支的变更流程。目前，分支设置提供以下选项：

  • 是否对分支进行保护？若分支受保护，任何人均无法直接对其进行编辑，所有变更必须通过其他分支合并引入。请注意，如果分支受保护，向其合并内容需要具备该工作簿的数据主人权限。
  • 是否允许在该分支上运行任务？若分支受保护且此设置处于关闭状态，该分支上的数据集必须通过 Foundry 构建进行物化。这能够防止任何人发起交互式运行，从而避免阻断正常构建任务的成功执行。
