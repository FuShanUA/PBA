概念：列

要遵循本文档中的示例，请添加: from pyspark.sql import functions as F.

列由 PySpark 类: pyspark.sql.Column 管理。每当您直接引用或从现有列推导出一个表达式时，就会创建列实例。您可以通过以下任何一种方式引用列：

  • F.col("column_name")
  • F.column("column_name")

引用列不等同于执行一个 select，因为“选择”列指的是子集化（和重新排序）您希望出现在结果数据集中的列。

获取模式

DataFrame.columns

返回所有列名作为Python列表

Copied!
1columns = df.columns # ['age', 'name']

DataFrame.dtypes

返回所有列名及其数据类型作为元组列表

Copied!
1dtypes = df.dtypes # [('age', 'int'), ('name', 'string')]

选择

DataFrame.select(*cols)

返回一个新 DataFrame，该 DataFrame 包含来自原始 DataFrame 的部分列。

例如，我们有一个包含 6 个命名列的 DataFrame：id（列名）, first_name（列名）, last_name（列名）, phone_number（列名）, address（列名）, is_active_member（列名）

创建，更新

DataFrame.withColumn(name, column)

Copied!
1new_df = old_df.withColumn("column_name", derived_column)
  • new_df: 包含 old_df 所有列的结果数据集（DataFrame），但新增了 new_column_name。
  • old_df: 我们想要添加新列的数据集（DataFrame）
  • column_name: 您想要创建的列名称（如果该列在 old_df 中不存在）或更新的列名称（如果该列已经在 old_df 中存在）。
  • derived_column: 推导出一个列的表达式，它将应用于 column_name 下的每一行（或者你给该列起的任何名称）。

给定一个现有的 DataFrame，您可以使用 withColumn 方法来创建新的列或用新的或修改后的值更新现有的列。这特别适用于以下目标：

  1. 基于现有值推导出新值

    Copied!
    1df = df.withColumn("times_two", F.col("number") * 2) # times_two = number * 2
    Copied!
    1df = df.withColumn("concat", F.concat(F.col("string1"), F.col("string2")))

  2. 将值从一种类型转换为另一种类型

    Copied!
    1# cast `start_timestamp` to DateType and store the new value in `start_date`
2df = df.withColumn("start_date", F.col("start_timestamp").cast("date"))

  3. 更新列

    Copied!
    1# update column `string` with an all-lowercased version of itself
2df = df.withColumn("string", F.lower(F.col("string")))

重命名，别名

DataFrame.withColumnRenamed(name, rename)

使用 .withColumnRenamed() 来重命名一列：

Copied!
1df = df.withColumnRenamed("old_name", "new_name")

另一种看待重命名列任务的方式，这应该能让你了解 PySpark 如何优化转换语句，是：

Copied!
1df = df.withColumn("new_name", F.col("old_name")).drop("old_name")

但也有几种情况你需要推导出一个新列而无需 withColumn，但仍需要为其命名。这就是 alias（或其方法别名 name）派上用场的地方。这里有几个使用示例：

Copied!
1df = df.select(derived_column.alias("new_name"))
2df = df.select(derived_column.name("new_name")) # same as .alias("new_name")
3df = df.groupBy("group") \
4	.agg(F.sum("number").alias("sum_of_numbers"), F.count("*").alias("count"))

我们也可以一次性重命名多个列：

Copied!
1renames = {
2    "column": "column_renamed",
3    "data": "data_renamed",
4}
5
6for colname, rename in renames.items():
7    df = df.withColumnRenamed(colname, rename)

删除

DataFrame.drop(*cols)

返回一个新的 DataFrame，包含来自原始 DataFrame 的一部分列，并删除指定的列。（如果模式不包含给定的列名，则会失败。）

有两种删除列的方法：直接方法和间接方法。间接方法是使用 select，你选择要保留的一组列。直接方法是使用 drop，你提供要删除的一组列。它们的使用语法类似，但这里顺序无关紧要。一些示例：

转换

Column.cast(type)

这里列出了所有存在的数据类型：NullType, StringType, BinaryType, BooleanType, DateType, TimestampType, DecimalType, DoubleType, FloatType, ByteType, IntegerType, LongType, ShortType, ArrayType, MapType, StructType, StructField

一般来说，你可以使用列上的 cast 方法将大多数数据类型相互转换：

Copied!
1from pyspark.sql.types import StringType
2df.select(df.age.cast(StringType()).alias("age"))
3# assume df.age is of IntegerType()

或者

Copied!
1df.select(df.age.cast("string").alias("age"))
2# effectively the same as using StringType().
age
"2"
"5"

强制转换基本上会创建一个新派生的列，你可以在其上直接执行 select, withColumn, filter 等操作。PySpark 中也存在“向下转换”和“向上转换”的概念，因此你可能会丢失之前数据类型中存储的更细粒度的信息，或者获得垃圾信息。

当...否则

F.when(condition, value).otherwise(value2)

参数：

  • condition - 布尔型 Column 表达式
  • value - 字面值或 Column 表达式

评估为一个列表达式，该表达式与 value 或 value2 参数相同。如果未调用 Column.otherwise()，则对于不匹配的条件返回 None (null) 的列表达式。

when、otherwise 运算符提供了类似于 if-else 语句的类比，该语句计算新的列值。基本用法是：

Copied!
1# CASE WHEN (age >= 21) THEN true ELSE false END
2at_least_21 = F.when(F.col("age") >= 21, True).otherwise(False)
3# CASE WHEN (last_name != "") THEN last_name ELSE null
4last_name = F.when(F.col("last_name") != "", F.col("last_name")).otherwise(None)
5df = df.select(at_least_21.alias("at_least_21"), last_name.alias("last_name"))

你也可以多次链式调用 when 语句：

Copied!
1switch = F.when(F.col("age") >= 35, "A").when(F.col("age") >= 21, "B").otherwise("C")

这些评估可以分配给列，或用于过滤器中：

Copied!
1df = df.withColumn("switch", switch) # switch=A, B, or C
2df = df.where(~F.isnull(last_name)) # filter for rows where the last_name (after empty strings were evaluated as null values) is not null
