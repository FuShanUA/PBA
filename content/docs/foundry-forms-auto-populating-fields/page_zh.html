自动填充字段

Foundry Forms 不再是 Foundry 上数据录入或写回工作流的推荐方法。相反，使用 Foundry 本体论构建用户输入工作流，将相关数据结构表示为对象类型，并配置与操作（Action）的写回交互。在 表单概览 文档中了解更多。

您可以使用自动填充表单字段来捕获有关表单的元数据，例如谁创建了响应以及何时创建。此页面讨论了添加到表单时可用的七种自动填充字段类型。

计算

计算 字段会根据其他单值字段显示一个计算出的值。用户可以配置以下选项：

  • 创建一个引用其他字段值的公式。
  • 自定义公式计算结果为 NaN、infinity 或其他错误时存储的值。
  • 指定一个 Output（数字、日期或时间戳）。
  • 设置一个 Number precision。
  • 为每个参数提供一个 Default missing value。
  • 设置一个 Unit label（例如，kg 或 lbs）。
  • 使用 代码编辑器 来设置 placeholder: string。

用户可以创建 expr-eval 库支持的任何公式 ↗。

创建于

创建于 字段显示条目的创建或修改时间。用户可以配置以下选项：

  • 指定一个 Recorded format（例如，“YYYY/MM/DD” 或 “DD.MM.YYYY”）。
  • 以包含时区的时间戳形式记录该值，或者以仅存储该位置当前时间的简单形式记录。
  • 记录表单打开的时间，而不是提交的时间。

如果您在字段配置中开启按本地日期或时间戳记录，该字段将存储用户所在时区的当前时间。因此，不同条目中的时间可能不能真实反映它们实际发生的时间顺序。如果您想表达条目创建所在的时区，可以在时间戳格式中包含z选项。

创建时间字段可用于跟踪哪些条目是最新的，随时间推移创建了多少条目，或者它们通常在一天中的什么时间被创建。

由谁创建

由谁创建字段显示条目的创建者或修改者的用户ID。

此字段可以与用户属性字段结合使用，以获取当前用户的姓名或电子邮件地址。

修改时间

修改时间字段显示条目的最新修改时间。用户可以配置以下选项：

  • 指定 Recorded format（例如，“YYYY/MM/DD” 或 “DD.MM.YYYY”）。
  • 将值记录为也表示时区的时间戳，或者作为简单的表示形式，存储该位置的当前时间。
  • 记录表单打开的时间而不是提交的时间。

如果您启用 记录为本地日期或时间戳 在字段配置中，该字段将存储用户所在时区的时间。因此，不同条目中的时间可能不代表它们实际发生的顺序。如果您在时间戳格式中包含z选项，可以表示该条目创建的时区。

修改时间字段对于跟踪哪些条目是最新的、条目被编辑的频率，或者自创建以来发生了什么变化并需要后续更新非常有用。

修改人

修改人字段显示条目修改者的用户ID。

此字段可以与用户属性字段结合使用，以获取当前用户的姓名或电子邮件地址。

模板

模板 字段根据其他单值字段的值显示一个组合后的值。用户可以配置以下选项：

  • 创建一个引用其他字段值的模板。
  • 使用代码编辑器设置placeholder: string和errorValue: string。

示例

  • 假设您想在问卷中结合分数字段和评论字段。您可以将“score”作为变量a，将“comment”作为b。然后创建模板{{a}} - {{b}}，并得到形式为5 - very satisfied的结果。

  • 要使用表单中引用的对象的属性，可以使用对象属性显示字段来选择属性，隐藏该字段，并从模板字段中引用它。

  • 您可以使用字段配置转换，根据表单中其他值的条件来更改模板使用的字段或格式。例如，您可以要求受访者从下拉菜单中选择一个标准响应，或者让他们在文本字段中编写自定义响应。将根据受访者的操作使用不同的模板。
