自动填充字段

对于 Foundry 上的数据录入或回写工作流，Foundry 表单已不再是推荐方案。建议改用 Foundry 本体来构建用户输入工作流，将相关数据结构表示为对象类型，并通过 Actions 配置回写交互。详情请参阅《表单概述》文档。

利用自动填充表单字段，可以采集有关表单的元数据，例如响应的创建者及创建时间。本文将探讨向表单添加字段时可用的七种自动填充字段类型。

计算

计算字段会根据其他单值字段显示一个计算后的值。用户可配置以下选项：

  • 创建引用其他字段值的公式。
  • 自定义公式求值结果为 NaN、infinity 或出现其他错误时所存储的值。
  • 指定 输出 类型（数字、日期或时间戳）。
  • 设置 数字精度。
  • 为每个参数提供 默认缺失值。
  • 设置 单位标签（例如 kg 或 lbs）。
  • 使用《代码编辑器》来设置 placeholder: string。

用户可创建 expr-eval 库 ↗ 支持的任意公式。

创建时间

创建时间字段用于显示条目的创建或修改时间。用户可配置以下选项：

  • 指定 记录格式（例如 "YYYY/MM/DD" 或 "DD.MM.YYYY"）。
  • 将值记录为带有时区信息的时间戳，或记录为仅存储当地时间的简单表示形式。
  • 记录打开表单的时间，而非提交表单的时间。

如果在字段配置中开启 记录为本地日期或时间戳，该字段将直接存储用户所在时区的当前时间。因此，不同条目上的时间可能无法反映它们实际发生的先后顺序。若要在时间戳格式中体现创建条目时所处时区，可加入 z 选项。

创建时间字段有助于追踪哪些条目是最新创建的、随时间推移创建了多少条目，以及条目往往在一天中的哪些时段被创建。

创建者

创建者字段用于显示条目创建者或修改者的用户 ID。

该字段可与用户属性字段结合使用，以获取当前用户的姓名或电子邮件地址。

修改时间

修改时间字段用于显示条目最近一次修改的时间。用户可配置以下选项：

  • 指定 记录格式（例如 "YYYY/MM/DD" 或 "DD.MM.YYYY"）。
  • 将值记录为带有时区信息的时间戳，或记录为仅存储当地时间的简单表示形式。
  • 记录打开表单的时间，而非提交表单的时间。

如果在字段配置中开启 记录为本地日期或时间戳，该字段将直接存储用户所在时区的当前时间。因此，不同条目上的时间可能无法反映它们实际发生的先后顺序。若要在时间戳格式中体现创建条目时所处时区，可加入 z 选项。

修改时间字段有助于追踪哪些条目是最新修改的、条目的编辑频率，或者创建以来发生了哪些需要下游更新的变更。

修改者

修改者字段用于显示条目修改者的用户 ID。

该字段可与用户属性字段结合使用，以获取当前用户的姓名或电子邮件地址。

模板

模板字段用于显示基于其他单值字段值拼接而成的结果。用户可配置以下选项：

  • 创建一个引用其他字段值的模板。
  • 使用《代码编辑器》来设置 placeholder: string 和 errorValue: string。

示例

  • 假设需要将问卷中的评分字段与评论字段合并，可以将“score”设为变量 a，将“comment”设为 b。随后，创建模板 {{a}} - {{b}}，即可获得形如 5 - 非常满意 的结果。

  • 若需使用表单中所引用对象的属性，可通过对象属性显示字段选择该属性，隐藏该字段，并在模板字段中对其进行引用。

  • 可利用字段配置转换功能，根据表单值的其他条件，更改模板所使用的字段或格式。例如，可要求回复者从下拉菜单中选择标准回复，或允许其在文本框中撰写自定义回复。根据回复者的操作，将应用不同的模板。
