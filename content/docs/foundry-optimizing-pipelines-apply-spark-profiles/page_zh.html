应用 Spark 配置文件

您可能需要为转换作业应用自定义的 Spark 属性。

若要将 Spark 属性应用于特定作业：

  1. 按照指南将 Spark 配置文件导入您的代码库。
  2. 按照下文所述，在代码中引用转换配置文件。

您可以在《Spark 配置文件参考》部分详细了解默认 Spark 配置文件的特性。

另请注意调整 Spark 配置文件的最佳实践建议。

转换配置文件语法

所有支持的语言均支持指定自定义 Spark 配置文件。在以下所有情况中，设置的求值顺序均从左到右。如果多个配置文件指定了同一设置，则更接近列表末尾的配置将优先生效。

Python

您可以在 Python 代码中引用 profile1 和 profile2 配置文件，方法是使用 configure 装饰器包装您的转换对象。该装饰器接收一个 profile 参数，该参数指向您的自定义转换配置文件列表：


自动注册的转换可以在你的 Java 代码中引用 `profile1` 和 `profile2` 配置，方法是在你的计算函数中使用 `TransformProfiles` 注解。该注解接收一个参数，用于指定你的自定义 Spark 配置数组：


已复制！
1import com.palantir.transforms.lang.java.api.TransformProfiles;
2
3/**
4* 这是一个用于自动注册的底层 Transform 示例。
5*/
6public final class LowLevel {
7     @Compute
8     @TransformProfiles({ "profile1", "profile2" })
9     public void myComputeFunction(
10             @Input("/path/to/input/dataset") FoundryInput myInput,
11             @Output("/path/to/output/dataset") FoundryOutput myOutput) {
12         Dataset<Row> limited = myInput.asDataFrame().read().limit(10);
13         myOutput.getDataFrameWriter(limited).write();
14     }
15 }

SQL

你可以通过为表设置 foundry_transform_profiles 属性，在 SQL 代码中引用 profile1 和 profile2 配置：

Copied!
1CREATE TABLE `/path/to/output` TBLPROPERTIES (foundry_transform_profiles = 'profile1, profile2')
2    AS SELECT * FROM `/path/to/input`

以下是使用替代 SQL 语法的另一个示例：

Copied!
1CREATE TABLE `/path/to/output` USING foo_bar OPTIONS (foundry_transform_profiles = 'profile1, profile2')
2    AS SELECT * FROM `/path/to/input`;

请注意，ANSI SQL 模式当前不支持指定自定义转换配置。
