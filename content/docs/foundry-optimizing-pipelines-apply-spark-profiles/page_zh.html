应用 Spark 配置文件

您可能需要将自定义的 Spark 属性应用于您的转换作业。

要将 Spark 属性应用于特定作业：

  1. 按照 将 Spark 配置文件导入到您的代码仓库 的指南进行操作。
  2. 如 下文 所述，在您的代码中引用转换配置文件。

您可以在 Spark 配置文件参考 部分了解更多有关默认 Spark 配置文件特性的信息。

请注意 调整 Spark 配置文件的推荐最佳实践。

转换配置文件语法

所有 语言 均支持指定自定义的 Spark 配置文件。 . 在以下所有情况下，设置都是从左到右进行评估的。如果多个配置文件指定了相同的设置，则列表中更接近末尾的那个将具有优先权。

Python

您可以通过使用 profile1 装饰器来包装您的转换对象，在 Python 代码中引用 profile2 和 configure 配置文件。该装饰器接受一个 profile 参数，该参数指的是您的自定义转换配置文件列表：

Copied!
1from transforms.api import transform, Input, Output
2
3@transform.spark.using(
4    # your input dataset(s)
5    my_input=Input("/path/to/input/dataset"),
6    # your output dataset
7    my_output=Output("/path/to/output/dataset"),
8).with_config(profiles=['profile1', 'profile2'])
9# your data transformation code
10def my_compute_function(my_input, my_output):
11    return my_output.write_dataframe(my_input.dataframe())

Java

自动注册的转换可以使用 profile1 注解在您的 Java 代码中引用 profile2 和 TransformProfiles 配置文件。该注解接受一个参数，该参数指的是您的自定义 Spark 配置文件数组：

Copied!
1import com.palantir.transforms.lang.java.api.TransformProfiles;
2
3/**
4* This is an example low-level Transform intended for automatic registration.
5*/
6public final class LowLevel {
7     @Compute
8     @TransformProfiles({ "profile1", "profile2" })
9     public void myComputeFunction(
10             @Input("/path/to/input/dataset") FoundryInput myInput,
11             @Output("/path/to/output/dataset") FoundryOutput myOutput) {
12         Dataset<Row> limited = myInput.asDataFrame().read().limit(10);
13         myOutput.getDataFrameWriter(limited).write();
14     }
15 }

或者，如果您使用手动注册，可以使用 builder 方法 transformProfiles() ：

Copied!
1public final class MyPipelineDefiner implements PipelineDefiner {
2
3    @Override
4    public void define(Pipeline pipeline) {
5        LowLevelTransform lowLevelManualTransform = LowLevelTransform.builder()
6               .transformProfiles(ImmutableList.of("profile1", "profile2"))
7               // Pass in the compute function to use. Here, "LowLevelManualFunction" corresponds
8               // to the class name for a compute function for a low-level Transform.
9               .computeFunctionInstance(new LowLevelManualFunction())
10               .putParameterToInputAlias("myInput", "/path/to/input/dataset")
11               .putParameterToOutputAlias("myOutput", "/path/to/output/dataset")
12               .build();
13        pipeline.register(lowLevelManualTransform);
14    }
15}

SQL

您可以通过为表设置foundry_transform_profiles属性，在SQL代码中引用profile1和profile2配置文件：

Copied!
1CREATE TABLE `/path/to/output` TBLPROPERTIES (foundry_transform_profiles = 'profile1, profile2')
2    AS SELECT * FROM `/path/to/input`

这是使用其他SQL语法的另一个示例：

Copied!
1CREATE TABLE `/path/to/output` USING foo_bar OPTIONS (foundry_transform_profiles = 'profile1, profile2')
2    AS SELECT * FROM `/path/to/input`;

请注意，目前在ANSI SQL模式下不支持指定自定义转换配置文件。
