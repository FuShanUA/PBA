设置批同步

批同步功能支持将外部系统的数据同步至 Foundry 数据集。这是支持最广泛的功能，几乎适用于所有连接器。批同步既能同步带有模式的结构化数据，也能同步无模式的原始文件。

创建批同步时，系统会同步新建一个 Foundry 数据集，用于存放写入的同步数据。完成配置后，你可以手动运行同步，也可以设置调度来触发构建，从而从外部系统读取数据并将其写入输出数据集。

请按照以下步骤设置批同步。本指南假定你已成功配置了支持 批同步 的数据源连接。

创建新的批同步

首先，在数据连接应用中进入你的数据源连接，然后在概览页面选择新建批同步。如果这是刚配置好的数据源，你会看到下方所示的可用功能，点击批同步选项旁边的创建即可。

如果你的连接器支持数据源探索，你也可以选择探索并创建同步，在探索视图中浏览数据源并直接开始创建同步。具体细节请参阅数据源探索文档。

指定输出位置

输出位置决定了同步数据的数据集将在何处创建，并根据项目级权限确定谁有权访问最终数据。你可以为数据源指定默认的输出文件夹，也可以按需为每次同步覆盖该设置。

创建同步数据集时，推荐的最佳实践是将其保存在连接器所在位置。这样可以实现对同一连接器所有数据的统一权限管理，对构建数据管道很有帮助。了解更多关于数据管道的推荐项目结构。

系统支持同步至现有数据集，但不建议这么做，因为同步操作可能会覆盖所选数据集中已有的数据。

配置批同步

在目标位置所在的页面上，你可以看到用于配置批同步的各项设置。

根据数据源的不同，可用的选项也会有所区别。最常见的两种批同步类型是：

  • 文件批同步，即将文件同步至无模式的数据集。
  • 表格批同步，即同步带有模式的数据表。

大多数系统要么支持文件批同步，要么支持表格批同步，但部分系统可能同时支持两者。

其他选项

大多数批量同步任务还提供许多其他选项，具体视连接器而定。以下列出了一些批量同步中常见的配置选项示例：

  • 事务类型决定了接入的数据是覆盖先前接入的数据（SNAPSHOT），还是进行增量添加（APPEND）。了解有关 增量同步 的更多信息。
  • 计划 允许你配置使用 Foundry 构建系统同步数据的频率。建议为新建的同步任务设置执行计划。了解有关 计划最佳实践 的更多信息。
  • 构建策略允许你限制同步任务的允许运行时间，而不受已配置计划的影响。
  • 最大持续时间允许你自动取消超出指定时间限制的同步任务。所有 在 Foundry worker 上运行 的同步任务，如果运行时间超过约 48 小时，将被自动取消。

以下选项仅适用于表批量同步：

  • 无时区时间戳设置允许你自定义同步至 Foundry 时处理无时区时间戳数据的方式。默认情况下，无时区的时间戳会作为字符串进行同步，但你也可以选择将其作为手动指定时区的 timestamp 或作为 long 类型进行同步。
  • 允许模式变更。此设置允许你在系统发生模式变更时阻止批量同步运行。默认情况下，不允许发生模式变更。

预览同步输出

在继续操作之前，你可以根据已配置的设置运行预览，查看将要同步的数据。你应该通过此功能验证同步配置是否符合预期。

  • 对于文件批量同步，预览将显示文件列表。
  • 对于表批量同步，预览将显示选定的表结果，仅限前 20 行。
