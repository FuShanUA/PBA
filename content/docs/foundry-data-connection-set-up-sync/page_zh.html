设置批量同步

批量同步功能允许将外部系统的数据同步到Foundry 数据集（dataset）。批量同步是最广泛支持的功能，几乎在所有 连接器（connectors） 上都可用。批量同步可以同步具有模式的表格数据以及没有模式的原始文件。

创建一个批量同步还将创建一个新的Foundry 数据集（dataset），同步的数据将写入其中。一旦配置了同步，您可以手动运行它，或者设置一个 计划（schedule） 来触发 构建（build），该构建将从外部系统读取数据并将其写入输出数据集。

按照以下步骤设置批量同步。此设置指南假设您已经成功 配置了一个源连接（source connection），该连接支持 批量同步（batch sync） 功能。

创建新的批量同步

首先，导航到数据连接应用程序中的源连接，然后从概览页面选择新批量同步。如果这是新配置的源，则应该看到如下所示的可用功能，并在批量同步选项旁边选择创建。

如果您的连接器支持源探索，您还可以选择探索并创建同步来探索您的数据源，并直接从探索视图开始创建同步。有关详细信息，请参阅源探索文档。

指定输出位置

输出位置定义了同步数据的数据集将被创建的位置，并将根据 项目级 权限确定谁有权访问结果数据。可以为源指定默认的输出文件夹，如果需要的话，也可以为每个同步覆盖该设置。

创建同步数据集时，推荐的最佳做法是将其与连接器一起保存。这使得对来自给定连接器的所有数据进行统一权限设置成为可能，这对于创建数据管道非常有帮助。了解更多关于 数据管道的推荐项目结构。

支持对现有数据集进行同步，但不建议这样做，因为同步可能会覆盖已存在于所选数据集中的任何数据。

配置批量同步

在目标页面上，您将看到各种用于配置批量同步的设置。

根据源系统，可能会有不同的选项可用。最常见的两种批量同步类型是：

  • 文件批量同步，其中文件被同步到一个没有模式的数据集。
  • 表批量同步，其中具有模式的表被同步。

大多数系统支持文件或表批量同步，但某些系统可能同时支持这两种。

附加选项

对于大多数批次同步，有其他一些选项可用，并可能因连接器而异。下面列出了一些广泛可用的批次同步配置选项示例：

  • 事务类型（transaction type） 确定已摄取的数据是否覆盖先前摄取的数据（SNAPSHOT）或者是否是按增量方式添加（APPEND）。了解更多关于增量同步的信息。
  • 计划允许您使用Foundry的构建系统配置数据同步的频率。我们建议为新创建的同步设置一个计划。了解更多关于计划最佳实践的信息。
  • 构建策略允许您限制同步运行的时间，无论配置的计划如何。
  • 最大持续时间允许您自动取消超过指定时间限制的同步。如果同步在Foundry工作器上运行超过大约48小时，所有同步将被自动取消。

以下选项仅适用于表批量同步：

  • 不带时区设置的时间戳允许您自定义在将没有时区的时间戳数据同步到Foundry时的处理方式。默认情况下，没有时区的时间戳会作为字符串同步，但您可以选择将其作为timestamp同步，手动指定时区，或作为long同步。
  • 允许模式更改。此设置允许您在外部系统中存在模式更改时阻止批量同步运行。默认情况下，不允许进行模式更改。

在继续之前，您可以根据已配置的设置运行数据同步的预览。您应该使用此功能来验证您的同步是否按预期配置。

  • 对于文件批量同步，预览将显示一个文件列表。
  • 对于表批量同步，预览将显示所选表的结果，仅限于前20行。
