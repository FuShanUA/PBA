Workshop 部分和页面的自定义背景颜色

发布日期：2026-02-26

工作区的各个部分和页面现在支持自定义背景颜色，使应用程序构建者能够更好地控制其模块的视觉设计。除了现有的预设色调外，您现在可以将任何十六进制颜色或Blueprint颜色应用于部分和页面的背景。在模块级别定义的已保存颜色也作为部分和页面背景选项可用，从而可以轻松地在整个应用程序中保持一致的颜色调色板。

显示自定义背景颜色的工作区应用程序页面。

包含内容

  • 部分和页面的自定义十六进制颜色: 使用十六进制代码将任何颜色设置为部分或页面的背景，或者从Blueprint颜色快捷方式中选择以快速应用标准颜色值。
  • 已保存颜色支持: 使用 已保存颜色 作为部分和页面的背景。编辑已保存的颜色会自动更新所有引用该颜色的部分和页面。
  • 自动主题适应：当应用自定义背景颜色时，部分内的小部件会根据背景颜色的明暗程度自动在浅色和深色模式之间切换，以确保文本和控件保持可读性。
  • 跨布局的一致行为：自定义背景颜色适用于部分内边距、部分标题和拆分部分。当拆分一个具有自定义背景颜色的部分时，两个结果部分都会继承原始颜色。

如何使用自定义背景颜色

要设置自定义背景颜色，请在编辑模式下选择一个部分或页面，打开样式配置，然后从背景颜色选项中选择一种颜色。选择自定义颜色瓷砖以打开颜色选择器，在这里您可以输入十六进制代码或从Blueprint颜色快捷方式中选择。

使用 Cipher 自动屏蔽敏感数据扫描器中的敏感数据

发布日期：2026-02-24

敏感数据扫描器（SDS）是一款数据治理工具，可在Foundry中检测可能的敏感数据，例如电话号码、社会安全号码或用户定义的模式，并在检测到匹配时自动执行匹配操作。SDS现在支持一种新的匹配操作类型，混淆数据，允许您使用Cipher自动对匹配的敏感数据进行加密或哈希处理。此功能加入了现有的匹配操作类型，应用标记和创建问题，使数据治理用户在检测到匹配时能够更直接地控制敏感数据的处理方式。

设置一个混淆数据匹配操作

要使用此匹配操作，您需要一个 Cipher Channel（通道） 和一个 Cipher License（许可证）。Cipher Channel 确定 SDS 在混淆匹配数据时将使用的加密算法。以下算法受支持：

  • 确定性加密（AES SIV）
  • 概率加密（AES GCM SIV）
  • 哈希（SHA512 和 SHA256）

一个加密许可证授予与加密通道交互所需的权限。SDS 特别需要一个管理员许可证，因为需要访问加密密钥以执行混淆操作。如果您还没有加密通道或加密许可证，可以在加密工具中创建这些内容。

《创建匹配操作》对话框，用于创建新的混淆数据匹配操作。

混淆模式

默认情况下，当找到匹配项时，SDS 会混淆整个列。您可以通过在 创建匹配操作 对话框中展开 显示高级配置 来自定义此行为，并从四种混淆模式中进行选择：

  • 整个列 （默认）：如果找到匹配项，则对整个列进行混淆处理。
  • 整行: 如果找到匹配项，则对整行进行混淆处理。
  • 仅匹配的单元格: 仅对找到匹配项的单元格进行混淆处理。
  • 仅匹配的段落: 仅对符合匹配条件的文本段落进行混淆处理。

设置对话框中可用的四种混淆模式选项。

对于每个扫描的资源，SDS 将创建一个混淆后的输出数据集。这些输出数据集直接链接到 SDS 扫描结果接口。

显示链接到混淆后输出数据集的 SDS 扫描结果。

一个经过混淆的输出数据集示例，用于加密检测到的电子邮件地址和电话号码。

如需更多信息，请查看 匹配操作文档 以了解敏感数据扫描器。

核心对象视图现已普遍可用

发布日期：2026-02-19

当你在本体论（Ontology）中创建并配置一个对象类型时，Foundry 会自动创建一个核心 对象视图（Object View） 为所有对象提供标准化的表示形式，确保其他用户无需额外配置即可全面理解其模式和关联。核心对象视图在创建对象类型后提供一致且完整的对象数据展示，并现已在Foundry注册中普遍可用。

为什么在您创建对象类型后，Foundry会配置核心对象视图？

提供一个默认的对象视图来显示对象的重要和标准属性，可以节省本体论构建者的时间，同时确保其他用户从与自动生成的核心表示的交互中获得最大价值。通过让Foundry处理核心对象视图的创建，构建者可以节省时间来创建他们自己的工作流特定的自定义对象视图，这些视图根据需要配置用于更广泛的用例。 了解更多关于使用Foundry分支构建自定义对象视图的信息。

在基础类型特定组件中查看突出属性

Foundry 在核心对象视图顶部显示 突出属性，以提供关于对象最重要的信息的即时上下文。

被标记为突出的属性会根据其类型获得增强的视觉处理：

  • 媒体引用属性会在专用的媒体查看器中显示。
  • 时间序列属性会以交互式 Quiver 图表显示，展示时间数据模式。
  • 地理空间属性会自动在 Foundry 地图 上显示。

所有其他突出属性都会以较大的卡片式格式显示，高于显示其余标准属性的表格。

Model Studio（模型工作室），我们的无代码模型训练工具，现已全面发布

发布日期：2026-02-12

Model Studio（模型工作室）是一个新的工作区，允许用户训练和部署机器学习模型，将在2月9日当周全面发布并可供使用。这继2025年10月开始的成功公开测试期之后。

Model Studio 将构建生产级模型这一复杂任务转化为一个简化的无代码流程，使高级机器学习更加易于使用。无论您是希望加快工作流程的数据科学家，还是渴望从数据中挖掘洞察力的业务用户，Model Studio 都提供了必要的工具和用户友好的界面，简化了从数据到模型的整个过程。

Model Studio 应用程序首页，显示最近的训练运行和运行详情。

什么是 Model Studio？

Model Studio 是一种无代码模型开发工具，允许您在预测、分类和回归等任务中训练模型。通过 Model Studio，您可以使用自定义数据训练模型，同时通过可选的参数配置保留对训练过程的定制和控制，以最大化模型性能。

构建实用、可投入生产的模型传统上需要深厚的技术专业知识和大量时间投入，但 Model Studio 通过提供以下功能改变了这一现状：

  • 简化的点击式界面，用于配置模型训练任务；无需编写代码。
  • 内置的生产级模型训练器，针对常见用例（如时间序列预测、回归和分类）进行了优化。
  • 智能默认设置和引导式工作流程，使您即使在机器学习领域是新手也能快速上手。
  • 深入的实验跟踪，集成性能指标，使您能够自信地监控和优化您的模型。
  • 完整的数据溯源和安全访问控制，基于 Palantir 平台构建，确保每一步都具有透明度和安全性。

谁应该使用 Model Studio？

Model Studio 适用于技术用户和非技术用户。想要利用机器学习而不进行编码的业务用户以及想要加速原型设计和模型部署的数据科学家都可以从 Model Studio 的工具和简化流程中受益。此外，组织也可以通过 Model Studio 降低 AI 采用的门槛，并使更多团队能够构建和使用模型。

开始使用

要开始模型训练，请打开 Model Studio 应用程序并按照以下步骤操作：

  • 为您的使用场景选择最佳模型训练器（时间序列预测、分类或回归）。
  • 选择您的输入数据集。
  • 使用直观的选项配置您的模型，或者坚持推荐的默认设置。

在配置完您的模型后，您可以启动一次训练运行，并通过清晰的指标和实验跟踪实时查看模型性能。

开发路线图的下一步是什么？
