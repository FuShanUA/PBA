网络访问与身份认证

在 Workshop 中开发的移动应用本质上是为移动浏览器优化过的 Web 应用。因此，用户无需安装任何程序，即可直接在平板电脑或手机上访问这些 Web 应用。不过，仍需确保在网络层面，用户的设备能够访问 Foundry 环境，并且用户能够成功完成 Foundry 的身份认证。

本页将介绍如何规划网络访问与身份认证、常见问题及其可能的解决方案。排查和修复问题时，尤其是涉及网络访问的问题，你可能需要与 IT 部门的同事协同处理。

从宏观层面来看，确保用户访问能力主要分为以下两大环节：

  • 确保网络访问
  • 确保用户能够进行身份认证

确保网络访问

Foundry 支持多种部署形态，既可部署于 Palantir 托管的 SaaS 环境，也可进行本地化部署，因此无法用单一方案保证所有用户都能通过移动设备访问。本部分将提供一系列方案，帮助你为用户打通访问链路。

实现 Foundry 移动端访问最常见的方法包括：

  • 使用企业级 Web 浏览器
  • 使用移动端 VPN
  • 将 IP 区域加入白名单

使用企业级 Web 浏览器

许多组织部署了 MDM（移动设备管理）解决方案来统一管理移动设备，且大多数 MDM 解决方案内置了安全 Web 浏览器，专用于访问公司内部资源。如果组织已建立 MDM 体系，可以直接使用受 MDM 管理的浏览器来访问 Foundry。

该方案的主要缺陷在于，受 MDM 管理的浏览器在渲染 Web 组件时，效果可能与 Safari、Chrome 或 Edge 等标准浏览器存在差异，导致用户体验不及常用浏览器。此外，设备未纳入组织管理的用户将无法访问 Foundry。

若已知目标用户群体的设备均已接入组织 MDM，那么使用 MDM 支持的浏览器无疑是最直接、最安全的路径。

后续步骤：联系 IT 部门，确认是否已部署企业级 MDM，以及能否配置 MDM 浏览器以实现对 Foundry 环境的访问。

使用移动端 VPN

组织可能支持员工在非办公场所通过 VPN 接入公司内部资源。许多 VPN 解决方案同样支持移动设备，用户可先连接 VPN，再通过设备自带的标准浏览器访问内部资源。

此方案的优势在于用户可继续使用 Safari 或 Chrome 等标准浏览器。缺点是除了初次配置 VPN 客户端外，用户每次使用 Foundry 前都必须先连接 VPN。对于习惯了该操作流程的用户而言，这不构成障碍；但若要将应用推送给此前未用过移动端 VPN 的用户群体，可能会引发推广阻碍。

后续步骤：联系 IT 部门，确认是否提供移动端 VPN 客户端。若有，请验证用户是否具备 VPN 访问权限，并确认通过该 VPN 能否顺利接入 Foundry 环境。

将 IP 区域加入白名单

通过在控制面板中配置网络入口，你可以选择允许特定国家或地区的广泛网络访问，以覆盖你的目标用户群体。

这种方法的优点是，白名单区域内的用户无需使用 MDM 管理的设备或 VPN 客户端（如果之前使用的是严格的基于 IP 的白名单），即可从任何设备访问您的 Foundry 环境。此方法为用户提供了更加无缝的登录体验。然而，您的组织可能不希望允许从更广泛的网络访问您的 Foundry 环境。

确保用户可以进行身份验证

用户在网络层面访问 Foundry 后，需要导航至移动应用启动器并进行身份验证。这要求用户拥有账户，并完成组织的 SSO 身份验证流程。下面我们将针对这两个步骤探讨一些注意事项。

确保用户拥有账户

访问 Foundry 必须拥有用户账户。针对移动端用户的账户相关注意事项，可能与桌面端用户有所不同。根据我们的经验，某些面向移动设备的使用场景可能会涉及异常复杂的设备环境：

  • 某些场景要求承包商能够从 Foundry 访问数据或向其提交数据。如果您的使用场景面向非本组织员工的广泛用户群体，应确保这些用户拥有账户，并能够向 Foundry 进行身份验证。
  • 某些移动端使用场景面向共享设备。例如，工厂车间可能会使用平板设备访问 Foundry 中的数据。这些设备不归单一用户所有，而是在工作环境中供多人共享。在这种情况下，应明确哪些用户将登录每台设备。如有必要，可以在身份提供商处或使用 Foundry 的内部域为每个工位配置专用用户账户。若需启用此类身份验证模式，请联系您的 Palantir 代表以了解更多信息。

验证最终用户身份验证流程

由于身份验证流程是端到端用户体验的关键环节，您应当验证最终用户在移动设备上的身份验证体验。用户每次进行身份验证时都需要完成组织的 SSO 流程，并经历 MFA（多因素身份验证）过程。

此外，组织的身份提供商决定了用户需要多久重新向 Foundry 进行一次身份验证。如果登录超时频率过高，可能会给最终用户体验带来额外的阻力，因为用户在一天内可能需要频繁进行身份验证。如果这给您的用户带来了困扰，请联系您的 IT 部门，商讨在身份提供商处延长会话时长的事宜。
