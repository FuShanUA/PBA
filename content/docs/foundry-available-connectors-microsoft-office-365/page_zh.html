Microsoft Office 365

Microsoft Office 365 连接器是 Palantir 提供的用于 Microsoft Office 365 的驱动程序。

要创建新的 Microsoft Office 365 源，请遵循 Palantir 提供的驱动的标准设置流程，然后使用以下章节进行 Microsoft Office 365 特定的配置和网络设置。有关完整的属性参考，请参阅《官方 Microsoft Office 365 驱动程序文档》 ↗。

配置

以下属性为必填或建议填写。

属性是否必填？描述默认值
DefaultGroups ↗必填确定访问 Microsoft Office 365 中组范围资源时的默认组上下文。CurrentUser
DefaultUser ↗必填确定访问 Microsoft Office 365 中用户范围资源时的默认用户上下文。CurrentUser
MessageContentType ↗必填确定返回的消息是 HTML 格式还是文本格式。html
AuthScheme ↗建议指定连接到 Microsoft Office 365 时使用的身份验证类型。如果此属性留空，则使用默认身份验证。AzureServicePrincipal
AzureTenant ↗建议标识用于访问数据的 Microsoft Office 365 租户。接受租户的域名（例如 contoso.onmicrosoft.com）或其目录（租户）ID。—
GroupId ↗建议指定要访问其数据的 Microsoft Office 365 组的 ID。—
InitiateOAuth ↗建议指定获取或刷新 OAuth 访问令牌的流程，以便在经过身份验证和授权的用户工作时保持用户的访问权限。GETANDREFRESH
OAuthClientId ↗建议指定分配给自定义 OAuth 应用程序的客户端 ID（也称为消费者密钥）。在身份验证期间，需要此 ID 向 OAuth 授权服务器标识应用程序。—
OAuthClientSecret ↗建议指定分配给自定义 OAuth 应用程序的客户端密钥。此机密值用于向 OAuth 授权服务器验证应用程序身份。（仅限自定义 OAuth 应用程序。）—
UserId ↗建议指定要访问其数据的 Microsoft Office 365 用户的 ID。—

网络连接

下表列出了为成功运行而需要源能够访问的域。

对于每个域，添加相应的出站策略。如果源托管在本地并且无法从 Foundry 直接访问，请改用代理出站策略；代理主机本身也必须能够访问所列出的域。有关详细信息，请参阅《使用代理作为代理》。

域是否需要
graph.microsoft.com如果 AzureEnvironment=GLOBAL（默认）
login.microsoftonline.com如果 AuthScheme=AzureAD（默认）、AzureServicePrincipal、AzureServicePrincipalCert 并且 AzureEnvironment=GLOBAL
microsoftgraph.chinacloudapi.cn如果 AzureEnvironment=CHINA
login.chinacloudapi.cn如果 AuthScheme=AzureAD（默认）、AzureServicePrincipal 、AzureServicePrincipalCert 并且 AzureEnvironment=CHINA
graph.microsoft.us如果 AzureEnvironment=USGOVT
login.microsoftonline.us如果 AuthScheme=AzureAD（默认）、AzureServicePrincipal、AzureServicePrincipalCert 并且 AzureEnvironment=USGOVT 或 USGOVTDOD
dod-graph.microsoft.us如果 AzureEnvironment=USGOVTDOD
