将数据连接到机器

通过将一个流程连接到数据源，您可以弥合抽象流程定义与现实世界观察之间的差距。这些数据可用于进行初始的 流程挖掘 或者 监控性能并查找瓶颈。

机器连接到本体论中的两种对象数据： 流程对象 和 日志对象。数据配置的侧边栏可以通过主工具箱或从流程容器中访问。

流程对象

流程对象是经历流程的实体。

例如，这可能是一个 Employee（员工）用于入职流程，或者是一个 Invoice（发票）用于采购到付款流程。该实体的状态由一个字符串属性显式跟踪，通常称为 state（状态）。state 属性的值（例如，“created” 或 “approved”）在图上表示为状态节点。

状态值可能有依赖关系，例如操作或自动化中的提交条件，这些条件由特定的状态条件触发。在Machinery中更改状态值不会更新这些依赖项，也不会更改本体论数据中的值。

在多进程设置中，每个进程容器仅捕获对象类型的恰好一个状态属性。

如上图资源示例截图所示，一个对象类型可以有多个状态属性；例如，一个粗略状态和一个细粒度状态。

进程对象类型表示实体的最新状态，因此无法提供状态转换和时间模式的信息。

日志对象

A log object represents an individual change to an entity’s state. Log objects contain a reference to the process entity, its previous state, its new state, and the timing of the transitions.

  • Log ID (string) [required]: The primary key of the log object.
  • Process ID (string) [required]: The primary key of the process object that is being tracked. Use when setting up an 本体论 link between the process object and the Log object.
  • Old state (string) [required]: The start state of the transition.
  • New state (string) [required]: The end state of the transition.
  • Timestamp (timestamp) [required]: Timestamp of entering the end state.
  • isLatest (boolean) [optional]: True if this log is the most recent for the process object, otherwise False.
  • Duration (long) [optional]: Duration in milliseconds since entering the old state.
  • Path (string) [optional]: A list of all states encountered so far, including the current state. Must be a serialized JSON string.
  • Action type RID (string) [optional]: An identifier of the action type that caused the transition. Typically, NULL for external changes.
  • 拥有RID（字符串）[可选]： 从该操作执行的应用程序的标识符。允许区分手动和自动操作。NULL 表示外部更改。

在进行编辑时，维护这样的日志需要协调。Machinery 提供了一个标准解决方案，可以从应用程序内部安装。设置日志对象类型需要一个流程对象类型。

将有一个对话框引导您完成设置。您可以选择要跟踪的编辑来源：

  • 如果流程对象类型从外部数据源接收更改，您可以选择并配置一个变更日志数据集。
  • 如果流程对象类型可以在平台上进行编辑，您可以选择 在流程对象类型上启用编辑历史记录，并将这些编辑包含在日志对象类型中。Machinery 会自动生成一个编辑历史记录的物化数据集，以便进行汇总分析。

目前不支持对具有多个数据源或行级权限的对象类型进行平台编辑的日志跟踪。

如果要跟踪来自外部数据源的更改，您必须选择一个以标准变更日志格式表示的数据集，这是Machinery的日志对象类型模式的简化版本。该数据集通常位于对象类型的上游数据源，包含以下列：

  • processId（字符串）[必需]：流程实体的唯一标识符。
  • state（字符串）[必需]：流程进入的新状态。
  • timestamp（时间戳）[必需]：转换发生的时间戳。
  • isDeleted（布尔值）[可选]：用于标记实体为已删除的属性。如果最新日志中的isDeleted为True，则该对象不会出现在本体论中。

确认后，Machinery会将一个Marketplace产品安装到您选择的文件夹中。此产品包含一个Pipeline Builder管道，该管道会从配置的源中计算日志条目并将其转换为正确的格式。该产品还会部署日志对象类型本身，并与您的流程对象类型建立本体论链接。

安装完成后，管道需要完成初始构建，日志对象类型需要在数据库中建立索引，数据才能用于挖掘和监控。

已部署的管道会在其任何数据源发生更改时安排运行。因此，日志对象类型会比进程对象类型落后一个构建和一个索引作业，通常大约2-5分钟。如果您需要实时更新，需要维护一个自定义的日志对象类型。

如果自定义日志对象类型满足最小模式，您可以手动映射它。
