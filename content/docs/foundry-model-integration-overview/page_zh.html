模型连接与开发

在 Palantir 平台中，模型是封装任意机器学习逻辑的产物。模型可广泛应用于数据管道、本体和应用层的工作流中，以支持丰富多样的业务场景。

Palantir 平台采用极具弹性的模型集成策略。模型既可以直接在平台上基于集成数据开发，也能在外部环境开发后以库、产物、容器或 API 的形式导入。

完成集成后，模型即可配合平台工具进行推理、部署、数据治理、机器学习运维及业务落地。模型从创建到调用的每一个环节，均受平台机制严格保障，涵盖血缘关系、安全、版本控制、可复现性及审计追踪。

来源

任何形式的模型均可进行平台内开发（例如借助 scikit-learn、TensorFlow 和 OR-tools 等开源工具，或通过自定义库实现），或从外部环境导入（如笔记本、第三方数据科学产品或容器镜像库），亦能直接配置为外部托管的 API。

建模目标

建模目标充当“任务控制中心”，旨在针对特定问题，简化模型的管理、评估、审查、发布和部署流程。除了提供便捷的用户界面，建模目标还提供了数据治理与权限控制层、自动化层（例如对候选模型进行统一评估）以及 CI/CD 层，支持模型实现持续且无停机的部署。

建模目标能够支撑任意建模问题下的完整模型生命周期，涵盖仿真与优化等传统机器学习运维工具难以覆盖的范畴。

借助建模目标，企业不仅能停留在搭建数据管道的层面，更能安全地向用户及决策系统进行模型的业务化部署。模型投入业务运行后，Foundry 平台可基于生产数据、业务结果、应用程序及用户行为，构建机器学习反馈闭环。这些反馈闭环为建模团队提供了强大的数据资产，便于监控、理解并改善生产环境中的模型表现，从而灵活应对新变化。

本体中的模型

本体是 Palantir 平台的业务运行层。它将数字资产与其现实世界中的对应实体相连接，从而赋能各类业务场景。

模型完成平台集成后，即可通过建模目标进行部署，并注册到本体层中调用。这使得高可信模型能够为业务交互式工作流提供支撑，驱动快速且关键的业务决策。
