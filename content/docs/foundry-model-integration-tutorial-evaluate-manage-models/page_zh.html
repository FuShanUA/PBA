3. 教程 - 在建模目标应用中评估模型

在开始本教程之前，您应该已经完成了 建模项目设置 并在 Jupyter® 笔记本 或 代码仓库 中训练了模型；此时，您应该至少有一个模型在您的建模目标中。

在本教程的这一步中，我们将评估我们模型的性能，并将该模型发布到建模目标中。此步骤是推荐的，但不会影响本教程后续步骤，可以稍后返回。这将涵盖：

  1. 什么是建模目标？
  2. 配置自动模型评估
  3. 构建指标管道
  4. 在评估仪表板中评估模型

3.1 什么是建模目标？

建模目标可以看作是潜在的、可投入生产的模型版本的目录。将模型提交到一个目标中，会将该模型添加到该目录，并使其在特定的建模问题或目标背景下可用于评估和审查。无论最终是否投入生产，每次模型提交都有助于跟踪建模项目的进展，并维护项目空间周围的实验和学习历史。

本教程的这一步没有必需的操作。

3.2 如何配置自动模型评估

现在我们已经有一个模型候选在我们的建模目标中，我们可以通过在此建模目标内生成模型性能指标来评估该模型的表现。性能指标是理解你的模型表现如何以及为什么模型会以这种方式运作的重要工具。

由于本教程的目标是估算一个数字（美国人口普查区的平均房价），我们的建模问题可以归类为回归建模问题。对于回归建模问题，通常会查看诸如平均绝对误差、均方根误差等评估指标。这些指标包含在Foundry的默认回归评估器中，因此我们将使用此库来评估我们模型提交的性能。

操作：从建模目标中选择配置评估仪表板。

配置模型评估

自动模型评估是一种确保模型以标准化方式评估的有用方法。标准化确保模型比较的一致性，使您能够自信地选择最适合在生产中使用的模型。

如果启用了评估管道管理；Foundry将为每种模型提交和评估数据集的组合自动生成一个推理数据集。推理数据集是您的模型对评估数据集运行推理（生成预测）的结果。评估数据集由用户定义为模型的标准测试集，需要包含特征（用于生成预测）和标签（用于将模型推理与真实标签进行比较）。

操作： 要配置管道管理，请选择 编辑，然后选择以下两个选项： 生成推理和指标管道 和 在模型提交时自动运行推理和指标。然后，保存 以确认您的管道管理设置。

操作（Action）： 要配置评估数据集，请选择 添加评估数据集，然后将您在 模型训练 教程中创建的 housing_test_data 数据集 作为您的评估数据集。选择您的 data 文件夹作为推理目的地和指标目的地。通过选择 选择数据集和文件夹 确认您的选择。

配置评估库

评估库是一个可参数化的 Foundry 库，可用于对推理数据集进行评估并生成评估指标，这些指标将被添加到建模目标中的评估仪表板。Foundry 为 回归 和 二分类 提供了默认的评估库，但也可以 创建自定义评估库。 针对您特定的建模问题。

为了评估添加到此建模目标的所有模型，所有模型提交必须一致地生成其评估分数。在此建模目标中，我们将期望所有模型生成一个名为prediction的推理列，该列为float类型。

操作：选择选择评估库，然后选择回归默认库。将推理字段配置为prediction，类型为float，实际字段（我们试图估计的属性）为median_house_value，并将直方图分箱留空。保存以保存评估库配置。

配置评估子集

设置评估子集是模型评估中的一个可选步骤，允许您为评估数据的特定部分独立生成指标。这些指标都可以在评估仪表板中单独分析。

如果您希望启用评估子集，请参见：

  • 您希望了解在数据的哪些部分，您的模型表现优于其他部分。这可以帮助确定该模型在生产环境中应使用的范围。
  • 您希望了解模型表现较差的区域，以便将未来开发工作的重点放在这些地方。
  • 您希望确保在评估数据中，您的模型不会对受保护群体产生偏见。

在这种情况下，我们将检查当平均房屋年龄小于5年或超过30年时，模型的表现情况。

操作：选择添加评估子集，然后选择housing_median_age字段。由于这是一个数值字段，我们可以定义定量分桶策略。在此示例中，我们将使用一个范围截断，分桶为5和30。

操作：保存子集配置。

此子集配置将在每个评估数据集上对模型进行四个不同数据集的评估。

  • Overall: 这是完整的评估数据集。
  • housing_median_age (<5): 评估数据集过滤后，其中 housing_median_age 小于 5。
  • housing_median_age (>= 5, < 30): 评估数据集过滤后，其中 housing_median_age 大于等于 5 但小于 30。
  • housing_median_age (>= 30): 评估数据集过滤后，其中 housing_median_age 大于等于 30。

这将使我们能够确定模型在 housing_median_age 不同的记录中是否表现出相似的行为。

操作（Action）: 在页面右上角选择保存配置（Save configuration）以保存并返回到评估仪表板。从现在起，您提交到此目标的任何模型都将自动生成并构建推理和指标数据集，您可以使用这些数据集来评估您的模型。

3.3 如何构建指标管道

配置完指标管道后，每次向该建模目标提交模型时，都会创建并启动一个推理数据集和一个指标数据集。如果已配置，Foundry 还会自动运行这些数据集，并将指标添加到您的建模目标的评估仪表板中。

在这种情况下，因为我们已经将模型添加到了此目标，所以需要手动启动这些数据集的构建。

操作：在评估仪表板的右上角选择构建评估，然后选择housing_test_data作为评估仪表板，linear_regression_model作为要评估的模型。然后，点击构建以开始推理和指标的构建。

注意

您的评估管道可能需要几分钟时间才能创建完成；您可能需要等待直到构建操作变为可用状态。

一旦您的构建开始，您可以通过查看评估仪表板顶部右侧的最近构建下拉菜单，来查看这些构建的进度。

注意

根据您的Foundry实例的负载情况，运行评估管道可能需要几分钟时间。

3.4 如何在评估仪表板中评估模型

在继续本教程之前，您的评估仪表板应已成功完成您之前创建的推理和指标数据集的构建。一旦指标完成，您将能够查看并比较所有已添加到此建模目标中的模型的指标。这为您的建模项目性能提供了一个集中来源。

在回归评估库中，我们生成了一系列可在评估仪表板中查看的指标。这些指标使我们能够了解模型在未见过的测试数据上预测标签（人口普查区的中位房价）的准确性。 确定使用哪些指标以及什么算是良好的性能会因项目而异。此过程通常需要与利益相关者进行讨论，但就我们的虚构示例而言，我们将认为该模型表现得足够好。在这种情况下，均方根误差为82639.10意味着，平均而言，模型预测值与我们未见过的测试数据中的标签相差82,639.10美元。

操作：刷新页面，在左侧栏的< strong >housing_test_dataset 数据集选择器中选择< strong >数据集，然后从模型选择器中选择< strong >线性回归模型。

评估仪表板还会显示我们之前定义的子集的模型性能。我们的评估仪表板中的标签页反映了我们可以查看指标的可用子集组。在这种情况下，我们可以看到当房屋中位年龄在5到30岁之间时，我们的模型表现最好。

操作： 选择评估仪表板顶部的housing_median_age标签页。
