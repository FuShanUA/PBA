在本地开发中预览转换

使用 VS Code 在本地开发中预览转换主要有两种方式：

  • 使用 Palantir 的 Visual Studio Code 扩展预览 Python 转换
  • 使用基于 Gradle 的本地预览预览 Java 转换

使用 Palantir 的 Visual Studio Code 扩展进行预览（仅限 Python）

Palantir 的 Visual Studio Code 扩展支持本地预览功能。有关安装说明，请参阅《扩展文档》。安装扩展并准备好预览环境后，您的转换将自动在预览选项卡中被发现，如下图所示。

您也可以在本地运行 Python 转换的预览。在本地运行预览时，部分数据集会流式传输到您的计算机上。有关更多信息，请查阅《转换预览文档》。

基于 Gradle 的 Java 本地预览

本节详细介绍了在本地开发中预览 Python 和 Java 转换所需的步骤。获取更多上下文信息，请查阅我们的《Java 本地开发文档》。您还可以了解更多关于《如何预览转换》的内容。基于 Gradle 的本地预览会在 Foundry 内部远程执行预览。

前提条件与限制

除本地开发的现有前提条件外，本地预览支持还要求本地分支必须跟踪远程分支，这意味着本地分支至少需要推送一次。请注意以下额外限制：

  • 预览的 URI 只能由运行预览的用户访问，并且仅在临时基础上可用。

运行数据集预览

在运行预览之前，您必须设置本地开发环境，并确保您的代码库已升级至最新模板版本。

  1. 运行 ./gradlew displayTransformsList，该命令将返回所有可用转换的列表。

  2. 运行 ./gradlew datasetPreview --transformId=<transformId>，将 <transformId> 替换为其中一个转换 ID（即上方截图中的蓝色文本），该命令将返回一个指向 Foundry 的链接，您可以在那里访问已计算完成的预览。

  3. （可选）在上述命令中添加 --printMode=table 标志，以直接在终端中打印所有预览数据集的前 10 行，而不是获取预览链接。

  4. （可选）要在预览中包含输入文件，请添加 --inputFiles=<datasetAlias>:<path>，其中 <datasetAlias> 是所选转换函数的其中一个输入数据集，<path> 是该输入数据集内的文件路径。

  5. （可选）要在预览中包含输出文件，请添加 --outputFiles=<datasetAlias>:<path>，其中 <datasetAlias> 是所选转换函数的其中一个输出数据集，<path> 是该输出数据集内的文件路径。
