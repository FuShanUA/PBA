创建或选择时序对象类型

若要向现有对象类型添加时序属性，请在设置助手中遵循选择现有对象类型路径。后续步骤请参阅有关如何设置时序属性的章节。

要创建新的对象类型，首先必须具备一个时序对象类型支持数据集。如果当前没有符合所需模式的数据集，则需要在 Pipeline Builder 中创建一个。

虽然可以在 Pipeline Builder 中将其作为本体输出创建新的对象类型，但我们建议在 Pipeline Builder 中创建时序对象类型支持数据集，然后通过设置助手创建新的对象类型。请按照以下步骤在 Pipeline Builder 中准备数据集。

准备时序对象类型支持数据集

创建新的时序对象类型之前，必须先准备一个时序对象类型支持数据集。以下说明介绍了如何在 Pipeline Builder 中创建此类数据集。

  1. 首先，重点创建一个数据集，其中每一行代表新对象类型中的单个对象。该数据集需要包含一个主键列以唯一标识对象，并且对象的每个非时序属性都需要对应一列。
  2. 接下来，通过为每个时序属性添加序列 ID，使该对象类型支持数据集支持时序。根据数据的具体结构，您可以通过 Pipeline Builder 中的以下转换操作之一来完成添加：
  • 针对不同测量/传感器类型的时序数据存储在不同数据集中，且需要手动创建新的序列 ID 列。
  • 所有测量数据共用单个数据集，且涉及的序列数量庞大。
  • 单个测量/传感器类型对应多个数据集。

多数据集（手动创建新的序列 ID 列）

从包含对象信息的数据集开始（例如下图中的机器信息）：

  1. 在 Pipeline Builder 中添加 Concatenate strings 转换。
    1. 选择一个通用的分隔符，例如下划线 (_)。
    2. 配置表达式：
      1. 输入一个 Value 类型的参数，并将其值设定为序列名称（例如 temperature）。
      2. 输入一个 Column 类型的参数，并将其设定为主键/对象键。
  2. 为这个新列命名，以便于将其识别为特定序列的序列 ID（例如 temperature 或 temperature_series_id）。



单数据集或大量序列

通过联接操作生成一个将各序列名称作为列名的数据集，即可避免手动创建每个新的序列 ID 列。获得此单个数据集后，请按照以下说明操作：

  1. 添加 Unpivot 转换。

    1. 将 Output unpivoted column name 设置为 series_name。
    2. 将 Unpivoted values output column name 设置为 series_value。
    3. 在 Columns to unpivot 字段中，选取所有包含序列名称的列。



  2. 添加 concatenate strings 转换以生成序列 ID。

    1. 选择一个通用的分隔符，例如下划线 (_)。
    2. 配置表达式：
      1. 输入 series_name 作为第一个参数。
      2. 输入对象键作为第二个参数（如下方截图中的 machine_id）。

  3. 将这个新的输出命名为 series_id。



  4. 将序列 ID 列通过对象键（下图中的 `machine_id`）重新关联回您的对象类型底层数据集。此时，您新建的时间序列对象类型底层数据集中，除了包含其他对象类型元数据外，还包含了您刚创建的 `series_id`。



单个测量类型的多个数据集

您的对象类型必须位于 Object Storage V2 中，才能使用多个时间序列同步来支持时间序列属性。

由于传感器数据通常来源于多个数据源，要将所有传感器数据归一化并转换到同一个数据集中往往颇具挑战。有时甚至无法实现，因为部分传感器持有分类数据，而其他传感器包含数值数据；不同的数据类型无法存在于同一个 时间序列同步 中。为了避免将所有传感器数据转换并统一到单一时间序列数据集中，您可以将 时间序列属性 关联到多个时间序列同步。为此，您的对象类型底层数据集中必须包含一列 限定序列 ID创建时间序列同步。

  1. 在 Pipeline Builder 中选择 添加数据，将支持您时间序列属性的每个时间序列同步的底层数据添加到流水线中。

  2. 对于每个时间序列同步数据集，仅选择序列 ID 列，并对生成的单列数据集进行去重。

  3. 添加 创建时间序列参考值 转换。将序列 ID 列作为 序列标识符，并选择相应的时间序列同步作为 时间序列同步 RID。将新列命名为 qualified_time_series_id 或类似名称。



  4. 将限定序列 ID 列 关联 回您的对象类型底层数据集。此步骤要求您的对象类型底层数据集中的序列 ID 必须唯一。

生成的数据集应如下图所示。`seriesId` 对应同步数据集中的序列标识符，而 `syncRid` 对应存储该序列的同步 RID。

创建新的时间序列对象类型

准备好时间序列对象类型底层数据集后，请按照设置助手中的路径执行 创建新对象类型。此路径会将您重定向至 本体管理器对象创建设置助手，您可以在其中选择新数据集作为底层底层数据源。完成设置助手对话框后，您即可开始 设置时间序列属性。

如果您直接从本体管理器主页启动对象创建设置助手（即并非从时间序列设置助手启动），助手在完成后将不会把您重定向至新对象类型的 能力 选项卡。
