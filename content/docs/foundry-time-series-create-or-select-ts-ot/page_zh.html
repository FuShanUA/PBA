创建或选择一个时间序列对象类型

要将 时间序列属性 添加到现有对象类型，按照设置助手中的 选择现有对象类型 路径进行操作。继续阅读有关如何 设置时间序列属性 的部分以了解下一步。

要创建一个新的对象类型，您首先需要一个 时间序列对象类型支撑数据集。如果您还没有符合此所需模式的数据集，则需要在 Pipeline Builder 中创建一个。

虽然可以在 Pipeline Builder 中作为本体论输出 创建新的对象类型，但我们建议在 Pipeline Builder 中创建时间序列对象类型的支撑数据集，然后按照设置助手来创建新的对象类型。请按照以下步骤准备 Pipeline Builder 中的数据集。

准备时间序列对象类型支撑数据集

在创建新的时间序列对象类型之前，您必须首先拥有一个时间序列对象类型支撑数据集。以下说明描述了如何在Pipeline Builder中创建时间序列对象类型支撑数据集。

  1. 首先，专注于创建一个数据集，其中每一行代表新对象类型的单个对象。此数据集需要一个主键列，可用于唯一标识一个对象，并且每个非时间序列 属性（属性） 需要一个列。
  2. 接下来，通过为每个 时间序列属性（时间序列属性） 添加 系列ID（系列ID），使此对象类型支撑数据集支持时间序列。您可能通过Pipeline Builder中的以下转换之一来添加此内容，具体取决于您的数据结构：
  • 您的不同测量/传感器类型的时序数据存储在不同的数据集中，您希望手动创建新的系列ID列。
  • 您有一个用于所有测量的单一数据集和/或大量系列。
  • 您有一个单一测量/传感器类型的多个数据集。

多个数据集（手动创建新的系列ID列）

从包含有关对象信息的数据集开始（例如，下面图像中的机器信息）：

  1. 在Pipeline Builder中添加一个Concatenate strings转换。
    1. 选择一个通用的分隔符，例如下划线（_）。
    2. 配置您的表达式：
      1. 输入一个Value类型输入，并将此值设置为您的系列名称（例如，temperature）。
      2. 输入一个Column 类型输入，并将此设置为您的主键/对象键。
  2. 为此新列命名，以便轻松将其识别为该特定系列的系列ID（例如，temperature 或 temperature_series_id）。



单个数据集或大量系列

通过使用连接来创建一个数据集，其中每个系列名称作为列名，避免手动创建每个新的系列ID列。一旦您有了这个单一数据集，请按照以下说明操作：

  1. 添加一个 Unpivot 转换。

    1. 将 输出未旋转列名 设置为 series_name。
    2. 将 未旋转值输出列名 设置为 series_value。
    3. 在 要未旋转的列 字段中，选择所有具有系列名称的列。



  2. 添加一个 concatenate strings 转换以生成系列ID。

    1. 选择一个通用的分隔符，例如下划线（_）。
    2. 配置您的表达式：
      1. 输入 series_name 作为第一个输入。
      2. 输入对象键作为第二个输入（如下图中的 machine_id）。

  3. 将此新输出命名为 series_id.



  4. 将系列ID列与您的对象类型支撑数据集在对象键上进行关联（如下图中的 machine_id）。您新时间序列对象类型的支撑数据集现在除了其他对象类型元数据外，还具有您创建的 series_id。



多个数据集用于单一测量类型

您的对象类型必须在 对象存储 V2 中，以便通过多个时间序列同步来支撑时间序列属性。

由于传感器数据通常由多个数据源提供，因此在一个数据集中对所有传感器数据进行标准化和转换可能会很困难。有时，这可能无法实现，因为某些传感器包含分类数据，而其他传感器包含数值数据；不同的数据类型不能存在于一个 时间序列同步 中。为了避免需要将所有传感器数据转换并统一到一个时间序列数据集中，您可以将 时间序列属性 链接到多个时间序列同步。要做到这一点，您必须在支持时间序列属性的对象类型数据集中有一列 合格系列 ID。按照以下步骤创建合格系列 ID。请注意，在遵循这些步骤之前，您需要 创建您的时间序列同步。

  1. 通过在Pipeline Builder中选择添加数据，为每个用于支持你的时间序列属性的时间序列同步添加支撑数据。

  2. 对于每个时间序列同步数据集，仅选择系列ID列并去重生成的单列数据集。

  3. 添加一个Create time series reference values转换。使用系列ID列作为系列标识符，并选择适当的时间序列同步作为时间序列同步RID。将新列命名为qualified_time_series_id或类似名称。



  4. 关联已筛选的系列ID列回你的对象类型支撑数据集。此步骤要求你的对象类型支撑数据集中的系列ID是唯一的。

创建一个新的时间序列对象类型

在您准备好了时间序列对象类型支撑数据集后，按照设置助手中的路径来< strong >创建一个新的对象类型。此路径将引导您进入< a href="/docs/foundry/object-link-types/create-object-type/">本体论管理器对象创建设置助手，您将在其中选择新的数据集作为您的支撑数据源。完成助手对话后，您就可以< a href="/docs/foundry/time-series/time-series-properties/#time-series-property-setup">设置时间序列属性。

如果您直接从本体论管理器首页启动对象创建设置助手（即不从时间序列设置助手启动），则助手在完成后不会引导您到新对象类型的功能选项卡。
