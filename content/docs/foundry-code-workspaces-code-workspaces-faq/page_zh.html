常见问题

是否可以在代码工作区中使用 PySpark？

是的，您可以在 Jupyter® 代码工作区中使用 PySpark。但是，它在您的工作区容器内以本地模式运行，并且只使用可用的 CPU 和内存；它不会在分布式 Spark 集群上运行。我们建议使用 Foundry 应用程序，如 管道构建器（Pipeline Builder） 和 代码仓库（Code Repositories） 来运行分布式 Spark 工作负载。

要在您的 JupyterLab® 笔记本中安装 PySpark，请转到 库（Libraries） 选项卡并安装 pyspark 和 openjdk。以下 OpenJDK 版本受支持：

  • PySpark < 4.0.0: OpenJDK 8.x, 11.x 或 17.x
  • PySpark 4.0.0 及以上版本: OpenJDK 17.x 或 21.x

如果遇到如java.lang.UnsupportedOperationException: getSubject is supported only if a security manager is allowed之类的错误，请确保为您的 PySpark 版本将 OpenJDK 固定到受支持的版本。

使用以下代码创建 Spark 会话，将原始 parquet 文件读入 Spark 数据框，然后应用转换。这种方法对于在 Code Repositories PySpark 转换中要应用的 PySpark 转换很有用。

Copied!
1from foundry.transforms import Dataset
2from pyspark.sql import SparkSession
3import pyspark.sql.functions as F
4
5# Create the Spark context. The example below shows a 2 CPU, 16 GB workspace.
6spark = (SparkSession.builder
7    .appName("ParquetExample")
8    .master("local[2]")  # Set to the CPUs available.
9    .config("spark.driver.memory", "10g")  # Define the driver memory, around 50-65% of the container's total.
10    .config("spark.driver.memoryOverhead", "4g")  # Define the overhead memory, about 20-35% of the container's total.
11    .config("spark.sql.shuffle.partitions", "6")  # Optionally define partition count, ~3x the CPU total.
12    .getOrCreate())
13sc = spark.sparkContext
14
15# Download a Foundry dataset of parquet files into the notebook.
16parquet_foundry = Dataset.get("parquet_foundry_dataset")
17files = parquet_foundry.files().download()
18
19# Read the files into a Spark dataframe.
20spark_df = spark.read.parquet(*list(files.values()))
21
22# Apply PySpark transformations.
23df_processed = spark_df.filter(F.col("col1") == 1)
24
25# Write your output dataset.
26spark_output = Dataset.get("spark_output")
27spark_output.write_table(df_processed.toArrow())

可以在 Code Workspaces 中进行 API 调用吗？

是的，您可以在 Code Workspaces 中使用 外部系统 进行 API 调用，这些系统具有网络策略，作为从工作区出站的首选方法。对于 启用了 CBAC 的环境，您应使用在 控制面板 中配置的网络策略。

Code Workspaces 支持哪些 IDE？

Code Workspaces 目前支持 JupyterLab® 和 RStudio® Workbench。

Code Workspaces 不支持哪些包？

由于安全原因，以下 Python 包不被支持：

  • folium
  • pandasgui

如果您对上述包有任何担忧，请联系 Palantir 的代表。

我的代码仓库中的代码为什么是 JSON 格式？

Code Repositories 应用程序以 IPython 格式从关联的 Code Workspaces 接收代码，这会导致代码以 JSON 格式在单元格级别显示。

我可以使用自己的包吗？

可以；请参阅文档中关于 导入包 的内容。 如果您的包托管在一个组织的 Conda/PyPI/CRAN 通道上，Foundry 可以代理该通道，并将其提供给您的项目。如需更多信息，请联系您的 Palantir 代表。

为什么支撑我的代码工作区的代码仓库没有 库 选项卡？

要将库导入您的代码工作区，请使用工作区左侧面板中的 包 选项卡。

我能否在不属于我的代码工作区的代码仓库中编写或编辑代码？

是的，当代码来源于代码工作区时，您可以在代码仓库中直接编辑代码。提交后，您可以使用 同步 或 重置更改 Code Workspaces 中的功能可以检测到工作区中的远程更改。

从概念上讲，你可以将 Code Repositories 视为 Code Workspaces 的版本控制管理器，处理拉取请求、冲突解决和管理，而代码开发可以在 Code Workspaces 中进行。

为什么我的同事看到的视图与我不同？

出于安全原因，用户在使用 JupyterLab® 或 RStudio® 时是相互隔离的。这意味着每个访问相同 Code Workspaces 的用户都将拥有自己的环境。协作通过 git 工作流完成：如果您希望让最新的代码对同事可用，请选择 同步更改 以将您的更改与支撑代码仓库同步，当同事选择 同步 或 重置更改 时，这些更改将变得可用。当多个用户在同一工作区工作时，我们建议他们使用独立的分支。

请注意，我们默认使用 .gitignore 忽略某些文件，以确保没有数据与 git 仓库同步，并限制 git 仓库的大小。我们还会从 JupyterLab® .ipynb 文件中删除所有输出。

我可以导入额外的 LaTeX 包来渲染我的 R Markdown 文件吗？

可以。默认情况下，代码工作区（Code Workspaces）仅预安装用于渲染常见 R Markdown 文件所需的依赖项（如 TinyTex-1 ↗ 所定义的），但您也可以自行安装其他 LaTeX 包：

  1. 下载或创建 TDS 归档文件。例如，您可以从 CTAN ↗ 下载一个 .tds.zip 文件。
  2. 将 TDS 归档文件上传到新或现有的 Foundry 数据集。您可以通过拖放文件夹中的文件来上传它们到新的 Foundry 数据集。
  3. 选择 不带模式的数据集上传。您可以同时将多个 TDS 归档文件上传到同一数据集。
  4. 在代码工作区中，转到数据选项卡，然后选择读取现有数据集。选择包含TDS归档的数据集。
  5. 可选地，更新数据集别名。在这个例子中，我们假设您将数据集别名命名为my_latex_packages。
  6. 复制代码片段，将其粘贴到您的.Rprofile中，并更新它以将文件提取到正确的位置。您需要在下面的代码片段中将my_latex_packages替换为您的数据集别名：
library(foundry)
my_latex_packages_files <- datasets.list_files("my_latex_packages")
my_latex_packages_local_files <- datasets.download_files("my_latex_packages", my_latex_packages_files)
texmflocal <- system("kpsewhich --var-value TEXMFLOCAL", intern = TRUE)
sapply(my_latex_packages_local_files, function(tds_file) { unzip(tds_file, exdir = texmflocal) })
  1. 现在，您可以将您的包包含在R Markdown标题中以使用它：
headers-include:
 - \usepackage{my_package}

我可以在R Markdown文件中使用自定义字体吗？

可以。

  1. 下载或创建您想要使用的TTF文件。
  2. 将TTF文件上传到新的或现有的Foundry数据集。您可以将文件夹中的文件拖放以上传到新的Foundry数据集。
  3. 选择上传到没有模式的数据集 . 您可以同时上传多个TTF文件到同一个数据集。
  4. 在代码工作区中，转到数据选项卡，然后选择读取现有数据集。选择包含TTF文件的数据集。
  5. 可选地，更新数据集别名。在此示例中，我们假设您将数据集别名命名为my_fonts。
  6. 复制代码片段，在R控制台中运行它，并将文件复制到您的仓库中，以便版本控制可以跟踪它们。您需要在下面的代码片段中将my_fonts替换为您的数据集别名：
my_fonts_files <- datasets.list_files("my_fonts")
my_fonts_local_files <- datasets.download_files("my_fonts", my_fonts_files)
dir.create("fonts")
file.copy(unlist(my_fonts_local_files), "fonts")
  1. 现在您可以引用/home/user/repo/fonts目录中的字体，按照R Markdown建议，使它们可以在HTML或PDF输出中使用。

要更改HTML的字体，请在 自定义CSS ↗ 中设置 font-family 。

要更改PDF的字体，可以使用 自定义LaTeX代码 ↗ 在页眉中通过 \newfontfamily 加载字体。

如何防止我的 Jupyter® 工作区在代码运行时暂停？

您可以利用 %%keep_alive 单元格魔法来防止 Code Workspaces 在单元格中的代码运行时暂停 Jupyter® 笔记本。

Copied!
1%%keep_alive
2
3long_running_process()

这将在单元格代码运行期间保持单元格活跃长达 24 小时。如果您的浏览器标签页关闭或闲置太久，JupyterLab® 将停止显示和保存单元格的输出。如果您想稍后查看单元格的输出，可以使用 %%capture 单元格魔法将输出存储在一个变量中：

Copied!
1%%keep_alive
2%%capture cell_output
3
4long_running_process()

在另一个单元格中，将上述变量的输出写入文件以便以后查看：

Copied!
1with open('cell_output.txt', 'w+') as f:
2    f.write(cell_output.stdout)
3    f.write(cell_output.stderr)

如何从我的 Jupyter 或 RStudio 代码工作区下载文件？

Palantir 平台中禁用了 IDE 原生文件下载功能，以确保管理员配置的下载限制得到执行。

为了下载文件，请将文件写入 Foundry 数据集。为此，在 Code Workspaces 侧边栏中，转到 数据 选项卡，然后选择 将数据写入新数据集，并为新数据集输入名称。确认数据集别名后，选择 非表格数据集 作为数据集类型，输入您要上传的文件或目录的位置，并运行生成的代码片段。

一旦文件已上传至 Foundry 数据集，您可以通过点击数据集链接进入数据集预览，并从那里下载文件。

请记住，您的管理员可能在从平台导出数据前要求提供理由，或者限制导出的数据量。
