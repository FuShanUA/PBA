为SAP应用安装Palantir Foundry连接器2.0

Palantir Foundry连接器2.0 for SAP Applications（“连接器”）作为SAP附加组件提供，可以通过SAINT（SAP附加组件安装工具）进行安装。该附加组件以SAR格式交付，文件名格式如下：FOUNDRY-SAPCONN-INST-SP00SPXX.SAR

SP00SPXX表示从SP00到SPXX的安装文件，其中XX是支持包级别。

如果在BW/4HANA或S/4HANA上安装连接器，请包括属性更改包。从SP22开始，这些包已包含在安装文件中。

这是设置连接器所需的高级步骤：

  1. 下载安装包。
  2. 通过SAINT安装连接器附加组件。
  3. 创建一个系统类型（类型 B）的技术用户用于Foundry在 SU01 中。此用户由Foundry代理用于系统到系统的调用，不会用于登录SAP GUI。如果您计划以后启用 通过OAuth 2.0的用户属性回传，用户类型 必须 是 System。
  4. 运行安装后向导。

安装过程中涉及两个不同的SAP用户。在本页面中它们分别被称为：

术语定义
执行用户（基础管理员）一个人类SAP管理员（对话用户），通过交互方式登录以运行 SAINT、安装后向导和 PFCG。需要 SAINT / PFCG / SU01 授权。
Foundry技术用户一个系统类型（类型 B）的用户，不支持交互式登录，由Foundry代理用于RFC/HTTPS调用。接收由安装后向导分配的 /PALANTIR/* 角色。

前提条件

  • SAP_BASIS（技术组件版本，适用于SAP NetWeaver和SAP S/4HANA系统）7.4 SP5或更高版本

或者

  • SAP_BASIS 7.5（无最低SP级别要求）

这些是最低版本。连接器支持所有更高的SAP_BASIS版本，包括基于SAP_BASIS 8.x的SAP S/4HANA发行版（例如，SAP_BASIS 816）。

请确保已阅读以下SAP注释并根据适用情况执行相关步骤：

  • SAP OSS 注释 2645739 - ABAP Add-On OCS 包未经过数字签名

安装连接器附加组件

  1. 使用具有使用 SAINT 权限的用户登录到 SAP 系统客户端 000。
  2. 运行 SAINT 事务。
  3. 将 FOUNDRY-SAPCONN-INST-SP00SPXX.SAR 导入 SAP 服务器；选择 安装包 > 加载包 > 从前端。

对于某些 SAINT/SPAM 版本，SPAM 或 SAINT 设置可能会影响安装过程。在 导入期间的检查 部分下禁用 无法进行签名检查，SAP 注释 2520826 尚未实现 项目。请注意，对于某些 SPAM 版本，此项目完全没有描述，但仍应将其禁用。

  1. 选择 开始 进行安装。
  2. 将列出可用的包。如果没有，请点击过滤图标取消过滤。
  3. 选择 继续 进入下一步。
  4. 对于连接器安装，选择 PALANTIR 和 PALCONN。
  5. 选择 继续 进入支持包选择。
  6. 从列表中选择最高可用的支持包，并确保两个组件选择了相同的 SP 级别。
  7. 确认安装队列并点击 继续。
  8. 选择 在对话模式下继续 进入准备阶段，立即在后台继续 对于其他阶段，然后开始安装。

如果在安装过程中出现警告，请按照警告信息中描述的步骤进行解决。在大多数情况下，可以忽略连接器和远程代理安装的警告信息。特别是标题为“开放数据提取请求”的警告信息可以被忽略，因为安装不会对DDIC结构进行更改，因此不会导致开放数据提取请求终止。

  1. 选择完成以完成安装。

运行安装后向导

连接器安装后向导简化了连接器附加组件安装后的设置活动。要执行安装后和配置操作，基础管理员（执行用户，不是Foundry技术用户）通过SAP GUI登录到主客户端（不是000）。安装后向导在该管理员的会话下运行并配置 Foundry 技术用户；Foundry 技术用户本身不会以交互方式登录。可以使用事务代码 /n/PALANTIR/POST_INST 或从连接器菜单（事务代码：/n/PALANTIR/）访问安装向导。

向导步骤

共有10个步骤，可以一起运行或单独运行。要完成安装，所有步骤都需要启用。如果不是SLT安装，请禁用创建SLT配置。SLT配置在第7步中有详细说明。

执行用户（运行向导的Basis管理员）需要有足够的授权，以便将所需的授权角色分配给Foundry技术用户。此过程类似于通过SU01事务代码维护用户。如果单独进行用户分配，请禁用Assign Roles to Foundry User和Perform Healthcheck。

1. 运行卸载修正

此步骤运行一个后台程序，用于修复连接器对象目录中的SAP包。此修正程序可以在任何时间运行，不会产生任何副作用。

2. 激活SICF服务

此步骤激活两个连接器服务，这些服务是数据传输所必需的。请注意，如果需要，可以使用SICF事务代码手动执行。这两个服务是：

  • /default_host/sap/palantir
  • /default_host/sap/opu/odata/palantir

3. 生成角色

连接器有自己的角色集，在安装期间会导入这些角色。这些角色在安装后仍处于未生成状态。如果需要，可以使用PFCG事务代码和Utilities > Mass Generation手动批量生成这些角色。

  • /PALANTIR/CONTENT_BEX_ALL
  • /PALANTIR/CONTENT_CDS_ALL
  • /PALANTIR/CONTENT_HANA_ALL
  • /PALANTIR/CONTENT_DM_ALL
  • /PALANTIR/CONTENT_EXT_ALL
  • /PALANTIR/CONTENT_FUNCTION_ALL
  • /PALANTIR/CONTENT_INFOPROV_ALL
  • /PALANTIR/CONTENT_SLT_ALL
  • /PALANTIR/CONTENT_TABLE_ALL
  • /PALANTIR/CONTENT_TCODE_ALL
  • /PALANTIR/DEBUG_USER
  • /PALANTIR/MONITORING
  • /PALANTIR/OAUTH_CLIENT
  • /PALANTIR/SERVICE_SLT
  • /PALANTIR/SERVICE_SLT_740
  • /PALANTIR/SERVICE_USER

4. 将角色分配给Foundry技术用户

步骤3中的所有Palantir角色将被分配给在选择屏幕中定义的SAP用户。这应该是上述高层次步骤中第3步创建的系统类型Foundry技术用户，而不是执行用户。

如果此步骤生成有关角色分配、无法检索用户详细信息或缺少授权的错误消息，这表明执行用户没有足够的权限。在这种情况下，请按照错误消息中的建议更正执行用户的授权并重新运行程序。

如果此步骤生成表明Foundry技术用户被锁定的错误消息，请联系锁定的拥有者，在重新运行程序之前解除锁定。

5. 维护默认参数

可以通过这些选择设置资源检查和持续资源检查参数。默认值在选择屏幕上，可以根据您的要求进行更改。当使用此选择运行程序时，连接器参数将根据所选参数进行修改。这些参数也可以使用事务代码 /n/PALANTIR/PARAM 进行维护。

了解 性能参数。

6. 检查ICM设置

此步骤验证ICM设置、主机名和端口（HTTP和HTTPS），并创建可用于测试连接的URL。

7. 创建SLT配置

此步骤仅在通过SAP SLT复制服务器连接到SAP ERP系统时相关。

如果连接器安装在SAP SLT实例之上，可以通过安装后向导创建一个新的SLT配置（ODP）。您也可以使用LTRC事务代码创建SLT配置。

SLT配置的参数如下：

Context Description上下文描述，将在SAP事务LTRC和Foundry中显示给用户。
Data Transfer Jobs数据传输作业的数量。

8. 执行健康检查

连接器在安装的SAP系统中有各种指标来衡量健康状况。连接器会在以下类别中执行健康检查：

  • AGENT: 检查RFC连接和源系统授权。 仅适用于通过网关的远程连接。
  • AUTHORIZATION: 检查Foundry用户在SM53中是否有任何缺失的授权。
  • CONNECTOR: 检查连接器的维护作业是否在系统中安排。
  • ROLE: 检查Foundry用户的角色和授权配置文件。
  • SLT: 检查SLT配置是否正确且健康；例如，BADI_Implementation是否激活、上下文是否激活、SLT维护作业是否安排，或源系统连接是否正常。

9. 浏览器测试URL

安装后，通过使用网页浏览器进行连接性测试来检查连接器是否正常运行非常重要。所需的URL由安装后向导生成。

有几种不同的测试可用。第一和第二个测试是对连接器服务的同步调用；第三和第四个用于批量提取。

  • 关于： 连接器的关于页面。
  • T000 表： 使用 table 对象类型直接提取T000表。
  • T000 带分页-初始化： 使用 table 对象类型初始化T000表的后台提取。
  • T000 带分页-读取第一页： 将检索由上一个URL初始化过程的第一页。

10. Foundry 的源定义模板

运行带有Foundry中“Source Definition”选项的程序后，将显示一个名为“Source Definition”的新标签页，其中包含Foundry中的源配置详细信息。

性能参数

连接器具有以下性能参数，以在系统资源不足以服务Foundry时避免不必要的系统负载。这些设置与ST06值进行比较。

以下列出的默认参数值适用于最新连接器版本的全新安装。

  • MEMORY_FREE: 可用内存百分比。如果此百分比低于系统默认值或用户定义的值，提取操作将停止。连接器默认值为 5%。
  • CPU_IDLE: CPU 空闲百分比。连接器默认值为 5%。
  • CPU_USER: 由用户事务使用的 CPU 百分比。连接器默认值为 80%。
  • CPU_LOAD: 总 CPU 使用率。连接器默认值为 80%。

ST06 关键指标由 SAP 提供给连接器。如果激活了虚拟化增强监控，操作系统特定的信息将被虚拟化特定信息所补充。这可能会影响 ST06 指标并误导连接器。请参阅 SAP OSS 文章 2266266 获取更多详细信息。

初始系统默认值是故意保守的。如果您希望覆盖它们以使其更宽松，请通过 /n/PALANTIR/PARAM 交易进行维护。

以下参数用于启用或禁用资源检查：

参数 ID参数名称参数值默认值描述

SYSTEMMEMORY_CHECK TRUE / FALSETRUE启用或禁用 MEMORY 检查。
