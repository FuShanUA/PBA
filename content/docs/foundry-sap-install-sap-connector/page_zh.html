为 SAP 应用程序安装 Palantir Foundry 连接器 2.0

面向 SAP 应用程序的 Palantir Foundry 连接器 2.0（“连接器”）作为 SAP 附加组件提供，需通过 SAINT（SAP 附加组件安装工具）进行安装。该附加组件以 SAR 格式交付，文件名格式如下：FOUNDRY-SAPCONN-INST-SP00SPXX.SAR

SP00SPXX 代表从 SP00 到 SPXX 的安装文件，其中 XX 为支持补丁包级别。

如果要将连接器安装在 BW/4HANA 或 S/4HANA 上，请在安装中包含属性变更包。自 SP22 起，这些包已包含在安装文件中。

设置连接器所需的高层级步骤如下：

  1. 下载安装包。
  2. 通过 SAINT 安装连接器附加组件。
  3. 在 SU01 中为 Foundry 创建系统类型（类型 B）的技术用户。此用户供 Foundry 智能体用于系统间调用，不用于登录 SAP GUI。如果计划后续启用通过 OAuth 2.0 进行用户归属回写，用户类型必须为 System。
  4. 运行安装后向导。

安装过程涉及两个不同的 SAP 用户。在本页面中，它们分别被称为：

术语定义
执行用户（Basis 管理员）人工 SAP 管理员（对话用户），通过交互式登录运行 SAINT、安装后向导和 PFCG。需要 SAINT / PFCG / SU01 授权。
Foundry 技术用户系统类型（类型 B）用户，无交互式登录，由 Foundry 智能体用于 RFC/HTTPS 调用。接收由安装后向导分配的 /PALANTIR/* 角色。

先决条件

  • SAP_BASIS（技术组件版本，适用于 SAP NetWeaver 和 SAP S/4HANA 系统）7.4 SP5 或更高版本

或者

  • SAP_BASIS 7.5（无最低 SP 级别要求）

这些为最低版本要求。连接器支持所有更高版本的 SAP_BASIS，包括基于 SAP_BASIS 8.x 的 SAP S/4HANA 发行版（例如 SAP_BASIS 816）。

请确保已阅读以下 SAP Note，并在适用时遵循相关步骤：

  • SAP OSS Note 2645739 - 《ABAP 附加组件 OCS 包未进行数字签名》

安装连接器附加组件

  1. 使用具有 SAINT 使用权限的用户登录 SAP 系统 client 000。
  2. 运行 SAINT 事务。
  3. 将 FOUNDRY-SAPCONN-INST-SP00SPXX.SAR 导入 SAP 服务器；选择 Installation package > Load Packages > From Front End。

对于某些 SAINT/SPAM 版本，SPAM 或 SAINT 设置可能会影响安装过程。请在 Checks during import 部分下禁用 Signature check not possible, SAP note 2520826 is not implemented 项。请注意，对于某些 SPAM 版本，该项可能没有任何描述，但仍应将其禁用。

  1. 选择 Start 开始安装。
  2. 系统将列出可用包。如果没有列出，请点击过滤器图标取消过滤。
  3. 选择 Continue 进入下一步。
  4. 对于连接器安装，请选择 PALANTIR 和 PALCONN。
  5. 选择 Continue 进入支持补丁包选择。
  6. 从列表中选择最高可用支持补丁包，并确保两个组件选择了相同的 SP 级别。
  7. 确认安装队列并点击 Continue。
  8. 在准备阶段选择 Continue in Dialog mode，在其他阶段选择 Continue in Background Immediately，然后开始安装。

如果安装过程中出现警告，请按照警告信息中的步骤进行处理。对于连接器和远程智能体安装，大多数警告信息可以忽略。特别是标题为“Open Data Extraction Requests”的警告信息可以忽略，因为安装不会对 DDIC 结构进行更改，因此不会导致打开的数据提取请求终止。

  1. 选择 Finish 完成安装。

运行安装后向导

Connector 安装后向导简化了 Connector 插件安装后的设置工作。要执行安装后配置，Basis 管理员（即执行操作的用户，而非 Foundry 技术用户）需通过 SAP GUI 登录主客户端（非 000）。安装后向导将在该管理员的会话下运行，并配置 Foundry 技术用户；Foundry 技术用户本身绝不会进行交互式登录。可通过事务码 /n/PALANTIR/POST_INST 或从 Connector 菜单（事务码：/n/PALANTIR/）访问安装后向导。

向导步骤

共有 10 个步骤，既可以一起运行，也可以单独运行。要完成安装，必须启用所有步骤。如果这不是 SLT 安装，请禁用创建 SLT 配置。SLT 配置详见步骤 7。

执行操作的用户（运行向导的 Basis 管理员）需要具备足够的权限，以便为 Foundry 技术用户分配所需的角色权限。此过程类似于通过 SU01 事务码维护用户。如果单独进行用户分配，请禁用为 Foundry 用户分配角色和执行健康检查。

1. 运行卸载纠正程序

此步骤会运行一个后台程序，用于修复 Connector 对象目录中的 SAP 软件包。该纠正程序可随时运行，且不会产生任何负面影响。

2. 激活 SICF 服务

此步骤会激活数据传输所需的两个 Connector 服务。请注意，如有需要，也可使用 SICF 事务码手动完成此操作。这两个服务分别为：

  • /default_host/sap/palantir
  • /default_host/sap/opu/odata/palantir

3. 生成角色

Connector 拥有一套自身的角色，这些角色在安装期间导入。安装完成后，这些角色处于未生成状态。如有需要，可使用 PFCG 事务码通过实用程序 > 批量生成手动批量生成这些角色。

  • /PALANTIR/CONTENT_BEX_ALL
  • /PALANTIR/CONTENT_CDS_ALL
  • /PALANTIR/CONTENT_HANA_ALL
  • /PALANTIR/CONTENT_DM_ALL
  • /PALANTIR/CONTENT_EXT_ALL
  • /PALANTIR/CONTENT_FUNCTION_ALL
  • /PALANTIR/CONTENT_INFOPROV_ALL
  • /PALANTIR/CONTENT_SLT_ALL
  • /PALANTIR/CONTENT_TABLE_ALL
  • /PALANTIR/CONTENT_TCODE_ALL
  • /PALANTIR/DEBUG_USER
  • /PALANTIR/MONITORING
  • /PALANTIR/OAUTH_CLIENT
  • /PALANTIR/SERVICE_SLT
  • /PALANTIR/SERVICE_SLT_740
  • /PALANTIR/SERVICE_USER

4. 为 Foundry 技术用户分配角色

步骤 3 中的所有 Palantir 角色将分配给在选择屏幕中定义的 SAP 用户。该用户应为上述高层步骤 3 中创建的 System 类型 Foundry 技术用户，而非执行操作的用户。

如果此步骤生成了有关角色分配、无法获取用户详情或权限不足的错误消息，表明执行操作的用户没有足够的权限。遇到这种情况，请按照错误消息中的建议纠正执行操作用户的权限，然后重新运行程序。

如果此步骤生成的错误消息提示 Foundry 技术用户被锁定，请联系锁定所有者解除锁定，然后再重新运行程序。

5. 维护默认参数

资源检查和持续资源检查参数可通过这些选项进行设置。默认值位于选择屏幕中，可根据需要进行更改。运行带有此选项的程序时，Connector 参数将根据所选参数进行修改。也可使用事务码 /n/PALANTIR/PARAM 维护这些参数。

8. 执行健康检查

Connector 提供了多种指标来衡量其在已安装 SAP 系统中的运行状况。Connector 会在以下类别中执行健康检查：

  • AGENT：检查 RFC 连接和源系统权限。仅与通过网关的远程连接相关。
  • AUTHORIZATION：检查 Foundry 用户在 SM53 中是否存在缺失的权限。
  • CONNECTOR：检查系统中是否已计划 Connector 清理作业。
  • ROLE：检查 Foundry 用户角色和权限配置文件。
  • SLT：检查 SLT 配置是否正确且健康；例如，BADI_Implementation 是否激活、Context 是否激活、SLT 清理作业是否已计划，或源系统连接是否正常。

9. 浏览器测试 URL

安装完成后，通过 Web 浏览器进行连接测试以检查 Connector 是否正常运行非常重要。所需的 URL 由安装后向导生成。

有多种不同的测试可供使用。第一项和第二项测试是对 Connector 服务的同步调用；第三项和第四项用于批量提取。

  • 关于： Connector 的关于页面。
  • T000 表： 使用 table 对象类型直接提取 T000 表。
  • T000 分页初始化： 使用 table 对象类型对 T000 表的后台提取进行初始化。
  • T000 分页读取第 1 页： 检索上一个 URL 初始化的进程的第一页数据。

性能参数

Connector 提供了以下性能参数，以避免在系统资源不足以服务 Foundry 时产生不必要的系统负载。这些设置会与 ST06 的值进行比较。

下面列出的默认参数值适用于最新版本 Connector 的全新安装。

  • MEMORY_FREE： 空闲内存百分比。如果该百分比低于系统默认值或用户定义值，提取操作将停止。Connector 默认值为 5%。
  • CPU_IDLE： CPU 空闲百分比。Connector 默认值为 5%。
  • CPU_USER： 用户事务的 CPU 利用率。Connector 默认值为 80%。
  • CPU_LOAD： CPU 总利用率。Connector 默认值为 80%。

SAP 会向 Connector 提供 ST06 关键指标数据。如果激活了虚拟化增强监控，操作系统特定信息会加入虚拟化特定信息进行扩充。这可能会影响 ST06 数据并对 Connector 产生误导。有关详细信息，请参阅 SAP OSS 文章 2266266。

初始系统默认值是刻意保持保守的。如果您想覆盖这些设置以放宽限制，请通过 /n/PALANTIR/PARAM 事务进行维护。

以下参数用于启用或禁用资源检查：

参数 ID参数名称参数取值默认值描述
