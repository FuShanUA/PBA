入门

本指南将向您展示如何在Python转换仓库中设置数据预期。有关数据预期的概述，请参阅 文档。

仓库设置：transforms-expectations库

打开代码仓库左侧的库搜索面板。transforms-expectations 库应该已经安装。通过检查已安装库的列表来验证这一点。如果 transforms-expectations 尚未安装，请现在搜索并安装它。

转换设置

将 expectations 和 Check 导入到您的转换文件中：

Copied!
1from transforms.api import transform_df, Input, Output, Check
2from transforms import expectations as E

对于一些常见的模式和列期望，您可能还需要导入 types：

Copied!
1from pyspark.sql import types as T

Create checks

单个检查的基本结构：

Copied!
1Check(expectation, 'Check unique name', on_error='WARN/FAIL')
  • expectation（期望） - 单个 expectation（期望），可以是多个子期望的复合期望（例如使用 any/all 操作符）
  • Check unique name（检查唯一名称） - 此名称在转换中必须唯一（同一名称不能在输出和输入之间共享），并且将在应用程序中标识该检查（例如 Data Health、Builds 应用程序）
  • on_error（错误处理） - 定义当期望未满足时作业的行为：
    • FAIL（默认） - 如果检查失败，作业将被中止
    • WARN - 作业将继续运行，并生成警告，由 Data Health 处理
警告

设置 on_error='FAIL' 不支持用于单节点转换中的 虚拟表 输出。

将检查分配给数据集

每个检查应传递给一个输入或输出。将单个检查作为 checks=check1 或多个检查以数组形式传递: checks=[check1, check2, ...]

多个检查

使用多个检查来创建更清晰的期望结构，并分别控制每个有意义检查的行为。

在输出上进行简单主键检查的示例：

Copied!
1from transforms.api import transform, Input, Output, Check
2from transforms import expectations as E
3import polars as pl
4
5
6@transform.using(
7    output=Output(
8        "/path/dataset",
9        checks=Check(E.primary_key('id'), 'Primary Key', on_error='FAIL')
10    ),
11    input=Input("/path/input"),
12)
13def compute(output, input):
14    input_df = input.polars()
15    return output.write_table(input_df)

复杂检查

你也可以使用复合期望添加更复杂的检查。例如，让我们检查在给定范围内列age是否不为 null。请注意，我们可以定义复合期望并在转换中多个检查中使用它，对错误应用不同的行为。

即使检查由复合期望组成，也会将其作为一个整体进行监控。如果你想监控（即监视并获取通知）复合期望的特定部分，建议将它拆分成多个不同的检查。

Copied!
1from transforms.api import transform, Input, Output, Check
2from transforms import expectations as E
3import polars as pl
4
5
6# We assume an age is valid if it is between 0 and 200.
7expect_valid_age = E.all(
8    E.col('age').non_null(),
9    E.col('age').gte(0),
10    E.col('age').lt(200)
11)
12
13
14@transform.using(
15    output=Output(
16        "/path/dataset",
17        checks=[
18            Check(E.primary_key('id'), 'Primary Key', on_error='FAIL'),
19            Check(expect_valid_age, 'Valid age on output', on_error='FAIL')
20        ]
21    ),
22    input=Input(
23        "Users/data/input",
24        checks=Check(expect_valid_age, 'Valid age on input', on_error='WARN')
25    )
26)
27def compute(output, input):
28    input_df = input.polars()
29    return output.write_table(input_df)
