快速入门

本指南将向您展示如何在 Python transforms 代码仓库中设置数据期望。有关数据期望的宏观概述，请参阅《文档》。

代码仓库设置：transforms-expectations 库

打开代码仓库左侧的库搜索面板。`transforms-expectations` 库应该已经安装。通过查看已安装库列表来验证这一点。如果尚未安装 `transforms-expectations`，请立即搜索并安装它。

创建检查

单个检查的基本结构：



已复制！
1Check(expectation, 'Check unique name', on_error='WARN/FAIL')
  • expectation - 单个 预期，可以是多个子预期的复合预期（例如使用 any/all 操作符）
  • Check unique name - 该名称在转换中必须唯一（输出和输入之间不能共用相同名称），并将在各应用（例如 Data Health、Builds 应用）中标识此检查
  • on_error - 定义当不满足预期时作业的行为：
    • FAIL（默认） - 检查失败时作业将中止
    • WARN - 作业将继续运行并生成警告，由 Data Health 处理
警告

在单节点转换中，虚拟表输出不支持设置 on_error='FAIL'。

将检查分配给数据集

每个检查应传递给单个输入或输出。将单个检查作为 checks=check1 传递，或将多个检查作为数组传递：checks=[check1, check2, ...]

多重检查

使用多重检查可以创建更具可读性的预期结构，并分别控制每个有意义检查的行为。

以下是对输出进行简单主键检查的示例：

Copied!
1from transforms.api import transform, Input, Output, Check
2from transforms import expectations as E
3import polars as pl
4
5
6@transform.using(
7    output=Output(
8        "/path/dataset",
9        checks=Check(E.primary_key('id'), 'Primary Key', on_error='FAIL')
10    ),
11    input=Input("/path/input"),
12)
13def compute(output, input):
14    input_df = input.polars()
15    return output.write_table(input_df)

复杂检查

你还可以使用复合期望来添加更复杂的检查。例如，检查 age 列在给定范围内不为空。请注意，我们可以定义复合期望，并在转换过程中的多个检查中复用它，从而在发生错误时应用不同的行为。

即使检查由复合期望构成，它也会作为一个整体受到监控。若要监控（即观察并接收通知）复合期望中的特定部分，建议将其拆分为多个独立的检查。

Copied!
1from transforms.api import transform, Input, Output, Check
2from transforms import expectations as E
3import polars as pl
4
5
6# We assume an age is valid if it is between 0 and 200.
7expect_valid_age = E.all(
8    E.col('age').non_null(),
9    E.col('age').gte(0),
10    E.col('age').lt(200)
11)
12
13
14@transform.using(
15    output=Output(
16        "/path/dataset",
17        checks=[
18            Check(E.primary_key('id'), 'Primary Key', on_error='FAIL'),
19            Check(expect_valid_age, 'Valid age on output', on_error='FAIL')
20        ]
21    ),
22    input=Input(
23        "Users/data/input",
24        checks=Check(expect_valid_age, 'Valid age on input', on_error='WARN')
25    )
26)
27def compute(output, input):
28    input_df = input.polars()
29    return output.write_table(input_df)
