Transforms Excel Parser

There are many ways to extract tabular data from a schemaless dataset containing Microsoft Excel files in Foundry, including Pipeline Builder, Python file-based transforms using open-source libraries like openpyxl ↗, and Java file-based transforms using open-source libraries like Apache POI ↗.

In addition to these options, Palantir provides a library called transforms-excel-parser, which wraps Apache POI with sensible default behavior to make it easy to use in a transforms-java repository with minimal configuration.

Some examples of the useful features and behavior provided by this library include the following:

  • Processing an input dataset containing files with partially overlapping but inconsistent schemas, inferring the schemas from the table headers.
  • Extracting data from multiple sheets (or different tables from the same sheet) and writing the result into multiple output datasets while only reading the file into memory once.
  • Providing a fluent API for defining extraction of fields from non-tabular, "form-style" sheets (where data is located above, below, or adjacent to labels).
  • Setting appropriate global parameters that resolve common Apache POI issues such as incorrect "zip bomb" detection or failures due to exceeding "max byte array size."
  • Supporting both fail-fast and fail-safe behavior with the errorDataframe() method on the ParseResult class, which can be checked at runtime to fail a job or alternatively written to a separate output and checked asynchronously.
  • Providing appropriate configuration options and utility functions for incremental pipelines in order to handle common edge cases such as inconsistent schemas between incremental batches.

Setup

1. Confirm availability of transforms-excel-parser-bundle and add it as a backing repository

Search for transforms-excel-parser in the repository's Maven library panel.

Select the latest version available.

This will show a dialog for importing transforms-excel-parser-bundle as an additional backing repository. Select "Add library".

You may see eddie-spark-module-bundle and/or ri.eddie.artifacts.repository.bundles in addition to transforms-excel-parser-bundle as dropdown options.

The backing repositories with eddie in their name are intended for the exclusive use of the Pipeline Builder application, so they are not the appropriate choice, and using them may lead to issues in the future. If you do not see transforms-excel-parser-bundle as an option, contact your Palantir representative for installation.

2. Add dependency to build.gradle

Add the latest version available to your transforms-java/build.gradle file as below.

transforms-java/build.gradle is a hidden file, so you will need to toggle the Show hidden files setting in order to see it.

API documentation

For detailed API documentation, download the javadoc archive, unzip it, and view the contained HTML files in a web browser. The best place to start when reading the javadoc is com/palantir/transforms/excel/package-summary.html.

Known issues and caveats

Supported file types

The following file formats are currently supported:

  • xls
  • xlt
  • xltm
  • xltx
  • xlsx
  • xlsm

Note that xlsb files are not currently supported.

Code Assist preview instability

When running Code Assist preview, you may observe an issue where the first run after workspace startup succeeds and the second run fails with an error similar to the below:

java.lang.ClassCastException: class com.palantir.transforms.excel.KeyedParsedRecord cannot be cast to class com.palantir.transforms.excel.KeyedParsedRecord (com.palantir.transforms.excel.KeyedParsedRecord is in unnamed module of loader java.net.URLClassLoader @5a5d825a; com.palantir.transforms.excel.KeyedParsedRecord is in unnamed module of loader java.net.URLClassLoader @53dafc50)

This issue is exclusive to the Code Assist preview functionality and does not lead to issues at build time. Refreshing your browser window should allow you to preview again without performing a full Code Assist workspace rebuild.

Memory requirements

The Apache POI library is known for its high memory consumption, which means that even relatively small Excel files can result in a considerable memory footprint when opened. As a result, it is common for default transform Spark profile settings to provide insufficient memory-per-task to accomodate the in-memory objects. Insufficient memory can result in the following issues:

  • The transform job fails with an error like Spark module '{module_rid}' died while job '{job_rid}' was using it.
  • The transform job stalls for an extended period, neither succeeding nor failing.

A rough guideline for identifying if a job is stalling is whether it takes more than 10 minutes to process a single file, since given sufficient memory, a very large Excel file can take about 8 minutes to process. Note that a single Spark task can process multiple input files, so it is not always straightforward to apply this rule.

Whether the symptom of insufficient memory is job failure or job stalling, it is advisable to resolve the issue by switching to a spark profile combination that provides more memory per task, such as EXECUTOR_MEMORY_LARGE and EXECUTOR_CORES_EXTRA_SMALL.

Usage examples

Simple tabular Excel files

Copied!
1package myproject.datasets;
2
3import com.palantir.transforms.excel.ParseResult;
4import com.palantir.transforms.excel.Parser;
5import com.palantir.transforms.excel.TransformsExcelParser;
6import com.palantir.transforms.excel.table.SimpleHeaderExtractor;
7import com.palantir.transforms.excel.table.TableParser;
8import com.palantir.transforms.lang.java.api.Compute;
9import com.palantir.transforms.lang.java.api.FoundryInput;
10import com.palantir.transforms.lang.java.api.FoundryOutput;
11import com.palantir.transforms.lang.java.api.Input;
12import com.palantir.transforms.lang.java.api.Output;
13import java.util.Optional;
14import org.apache.spark.sql.Dataset;
15import org.apache.spark.sql.Row;
16
17public final class SimpleTabularExcel {
18
19    @Compute
20    public void myComputeFunction(
21            @Input("<input_dataset_rid>") FoundryInput myInput,
22            @Output("<output_dataset_rid>") FoundryOutput myOutput,
23            @Output("<error_output_dataset_rid>") FoundryOutput errorOutput
24        ) {
25        // Create a TableParser with an appropriately configured SimpleHeaderExtractor
26        // In this example, the header of the file is on the second row.
27        // If the header were on the first row, we would not need to
28        // specify rowsToSkip, since the default value is 0, and in fact
29        // we could just do TableParser.builder().build() in that case.
30        Parser tableParser = TableParser.builder()
31                .headerExtractor(
32                        SimpleHeaderExtractor.builder().rowsToSkip(1).build())
33                .build();
34
35        // Create a TransformsExcelParser with the TableParser
36        TransformsExcelParser transformsParser = TransformsExcelParser.of(tableParser);
37
38        // Parse input
39        ParseResult result =
40                transformsParser.parse(myInput.asFiles().getFileSystem().filesAsDataset());
41
42        // Get the parsed data, which may be empty if there were no rows in the input
43        // or an error occurred
44        Optional<Dataset<Row>> maybeDf = result.singleResult();
45
46        // If parsed data is not empty, write it to the output dataset
47        maybeDf.ifPresent(df -> myOutput.getDataFrameWriter(df).write());
48
49        // Write error information to the error output
50        errorOutput.getDataFrameWriter(result.errorDataframe()).write();
51}

Excel files with forms

In this example, we register multiple FormParser instances, but it is also possible to register a mix of FormParser and TableParser instances, and that is a common pattern with complex forms that include tabular elements (within the same sheet or across sheets).

Copied!
1package myproject.datasets;
2
3import com.palantir.transforms.excel.TransformsExcelParser;
4import com.palantir.transforms.excel.ParseResult;
5import com.palantir.transforms.excel.Parser;
6import com.palantir.transforms.excel.form.FieldSpec;
7import com.palantir.transforms.excel.form.FormParser;
8import com.palantir.transforms.excel.form.Location;
9import com.palantir.transforms.excel.form.cellvalue.AdjacentCellAssertion;
10import com.palantir.transforms.excel.form.cellvalue.CellValue;
11import com.palantir.transforms.excel.functions.RegexSubstringMatchingSheetSelector;
12import com.palantir.transforms.lang.java.api.Compute;
13import com.palantir.transforms.lang.java.api.FoundryInput;
14import com.palantir.transforms.lang.java.api.FoundryOutput;
15import com.palantir.transforms.lang.java.api.Input;
16import com.palantir.transforms.lang.java.api.Output;
17
18public final class FormStyleExcel {
19    private static final String FORM_A_KEY = "FORM_A";
20    private static final String FORM_B_KEY = "FORM_B";
21
22    @Compute
23    public void myComputeFunction(
24            @Input("<input_dataset_rid") FoundryInput myInput,
25            @Output("<form_a_output_dataset_rid>") FoundryOutput formAOutput,
26            @Output("<form_b_output_dataset_rid>") FoundryOutput formBOutput,
27            @Output("<error_output_dataset_rid>") FoundryOutput errorOutput) {
28        // Form A parser configuration
29        Parser formAParser = FormParser.builder()
30                .sheetSelector(new RegexSubstringMatchingSheetSelector("Form_A"))
31                .addFieldSpecs(createFieldSpec("form_a_field_1", "B1"))
32                .addFieldSpecs(createFieldSpec("form_a_field_2", "B2"))
33                .build();
34
35        // Form B parser configuration
36        Parser formBParser = FormParser.builder()
37                .sheetSelector(new RegexSubstringMatchingSheetSelector("Form_B"))
38                .addFieldSpecs(createFieldSpec("form_b_field_1", "B1"))
39                .addFieldSpecs(createFieldSpec("form_b_field_2", "B2"))
40                .build();
41
42        // TransformsExcelParser with both Form A and Form B parsers
43        TransformsExcelParser transformsParser = TransformsExcelParser.builder()
44                .putKeyToParser(FORM_A_KEY, formAParser)
45                .putKeyToParser(FORM_B_KEY, formBParser)
46                .build();
47
48        // Parse input
49        ParseResult result =
50                transformsParser.parse(myInput.asFiles().getFileSystem().filesAsDataset());
51
52        // Write parsed data to the output datasets
53        result.dataframeForKey(FORM_A_KEY)
54                .ifPresent(df -> formAOutput.getDataFrameWriter(df).write());
55        result.dataframeForKey(FORM_B_KEY)
56                .ifPresent(df -> formBOutput.getDataFrameWriter(df).write());
57
58        // Write error information to the error output
59        errorOutput.getDataFrameWriter(result.errorDataframe()).write();
60    }
61
62    // Helper method to concisely create a FieldSpec with an appropriate assertion
63    private static FieldSpec createFieldSpec(String fieldName, String cellLocation) {
64        return FieldSpec.of(
65                fieldName,
66                CellValue.builder()
67                        .addAssertions(AdjacentCellAssertion.left(1, fieldName))
68                        .location(Location.of(cellLocation))
69                        .build());
70    }
71}

Incremental processing with multiple outputs

Copied!
1package myproject.datasets;
2
3import com.palantir.transforms.excel.ParseResult;
4import com.palantir.transforms.excel.Parser;
5import com.palantir.transforms.excel.TransformsExcelParser;
6import com.palantir.transforms.excel.functions.RegexSubstringMatchingSheetSelector;
7import com.palantir.transforms.excel.table.CaseNormalizationOption;
8import com.palantir.transforms.excel.table.SimpleHeaderExtractor;
9import com.palantir.transforms.excel.table.TableParser;
10import com.palantir.transforms.excel.utils.IncrementalUtils;
11import com.palantir.transforms.lang.java.api.*;
12
13public final class IncrementalTransform {
14
15    @Compute
16    public void myComputeFunction(
17            @Input("<input_dataset_rid>") FoundryInput myInput,
18            @Output("<sheet_1_output_dataset_rid>") FoundryOutput sheet1Output,
19            @Output("<sheet_2_output_dataset_rid>") FoundryOutput sheet2Output) {
20        // Define the parsers
21        // Specifying either CONVERT_TO_LOWERCASE or CONVERT_TO_UPPERCASE for a
22        // CaseNormalizationOption is especially important with incremental processing
23        // to avoid subtle issues due to inconsistent casing between input files across
24        // incremental batches.
25        Parser sheet1Parser = TableParser.builder()
26                .headerExtractor(SimpleHeaderExtractor.builder()
27                        .caseNormalizationOption(CaseNormalizationOption.CONVERT_TO_LOWERCASE).build())
28                .sheetSelector(new RegexSubstringMatchingSheetSelector("Sheet1")).build();
29        Parser sheet2Parser = TableParser.builder()
30                .headerExtractor(SimpleHeaderExtractor.builder()
31                        .caseNormalizationOption(CaseNormalizationOption.CONVERT_TO_LOWERCASE).build())
32                .sheetSelector(new RegexSubstringMatchingSheetSelector("Sheet2")).build();
33        TransformsExcelParser transformsParser = TransformsExcelParser.builder().putKeyToParser("Sheet1", sheet1Parser)
34                .putKeyToParser("Sheet2", sheet2Parser).build();
35
36        // Parse the data
37        FoundryFiles foundryFiles = myInput.asFiles();
38        ParseResult result = transformsParser.parse(foundryFiles.getFileSystem(ReadRange.UNPROCESSED).filesAsDataset());
39
40        // Check for errors and fail fast
41        // With incremental processing in particular, it is often better to fail fast
42        // instead of writing the error dataframe to a separate output and
43        // checking it asynchronously. If failing fast is not an option and you
44        // adopt the "write error dataframe to separate output" approach, note
45        // that you will need to either ① re-upload files that had parse errors to the input dataset
46        // or ② force a snapshot build of this transform via a manual dummy transaction
47        // on one of the output datasets in order to trigger the reprocessing of the
48        // files that had parse errors.
49        if (result.errorDataframe().count() > 0) {
50            throw new RuntimeException("Errors: " + result.errorDataframe().collectAsList().toString());
51        }
52
53        // Write parsed data incrementally to the outputs via an APPEND transaction if possible
54        // and a merge-and-replace SNAPSHOT transaction if not
55        // The below implementation assumes that it is normal and expected for a subset of the
56        // parsers to find no data in a given incremental batch of files. If that is not
57        // the case, you may want to raise an exception if a subset of the dataframes is
58        // absent and there were a non-zero number of unprocessed files in the input.
59        // An absent result does not necessarily mean that an error will be present
60        // in the error dataframe (for example, a SheetSelector returning an empty
61        // collection of sheets is not considered an error).
62        FilesModificationType filesModificationType = foundryFiles.modificationType();
63        result.dataframeForKey("Sheet1").ifPresent(
64                dataframe -> IncrementalUtils.writeAppendingIfPossible(filesModificationType, dataframe, sheet1Output));
65        result.dataframeForKey("Sheet2").ifPresent(
66                dataframe -> IncrementalUtils.writeAppendingIfPossible(filesModificationType, dataframe, sheet2Output));
67    }
68}

Custom interface implementations

Most of the configuration methods in the TableParser and FormParser builders, such as sheetSelector and headerExtractor, specify interfaces instead of classes in their argument signatures. Therefore, you can define custom implementations when the built-in implementations, such as RegexSubstringMatchingSheetSelector, SimpleHeaderExtractor, and MultilayerMergedHeaderExtractor, are not sufficient. Additionally, since most of these interfaces have only one method, you can usually use Java lambda expressions ↗ when defining a custom implementation in this way.

For example, the built-in MultilayerMergedHeaderExtractor requires you to specify the top-left and bottom-right cell names statically. However, inconsistencies may exist between input files in the number of empty rows before the header begins. The example below shows how you can define a custom implementation of the HeaderExtractor interface that dynamically identifies the start of the header, uses that information to define a local instance of MultilayerMergedHeaderExtractor, and then returns the result of calling the local MultilayerMergedHeaderExtractor's extractHeader method.

Consult the API documentation for detailed definitions of HeaderExtractor and other interfaces.

Copied!
1package myproject.datasets;
2
3import com.palantir.transforms.excel.ParseResult;
4import com.palantir.transforms.excel.Parser;
5import com.palantir.transforms.excel.TransformsExcelParser;
6import com.palantir.transforms.excel.table.HeaderExtractor;
7import com.palantir.transforms.excel.table.MultilayerMergedHeaderExtractor;
8import com.palantir.transforms.excel.table.TableParser;
9import com.palantir.transforms.excel.utils.WorkbookUtils;
10import com.palantir.transforms.lang.java.api.*;
11import org.apache.poi.ss.usermodel.Cell;
12import org.apache.poi.ss.usermodel.Row;
13import org.apache.poi.ss.usermodel.Sheet;
14import org.apache.poi.ss.util.CellReference;
15import org.apache.spark.sql.Dataset;
16
17import java.util.Optional;
18
19public class CustomHeaderExtractorUsageExample {
20    private static Optional<String> getFirstCellValue(Sheet sheet, int rowIndex) {
21        Row row = sheet.getRow(rowIndex);
22        if (row == null) {
23            return Optional.empty();
24        }
25        Cell cell = row.getCell(0);
26        if (cell == null) {
27            return Optional.empty();
28        }
29        String value = WorkbookUtils.getValueAsString(cell);
30        if (value == null) {
31            return Optional.empty();
32        }
33        return Optional.of(value);
34    }
35
36    @Compute
37    public void myComputeFunction(
38            @Input("<input_dataset_rid>>") FoundryInput myInput,
39            @Output("<output_dataset_rid>") FoundryOutput myOutput,
40            @Output("<error_output_dataset_rid>") FoundryOutput errorOutput
41    ) {
42        HeaderExtractor headerExtractor = sheet -> {
43            int rowsInSheet = sheet.getLastRowNum();
44            Integer firstHeaderRow = null;
45            for (int currentRowIndex = 0; currentRowIndex < rowsInSheet; currentRowIndex += 1) {
46                Optional<String> firstCellValue = getFirstCellValue(sheet, currentRowIndex);
47                // Assume that we can always identify the start of the header by looking for the first row
48                // in which the value in the first cell is the string "record_id".
49                if (firstCellValue.isPresent() && firstCellValue.get().equals("record_id")) {
50                    firstHeaderRow = currentRowIndex;
51                    break;
52                }
53            }
54            if (firstHeaderRow == null) {
55                return Optional.empty();
56            }
57            // Assume that the height of the header is always two rows. We could implement
58            // additional logic to attempt to find the height on a per-file basis
59            // if that assumption were not valid.
60            int lastHeaderRow = firstHeaderRow + 1;
61            return MultilayerMergedHeaderExtractor
62                    .builder()
63                    .topLeftCellName(new CellReference(firstHeaderRow, 0).formatAsString())
64                    // Assume that the width of the header is always five columns (the indices in the CellReference
65                    // constructor are zero-indexed). We could implement additional logic to attempt to find
66                    // the width on a per-file basis if that assumption were not valid.
67                    .bottomRightCellName(new CellReference(lastHeaderRow, 4).formatAsString())
68                    .build()
69                    .extractHeader(sheet);
70        };
71
72        Parser tableParser = TableParser.builder()
73                .headerExtractor(headerExtractor)
74                .build();
75
76        TransformsExcelParser transformsParser = TransformsExcelParser.of(tableParser);
77
78        ParseResult result =
79                transformsParser.parse(myInput.asFiles().getFileSystem().filesAsDataset());
80
81        Optional<Dataset<org.apache.spark.sql.Row>> maybeDf = result.singleResult();
82
83        maybeDf.ifPresent(df -> myOutput.getDataFrameWriter(df).write());
84        errorOutput.getDataFrameWriter(result.errorDataframe()).write();
85    }
