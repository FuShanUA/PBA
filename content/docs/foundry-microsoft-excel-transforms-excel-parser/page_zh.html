Transforms Excel 解析器

在 Foundry 中，从包含 Microsoft Excel 文件的无模式（schemaless）数据集里提取表格数据有多种方式，包括使用流水线构建器（Pipeline Builder）、使用 openpyxl ↗ 等开源库进行基于 Python 的文件转换，以及使用 Apache POI ↗ 等开源库进行基于 Java 的文件转换。

除了上述选项之外，Palantir 还提供了一个名为 transforms-excel-parser 的库。该库对 Apache POI 进行了封装，并提供了合理的默认行为，使其能够在基于 Java 的转换（transforms-java）代码库中通过极简的配置即可轻松上手。

该库提供的一些实用功能和行为示例如下：

  • 处理包含部分重叠但模式不一致文件的输入数据集，并根据表头推断模式。
  • 从多个工作表（或同一工作表中的不同表格）中提取数据，并将结果写入多个输出数据集，而整个过程只需将文件加载到内存一次。
  • 提供流式 API，用于定义从非表格化的“表单式”工作表（即数据位于标签的上方、下方或相邻位置）中提取字段。
  • 设置适当的全局参数，以解决 Apache POI 的常见问题，例如错误的“压缩炸弹（zip bomb）”检测或因超出“最大字节数组大小”而导致的失败。
  • 通过 ParseResult 类的 errorDataframe() 方法同时支持快速失败（fail-fast）和故障安全（fail-safe）行为。该方法可在运行时进行检查以使作业失败，或者将其写入单独的输出中进行异步检查。
  • 为增量管道提供适当的配置选项和实用工具函数，以处理诸如增量批次间模式不一致等常见的边缘情况。

已知问题与注意事项

支持的文件类型

目前支持以下文件格式：

  • xls
  • xlt
  • xltm
  • xltx
  • xlsx
  • xlsm

请注意，目前不支持 xlsb 文件。

代码辅助预览的不稳定性

在运行 Code Assist 预览时，您可能会遇到这样的问题：工作区启动后的首次运行成功，但第二次运行会失败，并报出类似以下的错误：

java.lang.ClassCastException: class com.palantir.transforms.excel.KeyedParsedRecord cannot be cast to class com.palantir.transforms.excel.KeyedParsedRecord (com.palantir.transforms.excel.KeyedParsedRecord is in unnamed module of loader java.net.URLClassLoader @5a5d825a; com.palantir.transforms.excel.KeyedParsedRecord is in unnamed module of loader java.net.URLClassLoader @53dafc50)

此问题仅存在于 Code Assist 预览功能中，不会在构建时引发故障。 刷新浏览器窗口即可重新预览，无需完整重建 Code Assist 工作区。

内存要求

Apache POI 库内存消耗较大，这意味着即使是体积相对较小的 Excel 文件，在打开时也会占用大量内存。因此，默认的转换 Spark 配置通常无法为任务提供足够的内存来处理内存中的对象。内存不足可能导致以下问题：

  • 转换作业失败并报错，例如 Spark module '{module_rid}' died while job '{job_rid}' was using it。
  • 转换作业长时间卡顿，既不成功也不失败。

判断作业是否卡顿的一个粗略参考标准是：处理单个文件的时间是否超过 10 分钟。因为在内存充足的情况下，处理一个非常大的 Excel 文件大约需要 8 分钟。需要注意的是，单个 Spark 任务可能会处理多个输入文件，因此这条规则并不总是适用。

无论内存不足的表现是作业失败还是作业卡顿，建议通过切换到为每个任务提供更多内存的 spark 配置组合来解决此问题，例如 EXECUTOR_MEMORY_LARGE 和 EXECUTOR_CORES_EXTRA_SMALL。

自定义接口实现

TableParser 和 FormParser 构建器中的大多数配置方法（例如 sheetSelector 和 headerExtractor），在其参数签名中指定的是接口而非具体的类。因此，当内置实现（如 RegexSubstringMatchingSheetSelector、SimpleHeaderExtractor 和 MultilayerMergedHeaderExtractor）无法满足需求时，你可以定义自定义实现。此外，由于这些接口大多只包含一个方法，因此在以这种方式定义自定义实现时，通常可以直接使用 Java 的 lambda 表达式 ↗。

例如，内置的 MultilayerMergedHeaderExtractor 要求你静态指定左上角和右下角的单元格名称。然而，不同的输入文件在表头开始前的空行数量上可能存在差异。下面的示例展示了如何定义 HeaderExtractor 接口的自定义实现：动态识别表头的起始位置，利用该信息创建 MultilayerMergedHeaderExtractor 的局部实例，并返回调用该局部实例 extractHeader 方法后的结果。

有关 HeaderExtractor 及其他接口的详细定义，请查阅 API 文档。

1package myproject.datasets;
2
3import com.palantir.transforms.excel.ParseResult;
4import com.palantir.transforms.excel.Parser;
5import com.palantir.transforms.excel.TransformsExcelParser;
6import com.palantir.transforms.excel.table.HeaderExtractor;
7import com.palantir.transforms.excel.table.MultilayerMergedHeaderExtractor;
8import com.palantir.transforms.excel.table.TableParser;
9import com.palantir.transforms.excel.utils.WorkbookUtils;
10import com.palantir.transforms.lang.java.api.*;
11import org.apache.poi.ss.usermodel.Cell;
12import org.apache.poi.ss.usermodel.Row;
13import org.apache.poi.ss.usermodel.Sheet;
14import org.apache.poi.ss.util.CellReference;
15import org.apache.spark.sql.Dataset;
16
17import java.util.Optional;
18
19public class CustomHeaderExtractorUsageExample {
20    private static Optional<String> getFirstCellValue(Sheet sheet, int rowIndex) {
21        Row row = sheet.getRow(rowIndex);
22        if (row == null) {
23            return Optional.empty();
24        }
25        Cell cell = row.getCell(0);
26        if (cell == null) {
27            return Optional.empty();
28        }
29        String value = WorkbookUtils.getValueAsString(cell);
30        if (value == null) {
31            return Optional.empty();
32        }
33        return Optional.of(value);
34    }
35
36    @Compute
37    public void myComputeFunction(
38            @Input("<input_dataset_rid>>") FoundryInput myInput,
39            @Output("<output_dataset_rid>") FoundryOutput myOutput,
40            @Output("<error_output_dataset_rid>") FoundryOutput errorOutput
41    ) {
42        HeaderExtractor headerExtractor = sheet -> {
43            int rowsInSheet = sheet.getLastRowNum();
44            Integer firstHeaderRow = null;
45            for (int currentRowIndex = 0; currentRowIndex < rowsInSheet; currentRowIndex += 1) {
46                Optional<String> firstCellValue = getFirstCellValue(sheet, currentRowIndex);
47                // Assume that we can always identify the start of the header by looking for the first row
48                // in which the value in the first cell is the string "record_id".
49                if (firstCellValue.isPresent() && firstCellValue.get().equals("record_id")) {
50                    firstHeaderRow = currentRowIndex;
51                    break;
52                }
53            }
54            if (firstHeaderRow == null) {
55                return Optional.empty();
56            }
57            // Assume that the height of the header is always two rows. We could implement
58            // additional logic to attempt to find the height on a per-file basis
59            // if that assumption were not valid.
60            int lastHeaderRow = firstHeaderRow + 1;
61            return MultilayerMergedHeaderExtractor
62                    .builder()
63                    .topLeftCellName(new CellReference(firstHeaderRow, 0).formatAsString())
64                    // Assume that the width of the header is always five columns (the indices in the CellReference
65                    // constructor are zero-indexed). We could implement additional logic to attempt to find
66                    // the width on a per-file basis if that assumption were not valid.
67                    .bottomRightCellName(new CellReference(lastHeaderRow, 4).formatAsString())
68                    .build()
69                    .extractHeader(sheet);
70        };
71
72        Parser tableParser = TableParser.builder()
73                .headerExtractor(headerExtractor)
74                .build();
75
76        TransformsExcelParser transformsParser = TransformsExcelParser.of(tableParser);
77
78        ParseResult result =
79                transformsParser.parse(myInput.asFiles().getFileSystem().filesAsDataset());
80
81        Optional<Dataset<org.apache.spark.sql.Row>> maybeDf = result.singleResult();
82
83        maybeDf.ifPresent(df -> myOutput.getDataFrameWriter(df).write());
84        errorOutput.getDataFrameWriter(result.errorDataframe()).write();
85    }
