配置并应用样式

Slate 是基于 Palantir 开源的 Blueprint 框架构建的，和任何其他网站一样，它使用 CSS 来设置 DOM 的样式。这为小部件提供了统一的外观和感觉，并内置了“深色模式”的切换功能。这些并不是“皮肤”或“模板”，而是每个 Slate 小部件内置的。

这意味着如果你的应用程序设计需要特定的用户界面，你的开发工作将集中在提供一组样式覆盖规则上，以提供新的规则。这增加了一定的复杂性，因为你必须了解 Slate 默认样式被应用的位置，然后生成一个具有正确特异性水平的 CSS 选择器来覆盖它。

在开发应用程序样式时，你最好的助手将是 Chrome 开发者工具 - 特别是“检查器” - 它允许你查看网页中任何元素的渲染后的 HTML 和 CSS。右键点击 → “检查” 将以高亮显示点击的元素打开检查器。这对于识别哪些类被应用到哪个元素上非常重要，因此可以确定需要扩展或覆盖哪些类。

关于 LESS 的说明

从技术角度来说，在 Slate 应用中编写的 CSS 是一种预编译方言，称为 Less ↗。Less 提供了简化冗长 CSS 的语法，并且是一个强大的工具，用于编写复杂的样式同时保持可读性。在页面加载时，所有的 Less 都会被编译成 CSS，因此当你检查渲染后的页面时，如果使用了 Less 功能，可能会看到一些差异。阅读 Less 文档以了解有关常见模式和有用功能的更多信息。

小部件样式

在每个单独的小部件的显示配置中的样式输入区域，您可以定义小部件级别的不同样式和类。这些样式仅在所选的小部件上可用。如果小部件是容器，则其所有嵌套子项也将使用这些样式。

在小部件中编写样式可能难以管理或跟踪，与将样式存储在中心位置相比。对于较大的项目，您应避免将样式放入单个小部件中，而应通过样式面板来管理样式。

样式表

最清晰的模式是在样式面板中定义新类，并使用附加类配置将它们应用到各个小部件。有两类样式表可供使用，以支持更复杂的样式。

本地样式表

本地样式表在应用级别定义样式和类。每个Slate应用只有一个本地样式表，可以在整个应用中引用。

蓝图样式

Slate 使用由 Blueprint ↗ 提供的样式，这是一个由 Palantir 创建的开源框架。

您可以在应用程序中使用许多 Blueprint 类和图标来构建常见的 HTML 用户界面元素，并帮助您的应用程序更无缝地匹配平台其余部分，从而实现专业、生产级的外观和感觉。

颜色

Blueprint 提供了一系列 核心和扩展颜色名称 ↗，这些颜色名称在选择时已考虑到 WCAG 2.0 的合规性（用于可访问的应用程序设计）。

这些可以直接在Slate样式中使用LESS变量进行引用。

组件

Blueprint中有许多组件，可用于创建动态用户交互元素或为您的应用程序增添精致感，从 按钮（按钮） ↗ 到 进度条（进度条） ↗ 再到 标签（标签） ↗。

要在Slate中使用这些组件，请注意每个组件的 CSS API（CSS接口）（而不是JavaScript API）部分，以了解是否可以使用相关的 pt-x 类和HTML元素在Slate中创建类似的组件。

动态样式模式

由于Slate的样式需要在页面加载时确定，所有CSS样式必须是静态的。这意味着，例如，您不能将Handlebars语句放入您的CSS类中以尝试使小部件改变颜色或更改任何其他属性。但是，有几种不同的模式可用于实现动态样式。

动态“附加类”

尽管您不能生成或修改 在运行时动态添加类，你可以使用 Handlebars 为每个小部件的 附加类 配置进行模板化。常见的模式是预先定义多个不同的类，然后使用一个函数来确定在特定情况下应该应用哪些类。如果你正在构建一个大型、有状态的应用程序 - 请参见上面的章节 - 那么你可能需要一个中心函数，用于确定所有正确的类，以应用于任何给定条件下的所有不同小部件，然后返回一个更复杂的映射，将小部件与应应用的正确类关联起来。

通过这种方式，你可以轻松地构建一个应用程序，它可以通过更改颜色、图标或其他显示属性，甚至通过显示和隐藏整个小部件来响应并调整自身。有关此特定模式的更多信息，请参见上面的“响应式布局”部分。

Style 属性

虽然 CSS 对于您的应用程序，必须预先编译，HTML 小部件上的样式属性可以使用 Handlebars 进行模板化。这仅适用于您提供要在单元格中显示的 HTML 的 HTML 小部件或表格；在这些情况下，您可以动态生成 CSS 并将其放在 style 属性 ↗ 中该元素的属性。

暗色模式加载屏幕

在初始页面加载期间，您通常会看到一个深色侧边栏加载一个白色屏幕。要加载一个完全的深色屏幕，您可以使用查询参数 ?$theme=dark 来以与侧边栏一致的深色主题加载整个页面。

请注意，这是一次性加载，如果没有添加查询参数，它不会在页面刷新后保留。另外请注意，这只有在 Slate 应用程序开始加载时才适用，而不是在浏览器仍在路由到应用程序时。

自定义字体

Slate 支持使用自定义字体，以进一步定制应用程序的外观以适应您的组织。要使用自定义字体，首先必须将字体作为 otf 文件上传到 Foundry。上传后，现在可以通过样式表导入字体，并使用字体文件的 rid 将其分配给一个字体系列。
