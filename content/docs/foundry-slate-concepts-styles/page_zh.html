配置并应用样式

Slate 构建于 Palantir 开源的 Blueprint 框架之上，并且与其他网站一样，使用 CSS 为 DOM 设置样式。这为小部件提供了一致的外观和体验，并内置了“深色模式”切换功能。这些并非“皮肤”或“模板”，而是直接内置于每个 Slate 小部件中。

这意味着，如果您的应用设计需要特定的 UI，您的开发工作将集中于提供一组新的样式覆盖规则。这增加了一定的复杂性，因为您必须了解 Slate 默认样式应用在何处，然后生成具有正确特异性级别的 CSS 选择器来覆盖它。

在应用中开发样式时，您最好的帮手将是 Chrome 开发者工具（Chrome Developer Tools）——特别是“检查器”——它允许您查看网页中任何元素的已渲染 HTML 和 CSS。右键点击 →“检查”将打开检查器，并高亮显示所点击的元素。这对于识别哪些类被应用到了哪些元素上，从而确定哪些类需要被扩展或覆盖，是至关重要的。

关于 LESS 的说明

从技术上讲，您在 Slate 应用中编写的 CSS 是一种名为 《Less》↗ 的预编译器方言。Less 提供了精简冗长 CSS 的语法，是编写复杂样式同时保持可读性的强大工具。在页面加载时，所有的 LESS 都会被编译为 CSS，因此当您检查已渲染的页面时，如果您一直使用 LESS 功能，可能会看到一些差异。请阅读 LESS 文档以获取有关常见模式和实用功能的更多信息。

小部件样式

在每个单独小部件的显示（Display）配置中的样式（Styles）输入区，您可以在小部件级别定义不同的样式和类。这些样式仅在所选小部件上可用。如果该小部件是一个容器，这些样式也将适用于其所有嵌套的子小部件。

与将样式集中存储相比，在小部件中编写的样式可能难以管理或跟踪。对于较大的项目，您应避免将样式放入单独的小部件中，而是通过样式面板来管理样式。

样式表

最整洁的模式是在样式面板中定义新类，然后使用附加类（Additional Classes）配置将它们应用到各个小部件上。目前有两种不同类型的样式表可用于支持更复杂的样式。

本地样式表

本地样式表在应用级别定义样式和类。每个 Slate 应用都恰好拥有一个本地样式表，可以从整个应用中引用它。

Blueprint 样式

Slate 使用了由 《Blueprint》↗ 提供的样式，这是由 Palantir 创建的一个开源框架。

您可以在应用中使用许多 Blueprint 类和图标，以便在 HTML 中构建常见的 UI 元素，帮助您的应用更无缝地与平台的其余部分匹配，并实现专业、生产级的外观和体验。

颜色

Blueprint 提供了一系列核心及扩展颜色名称↗，这些颜色的选择充分考虑了 WCAG 2.0 标准的合规性（面向无障碍应用设计）。

这些颜色可以通过 LESS 变量在 Slate 样式中直接引用。

组件

Blueprint 中有许多组件可用于创建动态用户交互元素或为您的应用增添一份精致感，从 《按钮》 ↗ 到 《进度条》 ↗ 再到 《标签》 ↗ 应有尽有。

要在 Slate 中使用这些组件，请注意每个组件的 CSS API（而非 JavaScript API）部分，以了解您是否可以使用相关的 pt-x 类和 HTML 元素在 Slate 中创建类似的组件。

图标

Blueprint 还包含了一套详尽的图标库，可以轻松作为徽标或小型 UI 元素集成到您的应用中。

引用 Blueprint 图标的方法因版本而异。要引用 Blueprint 图标，请遵循特定版本对应的图标 CSS API 说明。

动态样式模式

由于 Slate 的样式需求必须在页面加载时确定，所有的 CSS 样式必须是静态的。举例来说，这意味着您不能在 CSS 类中插入 Handlebars 语句来试图改变组件的颜色或其他任何属性。不过，有多种模式可以实现动态样式。

动态“附加类”

虽然您无法动态生成或修改类，但您可以使用 Handlebars 为每个组件的附加类配置进行模板化处理。常见模式是预先定义多个不同的类，然后通过函数决定在特定情况下应用哪些类。如果您正在构建一个大型有状态应用——参见前文相关章节——您可能需要一个中心函数，为所有不同条件下的各类组件确定正确的应用类，并返回组件与正确应用类之间更为复杂的映射关系。

通过这种方式，您可以轻松构建出能够通过改变颜色、图标或其他显示属性来响应交互的应用，甚至可以通过显示和隐藏整个组件来实现自适应尺寸。更多关于此特定模式的信息，请参阅前文的“响应式布局”章节。

style 属性

虽然应用的 CSS 必须经过预编译，但您可以使用 Handlebars 对 HTML 组件的 style 属性进行模板化处理。这仅适用于您自行提供要在单元格中显示的 HTML 的 HTML 组件或表格；在这些情况下，您可以动态生成 CSS 并将其放入该元素的 style 属性 ↗ 中。

深色模式加载屏幕

在初始页面加载期间，您通常会看到深色侧边栏加载白色屏幕。要加载全深色屏幕，您可以使用查询参数 ?$theme=dark，以与侧边栏一致的深色主题加载整个页面。

请注意，这是一次性加载，若未添加该查询参数，页面刷新后将无法保持此状态。此外还需注意，此参数仅在 Slate 应用开始加载时生效，而在浏览器仍在路由至该应用的过程中无效。

自定义字体

Slate 支持使用自定义字体，以便进一步根据您的组织需求定制应用程序的外观。要使用自定义字体，必须先将该字体文件作为 otf 格式上传至 Foundry。上传完成后，即可通过样式表导入该字体，并利用字体文件的 rid 将其分配给特定的字体族。
