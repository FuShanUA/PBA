入门指南

本教程将引导您构建一个简单的移动端 Workshop 应用程序，让用户能够查看航班数据、筛选航班并查看每个航班的详细信息。本教程使用大多数 Foundry 环境中内置的示例数据。由于 Foundry 本体已根据您的具体需求与数据完成定制，您可能无法访问本教程中描述的完全相同的对象类型。

本教程重点介绍创建移动端 Workshop 应用程序的必要步骤，并假定您已对构建 Workshop 应用程序及使用本体操作配置回写有一定的了解。如果您从未使用过 Workshop 或操作，建议先从以下常规的 Workshop 和操作类型教程入手：

  • 《Workshop 入门指南》
  • 《操作类型入门指南》

第一部分：初始设置

首先，我们将按照以下步骤创建一个可用于滚动浏览航班数据的基础应用程序：

  1. 创建一个新的 Workshop 模块。

  2. 在左侧边栏中选择 设置，并将设备类型切换为 移动端。编辑器的尺寸将切换为移动设备尺寸，同时布局将更新为单列。

  3. 暂时先关闭导航栏，方法是禁用右上角 模块导航栏可见 旁边的开关。

  4. 在左侧边栏中选择 概览，然后选择 添加对象集变量，开始向应用程序添加数据。本例中，请将 [示例数据] 航班 对象类型添加到您的模块中。

  5. 选择 + 添加组件，向应用程序中添加一个 对象列表 组件。配置该组件以使用您的航班对象集变量，随后选择要在列表中显示的属性。本例中，我们将把 承运人名称、出发城市名称、到达城市名称、出发时间 和 到达时间 属性添加到列表中。

至此，您已创建好一个可用于滚动浏览航班数据的基础应用程序。下一步，我们将为该应用程序添加一些交互功能。

第二部分：添加筛选功能

在本部分的教程中，我们将添加筛选航班列表的功能。为了给用户提供足够的屏幕空间来应用筛选器，我们将把筛选功能嵌套在 Workshop 抽屉中。

  1. 在左侧边栏中选择 布局，接着选择 + 并向应用程序添加一个 新抽屉。将抽屉的标题和名称更新为“筛选航班”，并将其图标设置为筛选图标。

  2. 在抽屉内，选择 + 添加组件 并添加一个 筛选列表 组件。将筛选列表的输入数据配置为您先前创建的航班对象集变量。

  3. 在 筛选器配置 下，选择几个允许用户筛选的属性。本例中，我们将为 出发城市名称、到达城市名称 和 取消原因 属性添加筛选器。

  4. 点击编辑器视图右上角的 X 关闭抽屉。返回航班视图后，点击顶部工具栏中的 + 在工具栏中添加一个组件。选择 按钮组 组件。

  5. 将按钮设置为 极简样式 和 大号样式，随后选择 按钮 1 来配置该按钮及其行为。本例中，我们希望该按钮仅显示筛选图标。移除按钮文本，然后在 左侧图标 下选择筛选图标。选择 + 添加事件，接着选择 打开 筛选航班。

  6. 接下来，更新对象列表以使其遵循筛选列表组件生成的筛选条件。选择对象列表组件，然后点击 输入数据 下的 X 移除对象集变量配置。

  7. 选择 新建对象集变量，并将新变量命名为“Filtered flights”。

  8. 在 起始对象集 下，选择名为“[Example Data] Flight”的现有对象集变量。接着，选择 使用变量过滤，并选择筛选列表组件的输出。

此时，用户已经可以根据您配置的一组属性来筛选航班列表了。为避免界面拥挤，这些筛选器会显示在整页抽屉中，方便用户在返回筛选后的列表前，一次性配置多个筛选条件。

试着打开抽屉，筛选出从 丹佛 (Denver, CO) 出发且因 天气 原因取消的航班。关闭抽屉，您会看到航班列表已相应更新。

下一步，我们将让用户能够浏览所选航班的更多详细信息。

第 3 部分：添加所选对象视图

在本部分的教程中，我们将实现让用户从列表中选择航班时，能够导航到其他页面查看详细信息。

  1. 在左侧边栏选择 布局，然后点击 + 并向应用添加 新页面。将页面名称更改为“Flight details”。在新页面的页眉中，删除版块标题。

  2. 点击工具栏左侧的 +，添加 对象集标题 组件。配置该组件，将“Object list 1 Active object”作为其输入对象集。

  3. 在页面内，选择 设置布局，然后选择 流式布局。这会为页面添加一个可滚动的主体，能够容纳多个组件。

  4. 接下来，我们将展示当前航班的一些属性：

  • 选择新建的流式布局中的版块，将其标题更改为“Properties”。

  • 选择 + 添加组件，并添加属性列表组件

  • 配置属性列表组件，同样将“Object list 1 Active object”作为其输入对象集

  • 更新属性列表组件，使用堆叠标签布局并设置为 1 列，随后选择几个要展示的属性。这里，我们将展示 Operating carrier、Carrier Name、Cancelled 和 Cancellation Reason。

  1. 接着，我们将展示出发机场和到达机场的信息：
  • 在流式布局底部选择 + 添加版块。将新版块的标题更改为“Origin Airport”
  • 选择 + 添加组件，并添加属性列表组件
  • 点击 选择对象集变量...，然后点击 + 新建对象集变量。以“Object list 1 Active object”变量为起点，接着使用 [Example Data] Origin Airport 链接进行 搜索。将这个新变量重命名为“Active flight origin airport”。
  • 更新属性列表组件，使用堆叠标签布局并设置为 1 列，随后选择几个要展示的属性。这里，我们将展示 Display Name、Number of Carriers、Number of Departing Flights、Number of Destinations 属性。

  1. 重复上述步骤，添加一个展示到达机场信息的版块。

  2. 使用左侧的 布局 边栏切回显示航班对象列表的原始页面。选择您的对象列表，然后在配置选项的 对象活动选择时 下选择 + 添加事件。选择 切换至 Flight details。

至此，您已经添加了导航到不同页面查看列表中各航班详细信息的功能。试着在对象列表中选择一个航班，导航至您刚刚创建的“Flight details”页面。

最后，我们将在桌面浏览器和手机上预览您的移动应用。

第 4 部分：查看并使用模块

在右上角点击 保存并发布 保存模块，然后选择 查看 ↗。这将在 Foundry 的移动应用启动器中打开您的应用。

此时，你可以通过以下几种方式在桌面浏览器和手机上预览移动应用：

  • 桌面浏览器预览。若要预览应用在真实手机上的显示效果，请使用浏览器自带的移动设备模拟功能：

    • Chrome 设备模式 ↗

    • Safari 响应式设计模式 ↗

    • Edge 模拟移动设备 ↗

  • 在移动设备上预览。如果你的移动设备可以访问 Foundry 环境的网络，也可以直接在手机上打开 URL 来访问应用。在 Chrome 和 Edge 等浏览器中，只需右键点击页面并选择为此页面创建二维码即可轻松实现。随后，用移动设备扫描该二维码便可打开应用。

请注意，在移动应用启动器中，你可以使用浏览器的“后退”和“前进”功能来浏览页面。在 Android 设备上，可以使用系统的“返回”按钮进行导航；而在 iOS 设备上，则可以按如下所示从左向右或从右向左滑动屏幕。
