基于分类的访问控制

基于分类的访问控制（Classification-based Access Controls）在 Foundry 中默认未启用。不同机构之间的分类标记可能有所不同，因此可以在 Palantir 环境中进行不同的配置。分类标记的配置需要 Palantir 的参与。

基于分类的访问控制（CBAC）是用于保护敏感政府信息的强制性控制措施。分类标记也称为分类标记，通过要求用户具有特定的分类标记才能访问信息来限制访问。

对分类标记的访问可能与 Palantir 平台外进行的安全许可流程相关联。与一般强制性控制一样，分类标记可以与其他访问要求结合使用，例如 自主角色 和 强制标记，这些标记可能应用于某个资源。

分类标记的关键特征

分类标记有三个显著的特征：

  • 层级性： 分类标记的一个常见用例是限制对敏感信息的访问，其中信息的敏感性以分层方式定义。例如，您可能有一组用户，他们只能访问标记为“机密”或更低级别的敏感数据。另一组用户可以访问标记为“绝密”或更低级别的敏感信息，这可能包括“机密”数据。
  • 分类标记中的排斥性元素： 非分类标记是联合使用的 - 用户需要拥有资源上所有标记才能获得访问权限。分类标记可以具有排斥性组件，其中属于分类标记排斥性组件中任一组的用户可以满足CBAC访问条件。这通常用于定义可发布性，例如在不同组织或国家之间共享。例如，来自country A 或 country B 的用户可以满足分类标记中的排斥性元素（参见 下文）。
  • 分类标记的普遍性： 使用CBAC强制控制的环境要求所有项目都设置一个项目分类。此外，所有数据集都必须具有数据分类，这意味着原始数据集（即没有输入的数据集）必须具有文件分类。

关键概念

分类标记

分类标记是在类别中配置的。例如，一个类别可能定义数据的类型，另一个类别可能描述该数据应如何传播（即共享）。

一个分类可以有多个分类标记。一个分类可以由不同类别的分类标记组成。关于哪些分类标记组合是有效的规则可以由Palantir配置并在平台上执行。

连接和分离分类标记类别

分类标记类别的一个定义特征是它们可以支持分离（OR）行为。当一个类别是连接（AND）时，用户必须拥有该类别中所有使用的分类标记才能访问分类数据。当一个类别是分离时，用户只需拥有该类别中的任意一个标记即可被授权访问标记的分类数据。

整个分类的组成部分是连接在一起的。这意味着如果一个分类包含多个类别的分类标记，用户必须满足每个分类标记类别的所有组件才能访问数据。

文件和数据分类

文件分类是指用户为了发现文件必须满足的分类。这种限制是除其他要求（如项目的分类和项目或文件的强制标记）之外的额外条件。

数据分类适用于某些类型的文件，例如数据集。数据分类是指用户在查看文件中的数据时必须满足的分类。用户必须满足数据分类才能查看文件中的数据，但这不会影响他们发现数据集的存在并查看其元数据（如名称、描述和模式）。数据分类不能直接编辑；相反，数据分类是通过以下方式组合形成的：

  • 如果已设置资源的文件分类。
  • 所有上游 数据依赖项 的数据分类。

这意味着数据分类始终至少和文件分类以及所有上游数据依赖的数据分类一样严格。

文件、数据和项目分类在 Palantir 平台上与其他适用的访问要求一起进行沟通。与数据分类不同，数据分类会自动继承，而文件分类可以在资源侧边栏中进行编辑，如下所示。

项目分类

项目分类控制谁能够发现该项目并访问其中的资源。要访问一个资源，用户必须满足该资源的项目分类以及资源的文件和数据分类。在使用分类标记的环境中，所有项目都必须具有项目分类。在创建项目时必须选择一个分类。请注意，分类可以更新，但不能删除。

项目分类不会影响项目中数据集的数据分类，因此项目分类不会沿着数据依赖关系继承。如果其他项目中有派生的下游数据集，只有数据分类会被继承。这与项目标记的行为不同，项目标记会被下游数据集继承。

项目最大分类

项目最大分类指定项目内所有资源的最大分类。项目最大分类也称为“允许标记限制”。具有更高数据或文件分类的资源不能被创建或移动到具有较低最大分类的项目中。分类低于或等于项目最大分类的资源可以存在于项目中，但这些资源只能被满足整体项目分类的用户看到。

默认情况下，项目最大分类在项目创建时等于项目分类，但它们可以独立于项目分类进行编辑或删除。

删除项目的最大分类可能需要将对象或链接类型添加到项目中。不过要注意的是，如果一个对象或链接类型位于项目中，当它缺乏文件分类时，将无法实例化。

如果在另一个项目中的上游数据集上添加了更高的分类级别，并且被该项目内的数据集继承为数据标记，那么该数据标记将违反项目的最大分类。如果发生这种情况，数据将继续受到更高分类的保护，但会显示警告信息，并且在解决违规之前无法在该项目中构建数据集或任何下游资源。可以通过修复上游数据集上的分类并移除该上游数据集作为输入并重新构建数据集，或者通过更新项目的最大分类来解决此违规。这种行为与项目约束违规相同。
