项目限制

项目限制允许项目数据主人对项目中文件可应用或不可应用的标记进行设限。项目限制可防止用户将违规文件保存到项目中；如需详细了解如何管理项目限制，请参阅相关文档。请注意，如果数据集违反了其所在项目的限制，在违规问题解决之前，该数据集将无法构建。

项目限制分为以下三种类型：

  • 无限制（默认）：项目中允许所有标记，并可将这些标记设为访问要求。
  • 允许的标记：项目中仅允许特定的标记，并可在项目文件中将其设为访问要求。
  • 禁止的标记：该项目不允许使用指定的标记，且无法在项目文件中将其设为访问要求。此限制实际上允许在项目中使用除所列标记之外的任何标记的数据。

项目限制通常用于防止用户意外关联数据。在某些场景下，用户可能需要访问多种标记，但不应允许特定标记的数据进行组合关联。为了提供这种保护，管理员可以对相关项目设置项目限制，避免用户意外关联不应组合的数据。

例如，出于合规性考虑，银行可能要求敏感的投资数据绝不能与研究数据进行关联。然而，合规人员可能需要分别访问投资数据和研究数据才能完成工作。为了防止合规人员意外关联投资数据和研究数据，平台管理员可以在同时包含投资数据和研究数据的项目上应用“允许的标记”类型的限制，以保护项目数据。
