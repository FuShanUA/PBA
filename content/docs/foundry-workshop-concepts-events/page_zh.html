事件

Workshop 模块中的事件使您能够在用户执行特定操作时触发相应行为。许多微件都可以触发事件，包括 按钮组、在行选择时的 对象表、在选中或取消选中时的 字符串下拉菜单，以及 选项卡 微件。详情请参阅我们的《事件触发与导航微件》文档。可用事件的完整列表如下文所述，但可用的事件类型可能因微件而异。

事件执行顺序

Workshop 中的事件会根据其配置顺序依次执行。然而，事件不会等待前一个事件的下游计算完成后再执行。

关键行为

当 设置变量值 事件触发时：

  • 源变量值会立即复制到目标变量值。
  • 在下一个配置的事件执行时，目标变量值将处于最新状态。
  • 在下一个配置的事件执行之前，依赖于目标变量的下游变量将不会处于最新状态。

此行为对于其他更新变量值的事件类型同样适用。

限制

Workshop 不支持强制事件在进入下一个事件之前等待所有下游更新完成。

如果您的工作流要求在事件之间完全传播下游更新，请将这些事件拆分为多个由用户触发的事件。这样一来，在下一个事件触发之前，下游计算就能得以完成。

事件类型

层

层事件会触发 Workshop 模块中《叠加层》（抽屉和模态框）在屏幕显示状态上的变化。

打开/关闭 {叠加层名称}

对于模块中的每个叠加层，提供以下两个事件：

  • 打开： 打开事件名称中指定的叠加层。
  • 关闭： 关闭事件名称中指定的叠加层。

布局

布局事件会触发 Workshop 模块内屏幕显示的变化，例如切换所选页面、展开或折叠特定部分，以及在《选项卡》布局中切换当前激活的选项卡。

切换至 {页面名称}

对于模块中的每个页面，都可以使用一个事件在触发时切换至所选页面。如果模块正在使用字符串变量来实现《基于变量的页面选择》选项，则该变量的值不会因为切换至页面事件而发生更新。若希望保持此变量值与所选页面同步，可以改用《设置变量值》事件。

展开/折叠/切换 {部分名称}

对于模块中的每个可折叠部分，提供以下三个事件：

  • 展开： 展开事件名称中指定的部分。
  • 折叠： 折叠事件名称中指定的部分。
  • 切换： 若当前已折叠，则展开事件名称中指定的部分；若当前已展开，则折叠指定的部分。

如果指定部分有一个布尔变量支撑其折叠状态，则该变量的值不会因为这些事件而发生更新。若希望保持此变量值与该部分的折叠状态同步，可以改用《设置变量值》事件。

切换至 {选项卡名称}

对于模块中的每个选项卡区域，系统会为该区域内的每个选项卡添加一个切换至 {选项卡名称}事件。与切换至 {页面名称}和区域折叠状态事件不同，更改所选选项卡的事件还会更新为基于变量的选项卡选择配置的字符串变量值（前提是已配置该变量）。

变量

变量事件为应用构建者提供了在 Workshop 模块内更改变量值的方法。

重置 {变量名称} 值

重置 {变量名称} 值事件会将所选变量的值设为其默认值，即在变量定义中配置的值。此选项适用于静态变量。

重新计算 {变量名称}

重新计算 {变量名称}事件会根据所选变量的输入值和变量定义，重新计算其值。此选项适用于非静态变量类型，对于管理未启用自动重新计算行为的变量尤为实用。

设置变量值

设置变量值事件会将所选源变量的当前值赋给所选目标变量。

配置选项
  • 源变量：当触发设置变量值事件时，该变量值会立即复制，无需等待由同时启动的其他事件可能引发的输入变量重新计算。
  • 目标变量：该变量值将被源变量的值覆盖。

将 LLM 响应流式传输到变量

将 LLM 响应流式传输到变量事件支持在 Workshop 模块中实时展示大语言模型的响应。

配置选项
  • 目标变量：用于流式接收响应的字符串变量。
  • 提示词：发送给大语言模型的提示词。可以是字符串变量，也可以是返回字符串的函数。
  • 模型：要使用的语言模型。支持三种 OpenAI 模型：GPT-3、GPT-4 和 GPT-4 32K。
  • 温度：与模型配合使用的温度值，介于 0 和 1 之间的数字。较高的值（如 0.8）会使输出更加随机，而较低的值（如 0.2）则会使输出更加集中和确定。
  • 变量更新延迟：变量更新的频率，以毫秒为单位。

AIP Assist

`Send to AIP Assist` 事件提供了一种将提示词以静态文本或字符串变量形式发送至 AIP Assist 并自动执行查询的方法。触发该事件时，若 AIP Assist 侧边栏尚未开启，系统将自动展开。为进一步优化用户体验，构建者还可指定默认聊天机器人——一种配备了企业专属知识、由大语言模型驱动的助手——来解答关于自定义业务操作议题的查询。AIP Assist 聊天机器人以自定义内容作为搜索上下文，可在《Chatbot Studio》（前身为 Agent Studio）[Beta] 中进行配置。

在事件配置阶段即可选定默认的 AIP Assist 聊天机器人，便于构建者根据特定场景选择最合适的助手。聊天机器人可接入自定义内容，例如当前应用文档或其他操作流程，从而在关键时刻提供针对性辅助。选择聊天机器人并非必选步骤，用户也能在 AIP Assist 侧边栏中自行挑选专用机器人。通过设定默认聊天机器人，构建者可确保任务调用的是正确助手，免去依赖终端用户手动选取的麻烦。事件中发送的提示词支持静态文本或动态变量，构建者可根据提示词内容与接收方智能体灵活定制用户体验。

配置

可通过《按钮组小部件》来添加 `Send to AIP Assist` 事件。以下将演示如何使用按钮组小部件创建该事件。选中该小部件后，系统会展开 AIP Assist 侧边栏并发送已配置的提示词。

  1. 选择 + Add widget（+ 添加小部件），并从组件列表中选取 Button group（按钮组）以完成添加。

  2. 在 Button Configuration（按钮配置）区域，根据需要设定按钮名称、图标及描述。

  3. 在按钮配置面板的 On Click（点击时）区域，从下拉菜单中选择 Event（事件）。

  4. 点击 +Add event（+添加事件），并从选项列表中选择 Send to AIP Assist。

  5. 配置该事件，选择提示词采用 Static（静态）文本还是计算得出的 Variable（变量）。

    • Static: 选择此项后，直接输入提示词文本即可。点击按钮时，该文本将作为提示词发送至 AIP Assist。例如：“库存管理应用判断何时生成‘低库存预警’的阈值标准是什么？”
    • Variable: 选择此项后，需为 `Send to AIP Assist` 事件配置一个字符串类型的变量，并通过 Select a variable（选择变量）下拉菜单将其选中。

  6. （可选）设置事件触发时默认调用的智能体。此功能使构建者能够针对特定场景选择合适的 AIP Assist 智能体。需注意，此步骤对用户体验影响显著，因为用户虽可在 AIP Assist 侧边栏中选用专用智能体，但在配置阶段预设默认项，可确保任务匹配到正确的智能体，无需终端用户手动选择。该选项还能让用户接入包含自定义内容的智能体。

有关 AIP Assist 智能体配置的更多信息，请参阅 AIP Assist 智能体相关文档。

为 `Send to AIP Assist` 事件配置变量时，请确保变量值在调用前处于最新状态。变量配置方式多样，其中 String → Variable Transformation（字符串至变量转换）类型支持根据 Workshop 应用的活动状态来配置复杂的动态提示词。以下是一组串联变量的配置示例，它们会依据 Workshop 的活动状态进行程序化变更。在此例中，从活动对象表中选取的 `Inventory Material`（库存物料）对象采用 String concatenation Variable Transform（字符串拼接变量转换），并与自由文本相结合，最终构成可发送至 AIP Assist 的提示词。
