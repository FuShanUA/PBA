事件

在Workshop模块中的事件允许您在用户执行特定操作时触发特定行为。事件可以从许多小部件中触发，包括 按钮组、对象表（在行选择时）、字符串下拉框（在选择或取消选择时）和 标签页 小部件。了解更多请参阅我们的 事件触发和导航小部件 文档。下面列出了所有可用的事件，但可用的事件类型可能因小部件而异。

事件执行顺序

Workshop中的事件根据其配置顺序依次执行。但是，在执行事件之前，事件不会等待前一个事件的下游计算完成。

关键行为

当一个 设置变量值 事件触发时：

  • 源变量值会立即复制到目标变量值。
  • 在下一个配置的事件执行时，目标变量值将是最新的。
  • 依赖于目标变量的下游变量在下一个配置的事件执行前不会是最新的。

对于其他更新变量值的事件类型，这种行为类似。

限制

Workshop 不支持强制事件在继续到下一个事件之前等待所有下游更新完成。

如果您需要在事件之间完全传播下游更新，请将事件拆分为多个用户触发的事件。这允许下游计算在下一个事件触发前完成。

事件类型

图层

图层事件会触发Workshop模块中屏幕显示状态的更改，包括 覆盖层（抽屉和模态框）。

打开/关闭 {覆盖层名称}

对于模块中的每个覆盖层，以下两个事件可用：

  • 打开: 打开事件名称中指定的覆盖层。
  • 关闭: 关闭事件名称中指定的覆盖层。

布局

布局事件会触发Workshop模块内屏幕显示的变化，例如切换选定的页面、展开或折叠给定的部分，以及在Tabs布局中切换活动标签页。

切换到{页面名称}

对于模块中的每个页面，当事件被触发时，有一个事件可用于切换到选定的页面。如果模块使用字符串变量作为基于变量的页面选择选项，此变量的值不会因“切换到页面”事件而更新。如果您希望保持该变量值与所选页面同步，可以改用设置变量值事件。

展开/折叠/切换{部分名称}

对于模块中的每个可折叠部分，有以下三个事件可用：

  • 展开： 展开事件名称中指定的区域。
  • 折叠： 折叠事件名称中指定的区域。
  • 切换： 如果当前区域是折叠状态，则展开事件名称中指定的区域；如果当前区域是展开状态，则折叠该指定区域。

如果指定的区域有一个布尔变量来支撑折叠状态，由于这些事件之一，此变量的值不会被更新。如果您希望保持此变量的值与区域的折叠状态同步，可以改用 设置变量值 事件。

变量

变量事件为应用构建者提供了在Workshop模块中更改变量值的方法。

重置{variable name}值

重置{variable name}值事件会将所选变量的值设置为其默认值，即变量定义中配置的值。此选项适用于静态变量。

重新计算{variable name}

重新计算{variable name} 事件将根据其输入值和变量定义重新计算所选变量的值。此选项适用于非静态变量类型，对于管理没有Automatic重新计算行为的变量的重新计算特别有用。

设置变量值

设置变量值事件将把所选源变量的当前值赋给所选目标变量的值。

配置选项
  • 源变量：这是在触发设置变量值事件时立即复制的变量值，而无需等待其他同时开始的事件对其输入变量进行重新计算。
  • 目标变量：该变量值将被源变量的值覆盖

将 LLM 响应流式传输到变量

将 LLM 响应流式传输到变量 事件允许在 Workshop 模块中实时显示 LLM 的响应。

配置选项
  • 目标变量： 将响应流式传输到的字符串变量。
  • 提示： 发送到 LLM 的提示。这可以是一个字符串变量或返回字符串的函数。
  • 模型： 要使用的语言模型。支持三个 OpenAI 模型：GPT-3，GPT-4 和 GPT-4 32K。
  • 温度： 与模型一起使用的温度值，一个介于 0 和 1 之间的数字。较高的值，如 0.8，会使输出更加随机，而较低的值，如 0.2，会使输出更集中和确定。
  • 变量更新延迟： 变量更新的频率，单位为毫秒。

AIP Assist

Send to AIP Assist事件提供了一种方式，可以将提示作为静态文本或字符串变量发送到AIP Assist并自动运行查询。当触发此事件时，如果AIP Assist侧边栏尚未打开，它会自动打开。为了进一步定制用户体验，构建者还可以提供一个默认的聊天机器人，这是一个配备了企业特定知识的LLM驱动助手，用于回答有关自定义操作主题的问题。AIP Assist聊天机器人使用自定义内容作为其搜索上下文，并可在 Chatbot Studio（以前称为Agent Studio）[Beta] 中进行配置。

可以在事件配置期间选择一个默认的AIP Assist聊天机器人，使构建者能够为特定上下文选择合适的聊天机器人。聊天机器人可以访问自定义内容，例如正在使用的应用程序的文档或其他操作流程，以便在相关时间提供有针对性的帮助。选择聊天机器人是可选的，用户也可以在AIP Assist侧边栏中选择专用的聊天机器人。通过选择默认的聊天机器人，构建者可以确保已选择了适用于该任务的正确聊天机器人，而无需依赖最终用户进行手动选择。事件中发送的提示可以是静态文本或动态变量，使构建者可以根据提示和接收代理的情况自由定制用户体验。

配置

Send to AIP Assist事件可以通过使用按钮组小部件添加。以下是对Send to AIP Assist事件创建的逐步说明，使用了按钮组小部件。当被选中时，此小部件将打开AIP Assist侧边栏并发送配置好的提示。

  1. 通过选择+ 添加小部件，然后从小部件列表中选择按钮组来添加一个按钮组小部件。

  2. 在按钮配置部分下，按照需要配置按钮的名称、图标和描述。

  3. 在按钮配置面板的点击时部分，从下拉菜单中选择事件。

  4. 选择+ 添加事件，然后从选项列表中选择发送到AIP Assist。

  5. 通过选择提示应该是静态文本还是计算的变量来配置事件。

    • 静态：对于此选项，只需输入提示文本。当按钮被选中时，此文本将作为提示发送到AIP Assist。例如：“库存管理应用程序使用什么阈值来确定何时创建‘库存不足警报’？”
    • 变量： 对于此选项，您必须 配置一个字符串类型变量 用于 Send to AIP Assist 事件，并使用 选择一个变量 下拉菜单进行选择。

  6. （可选）选择在事件触发时默认使用的代理。此功能使构建者可以为给定上下文选择适当的 AIP Assist 代理。请注意，此步骤可能对用户体验产生重大影响，因为用户也可以在 AIP Assist 侧边栏中选择专用代理，但在配置期间选择默认代理可以确保已选择了任务的正确代理，而不依赖最终用户进行手动选择。此选项可能让用户访问具有自定义内容访问权限的代理。

有关 AIP Assist 代理配置的更多信息，请参阅 AIP Assist 代理的 文档。

数据过时

刷新模块中的数据事件允许在触发此事件时重新加载模块中的所有数据。

Module appearance

The Toggle between light and dark mode event allows the theme of the module to be changed by the user when this event is triggered.
