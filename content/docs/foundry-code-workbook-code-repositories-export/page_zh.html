导出至代码仓库

Python 和 SQL 代码可以从 Code Workbook 导出至代码仓库。将代码迁移至代码仓库对生产流水线很有帮助，因为它提供了完整的 git 版本控制和多种高级流水线工具。了解有关代码仓库的更多信息。

导出时，仓库中将创建一个新分支，Code Workbook 中的代码会被转换为对应的代码仓库格式。由于这只是尽力转换，您可能需要调整这些代码，例如移动文件、更改数据集名称和路径、调整导入的包等。

导出是一次性的单向导出。后续对 Code Workbook 的编辑不会自动推送到代码仓库，且不支持将相同数据集重复导出到同一仓库。代码无法从代码仓库导回至 Code Workbook。此外，执行导出的用户必须对目标代码仓库至少具有编辑者权限。

支持的类型

目前，仅支持 SQL 节点或使用 Pandas 或 Spark dataframe 输入输出的 Python 代码节点。导出的节点集必须相互连接，且导入的数据集位于根节点。

不支持输入类型为 Python transform input 或 object 的节点。

在将代码导出到代码仓库之前，请确保代码仓库支持所需的语言。例如，如果您同时导出 SQL 和 Python 节点，可能需要在仅限 SQL 或仅限 Python 的仓库中添加一个新子项目。

如何导出

要从 Code Workbook 导出，请转到设置齿轮图标 > 导出至代码仓库助手：

点击选择仓库并选择一个代码仓库：

使用左侧面板中的复选框选择要导出的数据集。

图表显示了哪些数据集可供导出（白色），哪些不可导出（灰色）。导出操作必须是一个连通图，因此将节点添加到选择中可能会使更多节点可供导出。将鼠标悬停在已禁用的节点上会解释其无法选中的原因。

准备好导出后，点击创建拉取请求

选定的代码仓库中将创建一个新分支，其中包含从 Code Workbook 导出的代码。点击“查看拉取请求”以在代码仓库中打开该拉取请求。

您可以在此处检查导出的代码，并进行所需的任何编辑，例如数据集路径和名称。

验证 transforms-python/src/codeWorkbookExport 中导出的数据集是否已导入，并按需添加到 transforms-python/src/myproject/pipeline.py 的流水线中。

Code Workbook 中的任何自定义包版本都将添加到 transforms-python/conda_recipe/meta.yml 中。

Code Workbook 和代码仓库支持的包集合并未完全一致。虽然大多数包应该可以正常工作，但部分包可能无法通过仓库检查。

准备好后，即可创建 PR 并按照常规流程将导出的代码合并到流水线中。

写入相同的数据集

默认情况下，导出到代码仓库会写入新的数据集。如果您希望写入与 Code Workbook 相同的数据集，请按照以下步骤操作：

  1. 导航至通过导出至代码仓库操作在仓库中创建的分支。将输出数据集路径更改为所需数据集的路径。
  2. 针对每个数据集，进入数据集页面的 Details 标签页，删除该数据集的作业规格。此操作旨在让代码仓库接管该数据集的数据权属。 创建 PR，并按常规流程将代码合并至流水线。当仓库中的 CI 检查运行时，目标数据集上将生成新的作业规格。 请注意，将这些数据集转移至仓库后，您将无法继续使用原始的 Code Workbook 向这些数据集写入数据。为避免混淆，建议您删除已导出的节点。
