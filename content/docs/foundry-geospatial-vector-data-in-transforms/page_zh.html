在转换中使用向量数据 [旧版]

geospatial-tools 库不再积极开发；相反，请使用 Pipeline Builder 来加载、转换和处理地理空间数据。本页面上描述的用于操作地理空间数据类型的函数功能已被 Pipeline Builder 的地理空间功能所取代。 了解有关 Pipeline Builder 地理空间功能的更多信息。

Geospatial Tools 是一个 Python 库，用于托管在 Foundry 中对向量地理空间数据管道进行常见转换的可重用逻辑。此库提供的函数旨在用于 Python 转换，以简化这些常见操作与您管道其余部分逻辑的集成。

请注意，此功能已被 Pipeline Builder 的地理空间功能 所取代。您应使用 Pipeline Builder 而不是 geospatial-tools 库，因为 geospatial-tools 库不再维护，可能无法按预期工作。

本页面上的文档将作为参考信息保留，直到geospatial-tools库正式弃用。

该库目前包含以下辅助函数：

  • 清理地理空间数据
  • 空间函数
  • 空间连接

安装

geospatial-tools库已不再积极开发；请改用Pipeline Builder。

导入 geospatial-tools 和 Spark 配置文件 GEOSPARK 到您的 Python 仓库中。如需安装帮助，请联系您的 Palantir 代表。

切换隐藏文件和文件夹，并在 /transforms-python/build.gradle 的最底部添加以下代码块：

dependencies {
    condaJars "org.apache.sedona:sedona-spark-shaded-3.4_2.12:1.7.0"
    condaJars "org.datasyslab:geotools-wrapper:1.7.0-28.5"
}

使用方法

所有使用 geospatial-tools 库的转换都需要使用 @geospatial() 装饰器（在任何其他装饰器之前，例如 @transform_df() 和 @configure()）。这将在您的 Spark 环境中启用 Apache Sedona，并为在 PySpark DataFrame 上进行空间连接添加绑定。

Copied!
1from transforms.api import transform_df, configure, Input, Output
2from geospatial_tools import geospatial
3
4@geospatial()
5@configure(...)
6@transform_df(...)
7def compute(input_df):
8    ...

清理地理空间数据

在 Foundry 中清理地理空间数据是：

  1. 表格形式，因此数据可以用于 Spark 转换
  2. 格式为有效的 GeoJSON ↗ 或 geohash，因此数据可以用于 Foundry 本体论
  3. 使用 EPSG:4326 ↗ CRS 投影，这样空间连接的两边都使用相同的投影，Foundry 地图将正确渲染特征。

geospatial-tools 库提供了 clean_geometry() 和 lat_long_to_geometry() 来执行步骤（2）和（3）。如果您的原始数据是 GeoJSON、Shapefile、GDB、KML 或 KMZ，您可以使用 内置解析帮助函数 来执行步骤（1）。

clean_geometry()

clean_geometry(geometry, input_crs, geometry_format='geojson', lat_long=False, output_format='geojson')

通过使其有效并将其CRS标准化为EPSG:4326来清理空间几何列。

  • 参数
    • geometry (str): 输入的几何列名称。
    • input_crs (str): 输入的几何CRS。
    • geometry_format (str, 可选): 输入的几何格式。支持的格式包括 geojson, geohash (包含 latitude,longitude 的字符串), wkt, 和 wkb。默认值为 geojson。
    • lat_long (bool, 可选): 如果输入的几何坐标顺序为 latitude,longitude, 则 True。这很可能适用于 geohash 输入，但不适用于 geojson 或 wkt。默认值为 False。
    • output_format (str, 可选): 输出的几何格式（geojson 或 geohash）。默认值为 geojson。
  • 示例 
    Copied!
    1from transforms.api import transform_df, Input, Output
2from geospatial_tools import geospatial
3from geospatial_tools.functions import clean_geometry
4
5@geospatial()
6@transform_df(
7    Output('/my/output/dataset'),
8    raw_df=Input('/my/input/dataset'),
9)
10def compute(raw_df):
11    clean_df = raw_df.withColumn('geometry', clean_geometry('geometry', 'EPSG:26910'))
12    return clean_df

lat_long_to_geometry()

lat_long_to_geometry(lat, long, input_crs, output_format='geojson')

将纬度和经度列转换为清理后的几何列。

  • 参数
    • lat (str): 输入的纬度列名称。
    • long（str）：输入的经度列名称。
    • input_crs（str）：输入的几何 CRS。
    • output_format（str，可选）：输出几何格式（geojson 或 geohash）。默认为 geojson。
  • 示例 
    Copied!
    1from transforms.api import transform_df, Input, Output
2from geospatial_tools import geospatial
3from geospatial_tools.functions import lat_long_to_geometry
4
5@geospatial()
6@transform_df(
7    Output('/my/output/dataset'),
8    raw_df=Input('/my/input/dataset'),
9)
10def compute(raw_df):
11    clean_df = raw_df.withColumn('geohash', lat_long_to_geometry('latitude', 'longitude', 'EPSG:26910', output_format='geohash'))
12    return clean_df.drop('latitude', 'longitude')

解析辅助函数

geojson_to_dataframe()

geojson_to_dataframe(dataset, properties=None, glob='*.geojson', batch_size=100000)

将 GeoJSON 文件转换为数据框。使用流处理来处理非常大的文件而不会耗尽驱动内存。

  • 参数
    • dataset（transforms.api.Input）：包含 GeoJSON 文件的输入数据集。
    • properties（list[str]，可选）：要创建列的属性列表。如果没有提供，则所有属性都将包含在 JSON "properties" 列中。
    • glob（str，可选）：用于在数据集中识别 GeoJSON 文件的 glob 模式。默认为 *.geojson。
    • batch_size (int, 可选): 每批处理的记录数。如果驱动程序内存不足，应减少批次大小。默认值为100000。
  • 示例 
    Copied!
    1from transforms.api import transform, Input, Output
2from geospatial_tools import geospatial
3from geospatial_tools.parsers import geojson_to_dataframe
4
5@geospatial()
6@transform(
7    output=Output('/my/output/dataset'),
8    raw=Input('/my/input/dataset'),
9)
10def compute(raw, output):
11    return output.write_dataframe(
12        geojson_to_dataframe(raw)
13    )

buffer()

buffer(geometry, meters, metric_crs='EPSG:3395')

根据给定的距离对 GeoJSON 进行缓冲处理。

  • 参数
    • geometry (str): 地理JSON几何列的名称。
    • meters (double): 缓冲区（米）。
    • metric_crs (str, 可选): 用于转换和缓冲的度量坐标参考系统（注意：此投影用于中间步骤。输出将保持与输入相同的投影）。应靠近目标区域以最小化失真。默认为 EPSG:3395。
  • 示例 
    Copied!
    1from transforms.api import transform_df, Input, Output
2from geospatial_tools import geospatial
3from geospatial_tools.functions import buffer
4
5@geospatial()
6@transform_df(
7    Output('/my/output/dataset'),
8    df=Input('/my/input/dataset'),
9)
10def compute(df):
11    return df.withColumn('geometry', buffer('geometry', meters=10))

DataFrame.spatial_join()

DataFrame.spatial_join(right_df, on, how='inner', **kwargs)

使用空间交集连接两个数据框。

  • 参数
    • right_df (PySpark DataFrame): 连接的右侧。
    • on (tuple<str, str>): (left_geometry_column_name, right_geometry_column_name) 的元组。
    • how (str, 可选): 连接类型（支持与常规 PySpark 连接相同的连接类型）。默认为 inner。

geopandas_spatial_join()

geopandas_spatial_join(df_left, df_right, geometry_left, geometry_right, how='inner', op='intersects')

计算两个Geopandas数据框的空间连接。实现Geopandas的'sjoin'方法。期望两个数据框都包含一个GeoJSON几何列，其名称作为geometry_left和geometry_right参数传递。

  • 对于大规模数据，使用上面文档中的DataFrame.spatial_join函数。
  • 参数
    • df_left (pandas.DataFrame ↗): 左边输入的数据框。
    • df_right (pandas.DataFrame ↗): 右边输入的数据框。
    • geometry_left (str): 左边数据框的几何列的名称。
    • geometry_right (str): 右边数据框的几何列的名称。
    • how (str, 可选): 连接类型，可以是 {left, right, inner} 中的一个。默认为 inner。
    • op (str, 可选): 连接条件，可以是 {intersects, contains, within} 中的一个。默认为 intersects。
  • 示例 
    Copied!
    1from transforms.api import transform, Input, Output
2from geospatial_tools.spatial_joins import geopandas_spatial_join
3
4@transform(
5    output=Output('/my/output/dataset'),
6    input_df_left=Input('/my/input/dataset/left'),
7    input_df_right=Input('/my/input/dataset/right'),
8)
9def join(ctx, output, input_df_left, input_df_right):
10  df_left = input_df_left.dataframe().toPandas()
11  df_right = input_df_right.dataframe().toPandas()
12  geometry_left = 'geometry'
13  geometry_right = 'geometry'
14
15  joined = geopandas_spatial_join(
16        df_left,
17        df_right,
18        geometry_left,
19        geometry_right,
20        how='inner',
21        op='intersects'
22    )
23    joined = ctx.spark_session.createDataFrame(joined)
24    output.write_dataframe(joined)
