调试转换

在代码仓库中使用调试器工具，可以在数据转换运行时检查其行为。通过设置断点来暂停转换的执行，以便检查变量、查看数据框，并理解函数和库。

该调试器仅支持 Python。

设置断点

要使用调试器，需要设置断点。断点会向调试器发出信号，指示其应在何处暂停代码执行，从而允许您与变量和数据框进行交互。

点击每行代码边缘处褪色的红点即可设置断点。调试器会在标记行运行之前暂停执行。如有需要，可以跨多个文件设置多个断点。

使用内部库断点时，控制台功能可能会受限。在这种情况下，断点会显示为灰色，调试器会提示您是忽略该断点，还是使用受限的控制台功能。

运行调试器

在代码中添加断点后，点击代码编辑器操作栏中的预览并调试。调试器面板随即打开，并在遇到第一个断点时暂停。调试器的左侧栏可用于导航代码、移除断点，以及结束或停止调试会话。

您可以开启导航内部库的功能。找到内部库调试已禁用部分，然后选择启用内部库调试。

在代码中导航时，编辑器会高亮显示即将执行的代码行。使用以下按钮来推进调试器：

  1. 恢复执行：继续执行直至完成，或直至被下一个断点暂停。
  2. 单步跳过：执行该行代码，但不进入内部函数。
  3. 单步进入：如果该行代码中存在内部函数，则进入该函数。
  4. 单步跳出：跳出当前内部函数并推进调试器。
  5. 停止执行：完全停止调试器。
  6. 移除断点：从仓库中移除所有断点，并在运行预览时不暂停执行。
  7. 设置：开启或关闭调试器（不清除断点）。
  8. 文档：打开文档以获取更多详细信息。

检查变量

在调试器运行期间，您可以在代码执行的确切位置检查变量和数据。

帧

帧表示调试器处于活动状态或存在断点的函数。每个帧都指示了函数名，以及该函数所在的文件名和行号。

选择一个帧以检查其中的变量，并对其运行控制台命令。

变量

变量部分显示了转换执行期间存储在局部变量和全局变量中的值。

Dataframe 的值基于预览样本，可能无法代表完整数据集。这些值可用于理解和调试代码，但不能作为转换输出的依据。

控制台

运行调试器时，可通过控制台使用 PySpark 命令与数据进行交互。控制台常用操作模式有两种：

  • 在控制台选项卡底部的命令行中直接对 dataframe 和变量执行命令，按 Enter 或 Return 键发起命令。
  • 在转换代码中调用 print 函数，向控制台发送提示信息。

注意，控制台基于选定的帧运行。尝试对属于不同帧的局部变量执行命令将导致 NameError。

配置调试器

切换至调试器选项卡并点击设置齿轮，即可开启或关闭调试器功能。若要在运行预览时不于断点处暂停，可关闭调试器。

调试器配置会应用于整个代码仓库，但仓库内可能存在不受调试器支持的语言。若某语言不受调试器支持，无论调试器如何设置，预览功能均会正常运行。

也可在 设置 选项卡下的 首选项 > 调试器 中配置调试器。
