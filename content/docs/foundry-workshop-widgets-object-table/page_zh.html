对象表

对象表小部件用于以表格格式显示对象数据。配置对象表小部件的模块构建者可以使用包括以下功能：

  • 在一个或多个对象类型上显示数据。
  • 选择要显示的列，包括使用时间序列属性的时间序列列，以及通过函数生成的派生列。
  • 通过一个或多个列进行排序。
  • 设置列大小和行高。
  • 显示在本体论管理器中配置的条件格式和数字格式选项。
  • 允许在表格中进行单选或多选。
  • 内联编辑，以在表格中实现单元格级别的写回。
  • 在表格中选择行时触发Workshop事件。
  • 在右键菜单中添加自定义行操作。

配置选项

如果你的对象有一个属性存储了图片的URL，你可以添加类型类 hubble:icon 来显示图片而不是在设置对象类型时选择的图标。此功能允许你展示图示或图片以与你的数据一起显示。

对于对象表小部件，核心配置选项如下：

  • 输入数据

    • 对象集：这是对象表小部件的输入变量，并决定了将在对象表中显示的数据。这允许模块构建者定义一个新的对象集变量或重用在此工作区模块其他地方创建的现有对象集变量。

  • 列配置

    • 列： 本节确定将在对象表中显示的列。此配置选项会更详细地显示初始对象集中的属性类型，一旦该对象集被填充后即可看到。此外，特殊列类型允许模块构建者显示关联的对象、带参数的URL链接、时间序列数据以及通过函数实时生成的派生列。处于查看模式的用户也可以选择配置他们看到的列。通过选择列标题旁边的箭头中的配置列，查看者可以选择要在对象表中显示的列及其显示顺序。有关时间序列数据的更多信息，请参阅下面的时间序列属性部分，有关如何配置派生列的更多信息，请参阅函数支撑的列部分。

      显示的列名是一个文本字段，当添加时默认使用对象类型的属性名称。 如果在本体论中之后更新了对象类型的属性名称，则对象表中的显示列名不会自动更改。

    • 不支持的属性类型：某些大型属性（例如，Geoshape 和 Vector）默认不会加载以保持性能。在查看模式下，用户可以点击单元格中的“...”溢出选项，按需加载不支持的属性值。在小部件编辑器中，不支持的属性会用警告图标和提示工具栏标记，以指示潜在的性能影响。

    • 启用内联编辑：启用后，此切换允许在对象表中配置单元格级别的编辑。有关如何配置此高级功能的更多信息，请参阅下面的 内联编辑（单元格级回写） 部分。

    • 默认排序： 此设置允许对表格应用一个或多个默认排序。模块构建者可以在表格中显示的可见属性类型或未显示的隐藏属性类型上进行排序。如果没有应用排序，数据将不会被排序。我们建议在必要时指定排序。

  • 右键菜单

    • 启用导出为CSV: 启用后，此开关允许用户从行的右键菜单中将对象表格数据导出为CSV格式。此功能支持导出函数支撑的列和关联对象属性，并且每次可以导出最多10,000行。
    • 启用导出为Excel: 启用后，此开关允许用户从行的右键菜单中将对象表格数据导出为Excel格式。请注意，此功能每次最多支持导出200,000行。
    • 自定义右键菜单: 此选项允许模块构建者使用自定义操作配置行的右键菜单。有关如何配置此功能的更多信息，请参阅下面的自定义右键菜单部分。

  • 选择

    • 活动对象：这是对象表中的两个输出变量之一，输出当前活动/高亮对象的对象集。然后可以将此对象集用于当前模块内的下游小部件。
    • 禁用活动对象自动选择：默认情况下，表格中的第一行会在加载时自动设置为活动对象。禁用此设置将在加载时导致空的活动对象。请注意，只有在小部件可见时才会触发自动选择；如果对象表位于折叠的区域中，则在该区域展开并使小部件可见之前，不会发生自动选择。
    • 启用多选：启用此切换后，允许在表格中勾选/选择多个对象，并通过已选择对象对象集变量输出。
    • 已选择对象： 这是对象表中的第二个输出变量，输出当前选中/勾选对象的集合。此对象集合随后可在当前模块内的下游小部件中使用。注意：如果启用多选开关设置为true，此输出变量才会被使用并填充。
    • 在激活对象选择时： 此选项允许模块构建者配置Workshop事件，以便在表格中选择某一行时触发（例如，导致一个显示更详细对象视图的侧边栏出现）。

  • 显示与格式

    • 每行显示的行数： 此数字控制每行表格的高度。
    • 启用值换行： 启用后，允许文本内容在单元格内换行。此选项支持结构体属性和结构体属性数组的换行。当内容过长无法完全显示时，结构体值会显示省略号截断，并且悬停时显示完整的值。
    • 固定列数量： 这个数字决定了表格左侧固定列的数量，这些列在用户向右滚动时仍会保持可见。
    • 空状态消息：配置当支撑该控件的对象集为空时，控件显示的内容。通过默认，控件将显示一个通用的表格图标以及“未找到对象”的信息。要自定义显示的图标和信息，请选择自定义选项。
    • 自定义“无值”显示：启用后，将覆盖当单元格中没有值时在表格中显示的内容。默认情况下，将显示“无值”。
    • 合并多种对象类型：此设置仅影响显示多种对象类型的表格。当禁用时，每种对象类型将在其自己的标签页中显示。当启用时，所有对象类型将在一个表格中显示，并且在不同对象类型之间，具有相同显示名称和ID的属性类型将合并为一列。请注意，当启用内联编辑设置为true时，此选项不可用。
    • 变量支撑的列可见性： 启用后，可以通过包含可见列的API名称的字符串数组变量来控制哪些列可见。该字符串数组变量还控制列出现的顺序。如果字符串数组为空，则会显示所有配置的列。
    • 水平适应列：启用后，列将自动调整大小以填满表格的当前宽度。
    • 启用窄表头：启用后，表格表头将从50像素缩小到30像素。
    • 条件格式化整个单元格：启用后，条件格式化将对整个单元格进行着色。注意：条件格式化在Ontology管理器中为常规属性类型进行配置，并在函数支撑属性的Object Table小部件配置面板中进行配置。
    • 隐藏列配置：启用后，将隐藏表格标题菜单中视图模式下的配置列选项。
    • 显示安全标记：启用后，属性安全标记将显示为一个压缩的灰色药丸，在选中时可以展开查看。

  • 场景

    • 要加载数据的场景： 选择要加载数据的场景以用于对象表。此输入还会影响对象表中显示的对象及其各自的顺序。
    • 与场景进行比较：启用此开关以选择要比较数据的场景数组变量。这将通过在列中并排显示有修改的值，将表格中的数据与数组中场景的值进行比较。
    • 有关更多信息，请参阅 场景文档。

保存用户列配置

要保存用户的列配置，请按照以下步骤使用具有状态保存功能的字符串数组变量：

  1. 确保已启用 状态保存。
  2. 使用新的静态字符串数组变量启用基于变量的列可见性（该数组变量可以为空）。这将存储哪些列是可见的以及它们的顺序。
  3. 在设置面板中为字符串数组变量添加一个外部ID，并确保它已启用以用于状态保存。

函数支撑的列

下面的教程引用了一个可能在您的Foundry环境中不可用的Flight Alert对象类型。请将以下内容作为指南，然后使用您Foundry实例中的相应对象类型进行操作。有关Functions的更多信息，请参阅专用的 Functions文档。

总结

要创建一个返回一个或多个函数支撑列的函数，您需要满足以下规范：

  1. 函数的输入参数必须包含一个 ObjectSet<ObjectType> 参数（也可以选择性地包含其他输入参数）。这个 ObjectSet<ObjectType> 参数将使表格中显示的对象传递到函数中，以生成所需的派生列。请注意，ObjectType[] 参数在这里也可以使用，但这种性能较低的选项不建议使用。

  2. 函数的返回类型必须是从对象类型到值或自定义类型的映射，使得函数能够返回对象到值或自定义类型的映射，该映射可能包含一个或多个字段：

    • 在 TypeScript v1 中，使用 FunctionsMap<ObjectType, CustomType>。
    • 在 TypeScript v2 中，使用 Record<ObjectSpecifier<ObjectType>, CustomType>。
    • 在 Python 中，使用 dict[ObjectType, CustomType]。

    了解有关自定义类型的更多信息。

一旦配置并发布了符合上述条件的函数，您就可以成功在对象表配置中配置一个基于函数的属性列。有关更详细的教程，请参见 下方。

函数支撑属性的功能

对象表小部件支持显示通过函数实时计算的函数支撑属性。这为模块构建者提供了极大的灵活性，以显示例如以下内容的列：

  • 计算两个或多个其他属性之间的总和、差值或其他数学运算。
  • 展示一个统计模型的输出，该模型根据工作区模块中其他地方的数据输入进行更新。
  • 显示来自关联对象的属性
  • 根据工作区模块中其他地方的用户输入更改显示内容。

配置单个函数支撑属性

此示例使用一个函数支持的属性，以帮助用户查看和处理Flight Alert票证的优先级。在此处，目标是检查一个名为Time of Delay（以小时为单位）的现有属性，并从该属性生成一个新的、由函数支持的列，称为Urgency。派生的Urgency列将超过4小时的延迟标记为“高”紧急程度，2到4小时之间的延迟标记为“中”紧急程度，少于2小时的延迟标记为“低”紧急程度。这个Urgency派生属性应该使用户更容易快速浏览对象表并确定首先采取行动的警报。

要配置此函数支持的属性，第一步是配置一个函数，该函数接收预期的输入（例如，Flight Alert对象的对象集），并返回对象表用于函数支持属性所需的预期映射输出（例如，Flight Alert对象到紧急程度字符串的映射）。在本示例中，计算紧急程度的逻辑直接硬编码在函数本身中，如下所示：

Copied!
1import { Function, FunctionsMap } from "@foundry/functions-api";
2import { FlightAlertWorkshopTutorial, ObjectSet } from "@foundry/ontology-api";
3
4export class MyFunctions {
5    @Function()
6    public flightAlertCalculateUrgency(flightAlerts: ObjectSet<FlightAlertWorkshopTutorial>):
7    FunctionsMap<FlightAlertWorkshopTutorial, string>{
8        const map = new FunctionsMap<FlightAlertWorkshopTutorial, string>();
9        flightAlerts.all().forEach(flightAlert => {
10            var hoursDelayed = flightAlert.timeOfDelayHours
11            if (hoursDelayed! > 4) {
12                map.set(flightAlert, "High")
13            }
14            else if (hoursDelayed! > 2) {
15                map.set(flightAlert, "Medium")
16            }
17            else {
18                map.set(flightAlert, "Low")
19            }
20        });
21        return map;
22    }
23}

接下来，确认函数版本，然后配置该函数的必要输入。让我们选择使用运行时输入，仅传递当前在对象表中显示的对象，从而优化我们的函数性能。或者，您还可以选择传递一个对象集变量（即支持对象表的相同对象集）。

使用运行时输入选项将动态地仅将对象表中当前显示的对象（而不是整个对象集）传递到派生列函数中，并提供更快的性能。附加的使用变量输入选项允许传递整个对象集变量（例如，支持对象表的输入对象集变量），但可能会导致性能下降。

最后，选择函数支撑属性列单元格，将此列重命名为更具描述性的名称，如“紧急程度”。

最终结果将是以下对象表，它能够实时计算新的派生“紧急程度”列，并向用户显示这一有价值的附加信息。

配置多个函数支撑属性

3
4export class MyFunctions {
5    @Function()
6    public getColumnsFromLinkedDepartureAirport(flightAlerts: ObjectSet<FlightAlertWorkshopTutorial>):
7    FunctionsMap<FlightAlertWorkshopTutorial, LinkedDepartureAirportProperties> {
8        // Fetch all linked airports and store in dictionary, keyed by iata
9        const iataToAirport: { [iata: string]: AirportObject } = {};
10        flightAlerts.searchAroundDepartureAirport().all().forEach(airport => {
11            iataToAirport[airport.iata!] = airport;
12        });
13
14        const map = new FunctionsMap<FlightAlertWorkshopTutorial, LinkedDepartureAirportProperties>();
15        flightAlerts.all().forEach(alert => {
16            // Get airport for this alert
17            const airport = iataToAirport[alert.departureAirportCode!];
18            // Skip if airport not found
19            if (!airport) {
20                return;
21            }
22
23            map.set(alert, {
24                airport: airport.airport!,
25                city: airport.city!,
26                country: airport.country!,
27            });
28        });
29

31    }

内联编辑（单元格级写回）

内联编辑的编辑选项是通过在本体论中配置的操作定义的。有关操作的更多信息，请参阅 操作教程。有关进一步的内联编辑注意事项，请参阅 操作内联编辑文档。

摘要

启用内联编辑允许模块用户修改对象表中显示的单元格级数据，然后将这些编辑保存到对象数据。编辑选项通过在本体论（Ontology）中配置的操作定义，该操作必须满足以下条件才能与内联编辑兼容：

  • 该操作应使用单一的“修改对象”规则或由函数支持。
  • 操作中所有修改的属性必须定义为“来自参数”（而不是“作为静态值”或“来自对象参数的属性”）。
  • 属性参数必须是单个原始类型（例如，布尔值、整数或字符串，而不是对象引用或数组）。
  • 参数的可见性选项不应设置为“隐藏”（因为每个参数都将与表格中的可见列相关联）。

附加说明：

  • 如果一个参数有从用户可选择的枚举值中定义的值，这些选项将被尊重并在用户修改该参数时在单元格下拉菜单中显示。
  • 如果一个参数或操作有验证条件（例如，给定的数字参数必须在1到10之间），这些验证条件将被强制执行。
  • 在内联编辑期间，参数的显示选项将不会被尊重（例如，文本输入与文本区域）。
  • 为了更轻松的配置体验，操作参数ID应与表格中显示的属性ID匹配。这将允许操作参数到表格列的自动映射。
  • 为了保持一致的显示效果，Multipass用户参数应使用本体论中的Multipass值格式映射到列/属性类型。有关值格式的更多信息，请参阅 本体论文档。

自定义右键菜单

摘要

将自定义行操作添加到右键菜单中，允许用户对从对象表中右键点击的对象执行操作或事件。

配置和使用内联编辑

要将自定义行操作添加到右键菜单中，请在右键菜单部分中启用自定义右键菜单切换。将此切换设置为true将提示您创建一个右键单击对象，该对象输出表格中当前右键单击的对象。然后您可以选择添加项来向菜单中添加自定义项。

当您选择添加项时 ，将出现一个新菜单（如下面的截图所示），您可以在其中自定义菜单项在右键菜单中的显示方式。您还可以通过从点击时下拉菜单中选择一个选项，为您的菜单项分配一个操作。这允许您选择您的菜单项是触发一个操作还是事件。此外，您还可以在操作或事件中使用之前设置的右键单击的对象。

配置完成后，当您右键单击对象表中的某一行时，将看到您的更改。
