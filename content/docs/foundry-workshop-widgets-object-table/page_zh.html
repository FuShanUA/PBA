对象表格

对象表格微件用于以表格形式展示对象数据。配置对象表格微件的模块构建者可以使用以下功能：

  • 展示一种或多种对象类型的数据。
  • 选择要显示的列，包括使用时间序列属性的时间序列列，以及通过函数即时生成的派生列。
  • 按一列或多列进行排序。
  • 设置列宽和行高。
  • 展示在 Ontology Manager 中配置的条件格式和数字格式选项。
  • 允许在表格内进行单选或多选。
  • 行内编辑，支持在表格内进行单元格级别的回写。
  • 在表格内选择行时触发 Workshop 事件。
  • 在右键菜单中添加自定义行操作。

配置选项

如果对象具有存储图像 URL 的属性，可以添加类型类 hubble:icon 来显示该图像，而不是显示设置对象类型时所选择的图标。此功能允许展示图标或图像以随数据一同显示。

对于对象表格微件，核心配置选项如下：

  • 输入数据

    • 对象集：这是对象表格微件的输入变量，决定了对象表格中将显示的数据。模块构建者可以通过此选项定义新的对象集变量，或复用在此 Workshop 模块其他位置创建的现有对象集变量。

  • 列配置

    • 列： 此部分用于决定对象表格中将要显示的列。一旦初始对象集完成数据填充，此配置项会进一步展开，展示该初始对象集内包含的属性类型。此外，借助特殊的列类型，模块构建者可以展示关联对象、带参数的 URL 链接、时间序列数据，以及通过函数即时生成的派生列。处于查看模式的用户同样可以自行配置要显示的列。点击列标题旁边的箭头并选择配置列，查看者即可挑选要在对象表格中显示的列及其排列顺序。有关时间序列数据的更多信息，请查阅下文的时间序列属性部分；有关如何配置派生列的更多详情，请参阅函数支持列部分。

      显示的列名是一个文本字段，在添加时默认使用对象类型的属性名称。如果随后在 Ontology 中更新了该对象类型的属性名称，对象表格中显示的列名不会自动同步更改。

    • 不支持的属性类型： 为保障性能，某些大型属性（例如 Geoshape 和 Vector）默认不予加载。在查看模式下，用户可以点击单元格上的“...”溢出菜单，按需加载这些不支持的属性值。在微件编辑器中，不支持的属性会带有警告图标和提示信息，以标明可能存在的性能影响。

    • 启用行内编辑： 开启此开关后，允许在对象表格中配置单元格级别的编辑操作。有关如何配置此项高级功能，请参阅下文的行内编辑（单元格级别回写）部分。

    • 默认排序： 此设置支持对表格应用一个或多个默认排序规则。模块构建者既可针对表格中显示的可见属性类型进行排序，也可针对未显示的隐藏属性类型进行排序。若未应用排序，数据将保持无序状态。建议在必要时明确指定排序规则。

  • 右键菜单

    • 启用导出为 CSV： 开启此开关后，用户可以通过行右键菜单将对象表格数据导出为 CSV 格式。此功能支持导出函数支持列和关联对象属性，且单次最多可导出 10,000 行数据。
    • 启用导出至 Excel：开启此开关后，用户可通过行右键菜单将对象表格数据导出为 Excel 格式。请注意，此功能单次最多支持导出 200,000 行数据。
    • 自定义右键菜单：此选项允许模块构建者配置带有自定义操作的行右键菜单。有关如何配置此功能的更多信息，请参阅下方的自定义右键菜单章节。

  • 选择

    • 活动对象：这是对象表格的两个输出变量中的第一个，用于输出当前活动或高亮显示对象的对象集合。随后，可在当前模块的下游微件中使用该对象集合。
    • 禁用活动对象自动选择：默认情况下，表格的首行会在加载时自动设为活动对象。禁用此设置可阻止该行为，使得加载时活动对象为空。请注意，自动选择仅在微件可见时触发；如果对象表格处于折叠的区块内，则在该区块展开且微件可见之前，不会触发自动选择。
    • 启用多选：开启此开关后，允许在表格中勾选或选择多个对象，并通过 Selected objects 对象集合变量进行输出。
    • Selected objects：这是对象表格的两个输出变量中的第二个，用于输出当前已勾选或已选对象的对象集合。随后，可在当前模块的下游微件中使用该对象集合。注意：仅当 启用多选 开关设为 true 时，此输出变量才会生效并赋值。
    • 活动对象选择时：此选项允许模块构建者配置 Workshop 事件，使其在表格中选中某一行时触发（例如，触发显示包含更详细对象视图的抽屉）。

  • 显示与格式化

    • 每行显示的行数：此数值用于控制表格各行的高度。
    • 启用值换行：开启后，允许文本内容在单元格内换行。此选项支持对结构体属性及结构体属性数组进行换行处理。当内容过长无法完全显示时，结构体值将以省略号截断，并在悬停时通过提示框显示完整内容。
    • 冻结列数： 此数字决定锚定在表格左侧的冻结列数，用户向右滚动时这些列将保持可见。
    • 空状态提示信息： 配置组件支持的对象集为空时所显示的内容。默认情况下，组件将显示一个通用的表格图标以及“未找到对象”提示信息。如需自定义显示图标和提示信息，请选择自定义选项。
    • 自定义“无值”显示内容： 启用后，当单元格没有值时，将覆盖表格中的默认显示内容。默认显示“无值”。
    • 合并多种对象类型： 此设置仅影响显示多种对象类型的表格。禁用时，每种对象类型将显示在各自的选项卡中。启用时，所有对象类型将显示在单个表格中；并且对于不同对象类型，如果属性类型的显示名称和 ID 均相同，将合并为同一列。请注意，当 启用行内编辑 设为 true 时，此选项不可用。
    • 基于变量的列可见性： 启用后，允许使用包含可见列 API 名称的字符串数组变量来控制哪些列可见。此数组变量同时控制列的显示顺序。如果该字符串数组为空，表格将显示所有已配置的列。
    • 水平调整列宽： 启用后，列将自动调整大小以填满表格当前的宽度。
    • 启用窄表头： 启用后，表格表头宽度将从 50 像素缩小至 30 像素。
    • 条件格式着色整个单元格： 启用后，条件格式将对整个单元格进行着色。注意：对于普通属性类型，条件格式在 Ontology Manager 中配置；对于函数支持的属性类型，条件格式在对象表格组件配置面板中配置。
    • 隐藏列配置： 启用后，将从表格表头菜单中隐藏查看模式下的 配置列 选项。
    • 显示安全标记： 启用后，属性安全标记 将渲染为紧凑的灰色胶囊状图标，选中时会展开窗口视图。

  • 情景

    • 加载数据的情景： 选择为对象表格加载数据的情景。此输入项还会影响对象表格中显示的对象及其各自排列的顺序。
    • 与场景对比：启用此开关可选择要对比数据的场景数组变量。此功能会将表中的数据与数组中各场景的值进行对比，并在发生修改的列中将这些值并排显示。
    • 有关更多信息，请参阅《场景文档》。

保存用户列配置

要保存用户的列配置，请按照以下步骤使用带有状态保存功能的字符串数组变量：

  1. 确保已启用状态保存。
  2. 启用变量支持列可见性，并配置一个新的静态字符串数组变量（该数组变量可以为空）。此变量将记录哪些列可见及其排列顺序。
  3. 在设置面板中为该字符串数组变量添加外部 ID，并确保已启用其状态保存功能。

函数支持列

下面的教程引用了“航班警报”对象类型，该类型在您的 Foundry 环境中可能不可用。请将以下内容作为参考，并使用您 Foundry 实例中可比的对象类型进行实际操作。要了解有关函数的更多信息，请参阅专门的《函数文档》。

摘要

要创建返回一个或多个函数支持列的函数，您需要满足以下规范：

  1. 该函数的输入参数必须包含一个 ObjectSet<ObjectType> 参数（也可选择包含其他输入参数）。此 ObjectSet<ObjectType> 参数允许将表中显示的对象传递给函数，进而生成所需的派生列。请注意，此处使用 ObjectType[] 参数同样可行，但不推荐这种性能较差的方式。

  2. 该函数的返回类型必须是从对象类型到值或自定义类型的映射，从而允许函数返回包含一个或多个字段的“对象到值或自定义类型”的映射关系：

    • 在 《TypeScript v1 入门》 中，使用 FunctionsMap<ObjectType, CustomType>。
    • 在 《TypeScript v2 入门》 中，使用 Record<ObjectSpecifier<ObjectType>, CustomType>。
    • 在 《Python 入门》 中，使用 dict[ObjectType, CustomType]。

    了解有关自定义类型的更多信息。

配置并发布满足上述条件的函数后，即可在对象表配置中成功配置函数支持的属性列。更详细的教程请参阅下文。

函数支持的属性特性

对象表组件支持显示由函数即时计算得出的函数支持的属性。这为模块构建者提供了极大的灵活性，可用于显示以下列，例如：

  • 计算两个或多个其他属性之间的和、差或其他数学运算结果。
  • 展示根据 Workshop 模块其他位置的数据输入进行更新的统计模型输出。
  • 显示来自关联对象的属性
  • 根据 Workshop 模块其他位置的用户输入更改显示内容。

配置单个函数支持的属性

本示例使用函数支持的属性来帮助用户查看和处理 Flight Alert 工单的优先级。此处目标是检查名为 Time of Delay（以小时为单位）的现有属性，并基于该属性生成名为 Urgency 的新的函数支持列。派生的 Urgency 列将长时间延误（超过4小时）标记为“高”紧急程度，中度延误（2到4小时之间）标记为“中”紧急程度，较短时间延误（少于2小时）标记为“低”紧急程度。此 Urgency 派生属性应能让用户更轻松地快速浏览对象表，并确定优先处理哪些警报。

要配置此函数支持的属性，第一步是配置一个函数，该函数接收预期输入（例如，Flight Alert 对象的对象集），并返回对象表所需的预期映射输出，以支持函数属性（例如，Flight Alert 对象到紧急程度字符串的映射）。在本示例中，计算紧急程度的逻辑硬编码在函数本身中，如下所示：

Copied!
1import { Function, FunctionsMap } from "@foundry/functions-api";
2import { FlightAlertWorkshopTutorial, ObjectSet } from "@foundry/ontology-api";
3
4export class MyFunctions {
5    @Function()
6    public flightAlertCalculateUrgency(flightAlerts: ObjectSet<FlightAlertWorkshopTutorial>):
7    FunctionsMap<FlightAlertWorkshopTutorial, string>{
8        const map = new FunctionsMap<FlightAlertWorkshopTutorial, string>();
9        flightAlerts.all().forEach(flightAlert => {
10            var hoursDelayed = flightAlert.timeOfDelayHours
11            if (hoursDelayed! > 4) {
12                map.set(flightAlert, "High")
13            }
14            else if (hoursDelayed! > 2) {
15                map.set(flightAlert, "Medium")
16            }
17            else {
18                map.set(flightAlert, "Low")
19            }
20        });
21        return map;
22    }
23}

在列列表中显示的 函数支撑属性 选项内，选择“fx”函数图标，然后选择所需的函数。在此示例中，我们选择 flightAlertCalculateUrgency 函数。

接下来，确认函数版本，然后配置函数所需的输入参数。我们选择 使用运行时输入，以便仅传入对象表中当前显示的对象，从而优化函数性能。或者，您也可以选择传入对象集变量（即支撑对象表的同一对象集）。

使用运行时输入 选项会仅将对象表中当前显示的对象（而非整个对象集）动态传递给派生列函数，从而提供更快的性能。此外，使用变量输入 选项允许传入整个对象集变量（例如支撑对象表的输入对象集变量），但可能会导致性能下降。

最后，选择 函数支撑属性 列的单元格，将此列重命名为更具描述性的名称，例如“Urgency”。

配置多个函数支撑属性

接下来，我们来看一个进阶示例：通过使用自定义返回类型，创建一个生成多个函数支撑属性的单一函数。了解有关自定义类型的更多信息。 使用单一函数返回多个函数支撑属性具有多项优势，包括提升性能以及更清晰地组织相关派生属性的代码。

在此示例中，使用一个接收 ObjectSet<FlightAlerts> 的函数，遍历 出发机场 链接以获取关联的 Airport 对象，随后从该关联的 Airport 对象中返回三个属性。这将丰富对象表格中显示的数据，使其包含来自关联对象类型的相关信息。

1import { Function, FunctionsMap } from "@foundry/functions-api";
2import { AirportObject, FlightAlertWorkshopTutorial, ObjectSet } from "@foundry/ontology-api";
3
4export class MyFunctions {
5    @Function()
6    public getColumnsFromLinkedDepartureAirport(flightAlerts: ObjectSet<FlightAlertWorkshopTutorial>):
7    FunctionsMap<FlightAlertWorkshopTutorial, LinkedDepartureAirportProperties> {
8        // Fetch all linked airports and store in dictionary, keyed by iata
9        const iataToAirport: { [iata: string]: AirportObject } = {};
10        flightAlerts.searchAroundDepartureAirport().all().forEach(airport => {
11            iataToAirport[airport.iata!] = airport;
12        });
13
14        const map = new FunctionsMap<FlightAlertWorkshopTutorial, LinkedDepartureAirportProperties>();
15        flightAlerts.all().forEach(alert => {
16            // Get airport for this alert
17            const airport = iataToAirport[alert.departureAirportCode!];
18            // Skip if airport not found
19            if (!airport) {
20                return;
21            }
22
23            map.set(alert, {
24                airport: airport.airport!,
25                city: airport.city!,
26                country: airport.country!,
27            });
28        });
29
30        return map;
31    }
32}

上述函数发布后，即可在 Workshop 中用于添加派生列。按照上一节的操作方式，向包含 Flight Alert 对象的对象表格（Object Table）组件中添加一个新的 Function-backed property 列。

在列列表中出现的新列中，选择 fx 图标，挑选用于支持派生列的函数。本示例中，请选择本教程前文定义的 getColumnsFromLinkedDepartureAirport 函数。

接下来，为所选函数配置输入参数。本示例中，该函数接收一个单一输入参数，即名为 flightAlerts 的 Flight Alert 对象集合。我们可以选择 Use a runtime input 选项，将对象表格中当前显示的对象传递给该函数。

Use a runtime input 选项会将对象表格中当前显示的对象（而非整个对象集合）动态传递给派生列函数，从而提供更快的性能。另一个 Use a variable input 选项则允许传入完整的对象集合变量（例如支持对象表格的输入对象集合变量），但这可能会导致性能下降。

完成上述步骤后，函数应能成功运行，并在 Object Table: 中显示预期的三个列：Departure Airport、Departure City 和 Departure Country。我们可以就此止步，或者编辑列的显示名称与排列顺序，以优化列在应用中的呈现效果。

时间序列属性

时间序列属性（包括由时间序列转换生成的数据）可以在对象表格中与常规属性一起查看。时间序列属性是一种对象属性，用于存储带有时间戳的值的历史记录。更多信息请参阅《Workshop 中的时间序列属性》。

在下方的示例中，Country 对象包含一个常规的字符串属性 Name，用于存储国家名称；还包含一个时间序列属性 New Cases，用于记录该国每日新增的 COVID-19 确诊病例历史数据。对象表格在左侧前两列中展示了这两个属性，同时通过时间序列变换，基于 New Cases 属性派生出了三个时间序列：Case Acceleration、Weekly Cases 和 Total Cases。每个时间序列列的左侧显示该序列的最新观测值，右侧则用迷你图直观呈现该时间序列的历史趋势。

为了演示如何在对象表格中配置时间序列属性列，我们将详细介绍 Weekly Cases 列的设置过程。屏幕右侧的面板分为上下两部分，展示了 Weekly Cases 列的配置内容。这些配置面板与列本身展示在同一界面上，方便您将配置项与最终结果进行全局对照。

时间序列变换

首先，我们需要使用时间序列变换来生成每周 COVID-19 病例数的时间序列。时间序列变换会对输入的时间序列数据执行数学运算，从而生成一条新的输出时间序列。有关时间序列变换的更多详细信息，请参阅《Workshop 中的时间序列属性》。

在此，我们可以使用聚合变换来创建所需的时间序列。点击 Add transform 按钮会打开变换配置界面，我们可以在其中进行设置，以生成所需的时间序列。

时间序列汇总器

接下来，我们可以使用时间序列汇总器（红框）来生成列中显示的数值。时间序列汇总器用于为时间序列数据配置摘要统计量，即一个反映时间序列状态的值。此处，我们使用单值汇总器生成所需显示的值，该值将为最近一周的平均病例数。有关时间序列汇总器的更多信息，请参阅《Workshop 中的时间序列属性》。

随后，我们可以在“数字格式”标题下配置数值格式（蓝框），并在“条件格式”标题下配置条件格式（绿框），以此对该值的显示样式进行设置。数值格式控制数值中数字的显示方式，而条件格式则基于值的规则系统改变显示颜色。本例中，我们希望使用紧凑记数法，并且如果每周平均 COVID-19 病例数超过 100，则将该值显示为红色。有关数值格式和条件格式的更多信息，请参阅《Workshop 中的格式设置》。

迷你图

最后，我们可以配置迷你图可视化，以显示每周平均病例时间序列的历史数据。

时间范围选择器（红框）允许我们指定可视化时间序列的时间窗口。有关时间范围的更多信息，请参阅《Workshop 中的时间序列属性》。

我们还可以在“条件格式”标题下添加条件格式（蓝框），利用基于值的规则系统改变迷你图的颜色。有关条件格式的更多信息，请参阅《Workshop 中的格式设置》。

最后，我们可以配置基准线（绿框）来辅助解读迷你图。有关配置基准线的更多信息，请参阅《Workshop 中的时间序列属性》。

行内编辑（单元格级回写）

行内编辑的编辑选项通过本体中配置的操作来定义。有关操作的更多信息，请参阅操作教程。有关行内编辑的更多注意事项，请参阅操作行内编辑文档。

总结

启用行内编辑允许模块用户修改对象表格中显示的单元格级别数据，随后将这些编辑保存到对象数据中。编辑选项通过在本体中配置的操作来定义，该操作必须满足以下条件才能与行内编辑兼容：

  • 该操作应使用单一的“修改对象”规则，或由函数提供支持。
  • 操作内所有修改的属性必须定义为“来自参数”（而非“作为静态值”或“来自对象参数的属性”）。
  • 属性参数必须为单一的基本类型（例如布尔值、整数或字符串，不能是对象引用或数组）。
  • 参数的可见性选项不应设置为“隐藏”（因为每个参数将与表格中的可见列绑定）。

补充说明：

  • 如果某个参数已定义供用户选择的枚举值，那么当用户修改该参数时，将遵循这些选项并在单元格内的下拉菜单中显示。
  • 如果参数或操作带有校验条件（例如某个数字参数必须介于 1 和 10 之间），将强制执行这些校验条件。
  • 行内编辑时不会遵循参数的显示选项（例如文本输入框与文本区域的区别）。
  • 为获得更简便的配置体验，操作参数 ID 应与表格中显示的属性 ID 相匹配。这样可以实现操作参数到表格列的自动映射。
  • 为了显示一致，Multipass 用户参数应在本体内使用 Multipass 值格式化映射到列/属性类型。有关值格式化的更多信息，请参阅本体文档。

配置并使用行内编辑

配置对象表格（Object Table）小部件时，列列表下方的列配置区域会显示启用行内编辑开关。开启该开关即可启用行内编辑，系统会提示你选择先前在 Ontology 中配置好的操作，用于修改当前显示的对象类型。在下方的示例中，通过航班预警：行内编辑操作启用了行内编辑功能。在操作选择器下方，可通过下拉菜单将操作参数映射至对应的表格列。你也可以将变量作为操作参数传入，系统会自动将其带入操作中，无需用户在表格中手动填写字段。

完成上述配置后，用户点击表格底部的编辑表格按钮即可进入编辑模式。你可以在自定义按钮文字输入框中修改该按钮的显示文本。开启默认启用编辑模式选项，可让表格始终保持在行内编辑状态。进入编辑模式后，用户即可对映射到操作参数的任一可修改列进行编辑，效果如下图所示。对于基于函数的操作，用户一次最多可暂存 20 行的编辑内容；对于非基于函数的操作，一次最多可暂存 200 行。所有暂存的编辑内容均可通过 撤销按钮（见下方截图表格最左侧列）予以撤销。

完成编辑并准备好提交更改后，可以点击表格右下角的提交按钮。系统会弹出确认对话框，再次点击提交即可提交更改。若你希望使用一键提交功能并禁用该确认对话框，可开启一键提交开关。

自定义右键菜单

摘要

通过向右键菜单添加自定义行操作，用户可以直接在对象表中右键点击对象，并对其执行操作或触发事件。

配置并使用内联编辑

要向右键菜单添加自定义行操作，请在右键菜单部分开启自定义右键菜单开关。开启该开关后，系统会提示您创建一个被右键点击的对象，用于输出表中当前被右键点击的对象。随后，您可以通过选择添加项目来向菜单中添加自定义项。

选择添加项目后，将弹出一个新菜单（如下图所示），您可以在其中自定义菜单项在右键菜单中的显示方式。您还可以通过从点击时下拉菜单中选择相应选项，为菜单项分配操作。这允许您决定菜单项是触发操作还是事件。此外，您可以在操作或事件中使用先前设置的被右键点击对象。

配置完成后，在对象表中右键点击某一行，即可看到相应的变化。
