入门指南

本页内容探讨如何实施 Power BI® 连接器，以便从 Power Query 接口访问 Foundry 资源。如需了解用于数据集成的 Microsoft Power BI® XMLA 连接器相关信息，请查阅我们的《数据连接文档》。

本指南将向您展示如何通过 Power BI® 完成 Foundry 身份验证、选择数据集，并开始构建您的首个报告。

在 Power BI® 中将 Foundry 选为数据源

  • 在 Power BI® 中，点击功能区中的“获取数据”，然后选择“更多”。
  • 在数据源列表中搜索“Palantir Foundry”，或者在“在线服务”下选择它。
  • 如果在此步骤遇到错误，请确保您已完成《安装指南》。

配置连接设置

系统将提示您输入有关 Foundry 连接的一些详细信息：

  • 基础 URL：在“基础 URL”处输入您的 Foundry 连接 URL。该 URL 即为您平时访问 Foundry 所用的链接。您可以通过右键单击并复制链接地址来获取此路径：基础 URL。（请注意，如果在浏览器中加载此 URL，可能会重定向至其他页面。遇到这种情况，只需删除地址中 “.com” 之后的所有内容，即可恢复为基础 URL。）
  • （可选）数据集 RID 与分支：如果您已经知道想要访问的数据集 RID 或分支，可以在此处输入该信息。（参见《指南：在 Foundry 中识别数据集的 RID 或文件路径》。）否则，请将这些字段留空。后续步骤中将提供数据集浏览器供您选择数据。
  • 数据连接模式：选择您希望使用“导入”还是“DirectQuery”模式。

点击“确定”进入下一步。

通过 Foundry 进行身份验证

通过 Foundry 进行身份验证共有三种方式：Foundry OAuth、Foundry 令牌和 Foundry 客户端凭据。OAuth 是 Power BI® 桌面版推荐且最简单的身份验证方法。对于在 Power BI® 服务上配置身份验证的管理员，我们推荐使用客户端凭据。

您可以从 Power BI® 身份验证对话框的左侧选择要使用的方法。以下是对这些身份验证选项的使用说明。

Foundry OAuth 身份验证（推荐用于 Power BI® 桌面版）

OAuth 是授权 Power BI® 桌面版连接至 Foundry 的推荐方法。如果使用 OAuth 身份验证，请从 Power BI® OAuth 对话框中选择登录。随后将打开一个新窗口，显示您的 Foundry 登录界面。

如果您是首次将 Power BI® 与 Foundry 配合使用，系统会提示您批准 Power BI® 访问您的 Foundry 账户。点击“允许” Power BI® 的访问请求，然后照常登录 Foundry 即可。

如果通过 OAuth 登录时收到错误页面，可能是您的组织尚未为 Power BI® 启用 OAuth 登录功能。在这种情况下，请联系您的 Foundry 管理员，以便将 Power BI® 启用为第三方应用程序。在此期间，您也可以使用下文所述的基于令牌的替代身份验证选项。

Foundry 令牌身份验证

如果使用基于令牌的身份验证，请按照《生成令牌》中的说明，在 Foundry 内部生成一个私有身份验证令牌。获取令牌后，您可以将其粘贴到 Power BI® 的提示框中。

你的凭证现已保存在 Power BI® 中，只要凭证保持有效即可持续使用。在令牌过期之前，系统不会再次提示你输入令牌。届时，你可以按照上述说明重新生成新令牌。

Foundry 第三方应用程序客户端凭证（推荐 Power BI® 服务中的 Foundry 管理员使用）

对于 Foundry 管理员而言，第三方应用程序客户端凭证是授权发布到 Power BI® 服务的报告的推荐方法。此类凭证没有过期时间。

首先，你必须在 Foundry 中配置一个第三方应用程序。按照说明配置该应用程序。选择 机密客户端 选项，并确保启用 客户端凭证授权。请不要启用 Ontology SDK。

然后，向该第三方应用程序的服务用户授予适当的权限。Power BI® 中的数据访问权限将反映该服务用户的访问级别。

最后，在 Power BI® 中，选择 Foundry 客户端凭证 身份验证选项，并输入你的第三方应用程序的客户端 ID 和密钥。

连接到 Foundry 并选择你的数据集

  • 使用左侧的导航器选择报告所需的数据集。
  • 完成选择后，选择“加载”或“转换数据”，然后照常继续构建报告。
