本体论清理

本体论清理工具是一种安全删除对象类型的方式，并提供了许多好处，包括：

  • 没有多余对象类型的更易于导航的本体论
  • 由于搜索和加载是在更少的对象类型上进行，因此性能更好的本体论
  • 没有多余存储成本负担的本体论

该工具旨在帮助本体论编辑者判断删除一个对象类型的可行性，并提供了一个弃用选项，用于通知对象类型用户其未来将被删除。

访问本体论清理工具

可以从本体论清理工具的首页访问该视图。

当您选择 开始清理 时 ，根据您的本体论的大小，该工具可能需要一些时间来查找清理候选对象。

对象类型的结果列表与其他从主页访问的显示列表的页面操作方式类似。可以对特定标志或您负责的对象类型组进行过滤。您还可以自定义表格中显示的列。

默认情况下，表格按对象类型触发的最高优先级标志排序。

清理您的本体论

这是一个例子，我们已过滤到“规划”工作流，这是一个正在开发但从未发布的流程。

使用内联复选框选择三个对象类型。

有三种管理这些对象类型的方式：

  • 静音：在可配置的时间内从清理队列中隐藏对象类型。静音是一种仅影响执行该操作的用户的操作。
  • 弃用：在显示对象类型状态的每个上下文中将对象类型显示为已弃用。此选项会通知用户转移到其他对象类型，或标记该对象类型仍然有用。您可以设置一个截止日期以及弃用时间，这样用户就知道他们还有多长时间不能使用这些对象类型。
  • 删除：从本体论中删除对象类型，并从对象存储中删除相关数据。

一旦您对队列中的对象类型进行操作，它就会从队列中消失。使用表格过滤器查看所有已选择的操作。

弃用和删除的步骤与正常的本体论修改相同。在上面的例子中，“工作项”对象类型包含用户编辑的内容，因此可以被弃用，而其他两个则被删除。选择保存 在右上角启用直接将更改保存到本体论或创建提案以请求其他用户的审核。

配置本体论清理

清理页面包含一个子页面，允许您自定义使用的标志及其相应的优先级。

您可以在此页面上配置标志设置，可以选择使用默认集或自定义标志。

与从队列中静音对象类型类似，这是个人的定制，不会影响其他本体论编辑器。

当您保存更改并返回主清理选项卡时，会提示您重新计算清理队列。

请注意，如果使用自定义标志设置，将来添加的新标志在使用默认标志集时不会自动启用。

本体论清理标志

以下标志列表旨在解决常见问题，但并非详尽无遗：

  • 已过期的弃用日期： 对象类型目前具有 deprecated 状态，并且弃用日期字段已经过了。
  • 数据源被删除： 任何支持该对象类型的 datasource（无论数据集、受限视图或其他）已在 Compass 中被删除。
  • 数据源在 [x] 天内未更新： 检查 Compass 中支持数据源的最后修改时间。
  • 缺少描述： 该对象类型没有描述。不检查该对象类型所有属性的描述。
  • 显示名称正则匹配字符串： \[test|deprecated\] 的默认值将匹配在显示名称中包含 [test] 或 [deprecated] 的对象类型。例如，如果在您的组织中，通常使用前缀 UAT - 或 Testing - 来标记用户验收测试中的对象类型，那么您将使用正则表达式 UAT -|Testing - 来查找符合此模式的所有对象类型。支持 ECMA（JavaScript）正则表达式语法。
  • Phonograph deindexed： 仅对对象存储 V1 中的对象类型应用的标志。对象存储 V2 没有相应的检查。
