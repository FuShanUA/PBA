在控制面板中配置代码仓库设置

您可以在控制面板中配置组织级别的多项代码仓库设置。如需修改这些设置，您需要具备 用户体验管理员 角色权限。

可用设置

  • 结合 Palantir Visual Studio Code 插件进行本地开发： 启用该设置后，组织内的用户能够使用 Palantir VS Code 插件在本地克隆代码仓库并进行开发。此设置为默认开启状态。
  • 通过 Palantir Visual Studio Code 插件进行本地预览： 启用该设置后，组织内的用户可以在本地预览数据集。本地数据集预览会将数据集的部分内容下载并临时存储到用户的计算机上。此设置为默认关闭状态。该设置已弃用，由通过 Palantir Visual Studio Code 插件进行的安全流式预览功能所取代。
  • 通过 Palantir Visual Studio Code 插件进行安全流式预览： 启用该设置后，组织内的用户可以使用安全流式预览方案在本地预览数据集。本地数据集预览涉及将部分数据集流式传输至用户计算机的系统内存中。此设置为默认开启状态。该设置目前尚未生效，需待平台介入后方可正式启用。
  • 结合 Palantir MCP Server 进行本地开发： 在 Claude Code、GitHub Copilot 或 Cursor 等本地 IDE 中使用 Palantir MCP，外部 AI 系统即可利用已配置用户令牌的权限，通过 AI 友好型 API 端点读取 Foundry 中的数据和元数据并进行交互。一旦外部系统通过 Palantir MCP 获取数据，其使用层面的治理将由 Palantir 转移至该外部系统。数据安全取决于您与该外部系统之间的信任关系。使用 Palantir MCP 并不意味着会调用 Palantir 的 AI 模型；实际使用的 AI 模型提供商由您连接的外部系统决定（例如，GitHub Copilot 对应 Microsoft，Cursor 对应 Cursor）。

若要进行本地预览，必须通过上述本地预览设置授权用户操作，且用户需对所预览的输入数据具备 下载 权限。此类属于特权操作，允许用户将完整数据集下载至本地，授权时需谨慎行事。若无法向用户授予此权限，仍可通过 代码工作区使用 VS Code 提供的开发体验，该方式可确保更严格的数据管控。有关本地预览的更多细节，请参阅安全注意事项。

您可以通过我们的文档，进一步了解本地开发、预览和调试 Python 转换，以及如何使用 Palantir 插件等操作。
