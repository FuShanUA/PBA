在控制面板中配置代码仓库设置

您可以在 控制面板 中配置许多组织级别的代码仓库设置。要修改这些设置，您需要 User Experience Administrator 角色。

可用的设置

  • 使用 Palantir 扩展进行本地开发（Visual Studio Code）: 启用后，您组织的用户将能够通过 Palantir 扩展 for VS Code 克隆代码仓库并在本地进行工作。此设置默认已启用。
  • 通过 Palantir 扩展进行本地预览（Visual Studio Code）: 当启用此设置时，您组织的用户可以本地预览数据集。本地数据集预览涉及将数据集的部分内容下载并临时存储到用户的机器上。此设置默认是禁用的。此设置已弃用，已被通过 Visual Studio Code 的 Palantir 扩展进行的安全流式预览所取代。
  • 通过 Visual Studio Code 的 Palantir 扩展进行安全流式预览：当启用此设置时，您组织的用户可以使用 安全流式 预览实现来本地预览数据集。本地数据集预览涉及将数据集的部分内容流式传输到用户的系统内存中。此设置默认是启用的。此设置尚未生效，将在平台干预后生效。
  • 通过 Palantir MCP 服务器进行本地开发： 在本地 IDE（如 Claude Code、GitHub Copilot 或 Cursor）中使用 Palantir MCP，可使外部 AI 系统通过 AI 友好的 API 端点读取 Foundry 中的数据和元数据，并使用配置用户令牌的权限进行交互。一旦数据通过 Palantir MCP 被外部系统访问，其使用的管理权将从 Palantir 转移到外部系统。数据安全性将取决于你与该外部系统的合作关系。使用 Palantir MCP 并不意味着会使用 Palantir 的 AI 模型；AI 模型供应商由你连接的外部系统决定（例如，GitHub Copilot 对应 Microsoft，Cursor 对应 Cursor）。

要进行本地预览，用户必须通过上述本地预览设置被允许执行此操作，并且对预览的输入具有 Download 操作权限。这些是特权操作，允许用户本地下载整个数据集，应谨慎授予。如果无法授予此权限，仍然可以通过 代码工作区 访问 VS Code 开发者体验。 ，这保证了更严格的数据控制。有关详细信息，请参阅 本地预览的安全性注意事项。

您可以在我们的文档中了解更多关于本地开发、预览和调试 Python 转换，以及使用 Palantir 扩展 的信息。
