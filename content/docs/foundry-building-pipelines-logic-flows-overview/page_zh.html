Logic Flows [已停止维护]

Logic Flows 允许你在 Foundry 中自动化常见的工作流。Logic Flows 与 Foundry 的构建和调度基础设施相集成，从而将其纳入你的数据管道。

已停止维护

自 2024 年 8 月起，Logic Flows 处于已停止维护阶段，不应用于新的开发。

已停止维护

Logic Flows 处于开发的已停止维护阶段，并将在未来被废弃。目前仍提供全面支持。我们建议你将工作流迁移至其他应用和工具，以满足你的用例需求：

  • AIP Logic： 利用大语言模型的力量来自动化和支持关键任务，包括将来自非结构化输入的关键信息连接到你的本体、解决调度冲突、通过寻找最佳分配来优化资产性能、应对供应链中的中断等。
  • Actions： 构建自动化事务，基于你定义的逻辑和标准更改对象属性。

核心概念

自动化是一种使用构建与 Foundry 服务 API 交互的脚本，旨在替代平台中手动重复的操作。它的行为类似于 Foundry 中的其他作业，但不使用数据集作为输入或输出。Palantir 维护着一个经过筛选的自动化库。

自动化将资源作为参数接收，并接收一个 JSON 配置。

自动化的具体实例称为已连接流。已连接流通过项目、参数和配置创建。

已连接流：

  • 在单个项目上定义并执行
  • 根据参数和配置运行，这些参数和配置在创建已连接流时进行验证
  • 可作为调度的一部分运行
  • 可通过 Logic Flows UI、数据血缘关系或 Builds 应用手动运行
  • 不会在运行之间存储信息，因此创建新的已连接流不会影响后续运行的结果
  • 无法编辑
  • 可归档
