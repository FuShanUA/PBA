使用 Pipeline Builder 创建时间序列数据

按照本指南创建的数据管道将生成时序数据，该数据支持时序同步，以便与 Carrier、Route 和 Airport 对象类型上的时序属性相关联，从而创建新的时序对象。由于我们需要对非时序数据进行计算来生成时序数据，因此该数据管道的设置比从时序数据集到时序属性的标准映射更为复杂。请查阅《Pipeline Builder 文档》以获取有关常规数据管道指导的信息。

我们正在使用的航班数据集包含以下列，我们将用它们来创建时序属性：

  • flight_id： string | 用于标识航班及数据集中每一行的唯一字符串。
  • date： date | 航班执行的日期。
  • destination_airport_id： string | 用于标识目的地机场的字符串。
  • airline_id： string | 航空公司的 ID。
  • origin_airport_id： string | 出发地机场的 ID。
  • dep_delay： integer | 出发延误的分钟数。
  • arr_delay： integer | 到达延误的分钟数。
  • route_id： string| 用于标识路线的唯一字符串。

第一部分：生成时序数据

使用同样用于支持 Flights 对象类型的航班数据集，我们可以执行一些聚合转换，并根据航班指标生成时序数据。请注意，如果您的时序数据是通过历史数据库或边缘传感器接入 Foundry 的，则无需执行此步骤。您可以直接进入下一环节生成时序同步。

1. 对 Carrier 和 Route 对象类型应用转换

根据航班数据集，按照以下步骤应用转换。您需要对 Carrier 和 Route 对象类型分别执行此操作。

聚合数据

使用聚合转换按日期和对象 ID 进行分组（在本例中，对 Route 对象类型使用 route_id；您还需要对 Carrier 对象类型使用 airline_id 单独执行相同操作），并计算平均到达延误、平均出发延误和每日航班数量。

聚合后，数据集应预览为以下架构：

转换为新的数据类型

要将这些新数据用作时序数据，我们必须创建一个时间戳类型的列。为此，请使用类型转换将 date 列转换为时间戳类型的列。我们接下来还将应用逆透视转换，以将 daily_avg_dep_delay、daily_avg_arr_delay 和 daily_count_of_flights 的值合并到一列中。由于此函数要求所有值必须为相同的数据类型，我们还必须将每日航班数量指标转换为双精度浮点型（即与平均延误指标相同的数据类型）。

逆透视以合并时序值

由于此数据集在不同列中包含时序数据，我们必须使用逆透视转换将其合并为一个值列，以便数据能够符合时序同步所需的架构，如下所示：

  • series ID： string | 由 TSP 引用的一组时间戳和值对的序列 ID，必须与 TSP 的序列 ID 匹配。
  • timestamp： timestamp 或 long | 测量该数值的时间。
  • value: integer、float、double、string | 测量点处该量的数值。若为 string 类型，则表示分类时间序列；每个分类时间序列最多可有 10,000 种唯一变体。

下方的逆透视转换将 daily_avg_dep_delay、daily_avg_arr_delay 和 daily_count_of_flights 的值放入同一个 series_value 列中。这些原始列名将输出到新的 series_name 列中，供 series_id 使用。

Unpivot to merge series values

Since this dataset contains time series data in different columns, we must use an unpivot transform to merge it into one value column so the data can match the required schema for a time series sync, as shown below:

  • series ID: string | The series ID for the set of timestamp and value pairs referred to by a TSP, which must match the TSP's series ID.
  • timestamp: timestamp or long | The time at which the quantity is measured.
  • value: integer, float, double, string | The value of the quantity at the point that it is measured. A string type indicates a categorical time series; each categorical time series can have, at most, 10,000 unique variants.

The unpivot transform shown below places values for daily_avg_dep_delay, daily_avg_arr_delay, and daily_count_of_flights into the same series_value column. Those original column names are outputs to the new series_name column that will be used in the series ID.

Part II: Create the time series sync

1. Remove null values

Apply a filter transform on the resulting dataset to remove any null values.

2. 配置时序同步

现在，在屏幕右侧的管道输出部分选择 Add，创建一个时序同步。接着，选择 Time series sync。为新建的时序同步填写必要的数据，请注意以下几点：

  • 标题“[示例] 时序同步 | 事件”将对应于你 Palantir 文件系统文件夹中生成的资源。
  • 为 Series ID 字段选择 series_id 列。
  • 在 Time 字段中添加已创建的 date 时间戳列。
  • 将 series_value 添加到 Value 字段中。

现在，保存并构建管道。输出内容将在管道所在的同一文件夹中生成。

3. 使用时序同步向对象类型添加属性

创建带有时间序列同步的管道后，你就可以使用该同步向 Route、Carrier 和 Airport 对象类型添加时间序列属性了。如需更多指导，请参阅我们关于向对象类型添加时间序列属性的文档。
