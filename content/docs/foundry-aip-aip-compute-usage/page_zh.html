使用 AIP 进行计算

AIP 的计算使用涉及大型语言模型（LLMs）。从根本上说，LLMs 以文本作为输入，并以文本作为输出。输入和输出的文本量以 标记 来衡量。 LLM 的计算使用 以每一定数量的标记的计算秒数来衡量。不同的模型可能有不同的计算使用率，如 下文 所述。

AIP 中的标记

标记是 LLM 用来处理和理解输入的基本文本单位。一个标记可以短至一个字符，也可以长至一个完整的单词，这取决于语言和具体的模型。

重要的是，标记并不一一对应于单词。例如，常见单词可能是一个标记，但较长或不常见的单词可能被拆分为多个标记。甚至标点符号和空格也可以被视为标记。

不同模型提供商对“token”的定义各不相同；例如，OpenAI ↗ 和 Anthropic ↗。平均而言，每个 token 大约有 4 个字符长，一个字符可以是单个字母或标点符号。

在 AIP 中，当应用程序向 LLM 发送提示并接收提示时会消耗 token。这些提示和响应都包含可测量的 token 数量。这些 token 可以发送到多个 LLM 提供商；由于各供应商之间的差异，这些 token 会被转换为 计算秒数，以匹配底层模型提供商的价格。

所有提供 LLM 支持功能的应用程序在使用时都会消耗 token。请参阅以下列表，了解在您与其 LLM 支持功能进行交互时可能会使用 token 的应用集合。

  • AIP Assist
  • AIP Logic
  • AIP Error Enhancer
  • AIP Code Assist
  • AIP Analyst
  • AI FDE
  • Workshop LLM 支持工具
  • Quiver LLM 支持工具
  • Pipeline Builder LLM 支持工具
  • 直接调用语言模型服务（包括 Python 和 TypeScript 库）

AIP 直接将文本路由到支撑的 LLM，这些 LLM 会自行进行分词。文本的大小将决定支撑模型在提供响应时使用的计算量。

以下是一个发送到 GPT-4o 模型的示例句子。

AIP 结合了 Palantir 所有先进的安全措施，以确保敏感数据在符合行业法规的前提下得到保护。

这个句子包含 140 个字符，其分词方式如下，用 | 字符分隔每个标记。请注意，一个标记并不总是等同于一个单词；有些单词会被拆分成多个标记，例如下面例子中的 AIP 和 Palantir。

A|IP| incorporates| all| of| Pal|ant|ir|'s| advanced| security| measures| for| the| protection| of| sensitive| data| in| compliance| with| industry| regulations|.

这个句子包含 24 个标记，并将使用以下计算秒数：

compute-seconds = 24 tokens * 43 compute-seconds / 10,000 tokens
compute-seconds = 24 * 43 / 10,000
compute-seconds = 0.1032

上述句子中的标记和字符数量已通过 OpenAI 的分词功能 ↗ 进行验证。

使用 AIP 理解计算使用情况

由于 LLM 标记产生的计算秒数直接与请求该使用的单个应用程序资源相关联。例如，如果您使用 AIP 在 Pipeline Builder 中自动解释一个管道，则 LLM 生成该解释所使用的计算秒数将归因于该特定的管道。这种归因关系在平台上是通用的；记住这一点将有助于您追踪令牌的使用位置。

在某些情况下，计算使用情况无法归因于平台上的单一资源；例如包括 AIP 协助和错误解释器等。当使用情况无法归因于单一资源时，令牌将被归因于发起令牌使用的用户文件夹。

我们建议您注意那些代表您发送到 LLM 的令牌。通常来说，您在使用 LLM 时包含的信息越多，所使用的计算秒数就越多。例如，以下场景描述了使用计算秒数的不同方式。

  • 在 Pipeline Builder 中，您可以要求 AIP 解释您的转换节点；所选节点的数量会影响 LLM 生成响应所使用的 token 数量，从而影响计算时间。这是因为随着节点数量的增加，LLM 必须处理的有关这些节点配置的文本量也会增加。
  • 在 AIP Assist 中，要求 LLM 生成大量代码块需要更多的输出 token。较短的响应使用更少的 token，因此计算时间也更少。
  • 在 AIP Logic 中，将大量文本与您的提示一起发送需要更多的 token，因此也需要更多的计算时间。

导出 AIP token 使用数据

为了详细分析您在 AIP 中的 LLM 使用情况，您可以从控制面板中的 内部数据集导出 部分导出 AIP Token Usage 数据集。该数据集提供了按模型和资源划分的每日 token 消耗情况，以及相应的计算时间（compute-seconds）和货币消耗。如需更多信息，请参阅 内部数据集导出。
