AIP 计算使用

AIP 计算使用涉及大型语言模型（LLMs）。从根本上讲，LLMs 以文本作为输入，并以文本作为输出。输入和输出的文本量以 标记 来衡量。 使用 AIP 测量计算 以每一定数量的标记的计算秒数来衡量。不同的模型可能有不同的计算使用率，如下所述。

AIP 中的标记

标记是 LLM 用来处理和理解输入的基本文本单位。根据语言和特定模型，一个标记可以短至一个字符，也可以长至一个完整的单词。

重要的是，标记（token）并不总是与单词一一对应。例如，常见单词可能是一个标记，但较长或不常见的单词可能会被拆分为多个标记。甚至标点符号和空格也可能是标记。

不同的模型提供商对什么是标记有不同的定义；例如，OpenAI ↗ 和 Anthropic ↗。平均而言，标记大约有 4 个字符长，一个字符可以是单个字母或标点符号。

在 AIP 中，当应用程序向 LLM 发送提示并接收提示时，会消耗标记（token）。这些提示和响应都包含可测量数量的标记。这些标记可以发送到多个 LLM 提供商；由于供应商之间的差异，这些标记会被转换为 计算秒数，以匹配底层模型供应商的价格。

所有提供 LLM 支持功能的应用程序在使用时都会消耗标记（token）。查看以下列表，了解您与其 LLM 支持功能交互时可能使用标记的应用程序集合。

  • AIP Assist
  • AIP Logic
  • AIP Error Enhancer
  • AIP Code Assist
  • AIP 分析员
  • AI FDE
  • Workshop LLM 支撑工具
  • Quiver LLM 支撑工具
  • Pipeline Builder LLM 支撑工具
  • 直接调用语言模型服务（包括 Python 和 TypeScript 库）

AIP 直接将文本路由到支撑 LLM，这些 LLM 会自行进行分词。文本的大小将决定支撑模型用于生成响应的计算量。

以下是一个发送到 GPT-4o 模型的示例句子。

AIP incorporates all of Palantir's advanced security measures for the protection of sensitive data in compliance with industry regulations.

这个句子包含 140 个字符，并将以以下方式分词，其中 | 字符分隔每个标记。请注意，一个标记并不总是等同于一个单词；一些单词会被拆分为多个标记，例如下面示例中的 AIP 和 Palantir。

A|IP| incorporates| all| of| Pal|ant|ir|'s| advanced| security| measures| for| the| protection| of| sensitive| data| in| compliance| with| industry| regulations|.

这个句子包含 24 个标记，并将使用以下计算秒数：

compute-seconds = 24 tokens * 43 compute-seconds / 10,000 tokens
compute-seconds = 24 * 43 / 10,000
compute-seconds = 0.1032

上述句子中的标记和字符数量已通过 OpenAI 的分词功能 ↗ 验证。

通过AIP了解计算使用情况

由LLM标记产生的计算秒数直接关联到请求该使用的具体应用资源。例如，如果您使用AIP在Pipeline Builder中自动解释一个管道，则LLM生成该解释所使用的计算秒数将归因于该特定的管道。这种归因方式在整个平台上都适用；记住这一点将有助于您追踪您正在使用标记的位置。

在某些情况下，计算使用情况无法归因于平台上的单个资源；包括AIP Assist和Error Explainer等示例。当使用情况无法归因于单个资源时，标记将归因于发起标记使用的用户文件夹。

我们建议您留意代表您发送到LLMs的标记。一般来说，您在使用LLMs时包含的信息越多，使用的计算秒数就越多。例如，以下场景描述了不同使用计算秒数的方式。

  • 在Pipeline Builder中，您可以要求AIP解释您的转换节点；所选节点的数量会影响LLM生成响应时使用的token数量，从而影响计算秒数的使用。这是因为随着节点数量的增加，LLM必须处理的关于这些节点配置的文本量也会增加。
  • 在AIP Assist中，要求LLM生成大量代码块需要更多的输出token。较短的响应使用较少的token，因此消耗更少的计算资源。
  • 在AIP Logic中，与提示一起发送大量文本需要更多的token，因此需要更多的计算秒数。

导出AIP token使用数据

要详细分析您注册的LLM使用情况，可以从控制面板中的内部数据集导出部分导出AIP Token Usage数据集。该数据集提供了按模型和资源划分的每日token消耗情况，以及相应的计算秒数和货币使用情况。有关更多信息，请参见 内部数据集导出。
