创建和删除 Kubernetes 命名空间

要为 Environment 管理 Kubernetes 命名空间，请导航到 K8s 命名空间 部分的 设置 选项卡。

K8s 命名空间 部分仅在 Environment 的 k8s-namespace-agent 代理成功向 Apollo 报告后才会显示。 这需要 helm-chart-operator 代理和 apollo-auth-broker 代理先向 Apollo 报告。

创建 Kubernetes 命名空间

要创建一个新的 Kubernetes 命名空间，请选择 新建 K8s 命名空间 按钮。这将打开一个对话框，您可以在其中提供新命名空间的名称和可选描述。所有命名空间名称必须是有效的 RFC 1123 DNS 标签 ↗（例如 example-namespace 是有效的，但 ExampleNamespace 不是）。

提交后，将生成一个变更请求。一旦由具有 Environment 编辑器角色的用户批准，Apollo 将生成一个计划，在您的 Environment 中创建命名空间。

您可以然后 将实体安装到新创建的命名空间中。

删除 Kubernetes 命名空间

Apollo 仅在命名空间中没有安装任何实体时才会删除 Kubernetes 命名空间。如果命名空间中有已安装的实体，您必须先卸载这些实体，Apollo 才能删除命名空间。

要删除现有的 Kubernetes 命名空间，您首先必须将其标记为不受保护。保护标记是一种额外的安全层，用于防止意外删除，默认情况下是启用的。您可以选择要删除的命名空间旁边的三个点菜单，然后选择 更改保护标记。如果您改变主意，稍后可以再次切换回保护标记。

这将打开一个对话框，您可以确认对保护标记的更改。

这将创建一个变更请求，必须由具有环境编辑器角色的用户进行批准。一旦获得批准，命名空间将显示为 等待保护状态更改，直到 Apollo 发布了一个取消保护的计划。

一旦命名空间不再显示为 等待保护状态更改，您可以使用 更改删除标记 操作来删除它。

这将打开一个对话框，您可以在其中确认删除操作。如果您在该命名空间中安装了任何实体，Apollo 将会提示您，在这些实体被卸载之前，无法删除该命名空间。

提交后，将生成一个变更请求。一旦获得具有 Environment 编辑器角色的用户批准，Apollo 将生成一个删除命名空间的计划，您可以在 Environment 的 活动 选项卡中查看该计划。
