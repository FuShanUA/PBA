从 foundry_ml [计划弃用] 迁移至 palantir_models

计划弃用

自 2025 年 10 月 31 日起，foundry_ml 库及基于数据集的模型已完全弃用并停止使用。所有遗留的工作流均应迁移至 palantir_models 库，方式包括训练新模型或将现有模型封装在模型适配器中。此外，在 Code Workbooks 中使用 foundry_ml 构建的模型必须在 Jupyter® Code Workspaces 或 Code Repositories 中重新构建。有关使用 palantir_models 构建新模型的指南，请查阅《如何在 Code Repositories 中训练模型》或《如何在 Jupyter® notebooks 中训练模型》。如需迁移工作流方面的额外帮助，请联系 Palantir 支持团队。

从 0.1599.0 版本开始，palantir_models 库提供了对 Spark ML 模型的支持，但我们建议用户在此次迁移过程中，尽可能将此类模型编写为 Scikit-learn 或类似单节点框架，以便在 Live Inference 中获得更好的体验。此外，我们建议尽可能使用实验来替换 MetricSets。请注意，实验目前已支持图像指标，但尚不支持图表指标。

升级活动

模型迁移活动

活动详情

作为两项活动中的首项，升级助手（Upgrade Assistant）中已发布一项活动，旨在帮助用户从基于数据集的模型迁移出来。仅仍在使用的模型会被标记以供审查，其余模型则标记为 Ignored，并默认从活动视图中过滤掉。

用户可以直接在基于数据集的模型页面中，为该模型指定一个替代的模型资产。升级助手会利用此信息来确定资源的迁移状态。已明确指定替代模型的模型状态为 Completed，默认也会从活动视图中过滤掉。总体而言，建议使用此功能，以便将模型的下游使用者引导至新框架中的替代模型。

了解更多信息：

  • 查阅此示例，了解如何在两个框架之间进行迁移；
  • 查阅《迁移常见问题解答》，获取有关迁移工作原理的更多详情。

AIP 驱动的代码建议

在已启用 AIP 的环境中，系统会提供由大语言模型（LLM）通过 AIP 驱动的代码迁移建议。选择下图所示的紫色图标即可查看这些建议。随后，可使用每个文件上方的剪贴板图标来复制生成的代码。

尽管 LLM 能够帮助用户开启迁移工作，但您很可能需要对 LLM 生成的代码进行修改，才能通过各项检查并生成可正常运行的模型。请务必仔细审查代码和模型输出。

消费资源迁移活动

系统已发布第二项活动，用于展示正在消费已弃用的基于数据集模型的资源。这些资源包括：

  • 将基于数据集的模型作为输入的转换作业；
  • 生成基于数据集的模型及其他输出的 Code workbooks 或转换作业；
  • 包含基于数据集模型的实时或批量部署的建模目标。
