安装

Palantir 模型上下文协议 (MCP) 使外部 AI 系统能够从 Foundry 读取数据和元数据，并通过配置的用户令牌权限使用 AI 友好的 API 端点进行交互。一旦外部系统通过 Palantir MCP 访问了数据，其使用的管理将从 Palantir 转移到外部系统。然后，数据安全将取决于你与该外部系统的合作关系。使用 Palantir MCP 并不意味着会使用 Palantir AI 模型；AI 模型供应商由你连接的外部系统决定，例如 GitHub Copilot 的 Microsoft 或 Cursor。

以下说明解释了如何在支持的 IDE 中设置 Palantir MCP。或者，使用 AI 开发工具，可在 VS Code 工作区中访问 Palantir MCP。

1. 在控制面板中启用 Palantir MCP

Platform administrators must first enable Palantir MCP for use through Control Panel. Access can be configured for a specific subset of users or groups. Palantir MCP is in Control Panel under Code Repositories.

2. Configure your IDE to use Palantir MCP

See instructions for:

  • Claude Code
  • Cline
  • Continue
  • Cursor
  • VS Code Copilot
  • Windsurf

Claude Code

  1. Generate a user token from Palantir, then add this in place of <token> in the code below.
  2. Replace <your-foundry-hostname> with your Foundry hostname (for example, mycompany.palantirfoundry.com). Do not prefix with "https://".
  3. Review the Claude Code documentation ↗ to install Palantir MCP in your Claude Code CLI, particularly the documentation around scoping where MCP should be installed. We recommend installing the MCP at the project scope ↗ level so that the MCP server configuration is scoped to the current project.
  4. Run the following commands in your terminal:
Copied!
1export FOUNDRY_HOST="<your-foundry-hostname>"
2export FOUNDRY_TOKEN=<token>
3claude mcp add palantir-mcp \
4    --scope project \
5    -e FOUNDRY_TOKEN='${FOUNDRY_TOKEN}' \
6    -- npx "-y" "palantir-mcp" "--foundry-api-url" "https://$FOUNDRY_HOST"

Cline

  1. Generate a user token from Palantir, then add this in place of <token> in the code below.
  2. Replace <your-foundry-hostname> with your Foundry hostname (for example, mycompany.palantirfoundry.com).
  3. Review the Cline documentation ↗ to install Palantir MCP into your Cline extension with the configuration below.
Copied!
1{
2    "mcpServers": {
3        "palantir-mcp": {
4            "command": "npx",
5            "args": [
6                "-y",
7                "palantir-mcp",
8                "--foundry-api-url",
9                "https://<your-foundry-hostname>"
10            ],
11            "env": {
12                "FOUNDRY_TOKEN": "<token>"
13            },
14            "disabled": false
15        }
16    }
17}

Continue

  1. Generate a user token from Palantir, then add this in place of <token> in the code below.
  2. Replace <your-foundry-hostname> with your Foundry hostname (for example, mycompany.palantirfoundry.com).
  3. Review the Continue documentation ↗ to install Palantir MCP into your Continue extension with the configuration below.
Copied!
1mcpServers:
2  - name: palantir-mcp
3    command: npx
4    args:
5      - "-y"
6      - "palantir-mcp"
7      - "--foundry-api-url"
8      - "https://<your-foundry-hostname>"
9    env:
10        FOUNDRY_TOKEN: "<token>"

Github Copilot in VS Code

  1. Generate a user token from Palantir, then add this in place of <token> in the code below.
  2. Replace <your-foundry-hostname> with your Foundry hostname (for example, mycompany.palantirfoundry.com).
  3. Review the VS Code documentation ↗ to install Palantir MCP into your VS Code IDE with the configuration below.
Copied!
1"mcp": {
2    "inputs": [
3        {
4            "type": "promptString",
5            "id": "foundry-token",
6            "description": "Foundry user token",
7            "password": true
8        }
9    ],
10    "servers": {
11        "palantir-mcp": {
12            "type": "stdio",
13            "command": "npx",
14            "args": [
15                "-y",
16                "palantir-mcp",
17                "--foundry-api-url",
18                "https://<your-foundry-hostname>"
19            ],
20            "env": {
21                "FOUNDRY_TOKEN": "${input:foundry-token}"
22            }
23        }
24    }
25}
