设置

  1. 打开 数据连接 应用程序，并在屏幕右上角选择 + 新源。
  2. 选择 NetSuite SuiteTalk 从可用的连接器类型中选择。
  3. 按照后续的配置提示，使用下面部分的信息继续设置您的连接器。

在 NetSuite 中配置用户角色和权限

NetSuite中的访问控制是通过将角色分配给用户来配置的；每个角色是一组权限的集合，这些权限定义了用户可以执行哪些任务以及可以访问哪些数据。我们建议为将连接到Foundry的用户进行以下配置：

  1. 创建一个具有适当权限的专用角色。

    1. 从NetSuite的工具栏中，选择设置 > 用户/角色 > 管理角色 > 新建，并为该角色提供一个明确的名称。我们建议使用foundry-role。
      • 您可以选择勾选仅限网络服务配置框。
    2. 通过导航到角色页面底部并选择权限 > 设置，为该角色添加系统范围内的权限。您需要添加的最小必要权限包括：
      • 使用访问令牌登录,
      • SOAP网络服务,
      • 其他自定义字段,
      • 自定义主体字段,
      • 自定义商品字段,
      • 记得要保存权限。
    3. 将表权限添加到角色中，以便能够从Foundry查询这些表。通过导航到角色页面底部并选择权限 > 列表。选择您需要的表，选择添加然后选择保存。

  2. 将新角色分配给用户。

    1. 从NetSuite的工具栏中，选择设置 > 用户/角色 > 管理用户。选择您要用于连接到Foundry的用户，并选择编辑。
    2. 导航到访问选项卡并确保选中提供访问复选框。
    3. 在角色选项卡中，从下拉列表中选择新创建的角色（foundry-role），选择添加然后选择保存。

为了验证是否已添加了正确的权限，请以分配了新角色的用户身份登录，并检查您是否可以查看所有预期的数据。

在NetSuite中配置集成和访问令牌

集成记录用于在NetSuite中管理与外部系统的连接。我们建议以下配置以连接到Foundry：

  1. 使用TBA创建一个新的集成记录（参见 更多详情 ↗）。
    1. 从NetSuite的工具栏中，选择 设置 > 集成 > 管理集成 > 新建，并为集成提供一个明确的名称。我们推荐 foundry-integration。
    2. 确保 状态 是 已启用，并且仅选中 基于令牌的身份验证。其他所有选项应取消选中。
    3. 保存后，请记下 CLIENT ID 和 CLIENT SECRET。您需要它们来配置Foundry。

CLIENT ID 和 CLIENT SECRET 仅在您第一次保存集成记录时显示。要获取新值，您必须重置 CLIENT ID 和 CLIENT SECRET，这将使之前的值失效。

  1. 创建并分配一个 TBA 令牌。
    1. 从 NetSuite 的工具栏中，选择 设置 > 用户/角色 > 访问令牌 > 新建。
      • 如果您无法为其他用户管理令牌，请导航到 NetSuite 首页的左下角 设置 面板并选择 管理访问令牌。
    2. 选择新创建的 应用（在我们的示例中为 foundry-integration），您已分配新创建角色的用户（foundry-role）以及新创建的角色。
      • 如果您无法为其他用户管理令牌，您的用户将默认被选中。确保您的用户已分配新创建的角色（foundry-role）。
    3. 保存后，记下 TOKEN ID 和 TOKEN SECRET。您需要它们来配置 Foundry。

TOKEN ID 和 TOKEN SECRET 只会在你第一次保存令牌时显示。你需要创建一个新的令牌以获得新的 TOKEN ID 和 TOKEN SECRET。

网络连接

NetSuite SuiteTalk 连接器需要访问你想要连接的 NetSuite 实例的网络。

Foundry 工作节点连接

如果你的连接在 Foundry 工作节点 上运行，你需要将以下 出站策略 添加到源中：

  • <ACCOUNT_ID>.suitetalk.api.netsuite.com 在端口 443 上；当你连接到 NetSuite 时，可以在 URL 中找到你的账户 ID。
  • webservices.netsuite.com 在端口 443 上。

如果这些出站策略不存在，您可以 请求它们；否则您可以 添加它们。

连接详情

选项是否必需？描述
