Oracle NetSuite SuiteTalk

使用 SuiteTalk 框架将 Foundry 连接到 Oracle NetSuite，并开始将数据从您的 NetSuite ERP 同步至 Foundry。

设置

  1. 打开《数据连接》应用程序，并在屏幕右上角选择 + 新建源。
  2. 从可用的连接器类型中选择 NetSuite SuiteTalk。
  3. 按照后续的配置提示，结合下方各章节的信息继续完成连接器的设置。

了解有关在 Foundry 中《设置连接器》的更多信息。

身份验证

NetSuite SuiteTalk 源使用《基于令牌的身份验证 (TBA) ↗》。

您的账户必须启用基于令牌的身份验证功能。要启用 TBA，请参阅《NetSuite 文档 ↗》。

在 NetSuite 中配置用户角色和权限

NetSuite 中的访问控制通过为用户分配角色来配置；每个角色是权限的集合，定义了用户可以执行的任务以及可以访问的数据。对于将连接到 Foundry 的用户，我们建议采用以下配置：

  1. 创建具有适当权限的专用角色。

    1. 从 NetSuite 的工具栏中，选择 设置 > 用户/角色 > 管理角色 > 新建，并为该角色提供一个明确的名称。我们建议使用 foundry-role。
      • 您可以选择勾选 仅限 Web 服务 配置框。
    2. 导航至角色页面底部，选择 权限 > 设置，为该角色添加系统级权限。您需要添加的最低权限包括：
      • 使用访问令牌登录，
      • SOAP Web 服务，
      • 其他自定义字段，
      • 自定义主体字段，
      • 自定义项目字段，
      • 记得保存这些权限。
    3. 导航至角色页面底部，选择 权限 > 列表，为该角色添加您希望从 Foundry 查询的表权限。选择所需的表，点击 添加，然后点击 保存。

  2. 将新角色分配给用户。

    1. 从 NetSuite 的工具栏中，选择 设置 > 用户/角色 > 管理用户。选择您要用于连接 Foundry 的用户，然后选择 编辑。
    2. 导航至 访问 标签页，并确保已勾选 授予访问权限 复选框。
    3. 在 角色 标签页中，从下拉列表选择新创建的角色（foundry-role），点击 添加，然后点击 保存。

要验证是否添加了正确的权限，请以分配了新角色的用户身份登录，并检查您是否能查看所有预期的数据。

在 NetSuite 中配置集成和访问令牌

NetSuite 中的集成记录用于管理与外部系统的连接。我们建议使用以下配置来连接 Foundry：

  1. 使用 TBA 创建新的集成记录（参见《更多详情 ↗》）。
    1. 从 NetSuite 的工具栏中，选择 设置 > 集成 > 管理集成 > 新建，并为该集成提供一个明确的名称。我们建议使用 foundry-integration。
    2. 确保 状态 为 已启用，并且仅勾选 基于令牌的身份验证。取消勾选所有其他复选框。
    3. 保存 后，请记录下 CLIENT ID 和 CLIENT SECRET。您将需要它们来配置 Foundry。

CLIENT ID 和 CLIENT SECRET 仅在您首次保存集成记录时显示。要获取新值，您必须重置 CLIENT ID 和 CLIENT SECRET，这将使之前的值失效。

  1. 创建并分配 TBA 令牌。
    1. 从 NetSuite 的工具栏中，选择 设置 > 用户/角色 > 访问令牌 > 新建。
      • 如果您无法为其他用户管理令牌，请导航至 NetSuite 主页左下角的 设置 面板，并选择 管理访问令牌。
    2. 选择新创建的应用程序（本例中为 foundry-integration）、分配了新创建角色的用户（foundry-role）以及新创建的角色。
      • 如果无法管理其他用户的令牌，默认将选择您的用户。请确保您的用户已分配新创建的角色（foundry-role）。
    3. 保存后，请记下 TOKEN ID 和 TOKEN SECRET。配置 Foundry 时将需要这些信息。

TOKEN ID 和 TOKEN SECRET 仅在首次保存令牌时显示。如需获取新的 TOKEN ID 和 TOKEN SECRET，必须创建新令牌。

网络连接

NetSuite SuiteTalk 连接器需要网络访问权限，才能连接到目标 NetSuite 实例。

Foundry worker 连接

如果连接在 Foundry worker 上运行，需向源添加以下出口策略：

  • 端口 443 上的 <ACCOUNT_ID>.suitetalk.api.netsuite.com；连接到 NetSuite 时，可在 URL 中找到账户 ID。
  • 端口 443 上的 webservices.netsuite.com。

若出口策略不存在，可申请策略；也可自行添加策略。

选项是否必填？描述

访问令牌密钥是创建 TBA 令牌 时复制的 TOKEN SECRET
