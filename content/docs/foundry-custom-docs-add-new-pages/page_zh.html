将新页面添加到自定义文档

平台内的自定义文档是用 Markdown 编写的。要向自定义文档包中添加新页面，请在文档仓库中创建一个新的 Markdown 文件（.md），并确保满足以下要求：

  • 页面必须使用 @title 作为页面标题
  • 页面必须列在 ordering.md 中
  • 可选：为目录指定一个标签

文档仓库中的每个 Markdown 文件（.md）都会被发布为单独的文档页面，按包分组并通过自动生成的目录链接。

要编辑自定义文档页面的内容，请使用代码仓库创建文档仓库的新分支，修改 Markdown 文件，并提交/构建您的更改。

指定页面标题

Foundry -> Foundry Apollo -> Apollo Gotham -> Gotham AIP -> AIP Ontology -> 本体论 ontology -> 本体论 Pipeline -> 管道 pipeline -> 管道 Transform -> 转换 transform -> 转换 Workshop -> Workshop workshop -> Workshop Code Repositories -> 代码仓库 Code Workspaces -> 代码工作区 Data Integration -> 数据集成 Model Integration -> 模型集成 Object -> 对象 object -> 对象 Dataset -> 数据集 dataset -> 数据集 Tenant -> 租户 tenant -> 租户 Marking -> 标记 marking -> 标记 Permission -> 权限 permission -> 权限 Role -> 角色 role -> 角色 Function -> 函数 function -> 函数 Action -> 操作 action -> 操作 Property -> 属性 property -> 属性 Interface -> 接口 interface -> 接口 内容: 文档包中的所有内容页面都需要指定页面标题。这是通过在每个 Markdown 文件的顶部使用 @title 注释来完成的（除了 overview.md，它使用 @name）。

例如：

@title 示例标题

这是名为“示例标题”的文档页面的内容。

请注意，实际文件名和使用 @title 注释指定的页面名称不需要相同。

将新页面添加到 ordering.md

文档包中的每个页面（除必需的 overview.md 外）都必须在文档仓库中的 ordering.md 文件中进行指定。ordering.md 允许您使用 章节注释 来构建和组织文档包。左侧的目录是通过这些章节注释自动生成的。

了解更多关于组织您的文档包的信息。

可选：在目录中使用不同的标签

内容页面的文件名用于生成该页面的URL，但@title页面标题用于左侧目录。如果您希望目录中的页面标签与页面标题不同（例如，如果页面标题很长），可以使用@toc注释来指定目录标签。

基于上面的例子，我们可能会有一个如下文件：

@title 示例长标题（首字母大写）
@toc 示例

这是名为“示例长标题（首字母大写）”的文档页面的内容。该页面的标题在目录中将显示为“示例”。
