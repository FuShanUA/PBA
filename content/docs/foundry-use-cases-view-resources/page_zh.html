查看资源

创建新用例后，可以在用例概览页面查看已连接的应用程序和本体资源。如果是基于现有后端项目创建的用例，这些资源会自动显示在用例中。如果是随新用例一起创建的新项目，则需要先添加应用程序资源或数据源，随后才能查看这些资源及其关联的本体组件。

用例概览提供了元数据，包括用例工作流的描述、用例数据主人的姓名以及用例的状态。该概览还包含“应用程序”、“本体”和“后端数据”资源板块。本体资源会按对象类型和操作类型进行分类。选择概览或左侧边栏中的板块标题，即可打开相应的资源页面。

查看应用程序

在“应用程序”页面中，可以找到为用例创建的所有应用程序资源，或通过现有项目引入的资源，包括 Workshop 模块、Slate 文档和 Quiver 分析。在下面的示例中，我们的用例包含两个 Workshop 模块和一个 Slate 文档。

选择应用程序资源的名称即可打开该应用程序页面。

在应用程序页面中，可以查看该应用程序使用的对象类型，以及使用了该对象类型的其他任何应用程序。

用例应用程序表格列表中的“操作”菜单支持直接跳转至用于创建该资源的 Foundry 应用程序。要修改应用程序资源，请将鼠标悬停在表格中的任意行上，然后选择“编辑应用程序”。

查看本体组件

用例本体资源包括对象类型和操作类型。在用例应用程序中引用的对象类型和操作类型，会自动作为本体资源填充到用例中。

了解更多关于创建本体组件的信息。

对象类型

在“对象类型”页面中，可以找到用例中任何应用程序使用的所有对象类型。继续上面的示例，我们的两个 Workshop 模块应用程序中使用了几种对象类型。

对象类型表格列表中的“操作”菜单支持直接跳转至 Ontology Manager 应用程序，以编辑该对象类型的定义。如果某些手动添加的对象类型未被用例中的任何应用程序使用，也可以选择“移除”将其删除。

表格中列出的对象类型可能包含以下指示标志：

  • 对象类型状态：处于“实验性”或“已弃用”状态的对象类型会在该行显示状态标签。没有状态标签的对象类型处于“活跃”状态。
  • 共享/本地状态：由用例项目中的数据源作为后端支撑的对象类型会显示“本地”标签。没有“本地”标签的对象类型由非用例项目中的数据源作为后端支撑。

当对象类型标有“本地”标签，但同时被非当前用例的应用程序使用时，系统将显示警告标志，建议将用例的所有后端数据集迁移至共享的本体项目中。

选择一个对象类型，以查看有关该资源的更多详细信息，包括属性、数据源和源系统详细信息。在下面的示例中，我们可以看到有关 Flight 对象类型的更多详细信息。

单击 View and edit in Ontology Manger 以在 Ontology Manager 应用程序中打开该对象类型。

操作类型

在 Action types 页面，您将找到本体中连接到用例中应用程序资源的所有操作类型。

操作类型表格列表中的 Actions 菜单允许您直接导航至 Ontology Manager 应用程序以编辑操作类型的定义。如果手动添加的操作类型未被用例中的任何应用程序使用，您还可以选择 Remove 将其移除。

表格中列出的操作类型可能包含以下指示器：

操作类型状态：处于 Experimental 或 Deprecated 状态的操作类型将在该行上显示状态标签。没有状态标签的操作类型处于 Active 状态。

详细了解有关创建本体的更多信息。

查看数据源

Backing data 部分提供为您的用例中所使用的本体对象类型提供数据填充的数据源列表。选择一个数据源以在数据集预览中查看它。

底层数据表格列表中的 Actions 菜单允许您打开输入和回写数据源的数据血缘关系图。您还可以查看馈送此数据源的 Update via 资源。

表格中列出的底层数据源可能包含以下指示器：

对象类型状态：处于 Experimental 或 Deprecated 状态的对象类型将在该行上显示一个标签。请注意，没有状态标签的对象类型处于 Active 状态。

回写指示器：当存在回写数据源但其位于不同的项目中时，输入数据源和一个信息指示器将显示在该行上。
