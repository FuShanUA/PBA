S3兼容API for Foundry数据集

Foundry数据集的S3兼容API允许您将Foundry数据集当作S3存储桶来操作。了解通过API访问数据集时的行为，并查看设置指南和示例。

Foundry公开了亚马逊简单存储服务（S3）API ↗的一部分，使您可以使用能够与S3存储服务通信的客户端与Foundry数据集进行交互。例如包括AWS CLI、AWS SDKs、Hadoop S3文件系统和Cyberduck。

S3 API 并未完全实现，因为并非所有的 S3 概念都能自然地映射到 Foundry 中的概念。例如，创建和删除存储桶（代表 Foundry 数据集）目前不被支持；在使用 API 之前，应在 Foundry 中先创建数据集。然而，大多数文件的读取、写入和删除工作流是受支持的，包括多部分上传。有关哪些 S3 操作受支持的列表，请参见 支持的操作。

概念

本节概述了 S3 概念如何映射到 Foundry 数据集概念。

S3 存储桶对应 Foundry 数据集

S3 存储桶 对应 Foundry 数据集，存储桶名称是 Foundry 数据集的唯一标识符（例如，ri.foundry.main.dataset.bafb7e96-84f1-4d23-a4a5-40b17c6912e7）。

S3 对象键 对应 Foundry 数据集内文件的逻辑路径 （例如，top-level-file.csv 或 subfolder/nested-file.csv）。

分支

API 支持通过字母数字名称（仅包含 a-z、A-Z 或 0-9）或特殊字符 - 和 _ 访问数据集的分支。要指定一个分支，可以通过在存储桶名称后追加分支名称，用点分隔：<dataset-rid>.<branch-name>。如果没有指定分支，则使用默认分支。

例如，要访问具有 RID ri.foundry.main.dataset.bafb7e96-84f1-4d23-a4a5-40b17c6912e7 的数据集的 mybranch 分支，请使用存储桶名称 ri.foundry.main.dataset.bafb7e96-84f1-4d23-a4a5-40b17c6912e7.mybranch。

S3 存储桶名称验证施加了之前描述的字符限制。如果您的分支包含不允许的字符，如 /，可以使用 Base64 编码对分支名称进行编码。您应省略任何尾随的 = 字符。例如，feature/my-branch 会被 Base64 编码为 ZmVhdHVyZS9teS1icmFuY2g=。ZmVhdHVyZS9teS1icmFuY2g（不带 = 字符）可用于在存储桶名称中引用此分支。

API 不支持创建分支；指定的分支必须已经在数据集中存在。

事务

该API支持以类似分支的方式访问数据集事务。这允许用户访问数据集的历史版本。例如，要访问RID为ri.foundry.main.dataset.bafb7e96-84f1-4d23-a4a5-40b17c6912e7的数据集的ri.foundry.main.transaction.0cdfe8c9-f595-4859-a194-7daecff9d6fe事务，请使用存储桶名称ri.foundry.main.dataset.bafb7e96-84f1-4d23-a4a5-40b17c6912e7.ri.foundry.main.transaction.0cdfe8c9-f595-4859-a194-7daecff9d6fe。

只有已提交的事务才能以这种方式访问。因此，存储桶将是只读的；当使用包含事务标识符的存储桶名称时，无法放入或删除对象。

事务管理

S3没有事务的概念，因此 Foundry数据集事务 会自动处理，具有以下行为：

  1. 当用户写入或删除文件时，事务才会被延迟打开。
  2. 该API支持同时写入多个文件 或 同时删除多个文件。文件将在同一笔打开的事务中被修改，写入操作使用UPDATE事务，删除操作使用DELETE事务。
  3. 由于这需要同时激活一个UPDATE和一个DELETE事务，因此API不支持同时进行写入和删除操作。
  4. 在写入或删除操作之后，如果有一段短时间的不活动，事务会自动提交。这允许您在同一事务中上传多个文件，避免出现许多只包含少量文件的事务。为了避免事务无限期地保持打开状态，例如当文件持续上传而没有不活动时间段时，一旦没有活动上传，事务将在一定时间后提交。如果文件是并行连续上传的，则事务将保持打开状态，直到所有上传完成。
  5. 如果在事务打开期间的静默期内接收到读取请求，则可能立即提交UPDATE和DELETE事务。这旨在保持S3中通常预期的读后写语义。然而，如果在仍有活动的写入请求到打开的事务时尝试读取，读取将从最新的已提交视图中进行。为了确保事务在所有写入或删除完成后提交，您可以在任何新的写入或删除请求之前发出后续的读取请求。

鉴于上述行为，不能保证读后写语义。但是，会尽最大努力在可能的情况下提供这些语义。

认证

通过API的连接使用访问密钥ID和秘密访问密钥凭据进行认证。

静态凭据

静态凭据类似于标准的AWS访问密钥ID和秘密访问密钥凭据。静态凭据是长期有效的，并且在Foundry的情况下，与在Foundry控制面板中注册的第三方应用程序的服务用户相关联。

当使用静态访问密钥ID和秘密访问密钥通过S3兼容API连接到数据集时，访问级别由授予第三方应用程序服务用户的访问权限决定。静态凭据必须限制为单个 项目（Project）。项目限制是在生成新一组凭据时指定的。只有在指定项目内的数据集才能使用生成的凭据进行访问。

有关使用 /io/s3/api/v2/credentials API端点生成这些凭据的指导，请参见下面的 设置指南（Setup Guide）。

临时凭据（Temporary credentials）

我们建议使用 静态凭据（Static credentials） 在任何需要长期有效的凭据或有利于将访问权限绑定到服务用户的流程中。如果您希望以常规的Foundry用户身份对API进行身份验证，我们支持将 用户身份验证令牌 交换为临时S3凭据。此令牌也可以通过 OAuth2 的一种 第三方应用程序 授权方式获得。

临时凭据是通过标准的 AssumeRoleWithWebIdentity ↗ 安全令牌服务（STS）API 获取的。我们只需要 WebIdentityToken 请求参数存在，并按照上述说明配置为常规的Foundry令牌。返回的临时凭据将具有与提供的令牌相同的范围。 DurationSeconds 参数可以提供以指定凭据的有效期。凭据的最大有效期为一小时，并且永远不会超过用于获取临时凭据的Foundry令牌的有效期。

STS API 可以在 https://<FOUNDRY_URL>/io/s3 访问。如果您希望通过编程方式获取STS凭据，应将此URL配置为标准STS客户端或凭证提供程序的端点配置。例如：

Copied!
1import boto3
2
3endpoint = "https://<FOUNDRY_URL>/io/s3"
4
5session = boto3.session.Session()
6client = session.client(service_name='sts', endpoint_url=endpoint)
7
8# RoleArn and RoleSessionName are required parameters in boto3 despite being unused
9credentials = client.assume_role_with_web_identity(
10    RoleArn="xxxxxxxxxxxxxxxxxxxx",
11    RoleSessionName="xxxxx",
12    WebIdentityToken=token
13)["Credentials"]

一旦获得临时凭证，请按照下面的< a href="#setup-guide">设置指南中的第四步来获取配置S3客户端的指导。您不需要遵循任何与第三方应用程序相关的步骤。在配置S3客户端时，必须提供会话令牌、访问密钥ID和秘密访问密钥。

要通过S3兼容API读取或写入数据，用户需要s3-proxy:datasets-read和s3-proxy:datasets-write权限，根据默认设置，分别授予Viewer和Editor角色。使用静态凭证时，对应第三方应用的服务用户需要被授予相关角色。使用临时凭证时，获得凭证的用户需要具有相关角色。

路径风格的URL访问

API仅支持 路径风格 ↗ 的存储桶访问。您的存储桶URL将采用 https://<FOUNDRY_URL>/io/s3/<bucket-name>/<key-name> 格式。

在Foundry术语中，这意味着 https://<FOUNDRY_URL>/io/s3/<dataset-rid>/<logical-filepath>。

虚拟托管风格的存储桶访问（其中存储桶名称包含在URL的子域名中）目前不被支持。

预签名的URL

预签名的URL支持 putObject 操作。

设置指南

按照以下步骤从您的S3客户端设置与API的连接。

  1. 在Foundry中注册第三方应用程序
  2. 授予您的第三方应用程序的服务用户权限
  3. 生成S3访问密钥ID和秘密访问密钥
  4. 配置 S3 客户端

查看特定 S3 客户端的示例配置设置。

步骤 1：在 Foundry 中注册第三方应用程序

为了获得 S3 兼容 API 的凭据，您首先需要获取已在 Foundry 控制面板中创建的 第三方应用程序 的客户端 ID 和密钥：

  1. 在 Foundry 中打开 控制面板。
  2. 在侧边栏中选择 第三方应用程序。
  3. 选择 新建应用程序，然后使用以下参数完成设置向导：
    • 客户端类型：选择 保密客户端。
    • 授权授予类型：启用 客户端凭证授予。
  4. 在 摘要 页面右下角选择 注册应用程序。
  5. 在完成屏幕上，记录 客户端 ID 这样将在下一步中需要用到。然后选择启用应用程序使用，并使用切换开关来启用它。

查看 概念：认证 以了解项目限制的要求和行为，以及生成的凭据的范围。

步骤 2：为您的第三方应用程序的服务用户授予权限

用户应使用 开发者控制台 来管理他们的应用程序配置。只有在 开发者控制台 未为用户启用时，控制面板 视图才适用。

在上一步中，当你创建第三方应用程序时，Foundry会自动生成一个服务用户。要通过S3 API访问数据集，此服务用户必须在相关项目和标记上具有足够的权限。

设置服务用户的权限：

  1. 查找服务用户的名称和ID。您可以在应用程序的“管理应用程序”页面上找到这些信息，位于授权类型 > 客户端凭证授权 > 服务用户下。
  2. 为服务用户授予权限，可以通过直接将用户添加到项目和标记中，或者将用户添加到已获得这些权限的组中。

第3步：生成S3访问密钥ID和秘密访问密钥

要生成凭据，您需要在< a href="/docs/foundry/administration/enrollments-and-organizations-permissions/">控制面板中的组织上拥有User experience administrator角色

。

运行下面的终端命令（使用 curl 或 Powershell）以接收访问密钥 ID 和秘密访问密钥。将 <TOKEN> 替换为您的用户帐户的活动令牌，将 <CLIENT_ID> 替换为上一步中生成的第三方应用程序的客户端 ID。此外，您必须将 <PROJECT_RID> 替换为凭据有访问权限的项目的 RID。projectRestrictions 值可以包含多个项目，如果需要的话，您可以列出更多的项目 RID。至少需要指定一个项目。项目 RID 应该是 ri.compass.main.folder.{RID_VALUE} 的形式。

选项 1：curl

Copied!
1curl -X POST \
2    -H "Authorization: Bearer <TOKEN>" \
3    -H "Content-type: application/json" \
4    --data '{"clientId":"<CLIENT_ID>","projectRestrictions":["<PROJECT_RID>"]}' \
5    https://<FOUNDRY_URL>/io/s3/api/v2/credentials

选项 2：Powershell（Windows）

Copied!
1$headers = @{
2    "Authorization" = "Bearer <TOKEN>"
3    "Content-type" = "application/json"
4}
5$body = @{
6    "clientId" = "<CLIENT_ID>"

8} | ConvertTo-Json
9
10Invoke-WebRequest -Uri "https://<FOUNDRY_URL>/io/s3/api/v2/credentials" -Method POST -Headers $headers -Body $body

将您在此请求响应中收到的访问密钥和秘密密钥安全地存储起来。您必须用这些凭证配置客户端，而不是第三方应用程序的客户端 ID 和密钥。

指定组织

默认情况下，/v2/credentials 端点假定认证用户正在为他们自己的 组织（Organization） 中的第三方应用程序生成凭据。如果第三方应用程序位于不同的组织中，请在 URL 中将组织 ID 作为查询参数指定：https://<FOUNDRY_URL>/io/s3/api/v2/credentials?organizationRid=<ORGANIZATION_ID>。

吊销凭证

如果您需要吊销访问密钥和秘密访问密钥，请调用以下端点，并将 <ACCESS_KEY_ID> 替换为您希望吊销的访问密钥 ID：

Copied!
1curl -X DELETE \
2    -H "Authorization: Bearer <TOKEN>" \
3    https://<FOUNDRY_URL>/io/s3/api/v2/credentials/<ACCESS_KEY_ID>

列出凭证

您可以使用以下端点来检索活动（未吊销）访问密钥列表，包括它们的客户端 ID 和项目限制。

Copied!
1curl -X GET \
2    -H "Authorization: Bearer <TOKEN>" \
3    https://<FOUNDRY_URL>/io/s3/api/v2/credentials

DuckDB
DuckDB ↗

'S 对 S3 的支持可用于查询 Foundry 数据集。您可以使用 DuckDB 秘密来管理凭据，并使用以 s3:// 为前缀的 URL 查询数据集。


CREATE SECRET foundry_secret (
    TYPE S3,
    KEY_ID '<ACCESS_KEY_ID>',
    SECRET '<SECRET_ACCESS_KEY>',
    REGION 'foundry',
    ENDPOINT '<FOUNDRY_URL>/io/s3',
    URL_STYLE 'path'
);

CREATE TABLE new_tbl AS SELECT * FROM 's3://ri.foundry.main.dataset.<uuid>/spark/*.parquet';



有关更多信息，请参阅 DuckDB 文档 ↗。
在上述秘密配置中，ENDPOINT 配置参数不应包含 https:// 方案。URL 方案由 USE_SSL 参数自动处理，默认值为 true。
