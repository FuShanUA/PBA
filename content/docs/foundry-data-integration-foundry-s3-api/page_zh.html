Foundry 数据集的 S3 兼容 API

Foundry 数据集的 S3 兼容 API 允许你像操作 S3 存储桶一样与 Foundry 数据集进行交互。了解通过该 API 访问时数据集的具体行为，并查看设置指南和示例。

Foundry 暴露了《Amazon Simple Storage Service (S3) API ↗》的一个子集，让你能够使用支持与 S3 存储服务通信的客户端来操作 Foundry 数据集。典型的客户端包括 AWS CLI、AWS SDK、Hadoop S3 文件系统以及 Cyberduck。

S3 API 并未完全实现，因为并非所有的 S3 概念都能自然映射到 Foundry 的概念上。例如，目前不支持创建和删除存储桶（对应 Foundry 数据集）；在使用该 API 之前，应先在 Foundry 中创建好数据集。不过，绝大多数文件读/写/删除工作流均受支持，包括分片上传。有关受支持的 S3 操作列表，请参阅支持的操作。

概念

本部分概述了 S3 概念如何映射到 Foundry 数据集概念。

S3 存储桶对应 Foundry 数据集

S3 存储桶对应Foundry 数据集，存储桶名称即为 Foundry 数据集的唯一标识符（例如 ri.foundry.main.dataset.bafb7e96-84f1-4d23-a4a5-40b17c6912e7）。

S3 对象键对应Foundry 数据集内文件的逻辑路径（例如 top-level-file.csv 或 subfolder/nested-file.csv）。

分支

该 API 支持访问名称仅包含字母数字（即只含 a-z、A-Z 或 0-9）或特殊字符 - 和 _ 的数据集分支。若要指定分支，请在存储桶名称后追加分支名称，并以句点分隔：<dataset-rid>.<branch-name>。若未指定分支，则使用默认分支。

例如，要访问 RID 为 ri.foundry.main.dataset.bafb7e96-84f1-4d23-a4a5-40b17c6912e7 的数据集的 mybranch 分支，请使用存储桶名称 ri.foundry.main.dataset.bafb7e96-84f1-4d23-a4a5-40b17c6912e7.mybranch。

S3 存储桶名称验证会施加前文所述的字符限制。如果您的分支包含不允许的字符（例如 /），您可以使用 Base64 对分支名称进行编码。请省略所有末尾的 = 字符。例如，feature/my-branch 的 Base64 编码为 ZmVhdHVyZS9teS1icmFuY2g=。可以使用 ZmVhdHVyZS9teS1icmFuY2g（不含 = 字符）在存储桶名称中引用此分支。

API 不支持创建分支；指定的分支必须已在数据集上存在。

事务

API 支持以类似于分支的方式访问数据集事务。这允许用户访问数据集的历史版本。例如，要访问 RID 为 ri.foundry.main.dataset.bafb7e96-84f1-4d23-a4a5-40b17c6912e7 的数据集的 ri.foundry.main.transaction.0cdfe8c9-f595-4859-a194-7daecff9d6fe 事务，请使用存储桶名称 ri.foundry.main.dataset.bafb7e96-84f1-4d23-a4a5-40b17c6912e7.ri.foundry.main.transaction.0cdfe8c9-f595-4859-a194-7daecff9d6fe。

只有已提交的事务才能以这种方式访问。因此，该存储桶将是只读的；当使用包含事务标识符的存储桶名称时，无法放置或删除对象。

事务管理

S3 没有事务的概念，因此 Foundry 数据集事务 会自动按以下行为进行处理：

  1. 当用户写入或删除文件时，事务会延迟开启。
  2. API 支持并发写入多个文件或并发删除多个文件。这些文件将在同一个开启的事务中被修改，其中写入使用 UPDATE 事务，删除使用 DELETE 事务。
  3. API 不支持并发写入和删除，因为这需要 UPDATE 和 DELETE 事务同时处于活动状态。
  4. 在写入或删除之后的一小段不活动期后，事务会自动提交。这允许您在同一个事务中上传多个文件，从而避免产生许多只包含极少文件的事务。为避免事务被无限期保持开启（例如，如果持续上传文件导致没有不活动期），一旦没有活跃的上传操作，事务将在一定时间后被提交。如果并行持续上传文件，则事务将保持开启，直到所有上传完成。
  5. 若在事务保持开启的闲置期间收到读取请求，系统会在条件允许时立即提交 UPDATE 和 DELETE 事务。此举旨在维持 S3 通常预期的读后写语义。然而，若在已开启的事务仍有活跃写入请求时尝试读取，读取操作将基于最新已提交的视图进行。若要确保事务在所有写入或删除操作完成后才提交，可在发起任何新的写入或删除请求前，先执行一次后续的读取请求。

鉴于上述行为，读后写语义并未得到绝对保证。但系统会在可能的情况下尽最大努力予以提供。

身份验证

通过 API 建立的连接使用访问密钥 ID 和秘密访问密钥凭据进行身份验证。

静态凭据

静态凭据类似于标准的 AWS 访问密钥 ID 和秘密访问密钥凭据。静态凭据具有长期有效性，在 Foundry 场景下，它们与在 Foundry 控制面板中注册的第三方应用的服务用户相关联。

当使用静态访问密钥 ID 和秘密访问密钥通过兼容 S3 的 API 连接数据集时，访问权限级别由授予该第三方应用服务用户的访问权限决定。静态凭据必须限制在特定的项目范围内。项目限制在生成新凭据时指定。使用生成的凭据仅能访问指定项目内的数据集。

有关使用 /io/s3/api/v2/credentials API 端点生成这些凭据的指引，请参阅下方的设置指南。

临时凭据

在任何需要长效凭据的工作流中，或者当将访问权限与服务用户绑定更具优势时，我们建议使用静态凭据。若您希望以常规 Foundry 用户身份对 API 进行身份验证，我们支持将用户身份验证令牌换取临时 S3 凭据。该令牌也可以通过第三方应用的其中一种OAuth2 授权方式获取。

使用标准的 AssumeRoleWithWebIdentity ↗ 安全令牌服务（STS）API 获取临时凭证。仅需提供 WebIdentityToken 请求参数，并按上述说明配置常规的 Foundry 令牌即可。返回的临时凭证与所提供令牌的作用域完全一致。可提供 DurationSeconds 参数来指定凭证的有效期。凭证的最长有效期限为一小时，且绝不会超过用于获取临时凭证的 Foundry 令牌的有效期。

可通过 https://<FOUNDRY_URL>/io/s3 访问 STS API。若需通过编程方式获取 STS 凭证，应将此 URL 配置到标准 STS 客户端或凭证提供程序的端点配置中。例如：

路径样式 URL 访问

该 API 仅支持路径样式存储桶访问。您的存储桶 URL 格式为 https://<FOUNDRY_URL>/io/s3/<bucket-name>/<key-name>。 在 Foundry 术语中，即 https://<FOUNDRY_URL>/io/s3/<dataset-rid>/<logical-filepath>。 目前不支持虚拟主机样式存储桶访问（即在 URL 的子域中包含存储桶名称）。

预签名 URL

putObject 操作支持预签名 URL。

设置指南

请按照以下步骤设置从您的 S3 客户端到该 API 的连接。 - 在 Foundry 中注册第三方应用程序 - 向您的第三方应用程序的服务用户授予权限 - 生成 S3 访问密钥 ID 和秘密访问密钥 - 配置 S3 客户端 查看特定 S3 客户端的示例配置设置。

步骤 1：在 Foundry 中注册第三方应用程序

要获取兼容 S3 的 API 凭证，您首先需要获取在 Foundry 的控制面板中创建的第三方应用程序的客户端 ID 和密钥： - 在 Foundry 中打开控制面板。 - 在侧边栏中选择第三方应用程序。 - 选择新建应用程序，然后使用以下参数完成设置向导：
  • 客户端类型：选择机密客户端。
  • 授权许可类型：启用客户端凭证许可。
  • 在摘要页面右下角，选择注册应用程序。
  • 在完成界面上，记录客户端 ID，后续步骤中将会用到。随后选择启用应用程序，并使用开关将其启用。

    • 查看《概念：身份验证》，以了解项目限制的要求与行为，以及所生成凭证的范围。

    第 2 步：为第三方应用程序的服务用户授权

    用户应使用开发者控制台来管理其应用程序配置。控制面板视图仅在尚未为用户启用开发者控制台时适用。

    在上一步创建第三方应用程序时，Foundry 会自动创建一个服务用户。若要通过 S3 API 访问数据集，该服务用户必须对相关项目和标记具有足够的权限。

    为服务用户设置权限：

    1. 查找服务用户的名称和 ID。您可以在应用程序的“管理应用程序”页面上，依次进入授权许可类型 > 客户端凭证许可 > 服务用户找到这些详细信息。
    2. 为服务用户授权，既可以直接将用户添加到项目和标记中，也可以将用户添加到已被授予这些权限的群组中。

    第 3 步：生成 S3 访问密钥 ID 和秘密访问密钥

    要生成凭证，您需要在控制面板的组织中拥有用户体验管理员角色。

    运行以下终端命令（使用 curl 或 PowerShell）以获取访问密钥 ID 和秘密访问密钥。将 <TOKEN> 替换为您用户账户的有效令牌，将 <CLIENT_ID> 替换为上一步生成的第三方应用程序客户端 ID。此外，必须将 <PROJECT_RID> 替换为凭证有权访问的项目的 RID。projectRestrictions 值可以包含多个项目，如有必要，您可以列出更多的项目 RID。必须至少指定一个项目。项目 RID 的格式应为 ri.compass.main.folder.{RID_VALUE}。

    Spark

    
```python
from pyspark.sql import SparkSession

hostname = "https:///io/s3"
access_key_id = ""
secret_access_key = ""

# 确保数据集 RID 能被解析为有效的主机名
dataset_rid = "ri.foundry.main.dataset.".replace('.', '-')

spark_session = (
    SparkSession.builder
        .config("fs.s3a.access.key", access_key_id)
        .config("fs.s3a.secret.key", secret_access_key)
        # .config("fs.s3a.session.token", session_token) 仅在使用临时凭证时需要
        .config("fs.s3a.endpoint", hostname)
        .config("fs.s3a.endpoint.region", "foundry")
        .config("fs.s3a.path.style.access", "true")
        .getOrCreate()
)

df = spark_session.read.parquet(f"s3a://{dataset_rid}/*")
df.show()
```

    DuckDB
    

    可以利用 DuckDB ↗ 对 S3 的支持来查询 Foundry 数据集。你可以使用 DuckDB 密钥管理凭证，并通过带 s3:// 前缀的 URL 查询数据集。
    

    CREATE SECRET foundry_secret (
    TYPE S3,
    KEY_ID '<ACCESS_KEY_ID>',
    SECRET '<SECRET_ACCESS_KEY>',
    REGION 'foundry',
    ENDPOINT '<FOUNDRY_URL>/io/s3',
    URL_STYLE 'path'
);

CREATE TABLE new_tbl AS SELECT * FROM 's3://ri.foundry.main.dataset.<uuid>/spark/*.parquet';

    

    更多信息请参阅 DuckDB 文档 ↗。
    

    在上述密钥配置中，ENDPOINT 配置参数不应包含 https:// 协议。URL 协议由 USE_SSL 参数自动处理，其默认值为 true。
