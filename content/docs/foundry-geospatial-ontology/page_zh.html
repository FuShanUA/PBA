在本体论中使用地理空间数据

geospatial-tools库不再积极开发；而是使用管道构建器（Pipeline Builder）来加载、转换和处理地理空间数据。本页面上描述的用于操作地理空间数据类型的功能已被管道构建器的地理空间功能所取代。 了解有关管道构建器的地理空间特性的更多信息。

如何使用矢量数据配置本体论数据集

Foundry的 geospatial-tools库 用于解析、清理和将矢量数据格式（如shapefiles和geoJSON）转换为可以添加到本体论中的数据集。然而，此功能已被 Pipeline Builder的地理空间功能 取代 . 您应使用Pipeline Builder而不是geospatial-tools库，因为geospatial-tools库不再维护，可能无法正常工作。

本页面上的文档将继续作为参考，直到geospatial-tools库正式弃用。

点

可以使用geopoint属性类型来指定点几何。geopoint属性的内容应该是一个字符串，包括：

  • latitude,longitude：例如，57.64911,10.40744. 坐标必须使用WGS 84 CRS（标准纬度和经度）。
  • 一个Geohash ↗：例如，u4pruydqqvj. Geohashes将被转换为点，使用Geohash矩形的左下角。

具有geopoint属性的对象会进行地理空间搜索索引。

多边形和线条

多边形和线几何可以通过geoshape属性类型进行指定。geoshape属性的内容必须是符合以下要求的GeoJSON几何字符串：

  • 必须是GeoJSON LineString、Polygon、MultiLineString、MultiPolygon、MultiPoint或Point。但是，点几何不应该使用geoshape属性类型；请使用geopoint属性类型来表示点几何。
  • 不能是Feature、FeatureCollection或GeometryCollection。
  • 必须遵守GeoJSON规范（RFC 7946） ↗。
  • 必须使用WGS 84坐标。
  • 多边形和MultiPolygons必须根据GeoJSON规范有效。多边形和MultiPolygons必须闭合，外部环必须使用右手/逆时针方向，并且没有自相交。

这是geoshape属性的有效GeoJSON示例：

Copied!
1{ "type": "LineString", "coordinates": [ [100.0, 0.0], [101.0, 1.0] ] }

具有geoshape属性的对象将被索引以进行地理空间搜索。

如何可视化、探索和发布地理空间数据

一旦你的地理空间数据被加载到本体论（Ontology）中，有两大主要的本体论（Ontology）映射工具可以使用你的数据：

  • 地图应用 提供沉浸式的地理空间分析和调查功能，包括时间序列叠加、关联分析和模拟。
  • Workshop 是一个专为操作用户设计的应用程序构建框架，包含一个强大的地图小部件。了解更多关于 在 Workshop 中创建地图 的信息。

地理空间功能也可在其他 Foundry 应用程序中使用，包括 Slate、Quiver、对象浏览器 和 Contour。

在地图中使用外部图层

在某些情况下，Foundry 还支持将外部图层添加到地图中；例如，当您有一个外部要素服务或瓦片服务器，想要直接连接到 Foundry 地图，而无需先将所有数据加载到数据集中时。

如需更多详情，请联系您的 Palantir 代表，因为这需要额外的配置才能启用。

Mapbox 边界

与 Mapbox ↗ 合作，Foundry 提供了内置支持，可以使用多种可用选项创建类似统计地图（choropleth）的图层来定义区域边界。

通过边界功能，您可以将您的 Foundry 数据集和/或本体论（ontology）与一组管理好的地理特征关联起来，这些特征表示各种类型的边界，例如行政划分或邮政编码区域。Mapbox 边界中的多边形是边缘匹配的，以实现无缝的全球覆盖，因此您可以创建精确的数据可视化和准确的分析。

图层类型、层级和世界观

Mapbox 提供了跨多种类型、层级和世界观的边界数据。你可以通过 边界探索应用 ↗ 来探索可用的边界类型和层级，以及 关于不同边界类型的更多文档 ↗。

使用 Mapbox 边界

以下说明是用于利用 Mapbox 边界最新版本（v4）的。如果您已经在使用之前的版本，请先 将您的管道和应用程序迁移到 v4。

安装 Mapbox 边界定义

Mapbox 边界定义 v4 可以在 Foundry 市场中下载。要从您的 Foundry 实例访问：

  1. 导航到 /workspace/marketplace/discovery 在您的 Foundry 实例上。
  2. 搜索“Mapbox”或使用左侧面板中的商店筛选器过滤到“Foundry Store”。
  3. 选择Mapbox 边界定义 v4产品。
  4. 点击安装。如果有人已经安装了这些边界，您将看到打开，您可以从中导航到当前安装。

  1. 在安装时：
    1. 选择一个空间进行安装。
    2. 由于该产品仅包含数据集，因此不需要本体论（Ontology）。
    3. 启用安装后缀以自定义数据集将被安装的项目名称。
    4. 在左侧面板底部选择安装。
  2. 安装作业完成后，点击产品名称右侧的查看安装以导航到安装位置。

  1. 你可以通过选择右侧面板中的位置链接来访问已安装的边界数据集。默认情况下，只有你才能访问此项目，但你可以 在右侧的项目详细信息面板中添加其他用户和组。

当有新的小版本边界数据集可用时，你将能够通过出现在安装顶部的横幅进行升级。主要版本发布将作为新的数据源产品提供。

请注意，产品会安装到一个新项目中，该项目默认禁用编辑功能以确保安全升级（例如，如果出现额外的边界数据集）。鉴于此，我们建议将利用这些数据集的内容（例如，分析、管道等）保存在另一个项目中。如果你需要编辑该项目本身，可以通过选择左侧面板中的 设置 然后 解锁，如下所示。

将 Mapbox ID 集成到管道中

为了利用 Mapbox 边界，您需要一个 mapbox ID，该字段用于在特定级别上标识特定类型的边界区域。这些 ID 可以在 边界元数据文件 ↗ 中找到，这些文件作为 Mapbox 边界定义产品的一部分自动提供。有关说明，请参见 安装 Mapbox 定义。

一旦安装了元数据文件，就可以在 Foundry 中将它们与您的数据进行连接，以添加 mapbox_id 列。根据边界类型和级别，元数据文件中将包含其他 ID 和/或名称列，您可以使用这些列作为与自己数据的连接键。

此 mapbox_id 列将用于配置下游应用程序以显示基于边界的地图。

配置下游应用

Mapbox 边界目前可用于以下任何产品的着色图（Choropleth）样式可视化：

  • Workshop
  • Contour
  • 地图应用
  • 对象视图
  • 对象浏览器
  • Quiver

每个消费应用的具体配置选项可能会略有不同，但通用的配置选项将包括：

  • 您数据中包含 mapbox_id 的字段或属性
  • 用于选择特定 世界观 ↗ 的选项，以便根据该世界观显示数据
  • 要使用的边界类型和级别

然后，您可以使用数据集和/或本体论中的数据来驱动这些边界的样式，例如多边形的颜色。有关这些额外样式选项的更具体文档，请访问特定下游应用的文档。

从 Mapbox 边界 v3 迁移到 v4

如果您当前正在使用 Mapbox 的旧版边界数据（如 V2 或 V3），我们强烈建议升级到 v4。除了边界本身的更新外，v4 版本还包括更可靠的工具提示中显示区域名称的支持，以及对基于 Mapbox 边界的地图图层自动缩放至合适的支持等其他改进。

在从 v3 迁移到 v4 时，有一些边界会有破坏性更改。在继续之前，请查看 破坏性更改的详细变更日志 ↗。

要迁移您管道中的数据，您可以：

  • 利用 v3 到 v4 ID 映射表 ↗，这些表格包含在 Mapbox 边界定义 V4 产品 中。 . 使用V3中的feature_id列，将其与您现有的边界数据进行关联，并引入随着v4新增的mapbox_id列。确保手动处理上述提到的任何破坏性更改。
  • 或者，您可以直接按照上述说明操作，并使用边界元数据CSV文件从头开始关联mapbox_id列。

一旦您将mapbox_id列添加到您的管道/本体论中，请重新配置下游应用程序以利用v4边界，并选择此新列以及相应的边界类型/层级。
