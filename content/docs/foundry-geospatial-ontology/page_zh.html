在本体中使用地理空间数据

geospatial-tools 库已不再积极开发；请改用 Pipeline Builder 来加载、转换和处理地理空间数据。本页所述用于处理地理空间数据类型的功能已被 Pipeline Builder 的地理空间功能所取代。了解有关 Pipeline Builder 地理空间功能的更多信息。

如何使用矢量数据配置本体数据集

Foundry 的 geospatial-tools 库旨在解析、清理 shapefile 和 geoJSON 等矢量数据格式，并将其转换为可添加到本体中的数据集。然而，此功能已被 Pipeline Builder 的地理空间功能所取代。您应使用 Pipeline Builder 而非 geospatial-tools 库，因为 geospatial-tools 库已不再维护，可能无法按预期运行。

在 geospatial-tools 库正式废弃之前，本页的文档将作为参考保留。

点

点几何可以通过 geopoint 属性类型来指定。geopoint 属性的内容应为以下字符串之一：

  • latitude,longitude：例如 57.64911,10.40744。坐标必须使用 WGS 84 坐标参考系统（标准经纬度）。
  • Geohash ↗：例如 u4pruydqqvj。Geohash 将使用该 Geohash 矩形的左下角转换为点。

具有 geopoint 属性的对象会被建立索引，以便进行地理空间搜索。

多边形与线

多边形和线几何可以通过 geoshape 属性类型来指定。geoshape 属性的内容必须是符合以下要求的 GeoJSON Geometry 字符串：

  • 必须是 GeoJSON 的 LineString、Polygon、MultiLineString、MultiPolygon、MultiPoint 或 Point。但是，Point 几何不应使用 geoshape 属性类型；请对 Point 几何使用 geopoint 属性类型。
  • 不得是 Feature、FeatureCollection 或 GeometryCollection。
  • 必须符合 GeoJSON 规范 (RFC 7946) ↗。
  • 必须使用 WGS 84 坐标。
  • 多边形和 MultiPolygon 必须符合 GeoJSON 规范。多边形和 MultiPolygon 必须闭合，外环使用右手/逆时针环绕顺序，且不得自交。

以下是 geoshape 属性的有效 GeoJSON 示例：

已复制！
1{ "type": "LineString", "coordinates": [ [100.0, 0.0], [101.0, 1.0] ] }

具有 geoshape 属性的对象会被建立索引，以便进行地理空间搜索。

如何可视化、探索和发布地理空间数据

将地理空间数据加载到本体后，有两个主要的本体映射工具可以使用这些数据：

  • 《地图应用》提供沉浸式的地理空间分析与调查功能，包括时间序列叠加、关联分析和模拟。
  • 《Workshop》是一个面向业务用户的应用构建框架，包含功能强大的地图组件。如需了解更多信息，请参阅《在 Workshop 中创建地图》。

其他 Foundry 应用也提供地理空间功能，包括 Slate、Quiver、Object Explorer 和 Contour。

在地图中使用外部图层

在某些情况下，Foundry 也支持将外部图层添加到地图中；例如，当你拥有外部要素服务或瓦片服务器，并希望将其直接连接到 Foundry 地图，而无需先将所有数据加载到数据集中时。

如需了解更多详细信息，请联系您的 Palantir 代表，因为此功能需要进行额外配置才能启用。

Mapbox 边界

Foundry 与 Mapbox ↗ 合作，内置支持使用各种可用选项来定义区域边界，从而创建分级统计图层。

通过边界功能，你可以将 Foundry 数据集和/或本体与一组受管的地理要素连接起来，这些要素代表各种类型的边界，如行政区划或邮政编码区。Mapbox 边界中的多边形经过边缘匹配，可实现无缝的全球覆盖，因此你可以创建精确的数据可视化和准确的分析。

图层类型、级别和世界观

Mapbox 提供跨多种类型、级别和世界观的边界数据。你可以使用《边界探索器应用》 ↗ 探索可用的边界类型和级别，以及《关于不同边界类型的补充文档》 ↗。

将 Mapbox ID 集成到数据管道中

为了利用 Mapbox 边界，你需要一个 mapbox ID，用于标识特定层级下特定类型的边界区域。这些 ID 可以在《边界元数据文件》 ↗ 中找到，该文件作为 Mapbox 边界定义产品的一部分自动提供。相关说明请参阅《安装 Mapbox 定义》。

安装元数据文件后，请在 Foundry 中将其与你的数据进行连接，以添加 mapbox_id 列。根据边界的类型和层级，元数据文件将包含其他 ID 和/或名称列，你可以将其作为连接键与自己的数据进行匹配。

此 mapbox_id 列将用于配置下游应用，以展示基于边界的地图。

配置下游应用

目前，Mapbox 边界可用于在以下任一产品中生成分层设色图样式的可视化效果：

  • 《Workshop》
  • 《Contour》
  • 《地图应用》
  • 《对象视图》
  • 《对象浏览器》
  • 《Quiver》

各消费应用的具体配置选项可能略有不同，但通用配置选项通常包括：

  • 数据中包含 mapbox_id 的字段或属性
  • 选择特定的《世界视图》 ↗，作为展示数据的参考基准
  • 要使用的边界类型和层级

随后，你可以使用来自数据集和/或本体中的数据来驱动这些边界的样式，例如多边形的着色。有关这些附加样式选项的更具体文档，请访问特定下游应用的文档。

从 Mapbox 边界 v3 迁移至 v4

如果你当前使用的是旧版 Mapbox 边界数据（如 V2 或 V3），我们强烈建议你升级到 v4。除了获取边界本身的更新外，V4 版本还更可靠地支持在提示框中显示区域名称，并支持自动缩放以适应基于 Mapbox 边界的地图图层，以及其他改进。

从 v3 迁移到 v4 时，部分边界存在破坏性变更。在继续操作之前，请查阅《破坏性变更的详细更新日志》 ↗。

要迁移数据管道中的数据，你可以：

  • 利用《V3 到 V4 的 ID 映射表》 ↗，该表包含在《Mapbox 边界定义 V4 产品》中。使用 V3 的 feature_id 列将其与现有的边界数据进行连接，并引入 v4 新增的 mapbox_id 列。确保手动处理上述提到的任何破坏性变更。
  • 或者，你可以直接按照上述说明操作，使用边界元数据 CSV 文件从头连接 mapbox_id 列。

将 mapbox_id 列添加到数据管道/本体后，请重新配置下游应用以利用 v4 边界，并选择此新列及相应的边界类型/层级。
