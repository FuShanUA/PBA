数据支撑字段

Foundry 表单不再是 Foundry 中数据输入或写回工作流的推荐方法。相反，使用 Foundry 本体论构建用户输入工作流，将相关数据结构表示为对象类型，并配置与操作的写回交互。在 表单概述 文档中了解更多。

数据支撑字段将表单与已存在于 Foundry 中的数据关联起来。此页面讨论了 Foundry 表单中可用的七种数据支撑字段。

用户下拉框

用户下拉框 字段允许受访者选择一个 Foundry 用户。该字段显示用户名称，但记录的是用户 ID。用户可以配置以下选项：

  • 允许 多选。
  • 如果只有一个值可用，则预填该字段。
  • 设置一个 Placeholder。
  • 在下拉框下方隐藏所选用户的列表。
  • 使用 代码编辑器 来设置 onlyUsersInGroupIds: list<string>。

对象下拉菜单

对象下拉菜单 字段允许受访者选择给定类型的对象。此字段显示对象的标题，但记录其主键。用户可以配置以下选项：

  • 根据属性的值指定过滤条件。
  • 指定应搜索哪些属性。
  • 允许 多选。
  • 如果只有一个值可用，则预填该字段。
  • 显示悬停对象属性的预览。
  • 隐藏对象类型图标。
  • 设置一个 Placeholder。
  • 使用 代码编辑器 来设置 noResultsText: string 和 displayedProperties: list<string>。

要进行过滤，属性必须在本体论（Ontology）中配置“低基数”渲染提示。

对象属性下拉菜单

对象属性下拉菜单（object property dropdown）字段允许受访者选择给定对象类型的属性值。用户可以配置以下选项：

  • 基于另一个属性的值指定过滤条件。
  • 允许 多选。
  • 如果只有一个可用值，则预填充该字段。
  • 允许创建除给定值以外的值。
  • 设置一个 Placeholder。
  • 使用 代码编辑器（Code Editor） 来设置 noResultsText: string。

数据集值下拉菜单

数据集值下拉菜单（dataset value dropdown） 字段允许受访者选择给定数据集中的列值。用户可以配置以下选项：

  • 根据另一列的值指定筛选条件。
  • 允许 多选。
  • 如果只有一个值可用，则预填该字段。
  • 允许创建给定值以外的值。
  • 设置 Placeholder。
  • 使用 代码编辑器 来设置 noResultsText: string。

要使用此字段，数据集必须被 索引到Lime中。

对象提供者

对象提供者 字段是只读的，并显示指定类型的选定对象。用户可以配置以下选项：

  • 根据属性的值指定筛选条件。
  • 显示摘要视图或所有对象的可折叠列表。
  • 使用 代码编辑器 来设置 maxObjects: integer。

对象属性显示

对象属性显示（Object property display） 字段是只读的，显示所选对象的属性。用户可以配置以下选项：

  • 指定对象的主键或引用它的字段。
  • 隐藏对象类型图标。
  • 自定义以下情况下的消息：
    • 未选择对象。
    • 值为空。
    • 请求失败。
    • 对象“缺失”（不存在或受访者没有适当的权限）。
  • 指定以下情况下的操作：
    • 请求失败。
    • 对象“缺失”。

如果引用了一个启用了多选的字段，对象属性显示将显示所有对象的属性，用逗号分隔。

用户属性

用户属性（user properties） 字段是只读的，显示所选Foundry用户的选定属性。用户可以配置以下选项：

  • 指定用户ID或引用它的字段。
  • 指定格式（例如，Last name, first name）。
  • 自定义消息情况如下：
    • 未选择用户。
    • 值为空。
    • 请求失败。
    • 用户“缺失”（既不存在，或者受访者没有相应的权限）。
  • 指定在以下情况下的操作：
    • 请求失败。
    • 用户“缺失”。

如果引用的是启用了多选的字段，则用户属性显示将显示所有用户的属性，并用逗号分隔。
