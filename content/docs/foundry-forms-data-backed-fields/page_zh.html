数据支撑字段

Foundry Forms 不再是 Foundry 上进行数据录入或回写工作流的推荐方案。请改用 Foundry Ontology 构建用户输入工作流，将相关数据结构表示为对象类型，并通过 Actions 配置回写交互。详情请参阅《表单概览》文档。

数据支撑字段用于将表单链接到 Foundry 中已存在的数据。本页将讨论 Foundry Forms 中可用的七种数据支撑字段。

用户下拉列表

用户下拉列表字段允许填写者选择一名 Foundry 用户。该字段会显示用户名，但记录的是用户 ID。用户可以配置以下选项：

  • 允许多选。
  • 当仅有一个可用值时，自动预填该字段。
  • 设置 Placeholder。
  • 隐藏下拉列表下方已选用户列表。
  • 使用代码编辑器设置 onlyUsersInGroupIds: list<string>。

对象下拉列表

对象下拉列表字段允许填写者选择指定类型的对象。该字段会显示对象标题，但记录其主键。用户可以配置以下选项：

  • 基于属性值指定过滤器。
  • 指定要搜索的属性。
  • 允许多选。
  • 当仅有一个可用值时，自动预填该字段。
  • 显示鼠标悬停对象的属性预览。
  • 隐藏对象类型图标。
  • 设置 Placeholder。
  • 使用代码编辑器设置 noResultsText: string 和 displayedProperties: list<string>。

若要作为过滤条件，必须在 Ontology 中将属性配置为“低基数”渲染提示。

对象属性下拉列表

对象属性下拉列表字段允许填写者选择指定对象类型的属性值。用户可以配置以下选项：

  • 基于另一个属性的值指定过滤器。
  • 允许多选。
  • 当仅有一个可用值时，自动预填该字段。
  • 允许创建给定选项以外的值。
  • 设置 Placeholder。
  • 使用代码编辑器设置 noResultsText: string。

数据集值下拉列表

数据集值下拉列表字段允许填写者选择指定数据集的列值。用户可以配置以下选项：

  • 基于另一列的值指定过滤器。
  • 允许多选。
  • 当仅有一个可用值时，自动预填该字段。
  • 允许创建给定选项以外的值。
  • 设置 Placeholder。
  • 使用代码编辑器设置 noResultsText: string。

要使用此字段，必须将数据集索引到 Lime。

对象提供程序

对象提供程序字段为只读状态，用于显示指定类型的已选对象。用户可以配置以下选项：

  • 基于属性值指定过滤器。
  • 显示所有对象的摘要视图或可折叠列表。
  • 使用代码编辑器设置 maxObjects: integer。

对象属性显示

对象属性显示字段为只读状态，用于显示所选对象的指定属性。用户可以配置以下选项：

  • 指定对象的主键或引用该主键的字段。
  • 隐藏对象类型图标。
  • 自定义以下情况的消息：
    • 未选择任何对象时。
    • 值为空时。
    • 请求失败时。
    • 对象“缺失”时（不存在，或填写者没有相应权限）。
  • 指定以下情况的处理方式：
    • 请求失败时。
    • 对象“缺失”时。

如果引用启用了多选的字段，对象属性展示将显示所有对象的属性，以逗号分隔。

用户属性

用户属性字段为只读，会显示所选 Foundry 用户的指定属性。用户可以配置以下选项：

  • 指定用户 ID 或引用该 ID 的字段。
  • 指定格式（例如 姓，名）。
  • 在以下情况自定义提示信息：
    • 未选择用户时。
    • 值为空时。
    • 请求失败时。
    • 用户“缺失”时（即用户不存在，或受访者没有相应权限）。
  • 指定在以下情况采取的操作：
    • 请求失败时。
    • 用户“缺失”时。

如果引用启用了多选的字段，用户属性展示将显示所有用户的属性，以逗号分隔。
