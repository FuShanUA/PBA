快速搜索

快速搜索是 Palantir 平台 中用于快速搜索、导航和发现内容的工具。在左侧边栏点击 搜索 图标，或使用 Cmd+J (macOS) 或 Ctrl+J (Windows) 即可访问快速搜索。快速搜索包含两种视图模式：

  1. 直达模式： 提供一份个性化的简短结果列表，引导用户直接访问主要类型的可用内容：平台应用程序、自定义应用程序、对象、数据集及其他资源。
  2. 完整结果视图： 旨在帮助用户查找内容并发现平台中的已有资源。用户可以使用高级过滤器搜索 Palantir 应用程序、对象、数据集及其他文件，并找到最相关的结果及丰富的元数据。

直达模式

  • 易于访问： 若要打开快速搜索，请在导航侧边栏选择 搜索...，或使用快捷键 Cmd+J (macOS) 或 Ctrl+J (Windows)。
  • 聚焦导航： 初始对话框和下拉菜单以导航为导向。输入内容即可获取建议。使用键盘可跳转至平台中的任意页面。
  • 仅搜索标题： 直达模式仅搜索应用程序、资源和对象的标题。如需搜索其他字段和元数据，请使用完整结果视图。
  • 一键查看完整结果： 在对话框和下拉菜单中，按 Enter/Return 键或按 应用程序、对象、数据集 或 文件 进行过滤，即可切换至高级搜索模式。
  • 个性化结果： 快速搜索中的结果会根据您最近访问或收藏的资源等进行个性化推荐。

完整结果视图

  • 聚焦发现： 专为搜索和发现而设计，用户可以对比多个搜索结果，找到对其最有用的资源。
  • 多字段搜索： 支持搜索描述、列名、文件路径等内容。此外，还支持按创建者、标签、项目、文件夹等进行过滤。
  • 选项卡： 界面包含一个“热门结果”选项卡以及四个用于过滤的选项卡：应用程序、对象、数据集 或 文件。
  • 元数据： 用户可以查看每个结果的相关信息，以评估该资源是否与自身相关。支持按关键词高亮、文件路径、查看次数、最后更新时间、关键安全元数据等进行搜索。
  • 排序： 结果基于包含文本匹配及其他加权参数的算法进行排序。
  • 过滤器： 提供过滤器用于高级搜索。此外，通过在初始搜索下拉菜单中选择过滤器选项卡，用户可以执行复杂的过滤搜索（例如，“项目 X 下由我创建的所有报告”）。
  • 权限： 快速搜索严格遵守平台中的所有现有权限。对于具有 发现 权限的内容，会弹出 申请访问权限 提示。用户无法看到其无权访问的内容。

快速搜索 不会 搜索平台中的所有对象实例。快速搜索仅限于搜索 250 种对象类型上的实例，优先级依次为 活跃 且状态为 重要 的对象类型，其次是 正常，最后是 实验性 状态（已弃用和隐藏的对象类型不在搜索范围内）。如果用户找不到所需内容，系统会提示其前往 对象浏览器 中尝试搜索，在此可以将搜索范围调整为特定的对象类型组，甚至是特定的对象类型。

过滤器

通过快速搜索中的过滤器，您可以快速找到平台内所需的资源和数据。从 应用程序、对象、数据集 或 文件 这些结果类型中进行选择，即可过滤搜索结果。您还可以通过在文件路径、标签或项目内进行搜索，进一步细化过滤条件。应用过滤器后，搜索结果视图将仅显示您所选的结果类型。

举例来说，我们希望在注册信息中搜索“汽车”相关的资源。

应用程序

若要搜索模块、工作区或通过 Palantir 应用构建界面创建的其他应用程序，您可以使用 应用程序 进行过滤。

在我们的“汽车”搜索示例中，可以看到三个 Workshop 模块、一个 Carbon 工作区和一个 Slate 文档，它们均被正确识别为 Palantir 中构建的应用程序。

对象

如果只想查看基于本体对象类型的结果，请按对象进行筛选。在此搜索结果视图中，您可以从与搜索词匹配的对象类型中进行选择，对对象类型进行编辑或查看其血缘关系，并进一步下钻到单个对象。

在搜索“汽车”时，我们会看到多个对象类型和对象结果，并且能够进一步筛选为 Award 或 Auto Parts 等特定对象类型。

数据集

按数据集筛选时，结果视图将显示平台中可用的数据集列表，这些数据集的名称或列名与搜索词相匹配。

沿用“汽车”这个示例，我们可以看到一个包含 38 个数据集的列表，其数据集名称和列名中均包含匹配的词条。

文件

使用快速搜索中的文件选项卡，可快速查找已添加到平台或在平台中创建的单个资源文件。利用左侧面板中的筛选器，可以专门搜索特定的文件或资源类型。

在搜索“汽车”时，我们会得到多个文件结果，包括文件夹、图谱、图像和建模目标。
