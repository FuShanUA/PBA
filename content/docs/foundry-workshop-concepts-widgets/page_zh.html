组件

组件向 Workshop 用户展示内容，是模块用户界面的核心构建块。常用组件包括筛选器列表、对象表格和按钮组组件。模块构建者在 Workshop 中构建应用程序时，会将组件放入模块的布局中，并根据其用例需求配置所需的输入和输出变量、显示选项及操作。

要向模块添加组件，请将鼠标悬停在任何空白区域以显示+ 添加组件按钮，然后将其选中。您也可以选择某个区域或模块标题中的+按钮。随后，在弹出的组件选择器对话框中选择所需的组件。

组件配置

将组件添加到 Workshop 模块后，选中该组件即可在屏幕右侧查看其配置选项。以下示例将通过展示如何配置筛选器列表组件，来讲解 Workshop 组件的核心通用概念。

组件设置

组件的核心配置选项位于组件设置选项卡中。模块构建者在此处配置组件的输入和输出变量（即初始填充到组件中的数据，以及在适用情况下组件随后生成和输出的数据），以及其他附加配置和显示选项。

在上方所示的筛选器列表组件示例中，核心配置选项如下：

  • 对象集：这是筛选器列表组件的输入变量，决定了可用的筛选选项。此配置既支持采用在该 Workshop 模块其他位置定义的现有对象集变量，也支持创建新的对象集变量。
  • 筛选器列表：用于决定组件中显示的筛选选项。一旦对象集完成填充，此配置选项将展示更多细节；随后它会显示初始对象集内的属性类型，并允许按关键字在所有属性中进行搜索，以及根据关联对象的存在与否进行筛选。
  • 允许用户添加和移除筛选器：该选项为组件添加了一个用户界面，使模块用户能够添加和移除所显示的筛选器。
  • 筛选器输出：该选项是一个作为组件输出的对象集筛选器变量。此筛选器可应用于另一个对象集变量，以对指定变量内的对象进行筛选。默认情况下，将筛选器列表组件添加到模块时会创建一个输出变量。

元数据

元数据选项卡支持重命名组件，并允许访问组件的原始 JSON 配置。

修改组件名称选项会影响 Workshop 中对当前组件的引用方式，最显著的影响体现在作为布局面板中的组件时，以及默认变量名中。

原始组件配置展示了当前组件设置以 JSON 格式存储的形式，为高级模块构建者提供了快速查看、编辑或复制原始格式配置的选项。

显示

显示选项卡支持配置当前组件的尺寸，并允许模块构建者在自动（最大）、绝对和弹性尺寸之间进行切换。

  • 自动（最大）：根据组件内容自动调整其高度，同时设定最大高度。在列布局中，该选项不可用于设置组件的宽度。
  • 绝对：为组件指定固定的像素尺寸。
  • Flex：指定当前模块中此组件的高度或宽度与其他组件的比例，以便在屏幕尺寸或分辨率发生变化时，组件能够进行适当缩放。

复用组件

完成组件配置后，您可以将其复制到模块内的任意位置进行复用。此功能在重新创建复杂的组件配置时尤为实用；但是，如果组件配置后续会频繁变更，采用嵌入式 Workshop 模块组件可能更为合适。要复制组件以供复用，请选中该组件并使用 Cmd+C (macOS) 或 Ctrl+C (Windows)。接着，使用 Cmd+V (macOS) 或 Ctrl+V (Windows) 将组件粘贴到左侧面板 布局 部分底部的 未使用的组件 区域。通过选择 + 添加组件 将该组件加入您的模块，随后即可在组件选择器弹窗的 未使用的组件 标签页中找到它。

此外，您也可以借助复制分区中的控制选项来实现组件的复制、剪切和粘贴。

输入和输出变量

输入和输出变量用于控制数据在 Workshop 模块内部的流转方式。

输入变量定义了传入特定组件的数据，这些数据通常随后会在组件中进行展示。对象集输入便是一个输入变量的示例，它会传递给对象表、筛选列表、图表和地图组件。

输出变量定义了从特定组件传出的数据，这些数据随后会被下游组件或其他变量消费。输出变量的示例包括对象表组件生成的“选中对象”对象集输出，或字符串下拉组件生成的“选中值”字符串输出。

上文的组件配置示例详细说明了如何为对象表组件配置输入变量（对象集）和输出变量（活动对象和选中对象）。有关变量的更多通用细节，请参阅《变量文档》。

组件类型

组件提供了丰富的用途，从信息可视化到支持在 Workshop 模块中触发事件和数据回写。以下分类对 Workshop 组件的功能进行了大致划分：

  • 核心展示组件
  • 可视化组件
  • 筛选组件
  • 事件触发与导航组件
  • 嵌入式 Foundry 应用
