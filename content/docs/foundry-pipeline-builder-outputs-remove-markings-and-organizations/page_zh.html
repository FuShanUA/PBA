从输出中移除标记和组织

平台资源的访问权限由标记和组织控制。标记采用全有或全无的方式限制访问：用户必须拥有应用于资源的所有标记的成员身份才能进行访问。此外，标记会通过文件层级和直接依赖关系继承。另一方面，对于组织，用户必须是应用于项目的至少一个组织的成员或访客成员，才能满足访问要求。组织通过文件层级和直接依赖关系继承。

如果你对特定标记拥有 Remove marking 权限，现在可以在 Pipeline Builder 中从输出移除该继承的标记。这等同于代码仓库中的 stop_propagation 参数。

在输出上移除标记等同于停止从输入传播标记。

前提条件

在使用 Pipeline Builder 移除标记或组织之前，必须完成以下步骤。

启用分支保护

  1. 在 Pipeline Builder 中，选择 Settings，然后选择 Manage branches。

  1. 选择 Branch protection 选项卡。

要求代码审批

  1. 在 Branch protection 选项卡中，勾选 Require proposals to update protected branches 和 Require approval before merging 复选框。
  2. 指定所需的审批策略。下方展示了一个审批策略示例。

在管道设置中启用对安全标记的更改

导航至 Security approvals 选项卡，勾选 Allow changes to security markings in this pipeline 旁边的复选框。你必须拥有该管道的 Owner 角色才能完成此步骤。

一旦你启用了 Allow changes to security markings in this pipeline，便无法禁用分支保护或移除代码审批要求。你必须禁用 Allow changes to security markings in this pipeline 才能禁用这些功能。

一旦你在受保护分支中移除了标记，便无法从 Security approvals 选项卡禁用 Allow changes to security markings in this pipeline。你必须先撤销对该标记的移除操作，才能禁用此设置。

移除标记或组织

  1. 从受保护分支创建分支。

  2. 导航至屏幕右侧的 Pipeline outputs，将鼠标悬停在要移除标记的输出上。然后，选择 Edit。

  1. 选择输出数据集下的 Configure markings 下拉菜单。

  1. 移除标记：在弹出菜单的 Markings 选项卡下，选择要移除的标记旁边的红色移除图标。

移除的标记现在将显示在对话框的 Markings removed 部分下。

  1. 要移除组织，请选择 Organizations 选项卡。要彻底移除组织标记，必须移除包含所需目标组织的所有输入。如果要移除所有输入中的所有组织，请选择 Remove all inputs。

输出会继承其所在项目的组织。若需移除当前项目上已有的组织，请将输出移动至单独的项目中。

  1. 选择 Apply。此时，在输出看板的左上角会出现一个带有负数的盾牌图标，表示你要移除的标记和组织数量。

对输出上的标记和组织所作的更改，在分支成功合并并部署至受保护分支前不会生效。如果你尝试在当前分支上构建数据集，它仍会显示原有的标记。

组织移除操作仅影响移除时输入中已存在的组织。如果与该输入关联的组织自上次移除后发生了变化，系统将显示一个警告图标，提示该输入的组织自上次移除操作以来已被修改。

提交更改

  1. 为使管道输出上的标记和组织更改生效，请创建一个提案，将更改合并至受保护分支。该提案将包含一个用于批准移除标记的区块，其功能类似于管道代码审批。

你必须拥有 Remove marking 权限才能批准更改。移除标记提案的审批者无需是管道数据主人，仅需对该提案具有 View 访问权限。

每个被移除的标记或组织都需要单独审查，这意味着一个提案中可能包含多项审查。当你批准移除某个标记时，你的批准将适用于你有权审查的所有标记。

一旦获得所有必要的批准，提案即可合并。部署该版本后，标记移除操作即会生效。

撤销从管道输出中移除标记或组织

  1. 要撤销移除标记，请导航至屏幕右侧的 Pipeline outputs，将鼠标悬停在已移除标记的输出上。

  2. 选择 Edit。

  1. 选择输出数据集下方的 Configure markings 下拉菜单。

  1. 对于标记：在弹出的对话框中，选择 Markings not propagated 旁边的撤销图标。

  1. 对于组织，在与该组织关联的输入旁边选择撤销图标。

  1. 选择 Apply，然后从屏幕右上角保存你的管道。

  1. 提交你的更改以开始审批检查。

  2. 一旦获批，即可部署你的管道。

撤销移除标记无需提升权限，这与移除标记所需的权限不同。

标记与作业组

在作业组中，所有输入的标记将被同一作业组内的所有输出继承。要查看示例并详细了解作业组，请查阅我们的文档。

标记与多个受保护分支

若任一分支上存在移除标记的操作，在保护或取消保护分支之前，必须停止移除流水线中所有分支的标记。多个分支受保护时，移除标记的操作将作用于所有受保护分支。

启用安全审批设置后，将无法更改分支保护设置，包括保护或取消保护分支。
