AIP架构

如在AIP、Foundry和Apollo的概述中所述，AIP是Palantir用于将生成式AI整合到操作领域的平台。虽然AIP和Foundry作为共享服务网格的一部分协同工作，由Apollo支持，并部署在Rubix中，但本页面从以AI为中心的视角提供了AIP端到端架构的视图。这包括安全连接到所有范围内的大型语言模型的能力，持续将上下文集成到本体论中，构建代理和自动化，观察和评估已部署代理的持续性能，以及管理AI驱动产品的完整开发工具链。

AIP架构可以概括为12个一般能力类别：

  1. 安全的LLM集成与访问：通过Palantir管理的基础设施，实现对全部商业LLM（例如GPT、Gemini、Claude、Grok模型）和开源模型（例如Llama）的安全访问，确保传输的数据不会被第三方提供商保留，也不会被模型提供商用于重新训练。企业还可以集成其现有的模型，无论是现有的模型订阅、微调后的模型，还是领域特定的模型。
  2. 端到端的可观测性：为AI驱动的工作流和代理流程的每一步提供监控工具。这包括针对所有输入本体论的数据流的细粒度监控，对用户或AI代理所执行的每个操作的日志记录，以及在工作流中追踪链式执行的级联过程。这种可观测性还扩展到令牌消耗和其他资源使用方面的内容。
  3. 上下文工程：为开发者提供无代码、低代码和专业代码工具，用于整合支撑本体论及所有依赖工作流的上下文数据、逻辑和操作。可以通过任何运行时（例如Spark、Flink、DataFusion、Polars）利用所有类型的数据集成（例如批量处理、流处理、通过CDC的实时复制），同时遵守统一的安全性、治理、溯源跟踪和其他关键保证。
  4. 本体论系统： 通过将分散的数据、逻辑、操作和安全集成到企业决策的统一表示中，激活上下文。 了解更多关于本体论的信息。
    • 本体论的语言将操作流程中的“名词”和“动词”建模为人类和代理都可以理解的形式。
    • 本体论的引擎支持对数十亿个对象进行查询，协调数以万计的操作，并持续融入基于反馈的学习。
    • 本体论的工具链使开发人员能够在共同的基础上构建多样化且复杂的AI驱动应用程序。
  5. 向量、计算、工具服务： 提供生产并管理嵌入所需的集成向量化服务；一个可扩展的计算框架，可以利用多节点引擎（如 Spark、Flink）、高效的单节点引擎（如 DuckDB、Polars）以及任何容器化的“BYO”引擎；以及一组与本体论系统协同工作的工具服务，作为不断演进的工具工厂。该平台在模型、计算引擎和接口方面设计为模块化和可扩展的。
  6. 安全与治理： 确保人类和代理执行的每一项操作都遵循严格的基于角色、标记和目的的控制。这需要基础设施、平台和企业级安全控制的结合。这些控制可以进行细粒度配置并动态查询，并在丰富的审计日志中进行分类。治理功能在平台界面内以及通过API/SDK进行程序化操作的所有操作性、工程性和开发者活动上统一扩展。
  7. 代理生命周期： 为生产环境中的代理的互联构建、编排和评估流程提供动力。代理可以使用无代码、低代码和专业代码工作区进行构建。可以通过低代码接口如AIP Logic，或专业代码接口如Code Workspaces来配置和管理持久化的编排。集成的评估框架（AIP Evals）与本体论无缝协作，使您能够创建测试用例、调试和迭代代理定义、在不同LLM之间比较性能、检查执行间的差异等。
  8. 操作自动化： 促进工作流程内部和跨工作流程所需的各类自动化模式。这包括可扩展的基于计划的自动化、处理流数据的接近实时事件驱动的自动化，以及与基于API的操作紧密集成的自动化。无论采用哪种方式，每种自动化都可以利用本体论系统中丰富的数据、逻辑和操作基本要素，以及广泛执行和通知配置选项。
  9. 开发环境： 赋能开发者按照他们想要的方式和地点构建代理和自动化。AIP 提供集成开发环境（例如 VS Code、JupyterLab），这些环境与基于本体论的应用程序提供无缝连接，并集成了测试和评估框架。同样地，平台 SDK ↗ 和 本体论 SDK，结合 Palantir 的 VS Code 插件，将相同的核心功能带入现有的环境和开发者工具链中。此外，Palantir MCP 提供了一个安全接口用于代理开发（类似于在平台上使用 AI FDE 所能实现的功能）。
  10. 人机+AI应用： 提供全方位的AI驱动体验；从面向对象的分析，到实时应用构建，到多模态治理工作流，再到核心平台功能的管理。操作用户、合规团队、工程师、分析师和其他关键角色都有开箱即用的应用程序，专门针对其工作流程进行了定制，同时还具备快速构建新的本体论驱动应用程序的能力。在每种情况下，AI的融合都可以被仔细控制并透明评估，确保从增强到自动化的平稳过渡。
  11. 打包、发布、部署： 允许开发人员超越点分析和应用程序，构建充分利用集成DevOps工具链的完整功能AI产品。可以将数据管道、本体论定义、自动化和预构建应用程序等端到端集合打包、发布并在异构目标环境中部署。开发人员可以指定“最后一英里”定制的权限，下游团队可以安全地接收更新，随着产品定义的演变以及更改在各个发布渠道中得到验证和推广。
  12. 企业自动化：使所有背景和角色的构建者能够使用专门的AI代理（例如，AI FDE，AIP 分析师），构建数据管道、编写业务逻辑、训练模型、构建本体论、生成分析并开发端到端的应用程序。关键的是，这些代理与人类用户在同一基础之上运行，这意味着它们遵循相同的集成变更管理功能（例如，全局分支），并且可以无缝地将人机协作的工作流程与完全自主的操作结合在一起。
