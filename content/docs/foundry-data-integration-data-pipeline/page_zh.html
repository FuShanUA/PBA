什么是数据管道？

Foundry 中数据集成的总体目标是提供组织内部客观现实的数字化视图。要实现这一目标，通常需要同步多个源系统的数据，应用统一的模式，将数据集整合在一起，并使团队能够在通用数据基础之上构建用例。

在此背景下，“数据管道”一词被广泛使用，指代数据从源系统流经中间数据集，最终产出高质量、精心治理的数据集的过程；这些数据集可以结构化为本体（Ontology），或作为机器学习和分析工作流的基础。

尽管在 Foundry 中，通过转换逻辑连接在一起的任意两个数据集都可以被视为一个管道，但在实际应用中，我们所说的“数据管道”范围更窄。通常，管道具有数据权属的概念——即由某个人或团队负责监督该管道，确保数据定期、可靠地流经管道，以支撑业务流程。

除了数据权属的概念外，高质量的、可用于生产环境的数据管道还有其他几个特征。我们将在本文档的后续部分探讨这些理念，并提供相关资源的链接以供深入了解：

除了所有管道共有的特征外，还应根据数据规模、延迟要求和维护复杂度等因素，考虑为您的数据基础创建哪种类型的管道。Foundry 提供三种主要类型的管道：批处理、增量和流式处理。了解更多关于管道类型的信息。

管道设置

Foundry 的管道构建器（Pipeline Builder）让用户能够通过简化的点击式界面，快速、轻松地设置管道。借助管道构建器，用户可以享受 Git 风格的变更管理、数据健康检查、多模态安全以及细粒度数据审计等诸多益处。

技术人员如今可以比以往更快速地构建和维护管道，将精力集中在端到端管道和期望输出的声明式描述上。此外，管道构建器基于表单的点击式界面，使非技术背景的用户也能通过简化的方式创建管道。

构建调度

简而言之，一系列数据转换必须定期运行，才能称之为数据管道。在 Foundry 中定义构建调度是构建管道的基本步骤，因为下游数据消费者期望数据能够定期更新。数据流经管道的频率取决于组织需求：有些管道可能仅按周或按日运行，而有些则按小时甚至更高频率运行。

数据质量

在定义管道的初始阶段，我们建议在每个步骤频繁检查输入和输出的质量。从源系统同步的数据通常包含未定义的值，以及格式不规范或不一致的数据。清洗和规范化数据是管道构建过程的核心环节。

Foundry 中提供了多种工具，用于检验对数据集的假设：

  • 数据集预览支持对数据集的任意列计算统计信息，并支持筛选部分行以快速验证预期。
  • 代码仓库对转换调试的支持，可用于在编写转换逻辑时检查输入数据集的结构是否符合预期。
  • Foundry 分析套件中的应用，尤其是 Contour，能够以点击的方式验证对数据集的假设，十分实用。

管道建立后，健康检查是验证数据长期保持高质量的首选方法。以下是一些帮助您入门健康检查的资源：

安全与治理

Foundry 的平台安全原语提供了一流的能力，既能保障数据底座的安全，又能确保敏感数据得到妥善处理。其中，《项目》与《标记》这两大横切概念，分别支撑自主访问控制与强制访问控制，足以满足各种维度的数据治理要求。
