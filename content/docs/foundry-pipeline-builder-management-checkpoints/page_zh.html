检查点

在构建管道时，您通常会在多个输出之间使用共享的转换节点。此逻辑通常针对每个输出重新计算一次。借助 Pipeline Builder 中的检查点功能，您可以将转换节点标记为“检查点”，以便在下一次构建期间保存中间结果。该检查点节点上游的逻辑将针对其所有共享输出仅计算一次，从而节省计算资源并缩短构建时间。

请注意，检查点有时会导致性能下降。例如，将数据写入磁盘然后再次读取的成本，可能低于多次重新计算逻辑的成本。此外，将查询拆分为多个部分可能会减少可对管道执行的优化数量。

输出必须位于同一个作业组中，检查点节点才能提高管道效率。详细了解Pipeline Builder 中的作业组。

添加检查点节点

以下是一个生成两个输出的管道示例：Attachment 和 Request。转换节点 Checkpoint 在这两个输出之间共享。在当前状态下，逻辑节点 Clean 和 Checkpoint 将被计算两次，每个输出各计算一次。

但是，我们希望针对这两个输出仅计算一次 Clean 和 Checkpoint。为此，请右键点击 Checkpoint 并选择标记为检查点。

检查点策略

有三种可用的检查点策略：

  • 保存至短期磁盘 [默认]：将每个输出临时存储在磁盘上。这是标记为检查点的节点的默认策略。
  • 保存至内存 [最快]：将结果临时存储在内存中以便更快访问；这对于小型数据集特别有用。请注意，由于内存限制，此选项对于大型数据集可能会失败。
  • 保存为输出数据：将检查点的最新事务保存在专用数据集中。该数据集在首次部署检查点时创建。

快速管道不支持保存至内存检查点策略。

默认情况下，标记为检查点的节点将对每个输出使用保存至短期磁盘策略。要配置此项，请选择检查点，然后选择配置策略。

当转换具有多个输出时（例如，如果设置了检查点的节点是拆分转换），可以选择为每个输出选择一种检查点策略。

对于只有一个输出的节点，您必须选择一种检查点策略或取消该节点的检查点标记。

现在，Checkpoint 节点的顶部角落将出现一个浅蓝色徽章。

现在，将两个输出添加到同一个作业组以验证检查点行为。右键点击其中一个输出（Request）以分配作业组。选择新建组以打开构建设置面板。

由于默认情况下数据集位于不同的作业组中，因此检查点将针对每个输出重新计算，从而抵消任何好处。要解决此问题，请通过选择另一个输出（Attachment），然后选择面板底部的添加到组...，将其添加到同一个作业组中。

检查点存储成本

对于 保存至短期磁盘 检查点策略，检查点会将转换的全量结果推送到存储系统，例如 Hadoop 分布式文件系统（HDFS）。举例来说，如果你使用该策略对连接操作设置检查点，连接的完整结果将全部输出到存储中。这样一来，即使最终的数据集输出量很小，也可能会导致存储大量数据。
