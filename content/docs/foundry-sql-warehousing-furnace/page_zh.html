Furnace

Furnace 是 Foundry 的现代 SQL 数据仓库引擎，专为灵活性、性能和可扩展性而设计。Furnace 让您可以在数据上运行强大且灵活的 SQL 查询，使用最新的行业标准进行计算和互操作性。通过 Furnace，您可以在 Foundry 计算之上运行读写 SQL，既可以在 Foundry 内部原生运行，也可以从您选择的第三方分析工具中运行。Furnace 从第一原理出发进行设计，支持多种计算引擎，使用统一的语法，为您的 SQL 工作负载提供未来保障和弹性。

Furnace 架构

Furnace 的架构将查询解析和规划与查询执行分离。这种分离是 Furnace 灵活性和可扩展性的关键：

  • 查询解析与规划： 所有 SQL 查询都使用统一的无引擎依赖层进行解析和规划。该层利用 Apache Calcite 将 SQL 解析为抽象语法树（AST），验证它，并优化查询，无论最终的计算引擎是什么。
  • 计算引擎抽象: 在规划之后，Furnace 确定哪个计算引擎应该执行查询。计算引擎负责运行查询、读取数据并返回结果。
  • 引擎选择: 默认情况下，Furnace 根据工作负载、兼容性和性能，智能地为每个查询选择最佳的可用引擎。引擎选择是自动的，不需要用户进行配置。
  • 统一的用户体验: 无论使用哪个计算引擎，或者查询入口是在 Foundry 平台内部还是外部，用户都通过相同的 API 和接口（包括 Arrow Flight SQL）与 Furnace 交互，并受益于对现代表格式（如 Iceberg）的一致支持。

统一的 SQL 体验

Furnace 提供了跨所有连接方式的统一 SQL 体验：

  • 平台内查询 通过 SQL Studio 和 SQL Console
  • 外部 BI 工具 通过现有的 ODBC/JDBC 驱动程序
  • 现代集成 通过 Arrow Flight SQL

无论您通过何种方式连接到 Foundry，所有连接方式都能受益于 Furnace 的智能引擎选择和性能优化。
