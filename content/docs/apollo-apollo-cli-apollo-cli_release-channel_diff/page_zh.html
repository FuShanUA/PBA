apollo-cli 发布渠道差异 [实验性]

该命令为 实验性。要 启用此命令，请运行 apollo-cli configure 命令并启用 V2 实验性命令。

比较环境中产品在两个发布渠道之间的版本

比较两个发布渠道，并显示在源渠道上存在但不在目标渠道上的产品版本，仅限于特定环境中使用的产品。

这对于识别需要从一个渠道推广到另一个渠道的版本很有用。

示例用法：

比较环境中的 DEV 渠道与 STABLE 渠道

apollo-cli release-channel diff -e apollo-env-123 -s DEV -t STABLE

输出为 JSON

apollo-cli 发布渠道差异 -e apollo-env-123 -s DEV -t STABLE -o json

显示所有产品，包括已经更新到最新版本的产品

apollo-cli 发布渠道差异 -e apollo-env-123 -s DEV -t STABLE --all

标志用途

-s, --source-channel要比较的源发布渠道
-t, --target-channel要对比的目标发布渠道

从父命令继承的标志

标志用途
--apollo-client-id用于生成Bearer Token的客户端ID
--apollo-client-secret用于生成Bearer Token的客户端密钥
--apollo-token用于认证端点的Bearer Token
--apollo-token-provider指定用于认证Apollo端点调用的Bearer Token的提供方式。有效值为 "auto"、"static"、"service-user" 或 "sso"（默认为 "auto"）。如果指定 "auto"，则根据配置选择模式：如果设置了 "apollo-token"，则使用 "static"；如果同时设置了 "apollo-client-id" 和 "apollo-client-secret"，则使用 "service-user"；否则使用 "sso"（交互式浏览器登录）。如果同时设置了Bearer Token和（客户端ID或客户端密钥），将报错。如果指定 "static"，则使用 "apollo-token" 提供的令牌。如果指定 "service-user"，则使用 "apollo-client-id" 和 "apollo-client-secret" 从Multipass生成令牌。如果指定 "sso"，则使用针对Multipass的交互式浏览器SSO登录（支持硬件密钥如YubiKey）；生成的令牌将按配置文件缓存并静默刷新。当首次需要令牌且没有有效的缓存令牌时，会自动触发登录流程。
--apollo-urlApollo的基础URL，用于推导API端点
--debug启用调试级别日志记录
--http-timeout所有apollo请求的超时时间（分钟）
-k, --insecure-skip-verify跳过服务器证书验证
-o, --output输出格式（json,yaml,pretty）
--profile从配置文件中使用特定的配置文件
--quiet不将日志输出到标准错误
--space-id用于某些作用域为空间的命令的空间ID
