本地开发的安全考虑

确保在转换预览期间数据不离开平台的最佳方法是通过 VS Code 工作区 执行。采用这种方法，数据不会直接发送到您的机器。

当使用某些第三方工具、库和扩展时，可能更倾向于本地开发而不是 VS Code 工作区。当在本地开发中使用 Palantir 的 VS Code 扩展时，可以进行转换预览。然而，由于将部分数据集流式传输到用户机器的系统内存以本地执行预览的管道逻辑会带来数据安全风险，因此本地预览可能仅对部分用户启用。

执行本地预览的要求

此外，平台管理员必须在 控制面板 中的代码仓库设置页面启用本地预览功能。只有当两个条件都满足时，才能进行本地预览。这确保了不能通过本地预览来提升任何用户对数据集的权限。

本地预览期间的数据生命周期

在预览期间，不会将输入数据集存储在磁盘上。预览引擎使用 Foundry 的 S3 兼容 API 建立与 Foundry 的 S3 连接。这种实现方式导致每次本地预览都会在 审计日志 中生成事件，从而实现完全的可见性。

PySpark（用于旧版转换）或 Polars（用于轻量级转换）然后将数据集的所需部分流式传输到内存中。流式传输的数据子集由您选择的代码定义的输入过滤器和预览的管道逻辑决定。预览执行期间的所有缓存都位于系统内存中。

在预览调用结束时，最多 10000 行由转换写入的结果数据集会被存储在缓存文件夹中，以便在 VS Code 的预览表格中显示。您可以在 Palantir 扩展的设置中通过 Palantir > 缓存：路径 键来设置此缓存文件夹的位置。

结果缓存仅在必要的时间内保留，同时保持 Palantir 扩展的功能。结果缓存会被删除：

  • 在每次新的预览运行开始时。
  • 当 VS Code 正常退出时，例如当窗口的 关闭 按钮被选中时。
  • 在每次 VS Code 启动时。

此外，您可以通过在命令面板中调用 Delete all locally stored Python transforms preview results 命令手动删除缓存。也可以通过 palantir.transforms.datasets.cleanUp 标识符将该命令分配给快捷键。

风险评估

如上所述，安全流本地预览的数据驻留功能旨在防止意外和非故意的数据泄露。这些功能并非用于且无法阻止有意滥用。

例如，没有机制可以阻止用户在预览执行期间故意将数据写入其机器上的任何文件夹。拥有对数据集的 Download 操作权限的用户，在使用 Foundry 的公共数据访问端点以及执行本地预览时，应合理判断如何处理相关数据集。

有关本地预览的技术设置说明，请查看我们关于 在本地开发中预览转换 的文档。
