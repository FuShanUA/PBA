创建模型适配器

创建模型适配器主要有以下两种方法：

  • 通过独立的《模型适配器库》模板，适配器将作为 Python 库发布到您的 Foundry 环境中，供多个消费代码仓库使用。
  • 在发布模型时，将适配器类本地定义在相应资源中：
    • 在使用《模型训练模板》的代码仓库中定义。
    • 在 Jupyter® 代码工作区中，通过在 Notebook 之外单独定义一个 Python 文件来实现。

以下情况建议使用《模型适配器库》模板：

  1. 容器化模型
  2. 外部模型
  3. 需要创建可跨代码仓库共享的模型适配器

对于在 Foundry 中通过代码仓库中的 Python 转换创建的大多数模型，建议使用《模型训练》模板。通过将模型适配器的定义整合到转换代码仓库中，相比使用 Model Adapter Library 模板，您可以更快速地迭代适配器和训练逻辑。

在 Jupyter® 工作区训练模型时，必须将适配器定义在单独的 .py 文件中，然后将其导入到 Notebook 里。

下表对比了这两种方法，并提供了使用场景的指引：

《模型训练模板》Jupyter® 工作区《模型适配器库》
创作环境代码仓库Jupyter® notebook代码仓库
支持的模型类型Foundry 产出Foundry 产出Python 转换、容器、外部

模型适配器库模板

要创建容器化、外部或可复用的 Python 转换模型适配器，请打开《代码仓库应用程序》并选择《模型适配器库》。

《模型适配器库》在 src/model_adapter/ 目录下包含了一些示例实现。使用此模板生成的模型适配器具有可配置的包名，初始包名由代码仓库名称派生。您可以在隐藏的 gradle.properties 文件中查看和修改 condaPackageName 值（请注意，空格及其他特殊字符在已发布的库中将被替换为 -）。

与其他 Python 库代码仓库一样，对提交打标签后将发布新版本的库。这些已发布的库可导入到其他 Python 转换代码仓库中，用于模型训练和推理。

一个代码仓库可以发布多个模型适配器。任何额外的适配器实现文件必须作为引用添加到适配器模板的 build.gradle 隐藏文件中的模型适配器模块列表内。默认情况下，只会发布 model_adapter.example_adapter。

适配器的实现取决于所生成模型的类型。详情请参阅《外部模型适配器文档》，或查看《容器模型适配器示例》。

模型训练模板

要直接在 Foundry 中使用代码仓库创建模型，请打开《代码仓库应用程序》并选择《模型训练》模板。该代码仓库支持您编写《Python 数据转换》来训练模型并执行推理。

使用此模板生成的模型适配器无法自定义包名，也无法打标签。适配器将发布在名为 transforms-model-training-<repository rid> 的库中，版本号由 Git 提交记录派生。包的完整名称和版本可在 Palantir 模型页面上查看。必须将此库添加到下游代码仓库中，才能将模型作为 ModelInput 加载。如果这种方式不适合您的使用场景，模型适配器库模板允许您同时配置版本号和包名。

了解更多信息，请阅读有关《在 Foundry 中使用训练模板训练模型》的文档。

代码工作区

要在 Jupyter® 工作区中创建模型，请打开左侧边栏的模型选项卡，并选择添加模型 > 创建新模型。系统将以 .py 文件的形式生成一个模型适配器骨架，该文件以您为模型提供的别生命名。

在 Jupyter® 工作区中，适配器类必须定义在独立的 .py 文件中，并导入到笔记本内。不支持直接在笔记本单元格中定义适配器类。发布模型时，适配器及工作目录下的其他 Python 文件会打包成名为 cws_adapter 的库，版本号取自最新的 Git 标签。

如需了解更多，请阅读有关在 Jupyter® 笔记本中训练模型的文档。
