本体论 MCP（OMCP）的示例用例

以下各节描述了本体论 MCP（OMCP）的常见使用模式。

桌面代理集成

您可以将本体论 MCP 与桌面代理（如 Claude.ai、Microsoft Copilot Studio 和 Gemini Enterprise）一起使用。在这些用例中，本体论 MCP 为桌面代理增添了连接到您组织的权威数据源、读取和写入数据以及执行受您的本体论控制的操作的能力。

这种模式实现了人机共生：人类用户和 AI 代理都在相同的共享数据、逻辑、操作和治理上运行。例如，一名在 Microsoft Teams 中工作的用户可以要求 Copilot Studio 代理在本体论中创建一个任务，结果数据会立即对共享同一本体论的其他用户和应用程序可见。

外部代理工作流集成

使用 Google Agent Development Kit (ADK)、Microsoft Agent Framework 或 OpenAI SDK 等外部框架运行代理工作流的组织可以使用本体论 MCP 通过通用接口访问 Foundry 数据和逻辑。这些框架中的代理与本体论 MCP 的交互方式与与其他系统交互的方式相同，将 MCP 作为标准协议。

开发者可以组合多个 MCP 服务器以构建更丰富的代理功能。例如，代理可以使用 Perplexity MCP 进行网络搜索，同时使用本体论 MCP 读取和写入本体数据，从而实现跨越外部知识和您组织的真实数据源的工作流程。

无头代理

本体论 MCP 可以作为无头代理工作流的记忆和跟踪层。无头代理在没有直接人工监督的情况下运行，并且可以使用本体论 MCP 从组织的真实数据源中读取和写入数据。

由于通过Ontology MCP执行的每个操作都会记录在本体论中，这种模式为无监督操作提供了可审计性。您可以使用任何代理框架（如Anthropic SDK或Google ADK）来构建无头工作流，这些工作流会根据本体论的变化运行或按计划运行。
