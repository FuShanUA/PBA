在平台内固定 Spark 模块

代码仓库支持为特定仓库固定 Spark 模块，以此强制使用指定版本的 Spark。无论是新建构建还是现有构建，均可执行此固定操作。

建议始终使用最新版本的 Spark，以获取最前沿的性能与安全特性。固定 Spark 模块仅应作为临时手段。

如何固定 Spark 模块

进入代码仓库的“设置”选项卡，随后选择“运行时覆盖”。

创建固定配置

在“运行时覆盖”选项卡中，选择“设置固定”。

您可以选择在所有分支或特定分支上创建固定配置。您可以选定需要固定的 Spark 模块版本（适用于当前仓库中的各类模块类型），并设置到期日期。到期日期距当前日期不得超过 90 天。固定配置会在 90 天期满或您设定的到期日期（以两者中较早的时间为准）失效。超过到期日期后，构建操作可能会失败。

在所有分支上创建固定配置

首先指定需要固定的 Spark 版本及到期日期。

不稳定及非推荐版本前会带有警告标志，除非确有必要，否则不建议使用这些版本。

设置完成后，选择“保存”以应用固定配置。

在特定分支上创建固定配置

您可以为各个分支分别固定版本，也可以为未指定的分支设定版本。

设置完成后，选择“保存”以应用固定配置。在下方示例截图中，我们为所有分支固定了 1.916.0 版本，并将其设置为于 2023 年 12 月 20 日凌晨 12:00 到期。

设置固定配置后，您可以查看到固定配置已创建的确认信息。

编辑现有固定配置

选择“编辑”可修改固定配置。您可以按照创建固定配置的相同流程来更改分支、版本及到期日期。选择“归档”则会移除固定配置，并在先前设置的固定项上添加“已失效”标签。

您可以选择“恢复”来重新创建已归档的固定配置。

查看您的固定配置

在“构建预览 > 查看详情”页面，您可以查看为该仓库固定的版本。

在代码仓库 CI 中使用固定配置控制 transforms 版本

在代码仓库中为构建运行持续集成（CI）时，系统会根据仓库模板的当前版本选择相应版本的《transforms 库》。若已应用固定配置，且该固定版本高于模板所声明的最低版本，CI 将转而使用该固定版本的库。用户可利用固定配置，更快地接入新版 API 及代码辅助与本地开发的新功能。

常见问题

1. 为什么版本锁定会在 90 天后过期？为什么我不能将其锁定更长时间？

我们建议您始终使用最新版本的 Spark。如果不这样做，您可能会错过性能和安全方面的修复与改进，从而对您的构建产生不利影响。

话虽如此，我们也理解某些特定用例确实需要锁定版本，因此我们提供了此项功能，但前提是只能将其作为临时手段使用。在锁定期限到期之前，我们建议您在本地进行必要的调整，以便与最新版本的 Spark 保持兼容。

2. 当我的版本锁定即将到期时，能否通过 UI 界面接收通知？

该功能正在开发中。

3. 能否通过 Job Tracker UI 界面锁定单个构建？

该功能正在开发中。

4. 要使用此功能，我是否需要升级或在本地进行调整？

您无需进行任何升级或操作。

5. 此功能是仅适用于新构建，还是也可以应用于现有构建？

新构建和现有构建均可使用此功能。

6. 如果我的作业已经在 cdconfig 中锁定了版本会怎样？

平台内的版本锁定将始终优先生效。
