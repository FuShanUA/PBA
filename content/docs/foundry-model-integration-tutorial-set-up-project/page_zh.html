1. 教程 - 在Foundry中设置一个机器学习项目

在本教程的这一步中，您将在Foundry中创建一个机器学习项目。这一步是必需的，并将涵盖以下内容：

  1. 为机器学习构建Foundry项目结构
  2. 管理用于机器学习的数据
  3. 管理机器学习模型
  4. 管理Foundry中的机器学习项目

1.1 如何为机器学习构建Foundry项目结构

Foundry项目是用于存储相关工作的文件夹结构。我们建议为每个机器学习项目单独创建一个Foundry项目。该项目应包含：

  • 一个 data 文件夹，用于存储此项目使用的数据集，
  • 一个 models 文件夹，用于存储此项目中的模型，
  • 一个 code 文件夹，用于存储此项目中使用的模型训练逻辑，以及，
  • 一个建模目标，用于管理和部署生产模型。

如果您没有权限创建新项目，可以在现有项目中创建一个新文件夹，作为您的机器学习项目的根目录。

操作：为本教程创建一个新的 Foundry 项目，并创建上述文件夹 - 查看如何操作。如果您无法创建新的 Foundry 项目，请在现有项目中创建一个空文件夹，以模拟新项目的根目录。

操作：在您的 Foundry 项目中，选择 +New > 建模目标。建模目标应根据您试图解决的机器学习问题的名称进行命名。在这种情况下，将目标命名为“房价预测目标”。

1.2 如何管理机器学习数据

在本教程中，我们将建立一个机器学习模型来估算美国人口普查区的中位数房价。

我们将使用特征数据（关于美国人口普查区的历史细节）和标签（该人口普查区当时的中位数房屋价格）来揭示特征与标签之间的关系，然后将这种关系保存为可在Foundry中重复使用的模型。将来，当我们有最新的特征数据（关于美国人口普查区的详细信息）但没有最新的标签（中位数房屋价格）时，我们可以将该模型应用于人口普查区的特征数据，以找到该人口普查区房屋价格的估算值。这种类型的机器学习项目称为监督式机器学习，也是最常见的机器学习项目类型。

在Foundry中，一个监督式机器学习项目应包含两个数据集：

  1. 一个可用于模型训练和测试的< strong >带标签的数据集，以及
  2. 一个< strong >不带标签的数据集，其中包含最新特征但没有标签。我们将对该数据集应用模型，以生成推断（我们标签的预测）。

这些数据集通常来自生产源的数据连接或您的本体论 . 但是，在本教程中，我们将上传CSV文件以模拟这些生产数据源。

操作： 下载 标记的美国住房数据源 并将其上传到 data 文件夹中，作为 housing_features_and_labels. 下载 未标记的美国人口普查数据源 并将其上传到 data 文件夹中，作为 housing_inference_data. 您可以通过将文件拖放到文件夹结构中来将CSV文件上传到Foundry - 在本教程中，将其作为结构化数据集上传。

1.3 如何管理机器学习模型

在Foundry中训练的模型会与用于训练它们的数据、代码和开发环境相关联。这很有用，因为它可以提供一个有管理的记录，说明所有模型是如何生成的，以及记录和分享历史实验的细节。

机器学习模型可以在Code Repositories应用程序中在Foundry中进行训练。

代码仓库

代码仓库应用是一个基于网络的开发环境，用于创建数据管道和机器学习逻辑。Foundry为机器学习提供了一个模板仓库，称为Model Training模板。

代码仓库支持Git用于本地代码迭代，但在Foundry内部运行构建时需要提交的代码。代码仓库应用最适合创建生产级和可重复的数据管道和机器学习逻辑。

集成现有模型

如果你已经有一个想要在Foundry中使用的模型，可以通过以下方式集成该现有模型：

  • 上传一个 容器镜像作为模型
  • 创建一个代理 外部托管模型 的模型
  • 导入预训练的模型文件，例如 从Hugging Face 导入

本教程此步骤没有必需的操作。

1.4 如何管理机器学习项目

在Foundry中，机器学习项目通过Modeling Objectives应用程序进行管理。一个建模目标通过以下方式建议管理机器学习项目的最佳实践：

  • 将机器学习项目围绕特定问题进行定位
  • 创建系统化模型评估的标准
  • 在生产使用前启用多方对模型的审查
  • 保留所有用于生产的模型的历史记录
  • 将模型开发与部署相结合，以批量管道或实时托管推理的方式

在本教程中，建模目标是预测一个人口普查区的中位数房屋价格。

操作：导航到之前创建的“房价预测目标”建模目标。在建模目标的标题部分添加项目上下文，以描述问题供其他团队了解。点击标题右侧的笔图标进入编辑模式，并为您的目标添加摘要和描述。描述字段支持Markdown。建议内容的示例如下：

#### Goal: Build forecasting model to predict median house prices across America.

#### Data

This dataset was derived from the California responses in the 1990 U.S. census, using one row per census block group. A block group is the smallest geographical unit for which the U.S. Census Bureau publishes sample data (a block group typically has a population of 600 to 3,000 people).

The target variable is the **median_house_value** for California districts.

#### Reference

Pace, R. Kelley, and Ronald Barry, "Sparse Spatial Autoregressions," Statistics and Probability Letters,
Volume 33, Number 3, May 5 1997, p. 291-297.

Data derived from StatLib repository. <http://lib.stat.cmu.edu/datasets/>
