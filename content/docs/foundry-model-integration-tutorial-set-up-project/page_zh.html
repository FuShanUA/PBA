1. 教程 - 在 Foundry 中搭建机器学习项目

在本教程的这一步中，您将在 Foundry 中创建一个机器学习项目。此步骤为必做项，内容涵盖：

  1. 为机器学习构建 Foundry 项目结构
  2. 管理机器学习数据
  3. 在 Foundry 中管理机器学习模型
  4. 在 Foundry 中管理机器学习项目

1.1 如何为机器学习构建 Foundry 项目结构

Foundry 项目是用于存储相关工作的文件夹结构。我们建议为每个机器学习项目单独设立一个 Foundry 项目。该项目应包含：

  • 一个 data 文件夹，用于存储本项目使用的数据集；
  • 一个 models 文件夹，用于存储本项目中的模型；
  • 一个 code 文件夹，用于存储本项目中的模型训练逻辑；以及
  • 一个建模目标，用于管理和部署生产模型。

如果您没有创建新项目的权限，可以在现有项目中创建一个新文件夹，作为您机器学习项目的根目录。

操作：为本教程创建一个《新建 Foundry 项目》，并创建上述文件夹——《查看如何操作》。如果您无法创建新的 Foundry 项目，请在现有项目中创建一个空文件夹，以模拟新项目的根目录。

操作：在您的 Foundry 项目中，选择 +新建 > 建模目标。建模目标的命名应与您尝试解决的机器学习问题相关。本例中，请将该目标命名为“房价预测目标”。

1.2 如何管理机器学习数据

在本教程中，我们将构建一个机器学习模型，用于估算美国人口普查区的房屋中位价。

我们将获取特征数据（关于美国人口普查区的历史详情）和标签（该人口普查区当时的房屋中位价），揭示特征与标签之间的关系，然后将这种关系作为可复用的模型保存在 Foundry 中。未来，当我们拥有最新的特征数据（关于某个美国人口普查区的详情）但没有最新的标签（房屋中位价）时，我们可以将该模型应用于该人口普查区的特征数据，从而估算出该人口普查区的房屋价格。这种类型的机器学习项目被称为监督学习，也是最常见的机器学习项目类型。

在 Foundry 中，一个监督式机器学习项目应包含两个数据集：

  1. 一个带标签数据集，可用于模型的训练和测试；以及
  2. 一个无标签数据集，包含最新特征但没有标签。我们将把模型应用于此数据集以生成推断（对标签的预测）。

这些数据集通常来自《生产源数据连接》或您的《本体》。然而，在本教程中，我们将上传 CSV 文件来模拟这些生产源。

操作：下载《带标签的美国住房数据源》，并将其作为 housing_features_and_labels 上传到 data 文件夹中。下载《无标签的美国人口普查数据源》，并将其作为 housing_inference_data 上传到 data 文件夹中。您可以通过将 CSV 文件拖入文件夹结构的方式将其上传至 Foundry——在本教程中，请将其作为结构化数据集上传。

1.3 如何管理机器学习模型

在 Foundry 中训练的模型会关联至训练时使用的数据、代码及开发环境。这有助于为所有模型的生成过程提供受治理的记录，并记录和共享历史实验的细节。

可以在 Foundry 的代码仓库（Code Repositories）应用中训练机器学习模型。

代码仓库

代码仓库应用是一个基于 Web 的开发环境，用于编写数据管道和机器学习逻辑。Foundry 为机器学习提供了一个名为模型训练的模板仓库。

代码仓库支持使用 Git 进行本地代码迭代，但需要提交代码才能在 Foundry 中运行构建。代码仓库应用最适合用于编写生产级且可复现的数据管道和机器学习逻辑。

集成现有模型

如果您已有想在 Foundry 中使用的模型，可以通过以下方式集成该现有模型：

  • 上传容器镜像作为模型
  • 创建模型以代理外部托管的模型
  • 导入预训练模型文件，例如来自 Hugging Face 的模型

本教程的这一步骤无需执行任何操作。

1.4 如何管理机器学习项目

在 Foundry 中，机器学习项目通过“建模目标”（Modeling Objectives）应用进行管理。建模目标提出了管理机器学习项目的最佳实践，具体包括：

  • 围绕特定问题定位机器学习项目
  • 创建系统化模型评估标准
  • 在投入生产使用前启用多方模型评审
  • 维护生产环境中使用的所有模型的历史记录
  • 将模型开发与部署集成至批处理管道或实时托管推理中

在本教程中，建模目标是预测人口普查区内的房屋中位价。

操作：导航至先前创建的“房价预测目标”建模目标。在建模目标的头部区域添加项目背景，为其他团队描述该问题。选择头部右侧的钢笔图标进入编辑模式，为您的目标添加摘要和描述。描述字段支持 Markdown。建议的内容示例如下：

#### 目标：构建预测模型以预测全美各地的房屋中位价。

#### 数据

该数据集源自 1990 年美国人口普查中加利福尼亚州的反馈数据，每个人口普查块组对应一行。块组是美国人口普查局发布样本数据的最小地理单元（一个块组通常包含 600 至 3,000 人）。

目标变量是加利福尼亚各区的 **median_house_value**。

#### 参考

Pace, R. Kelley, 与 Ronald Barry, 《稀疏空间自回归》, 统计与概率快报,
第 33 卷, 第 3 期, 1997 年 5 月 5 日, p. 291-297.

数据派生自 StatLib 仓库。 <http://lib.stat.cmu.edu/datasets/>
