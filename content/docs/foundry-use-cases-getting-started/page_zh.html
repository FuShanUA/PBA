构建新的用例

在本教程中，我们将使用“用例”应用程序创建一个包含各种应用和本体资源的新用例。随后，我们可以直接从用例概览页面导航至 Workshop 和 Ontology Manager 等应用程序，并添加更多应用程序。

第一部分：创建用例

首先，我们需要访问“用例”应用程序。

登录 Foundry 后，在左侧导航栏的 Platform Apps 下访问“用例”应用。如果看不到该应用，请选择 View all，并在弹出的 Platform apps 搜索窗口的 Operational applications 部分下找到 Use Cases。

现在您将进入“用例”应用的首页。如果您尚未创建任何用例，此页面将为空。

在 All use cases 部分的右侧，点击 + Create new。这将打开 Create use case 模态框。添加用例名称和描述。在我们的示例中，用例名称为“航班警报用例”，描述为“用于管理航班警报和更新时刻表的用例。”

在本教程中，我们将在文件系统中创建一个新的支持项目。点击右下角的 + Create new 打开 Create new project 模态框，您可以通过该模态框向 Foundry 文件系统添加新项目。为您的项目添加名称、可选描述和空间。

您的空间将决定组织中用户的默认组织和角色。在我们的示例中，项目命名为“航班警报项目”，描述为“航班警报用例的支持项目”。我们的空间是 Palantir，组织是 Palantir，默认角色是 Viewer。

选择 Create 创建新项目并返回 Create use case 模态框。再次选择 Create 创建您的用例并进入用例概览页面。

详细了解组织、角色以及其他平台安全概念。

第二部分：编辑元数据

创建用例和支持项目后，您可以修改有助于定义用例的元数据。这些元数据包括用例名称、描述、数据主人和状态。

此时，您的用例概览页面应如下所示：

请注意，默认情况下您将成为该用例的技术数据主人。在我们的示例中，我们使用的是 test 用户。

名称

如果需要更改用例名称，您可以在用例概览页面进行操作。点击页面顶部的用例名称，在文本框内进行编辑。

描述

您可以直接点击描述内容，在文本框内编辑，从而轻松更改用例的描述。

数据主人

您可以为用例分配两种类型的数据主人：技术和业务。

  • 技术数据主人：技术数据主人通常由应用构建者担任，负责使用 Foundry 资源为用例创建应用输出。
    • 您是所创建用例的默认技术数据主人。
  • 业务数据主人：业务数据主人通常是跟踪用例工作流进度和财务细节的用户。
    • 当前用例尚未指派默认用户作为业务数据主人。

要更改用例的数据主人，请将鼠标悬停在名称或空白字段上，并点击铅笔图标。系统会弹出一个搜索下拉菜单，你可以滚动浏览或搜索可用用户。由于我们的用例目前还没有业务数据主人，现在我们将进行指派。

我们已将用户 test2 添加为该用例的业务数据主人。

状态

你可以根据用例的运行情况，为其分配相应的状态：

  • Active（活跃）：已发布并在你的 Foundry 实例中运行。
  • Experimental（实验性）：开发中且尚未正式发布。
  • Deprecated（已弃用）：不再运行。

下面我们将用例的状态从 Experimental 更改为 Active。点击状态下拉菜单，选择 Active，然后点击 Change status。

第 3 部分：添加资源

既然我们已经创建了一个带有支持项目的新用例，接下来就可以开始添加应用资源了。让我们先从创建一个新的 Workshop 模块开始。

在用例概览页面中，找到中间的 Applications 区域，点击 + Add new application。选择 Workshop 选项，然后点击 Create。

系统将创建并打开新的 Workshop 模块。在这里，你可以为模块命名，并从各种小组件中进行配置和使用。如需了解更多关于在 Workshop 中创建交互式模块的信息，请参阅相关文档。

你可以点击 Workshop 模块名称左上角的标题，导航返回至该用例。
