视图

一个视图（View）的行为类似于Foundry的数据集视图（dataset view），但它不包含任何包含数据的文件。相反，当它被读取时，它由其他数据集（称为支撑数据集）的并集组成。视图可以被认为“指向”它们的支撑数据集。

视图还可以通过添加主键自动执行数据去重。如果视图的任一支撑数据集有新数据，将会自动触发构建以确保在去重后可用最新的数据。视图的构建方式与常规数据集类似，但由于构建过程中实际上并不读取或写入数据，因此几乎可以立即完成。

通常情况下，您可以像使用常规数据集一样使用视图。但是，视图不能作为有效的转换（transform）输出（outputs）指定；相反，它们可以作为有效的转换（transform）输入（inputs）指定。

此外，视图只能用于具有模式的数据集，因为视图是基于行进行操作的。

基于增量数据集构建的视图不应作为下游转换中的增量输入。当以增量方式读取视图时，去重是在增量读取的事务范围内完成的，而不是在整个视图中完成。

视图的一些主要用例包括以下内容：

  • 自动更新： 创建一个始终最新的联合数据集，而无需执行转换。
  • 文件夹组织： 在不同位置复制数据集而不产生额外的计算和存储成本。
  • 数据唯一性： 在不执行转换的情况下，自动保证主键去重的数据。

创建视图

要创建一个新视图，请从左侧工作区导航面板中打开文件，然后找到您所需的项目或文件夹。到达那里后，在右上角选择新建 > 视图，在当前项目或文件夹中创建一个新视图：

为视图选择一个名称和位置，然后进行配置。当视图被创建时，将在后台自动创建一个构建计划，该计划将在支撑数据集更新时重新构建视图。如果您希望在创建或修改视图后立即读取视图，必须手动构建视图。

选择支撑数据集

因为一个视图是由其支撑数据集的并集组成的，所以给定视图的每个支撑数据集必须具有相同的列名和类型（尽管列的顺序可以不同）。

如果您从支撑数据集中添加或删除列，视图可能会出现 View:MismatchedNumberOfColumns 错误。要解决此问题，请导航至 详情 > 数据集视图设置 并启用 模式演变。这允许视图自动适应其支撑数据集中的模式更改，而无需创建新的视图。

需要一个或多个支撑数据集来构建视图。初始配置后，可以通过导航到 详情 > 数据集视图设置 来修改视图的支撑数据集。

添加主键

数据去重是 Views 的一个关键功能，您可以通过添加主键来启用它。一旦配置了主键，View 将保证主键的唯一性；View 中存在的每个主键只会出现一次。添加主键可以确保在读取 View 时，仅返回每个主键的最新数据。这种自动去重在 View 的支撑数据集可能包含不需要的重复行的情况下尤其有用。

一旦设置了主键，就无法在不创建新 View 的情况下进行修改。

要添加主键，请选择一个或多个列，这些列组合起来将形成主键。所选列不能包含空值。

如果您在初始配置期间没有添加主键，可以通过导航到 详细信息 > 数据集视图设置 来稍后添加。

解决重复键

要添加主键，还必须指定一个列（或列的组合）以解决重复的主键。通过保留这些列中值最高的行并丢弃其他行来对具有相同主键的行进行去重。这确保了View中出现的每个主键都是唯一的。

选择删除列

您还可以选择性地指定一个删除列，该列必须包含布尔值。去重后，如果给定主键的最终剩余行在删除列中具有true，则该行将被排除在View之外。

实际中的去重和删除

在使用删除列排除行之前，会先解决重复的主键。因此，对于给定的主键，“最新”行中的删除列值才有效。

去重和删除适用于所有的支撑数据集：

去重和删除是通过以下组合实现的：

  • 在视图上适当配置并自动创建的投影（projection）。
  • 基于窗口函数的过滤逻辑，在每次读取时进行评估。

尽管过滤逻辑会在每次读取时进行评估，但只要投影是最新的，并且视图的支撑数据集自上次投影构建以来只进行了APPEND或UPDATE事务，这种逻辑的评估对运行时间和计算成本的影响可以忽略不计。

考虑到这种架构，我们建议仅在以下情况下创建带有主键配置的视图。

  • 所有支撑数据集都通过 APPEND 或严格追加 UPDATE 事务进行更新。
  • 部分支撑数据集通过 APPEND 或严格追加 UPDATE 事务进行更新，部分则通过 SNAPSHOT 事务进行更新，但那些通过 SNAPSHOT 事务更新的数据集构建频率较低。

如果对支撑数据集的更新需要重新生成投影，则会产生较高的计算成本，因为读者（Contour、转换任务等）在投影重新生成完成之前必须在没有投影优势的情况下应用过滤逻辑。

使用视图

视图在Foundry中可见，就像常规的Foundry数据集一样，可以在任何可以使用常规数据集的地方使用。唯一例外的是视图不是有效的转换输出（尽管它们是有效的转换输入）。视图的一个用例是使用主键高效地创建多个数据集的并集，以确保结果视图具有数据唯一性。

使用视图移除标记

视图不支持从数据集中移除组织，仅支持强制控制标记。

使用视图移除标记可能未在您的Foundry注册中启用。请联系Palantir支持以启用此功能。

您可以使用视图来停止在支撑该视图的数据集上传播标记。您可以通过导航到视图的详细信息选项卡中的数据集视图设置，然后选择编辑 > 管理标记传播。

只有拥有 Remove Marking 权限的用户才能停止标记的传播。对标记传播的更改将在支撑数据集发生任何更新时生效。

有关更多信息，请查看我们的 关于移除标记的指导。
