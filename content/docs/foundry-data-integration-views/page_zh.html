视图

视图的行为与 Foundry 的数据集视图类似，但它不保存任何包含数据的文件。相反，它由其他数据集（称为底层数据集）在读取时合并而成。可以将视图视为“指向”其底层数据集。

通过添加主键，视图还能自动对数据执行去重操作。如果视图的任一底层数据集包含新数据，系统会自动触发构建，以确保去重后能获取到最新数据。视图的构建方式与常规数据集类似，但由于构建过程中不涉及实际的数据读写，其构建过程几乎瞬间完成。

通常，你可以像使用常规数据集一样使用视图。不过，视图不能作为有效的转换输出；相反，它只能作为有效的转换输入。

此外，视图只能与具备 Schema 的数据集配合使用，因为视图是按行操作的。

在增量数据集上构建的视图，不应用作下游转换的增量输入。当以增量方式读取视图时，去重操作仅限于增量读取的事务范围内执行，而不是在整个视图范围内进行。

视图的一些主要用例包括：

  • 自动更新：创建一个始终保持最新的合并数据集，无需执行转换操作。
  • 文件夹组织：在不同位置复制数据集，且不会产生额外的计算和存储成本。
  • 数据唯一性：通过主键去重，自动保证数据唯一性，无需进行转换。

创建视图

要创建新视图，请从左侧工作区导航面板打开文件，然后找到目标项目或文件夹。进入后，在右上角选择新建 > 视图，即可在当前项目或文件夹中创建新视图：

为视图选择名称和位置，然后继续配置。视图创建完成后，后台会自动生成构建计划，只要底层数据集发生更新，就会重新构建视图。如果在创建或修改视图后想立即读取数据，必须手动构建视图。

选择底层数据集

由于视图由其底层数据集合并而成，因此给定视图的每个底层数据集必须具备相同的列名和类型集合（尽管列的顺序可以不同）。

如果在底层数据集中添加或删除列，视图可能会报错并抛出 View:MismatchedNumberOfColumns 错误。要解决此问题，请导航至详情 > 数据集视图设置，并启用模式演进。这允许视图自动适应底层数据集中的模式变化，而无需创建新视图。

构建视图至少需要一个底层数据集。初始配置完成后，可以通过导航至详情 > 数据集视图设置来修改视图的底层数据集。

添加主键

数据去重是视图的一项核心功能，你可以通过添加主键来启用该功能。主键配置完成后，视图将保证主键的唯一性；视图中的每个主键仅出现在一行中。添加主键可确保在读取视图时，每个主键仅返回最新的数据。当视图的底层数据集可能包含不需要的重复行时，这种自动去重功能会显得尤为实用。

主键一旦设置，除非创建新视图，否则无法修改。

要添加主键，请选择一列或多列，这些列组合起来将构成主键。所选列不能包含空值。

如果初始配置时未添加主键，后续可以通过导航至 详情 > 数据集视图设置 来完成添加。

解决重复键

添加主键时，还必须指定用于解决重复主键的列（或多列组合）。对于具有相同主键的行，去重操作会保留这些列中按顺序排列值最大的行，并丢弃其他行。这能确保视图中出现的每个主键都是唯一的。

选择删除列

您还可以选择指定一个删除列，该列必须包含布尔值。去重后，如果某个给定主键最终保留的行在删除列中的值为 true，该行将从视图中排除。

实际操作中的去重与删除

重复主键的解决发生在使用删除列排除行之前。因此，只有给定主键对应“最新”行中的删除列值才会起作用。

去重和删除通过以下组合方式实现：

  • 视图上经过适当配置且自动创建的投影。
  • 基于窗口函数并在每次读取时进行评估的过滤逻辑。

尽管每次读取都会评估过滤逻辑，但只要投影保持最新，且视图的底层数据集自上次投影构建以来仅发生 APPEND 或 UPDATE 事务，该逻辑的评估对运行时间和计算成本的影响微乎其微。

鉴于这种架构，我们建议仅在以下情况配置主键创建视图：

  • 所有底层数据集均通过 APPEND 或严格追加的 UPDATE 事务进行更新。
  • 部分底层数据集通过 APPEND 或严格追加的 UPDATE 事务更新，而另一部分通过 SNAPSHOT 事务更新，但后者构建频率较低。

如果底层数据集的更新需要完全重建投影，将产生大量的计算成本，因为在投影重建完成之前，读取者（Contour、转换作业等）必须在缺少投影支持的情况下应用过滤逻辑。

使用视图

在 Foundry 中，视图与常规 Foundry 数据集一样可见，并且可以在任何使用常规数据集的地方使用。唯一的例外是视图不能作为转换输出（但可以作为转换输入）。视图的一个典型用例是高效合并多个数据集，并利用主键确保生成的视图数据具有唯一性。

使用视图移除标记

视图不支持从数据集中移除组织，仅支持移除强制控制标记。

您的 Foundry 部署中可能未启用使用视图移除标记的功能。请联系 Palantir 支持团队以启用此功能。

您可以使用视图来停止在底层数据集上传播标记。操作方法为：导航至视图的 详情 标签页中的 数据集视图设置，然后选择 编辑 > 管理标记传播。

只有拥有 移除标记 权限的用户才能停止标记的传播。对标记传播的修改将在底层数据集发生任何更新时生效。

请查阅我们的 《移除标记指南》 以了解更多信息。
