创建发布推广管道

软件推广是常见的DevOps概念。在Apollo中，发布通道贡献者使用发布通道将产品发布添加到环境中。一旦满足定义的标准，产品可以自动或手动推广到这些通道。然后，环境编辑器可以订阅他们的环境以根据其风险承受能力和功能需求接收产品发布。

在Apollo中，发布推广管道定义了您的产品发布应如何从一个发布通道推广到另一个发布通道以及每个推广阶段必须满足的标准。这使得可以在您的环境中实现自动化、稳定的最新发布部署。

在此任务中，您将创建一个两阶段的RELEASE → STABLE管道并定义基本的推广标准。然后，您将订阅您的环境以从其中一个通道接收产品发布升级。

有关Apollo默认发布通道及其如何使用产品版本字符串来确定发布的默认通道的信息，请阅读以下文档条目：

  • 默认发布通道
  • 产品版本规范

创建新的稳定发布通道

  1. 从左侧边栏中选择 设置。
  2. 导航到 发布通道 选项卡，然后选择 创建发布通道。
  3. 在 详细信息 选项卡中，将发布通道命名为 STABLE 并输入描述，例如“适用于生产的发布”。然后选择 下一步。
  4. 在 要求 选项卡中，您可以为发布通道设置标签要求。在此任务中，您不需要设置任何标签要求，但可以在 Apollo 文档 中了解更多有关标签的信息。一些标签要求的示例包括确保安全扫描通过、验证容器镜像是否存在以及确认某个遥测值为真。
  5. 在 权限 选项卡中，您可以指定哪些团队可以使用或管理该通道。您可以暂时保留默认设置，然后选择 创建通道。

设置您的管道

  1. 导航到您的Helm chart或Nginx的产品页面。注意发布管道模块中的未配置状态。
  2. 在发布管道模块中选择编辑。
  3. 选择添加阶段以添加一个发布阶段。输入RELEASE作为源通道和STABLE作为目标通道。然后输入以下设置。您可以将鼠标悬停在字段标签上以了解更多信息。
    • 测试环境: 留空
    • 浸泡时间: 1小时
    • 发布阈值: 1
    • 召回阈值: 1
    • 浸泡比例: 0.95
  4. 完成后选择确认。

测试您的发布管道

如果您连接了多个环境，您可以将其中一个设置为在RELEASE上接收产品的发布，另一个设置为在STABLE上接收它们。在Apollo将发布从RELEASE发布到STABLE之前，它会根据您在上一节中定义的发布评估要求来评估该发布。

如果您连接了一个单一的Spoke环境，请按照下面的说明将其订阅到RELEASE频道。

通常情况下，Apollo Hub 可能会连接多个 Environment。在这些情况下，可能不需要发布提升管道，因为没有测试 Environment 来进行发布评估。

在继续之前，请注意您可以通过以下方式设置和更改 Environment 的默认发布通道：

  1. 打开其中一个连接的 Environment。
  2. 从 操作 下拉菜单中选择 设置默认发布通道...。
  3. 从下拉菜单中选择一个默认的发布通道。

您 Environment 中所有产品的安装，称为“实体”，现在将从默认发布通道接收发布升级。您可以为某个实体指定不同的通道，如果该产品不在默认通道上，或者您想为安装编码不同的风险承受能力。

如果您连接了多个 Environment，您可以按照以下步骤来测试您的管道。

  1. 导航到其中一个 Environment 的 Environment 页面，我们称之为 Environment 1。
  2. 在 实体 选项卡中找到您的产品或 Nginx。
  3. CHANNEL 列中的值应为 RELEASE；Environment 1 已订阅从 RELEASE 通道接收发布内容。
  4. 导航到另一个环境的环境页面，我们将称之为 Environment 2。在 实体 选项卡中找到您的产品或 Nginx。 CHANNEL 列中的值可能是 RELEASE，因为这是该环境的默认通道。 我们将此实体的发布通道更改为 STABLE。
  5. 在 实体 表中选择产品（您的或 Nginx 的），以打开 Nginx on ${environmentName} 实体页面。
  6. 从 操作 下拉菜单中选择 设置默认发布通道...。
  7. 将值从 RELEASE 更改为 STABLE，然后选择 更改通道。
  8. 返回到 Environment 2 的 实体 选项卡。您可能需要刷新浏览器。确认您的产品或 Nginx 在 CHANNEL 列中的值现在为 STABLE。

现在您可以观看您的发布管道运行情况。按照之前遵循的步骤创建您的产品或 Nginx 产品的新的发布版本。请确保在配置中使用小写字母作为产品和 Helm 图表名称。

您还应适当增加版本号。例如，如果您的最新发布版本是 13.1.5，请将新版本设置为 13.1.6。您可以在 Artifact Hub ↗ 查看可用的 Nginx Helm 图表版本。

Apollo 将会自动在 环境 1 上安装新的发布版本，因为该环境已订阅接收您产品在 RELEASE 通道中的新版本。Apollo 会在 环境 1 上对发布版本进行评估后，将发布版本推广到 STABLE 通道，并仅在那时在 环境 2 上安装新版本。

您可以通过打开新发布版本的页面来监控推广进度。

请注意，每个发布阶段表示 1 个总实体，即每个通道有一个环境订阅。在配置的一小时成功浸泡时间之后，发布版本将自动推广到 STABLE，然后 Apollo 才会在 环境 2 上安装该发布版本。

您可以重复此过程，以处理您可能添加到产品目录中的任何其他 Helm 图表。
