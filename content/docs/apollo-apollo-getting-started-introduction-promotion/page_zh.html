创建一个发布提升管道

软件提升是常见的DevOps概念。在Apollo中，发布通道贡献者使用 发布通道 将产品发布到环境中。一旦满足定义的标准，产品可以自动或手动提升到这些通道。然后，环境编辑器可以根据他们的风险承受能力和功能需求，订阅其环境以接收产品发布。

在Apollo中，发布提升管道定义了您的产品发布应如何从一个发布通道提升到另一个发布通道以及每个提升阶段必须满足的标准。这使您能够在所有环境中实现自动化、稳定的发布。

在此任务中，您将创建一个两阶段的 RELEASE → STABLE 管道并定义基本的提升标准。然后，您可以订阅您的环境以从其中一个通道接收产品发布升级。

关于Apollo的默认发布通道以及它如何使用产品版本字符串来确定发布的默认通道，请阅读以下文档条目：

  • 默认发布通道
  • 产品版本规范

创建一个新的稳定发布通道

  1. 从左侧边栏中选择设置。
  2. 导航到发布通道选项卡，然后选择创建发布通道。
  3. 在详情选项卡中，将发布通道命名为STABLE，并输入描述，例如“适用于生产的发布”。然后选择下一步。
  4. 在要求选项卡中，您可以为发布通道设置标签要求。在这个任务中您不需要设置任何标签要求，但可以在Apollo文档中了解更多有关标签的信息。 . 一些标签要求的示例包括确保安全扫描通过、验证容器镜像存在，并确认某个遥测值为真。
  5. 在权限选项卡中，您可以指定哪些团队可以使用或管理该频道。您现在可以保留默认设置，并选择创建频道。

设置您的管道

  1. 导航到您的Helm图表或Nginx的产品页面。注意推广管道块中的Not configured状态。
  2. 在推广管道块中选择编辑。
  3. 选择添加阶段以添加一个推广阶段。输入RELEASE作为源频道和STABLE作为目标频道。然后输入以下设置。您可以将鼠标悬停在字段标签上以了解更多。
    • 测试环境: 留空
    • 浸泡时间: 1小时
    • 推广阈值: 1
    • 回忆阈值: 1
    • 浸泡比例: 0.95
  4. 当您完成后选择确认。

测试您的发布管道

如果您连接了多个环境，您可以设置一个环境在RELEASE上接收产品的发布，另一个环境在STABLE上接收它们。在Apollo将发布从RELEASE提升到STABLE之前，它会根据您在上一节中定义的发布评估要求来评估该发布。

如果您连接了一个单个的 spoke 环境，请使用以下说明将其订阅到RELEASE频道。

Apollo Hub 只连接一个环境的情况很少见。在这种情况下，可能不需要发布管道，因为没有测试环境用于发布评估。

在继续之前，请注意，您可以通过以下步骤为您的环境设置和更改默认发布通道：

  1. 打开您连接的其中一个环境。
  2. 从操作下拉菜单中选择设置默认发布通道...。
  3. 从下拉菜单中选择一个默认的发布通道。

您环境中所有产品的安装，称为“实体”，现在将从默认发布通道接收发布升级。您可以为某个实体指定不同的通道，如果该产品在默认通道上不可用，或者您想为该安装编码不同的风险容忍度。

如果您连接了多个环境，可以按照以下步骤测试您的管道。

  1. 导航到其中一个环境的环境页面，我们将称之为Environment 1。
  2. 在实体选项卡中找到您的产品或Nginx。
  3. CHANNEL列中的值应为RELEASE；Environment 1已订阅从RELEASE通道接收发布。
  4. 导航到其他环境页面，我们将其称为Environment 2。在实体选项卡中找到您的产品或 Nginx。CHANNEL列中的值可能是RELEASE，因为这是此环境的默认频道。 我们将此实体的发布频道更改为STABLE。
  5. 在实体表中选择产品，您的或 Nginx，以打开Nginx on ${environmentName}实体页面。
  6. 从操作下拉菜单中选择设置默认发布频道...。
  7. 将值从RELEASE更改为STABLE，然后选择更改频道。
  8. 返回到Environment 2的实体选项卡。您可能需要刷新浏览器。确认您的产品或 Nginx 的CHANNEL列中的值现在为STABLE。

现在您可以观看您的发布管道运行了。按照之前遵循的步骤为您的产品或 Nginx 产品创建一个新发布。请确保在配置中使用小写字母作为产品和 Helm 图表名称。

您还应适当增加版本号。例如，如果您的最新发布版本是 13.1.5，请将新版本设置为 13.1.6。您可以查阅 Artifact Hub ↗ 以查看可用的 Nginx Helm 图表版本。

Apollo 将在 Environment 1 上自动安装新发布版本，因为 Environment 已订阅接收您产品的新版本，通过 RELEASE 通道。Apollo 将在 STABLE 通道中推广该发布版本，并且只有在该发布版本已在 Environment 1 上评估后，才会在 Environment 2 安装新版本。

您可以通过打开新发布版本的页面来监控推广进度。

请注意，每个发布阶段表示 1 total entity，这意味着每个通道都有一个 Environment 订阅。在成功浸泡时间配置的一小时后，发布版本将自动推广到 STABLE，然后 Apollo 才会在 Environment 2 上安装该发布版本。

您可以对产品目录中添加的任何其他 Helm 图表重复此过程。
