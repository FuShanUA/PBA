添加数据集输出

在管道构建器中，您可以选择添加数据集输出，以引导管道集成生成清洗、转换后的数据。了解更多关于不同输出类型的信息。

创建数据集输出

首先，点击图形右侧输出面板中数据集类型旁边的添加。

现在您已创建了一个新的输出数据集。首次构建管道后，您的数据集输出将创建在与管道相同的文件夹中。例如，演示管道的 Vendor 数据集输出将具有以下文件路径：/Palantir/Pipeline Builder/Demo Pipeline/Vendor。

点击名称字段可重命名输出数据集。选择添加列可手动向输出模式中添加列，或者连接转换节点并使用使用更新后的模式来采用其输出模式。

添加输出模式后，使用搜索列...字段可快速查找数据集中的列。要仅查看输出模式中的错误，请切换仅显示错误按钮。

添加输出数据集后，点击返回所有输出可查看管道中所有输出的列表。一目了然地了解每个输出的状态，包括输出模式是否与输入转换节点模式匹配。以下三个输出展示了输出模式可能具有的不同状态：

  • 数据集 1包含 5/5 个必需列，这意味着输入转换节点中的每一列都将构建在输出数据集中。
  • 数据集 2包含 3/3 个必需列，其中 2 列已丢弃，这意味着输入转换节点有 5 列，但只有 3 列将构建在输出数据集中。这在输入转换节点中存在不需要的列时非常适用。
  • 数据集 3包含 5/7 个必需列，这是一种错误状态。在 2 个缺失列映射到输入转换节点中的列之前，您将无法部署管道。

点击编辑可随时更新输出模式。

在数据集成中了解更多关于数据集模式的信息。

配置输出设置

除了模式配置外，每个单独的输出还有各种可自定义的默认设置。

配置预期

在输出数据集上添加预期以保障管道稳定性。如果管道构建期间任何检查失败，构建将失败。

配置写入模式

定义在未来的部署中如何将数据添加到数据集输出。

默认：建议用于大多数批处理管道，包括快照和增量模式。如果至少有一个输入被标记为增量，并且所有被标记为增量的输入自上次构建以来仅发生过 APPEND 或附加性 UPDATE 事务，则默认写入模式会将结果作为 APPEND 事务输出。否则，默认写入模式会将结果作为 SNAPSHOT 事务输出。了解更多关于事务类型的信息。

始终追加行：将结果作为 APPEND 事务输出。

仅追加新行：将结果作为 APPEND 事务输出，其中只有新行（定义为新出现的主键）会被添加到输出中。如果当前事务中存在重复行，则会随机丢弃其中一行。主键已存在于先前输出中的行将被丢弃。

变更日志：输出一系列包含所有记录完整变更历史的 APPEND 事务。在 对象存储 V1 中，通过将以此方式配置的变更日志数据集设为底层数据源，可以优化对象类型的同步性能。详细了解变更日志数据集。

快照差异：将结果输出为 SNAPSHOT 事务，其中仅保留主键为新出现的行。如果当前事务内存在重复行，这些行将被保留，而所有其他行均会被丢弃。

快照替换：将结果输出为 SNAPSHOT 事务，其中新数据将与先前的输出合并。先前输出中已存在的主键将被丢弃，以让位于新行。如果当前事务内存在重复行，将随机丢弃多余的行仅保留一条，从而确保输出中每个主键仅对应一行。

快照替换并移除：这等同于快照替换，随后进行后置过滤阶段以选择性地从旧数据中移除行。将结果输出为 SNAPSHOT 事务，其中新数据与先前输出合并后，随后进行后置过滤阶段，基于提供的布尔值 post_filtering_column 移除先前事务中的行。若 post_filtering_column = TRUE，先前输出中已存在的主键将被丢弃，以让位于新行。然而，如果当前事务中存在某主键对应行的 post_filtering_column = FALSE，那么旧数据中相应的行将被过滤掉（尽管这不会覆盖对 post_filtering_column = TRUE 的新行的存储）。如果当前事务中存在多个 post_filtering_column = TRUE 的重复行，将随机丢弃多余的行仅保留一条，从而确保输出中每个主键仅对应一行。

数据集写入格式

数据集的输出文件格式可以在初次部署后更改，并在下一次部署管道时生效。详细了解文件格式。

覆盖数据集

这是一项一次性操作，用于将现有数据集的数据权属授予管道构建器中的新输出。请注意，此操作可能需要在管道构建器之外执行额外操作。

构建数据集输出

将数据集输出添加到管道后，请务必保存更改。如果数据转换和管道工作流定义均已完毕，即可部署管道并构建数据集输出。部署管道后，请在本体管理器中使用最终的数据集输出作为构建本体的基础。
