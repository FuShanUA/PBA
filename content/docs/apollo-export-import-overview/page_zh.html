导出和导入

核心概念

  • 源中心: 一个将产品目录数据、图像和配置传输到目标中心的Apollo中心。源中心也可以作为目标中心。
  • 目标中心: 一个从源中心接收产品目录数据、图像和配置的Apollo中心。目标中心也可以作为源中心。
  • 包: 目标中心的新更新压缩包。
  • 导出管道: 从源中心导出的一组资源。

概述

Apollo 提供了一种无缝的方式，可在缺乏网络访问或无法连接到您的源 Apollo Hub 的环境中部署软件。通过在大多数用户可访问的源 Hub 中管理软件目录和配置的真实来源，它消除了仅依赖有权访问目标 Hub 的用户的需要。这使得开发人员、运维人员和其他 Apollo 用户能够在各种网络边界上对软件的配置、管理和监控拥有统一视图。

为了实现这一点，请从另一个 Apollo Hub 指定一个目标 Hub，它将作为源 Hub。源 Hub 将作为目标 Hub 的一些或全部产品发布信息的真实来源。

更改首先在源 Hub 中进行，该 Hub 被配置为真实来源，然后转移到目标 Hub。例如，应该首先在源 Hub 中 创建 一个发布。

导出管道

一个 导出管道 在 Apollo 中定义了从源中心导出为 Bundle 的资源集合。导出管道定义了构建 Bundle 的频率，并且还允许手动构建。

导出管道提供了选择从源中心导出哪些数据的灵活性。例如，您可以创建一个导出管道以支持特定的事件响应团队，其中仅导出修复所需的 Releases 和相关资源。您还可以设置导出管道，根据模块导出自上次构建以来的所有新更新。

从导出管道构建的 Bundle 可以导入到任何 Apollo 中心。在上面的例子中，您可以从事件响应导出管道导出一个 Bundle，并将其导入所有受该事件影响的目标中心。

Bundle 内容

在源中心，Apollo 会构建一个称为“Bundle”的包。您可以通过配置一个 导出管道，并设置一个或多个 数据选择规则 来定义 Bundle 的内容。一个 Bundle 可以包含：

  • 源中心产品目录中的产品版本。这还包括产品版本的元数据，例如它们属于哪个 发布渠道，是否是 模块 的一部分，或者是否已被召回。
  • 将目标中心的实体升级到相应发布渠道上最新版本所需的最小图像集。

了解更多关于 导出 和 导入 Bundle 的信息。

架构

下图概述了从源中心到目标中心的 Bundle 传输过程。

从源中心：

  1. 导出过程可以手动触发，也可以通过 导出管道计划 触发。
  2. Apollo 根据 数据选择规则 检索需要更新的版本信息。
  3. 如果 Apollo 可以访问您的构件存储库，它将使用这些信息从您的构件存储库中拉取正确的构件，并进一步压缩这些构件，使得生成的 Bundle 尽可能小。如果 Apollo 无法访问您的构件存储库，只有元数据会被打包，此时您组织的责任是确保这些镜像被传送到目标中心的构件存储库。
  4. Apollo 然后会将所有内容打包成一个 Bundle，您可以将其导出。

从目标中心：

  1. 使用适用于该环境的正确传输流程，将 Bundle 导入目标中心。
  2. Apollo 使用新旧版本元数据之间的差异重新组装版本镜像，并将它们发布到目标中心环境中的构件存储库。

一旦此过程完成，目标中心将根据 计划 发布升级、版本召回、配置更改以及其他所有维护操作的指令，前提是所有相关的 约束 已经满足。
