Functions 中的时间序列

Functions 支持对时间序列属性进行操作。本页将介绍如何在函数中设置和使用时间序列属性。

初始设置

要在 Functions 中使用时间序列，必须先将时间序列存储在本体中。您可以按照此处概述的步骤开始操作。

将时间序列存储在本体中后，我们需要为时间序列 Functions 创建代码仓库。在此仓库中，首先导入本体类型，以便引用存储在这些本体类型中的时间序列。

现在您已准备好在 Functions 中处理时间序列了。

在 Functions 中处理时间序列

要访问 Functions 中的时间序列，请先在代码仓库中创建对象支持的函数。这些函数可以直接访问本体中的时间序列属性。编写好函数后，有两种方式可以运行新函数。您可以在实时预览中测试函数，也可以发布函数并开始在平台的其他应用中使用它。

28

30    df["value"] = (df["value"].diff() / time_diff_seconds).fillna(0)
31
32    max_abs_rate = df["value"].abs().max()
33    if max_abs_rate > 0:

35
36    return TimeSeries.serialize(df)

TypeScript 函数

以下各节提供了常见 TypeScript 函数操作的示例。

返回第一个或最后一个点

由于函数没有针对时间序列点的内置类型，因此你可以改为返回值或时间戳。例如，以下函数读取机器上的最新温度：

求导

基于上述示例，您还可以通过以下计算函数，获取同一台设备的平均温度变化量：


```text
    @Function()
    public async getAverageTemperature(machine: MachineRoot): Promise {
        const aggregation = await machine.temperatureId?.derivative()
            .aggregate()
            .overEntireRange()
            .mean()
            .compute();
        return aggregation?.mean;
    }
```

指定时间范围

除了求导运算外，你还可以对时间序列应用其他转换操作。以下示例展示了如何将时间戳参数作为范围应用于时间序列：


@Function()
public async getAverageTemperatureOverRange(
    machine: MachineRoot,
    start: Timestamp,
    end: Timestamp): Promise
{
    const latest = await machine.temperatureId?.timeRange({min: start, max: end})
        .aggregate()
        .overEntireRange()
        .mean()
        .compute();
    return latest?.mean;
}
