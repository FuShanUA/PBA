对象视图版本控制

对对象视图的已保存编辑将作为新版本存储。一个版本可以包含多个更改，例如添加、编辑和删除选项卡。版本控制使您可以：

  • 通过逐步保存更改安全地迭代对象视图
  • 控制哪个版本被发布并可见于用户
  • 在较新版本包含错误时重新发布旧版本
  • 通过为版本添加描述与其他编辑者协作

对象视图中包含的每个工作区模块都有单独的版本。两个版本号都出现在对象视图编辑器标题中。

禁用自动发布新版本将显示两个单独的按钮用于保存和发布。 保存按钮将保存标签和工作区模块的更改，而 发布按钮将把这两项更改发布给用户。

在编辑工作区模块时，它会像其他工作区模块一样定期自动保存，但这些更改在对象视图发布之前不会对用户可见。

如果您的对象视图被许多用户频繁使用或有多个编辑者共同协作，则我们建议禁用自动发布。这可以改善协作，并降低破坏用户工作流程的可能性。

访问早期版本

要查看对象视图的早期版本列表，请在对象视图编辑器的标题栏中选择蓝色的对象视图版本号。这将打开一个对话框，显示所有早期版本的日期、作者和描述。从这个对话框中，可以预览所有版本。

  • 当前已发布的版本会用绿色对勾标记。
  • 早期已发布的版本会显示灰色对勾。
  • 从未发布的版本不会显示对勾。
