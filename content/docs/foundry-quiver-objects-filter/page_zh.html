过滤对象集

在 Quiver 中，过滤对象最常用的方法是使用过滤对象集卡片。将鼠标悬停于任意对象集卡片上，便会在卡片下方显示下一操作菜单。随后，选择 Filter > Filter object set 即可添加过滤卡片。

下例运用过滤器，仅纳入建于 2010 年至 2020 年间且类型为 building 或 garage 的建筑对象，从而在输入对象集（总计 1,100,640 个对象）中筛选出包含 11,617 栋建筑的新对象集。

你还可以直接通过对象集卡片表格中的列标题，为特定属性创建过滤卡片。具体操作如下：点击某属性列标题右侧的下拉菜单，随后将光标移至 Add filter where... 上方。各属性列标题均提供针对精确匹配和空值过滤的快捷选项。采用此方式添加过滤器后，系统将生成一张过滤对象集卡片，并预先配置好所选属性。

查看过滤结果

选择 Show/Hide data 按钮，即可在 results 视图与卡片 configuration 视图之间进行切换。

基于关联属性过滤

过滤对象集卡片同样支持按关联对象属性进行过滤。若你的过滤条件并非针对当前操作对象自身的属性，而是针对与其关联的对象属性，此功能便十分实用。

要添加关联属性过滤器，需在添加过滤器时从 Filter by linked objects 列表中选择关联对象类型，随后点击该关联属性即可将其添加。

下例对茶叶批次的 object set 进行了过滤。每个茶叶批次均关联至一个 Tea Tasting 对象，该对象会按 1 至 10 的等级对批次进行评价。本例中，我们旨在寻找优质批次，因此你可以将条件设置为仅保留评价等级大于 8 的 Tea Tasting 所关联的 Tea Batches。

按关联属性进行过滤时，若初始对象集规模在 10 万以下，则过滤规模不受限制。若初始对象集规模超过 10 万，则关联对象集的规模必须小于 1000 万。

嵌套过滤器

对于需要同时使用 AND 和 OR 运算符的高级过滤工作流，过滤卡片支持嵌套过滤器。操作时，点击某属性过滤器右侧的 Insert Nested Filter 按钮即可。此举将弹出一个菜单，供你选择嵌套过滤器应采用 AND 还是 OR 条件，以及需过滤的属性。虽然过滤器支持无限嵌套，但为保证可读性，建议控制嵌套深度。

在下例中，我们原有的过滤器纳入了建于 2010 年至 2020 年间且类型为 skybridge 的建筑对象。我们希望同时纳入建于 1990 年至 2000 年间的空中连廊，因此在建设年份过滤器上嵌套了一个 OR 条件。

基于派生属性过滤

若需对在分析中本地派生的属性进行过滤，可通过转换表格或物化视图来实现。这两种方式在规模与性能上各有取舍，但均支持在本地派生属性并对其进行过滤。

下面的示例展示了如何使用转换表派生新的高度属性：用建筑物的地面高程减去屋顶高度，随后筛选出该新高度属性大于 50 的记录。
