准备数据集以供下载

以下导出流程属于高级工作流，仅在无法通过操作菜单直接从 Foundry 界面下载数据，或无法从另一个 Foundry 应用程序导出数据时才应执行。

本指南介绍如何使用《代码仓库》或《流水线构建器》中的转换功能来准备可供下载的 CSV 文件。

准备数据

准备 CSV 下载文件的第一步是创建经过过滤和清理的数据集。我们建议执行以下步骤：

  1. 确保数据样本可以导出，并遵循数据导出控制规则。具体而言，您应验证导出操作是否符合您所在组织的数据治理策略。
  2. 过滤数据，使其在满足必要目标的前提下尽可能小。为获得最佳性能，CSV 格式数据的未压缩大小应小于默认的 HDFS 块大小（128 MB）。为此，您应仅选择必要的列并尽量减少行数。您可以通过筛选特定值，或抽取任意行数（例如 1000 行）的随机样本来减少行数。尝试创建大于 128 MB 的 CSV 文件可能会消耗更多时间，并且可能需要额外的 Spark 执行器内存才能成功。
  3. 将列类型更改为 string。由于 CSV 格式缺乏架构（列的类型和标签不受限制），建议将所有列转换为字符串。这对于时间戳列尤其重要。

以下 Python (PySpark) 示例代码说明了将上述部分原则应用于纽约市出租车行程数据集的情况。

def prepare_input(my_input_df):
    from pyspark.sql import functions as F

    filter_column = "vendor_id"
    filter_value = "CMT"
    df_filtered = my_input_df.filter(filter_value == F.col(filter_column))

    approx_number_of_rows = 1000
    sample_percent = float(approx_number_of_rows) / df_filtered.count()

    df_sampled = df_filtered.sample(False, sample_percent, seed=0)

    important_columns = ["medallion", "tip_amount"]

    return df_sampled.select([F.col(c).cast(F.StringType()).alias(c) for c in important_columns])

运用类似的逻辑和 Spark 概念，您也可以在《流水线构建器》或其他 Spark API（如 SQL 或 Java）中实现数据准备工作。

缩减为单个分区并设置输出格式

数据准备好导出后，您需要将其重新分区或合并为单个分区，以便输出的下载内容仅包含一个文件。接着，将输出格式设置为 CSV 或其他受支持的输出格式，如 JSON、ORC、Parquet 或 Avro。以下示例展示了如何在《流水线构建器》、Python 和 SQL 中重新分区数据并设置输出格式。

repartition(1) 和 coalesce(1) 都能将数据缩减为单个分区，但如果在同一转换中执行了过滤操作，则应避免使用 coalesce，因为这会合并上游任务，可能导致过滤操作以非分布式方式运行。

在上述示例中，我们将分区缩减为一个，从而确保输出数据集只生成一个可供下载的 CSV 文件。如此一来，全部数据都必须能够容纳在单个 Spark 执行器的内存中，因此我们建议先对数据进行过滤或采样。如果无法进行过滤或采样，且数据量过大而无法塞进单个执行器的内存中，可以通过配置合适的《Spark 配置文件》来增加执行器内存。或者，也可以将分区数设置为大于 1 的值。使用大于 1 的 repartition 值会减少单个执行器内存中需要保留的数据量，但这同时也意味着输出数据集会生成多个 CSV 文件，需要逐个下载而非一次性下载。

访问文件进行下载

数据集构建完成后，请前往数据集页面的“详情”选项卡。此时即可看到可供下载的 CSV 文件。
