准备数据集以供下载

以下的导出流程是一个高级工作流，只有在您无法通过 操作（Actions） 菜单或 从另一个 Foundry 应用程序导出数据 直接从 Foundry 界面下载数据时，才应执行。

本指南将解释如何使用 代码仓库（Code Repositories） 或 管道构建器（Pipeline Builder） 中的转换功能准备 CSV 文件以便下载。

准备数据

准备 CSV 以供下载的第一步是创建一个筛选并清理后的数据集。我们建议执行以下步骤：

  1. 确保数据样本可以导出，并遵循数据导出控制规则。具体来说，您应验证导出是否符合您组织的数据治理策略。
  2. 对数据进行筛选，使其尽可能小以满足必要的目标。为了获得最佳性能，CSV格式中数据的未压缩大小应小于默认的HDFS块大小（128 MB）。为实现此目的，您应仅选择必要的列并最小化行数。您可以通过筛选特定值来减少行数，或采用任意行数的随机样本（例如1000行）。尝试创建大于128 MB的CSV可能会消耗更多时间，并可能需要额外的Spark执行器内存才能成功。
  3. 将列类型更改为string。由于CSV格式缺乏模式（没有强制的类型和列标签），建议将所有列转换为字符串。这对时间戳列尤其重要。

以下示例Python（PySpark）代码说明了上述一些原则在纽约市出租车行程数据集中的应用。

def prepare_input(my_input_df):
    from pyspark.sql import functions as F

    filter_column = "vendor_id"
    filter_value = "CMT"
    df_filtered = my_input_df.filter(filter_value == F.col(filter_column))

    approx_number_of_rows = 1000
    sample_percent = float(approx_number_of_rows) / df_filtered.count()

    df_sampled = df_filtered.sample(False, sample_percent, seed=0)

    important_columns = ["medallion", "tip_amount"]

    return df_sampled.select([F.col(c).cast(F.StringType()).alias(c) for c in important_columns])

使用类似的逻辑和Spark概念，您还可以在Pipeline Builder或其他Spark API（如SQL或Java）中实现准备操作。

减少到一个分区并设置输出格式

数据准备完成后，需要将数据重新分区或合并为一个分区，这样输出时就会有一个文件供下载。然后，将输出格式设置为CSV或其他支持的输出格式，例如JSON、ORC、Parquet或Avro。下面的例子展示了如何在Pipeline Builder、Python和SQL中对数据进行重新分区并设置输出格式。

repartition(1) 和 coalesce(1) 都会将数据减少到一个分区，但在作为相同转换的一部分执行过滤操作时，应避免使用 coalesce，因为 coalesce 会压缩上游任务，可能导致过滤操作以非分布式方式运行。

在上面的示例中，我们将数据集减少到一个分区，以便输出数据集正好有一个CSV文件可供下载。因此，所有数据必须能够放入一个Spark执行器的内存中，这就是我们建议首先对数据进行过滤或采样的原因。如果无法对数据进行过滤或采样，但数据太大而无法放入一个执行器的内存中，您可以使用适当的 Spark配置 来增加执行器内存。或者，您可以为分区数使用大于1的值。使用 repartition 并将值设为大于1会减少每个执行器必须在内存中保存的数据量，但也会导致输出数据集包含多个CSV文件，需要分别下载而不是一个。

访问用于下载的文件

数据集构建完成后，导航到数据集页面的详细信息选项卡。CSV文件应显示为可下载。
