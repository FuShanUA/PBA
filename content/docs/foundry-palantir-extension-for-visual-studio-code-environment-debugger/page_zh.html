调试 Python 环境

环境调试器有助于理解 Python 转换环境中的依赖关系，适用于 VS Code。当依赖项安装失败时，环境调试器会显示冲突并揭示根本问题，使您可以立即采取行动解决冲突。在安装成功的情况下，环境调试器允许用户了解将给定软件包引入环境的传递依赖。

可视化冲突

当遇到解决冲突时，会显示在环境调试器中解决选项。这由以下情况触发：

  • 使用库面板安装依赖项。
  • 在不可解决的环境中启动预览。

选择此选项会在 VS Code 中打开一个新标签页，以交互式图表可视化环境冲突。

请注意以下图形特征：

  • 该图的根是用户库。
  • 图中的节点对应于包版本需求。需求与其依赖的需求相连。
  • 冲突的包需求以红色突出显示。这些强调了冲突的根本问题。
  • 来自您库中的包标记为 user-defined（用户定义）。用户定义的包将显示可用的操作；例如，删除依赖项或更改版本。这使您可以在不离开环境调试器的情况下解决冲突。

可视化依赖来源

当 Python 环境成功解析后，环境调试器可以帮助追踪包的来源并了解是哪些依赖导致它们被添加到环境中。要查看包，请在 .maestro 文件夹中打开 hawk 锁文件。然后您可以选择一个包，在该包上方将显示 在环境调试器中查看 [包名称] 选项。选择此选项以打开环境调试器。

环境调试器包含一个图表，显示所选包与导入它的包之间的关系。所有导致该包被需要的传递路径及其相应版本都会显示出来。这有助于追踪所选包的来源，使用户能够可视化依赖于所选包和版本的导入。
