终端

Apollo 允许您直接从 Apollo Hub 在您的环境中启动交互式调试终端。 您可以使用终端通过运行 kubectl 命令来检查在 Kubernetes 中运行的容器的配置、健康状况和日志，以调试您的应用程序。 要启动调试终端，您必须在环境中拥有 Environment Operator 角色。 每个调试终端会话都有时间限制（四小时），并且与其他用户的终端会话隔离。

打开一个终端会话

要开始使用调试终端：

  1. 导航到您想要调试的 Apollo 环境并选择 终端 选项卡。



  2. 选择 新建终端会话 并确认该操作。



  3. 等待Apollo创建终端会话，如果您的环境没有缓存的容器镜像，这可能需要长达一分钟。



  4. 在新窗口中与终端会话进行交互。您将根据Apollo角色的作用域缩小Kubernetes RBAC。

    有关更多详细信息，请查看Kubernetes集群中的故障排除步骤。



  5. 会话的存活时间为四小时，当您关闭会话窗口时会自动删除。您可以在终端页面上每个会话旁边的菜单中选择删除终端会话来清理任何卡住的会话。

Kubernetes RBAC

在活动的调试终端会话中，您将对一组有限的Kubernetes资源（如Pods、Services和ConfigMaps）拥有只读访问权限。

授予的确切权限集将取决于您在Environment中部署的内容，但基本权限包括：
