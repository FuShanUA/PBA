变量

变量供模块构建者配置数据在 Workshop 模块中的流转方式。下方列出了可用的变量类型。

要访问 Workshop 模块中的现有变量或定义新变量，请打开左侧边栏中的变量菜单。

变量面板（如上图所示）会显示一个列表，包含模块内现有的变量、用于添加新变量的加号+选项、用于按名称或唯一 ID 搜索变量的输入框、用于打开变量血缘关系图的选项，以及用于根据定义类型或启用的设置来筛选变量的过滤器。变量列表包含分区，有助于快速找到相关变量：选中某个组件时，分区会显示该组件使用的变量；未选中任何组件时，分区会显示当前活动页面使用的变量。

从左侧列表中选择某个变量后，可以在变量列表右侧弹出的窗口中查看并修改该变量的配置。可用的配置选项如下：

  • 变量名称：选择变量名称或铅笔图标即可编辑变量名称。我们建议使用具有描述性的变量名称，这有助于记录 Workshop 模块的配置，并在后续的配置过程中更轻松地找到目标变量。
  • 变量定义类型：变量名称旁边是一个下拉菜单，提供了有关如何定义给定变量并为其填充数据的选项。可选选项会因所选的变量类型而异，可能包含以下选项：
    • 静态：用于手动设置的变量值
    • 函数：用于由函数支持、动态计算的变量
    • 对象集聚合：用于从对象集聚合派生而来的变量
    • 对象属性：用于绑定到单个对象所选属性值的变量
      • 注意：Workshop 不支持将复杂的非高性能对象属性作为变量。
    • 对象集定义：专用于通过选定对象类型、过滤器和链接对象遍历来定义的对象集变量
    • 变量转换：用于将变量值定义为一系列常见操作，可能会引用其他变量；请查阅变量转换文档以获取更多信息。
  • 删除变量（垃圾桶图标）：可以删除模块中未被使用（未被组件或下游变量使用）的变量。
  • 从当前变量新建（仅限对象集变量）：在复制变量按钮旁边，从当前变量新建按钮允许你创建一个新的对象集变量，该变量会自动将当前对象集作为其输入。当你在保持对源变量引用的同时，希望在现有对象集配置的基础上进行构建时，此功能非常有用。
  • 变量定义配置：屏幕的主要部分，用于配置变量的具体细节。此处提供的选项会根据所选的变量类型和变量定义类型而变化，并会提供屏幕操作指引。
  • 变量设置：变量设置面板允许配置外部 ID，并决定是否为给定变量启用以下功能。
    • 模块接口
    • 路由
    • 状态保存

变量延迟加载

在查看和编辑模式下，Workshop 变量仅在由可见组件或布局显示时才会进行延迟计算和重新计算。这意味着，在不可见页面、选项卡、覆盖层或循环布局的不可见页面中使用的变量，在显示之前不会进行计算。对于嵌入模块中使用的不可见变量，此行为同样适用。

变量类型

Workshop 支持以下变量类型：

  • 数组：接受由布尔值、日期、数字、地理坐标点、地理形状、字符串、时间戳或结构体组成的数组。
  • 布尔值：接受 true 或 false 值。通过静态值或函数、聚合或对象属性的输出进行初始化。
  • 日期：接受日期。通过静态值或函数、聚合或对象属性的输出进行初始化。
  • 地理点：接受地理点。通过对象属性进行初始化。
  • 地理形状：接受地理形状。通过对象属性或函数的输出进行初始化。
  • 数值：接受数值，包括整数和浮点数。通过静态值或函数、聚合或对象属性的输出进行初始化。
  • 对象集：存储一个或多个对象的集合。通过整个对象类型或其他对象集变量进行初始化，随后可选择按属性值或过滤变量进行过滤，或通过搜索Around选择性透视至关联对象。你也可以通过共享属性类型或对象支持链接类型，跨本体透视至其他对象类型。
  • 对象集过滤器：存储一组用于过滤对象集变量的属性类型/属性值对。通过对象类型以及一组属性类型/属性值过滤对进行初始化。有关更多信息，请参阅对象集过滤变量。
  • 字符串：接受文本。通过静态值或函数、聚合或对象属性的输出进行初始化。
  • 结构体：存储将字符串字段ID映射到值的复合类型。目前不支持嵌套结构体，但接受结构体字段值使用任何其他 Workshop 变量类型。通过函数的输出进行初始化。有关更多信息，请参阅结构体变量。
  • 时间戳：接受时间戳。通过静态值或函数、聚合或对象属性的输出进行初始化。
  • 时间序列集：存储单个对象的时间序列属性，并允许选择性地对其应用时间序列转换。有关更多信息，请参阅 Workshop 中的时间序列属性。

重新计算变量值

Workshop 为以下变量定义类型提供了可配置的重新计算行为选项：

  • 函数
  • 对象集聚合
  • 对象属性
  • 变量转换
  • 对象集过滤器

重新计算选项的行为方式如下：

  • 自动：当其依赖的任何变量值发生变化时，变量值会自动重新计算。这是默认选项。请注意，在某些情况下，具有自动重新计算行为的变量可能会在没有上游值发生变化时也进行重新计算。这种情况可能发生在上游对象重新加载时，例如在提交操作或其他数据重新加载触发器（如自动刷新）之后。
  • 仅当由事件触发时：变量值仅在由重新计算 {变量名} 事件明确触发时才进行重新计算。
  • 在模块加载时以及由事件触发时：变量值在模块初始加载时进行重新计算，并在由重新计算 {变量名} 事件明确触发时再次进行重新计算。

对象集定义 变量定义类型不支持重新计算行为配置，其功能类似于 自动 重新计算行为。如果不希望如此，可以在上游变量上设置重新计算行为。如果你希望显式管理对象集变量的重新计算行为，可以使用函数支持的变量。

变量血缘关系图

使用 变量 面板头部的 变量血缘关系图 选项，可以可视化模块中变量和微件之间的相互依赖关系。利用它可以调试重新计算行为，追踪哪些微件读取或写入变量，并更好地理解应用元素之间的复杂关系。

展开图形

图中的每个节点代表一个变量或微件。具有依赖关系的节点在其顶部和底部边缘有箭头标志。选择箭头可以展开节点的父节点（上游依赖）或子节点（下游消费者），从而追踪模块中的关系链。使用图形头部的 显示全部 操作可以展开至完整的应用图形，或选择 清除 来移除所有节点。

图头部的撤销和重做选项可以回溯和前进展开、折叠及选择操作，这与模块自身的撤销/重做控件功能一致。

每个节点可以显示变量被引用的页面和叠加层，以及变量计算完成的时间。

页面与叠加层

节点会标记其被引用的页面和叠加层。在头部切换 Show pages and overlays 即可显示图例，列出正在使用的每个页面和叠加层。该图例仅列出可见图节点上出现的页面和叠加层，因此列表范围始终与图中可见内容保持一致。

计算时间

在头部切换 Show computation time 即可显示每个变量的耗时信息。计算时间是指应用程序中变量值最后一次计算的时间（即用户所见的时间），而非应用程序外部底层数据最后一次更新的时间。重新计算耗时较长的变量可能需要重构：例如，将复杂的函数支持变量拆分为更小的部分，或者更改上游变量的重新计算行为，以避免不必要的开销。
