变量

变量 由模块构建者使用，以配置数据在 Workshop 模块中的流动方式。下面列出了可用的 变量类型。

要访问 Workshop 模块中的现有变量或定义新变量，请打开左侧栏中的 变量 菜单。

变量 面板（如上所示）显示了模块中当前存在的变量列表，一个用于添加新变量的加号 + 选项，一个通过名称或唯一 ID 搜索变量的输入框，以及一个打开 变量谱系图 的选项。 ，以及一个过滤器，可以根据变量的定义类型或启用的设置来显示变量。变量列表包含分区，帮助您快速找到相关的变量：当选择一个部件时，分区会显示该部件使用的变量；当没有选择任何部件时，分区会显示当前页面中使用的变量。

从左侧的列表中选择一个变量，可以在变量列表右侧打开一个窗口，查看和修改该变量的配置。以下配置选项可用：

  • 变量名称： 选择变量名称或铅笔图标将允许您编辑变量的名称。我们建议使用描述性的变量名称，以帮助记录您的 Workshop 模块的配置，并在配置过程中更容易找到所需的变量。
  • 变量定义类型： 在变量名称旁边有一个下拉菜单，提供有关如何定义和填充数据的选项。可用的选择将根据所选变量类型而有所不同，可能包括以下选项：
    • 静态: 用于手动设置的变量值
    • 函数: 用于由函数支撑的动态计算的变量
    • 对象集聚合: 用于从对象集聚合中派生的变量
    • 对象属性: 用于与单个对象的选定属性值相关联的变量
      • 注意:复杂且性能不佳的对象属性在Workshop中不支持作为变量。
    • 对象集定义: 专门用于由所选对象类型、过滤器和链接对象遍历定义的对象集变量
    • 变量转换: 用于将变量值定义为一系列常见操作，可能引用其他变量；请查阅 变量转换 文档以获取更多信息。
  • 删除变量（垃圾桶图标）: 在模块中未被使用（通过小部件或下游变量）的变量可以被删除。
  • 从当前创建新变量（仅限对象集变量）: 在复制变量按钮旁边，从当前创建新变量 按钮允许您创建一个新的对象集变量，该变量会自动将当前对象集作为其输入。当您希望在现有对象集配置的基础上进行构建，同时保持对源变量的引用时，这非常有用。
  • 变量定义配置： 屏幕的主要部分，用于配置变量的详细信息。此处提供的选项将根据所选的变量类型和变量定义类型而变化，并将显示屏幕上的说明。
  • 变量设置： 变量设置面板允许配置外部ID以及为给定变量启用以下哪些功能。
    • 模块接口
    • 路由
    • 状态保存

延迟变量加载

在视图模式和编辑模式下，Workshop 变量仅在被可见的 widget 或 layout 显示时才会进行计算和重新计算。这意味着在非可见页面、标签页、覆盖层或循环布局的非可见页面中使用的变量不会被计算，直到它们被显示出来。这种行为对于嵌入式模块中使用的非可见变量也是一样的。

变量类型

Workshop 支持以下变量类型：

  • 数组: 接受布尔值、日期、数字、地理点、地理形状、字符串、时间戳或结构值的数组。
  • 布尔值: 接受 true 或 false 值。从静态值或函数、聚合或对象属性的输出初始化。
  • 日期： 接受一个日期。可以从静态值或函数、聚合或对象属性的输出初始化。
  • 地理点位： 接受一个地理点位。从对象属性初始化。
  • 地理形状： 接受一个地理形状。从对象属性或函数的输出初始化。
  • 数值： 接受数值，包括整数和浮点数。可以从静态值或函数、聚合或对象属性的输出初始化。
  • 对象集合： 存储一个或多个对象的集合。可以从整个对象类型或其他对象集合变量初始化，然后可以通过属性值或过滤器变量进行可选过滤，或者通过“搜索周边”操作转换为关联对象。您还可以通过 共享属性类型 或 基于对象的关联类型 跨本体论转换到其他对象类型。
  • 对象集合过滤器： 存储用于过滤对象集合变量的属性类型/属性值对。从对象类型初始化，然后设置一组属性类型/属性值过滤器对。参见 对象设置过滤器变量以获取更多信息。
  • 字符串：接受文本。从静态值或函数、聚合或对象属性的输出初始化。
  • 结构体：存储一个映射字符串字段ID到值的复合类型。目前不支持嵌套的结构体，但结构体字段的值可以接受任何其他Workshop变量类型。从函数的输出初始化。请参阅 结构体变量 以获取更多信息。
  • 时间戳：接受时间戳。从静态值或函数、聚合或对象属性的输出初始化。
  • 时间序列集：存储单个对象的时间序列属性，可选择对其应用时间序列转换。有关更多信息，请参阅 Workshop中的时间序列属性。

重新计算变量值

Workshop 为变量定义类型提供了以下可配置的重新计算行为选项：

  • 函数
  • 对象集聚合
  • 对象属性
  • 变量转换
  • 对象集过滤

重新计算选项的行为如下：

  • 自动: 当该变量所依赖的任何变量的值发生变化时，变量值会自动重新计算。这是默认选项。请注意，在某些情况下，具有自动重新计算行为的变量可能在没有上游值更改时也会重新计算。例如，当上游对象重新加载时（如操作提交后或其他数据重新加载触发器，如自动刷新）。
  • 仅当由事件触发时: 变量值仅在通过 重新计算 {variable name} 事件显式触发时才重新计算。
  • 在模块加载时以及由事件触发时: 当模块最初加载时，以及当通过 重新计算 {variable name} 事件显式触发时，变量值会重新计算。

Object set definition 变量定义类型不提供重新计算行为配置，并且类似于 Automatic 重新计算行为。如果这不希望发生，可以在上游变量上设置重新计算行为。如果您希望显式管理对象集变量的重新计算行为，可以使用函数支持的变量。

变量血缘图

在 Variables 面板的标题中找到 Variable lineage graph 选项，以可视化您的模块中变量和小部件之间的依赖关系。用它来调试重新计算行为，追踪哪些小部件读取或写入一个变量，并更好地理解应用程序元素之间的复杂关系。

扩展图谱

图谱中的每个节点代表一个变量或小部件。具有依赖关系的节点在其顶部和底部边缘有箭头符号。选择一个箭头以展开节点的父级（上游依赖项）或子级（下游消费者），并通过模块追踪一系列关系。在图谱标题中使用显示所有操作，以展开到完整的应用程序图谱，或使用清除来移除所有节点。

图谱标题中的撤销和重做选项可以回退和前进通过展开、折叠和选择操作，与模块自身的撤销/重做控件相匹配。

每个节点可以显示变量被使用的页面和覆盖层，以及变量被计算的时间。

页面和覆盖层

节点会标记它们被引用的页面和覆盖层。在标题中切换显示页面和覆盖层，以显示一个图例，列出正在使用的每个页面和覆盖层。该图例只列出可见图节点上出现的页面和覆盖层，因此列表范围仅限于您在图中可以看到的内容。

计算时间

在标题中切换显示计算时间，以显示每个变量的时间信息。计算时间指的是变量值在应用程序中最后一次被计算的时间（对用户而言），而不是在应用程序外部数据最后一次更新的时间。计算时间较长的变量可能需要重新设计：例如，将一个基于函数的复杂变量拆分为更小的部分，或改变上游变量的重新计算行为以避免不必要的工作。
