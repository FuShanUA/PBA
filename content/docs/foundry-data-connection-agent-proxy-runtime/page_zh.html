智能体代理配置参考

本页面介绍了在 Foundry worker 连接中，通过智能体代理出站策略将智能体用作网络代理时可用的配置选项。阅读本页内容前，假定您已熟悉数据连接智能体及其通用配置选项。

用作代理的智能体可为 Foundry 中的服务提供网络路径，使其能够访问 Foundry 网络无法直接触达的独立网络内的系统。 当 Foundry 部署在云端时，必须通过代理才能访问无法经互联网访问的系统。当 Foundry 部署在私有网络中时，若需访问其他网络中的系统（在无法直接访问的情况下可能包括公共互联网），同样需要借助代理。

智能体代理使得 Foundry 中的应用能够像直接通过 Foundry 网络连接数据源一样运作，并且支持配置多个智能体以错开维护窗口，避免停机。请按照以下指南配置和管理用作代理的智能体。

将智能体作为代理用于数据连接源

将智能体用作代理前，必须先进行设置。已经用于 agent worker 源的现有智能体可直接复用，无需额外配置。

设置好智能体后，即可使用该智能体创建智能体代理出站策略。随后，您可以创建源并将该策略分配给它。

智能体代理的专属安全控制

为确保通过代理建立的连接仅访问所需资源，系统提供了多种安全控制手段。

安全控制分为两类：一类是强制且无法禁用的，另一类是可选且默认不启用的。

可用的安全控制配置如下：

安全控制是否必选？描述
源配置强制执行是智能体代理始终将流量限制在所分配源上配置的主机名和端口范围内。
智能体允许列表 (Foundry)可选在 Foundry 中配置允许列表，在流量到达智能体之前进行过滤。仅智能体资源数据主人可修改此设置。
智能体允许列表（本地文件）可选在智能体主机上创建允许列表，供本地智能体进程在您的网络中执行。仅拥有智能体主机 SSH 访问权限的用户可修改此设置。
智能体主机防火墙可选设置主机级防火墙控制，限制智能体仅与预期目标系统通信（强烈建议）。这些防火墙独立于 Foundry 运行，可提供额外的安全层。

源配置强制执行

连接智能体代理时，源 URL 中定义的主机名和端口将访问范围限制在该主机名和端口内。尝试连接任何其他主机名或端口都将导致代理返回 HTTP 403（未授权）响应码。

这能在代码中导入和使用该连接时防止发生未授权的连接。

若仅使用此项安全控制，请确保获准将智能体分配给特定源的用户，同样受到信任，能够连接到自智能体主机可触及的任何系统。

智能体允许列表 (Foundry)

在 Foundry 中配置允许列表，以限制特定智能体仅能连接到特定的 IP 地址或 CIDR 块，无论已分配的源如何。

如需进行配置，请导航至智能体设置，然后在管理配置部分切换高级选项。在 YAML 文件中，于与 security 块相同的缩进级别添加一个 agentProxyConfiguration 块。

列出您希望允许智能体连接的 CIDR 块和端口。例如：


```yaml
agentProxyConfiguration:
  allowListedCidrs:
    - cidrBlock: '192.168.1.1/32'
      port: 7000
      endPort: 9000
    - cidrBlock: '192.168.2.2/24'
      port: 443
```

### 智能体白名单（本地文件）

用于在 Foundry 中配置智能体白名单的同一套配置，也可以直接放在智能体宿主机上的文件中，从而防止 Foundry 中的用户编辑该配置。

要在本地配置智能体白名单，请按照以下步骤操作：

通过 SSH 登录到智能体的宿主机。
在包含 `magritte-bootvisor-` 的目录（根目录）下，创建一个名为 `agentProxyConfig` 的文件夹。
在新创建的 `agentProxyConfig` 文件夹中，创建一个名为 `agentProxyConfig.yml` 的文件。
`agentProxyConfig.yml` 文件和 `agentProxyConfig` 文件夹*必须*由 root 用户创建，并且必须设置文件权限以防止智能体写入此文件或文件夹。如果 `agentProxyConfig.yml` 文件存在且可写，或者其某个父目录可被智能体写入，智能体将*不会*运行智能体代理功能。
此文件的内容与智能体设置界面中提供的配置相同。

例如：

```yaml
allowListedCidrs:
  - cidrBlock: '192.168.1.1/32'
    port: 7000
    endPort: 9000
  - cidrBlock: '192.168.2.2/24'
    port: 443
```

智能体主机防火墙


与任何智能体一样，我们建议在主机上设置防火墙，将连接限制在必要范围内。


连接池配置


智能体代理维护一个用于处理请求的连接池。您可以根据工作负载配置连接池大小，从而优化性能。


要配置连接池，请导航至 智能体设置，然后在 管理配置 部分切换 高级 选项。在 YAML 文件中，将连接池设置添加到 agentProxyConfiguration 块内：


已复制！
1agentProxyConfiguration:
2  maxConnections: 250
3  coreConnections: 100






















选项默认值描述
maxConnections300智能体代理可打开的最大连接数。
coreConnections20连接池中维护的最小连接数。





部分连接器（例如 Databricks 平台）可能会同时打开大量连接。如果在使用智能体代理策略时观察到智能体出现连接池耗尽错误，建议考虑调大 maxConnections 和 coreConnections 的值。
