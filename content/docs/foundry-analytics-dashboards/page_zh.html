仪表板

基于 Foundry 分析结果构建交互式仪表板主要有两种方式：Contour 仪表板 和 Quiver 仪表板。在特定场景下，您也可以选择使用 代码工作空间 (Code Workspaces) 仪表板、记事本 (Notepad) 或 自定义应用。本页将探讨可用的工具及其适用场景。

了解有关 Contour 与 Quiver 区别的更多信息。

Contour 仪表板

Contour 仪表板用于展示 Contour 分析的内容，例如分析结果和发现。这些仪表板支持图表间联动过滤、内联参数引用、全屏演示视图以及 PDF 导出功能。

Quiver 仪表板

Quiver 仪表板用于以只读的交互式仪表板形式展示 Quiver 分析的内容。

Quiver 仪表板既可以独立使用，也可以嵌入到 对象视图 (Object Views)、Workshop 应用 或 记事本文档 (Notepad documents) 中。Quiver 仪表板还可以在 Carbon 中展示，并直接推送给一线业务用户。

了解有关 Quiver 及 Quiver 仪表板的更多信息。

代码工作空间 (Code Workspaces) 应用

对于 Python 应用，代码工作空间目前支持 Dash ↗ 和 Streamlit ↗；对于 R 应用，则支持 Shiny® ↗。用户可以直接在代码工作空间中创建应用工作流，并使用 Foundry 内置的版本控制、分支管理以及数据治理功能。

其他展示工具

记事本 (Notepad)

Contour 仪表板和 Quiver 模板最适合直接在 Foundry 中使用，其交互式设计允许用户通过参数和图表选择来处理实时数据。相比之下，记事本专为侧重于创建可导出静态内容的报告工作流而设计。此外，记事本文档可以包含来自多个应用的内容（包括 Contour 看板和 Quiver 卡片），而 Contour 仪表板和 Quiver 模板仅支持各自应用内的内容。由于侧重于静态内容，用户无法直接在记事本中进行分析或创建图表。

了解有关报告工作流的更多信息。

自定义应用

Contour 仪表板和 Quiver 模板经过优化，能够轻松以交互方式展示分析结果，而应用构建工具（如 Workshop）在某些场景下可能更为合适。具体而言，如果您的使用场景需要完全自定义和灵活的布局、多步骤工作流或数据回写，则应考虑使用 Foundry 的 应用构建工具。

Quiver 仪表板可以嵌入到 Workshop 应用中，从而为一线业务用户提供更复杂的分析内容。
