添加对象

对象类型是本体的核心元素，用于定义现实世界实体或事件的模式。在本体中，对象是某个对象类型的单一实例，对应现实世界中的单个实体或事件。与单个对象相对，对象集是指多个对象实例的集合，代表一组现实世界中的实体或事件。

Quiver 原生支持多种卡片，用于对对象数据进行过滤、转换和可视化。分析对象数据的首要步骤是将对象添加到分析中。

本文档介绍了将对象数据添加到 Quiver 分析中的几种不同方法。熟悉添加对象的操作后，你可以进一步了解如何过滤对象以及搜索关联的对象。

添加对象类型

将对象数据添加到 Quiver 分析中最简单且最常用的方式，是添加给定对象类型下的所有对象。你可以在屏幕中央选择添加对象来完成此操作。随时可以通过分析顶部栏的添加数据部分中的对象按钮，添加额外的对象数据。

选择添加对象会打开 Quiver 的搜索栏，以便在本体中浏览对象类型。对象类型搜索菜单支持关键字检索。搜索菜单左侧显示了对象类型分组标签页，以及常用对象类型和时序对象类型的标签页。

选择一个对象类型将其添加到分析中。添加对象类型后，选择右上角的 X 关闭搜索栏。此时分析中会出现一个对象集卡片，包含所选类型的所有对象。该卡片会展示集合内对象的表格预览，并在右上角显示所有对象的总数。

添加单个对象

你也可以将单个对象添加到分析中。操作方法如下：打开对象类型搜索菜单后，在菜单顶部切换到对象标签页。在对象标签页下，可以通过属性过滤或关键字，在特定对象类型中搜索单个对象。在结果表格中选中某个对象，即可将其作为单个对象卡片添加到分析中。

从对象浏览器中打开对象

对象分析通常从对象浏览器中的探索开始。如果你已经在对象浏览器中开始了工作流，可以通过选择浏览界面右上角的在 Quiver 中分析按钮，无缝切换至 Quiver。这会新建一个 Quiver 分析，其中包含你正在浏览的对象。

导入已保存的对象集

将对象集添加到 Quiver 分析的另一种方法是导入已保存的对象集。操作时，在对象类型或对象搜索菜单的右上角选择导入对象集按钮。这会打开文件选择器，允许你选择已保存的对象集并将其添加到分析中。

使用 URL 预加载对象

新建分析时，也可以通过 URL 预加载指定的对象或对象集。可通过以下端点实现：

  • 对象：/quiver/views/load/object/<object_rid>
  • 对象集：/quiver/views/load/object-set/<object_set_rid>
  • 对象类型：/quiver/view/load/object-type/<object_type_id>
