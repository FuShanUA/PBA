虚拟表上的 Python 转换

虚拟表允许您查询并写入受支持数据平台中的表，而无需将数据存储在 Foundry 中。您可以使用 transforms-tables 库在 Python 转换中与虚拟表进行交互。

Foundry 支持虚拟表的两种计算模式：

  1. Foundry 原生 Spark 计算：可与来自一个或多个数据源的虚拟表输入或输出配合使用，并与 Foundry 原生数据集的输入和输出组合使用。请注意，虚拟表不支持 Foundry 中的 轻量级计算引擎，例如 Polars 和 Pandas。
  2. 计算下推：在此模式下，Foundry 会通过轻量级编排，在源系统中执行外部计算。要使用计算下推，所有输入和输出必须是与同一数据源关联的虚拟表，且该数据源必须兼容计算下推。

使用计算下推时，目前不支持使用 @incremental 装饰器进行增量计算。

前提条件

要在 Python 转换中使用虚拟表，您必须：

  1. 升级您的 Python 仓库至最新版本。
  2. 从库选项卡安装 transforms-tables。

使用 use_external_systems 装饰器的转换与虚拟表不兼容。请改用基于源的外部转换，或者将您的转换拆分为多个转换：一个使用虚拟表，另一个使用 use_external_systems 装饰器。

在代码仓库中使用虚拟表时，使用这些虚拟表的转换将根据在数据源上配置的出站策略自动获取网络出站权限。

必须在数据源上启用以下设置：

  • 代码导入：允许将数据源导入并在代码仓库中使用。有关此设置的更多信息，请参阅将数据源导入代码的文档。
  • 导出控制：控制允许哪些安全标记和组织作用于使用虚拟表的 Python 转换的输入。有关更多信息，请查阅在数据源上配置导出控制的文档。

在 Python 转换中作为输入和输出使用的表无需来自同一数据源，甚至无需来自同一平台。默认情况下，Python 转换将使用 Foundry 的计算能力来读写表，这允许跨不同的外部系统进行查询。有关将计算完全联邦化至外部系统的详细信息，请参阅计算下推部分。

请记住使用侧边栏将输出数据源导入您的仓库。

以下示例使用 Snowflake 平台 数据源，该源接收虚拟表输入，并使用 Foundry 原生 Spark 计算生成虚拟表输出。

JDBC 分区

通过 JDBC 连接读取虚拟表数据时，可以在 Python 转换中配置分区，以实现并行读取并提升表加载性能。

以下示例展示了在加载 input_table 时添加分区的操作。该分区基于 column_name 列，将在下界 1 和上界 100 之间划分为三个等大的分区。

Copied!
1@transform(
2    input_table=TableInput("/path/input_table_name"),
3    output_df=Output("/path/output_dataset_name")
4)
5def compute(input_table: TableTransformInput, output_df: TransformOutput):
6    input_df = input_table.dataframe(
7        options={
8            "numPartitions": "3",
9            "partitionColumn": "column_name",
10            "lowerBound": "1",
11            "upperBound": "100"
12        }
13    )
14
15    output_df.write_dataframe(input_df)

调用 .dataframe() 加载虚拟表输入时，应通过 options 参数传递分区配置。partitionColumn 必须是数字、日期或时间戳列。数据将分配到由 numPartitions 指定数量的分区中。这些分区将在 lowerBound 和 upperBound 选项之间形成大小均匀的区间。

请注意，lowerBound 和 upperBound 仅用于计算分区步长，不用于过滤行。表中的所有行都将返回，包括值超出指定边界的行。

有关可用分区选项的更多详细信息，请参阅 《Spark JDBC 文档》 ↗。

文件模板配置向导 [Beta]

Beta

文件模板配置向导中的虚拟表输出目前处于 Beta 开发阶段。在活跃开发期间，其功能可能会发生变化。

可以在 《代码仓库文件模板配置向导》 中使用 virtual table 模板变量类型来配置虚拟表输入和输出。创建虚拟表输出时，向导将引导您选择要写入的输出源以及虚拟表的 Foundry 位置。
