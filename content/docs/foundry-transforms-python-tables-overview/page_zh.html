Python 在虚拟表上的转换

虚拟表 允许您查询和写入支持的数据平台中的表，而无需在 Foundry 中存储数据。您可以使用 transforms-tables 库在 Python 转换中与虚拟表进行交互。

Foundry 支持虚拟表的两种计算模式：

  1. Foundry 原生 Spark 计算： 这可用于一个或多个来源的虚拟表输入或输出，并可以与 Foundry 原生数据集输入和输出组合在一起。请注意，虚拟表不支持 Foundry 中的轻量级计算引擎，如 Polars 和 Pandas。
  2. 计算下推： 在此模式下，通过轻量级计算，Foundry 在源系统中协调外部计算。为了使用计算下推，所有输入和输出必须是与同一源相关联的虚拟表，并且源必须兼容计算下推。

使用 @incremental 装饰器的增量计算目前在使用 compute pushdown 时不受支持。

前提条件

要在 Python 转换中使用虚拟表，您必须：

  1. 升级您的 Python 代码仓库到最新版本。
  2. 从 库选项卡 安装 transforms-tables。

使用 use_external_systems 装饰器的转换不与虚拟表兼容。请改用 基于源的外部转换 或将您的转换拆分为多个转换，其中一个使用虚拟表，另一个使用 use_external_systems 装饰器。

当在代码仓库中使用虚拟表时，消耗它们的转换将根据源上配置的 出站策略 自动获得网络出口。

以下设置必须在源上启用：

  • 代码导入： 这允许源被导入并用于代码仓库中。有关此设置的更多信息，请参阅文档中的 将源导入代码。
  • 导出控制： 这控制哪些 安全标记 和组织将被允许在使用虚拟表的 Python 转换输入上。有关更多详细信息，请查阅文档中关于 在源上配置导出控制 的内容。

在 Python 转换中用作输入和输出的表不需要来自同一来源，甚至不需要来自同一平台。默认情况下，Python 转换将使用 Foundry 的计算来读取和写入表，这允许跨不同外部系统进行查询。请参阅 计算下推 部分，了解有关将计算完全联合到外部系统的详细信息。

快速入门示例

请使用侧边栏将输出源导入到您的仓库中。

以下是使用 Snowflake 源的示例，该源采用虚拟表输入，并使用 Foundry 原生 Spark 计算生成虚拟表输出。

Copied!
1from transforms.api import transform
2from transforms.tables import TableInput, TableOutput, TableTransformInput, TableTransformOutput, SnowflakeTable
3
4
5@transform.spark.using(
6    input_table=TableInput('/path/input_table_name'),
7    output_table=TableOutput(
8        '/path/output_table_name',  # Virtual table output in Foundry
9        'ri.magritte..source.123...',  # The output source RID
10        SnowflakeTable('DATABASE', 'SCHEMA', 'TABLE'),  # The output storage location
11    ),
12)
13def compute(input_table: TableTransformInput, output_table: TableTransformOutput):
14    output_table.write_dataframe(input_table.dataframe())

Foundry 数据集也可以与虚拟表结合使用。下面的示例采用 Foundry 数据集作为输入，并将虚拟表输出写入外部源。

Copied!
1from transforms.api import transform, Input, TransformInput
2from transforms.tables import TableOutput, TableTransformOutput, SnowflakeTable
3
4
5@transform.spark.using(
6    input_dataset=Input('/path/input_dataset_name'),
7    output_table=TableOutput(
8        '/path/output_table_name',  # Virtual table output in Foundry
9        'ri.magritte..source.123...',  # Output source RID
10        SnowflakeTable('DATABASE', 'SCHEMA', 'TABLE'),  # Output storage location
11    ),
12)
13def compute(input_dataset: TransformInput, output_table: TableTransformOutput):
14    output_table.write_dataframe(input_dataset.dataframe())

最后，下面是使用计算下推的虚拟表输入和输出的示例。

Copied!
1from snowflake.snowpark.functions import lit
2import snowflake.snowpark as snow
3from transforms.api import transform
4from transforms.tables import (
5    SnowflakeTable,
6    TableInput,
7    TableOutput,
8    SnowflakeInput,
9    SnowflakeOutput,
10)
11
12@transform.snowflake.using(
13    input_table=TableInput('/path/input_table_name'),
14    output_table=TableOutput(
15        '/path/output_table_name',  # Virtual table output in Foundry
16        'ri.magritte..source.123...',  # Output source RID
17        SnowflakeTable('DATABASE', 'SCHEMA', 'TABLE'),  # Output storage location
18    ),
19)
20def compute_in_snowflake(input_table: SnowflakeInput, output_table: SnowflakeOutput):
21
22    # Get a Snowpark DataFrame instance
23    df: snow.DataFrame = input_table.dataframe()
24
25    # Example data transformation - create a new column
26    df = df.with_column("NEW_COLUMN", lit("ABC"))
27
28    # Write back to the new table
29    output_table.write(df)

JDBC 分区

当使用 JDBC 连接读取虚拟表数据时，您可以在 Python 转换中配置分区以并行读取并提高加载表的性能。

以下是一个在加载 input_table 时添加分区的示例。这将基于 column_name，该值将在下限为1和上限为100之间分成三个等大小的分区。

Copied!
1@transform(
2    input_table=TableInput("/path/input_table_name"),
3    output_df=Output("/path/output_dataset_name")
4)
5def compute(input_table: TableTransformInput, output_df: TransformOutput):
6    input_df = input_table.dataframe(
7        options={
8            "numPartitions": "3",
9            "partitionColumn": "column_name",
10            "lowerBound": "1",
11            "upperBound": "100"
12        }
13    )
14
15    output_df.write_dataframe(input_df)

分区配置应通过 options 参数传递给 .dataframe()，以加载虚拟表输入。partitionColumn 必须是数值、日期或时间戳列。数据将被分配到由 numPartitions 指定数量的分区中。这些分区将是 lowerBound 和 upperBound 选项之间的均匀大小范围。

请注意，lowerBound 和 upperBound 仅用于计算分区步长，而不是用于过滤行。表中的所有行都将返回，包括那些值超出指定范围的行。

有关可用分区选项的更多详细信息，请参阅 Spark JDBC 文档 ↗。

测试版

文件模板配置向导中的虚拟表输出处于开发的 测试版 阶段。在积极开发期间，功能可能会发生变化。

虚拟表输入和输出可以在 代码仓库文件模板配置向导 中使用 virtual table 模板变量类型进行配置。创建虚拟表输出时，向导会引导您选择要写入的输出 源 以及虚拟表的 Foundry 位置。
