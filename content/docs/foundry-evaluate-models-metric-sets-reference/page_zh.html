计划弃用

指标集是为使用 foundry_ml 的数据集支持模型构建的，该库已于 2025 年 10 月 31 日起正式弃用。对于新实现，建议改用《实验》。虽然对于使用 palantir_models 构建的模型，指标集不会显示在模型页面上，但仍然可以针对模型编写指标集，并在《建模目标》中查看指标。

指标集作为命名指标的容器，我们将这些指标定义为模型输出的任何汇总。支持的指标包括数值、图表和图像。

在 Python 中，指标集通过 foundry_ml_metrics 包中的 MetricSet 类实现（要求环境配置中包含 foundry_ml）。

所有指标都可以通过 MetricSet.add 方法添加，该方法包含以下参数：

  • name 指标的字符串名称
  • value 要添加的指标值
  • subset 一个 dict<str, str>，用于描述该指标所对应的数据子集
  • stage Foundry ML 模型阶段或阶段 uuid，用于将其绑定到特定阶段，默认为最后一个模型阶段

下面是一个创建空指标集的示例，用于说明该概念。

创建时，必须向 MetricSet 传递 model 和 input_data 参数。

输入数据集


输入数据集明确引用了模型所应用的具体数据版本，随后便基于该版本计算各项指标。
通过手动追踪这些数据集，你可以建立模型性能的记录并保留完整的溯源链。在“建模目标”中相互对比评估多个模型时，此功能必不可少，它能确保所有模型都在相同的数据上进行评估。


输入数据集必须以 Python transform input 类型传入 MetricSets，才能获取相应的元数据。注意：Code Workbook 仅支持对导入的数据集执行此操作。按照最佳实践，我们必须在单独的工作簿中进行验证，与测试/训练集的拆分分开进行。




指标类型


数值型指标


数值型指标是最基础的指标类型，通常是分析师了解模型性能的第一步洞察。
数值型指标就是简单的 Python int 或 float 类型，直接添加到指标集中即可。

验证数据子集


模型的性能通常会在数据集的不同子集上存在差异。
为了便于分析这一问题，MetricSet.add 方法支持传入一个 subset 参数，这是一个字符串到字符串的 Python 字典，用于描述您所应用的任何过滤条件。为过滤条件赋予语义化的名称，有助于用户理解模型在代表实际业务问题的不同数据切片上的性能表现。


指标和子集将显示在“建模目标”中。


例如，在经典的开源鸢尾花数据集上，您可能需要针对低于平均水平和高于平均水平的花瓣长度分别计算模型的准确率。


可以通过以下方式实现：

更新指标


指标集以数据集的形式表示，并通过 Foundry 构建进行计算。当模型原地更新，或有新的输入数据版本可用时，必须重新构建指标集，因为它与特定版本的模型和输入数据集相关联。


需要注意的是，如果更新并重新运行模型，还需要重新构建关联的指标集，以避免信息滞后，因为它们与特定的模型（和输入数据集）版本绑定。否则，你将无法在模型预览中看到指标。
