可观测性

Palantir 平台提供内置工具，用于监控资源的健康状况、在开发和生产环境中调试问题、跨服务跟踪执行，并大规模分析遥测数据。

监控: 您可以使用 数据健康 工具来监控平台。通过数据健康，您可以设置规则并定义失败、延迟等的阈值；为每个资源或跨项目按规模配置监视器；通过 PagerDuty、Slack、webhook 或 Foundry 通知接收警报；并查看过去 30 天的执行次数和 P95 延迟 指标。

调试: Workflow Lineage 工具可让您探索和调查平台的历史和日志。您可以查看七天的 执行历史记录，按状态、用户、持续时间或版本进行过滤，并准确找到需要关注的执行内容。Workflow Lineage 还允许您 跨所有执行的日志进行搜索，以查找特定的日志消息、错误或模式。

追踪: 要可视化跨函数、操作和LLM调用的完整请求流程，您可以使用 追踪视图。追踪视图使您能够深入查看任何操作，以查看持续时间、输入、输出和错误。

分析: 为了对日志数据进行进一步分析，您可以 导出 Foundry 日志、指标和追踪信息到流数据集，以支持您自己的仪表板、管道或自定义可观测性工作流程。

监控

监控工具可以帮助您跟踪资源随时间的健康状况和稳定性，主动检测问题，并在出现问题时接收警报。

数据健康 是用于监控平台资源健康的主应用程序。数据健康提供两个功能集：

  • 监控视图: 您可以使用基于范围的监控规则，在项目、文件夹、应用或单个资源上大规模监控Foundry资源。
  • 健康检查: 可以在单个资源上配置详细的健康检查，包括数据集、计划和表的内容和模式验证。

监控视图和健康检查在检测到问题时会生成警报。警报可以通过Foundry通知或通过外部系统（如PagerDuty或Slack）传递。

指标

Foundry 提供了多种资源类型的指标，帮助您随时间监控健康状况和性能。

  • Functions, actions 和 AIP Logic: 通过 本体论管理器 或 工作流谱系 查看过去 30 天内接近实时的成功/失败计数和 P95 执行持续时间。
  • 流和计算模块： 在 指标 选项卡中访问指标，以监控长时间运行的计算工作负载的健康状况和稳定性。

调试

调试工具可以帮助您在开发和生产环境中调查问题，提供执行细节、日志和跟踪信息的可见性。

AIP 可观测性

AIP 可观测性 功能在 工作流血缘 中启用，使您能够全面了解 AIP 和 本体论 工作流的执行情况。使用 AIP 可观测性功能来了解代理、函数、语言模型、自动化、操作和 本体论 的性能。

AIP 可观测性功能可用于查看 指标、执行历史、分布式追踪、日志记录 和 日志搜索。

日志

您可以查看用于运行代码的第三方库（例如使用流时的 Kafka）的日志，以及您的代码发出的日志。日志可在多种类型的作业中使用：

  • 批量和流转换: 可以在转换的作业报告中查看实时日志。
  • 计算模块: 可以在计算模块概览页面查看日志。
  • 函数: 可以在工作流血缘中查看日志。

You can also search across logs from all executions for a source executor to find specific log messages, errors, or patterns. A source executor is the first executable resource in the call chain, such as a function, action, automation, AIP Logic, AIP agent, or model live deployment.

Log export

Log exporting may not be available in your enrollment. Contact Palantir Support for more information.

To allow for arbitrary processing outside the current capabilities of in-platform tools, you can create a stream in a specified folder containing all telemetry for an organization. This includes logs, metrics, and traces. Data in this stream can be analyzed using Foundry's suite of data analysis tools or exported to third-party systems.

Learn more about exporting logs to a stream on the Configure logging page.
