自定义函数

自定义函数包括自定义表达式和自定义转换。这两种功能都有助于在管道中重复使用逻辑，同时在一个位置进行统一管理。

自定义转换允许将一系列转换看板保存为单个转换，以便在整个管道中重复使用。

自定义表达式允许保存带有可选参数的单个表达式，以便在整个管道中重复使用。自定义表达式可用于过滤器或条件语句等其他转换内部。

自定义函数通常由名称（必填）、描述（选填）、函数参数（选填）和函数定义（必填）组成。

创建自定义表达式

要创建自定义表达式，请选择图表顶部的可复用项下拉菜单，然后选择自定义函数。

选择添加自定义表达式。

在新窗口中，配置自定义表达式。在左上角输入名称，使用添加参数选项在参数窗口中添加所需参数，并定义表达式。完成后选择应用。

创建自定义表达式后，可以像在转换节点或转换中搜索其他任何表达式一样搜索它。

创建自定义转换

创建自定义转换需要两个步骤：

  1. 将逻辑定义为一系列转换看板。
  2. 将这些转换看板转换为新的自定义函数。

下面将逐步介绍每个步骤的示例。

定义一系列转换看板

假设有一个 users 表，需要创建一个主键来唯一标识每个用户。已知每个用户的 first_name、last_name 和 first_login_date 组合是唯一的。需要在数据集中添加一个 String 类型的 primary_key 列，该列是这三列的哈希值，然后删除 first_name、last_name 和 first_login_date。最后，如果存在重复项，希望每个用户只保留一行，即保留 age 值最小的那一行。

首先，将 first_name、last_name 和 first_login_date 合并为一列。可以使用 Concatenate strings 转换将这三列拼接起来。

在 Separator 字段中输入 -。然后，在 Expressions 下拉菜单中选择每一列。前两列分别选择 first_name 和 last_name。但是，first_login_date 列无法作为第三个字段的选项。这是因为它的类型是 Date，而 Concatenate strings 函数仅接受 String 类型。

要解决此问题，请从 Expressions 选项卡中插入 Cast 表达式。参数设置为：Expression 选 first_login_date，Type 转换为 String。这样可以避免全局修改 first_login_date，从而影响所有下游转换看板。

该选项会保留现有值，并将其作为新转换的第一个输入。

将一系列转换面板转换为自定义函数

现在，假设您要在管道中的三个不同位置复用此逻辑。我们可以通过 Shift + Down Arrow 选中两个面板，然后选择顶部栏中的 + 按钮，将逻辑转换为自定义函数。

这将跳转至自定义函数创建页面。您会注意到列输入 first_name、last_name、first_login_date 和 age 带有删除线；这是因为您创建的函数对于管道中任何类型正确的四个输入都是通用的，无需考虑列名。

要定义这些输入，请选择 Add argument 四次。点击黄色框即可配置参数名称。

接着，点击每个参数右侧的齿轮图标来配置参数类型。这里，我们希望前两个参数为 Expression<String> 类型以代表 String 类型的列，第三个参数为 Expression<Date> 类型，最后一个参数为 Expression<Integer> 类型。

现在，您可以像之前一样将新参数添加到两个面板中。不过，它们现在会显示在 Parameters 部分，而不是 Columns 部分。

完成配置后，将函数命名为 Primary key generator，填写可选描述，然后选择 Apply all changes。

新函数现在可在任何转换路径中使用。下次需要使用此模式创建主键时，您可以在转换下拉菜单中搜索 Primary key generator。

您可以使用原始列填充自定义函数：first_name、last_name、first_login_date 和 age。

通过将相关列名填充到自定义函数中，您获得的结果与在路径中使用相同转换创建函数的结果一致。但是，通过创建自定义函数，您可以保存函数逻辑以便在整个管道中复用，而无需通过单独的转换面板重新构建逻辑。
