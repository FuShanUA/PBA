应用参考

你可以通过侧边栏访问的应用程序与 Palantir 平台进行交互。本页提供了可用应用程序范围的参考，并描述了何时你可能需要使用每个应用程序。

数据连接与集成

Palantir 平台中的数据连接与集成超越了传统的 ETL/ELT 解决方案，提供了一套全面的工具集，旨在提升数据团队的能力并随着时间的推移降低集成成本。Foundry 作为复杂环境下强大的数据集成骨干，提供了一个可扩展框架，可连接到各种源系统并支持多种数据传输方式。凭借集成的数据转换、管理和管道功能，Foundry 确保为关键操作提供安全、可扩展且高质量的数据交付。

应用程序描述用途
数据血缘数据血缘展示了不同资源如何在平台内交互及流动的图形。探索 Palantir 平台中任何数据或资源的来源或下游使用情况。
管道构建器管道构建器利用大语言模型和其他内置数据转换功能，创建从数据源到最终输出的端到端管道。通过批处理和流处理管道集成数据，用于分析和应用程序构建。
代码仓库 [1] [2]代码仓库是一个基于 Web 的代码编写环境，支持版本控制和协作。创建数据管道或在本体中编写函数。
数据集预览数据集预览显示数据集的内容和历史。浏览数据集并了解其历史和其他元数据。
数据健康数据健康允许你定义健康检查，以确保数据集的高质量。在数据集上添加或监控健康检查。
数据连接数据连接允许你连接到数据源并将数据同步到 Palantir 平台。连接到组织数据源或将新数据集同步到 Palantir 平台。
HyperAuto (SDDI)HyperAuto 在常见的 ERP 系统之上生成端到端数据管道。无需手动开发管道，即可从企业系统生成本体。
LinterLinter 分析你的租户状态以识别反模式，并提供优化资源、提升成本效益以及改善管道稳定性和弹性的建议。识别影响最大的建议，然后使用引导式或自动化工作流将其分配给用户采取行动。使用影响摘要功能定期审查预期影响。

[1] 对于某些数据科学工作流，代码工作簿或代码工作空间可能更合适。了解有关代码工作簿、代码工作空间和代码仓库之间差异的更多信息。
[2] 如果你是技术背景较弱的用户，管道构建器可能更适合你。

模型连接与开发

模型连接与开发涉及创建和集成机器学习模型，这些模型可以跨数据管道、本体和应用层使用，以支持多样化的用例。模型可以在平台内使用开源工具开发，也可以从外部环境导入，一旦集成，它们就能从推理、部署、治理和 ML Ops 等平台功能中受益。该平台确保模型开发和部署的每一步都严格遵守血缘关系、安全、版本控制和审计标准，从而实现稳健的 operationalization 并为模型改进提供持续的反馈回路。

应用程序描述用途
模型资产模型资产支持将多种模型类型集成到 Palantir 平台中。在 Palantir 平台中训练模型，并连接到外部托管的模型。
建模目标建模目标允许组织利益相关者和模型开发者协作并部署机器学习模型。提交模型；讨论建模目标，并将模型部署到生产环境。

本体构建

Palantir 平台中的本体构建涉及创建一个操作层，将数据集和模型映射到其现实世界的对应物，作为组织的数字孪生，包含语义和动态元素。通过定义对象类型、链接类型、操作类型和函数，本体支持强大的最终用户工作流、全面的元数据管理以及动态的安全与治理。这种集成使组织能够通过 Palantir 的分析和操作工具大规模提升决策能力，实现高效的数据探索、分析和应用程序开发。

应用描述用途
本体管理器本体管理器用于定义组织的本体。创建新的本体资源，例如对象、链接或操作类型。
对象视图对象视图代表了展示对象类型的规范方式。定义可跨用例使用的用户界面。
对象浏览器对象浏览器用于可视化本体。搜索、浏览和分析本体。
VertexVertex 用于探索对象关系并运行模拟。创建相关对象的系统图，并使用模型运行端到端模拟。
MachineryMachinery 是一个框架，通过识别不良行为并促进改进来理解和管理流程，同时利用 AIP 的大语言模型能力在 AIP 工作流中编排多步骤自动化。用于优化采购、销售和客户服务等各个领域的流程，通过解决低效问题、管理 AI 驱动的工作流、挖掘事件日志、监控性能指标以及实现实时人工干预来实现。
Foundry 规则Foundry 规则使用户能够在平台中主动管理复杂的业务逻辑。为数据集、对象和时间序列创建并应用规则，以适用于各种用例。
地图地图提供强大的地理空间和时间分析及可视化能力。将来自整个平台的数据集成到统一的地理空间体验中。
动态调度动态调度能力允许构建者通过利用本体并对资源约束和相互依赖关系进行建模，创建量身定制的调度和资源分配工作流，以满足其组织的复杂需求。通过提供交互式调度工具、实时解决方案和机器学习支持的推荐，优化员工日程，增强运输和物流的弹性，并提高制造产量。

开发者工具链

Palantir 平台的开发者工具链提供了一套全面的工具，包括核心 API 和 SDK、开发环境以及计算模块，使开发者能够构建充分利用本体各项功能的应用程序。本体 SDK (OSDK) 支持 Python、Java 和 TypeScript，允许开发者与本体数据交互并执行 AIP 逻辑函数，同时 API 促进了平台访问和数据编程化管理。此外，像 VS Code 这样的集成开发环境和计算模块允许进行可扩展的容器化部署，增强了第三方应用程序和工作流的开发与集成。

应用描述用途
本体 SDK本体软件开发工具包 (SDK) 使开发者能够从其开发环境直接访问本体的全部功能，支持多种语言，并实现与 Palantir API 的无缝集成，以进行应用程序的创建和管理。通过提供对本体 API 的便捷访问，加速应用程序开发，确保类型安全，减少维护工作并增强安全性，使开发者能够高效地构建由 Palantir 平台支持的应用程序。
计算模块 [Beta]计算模块允许您在 Palantir 平台上部署交互式容器，使您能够运行现有的代码库，无论使用何种语言，都可以作为无服务器 Docker 镜像在 Workshop 和 Slate 等应用程序中根据负载进行横向扩展。使用计算模块通过启用容器支持的功能、数据集成和托管模型来与代码或第三方代码交互，同时以动态扩展和连接性集成任何语言的现有代码库。
代码工作区 [3]代码工作区将 JupyterLab® 和 RStudio® Workbench 第三方 IDE 引入了 Palantir。利用集成到 Palantir 平台中的行业标准工具，加速数据科学、统计和基于代码的工作流。
VS Code 工作区Palantir 平台上的 VS Code 工作区集成使用开源的 VS Code 提供一个用于编写和协作生产级代码的 IDE，原生支持 Python 转换和 OSDK React 应用程序。直接从 Palantir 平台访问预配置的 VS Code 环境来高效开发代码，利用预览集成、调试支持和 Python 环境自动设置等功能，并使用了解 Foundry 及其 API 的预配置 AI 编码助手。
Visual Studio Code 的 Palantir 扩展Visual Studio Code 的 Palantir 扩展将代码仓库的功能集成到 VS Code 中，目前专注于支持 Python 转换，在 Palantir 生态系统内提供无缝的开发体验。通过直接在 VS Code 中访问 Palantir 仓库来增强编码工作流，实现编写 Python 转换和在 Foundry 中运行构建等功能。
Palantir MCPPalantir MCP 使外部 AI IDE 和智能体能够连接到 Palantir 平台，并获取有关您的本体和 Foundry 工具的上下文。使用 Palantir MCP 让外部 AI 系统查询数据、访问文档并更高效地构建应用程序。
本体 MCP (OMCP)本体 MCP (OMCP) 将应用程序本体资源公开为 MCP 工具，使外部 AI 智能体能够通过受控访问读取对象、执行操作和查询本体数据。使用本体 MCP 使外部 AI 智能体能够通过预定义的操作和应用程序限制安全地与生产本体数据进行交互。

分析

Palantir 为企业内的所有用户提供全面的分析能力，与 Foundry 本体无缝集成，支持点击式和基于代码的分析，例如基于表格、地理空间和时间序列的分析。其核心应用程序——Contour、Quiver、Code Workbook、Notepad 和 Fusion——支持多种分析任务，从自上而下的数据探索和多模态图表绘制，到机器学习和电子表格驱动的计算，同时支持将数据写回本体以丰富分析洞察。此外，Foundry 通过开箱即用的连接器、REST API、ODBC/JDBC 驱动程序以及 Python 和 R 的 SDK，支持与现有分析工具的深度集成，增强了与 Power BI、Tableau 和 Excel 等平台的连通性。

详细了解分析应用程序及平台中可用的分析类型。

Marketplace 进一步提升了产品交付能力，提供了一个店面，便于用户轻松发现并安装已发布的数据产品，同时提供引导式安装和相关产品推荐功能。

安全与治理

Palantir 平台的设计以安全与治理为基础支柱。这种方式为医疗、金融和政府等行业的敏感数据提供了强大的保护。此外，Palantir 的解决方案与 HIPAA、GDPR 和 ITAR 等全球监管框架保持一致，确保实现全面的合规性，并保障敏感信息的安全。

该平台通过精细的安全模型实施严格的访问控制，该模型包含强制权限与自主权限、细粒度的访问控制以及强制加密，确保用户仅能访问经授权的数据。Palantir 的企业级安全功能包括强大的身份验证措施、全面的审计日志以及广泛的信息治理。

应用程序描述用途
ApprovalsApprovals 应用程序负责管理在 Foundry 中请求、批准和调用变更的工作流，整合了合规性、治理和同行评审流程，从而高效管理项目访问请求和本体提案等任务。通过审查和批准任务、跟踪请求状态，并确保符合治理策略，来处理 Palantir 平台中的变更请求；同时通过有关请求进度和所需操作的通知随时掌握动态。

管理与赋能

管理与赋能能力集中于控制面板，提供了一套涵盖数据治理、资源管理和安全管理的综合工具集，支持数据网格和数据编织等企业数据架构。控制面板允许管理员配置和管理注册，将管理功能映射至现有数据治理体系，并通过 SAML 2.0 协议依托身份提供商进行访问控制，从而在平台规模扩展时确保无缝集成与安全。此外，资源管理工具可对资源使用情况进行精细化洞察，而可定制的平台体验则确保与组织品牌形象及特定用户需求保持一致。
