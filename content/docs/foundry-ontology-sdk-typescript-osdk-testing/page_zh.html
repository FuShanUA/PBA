单元测试 TypeScript OSDK 代码

实验性

@osdk/unit-testing 包是实验性的。该包被发布为 @osdk/unit-testing，并且仅通过 /experimental 子路径导出。API 表面在提升到稳定名称之前可能会发生变化。

@osdk/unit-testing 包允许您对使用 OSDK Client 的代码进行单元测试，包括 Foundry 函数，而无需向 Foundry 发送请求。该包提供：

  • createMockClient: 一个你可以用流畅的 .when、.whenObjectSet 和 .whenQuery 匹配器进行存根的 Client。
  • createMockOsdkObject: 构建完全格式化的 Osdk.Instance 值，包括 $primaryKey、$title、$rid、$link 和 $clone。
  • createMockObjectSet: 一个独立的 ObjectSet，你可以将其传递给任何真实对象应该去的地方，或作为聚合的多链接目标。
  • createMockAttachment: 附件值的占位符。

安装

将该包作为开发依赖项安装：

Copied!
1npm install --save-dev @osdk/unit-testing

该包有以下对等依赖项，这些依赖项应该已经在你的项目中安装：

  • @osdk/api
  • @osdk/client
  • @osdk/functions

该包在内部使用 vitest 进行示例测试；你可以在自己的代码中使用任何测试运行器。

编写你的第一个测试

考虑一个 Foundry 函数，它从页面中读取第一个 Employee：

Copied!
1import type { Osdk } from "@osdk/api";
2import type { Client } from "@osdk/client";
3import { Employee } from "your-app-sdk";
4
5export async function basicFetchPage(
6  client: Client,
7): Promise<Osdk.Instance<Employee>> {
8  const objects = await client(Employee).fetchPage();
9  const object = objects.data[0];
10  if (object == null) throw new Error("No objects returned");
11  return object;
12}

下面是一个带有模拟客户端的单元测试：

Copied!
1import {
2  createMockClient,
3  createMockOsdkObject,
4} from "@osdk/unit-testing/experimental";
5import { describe, expect, it } from "vitest";
6import { Employee } from "your-app-sdk";
7import { basicFetchPage } from "./basicFetchPage.js";
8
9describe("basicFetchPage", () => {
10  it("returns the first Employee", async () => {
11    const mockClient = createMockClient();
12    const mockEmployee = createMockOsdkObject(Employee, {
13      employeeId: 1,
14      fullName: "John",
15    });
16
17    mockClient
18      .when((stub) => stub(Employee).fetchPage())
19      .thenReturnObjects([mockEmployee]);
20
21    const actual = await basicFetchPage(mockClient);
22    expect(actual).toEqual(mockEmployee);
23  });
24});

该测试做了三件事：

  1. createMockClient() 返回一个满足 MockClient 接口的 Client；将其传递给任何需要真实客户端的地方。
  2. createMockOsdkObject(Employee, { ... }) 构建一个真实形状的 Osdk.Instance。
  3. mockClient.when(stub => stub(Employee).fetchPage()).thenReturnObjects([...]) 记录一个模拟对象。stub 参数是一个类似 Client 的工厂；重新构建您的测试代码将执行的调用链。

模拟对象和关联

createMockOsdkObject 构建一个完整的 Osdk.Instance<T>，您可以将其传递给您的测试代码或放入 .thenReturnObjects([...]) 模拟对象中。以下部分涵盖了对象结构、links 选项（单个、多个、错误和模拟对象集）以及 createMockAttachment。

参数

createMockOsdkObject 有三个参数：

Copied!
1createMockOsdkObject(objectType, properties, options);
  1. objectType: 从您的 SDK 生成的对象类型常量（例如，Employee）。模拟对象会从该值中读取其 apiName 和 primaryKeyApiName。
  2. properties: 你希望在模拟对象上拥有的属性值。必须包含主键属性。其他属性是可选的，仅在你的代码读取它们时才有意义。
  3. options: 三个可选字段：
    • links: 对象的 $link 访问器的模拟数据。参见 链接。
    • titlePropertyApiName: 应该支持 $title 的属性的API名称。参见 设置标题属性。
    • $rid: 覆盖自动生成的 $rid。默认值为 "ri.mock.main.object.<apiName>.<primaryKey>"。

返回的模拟对象与真实OSDK实例具有相同的结构：

字段描述
$apiName, $objectType对象类型的API名称。
$primaryKey在 properties 中主键属性的值。
$titletitlePropertyApiName 属性的值；如果没有设置，则为 undefined。
$rid如果提供了 options.$rid，否则为自动生成的模拟RID。
$objectSpecifier"<apiName>:<primaryKey>"。
$link 由options.links支撑的代理。
$clone(updates?)返回一个合并属性值的新模拟对象。

模拟对象不模拟$as和$__EXPERIMENTAL__NOT_SUPPORTED_YET__*访问器；访问它们会抛出错误。

基本用法

Copied!
1import { createMockOsdkObject } from "@osdk/unit-testing/experimental";
2import { Employee } from "your-app-sdk";
3
4const emp = createMockOsdkObject(
5  Employee,
6  { employeeId: 1, fullName: "John Doe" },
7  { titlePropertyApiName: "fullName" },
8);
9
10emp.$primaryKey; // 1
11emp.$title; // "John Doe"
12emp.$objectSpecifier; // "Employee:1"

您必须包含主键属性。createMockOsdkObject读取objectType.primaryKeyApiName，如果该键不在properties中，则会抛出错误。

设置标题属性

在测试环境中，OSDK 不知道给定对象类型中的哪个属性是其标题。如果您的被测代码读取obj.$title，您必须通过传递titlePropertyApiName来告诉模拟对象在此处显示哪个属性：

Copied!
1const emp = createMockOsdkObject(
2  Employee,
3  { employeeId: 1, fullName: "John Doe" },
4  { titlePropertyApiName: "fullName" },
5);
6
7emp.$title; // "John Doe"

titlePropertyApiName 必须命名一个你在 properties 中实际包含的属性；如果缺少 createMockOsdkObject 会抛出错误。如果你完全省略 titlePropertyApiName，则 $title 是 undefined。

关联

links 选项与对象类型中的关联 API 名称相匹配。每个值可以是以下之一：

关联多重性允许的值
单个一个模拟对象，或一个 Error 实例。
多个一个模拟对象数组，或一个 MockObjectSet（参见 模拟对象集）。

成功获取关联

Copied!
1const office = createMockOsdkObject(Office, {
2  officeId: "nyc",
3  name: "New York Office",
4});
5
6const employee = createMockOsdkObject(
7  Employee,
8  { employeeId: 1, fullName: "John Doe" },
9  { links: { officeLink: office } },
10);
11
12await employee.$link.officeLink.fetchOne();
13// → office
14
15(await employee.$link.officeLink.fetchOneWithErrors()).value;
16// → office

获取关联失败

将一个 Error 实例传递给调用代码，以测试失败分支。fetchOne() 会以错误拒绝，而 fetchOneWithErrors() 会解析为 { error }：

Copied!
1const employee = createMockOsdkObject(
2  Employee,
3  { employeeId: 1 },
4  { links: { officeLink: new Error("link unavailable") } },
5);
6
7await expect(employee.$link.officeLink.fetchOne()).rejects.toThrow(
8  "link unavailable",
9);
10
11const result = await employee.$link.officeLink.fetchOneWithErrors();
12result.error; // the Error
13result.value; // undefined

缺失的关联

如果你的代码访问了 $link.someLink 但没有配置它，访问器仍然存在。fetchOne 方法会拒绝，而 fetchOneWithErrors 会用 { error } 解析，其中包含关联名称、对象类型和主键：

Copied!
1const employee = createMockOsdkObject(Employee, { employeeId: 1 });
2
3await employee.$link.officeLink.fetchOne();
4// rejects

使用数组的多关联

传入一个数组；$link 访问器暴露与真实多关联相同的调用形状：

Copied!
1const peep1 = createMockOsdkObject(Employee, {
2  employeeId: 10,
3  fullName: "Alice",
4});
5const peep2 = createMockOsdkObject(Employee, {
6  employeeId: 11,
7  fullName: "Bob",
8});
9
10const employee = createMockOsdkObject(
11  Employee,
12  { employeeId: 1 },
13  { links: { peeps: [peep1, peep2] } },
14);
15
16(await employee.$link.peeps.fetchPage()).data;
17// → [peep1, peep2]
18
19await employee.$link.peeps.fetchOne(11);
20// → peep2 (matched by $primaryKey)
21
22for await (const peep of employee.$link.peeps.asyncIter()) {
23  // peep1, peep2
24}

当没有数组元素具有匹配的 $primaryKey 时，fetchOne(primaryKey) 会抛出错误。aggregate() 方法在数组形式上 不支持；请改用 MockObjectSet。

使用 MockObjectSet 的多关联

Many-link 与一个 MockObjectSet

Many-link 也可以由一个 MockObjectSet 而不是数组来支撑。当你的代码在链接上调用 aggregate()、where() 或其他对象集方法时，请使用这种方法：

Copied!
1const employee = createMockOsdkObject(
2  Employee,
3  { employeeId: 1 },
4  { links: { peeps: peepsSet } },
5);

有关如何构建 peepsSet 并在其上设置模拟调用，请参见 模拟对象集。

模拟对象集

createMockObjectSet(objectType) 返回一个 ObjectSet<T>，你可以将它传递到任何实际对象会去的地方：直接传入被测试的函数中，或者作为 createMockOsdkObject 中 many-link 的值。单独使用时，模拟对象集不包含任何数据；你通过在一个 MockClient 上注册模拟方法来配置它的行为。

Copied!
1import {
2  createMockClient,
3  createMockObjectSet,
4  createMockOsdkObject,
5} from "@osdk/unit-testing/experimental";
6
7const mockClient = createMockClient();
8const peepsSet = createMockObjectSet(Employee);
9
10mockClient
11  .whenObjectSet(
12    peepsSet,
13    (os) => os.aggregate({ $select: { $count: "unordered" } }),
14  )
15  .thenReturnAggregation({ $count: 7 });

现在 peepsSet.aggregate(...) 解析为 { $count: 7 }。你可以将同一个集合作为 many-link 添加到父级模拟对象上：

Copied!
1const employee = createMockOsdkObject(
2  Employee,
3  { employeeId: 1 },
4  { links: { peeps: peepsSet } },
5);
6
7const result = await employee.$link.peeps.aggregate({
8  $select: { $count: "unordered" },
9});
10result.$count; // 7

你可以在同一个集合上注册 fetchPage、where 和其他调用形式；有关完整的构建器参考，请参见 模拟对象集上的模拟调用。

模拟附件

对于类型为 Attachment 的函数输入，createMockAttachment 返回一个占位值，其表面可供您的代码调用。使用方式与 createMockOsdkObject 相同：

Copied!
1import { createMockAttachment } from "@osdk/unit-testing/experimental";
2
3const blob = new Blob(["hello world!"], { type: "text/plain" });
4const attachment = createMockAttachment("ri.attachments.main.attachment.abc", blob);

克隆和更新

$clone 是支持的，并返回一个合并属性的新冻结模拟。将主键更新为不同的值会抛出错误。

Copied!
1const updated = employee.$clone({ fullName: "Jane Doe" });
2updated.$primaryKey; // unchanged
3updated.fullName; // "Jane Doe"

模拟客户端调用

createMockClient() 返回一个满足 OSDK Client 接口的值，以及四个用于设置模拟的方法：

  • client.when(callback): 在客户端根部存根调用。传递一个回调函数，该函数重新构建您的测试代码将进行的链（例如，stub(Employee).where(...).fetchPage()）。返回一个构建器，其 .thenReturn* 匹配器取决于调用结构。
  • client.whenObjectSet(set, callback): 在特定 MockObjectSet（使用 createMockObjectSet 创建）上存根调用。当您的代码直接接收到一个对象集，或者对于由模拟对象集支持的多链接，使用此方法。
  • client.whenQuery(query, params?): 存根查询调用（本体论上的生成函数）。返回一个带有 .thenReturn(value) 和 .thenThrow(error) 的构建器。
  • client.clearStubs(): 移除在此客户端上注册的所有存根。

每个注册器都在以下部分中介绍。

在客户端根部的存根调用

使用 client.when(callback) 来重建您的测试代码将进行的调用链。参数是一个类似 Client 的工厂；链式 where、aggregate、fetchPage、fetchOne 等，就像您的代码会做的那样。

fetchPage 与 thenReturnObjects

Copied!
1const mockClient = createMockClient();
2const emp = createMockOsdkObject(Employee, { employeeId: 1, fullName: "John" });
3
4mockClient
5  .when((stub) => stub(Employee).fetchPage())
6  .thenReturnObjects([emp]);
7
8const page = await mockClient(Employee).fetchPage();
9page.data; // [emp]

thenReturnObjects 还会连接 asyncIter. 如果您的代码是迭代而不是分页，那么同一个 stub 可以满足这两种调用方式。

fetchOne 与 thenReturnObject

Copied!
1mockClient
2  .when((stub) => stub(Employee).fetchOne(1))
3  .thenReturnObject(emp);

aggregate 与 thenReturnAggregation

Copied!
1mockClient
2  .when((stub) =>
3    stub(Employee)
4      .where({ employeeId: { $eq: 5 } })
5      .aggregate({ $select: { "employeeLocation:exactDistinct": "asc" } })
6  )
7  .thenReturnAggregation({ employeeLocation: { exactDistinct: 3 } });

Stub $groupBy 以相同的方式聚合数据；返回一组分组行：

Copied!
1mockClient
2  .when((stub) =>
3    stub(Employee).aggregate({
4      $select: { "employeeId:max": "unordered" },
5      $groupBy: { employeeId: "exact" },
6    })
7  )
8  .thenReturnAggregation([
9    { $group: { employeeId: 5 }, employeeId: { max: 5 } },
10  ]);

同一客户端上的多个 stub

您可以注册任意数量的 stub。stub 会匹配您代码中的调用；注册顺序不影响匹配。

在设置的模拟对象上进行存根调用

当代码直接接收一个 ObjectSet（不是由 client(Type) 构建的），或者当你为一个由 MockObjectSet 支持的多链接注册存根时，将存根注册到该集合本身：

Copied!
1import { createMockObjectSet } from "@osdk/unit-testing/experimental";
2
3const empSet = createMockObjectSet(Employee);
4const emp1 = createMockOsdkObject(Employee, {
5  employeeId: 1,
6  fullName: "Alice",
7});
8const emp2 = createMockOsdkObject(Employee, { employeeId: 2, fullName: "Bob" });
9
10mockClient
11  .whenObjectSet(empSet, (os) => os.fetchPage())
12  .thenReturnObjects([emp1, emp2]);
13
14mockClient
15  .whenObjectSet(
16    empSet,
17    (os) => os.aggregate({ $select: { $count: "unordered" } }),
18  )
19  .thenReturnAggregation({ $count: 42 });

然后可以将相同的 empSet 传递给任何你的代码期望一个 ObjectSet<Employee> 的地方：传入被测试的函数，或作为父模拟对象上的多链接目标。

存根 Foundry 查询

查询（本体论上的函数）也可以被存根：

Copied!
1import { addOne } from "your-app-sdk";
2
3mockClient.whenQuery(addOne, { n: 5 }).thenReturn(6);
4mockClient.whenQuery(addOne, { n: 99 }).thenThrow(new Error("boom"));

thenReturn(value) 将查询 promise 解析为 value，而 thenThrow(error) 则会拒绝它。不同的参数对象可以分别存根：

Copied!
1mockClient.whenQuery(addOne, { n: 10 }).thenReturn(11);
2mockClient.whenQuery(addOne, { n: 20 }).thenReturn(21);

带有数组参数的查询遵循相同的模式；匹配你的代码传递的参数：

Copied!
1mockClient
2  .whenQuery(queryTypeReturnsArray, { people: ["Alice", "Bob"] })
3  .thenReturn(["Alice - processed", "Bob - processed"]);

在测试之间重置桩

mockClient.clearStubs() 会移除客户端上注册的所有桩。如果您在多个 it 块中重用客户端，这很有用；否则，为每个测试构建一个全新的 createMockClient() 以实现隔离。

使用MSW测试Foundry平台API

createMockClient 仅对本体论调用（对象类型、查询和对象集）进行桩处理。它不会拦截 @osdk/foundry.* 和 @osdk/internal.foundry.* 中的Foundry平台API；这些调用将通过常规的 fetch 路径。为了在测试中将其从网络中移除，请使用 MSW ↗ 进行拦截。

它是如何工作的

您使用网络请求桩库来桩处理平台SDK请求。它使用一个占位符基础URL：

https://mock.invalid/

每个平台调用您的代码都会针对该来源进行解析。为您的函数调用的特定路径提供MSW处理程序。

模拟Platform调用

一个加载当前用户并根据用户名后缀进行控制的函数：

Copied!
1import type { Client } from "@osdk/client";
2import { Users } from "@osdk/foundry.admin";
3
4export async function requireAdminUser(client: Client): Promise<string> {
5  const user = await Users.getCurrent(client);
6  if (!user.username.endsWith("@admin")) {
7    throw new Error(`User ${user.username} is not an admin`);
8  }
9  return user.username;
10}

按照真实平台类型输入你的fixture

MSW响应体是一个JSON字面量，因此body可能会偏离实际的@osdk/foundry.*结构（重命名字段、新增必填属性等），仅在运行时失败。

为了使fixture与真实类型保持一致，可以从Platform函数本身派生：

Copied!
1import type { somePlatformFn } from "@osdk/foundry.<service>/<Resource>";
2
3type ResponseShape = Awaited<ReturnType<typeof somePlatformFn>>;

然后使用satisfies ResponseShape断言响应体：

Copied!
1HttpResponse.json(
2  {/* ...response fields... */} satisfies ResponseShape,
3);

Awaited<ReturnType<typeof fn>>会解开由async函数返回的Promise<T>，直接给你T，这确保了模拟的响应与真实调用返回的内容完全匹配。

在上面的例子中，这个模式的具体实例是type User = Awaited<ReturnType<typeof getCurrent>>；根据端点返回的内容来命名别名。

技巧

  • 使用onUnhandledRequest: "error"。 发现缺失的处理器比调试卡住的测试要简单。
  • 在处理程序之间复用同一个服务器。 在模块作用域中调用 setupServer() 一次，然后在每个 it 块中调用 server.use(...) 以处理每个测试的句柄。使用 afterEach(server.resetHandlers) 来清除它们。
  • 将本体论（Ontology）存根与平台（Platform）存根合并。 一个 mockClient 可同时处理这两种调用；本体论（Ontology）调用通过 when、whenObjectSet 和 whenQuery 进行，而平台（Platform）调用则通过 MSW。
