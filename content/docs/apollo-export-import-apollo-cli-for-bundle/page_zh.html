Apollo CLI 命令用于捆绑包

以下命令允许您从终端下载和上传一个捆绑包。

要使用这些命令，您可能需要从 Apollo Hub 主页的 快速操作 > 下载 Apollo CLI 下载最新的 apollo-cli 版本。所需的最低 apollo-cli 版本是 0.425.0。您可以使用 ./apollo-cli 版本命令来检查您正在运行的 apollo-cli 版本。

相同的 apollo-cli 命令用于捆绑包和捆绑包块。有关命令的帮助，您可以添加 -h 标志。

有关 Apollo CLI 入门的更多信息，请参阅 公开文档。

下载捆绑包

默认情况下，Apollo CLI 会通过下载所有块并组装 Bundle 来下载整个 Bundle。然而，如果 Bundle 没有被分块，CLI 将直接下载整个 Bundle。

  • 如果 Bundle 被分块，并且您只想下载块而不组装 Bundle，请在命令中添加 --chunks-only 标志。
  • 如果您只想下载 Bundle 的索引和引用者，请使用 --index-only 标志。
  • 如果您想生成下载 Bundle 的 URL，请使用 --generate-urls 标志。

使用 apollo-cli bundle download 命令下载 Bundle。您可以在 Pipeline 收件箱中 Bundle 页面的 URL 路径中找到 bundle-rid，其格式应为 workspace/apollo/export/app/pipelines/{pipeline-id}/bundles/{bundle-rid}。如果没有指定输出文件，Bundle 将被下载到默认的 tar 文件 pipeline_id-bundle_version.tar 中。

$ ./apollo-cli bundle download [bundle-rid]\
   --apollo-token "$APOLLO_TOKEN" \
   --apollo-url "$APOLLO_URL"

下载的Bundle将是一个未压缩的tar格式文件。如果要使用gzip压缩Bundle（即作为.tar.gz文件），可以向命令中添加--gzip-compressed标志。请注意，这可能会显著增加组装Bundle所需的时间。

下载非分块捆绑包时，可以使用--retry标志来指定失败下载的最大重试次数。当下载分块捆绑包时，不支持重新下载失败的分块，但未下载的分块编号会包含在错误信息中。

CLI 还支持通过使用 --transfer-target 标志实现分层去重的 Bundle 传输。这可能会减少传输到特定目标 Hub 的数据量。 了解有关使用 Apollo CLI 进行分层去重的更多信息。

上传 Bundle

上传捆绑包

您可以使用 apollo-cli bundle upload 命令来上传一个捆绑包或批量上传所有捆绑包块。使用 --bundle-path 标志来指定您要上传的捆绑包的路径。这可以是一个单独的 tar 文件或一个块目录。

$ ./apollo-cli bundle upload \
   --apollo-token "$APOLLO_TOKEN" \
   --apollo-url "$APOLLO_URL" \
   --bundle-path $PIPELINE_ID-$BUNDLE_VERSION

将块组装为捆绑包

默认情况下，apollo-cli bundle download 会尽可能地从各个数据块中下载并组装一个完整的 Bundle。这种方法由于每个下载的数据块都会直接流式传输到输出的 Bundle 归档文件中，并且只写入磁盘一次，因此占用的磁盘空间较小。然而，如果任何一个数据块下载失败，整个命令都会失败。当一个 Bundle 包含大量数据块或某个数据块过大时，这会使该操作容易出现故障。

为了可靠地从数据块中下载完整的 Bundle，您可以：

  1. 使用一个用于组装的基础 Bundle 归档文件下载所有数据块。

$ ./apollo-cli bundle download [bundle-rid] \
  --apollo-token "$APOLLO_TOKEN" \
  --apollo-url "$APOLLO_URL" \
  --chunks-only

  2. 下载索引和引用者。

 $ ./apollo-cli bundle download [bundle-rid] \
   --apollo-token "$APOLLO_TOKEN" \
   --apollo-url "$APOLLO_URL" \
   --index-only

  3. 将所有 Bundle 块归档和索引合并到一个保存在 pipeline_id-bundle_version-index.tar 的 Bundle 中。

    

    已复制！

$ ./apollo-cli bundle assemble \
  --archive-dir $PIPELINE_ID-$BUNDLE_VERSION \
  --base-bundle-archive $PIPELINE_ID-$BUNDLE_VERSION-index.tar

如果您希望分次将较少的数据传输到目标中心，可以先将步骤1和步骤2中下载的块、索引和引用者上传到目标中心，然后在目标中心运行 apollo-cli bundle assemble。

apollo-cli bundle assemble 命令将以未压缩的归档格式（.tar 文件）生成 Bundle。若要使用 gzip 压缩 Bundle，将生成 .tar.gz 文件，可以向命令中添加此标志： --gzip-compressed true。请注意，这可能会**显著**增加生成 Bundle 所需的时间。
