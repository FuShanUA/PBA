Apollo CLI 命令用于捆绑包

以下命令允许您从终端下载和上传一个捆绑包。

要使用这些命令，您可能需要从 Apollo Hub 主页的 快速操作 > 下载 Apollo CLI 下载最新的 apollo-cli 版本。所需的最低 apollo-cli 版本为 0.425.0。您可以使用 ./apollo-cli version 命令来检查您正在运行的 apollo-cli 版本。

相同的 apollo-cli 命令用于捆绑包和捆绑包块。有关命令的帮助，您可以添加 -h 标志。

有关 Apollo CLI 入门的更多信息，请参阅 公共文档。

下载捆绑包

默认情况下，Apollo CLI 会通过下载所有块并组装 Bundle 来下载整个 Bundle。然而，如果 Bundle 没有被分块，则 CLI 会直接下载整个 Bundle。

  • 如果 Bundle 被分块，并且您只想下载块而不组装 Bundle，请在命令中添加 --chunks-only 标志。
  • 如果您只想下载 Bundle 的索引和引用者，请使用 --index-only 标志。
  • 如果您想生成下载 Bundle 的 URL，请使用 --generate-urls 标志。

使用 apollo-cli bundle download 命令下载 Bundle。您可以在其 Pipeline 邮件箱中的 Bundle 页面的 URL 路径中找到 bundle-rid，它应遵循 workspace/apollo/export/app/pipelines/{pipeline-id}/bundles/{bundle-rid} 格式。如果没有指定输出文件，Bundle 将被下载到默认的 tar 文件 pipeline_id-bundle_version.tar 中。

Copied!
1
2
3
$ ./apollo-cli bundle download [bundle-rid]\
   --apollo-token "$APOLLO_TOKEN" \
   --apollo-url "$APOLLO_URL"

下载的 Bundle 将是 .tar 格式的未压缩档案。要使用 gzip 压缩 Bundle（即作为 .tar.gz 文件），您可以向命令中添加 --gzip-compressed 标志。请注意，这可能会 显著 增加组装 Bundle 所需的时间。

当下载非分块的包（Bundles）时，可以使用 --retry 标志来指定失败下载的最大重试次数。当下载分块的包时，不支持重试失败的分块，但未下载的分块编号会包含在错误信息中。

Copied!
1
Error: failed to download all chunks, chunk numbers not downloaded: 1,8,9,10,11,12, downloaded: 2,3,4,5,6,7,13,14,15,16,17,18: http2: client connection lost

通过提供 --chunk-ids 参数的列表来重新下载失败的分块。

1
2
3

$ ./apollo-cli bundle download [bundle-rid]\
   --apollo-token "$APOLLO_TOKEN" \
   --apollo-url "$APOLLO_URL" \
   --chunk-ids 2,3,4,5,6,7,13,14,15,16,17,18

CLI 还支持通过使用 --transfer-target 标志进行分层去重的包传输。这可能会减少传输到特定目标 Hub 的数据量。了解有关使用 Apollo CLI 进行分层去重的更多信息。

上传包

您可以使用 apollo-cli bundle upload 命令来上传一个包和批量上传所有包分块。使用 --bundle-path 标志来指定您要上传的包的路径。这可以是一个单独的 tar 文件或一个分块目录。

Copied!
1
2
3
4
$ ./apollo-cli bundle upload \
   --apollo-token "$APOLLO_TOKEN" \
   --apollo-url "$APOLLO_URL" \
   --bundle-path $PIPELINE_ID-$BUNDLE_VERSION

将块组合为 Bundle

默认情况下，apollo-cli bundle download 会在可能的情况下从块下载并组合一个完整的 Bundle。这种方法由于每个下载的块直接流到输出的 Bundle 归档文件中，并且只写入磁盘一次，因此使用的小磁盘占用空间较小。然而，如果任何一个块下载失败，则整个命令将失败。当 Bundle 包含许多块或单个块太大时，这使其容易失败。

要可靠地从块下载整个 Bundle，您可以：

  1. 使用用于组装的基 Bundle 归档文件下载所有块。

    Copied!
    1
2
3
4
$ ./apollo-cli bundle download [bundle-rid] \
  --apollo-token "$APOLLO_TOKEN" \
  --apollo-url "$APOLLO_URL" \
  --chunks-only

  2. 下载索引和引用项。

    Copied!
    1
2
3
4
 $ ./apollo-cli bundle download [bundle-rid] \
   --apollo-token "$APOLLO_TOKEN" \
   --apollo-url "$APOLLO_URL" \
   --index-only

  3. 将所有 Bundle 块归档和索引组合成一个保存在 pipeline_id-bundle_version-index.tar 的 Bundle。

    Copied!
    1
2
3
$ ./apollo-cli bundle assemble \
  --archive-dir $PIPELINE_ID-$BUNDLE_VERSION \
  --base-bundle-archive $PIPELINE_ID-$BUNDLE_VERSION-index.tar

如果您希望每次向目标中心传输更少的数据量，可以先将步骤 1 和 2 下载的块、索引和引用项上传到目标中心，然后在目标中心运行 apollo-cli bundle assemble。

apollo-cli bundle assemble 命令会以未压缩的归档格式组装 Bundle，即 .tar 文件。要使用 gzip 压缩 Bundle，这将创建一个 .tar.gz 文件，您可以将此标志添加到您的命令中：--gzip-compressed true。请注意，这可能会 显著 增加组装 Bundle 所需的时间。
