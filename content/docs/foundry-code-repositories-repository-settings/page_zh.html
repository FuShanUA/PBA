提交信息设置

默认情况下，点击提交按钮或构建数据集时，系统会自动生成提交信息。您可以禁用此选项，以鼓励编写更有意义的提交信息。禁用后，每次提交前都会打开提交信息对话框，并要求输入提交信息。

数据集别名

在代码中引用数据集时，可以使用其在 Foundry 中的确切位置（路径）或其唯一资源标识符（RID）。虽然这两种方式都有效，但建议尽可能使用 RID，因为这样可以在将资源从一个位置移动到另一个位置时，无需更新代码库中的代码。

代码库编辑器的自动补全功能允许您在输入时搜索数据，无论您输入的是路径、RID 还是自由文本。当您从结果列表中选择数据集时，编辑器会根据设置插入其路径或 RID：

  • 尽可能自动更改为 RID - 此选项将插入数据集的 RID，并提议将转换中的任何路径转换为 RID。编辑器会在 RID 上方显示数据集名称，并允许根据需要进行编辑。
  • 不自动将位置更改为 RID - 此选项将插入数据集的路径，并在可能的情况下允许将 RID 转换为路径。

外部系统

信息安全官可以启用代码库以与外部系统进行交互。这是一项高级功能，允许开发者编写外部转换，即通过 agent-proxy 与开放互联网或本地系统上的 Web 服务和 API 进行交互的 Python 代码。此功能仅应在面向高度信任的开发者的代码库中启用，因为无法保证与外部系统的交互会遵循 Foundry 严格的数据来源和访问控制实践。安全官可以通过配置允许导出的强制控制标记，来限制允许与外部系统交互的数据范围。
