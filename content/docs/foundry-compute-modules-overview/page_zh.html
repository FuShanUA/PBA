计算模块

应用程序访问权限

要启用计算模块，请联系您的平台管理员在控制面板中修改应用程序访问权限。

您也可以在平台的计算模块应用程序中直接查看此文档，以获得更高效的开发者体验。

计算模块允许您在 Palantir 平台上部署交互式容器。您可以将现有的代码库（无论使用何种编程语言）接入平台并在其中运行。具体而言，您可以在 Palantir 平台上将无服务器 Docker 镜像作为计算模块运行，并根据前端应用（例如 Workshop 和 Slate）的负载情况对其进行水平扩容或缩容。

计算模块入门

您可以通过我们的计算模块快速入门指南开始开发：

  • 从 Workshop 模块或 Slate 应用程序调用自定义容器或模型： 编写一个可供 Workshop 或 Slate 查询的容器函数。查阅有关构建计算模块支持函数的文档，开始使用 Palantir 的 TypeScript 或 Python SDK。
  • 使用自定义容器将数据同步至 Foundry： 编写一个容器，用于将来自自定义源的数据添加到流、数据集或媒体集中。查阅有关构建计算模块支持管道的文档，开始使用 Palantir 的 TypeScript 或 Python SDK。
  • [高级] 通过编写自定义客户端与任意语言集成： 如果要在不使用专用 SDK 的情况下集成某种语言的容器，您可以编写一个自定义客户端来实现计算模块客户端规范。

计算模块的用例

计算模块为您提供了在平台中与自有代码或第三方代码交互的新方式，支持以下用例：

  • 容器支持函数： 编写可供 Workshop 或 Slate 等应用程序查询的容器函数。
  • 基于容器的数据集成： 连接任意数据源，并将数据摄取到流、数据集或媒体集中。
  • 托管自定义模型： 托管自定义或开源模型，并从 Foundry 应用程序中进行交互式查询。

为什么要使用计算模块？

计算模块具有以下几项核心优势：

  • 集成现有代码库： 如果您的关键业务代码在 Foundry 中重写存在风险或成本过高，您可以将其容器化为 Docker 镜像，并作为计算模块运行。
  • 使用任意编程语言： 只要代码能够被容器化，无论使用何种语言均可运行。这意味着您不会受限于 Foundry 原生支持的语言。
  • 动态预测性水平扩缩容： 如果您需要处理数量不一的请求，计算模块可以根据当前和历史负载动态增加或减少可用副本的数量，从而确保更高的可用性。
  • 平台内外连接： 利用 Palantir 产品编写自定义逻辑。例如，您可以读取和写入数据或媒体集，也可以连接外部系统。

请注意，计算模块并不适用于所有场景。我们不建议在以下情况中使用计算模块：

  • 动态垂直扩缩容： 如果您预期单个请求的规模会发生剧烈变化（例如从 1MB 增至 100GB），并且希望支持动态垂直扩缩容，计算模块在“开箱即用”状态下的表现可能不佳，因为其分配的资源量是静态且需手动定义的。虽然可以为同一个计算模块创建不同资源配置的层级并在它们之间进行多路复用，但这种方案可能会更加复杂和繁琐。
  • 替代 Foundry 原生支持的现有功能： 理论上，计算模块可用于构建任何所需功能。然而，由于其高度通用且功能强大，这可能会导致最终方案变得更加复杂。

架构

每个计算模块由若干副本组成。副本数量会随请求量的变化而相应调整。

每个副本包含相同的一组或多组相互隔离的容器。其中一个容器作为入口点，它必须实现一个持续轮询待处理事件的客户端。其他容器则可以包含任何内容。

默认情况下，设置多容器计算模块的防护措施较少。一种建议的方法是让它们使用标准网络协议进行通信；另一种建议是使用共享卷挂载。同一副本中的容器可以通过这些方法（以及更多其他方法）进行通信，但容器无法跨副本通信，因此不应依赖它们可能累积的任何状态。

要开始使用，请查阅《在 Palantir 平台中构建计算模块》指南。
