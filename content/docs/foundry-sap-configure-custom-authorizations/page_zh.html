配置自定义授权与角色管理

面向 SAP 应用的 Palantir Foundry Connector 2.0 自 SP16 起支持自定义授权管理。可以直接在透明表上定义角色和对象访问权限，而无需依赖 SAP 标准授权管理（PFCG）。

启用此功能需运行 /PALANTIR/PARAM 事务码，并维护以下参数值：

  • 参数 ID： SYSTEM
  • 参数名称： AUTH_CHECK_SOURCE
  • 参数值： TABLE

开启该功能后，系统将不再检查现有的内容角色。如需停用此功能，请删除该参数或将参数值从 TABLE 改为 PFCG。

创建自定义角色步骤如下：

  • 运行 /PALANTIR/AUTH_01 事务码定义新角色。

    • 角色 ID 是该角色的唯一标识符，可跨所有上下文使用。

    • 对象类型是 Foundry SAP 连接器支持的对象类型：

      • TABLE
      • REMOTETABLE
      • INFOPROVIDER
      • REMOTEINFOPROVIDER
      • BEX
      • FUNCTION
      • REMOTEFUNCTION
      • SLT
      • EXTRACTOR

    • 对象即为主要抽取对象。例如，若对象类型为 TABLE，则对象应填写表名（BSEG 或 B*；支持通配符）。

    • 配置 Exc/Inc 设置以允许或拒绝访问。选择 Exclude 可拒绝访问相应对象。

  • 运行 /PALANTIR/AUTH_02 事务码，将角色分配给用户和上下文。

    • 用户是指 Foundry 用于连接 SAP 的用户，在 Foundry Source 配置中定义。
    • 若不存在远程智能体、抽取器或 SLT，则上下文应留空。
    • 同一角色可应用于多个上下文和用户。
