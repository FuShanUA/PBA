核心概念

在使用或配置您的注册中的数据生命周期之前，了解以下概念很重要。

数据生命周期策略

数据生命周期允许您在命名空间级别定义“谱系感知”保留策略，并将它们应用于这些命名空间内的数据集。一旦应用了策略，数据生命周期会根据策略配置为数据集中的所有事务分配删除日期。可以配置两种类型的策略：

  • 固定删除日期: 默认情况下，根数据集中的所有事务都会被分配相同的指定删除时间（除非配置了 截止日期）。
  • 仅最新视图: 所有根数据集中的事务，如果不在该数据集当前视图中，将被分配一个等于当前时间的删除日期。这意味着所有历史事务将被标记为立即删除，而数据集最新视图中的事务则不会被分配删除日期。

保留策略与谱系感知保留策略

Foundry 提供了两种系统化从系统中删除数据的方法：

  • 保留策略，在 保留 应用程序中定义，基于特定规则应用于数据集事务，并可以系统化地删除数据。然而，这些策略 不 具备谱系感知功能，因此不会传播到下游数据集。了解更多关于 保留策略 的信息。

  • 数据生命周期策略 与 保留 策略不同。数据生命周期策略的 谱系感知 删除机制确保当一个事务被删除时，从该事务派生的所有下游事务也会被删除。这两种方法之间的关键区别在于，数据生命周期建议将策略应用于根数据集或上游数据集，而通过 保留 管理的策略没有此要求。

尽管可以在同一注册上同时配置策略，但数据生命周期在显示事务的删除日期时不会考虑其他保留策略。例如，如果一个保留策略计划在星期二删除特定事务，而数据生命周期设置为在星期三删除同一事务，那么数据生命周期会将该事务的删除日期报告为星期三。即使实际上根据保留策略该事务会在星期二被删除，这一情况仍然成立。

权限和角色

了解权限和角色是使用数据生命周期的重要部分。了解更多关于 保护策略 的重要性。

数据生命周期中的权限默认角色如下所述：

  • 命名空间查看器： 查看保留策略。
  • 数据集编辑器： 在数据集上设置/删除保留策略。
  • 数据治理官员：
    • 为他们可以查看的命名空间创建/更新/删除保留策略。
    • 为他们可以查看的数据集设置/删除保留策略和策略覆盖。

“数据治理官员”角色对于管理和保护数据生命周期策略尤其重要。了解更多关于 在控制面板内分配权限 的信息。此外，我们建议所有组织审查并理解我们在平台内的 数据治理原则与实施。
