检查资源权限

你可以使用数据血缘关系，通过“权限”着色选项来检查用户查看数据集或产物的权限。首先，通过侧边栏的搜索辅助工具将节点添加到图表中。

接着展开图表，查看指向目标资源的血缘关系（阅读更多关于探索血缘关系的内容）。

完成上述操作后，在节点颜色选项下拉菜单中选择权限配色方案。

从查看身份下拉菜单中选择用户名。这样就能查看该用户对图表中各个节点的权限。

你可以通过以下两种权限类型进行着色：

  • 数据集中的数据访问权限
  • 资源访问权限

数据集中的数据访问权限

使用此选项排查权限问题。请注意，用户的数据访问权限受数据血缘关系影响（参见平台安全）。根据用户的数据访问权限为节点着色，可以直观地看出哪些上游数据集可能限制了用户的数据访问。

请注意，此选项仅对数据集节点生效。

资源访问权限

此选项可展示为选定用户在选定资源上设置的角色（如编辑者、查看者等）。

使用此选项查看用户对你产物的访问级别。

角色与数据血缘关系的对应方式不同于数据访问权限。例如，用户在 Contour Analysis 上拥有“编辑者”身份，并不能保证其拥有查看该分析所依赖数据的权限。在向用户共享资源时，请确保他们能够访问底层数据。
