使用动作写回数据

您可能需要配置 Slate 应用，以便将用户在交互式应用中的操作“写回”至 Foundry 的底层数据资产。在 Slate 中执行写回操作的推荐方式是使用本体动作。若要直接写回数据，可以使用 OSDK 动作或 API 网关查询。如果您倾向于为此使用提交表单，请考虑使用动作小部件。

除了通过平台选项卡从本体层读取数据外，Slate 还支持使用动作小部件通过动作写回至本体。

Slate 向本体的写回操作通过小部件进行配置，而非通过平台选项卡，这使开发者能够选择性地利用自动生成的动作表单。如此一来，开发者便无需使用 Slate 小部件重新创建所有表单元素。

在此设置中，通常会将动作小部件置于对话框容器内，并在需要显示表单时，使用 Slate 事件触发该容器打开。

或者，可以关闭表单显示，此时小部件配置将作为模板，根据应用内其他小部件中的用户交互和输入来收集待提交的值。随后通过事件触发提交，该事件通常来自按钮小部件。

动作小部件还为 Slate 事件提供了钩子，用于广播动作的验证状态及其提交成功或失败的结果。

请查阅动作相关文档以获取有关如何构建和配置动作的更多详细信息，并查阅动作小部件文档以了解 Slate 中的小部件配置详情。
