计算模块的使用与定价

计算模块允许您部署交互式容器。要了解其计算资源使用情况，您需要知道这些部署可以自动进行水平扩缩容，并且由于预测性自动扩缩容机制，即使在没有请求的情况下也可能发生扩缩容。只要您点击构建并且部署正在运行，就可能产生资源使用量。

在运行期间，任何消耗的计算资源使用量都将归属于计算模块本身。只要有“副本”正在启动或运行，就会消耗计算资源使用量；如果没有副本处于启动或运行状态，则不会消耗。使用量的多少取决于副本数量及其随时间推移产生的关联资源使用情况。

为了更好地理解您的使用情况，您需要了解：一个计算模块可以拥有多个副本，而每个副本又可以包含多个容器。

定价

计算模块的使用量以 Foundry 计算秒为单位进行跟踪。请查阅我们的《使用量类型文档》以获取更多详细信息。

只要副本处于启动或活动状态，就会计量计算秒。确定计算秒的因素有以下几点：

  • 每个副本的 vCPU 数量
  • 每个副本的 RAM 大小（GiB）
  • 每个副本的 GPU 数量
  • 副本数量
    • 任意时刻的副本数量是动态的，但它们都拥有相同的资源配置

在为 Foundry 使用量付费时，默认的使用费率如下：

vCPU / GPU使用费率
vCPU0.2
T4 GPU1.2
V100 GPU3
A100 GPU1.3
A10G GPU1.5
L4 GPU2.1
H100 GPU4.7

如果您与 Palantir 签有企业合同，在进行计算使用量核算之前，请联系您的 Palantir 代表。

vcpu_compute_seconds = max(vCPUs_per_replica, GiB_RAM_per_replica / 7.5) * num_replicas * vcpu_usage_rate * time_active_in_seconds

以下公式用于衡量 GPU 计算秒：

gpu_compute_seconds = GPUs_per_replica * num_replicas * gpu_usage_rate * time_active_in_seconds

计算模块的扩缩容配置

您可以直接在我们的配置页面上配置允许的最小和最大副本数量。

这个范围让您能够对自动水平扩缩容进行一定程度的控制。本质上，我们将被允许在任何时候扩缩容至您配置范围内的任意副本数量。您还可以设置每个副本的并发限制，这会对扩缩容产生影响。在相同负载下，较低的并发限制可能会导致更激进的水平扩容。

如果您将最小副本数量设置为非零值，即使您没有 actively 使用您的部署，也会消耗计算资源使用量。如果您将最小值设置为零，并且在一段时间内没有向您的部署发送任何请求，我们可以缩减至零，这种情况下您将不会消耗使用量。但是，只要发送请求、首次部署时，或者我们预测可能会有负载的任何时候，我们都会立即从零开始扩容。

计算模块具备预测性扩缩容功能，我们会跟踪部署的历史查询负载，并尝试先发制人地进行扩容以满足预测需求。如果我们的预测不准确，我们会利用它来优化下一次预测，并且会相对快速地缩减。该系统会遵循您配置的最大副本数量，因此您应该随时间推移监控部署的扩缩容情况，并相应地调整扩缩容设置。

计算模块的资源配额配置

计算模块应用界面允许您按副本为每个容器配置资源。CPU、内存和 GPU 资源的请求量和限制量均可按副本进行配置。

这些资源配置是可选的；在未进行配置的情况下，副本将默认使用 1 个 vCPU、4 GB 内存且无 GPU。每个副本将请求配置的资源量，并遵循已设置的限制。这意味着，如果您使用默认资源（1 个 vCPU 和 4 GB 内存且无限制）并且您有四个副本，您可能会消耗 4 个 vCPU 和 16 GB 内存（不过根据请求负载的不同，实际消耗量可能偏多或偏少）。
