提议更改

在创建分支以在工作流中进行协作后，你可以进行编辑并向 Main 分支提议更改。

要提议更改，请点击 Save 保存编辑内容，然后在顶部工具栏中选择 Propose。

在 Proposal 视图中，为你的提议命名，并提供任何相关细节，向审批者解释你的提议。在下面的示例中，我们提议添加一个 Rename column 转换，以清理来自 Facility dataset 的原始数据。

选择 Create proposal，向 Main 分支审批者发起请求。

处理更改

为了尽量减少合并冲突，你可以在完成最终合并之前，将目标分支合并到当前分支中。为此，请选择分支旁边的 Resolve changes。

系统将打开一个窗口，你可以在其中选择要合并到管道中的目标分支。选择分支后，点击 Apply changes。

重置分支

你还可以重置分支，使其与另一个分支保持一致。为此，请从分支名左侧的下拉菜单中选择 Reset branch。然后，选择要重置到的目标分支，并再次选择 Reset branch 进行确认。

请注意，如果两个分支的版本相同，该按钮将被禁用。

分支重置操作无法撤销。重置后，原分支上的所有更改都将丢失，包括已保存和未保存的更改。

重置后，当前分支将与所选分支保持相同的逻辑和版本历史。
