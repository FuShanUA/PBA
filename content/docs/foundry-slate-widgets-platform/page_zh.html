平台

平台小部件类别包括以下小部件：

  • 对象卡（Object Card）
  • 资源选择器（Resource Selector）
  • 时间序列（Time Series）
  • 操作（Action）

对象卡（Object Card）

对象卡小部件以ObjectRID作为输入。它将根据本体论（Ontology）应用程序中定义的迷你对象视图（卡片）进行渲染。对象卡将链接到其对象浏览器（Object Explorer）对象视图。

表格格式

可以使用此小部件结合可重复容器来填充对象视图的表格。

例如，假设您有一个返回RID数组的Slate函数。您可以将对象卡片小部件插入到可重复容器中以创建“表格视图”。您需要使用可重复容器的索引来对对象小部件进行索引，因此在“对象RID”中，您将指定：{{lookup f_function1 w_repeating_container.index}}。

这会基于您的函数中的RID和可重复容器的索引，使用内置查找函数建立列表。

获取对象RID

要获得ObjectRID输入，您有几个选项：

选项1：Slate提供了一个点选式UI，即对象集面板，您可以在其中进行对象查询而无需手动构建查询。

选项 2：您可以通过导航到特定对象的对象视图并选择操作下拉菜单来获取该对象的 ObjectRid。在剪贴板中，它将允许您复制一个 URL 链接。复制 URL 链接的最后一部分，即 ?objectId={ObjectRID} 之后的部分（例如，如果 URL 是 ...?objectId=ri.phonograph2-objects.main.object.09d2e0e9-dd3c-49b2-8b96-0cb1bf005c1d，则您的 ObjectRID = ri.phonograph2-objects.main.object.09d2e0e9-dd3c-49b2-8b96-0cb1bf005c1d）。

选项 3：要从 ObjectTypeId 和 ObjectPrimaryKey 获取 ObjectRID，您需要使用 Get Object by Locator 端点。虽然 ObjectTypeId 和 ObjectPrimaryKey 可以在其他地方用于参数化对象资源管理器 URL 链接，但在 Slate 中需要将它们解析为 ObjectRIDs。您可以动态组合 ObjectTypeId 和 ObjectPrimaryKey，并将其传递给 Slate 查询以检索 ObjectRID（前提是您的实例上已公开这些端点，并且您已将其设置为数据源）。

资源选择器

资源选择器小部件允许您在Compass中选择一个文件。

此小部件不会在您的页面上创建可视元素。相反，它会暴露一个操作以打开资源选择器，并提供一些事件来处理所选资源。

事件

  • w_resource_selector_1.didClose: 当资源选择器关闭时触发此事件。
  • w_resource_selector_1.selectedRid.changed: 当资源选择器中选择了一个资源时触发此事件。{{sl_event_value}} 被设置为所选资源的RID。

操作

时间序列 时间序列

Slate 的时间序列小部件提供了一种方便的方式来可视化时间序列图。您可以使用系列 ID 将多个系列添加到同一图表中。您也可以通过以下方式配置可视化效果：

  • 指定标题、单位和颜色，或者禁用十字线或图例。
  • 动态定义时间范围以指定查看窗口。
  • 为数据开启“流模式”。

Action

Slate 的 Action 小部件（操作小部件） 允许 Slate 执行预配置在 Action 中的业务逻辑。在将小部件添加到您的页面后，您可以：

  • 选择您有权限访问的 Foundry 操作（Action）,
  • 直接或通过 Slate 的 handlebars 向您的操作表单传递默认参数，并且
  • 通过 Submit 按钮直接提交操作表单，或通过 Slate 事件-操作对 w_widget.submit 间接提交。

此外，Slate 还提供了以下与操作表单相关的 Slate 事件：

  • 提交 (w_widget.success/w_widget.failure)
  • 验证状态 (w_widget.ValidationSuccess/w_widget.ValidationFailure)，以及
  • 渲染更改 (w_widget.transitioned 和 w_widget.cssClassesUpdated)。

最后，如果您需要将 UI 从用户那里抽象出来并通过 Slate 事件触发提交，也可以通过切换控件防止显示操作表单。有关更多信息，请参阅 Slate 事件。

在使用 Slate 之前，必须先创建 Foundry 操作。有关更多信息，请参阅 操作文档。

默认参数

Foundry 操作的参数可以由用户直接提供，也可以通过使用默认参数进行编程设置，这些默认参数可以在小部件的配置中设置。

日期预填充

日期应为 ISO 8601 格式；例如，YYYY-MM-DD（如 1990-01-12）是一个有效的日期格式。

时间戳预填充

时间戳应遵循 ISO 8601 格式：YYYY-MM-DD[T]HH:mm:ss.SSS[Z]（例如 1990-01-12T23:00:00.000Z）。
