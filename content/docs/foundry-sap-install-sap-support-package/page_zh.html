安装支持补丁包

支持补丁包是初始安装包发布后提供的错误修复和新功能集合。

针对 SAP 应用程序的 Palantir Foundry Connector 2.0（下称“连接器”），应通过 SPAM（支持补丁管理器）执行标准的支持补丁包安装流程。

安装支持补丁包期间，必须暂停日常维护作业调度，以及所有使用该连接器的 Foundry 数据同步任务。否则，后台进程持有的数据库锁可能导致安装失败。

请使用 SAR 格式（FOUNDRY-SAPCONN-INST-SP00SPXX.SAR）的安装文件。XX 代表支持补丁包的版本级别。SPAM 会根据系统已安装的支持补丁包级别，加载所需文件以执行升级。

  1. 下载安装包。
  2. 使用具有 SPAM 操作权限的用户登录 SAP 系统 000 客户端。
  3. 运行 SPAM 事务。
  4. 从应用服务器导入软件包。
  5. 在工具栏菜单中，选择 安装包 > 加载包 > 从前端 执行导入操作。
  6. 定义支持补丁包的安装队列。

所有连接器组件须同步升级。请将 PALCONN 与 PALANTIR 组件，或者 PALANTIR 与 PALAGENT 组件一同升级。

  1. 选择应用组件。若要一同升级 PALANTIR 组件，请选择 所有组件，随后仅选中需要升级至目标支持补丁包级别的 PALANTIR 组件。确保升级过程中未选中其他任何组件。
  2. 在 SPAM: 导入: 队列 对话框中，将 准备 设为 在对话框中开始，其余所有步骤设为 在后台继续。

若安装过程中出现警告提示，请按照警告信息中的步骤进行处理。对于连接器和远程智能体的安装，绝大多数警告信息均可忽略。尤其是标题为“开放数据提取请求”的警告信息，完全可以安全忽略。

  1. 导入成功完成后，确认该队列。

保持 SPAM/SAINT SP 级别以及 tp/R3Trans 补丁处于最新状态至关重要。如果在 DDIC_ACTIVATION 期间出现问题，请使用 SAP 提供的批量激活程序（RADMASG0），并启用 强制激活标记。

各支持补丁包之间可能包含 SAP 安全更新，这会引入新的授权对象。因此，若使用自定义角色，强烈建议将 /PALANTIR/* 角色与自定义角色的内容进行比对。若正在使用 /PALANTIR/* 角色，请确保所有用户对比均已完成，且授权 profile 已生成并显示为绿色状态。
