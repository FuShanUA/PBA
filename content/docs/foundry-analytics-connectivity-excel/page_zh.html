Excel

本指南将介绍如何使用 Foundry ODBC 驱动程序将数据集导入 Excel。

配置 Foundry 连接

  1. 打开 Windows ODBC 数据源管理器（在 Windows 搜索栏中搜索“odbc”，打开与您的 Excel 版本匹配的 32 位或 64 位版本）。
  2. 创建新的用户 DSN，选择 FoundrySqlDriver 驱动程序。为其指定一个具有明确含义的名称，例如 Foundry Excel。
  3. 在服务器处，输入您的 Foundry URL（例如：myorganization.palantirfoundry.com）。
  4. 可选：如果您使用 OAuth 进行身份验证，请设置 OAuth 属性。
  5. 可选：您可以在此处保存身份验证令牌，但建议稍后在 Excel 中导入数据时根据提示再输入。
  6. 点击确定保存 DSN。

通过 SQL 导入数据

  1. 在 Excel 中，打开数据选项卡，点击功能区上的获取数据。选择自其他源 -> 自 ODBC。
  2. 选择您在先前步骤中配置的 DSN。
  3. 在“高级选项”下，输入用于导入数据的 SQL 查询语句。
    • 若要导入数据集 /YourProject/yourdataset，可输入 SELECT * FROM "/YourProject/yourdataset"。
    • 如果您熟悉 SQL，可在此处输入包含筛选和聚合等操作的更复杂查询。
  4. 点击确定。
  5. 如果是首次导入数据，系统将提示您输入凭据。
    • 若使用 OAuth，请选择“默认或自定义”凭据类型，留空该字段并点击“连接”。
    • 否则，您需要使用身份验证令牌。选择“数据库”，并输入您的用户名。在“密码”字段中输入您的令牌，而不是您的 Foundry 密码。
  6. 点击连接。

通过表浏览器导入数据

默认情况下，如果您按照上述步骤操作但未输入 SQL 查询，表浏览器将显示为空状态。通过在 DSN 中添加 数据目录 属性并将其设置为完整的项目路径（如 MyOrg/MyProject），将连接限制为单个项目，即可解决此问题。您可以在驱动程序 DSN 设置窗口中点击附加属性，然后点击添加来完成此操作。随后，表浏览器应能正常显示，但每个 DSN 将仅限于浏览单个项目。

将数据导入 Microsoft Access

同样的 Foundry Excel 连接器也可用于将数据导入 Access 数据库。

  1. 在 Access 中，打开外部数据选项卡，点击功能区上的新数据源。选择自其他源 -> ODBC 数据库。
  2. 选择是导入源数据还是链接到源数据。
  3. 在计算机数据源选项卡下，选择您在本指南第一步中配置的 DSN。
    • 您必须已在驱动程序 DSN 设置窗口中保存了身份验证令牌。
    • 您必须已按照先前步骤的说明设置了 数据目录 属性。
    • 您的项目路径必须符合 《Access 表名限制规则》。
  4. 从表浏览器中选择您的表，然后点击确定。
