推出导出与导入功能

Apollo 导出与导入功能现已在所有 Apollo Hubs 上可用。这些工作流程的文档现在可在 公开文档 中找到。

导出和导入提供了一种无缝的方式，可在没有网络访问或无法连接到您的源Apollo Hub的环境中部署软件。通过在大多数用户都可以访问的源Hub中管理软件目录和配置的真实来源，您不需要完全依赖具有受限环境访问权限的用户。这使得开发人员、操作员和其他Apollo用户可以在各种网络边界之间对软件的配置、管理和监控拥有统一视图。

为此，从另一个Apollo Hub指定一个目标Hub，它将作为源Hub。源Hub将作为目标Hub的部分或全部产品发布信息的真实来源。更改首先在源Hub中进行，然后通过导出管道传输到目标Hub。

通过创建导出管道开始使用。

使用推广管道和发布渠道跟踪来配置模块发布

模块控制 Apollo 环境的关键属性，例如安装了哪些实体及其配置。以前，模块更新会立即传播到所有已安装该模块的环境中。这意味着发布新模块版本会对环境产生重大影响，如果出现问题则没有明确的缓解路径。

以下更新使您能够控制模块更新在您的环境中发布的流程。

模块提升管道

模块发布现在支持提升管道，可以提升到像产品一样的发布通道。请注意，目前仅支持基于时间的提升阶段，不过添加对基于金丝雀的提升的支持已在开发计划中。

您可以通过导航到模块页面的发布选项卡，并从提升管道部分选择编辑来配置模块的提升管道。

已配置提升管道的模块。

模块安装跟踪发布通道

所有模块安装现在都会跟踪环境默认发布通道。Apollo只有在模块发布被提升到该通道后才会提出模块更新。您可以覆盖新旧模块安装所跟踪的发布通道。

使用选择发行通道下拉菜单覆盖新模块安装的默认发行通道。

通过选择 更新发行通道 来覆盖现有模块安装的默认发行通道。

我们建议保持默认通道，因为模块安装应与其环境具有相同的风险承受能力。

召回模块

召回模块发布将取消所有针对该版本的更新安装的开放更改请求，并阻止Apollo建议对该版本进行升级。

Apollo 不会自动将已召回版本的模块安装降级到非召回版本。

从模块页面通过选择召回版本从操作下拉菜单。

2022-06-10

The Payload Bundler

Many Apollo-supported installations of software are only accessible from specific air-gapped secure networks with strict data controls. That means that new releases of Apollo products need to go through a special process in order to be deployed into these environments.

Apollo supports this process through an application now available in the primary sidebar navigation called the Payload Bundler. Namespace owners are the primary users of the Payload Bundler: software operators who use Apollo to make sure the environments in these secure spaces are healthy and up-to-date. They can easily prepare and manage "bundles" of software with prepared updates for all tracked environments in a remote namespace. Apollo ensures these bundles contain the releases, settings, and metadata that are critical to keeping these critical applications and environments working.
