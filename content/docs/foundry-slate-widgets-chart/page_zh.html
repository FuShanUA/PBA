图表

图表组件类别包含以下组件：

  • 柱状图
    • 带图例的柱状图
    • 堆叠柱状图
    • 横向柱状图
    • 多系列图表
  • 甘特图
  • 热力网格
  • 折线图
  • 饼图
  • 散点图
  • 矩形树图

本页包含有关图表组件可用的属性信息，以及组件的示例和常用 CSS。

图表组件属性

下表提供了图表组件可用属性的使用详情。

属性描述类型必填修改者
dataSelectionEnabled指定用户是否可以选择柱状图和散点图数据点。在“多选”模式下也可进行图例选择。所选数据通过图表的 selection.data 属性公开。（若启用了平移/缩放，则禁用此功能）boolean是直接编辑
dataSelectionMode指定数据选择行为。“单选”模式仅允许单击交互。“多选”模式允许矩形框选以及 cmd/ctrl+点击交互。（若启用了平移/缩放，则禁用此功能）string否直接编辑
animate指定图表数据在加载、更改和刷新时是否显示动画。boolean是直接编辑
areaSelectionEnabled指定用户是否可以在图表上绘制区域选择。（若轴类型为“类目”、数据选择为“多选”、存在多个坐标轴，或启用了平移/缩放，则禁用此功能）boolean是直接编辑
autorangePanZoomEnabled当为单个坐标轴启用平移和缩放时，可应用自动范围调整，以动态缩放第二个坐标轴来适应数据。boolean是直接编辑
datasets请参阅下方的 IDatasetModel 及其子类型。IDatasetModel[]是直接编辑
zeroBoundYAxisEnabled允许 Y 值范围不跨越 0 的数据集所在的量化图表将其 Y 轴边界固定为 0。警告：禁用此选项可能会导致图表在视觉上产生误导。boolean是直接编辑
hover当 tooltipsEnabled = true 时，此属性用于确定鼠标悬停所在的值。为了在其他地方链接到此变量，必须首先通过模板“hover”: { “xValue”: null, “yValue”: null } 声明它。有关更多信息，请参阅 IHover。请注意，悬停功能适用于除堆叠面积图 (stackedArea) 外的所有图表。另外请注意，悬停默认对应 yValue。IHover否用户交互
labelsEnabled启用静态标签。仅适用于柱状图。选项包括“起始”、“中间”、“末端”或“外侧”。boolean是直接编辑
labelsPosition指定柱状图标签的位置。选项包括“起始”、“中间”、“末端”或“外侧”。string否直接编辑
legendEnabled启用图例显示。可用的位置选项包括“顶部”、“底部”、“左侧”或“右侧”。boolean是直接编辑
legendPosition图例的位置。可用的选项包括“顶部”、“底部”、“左侧”或“右侧”。string否直接编辑
panZoomEnabled指定图表是否允许平移和缩放。仅适用于数值轴。禁用此选项将重置缩放比例。缩放比例在保存期间不会保留。（若启用了数据选择、区域选择，或所有轴类型均为“类目”，则禁用此功能）boolean是直接编辑
panZoomAxes启用平移和缩放的坐标轴。可用选项为“X 和 Y 轴”、“仅 X 轴”或“仅 Y 轴”。string否直接编辑
selection.area当前的区域选择。仅在启用区域选择时有效。请参阅下方的 IAreaSelection。IAreaSelection否用户交互
selection.data当前选中的图表数据。仅在启用数据选择时有效。请参阅下方的 IDataSelection。IDataSelection否用户交互
tooltipsEnabled指定是否启用工具提示。工具提示将显示由 hover 设置的数据值（默认为 y 值）。boolean是直接编辑
tooltipText工具提示中显示的文本（必须设置 tooltipsEnabled 为 true）。通常用于显示悬停值。如果省略 tooltipText 或其为空字符串，则在除水平柱状图外的所有图表中，工具提示将显示 yValue（水平柱状图将显示 xValue）。string否直接编辑
xAxesX 轴数组。IAxis[]是直接编辑
yAxesY 轴数组。IAxis[]是直接编辑
title图表的标题。string否直接编辑

属性描述类型必填修改方
formatter格式化坐标轴刻度。格式化字符串取决于坐标轴类型：对于线性/对数轴，请参阅 Numeral.js 文档 ↗；对于时间序列，请参阅 moment.js 文档 ↗；分类轴不支持此项；小数精度限制为 20 位。string否直接编辑
gridlinesEnabled指定是否显示坐标轴的网格线。目前仅适用于非分类轴boolean是直接编辑
betweenTicks指定坐标轴网格线的对齐方式是否在刻度之间，默认为 false，并将网格线渲染在刻度上boolean是直接编辑
name坐标轴名称，在 ISeries 中由 xAxisName 和 yAxisName 引用。string是直接编辑
position坐标轴位置。对于 X 轴，位置可以是顶部或底部。对于 Y 轴，可以是左侧或右侧。string是直接编辑
label与坐标轴关联的标签。string否直接编辑
scale坐标轴类型。对于 Y 轴（左侧和右侧位置），刻度类型可以是 linear、modifiedlog 或 category。对于 X 轴（顶部和底部位置），刻度类型还可以是 timeseries。注意：目前 timeseries 刻度要求日期为表示距离纪元毫秒数的整数。SQL 日期格式传入 Slate 时通常是字符串，如果时间图表无法正常工作，请尝试按如下方式将其转换为整数：MySQL: SELECT UNIX_TIMESTAMP(date_column)*1000 AS date_number 或 Postgres: SELECT EXTRACT(epoch FROM date_column)*1000 AS date_number。请记得在上述示例中将 xValues 更新为新创建的数字列，即“{{query1.date_number}}}”string否直接编辑
scaleMax坐标轴的最大值（如未指定，坐标轴将自动调整范围）。number否直接编辑
scaleMin坐标轴的最小值（如未指定，坐标轴将自动调整范围）。number否直接编辑
tickInterval刻度值之间的间隔。仅适用于采用线性刻度的坐标轴。number否直接编辑
tickLabelAngle旋转分类轴的刻度标签：90、0 和 90。仅适用于采用分类刻度的坐标轴。number否直接编辑
visible指定是否显示坐标轴。boolean是直接编辑
maxZoomSpan缩小操作时坐标轴显示的最大数值跨度。无法在此值之外进一步缩小。number否直接编辑
minZoomSpan放大操作时坐标轴显示的最小数值跨度。无法在此值之外进一步放大。number否直接编辑

教程：添加散点图部件


假设你要创建一个散点图部件，用于展示包含距离和时间值的路线指标。选择左上角的部件按钮，然后选择图表 > 散点图，即可向你的应用中添加一个散点图部件。该部件会预填充一些示例数据，你可以在继续操作前从右侧编辑器面板中清除这些数据。


首先，将该图表部件重命名为易于识别的名称，例如 w_routeMetrics。


我们希望沿 x 轴绘制路线的平均距离。这意味着你必须将X 值设置为 "{{q_routeMetrics.avg_distance}}"。


接下来，沿 y 轴绘制路线的持续时间。在数据选项卡中将Y 值设置为 "{{q_routeMetrics.avg_time}}"。


为了使散点图更易于阅读，可以为坐标轴添加标签。选择坐标轴选项卡，将第一个X 轴对象的名称从 x1 改为 Distance，并添加诸如 Distance (miles) 的标签来描述所绘制的信息。你还可以开启网格线，以便更好地识别图表上各个点的位置。




将你的Y 轴命名为 Duration，并进行如下配置：




最后，为你的数据点设置不同的大小。我们希望点的大小能根据路线的繁忙程度而变化。为此，切换回数据选项卡，并将半径设置为 "{{q_routeMetrics.num_flights}}"。


再次查看图表。似乎有些不对劲，因为图表现在被填满了。这是由于半径值是按照提供的值与半径像素数 1:1 的比例绘制的，而我们有些路线的航班数量超过了 150。你必须对半径值进行缩放。你可以通过在查询中计算一个“显示”值来实现这一点。你可以将此值硬编码，但使用变量能让你在后续无需编辑查询即可轻松进行修改。


选择顶部栏的变量选项卡，打开变量窗口。在右下角创建一个新变量。将新变量命名为 v_routeCountDisplayScale，并将值设置为 100。


返回到 q_routeMetrics 部件并添加一个新列：




COUNT(flight_id)/{{v_routeCountDisplayScale}} as num_flights_disp


在此处，同时为每一行生成一个显示标签：


CONCAT(origin, ' -> ', dest) as route_name


现在，你的完整查询语句如下所示：




已复制！
1SELECT
2    origin,
3    dest,
4    AVG(distance) as avg_distance,
5    AVG(actual_elapsed_time) as avg_time,
6    COUNT(flight_id) as num_flights,
7    COUNT(flight_id)/{{v_routeCountDisplayScale}} as num_flights_disp,
8    CONCAT(origin, ' -> ', dest) as route_name
9FROM "foundry_sync"."{{v_flightTable}}"
10GROUP BY
11    origin,
12    dest
13ORDER BY
14    COUNT(flight_id) DESC
15LIMIT 50



返回 w_routeMetrics 控件，调整 Radius 配置，使其引用 num_flights_disp 列而非 num_flights。


为完成图表配置，在 w_routeMetrics 配置面板顶部的 Title 输入框中添加标题 Route Metrics。

甘特图


甘特图将基于时间的数据显示为沿分类 Y 轴分布的水平条形图。你可以使用甘特图来可视化日程安排、项目时间线、事件持续时间，以及任何包含项目起始时间和结束时间的数据集。甘特图具备上文记录的图表组件属性。




属性


IGanttDatasetModel


甘特图数据集模型扩展了 IRangeDatasetModel 和 ILabelsModel。startValues、endValues 和 labels 属性已在上文 IRangeDatasetModel 和 ILabelsModel 表格中说明。共享的 name、renderer、seriesColors、seriesNames 和 seriesValues 属性已在 IDatasetModel 表格中说明。





















属性说明类型必填修改方式
yValues分类值数组，用于决定每个条形的 Y 轴位置。每个值对应 Y 轴上的一行。该数组的长度必须与 startValues 和 endValues 的长度相匹配。any[]是直接编辑


X 轴必须使用 timeseries 刻度，Y 轴必须使用 category 刻度，并且数据集的 renderer 必须设置为 "gantt"。startValues、endValues 和 yValues 数组的长度必须全部相同。如果提供了 labels，其长度也必须与 yValues 相匹配。

树状图


树状图小部件提供了一种灵活的方式，通过一组嵌套的矩形来可视化层级数据。它有助于发现其他可视化方式中难以揭示的模式。


树状图用于可视化给定实体（label）的绝对数量（size），该实体可能属于某个（可选的）category，并带有某个（可选的）相对数量（density）。
每个单元格矩形的属性包括：面积与size成正比；颜色由density和category决定；名称由label指定。


使用此图表可视化的数据集示例：


    

  • 属于某个行业（category）的股票（label），具有一定的市场份额（size）和一定的价格涨跌幅（density）
    • 





    

  • 按地区（category）划分的工厂位置（label），生产一定数量的产品（size），并具有一定的误报率（density）
    • 





    

  • 集群的节点（label），具有存储空间（size）和一定的存储空间利用率（density）
    • 





    

  • 属于某个世界地区（category）的国家（label）的人口规模（size）
    • 





数据配置


    

  • label（可选）：显示在每个单元格上的名称
    • 

  • size：决定单元格矩形的大小
    • 

  • density（可选）：决定单元格颜色的深浅
    • 

  • category（可选）：决定单元格的分组，进而决定单元格的颜色
    • 



在默认示例中，单元格矩形对应于单元格的size。有两个类别，I和II。请注意，H的值最大（45），因此它是最大的矩形。H的密度最小（3），所以它的颜色最深（可以通过颜色面板下的复选框反转渐变）。









颜色配置


根据是否启用 categories 和/或 densities，可以使用不同的颜色配置。


启用类别和启用密度


同时启用 categories 和 densities 时，单元格的颜色将根据 色相-饱和度-明度 (HSV) ↗ 颜色空间中的渐变来定义。每个类别都会定义一个色相。可以提供色相列表（0 到 360 之间的数字），或者提供将类别名称映射到色相的字典。随后，所有类别的渐变都将使用相同的已定义起始饱和度和明度。渐变通过渐变类型（变亮、变暗、增加饱和度、降低饱和度）以及渐变强度来定义。






仅启用类别


仅启用 categories 时，你可以为各个类别指定颜色列表，或者提供将类别名称映射到颜色的字典。


颜色可以是十六进制格式（#FF0000、#00FF00），也可以是标准颜色名称（例如 red、green）。






仅启用密度


仅启用 densities 时，你可以定义一个颜色列表，这些颜色将作为带渐变的范围应用。你必须至少提供两种颜色来定义颜色渐变，不过可以提供更多颜色以更精细地定义此颜色渐变。


颜色可以是十六进制格式（#FF0000、#00FF00），也可以是标准颜色名称（例如 red、green）。






禁用类别和禁用密度




在禁用 `categories` 和 `densities` 的情况下，你可以定义一个颜色列表作为调色板。该列表的长度必须与数据点的数量保持一致。

颜色可以使用十六进制格式（如 #FF0000、#00FF00）或标准命名格式（如 red、green）。





杂项配置

支持配置以下可选的杂项参数：

- 单元格样式
  - 每个单元格的边框宽度
  - 边框颜色
  - 单元格标签
  - 单元格工具提示
- 图例
  - 图例位置（顶部、底部、左侧、右侧）
  - 图例标签（用于图例标题）
- 单元格生成策略。平铺策略由 d3 ↗ 定义：Binary、Dice、Resquarify、Slice、Slice Dice、Squarify。
- 选择模式（多选或单选）
