通用连接器

通用连接器可用于表示与任意外部系统的连接。作为连接器，它不直接支持其他连接器中提供的标准< a href="/docs/foundry/data-connection/core-concepts/#capabilities">功能（capabilities）集合。然而，当仅作为< a href="/docs/foundry/data-connection/core-concepts/#use-in-code">在代码中使用（Use in code）的连接器时，可以用来创建这些标准功能的代码替代方案，包括批量同步、文件和表导出、流式同步、流式导出、媒体同步、Webhook等。

通过配置适当的worker（工作进程）、凭据和导出控制，开发人员可以使用它来编写连接到任意外部系统的代码。当源所有者一起配置网络、worker（工作进程）、凭据和导出控制时，可以确保凭据仅用于指定的外部系统，并在建立连接的不同代码环境中强制执行导出标记。

设置

通用连接器应该从代码仓库环境侧边栏中的 新建连接 选项创建，而不是通过数据连接应用程序。然而，任何配置好的通用连接器都会与其他连接器一起出现在数据连接中。

有关如何直接从代码仓库中设置通用连接器的更多信息，请参阅 我们的外部转换教程。

网络

由于通用连接器必须在代码中使用，只有 Foundry 工作员 连接可以使用，既可以使用 直接连接出站策略 用于可以接受来自 Palantir 的入站流量的系统，也可以使用 代理代理出站策略 用于位于无法接受来自 Palantir 的入站流量的私有网络中的系统。

配置选项

从概念上讲，你可以将通用连接器视为一组 网络配置、凭证 和 可导出的标记，这些配置旨在一起使用。

  • 网络配置 指定使用此通用连接器时应能访问的系统。
  • 凭证 指定必须可用以成功连接的秘密值。
  • 可导出的标记 指定哪些数据可以安全地离开 Palantir 平台到指定的网络目标。

为了提供上述功能，以下配置选项可用于通用连接器：

选项是否必需？ 描述
Networking是一组 出站策略，规定哪些目标地址或IP应该可访问。
Credentials否一组键值对可用于存储凭据。目前，只能存储密钥值。不支持明文值。
Exportable control markings否如果将通用连接器与Foundry数据输入一起使用，则必须启用允许导出的设置，并指定可导出的标记集。有关数据连接导出控制的更多信息，请参阅 我们的文档。

Python外部转换

可以将通用连接器导入代码仓库。使用此通用连接器，您可以编写代码以访问外部系统并获取源端的凭据。

了解有关基于源的外部转换的更多信息。

计算模块

通用连接器可以导入到一个 计算模块 中。使用此通用连接器，您可以编写与外部系统交互的计算模块。

TypeScript 函数

通用连接器可以导入到一个 TypeScript 函数代码仓库 中。使用这个通用连接器，您可以通过 fetch 直接调用外部系统。

Python 函数

通用连接器可以导入到一个 Python 函数代码仓库 中。使用这个通用连接器，您可以通过 Python 的 requests 库直接调用外部系统。

通用连接器的转换

通用连接器可以转换为 REST API 源，如果该源针对 DNS 地址和端口 443 仅有一个出站策略的话。

执行此转换将使开发人员能够使用 内置 HTTP 客户端，用于代码中的使用，使开发人员能够立即编写与外部系统交互的代码。
