通用连接器

通用连接器可用于表示与任意外部系统的连接。作为一种连接器，它不直接支持其他连接器中提供的标准功能集合。然而，当专门将其作为 代码中使用 连接器时，可利用它为这些标准功能创建基于代码的替代方案，包括批量同步、文件与表导出、流式同步、流式导出、媒体同步、Webhook 等。

通过为通用连接器配置合适的 Worker、凭据和导出控制，开发者可借此编写代码来连接任意外部系统。数据主人将网络、Worker、凭据和导出控制配置组合在一起后，即可确保凭据仅用于指定的外部系统，且导出标记会在建立连接的不同代码环境中得到严格执行。

设置

应从代码仓库环境侧边栏的 新建连接 选项创建通用连接器，而非通过数据连接应用程序创建。不过，任何已配置的通用连接器都会与数据连接中的其他连接器一起显示。

请查阅《外部转换教程》，详细了解如何直接从代码仓库设置通用连接器。

网络

由于通用连接器必须在代码中使用，因此只能使用 Foundry worker 连接：对于可接受来自 Palantir 入站流量的系统，可配合 直接连接出口策略 使用；对于位于私有网络且无法接受来自 Palantir 入站流量的系统，则可配合 代理出口策略 使用。

配置选项

从概念上讲，您可以将通用连接器视为一组旨在配合使用的 网络配置、凭据 和 可导出标记。

  • 网络配置 指定使用该通用连接器从代码中可访问的目标系统。
  • 凭据 指定建立成功连接所必需的密钥值。
  • 可导出标记 指定哪些数据可以安全地离开 Palantir 平台并流向指定的网络目的地。

为实现上述功能，通用连接器提供以下配置选项：

选项是否必填？说明
网络是一组 出口策略，用于指定应允许访问的目标地址或 IP。
凭据否可使用一组键值对来存储凭据。目前仅支持存储密钥值，不支持存储未加密的值。
可导出控制标记否如果通用连接器将与 Foundry 数据输入配合使用，必须启用允许导出的设置，并指定一组可导出标记。有关数据连接导出控制的更多信息，请参阅《相关文档》。

在代码中使用通用连接器

本节提供关于如何在 Foundry 的各种代码环境中使用通用连接器的更多信息。

代码环境说明
Python 外部转换使用通用连接器从 Python 转换仓库连接外部系统。
计算模块使用通用连接器从长期运行的计算模块建立连接。适用于流式同步与导出工作流，以及定制化的数据回写工作流。
TypeScript 函数正式发布。
Python 函数正式发布。

Python 外部转换

通用连接器可导入至代码仓库中。利用该通用连接器，您可以编写代码来访问外部系统并读取源上的凭据。

了解有关基于源的外部转换的更多信息。

计算模块

通用连接器可导入至 计算模块中。利用该通用连接器，您可以编写与外部系统交互的计算模块。

TypeScript 函数

通用连接器可导入 TypeScript 函数代码仓库。借助该通用连接器，可通过 fetch 直接调用外部系统。

Python 函数

通用连接器可导入 Python 函数代码仓库。借助该通用连接器，可通过 Python 的 requests 库直接调用外部系统。

通用连接器转换

如果源上仅绑定了一个针对 443 端口 DNS 地址的流出策略，即可将通用连接器转换为 REST API 源。

完成转换后，开发者可在代码中使用 内置 HTTP 客户端，进而直接编写与外部系统交互的代码。
