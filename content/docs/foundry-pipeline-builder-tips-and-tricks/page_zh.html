更快地构建

配置用户偏好

  1. 导航到Pipeline Builder视图右上角的设置菜单，然后选择用户偏好，配置并保存您的设置。





最常用的两个设置是 折叠转换样式 和 自动命名转换路径。在 用户首选项 中的 折叠转换样式 选择器中，将该设置切换为 伪代码，会以类似于编程语言的格式显示您的转换。伪代码不遵循任何特定编程语言的语法。

快捷键（自动布局、缩放、隐藏等）

  1. 在 Pipeline Builder 的左上角打开 帮助 下拉菜单。



  2. 选择 查看快捷键 打开Pipeline Builder的完整快捷键参考。



一些最常用的快捷键包括：

  • Up Arrow 以缩放并适应所有管道节点到屏幕
  • Cmd+H (macOS) 或 Ctrl+H (Windows) 隐藏所选节点
  • Cmd+K (macOS) 或 Ctrl+K (Windows) 显示所有隐藏的节点
  • Cmd+D (macOS) 或 Ctrl+D (Windows) 选择一个节点的子节点
  • Cmd+E (macOS) 或 Ctrl+E (Windows) 选择一个节点的父节点
  • Cmd+C (macOS) 或 Ctrl+C (Windows) 将节点复制到剪贴板

复制和粘贴节点

  1. 选择您要复制的节点。使用 选择 在右上角的按钮，或使用 Cmd+A（macOS）或 Ctrl+A（Windows）来选择当前管道中的所有节点。如果需要，您也可以选择单个节点，或者通过选择并拖动来选择一组节点。
  2. 通过右键点击并选择复制，或使用 Cmd+C（macOS）或 Ctrl+C（Windows）来复制您的选择。



  3. 导航到目标管道并通过右键点击并选择粘贴，或使用 Cmd+V（macOS）或 Ctrl+V（Windows）进行粘贴。

点击并拖动以连接节点

  1. 选择作为输入的节点的输出圆圈。



  2. 将该输出拖动到应接收其输出的节点的输入圆圈上。

查看未保存的更改

  1. 导航到屏幕右侧的侧边栏菜单并选择查看更改。在此处，您可以查看当前分支中的未保存更改，并与其他Pipeline Builder分支进行比较。

决定每个节点应放置多少转换

  • 每个节点上的转换数量没有固定限制。作为一般最佳实践，应将较大的转换函数跨多个节点连接起来，而不是将所有内容都放在一个节点中。这使您的管道更容易维护、调试和更新。
  • Pipeline Builder还支持 自定义函数，允许您将一系列转换板打包成一个可重复使用的转换，并在整个管道中重复使用。在转换节点中选择+图标以配置自定义转换路径。

使用大语言模型生成概念数据

  1. 在您手动输入的表格中添加列后，从右上角选择生成数据。
  2. 对于每一列，使用用下拉菜单选择数据类型，或者选择用大语言模型生成以输入自定义提示。
  3. 配置好您的提示后，在右下角选择生成表格来填充表格。

用于构建和理解的AI工具

要直接在您的管道中使用大型语言模型（LLMs），请参见 在Pipeline Builder中使用LLMs。

使用AI生成来构建新的转换节点

  1. 选择您想要构建的节点，并从菜单中选择生成。



  2. 接下来，在文本框中输入您的提示，并选择生成。



  3. Pipeline Builder会在所选节点之后生成新节点，您可以像其他节点一样重命名它。

使用AI解释生成您的管道逻辑摘要

  1. 选择您要解释的节点，并在图表视图的顶部中心选择紫色灯泡图标。Pipeline Builder会在文本框中生成所选节点逻辑的书面摘要。



  2. 您也可以直接在节点中使用解释功能，通过选择转换板顶部右侧的紫色灯泡。

使用AIP正则表达式助手通过LLM生成正则表达式模式

  1. 选择您想要运行正则表达式函数的数据集，然后打开转换面板。
  2. 在转换窗口中，选择您需要的正则表达式函数和目标列。
  3. 点击模式输入框右侧的紫色星星图标，然后输入您的提示。



  4. 选择生成，然后通过选择应用来验证输出结果。返回到图表视图并预览转换的输出以确认结果。

使用AI FDE支持管道创建和构建

  1. 导航到屏幕左侧的应用程序菜单，并搜索AI FDE。
  2. 打开AI FDE应用程序后，可以通过将链接复制粘贴到聊天框中，将您现有的管道作为资源添加。
  3. 从那里，您可以提示AI FDE在您的管道中构建额外的节点。

改进管道

更快的管道模式

  1. 创建新的批处理管道时，从 选择批处理计算 菜单中选择 更快。
  2. 您也可以将现有的标准批处理管道转换为更快的管道。在您的标准管道中的 设置 菜单中选择 转换为更快的管道 选项。



  3. 一旦选择了 转换为更快的管道 选项，请选择 转换管道 按钮以确认转换。

如果您的管道包含与更快的管道不兼容的功能，您将收到一条错误消息，指示您在转换前需要删除或更改的内容。您还可以打开转换检查，在底部面板中查看这些警告。

更快的管道功能支持

更快的管道支持标准批处理管道中可用功能的一个子集。了解有关 使用Pipeline Builder的更快的管道 的更多信息。

用于减少构建时间的计算配置文件

  1. 一旦您的管道创建完成，请导航到屏幕最右侧的构建设置菜单以进一步调整计算设置。
  2. 在构建设置菜单中，进入默认计算配置文件 并选择显示您当前计算配置大小的下拉菜单。
  3. 选择配置管理策略下拉菜单。有三个选项可用。选择最适合您需求的一个：
    • 标准: 默认的计算配置设置，没有额外优化。最适合具有可预测数据量的生产规模管道。
    • 热池: 持续运行虚拟机以减少管道启动时间。最适合小规模、迭代式管道和低延迟开发任务。
    • 托管配置: 根据您管道的最后五次运行自动调整计算资源分配。最适合数据量波动或需要计算成本管理的管道。

更快的管道支持计算配置

更快的管道仅支持标准配置管理策略。

筛选预览数据

  1. 右键点击要采样的输入数据集，并从下拉菜单中选择采样策略。您配置的采样策略将适用于后续的所有节点。



  2. 选择要在其上配置采样策略的管道中的数据集。过滤预览允许您根据任何表达式过滤行。在下面的例子中，只有当original_language等于en时，这些行才会被保留。

管道可靠性单元测试

  1. 通过选择右侧菜单中的图标，选择+ 新建测试按钮，并选择要测试的转换。




  • 接下来，通过选择一个模式（手动输入或从现有数据集获取），在单元测试输入节点中输入一行测试数据，并在单元测试输出节点中输入预期的输出来配置您的测试。



  • 一旦测试配置完成，选择运行测试并查看结果。有关更多信息，请参阅 Pipeline Builder 中的单元测试。

    • 清晰地组织

    文件夹和颜色组用于组织

    1. 要 在 Pipeline Builder 中为您的节点设置文件夹，请在屏幕最右侧选择文件图标。在 Pipeline 文件树 panel, you can create new folders, place nodes into folders, and show or hide nodes in specific folders.



    2. You can also color nodes and collapse multiple nodes into a single node based on the coloring.

    Text nodes for documentation

    1. Select Add new text node from the graph view toolbar. Text nodes support Markdown and are positioned behind any nodes in your graph. Learn more about text nodes.

    Find and replace column names

    1. Navigate to the search panel on the right-hand side of your screen. From there, you can search for and replace column names in your pipeline.

    Auto-layout nodes and grid snapping

    Use the automatic layout and grid snapping options from the graph view toolbar to align your nodes and improve readability.
