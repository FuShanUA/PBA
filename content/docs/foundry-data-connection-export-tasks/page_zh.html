导出任务（旧版）

我们通常不建议使用导出任务将数据写回外部源。然而，根据您在Foundry的注册情况，某些源类型可能提供并支持导出任务。

以下导出任务文档是为尚未迁移到我们< a href="/docs/foundry/data-connection/export-overview/">推荐的导出工作流的导出任务用户准备的。

概述

导出任务允许您将数据从Foundry导出到各种外部数据源。导出任务的配置包括两个部分：

  • 源: 定义代理如何连接到数据源。应以与数据集同步相同的方式进行配置。
  • 任务配置： 定义了导出的类型、输入数据集和导出参数。

当在任务输出数据集上启动作业时，会触发导出任务。

已知的导出任务限制

  • 导出任务不与标记和导出控制集成。通过导出任务导出的数据不需要在导出的数据集或流上具有取消标记权限。
  • 导出任务未针对性能进行优化。导出大量数据可能导致长时间运行的任务或无法完成的任务。
  • 导出任务没有用于配置的用户界面，而是必须使用YAML来提供所需的配置选项。并非所有导出任务选项都为自助服务使用进行了文档记录；在某些情况下，只能通过Palantir的支持来配置导出任务。
  • 导出任务仅适用于 代理-工作程序源。

入门

要配置导出任务：

  1. 在数据连接中配置一个源，以连接到你要导出的目标系统。
  2. 在Foundry文件系统中，导航到包含源的文件夹。右键单击资源并选择创建数据连接任务。
  3. 在任务编辑器UI中：
    • 保留名为inputSource的源
    • 添加名为inputDataset的输入数据集
    • 创建或选择名为outputDataset的输出数据集
    • 配置特定于你导出类型的YAML

输入源必须命名为inputSource，输出数据集必须命名为outputDataset。如果未使用这些特定名称，可能会导致任务错误。

支持的导出类型

数据连接导出任务支持写入广泛的常见企业系统：

  • Amazon S3
  • OneLake 和 Azure Blob 文件系统（ABFS）
  • HDFS
  • JDBC 兼容系统，包括：
    • 关系型数据库
      • PostgreSQL
      • Microsoft SQL Server
      • MySQL
    • 数据仓库
      • Teradata
      • Snowflake
      • Vertica
  • 文件系统，包括网络文件系统 挂载在中间代理上
  • SFTP

常用配置选项

增量导出

所有导出类型都支持 incrementalType 参数，该参数控制数据随时间的导出方式。对于JDBC导出，此参数是按数据集指定的，而不是全局的。

当设置为 快照（默认值）时，导出任务会导出当前输入数据集视图中可见的所有文件。这意味着每次导出都将包含完整的数据集，无论之前导出了什么。

当设置为 增量 时，导出行为将进行优化以提高效率。首次导出的行为类似于快照，导出所有可用的数据。在后续导出中，只有自上次导出以来新增的事务会被包括在内，前提是初始导出的事务仍存在于数据集中。如果初始事务不再可用（例如由于数据集重建），系统会自动回退到完整的快照导出。

基于文件的导出示例：

Copied!
1incrementalType: incremental

路径重写

基于文件的导出支持rewritePaths配置选项，用于在导出时自定义文件名和路径。此字段接受正则表达式和替换模板的映射：

Copied!
1rewritePaths:
2  "^spark/(.*)": "$1"  # Removes the spark/ prefix

替换模板支持几种动态替换模式：

  • 正则表达式捕获组，如$1、$2等，用于引用原始路径中匹配的部分。
  • 时间戳模式，如${dt:yyyy-MM-dd}，使用Java的DateTimeFormatter语法插入当前日期和时间。
  • 使用${transaction}的Foundry事务ID
  • 使用${dataset}的Foundry数据集ID
