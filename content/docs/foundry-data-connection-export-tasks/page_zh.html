导出任务（旧版）

通常情况下，我们不建议使用导出任务将数据回写到外部源。不过，根据您在 Foundry 平台上的订阅方案，部分源类型可能仍会提供并支持导出任务。

以下导出任务文档专为尚未迁移至我们《推荐导出工作流》的导出任务用户编写。

概述

导出任务支持将数据从 Foundry 平台导出至多种外部数据源。导出任务的配置包含两部分：

  • 源：定义智能体连接数据源的方式。其配置方式应与数据集同步保持一致。
  • 任务配置：定义导出类型、输入数据集以及导出参数。

在任务输出数据集上启动作业时，便会触发导出任务。

已知的导出任务限制

  • 导出任务未与标记和导出管控功能集成。通过导出任务导出的数据，无需在导出的数据集或数据流上具备取消标记的权限。
  • 导出任务未针对性能进行优化。导出大量数据可能会导致作业运行时间过长或无法完成。
  • 导出任务没有提供用户配置界面，必须通过编写 YAML 来设定所需的配置项。并非所有导出任务的选项都提供了可供自助参考的文档；在某些情况下，导出任务只能通过 Palantir 的支持协助进行配置。
  • 导出任务仅在《智能体工作节点源》上可用。

入门指南

配置导出任务的步骤如下：

  1. 在数据连接中配置源，以连接至目标导出系统。
  2. 在 Foundry 文件系统中，进入包含该源的文件夹。右键点击资源并选择创建数据连接任务。
  3. 在任务编辑器界面中：
    • 保留名为 inputSource 的源
    • 添加名为 inputDataset 的输入数据集
    • 创建或选择名为 outputDataset 的输出数据集
    • 针对特定的导出类型配置 YAML

输入源必须命名为 inputSource，输出数据集必须命名为 outputDataset。若不使用这些特定名称，可能会导致任务报错。

支持的导出类型

数据连接导出任务支持写入多种常见的企业级系统：

  • 《Amazon S3》
  • 《OneLake 与 Azure Blob 文件系统 (ABFS)》
  • 《HDFS》
  • 《兼容 JDBC 的系统》，包括：
    • 关系型数据库
      • 《PostgreSQL》
      • 《Microsoft SQL Server》
      • MySQL
    • 数据仓库
      • Teradata
      • 《Snowflake 平台》
      • Vertica
  • 文件系统，包括《挂载在中间智能体上》的网络文件系统
  • 《SFTP》

常见配置选项

增量导出

所有导出类型均支持 incrementalType 参数，该参数用于控制数据随时间推移的导出方式。对于 JDBC 导出，此参数按数据集单独指定，而非全局设定。

当设置为快照（默认值）时，导出任务会导出输入数据集当前视图中可见的所有文件。这意味着每次导出都会包含完整的数据集，无论此前曾导出过什么内容。

设置为 增量 时，导出行为会发生变化以优化效率。首次导出类似于快照，会导出所有可用数据。在后续导出中，只要初始导出的交易记录仍存在于数据集中，系统就只会包含自上次导出以来新增的交易记录。如果初始交易记录已不存在（例如由于数据集重建），系统会自动回退到完整快照导出。

基于文件的导出示例：

已复制！
1incrementalType: incremental

路径重写

基于文件的导出支持 rewritePaths 配置选项，用于在导出过程中自定义文件名和路径。该字段接受一个包含正则表达式和替换模板的映射：

已复制！
1rewritePaths:
2  "^spark/(.*)": "$1"  # 移除 spark/ 前缀

替换模板支持以下几种动态替换模式：

  • 正则表达式捕获组，例如 $1、$2 等，用于引用原始路径中匹配到的部分。
  • 时间戳模式，例如 ${dt:yyyy-MM-dd}，使用 Java 的 DateTimeFormatter 语法插入当前日期和时间。
  • 使用 ${transaction} 表示 Foundry 交易记录 ID
  • 使用 ${dataset} 表示 Foundry 数据集 ID
