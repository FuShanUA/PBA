传感器对象类型设置

本页包含有关如何设置和配置传感器对象类型的说明。

传感器对象或 TSP

如果预期时间序列数据仅出现在给定对象类型的一部分对象上，应创建传感器对象并将其链接回那些根对象类型。然而，如果预期时间序列数据将出现在给定对象类型的（几乎）所有对象上，则应直接在该对象类型上添加 TSP。如需详细了解这些设置选项，请参阅《时间序列文档》。

前提条件：时间序列对象类型底层数据集

首先，请按照《创建新的时间序列对象类型》中的说明进行操作。您的时间序列对象类型底层数据集应具有以下模式：

列类型描述
主键String[必填] 每行的主键。
序列 IDString[必填] 唯一 TSP 的序列 ID。请注意，如果您的传感器对象类型由多个时间序列同步作为底层数据，则将需要限定序列 ID。TSP 不能作为主键或标题属性。
传感器名称String[必填] 用于标识给定传感器对象的时间序列数据所代表含义的名称。链接到同一根对象的所有传感器对象的传感器名称必须唯一。
外键String[必填] 用于将传感器对象类型链接到根对象类型的外键。主键可起到此作用，但传感器对象类型至少需要一种链接类型。
是否为分类Boolean[如果 TSP 由数值和分类类型的多个同步作为底层数据，则为必填] 布尔值 true 表示给定的传感器对象具有分类时间序列数据，否则数据将被视为数值数据。如果您是从 Measures 设置迁移过来的，此项以前称为 is_enum。
单位String[可选] 给定传感器对象应显示的单位。
内部插值String[可选] 给定传感器对象应使用的内部插值方式。

设置传感器对象类型

在 Ontology Manager 的功能页面中，找到时间序列部分并选择开始使用。

按照对话框中的前两个步骤来设置您的 TSP。在时间序列设置对话框的最后一步中，选择传感器对象类型选项：

传感器对象类型配置选项

添加该属性后，您会在时间序列属性表格下方看到一个展开的传感器对象类型配置部分，其中包含以下配置选项：

  • 传感器链接 (必填)
    • 您必须至少选择一种链接类型，将此传感器对象类型链接到记录其时间序列数据的根对象类型。
    • 您还必须选择包含此链接类型对应的传感器名称的属性。
  • 主传感器链接 (如存在则为必填)
    • 仅当您的 Foundry 实例中仍可访问旧版本的 Quiver 时，此项才会出现。
    • 如果出现此项，您必须选择一种链接类型和传感器名称，以与上方传感器链接部分中已配置的条目之一相匹配。
  • 是否为分类？ (如果 TSP 由不同类型的多个同步作为底层数据，则为必填)
    • 如果您的 TSP 由数值和分类同步混合而成的多个同步作为底层数据，您必须选择一个布尔属性，用于指示传感器对象是否具有分类时间序列数据。
  • 单位 (可选，推荐)
    • 可选择包含每个传感器对象时间序列数据单位的属性。
  • 内部插值 (可选)
    • 可选择包含每个传感器对象时间序列数据应使用的内部插值类型的属性。
    • 有效值为：
      • LINEAR：在两点之间进行线性插值。仅适用于数值时间序列。
      • NEAREST：取最近点的值。
      • PREVIOUS：取前一个点的值。
      • NEXT：取下一个点的值。
      • NONE：不进行插值。

      • 传感器对象类型的单位和插值设置应在传感器对象类型配置部分完成，而非通过基础格式化器进行设置。

      从度量迁移

      如果在 Ontology Manager 中某个对象类型的功能页面看到如下所示的警告信息，说明该对象类型之前被配置为度量。目前度量功能已弃用，你可以改用传感器对象类型来完成类似的工作流。

      你的对象类型可能已经完成全部或部分向传感器对象类型的迁移。如果在 Ontology Manager 的功能页面中的时序部分看到开始使用按钮，说明你的度量尚未迁移。请按照以下步骤将度量迁移至传感器对象类型。如果在该部分的传感器对象类型区域看到绿色的勾选标记并伴随全部完成字样，则说明你的对象类型已完全迁移。相关示例见本页底部。如果你的设置与上述两种情况均不符，则说明迁移仅完成了一部分，请按照屏幕提示完成迁移操作。

      除非你确信自己清楚是哪个时序同步回了你的度量，否则不建议进行迁移。如果你不确定是哪个时序同步回了你的度量但仍希望执行迁移，请联系 Palantir 支持。

      1. 确定哪个时序同步回了你的度量。若有多个时序同步支撑该度量，则需生成限定序列 ID。详细了解如何设置限定序列 ID。
      2. 在度量对象类型中，序列 ID 列很可能被用作主键。传感器对象类型不允许这种做法，因为 TSP 不能作为主键使用。根据当前作为主键的列的具体情况，你可以选择以下方案：
        • 方案 1： 如果可以使用序列 ID 以外的其他列作为主键，则无需进一步操作。
        • 方案 2： 如果之前将序列 ID 列作为度量的主键，则需要创建一个新列作为主键。
          • 创建该列最简单的方法是打开生成支撑数据集的 Pipeline Builder 管道或代码仓库，然后创建一个与序列 ID 内容重复的列。
          • 注意： 任何不存在重复值的非时序属性均可用作主键。请根据你的实际用例选择最合适的列。
        • 方案 3： 如果由于度量由多个时序同步支撑而不得不重新格式化序列 ID 列，只要该列是唯一标识符，就可以将原先的序列 ID 列用作主键。
      3. 在 Ontology Manager 中打开该度量并进入功能页面。点击开始使用并按前文所述完成流程配置。请注意，传感器对象类型的开关应保持开启状态，且许多选项可能已自动为你填充完毕。
