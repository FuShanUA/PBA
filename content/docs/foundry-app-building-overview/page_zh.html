用例开发

Palantir 平台旨在为多元化的构建者社区提供一系列强大的用例开发工具，包括应用构建工具、工作流构建工具、集成的分析工具以及开发者工具。所有这些工具都依托于 Foundry 核心的安全、血缘关系、数据和计算原语，使团队能够专注于交付运营能力，而非管理基础设施。至关重要的是，Palantir 平台中的每一款工具都旨在持续、安全地丰富一组一致的数据和模型资产（包含在 Ontology 中）。这使得知识能够随着运营工作流在企业范围内的扩展而不断积累。

应用构建

Palantir 平台中的主要应用构建工具是 Workshop 和 Slate。

除了这些内置工具外，还可以使用开发者工具链和 Ontology SDK (OSDK) 在 Palantir 平台之上创建自定义的定制应用。

Workshop

Workshop 是一款灵活的、面向对象的应用构建工具。Workshop 利用 Ontology 中的语义原语（例如对象、链接）和动态原语（例如操作、函数），实现了高交互性桌面和移动应用的快速交付。Workshop 中的应用构建体验赋能用户通过无代码、低代码和基于代码的组件来创建强大的应用。在构建过程中，用户无需具备专业技术背景即可上手使用组件，将对象、链接和操作编织成由用户驱动的工作流，其功能远超仪表板或静态可视化。同时，通过函数实现的基于代码的增强功能可以无缝嵌入到 Workshop 组件中，从而支持复杂的交互、级联流程和复杂的数据捕获。

了解更多关于 Workshop 的信息。

Slate

Slate 为构建者提供了一套灵活的工具，用于快速创建运营应用和交互式仪表板。Slate 使应用开发者能够通过拖拽界面构建动态且响应式的应用，从而缩短开发时间并降低成本。Slate 不仅具备与 Foundry Ontology 无缝集成的能力，还允许开发者使用 HTML、CSS 和 JavaScript 对应用进行完全定制。借助定制的 Slate 应用，组织各层级的相关人员都能快速探索和了解其数据，进而做出更明智的决策。

工作流构建与管理

Palantir 平台中主要的工作流构建和管理工具包括工作流血缘关系、Automate、Solution Designer 和用例。

工作流血缘关系

工作流血缘关系提供了一个交互式工作区，用于理解和管理应用及其底层流程。借助工作流血缘关系，您可以探索工作流，并查看有关对象、操作、函数、大语言模型和应用的详细信息。对于负责创建、调试或维护工作流的应用构建者来说，工作流血缘关系尤为实用。在更改或扩展工作流时，来源图谱、更深层次的属性与 Workshop 组件/变量来源，以及升级工具都能提供极大的帮助。

Automate

Automate 为您设置和执行平台中的所有业务自动化提供了统一入口。Automate 应用允许用户定义条件和效果；系统会持续检查条件，并在满足指定条件时自动执行相应效果。

了解更多关于 Automate 的信息。

Carbon

Carbon 支持为特定用户群体配置定制化的平台体验，即所谓的工作区。对于技术背景较弱但需执行关键操作工作流的用户，Carbon 能提供专注度更高的使用体验。每个 Carbon 工作区都是应用程序和资源的精选集合，可对其进行配置，以优化特定的操作和终端用户工作流。例如，一个航空零部件维修工作区可能包含一个“ Workshop ”应用，其中列出了需维修零部件的动态更新清单，并提供基于本体（Ontology）的动作（Actions）对每个零部件进行分流；另一个应用则用于调查每个零部件的维修问题；此外还配有 Quiver 分析图表，展示随时间推移的维修趋势。Carbon 能够将 Foundry 丰富的应用矩阵和分析能力整合到聚焦的操作体验中。

了解更多关于 Carbon 的信息。

解决方案设计器

解决方案设计器（Solution Designer） 是一款交互式工具，用于创建基于 Palantir 平台构建的解决方案架构视图，其中包括对第一方和第三方集成点的呈现、平台资源链接、按需获取文档与最佳实践等功能。

了解更多关于解决方案设计器的信息。

用例

“用例（Use Cases）”应用允许构建者在单一的操作界面中统筹其工作。通过将文件系统视图与本体管理视图相结合，开发者可获取一个专属视图，该视图完全聚焦于其职责范围内的工作。

开发者工具链

Palantir 开发者工具链 支持您使用自有工具在 Palantir 平台之上构建专属应用。

Palantir 开发者工具链的核心是 本体 SDK (OSDK)。通过 Palantir 平台内的开发者控制台（Developer Console），您可以生成针对特定本体的 SDK。该本体 SDK 既可打包为适用于 TypeScript 的 NPM（Node 包管理器）包，也可通过 Pip 或 Conda 用于 Python，其中仅包含预先选定的本体子集。借助该 SDK，您可以访问对象类型、执行动作以更新本体中的数据、调用函数，并为 已启用 AIP 的环境运行 AIP 逻辑（AIP Logic）函数。开发者控制台还提供针对您应用所选实体的特定本体文档。应用程序将使用 OAuth 流程，作为公共或机密客户端来访问数据。
