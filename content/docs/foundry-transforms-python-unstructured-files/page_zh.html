读写非结构化文件

警告

非结构化文件访问属于高级主题。在阅读本页内容前，请确保您已熟悉本用户指南中的其余内容。

在数据转换过程中，您可能出于多种原因需要访问文件。如果您希望处理非表格格式（如 XML 或 JSON）或压缩格式（如 gz 或 zip）的文件，文件访问将尤为实用。

transforms Python 库允许用户在 Foundry 数据集中读写文件。transforms.api.TransformInput 会暴露一个只读的 FileSystem 对象，而 transforms.api.TransformOutput 则会暴露一个只写的 FileSystem 对象。这些 FileSystem 对象支持基于文件在 Foundry 数据集内的路径进行文件访问，同时对底层存储进行了抽象封装。

如果希望在数据转换中访问文件，您必须使用 transform() 装饰器构建 Transform 对象。这是因为 FileSystem 对象由 TransformInput 和 TransformOutput 对象暴露。transform() 是唯一一个要求其计算函数的输入和输出类型分别为 TransformInput 和 TransformOutput 的装饰器。

导入文件

可以通过手动文件导入或通过《数据连接》同步的方式将文件上传至 Foundry。结构化和非结构化文件均可导入至 Foundry 数据集，以便在下游应用中处理。文件也可以作为原始文件上传，无需修改扩展名。以下示例均指作为 Foundry 数据集上传的文件，而非原始文件。

Foundry 还具备为上传至数据集的特定文件类型自动推断模式的功能。例如，在导入 CSV 类型文件时，可以使用 应用模式 按钮自动应用模式。有关《手动上传数据》的更多信息，请参阅相关文档。

你可以通过手动操作多个进程来并行处理文件。

1from transforms.api import transform, Input, Output

5@transform.using(

7    hair_eye_color=Input("/Mint/Transforms/lightweight/abort/csvs"),
8)
9def example_computation(hair_eye_color, processed):
10    fs = hair_eye_color.filesystem()
11    # Step 1: List all CSV files
12    csv_files = list(fs.ls(glob="*.csv"))
13
14    # Step 2: Function to process a single file
15    def process_file(dataset_file):
16        path = dataset_file.path
17        with fs.open(path) as csv_file:
18            # Read CSV as Polars DataFrame
19            df = pl.read_csv(csv_file)
20            # Optionally, select or rename columns to match schema
21            return df.select(["student_id", "hair_color", "eye_color"])
22
23    # Step 3: Process files in parallel
24    with ThreadPoolExecutor() as executor:
25        dfs = list(executor.map(process_file, csv_files))
26
27    # Step 4: Concatenate all DataFrames
28    result_df = pl.concat(dfs)
29    processed.write_table(result_df)

## 使用 PySpark 进行并行化处理 与通过 `DataFrame` ↗ 对象表达的数据转换，或使用标准非 Spark 转换定义的数据处理不同，在 PySpark 中处理基于文件的转换时，必须厘清驱动器代码与执行器代码之间的差异。计算函数在驱动器上运行，而驱动器是一台单机。Spark 会根据自身判断，将 `DataFrame` ↗ 中的函数自动分发到执行器（多台机器）上。 若想从文件 API 的分布式处理中获益，我们必须借助 Spark 来分发计算任务。具体做法是：创建一个由 `FileStatus` 组成的 `DataFrame` ↗，并将其分发到各个执行器上。随后，执行器上的每个任务都会打开分配给它的文件并进行处理，最终由 Spark 汇总处理结果。 文件 API 提供了 `files()` 函数，该函数接收的参数与 `ls()` 函数相同，但返回的是由 `FileStatus` 对象组成的 `DataFrame` ↗。此 DataFrame 会根据文件大小进行分区，以便在文件大小不一的情况下均衡计算负载。可以通过以下两个 Spark 配置项来控制分区行为： - spark.sql.files.maxPartitionBytes ↗：读取文件时，打包到单个分区中的最大字节数。 - spark.sql.files.openCostInBytes ↗：打开文件的估算成本（以相同时间内可扫描的字节数来衡量）。该值会与文件大小相加，用于计算该文件在分区中占用的总字节数。 要修改这些属性的值，必须创建自定义的 Transforms 配置文件，并通过 `configure()` 装饰器将其应用到 Transform 上。有关详细信息，请参阅 Code Repositories 文档中关于定义 Transforms 配置文件的章节。 下面我们来看一个示例。假设我们需要解析并拼接多个 CSV 文件。我们可以使用 `flatMap()` ↗ 对每个 `FileStatus` 对象应用处理函数。该处理函数必须按照 `pyspark.sql.SparkSession.createDataFrame()` ↗ 的要求生成数据行。

13            r = csv.reader(f)
14            # 从表头行构建一个 pyspark.Row
15            header = next(r)
16            MyRow = Row(*header)
17            for row in r:
18                yield MyRow(*row)
19
20    schema = StructType([
21        StructField('student_id', StringType(), True),
22        StructField('hair_color', StringType(), True),
23        StructField('eye_color', StringType(), True),
24    ])
25
26    files_df = hair_eye_color.filesystem().files('**/*.csv')
27    processed_df = files_df.rdd.flatMap(process_file).toDF(schema)
28    processed.write_dataframe(processed_df)
警告

虽然可以在不传入 schema 的情况下调用 toDF()，但如果文件处理后的返回结果为零行，Spark 的 schema 推断将会失败，并抛出 ValueError: RDD is empty 异常。因此，建议始终手动指定 schema。
