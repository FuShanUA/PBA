读取和写入非结构化文件

警告

非结构化文件访问是一个高级主题。在阅读本页之前，请确保您熟悉本用户指南中的其余内容。

您可能出于各种原因想要访问数据转换中的文件。如果要处理非表格格式（如 XML 或 JSON）或压缩格式（如 gz 或 zip）的文件，文件访问特别有用。

transforms Python 库允许用户在 Foundry 数据集内读取和写入文件。transforms.api.TransformInput 暴露了一个只读的 FileSystem 对象，而 transforms.api.TransformOutput 暴露了一个只写的 FileSystem 对象。这些 FileSystem 对象允许根据 Foundry 数据集内文件的路径进行文件访问，从而抽象了底层存储。

如果要访问数据转换中的文件，您必须使用 Transform 装饰器来构造 transform() 对象。这是因为 FileSystem 对象由 TransformInput 和 TransformOutput 对象公开。transform() 是唯一一个期望其计算函数的输入和输出分别为 TransformInput 和 TransformOutput 类型的装饰器。

导入文件

可以使用手动文件导入或通过 数据连接 将文件上传到 Foundry。结构化和非结构化的文件可以导入到 Foundry 数据集，以便在下游应用程序中进行处理。也可以将文件作为原始文件上传而不修改扩展名。以下示例指的是作为 Foundry 数据集上传的文件，而不是作为原始文件上传的文件。

Foundry 还具有自动推断某些上传到数据集的文件类型的模式的功能。例如，当导入 CSV 类型的文件时，会提供 Apply a schema 按钮以自动应用模式。了解更多关于 手动上传数据。

您可以通过在多个进程中手动操作来并行处理文件。

1from transforms.api import transform, Input, Output
2import polars as pl
3from concurrent.futures import ThreadPoolExecutor

5@transform.using(
6    processed=Output("/Mint/Transforms/lightweight/abort/out"),
7    hair_eye_color=Input("/Mint/Transforms/lightweight/abort/csvs"),
8)
9def example_computation(hair_eye_color, processed):
10    fs = hair_eye_color.filesystem()
11    # Step 1: List all CSV files
12    csv_files = list(fs.ls(glob="*.csv"))
13
14    # Step 2: Function to process a single file
15    def process_file(dataset_file):
16        path = dataset_file.path
17        with fs.open(path) as csv_file:
18            # Read CSV as Polars DataFrame
19            df = pl.read_csv(csv_file)
20            # Optionally, select or rename columns to match schema
21            return df.select(["student_id", "hair_color", "eye_color"])
22
23    # Step 3: Process files in parallel
24    with ThreadPoolExecutor() as executor:
25        dfs = list(executor.map(process_file, csv_files))
26
27    # Step 4: Concatenate all DataFrames
28    result_df = pl.concat(dfs)
29    processed.write_table(result_df)

使用 PySpark 的并行处理

与以 DataFrame ↗ 对象表示的数据转换，或使用标准非 Spark 转换定义的转换不同，在 PySpark 中理解驱动程序代码和执行器代码之间的区别非常重要。计算函数是在驱动程序上执行的，它是一个单独的机器。Spark 会自动分发 DataFrame ↗ 将函数发送到执行器（许多机器）以按需进行。

为了利用文件API进行分布式处理，我们必须利用Spark来分发计算。为此，我们创建一个 DataFrame ↗ 的 FileStatus 并在我们的执行器之间分发。然后，每个执行器上的任务可以打开它被分配的文件，并用结果由Spark进行聚合。

文件API公开了 files() 函数，该函数接受与 ls() 函数相同的参数，但返回的是 DataFrame ↗ 的 FileStatus 对象。此DataFrame按文件大小进行分区，以便在文件大小不同时帮助平衡计算。可以使用两个Spark配置选项来控制分区：

  • spark.sql.files.maxPartitionBytes ↗ 是读取文件时打包到单个分区中的最大字节数。

  • spark.sql.files.openCostInBytes ↗ 是指打开文件的预估成本，以在相同时间内可以扫描的字节数来衡量。这将被添加到文件大小中，以计算文件在分区中使用的总字节数。

要修改这些属性的值，您必须创建一个自定义的转换（Transform）配置文件，并使用configure()装饰器将其应用到您的转换（Transform）中。有关更多信息，请参阅代码仓库文档中关于定义转换（Transform）配置文件的部分。

现在，让我们逐步演示一个示例。假设我们有CSV文件需要解析和连接。我们利用flatMap() ↗来对每个FileStatus对象应用处理函数。此处理函数必须根据pyspark.sql.SparkSession.createDataFrame() ↗生成行。

18                yield MyRow(*row)

21        StructField('student_id', StringType(), True),
