安全术语表

以下是你需要了解的所有安全术语。

  • 访问权限：指示用户是否能够知晓某项资源的存在。如果用户拥有访问权限，便可通过角色获得使用该资源的各种能力。如果用户没有访问权限，则无法知晓该资源的存在。 参见：资源、角色
  • 访问要求：决定哪些用户可以访问项目或资源的独特安全要求。这些要求由组织和/或标记组成。如果用户满足某个项目或资源的访问要求，则允许该用户查看该项目或资源的存在，并可被授予相应的角色。 参见：组织、标记、角色
  • 属性：有关用户的结构化信息类型，例如其姓名、电子邮件或职位。
  • 凭证收集器：负责收集用户凭证的系统，随后这些凭证将由身份验证源进行验证。主要的收集器类型为 SAML。
  • 默认角色（平台上）：Foundry 平台自带四种角色：数据主人、编辑者、查看者和发现者。角色管理员可以选择自定义这些角色，或基于这些默认角色创建全新的角色。角色定义还可以指定哪些角色能在同一资源上授予其他角色，从而构建起角色层级体系。 参见：角色
  • 默认角色（项目上）：项目上的默认角色定义了自动授予所有满足访问要求的用户的角色。如果项目没有默认角色，用户将只能看到项目的名称和描述并申请项目访问权限，但无法发现项目内容。为了鼓励协作与发现，除非另有指定，项目创建时会使用“查看者”作为默认角色。 参见：角色、项目
  • 自主控制：在用户访问权限的基础上扩展其整体能力，并通过角色进行授予。自主控制是叠加式的，意味着自主控制只能为用户增加权限，而不能限制用户的权限。可以通过数据共享工作流向用户授予自主控制权：例如，创建一份报告并与同事共享，便会授予她对该报告的查看权限。
  • 应用组织：允许用户将组织作为访问要求添加到资源上的权限。拥有此权限的用户可以将组织应用到资源、文件夹或项目上。 参见：组织、访问要求；另见 组织权限
  • 扩大访问范围：扩大访问范围是指对资源访问要求的任何更改，使得潜在可以访问该资源的受众群体得以扩大。对于标记而言，移除任何标记都属于扩大访问范围事件。对于组织而言，扩大访问范围可以通过以下两种方式实现：（1）在已应用至少一个组织的情况下添加额外的组织，或者（2）移除已应用的唯一组织。 参见：访问要求、组织、标记；另见 组织权限
  • 用户组：一组用户和/或其他用户组的集合。用户组可以是内部的（即在 Foundry 中定义），也可以是外部的（即由外部身份提供者（如 Active Directory）或用户管理器定义）。内部用户组可以包含外部用户组和用户。 参见：身份提供者、用户管理器
  • 身份提供者：一种系统，它赋予应用程序在用户或服务登录时对其进行验证的能力，并提供有关用户、用户组和属性的信息。身份提供者是用户、用户组信息和属性的来源。身份提供者使应用程序能够在用户或服务登录时进行验证，并理解关于这些用户的信息。 参见：属性、用户、用户组
  • 继承/继承的权限：在项目级别应用的访问要求会被项目内的其他文件和文件夹继承。同样，在项目级别授予的角色也会通过继承机制授予给项目中的所有资源。 参见：角色
  • 强制控制：一种全有或全无的访问限制。在实施强制控制的情况下，无论用户的角色如何，除非用户满足资源的强制控制要求，否则无法以任何方式访问该资源。这些控制以组织和标记的形式体现。例如，如果同事共享了一份报告，而支持该报告的数据集被标记为机密标记，那么除非用户拥有机密级别数据的权限，否则不会被授予对该报告的访问权。 参见：角色、组织、标记
  • 标记： 一种应用于资源、以全有或全无方式限制访问的访问要求。为满足访问要求，用户必须是该资源上所应用所有标记的成员。标记是一种强制控制措施。 参见：访问要求、资源、强制控制措施
  • 标记类别： 标记的集合。标记类别的可见性可限制于特定组织，并且可以处于可见或隐藏状态。对于所需组织的成员或访客成员而言，可见类别是可被发现的。隐藏类别仅对所需组织中、被明确授予该类别或其标记权限的用户可见且可被发现。 参见：标记、组织
  • 空间（Space，原称 Namespace）： 项目的高阶分组，可在其中定义统一的设置，例如文件系统或使用量追踪账户。 参见：项目
  • 组织： 一种应用于项目的访问要求，用于在用户组与资源之间实施严格隔离。每位用户仅隶属于一个组织。为满足访问要求，用户必须是应用于某项目的其中一个组织的成员。组织是一种强制控制措施。 参见：访问要求、资源、强制控制措施
  • 组织可发现性： 一个组织的用户如何查看其他组织，反之亦然。跨组织的可发现性是可配置的；组织的用户、组和/或资源可以被设置为可发现或隐藏。 参见：组织
  • 权限： 授予用户或组的一组能力，允许他们在平台中或资源上执行特定任务。 参见：角色、标记、资源
  • 项目： 项目是一个协作空间，旨在为特定目的组织人员与资源。项目是 Foundry 中的主要安全边界，代表一系列共享工作。项目中的用户对其内容拥有基本一致的访问权限（具体访问可能因自主控制措施而异），这意味着访问要求和角色应在项目级别应用。 参见：资源、访问要求
  • 域： Foundry 中的身份验证源。可在“平台设置”中查看每个用户的域。 参见：用户
  • 引用： 指向资源的链接，该链接会导致资源被纳入项目的范围内。在构建数据管道时，引用通常用于包含当前项目之外的数据集。 参见：项目
  • 资源： Foundry 平台内任何可唯一标识的事物，例如项目、文件夹、分析和数据集。资源通过访问要求和权限进行安全保护。 参见：项目、角色
  • 受限视图： 一种特殊类型的数据集，其中对文件内数据的细粒度访问由已定义的规则控制。这些规则基于用户属性，将根据用户的访问级别隐藏或显示数据集的行。 参见：属性、用户
  • 角色： 权限的集合，定义了用户可对给定资源执行的特定工作流。角色是一种自主控制措施，通常在项目级别授予，以对项目范围内的所有资源提供统一的能力。 参见：工作流、资源、自主控制措施
  • 标签： 可应用于资源的结构化元数据，用于分类和发现。标签被组织成多个类别，其可见性可限制为一个或多个组织。标签可作为一种有用的结构，但标签不会以任何方式增加或暗示安全性。
  • 用户： 经过身份验证且有权访问 Foundry 的个人。用户由外部身份提供者（例如 Active Directory 系统）定义。 参见：域
  • 用户管理器： 一个可选的 Foundry 模块，用于向登录流程添加自定义逻辑，通常用于在登录期间分配用户组和属性。例如，用户管理器可以允许特定用户登录，同时禁止所有其他用户登录。最简单的用户管理器是同步的，这意味着 Foundry 会在登录期间同步（同时）连接到该服务，无需用户交互。异步用户管理器 (AUM) 将重定向至某台服务器，该服务器可展示页面并支持用户交互（例如确认最终用户许可协议），随后才允许继续登录。 参见：用户
  • 工作流： 执行特定用户动作所需且应通过单个角色一并授予的权限集合。 参见：角色
