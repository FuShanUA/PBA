在 FoundryTS 中处理时间序列

FoundryTS 是一个 Python 库，用于对时间序列数据执行查询，例如直接根据时间序列或根据附加的本体属性进行过滤。FoundryTS 库可与《代码仓库》和《代码工作簿》集成。

查看《FoundryTS API 参考》以获取更多详细信息。

请注意，在转换中使用 FoundryTS 库的输出结果将是一个数据集。如果要输出新的时间序列，应当创建一个具备时间序列所需模式的数据集，然后使用时间序列数据目录或管道构建器将该数据集作为新时间序列的底层数据。

在代码仓库中开始使用

首先，请确保您已经《设置了 Python 代码仓库》。

使用库面板将 foundryts、transforms-timeseries 和 transforms-objects 库添加到您的仓库中。

然后，按照以下说明将正在查询的对象类型导入到仓库中，并更新项目引用：

  1. 转到仓库的设置选项卡。
  2. 选择本体。
  3. 添加您的对象类型。

导入资源

项目引用授权 FoundryTS 访问项目外部的资源。本节将引导您完成在资源位于项目外部时所需导入的资源。

如果通过时间序列属性访问序列，必须导入以下资源：

  • 支持该对象类型的数据集

如果通过序列 ID 或搜索查询访问序列，必须导入以下资源：

  • 时间序列同步（此资源的 RID 格式类似于：ri.time-series-catalog.main.sync.）
  • 支持该时间序列同步的数据集

在代码工作簿中开始使用

在代码工作簿中，通过从右上角工具栏选择环境，然后选择配置环境，将 foundryts 包添加到您的环境中。

在Conda 环境下，选择自定义配置文件，搜索并添加 foundryts。选择更新环境以保存更改。

详细了解代码工作簿中的《环境配置》。

设置工作簿输入

任何被查询的对象类型（通过时间序列属性访问）或时间序列数据目录同步（通过序列 ID 或搜索查询访问）都必须从左侧的内容面板添加为工作簿输入。

添加为工作簿输入的对象类型或时间序列数据目录同步还必须与工作簿导入到同一个项目中，包括其底层数据集。否则，在执行使用 foundryts 编写的转换时将会引发错误。如果有任何工作簿输入不在项目范围内，它们将会显示在工作簿工具栏右上角设置下拉菜单中的项目范围设置对话框内。
