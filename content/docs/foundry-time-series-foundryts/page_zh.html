在FoundryTS中处理时间序列

FoundryTS是一个Python库，用于对时间序列数据进行查询，例如直接过滤时间序列或从附加的本体论属性中过滤。FoundryTS库与 代码仓库（Code Repositories） 和 代码工作区（Code Workspaces） 集成。

有关更多详细信息，请查看 FoundryTS API参考（FoundryTS API reference）。

请注意，使用FoundryTS库进行转换时的输出将是一个数据集（dataset）。如果您想要输出一个新的时间序列（time series），您应该创建一个具有时间序列所需模式的数据集（dataset），然后使用该数据集通过时间序列目录（Time Series Catalog）或管道构建器（Pipeline Builder）来支持新的时间序列。

在代码仓库中开始

首先，确保您已经 设置了一个 Python 代码仓库。

使用库窗格将 foundryts、transforms-timeseries 和 transforms-objects 库添加到您的仓库中。

然后，按照以下说明将正在查询的对象类型导入仓库并更新项目引用：

  1. 转到仓库中的设置选项卡。
  2. 选择本体论。
  3. 添加您的对象类型。

导入资源

项目引用授予 FoundryTS 访问项目外资源的权限。本节将指导您导入那些位于项目外部的资源。

如果您通过时间序列属性访问您的系列，必须导入以下资源：

  • 对象类型的数据集

如果您通过系列ID或搜索查询访问您的系列，您必须导入以下资源：

  • 时间序列同步（此资源的RID格式如下：ri.time-series-catalog.main.sync.）
  • 支撑时间序列同步的数据集

在代码工作区中开始

在代码工作区中，通过从右上角工具栏选择环境，然后选择配置环境来将foundryts包添加到您的环境中。

在Conda 环境下，选择自定义配置文件并搜索和添加foundryts。选择更新环境以保存您的更改。

了解更多关于< a href="/docs/foundry/code-workbook/environment-overview/">代码工作区中的环境配置。

设置工作簿输入

任何查询的对象类型（通过时间序列属性访问）或时间序列目录同步（通过系列ID或搜索查询访问）必须从左侧的内容面板添加为工作簿输入。

作为工作簿输入添加的对象类型或时间序列目录同步也必须导入到与工作簿相同的项目中，包括它们的支撑数据集。否则在使用foundryts编写转换时会导致错误。如果任何工作簿输入不在项目范围内，它们将在工作簿工具栏右上角的设置下拉菜单中的项目范围设置对话框中显示出来。

示例：股票数据

在这个示例中，我们从 Stock series 对象类型（对象）的 Ticker name 属性（属性）开始。我们的目标是查找 Technology 行业（行业）中的所有系列，并计算它们的时间范围。

首先定义转换的输入和输出。我们声明 Stock series 对象类型（对象）作为对象输入，时间序列同步作为时间序列输入。

2    fts = FoundryTS()

接下来我们在 timeseries-demo-stock-series 行业（行业）中搜索 Technology 对象（对象）。对于每个搜索结果，我们将系列映射到其时间范围（最早和最晚点的时间戳）。

Copied!

2        (ontology('sector') == 'Technology'),
3        object_types=['timeseries-demo-stock-series']
4    ).map(F.time_extent())
