Kinesis 连接器会将消息内容解析为 unicode 字符串。请使用下游流式转换（例如，《管道构建器》中的 parse_json）来解析结构化数据。

  • partition_key 列将包含用于将消息发布到 Kinesis 的分区键。
  • kinesis_ingestion_timestamp 列将包含消息发布到 Kinesis 时的时间戳。
  • foundry_ingestion_timestamp 列将包含 Foundry 接收到该消息时的时间戳。

性能与限制

对于源 Kinesis 数据流上的每个活跃分片，该连接器始终使用单个消费者线程。

流式同步属于一致性长驻任务。根据预期目标，任何对流式同步过程的中断都可能引发服务中断。

目前，流式同步存在以下限制：

  • Foundry worker 上的任务每 48 小时至少重启一次。预期停机时间为个位数分钟（假设资源充足，允许任务立即重启）。
  • 对于传统的 agent worker 数据源，任务会在 agent 维护窗口期（通常为每周一次）重启以进行升级。预期停机时间不到五分钟。为实现高可用性，建议通过两个维护窗口互不重叠的 agent 进行连接。

消息顺序

Kinesis 连接器确保具有相同 partition_key 的消息按顺序投递。不同 partition_key 值的消息可能以任意顺序处理。

设置

  1. 打开《数据连接》应用程序，并在屏幕右上角选择 + 新建源。
  2. 从可用的连接器类型中选择 Kinesis。
  3. 按照后续的配置提示，结合下文各节的信息，继续完成连接器的设置。

AWS 实例

旧版

AWS 实例身份验证仅适用于旧版智能体工作节点源。Foundry 工作节点源无法使用 AWS 实例身份验证；请改用 IAM 访问密钥。

当您的 Foundry 智能体运行在配置了 IAM 角色的 AWS 资源（例如 EC2 实例）上时，Kinesis 连接器将使用该配置的 IAM 角色连接到 Kinesis 数据流。无需额外配置。

静态凭证

静态凭证是指使用绑定到 IAM 用户的访问密钥 ID 和秘密访问密钥的标准 AWS 身份验证方式。

参数是否必填？默认值
访问密钥 ID是无
秘密访问密钥是无

STS 角色

Kinesis 连接器可以选择在连接到 Kinesis 数据流之前扮演一个 STS 角色。有关这些参数的详细信息，请参阅《AWS 文档》↗。

参数是否必填？默认值
角色 ARN是无
角色会话名称是无
角色会话时长是900
外部 ID否无

网络

连接器必须有权访问 AWS Kinesis API，如果使用 STS 角色，则还需可选地访问 AWS STS API。

  • Kinesis API： https://kinesis.<region>.amazonaws.com
  • STS API： https://sts.<region>.amazonaws.com

将数据导出至 Kinesis

该连接器支持在数据连接中导出至外部 Kinesis 数据流。

要导出至 Kinesis，请首先为您的 Kinesis 连接器启用导出功能。然后，创建新导出。
