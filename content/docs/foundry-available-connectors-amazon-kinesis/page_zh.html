Amazon Kinesis

将 Foundry 连接到 Amazon Kinesis，以实时从 Kinesis 流读取数据到 Foundry 流中。

数据模型

partition_key（字符串）data（字符串）kinesis_ingestion_timestamp（时间戳）foundry_ingestion_timestamp（时间戳）
London {"firstName": "John", "lastName": "Doe"}2023-07-12T15:12:42.371Z2023-07-12T15:12:42.512Z
Paris{"firstName": "Jean", "lastName": "DuPont"}2023-07-12T15:12:42.418Z2023-07-12T15:12:42.512Z

Kinesis 连接器将消息内容解析为 Unicode 字符串。使用下游流转换（例如，在 Pipeline Builder 中的 parse_json）来解析结构化数据。

  • partition_key 列将包含用于将消息发布到 Kinesis 的分区键。
  • kinesis_ingestion_timestamp 列将包含消息发布到 Kinesis 时的时间戳。
  • foundry_ingestion_timestamp 列将包含消息被 Foundry 摄取时的时间戳。

性能和限制

连接器对源 Kinesis 流上的每个活跃分片始终使用单个消费者线程。

流同步旨在保持一致且长期运行的任务。任何流同步的中断都可能造成停机，具体取决于预期的结果。

目前，流式同步有以下限制：

  • 在 Foundry worker 上的任务每 48 小时至少重启一次。预计停机时间是单位数分钟（假设资源可用性允许任务立即重启）。
  • 对于旧版 agent worker 来源，任务在 agent 维护窗口期间重启（通常每周一次）以获取升级。预计停机时间小于五分钟。为了高可用性，请通过两个维护窗口不重叠的 agent 连接。

消息顺序

Kinesis 连接器保证具有相同 partition_key 的消息的传递顺序。具有不同 partition_key 值的消息可能以任何顺序处理。

设置

  1. 打开 数据连接 应用程序，并在屏幕右上角选择 + 新增源。
  2. 选择可用连接器类型中的Kinesis。
  3. 按照后续的配置提示继续完成使用以下部分信息的连接器设置。

了解更多关于在Foundry中< a href="/docs/foundry/data-connection/set-up-source/">设置连接器的信息。

AWS 实例

旧版

AWS 实例认证仅适用于旧版 代理工作者 源。Foundry 工作者源不能使用 AWS 实例认证；请改用 IAM 访问密钥。

当您的 Foundry 代理运行在具有预置 IAM 角色（例如 EC2 实例）的 AWS 资源上时，Kinesis 连接器将使用预置的 IAM 角色连接到 Kinesis 流。无需额外配置。

静态凭证

静态凭证是指与 IAM 用户相关联的标准 AWS 身份验证，包括访问密钥 ID 和秘密访问密钥。

参数必需？默认值
访问密钥 ID是否
秘密访问密钥是否

STS 角色

Kinesis 连接器可以在连接到 Kinesis 流之前选择性地假定一个 STS 角色。有关这些参数的详细信息，请参阅 AWS 文档 ↗。

参数必需？默认值

网络

连接器必须能够访问AWS Kinesis API，并且如果使用STS角色，还可以选择性地访问AWS STS API。

  • Kinesis API： https://kinesis.<region>.amazonaws.com
  • STS API： https://sts.<region>.amazonaws.com

将数据导出到Kinesis

该连接器支持在Data Connection中将数据导出到外部Kinesis流。

要导出到Kinesis，请先 启用导出 您的Kinesis连接器。然后，创建新的导出。
