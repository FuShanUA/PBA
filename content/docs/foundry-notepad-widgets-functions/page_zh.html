函数

此微件仅适用于文档模板。

对象函数微件允许你使用 TypeScript 函数计算值，并在基于模板生成文档时将其插入。

你可以将该函数与在生成时传递的输入参数连接。对象函数部分也可以嵌套在诸如分区生成器或表格行生成器等生成器部分中；这使你能够为对象集中的每个对象分别计算值。我们建议在生成器内部配置批处理函数。

批处理函数

批处理函数是接受对象集作为参数并返回该对象集中每个对象结果的函数。用户可以为嵌套在生成器部分内的对象函数部分配置批处理函数。

使用批处理函数可显著减少基于模板生成文档所需的时间。原因在于执行函数的开销：非批处理函数必须为对象集中的每个对象执行一次，而批处理函数整个对象集只需执行一次。

使用批处理函数配置对象函数部分时，作为输入的对象集可以是来自生成器的对象集。来自生成器的对象集指由外围生成器部分生成的对象集。请注意，批处理函数可以有多个输入。如果批处理函数包含在嵌套生成器中，作为输入的对象集最多只能由一个生成器生成：可以是嵌套生成器或顶层生成器，但不能同时来自两者。如果批处理函数有多个对象集函数输入，所有配置为来自生成器的对象集的对象集都应来自同一个外围生成器部分。

除了对象集参数外，批处理函数可能还有其他参数。

经验丰富的 Foundry 用户可能熟悉函数支持属性。批处理函数与函数支持属性类似。

编写批处理函数

批处理函数接受对象集作为参数，并返回一个 FunctionsMap，将对象集中的每个对象映射到对应的结果。有关更多详细信息，请参阅可接受的函数输入和输出部分。

例如，考虑一个非批处理函数，它接收一个 Flight Alert 对象并根据其延误时间计算警报的紧急程度：

1@Function()
2public getAlertUrgency(flightAlert: FlightAlertWorkshopTutorial): string {
3    const hoursDelayed = flightAlert.timeOfDelayHours
4    if (hoursDelayed! > 4) {
5        return "High";
6    }
7    else if (hoursDelayed! > 2) {
8        return "Medium";
9    }
10    else {
11        return "Low";
12    }
13}

该函数的批处理版本接收一个包含多个 FlightAlertWorkshopTutorial 对象的集合，并返回一个 FunctionsMap<FlightAlertWorkshopTutorial, string>。返回的 FunctionsMap 包含了 FlightAlertWorkshopTutorial 对象与其紧急程度之间的映射关系。



已复制！
1@Function()
2public flightAlertCalculateUrgency(flightAlerts: ObjectSet<FlightAlertWorkshopTutorial>):
3    FunctionsMap<FlightAlertWorkshopTutorial, string> {
4    const map = new FunctionsMap<FlightAlertWorkshopTutorial, string>();
5    flightAlerts.all().forEach(flightAlert => {
6        const hoursDelayed = flightAlert.timeOfDelayHours
7        if (hoursDelayed! > 4) {
8            map.set(flightAlert, "High")
9        }
10        else if (hoursDelayed! > 2) {
11            map.set(flightAlert, "Medium")
12        }
13        else {
14            map.set(flightAlert, "Low")
15        }
16    });
17    return map;
18}

批处理函数生成器对象集限制

在模板生成期间，区块生成器与表格行生成器的对象集设有数量上限。此阶段执行批处理函数时，会使用中间对象集，这些中间对象集是通过对提供的输入对象集进行大小限制后得到的。

函数接受的输入与输出

非批处理函数必需的输出类型

非批处理函数必须返回以下类型之一，方可在 Notepad 模板中使用：

  • Boolean
  • Date
  • Double
  • Float
  • Integer
  • Long
  • String

批处理函数必需的输入与输出

有效的批处理函数必须至少包含一个类型为 ObjectSet<K> 的参数，且返回值类型必须为 FunctionsMap<K, V>。类型 V 必须为以下列出的允许类型之一：

  • Boolean
  • Date
  • Double
  • Float
  • Integer
  • Long
  • String

将集合（sets）或列表（lists）作为参数的函数，目前不符合有效批处理函数的要求。

微件属性

  • 批处理/非批处理： 允许用户选择配置批处理还是非批处理函数。仅当函数微件嵌套在生成器区块内时才会显示。
  • 函数： 要运行的函数。若上方选择了批处理选项，则仅会提供批处理函数。
  • 函数版本： 要运行的函数版本。
  • 函数输入： 传入函数的值。可用参数由所选函数决定。
  • 函数结果： 函数执行后生成的输出。结果可采用纯文本或Markdown等多种格式进行解析。目前仅非批处理函数支持 Markdown 解析。

模板配置

  • 函数输入： 依据所选函数，可通过参数将不同的预期输入值模板化。

嵌入的对象函数区块也可以通过这种方式连接到生成器区块。
