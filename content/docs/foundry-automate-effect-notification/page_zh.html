通知效果

Automate 应用支持在满足特定条件时，自动向平台内的其他用户发送通知。

通知发送方式有两种：直接在平台内发送，或通过电子邮件发送。通知内容支持静态定义，也可通过函数动态生成。此外，通知还能携带 PDF 附件。本页涵盖了通知收件人、内容及附件的相关配置选项。

对象分组

若使用涉及受影响对象的条件，可选择对象分组方式，以决定在发送通知前如何对这些对象进行分组。

涉及受影响对象的条件包括：

  • 添加到对象集的对象
  • 从对象集中移除的对象
  • 对象集中修改的对象

针对所有对象执行一次

当多个对象同时触发条件时，每位收件人仅会收到一条通知。

以下方截图示例为例，三个不同的 Support Ticket 对象触发了自动化流程，仅发送一条通知。基于函数的通知可访问包含全部三个对象的对象集。

针对每组对象执行一次

当多个对象同时触发条件时，对象将根据所选属性集进行分组。每个分组将向每位收件人发送一条通知。

注意，分组基于属性值的精确匹配。对于数组类型属性，各值必须完全一致且顺序相同才能分在同一组。

以下方示例为例，三个不同的 Support Ticket 对象同时触发自动化流程。其中两个对象属于计费类别，另一个属于技术支持类别。

  • 由于选择了类别作为分组属性，因此将产生两条通知：一条面向计费组，另一条面向技术支持组。
  • 对于第一条通知，基于函数的通知可访问包含两个计费 Support Ticket 对象的对象集。
  • 对于第二条通知，基于函数的通知可访问包含技术支持对象的对象集。

针对每个对象执行一次

当多个对象同时触发条件时，每个对象均会触发一条独立通知。

例如，若三个 Support ticket 对象触发了自动化流程，则为每个 Support ticket 分别发送一条通知。基于函数的通知可访问单个对象及其属性值。

收件人

配置通知时，首先需定义通知的接收方。收件人可以是 Foundry 用户或用户组，支持通过静态收件人列表或通过对象属性动态进行定义。

若需具备接收自动化通知的资格，收件人必须拥有以下权限：

  • 对该自动化流程至少拥有查看者权限。静态与动态收件人均需满足此要求。
  • 对触发自动化流程的对象实例拥有查看者权限。
  • 若触发对象类型为多数据源对象类型，则需对该对象实例的所有属性拥有查看者权限。
  • 对基于函数的通知中函数执行所访问的所有对象实例拥有查看者权限。

上述要求适用于已激活用户与预注册用户。

手动执行自动化流程会绕过触发条件。因此，在手动运行期间，不会检查触发对象上的权限。

静态收件人列表

定义通知接收者的第一种方式是在 静态 选项下将其作为静态列表提供。如需定义该列表，请点击相应的文本字段，并选择希望作为接收者的用户或群组。

通过对象属性动态定义接收者

第二种方式是通过受影响对象的对象属性，动态定义通知接收者。此配置选项需要一个能够公开效应输入的对象集条件。

动态接收者定义允许您指定包含用户 ID 或群组 ID 的对象属性，从而在运行时确定通知接收者。因此，对象属性类型必须为 String 或 Array of String。

内容

定义通知内容的方式有两种：作为纯文本通知或函数生成通知。

系统会为每位接收者单独渲染通知。因此，最终生成的内容可能因用户而异。

纯文本通知

纯文本通知提供界面组件，便于您直接指定通知内容。您必须提供 标题 和 消息。此外，您还可以选择修改 URL 链接。默认情况下，URL 链接是指向平台内通知的链接，该链接仅在电子邮件中显示。

您还可以使用 高级电子邮件配置 为电子邮件设置不同的值。默认情况下，系统将使用您上方为平台内通知提供的 标题、消息 和 URL 链接 值。如有需要，还可以在高级电子邮件配置中使用 HTML。

添加内容时，您可以使用 / 键打开一个菜单，将通知视为模板，以便在发送通知时替换接收者姓名等值。如果正在监控对象集且选择了 按对象执行，关联对象属性的当前值和先前值也可作为模板值。请注意，只有在启用实时监控时，才能获取先前属性值。

您所做的所有更改都将在配置组件右侧进行实时预览。

最后，您可以为通知定义附件。

函数生成通知

作为使用纯文本通知的替代方案，您可以通过函数支持通知，由函数动态生成通知内容。仅当纯文本通知无法满足需求时，才应使用此方式。

如需创建自定义通知函数，请按照《函数文档》中的说明进行操作。该函数必须返回 Notification 或 Notification | undefined。当返回 undefined 时，Automate 将跳过该通知。您可以在函数逻辑中利用这一点，根据条件决定是否发送通知。

编写并发布通知函数后，您即可在通知效应中使用它。首先选择您的函数及版本。随后，界面将更新并根据函数定义公开所需的输入项。

使用条件效应输入和接收者

根据输入参数类型，您或许能够使用特殊的效应输入，而不仅仅是提供静态值。支持的效应输入包括：

  • 接收者： 可用于 User 类型的函数输入，将为其渲染通知的接收者传递给该函数。
  • 条件效应输入： 当条件公开效应输入时，您可以在此处使用这些效应输入。函数参数的类型必须与公开的效应输入类型保持一致。条件公开的类型取决于条件类型和执行模式。

附件

你可以配置通知操作，将来自 Notepad 文档或 Notepad 模板文档的 PDF 附件包含在通知邮件中。运行时，系统会为每位接收者自动生成 PDF 文件。

在附件区域选择 + 选择，并选取一个 Notepad 文档或 Notepad 模板。随后，指定附件的文件名。根据所选的资源类型，你可能需要提供更多值，具体说明见下文。

Notepad 模板

执行通知操作时，可以使用 Notepad 模板动态生成 Notepad 文档并将其导出为 PDF。这一过程通过 Notepad 模板输入完成。在生成步骤中，你可以通过 Notepad 模板输入向 Notepad 文档传递值。有关 Notepad 模板的更多功能，请参阅《模板文档》。

选择 Notepad 模板和模板版本后，系统会显示所需的模板输入。Automate 应用支持向 Notepad 模板输入传递静态值和条件操作输入。如果模板公开了对象或对象集模板输入，且条件公开了相同类型的条件操作输入，你就可以将该条件操作输入传递到模板中。

Notepad 模板不会公开对象或对象集模板输入的对象类型信息。你必须自行确保所提供对象值的类型相匹配。
