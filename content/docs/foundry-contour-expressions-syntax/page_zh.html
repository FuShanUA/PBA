Syntax and supported functions

For an orientation to the Expression board, see the guide on using the expression board. This document is a resource for using Contour's rich expression language and can be used as a reference for types, operations and functions.

Data types

You can use the following data types in expressions:

  • String
  • Integer
  • Double
  • Boolean
  • Date

String

Strings are expressed in single quotes:

'This is a string'

This is a string

Escape single quotes within the string by adding a second single quote:

'I''m a string'

I’m a string

Concatenate columns as strings with the || operator:

"Gender" || "MaritalStatus"

You can also concatenate columns with constants.

'$' || ("salaryColumn"/1000.0) || 'k'

Integer

Integers are non-floating point numbers:

5

Double

Doubles are floating point numbers:

5.7

Boolean

Boolean variables can be either true or false. Boolean variables are case-insensitive.

Date

You can cast strings in the format YYYY-MM or YYYY-MM-DD as dates:

CAST('2016-12-06' AS DATE)

You can also cast strings as timestamps:

CAST('2016-12-06` `01:12:34' AS TIMESTAMP)

When performing operations with timestamps, it is easiest to first cast the timestamp to a long, or to a double if you need millisecond precision. This gives you the number of seconds in UNIX time.

For example, say you have two timestamp columns: “start” and “end.” To determine the number of minutes between the two times, you can use the following expression:

(CAST("end" as LONG) - CAST("start" as LONG)) / 60.

Casting longs to timestamps assumes that that long values are in seconds. If your data is in milliseconds, divide by 1000 prior to casting.

Arithmetic

Add, subtract, multiply, and divide numeric columns (integers or doubles) by other numeric columns or by constants: +, -, *, /

  • "tipAmount" / "fare"
  • "diameter" * 3.14

Comparison

Compare columns of any type against each other and against constants with > (greater than), >= (greater than or equal to), < (less than), <= (less than or equal to). Results in a column of booleans.

  • "age" > "averageAge"
  • "totalDistance" < 5

Equality comparison

Check whether a row has the same value in two columns with ==, or different values with !=.

  • "cityOfBirth" == "cityOfResidence"

You can also compare column values with constants.

  • "Gender" == 'M'
String comparisons

When comparing two strings, lexicographical ordering is used. Note that this can be counterintuitive when comparing strings that contain numbers. For example:

  • 'Alligator' < 'Boat' => TRUE because the encoding of 'A' is less than that of 'B'
  • 'Alliance' < 'Alligator' => TRUE because 'a' is less than 'g'
  • 'Zoo' < 'alpha' => TRUE，因为 'Z' 小于 'a'
  • 'Zoo' < 'Zoologist' => TRUE，因为如果一个字符串是另一个的前缀，则较短的字符串被认为较小
  • '1' < '2' => TRUE，因为字符串字符 '1' 小于字符串字符 '2'
  • '10' < '2' => TRUE，因为其第一个字符串字符 '1' 小于字符串字符 '2'；请注意，这是对字符串 '10' 和 '2' 的比较，而不是对整数 10 和 2 的比较
  • '10' < '20' => TRUE，因为 '1' 小于 '2'
与 Spark SQL 的行为差异

在 Contour 表达式语言中，与 NULL 的相等比较具有以下行为。

  • NULL = 'DATA' => FALSE
  • NULL != 'DATA' => TRUE
  • NULL = NULL => TRUE
  • NULL != NULL => FALSE

然而，在 Spark SQL 中，所有这些表达式都会返回 NULL。在 Contour 表达式和 Spark SQL 之间进行转换时，必须考虑这种行为差异，才能产生一致的结果。

布尔比较

通过布尔逻辑对一个或多个列进行求值，派生出一个新的布尔列。

  • "Age" >=70 AND "Gender" = 'M'

类型转换

类型转换允许您更改列或表达式的类型。您可以转换为布尔值、整数、双精度浮点数、日期和时间戳。

  • CAST("startDate" AS DATE)
  • CAST("startTime" AS TIMESTAMP)
  • CAST("numParticipants" AS INTEGER)

Case/When/End

Case 语句允许您在一个表达式中评估多种可能性。每个语句按顺序进行评估，并在第一个评估结果为 true 的语句中执行 THEN 语句。

已复制！
1CASE
2    WHEN "age" < 13 THEN 'Child'
3    WHEN "age" > 19 THEN 'Adult'
4    ELSE 'Teenager'
5END


已复制！
1CASE MaritalStatus
2    WHEN 'S' THEN 'Single'
3    WHEN 'M' THEN 'Married'
4    ELSE 'Unknown'
5END

LIKE

在列中搜索指定模式。请参阅《SQL LIKE ↗》文档以了解更多信息。

空值检查

检查值是否为 IS NULL 或 IS NOT NULL。返回布尔值。

优先级

表达式中运算的优先级如下所示，从最高到最低排列。同一行的运算具有相同的优先级。

  1. +, -, NOT (一元)
  2. || (二元)
  3. *, /, % (二元)
  4. +, - (二元)
  5. <, <=, >, >= (二元)
  6. =, ==, !=, <>, IS, IS NOT, LIKE, RLIKE (二元)
  7. IS NULL, NOT NULL (一元)
  8. IN, NOT IN
  9. AND (二元)
  10. OR (二元)

数学与数值函数

  • ABS：计算绝对值。
  • CBRT：计算给定值的立方根。
  • CEIL：计算给定值的上限值。
  • COS：计算给定值的余弦值。
  • EXP：计算给定值的指数值。
  • FACTORIAL：计算给定值的阶乘。
  • FLOOR：计算给定值的下限值。
  • FORMAT_NUMBER：将数值列 arg0 的格式设为 ‘#,###,###.##’，并四舍五入到 arg1 位小数，然后返回字符串列形式的结果（注：arg1 必须是整数常量）。
  • ISNAN：当且仅当该列为 NaN 时返回 true。
  • LN：计算给定值的自然对数。
  • LOG：计算以 arg0 为底数时 arg1 的对数（注：arg0 必须是数值常量）。
  • POW：返回 arg0 的 arg1 次方的值。
  • RAND：根据提供的种子 arg0，为每一行返回一个 0 到 1 之间的随机数。
  • ROUND：返回将 arg0 四舍五入到 arg1 位小数后的值（注：arg1 必须是整数常量）。
  • RTRIM：去除指定字符串值末尾的空格。
  • SIN：计算给定列的正弦值。
  • SQRT：计算指定浮点值的平方根。
  • TAN：计算给定值的正切值。

字符串函数

  • CONCAT：将多个输入字符串列拼接成一个单一的字符串列。
  • CONCAT_WS：使用给定的分隔符将多个输入字符串列拼接成一个单一的字符串列（注意：arg0 必须是字符串字面量）。
  • FORMAT_STRING：根据《Java Formatter ↗》格式化参数，并将结果作为字符串列返回。
  • LENGTH：计算给定字符串或二进制列的长度。
  • LEVENSHTEIN：计算两个给定字符串列的莱文斯坦距离。
  • LOWER：将字符串列转换为小写。
  • LPAD：返回字符串列 arg0，使用字符串 arg2 在左侧填充至长度 arg1（注意：arg1 必须是整数字面量，arg2 必须是字符串字面量）。
  • LTRIM：去除指定字符串值左端的空格。
  • REGEXP_EXTRACT：将字符串列 arg0 与正则表达式 arg1 进行匹配，并提取特定的分组 arg2（从 1 开始索引）。如果正则表达式未匹配，或指定分组未匹配，则返回空字符串（注意：arg1 必须是字符串字面量，arg2 必须是整数字面量）。
  • REGEXP_REPLACE：将 arg0 中所有匹配 arg1 的子字符串替换为 arg2。
  • REVERSE：反转字符串列，并将其作为新的字符串列返回。
  • RPAD：返回字符串列 arg0，使用字符串 arg2 在右侧填充至长度 arg1（注意：arg1 必须是整数字面量，arg2 必须是字符串字面量）。
  • SPLIT：根据正则表达式字符串 arg1 拆分字符串列 arg0（注意：arg1 必须是字符串字面量）。
  • SUBSTRING：返回 arg0 的子字符串，起始位置为索引 arg1（从 1 开始索引），长度为 arg2（注意：arg1 和 arg2 必须是整数字面量）。
  • TRIM：去除指定字符串列两端的空格。
  • UPPER：将字符串列转换为大写。

日期/时间函数

有关推导相对日期的更多信息和示例，请参阅《关于推导相对日期的参考文档》。

  • ADD_MONTHS：返回 arg0 之后 arg1 个月的日期（注意：arg1 必须是整数字面量）。
  • CURRENT_DATE：以日期列的形式返回当前日期。这些值仅在路径重新计算时更新。为避免不同看板间的值不一致，建议每天更新路径。
  • CURRENT_TIMESTAMP：以时间戳列的形式返回当前时间戳。这些值仅在路径重新计算时更新。
  • CURRENT_UNIX_TIMESTAMP：以秒数的形式返回当前时间戳。这些值仅在路径重新计算时更新。
  • DATE_ADD：返回 arg0 之后 arg1 天的日期（注意：arg1 必须是整数字面量）。
  • DATE_FORMAT：将日期/时间戳 arg0 转换为 arg1 指定格式的字符串（注意：arg1 必须是字符串字面量）。
  • DATE_SUB：返回 arg0 之前 arg1 天的日期，并遵循《Java SimpleDateFormat ↗》（注意：arg1 必须是整数字面量）。
  • DATE_TRUNC：返回按格式 arg0 指定单位截断后的日期/时间戳 arg1。
  • DATE_DIFF：返回从 arg1 到 arg0 的天数。
  • DAY_OF_MONTH：从给定的日期/时间戳/字符串中提取月份中的天数作为整数。
  • DAY_OF_WEEK：从给定的日期/时间戳/字符串中提取星期中的天数作为整数。范围从 1（星期日）到 7（星期六）。
  • DAY_OF_YEAR：从给定的日期/时间戳/字符串中提取年份中的天数作为整数。
  • FROM_UTC_TIMESTAMP：给定一个对应于 UTC 中某一时刻的时间戳列 arg0，返回对应于给定时区 arg1 中同一时刻的另一个时间戳（注意：arg1 必须是字符串字面量）。
  • HOUR：从给定的日期/时间戳/字符串中提取小时数作为整数。
  • LAST_DAY：给定一个日期列，返回该日期所属月份的最后一天。
  • MINUTE：从给定的日期/时间戳/字符串中提取分钟数作为整数。
  • MONTH：从给定的日期/时间戳/字符串中提取月份作为整数。
  • MONTHS_BETWEEN：返回日期 arg0 和 arg1 之间的月份数。
  • QUARTER：从给定的日期/时间戳/字符串中提取季度作为整数。
  • SECOND：从给定的日期/时间戳/字符串中提取秒数作为整数。
  • TO_UNIX_TIMESTAMP：使用给定的模式字符串 arg1 转换时间字符串 arg0，失败则返回 null（注意：arg1 必须是字符串字面量）。
  • TO_UTC_TIMESTAMP：给定一个时间戳列 arg0（对应于给定时区 arg1 中的某个时刻），返回与 UTC 时间中同一时刻相对应的另一个时间戳（注意：arg1 必须为字符串字面量）。
  • WEEK_OF_YEAR：从给定的日期/时间戳/字符串中提取 ISO 周数 ↗，并以整数形式返回。
  • YEAR：从给定的日期/时间戳/字符串中提取年份，并以整数形式返回。

数组函数

有关数组函数的更多信息和示例，请参阅 数组函数参考文档。

  • ARRAY：创建包含所有输入列的数组。
  • ARRAY_CONTAINS：如果数组列 arg0 包含值 arg1，则返回 true（注意：arg1 必须为字面量）。
  • ARRAY_GET_AT_INDEX：从数组列 arg0 中返回索引 arg1（从 1 开始计数）处的元素（注意：arg1 必须为整数字面量）。需显式转换此函数的结果，方可在后续看板中使用该列。
  • ARRAY_LENGTH：返回给定数组的长度。输入为 null 时返回 -1。
  • ARRAY_JOIN：返回由分隔符字符串 arg1 连接数组列 arg0 后得到的字符串（注意：arg1 必须为字符串字面量）。
  • ARRAY_SORT：如果 arg1 为 true，则按升序对输入数组 arg0 进行排序；如果 arg1 为 false，则按降序排序（注意：arg1 必须为布尔字面量）。
警告

在窗口函数中使用 ROW_NUMBER、FIRST、LAST、ARRAY_AGG 或 ARRAY_AGG_DISTINCT 时，请注意非确定性。假设按列 A 分区并按列 B 排序。如果对于列 A 的同一个值，存在多行列 B 的值相同，那么这些窗口函数的结果可能是不确定的——在输入数据和逻辑相同的情况下，也可能产生不同的结果。

窗口函数

窗口函数需要使用 OVER 子句；有关更多信息和示例，请参阅 窗口函数参考文档。

  • CUME_DIST：返回窗口分区内的累积分布值，即排在当前行之下的行数比例。
  • DENSE_RANK：返回窗口分区内的行排名，且排名之间没有间隔。
  • EXPLODE：为给定数组列中的每个元素创建新行，并丢弃包含 null 值的行。需显式转换此函数的结果，方可在后续看板中使用该列。
  • EXPLODE_OUTER：为给定数组列中的每个元素创建新行。需显式转换此函数的结果，方可在后续看板中使用该列。
  • FIRST：返回组中的第一个值。
  • LAG：返回列 arg0 中当前行之前 arg1 行的值；如果当前行之前的行数少于 arg1 行，则返回 null。例如，arg1 为 1 时，将返回窗口分区中任意给定位置的前一行（注意：arg1 必须为整数字面量）。
  • LAST：返回组中的最后一个值。
  • LEAD：返回列 arg0 中当前行之后 arg1 行的值；如果当前行之后的行数少于 arg1 行，则返回 null。例如，arg1 为 1 时，将返回窗口分区中任意给定位置的后一行（注意：arg1 必须为整数字面量）。
  • NTILE：在已排序的窗口分区中返回 ntile 组的 ID（从 1 到 arg0，包含边界）。例如，如果 arg0 为 4，则前四分之一的行将获得值 1，第二个四分之一获得 2，第三个四分之一获得 3，最后一个四分之一获得 4（注意：arg0 必须为整数字面量）。
  • PERCENT_RANK：返回窗口分区内行的相对排名（即百分位数）。
  • RANK：返回窗口分区内行的排名。
  • ROW_NUMBER：返回窗口分区内从 1 开始的顺序号。

聚合函数

聚合函数可用于聚合表达式和窗口函数中。

  • ARRAY_AGG：返回由输入列的值聚合而成的数组。
  • ARRAY_AGG_DISTINCT：返回由输入列的去重后值聚合而成的数组。
  • AVG：返回组内各值的平均值。
  • COUNT：返回组中的项数。
  • COUNT_DISTINCT：返回组中的项目数量（注意：只能在表达式面板的 聚合 选项中使用。不能与 添加新列、筛选 或 替换列 一起使用）。
  • CORR：返回两列的皮尔逊相关系数。
  • MAX：返回组中表达式的最大值。
  • MEAN：返回组中值的平均值。
  • MIN：返回组中表达式的最小值。
  • STDDEV：返回组中表达式的样本标准差。
  • SUM：返回表达式中所有值的总和。
  • SUM_DISTINCT：返回表达式中不同值的总和（注意：不能用于窗口函数）。
  • VARIANCE：返回组中值的无偏方差。

其他函数

  • COALESCE：返回第一个非空的列；如果所有输入均为空，则返回空值。
  • GREATEST：返回值列表中的最大值，跳过空值。
  • HASH：计算给定列的哈希码，并将结果作为整型列返回。
  • ISNULL：当且仅当列为空时返回 true。
  • LEAST：返回值列表中的最小值，跳过空值。
  • MD5：计算二进制列的 MD5 摘要，并以 32 字符的十六进制字符串形式返回该值。
  • MONOTONICALLY_INCREASING_ID：返回单调递增的 ID。这些值在每次计算路径时可能会发生变化。
  • SHA1：计算二进制列的 SHA-1 摘要，并以 40 字符的十六进制字符串形式返回该值。
  • GET_JSON_OBJECT：根据指定的 JSON 路径 ↗ 从 JSON 字符串中提取 JSON 对象，并返回所提取 JSON 对象的 JSON 字符串。以下是一些示例：
    • $.field
    • $['field']
