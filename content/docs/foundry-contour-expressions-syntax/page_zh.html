语法和受支持的函数

有关表达式板的说明，请参阅 使用表达式板指南。本文档是用于使用Contour丰富的表达式语言的资源，可以作为类型、操作和函数的参考。

数据类型

您可以在表达式中使用以下数据类型：

  • String（字符串）
  • Integer（整数）
  • Double（双精度浮点数）
  • Boolean（布尔值）
  • Date（日期）

String（字符串）

字符串用单引号表示：

'This is a string'

这是一个字符串

在字符串中转义单引号，方法是添加第二个单引号：

'I''m a string'

我是一个字符串

使用 ___CODE_BLOCK_4 ___ 运算符将列连接为字符串：

"Gender" || "MaritalStatus"

您也可以使用常量连接列。

'$' || ("salaryColumn"/1000.0) || 'k'

整数

整数是不带小数点的数字：

5

双精度浮点数

双精度浮点数是带有小数点的数字：

5.7

布尔值

布尔变量可以是 ___CODE_BLOCK_9 ___ 或 ___CODE_BLOCK_10 ___。布尔变量对大小写不敏感。

日期

您可以将格式为 YYYY-MM 或 YYYY-MM-DD 的字符串转换为日期：

CAST('2016-12-06' AS DATE)

您也可以将字符串转换为时间戳：

CAST('2016-12-06` `01:12:34' AS TIMESTAMP)

在进行时间戳操作时，最简单的方法是首先将时间戳转换为长整型（long），或者如果需要毫秒精度的话转换为双精度浮点型（double）。这会给出 UNIX 时间的秒数。

例如，假设您有两个时间戳列：“start” 和 “end”。要确定两个时间之间的分钟数，可以使用以下表达式：

(CAST("end" as LONG) - CAST("start" as LONG)) / 60。

将长整型转换为时间戳时，假定这些长整型值是以秒为单位的。如果您的数据是以毫秒为单位的，在转换之前请先除以 1000。

算术运算

通过其他数值列或常量对数值列（整数或双精度浮点数）进行加、减、乘、除运算：+, -, *, /

  • "tipAmount" / "fare"
  • "diameter" * 3.14

比较

通过 >（大于）、>=（大于等于）、<（小于）、<=（小于等于）与其他类型列或常量进行比较。结果生成一个布尔值的列。

  • "age" > "averageAge"
  • "totalDistance" < 5

等值比较

使用 == 检查某一行在两个列中是否具有相同的值，或使用 != 检查是否具有不同的值。

  • "cityOfBirth" == "cityOfResidence"

您也可以将列值与常量进行比较。

  • "Gender" == 'M'
字符串比较

在比较两个字符串时，使用字典顺序。请注意，当比较包含数字的字符串时，这可能会产生反直觉的结果。例如：

  • 'Alligator' < 'Boat' => TRUE，因为 'A' 的编码小于 'B' 的编码
  • 'Alliance' < 'Alligator' => TRUE，因为 'a' 小于 'g'
  • 'Zoo' < 'alpha' => TRUE 因为 'Z' 小于 'a'
  • 'Zoo' < 'Zoologist' => TRUE 因为如果一个字符串是另一个字符串的前缀，较短的字符串被认为更小
  • '1' < '2' => TRUE 因为字符串字符 '1' 小于字符串字符 '2'
  • '10' < '2' => TRUE 因为第一个字符串字符 '1' 小于字符串字符 '2'; 请注意，这是对字符串 '10' 和 '2' 的比较，而不是整数 10 和 2
  • '10' < '20' => TRUE 因为 '1' 小于 '2'
与 Spark SQL 的行为差异

在 Contour 表达式语言中，与 NULL 的相等比较具有以下行为。

  • NULL = 'DATA' => FALSE
  • NULL != 'DATA' => TRUE
  • NULL = NULL => TRUE
  • NULL != NULL => FALSE

然而，在 Spark SQL 中，所有这些表达式都返回 NULL。在将 Contour 表达式和 Spark SQL 进行转换时，考虑到这种行为差异以产生一致的结果非常重要。

布尔比较

通过使用布尔逻辑评估一个或多个列来生成新的布尔列。

  • "Age" >=70 AND "Gender" = 'M'

转换

转换允许您更改列或表达式的类型。您可以将列或表达式转换为布尔值、整数、双精度浮点数、日期和时间戳。

  • CAST("startDate" AS DATE)
  • CAST("startTime" AS TIMESTAMP)
  • CAST("numParticipants" AS INTEGER)

条件/当/结束

条件语句让您在一个表达式中评估多个可能性。每个语句按顺序进行评估，并且第一个评估为真的 THEN 语句被执行。

Copied!
1CASE
2    WHEN "age" < 13 THEN 'Child'
3    WHEN "age" > 19 THEN 'Adult'
4    ELSE 'Teenager'
5END
Copied!
1CASE MaritalStatus
2    WHEN 'S' THEN 'Single'
3    WHEN 'M' THEN 'Married'
4    ELSE 'Unknown'
5END

匹配

在某一列中搜索指定的模式。查看 SQL LIKE ↗ 的文档以了解更多信息。

空值检查

检查一个值 IS NULL 或 IS NOT NULL 。返回一个布尔值。

优先级

表达式中操作的优先级在下列表中从高到低详细列出。在同一行的操作具有相同的优先级。

  1. +, -, NOT（一元）
  2. ||（二元）
  3. *, /, %（二元）
  4. +, -（二元）
  5. <, <=, >, >=（二元）
  6. =, ==, !=, <>, IS, IS NOT, LIKE, RLIKE（二元）
  7. IS NULL, NOT NULL（一元）
  8. IN, NOT IN
  9. AND（二元）
  10. OR（二元）

数学 & 数值函数

  • ABS：计算绝对值。
  • CBRT：计算给定值的立方根。
  • CEIL：计算给定值的向上取整。
  • COS：计算给定值的余弦。
  • EXP：计算给定值的指数。
  • FACTORIAL：计算给定值的阶乘。
  • FLOOR：计算给定值的下取整。
  • FORMAT_NUMBER：将数值列 arg0 格式化为类似 '#,###,###.##' 的格式，四舍五入到 arg1 位小数，并将结果作为字符串列返回（注意：arg1 必须是整数字面量）。
  • ISNAN：如果列是 NaN，则返回 true。
  • LN：计算给定值的自然对数。
  • LOG：计算以 arg0 为底的 arg1 的对数（注意：arg0 必须是数值字面量）。
  • POW：返回 arg0 的 arg1 次幂。
  • RAND：根据提供的种子 arg0，为每一行返回一个 0 到 1 之间的随机数。
  • ROUND：返回 arg0 四舍五入到 arg1 位小数后的值（注意：arg1 必须是整数字面量）。
  • RTRIM：从指定字符串值的右侧删除空格。
  • SIN：计算给定列的正弦值。
  • SQRT：计算指定浮点值的平方根。
  • TAN：计算给定值的正切值。

字符串函数

  • CONCAT：将多个输入字符串列合并为一个字符串列。
  • CONCAT_WS：将多个输入字符串列合并为一个字符串列，使用给定的分隔符（注意：arg0必须是一个字符串字面量）。
  • FORMAT_STRING：根据Java Formatter ↗格式化参数，并将结果作为字符串列返回。
  • LENGTH：计算给定字符串或二进制列的长度。
  • LEVENSHTEIN：计算两个给定字符串列的Levenshtein距离。
  • LOWER：将字符串列转换为小写。
  • LPAD：返回字符串列arg0，用字符串arg2在左边填充到指定长度arg1（注意：arg1必须是整数字面量，arg2必须是字符串字面量）。
  • LTRIM：从指定字符串值的左侧删除空格。
  • REGEXP_EXTRACT：将字符串列arg0与正则表达式arg1进行匹配，并提取特定的组arg2（从1开始计数）。如果正则表达式未匹配，或者指定的组未匹配，则返回空字符串。（注意：arg1必须是字符串字面量，arg2必须是整数字面量）。
  • REGEXP_REPLACE：将arg0中所有与arg1匹配的子字符串替换为arg2。
  • REVERSE：反转字符串列并将其作为新的字符串列返回。
  • RPAD：返回字符串列arg0，右对齐至长度arg1，使用字符串arg2（注意：arg1必须是整数字面量，arg2必须是字符串字面量）。
  • SPLIT：根据正则表达式字符串arg1分割字符串列arg0（注意：arg1必须是字符串字面量）。
  • SUBSTRING：从索引arg1（从1开始计数）开始，长度为arg2的arg0的子字符串（注意：arg1和arg2必须是整数字面量）。
  • TRIM：删除指定字符串列两端的空格。
  • UPPER：将字符串列转换为大写。

日期/时间函数

有关推导相对日期的更多信息和示例，请参阅 推导相对日期的参考文档。

  • ADD_MONTHS：返回arg0之后arg1个月的日期（注意：arg1必须是整数字面量）
  • CURRENT_DATE：以日期列的形式返回当前日期。这些值仅在路径重新计算时更新。为了避免跨看板的值不一致，建议每天更新路径。
  • CURRENT_TIMESTAMP：以时间戳列的形式返回当前时间戳。这些值仅在路径重新计算时更新。
  • CURRENT_UNIX_TIMESTAMP：以秒为单位的数字形式返回当前时间戳。这些值仅在路径重新计算时更新。
  • DATE_ADD：返回arg0之后arg1天的日期（注意：arg1必须是整数字面量）。
  • DATE_FORMAT：将日期/时间戳arg0转换为由arg1指定格式的字符串（注意：arg1必须是字符串字面量）。
  • DATE_SUB：返回arg1天前的日期，并根据 Java SimpleDateFormat ↗（注意：arg1必须是整数字面量）。
  • DATE_TRUNC：将 arg1 的日期/时间戳截断为由格式 arg0 指定的单位。
  • DATE_DIFF：返回从 arg1 到 arg0 的天数。
  • DAY_OF_MONTH：从给定的日期/时间戳/字符串中提取一个月中的某一天作为整数。
  • DAY_OF_WEEK：从给定的日期/时间戳/字符串中提取一周中的某一天作为整数。周日为 1，周六为 7。
  • DAY_OF_YEAR：从给定的日期/时间戳/字符串中提取一年中的某一天作为整数。
  • FROM_UTC_TIMESTAMP：给定一个时间戳列 arg0，它对应 UTC 时间中的某个时间点，返回另一个时间戳，该时间戳对应于给定时区 arg1 中的相同时间点。（注意：arg1 必须是字符串字面量）。
  • HOUR：从给定的日期/时间戳/字符串中提取小时作为整数。
  • LAST_DAY：给定一个日期列，返回该日期所属月份的最后一天。
  • MINUTE：从给定的日期/时间戳/字符串中提取分钟作为整数。
  • MONTH：从给定的日期/时间戳/字符串中提取月份作为整数。
  • MONTHS_BETWEEN：返回日期 arg0 和 arg1 之间的月份数量。
  • 季度：从给定的日期/时间戳/字符串中提取季度作为整数。
  • 秒：从给定的日期/时间戳/字符串中提取秒作为整数。
  • TO_UNIX_TIMESTAMP：使用给定的模式字符串 arg1 将时间字符串 arg0 转换为时间戳，如果失败则返回 null（注意：arg1 必须是字符串字面量）。
  • TO_UTC_TIMESTAMP：给定一个时间戳列 arg0，它对应于给定时区 arg1 中的某一时刻，返回另一个时间戳，该时间戳对应于 UTC 中的同一时刻。（注意：arg1 必须是字符串字面量）。
  • YEAR_OF_WEEK：从给定的日期/时间戳/字符串中提取 ISO 周 ↗ 编号作为整数。
  • 年：从给定的日期/时间戳/字符串中提取年份作为整数。

数组函数

有关数组函数的更多信息和示例，请参阅 数组函数参考文档。

  • ARRAY：创建包含所有输入列的数组。
  • ARRAY_CONTAINS：如果数组列 arg0 包含值 arg1，则返回 true（注意：arg1 必须是字面量）。
  • ARRAY_GET_AT_INDEX：从数组列 arg0 中返回索引 arg1（1-索引）处的元素（注意：arg1 必须是整数字面量）。显式地将此函数的结果转换为可以在后续看板中使用的列。
  • ARRAY_LENGTH：返回给定数组的长度。当输入为 null 时，返回 -1。
  • ARRAY_JOIN：返回由分隔符字符串 arg1 连接数组列 arg0 的字符串（注意：arg1 必须是字符串字面量）。
  • ARRAY_SORT：如果 arg1 为 true，则按升序对输入数组 arg0 进行排序；如果 arg1 为 false，则按降序进行排序（注意：arg1 必须是布尔字面量）。
警告

当在窗口函数中使用 ROW_NUMBER、FIRST、LAST、ARRAY_AGG 或 ARRAY_AGG_DISTINCT 时，要注意非确定性。想象一下我们按列 A 进行分区，并按列 B 排序。如果列 A 的相同值下有多个行具有相同的列 B 值，这些窗口函数的结果可能是不确定的——在相同的输入数据和逻辑下，可能会产生不同的结果。

窗口函数

窗口函数需要一个 OVER 子句；更多详细信息和示例请参阅 关于窗口函数的参考文档。

  • CUME_DIST：返回窗口分区内的值的累积分布，即低于当前行的行数所占的比例。
  • DENSE_RANK：返回窗口分区内的行的排名，没有间隔。
  • EXPLODE：为给定数组列中的每个元素创建一行，删除包含空值的行。显式地将此函数的结果转换为使用该列的后续工作区。
  • EXPLODE_OUTER: Creates a new row for each element in the given array column. Explicitly cast the result of this function to use the column in subsequent boards.
  • FIRST: Returns the first value in a group.
  • LAG: Returns the value that is arg1 rows before the current row in column arg0, and null if there are less than arg1 rows before the current row. For example, an arg1 of one will return the previous row at any given point in the window partition (note: arg1 must be an integer literal).
  • LAST: Returns the last value in a group.
  • LEAD: Returns the value that is arg1 rows after the current row in column arg0, and null if there are less than arg1 rows after the current row. For example, an arg1 of one will return the next row at any given point in the window partition (note: arg1 must be an integer literal).
  • NTILE: Returns the ntile group id (from 1 to arg0 inclusive) in an ordered window partition. For example, if arg0 is 4, the first quarter of the rows will get value 1, the second quarter will get 2, the third quarter will get 3, and the last quarter will get 4 (note: arg0 must be an integer literal).
  • PERCENT_RANK: Returns the relative rank (i.e. percentile) of rows within a window partition.
  • RANK: Returns the rank of rows within a window partition.
  • ROW_NUMBER: Returns a sequential number starting at 1 within a window partition.

Aggregate functions

Aggregate functions can be used in aggregation expressions and window functions.

  • ARRAY_AGG: Returns an array of aggregated values from the input column.
  • ARRAY_AGG_DISTINCT: Returns an array of distinct aggregated values from the input column.
  • AVG: Returns the average of the values in a group.
  • COUNT: Returns the number of items in a group.
  • COUNT_DISTINCT: Returns the number of items in a group (note: can only be used with the Aggregate option in the expression board. Cannot be used with Add new column, Filter or Replace Column).
  • CORR: Returns the Pearson Correlation Coefficient for two columns.
  • MAX: Returns the maximum value of the expression in a group.
  • MEAN: Returns the average of the values in a group.
  • MIN: Returns the minimum value of the expression in a group.
  • STDDEV: Returns the sample standard deviation of the expression in a group.
  • SUM: Returns the sum of all values in the expression.
  • SUM_DISTINCT: Returns the sum of distinct values in the expression (note: cannot be used in window functions).
  • VARIANCE: Returns the unbiased variance of the values in a group.

Misc functions

  • COALESCE: Returns the first column that is not null, or null if all inputs are null.
  • GREATEST: Returns the greatest value of the list of values, skipping null values.
  • HASH: Calculates the hash code of given columns, and returns the result as an int column.
  • ISNULL: Return true if, and only if, the column is null.
  • LEAST: Returns the least value of the list of values, skipping null values.
  • MD5: Calculates the MD5 digest of a binary column and returns the value as a 32 character hex string.
  • MONOTONICALLY_INCREASING_ID: Returns a monotonically increasing ID. These values may change each time the path is calculated.
  • SHA1: Calculates the SHA-1 digest of a binary column and returns the value as a 40 character hex string.
  • GET_JSON_OBJECT: Extracts a JSON object from the JSON string based on the JSON path ↗ specified, and returns a JSON string of the extracted JSON object. Some examples of this are:
    • $.field
    • $['field']
