Code Workspaces

Code Workspaces 将 JupyterLab®、RStudio® Workbench 和 VS Code 第三方集成开发环境（IDE）引入了 Palantir Foundry，让用户能够基于 Foundry 本体论的高质量数据，使用自己偏好的工具来提升生产力，并加速数据科学与统计工作流。Code Workspaces 容器与 Foundry 生态系统的其余部分原生集成，将熟悉的 IDE 与 Foundry 平台的优势结合起来，例如数据安全、分支管理、构建调度和资源管理。

Code Workspaces 为平台管理员提供了一种易于部署、完全托管、安全且可用于生产环境的方式，能够向用户提供 JupyterLab®、RStudio® Workbench 和 VS Code，同时内置了 Foundry 的数据治理以及对 FedRAMP、GxP 等标准的合规性。借助 Code Workspaces，用户可以在 Foundry 的访问控制和数据权限管理下，安全地连接现有的内部系统，并基于数据构建分析、转换、模型、应用程序或完整的工作流。

主要特性

Code Workspaces 的主要特性包括：

  • 安全： Code Workspaces 构建于支撑整个平台的 Foundry 安全 核心组件之上，具备完善的权限机制和细粒度的访问控制。这为 Code Workspaces 中可用的第三方 IDE 提供了 Foundry 的安全模型。例如，在 Foundry 中限制对某个数据集的访问，也会限制在 Code Workspaces IDE 中对该数据集的访问，从而确保跨工具的权限一致性。
  • 自定义环境： Code Workspaces 允许用户定义自定义环境配置文件，并根据需要增加或减少其工作空间的计算资源。
  • Git 工作流支持： Code Workspaces 以 Code Repositories 基础设施为底层支撑，提供分支、合并和提交历史等业界标准的版本控制功能。这些功能使多名用户能够在同一工作空间中更轻松、更安全地协同工作。
  • 应用程序： Code Workspaces 目前支持用于 Python 应用程序的 Dash ↗ 和 Streamlit ↗，以及用于 R 应用程序的 Shiny® ↗。用户可以直接在 Code Workspaces 中创建应用程序工作流，并使用 Foundry 内置的版本控制、分支管理和数据治理功能。
  • 模型集成： 用户可以在 Code Workspace 内部创建模型资产，并使用建模目标 跟踪这些资产。同一工作空间可以创建多个模型。
  • 转换/构建集成： Code Workspaces 可作为转换的开发环境。在 Code Workspaces 中编写的逻辑可以发布为数据转换管道，并与 Foundry 的数据集成 工具包无缝集成，涵盖构建、调度、数据血缘关系和健康检查。Code Workspaces 同时支持 R 转换和 Python/Jupyter® 转换。

何时使用 Code Workspaces

Foundry 提供了多种应用程序，可用于分析或编码目的。例如，如果您是一名分析师，使用 Contour 可能最为合适，这是 Foundry 提供的用于数据集分析的点击式低代码界面。

如果您需要编写大规模数据管道、设置数据连接或处理流数据，其他 Foundry 工具比 Code Workspaces 具备更强的功能；对于这些用例，我们建议分别使用管道构建器、数据连接 和 Foundry Streaming。

具体而言，Code Workspaces 在单节点上运行，而其他 Foundry 应用程序则利用 Spark 基础设施。因此，我们建议执行大规模数据转换的用户选择 管道构建器 或 Code Repositories，而不是 Code Workspaces。

Code Workspaces 专为构建机器学习模型或熟悉在 JupyterLab® 或 RStudio® Workbench 中工作的用户而设计。
