首先，选择您想使用的模板。接着，只需选定要在模板中使用的对象类型及相关属性。以下图片展示了创建地图应用的过程。

选定模块的保存位置并点击 创建模块 后，您将获得一个包含所输入数据且开箱即用的模板。

Workshop 设计中心

Workshop 设计中心是一款《市场》产品，汇聚了各类示例应用与模板，旨在帮助您使用低代码应用构建工具 Foundry Workshop 打造出更出色的应用。

设计中心包含六个高质量的 Workshop 应用示例，可为您构建自身工作流提供灵感。

设计中心模块旨在提供设计示例与灵感，而非即用型应用模板。这些示例模块自带模拟数据，提供端到端功能应用，供应用构建者进行逆向工程与分析。我们希望您能将设计中心模块中的概念与模式，应用到您新构建的应用或现有管理的应用中。

支持的 Workshop 应用

设计中心提供以下 Workshop 应用，各应用的独特功能与特性如下所示：

  • 告警收件箱：使用 Workshop 设计兼具美观与行动导向的收件箱。了解如何打造兼具吸引力与视觉美感、且功能高效实用的收件箱。
  • 引导式创建表单：构建无缝衔接的创建表单，引导用户走完设定的工作流。通过拆解为易于管理的步骤并掌握引导式表单的核心要素，降低用户操作失误与认知负担。
  • 指标看板：创建动态的高管概览与下钻指标看板以监控绩效表现。探索如何设计极具视觉信息量的看板，并有效追踪关键指标。
  • 通用作战图：在地图上直观呈现带有地理背景的指标。了解地图如何成为实现高效筛选的有效手段，并探索以地理方式展示数据的优势。
  • 租赁预订应用：浏览并筛选纽约市的租赁房源，预订下一次住宿。学习如何打造视觉体验丰富的探索型应用，并通过直观的筛选选项提升用户体验。
  • 插画应用：提供一组精简的 Workshop 插画示例，展示将视觉元素融入应用设计的潜力。

访问示例模块

要访问 Workshop 设计中心示例模块，请前往 Workshop 设计中心 项目。接着，将所需的 Workshop 模块复制到您指定的位置。

若要从您的 Foundry 环境中访问 Workshop 设计中心，请联系您的 Palantir 代表。

使用建议

默认情况下，Workshop 设计中心内的资产（本体对象、应用、数据集）均设为只读状态。我们强烈建议在针对特定用例进行修改前，先将 Workshop 模块复制到其他项目中。若您的管理员已将资产配置为读写模式，仍务必将设计中心内的本体要素（如对象类型或操作类型）及 Workshop 应用视作只读状态，因为定期的设计中心更新会覆盖对这些上述要素所做的本地更改。

收件箱

在 Workshop 中构建的一种常见应用类型是收件箱。通常，收件箱的核心组件包括：

  • 一组清晰的待审查告警或待办任务，以及在其中轻松进行筛选和优先级排序的能力。
  • 针对每条告警或任务的详细视图，支持深入审查。
  • 已配置的操作类型，支持对每条告警或任务做出决策。

下面的示例展示了一个 Workshop 收件箱，其中显示了有关航班告警的信息，这些告警需要进行分流、审查和操作。使用的核心组件包括重点术语筛选器、筛选列表、对象表和按钮组。此外，还启用了状态保存功能，允许用户保留最相关的收件箱设置，并将这些保存的视图与其他用户共享。

通用作战图

高层领导通常需要一种方式来实时了解其组织关键部分的运作情况。在 Workshop 中构建的通用作战图能够传达地理空间背景、详细图表和核心指标，这些都是迅速做出高层决策所必需的。

下面的示例展示了一个 Workshop 通用作战图，将新冠病例信息叠加在零售店销售和运营数据之上。目的是帮助店长了解门店绩效，并在本地病例数发生变化时更有效地调整计划。使用的核心组件包括指标卡、地图、XY 图表和对象表。
