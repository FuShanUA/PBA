Workshop 设计中心

Workshop 设计中心是一个 Marketplace 产品，它是一系列示例应用程序和模板的集合，旨在帮助您使用 Foundry Workshop（一种低代码应用程序构建工具）创建更优质的应用程序。

设计中心包含六个高质量的 Workshop 应用程序示例，您可以将其作为自己工作流的灵感来源。

设计中心的模块旨在作为设计示例和灵感，而不是现成的应用程序模板。这些示例模块包含示意性数据，并提供功能完整的端到端应用程序，可以被应用程序构建者进行逆向工程和分析。我们希望设计中心模块中的概念和模式能够应用于您构建的新应用程序或您管理的现有应用程序中。

支持的Workshop应用程序

以下Workshop应用程序可在Design Hub中使用，每个应用程序的独特功能和特性如下所述：

  • 警报收件箱（Alert Inbox）： 使用Workshop设计一个吸引人且具有操作性的收件箱。了解如何创建具有吸引力且视觉效果良好的收件箱，使其功能强大且高效。
  • 引导式创建表单（Guided Creation Form）： 构建一个无缝的创建表单，引导用户完成定义的工作流程。通过创建可管理的步骤并理解引导式表单的基本要素来减少用户错误和认知负担。
  • 指标仪表板（Metrics Dashboard）： 创建动态的高管概览和深入分析的指标仪表板以监控性能。了解如何设计视觉信息丰富的仪表板，并有效跟踪重要指标。
  • 共同操作图景（Common Operating Picture）： 在地图上可视化地理指标。学习如何利用地图作为有效的筛选工具，并探索以地理方式显示数据的好处。
  • 租赁预订应用： 浏览和筛选纽约市的租赁房源，预订您的下一次住宿。了解如何创建一个高度可视化的探索性应用程序，并通过直观的筛选选项来提升用户体验。
  • 插图应用： 一个小型的插图集合，在Workshop中展示将视觉元素融入您的应用程序设计中的潜力。

使用建议

默认情况下，Workshop 设计中心（本体论对象、应用、数据集）中的资产设置为只读，我们强烈建议在对特定用例进行修改之前，将任何 Workshop 模块复制到另一个项目。如果您的管理员已将资产配置为可读写，则仍需将设计中心内的本体论元素（如对象类型或操作类型）和 Workshop 应用程序视为只读，因为定期的设计中心更新会覆盖对这些前述元素所做的本地更改。

收件箱

在 Workshop 中构建的一种常见应用类型是收件箱。通常，收件箱的核心组件包括：

  • 一组清晰的警报或待完成的任务，并能够轻松地对其进行过滤和优先排序。
  • 每个单独警报或任务的详细视图，允许深入审查。
  • 配置好的操作类型，使您可以对每个警报或任务做出决策。

以下示例显示了一个Workshop收件箱，其中包含有关需要优先处理、审查和操作的航班警报的信息。核心小部件包括突出关键词过滤器、过滤列表、对象表格和按钮组。此外，启用了状态保存功能，以允许用户保存最符合他们需求的收件箱设置，并与其它用户共享这些已保存的视图。

常见操作图像

高级领导通常需要一种方式来了解其组织的关键部分在实时运行中的情况。常见操作图像在Workshop中构建，可以传达做出即时高层决策所需的地理空间上下文、详细图表和关键指标。

下面的示例展示了一个常见的操作图（Workshop）叠加了新冠疫情病例信息，使其在零售店销售和运营数据之上。其目的是帮助门店经理了解门店表现，并根据当地病例数量的变化更有效地调整他们的计划。核心使用的组件包括 指标卡片（Metric Card）、地图（Map）、图表：XY（Chart: XY） 和 对象表格（Object Table）。
