审核请求

作为 请求 的审核者，您可以决定请求中的哪些 任务 应该被批准或拒绝。在审核时，请考虑整个请求的影响，尽管您可以单独审核和批准每个任务。如果您需要更多关于为何某人需要其所请求访问权限的信息，请使用请求上的 评论 功能。

要开始审核，请在审批收件箱中选择 待审核的请求 过滤器。这将把可用的请求过滤为等待您审核的请求列表。

合格的审核者

默认情况下，能够执行相应操作的用户有资格审核相应的任务。用户可能有资格审核多个任务。

评论

您可以使用请求右侧的评论小部件对请求进行评论。默认情况下，将显示整个请求的评论，如All下拉菜单所示。

通过在评论小部件中的下拉菜单中进行筛选，可以对单个任务进行评论，或者选择任务右侧的评论图标。在下面的示例中，审核者将评论筛选为仅查看针对标记访问请求任务所做的评论。

备注小部件还允许您添加链接并上传可能需要支持请求的文件。例如，请求者可以分享Foundry或外部资源的链接，并上传审批所需的必要法律文件。

添加评审人

在大多数情况下，拥有批准任务权限的用户一旦请求创建就会自动被分配为审批人。如果您希望明确地将任务分配给一个未被自动分配的用户，请通过选择邀请评审人按钮来邀请他们。请注意，邀请评审人不会授予查看请求或批准任务的权限。被要求评审的用户将收到通知，并在其待评审请求部分看到该请求。

提供审核

一个 合格的审核人员 可以通过选择 + 审核 按钮来 批准 或 拒绝 该任务。

或者，使用 全部批准 或 全部拒绝 按钮在 操作 下拉菜单中，批准或拒绝您有资格审核的所有任务。
