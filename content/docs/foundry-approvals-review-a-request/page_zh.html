审核请求

作为 请求 的审核人，您可以决定请求中的哪些 任务 应该被批准或拒绝。在审核时，请考虑整个请求的影响，即使您可以单独审核和批准每个任务。如果您需要更多信息以了解某人为什么需要他们请求的访问权限，请使用请求中的 评论 功能。

要开始审核，请在审批收件箱中选择 待审核的请求 过滤器。这将筛选出所有待您审核的请求。

评论

您可以使用请求右侧的评论小部件对请求进行评论。默认情况下，评论将显示整个请求，如All（全部）下拉菜单所示。

通过在评论小部件中过滤下拉列表来对单个任务进行评论，或者选择任务右侧的评论图标。在下面的示例中，审核员将评论过滤为仅查看在标记访问请求任务上所做的评论。

添加审核人

在大多数情况下，有权批准任务的用户一旦请求创建就会自动被指定为审批人。如果您想明确地将任务分配给未被自动分配的用户，请通过选择邀请审核人 按钮。注意，邀请评审者不会授予查看请求或批准任务的权限。被要求进行评审的用户将收到通知，并在他们的“待审批请求”部分看到该请求。

提供评审

一个 合格的评审者 可以通过选择 + 评审 按钮来 批准 或 拒绝 该任务。

另外，可以使用 Actions 下拉菜单中的 全部批准 或 全部拒绝 按钮，来批准或拒绝您有资格评审的所有任务。
