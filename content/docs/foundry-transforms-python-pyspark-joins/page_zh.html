连接

在 PySpark 中，DataFrame 可以像 SQL 中的表一样与另一个 dataframe 或自身进行连接。dataframe 使用 .join() 方法与其他 dataframe 进行连接。它需要一个 DataFrame、一个连接约束（如要连接的列的名称）和一种方法（left、right、inner 等）

简单左连接

Copied!
1df_joined = df_left.join(df_right, 'key', 'left')

df_joined 现在是基于 left 连接 df_left.key == df_right.key 的结果。PySpark 会自动删除 key 列的一个副本，这样 df_joined 就只包含一个名为 key 的列。

如果要在 df_left 和 df_right 中进行连接的键（key）名称不相同，建议在执行连接之前先重命名它们。

冲突的名称

请确保重命名或删除任何你没有明确连接的字段，这些字段与这些字段名称相同，一旦连接完成就会发生冲突。可以在一个循环中将 DataFrame 中的所有列重命名为具有特定前缀，如下所示：

Copied!
1for column in df.columns:
2	df = df.withColumnRenamed(column, 'some_prefix_' + column)

在多个字段上连接

.join() 方法可以接受一个要连接的字段列表，而不是单个字段。

Copied!
1df_joined = df_left.join(df_right, ['column1', 'column2', 'column3'], 'left')

df_joined 现在是基于 column1、column2 和 column3 的连接。同样，这假设 df_left 和 df_right 中的列名是一致的。

高级任意连接约束

PySpark 支持使用任意表达式，通过逻辑运算符进行连接。假设我们想在列 ID 上进行连接，左表 DataFrame 中的日期 start 在右表 DataFrame 中的日期 end 之前，并且根据某个字段 X 的内容，可能需要或不需要要求右表 DataFrame 中的 Y 包含另一个值。

Copied!
1key_constraint = df_left.ID == df_right.ID
2date_constraint = df_left.start < df_right.end
3case_constraint = F.when(df_left.X == 'some_value', df_right.Y == 'some_other_value')\
4	.otherwise(True)
5combined_constraints = key_constraint & date_constraint & case_constraint
6df_joined = df_left.join(df_right, combined_constraints, 'left')

交叉连接（笛卡尔积）

使用交叉连接在两个数据框之间生成所有行组合，也称为笛卡尔积，没有任何基于键或其他约束的匹配。由于交叉连接可能导致内存和性能问题，应尽可能避免使用。

警告

如果打算立即对结果进行筛选，请不要使用交叉连接。相反，将筛选条件嵌入到连接约束中，以获得更高效的解决方案（请参见上文的 高级任意连接约束）。

您必须显式地 导入配置文件 CROSS_JOIN_ENABLED 才能在您的代码仓库中使用交叉连接。

Copied!
1from transforms.api import configure
2
3@configure(profile=["CROSS_JOIN_ENABLED"])
4@transform_df(
5	...
6)
7def my_compute_function(input_a, input_b):
8	return input_a.crossJoin(input_b)
