连接 (Joins)

在 PySpark 中，可以将一个 DataFrame 与另一个 DataFrame 进行连接，或者与其自身连接，这与 SQL 中的表连接类似。通过 .join() 方法，可以将 DataFrame 与其他 DataFrame 连接起来。该方法接收一个 DataFrame、一个连接约束条件（例如用于连接的列名）以及一种连接方式（left、right、inner 等）。

简单的左连接

已复制！
1df_joined = df_left.join(df_right, 'key', 'left')

此时，df_joined 便是基于 df_left.key == df_right.key 执行 left 连接后的结果。PySpark 会自动丢弃重复的 key 列，因此 df_joined 中只会包含一个名为 key 的列。

如果 df_left 和 df_right 中用于连接的键名不一致，建议在执行连接操作前先对它们进行重命名。

列名冲突

请务必重命名或丢弃那些未显式用于连接但名称相同的字段，因为一旦连接完成，这些同名字段就会发生冲突。可以通过类似如下的循环操作，为 DataFrame 中的所有列添加指定前缀：

已复制！
1for column in df.columns:
2	df = df.withColumnRenamed(column, 'some_prefix_' + column)

多字段连接

.join() 方法可以接收一个字段列表进行连接，而非仅限于单个字段。



已复制！
1df_joined = df_left.join(df_right, ['column1', 'column2', 'column3'], 'left')

此时 df_joined 已基于 column1、column2 和 column3 完成了连接。再次强调，这里的前提是 df_left 和 df_right 之间的列名保持一致。

进阶：任意连接约束

PySpark 支持利用逻辑运算符，通过任意表达式进行连接。假设我们需要基于列 ID 进行连接，要求左侧 DataFrame 中的日期 start 早于右侧 DataFrame 中的日期 end；同时，根据特定字段 X 的内容，决定是否要求右侧 DataFrame 中的 Y 包含另一个特定的值。



已复制！
1key_constraint = df_left.ID == df_right.ID
2date_constraint = df_left.start < df_right.end
3case_constraint = F.when(df_left.X == 'some_value', df_right.Y == 'some_other_value')\
4	.otherwise(True)
5combined_constraints = key_constraint & date_constraint & case_constraint
6df_joined = df_left.join(df_right, combined_constraints, 'left')

交叉连接（笛卡尔积）

使用交叉连接可以生成两个数据帧之间所有行的组合，也就是笛卡尔积，无需根据键或其他约束进行匹配。由于交叉连接极易引发内存与性能问题，应尽量避免使用。

警告

如果打算对结果立即进行过滤，请勿使用交叉连接。相反，应将过滤条件直接嵌入到连接约束中，以获得更高效的解决方案（参见上文的高级任意连接约束）。

要在代码仓库中使用交叉连接，必须显式导入配置项 CROSS_JOIN_ENABLED。


```python
from transforms.api import configure

@configure(profile=["CROSS_JOIN_ENABLED"])
@transform_df(
	...
)
def my_compute_function(input_a, input_b):
	return input_a.crossJoin(input_b)
```
