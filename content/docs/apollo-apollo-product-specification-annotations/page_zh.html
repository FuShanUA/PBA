标签和注解

本页描述了 Apollo 在 Spoke 环境中运行产品时添加的自定义 Kubernetes 标签（Label）与注解（Annotation）。

标签规范

apollo.palantir.com/status.liveness_and_readiness

为了使服务的 存活状态（liveness）和就绪状态（readiness） 被报告到 Apollo 平台，该标签 apollo.palantir.com/status.liveness_and_readiness 必须出现在服务的 Pod spec 模板上。该标签会自动被添加到使用提供的 Apollo Agent 部署的任何 Entity 上。

示例：

Copied!
1apollo.palantir.com/status.liveness_and_readiness: ""

注释规范

apollo.palantir.com/metadata.environment.id

值： 注释的值是实体所属的 Apollo 环境 ID。当使用提供的 Apollo 代理部署实体时，会自动添加 Apollo 环境 ID 注释。

示例：

Copied!
1apollo.palantir.com/metadata.environment.id: example-apollo-environment-id

apollo.palantir.com/metadata.entity.id

值： 注释的值是与 K8s 对象相关的 Apollo 实体 ID。Apollo 实体格式为 aeid:<apollo-environment-id>:<entity-type>:<entity-name>. Apollo 实体 ID 必须小于 255 个字符。当使用提供的 Apollo 代理部署实体时，会自动添加 Apollo 实体 ID 注释。

示例：

Copied!
1apollo.palantir.com/metadata.entity.id: "aeid:example-apollo-environment-id:helm-chart:prometheus-helm"

apollo.palantir.com/metadata.product.name

值： 注释的值是 K8s 对象所关联的 Apollo 实体的产品名称。产品名称在使用提供的 Apollo 代理部署实体时会自动添加。

示例：

Copied!
1apollo.palantir.com/metadata.product.name: prometheus

apollo.palantir.com/metadata.product.group

值： 注释的值是 K8s 对象所关联的 Apollo 实体的产品组。产品组在使用提供的 Apollo 代理部署实体时会自动添加。

示例：

Copied!
1apollo.palantir.com/metadata.product.group: com.palantir

apollo.palantir.com/metadata.product.version

值： 注释的值是 K8s 对象所关联的 Apollo 实体的产品版本。产品版本在使用提供的 Apollo 代理部署实体时会自动添加。

示例：

Copied!
1apollo.palantir.com/metadata.product.version: 2.0.0
