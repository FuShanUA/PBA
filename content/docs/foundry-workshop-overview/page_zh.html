Workshop

Workshop 赋能应用构建者为业务运营人员打造交互性强且体验优异的应用。

Workshop 遵循以下三大核心原则：

  • 对象数据
    • Workshop 将对象层作为核心基础构件，以此降低应用构建者的准入门槛。Workshop 应用中的所有数据均读取自对象数据层，这使得应用创建者能够充分利用对象类型间的关联等丰富特性。Workshop 利用操作实现对对象数据的回写，并借助函数处理对象数据上的业务逻辑。
  • 统一的设计规范
    • 所有 Workshop 组件均遵循统一的设计体系，可实现顺畅交互，外观与体验保持一致，从而为最终用户提供高质量的使用体验。
  • 交互性与复杂度
    • 相较于使用其他点击式工具创建的常规仪表板，基于 Workshop 构建的应用具备更强的动态性与交互性。依托于高质量的布局以及易用且功能强大的事件系统，Workshop 应用致力于达到与定制化 React 应用同等的高水准用户体验与卓越质量。

Workshop 是一款灵活的应用构建器，能够支持丰富多样的业务场景。Workshop 支持的常见应用模式包括：

  • 收件箱告警与任务管理
    • 收件箱常用于辅助业务运营人员对任务进行分类、定优及处理。例如，你可以利用 Workshop 应用对新接入的保修索赔进行归类，或者对告警进行分流与调查。
  • 通用作战图 (COPs)
    • 通用作战图 (COP) 是一种跨多个组织共享的运营信息展示形式，旨在提升态势感知能力并促进协同合作。例如，你可以构建一个 Workshop 应用作为“大屏展示”的当前态势视图，其中包含地图、相关统计图表、数据过滤或下钻途径，以及与其他视图或工作流的联动入口。
