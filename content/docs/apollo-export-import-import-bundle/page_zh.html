导入一个包

本节将介绍如何将一个 包 导入到目标中心。

在遵循此指南之前，您应该从源中心 创建并导出一个包。您还应使用适当的传输流程将包转移到目标中心环境。

导入该包

要导入一个包，请将鼠标悬停在左侧边栏的 应用 图标上，然后选择 导入。

如果没有正在进行的包传输，您可以从 操作 下拉菜单中选择 上传一个包 来导入一个新的包。您也可以拖放一个包或选择 从您的计算机中选择。

选择捆绑包文件，然后选择上传捆绑包。捆绑包会异步处理，其内容将随着时间的推移在目标中心中可用。

您可以通过导航到历史记录选项卡，并从下拉菜单中选择管道来查看导出管道的导入历史记录。

选择一个捆绑包版本以查看导入进度。

如果捆绑包导入不成功，您可以选择查看详细信息以查看失败的导入阶段。

环境导入时的变更请求

当一个包包含环境变更时，Apollo 会创建一个变更请求，在应用变更之前进行审核。一些变更，例如对网络安全规则的修改，需要手动审批。您可以从导入历史记录中查看每个包的审批状态。
