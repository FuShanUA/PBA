将建模资源添加到 Marketplace 产品中

您可以使用 Foundry DevOps 将建模资源包含到 Marketplace 产品中，供其他用户安装和复用。了解如何创建您的第一个产品。

支持的功能

模型可以作为 DevOps 中产品的一部分进行打包，并通过 Marketplace 进行部署，以实现发布管理目的。模型也支持作为产品输入，允许用户在安装过程中选择模型。

打包模型

模型可以作为输出（即将部署的内容）打包到 Marketplace 产品中，或者作为用户在安装期间选择的输入包含在内。

将模型包含在产品输出中有两种方式：随内容包含和随生产者包含。

随内容包含

随内容包含选项会将模型安装为静态资产，部署后无法重新构建。使用此选项时，以下内容将被复制到模型资源中：

  • 已保存的模型文件和权重。
  • 包含模型适配器类的 Python 库。
  • 用于在模型应用程序和建模目标中支持实时部署的容器镜像。
  • 转换中的 Sidecar 模型推理。

如果模型总大小（包括文件和底层容器镜像）不超过 8 GiB，则只能将模型部署为随内容包含。DevOps 应用程序将验证此条件，如果不满足将显示错误。

随生产者包含

随生产者包含选项会安装用于构建模型的逻辑，以便在部署后能够对其进行构建。这使用户能够部署可以按计划持续训练的模型。这适用于在代码仓库和发布为转换的 Jupyter® 笔记本中构建的模型，但不适用于从 Jupyter® 笔记本中以交互方式发布的模型。这是因为交互式发布的模型被视为静态资产，没有关联的逻辑。目前对在 Model Studio 中创建的模型的支持正在开发中。使用此选项时，建议打包模型的最新版本以避免混淆。

使用代码仓库生产者部署的模型默认没有定义容器镜像，因为该镜像是在仓库中运行检查时生成的，并用于后续生成模型的构建。安装产品时不会运行检查，因此已安装的模型构建无法为模型生成容器镜像。模型版本缺少容器镜像将导致无法在转换和实时部署中对已部署的版本使用 Sidecar 模型推理。

要解决此问题，请手动对生成模型的已安装仓库触发检查，以构建容器镜像并使其可用于后续的模型训练构建。必须手动触发检查，以确保 Python 适配器逻辑和容器中的环境与源模型中的相同。每次部署新版本的模型时都必须执行此操作。

将模型用作产品输入

除了将模型打包为输出外，产品构建者还可以将模型作为产品的输入包含在内。在安装过程中，用户通过模型选择器选择要使用的模型，这允许他们解析依赖组件所需的特定模型版本。这避免了打包模型本身的需要，减小了包大小，并让安装者能够灵活选择其注册中已有的模型。

在打包转换中将模型作为 ModelInput 使用时，转换操作必须设置 use_sidecar=True。sidecar 会在独立的容器中运行模型适配器，无需将适配器库导入代码库环境。建议在所有场景下使用 use_sidecar=True。有关《将模型作为 sidecar 容器运行》的更多信息，请查阅相关文档。

实时部署

通过 Marketplace 部署的模型可作为实时部署使用，但需遵循上述关于模型容器镜像的注意事项。实时部署只需在部署后手动配置一次，Marketplace 无法自动完成配置。如果通过常规构建方式或升级 Marketplace 产品来重新构建模型，实时部署将自动更新并指向最新创建的版本，具体请参阅《模型部署》文档中的说明。

模型函数

在已部署的模型上设置函数

与直接部署一样，模型函数只需在模型部署后配置一次，并在创建新模型版本时自动升级。

在另一个函数中使用该模型函数

由于 Marketplace 无法显式识别模型函数，因此当从 Workshop 应用程序或通过 Marketplace 部署的《另一个函数》中调用模型函数时，需要格外注意。在 Workshop 中使用或导入代码库的模型函数，会被识别为安装期间需要提供的输入项。相反，如果未将模型函数导入代码库，而是直接通过《平台 SDK》进行查询，则这些函数会被忽略。

以下是打包模型、模型函数以及调用该模型函数的外部函数的推荐设置方式。

  1. 创建一个包含模型的产品。首次部署后，配置实时部署并发布一个《注册到本体》的函数。
  2. 创建第二个产品，包含外部函数（TypeScript v1、TypeScript v2 或导入了模型函数的 Python）以及任何依赖的应用程序，并将模型函数作为输入项。安装此第二个产品时，请使用上一步中创建的函数。
    • 每次模型 API 发生变更时，作为输入项的模型函数都需要更新至最新版本，因为模型函数仅接受符合《模型 API 定义》的类型化对象。

对于 Python 和 TypeScript v2 函数，模型函数必须注册到本体才能导入代码库。如果您的模型函数仍受限于空间，请将其《迁移至受限于本体的函数》，以便利用结构化依赖跟踪和 Marketplace 输入映射功能。

不支持的建模功能与限制

  • 建模目标：请使用《直接模型部署》进行实时目标部署，并使用《模型转换》进行批量部署。
  • 暂不支持 Model Studio 模型。该功能计划于 2026 年 6 月底推出。
  • 包含内容的模型仅在总大小（包括权重、Python 环境和底层容器镜像）低于 8 GiB 时才能进行打包。
  • 《外部模型》不支持在 DevOps 或 Marketplace 中使用。
