创建匹配操作

匹配操作允许敏感数据扫描器在检测到匹配的数据集时代表您执行自动操作。有三种类型的匹配操作：

  • "应用标记" 匹配操作： 指示敏感数据扫描器对匹配任何指定匹配条件的数据集应用标记。
  • "创建问题" 匹配操作： 指示敏感数据扫描器在匹配任何指定匹配条件的列上创建问题。
  • "混淆数据" 匹配操作： 指示敏感数据扫描器使用 Cipher 对匹配的数据进行加密或哈希处理。
测试您的匹配条件

如果您的敏感数据扫描覆盖了大量数据集，我们建议您在继续之前测试匹配条件，因为配置错误的匹配条件可能会以不需要的问题或标记的形式产生误报。

您可以通过为扫描选择 无匹配操作 来测试匹配条件。一旦确认匹配条件符合数据的预期格式，您可以为未来的单次扫描或定期扫描选择其他匹配操作之一。

与创建匹配条件类似，有两种方法可以开始创建匹配操作 - 一种是从敏感数据扫描器的首页，另一种是在创建敏感数据扫描时。

从首页上，点击 添加 按钮，位于 可用的匹配操作 列表上方。

在创建敏感数据扫描时，在 选择匹配操作 页面上，您也可以通过点击 创建匹配操作 来立即创建一个匹配操作并将其用于您的扫描。

这两个起点会打开相同的匹配操作创建流程。从那里，您可以选择是创建一个 添加标记 的匹配操作，还是创建一个 创建问题 的匹配操作。

此外，重新应用已被手动移除的标记 选项未被选中，这意味着如果该标记之前由敏感数据扫描器（例如，在之前的扫描期间）应用过，并且被用户手动移除了，那么该标记将不会被重新应用。启用该选项允许重新应用该标记。

创建一个“创建问题”匹配操作

此为“创建问题匹配操作”创建过程的示例。在此，您会注意到已选择两位用户作为 Sensitive Data Scanner 在检测到匹配项时将创建的问题的分配者——“治理管理员”和“数据控制官员”。

此外，还提供了一些“创建问题”匹配操作的高级配置选项：

  • 问题文本：您可以自定义在执行匹配操作时问题将具有的文本内容。
  • 问题标签：您可以选择适用于您的 空间 的可用问题标签，以帮助有效分类问题。
  • 问题严重程度：您可以根据希望运行的扫描设置问题的优先级。
  • 当发现该问题时通知数据集创建者：如果选中此选项，创建该数据集的用户将收到通知，告知其创建的数据集中发现了敏感数据。
  • 如果满足以下条件则不创建新问题：
    • 已归档 — 如果选中此选项，当之前的扫描已经创建了一个问题并手动归档后，Sensitive Data Scanner 将不会创建新问题。
    • 已关闭 — 如果选中，当之前的扫描创建了一个问题并手动关闭后，敏感数据扫描器将不会创建新问题。

创建“混淆数据”匹配操作

混淆数据 匹配操作允许您使用 Cipher 自动对匹配的数据进行加密或哈希处理。对于每个扫描的资源，敏感数据扫描器将创建一个包含混淆数据的输出数据集。

要创建一个新的“混淆数据”匹配操作，您可以打开 创建匹配操作 对话框并选择 混淆数据。

您需要一个 Cipher 通道 和一个 Cipher 许可证。如果您尚未拥有这些，可以在 Cipher 中创建它们。

Cipher 通道指定了用于混淆的加密算法。敏感数据扫描器支持：

  • 概率加密：AES GCM SIV
  • 确定性加密：AES SIV
  • 哈希：SHA512 和 SHA256

请注意，视觉混淆通道在敏感数据扫描器中不受支持。

A Cipher 许可证授予对 Cipher 通道进行交互的一组权限。敏感数据扫描器需要一个 管理员许可证，因为它需要访问加密密钥以执行混淆操作。

在确认您拥有可用的 Cipher 许可证后，可以在 创建匹配操作 对话框中选择它。

匹配操作只能应用于与 Cipher 许可证位于同一项目的已扫描资源。

混淆数据操作将为每个已扫描的数据集创建一个输出数据集。根据扫描资源的数量，这可能导致生成大量输出数据集。我们建议在少量数据上运行此扫描。

通过敏感数据扫描器无法撤销混淆数据操作。由敏感数据扫描器创建的数据集必须手动删除。
