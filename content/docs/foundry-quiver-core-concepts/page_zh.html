核心概念

本页面将讨论理解 Quiver 应用所需的几个核心概念：

  • 卡片
  • 画布与图
  • 对象
  • 时间序列
  • 仪表板
  • 参数
  • 保存与版本控制

卡片

在 Quiver 中，探索与分析工作通过卡片完成，多张卡片可串联起来执行复杂的操作。部分卡片用于创建图表或执行计算，另一些则用于通过过滤、连接、派生新列等方式处理数据。

Quiver 中的每张卡片可接收零个或多个输入，并产生特定类型的输出，例如 对象集、时间序列、分类图表、数字 等。这些类型共同构成了 Quiver 的 数据模型，并定义了卡片之间如何串联。

选中卡片右上角的 图标，即可在编辑器面板中配置卡片。

画布与图

Quiver 提供两种视图模式来构建分析：画布模式和 图模式。

画布是一个页面，您可以在上面展示、重新排列和调整分析中卡片的大小。一个分析可以包含多个画布。

与 Contour 路径不同，Quiver 画布仅用于展示和组织。在画布上重新排列卡片不会影响底层数据转换的顺序。

在 图模式 下，卡片表示为节点，卡片之间的连接表示为链接，采用从左到右的布局，方便您理清数据流。您可以使用 颜色组 来组织图，并通过 隐藏或过滤节点 来聚焦于分析的特定区域。

在图模式下添加的卡片不会自动放置在画布上。您可以随时在画布上添加或移除卡片。

图模式最适合表格式分析，而画布模式最适合时间序列分析。无论数据类型如何，画布模式更适合展示图表和表格等最终输出结果，而图模式则更适合检查卡片依赖关系。

了解更多关于画布模式 和 图模式 的信息。

对象

在 Quiver 中，来自 本体 的对象用作表格式分析的主要数据输入。Quiver 原生支持多种用于过滤、转换和可视化对象数据的卡片。

对于对象数据的高级转换操作（例如派生属性和连接关联对象），用户还可以利用 Quiver 提供的 转换表、物化 和 函数 卡片套件。

了解更多关于对象分析的信息。

时间序列

Quiver 为 时间序列 分析提供了一流的支持。时间序列主要通过 时间序列属性 添加到 Quiver 中，但也可以直接查看 时间序列同步。

Quiver 为时间序列数据提供了丰富的 转换 和 可视化 库。Quiver 还支持高级时间序列工作流，例如 异常检测 和 事件分析。

此外，时间序列转换可以保存为 派生序列，或者用于通过 时间序列自动化 创建告警。

了解更多关于时间序列分析的信息。

仪表板

借助 Quiver，您可以构建交互式仪表板，以展示分析的结果和发现。这些仪表板可以作为独立视图使用，也可以嵌入到其他 Foundry 应用程序（例如 Workshop）中。

了解更多关于仪表板的信息。

参数

Quiver 参数允许您轻松切换数据和结果的不同视图。创建参数后，您可以在卡片中使用它们，并将其展示在仪表板中。这使得仪表板的最终用户能够与所展示的数据和结果进行实时交互。

了解有关参数的更多信息。

保存与版本管理

通过点击应用程序右上角的“保存”按钮，可以手动保存 Quiver 分析。系统还提供版本历史记录功能，方便您查看或将分析回退到先前保存的版本。此外，在每次执行“保存”操作之间，Quiver 会自动保存您的“工作”状态（将其存储在 state URL 变量中，例如 state=j05na7mun3）。这样，即使您未进行保存，刷新页面后也能恢复到准确的分析状态。请注意，如果您分享的 URL 链接中包含了 state 变量，打开该链接时将进入该工作状态，而非最新的分析版本。

如果多个用户同时操作同一分析任务，他们可以互不干扰地独立工作，但在保存更改时会相互覆盖已保存的内容。
