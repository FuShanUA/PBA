设置和使用实时部署

一个 实时部署 是您发布版本的持久化、可扩展的部署，可以通过 API 端点与其进行交互。 实时部署可以与您的批量部署一起在建模目标中进行管理，获得自动化升级、可观测性和权限结构的所有好处。 现在也可以直接在模型页面上配置实时部署，而无需需要目标。

要求

在创建新的实时部署之前，建模目标必须包含具有相应标签（Staging 或 Production）的现有发布版本。

创建部署

要创建一个新的实时部署，请导航至建模目标底部的部署部分，然后选择蓝色的+ 创建一个部署按钮。

填写部署名称、描述以及该部署是否基于当前的暂存或生产发布的模型。配置完成后，选择创建部署。

选择新创建的部署以打开详细信息，在这里您将看到中间状态，表明包含最新发布和推理代码的容器镜像正在被部署。

发布函数

您可以为部署发布一个函数（函数），从而在Workshop（工作区）、Vertex（顶点）和其他终端用户应用程序中使用模型进行实时推理。

要发布一个函数，请导航到部署的详情页面，然后选择发布函数。这将创建一个具有与您的模型相同输入和输出API的包装函数，可以从函数仓库导入并调用以添加自定义业务逻辑。

有关函数行为、版本升级和配置选项的详细信息，请参阅 模型函数开发人员指南。

状态和健康状况

部署更新完成后，状态和健康状况都应该显示为绿色对勾。这些表示部署已准备好查询，并且成功更新为包含最新发布的模型。

测试实时部署

您可以通过前往部署详情页面的查询选项卡来测试实时部署端点。您可以编写输入请求，将其发送到实时端点，并查看模型输出响应。有两种查询类型可用：单I/O和多I/O。

单I/O查询仅支持单表格输入/输出，而多I/O查询仅支持在模型资产中定义的ModelAdapter API。模型资产支持单I/O和多I/O查询类型，而基于数据集的部署仅支持单I/O查询。

示例命令

以下是通过curl使用转换端点进行单I/O查询的示例：

1curl -X POST https://<URL>/foundry-ml-live/api/inference/transform/ri.foundry-ml-live.main.live-deployment.<RID>
2-H "Authorization: <BEARER_TOKEN>" -H "Accept: application/json" -H "Content-Type: application/json" -d
3'{"requestData":[{"house-location":"New York", "bedrooms":3,"bathrooms":1.5}],
4 "requestParams":{}}'

或者，下面的示例展示了如何通过curl使用transformV2端点进行多I/O查询：

部署操作

管理操作

对于正在运行的实时部署，有两种选择：

  • 禁用以关闭部署，但仍允许重新启用以利用相同的 目标 RID。
  • 删除以完全移除部署。旧的目标 RID 将无法再被查询。

已禁用的实时部署仍会在一个目标上显示在“部署”页面上，并且可以再次启用。

资源配置

资源配置

实时部署需要配置副本数以及CPU和GPU的数量。

您可以在创建部署时配置资源需求，也可以在之后进行编辑。编辑运行时配置将自动触发部署的重新运行，这将在无停机时间的情况下升级端点。

要配置新的实时部署，请选择创建部署，在部署类型下选择实时，然后输入副本数和资源配置。

要编辑已配置的副本和资源，请导航到您的建模项目中的部署部分，从列出的部署中选择您的部署，并在运行时配置下选择编辑按钮以编辑副本或资源配置。

查看部署服务日志

从每个正在运行的部署中会发出一系列描述部署启动进度的日志；这些日志包括部署将尝试下载和安装的发布环境中的Conda包的URI，以及有关下载模型本身的详细信息。如果部署未能成功启动，可能会有相关的警告或错误级别日志，可以帮助您了解发生了什么。

要直接从正在运行的实时部署查看服务日志，请导航到部署详情页面上的日志与指标选项卡。您可以指定一个时间窗口以进行搜索，按任何字段过滤日志，并从视图中添加或删除列。您还可以使用右上角的下载按钮将所有日志作为文本文件下载。

因为您可能已配置部署以使用多个副本运行，因此可能会看到看似重复的日志——每组日志来自不同的副本。但是，每个日志都包含一个带有唯一 node_id 的标签，您可以根据需要使用该标签按副本进行过滤。您可以使用以下符号表示： 

Copied!
1tags.node_id:{Insert UUID here}

要直接从您的模型发出日志，可以使用标准的 python 日志模块。实时部署将使每一行日志可供在 日志与指标 选项卡内查询。

Copied!
1import logging
2
3log = logging.getLogger('model-logger')
4log.info("Emitting info directly from the model")

要从 基于容器的模型 发出日志，请在 配置模型版本 时启用遥测功能。

您实时部署的日志保留期为7天，之后将无法查看或下载。

查看部署指标

日志与指标 选项卡提供两种类型的指标：Kubernetes 主机 和 推理容器 . 请确保了解这些指标类型之间的区别，以便正确监控和调试您的模型。

Kubernetes 主机指标

Kubernetes 主机指标显示主机上所有进程使用的内存和 CPU 使用百分比，而不仅仅是与特定模型相关的进程。这些指标对于调试与调度和资源限制相关的问题非常重要。例如，如果您的模型性能较慢但主机指标为 100%，则您的模型可能被 Kubernetes 主机限制了资源。

推理容器指标

推理容器指标有助于调试 Python 模型和模型适配器逻辑中的资源使用情况。这些指标提供了推理容器的确切内存使用量和 CPU 核心使用量，与整个 Kubernetes 主机无关。目前，尚不可用基于容器的模型的使用指标。

如果您只能查看Kubernates主机指标，但无法查看推理容器指标，则您的容器可能运行在过时的实时部署版本上。请重新启动实时部署以更新版本并查看所有指标。

Spark模型支持

Beta

支持的Spark版本可能会在后续发行版中发生变化，并且不保证向后兼容性。如果您的模型与当前Spark版本不兼容，您可能需要重新构建它。

所有实时部署现在都已初始化，安装了JDK和Spark发行版。这使得Spark模型与实时部署兼容。在此交互式环境中，仅支持本地Spark，这意味着所有处理都在单个JVM中完成。

如果您的模型需要Spark DataFrame输入，由于请求以JSON形式发送，数据转换问题更可能发生。在开发需要Spark数据框的自定义模型时，您应该在预处理阶段或直接在transform函数中手动执行此转换。下面是一个简单的示例：

Copied!
1import pandas as pd
2from pyspark.sql import SparkSession
3
4def _transform(model, df):
5    if isinstance(df, pd.DataFrame):
6        df = SparkSession.builder.getOrCreate().createDataFrame(df)
7    return model.predict_spark_df(df)
