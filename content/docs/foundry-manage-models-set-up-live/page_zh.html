设置与使用实时部署

实时部署是针对您的发布版本的持久且可扩展的部署，可通过 API 端点与其交互。

实时部署可以与建模目标中的批量部署统一管理，同样享有自动升级、可观测性和权限结构等优势。现在也可以直接在模型页面上配置实时部署，无需依赖建模目标。

前提要求

在创建新的实时部署之前，建模目标中必须包含带有相应标签（Staging 或 Production）的现有发布版本。

创建部署

要创建新的实时部署，请前往建模目标底部的 Deployments（部署）部分，选择蓝色的 + Create a deployment（+ 创建部署）按钮。

填写部署名称、描述，并选择该部署是基于当前的预发布（Staging）还是生产（Production）已发布模型。配置完成后，选择 Create deployment（创建部署）。

选中新创建的部署以打开详情页面，您将看到中间状态，这表示正在部署包含最新发布版本和推理代码的容器镜像。

发布函数

您可以为部署发布一个函数，以便在 Workshop、Vertex 和其他终端用户应用中使用模型进行实时推理。

要发布函数，请前往部署的 Details（详情）页面并选择 Publish Function（发布函数）。这会创建一个包装函数，其输入和输出 API 与您的模型一致，该函数可以从函数库中导入并调用，以添加自定义业务逻辑。

有关函数行为、版本升级和配置选项的详细信息，请参阅《模型函数开发者指南》。

状态与健康度

部署更新完成后，状态和健康度应均显示为绿色的勾选标记。这表示部署已准备好接受查询，并且已成功更新为包含最新发布的模型。

测试实时部署

您可以通过进入部署详情页面的 Query（查询）选项卡来测试实时部署端点。您可以编写输入请求，将其发送到实时端点，并查看模型的输出响应。目前提供两种查询类型：单输入/输出和多输入/输出。

单输入/输出查询仅支持单个表格型的输入/输出，而 多输入/输出查询仅支持在模型资产中定义的 ModelAdapter API。模型资产同时支持单输入/输出和多输入/输出查询类型，而基于数据集的部署仅支持单输入/输出查询。

示例命令

以下示例展示了如何使用 curl 通过单输入/输出和转换（transform）端点查询实时部署：


```bash
curl -X POST https:///foundry-ml-live/api/inference/transform/ri.foundry-ml-live.main.live-deployment.
-H "Authorization: " -H "Accept: application/json" -H "Content-Type: application/json" -d
'{"requestData":[{"house-location":"New York", "bedrooms":3,"bathrooms":1.5}],
 "requestParams":{}}'
```

此外，以下示例展示了如何通过 `curl` 使用 transformV2 端点的多输入/输出功能来查询实时部署：

```bash
curl -X POST https:///foundry-ml-live/api/inference/transform/ri.foundry-ml-live.main.live-deployment./v2
-H "Authorization: " -H "Accept: application/json" -H "Content-Type: application/json" -d
'{"input_df":[{"house-location":"New York", "bedrooms":3,"bathrooms":1.5},{"house-location":"San Francisco", "bedrooms":2,"bathrooms":1}]}'
```

## 部署操作

### 管理操作

针对运行中的实时部署，可执行以下两种操作：

- **禁用**：关闭部署，但仍允许重新启用以继续使用相同的 *Target RID*。
- **删除**：彻底移除部署。原有的 Target RID 将无法再用于查询。



被禁用的实时部署仍会显示在目标的“部署”页面上，并且可以再次启用。



### 资源配置

实时部署需配置副本数量以及 CPU 和 GPU 数量。

您可以在创建部署时配置资源需求，也可以在创建后进行修改。编辑运行时配置会自动触发部署的重新运行，并在零停机时间内完成端点升级。

要配置新的实时部署，请选择 **创建部署**，在 **部署类型** 下选择 **实时**，然后输入副本数量和资源配置。




要编辑已配置的副本和资源，请前往建模项目中的部署部分，从列表中选择您的部署，并在运行时配置下选择编辑按钮，以修改副本或资源配置。




查看部署服务日志


每个实时部署都会输出一系列描述部署启动进度的日志；这些日志包含部署将尝试下载和安装的发布环境中 Conda 包的 URI，以及下载模型本身的详细信息。如果部署未能成功启动，可能会有相关的警告或错误级别日志，帮助您了解发生了什么。


要直接从运行中的实时部署查看服务日志，请前往部署详细信息页面的日志与指标选项卡。您可以指定搜索的时间窗口，按任意字段过滤日志，以及在视图中添加或删除列。您还可以使用右上角的下载按钮将所有日志下载为文本文件。




由于您可能已将部署配置为使用多个副本运行，因此您可能会看到看似重复的日志——每一组来自不同的副本。但是，每条日志都包含一个带有唯一 node_id 的标签，如有需要，您可以使用它按副本进行过滤。您可以为此使用以下表示法：


Copied!
1tags.node_id:{Insert UUID here}


要直接从您的模型输出日志，可以使用标准的 python logging 模块。实时部署会将每行日志提供出来，以便在日志与指标选项卡中进行查询。


Copied!
1import logging
2
3log = logging.getLogger('model-logger')
4log.info("Emitting info directly from the model")


要从容器支持模型输出日志，请在配置模型版本时启用遥测。


实时部署的日志保留期为 7 天，此后将无法查看或下载。



查看部署指标


日志与指标选项卡提供两类指标：Kubernetes 主机和推理容器。务必理解这些指标类型之间的差异，以便正确监控和调试模型。


Kubernetes 主机指标


Kubernetes 主机指标展示了主机上所有运行进程的内存和 CPU 使用率百分比，而非仅限于与特定模型相关的进程。这类指标对于排查调度和资源限制引起的问题至关重要。例如，如果模型运行缓慢且主机指标已达 100%，说明模型可能受到了 Kubernetes 主机的资源限制。


推理容器指标


推理容器指标有助于排查 Python 模型和模型适配器逻辑中的资源使用问题。这些指标提供了推理容器本身精确的内存和 CPU 核心使用情况，独立于整个 Kubernetes 主机。目前，基于容器的模型尚无法获取使用量指标。




如果只能查看 Kubernetes 主机指标而无法查看推理容器指标，说明您的容器运行的实时部署版本可能已过时。请重启实时部署以更新版本，从而查看所有指标。

支持的 Spark 版本可能会在后续版本中发生变化，且不保证向后兼容。如果您的模型与当前 Spark 版本不兼容，可能需要重新构建模型。


目前，所有实时部署在初始化时都会安装 JDK 和 Spark 发行版。这使得 Spark 模型能够与实时部署兼容。在此交互式环境中，仅支持本地 Spark，这意味着所有处理过程都在单个 JVM 内完成。


如果您的模型需要输入 Spark DataFrame，由于请求以 JSON 格式发送，更容易出现数据转换问题。在开发需要 Spark DataFrame 的自定义模型时，您应该在预处理阶段或直接在 transform 函数中手动执行此转换。以下是一个简单的示例：

6        df = SparkSession.builder.getOrCreate().createDataFrame(df)
7    return model.predict_spark_df(df)
