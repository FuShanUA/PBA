初始化新的 OSDK TypeScript 应用程序

本页将引导您完成使用 @osdk/create-app CLI 工具，在主流 JavaScript 框架上构建前端应用程序的过程。

若您希望为现有应用程序添加 OSDK 支持，请查阅我们关于《向现有应用程序添加 OSDK》的文档。

1：前提条件

创建开发者控制台应用程序

请按照《创建新的开发者控制台应用程序》页面中列出的步骤操作。

设置您的令牌

在本地环境中导出您的令牌。以下示例使用了一个个人访问令牌作为演示，您可以在开发者控制台中生成有效期更长的令牌。切勿将此令牌提交至源代码版本控制系统中，因为它是您的个人访问令牌。

Copied!
1export FOUNDRY_TOKEN=<YOUR-TOKEN-FROM-GETTING-STARTED-PAGE>

检查 Node 版本

Typescript SDK 需要 Node 18 或更高版本才能正常运行。要检查您当前使用的 Node 版本，请输入以下命令：

Copied!
1node --version

2. 快速上手 @osdk/create-app

创建您的前端应用程序

运行提供的命令并按照交互式提示自定义您的项目，包括项目名称和框架选择。在“入门”页面上，您可以找到特定于应用程序的信息，这些信息将为您预填到命令中。以下是此代码的示例，其中 < > 内的占位符将填入您的具体详细信息：



已复制！
1npm create @osdk/app@latest -- \
2    --application <RID OF YOUR DEVELOPER CONSOLE APPLICATION> \
3    --foundryUrl <YOUR FOUNDRY URL> \
4    --applicationUrl <SUBDOMAIN OF YOUR FOUNDRY URL USED FOR HOSTING> \
5    --clientId <YOUR CLIENT ID> \
6    --osdkPackage <YOUR PACKAGE NAME> \
7    --osdkRegistryUrl <YOUR PACKAGE HOSTING URL> \
8    --corsProxy false

在开发者控制台的 API 文档 部分的 入门 页面，或者 概览 页面上，可以找到预填好所有这些参数的命令。

开发您的前端应用

您的项目文件现已生成在一个基于您输入的项目名称的目录中。运行以下命令即可启动本地开发服务器：

已复制！
1cd <project-directory>
2npm install
3npm run dev
