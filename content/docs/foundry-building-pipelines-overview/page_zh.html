构建管道

创建数据管道的首要步骤，是将组织的数据源接入 Foundry 并让数据在系统中流转。起初，重点应放在验证数据是否具备高质量，从而为用例开发、模型开发和数据分析提供可靠的基础。

本部分文档重点关注管道创建的初始阶段，此时业务需求可能仍在调整，管道逻辑也会频繁变更。在此阶段，核心在于打下坚实基础——既要支撑目标用例，又要保障管道未来的可维护性。

初始步骤

在大多数情况下，管道开发应遵循以下初始步骤：

  • 设置推荐的项目结构，以便从开发流程的最初阶段就妥善管理数据安全和数据治理。
  • 在 Pipeline Builder 或 Code Repositories 中创建批处理管道，对输入数据集进行处理、清洗和过滤，并与其他数据集进行关联，生成高质量的数据集，将其输入本体以支持工作流开发。
  • 将最终的数据集映射为本体中的对象类型和链接类型。
  • 设置调度，使数据开始定期流转。

除上述步骤外，你还可以采取多项措施来提升管道的健壮性和可扩展性，包括添加单元测试、建立分支与发布流程以及定义健康检查。了解管道开发的最佳实践。

增量管道

如果流入管道的输入数据变更规模很大，最好创建一个增量管道，以高性能的方式处理变更数据。在大多数情况下，你可以从批处理管道起步，随后再引入增量管道来提升性能并降低延迟。

在某些场景下，最好从一开始就将管道设计为增量模式，特别是当你预判流入管道的新数据规模将非常庞大时。然而，增量管道的编写和维护远比批处理管道复杂。进一步了解 Foundry 中不同类型的管道。

流式管道

如果对数据延迟的要求极低，最好创建一个流式管道，以高性能的方式处理输入数据。鉴于流式管道的速度受限于最慢的组件，因此必须从设计之初就统筹规划，以确保管道达到目标延迟和吞吐量。阅读我们的流处理与批处理对比，获取更细致的分析。
