组件的自定义样式

组件的自定义样式允许用户在单个组件上应用自定义 CSS。这些样式不会与应用程序中的其他组件共享。选中组件时，可以从右侧边栏访问自定义样式。导航到侧边栏中的 Styles 选项卡，在面板底部找到自定义样式部分。

组件的自定义样式非常适合用于测试和开发样式。快速便捷地设置背景颜色、字体大小、边框圆角等属性。在单个组件上进行更改并在画布中预览结果，而不会冒更改应用程序其他部分的风险。

自定义样式可能会与样式表中的样式发生冲突，从而导致意外的结果。当两者尝试更改同一个类的相同 CSS 属性时（例如，样式表和自定义样式同时更改组件的背景颜色），就会发生这种情况。为了避免这些冲突，自定义样式应仅用于开发和迭代，而样式表则更适合管理复杂的样式。

本指南将带您完成文本组件的样式设置。

  1. 添加一个文本组件并输入 Hello World。

  2. 文本组件中通常需要更改的属性包括颜色、字体、对齐方式和边框。

    已复制！
    1#w1 {
2  background: #d9eef2;
3  font-size: 30px;
4  color: #1685ca;
5  border: 2px solid #89bff5;
6  font-family: Times;
7  text-align: center
8}





  1. 添加另一个文本组件并输入 Slate is awesome。

  2. 请注意，这些样式不会应用于这个新的文本组件，因为我们使用了 选择器 #w1 来指定仅将这些更改应用于第一个组件。我们可以简单地将 CSS 更改为 #w1, #w2，让样式同时应用于两个文本组件。但是，如果我们添加了另一个文本组件并希望应用相同的样式，该怎么办呢？

  3. 在 Styles 编辑器中进行以下更改：

    

    已复制！
    1sl-markdown {
2  background: #d9eef2;
3  font-size: 30px;
4  color: #1685ca;
5  border: 2px solid #89bff5;
6  font-family: Times;
7  text-align: center
8}

    sl-markdown 是元素标签，这段代码表示对所有 markdown 组件应用此样式。

  4. 添加第三个文本组件。请注意它会自动应用样式。



  5. 最后，保留当前的 CSS 并为 #w1 添加 CSS。

    
请注意，这正确地将 w1 的样式与其他两个组件区分开来。
