Foundry 规则

Foundry 规则（以前称为 Taurus）使用户能够通过直观的点按点击、低代码界面在 Foundry 中主动管理复杂的业务逻辑。使用 Foundry 规则，用户可以创建规则，并将这些规则应用于数据集、对象和时间序列，以实现各种用例，例如警报生成或数据分类。

Foundry 规则包含一系列用于创建、管理和应用规则的组件：

  • 一个 规则 是一组 条件，这些条件一起可以指定数据集中的特定行数据。
  • 构成规则的 条件 应用于数据集的列，可以从简单的过滤器到复杂的聚合、连接或其他操作符。

以下页面描述了几个 核心概念，并提供了如何 部署 和 自定义 Foundry 规则的说明。

示例用例

Foundry 规则可以简化管理涉及复杂规则集的用例过程，例如：

  • 反洗钱 (AML)：通过针对单笔交易和汇总指标的规则来标记可疑交易。
  • 设备监控：基于传感器数据（例如当某些测量值达到特定值时）发出潜在设备退化的警报。
  • 分组：根据规则对实体进行分类，形成“分组”。例如，创建具有特定特征的客户群，以实现更好的定向营销。
