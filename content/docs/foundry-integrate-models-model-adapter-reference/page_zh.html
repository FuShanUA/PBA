API: ModelAdapter reference

A model adapter is a published Python library that provides the communication layer between Foundry and stored model artifacts to enable Foundry to load, initialize, and run inference on any model.

To implement a ModelAdapter, you must understand the classes listed below:

ModelAdapter implementation

The ModelAdapter class is an abstract base class that all model adapter implementations must extend. There are four abstract methods that all model adapters must implement:

  1. load()
  2. save()
    • Note that load() and save() are not required for model adapters defined with default serializers.
  3. api()
  4. predict() for single-output, and run_inference() for multi-output

1import palantir_models as pm
2import models_api.models_api_executable as executable_api
3
4class ExampleModelAdapter(pm.ModelAdapter):

8        cls,
9        state_reader: pm.ModelStateReader,
10        container_context: Optional[executable_api.ContainerizedApplicationContext] = None,
11        external_model_context: Optional[executable_api.ExternalModelExecutionContext] = None
12    ) -> "pm.ModelAdapter":
13        """
14        Python or binary models:
15            This is the method that Foundry will call to deserialize your ModelAdapter. The author of this ModelAdapter is expected to write logic to load the state of their trained model(s) from the same location that the model was saved/serialized to in the save method, and also initialize the model.
16
17        Container models:
18            This is the method that Foundry will call after your container has been launched as a sidecar to this ModelAdapter. The author of this ModelAdapter is expected to use the contents of the container_context to initialize any class variables the adapter might need. For example, users will often extract the relevant service URIs to send POST requests to the container within #run_inference.
19
20        Externally hosted models:
21            This is the method that Foundry will call when your model adapter is initialized. The author of this ModelAdapter is expected to write logic to initialize and persist a connection to their externally hosted model as well as other required model configuration.
22
23
24        :param state_reader: A ModelStateReader object that can be used to read model files.
25        :param container_context: This is only provided for container backed models, and defaults to None. The container context
26          includes a mapping from container name to service URIs and the shared directory mount path.
27        :param external_model_context: This is only provided for externally hosted models, and defaults to None. The external_model_context includes references to the configuration a user defines when creating an externally hosted model that uses this Model Adapter.
28
29        :return: An instance of a ModelAdapter.
30        """
31
32    def save(self, state_writer: pm.ModelStateWriter) -> None:
33        """
34        This is the method that Foundry will call to serialize your model adapter. This method is only required if this ModelAdapter is being used to wrap a newly trained or refit model in Foundry.
35
36        The author of this ModelAdapter is expected to write logic to save the state of their trained model(s) and relevant metadata to a ModelStateWriter.
37
38        :param state_writer: The ModelStateWriter object to which the model is serialized and saved.
39        """
40
41    @classmethod
42    def api(cls) -> pm.ModelApi:
43        """
44        This defines the expected input and output data structures of this model.
45
46        :return: The ModelApi object for the model
47        """
48
49    def run_inference(self, inputs, outputs) -> None:
50        """
51        This method will be called with the relevant input and outputs defined in the ModelAdapter.api method.
52
53        Runs inference on the associated model.
54
55        :param inputs: A namedtuple of the inputs defined in the ModelAdapter.api method.
56        :param outputs: A namedtuple of the outputs defined in the ModelAdapter.api method.
57          Outputs should be written to in the run_inference method.
58        """
59
60    def predict(self, *args, **kwargs) -> Union[Tabular, Parameter]:
61        """
62        This method is used to perform inference on a multi (>=1) input to single output (tabular or parameter) model.
63        The inputs are expected to be written into the signature of this method by the name that defines them in the api() method. The resulting output is returned by this method.
64
65        Note that the run_inference() method is not defined by the user if predict() is used.
66        """

Model save() and load()

The palantir_models.serializers library provides many default serializers that can be used for model serialization (save) and deserialization (load) for common modeling frameworks.

In some cases, users may want to implement custom logic for model serialization or deserialization. This may be necessary when, for example, there is no default serializer available for a modeling framework you are using or when you require manual control over which models are loaded into memory at any given time.

In these more complex cases, please see the implementation of save and load below.

Save

ModelStateWriter

A ModelStateWriter is provided to the ModelAdapter.save method so a ModelAdapter can save/serialize model artifacts to Foundry storage.



Copied!

2
3from io import IOBase
4from typing import ContextManager
5
6class ModelStateWriter:
7    def open(self, asset_file_path: str, mode: str = "wb") -> ContextManager[IOBase]:
8        """
9        Open a file-like object to serialize model artifacts and parameters.
10        """
11
12    def put_file(self, local_file_path, asset_file_path=None):
13        """
14        Put a local file in this model's repository.
15        :param asset_file_path: If provided, the file will be placed at this path in the repository.
16            Otherwise the file will be placed in the root directory of the repository.
17        """
18
19    def put_directory(self, local_root_path, asset_root_path = "/"):
20        """
21        Put a local directory and its contents into this model's repository.
22        :param asset_root_path: The path relative to the root path of the repository to place this directory.
23        """

The save() method is called whenever a model adapter is published in a transform via model.publish(), upon which the ModelStateWriter's contents are packaged into a zip file that is persisted in an artifacts repository referenced by the newly-created model version.

The load() method is called whenever a model adapter is instantiated (via ModelInput in a transform, or launching a live or batch deployment that is backed by a model). This method then accesses the same artifacts repository that ModelStateWriter writes to, and provides access to its contents via ModelStateReader. load() is called before any transforms or inference logic is executed.

Example: ModelStateReader

The following example loads a model file and returns and instance of the model adapter initialized with it.

Copied!
1from palantir_models.models import ModelAdapter, ModelStateReader
2
3class ExampleModelAdapter(ModelAdapter):
4    def __init__(self, model):
5        self.model = model
6
7    @classmethod
8    def load(cls, state_reader: ModelStateReader):
9        with state_reader.open("model.pkl", "rb") as model_infile:
10            model = pickle.load(model_infile)
11        return cls(model)
12

When used in conjunction with the save() method shown above, this method would retrieve the same model.pkl object that was persisted in the save() method. A ModelStateReader object is also provided to containerized models, and is backed by the user's uploaded zip file.

ContainerizedApplicationContext

The ContainerizedApplicationContext is optional and will be provided to the ModelAdapter.load method if, and only if, the model is backed by a container image or images. The context object includes a shared directory mount path and a mapping from container name to their service URIs. Each container can have multiple service URIs as it is valid to have multiple open ports.

Copied!
1# Note that this class type does not need to be imported within an authored adapter
2class ContainerizedApplicationContext:

4        """
5        Mapping from individual container name to list of service URIs the container provides.
6        """
7
8    def shared_empty_dir_mount_path(self) -> str:
9        """
10        The mount path of a shared empty directory that is available inside all containers and to the model adapter.
11        The directory is readable and writable by containers and by the model entrypoint.
12        """

ExternalModelContext 是可选的，当且仅当模型为外部托管模型时，该对象才会被传递给 ModelAdapter.load 方法。此上下文对象包含一个代表该外部托管模型的对象，以及连接该模型所需的、由用户定义的已解密机密信息映射表。

Model inputs


The ModelInput type contains input types that can be defined in the ModelAdapter.api method. Model adapters support the following input types:


- 表格
- 参数
- 文件系统
- 媒体引用 [Beta]
- 对象
- 对象集


Beta
媒体引用支持目前处于 beta 开发阶段，仅支持在 Python 转换中进行推理，且您的租户环境可能无法使用该功能。在“建模目标”应用中，包含模型的媒体引用不支持自动评估、批量部署或实时部署。在活跃开发期间，相关功能可能会发生变更。

表格输入

TabularInput 用于指定传递给 model.transform() 的输入应为表格类型。在该模型适配器的推理逻辑中，此输入的类型将为 api() 方法中指定的 df_type 参数。必要时，系统会执行适当的类型转换。以下为允许的表格类型：

  1. TransformInput
  2. Pandas DataFrame
  3. Spark DataFrame
    • 注意：适配器 API 中仅支持指定 df_type=DFType.SPARK 的表格输入使用 Spark DataFrame。对于指定 DFType.PANDAS 的表格输入，不支持向 Spark DataFrame 进行转换。

列类型

TabularInputs 还需指定一系列 ModelApiColumn，用于描述表格输入预期的列模式。type 参数可取以下值之一：

  1. str
  2. int
  3. float
  4. bool
  5. list
  6. dict
  7. datetime.date
  8. datetime.time
  9. datetime.datetime
  10. typing.Any
  11. MediaReference [测试版]

同时，也接受 typing 库中的 List 和 Dict 类型别名。例如：List[str]；Dict[str, float]。在将模型发布为函数时，请查阅有关使用这些类型的具体要求。

参数输入

ParameterInput 用于指定传递给 model.transform() 的输入应为参数类型。参数属于固定值输入，其类型由 type 参数指定。参数输入支持以下类型：

  1. str
  2. int
  3. float
  4. bool
  5. list (*)
  6. dict (*)
  7. typing.Any (*)
  8. 用于 NumPy n 维数组 的 palantir_models.NDArray

参数输入还可指定一个 default 值；若未向 model.transform() 提供对应于模型适配器 api() 方法中所定义参数输入的值，则使用此默认值。

在将模型发布为函数时，请查阅有关使用这些类型的具体要求。

文件系统输入

FileSystemInput 用于指定传递给 model.transform() 的输入应为文件系统类型。FileSystemInput 仅支持以 TransformInput 作为输入类型。

媒体引用输入

MediaReferenceInput 用于指定传递给 model.transform() 的输入应为媒体引用。媒体引用须为 str 类型，且包含完整的媒体引用对象定义。模型适配器会将此媒体引用字符串转换为 MediaReference 对象，该对象包含与所引用的媒体项进行交互的方法。

对象与对象集输入

对于 Object 或 ObjectSet 输入，需在模型适配器 API 中定义输入时指定对象类型。该对象类型将从为所选本体生成的 本体 SDK 中导入。

执行模型推理时，可通过以下方式将对象或对象集传递给模型：

API 强制校验

作为一项通用原则，模型适配器 API 在推理阶段并不进行严格校验，尽管对于实时推理，尤其是对于强类型的函数而言，校验通常更为严格。

参数类型始终会接受强制校验，任何传递给 model.transform() 且不符合指定类型的参数输入都将抛出错误。

模型适配器允许在推理阶段的表格输入中传递 API 未指定的额外列。Foundry 在将输入传递给 predict 方法之前，不会丢弃这些额外列。这对于确保模型适配器与评估功能正常配合至关重要，正如《模型适配器 API 指南》中所述。在批量推理场景下，即使 API 中指定的某些列在输入中缺失，适配器也不会抛出错误。MediaReference 类型（目前处于 Beta 阶段）是一个例外。它要求列中的每个元素都是媒体引用字符串。在批量推理过程中，无论是批量推理还是实时推理，它都会在将每个元素传递给该模型适配器的推理逻辑之前，将其转换为 MediaReference 对象。

然而，在实时推理场景下，模型输入会被强制转换为其声明的类型（如果已定义），以确保类型安全。这适用于表格输入中的所有列。该行为的一个副作用是，如果输入数据中未找到具有指定类型的列，或者数据无法转换为指定类型，系统将抛出错误。

对于对象输入

  1. 在模型适配器 API 的 Object 输入中指定的 OntologyObject 类型的实例。
  2. 指定对象类型的主键。

对于对象集输入

  1. 在模型适配器 API 的 ObjectSet 输入中指定的 OntologyObject 类型的 ObjectSet 实例。
  2. 指定对象类型的对象集 RID。

模型推理

predict() 方法

对于输出为单个表格或参数类型的模型，可以使用 predict() 方法替代 run_inference() 方法来定义模型适配器的推理逻辑。该方法的参数名将与模型适配器 api() 方法中定义的输入对象名称一致。参数顺序无需与定义时保持一致，但名称必须匹配。

### `run_inference` 方法

run_inference() 方法

对于多输出模型或需写入文件系统的模型，必须通过 run_inference() 方法定义自定义推理逻辑。该方法接收两个参数：inputs 和 outputs。这两个参数均为 NamedTuples，其名称与 api() 方法中定义的输入和输出的 name 参数相对应。
