Asana

使用 CData JDBC 驱动程序将 Foundry 连接到 Asana，以将数据导入到 Foundry 数据集。

支持的功能

功能状态
探索🟢 通常可用
批量导入🟢 通常可用
增量🟢 通常可用

数据模型

CData 驱动将 Asana 数据建模为关系数据库中的一系列表格，可以使用标准 SQL 语句进行查询。有关可用表格及其内容的信息，请查看 CData 文档中的 Asana 表 ↗。

设置

  1. 打开 数据连接 应用程序，并在屏幕右上角选择 + 新数据源。
  2. 从可用的连接器类型中选择 Asana。
  3. 按照后续配置提示继续使用以下部分的信息设置您的连接器。

了解有关 在 Foundry 中设置连接器 的更多信息。

认证

对 Asana 的认证是通过个人或服务访问令牌完成的。

要生成服务账户或个人访问令牌，请按照 Asana 文档中的指南进行操作：

  • 服务账户令牌 ↗
    • 使用服务账户令牌创建持久且耐用的源，该源不直接关联到单个用户。
  • 个人访问令牌 ↗
    • 一些组织不允许用户创建访问令牌。如果您在获取访问令牌时遇到问题，请联系您的 Asana 管理员。

网络连接

Asana 连接器需要对 app.asana.com 的 443 端口具有网络访问权限。请确保为源配置了 出站策略。

增加日志详细程度可能会导致敏感信息被保存到Foundry构建日志中。在分享这些日志之前，请根据需要擦除任何密码。查看CData文档，了解有关不同 详细程度级别 ↗ 的更多信息。

配置Asana同步

Asana连接器允许进行高级同步配置，以确定哪些数据以及多少数据将被带入Foundry。

在浏览了您可用的表格并将其添加到连接后，导航到编辑同步。从左侧的同步面板中，找到您要配置的同步项，并选择右侧的>。

选项是否必需？描述
Table是选择应从 Asana 复制到 Foundry 的表格。参见 数据模型 了解支持的表格。
Column selection否选择要同步到 Foundry 的列的子集。请查看下面的 列选择 部分以获取更多信息。
Row filters否启用并添加过滤器，根据其列值删除不符合条件的行。请查看下面的 行过滤器 部分以获取更多信息。
Limit否将 SQL limit 子句添加到底层查询中。此设置可用于限制单次运行中同步到 Foundry 的行数。
Incremental否启用以小批量将数据同步到Foundry。有关更多信息，请查看下面的增量部分。

增量

通常，同步会从目标表中导入所有匹配的行，无论同步之间数据是否发生变化。相比之下，增量同步会保留关于最近一次同步的状态，并且只从目标表中导入新的匹配行。

当从Asana导入大型表时，可以使用增量同步。要使用增量同步，表中必须包含一个严格单调递增的列。此外，要读取的表中必须包含以下数据类型之一的列：

  • DATE
  • TIMESTAMP

数值列类型：

  • int
  • TINYINT
  • SMALLINT
  • INTEGER
  • long
  • BIGINT
  • NUMERIC
  • NUMBER
  • DECIMAL
  • DEC

字符串列类型：

  • string
  • VARCHAR
  • CHAR
  • NVARCHAR
  • NCHAR
  • LONGNVARCHAR
  • LONGVARCHAR

增量同步需要以下配置：

选项是否必需？描述
列是选择用于增量导入的列。如果表中不包含任何支持的列类型，则下拉菜单将为空。
Initial value是开始同步数据的值。

示例：一个5TB的表包含数十亿行，我们希望将其同步到Asana。该表有一个名为id的单调递增列。可以配置同步，每次使用id列作为增量列，初始值为-1，并设置5000万行的限制。

当首次运行同步时，将把包含id值大于-1的前5000万行（按id升序排列）导入Foundry。例如，如果此同步已经运行过几次，且在上一次同步中导入的最大id值为19384004822，则下一次同步将从第一个大于19384004822的id开始，导入接下来的5000万行，依此类推。

列选择

选择要同步到Foundry的列。如果没有选择任何列，则所有列都将被同步到Foundry。

所有列名必须以下划线或英文字母开头。名称只能包含下划线、英文字母或数字。

以下示例是有效的列名：

  • _colum_name
  • a_column_name_123

以下示例为无效的列名：

  • 1ColumnName; 以数字开头。
  • a-column-name*; 包含连字符和星号。
  • å-column-name; 包含非英文字符。

行筛选

可以添加筛选条件以减少同步的行数，或单独选择要同步到Foundry的行。要配置行筛选，表预览必须成功。在配置行筛选之前，我们建议使用界面底部的 Preview of inbound data 部分刷新预览。可以刷新预览以验证所添加的筛选条件的行为。

过滤器配置由一组条件组成，每一行必须通过这些条件才能同步到Foundry。每组条件可以评估为匹配，如果all条件匹配，或者如果at-least one条件匹配。条件组可以嵌套，以对要导入的内容提供更精细的控制并实现高度灵活性。这些相当于标准SQL中的AND和OR语句。

一个条件本身是对Asana表中某一列的值与可以在界面上输入的值进行简单比较。例如，条件CreatedAt等于12/04/2021将只导入CreatedAt列的值为12/04/2021的那些行。该列的数据类型决定了支持的比较操作。

比较操作

以下操作在过滤器中被支持：

  • EQUAL TO
  • GREATER THAN
  • LESS THAN
  • GREATER THAN OR EQUAL TO
  • LESS THAN OR EQUAL TO
  • NOT EQUAL TO
  • IS NULL
  • IS NOT NULL

这些比较操作的行为与标准SQL中的行为相同。

字符串类型列还支持Like和Not like运算符，这些运算符使用标准的SQL-92语法。
