导入、导出和复制应用程序

这些工作流不适用于共享或复制复杂的应用程序。若要跨 Foundry 实例共享应用程序，请改用 Marketplace 集成；若要在同一 Foundry 注册范围内复制应用程序，请使用另存为。

导入 Slate 应用程序

要导入 Slate 应用程序，请在编辑模式下选择文件下拉菜单，然后选择导入 JSON以打开上传对话框。从本地计算机中选择一个 JSON 文件，将其作为 Slate 文档上传。如果 JSON 文件中定义的永久链接是全新且唯一的，系统会提示您选择一个位置来创建新的 Slate 文档。如果 Slate 文档的 JSON 中定义的永久链接在 Foundry 中已存在，系统将使用导入的 JSON 文件内容创建现有 Slate 文档的新版本。

导入应用程序时，只会导入应用程序本身的逻辑。图像、文件、对象、数据集和数据源等支持资源不会自动创建。如果引用的数据源不可用，JSON 导入将会失败。对其他不存在的资源的引用会在打开应用程序时导致错误，但您可以在编辑模式下通过手动重新指向这些资源来解决此类错误。对于使用 Ontology SDK 的应用程序，导入后可能还需要手动重新生成 OSDK 包。

导出 Slate 应用程序

要导出 Slate 应用程序，请在编辑模式下从文件下拉菜单中选择导出。这会将包含 Slate 应用程序配置的 JSON 文件下载到您的本地计算机。此配置包括小部件、函数、查询逻辑、事件、变量和样式。它不包含 Foundry 中未包含在应用程序本身中的其他资源，例如全局样式表、对象、数据源、数据集、Actions、图像、视频、JavaScript 以及导入到代码沙盒中的 CSS 库。

复制 Slate 应用程序

在编辑模式下，从文件下拉菜单中选择另存为选项，即可复制 Slate 应用程序。此时会弹出一个窗口，要求您为复制的应用程序输入新的位置和名称。新应用程序仅包含原始应用程序的最新版本，并将其存储为 v1。对于使用 Ontology SDK 的应用程序，现有的 OSDK 包是从源应用程序复制而来的，而不是重新生成的。

复制 Slate 小部件

您还可以在 Slate 应用程序中重用小部件。要复制小部件，请先在 Slate 应用程序左侧的“布局”窗格中将其选中。然后，按下 Cmd+C (macOS) 或 Ctrl+C (Windows)。最后，切换到您的第二个 Slate 应用程序，按下 Cmd+V (macOS) 或 Ctrl+V (Windows) 粘贴所复制的小部件。

覆盖 Slate 应用程序

另存为选项也允许您覆盖现有的 Slate 应用程序。您可以利用此功能来暂存或提升应用程序。要覆盖 Slate 应用程序，请在文件浏览器中选择一个现有应用程序作为保存应用程序的位置。覆盖应用程序时，将创建一个新版本，并复制当前应用程序中所选版本的内容。
