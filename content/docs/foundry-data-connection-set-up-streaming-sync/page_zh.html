设置流式同步

同步是一项从源读取特定数据并将其接入 Foundry 的任务。例如，若源是一个包含多张表的关系型数据库，你可以配置同步，将其中某张表接入 Foundry。

流式同步与非流式（即批处理或增量）同步同步类似，但存在一些差异。主要差异在于：批处理或增量同步会定期运行，而流式同步持续运行，旨在以极低的延迟将数据拉取到 Foundry 中。

接下来，我们将探讨创建同步所需的步骤：

  1. 定义要从源同步的数据。
  2. 定义 Foundry 中发送数据的同步位置。
  3. 配置流式同步。
  4. 运行流式同步。

在本教程中，我们将使用 Kafka 源来设置同步。

第 1 部分：定义数据

首先，确定要同步到 Foundry 中的数据。在“数据连接”中选择你的流式源，然后选择右上角的可用操作：

  • 探索并创建同步：如果源类型支持源探索，则会显示此选项，允许在创建同步的同时探索数据源。
  • 创建同步：如果源类型不支持源探索，则会显示此选项。

探索并创建同步

如果源类型支持源探索，你将进入“数据连接”中的探索源页面，该页面会显示可供同步的数据。探索视图界面取决于所使用的源类型。例如，Kafka 源探索允许你查看 Kafka 代理上存在的主题 ↗，并预览这些主题中包含的数据。

在 Kafka 探索视图中，你可以通过页面左侧的列表查看现有主题。

选择某个主题后，即可预览该主题中的数据样本。

第 2 部分：定义同步位置

接下来，需要确定将同步的数据集保存在 Foundry 中的位置。基于项目级权限，数据集的位置将决定谁有权访问最终生成的数据集。

建议将同步的数据集与其源保存在同一个项目中，以便两者具有相同的权限；在构建数据管道时，保持数据集与源权限的一致性会大有裨益。进一步了解数据管道推荐的项目结构。

选择同步位置后，点击右上角的创建流式同步。

第 3 部分：配置流式同步

此时，你将进入“数据连接”的同步创建页面，在此可以定义同步的源特定配置和核心流式配置。

  • 源特定：位于配置页面顶部，这些选项取决于你的源类型，用于配置传递给你所连接的特定源的参数。
  • 核心流式：位于源特定配置下方，这些选项是所有流式同步通用的。核心配置包括吞吐量、架构和同步目标。

接下来，为你的流选择吞吐量。吞吐量决定了将要创建的分区数。选择更多的分区数可以获得更高的吞吐量。选择正常吞吐量将允许该流达到最高 5 MB/s 的速度。

然后指定输入数据的架构，默认情况下架构会从源推断，但如有必要也可以覆盖。

配置好同步任务后，在右上角选择创建同步。

同步任务创建完成后，系统将跳转至概览标签页。

第 4 部分. 运行同步

现在，您可以运行同步任务了。选择概览标签页以查看新建同步任务的摘要信息，包括输出数据集、存储位置以及可用操作。

点击开始，将外部流中的数据同步至 Foundry。

要查看流数据，请导航至创建同步任务时配置的流，进入流预览页面。您将看到来自 Kafka 主题的记录正在流入该流中。

后续步骤

成功运行同步任务后，您可以进一步了解如何调试异常流、如何通过推送式摄入将数据推送到流，或如何将流与本体集成。
