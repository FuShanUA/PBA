如何开始一项新分析？

在 Foundry 中，每位用户都有一个名为 你的文件 的专属文件夹。用户可以在此文件夹中处理数据进行原型设计，并与其他用户共享结果。在该文件夹内，用户可以创建他们的第一个分析：

  1. 前往工作区导航栏的 文件 部分。
  2. 进入 你的文件 标签页。
  3. 选择 +新建 并从下拉菜单中选择 分析 来创建一项新分析。
  4. 新分析生成后，选择 +创建新路径。
  5. 在相应的文件夹中选择您想要分析的数据集。

如何在平台中查找相关数据？

发现数据的主要方法有两种：

  • 搜索： Foundry 在页面左侧的工作区侧边栏中提供了一个全平台搜索工具。该工具会搜索平台中的所有资源，当您知道数据集名称时，这是查找数据的绝佳方法。请注意，搜索工具可用于查找平台中的任何资源，包括电子表格、Contour 分析和代码工作簿。

  • 数据目录: Foundry 数据目录 包含经过清理和治理、随时可供业务分析师和数据科学家使用的数据集。如果您想了解平台中已存在哪些数据，数据目录 是一个绝佳的起点，可直接从主页访问。您可以通过选择工作区侧边栏中的 主页 返回主页。

如何与同事分享分析？

与同事共享资源意味着他们必须拥有您正在工作的项目的访问权限。选择 分享 并发送共享链接，或者直接将同事添加到该资源中以便自动通知他们。如果您的同事收到 权限拒绝 错误，他们需要申请访问权限。一旦同事对该项目的访问权限获得批准，他们就能通过电子邮件查看您分享的分析。

如何创建用于分析的初始数据集？

鉴于数据在 Foundry 平台中通常以原始形式提供，在开始构建分析之前，了解如何过滤数据至关重要。数据过滤能让您专注于对分析至关重要的要素，免受无关数据的干扰。

  • 排序与过滤：直方图数据过滤：
    • 在 Foundry 中，排序与过滤是最常用的功能之一。您可以通过直方图面板查看给定列中的所有不同选项，然后选择要处理的具体类别。
  • 属性数据过滤：
    • 如果只想保留符合设定条件的数据，请使用 KEEP。
    • 如果只想排除符合设定条件的数据，请使用 REMOVE。
  • 多重数据过滤：
    • 如果要筛选同时满足多个条件的数据，请使用 AND MATCHING。
    • 如果要筛选满足多个条件、但无需同时满足/无需在同一行数据中满足的数据，请使用 OR MATCHING。
  • 调整过滤条件：
    • 当用户针对特定过滤条件（例如 carrier_code = DL）创建新分析时，其他用户只需将过滤条件更改为符合自身用例的条件（例如 carrier_code = UA），或彻底移除该过滤条件以获取全局分析结果，即可轻松复现其分析过程。

为便于分析大型数据集，分析操作默认应用于整列。如果只想对较小的行范围运行分析（类似于在 Excel 中选择特定单元格区域），请先将数据过滤至目标行，再应用相关操作。

如需了解更多过滤选项，请查阅我们的《过滤数据》文档。

如何更改分析视图？

更改分析视图共有四种可用选项。您可以执行以下操作：

  • 按升序或降序对列进行排序 (SORT)

  • 重排 (REORDER)列

  • 移除 (REMOVE)列

  • 添加 (ADD)列：请参阅《Foundry 中的常用 Microsoft Excel 分析等效功能》中的 VLOOKUP 部分

  • 创建自动化 Notepad 文档：您可以选择将 Contour 分析输出添加到自动化 Notepad 中，以便在高管摘要中展示数据。该报告会根据 Foundry 中刷新的数据自动更新，无需重新创建相同的报告。

如何应用计算？

要在 Contour 中执行新计算，您需要在表达式面板中选择添加列 (Add a Column)。但是，Foundry 执行的是列级操作，而非单元格级操作。它不是通过公式将 A1 乘以 B1 并将结果返回到单元格 C1，而是将 column1 乘以 column2（将 column1 和 column2 中对应的每行条目相乘）并返回 column3。

Foundry 中的常用 Microsoft Excel 分析等效功能有哪些？

以下是 Microsoft Excel 中一些最常用的函数及其在 Contour 中的表达式等效功能。您可以按照《如何应用计算？》中讨论的方法应用这些计算。

  • Excel: IF(logical_test, value_if_true, [value_if_false])

    • Foundry: 
      CASE
    WHEN logical_test THEN value_if_true
    ELSE value_if_false
END
    • 示例：如果我想创建一个列，在航班起飞于 JFK 时返回 yes，否则返回 no，表达式如下： 
      CASE
    WHEN "origin" = 'JFK' THEN 'yes'
    ELSE 'no'
END

  • Excel: CONCAT(cell1, [cell2],…)

    • Foundry: CONCAT("col1", ["col2"],...)
    • 示例：如果我想通过拼接 timestamp 和 order_ID_number 列来创建一个包含每个订单唯一键的列，表达式如下：
      • CONCAT("timestamp","order_ID_number")

  • Excel: VALUE(text)

    • Foundry: CAST("col1" AS DOUBLE)
    • 注意：您可以通过替换表达式中的 DOUBLE 类型，将列数据类型转换为 STRING、INTEGER、BOOLEAN、DATE、TIMESTAMP 或 LONG
    • 示例：如果我想对某些列执行乘法运算，但必需的 cost 列却被归类为字符串，表达式如下：
      • CAST("cost" AS DOUBLE)

  • Excel: LEFT(text, [num_chars])

    • Foundry: SUBSTRING("col1", num2, num3)
    • 注意：num2 是起始索引，num3 是子字符串的长度
    • 示例：如果我想从 airport_display_name 列中提取括号内的字母，例如 [ALB] Albany International + Albany, NY、[AZA] Phoenix - Mesa Gateway + Phoenix, AZ、[CLT] Charlotte Douglas International + Charlotte, NC，表达式将如下所示：
      • SUBSTRING("airport_display_name", 2, 3)，并将返回一个包含 ALB、AZA 和 CLT 的列。

阅读《表达式看板》和《支持的表达式语法》文档以获取更多信息。

  • Excel: VLOOKUP(value, table, col_index, [range_lookup]) & 从其他数据集中添加列
    • Foundry: JOIN 看板。
    • 连接看板允许您将当前工作数据集与另一个数据集连接起来，并将匹配的结果合并到您的数据中。 示例：您正在处理 flights 数据集，并希望从 aircraft 数据集中添加 manufacturer 和 number_of_seats 列。

有哪些方法可以检查我的分析结果？

您可以添加其他看板来检查生成的数据集。

  • 表格看板： 在分析后插入表格看板，您可以快速检查新增的列是否正确，或者前置看板的逻辑是否达到了预期效果。

在 Contour 中保存数据集与在 Code Repositories 或 Code Workbook 中创建数据集有何区别？

从 Contour 分析中保存数据集的过程，与从代码仓库或 Code Workbook 创建数据集的过程基本一致——其逻辑都会被转换为一系列 Spark 转换操作，在集群中执行，最终保存为存储在分布式文件系统中的数据集。将分析结果保存为新数据集时，没有行数限制。不过，数据规模越大，保存所需的时间就越长，因为底层计算的成本会更高。需要注意的是，从 Contour 导出数据时存在行数限制，这与将路径保存为数据集是两回事。

我尝试导出时遇到了 100,000 行的限制，这正常吗？

是的，这属于正常情况。从 Contour 导出数据时有 10 万行的限制。如果需要导出更多数据，您可以将 Contour 的结果保存为数据集，然后在该数据集预览页面的操作下拉菜单中进行下载。

根据合作伙伴的具体要求，不同 Foundry 环境中 Contour 导出和数据集下载的限制可能会有所不同。

能否自动更新 Contour 分析路径？

目前无法自动更新 Contour 的分析路径，必须手动完成。不过，您可以为分析生成的最终数据集设置调度计划。保存数据集后，您可以打开数据集预览页面，并在操作下拉菜单中选择管理调度。随后，该结果数据集将根据您配置的调度计划进行构建。

数据透视表未显示所有数据

数据透视表预览不会显示表内的所有数据。透视表会对整个数据集计算聚合结果，并将输出精简为前 100 列或 10,000 个值，以防止浏览器性能下降。要获取这些大型透视表的准确结果，你需要切换到透视数据。你可以在《透视表文档》中阅读更多相关信息。

要进行故障排查，请执行以下步骤：

  1. 尝试切换到透视数据，这将强制 Contour 计算整个数据集。
  2. 在表看板中查看数据。请注意，透视看板仍将不完整。
  3. 如果可能，请在透视表上方插入一个过滤看板，以缩减你传入的数据量（这也应能整体提升性能），但同样地，这并不能改变透视表看板将仅对透视表前 10,000 个单元格进行计算的事实（出于性能原因）。
  4. 要完整导出数据，请选择切换到透视数据，然后使用导出看板，或使用当前路径末尾的导出选项。

在透视表中发现重复列

我的透视表由于存在重复列而无法计算。这通常是因为列名除了大小写不同外完全相同。

要进行故障排查，请执行以下步骤：

  1. 检查透视表“列”部分中的任何列是否包含除大小写外完全相同的值（例如，Test 和 test）。Foundry 数据集的列名不区分大小写，因此当列被透视时，Test 和 test 列会被视为重复项。
  2. 将所有此类值映射为统一的大小写格式，这样在透视列时就不会发生冲突。

常规性能问题

你的 Contour 运行缓慢，并且你想找出导致性能下降的原因。

要进行故障排查，请执行以下步骤：

  1. 首先，检查确定你的分析中使用的输入数据集是否使用了 Parquet 或 Avro 文件；如果没有，请确保你使用的是数据集恰当且干净的版本。

  2. 检查你是否正在使用存储为 CSV 文件的原始导入数据集，这种格式性能极差。这是导致此问题的最常见原因。

  3. 检查输入数据集的分区。如果分析中使用的数据集分区不合理，将导致性能下降。

  4. 要检查输入数据集中的文件大小，请转到 数据集 → 详情 → 文件 → 数据集文件。

  5. 每个文件应至少为 128 MB。如果它们太小或太大，你需要重新分区。

  6. 如果你的路径非常长，你应该物化中间数据集，并创建以这些新物化的数据集为起点的新路径。这将减少逻辑执行中的冗余，因为每个看板都会执行创建该看板所需的完整查询路径（例如，所有先前看板中使用的转换逻辑，如果有的话）。

  7. 考虑减少分析中的路径数量。特别是在使用路径概览屏幕时，这会降低浏览器的性能。

如需进一步阅读，请查阅我们的《Contour 分析性能优化》文档。

count_distinct() 在表达式看板的窗口函数中失败

由于 Spark 的限制，count_distinct() 函数无法在窗口函数中使用。请查阅《官方 Spark 文档 ↗》。

你可以在透视表看板中实现相同的功能（取决于你要使用的窗口逻辑），该看板提供了唯一计数聚合选项。你可以将“窗口”定义为行/列组合，然后为每个交叉点生成唯一计数。
