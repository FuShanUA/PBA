配置受限制视图支持的对象类型

受限制视图（RVs） 可以对本体论数据进行行级访问控制。这允许比仅仅授予整个数据集或某种类型的所有对象的访问权限更细粒度的访问控制。

受限制视图类似于数据集，但会限制对数据集中 特定行 的访问。受限制视图在数据集级别进行配置，本体论对象将继承受限制视图策略中定义的细粒度权限。

使用受限视图来支持对象类型

用受限视图支持一个对象类型，可以控制用户可以看到的具体对象。例如，如果一个用户符合某个策略的要求，并且可以看到受限视图中的特定行，那么他们将能够看到相应的本体论对象。

要查看由数据集支持的对象类型中的对象，您还必须能够查看该数据集。

  • 要查看由对象存储V1（Phonograph）中的受限视图支持的对象类型中的对象，您必须能够查看该对象类型本身；您不一定需要看到受限视图。相反，您只需满足该受限视图的标记及其该行的读取策略即可。
  • 在对象存储V2中，您必须能够看到受限视图才能看到由该受限视图支持的对象类型中的对象。

当限制特定对象时，仅在本体论管理器中选择作为对象类型的数据源的受限视图。

本体论编辑

对本体论对象的访问可能受到用户编辑的影响，因为有可能编辑对象安全中引用的属性。在这些情况下，用户可能能够在支撑受限视图中看到一行数据，但无法在本体论中看到对应的对象，或者相反。

安全配置

Ontology 管理器中的数据源选项卡将显示额外的配置选项，用于编辑细粒度策略。 细粒度策略部分允许您为此类对象配置权限。

只能为使用Object Storage V1且不包含仅允许通过操作进行编辑的对象类型配置细粒度编辑策略 选项已选中。对于所有其他对象类型，编辑权限通过操作类型来控制对象类型。 了解有关操作权限的更多信息。

您可以配置这些策略以适应您希望用户根据其属性（如对象的属性）仅查看或编辑特定对象的情况。例如，您可能只希望来自 Europe（位于 region 列中）的用户查看和编辑欧洲对象，这可能与受限视图的策略不同。

有三种策略可以定义谁可以访问对象上的属性：

  • 读取： 该策略定义谁可以查看受限视图上的所有属性。使用上面的例子，这可能是一个策略，将 region 列与用户属性（Europe）进行比较，以确定用户可以看到哪些对象。
  • 编辑属性： 此策略定义了允许更新任何配置为可回写且未用于任何细粒度权限策略定义的属性的人员。使用上面的示例，此策略将控制谁可以编辑 name 属性，但不会控制谁可以编辑 region 属性，因为 region 属性被用于该策略。
  • 编辑策略属性： 此策略定义了允许更新任何配置为可回写且也用于任何细粒度权限策略定义的属性的人员。使用上面的示例，此策略将控制谁可以编辑 region 属性。

如果在对象类型注册到对象存储 V1（Phonograph）之后更改了查看策略，则必须通过对象类型 数据源 选项卡中的 Phonograph 部分的 更新 按钮来更新注册。 . 如果注册信息未更新，可能根据之前注册的策略提供受限视图的最新数据。对象存储V2默认提供自动策略传播功能。
