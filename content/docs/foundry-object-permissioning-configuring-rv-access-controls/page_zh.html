配置受限制视图支持的对象类型

受限视图（RVs） 可以对本体论数据进行行级访问控制。这允许比仅仅授予整个数据集或某种类型的所有对象的访问权限更细粒度的访问控制。

受限视图类似于数据集，但会限制对数据集中的 特定行 的访问。受限视图在数据集级别上进行配置，而本体论对象将继承受限视图策略中定义的细粒度权限。

使用受限视图支持对象类型

使用受限视图支持对象类型将控制用户可以看到的具体对象。例如，如果一个用户符合策略的要求，并且可以查看受限视图中的特定行，那么他们将能够看到相应的本体论对象。

要查看由数据集支持的对象类型中的对象，您还必须能够查看该数据集。

  • 要查看Object Storage V1（Phonograph）中受限制视图支持的对象类型中的对象，您必须能够查看该对象类型本身；您不一定需要看到受限制的视图。而是，您只需要满足该受限制视图的标记及其对该行的读取策略。
  • 在Object Storage V2中，您必须能够看到受限制视图才能看到由该受限制视图支持的对象类型中的对象。

当限制特定对象对用户的访问时，请仅在Ontology Manager中选择受限制视图作为对象类型的后备数据源。

本体论编辑

对本体论对象的访问可能因用户编辑而受到影响，因为有可能编辑对象的安全性中引用的属性。在这些情况下，用户可能能够在支持受限视图的行中看到内容，但无法在本体论中看到相应的对象，或者反之。

例如，考虑一个 Ticket 对象类型，其数据如下，其中受限视图上的策略要求您是 Assignee 列中的组成员才能看到该行。

Ticket IDTitleAssignee
101Ticket Onepalantir-support

如果对该对象的 Assignee 进行更改，将其更改为 customer-support，那么不是 customer-support 成员的 palantir-support 成员将失去对本体论中该对象的访问权限。然而，他们仍会看到受限视图中的该行，该行不受此操作的影响。不是 palantir-support 成员的 customer-support 成员将获得对本体论对象的访问权限，但仍无法看到受限视图中的该行。

安全配置

数据源选项卡在本体论管理器中将显示额外的配置选项以编辑细粒度策略。 细粒度策略部分允许您为此类对象的编辑配置权限。

只能对使用对象存储 V1 的对象类型配置编辑的细粒度策略，这些对象类型未选择 仅允许通过操作进行编辑 选项。对于所有其他对象类型，编辑权限由编辑对象类型的动作类型控制。 了解有关操作权限的更多信息。

您可以配置这些策略，以适应您希望用户根据其属性（如对象的属性）查看或编辑特定对象的情况。例如，您可能只希望来自 欧洲（位于 region 列中）的用户查看和编辑欧洲对象，这可能与受限视图的策略不同。

有三种策略可以定义谁可以访问对象上的属性：

  • 读取：此策略定义了谁可以查看受限视图上的所有属性。使用上面的示例，这可能是一项策略，该策略将region列与用户属性（Europe）进行比较，以确定用户可以看到哪些对象。
  • 编辑属性：此策略定义了谁被允许更新任何配置为可写回的属性，这些属性未用于任何细粒度权限策略定义中。使用上面的示例，此策略将控制谁可以编辑name属性，但不能控制谁可以编辑region属性，因为region属性已用于策略中。
  • 编辑策略属性：此策略定义了谁被允许更新任何配置为可写回的属性，这些属性也用于任何细粒度权限策略定义中。使用上面的示例，此策略将控制谁可以编辑region属性。

如果在对象类型注册到对象存储V1（Phonograph）后更改了视图策略，则必须通过对象类型 Ontology Manager 中的 Datasources 选项卡中的 Phonograph 部分的 更新 按钮来更新注册信息。如果没有更新注册信息，根据之前注册的策略，受限视图的最新数据可能会被提供。默认情况下，对象存储V2中可以自动传播策略。
