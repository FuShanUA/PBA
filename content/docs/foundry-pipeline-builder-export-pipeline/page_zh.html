导出流水线代码

在流水线构建器中进行构建时，您可以将流水线代码导出至现有的 Java 转换代码库 中。当您需要访问特定的 Java 库时，此导出功能非常实用。

在导出过程中，您的流水线将被转换为 Java 转换代码，随后推送至目标代码库。针对此过程，请注意以下事项：

  • 目标代码库中指定分支上的任何现有代码或文件都将被删除。
  • Java 转换代码的新输出结果并不总是与流水线构建器中流水线的输出结果完全一致。有关更多详细信息，请参阅 将流水线构建器导出为 Java 转换。
  • 此过程不可逆，这意味着对 Java 转换代码的任何更改都无法反向推送回流水线构建器的流水线中。
  • 部分流水线转换操作无法转换为代码。

打开您要导出的流水线，导航至 设置 > 导出代码。系统将弹出一个窗口，您可以在其中搜索并选择现有的目标 Java 转换代码库。接着，选择导出操作所源自的流水线构建器分支，并可选择在目标代码库中创建一个新分支以供使用。

导出的流水线将作为 PipelineLogic.java 和 PipelineOutputs.java 文件，存放于您代码库的 transforms-java/src/main/java/com/ 目录下以供使用。

将流水线构建器导出为 Java 转换

在将流水线构建器的流水线导出为 Java 代码时，务必要认识到新的输出结果并不总是与原始流水线的输出结果完全一致。这主要是由以下几个原因造成的：

  • 代码生成限制：某些功能（例如用户定义函数（UDF）和 LLM 调用）不受支持，需要手动实现。这些部分在生成的代码中将以 todo 的形式呈现。
  • 与原生 Spark 的差异：为了实现更高的可靠性和更出色的错误处理能力，流水线构建器中的某些表达式经过了优化，其实现方式与原生 Spark 有所不同。我们无法导出这些自定义优化，必须退回到原生 Spark 表达式，这在某些边缘场景下的表现可能会有所不同。

代码生成中所有其他受支持的表达式均已通过 Spark 测试用例的验证。导出为 Java 转换应被视为一个起点，用户需要对其进行手动验证，以确保完全准确。
