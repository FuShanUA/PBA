Transforms

Foundry 的 Transforms 是数据处理管道的核心构建块。它们定义了数据如何从输入流向输出，将计算逻辑和资源需求封装在一个单一的声明式单元中。

Transforms 注册在 Pipelines 中，而后者可以由多个 transform 定义组成。

定义 transforms

每个 transform 由三个关键组件构成：

  1. 装饰器： 定义输入、输出和配置
  2. 函数签名： 与装饰器中声明的输入和输出相匹配
  3. 计算逻辑： 实际的数据处理代码

以下是用于定义 transforms 的基本模式：

1from transforms.api import transform, Input, Output
2
3

7)
8def filter_hair_color(hair_eye_color, processed):  # 函数签名与输入/输出匹配
9    ... # 计算逻辑

Python 转换可以配置单节点（轻量级）或多节点（Spark）计算引擎。默认情况下，转换在单节点上运行，你可以将数据加载为 Polars ↗ 或 pandas ↗ DataFrame。Polars 是一个用于转换表格数据的 DataFrame 库，以其高性能、稳定性和易用性著称。Pandas 则是一款被广泛采用且易于使用的数据分析工具。

要获取 DataFrame，请对你的输入调用相应的方法，如下所示：

处理多种数据格式

轻量级转换 API 支持加载多种格式的数据，如下例所示：



已复制！
1@transform.using(output=Output('/Project/folder/output'), dataset=Input('/Project/folder/input'))
2def compute(ctx, output, dataset):
3    polars_df = dataset.polars()         # polars_df 是一个 polars.DataFrame 对象
4    lazy_df = dataset.polars(lazy=True)  # 激活流处理模式，lazy_df 是一个 polars.LazyFrame
5    pandas_df = dataset.pandas()         # pandas_df 是一个 pandas.DataFrame
6    arrow_table = dataset.arrow()        # arrow_table 是一个 pyarrow.Table
7
8    # 带有预注册输入数据集的 DuckDB 连接
9    conn = ctx.duckdb().conn
10    df_from_duckdb = conn.sql("SELECT * FROM dataset").fetchdf()  # dataset 已自动注册
11
12    output.write_table(lazy_df)          # 上述任何格式均可传递给 write_table

请注意，调用 dataset.pandas() 需要您的环境中已安装 pandas。同样，dataset.polars(...) 也要求环境中可以使用 Polars。

配置计算资源

您可以使用 .with_resources() 方法来控制分配给转换操作的计算资源。


```python
from transforms.api import transform, Input, Output


@transform.using(
    large_dataset=Input('/datasets/large_file'),
    processed=Output('/datasets/processed')
).with_resources(
    cpu_cores=8,          # Use 8 CPU cores
    memory_gb=16,         # Allocate 16GB memory
)
def process_large_dataset(large_dataset, processed):
    ...
```


    

  • cpu_cores: CPU 核心数。该值可以为小数，例如 0.5，默认值为 2。
    • 

  • memory_mb 或 memory_gb: 内存分配量；两者只能使用其一，不可同时使用。
    • 



转换上下文


有时，数据转换可能会依赖于输入数据集之外的其他事物。例如，转换过程可能需要获取当前的 Spark 会话，或与外部服务进行通信。在这种情况下，可以将 TransformContext 对象注入到转换过程中。


要注入 TransformContext 对象，您的计算函数必须接受一个名为 ctx 的参数。请注意，这也意味着任何输入或输出都不能命名为 ctx。有关 TransformContext 的完整功能列表，请参阅《API 参考文档 ↗》。

PySpark 转换


默认情况下，转换在单个计算节点上运行，并利用 pandas 或 Polars 等数据处理库。对于大数据计算工作流，可以在 Python 转换中使用 PySpark ↗。


PySpark 是一种封装语言，允许你与 Apache Spark 后端交互以快速处理数据。Spark 能够跨分布式服务器网络处理超大规模数据集，如果使用得当，可带来扩展性和可靠性方面的优势。然而，它也伴随着更高的开销和资源消耗，因此不适合处理中小规模的数据。


虽然此处提供了对 PySpark 转换 API 的高层概览，但更多细节请参阅《Python 转换》文档。

配置 PySpark 转换


在 PySpark 转换中，资源通过 Spark 配置文件进行配置。这些配置允许对大规模计算资源的使用实施严格的访问控制。


转换逻辑层级版本控制


警告
为了使 TLLV 正常工作，代码必须在模块级别声明所有导入，并且不应尝试修补或以其他方式修改其他模块中的对象。如果你的项目情况并非如此，你必须禁用 TLLV。有关更多信息，请参阅下面的代码示例。


TLLV 默认启用。要禁用 TLLV，请将 transformsPython 配置中的 tllv 设置为 false。此配置位于你的 Python 转换子项目中的 build.gradle 文件内。

transformsPython {
    tllv false
}



转换的版本是一个字符串，在评估逻辑是否过期时，用于比较转换的两个版本。如果转换的输入未更改且转换的版本未更改，则该转换的输出为最新状态。如果版本发生更改，转换的输出将失效并重新计算。


默认情况下，转换的版本包含以下内容：


    

  • 定义转换的模块
    • 

  • 转换依赖的所有模块
    • 

  • 任何项目依赖项
    • 



如果其中任何一项发生更改，版本字符串也会随之更改。
如果希望在未涵盖的文件发生更改时使输出失效，请在 transformsPython 配置中设置 tllvFiles。一个典型的用例是，当你正在读取文件的配置，并希望在配置更改时使输出失效。


事务性


转换在事务上下文中执行，该事务在输出上打开，并在作业成功时提交。如果输出上未定义逻辑，或者在写入期间捕获并吞没了错误，则成功的转换将在这些输出上产生空快照。
