开发者API有限支持策略（v1.0）

本页概述了Apollo中功能的生命周期状态和支持策略。

当某个“有限支持策略”适用于一个功能时，该功能的文档中将明确标记这一点。这将包括对每个类别策略的参考。所有支持类别的更改都将通过公告进行宣布。该公告的时间可能在下面的策略中被引用。如果可能，我们将根据其支持类别在最高级别的分组中对相关命令或API进行分组，以便您确切了解所获得的内容。

这些策略可能会随时更改，但策略本身是版本化的，并且受您现在阅读的相同声明策略的约束！我们将在对我们的标准和有限支持策略进行任何更改之前至少六个月通过公告通知您任何重大更改。

标准支持政策（隐含）

这是默认应用于Apollo功能的支持政策，除非某些有限支持标记另有说明。

Apollo平台开发者API描述了许多我们希望您使用的功能，以从Apollo中获得更多的价值！

如果开发者API中没有特别说明，这些功能将隐式地包含在我们的标准支持政策中，并且可以用于您的第三方开发。我们承诺为您提供高质量的命令、API和组件集，以便您在此基础上构建，并且在出现问题时提供适当的支援响应。这些功能可能会发生变化，但将以非破坏性的方式进行——根据需要添加功能或进行修复，以持续改进平台。

需要注意的是，我们计划在未来几年内快速演进我们的平台。应用了标准支持政策的功能可能会随时被标记为“有限支持：已弃用”。从那时起，它们将按照该支持类别的规定进行处理。

有限支持策略

已弃用

为了逐步构建一个更好的平台，我们可能需要移除某些功能。

这些功能被标记为“已弃用”，并且在宣布该功能弃用后，不会早于六个月被删除。我们预计在六个月的时间线之后积极删除这些已弃用的功能，以最好地支持Apollo平台的演进。届时我们将发布一个公告，表明该功能已被删除。

我们仍将继续支持这些API，直到它们从我们的API表面区域中被删除，并且所有相关文档也被删除。在六个月的弃用期之后，不应对任何已下载或缓存的我们的CLI实例，或任何调用已被删除API的脚本继续工作抱有任何期望。即使它们在一段时间内仍能正常工作，您也不应期望可以依赖它们。

实验性

随着时间的推移，我们可能会引入一些标记为实验性的新功能。

这些功能随时可能发生变化。用户应预期到可能出现的问题，并仅在这种情况是可以接受的地方使用这些功能。这可能包括彻底删除实验性命令或API，而不提供替代方案。我们对实验性命令或API的问题提供的支持更为有限。

这些命令可能会长期处于“实验性”状态，这并不代表它们的稳定性或过渡到标准支持政策的可能性。这些功能通常很令人兴奋，但我们建议您谨慎使用。
