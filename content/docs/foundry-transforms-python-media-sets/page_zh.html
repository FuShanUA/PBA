使用 Python 转换处理媒体集

您可以在 Python 转换中使用 媒体集，用于 PDF 文本提取、光学字符识别 (OCR)、图像平铺、元数据解析等。下文将介绍如何在您的 Python 代码仓库中设置媒体集，以及如何通过 Python 转换读取和写入媒体集。

以下是一个逐步示例，说明如何 使用 Code Repositories 创建一个使用媒体集的批处理管道。

您也可以学习如何 使用媒体集编写增量转换。

目前 Code Repository 的预览 功能不支持媒体转换。任何使用媒体集的转换可以构建但无法预览。

将 transforms-media 库导入到您的代码仓库中

要使用特定于媒体集的装饰器，您首先需要将 transforms-media 库导入到您的代码仓库中。您可以通过导航到代码仓库界面左侧的 库 文件夹来完成此操作。搜索 transforms-media，然后安装该库。

在您的代码仓库中添加对 transforms-media 的依赖。

当处理媒体集时，您必须使用 @transform 装饰器。媒体集输入和输出可以使用 transforms.mediasets.MediaSetInput 和 transforms.mediasets.MediaSetOutput 规范传递。在构建过程中，这些规范分别被解析为 transforms.mediasets.MediaSetInputParam 和 transforms.mediasets.MediaSetOutputParam 对象。这些 MediaSetInputParam 和 MediaSetOutputParam 对象提供对计算函数中的媒体集的访问。任何数量的媒体集输入或输出都可以与其他任何有效的转换输入和输出（如表格数据集）结合使用。例如：

Copied!
1from transforms.api import transform
2from transforms.mediasets import MediaSetInput, MediaSetOutput
3
4
5@transform.spark.using(
6    images=MediaSetInput('/examples/images'),
7    output_images=MediaSetOutput('/examples/output_images')
8)
9def translate_images(images, output_images):
10    ...

从媒体集读取

您可以通过文件路径或RID访问单个媒体项：

Copied!
1from transforms.api import transform
2from transforms.mediasets import MediaSetInput, MediaSetOutput
3
4
5@transform.spark.using(
6    images=MediaSetInput('/examples/images'),
7    output_images=MediaSetOutput('/examples/output_images')
8)
9def translate_images(images, output_images):
10    image1 = images.get_media_item_by_path("image1")
11    image2 = images.get_media_item("ri.mio.main.media-item.123")
12    ...

但是，您可能希望转换媒体集中的所有项。为此，您必须首先使用列出方法将这些项拉入数据框。在下面的示例中，我们列出输入媒体集中的所有项，并将结果数据框写入表格输出：

Copied!

2from transforms.mediasets import MediaSetInput
3
4
5@transform.spark.using(
6    images=MediaSetInput('/examples/images'),
7    listing_output=Output('/examples/listed_images')
8)
9def translate_images(ctx, images, listing_output):
10    media_items_listing = images.list_media_items_by_path_with_media_reference(ctx)
11
12    # You can perform regular PySpark transformations on media_items_listing
13
14    column_typeclasses = {'mediaReference': [{'kind': 'reference', 'name': 'media_reference'}]}
15    listing_output.write_dataframe(media_items_listing, column_typeclasses=column_typeclasses)

如果媒体集中有多个项位于特定路径下，则仅包含最新的那个项。该列表的结构如下：

+--------------------------+-----------+-------------------+
|        mediaItemRid      |    path   |  mediaReference   |
+--------------------------+-----------+-------------------+
| ri.mio.main.media-item.1 | item1.jpg |  {{reference1}}   |
| ri.mio.main.media-item.2 | item2.jpg |  {{reference2}}   |
| ri.mio.main.media-item.3 | item3.jpg |  {{reference3}}   |
+--------------------------+-----------+-------------------+

请注意，上面的示例只显示了列表的前三行。

设置mediaReference列的类型类允许该列被读取为媒体引用。

对get_media_item()、get_media_item_by_path()等的调用返回一个Python文件类似对象。所有由 io.open() ↗ 也受支持。请注意，项目是作为流读取的，这意味着不支持随机访问。

您也可以在不下载完整项目的情况下返回单个媒体项目的元数据。元数据将包括诸如图像的尺寸、音频的长度等信息。有关可用元数据的完整参考，请参见下面的附录。下面的示例在媒体项目列表中添加了一列，显示每个图像的元数据。

Copied!
1from transforms.api import transform, Output
2from transforms.mediasets import MediaSetInput
3from pyspark.sql import functions as F
4from pyspark.sql.types import StringType
5from conjure_python_client import ConjureEncoder
6
7
8@transform.spark.using(
9    images=MediaSetInput('/examples/images'),
10    listing_output_with_metadata=Output('/examples/listed_images_with_metadata')
11)
12def translate_images(ctx, images, listing_output_with_metadata):
13
14    def get_metadata(media_item_rid):
15        metadata = images.get_media_item_metadata(media_item_rid)
16        return ConjureEncoder().default(metadata)
17
18    metadata_udf = F.udf(get_metadata, StringType())
19
20    media_items_listing = images.list_media_items_by_path_with_media_reference(ctx)
21    listing_with_metadata = media_items_listing.withColumn('metadata', metadata_udf(F.col('mediaItemRid')))
22    column_typeclasses = {'mediaReference': [{'kind': 'reference', 'name': 'media_reference'}]}
23    listing_output_with_metadata.write_dataframe(listing_with_metadata, column_typeclasses=column_typeclasses)

媒体集支持一些内置的转换功能。有关API和支持的转换列表，请参见下面的附录。对这些转换的调用也将返回一个Python文件类流对象。要使用这些内置转换，请在媒体集输入上调用适当的方法。例如：

Copied!
1@transform.spark.using(
2    images=MediaSetInput('/examples/images'),
3    image_text_output=Output('/examples/listed_images_with_text')
4)
5def translate_images(ctx, images, image_text_output):
6
7    def get_ocr_on_image(media_item_rid):
8        return images.transform_image_to_text_ocr_output_text(media_item_rid).read().decode('utf-8')
9
10    ocr_on_image_udf = F.udf(get_ocr_on_image, StringType())
11
12    media_items_listing = images.list_media_items_by_path_with_media_reference(ctx)
13    listing_with_ocr = media_items_listing.withColumn('text', ocr_on_image_udf(F.col('mediaItemRid')))
14    column_typeclasses = {'mediaReference': [{'kind': 'reference', 'name': 'media_reference'}]}
15    image_text_output.write_dataframe(listing_with_ocr, column_typeclasses=column_typeclasses)

创建一个媒体集

除了使用媒体集创建对话框在项目或文件夹内创建媒体集外，您还可以直接在代码仓库中创建媒体集，以作为您的 Python 转换的输出。

首先，在 MediaSetOutput 规范中选择新媒体集的输出位置和名称。

选择您刚刚定义的路径，此时该路径应显示为红色下划线；悬停时，您应该会看到一个错误消息，表示该媒体集不存在。

从行左侧的灯泡图标中，选择 创建媒体集。

按照对话框步骤选择所需的 媒体集模式，并在您的新媒体集上完成其他任何必要的配置。

在选择 创建 之后，MediaSetOutput 规范将填充您提供的详细信息。这些标注字段定义了媒体集的创建方式。

在第一次为媒体集构建Python转换后，将创建新的媒体集，之后不应编辑注释字段。

写入媒体集

可以通过使用 MediaSetOutput 规范将媒体集用作转换的输出。

要上传一个项目，请在输出媒体集中调用 put_media_item() 端点。该端点接受任何类似文件的对象和一个路径，该路径将用于在输出媒体集中标识该项目。以下是一个基本示例：

Copied!
1from transforms.api import transform
2from transforms.mediasets import MediaSetInput, MediaSetOutput
3
4
5@transform.spark.using(
6    images=MediaSetInput('/examples/images'),
7    output_images=MediaSetOutput('/examples/output_images')
8)
9def upload_images(images, output_images):
10    with images.get_media_item_by_path("image1.jpg") as input_image:
11        output_images.put_media_item(input_image, "copied_image1.jpg")

在将项目从一个媒体集复制到另一个媒体集时，可以在输出中使用 fast_copy_media_item() 方法。这比下载并重新上传媒体项目更快且更高效：

Copied!
1@transform.spark.using(
2    images=MediaSetInput('/examples/images'),
3    output_images=MediaSetOutput('/examples/output_images')
4)
5def upload_images(images, output_images):
6    origin_media_item_rid = images.get_media_item_rid_by_path("image1.jpg").item
7    output_images.fast_copy_media_item(images, origin_media_item_rid, "fast_copied_image1.jpg")

项目可以在用户定义的函数（UDFs）中上传到媒体集，以实现更高的并行性。在下面的示例中，我们使用内置的PDF转JPEG转换功能将输入媒体集中的PDF转换为JPEG，并将这些JPEG上传到一个新的输出媒体集。然后，我们输出一个包含这些上传的JPEG媒体引用的表格数据集：

Copied!
1from transforms.api import transform, Output
2from transforms.mediasets import MediaSetInput, MediaSetOutput
3from pyspark.sql import functions as F
4from pyspark.sql.types import StringType
5
6
7@transform.spark.using(
8    pdfs=MediaSetInput('/examples/PDFs'),
9    output_images=MediaSetOutput('/examples/JPEGs'),
10    output_references=Output('/examples/JPEG listing')
11)
12def upload_images(ctx, pdfs, output_images, output_references):
13
14    def upload_jpg(media_item_rid, path):
15        with pdfs.transform_document_to_jpg(media_item_rid, 0) as jpeg:
16            response = output_images.put_media_item(jpeg, path)
17        return response.media_item_rid
18
19    upload_udf = F.udf(upload_jpg, StringType())
20
21    listed_pdfs = pdfs.list_media_items_by_path(ctx)
22    media_reference_template = output_images.media_reference_template()
23    uploaded_jpegs = listed_pdfs\
24        .withColumn('uploaded_media_item_rid', upload_udf(F.col('mediaItemRid'), F.col('path')))\
25        .select('path', 'uploaded_media_item_rid')\
26        .withColumn("mediaReference", F.format_string(media_reference_template,
27                                                      'uploaded_media_item_rid'))
28
29    column_typeclasses = {'mediaReference': [{'kind': 'reference', 'name': 'media_reference'}]}
30    output_references.write_dataframe(uploaded_jpegs, column_typeclasses=column_typeclasses)

媒体集写入模式

可以使用两种写入模式之一来写入媒体集：

  • modify: 上传的项目将被添加到媒体集分支中已有的项目上。
  • replace: 上传的项目将完全替换媒体集分支。

默认的写入模式取决于媒体集的 事务策略。事务性媒体集默认使用 replace。无事务性媒体集使用 modify 写入模式，且此模式无法更改，因为无事务性媒体集中的分支不能重置为空。

媒体集输出的写入模式可以在运行时动态更改。这在基于管道中自定义标准决定是否完全替换输出的情况下可能会有所帮助。

要更改媒体集的写入模式，可以使用媒体集输出上的 .set_write_mode() 方法。在项目上传到输出之前，可以在任何时间点更改写入模式。例如：

Copied!
1from transforms.api import transform, Input
2from transforms.mediasets import MediaSetOutput
3
4@transform.spark.using(
5    input_PNGs=Input('/examples/input_PNGs'),
6    output_PNGs=MediaSetOutput('/examples/output_PNGs'),
7)
8def upload_pngs(input_PNGs, output_PNGs):
9    if should_replace(input_PNGs):
10        output_PNGs.set_write_mode("replace")
11    else:
12        output_PNGs.set_write_mode("modify")
13
14    output_PNGs.put_dataset_files(input_PNGs)

从文件系统（目录）数据集中上传

使用 put_dataset_files

Python 媒体集 SDK 内置工具可将 Palantir 文件系统（称为目录）中的常规数据集的文件上传到媒体集中。例如：

Copied!
1from transforms.api import transform, Input
2from transforms.mediasets import MediaSetOutput
3
4
5@transform.spark.using(
6    pdfs_dataset=Input('/examples/PDFs'),
7    pdfs_media_set=MediaSetOutput('/examples/PDFs media set')
8)
9def upload_to_media_set(pdfs_dataset, pdfs_media_set):
10    pdfs_media_set.put_dataset_files(pdfs_dataset, ignore_items_not_matching_schema=False)

此转换将把数据集中的所有项目上传到媒体集。如果任何项目与媒体集的模式不匹配（例如，数据集中有JPEG文件），则构建将失败。通过设置ignore_items_not_matching_schema=True，这些不匹配项将被忽略。

使用put_media_item

也可以逐个上传文件。例如：

Copied!
1from transforms.api import transform, Input, Output, incremental
2from transforms.mediasets import MediaSetInput, MediaSetOutput
3import os
4
5
6@transform.spark.using(
7    output_media_set=MediaSetOutput("/path/media_set_output", should_snapshot=False),
8    input_dataset=Input("/path/dataset_of_raw_files"),
9)
10def compute(input_dataset, output_media_set):
11    all_files = list(input_dataset.filesystem().ls())
12    for current_file in all_files:
13        with input_dataset.filesystem().open(current_file.path, 'rb') as f:
14            filename = os.path.basename(current_file.path)
15            output_media_set.put_media_item(f, filename)

轻量级支持

可以使用轻量级转换来转换媒体集。API 与标准 Python 转换相同，但在列出媒体项时使用单节点库如 pandas 进行数据框操作。以下示例展示了一个轻量级转换，它会列出输入媒体集中的所有项，并将生成的数据框写入表格输出：

Copied!

2from transforms.mediasets import MediaSetInput, LightweightMediaSetInputParam
3
4@transform.using(
5    images=MediaSetInput('/examples/images'),
6    listing_output=Output('/examples/listed_images')
7)
8def translate_images(images: LightweightMediaSetInputParam, listing_output: LightweightOutput):
9    media_items_listing = images.list_media_items_by_path_with_media_reference().pandas()
10
11    # You can perform regular pandas transformations on media_items_listing
12
13    listing_output.write_table(media_items_listing)

某些操作在轻量级转换中不受支持。特别是put_dataset_files不被支持，因为它专门依赖于Spark的分布式处理来并行上传文件。

从文档中提取布局感知内容

在处理媒体集时，可以使用转换从文档中提取内容，例如段落、标题和表格，以及有关此内容布局的附加元数据。此提取转换可以在PDF和图像媒体集上运行。

使用模型提取边界框并传递给视觉模型，可能对特别复杂或晦涩的文档产生更好的效果。

要在您的转换中运行此提取，必须在您的注册中可用文档信息提取模型。您可以通过在模型目录中搜索来检查文档信息提取模型是否可用。如果文档信息提取模型不可用但您希望使用它，请联系Palantir代表。 输出将是一个“block”结构体数组，这些结构体对应于文档的区域。每个“block”将具有类型、置信度、ID、边界框、提取文本、HTML（如适用）中的提取表格、页码和语言信息。 以下是一个从PDF媒体集提取布局感知内容的Python转换示例： 

Copied!
1from transforms.api import transform, Output
2from transforms.mediasets import MediaSetInput, MediaSetInputParam
3from pyspark.sql.functions import udf
4
5
6@transform.spark.using(
7    output=Output('ri.foundry.main.dataset.0-1-2-3-4'),
8    media_input=MediaSetInput('/examples/input_pdfs'),
9)
10def compute(media_input: MediaSetInputParam, output):
11
12    def extract_all_text(media_item_rid):
13        metadata = media_input.get_media_item_metadata(media_item_rid)
14        pages = metadata.document.pages
15        if pages is None:
16            return ""
17
18        text = ""
19        for page in range(pages):
20            response = media_input.transform_media_item(media_item_rid, str(page), {
21                "type": "documentToText",
22                "documentToText": {
23                    "operation": {
24                        "type": "extractLayoutAwareContent",
25                        "extractLayoutAwareContent": {
26                            "parameters": {
27                                "languages": ["ENG"]
28                            }
29                        }
30                    }
31                }
32            })
33            text += str(response.json())
34        return text
35
36    extract_text_udf = udf(extract_all_text)
37
38    result = media_input.dataframe().withColumn("text", extract_text_udf("mediaItemRid"))
39
40    column_typeclasses = {
41        "mediaReference": [{"kind": "reference", "name": "media_reference"}]
42    }
43
44    output.write_dataframe(result, column_typeclasses=column_typeclasses)
我们建议如果您在许多文档或具有许多页面的文档上运行此提取转换，则使用并行处理以获得最佳性能。 有关速率限制、地区可用性和使用费率的更多信息，请参阅文档中的 速率限制、地区可用性 和 使用费率。

将PDF文档转换为JPEG图像

将文档页面转换为指定尺寸的图像。

参数：

  • media_item_rid（可选[str]）：如果指定了该参数，将对该指定项进行转换，而不是整个媒体集。默认值为 None。
  • start_page（可选[int]）：转换的起始页码（从零开始）。默认值为 0（第一页）。
  • end_page（可选[int]）：转换的结束页码（从零开始，不包括该页）。默认值为 None（最后一页）。
  • width（可选[int]）：输出图像的宽度。默认值为 1024。
  • height（可选[int]）：输出图像的高度。默认值为 1024。
  • output_format（字符串）：输出图像的格式，例如 PNG 或 JPEG。默认为 PNG。

返回：

  • 一个包含文档到图像转换的 MediaSetInputTransform 实例，允许进一步的转换。

示例：

Copied!
1transform = pdf_input.transform().convert_document_to_images("ri.mio.main.media-item.1", 0, 5, 600, 900, output_format="jpg")
2image_output.write(transform)

使用OCR将PDF文档集转换为文本

此转换使用传统的光学字符识别（OCR）从PDF文档中提取文本。请注意，这不是基于AI的OCR，后者使用视觉语言模型来执行提取。 了解有关使用视觉语言模型提取PDF文档内容的更多信息。。

参数：

  • languages（列表[字符串]）：用于OCR的语言列表。默认为英语。所有有效的代码都可以在 Tesseract 文档 ↗ 的语言部分找到。
  • scripts（可选[列表[字符串]] ): 用于OCR的脚本列表。默认值为 None。所有有效代码可在 Tesseract 文档 ↗ 的脚本部分找到。
  • media_item_rid (可选[str]): 如果指定，则对指定项而不是整个媒体集执行转换。默认值为 None。
  • start_page (可选[int]): OCR 的起始页（从零开始）。仅适用于PDF媒体集。默认值为 0（第一页）。
  • end_page (可选[int]): OCR 的结束页（从零开始，不包括该页）。仅适用于PDF媒体集。默认值为 None（最后一页）。
  • return_structure (str): item_per_row 或 page_per_row。仅适用于整个PDF媒体集的转换。默认值为 item_per_row。
  • suppress_errors (bool): 指定错误处理行为。如果为 True，则会捕获错误，并在输出中返回错误信息。如果为 False，则不会捕获任何错误，构建将失败。默认值为 True。仅适用于整个媒体集的转换。

返回：

  • DataFrame、字符串列表或单个字符串。
    • str：对单个图像的转换。
    • list[str]：对单个PDF的转换。
    • DataFrame：对整个媒体集的转换。
      • 对于PDF (item_per_row)：列是 media_item_rid, path, media_reference, extracted_text (list[str])。
      • 对于PDF (page_per_row) 或图像集：列是 media_item_rid, path, media_reference, page_number, extracted_text (str)。

示例：

Copied!
1df = media_set.transform().ocr()
2dataset_output.write_dataframe(df)

将音频转录为文本

将包含语音的音频文件转录为文本。

参数：

  • media_item_rid（可选[str]）：如果指定，将在指定项上运行转换，而不是整个媒体集。默认值为 None。
  • language（可选[str]）：用于转录的语言。默认为 None，在这种情况下，它将被自动检测。有效的语言可以在 Whisper GitHub 仓库 ↗ 的 LANGUAGES 下找到。
  • performance_mode（Literal["more_economical", "more_performant"]）：用于转录的性能模式。默认为more_economical。
  • output_format（Literal["text", "segments"]）：输出格式。默认为text。
  • add_timestamps（Optional[bool]）：控制是否在转录中添加时间戳。默认为False。仅当output_format为text时适用。
  • suppress_errors（bool）：指定错误处理行为。如果为True，则会捕获错误，并将在输出中返回错误信息。如果为False，任何错误都不会被捕获，并且构建将失败。默认为True。仅适用于整个媒体集上的转换。

返回：

  • 一个DataFrame或单个字符串。
    • str：适用于单个项目的转换。
      • text：转录文本。
      • segments：包含转录段、时间戳、段置信度等详细信息的JSON对象。
    • DataFrame：列包括media_item_rid、path、media_reference、transcription（str）。适用于整个媒体集上的转换。
