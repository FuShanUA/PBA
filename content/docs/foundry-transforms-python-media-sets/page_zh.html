在 Python 转换中使用媒体集

你可以在 Python 转换中使用媒体集来进行 PDF 文本提取、光学字符识别 (OCR)、图像分块、元数据解析等操作。以下章节将介绍如何在你的 Python 代码库中设置媒体集，以及如何通过 Python 转换对媒体集进行读写。

以下是一个分步示例，演示如何使用代码仓库创建媒体集批处理流水线。

你还可以了解如何编写带有媒体集的增量转换。

代码仓库的预览功能目前不支持媒体转换。任何使用媒体集的转换都可以构建，但无法预览。

将 transforms-media 库导入到你的代码库中

要使用特定于媒体集的装饰器，你首先需要将 transforms-media 库导入到你的代码库中。你可以在代码仓库界面左侧的 库 文件抽屉中完成此操作。搜索 transforms-media，然后安装该库。

在你的代码仓库中添加对 transforms-media 的依赖

在处理媒体集时，你必须使用 @transform 装饰器。可以使用 transforms.mediasets.MediaSetInput 和 transforms.mediasets.MediaSetOutput 规范来传入媒体集的输入和输出。在构建期间，这些规范会分别解析为 transforms.mediasets.MediaSetInputParam 和 transforms.mediasets.MediaSetOutputParam 对象。这些 MediaSetInputParam 和 MediaSetOutputParam 对象提供了在计算函数内访问媒体集的途径。任意数量的媒体集输入或输出都可以与任何其他有效的转换输入和输出（例如表格数据集）结合使用。例如：

创建媒体集

除了可以通过媒体集创建对话框在项目或文件夹内创建媒体集外，您还可以直接在 Code Repositories 中创建媒体集，将其作为 Python 转换的输出。

首先，在 MediaSetOutput 规范中为新媒体集选择输出位置和名称。

选中您刚才定义的路径，此时该路径下方应显示红色波浪线；将鼠标悬停其上，您会看到一条错误消息，提示该媒体集不存在。

在该行左侧的灯泡图标中，选择创建媒体集。

按照对话框步骤操作，选择所需的媒体集架构，并完成新媒体集所需的任何其他配置。

选择创建后，MediaSetOutput 规范将填充您提供的详细信息。这些注解字段定义了媒体集的创建方式。

首次构建 Python 转换后将创建新媒体集，此后不应再编辑这些注解字段。

在不同媒体集之间复制项目时，可以对输出使用 fast_copy_media_item() 方法。相比下载并重新上传媒体项目，这种方式速度更快，效率更高：


媒体集写入模式

写入媒体集时，可采用以下两种写入模式之一：

  • modify：上传的项目将追加至媒体集分支现有项目之后。
  • replace：上传的项目将完全替换媒体集分支。

默认写入模式取决于媒体集的[事务策略](/docs/foundry/media-sets-advanced-formats/media-set-settings/#transaction-policies)。事务型媒体集默认采用 replace 模式。非事务型媒体集采用 modify 写入模式，且无法更改，因为非事务型媒体集中的分支无法重置为空。

媒体集输出的写入模式支持在运行时动态更改。当是否完全替换输出取决于管道中的自定义判定条件时，此功能将非常实用。

如需更改媒体集的写入模式，可在媒体集输出上调用 .set_write_mode() 方法。在项目上传至输出之前，可随时更改写入模式。示例如下：

此转换操作会将数据集中的所有条目上传至媒体集。若存在任何条目与媒体集的模式不匹配（例如数据集中包含 JPEG 文件），构建过程将会失败。通过设置 ignore_items_not_matching_schema=True，此类不匹配的情况将被忽略。

从文档中提取具备布局感知能力的内容


在处理媒体集时，你可以使用转换操作从文档中提取内容（如段落、标题和表格），以及关于这些内容布局的额外元数据。此提取转换操作可同时在 PDF 和图像媒体集上运行。


对于特别复杂或模糊的文档，使用模型提取边界框并传递给视觉模型可能会产生更好的效果。


若要在你的转换操作中运行此提取功能，你的部署环境中必须提供“文档信息提取”模型。你可以通过在模型目录中搜索来检查该模型是否可用。如果“文档信息提取”模型不可用且你希望使用它，请联系 Palantir 代表。


输出结果将是一个“区块”结构体数组，对应文档中的各个区域。每个“区块”将包含类型、置信度、ID、边界框、提取的文本、HTML 格式的提取表格（如适用）、页码以及语言信息。


以下是一个从 PDF 媒体集中提取具备布局感知能力内容的 Python 转换操作示例：



```python
from transforms.api import transform, Output
from transforms.mediasets import MediaSetInput, MediaSetInputParam
from pyspark.sql.functions import udf


@transform.spark.using(
    output=Output('ri.foundry.main.dataset.0-1-2-3-4'),
    media_input=MediaSetInput('/examples/input_pdfs'),
)
def compute(media_input: MediaSetInputParam, output):

    def extract_all_text(media_item_rid):
        metadata = media_input.get_media_item_metadata(media_item_rid)
        pages = metadata.document.pages
        if pages is None:
            return ""

        text = ""
        for page in range(pages):
            response = media_input.transform_media_item(media_item_rid, str(page), {
                "type": "documentToText",
                "documentToText": {
                    "operation": {
                        "type": "extractLayoutAwareContent",
                        "extractLayoutAwareContent": {
                            "parameters": {
                                "languages": ["ENG"]
                            }
                        }
                    }
                }
            })
            text += str(response.json())
        return text

    extract_text_udf = udf(extract_all_text)

    result = media_input.dataframe().withColumn("text", extract_text_udf("mediaItemRid"))

    column_typeclasses = {
        "mediaReference": [{"kind": "reference", "name": "media_reference"}]
    }

    output.write_dataframe(result, column_typeclasses=column_typeclasses)
```


如果在多文档或含大量页面的文档上运行此提取转换，建议采用并行处理以获取最佳性能。

  • add_timestamps (Optional[bool])：控制是否向转录文本中添加时间戳。默认值为 False。仅在 output_format 为 text 时适用。
    • 

  • 

suppress_errors (bool)：指定错误处理行为。若为 True，将捕获错误并在输出中返回错误信息。若为 False，则不捕获任何错误，构建过程将失败。默认值为 True。仅适用于对整个媒体集的转换操作。
    • 



返回值：


    

  • 返回一个 DataFrame 或单个字符串。

      

    • str：适用于对单个项目的转换操作。

        

      • text：转录后的文本。
        • 

      • segments：包含转录片段的 JSON 对象，涵盖时间戳、片段置信度及更多细节。
        • 

      

      • 

    • DataFrame：包含的列为 media_item_rid、path、media_reference、transcription (str)。适用于对整个媒体集的转换操作。
      •
