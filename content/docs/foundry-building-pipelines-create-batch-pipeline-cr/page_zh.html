使用代码仓库创建数据集批处理流水线

本指南将通过一个简单的数据转换示例，带您了解如何使用代码仓库应用程序。您将学习如何编写和修改 SQL 代码，以及如何构建数据集。同时，您还将学习如何通过分支与同事协同工作。

1. 创建仓库

首先创建一个新仓库。在 Foundry 中导航至某个项目，点击右上角的+ 新建，然后选择代码仓库。

在本指南中，我们将编写一个 SQL 转换。为您的仓库命名，然后在语言模板下的下拉菜单中选择SQL。接着，选择初始化仓库。

2. 导入数据

如果您已经导入了要使用的原始数据集，可以直接进入下一步。否则，您可以下载以下示例数据集：

下载 titanic.csv

请查阅《手动上传数据》指南，了解如何将此数据集与您的仓库一起上传到项目中。

3. 创建分支

通过创建个人分支来进行更改，而不是直接修改代码的主版本，您可以安全地与同事在同一代码仓库上进行协作。您可以跟踪和撤销更改，也可以将更改合并到主代码中。代码支持逐行审查，这意味着团队成员可以轻松讨论对生产流水线的更改。要了解有关 Foundry 中分支的更多信息，请参阅《分支》页面。

导航至代码仓库时，默认会处于 master 分支。最佳做法是将 master 分支设为受保护状态，这意味着无法直接在该分支上修改代码。请注意，您可以读取受保护分支上的文件，但不能编辑或创建文件。

在向代码仓库添加更改之前，您必须先创建自己的分支，该分支包含 master 分支上的代码副本。要创建分支，请点击当前分支名称旁边的 图标。

此操作会打开一个对话框，用于选择现有分支并为新分支指定自定义名称：

创建新分支后，您会在左侧看到完全相同的文件树。您刚才只是为起始的 master 分支上的代码创建了一个副本。现在您可以在自己的分支上修改文件了。

4. 修改代码

创建新文件

既然您已经在自己的分支上工作，将鼠标悬停在某个文件夹上并点击省略号图标，然后选择新建文件，即可创建一个新的 SQL 文件。选择新建文件后，系统会提示您选择文件类型并为其命名。在本示例中，选择SQL 转换并指定一个文件名（不含空格或特殊字符）：

请注意，新的 SQL 文件会在文件树中高亮显示，其位置即为构建后生成数据集所在的位置。

如果文件名在文件树中显示为绿色，表示这是您分支中新创建的文件，而在起始的 master 分支上并不存在。如果文件名显示为橙色，则表示该文件在 master 分支上存在，但在您的分支中已被修改。

新创建的 .sql 文件会根据您提供的文件名声明一个输出数据集。例如，如果您的代码库位于 /Public/Authoring 目录下，并且您创建了 titanicAnalysis.sql，该文件将自动声明一个输出数据集 /Public/Authoring/titanicAnalysis。

编辑您的文件

接下来，将占位文本替换为您实际的数据转换代码。首先，将 `SOURCE_DATASET_PATH` 替换为实际输入数据集的路径。在本示例中，您将使用本教程第 (2) 步中导入的 titanic 数据集。

请注意，当您输入反引号时，自动补全功能会弹出一个交互式菜单，列出可供使用的数据集。当您从列表中选择数据集时，数据集引用将被替换为该数据集的唯一 ID。这样一来，即使数据集日后被移动，您的转换代码依然能够正常运行。

在反引号中输入项目名称，找到 titanic 数据集，并从菜单中将其选中。

继续编写 SQL 代码以对数据进行转换。在输入 SQL 函数时，您会看到各种帮助提示框。例如，假设您想在“titanic”数据集中新建一列，用于显示乘客性别的单字母缩写。您可以查看有关如何使用 SUBSTRING 函数的说明：

在继续下一步之前，请完成数据转换代码的编写，以选择“Name”、“Age”、“Survived”和“Ticket”列，以及一个名为“Gender”的派生列。“Gender”列代表乘客性别的缩写；要创建此列，需对“Sex”列调用 SUBSTRING 函数。

请注意，您必须为在 SQL 中创建的任何派生列定义别名。有关编写 SQL 数据转换的更多信息，请参阅《Spark SQL 语言参考》。

5. 测试您的更改

在 Foundry 中，数据集支持分支管理（类似于代码）。这对于测试多步骤数据管道的设计非常有用。例如，您可以独立测试数据管道各部分的更改，而不会破坏未依赖您分支的其他用户的下游依赖关系。

既然已经编写了数据转换代码，您应当测试一下在当前分支中所作的更改。在将更改合并到 master 分支之前，进行测试以确保代码按预期运行是至关重要的。

使用预览功能迭代您的更改

在编写代码时，您可以使用“预览”功能来加快开发周期并快速迭代更改。预览功能会对采样输入运行您的代码，并在无需提交更改、运行检查或在 Foundry 中物化数据集的情况下提供样本输出。样本输出可能无法完全代表构建结果，但它们能帮助您确认代码是否正常运行并产生了预期结果。

要使用此功能，请点击页眉处的“预览”，或在代码编辑器的底部栏打开“预览”助手。您可以预览基于文件的数据集或带有模式的数据集。

对于带有模式的数据集，您可以通过点击需要编辑的输入项旁边的“设置图标”，来自定义预览更改时使用的输入。可用的选项包括：

  • 原始输入：使用原始输入的样本。
  • 上一次预览：对于在同一代码库中生成的数据集，您可以将预览的更改串联起来，将某个数据集的预览结果作为您下一次预览的输入。
  • 应用自定义过滤器：在受支持的列上应用过滤器，以针对输入的特定子集测试您的更改。例如：“时间戳在给定窗口内的所有行”，或“具有给定字符串值的所有行”。
  • 选择其他数据集：选择另一个包含代码所需全部列的数据集，以测试特定情况或提供任何其他自定义样本作为您的输入。

首次在包含文件的数据集上运行预览时，必须配置样本中将使用的文件。选定样本文件后，可以通过选择相应的输入来重新配置。保存配置后，预览将在选定的样本文件上执行代码。

再次运行预览时，无需重新配置输入文件。预览执行完成后，您可以以行或文件的形式查看样本输出。如果您具备所需权限，还可以选择下载输出文件。

提交更改

编写新代码后，您可以提交更改。在代码仓库中，当您希望为已完成的工作打上标签时，即可提交更改。即使不执行提交，系统默认也会自动保存您的工作。提交操作会在您到达一个暂停点时，为这组更改明确打上标签。

提示

点击 提交 按钮会提交您的更改，并对代码运行自动检查。点击 构建 按钮同样会提交更改。具体而言，点击 构建 会运行自动代码检查，并开始构建输出数据集。如果要在不构建数据集的情况下快速测试更改，以确保代码通过检查，请点击 提交。否则，您可以直接跳转到构建您的数据集。

要提交所作的更改，请点击右上角的 按钮，并输入所作更改的摘要。提交更改会触发对代码的自动检查。右上角的图标会指示这些检查的状态；将鼠标悬停其上可查看更多详情。

在您的分支上构建数据集

要测试更改，请点击屏幕顶部的 按钮。

点击构建按钮后，将发生两件事：对代码运行自动检查，并开始构建输出数据集。在此期间，系统将根据您分支中的代码创建新的输出数据集，或更新现有输出数据集以反映您的更改。您可以在 构建 助手中查看正在运行的任务进度。任务完成后， 图标表示每个任务均已成功完成。如果看到的是 图标，请点击 构建 助手中的 详情 按钮，了解出错原因。这会跳转至 检查 选项卡，您可以在此查找错误信息并重新触发构建。

以下是关于测试更改和构建数据集的一些重要信息：

  • 点击右上角的 构建 按钮，等同于点击代码仓库界面底部 构建 助手中的 构建 按钮。
  • 在点击构建按钮运行检查并开始构建输出数据集之前，必须先选中包含数据转换逻辑的文件。
  • 每次在分支上触发构建时，该任务都会排入当前正在执行的构建任务队列之后。

预览您的数据集

任务成功完成且数据集构建完毕后，您可以在 构建 助手中预览构建好的数据集：

点击 构建 助手中指向您数据集的链接，即可打开完整的数据集。

现在，您已在自己的分支上构建了数据集。请继续阅读本指南的其余部分，了解如何将数据转换代码合并到主分支。

6. 提交更改以供审查

至此，您已经：

  • 创建了自己的分支以进行更改，
  • 创建了包含数据转换代码的新 SQL 文件，
  • 在分支中测试了更改并构建了数据集，并且
  • 预览了构建好的输出数据集。

现在，你需要提交更改供团队成员审查。完成更改测试并对输出的数据集感到满意后，即可提交更改供团队成员审查。

提示

拥有数据主人权限的用户在创建拉取请求时，可以启用“自动合并更改”选项。仅当仓库分支配置并通过至少一项必要检查时，此选项才可用。启用“自动合并更改”选项后，拉取请求在创建后会自动合并到主代码中。分支更改合并到主代码后，该分支也会自动删除。

要创建新的拉取请求，请点击右上角的 按钮。此时会打开“新建拉取请求”页面，你可以在此处填写更改说明，然后点击创建拉取请求按钮。

这将基于你提交的更改创建一个新的拉取请求。拉取请求页面提供了丰富的工具，用于审查提交的更改对数据管道的影响：

  • 文件更改选项卡允许团队成员逐行审查代码
  • 影响分析选项卡展示受影响的数据集在当前分支上的变动情况，包括数据集架构和健康检查的更改
  • 管道审查选项卡以图表形式展示数据管道，并直观高亮显示此拉取请求中的更改对管道的影响。

详细了解如何分析拉取请求中更改的影响。

通常，在更改合并到主分支之前，邀请他人进行审查至关重要。用户可以按文件逐个批准或拒绝，以便跟踪在拉取请求合并前仍需调整的文件。将鼠标悬停在相应的指示图标上，即可查看哪些用户已经批准或拒绝了特定文件。

单个文件审查按钮为可选操作，但拒绝某个文件会自动拒绝整个拉取请求。同理，批准拉取请求时，所有单个文件也会一并获批。

7. 将更改合并到 master 分支

完成更改

更改审查通过后，拥有相应权限的用户（默认为数据主人和编辑者）可以合并拉取请求中的更改，将其并入 master 分支。

在本教程中，请点击屏幕右下角的压缩并合并按钮，然后在确认对话框中再次选择压缩并合并继续操作。

代码仓库可能要求在合并更改前满足不同的策略。这些策略由仓库数据主人在分支设置页面中定义，并展示给拉取请求页面的代码作者。

验证更改是否已合并到 master 分支

提交的更改通过后，应验证在当前分支中所作的更改是否已反映到 master 分支。为此，请点击代码选项卡，选择 master 分支，然后浏览相关文件，确保能够看到之前所作的更改。

删除分支

警告

请勿删除非本人创建的分支。对于在同一代码仓库中工作的其他人员，这可能导致工作内容丢失！

现在可以删除本教程开始时创建的分支，以保持环境整洁。既然更改已合并到 master 分支，便无需保留该分支。转至分支选项卡，在“个人分支”下找到并点击 图标，删除所创建的分支：

拉取请求页面提供了“合并后删除分支”选项，以便快速清理分支。受保护的分支无法使用此选项。

要删除受保护的分支，首先需要取消保护，然后按照上述步骤操作。

8. 在 master 分支上构建数据集

最后一步是在 master 分支上构建新数据集。与在分支中构建数据集类似，选中 SQL 代码文件后，点击屏幕顶部的 按钮即可。

任务成功完成且数据集构建完毕后，你可以点击构建辅助工具中的数据集链接，打开完整的数据集。

9. 还原更改

如果发现已合并到 master 分支的代码更改存在问题，可以通过一种简单的方法撤销这些更改。你可以在 master 分支的提交历史中找到特定的提交并将其还原。在仓库的分支选项卡中，点击 master 即可查看按时间顺序排列的所有提交列表。

点击提交哈希值即可查看某次提交的代码更改。找到要还原的提交后，点击还原。系统将创建一个指向 master 分支的拉取请求，你可以对其进行审查并合并。
