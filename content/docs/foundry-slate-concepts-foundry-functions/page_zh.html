在 Slate 中使用 Foundry Functions

通过 Foundry Functions 面板，Slate 能够将 Functions 代码仓库中编写的业务逻辑作为“后端”服务来执行。在这种模式下，可以将 Slate 应用视作传统 Web 应用基础设施中的客户端，Foundry Functions 实现视作服务器，而本体论（Ontology）则视作数据库。查阅文档以获取更多关于如何《Foundry Functions 入门》或如何使用《基于对象的 Functions》的信息。

Foundry Functions 面板支持以下操作：

  • 选择您有权限访问的 Foundry Function；
  • 选择该 Function 的版本（例如版本 0.0.5）；
  • 如果 Function 允许，向其传递具有类型的输入（请注意，如果 Function 接受对象集作为输入，您可以通过 Handlebars 传入 Slate 定义的对象集作为输入并进行解析）；以及
  • 在 Slate 中引用 Function 的 Handlebars 句柄 {{ff_foundry_function1}}，从而使用该 Function 具有类型的输出。

在 Slate 中使用之前，必须先在《Function 代码仓库》中创建该 Function。

面向 Slate 的实施方案

在编写用于 Slate 应用的 Functions 时，应根据应用逻辑的实现位置，考虑以下两种方案。

  • 在轻量级 Functions方案中，Functions 仅作为访问本体论 API 数据的简单透传通道。任何额外的处理操作（例如，将数值进行适当缩放以用作散点图的 Radius 半径参数）均在 Slate 应用中通过常规的 JavaScript 函数完成。
  • 在重量级 Functions方案中，所有的业务和显示逻辑都会下沉至 Foundry Functions 层中，Slate 应用本身仅保留非常特定的格式化或显示逻辑，以及实际的布局和组件配置。

利用 Foundry Functions 来组织和管理复杂 Slate 应用的逻辑层具有显著优势，例如：

  • 强类型化：这是一项最佳实践，能够从源头上杜绝整类潜在的边缘情况。
  • 在代码仓库中开发：为应用的前端和后端提供了独立的版本控制与协作机制，同时也支持代码组织和结构化的标准实践。
  • 将应用逻辑与特定前端解耦：打造出可在其他应用中引用的复用组件。

Function 输入与输出类型

Foundry Functions 支持多种标准类型、本体论生成类型以及自定义的输入和输出类型，相关内容已在《Functions 输入与输出类型》文档中详细说明。

在重量级 Functions方案中，通常会定义《自定义输入与输出类型》，从而确保返回的数据格式能够完全匹配特定图表或 HTML 组件的填充需求。
