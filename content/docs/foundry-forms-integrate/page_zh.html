与其他Foundry应用程序集成

Foundry Forms 不再是 Foundry 上数据输入或写回工作流的推荐方法。相反，使用 Foundry 本体论构建用户输入工作流，将相关数据结构表示为对象类型，并配置与操作的写回交互。有关更多信息，请参阅 表单概述 文档。

表单可以与其他 Foundry 应用程序无缝集成。此页面讨论了 Foundry 表单如何与 Fusion、Slate 和 对象浏览器 一起使用。

Fusion

如果选择 Fusion 作为 响应目标 ，表单将立即创建支撑的电子表格。每次保存表单时，新需要的列将自动添加到表格区域。

用户也可以通过在Fusion中选择插入选项卡并选择表单来从现有的表格区域创建一个表单。

Slate

Slate小部件可以用来在Slate中嵌入表单：

  • Foundry 表单（Form）
  • iIframe

使用iFrame时，可以使用以下URL参数来自定义表单的显示：

  • embedded=true：当嵌入表单时必须提供。
  • noHeader=true：将从表单中移除标题。
  • progressOnly=true : 将仅以进度条作为标题；当noHeader=true时被忽略。
  • forceDarkMode=true: 将以深色模式显示表单。

此外，用户可以使用代码编辑器（Code Editor）预填值。 要为每个字段添加 urlParam：

Copied!
1fields:
2  - uri: display.Text
3    name: Text
4    type: Text
5    urlParam: text
6    options: {}

然后URL将显示为：workspace/fforms/f/new/<form-id>?embedded=true&progressOnly=true&text=prefilled。

对象资源管理器

可以使用三个 部分小部件 来在对象视图中嵌入一个表单：

  • 编辑对象表单
  • 创建关联对象表单
  • iframe

此外，表单还支持通过一个基于对象的表单来批量编辑多个对象。

编辑对象表单

要添加 编辑对象表单 部分：

  1. 创建 一个与对象类型 X 关联的表单。
  2. 导航到 X 的对象视图，并选择 操作 然后是 编辑对象视图。
  3. 添加 编辑对象表单 部分。

Default Form ID 可以在表单的 URL 中找到：/workspace/fforms/v1/entry/<form-id>/new。

在 Prefilled Values 部分：

  • 与字段相关的 URI 或 URL 参数可以使用 代码编辑器 找到。
  • 这些值本身可以是静态的（2000-01-01），也可以从当前对象的属性中获取（{{start_date}}，其中 start_date 是对象类型 X 的属性 ID）。

Conditional Form 部分可用于根据某些属性的值呈现不同的表单。

创建关联对象表单

添加和配置此部分非常类似于 编辑对象表单，但有两个显著的不同之处：

  1. 该部分应添加到 Y 的对象视图中，该对象视图与 X 相关联。
  2. 可以根据最后一个关联对象进行预填充，并且可以提供一个排序属性来自定义“最后一个”的定义。

iframe

此部分可以按照上述 描述 添加，并按照 Slate 中所述进行配置。

批量编辑多个对象

默认情况下，此功能在基于对象的表单中是禁用的。请通过在 Visual Editor 右侧的 设置（Settings） 选项卡中切换 允许此表单进行批量编辑（在操作菜单中） 来启用它。

启用后，请按照以下步骤批量编辑多个对象：

  1. 导航到对象资源管理器并选择您要编辑的对象。
  2. 选择 打开于 按钮，然后选择 在 [name-of-the-form] 中编辑 [number-of-selected-objects] 个对象。
  3. 表单会在一个对话框中打开，并允许您覆盖所有选定对象的属性。

处于批量编辑模式的表单将覆盖所有选定对象上该表单中的所有属性。我们建议在这种表单中仅使用必填字段。使用 required 验证器确保提交前已填写值。

默认情况下，通过一个表单最多可以编辑 200 个对象。如果您的使用场景需要更大的限制，请联系您的 Palantir 代表。
