与其他 Foundry 应用程序集成

Foundry Forms 不再是 Foundry 上进行数据录入或数据回写工作流的推荐方式。建议改用 Foundry Ontology 构建用户输入工作流，将相关数据结构表示为对象类型，并通过 Actions 配置回写交互。详情请参阅《Forms 概览》文档。

Forms 提供了与其他 Foundry 应用程序的无缝集成。本页将讨论如何将 Foundry Forms 与 Fusion、Slate 和 Object Explorer 结合使用。

Fusion

如果选择 Fusion 作为响应目标，Forms 会立即创建对应的数据表。每次保存表单时，新必填列都会自动添加到表格区域中。

用户也可以通过选择“插入”选项卡并选择“表单”，从 Fusion 中现有的表格区域创建表单。

Slate

可以使用两个 Slate 微件将表单嵌入 Slate 中：

  • Foundry Form
  • iIframe

使用 iframe 时，可以使用以下 URL 参数来自定义表单的显示：

  • embedded=true：通过 iframe 嵌入表单时必须提供此参数。
  • noHeader=true：将移除表单的页眉。
  • progressOnly=true：将仅渲染进度条作为页眉；当 noHeader=true 时忽略此参数。
  • forceDarkMode=true：将以深色模式渲染表单。

此外，用户可以使用代码编辑器为每个字段添加 urlParam 来预填值：

已复制！
1fields:
2  - uri: display.Text
3    name: Text
4    type: Text
5    urlParam: text
6    options: {}

此时 URL 将显示为：workspace/fforms/f/new/<form-id>?embedded=true&progressOnly=true&text=prefilled。

Object Explorer

可以使用三个分区微件将表单嵌入对象视图中：

  • 编辑对象表单
  • 创建链接对象表单
  • iframe

此外，Forms 还支持通过一个对象支持的表单批量编辑多个对象。

编辑对象表单

要添加“编辑对象表单”分区：

  1. 创建与对象类型 X 关联的表单。
  2. 导航至 X 的对象视图，依次选择 操作 和 编辑对象视图。
  3. 添加 编辑对象表单 部分。

可以在表单的 URL 中找到 默认表单 ID：/workspace/fforms/v1/entry/<form-id>/new。

在 预填值 部分中：

  • 可以使用代码编辑器查找与字段关联的 URI 或 URL 参数。
  • 这些值可以是静态值（2000-01-01），也可以取自当前对象的属性（{{start_date}}，其中 start_date 是对象类型 X 的属性 ID）。

可以使用 条件表单 部分根据某个属性的值来渲染不同的表单。

创建关联对象表单

添加和配置此部分的操作与编辑对象表单非常相似，但有两个显著区别：

  1. 该部分应添加到与 X 关联的 Y 的对象视图中。
  2. 表单可以根据上一个关联对象进行预填，并且可以提供一个排序属性来自定义“上一个”的定义。

iframe

可以如上文所述添加此部分，并如Slate中所述进行配置。

批量编辑多个对象

默认情况下，基于对象的表单会禁用此功能。在表单右侧的可视化编辑器中，打开 设置 选项卡并切换 允许此表单批量编辑（在操作菜单中） 即可启用该功能。

启用后，请按照以下步骤批量编辑多个对象：

  1. 导航至对象资源管理器，选择需要编辑的对象。
  2. 选择 打开方式 按钮，然后选择 在 [表单名称] 中编辑 [选中对象数量] 个对象。
  3. 表单将在对话框中打开，允许您覆盖所有选中对象上的属性。

处于批量编辑模式下的表单将覆盖所有选中对象在该表单中的所有属性。我们建议在此类表单中仅使用必填字段。请使用 required 验证器以确保在提交前已填写相关值。

默认情况下，一次最多可以通过一个表单编辑 200 个对象。如果您的用例需要更高的上限，请联系您的 Palantir 代表。
