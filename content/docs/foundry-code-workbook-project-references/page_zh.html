项目引用

在 Foundry 中，项目既界定了相关工作的概念边界，也构成了用于应用和管理访问权限的安全边界。跨项目边界使用数据时必须格外谨慎。

引用资源

项目引用为具备较高权限的用户（通常是数据集或流水线的数据主人）提供了一种机制，允许他们发现并使用其他项目中的数据。通过在数据集导入项目时进行明确确认，项目引用为跨项目移动数据增加了一层额外的审查机制。

要引用资源，请前往项目根级别的项目导航面板。点击 +添加 按钮以添加对数据集或模板的引用。在下图中，数据集 flights 和 training_data 以及代码工作簿模板 scatterplot template 被作为引用添加到了项目中。这意味着项目中的代码仓库和代码工作簿可以将 flights 和 training_data 用作输入，而 scatterplot template 模板则可供项目中的代码工作簿使用。

  • 要引用资源，您必须对该资源拥有 compass:import-resource-from 权限（通常由查看者角色扩展而来），并对目标项目拥有 compass:import-resource-to 权限（通常由编辑者角色扩展而来）。
  • 可以使用自定义角色对这些角色进行定制。

项目级工作簿

所有新建的代码工作簿均为项目级，这意味着它们会遵守项目边界，并使用范围限定于该项目的令牌。

在项目级工作簿中：

  • 所有输入数据集和模板必须处于项目范围内：它们必须与工作簿位于同一项目中，或者已作为引用添加到该项目中。
  • 所有输出数据集必须与工作簿位于同一项目中。
  • 使用作业令牌对 Foundry 服务发起的 API 调用将会失败。

如果项目级工作簿中存在任何不合规的输入或输出，这些转换操作将无法构建，且其作业规范也不会更新。

在项目级工作簿中，不合规的输入和输出将列示于项目范围对话框中。如果您具备必要的权限，可以在该对话框中为范围外的输入数据集和模板添加引用。或者，您也可以使用项目根级别的项目摘要面板来完成此操作。

在项目级工作簿中，不合规的输入和输出还会以红色图标进行标记。

在工作簿中启用项目范围限定

对于在默认启用项目范围限定功能之前创建的工作簿，系统会提示您启用该功能。项目范围对话框将列示所选分支上不合规的输入和输出。在主分支上启用项目范围限定之前，您必须先解决这些问题。如果您不启用项目范围限定，工作簿将继续像以前一样使用用户令牌运行。您仍可继续正常工作、编辑并在工作簿中运行构建。您依然能够访问 Foundry API、使用超出项目范围的导入数据集等。

项目范围限定只能在主分支上启用。一旦在主分支上启用了该功能，其他分支仍可能存在不合规的输入或输出。这些内容将列示在相应分支的项目范围对话框中。项目范围限定一经启用，便无法通过 UI 将其禁用。
