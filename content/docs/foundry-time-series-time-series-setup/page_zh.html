设置

本文档旨在指导您在 Foundry 应用程序中创建和配置用于分析的时序对象类型及属性。

在继续设置之前，请查看并了解在 Ontology 中设置时序的两种可用选项。如果您已经开始设置但不确定进行到哪一步，请查看下方的设置检查点章节，以了解从何处继续。

开始使用

在开始设置可用于分析的时序对象类型和属性之前，您需要在 Foundry 中拥有一个或多个包含您的时序的数据集。如果有多个数据集，一个系列的所有值应包含在同一个数据集中。常见起点的示例请参见设置检查点。

首先，导航到包含时间戳列的数据集预览，并从分析数据操作菜单中选择设置时序。

这将启动一个概述页面，并带您进入我们的时序设置助手。以下文档将针对助手引导您完成流程的过程中可能需要的任何数据转换提供更深入的解释。

您也可以直接导航至 https://<domain>/workspace/ontology/home/overview/time-series-setup 来启动设置助手。

设置检查点

使用以下决策树来确定您的起点，或在时序设置过程中恢复进度。

  • 您的原始时序数据是否已经在 Foundry 中？
    • 否：使用数据连接将您的数据同步到 Foundry。
  • 您的原始时序数据是否只有时间戳类型的时间戳列？
    • 否：在数据集应用中编辑架构
  • 您的原始数据是否类似于以下示例？

    • 单个数据集包含对象键、时间戳列以及多个列，每列包含该时间点的值。



    • 包含对象信息的数据集，加上多个数据集，每个数据集包含单个系列的键、时间戳列和值。



    • 否：您可能需要转换数据，使其具有类似于这些示例之一的架构。如果不这样做，您可能仍然可以继续，但您需要知道如何自行转换数据，以生成有效的时序对象类型底层数据集和时序同步。

  • 如何启动设置助手？

    • 查看时序数据集（即带有时间戳列的数据集），并从分析数据操作菜单中选择设置时序（参见开始使用）。这将显示向导对话框，随后进入我们的设置助手。



    • 或者，直接导航至 https://<domain>/workspace/ontology/home/overview/time-series-setup 来启动设置助手。

  • 您的对象类型是否已存在？
    • 是：启动设置助手并选择选择现有对象。
    • 否：启动设置助手并选择创建新对象类型。
  • 您是否有对象类型底层数据集？
    • 是：在创建新对象类型对话框中选择它。
    • 否：导航到管道构建器，将数据从通用形状转换为时序对象类型底层数据集的形状（参见创建或选择时序对象类型）。
  • 您是否已将时序属性 (TSP) 添加到您的对象类型中？
    • 是：您已准备好使用时序对象类型。有关在 Foundry 应用程序中使用时序的不同方法，请参见在 Foundry 中使用时序。
    • 否：有关后续步骤，请参见配置时序属性。
  • 您是否有时序同步？
    • 是：完成配置时间序列属性，并在对话框的第二步中选择您的同步。
    • 否：按照配置时间序列属性中有关创建时间序列同步的说明进行操作。
      • 确保时间序列同步的序列 ID 与支撑数据集的对象类型的序列 ID 相匹配
  • 如何使用我刚设置的时间序列属性？
    • 请参阅在 Foundry 中使用时间序列，探索在 Foundry 中分析时间序列的各种方法。
