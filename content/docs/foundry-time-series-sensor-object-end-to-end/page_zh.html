传感器对象类型用例

传感器对象类型是时间序列数据的一种更高级配置，其中传感器对象持有其链接父对象（也称为根对象）的传感器数据。查阅时间序列文档，以确定时间序列属性设置或传感器对象类型配置是否适合您的用例。

本指南将逐步说明如何在 Pipeline Builder 中编写数据管道、如何在 Ontology Manager 中设置传感器对象类型，以及如何利用示例航空本体和 Foundry 的时间序列功能，创建 Quiver 仪表板和 Workshop 模块。

航空本体由示例 Flight、Carrier、Route、Airport 和 Flight Sensor 对象类型构成。Flight 通过这些对象上的 flight_id 外键与 Aircraft、Flight Sensor、Route、Airport 和 Carrier 对象建立链接。

该航空本体源自概念数据的参考本体，您的部署实例可能无法使用。无论您的部署实例是否可用，基于此参考本体构建的这些示例，都将作为您使用传感器对象类型创建自己的管道、对象类型和 Workshop 模块时的参考。

依照本指南创建的 Workshop 模块，将允许您查看所选航班的传感器时间序列数据并进行交互。

以下指南将引导您完成创建和支持该 Workshop 模块的步骤：

  1. 在 Pipeline Builder 中创建传感器对象类型数据
  2. 使用 Ontology Manager 创建传感器对象类型
  3. 在 Workshop 和 Quiver 中使用传感器对象类型时间序列数据
