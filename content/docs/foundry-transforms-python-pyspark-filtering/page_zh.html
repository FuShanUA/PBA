过滤

Filter, where

DataFrame.filter(expression)

返回一个新的 DataFrame，其中包含由布尔表达式决定的行子集。expression 参数是一个布尔列表达式，可以通过多种方式派生。

请在转换操作的开头而非结尾处使用 filter，以减少不必要的计算开销并提升构建性能。

如果数据集包含大量记录，但您只关注符合特定条件的行子集。

已复制！
1df = df.filter(F.col("age") >= 21) # 将数据集限制为仅包含年龄超过 21 岁的人员

where 是 filter 的别名，两者的作用完全相同（具体使用哪个方法名取决于你或团队在代码可读性上的偏好）：

已复制！
1df = df.where(F.col("age") >= 21) # 将数据集限制为仅包含年龄大于等于 21 岁的人员

你还可以通过以下几种不同的方式串联过滤器：



已复制！
1df = df.filter(F.col("age") >= 21).filter(F.col("age") < 35) # 或者...
2df = df.filter((F.col("age") >= 21) & (F.col("age") < 35)) # 使用括号进行分组比较，以确保正确的求值顺序。

常量（字面量）

在将列与常量或“字面量”（例如硬编码的字符串、日期或数字）进行比较时，PySpark 实际上会将这种基础 Python 数据类型求值为一个“字面量”（与声明 F.lit(value) 的效果相同）。字面量本质上是一个带有静态值的列表达式。在深入探讨之前，认清这一点至关重要，因为它意味着所有与字面量的比较（无论是隐式还是显式）都可以替换为具名列。如此一来，你就能轻松实现依赖同一行中其他列的动态比较。

在某些上下文中，字面量无法被正确解析。例如，在与字符串进行比较时，系统可能无法明确区分你是指代以该字符串命名的列，还是指代字符串本身。



已复制！
1df = df.filter("X" == "Y") # X 和 Y 指的是列名还是字面量？
2dff = df.filter(F.col("X") == F.lit("Y")) # 表达明确。

逻辑运算

PySpark 提供了多种二元逻辑运算。这些运算最终都会计算为布尔列表达式实例，可用于组合各种条件。

由于逻辑运算符的参数是整列数据，而非 Python 原生基本类型，我们不能使用熟悉的 and 或 or Python 运算符。这些运算符要求两个参数都已经求值为单个布尔值。PySpark 能够解析 &（按位与）、|（按位或）运算符以及 ~ 取反（波浪号）符号，从而构建出可以在所有行上高效运行的 SQL 查询。

这种机制的一个优势在于，它允许使用命名变量来引用二元运算的结果。因此，为了提升可读性与清晰度，尤其是在基于多个属性进行过滤时，你可以为每个比较操作指定一个具有描述性的名称：



已复制！
1# 筛选用户年龄在 [21, 35) 区间内，或者 first_name="John"，或者 last_name=None (空值) 的记录
2at_least_21 = F.col("age") >= 21
3younger_than_35 = F.col("age") < 35
4within_age_range = at_least_21 & younger_than_35
5name_is_john = F.col("first_name") == "John"
6last_name_is_null = F.isnull(F.col("last_name"))
7df = df.where(within_age_range | name_is_john | last_name_is_null)
8return df

这种方式的另一个好处是，我们可以利用逻辑运算来提供过滤逻辑：

  • &：与

    已复制！
    1df = df.filter(condition_a & condition_b)

  • |：或

    

    已复制！
    1df = df.filter(condition_a | condition_b)

  • ^：异或

    已复制！
    1df = df.filter(condition_a ^ condition_b)

  • ~：非

    

    已复制！
    1df = df.filter(~condition_a)

你可以使用 Python 的 for 循环来生成条件判断，但务必复习一下《布尔代数 ↗》，以免产生不必要的错误。以下是如何利用此方法的示例：



已复制！
1def custom_func(col, value):
2    # 基础逻辑或 UDF
3    return output # True/False
4
5values = ["a", "b", "c", "d"]
6condition = F.lit(False)
7for x in values:
8    condition = condition | custom_func(F.col("column"), x)
9df = df.filter(condition)
10return df

like, rlike

like 和 rlike 方法分别支持使用 SQL LIKE 语法和正则表达式语法进行模式匹配。

  • 对于简单的子串搜索，请使用 like。
  • 对于更复杂的模式匹配，请使用 rlike。

Column.like(sql_like)

基于 SQL LIKE 匹配返回一个布尔列，匹配条件由字符串字面量或列提供：

已复制！
1df = df.filter(F.col('name').like('Al%'))
agename
2"Alice"

SQL LIKE 通配符：

  • %：表示 ≥ 0 个字符
  • _：表示单个字符

示例（来自 w3schools ↗）：

LIKE 操作符描述
Column.like('a%')查找所有以 "a" 开头的值
Column.like('%a')查找所有以 "a" 结尾的值
Column.like('%or%')查找任意位置包含 "or" 的值
Column.like('_r%')查找第二个字符为 "r" 的值
Column.like('a_%_%')查找所有以 "a" 开头且长度大于等于 3 个字符的值
Column.like('a%o')查找所有以 "a" 开头并以 "o" 结尾的值

Column.rlike(regex)

基于正则表达式匹配返回一个布尔列表达式，该正则表达式由字符串字面量或列提供：

Copied!
1df = df.filter(F.col('phone').rlike('[0-9]{3}(?:.+)?[0-9]{3}(?:.+)?[0-9]{4}'))

Starts with, ends with, contains

Column.startswith(string)

返回一个布尔类型的列表达式，用于指示该列的字符串值是否以参数中提供的字符串（字面量或其他列）开头。

Copied!
1df = df.filter(F.col("id").startswith("prefix-"))

Column.endswith(string)

返回一个布尔类型的列表达式，用于指示该列的字符串值是否以参数中提供的字符串（字面量或其他列）结尾。



已复制！
1df = df.filter(F.col("id").endswith("-suffix"))

Column.contains(string)

返回一个布尔列表达式，用于指示该列的字符串值是否包含参数中提供的字符串（字面量或其他列）。



已复制！
1df = df.filter(F.col("id").contains("string"))

Between

Column.between(lowerBound, upperBound)

返回一个布尔表达式，如果该表达式的值处于下界和上界（包含边界值）的字面量或列之间，则该布尔表达式求值结果为 True。



已复制！
1within_range = F.col("age").between(10, df.upperBound).alias("age_within_range")
2df = df.select(df.name, df.upperBound, df.age, within_range)
