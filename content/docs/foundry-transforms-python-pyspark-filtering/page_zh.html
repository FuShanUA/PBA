过滤

筛选，其中

DataFrame.filter(expression)

根据布尔表达式返回一个新的 DataFrame，其中包含由布尔表达式决定的行子集。 expression 参数是一个布尔列表达式，可以通过多种方式生成。

在转换的开始阶段使用 filter 而不是在末尾，以减少不必要的计算工作并提高构建时间性能。

如果您的数据集包含大量条目，但您只关心基于某种条件存在的行的一个子集。

Copied!
1df = df.filter(F.col("age") >= 21) # limit dataset to only people over age 21

where 是 filter 的别名，其功能完全相同（选择取决于你或你的团队对哪种方法名称更清晰的偏好）：

Copied!
1df = df.where(F.col("age") >= 21) # limit dataset to only people over age 21

你也可以以几种不同的方式链接过滤器：

Copied!
1df = df.filter(F.col("age") >= 21).filter(F.col("age") < 35) # or...
2df = df.filter((F.col("age") >= 21) & (F.col("age") < 35)) # group comparisons using parenthesis to ensure correct order of evaluation.

常量（字面量）

每当您将某一列与常量或“字面量”进行比较时，例如一个硬编码的字符串、日期或数字，PySpark 实际上会将这种基本的 Python 数据类型转换为一个“字面量”（与声明 F.lit(value) 相同）。字面量只是一个具有静态值的列表达式。在我们继续之前，了解这一重要区别很重要，因为这意味着所有与字面量的比较（无论是隐式的还是显式的）都可以用命名列来替代。这意味着您可以轻松地创建依赖于同一行中其他列的动态比较。

在某些情况下，字面量可能无法被正确解释。例如，当与字符串进行比较时，可能不清楚您是想引用名为该字符串的列还是字符串本身。

Copied!
1df = df.filter("X" == "Y") # Do X and Y refer to columns or literals?
2dff = df.filter(F.col("X") == F.lit("Y")) # Unambiguous.

逻辑操作

PySpark 提供了多种二进制逻辑操作。这些操作始终会被评估为布尔列表达式，并可用于组合条件。

由于我们的逻辑运算符的参数是整列，而不是 Python 原始类型，因此我们不能使用熟悉的 and 或 or Python 运算符。这些运算符期望两个参数都已经评估为单个布尔值。PySpark 能够将 &（按位与）、|（按位或）运算符和 ~ 否定（波浪号）符号解释为构建一个在所有行上运行非常高效的 SQL 查询。

一种帮助我们的方法是，它允许命名变量来引用二元运算的值。因此，为了提高可读性和清晰度，尤其是在您对多个属性进行筛选时，您可以为每个比较进行描述性命名：

Copied!
1# filter for records where users are aged [21, 35), or has first_name="John", or last_name=None (null)
2at_least_21 = F.col("age") >= 21
3younger_than_35 = F.col("age") < 35
4within_age_range = at_least_21 & younger_than_35
5name_is_john = F.col("first_name") == "John"
6last_name_is_null = F.isnull(F.col("last_name"))
7df = df.where(within_age_range | name_is_john | last_name_is_null)
8return df

另一种帮助我们的方法是我们可以利用逻辑运算来提供过滤逻辑：

  • &: 与

    Copied!
    1df = df.filter(condition_a & condition_b)

  • |: 或

    Copied!
    1df = df.filter(condition_a | condition_b)

  • ^: 异或

    Copied!
    1df = df.filter(condition_a ^ condition_b)

  • ~: 否定

    Copied!
    1df = df.filter(~condition_a)

可以自由使用 Python 的 for 循环生成条件语句，但请务必复习 布尔代数 ↗ 以避免不必要的错误。以下是此方法的一个应用示例：

Copied!
1def custom_func(col, value):
2    # basic logic or UDF
3    return output # True/False
4
5values = ["a", "b", "c", "d"]
6condition = F.lit(False)
7for x in values:
8    condition = condition | custom_func(F.col("column"), x)
9df = df.filter(condition)
10return df

Like，rlike

like 和 rlike 方法分别使用 SQL LIKE 和正则表达式语法进行模式匹配。

  • 对于简单的子字符串搜索，请使用 like。
  • 对于更复杂的模式匹配，请使用 rlike。

Column.like(sql_like)

根据SQL LIKE匹配返回一个布尔列，由字面字符串或列提供：

Copied!
1df = df.filter(F.col('name').like('Al%'))
agename
2"Alice"

SQL LIKE通配符：

  • %：表示≥0个字符
  • _：表示单个字符

示例（来自w3schools ↗）：

LIKE运算符描述
Column.like('a%')查找以"a"开头的任何值
Column.like('%a')查找以"a"结尾的任何值
Column.like('%or%')查找任何位置包含"or"的值
Column.like('_r%')查找第二个位置包含"r"的值
Column.like('a_%_%') 查找以"a"开头且长度≥3的值
Column.like('a%o')查找以"a"开头且以"o"结尾的值

Column.rlike(regex)

根据提供的字面字符串或列返回一个布尔列表达式：

Copied!
1df = df.filter(F.col('phone').rlike('[0-9]{3}(?:.+)?[0-9]{3}(?:.+)?[0-9]{4}'))
name（名称）phone（电话）
"Alice""412-512-1234"
"John""(555) 123-5123"
"Jane""4121234444"

Column.startswith(string)

返回一个布尔列表达式，表示该列的字符串值是否以参数中提供的字符串（字面量，或其它列）开头。

Copied!
1df = df.filter(F.col("id").startswith("prefix-"))

Column.endswith(string)

返回一个布尔列表达式，表示该列的字符串值是否以参数中提供的字符串（字面量，或其它列）结尾。

Copied!
1df = df.filter(F.col("id").endswith("-suffix"))

Column.contains(string)

返回一个布尔列表达式，表示该列的字符串值是否包含参数中提供的字符串（字面量，或其它列）。

Copied!
1df = df.filter(F.col("id").contains("string"))

子字符串

Column.substr(startPos, length)

返回一个字符串列表达式，该表达式评估列值的子字符串。

参数：

  • startPos（起始位置） - 起始位置，从1开始计数（整数或列）
  • length（长度） - 子字符串的长度（整数或列）

  1. 创建子字符串列

    Copied!
    1df = df.select(F.col("name").substr(1, 3).alias("col"))

  2. 根据子字符串进行筛选

    Copied!
    1df = df.filter(F.col("phone").substr(5, 3) == "555")

介于...之间

Column.between(lowerBound, upperBound)

返回一个布尔表达式，当表达式的值在 lowerBound 和 upperBound 文字或列（包括边界）之间时，该表达式被评估为 True。

Copied!
1within_range = F.col("age").between(10, df.upperBound).alias("age_within_range")
2df = df.select(df.name, df.upperBound, df.age, within_range)
