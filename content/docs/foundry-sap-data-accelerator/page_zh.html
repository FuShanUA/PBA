SAP 数据加速器

SAP 数据加速器是一个由 SAP 管理的连接服务，通过您的 SAP 云连接器将 Foundry 的流量路由到本地 SAP 系统。而不是直接拨号到 SAP 主机，Foundry 会将其流量通过 SAP 数据加速器进行隧道传输，然后将其传递给您的 SAP 云连接器 ↗，再传送到后端 SAP 系统。

要使用 SAP 数据加速器，请完成下面的一次性 SAP 设置，然后将现有的 SAP ERP 或 SAP SLT 源指向它。通过 SAP 数据加速器路由源不会改变您同步的数据、如何对 SAP 插件进行身份验证或您的同步配置；只有到 SAP 系统的网络路径会发生变化。

开始之前

SAP数据加速器仅支持使用 Foundry worker 的源。如果您的源使用代理工作程序，请先将其迁移到 Foundry worker。

使用 SAP 数据加速器需要 Palantir 颁发的客户端证书。请向您的 Palantir 代表请求该证书。

SAP 设置

在 SAP 的工具中完成 SAP 数据加速器的配置和设置，由您的 SAP 管理员负责。按照 SAP 的数据加速器文档 ↗ 来执行以下操作：

  • 配置 SAP 数据加速器。
  • 使用 Palantir 颁发的客户端证书 将 Foundry 注册为合作伙伴系统。
  • 将您的 SAP Cloud Connector ↗ 连接到 SAP 数据加速器，并将虚拟主机和端口映射到您希望 Foundry 访问的后端 SAP 系统。

完成 SAP 设置后，请确保您拥有以下值以 配置 Foundry 源：

  • 控制平面 URL： SAP 数据加速器控制平面的 URL（例如，https://<tenant>.sharing.hdl.<region>.hanacloud.ondemand.com）
  • 虚拟主机和虚拟端口： 在 SAP 云连接器中映射到后端 SAP 系统的值
  • 云连接器位置 ID： （可选）当同一 SAP 数据加速器租户注册了多个云连接器时需要

配置 Foundry 源

除了源的常规设置外，请在您的 SAP ERP 或 SAP SLT 源上进行以下配置。

1. 分配证书

从 Palantir 发行的客户端证书（您在 SAP 数据加速器中注册为合作伙伴系统的同一证书）创建一个 控制面板客户端证书，然后 将其附加到源。

根据您的SAP Data Accelerator配置，您可能还需要添加服务器证书，以便Foundry工作者可以信任与SAP Data Accelerator的TLS连接。如果SAP Data Accelerator端点提供的证书不是由公共信任的证书颁发机构签名的，请将相关的服务器证书添加到源中。

2. 将主机和端口设置为Cloud Connector虚拟主机

将源的主机（host）和端口（port）设置为在您的SAP Cloud Connector中定义的虚拟主机和虚拟端口。

3. 启用SAP Data Accelerator选项

将可选的SAP Data Accelerator属性添加到源，并提供以下内容：

字段（Field）是否必需（Required）描述（Description）
控制平面URL是您在SAP端设置期间收集的SAP数据加速器控制平面URL（例如，https://<tenant>.sharing.hdl.<region>.hanacloud.ondemand.com）。
云连接器位置ID否面向您SAP系统的云连接器的位置ID。仅当同一SAP数据加速器租户注册了多个云连接器时才需要；否则留空。

4. 配置出站

允许从Foundry工作节点到以下两者的出站HTTPS（端口443）：

  • 您上面配置的控制平面主机（例如，<tenant>.sharing.hdl.<region>.hanacloud.ondemand.com）。
  • 数据平面，它动态分配每个连接的子域名。允许SAP数据加速器域名下的通配符（例如，*..hanacloud.ondemand.com）。

有关如何配置网络访问，请参阅出站策略文档。
