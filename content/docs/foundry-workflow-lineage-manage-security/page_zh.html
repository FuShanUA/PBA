管理安全

之前称为 Workflow Builder 的应用程序现在称为 Workflow Lineage。

要开始使用 Workflow Lineage，请打开一个 Workshop 应用程序或函数仓库，并使用快捷键 Command + I（macOS）或 Ctrl + I（Windows）来查看显示应用程序后端对象、操作和函数的相关 Workflow Lineage 图表。

您也可以直接导航到 Workflow Lineage 应用程序，并手动将资源添加到图表中。

Workflow Lineage 中的颜色图例允许您查看本体论和资源权限。本体论权限有两种类型：

  • 定义：指定用户是否可以查看或编辑实体定义。
  • 数据和执行：指定用户是否可以查看对象数据或执行操作。

要了解特定用户的访问权限，您可以将用户名输入到查看为下拉菜单中以进行预览。

对资源的本体论角色进行批量更新

您也可以按照以下步骤对对象和操作上的本体论角色权限进行批量编辑：

  1. 导航到一个资源，右键点击它，然后从上下文菜单中选择编辑权限。

这将带您进入编辑本体论资源权限窗口，并显示所选资源。

  1. 在要授予的本体论角色部分中，搜索您想添加的组。选择角色后，您应该会看到它显示在所选组旁边。

  1. Confirm the action by selecting Grant roles in the bottom right. A dialog will appear with the prompt, Are you sure you want to share these resources?.

  2. Select Yes, share to proceed. Note that this action is immediate and cannot be undone.

批量更新操作提交标准

此外，您可以将操作的提交标准更新为与源操作的提交标准匹配。从工作流谱系图中，选择您希望更新的操作。然后，从底部面板导航到 更新提交标准。

在面板的左侧，选择您希望应用其提交标准的源操作。可以在所选源操作下查看源操作的提交标准。

当完成时，选择蓝色的更新 x 个操作按钮，其中x是要更新的操作数量。这将生成一个您可以批准并提交的建议，以使更改生效。
