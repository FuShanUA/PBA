按用户角色划分的后续步骤

既然您已对 Palantir 平台有了基本了解并掌握了其核心概念，接下来就可以探索与您角色最相关的平台功能了。

在 Palantir 平台中，角色之间的界限并非一成不变，某些职责和工作流可能无法完全归入单一角色。您可能需要承担多种角色；如果您的组织刚刚开始使用 Palantir，或者您在一个小型实施团队中工作，日常工作中可能会用到平台的许多不同模块。

此外，您还可以通过查阅《应用程序参考》来探索 Palantir 平台，该文档提供了主要平台应用程序的宏观概览。

标准用户角色

考虑到这一点，以下角色是大多数使用 Palantir 的组织通常采用的。下面我们将讨论这些高层级角色，以及各类用户如何入门：

  • 数据工程师
  • 应用程序构建者
  • 分析师
  • 数据科学家
  • 平台管理员
  • 数据治理

如果您想立即开始，可以直接进入 Palantir 平台。

数据工程师

Palantir 的数据集成层为平台中的所有其他工作奠定了基础。通过构建和维护数据管道，数据工程师能够产出高质量、强相关且高频更新的数据集，以满足组织的需求。平台提供了丰富的工具来保障数据管道长期的稳定运行，包括程序化的健康检查以及对底层计算的透明化监控。

数据工程师使用的主要工具包括用于编写数据管道的《管道构建器》和《代码仓库》，以及用于端到端可视化的《数据血缘》。数据工程师还应掌握如何使用《推荐健康检查》、《监控视图》以及血缘关系来运维已部署的管道。数据工程师需要熟悉数据管道的概念，并深入理解在平台中如何构建高质量的数据管道。

了解有关数据管道的更多信息。

应用程序构建者

Palantir 的本体和应用程序构建功能使您能够为最终用户创建量身定制的应用程序。这些最终用户通常是组织内的操作人员，他们基于数据做出决策。自定义应用程序不仅能够向用户展示数据，还能通过在本体中配置的操作类型，捕获用户输入的信息。

应用程序构建者需要熟悉 Palantir 的《本体》，这通常是应用程序构建者与数据工程师协作建立工作流开发数据基础的层。构建者可以在《本体管理器》中创建和维护组织的本体，然后使用《工作流血缘》来理解对象、操作、函数、大语言模型（LLM）和应用程序如何协同工作。

为了创建和交付应用程序，构建者可以使用《工作坊》在ontology之上进行点击式应用程序构建，使用《自定义小部件》扩展工作坊，或使用《Pilot》从自然语言描述开始构建。对于代码优先开发，构建者可以使用《OSDK》和《OSDK React 应用程序》，在《开发者控制台》中管理应用程序配置和 SDK 生成，编写《函数》来处理共享业务逻辑，或在《Slate》中构建应用程序。构建者还可以使用《AIP 逻辑》创建由大语言模型支持的工作流，并使用《Automate》触发本体驱动的工作。

了解有关应用程序构建的更多信息。

数据科学家

Palantir 平台支持使用代码进行数据分析，以及开发、评估和部署机器学习模型。此功能建立在数据集成层的严谨性之上，能够像对待数据集一样，为模型提供血缘关系和可复现性。最终形成的环境，使得分析和机器学习能够建立在高质量数据和共享的建模工作流之上。

在 Palantir 平台中，数据科学家通常使用 代码工作簿，这是一款专为基于代码的分析和机器学习模型开发而设计的应用。代码工作簿支持使用 Python、R 和 SQL 编写代码，以访问、归一化并分析由数据工程师准备的高质量数据集。生成的分析和模型随后可连接至模型集成，通过模型目录进行共享，利用AIP 评估进行评测，并集成到本体中，供应用程序和工作流使用。

或者，数据科学家也可以使用代码工作区，在自己偏好的第三方 IDE 中开展工作。代码工作区容器与 Palantir 生态系统的其余部分相集成，将 JupyterLab® 和 RStudio® Workbench IDE 与 Palantir 平台在安全、分支和资源管理方面的优势结合起来。

了解更多关于模型集成和基于代码的分析的信息。

分析师

Palantir 可用于构建安全且高质量的数据基础，分析师借此能够查找并探索与所需解答问题相关的数据。平台提供了丰富的工具，支持分析多种格式的数据——包括表格、关系、时序、地理空间等格式。分析产生洞察后，您可以通过创建仪表板使其具备可重复性，或使用报告工具展示分析结果。

分析师通常使用Contour 在平台中探索数据集并进行大规模开放式分析，并使用Quiver 分析本体中的数据及相关时间序列。分析师还可以使用AIP 分析师 通过自然语言探索基于本体的问题。这些应用程序支持将临时分析的结果转换为仪表板、报告或记事本文档，以便与同事共享结果。

了解更多关于分析功能的信息。

平台管理员

平台管理员可以使用 Palantir 专用的管理工具来配置平台、管理并了解平台的使用情况，同时确保组织数据得到安全管理。

平台管理员通常会设置身份验证以连接到组织的身份提供商，配置应用访问权限，然后设置数据连接以使数据流入平台。随着平台使用的深入，管理员可以使用资源管理来管理资源消耗，使用开发者控制台支持自定义应用开发，并使用监控视图帮助团队观察运行状况。当启用相关功能时，管理员还应了解Palantir MCP 和本体 MCP 的访问注意事项。

了解更多关于平台管理的信息。

数据治理

Palantir 提供了保护数据并为数据治理负责人提供透明度的工具。这些工具确保数据在 Palantir 平台中进行转换及用于面向用户的应用时得到妥善保护。同时，它们还保留了内省和验证“谁有权访问哪些信息”的能力。

承担数据治理角色的用户应了解平台中广泛的数据安全工作流，从保护数据基础到保护敏感数据。这些功能构建于 Palantir 的数据安全概念之上，即项目和标记。如果这些控制机制是组织工作流的一部分，治理负责人还应了解对象权限设置、应用限制、OSDK 权限、本体 MCP 限制以及工作流血缘关系安全可见性。
