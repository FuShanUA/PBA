为 Marketplace 产品添加数据集转换

使用 Foundry DevOps 将您的数据集转换纳入 Marketplace 产品，以便其他用户安装和重用。了解如何创建您的第一个产品。

支持的功能

在打包数据集转换（及其生成该转换的代码仓库）时，所有必需的依赖项都会作为产品的一部分进行存储；这保证了转换是自包含的，并且可以在任何地方成功运行。仓库可以引入 Maven、PyPI 和 Conda 依赖项。

支持 Python、Java 和 SQL 转换。转换必须基于最新模板的仓库生成，否则可能出现打包错误。要进行调试，请在 Code Repositories 应用程序中升级您的仓库。如果转换能够成功打包，则不会导致任何安装或运行时错误。

安装时作为输入项提供的数据集，其列名和变量类型必须与源数据集完全一致。

这是因为来自源输入数据集的所有数据集列（例如，用作数据集转换输入并随后包含在 Marketplace 产品中的 airplane 数据集）在安装时都将作为必需的输入项，无论这些列是否在数据集转换中被引用。

支持的功能包括：

  • 增量转换
  • 取消标记工作流
  • Spark 配置文件
  • 遥测
  • 库
  • 外部转换
  • 无模式数据集

将数据集转换添加到产品中

要将数据集转换添加到产品中，请先创建产品，然后添加输出项。选择添加文件选项，从 Compass 文件系统中导航至该数据集转换，并将其添加到您的产品中。

在某些情况下，一个转换可能会生成多个输出数据集。若遇到此情况，必须将所有生成的数据集包含在产品中。

仓库打包选项

打包仓库有三种方式。

  • 排除所有源代码： 仓库打包时不包含任何源代码。该仓库的唯一目的是保存运行转换时所需的依赖项。此方法包含已编译的用户代码和所有传递依赖项。
  • 包含最新源代码，排除版本历史： 仓库包含源代码和必要的构建产物；但是，Git 历史（包括标签）不会被保留。这是将仓库作为只读文档进行发布的推荐方式。
  • 包含源代码和完整版本历史： 仓库将按原样保留在产品中。整个 Git 历史会在打包时保存，并在安装时还原。这是唯一允许您在安装后从 Code Repositories 应用程序中运行检查并重新构建转换的模式。
