在变量中存储值

您在变量编辑器中定义的变量在整个Slate应用程序中都是可访问的。您可以在查询和小部件中引用它们，甚至可以通过事件或URL设置它们的值。

变量的有效类型为Number, String, Boolean, Array, Object, 和 Null。

创建一个变量

要创建一个变量，请按照以下步骤操作：

  1. 在左侧边栏中，选择(x)图标以打开变量编辑器。将鼠标悬停在(x)图标上，会显示单词变量。
  2. 选择+ 添加新变量。
  3. 选择变量的作用域范围 - 共享（可在每一页上访问），或本地（仅在创建变量时编辑的页面上可访问）。
  4. 选择名称列中的文本，并为变量输入新名称。

Slate 将检测变量名称冲突并阻止用户在输入有效名称之前保存变量。 共享变量名称在整个页面、小部件、事件、查询和函数中必须唯一。 本地变量名称必须在页面内唯一，且不能与 环境变量 冲突。

  1. 为变量输入默认值。Slate 将根据输入的值推断变量类型。

变量的值不会在页面加载之间持久化。当 Slate 应用程序重新加载时，将使用默认变量值。 要在页面加载之间持久化变量值，请使用 用户存储 变量。

变量编辑器不仅显示每个变量的页面范围、名称和默认值，还提供变量当前值的只读视图，以防它通过事件或直接从URL进行了更改。

表格变量

数据集（Dataset）面板已迁移到变量（Variables）面板。数据集现在作为共享变量存储，但其功能保持不变。

变量（Variables）面板支持将表格数据作为变量存储。您可以使用电子表格编辑器直接输入数据或上传CSV文件。

创建一个表格变量

按照以下说明创建一个表格变量：

  1. 在左侧边栏中，选择X图标以打开变量编辑器。
  2. 选择+ 添加新变量。
  3. 选择变量页面的作用域（共享或本地）。
  4. 为变量输入名称。
  5. 在默认值字段中，输入列导向的 JSON 以初始化表结构，例如：{"column1": ["row1", "row2"], "column2": ["row1", "row2"]}，或者如果值为空，则直接选择表格选项卡。

表格选项卡提供了一个电子表格编辑器，允许您直接编辑数据。

表格选项卡在变量值为空时启用，{}，或者包含格式为 { columnName: [value1, value2, ...], ... } 的列导向 JSON。所有列必须具有等长的数组。如果您的变量包含其他格式（如行导向 JSON，例如 [{...}, {...}]），则 表格 选项卡将被禁用。

上传CSV文件

您可以按照以下步骤上传CSV文件以填充表格变量：

  1. 选择一个现有的表格变量或创建一个新的。
  2. 打开表格选项卡。
  3. 使用上传选项从您的计算机中选择一个CSV文件。
  4. CSV数据将被解析并在电子表格编辑器中显示。

电子表格编辑器支持以下功能：

  • 直接编辑单元格值
  • 右键点击以获取上下文菜单，获得更多选项
  • 添加和删除行和列

Slate应用程序默认大小限制为2 MB。避免上传大型CSV文件，因为它们会占用此限制。如果您的应用程序超过大小限制，将无法保存更改。对于大型数据集，建议使用查询或Code Repositories（代码仓库）。 OSDK 获取数据。

通过事件更新变量

事件可以用来设置变量的新值。每个变量都会自动提供一个 .set 事件。要更新变量的值，只需在事件内容中返回一个新的、合法的值。事件逻辑也可以将变量的当前值作为输入。您还可以在更新时更改变量值的类型（例如，从字符串改为数字）；但是，这可能会破坏变量的依赖关系。

URL中的变量

你可以将变量用于URL查询参数（附加到URL以帮助设置Slate页面状态的额外信息）。URL参数始终会覆盖默认参数。

一般来说，使用以下语法将变量添加到URL中：

  • 单个变量：?variableName=value
  • 多个变量：?variableName=value&otherVariableName=otherValue

请注意，查询参数是区分大小写的。此外，从URL查询参数中检索到的变量值将是字符串，即使传递的值是数字、布尔值或对象。这些值可以使用 Handlebar helpers 或通过 Functions 转换为所需的类型。

Slate 系统变量

Slate 提供了两种类型的系统变量：

  1. 环境 变量。
  2. 用户存储 变量。

系统变量具有特殊的行为，不能通过 URL 设置。

环境变量

$global 变量提供了对环境特定信息的访问。

属性 描述
locale返回用户的语言区域设置。
app.isEditMode如果应用处于编辑模式，返回true；如果应用处于查看模式，返回false。
app.rid返回Slate文档的RID。
user.domain返回用户的认证域。
user.email返回用户的电子邮件地址。
user.familyName返回用户的姓氏。
user.firstName返回用户的名。
user.groups返回用户所属的所有认证组列表。
user.id返回用户的唯一标识符。
user.username返回用户的用户名。
window.origin返回当前URL的来源。

示例：在函数中使用环境变量

以下示例使用$global变量的不同属性，以适当的语言向用户致意：

// 'de' represents the German locale
// English is the default language
if ({{$global.locale}} == 'de') {
    return "Willkommen " + {{$global.user.firstName}}
} else {
    return "Welcome " + {{$global.user.firstName}}
}

用户存储变量

在Slate中的sl_user_storage变量在每次应用程序加载时都为每个用户保留其值，与其他在页面加载时重置为默认值的变量不同。

用户存储变量允许应用程序构建者在应用程序上下文中存储单个用户的信息，例如特定应用程序的用户偏好。

通过事件，可以在用户存储中存储值，这些值可以在应用程序的各个会话中访问。用户存储限制为10 kB，并可以通过slate.setUserStorage操作进行设置。当用户在多个浏览器标签或窗口中使用应用程序时，可以调用slate.refreshUserStorage。当在一个标签或窗口中将存储设置为新值时，可以通过调用刷新操作将该值拉入其他窗口。

市场参数变量

本地变量可以标记为市场参数，允许用户通过 市场 安装Slate应用程序时自定义其值。这对于在安装后无需代码更改即可使应用程序可配置非常有用。
