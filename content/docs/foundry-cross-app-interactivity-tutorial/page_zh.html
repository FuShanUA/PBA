创建拖放集成点

以下教程仅涵盖在传输或接受相同媒体类型的拖放区域之间进行拖放操作。例如，如果您的拖拽区域将 Foundry 对象 RID 媒体类型数据添加到 DataTransfer 中，那么您只能将其拖放到接受 Foundry 对象 RID 媒体类型的放置区域上。例如，如果您想在接受 Gotham 对象 媒体类型的放置区域上放置 Foundry 对象 RID 数据，您需要 为您的对象启用类型映射，并在本教程之后遵循 增强教程。

拖放功能允许用户在应用程序内和应用程序之间动态移动数据。以下教程将教您如何在您的应用程序和 Palantir 平台之间拖动数据，包括 Gotham 和 Foundry。只需向您的应用程序添加一个拖放集成点，即可让用户在您的应用程序和 Palantir 平台之间无缝执行工作流程。

实现拖放区域包含两个主要组件：

创建一个拖拽区域：

  1. 使 HTML 元素可拖拽
  2. 向 DataTransfer 添加数据

创建一个放置区域：

  1. 启用可以放置到您的放置区域的项目
  2. 处理放置操作
  3. 美化您的放置区域

我们建议安装我们平台的最新版本，以提高您的应用程序与Palantir平台之间的拖放互操作性。

创建一个拖拽区域

拖拽区域是一个交互式DOM元素，允许用户通过“拖动”它到另一个“放置区域”来“抓取”数据并将其转移到其他地方。拖拽区域会将数据写入DataTransfer对象，以便在用户拖动draggable元素时进行传输。以下步骤详细说明了如何使元素draggable，向DataTransfer对象添加数据以及处理拖拽事件。有关这些概念的详细信息，请参阅拖拽区域和DataTransfer对象。

使HTML元素可拖拽

通过将 draggable 属性设置为 true，可以将 HTML 元素设为拖拽区域。要将数据附加到拖拽事件中，请添加一个 ondragstart 处理程序，该处理程序会将数据设置在 DataTransfer 上。

Copied!
1<p id="p1" draggable="true" ondragstart="handleDragStart    (event)">
2    Draggable Element
3</p>

将数据添加到 DataTransfer

要将数据添加到拖拽事件中，请调用附加 DataTransfer 的 dataTransfer.setData 方法。此方法以给定的媒体类型格式将数据添加到 DataTransfer。有关更多信息，请参阅 MDN DataTransfer 文档 ↗。

Copied!
1function handleDragStart(event) {
2    event.dataTransfer.setData(
3        <media type>,
4        <data>
5    );
6
7   event.dataTransfer.effectAllowed = "move";
8}

将上面的 <media type> 替换为所拖拽数据的媒体类型，例如 application/x-vnd.palantir.rid.phonograph2-objects.object 用于 Foundry 对象 RID。使用 JSON.stringify 将数据转换为字符串，然后将 <data> 替换为数据的序列化版本。请参考 Palantir 媒体类型文档 来确定您用例的正确媒体类型。

在向DataTransfer添加数据时，请考虑安全问题。任何拖放区域都可以访问此数据。

以下是Foundry对象RID的拖动开始处理程序示例：

Copied!
1function handleDragStart(event) {
2    const foundryObjectRids = ["ri.phonograph2-objects.main.object.XXXXXXX", "ri.phonograph2-objects.main.object.YYYYYYY"]
3    event.dataTransfer.setData(
4        "application/x-vnd.palantir.rid.phonograph2-objects.object",
5        JSON.stringify(foundryObjectRids)
6    );
7
8    event.dataTransfer.effectAllowed = "move";
9}

此处理程序将Foundry对象RID媒体类型添加到DataTransfer中。任何接受Foundry对象RID数据的拖放区域在拖动DataTransfer被释放时，都可以接收foundryObjectRids中的数据。

此时，您应该能够从您的拖动区域拖放到接受DataTransfer中媒体类型的拖放区域。根据媒体类型参考Palantir媒体类型，查找接受您的拖动内容的拖放区域。

创建一个拖放区域

拖放区域是DOM中的交互元素，用于接收从拖拽区域“拖放”的数据。拖放区域通过监听ondrop事件，并在事件触发时从事件的DataTransfer中获取信息来工作。有关更多信息，请参阅 拖放区域 和 DataTransfer 对象。

您可以通过添加一个 ondragover 处理程序将HTML元素转换为拖放区域，这会阻止默认的事件行为。添加一个 ondrop 处理程序以处理已拖放的DataTransfer。

步骤如下：

  1. 启用可拖放到您的拖放区域的项目
  2. 处理拖放
  3. 美化您的拖放区域

本教程涵盖创建一个接受 Foundry 对象 RID 媒体类型的拖放区域，但您的实现可以将其替换为任意您选择的媒体类型。

允许将项目拖放到您的放置区域

  1. 编写函数以阻止默认的 dragover 和 dragenter 事件处理。默认事件行为不接受拖放数据，因此我们取消此行为以允许元素接受拖放数据，这对放置区域是必要的。
Copied!
1// this is necessary to ensure the drop event will fire
2// only prevent the default behavior if the drag payload is accepted
3
4function handleDragOver(event){
5    if(event.dataTransfer?.types.includes("application/x-vnd.palantir.rid.phonograph2-objects.object")){
6        event.preventDefault();
7    }
8}
9
10function handleDragEnter(event){
11   if(event.dataTransfer?.types.includes("application/x-vnd.palantir.rid.phonograph2-objects.object")){
12        event.preventDefault();
13    }
14}
  1. 将 handleDragOver 事件处理程序添加到您的放置区域
Copied!
1<div
2  id="my-cool-drop-zone"
3  ondragover="handleDragOver(event)"
4  ondragenter="handleDragEnter(event)"
5>
6  Drop Zone
7</div>

处理拖放

  1. 创建一个函数来处理拖放，通过访问DataTransfer在被拖放后获取数据，并阻止默认事件行为，即打开链接数据。调用DataTransfer getData 函数以字符串格式从DataTransfer返回数据。 getData 如果DataTransfer没有数据或该媒体类型没有数据，则返回空字符串。有关此方法的更多信息，请参阅 MDN getData文档 ↗。

    以下是使用 ondrop 回调从 DataTransfer 获取 Foundry 对象 RID 的示例。将 event.dataTransfer.getData 调用中的媒体类型替换为反映您要传输的数据类型的媒体类型。

Copied!
1function handleDrop(event) {
2    // prevent default open-link behavior
3    event.preventDefault();
4
5    // get Foundry object RIDs data
6    const data = event.dataTransfer.getData(
7        "application/x-vnd.palantir.rid.phonograph2-objects.object"
8    );
9
10    try {
11        if (data != null && data !== ""){
12            // try to parse the returned data
13            const dataParsed = JSON.parse(data);
14
15            // print the data
16            console.log(dataParsed);
17
18            // do something with the data
19            doCoolThingWithFoundryObjects(dataParsed);
20        }
21    } catch (error) {
22        console.error("Unable to parse data", error)
23    }
24}
  1. 按照以下所示，将 handleDrop 函数添加到您的拖放区域组件中：
Copied!
1<div
2  id="my-cool-drop-zone"
3  ondrop="handleDrop(event)"
4  ondragover="handleDragOver(event)"
5>
6  Drop Zone
7</div>

此时，您应该能够将数据拖放到您的拖放区域中。请参考 Palantir 媒体类型文档，查找可以拖放到您的拖放区域上的拖放区域。

  1. （推荐）如果可能的话，在从 DataTransfer 获取数据时建议使用运行时类型守卫/验证器。虽然存在媒体类型到媒体类型的数据连接作为合同，但添加类型守卫可以确保接收到的数据遵循该合同。

请参考以下文档以获取支持的 Palantir 媒体类型：

  • Foundry 对象
  • Foundry 对象集
  • Gotham 对象

自定义您的放置区域

为您的放置区域添加样式至关重要，这样用户可以看到放置区域是否接受拖拽内容。如果没有这个功能，您的放置区域的可用性将不理想。我们建议根据拖拽内容是否适用于放置区域，或者放置区域是否需要执行额外操作来确定是否接受拖拽内容，对放置区域应用不同的样式。在放置后应清除放置区域的样式，这将在下面的步骤中介绍。

本教程结束后，如果放置区域接受拖拽内容，它将会“亮起”，但本教程并未涵盖设计指南中提到的“无效”和“加载”样式。我们建议实现这些附加样式以获得更一致且用户友好的拖放交互体验。

在完成步骤一至四后，当用户将具有可接受媒体类型的负载拖放到您的应用程序上时，您的拖放区域应“亮起”。在完成步骤五至七后，当用户从您的应用程序内部开始拖动时，您的拖放区域也会“亮起”。在完成步骤八后，如下图所示的悬停样式将被应用到您的拖放区域。

  1. 复制CSS类实现。为了给您的拖放区域添加样式，请为您的样式创建一个CSS文件，并从设计指南中添加适当的拖放区域样式类。请注意，根据您的拖放区域的大小，我们有不同的样式建议。

  2. 当您的拖放区域接受拖动负载时添加样式，通过编写一个函数来有条件地添加样式，如果拖动负载包含可接受的媒体类型。该函数使用classList ↗属性，在拖动负载被接受时向我们的拖放区域添加另一个类名。参见设计指南 用于 CSS 拖拽悬停样式。

Copied!
1function conditionallyApplyStylingToDropZone(event, className){
2    // our drop zone accepts the drag payload if it has a Foundry object RIDs media type
3    if(event.dataTransfer?.types.includes("application/x-vnd.palantir.rid.phonograph2-objects.object")){
4        const dropzone = document.getElementById("my-cool-drop-zone");
5        if(dropzone != null){
6            // add the style's classname
7            dropzone.classList.add(className);
8        }
9    }
10}
  1. 编写一个函数，使用 classlist 属性从放置区域移除样式。
Copied!
1function removeStylingFromDropZone(){
2    const dropzone = document.getElementById("my-cool-drop-zone");
3    if(dropzone != null){
4        dropzone.classList.remove(className);
5    }
6}
  1. 当数据从外部拖拽区域拖到你的放置区域时，通过添加 dragenter 和 dragleave 事件处理程序来应用样式。请注意，由于子元素可能会阻止事件进一步向上传播，因此你希望在 capture 事件阶段监听这些事件。有关更多信息，请参阅 MDN 事件监听器捕获文档 ↗。

    在这里，我们需要本地跟踪每个事件触发的次数。这些事件将由页面上的所有子元素触发。
Copied!
1let dragInsideApplicationCount = 0;
2
3function handleDragEnter(event){
4    if(dragInsideApplicationCount === 0){
5        maybeApplyValidStylingToDropZone(event, "valid-small");
6    }
7
8    dragInsideApplicationCount += 1;
9}
10
11function handleDragLeave(event){
12    if(dragInsideApplicationCount === 1){
13        removeValidStylingFromDropZone("valid-small");
14    }
15
16    dragInsideApplicationCount -= 1;
17}
18
19document.addEventListener("drag-enter", handleDragEnter, {capture: true});
20document.addEventListener("drag-leave", handleDragLeave, {capture: true});

上述代码片段只有在从外部应用程序或窗口的拖拽区域启动拖拽时，才会使放置区域亮起。如果您希望从您的应用程序传入的拖拽数据使放置区域亮起，需要按照以下说明添加一个 dragstart 监听器。

此时，您应该能够从外部应用程序或窗口拖拽数据，并且如果您的放置区域接受该拖拽数据，它将亮起。参考 Palantir 媒体类型文档，查找可以拖拽到您的放置区域的拖拽区域。请注意，在拖拽数据被放下后，样式仍然会应用到您的放置区域；有关在放下时移除样式的说明将在后续步骤中提供。

  1. 编写一个全局 dragstart 事件监听器，当拖放目标区域接受拖放数据时，为其应用样式。此事件监听器会在用户从同一页面上的拖放区域开始拖动时触发。此样式表示您的拖放目标区域接受拖放数据。有关更多信息，请参阅 MDN 拖动开始文档 ↗。
Copied!
1document.addEventListener("dragstart", maybeApplyValidDropStylingToDropZone, {capture: true});
  1. 在拖放目标区域被拖过时更改其样式。为了在拖放数据位于拖放目标区域上方时向用户提供视觉变化，将 drag-enter 和 drag-leave 处理程序添加到您的拖放区域，以应用和移除这些样式。与上面的全局 drag-enter 和 drag-leave 事件监听器不同，这些事件仅由拖放目标区域上的 dragenter 和 dragleave 事件触发。
Copied!
1let dropHoveringOverDropZoneCount = 0;
2
3function handleDragEnter(event){
4    if (dropHoveringOverDropZoneCount === 0 && event.dataTransfer?.types.includes("application/x-vnd.palantir.rid.phonograph2-objects.object")){
5        // allow the drop and apply styling if the drop zone accepts the drag payload
6        event.preventDefault();
7        conditionallyApplyStylingToDropZone(event, "valid-small-hover");
8    }
9    dropHoveringOverDropZoneCount += 1;
10}
11
12function handleDragLeave(event){
13    if (dropHoveringOverDropZoneCount === 1){
14        removeStylingFromDropZone("valid-small-hover");
15    }
16    dropHoveringOverDropZoneCount -= 1;
17}
  1. 将 handleDragEnter 和 handleDragLeave 拖动事件处理程序添加到您的拖放目标区域组件中：
Copied!
1<div
2  id="my-cool-drop-zone"
3  ondragover="handleDragOver(event)"
4  ondrop="handleDrop(event)"
5  ondragenter="handleDragEnter(event)"
6  ondragleave="handleDragLeave(event)"
7>
8  Drop Zone
9</div>
  1. 更新下拉区域以执行样式清理。数据被放下后，从下拉区域中移除下拉样式CSS类。这是必要的，以便下拉区域不会保持在“点亮”状态。同时重置拖动计数器，以便从头开始处理新的拖动事件。
Copied!
1function handleDrop(event) {
2    // prevent default open-link behavior
3    event.preventDefault();
4
5    // perform some styling cleanup
6    // remove CSS classes now that there is no active drag payload
7    removeStylingFromDropZone("valid-small-hover");
8    removeStylingFromDropZone("valid-small");
9
10    // reset counters
11    dropHoveringOverDropZoneCount = 0;
12    dragInsideApplicationCount = 0;
13
14    // get Foundry object RIDs data
15    const data = event.dataTransfer.getData(
16        "application/x-vnd.palantir.rid.phonograph2-objects.object"
17    );
18
19    try {
20        if (data != null && data !== ""){
21            // try to parse the returned data
22            const dataParsed = JSON.parse(data);
23
24            // print the data
25            console.log("dropped data: ", dataParsed);
26
27            // do something with the data
28            doCoolThingWithFoundryObjects(dataParsed);
29        }
30    } catch (error) {
31        console.error("Unable to parse data", error)
32    }
33}

完成此步骤后，您应该有一个可工作的下拉区域。当拖拽负载接受数据时，下拉区域应该会亮起，并且在拖拽负载被放下后，数据应该打印到控制台中。您的下拉区域在拖拽负载被放下后也应移除样式。
