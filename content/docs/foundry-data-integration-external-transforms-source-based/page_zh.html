外部转换

数据连接源现可直接导入代码仓库，这是与外部系统交互的首选方法，已取代基于凭据的传统外部转换

外部转换允许从 Python 转换仓库连接到外部系统。

外部转换不支持使用出站应用程序进行身份验证的源。

当满足以下任一条件时，通常使用外部转换来执行批量同步、导出和媒体同步工作流：

  • 不存在可用的现有数据连接源类型。
  • 目标源类型不支持所需功能。
  • 数据连接用户界面提供的功能不具备所需特性。

针对这些情况的解决方案可能包括以下几种：

  • 通过互联网或私有网络连接 REST API。
  • 连接数据库，以实现数据连接用户界面当前无法支持的自定义查询逻辑。
  • 在同步或导出过程中按需转换数据。例如，在写入 Foundry 之前将文件打包合并，在传输过程中处理数据的自定义加密/解密等。

任何使用虚拟表的转换也被视为外部转换，因为转换作业必须能够访问包含虚拟化数据的外部系统。要在 Python 转换中使用虚拟表，请按照以下有关如何设置源的说明进行操作。

本设置指南中使用的字典 API 与 Palantir 无关，且可能随时发生变更。本教程并非对该 API 用于生产环境的认可、推荐或建议。

在外部转换中使用 Foundry 输入

外部转换通常需要使用 Foundry 的输入数据。例如，你可能需要查询 API 来为表格数据集中的每一行收集额外的元数据。或者，在某个工作流中，你需要将 Foundry 数据导出到外部的软件系统。

这类情况被视为受出口管控的工作流，因为它们可能会将安全的 Foundry 数据导出到另一个安全保证未知且数据来源关系断裂的系统中。在配置源连接时，数据主人必须明确是否允许导出 Foundry 中的数据，并提供允许导出的安全标记集合和组织。Foundry 提供了治理面中的管控措施，确保开发者能够清晰地编码安全意图，同时信息安全官可以审计与外部系统交互的工作流的作用域和意图。

在源端配置出口管控

导出操作通过安全标记进行管控。配置源时，导出配置用于指定哪些安全标记和组织可以安全地导出到外部系统。具体操作是：在数据连接应用中打开相应的源，然后进入连接设置 > 导出配置选项卡。接着，打开允许导出到此源选项，并选择可能需要导出的标记和组织集合。

>

此操作需要具备移除相关数据和组织标记的权限，因为在 Foundry 中，导出操作等同于移除数据上的标记。

必须开启允许导出到此源设置才能执行以下操作：

  • 在导入此源的 Python 转换代码中，将数据集、媒体集和流作为输入使用。
  • 在 Python 转换中使用在此源上注册的虚拟表。

18    words_df=Input("<input_dataset_rid>"),

20def compute(dictionary_api_source: ResolvedSource, words_df: DataFrame) -> DataFrame:
21    dictionary_api_url = dictionary_api_source.get_https_connection().url

23
24    words = words_df["word"].tolist()
25
26    phonetics= []

30

32            dictionary_api_url + "/api/v2/entries/en/" + word
33        )
34
35        if response.status_code == 200:
36            data = response.json()[0]
37

40            else:

46

48
49    return pd.DataFrame(phonetics)

外部转换与旧版外部转换的对比

以下是外部转换与旧版外部转换在工作流程上的一些主要差异：

  • 对于外部转换，导入数据源的标签页会始终自动显示。而在以前，只有当安全官在仓库设置中开启使用外部系统的功能后，才会显示添加流出策略和凭证的标签页。
  • 允许外部连接和使用输入的设置不再位于仓库设置中。相反，这些设置由每个数据源单独控制。
  • 凭证、流出策略和可导出标记不再在代码中指定。相反，这些设置会从导入到转换中的数据源获取，并自动应用于作业。
    • 如果此配置在数据源级别发生更改，导入该数据源的转换会自动获取更新，无需更改代码或升级版本。这实现了凭证、流出和可导出标记的集中式数据治理，并能立即向下游工作流传递。
    • 更改将在构建开始时生效，且不会影响正在运行的构建。
  • 装饰器已从 @use_external_systems() 更改为 @external_systems()。

外部转换的主要优势包括：

  • 支持连接无法从互联网访问的系统
  • 支持在无需更改代码的情况下轮换/更新凭证
  • 支持在多个仓库间共享连接配置
  • 为选定的数据源类型提供开箱即用的 Python 客户端
  • 改进并简化了用于启用和管理外部转换仓库的数据治理工作流
  • 在《数据血缘关系》中可视化连接到外部数据源的外部转换

迁移至外部转换

目前没有自动化的迁移路径可将旧版外部转换升级为外部转换。不过，对于大多数工作流而言，所需的手动操作预计极少。

以下是手动迁移至外部转换的主要步骤：

  1. 识别现有旧版外部转换代码中使用的凭证、流出策略和出口管制标记集合。
  2. 识别或配置数据连接源，使其连接到您希望从外部转换中访问的系统。确保这些数据源已配置为允许导入到代码中。
  3. 将步骤 2 中的相关数据源导入到您现有的 Python 转换仓库中。
  4. 修改代码以导入并使用带有数据源引用的新 @external_systems() 装饰器，随后移除所有 @use_external_systems() 装饰器实例。这通常涉及将转换逻辑中对凭证的引用更新为对您当前导入的数据源所获取凭证的引用。
  5. 在分支上测试您的更改，确保转换能够继续成功构建。
  6. 合并更新后的转换代码后，您即可取消勾选该仓库设置。

转换不能同时包含外部转换及其旧版本。为解决此问题，您可以将所有旧版外部转换迁移为使用基于数据源的外部转换（推荐），或者将您的转换拆分为多个转换。转换可以拆分为一个使用 use_external_systems 装饰器的转换，以及另一个使用 external_systems 装饰器的转换。

高级模式

结合内存感知缓冲将数据写入 Parquet 文件

外部转换从外部源拉取大量记录时，若在写入前将所有记录全驻留内存，极易引发内存溢出（OOM）错误。下方展示的 BufferedParquetWriter 类会在超出可配置的内存阈值时，周期性地将累积的数据行刷写至 Parquet 文件。该模式适用于任意外部数据源，并调用 filesystem().open() 写入原始文件，随后通过 put_metadata() 完成模式推断的收尾工作。

将数据集导出为 CSV


当需要从 Foundry 中将数据导出为 CSV 时（例如，向需要 CSV 格式文件的外部系统交付数据），可以使用外部转换读取输入数据集，并通过 filesystem().open() 将 CSV 文件写入输出端。随后，可通过数据连接导出该输出数据集。



```python
import csv

from transforms.api import Input, Output, transform, LightweightInput, LightweightOutput


@transform.using(
    source_data=Input(""),
    csv_output=Output(""),
)
def compute(source_data: LightweightInput, csv_output: LightweightOutput):
    OUTPUT_FILENAME = "export.csv"

    # 将输入数据集读取为 Polars DataFrame。
    df = source_data.polars()
    fieldnames = df.columns

    # 将数据写入为 CSV 文件。
    with csv_output.filesystem().open(OUTPUT_FILENAME, "w") as f:
        writer = csv.DictWriter(f, fieldnames=fieldnames)
        writer.writeheader()
        for row in df.iter_rows(named=True):
            writer.writerow(row)
```
