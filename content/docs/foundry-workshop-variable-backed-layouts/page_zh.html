变量驱动的布局

变量驱动的布局允许你通过变量动态控制布局组件的状态。借此，你可以构建响应式应用，使界面元素能够对用户交互、对象属性或函数中定义的自定义逻辑做出反应。

以下布局组件支持变量驱动：

  • 页面：控制多页面模块中显示的具体页面
  • 区块：控制可见性与折叠状态
  • 选项卡：控制选中的选项卡
  • 覆盖层：控制覆盖层处于打开还是关闭状态

变量驱动的页面选择

你可以将字符串变量绑定到多页面模块，以控制当前显示的页面。当变量值与某个页面的 ID 匹配时，该页面即变为活动状态。

配置变量驱动的页面选择步骤如下：

  1. 在编辑模式下，从左侧边栏打开模块设置面板。
  2. 在配置面板的 页面 下，找到 变量驱动的页面选择 设置项。
  3. 选择一个用于控制显示页面的字符串变量。
  4. 为每个页面设置唯一的 页面 ID，使其与变量将包含的值相匹配。

此功能适用于通过 URL 参数深度链接到特定页面、根据用户操作在页面间导航，或通过接口变量从父模块控制页面选择。

切换页面的 布局事件 不会更新为变量驱动的页面选择所配置的变量值。若要使变量与事件触发的页面更改保持同步，请改用 设置变量值 事件。

变量驱动的区块条件可见性

你可以将布尔变量绑定到区块，以控制其可见或隐藏。默认情况下，变量求值为 true 时区块隐藏，求值为 false 时区块可见。通过更改变量选择框右侧的布尔选择器，可以反转此默认行为。

配置变量驱动的区块可见性步骤如下：

  1. 在模块中选中目标区块。
  2. 找到并开启 条件可见性 设置。
  3. 在配置面板中，找到 满足条件时隐藏区块 设置项。
  4. 选择一个用于控制区块可见性的布尔变量。
  5. （可选）配置该区块在布尔变量为 false 时隐藏。

这与折叠状态功能不同：隐藏的区块在布局中完全不显示，而折叠的区块仍会展示其标题，且用户可将其展开。系统不会提供用于更新区块可见性的事件。请改用 设置变量值 事件。

变量驱动的区块折叠状态

你可以将布尔变量绑定到可折叠区块，以控制其展开或折叠。当变量求值为 true 时，区块折叠；求值为 false 时，区块展开。

配置变量驱动的折叠状态步骤如下：

  1. 在模块中选中目标区块。
  2. 在配置面板中，启用 可折叠 设置。
  3. 找到 变量驱动的折叠状态 设置项，并选择一个布尔变量。

此功能适用于仅在存在相关数据时展开区块，或协调多个区块的展开与折叠行为。

展开、折叠或切换区块的 布局事件 不会更新为变量驱动的折叠状态所配置的变量值。若要使变量与事件触发的区块状态更改保持同步，请改用 设置变量值 事件。

变量驱动的覆盖层打开状态

你可以将布尔变量绑定到覆盖层，以控制其打开或关闭。当变量求值为 true 时，覆盖层打开；求值为 false 时，覆盖层关闭。

配置变量驱动的覆盖层状态步骤如下：

  1. 在模块中选中目标覆盖层。
  2. 在配置面板中，找到 变量驱动的打开状态 设置项。
  3. 选择一个用于控制覆盖层状态的布尔变量。

此功能适用于响应事件打开覆盖层、通过接口变量从父模块控制覆盖层状态，或协调多个覆盖层的展示。

打开或关闭遮罩层的布局事件会更新为变量支持打开状态配置的变量值。你还可以配置关闭时事件，以便在遮罩层关闭时重置上游变量或执行其他操作。

变量支持的选中标签页

你可以将字符串变量绑定到标签页容器，以控制当前选中哪个标签页。当变量值与某个标签页的 ID 匹配时，该标签页即变为活动状态。此机制是双向的：选择标签页会更新变量，而修改变量则会选中对应的标签页。

配置变量支持的标签页选择步骤如下：

  1. 在模块中选择标签页容器。
  2. 在配置面板中，找到变量支持的标签页选择设置项。
  3. 选择一个字符串变量来控制选中的标签页。
  4. 为每个标签页设置唯一的标签页 ID，使其与变量将包含的值相匹配。

此功能适用于跨多个标签页容器同步标签页状态、通过模块接口变量将选中的标签页持久化到 URL 中，或者根据模块中其他位置的用户操作以编程方式切换标签页。

与页面选择和区块折叠状态不同，更改选中标签页的布局事件也会更新为变量支持的标签页选择配置的变量值。
