限制

Automate 应用程序实施了几项限制，以提升执行和触发效果的性能。下表列出了这些限制及其预期行为。

自动化规模限制

描述限制达到限制时的行为
计划执行时 Objects added 或 Objects removed 条件的最大输入大小100,000保存自动化时报错，或者在评估自动化时，如果输入集增长超过限制则触发运行时错误
计划执行时 Run on all objects 条件的最大输入大小1,000,000保存自动化时报错，或者在评估自动化时，如果输入集增长超过限制则触发运行时错误
每个自动化的最大收件人数量10,000保存或评估自动化时报错
实时执行时，每次自动化评估的最大对象数10,000处理前 10,000 个对象事件，或者抛出错误
启用按对象执行时，计划自动化每次自动化评估的最大对象数10,000在执行任何效果之前的评估期间发生运行时错误；不处理任何对象
手动执行时的对象集最大大小5,000,000执行自动化时报错
自动化的最大批量大小1,000运行时错误

自动化运行时限制

描述限制达到限制时的行为
自动化事件在效果执行队列中等待的最长时间可配置该事件终止，且不执行任何效果
自动化事件在执行队列中等待的最长时间45 分钟该事件终止，且不执行任何效果
自动化事件的最长运行时间4 小时该事件终止；超时前完成的效果会保留，但剩余的效果不再执行

自动化事件可能会在队列中等待，原因有以下几点：

  • 队列效果执行设置：启用 队列效果执行 后，事件将按顺序执行，必须等待前一个事件完成才能开始。
  • 内部并行限制：即使未开启 队列效果执行 设置，系统对可并行执行的事件数量也有内部限制。当大量事件被连续快速触发时，它们会按触发顺序进入队列并开始执行，但根据每个事件的运行时间，完成顺序可能与触发顺序不同。
