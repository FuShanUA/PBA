在 Workshop 中使用操作

Foundry 中的操作允许用户根据已定义的规则集（称为操作类型）来编辑、创建、删除和链接对象数据。例如，某个操作可能允许用户执行以下一项或多项操作：

  • 创建一个新的 Flight 对象，并填写其 Flight Number、Time of Departure、Origin 和 Destination 属性。
  • 修改现有的 Flight 对象，以更改 Destination 属性。
  • 将特定的 Flight 对象与特定的 Passenger 对象进行链接。
  • 删除误创建的 Flight 对象。

通过为复杂的对象数据编辑创建易于使用的封装，操作使应用创建者能够使用有意义且安全的构建块来实现向对象数据的用户回写。随后，这些构建块可以：

  • 在多个应用程序（如 Workshop 或对象视图）中使用，以支持业务流程工作流。
    • 系统会根据操作定义自动生成表单组件，因此向对象数据的回写及其关联的用户界面无需单独定义。Workshop 和操作的核心交互详见下文。
  • 进行细粒度的权限设置，以针对不同用户和条件支持不同的回写权限。
    • 例如，可能允许分析师启动新的调查，但关闭调查的权限可能仅限于经理用户组的成员。

在 Ontology Manager 中定义操作

要创建、配置或编辑操作，Palantir Foundry 用户必须属于 actions-admins 组。

有关定义操作的完整详细信息，请参阅《操作文档》。下文提供的文档简要说明了一个侧重于在 Workshop 中使用操作的操作配置示例。

操作管理员可以在 Ontology Manager 中创建、配置和编辑操作类型。要进行访问，请从 Apps 侧边栏中选择 Ontology manager。Action types 选项卡（如以下屏幕截图所示）提供了现有操作类型的列表，您可以从中选择进行查看和编辑。New action type 按钮允许您从头开始定义新的操作类型。此外，您也可以直接从其应基于的对象类型创建操作类型。导航至该对象类型，并在操作类型框中选择 Create new。

请注意，此示例仅用于说明，您可能无法完全按原文完成每一个步骤。由于您的 Foundry Ontology 是根据您的需求和数据定制的，您可能无法访问完成本教程所需的 Ontology 对象。

选择对象类型上的 Create new 按钮开始。在本示例中，我们将定义一个“修改航班目的地”操作，允许用户修改现有 Flight 对象的 Destination 属性。

创建向导将引导您完成引导新操作类型的最重要步骤。输入 Display name，并在 Change object(s) 选项下选择 Modify 来配置您的操作类型。从对象下拉菜单中，选择您先前创建的 Flight 对象，然后选择此操作类型应修改的 Comment 和 Destination 属性。

操作类型定义的第一个选项卡名为 Overview，允许您定义 API Name（唯一 ID）、面向用户的 Display Name、面向用户的 Description 以及操作的 Status。其中某些字段已根据上一步骤自动填充。请使用如下所示的值填充缺失的字段。API 名称在保存后无法编辑，因此在选择 API 名称时务必谨慎。

接下来，选择此操作类型定义的 Rules 选项卡。您可以在此配置操作类型的核心：其 Rules 或输出/执行。选择对象类型和您希望此操作类型执行的更改类型后，您可以看到配置为修改规则的规则以及添加的属性。如有需要，可从下拉菜单中添加其他属性。

表单选项卡列出了动作类型的参数或输入项。这些参数已由规则部分自动添加。

接下来，我们为动作类型配置一些基本的提交条件。在此示例中，我们要确认用户为修改后的目的地输入了有效的三字符机场代码（如 EWR）。导航至安全与提交条件选项卡，并在提交条件部分选择添加条件。在本例中，请使用参数条件模板，将 Destination 属性与 ^[A-Z]{3}$ 这样的正则表达式进行比较；这让我们能够确认目的地参数的输入值正好是三个字符。为了让用户了解提交为何可能未通过条件检查，请在侧边栏底部添加一条错误消息。

要测试新的动作类型，请导航回表单选项卡，并使用屏幕右侧的表单预览来测试您的动作类型。您可以输入不同的值并选择提交按钮。这仅用于测试动作是否可提交，并不会实际提交动作。

您已完成第一个动作类型的配置，需要将其保存到我们的本体中。为此，请选择屏幕右上角的保存按钮，查看模态框中出现的修改列表，然后再次选择两次保存以确认。

您的第一个动作类型现已保存，我们可以前往 Workshop 构建一个模块，让用户能够执行此动作类型。

提示

以下部分假设您已熟悉构建 Workshop 模块。有关涵盖这些主题的教程，请参阅此页面。

在 Workshop 中，可以通过多种不同的微件（包括按钮组和创建动作面板）向用户展示动作。以下示例从航班收件箱模块开始，描述如何从该 Workshop 模块中通过按钮组触发动作。

按钮组

Workshop 中的按钮组微件允许模块构建者向其模块添加一个或多个按钮。在本示例中，我们将重点介绍如何使用按钮组在 Workshop 中展示动作。

在按钮组中配置动作

将按钮组添加到您的 Workshop 模块后，单击屏幕上模块内的按钮组，将在屏幕右侧打开微件的配置面板。核心配置选项可在微件设置选项卡中找到，分为三个部分——布局、样式和按钮。布局和样式部分控制高级显示选项，但本示例将重点关注按钮部分，模块构建者可在此处配置基本显示选项，并确定选择按钮时如何应用动作。

在上述按钮部分中，您可以调整文本、颜色、左侧图标和右侧图标选项来控制按钮的显示。举例来说，让我们为打算通过该按钮触发的“修改目的地”动作配置这四个选项。

上述配置将生成一个如下所示的按钮。使用适当的文本、颜色和图标为按钮准确添加标签，有助于用户了解选择该按钮后会发生什么。您可以选择预设颜色（主要、成功、警告、危险），或通过十六进制代码等方式指定自定义颜色。

按钮完成标签设置后，我们进入点击时部分，连接预期操作。具体而言，点击选择操作...下拉菜单，筛选并选择“修改目的地”操作。随后会显示以下界面，其中包含我们在上文创建“修改目的地”操作时定义的所有三个操作参数。

参数默认值允许用户根据需要为操作参数设置默认值。再次提醒，每个参数都对应操作类型定义中的一种输入（如对象列表、对象引用、整数、字符串等）。对于每个参数，用户有两种选择：

  • 设置与 Workshop 模块中变量相等的默认值
    • 为此，我们可以选择所选航班参数的“无默认值”选项，将其设置为航班对象表输出的“活动对象”变量。设置完成后，将显示额外选项，可将参数设为可见（用户可在操作表单中进一步修改默认值）、隐藏（参数在操作表单中对用户完全隐藏）或禁用（参数在操作表单中对用户显示为只读）。在本示例中，我们保留默认选项可见。
  • 不留默认值
    • 无默认值的参数最初会在操作表单中显示为空白字段，其值将由用户输入。
    • 在本示例中，我们将“修改后的目的地”和“评论”字段留空，不设默认值。

从按钮组应用操作

在按钮组中配置好操作后，当用户在 Workshop 模块中选择修改目的地按钮时，将会看到一个操作表单弹出。字段的默认值基于上述配置；所选航班字段会根据模块变量自动填充，而修改后的目的地和评论字段则为空，等待用户输入。

在上方表单中，操作表单左下角标记了1 个问题。将鼠标悬停在1 个问题文本上，用户可以看到问题在于修改后的目的地字段当前不是有效的三字符机场代码。

填写完操作表单且所有参数通过校验后，即可提交操作（提交按钮将变为可用状态）。

点击提交后，用户会在模块顶部看到一个绿色的提示框，表明操作已成功完成，且所选的 Flight 对象已更新，反映了所输入的更改。

从媒体上传器组件触发操作

用户通过媒体上传器组件上传文件时，也可以触发操作。这对于依赖新上传媒体的工作流（如处理、丰富或链接工作流）非常有用。

在配置媒体上传器以在上传时触发操作时，可以在操作的参数配置中使用特殊的文件标识符值来引用上传的文件：

  • 对于上传到数据集的文件，文件标识符为上传的文件名。
  • 对于上传到 Compass 文件夹的文件，文件标识符为上传文件的 RID。
  • 对于上传到媒体集的文件，文件标识符为上传文件的媒体引用。

如果启用了多文件上传，配置的操作将针对每个上传的文件触发一次，文件标识符与正在处理的具体文件相对应。

有关设置详情（包括按钮配置、上传目标、允许的文件扩展名、动态目标和多文件选项），请参阅《媒体上传器组件文档》。
