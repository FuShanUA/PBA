设置批量部署

批量部署是配置在建模目标内部的一种特殊流水线，允许数据通过模型运行，并将结果输出到 Foundry 数据集。这些输出数据集可以通过构建计划来管理。

前提条件

创建新的批量部署前，需满足两个前提条件：

  1. 目标内部必须存在一个已发布的版本，且该版本带有预发布或生产环境对应的环境标签。

  2. 选择一个用于通过模型运行的输入数据集。理想情况下，该数据集应经过精心维护并定期更新——它代表了你试图解决的问题的新增信息。

创建部署

要创建新的批量部署，请转至建模目标主页底部的部署部分，并在右上角选择创建部署。

在提示中填写详细信息与相关内容。根据前提条件 1 选择部署环境，根据前提条件 2 选定输入数据集，并为输出数据集选择位置和名称。或者，也可以选择一个现有数据集作为输出数据集，而无需创建新的输出数据集。

在建模目标中直接设置批量部署仅兼容使用单个表格数据集输入的模型。如果你的模型适配器需要多个输入，可以改为在 Python 转换中设置批量推理。

资源配置

创建新的批量部署时，你可以为该单个部署配置所需资源。资源通过 Spark 配置文件进行配置，该配置将在推理期间应用于 Spark 环境。

如果创建新的批量部署，会在初始部署配置时显示 Spark 配置文件选择器。其行为与 Python 转换一致。若未作选择，则不会为该部署配置任何 Spark 配置文件，而是使用默认的资源配置文件。

要编辑现有批量部署上已配置的 Spark 配置文件，请转至建模目标中的部署部分，从列表中选择你的部署，并在运行时配置下选择编辑按钮以编辑 Spark 配置文件。这将自动更新部署。

自动构建部署输出

选择新创建的部署，即可转至部署视图。

在部署详情视图中选择数据集链接，即可转至输出数据集。

你可以在批量部署的输出数据集上创建计划，使其在每当有新模型发布到该部署环境时自动更新。这是通过创建一个新的逻辑计划来实现的，即每当该数据集的逻辑更新时，就构建输出数据集。当目标中发布新模型时，输出数据集的逻辑将随之更新，进而触发此计划。
