编辑环境管理设置

本节概述了您可以为环境配置的各种详细信息。在编辑您的环境管理设置之前，请确保您了解环境的 授权模型，并已创建一个将管理此环境的 团队。

要编辑现有环境的配置，请从 操作 下拉菜单中选择 管理环境…。

这将打开环境管理表单。

环境ID 唯一标识每个环境。创建环境后，无法更改。只允许使用小写字母和短横线。该ID在整个Apollo中用于引用环境。

< strong > 认证 会影响 Apollo 中的多个合规和审批流程。如果您的环境必须遵循严格的合规标准，如 FedRAMP、IL5 或 IL6，则需要对其进行适当的标记。认证将在环境主页顶部的标题栏中显示，并可能影响环境的 < a href = "/docs/apollo/managing-changes/change-requests/" > 变更请求 审批流程，包括从专门的合规团队添加必需的审查。

选择 < code > DEV 认证将导致环境的所有变更请求自动获得批准，这在初始环境设置期间非常有用。一旦环境开始用于生产，应使用不同的认证。对于没有特定合规要求的生产环境，应使用 < code > STANDARD 认证。在 < code > STANDARD 认证的环境中，变更请求需要根据变更所影响的内容，由产品或环境编辑者进行审批。

对于其他内置的认证选项，请选择 < strong > 其他认证选项 ，然后从下拉菜单中选择适当的认证。如果您需要尚未提供的认证选项，请联系您的 Palantir 代表以讨论添加该选项。

联系团队 指的是对环境负责并作为主要联系点的团队，尽管管理职责可以与其他被设置为 高级设置 下额外联系团队的团队共享。

启用操作责任 选项决定了警报和支持请求的路由方式。当启用操作责任时，每个已安装产品的联系团队将收到警报通知。当禁用操作责任时，通知则会转交给环境的联系团队。

您可以在 角色 选项卡中编辑环境的角色。

了解有关 环境可能的角色 的更多信息。

在编辑完成后选择 更新。这将创建一个更改请求以更新环境管理设置。
