数据集与对象集

数据集

在 Contour 和 Code Workbook 中，用户可以选择输出数据集（表格数据）来记录分析结果。这些数据集可用于 Contour、Code Workbook 和 Fusion 等应用程序，也可以与其他用户共享。

我应该使用哪个工具来构建流水线？

Contour 和 Code Workbook 并未针对创建生产流水线进行优化。如果要构建或维护生产流水线，请使用 Code Repositories 应用程序，该程序包含版本历史记录、分支和拉取请求等功能，这些功能对于构建稳健的流水线至关重要。更多相关信息，请参阅《对 Foundry 基于代码转换的编写工具的对比分析》。

Contour

在路径底部选择另存为数据集，即可保存 Contour 路径的结果。为数据集命名并选择保存位置后，系统将构建包含分析结果的数据集。《详细了解如何从 Contour 中保存数据集》。

Code Workbook

在 Code Workbook 中，可以通过选择另存为数据集，将转换结果保存为数据集。默认情况下，新建的转换不会保存为数据集。《详细了解如何从 Code Workbook 中保存数据集》。

什么情况下更适合使用 Code Repositories？

我们建议使用 Code Repositories 应用程序来创建稳健的生产流水线，并支持需要额外数据治理和监控的工作流。通过 Code Repositories，数据工程师可以批量创建高效的流水线。

适合使用 Code Repositories 的工作流示例包括：

  • 数据规模庞大且需要增量计算的每日流水线。
  • 具有严格数据治理要求的高可见性流水线，需要具备回滚至历史代码版本的能力，或在单元测试成功通过前拦截代码变更。

适合在 Contour 或 Code Workbook 中保存数据集的工作流示例包括：

  • 一次性捕获数据，随后将其用于其他分析应用程序。

虽然可以为在 Contour 和 Code Workbook 中创建的数据集设置构建计划，但流水线工作流通常应交由 Code Repositories 处理。

对象集

对象集是现实世界实体的列表，保存后可供支持对象的 Foundry 应用程序在未来引用和使用。对象集作为资源保存，便于与协作者共享。

对象集分为两种类型：

  • 静态对象集：静态对象集以主键列表的形式保存，无论输入数据如何变化，其内容都将保持不变。
  • 动态对象集：动态对象集以创建对象集时所应用过滤规则的表示形式保存。当新数据匹配这些过滤规则时，对象集会自动更新。

在 Quiver 分析中创建的对象集可以保存到 Foundry 中。保存后，此类对象集可以导入到新的 Quiver 分析中，或者在 Object Explorer 等其他 Foundry 应用程序中打开。

要导出对象集，请点击卡片右上角的图标以打开对象集卡片的编辑器，然后导航至“导出”选项卡。

要导入对象集，请使用导入已保存对象引用卡片。

详情请阅读《如何在 Quiver 中导入和导出对象集》。

对象集也可以在 Object Explorer 中进行保存、更新和比较。详情请阅读《如何在 Object Explorer 中保存对象集》。
