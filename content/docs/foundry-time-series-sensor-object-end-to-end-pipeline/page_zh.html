第一部分：丰富传感器读数数据

将传感器读数数据集添加至新管道后，您需要添加一些元数据。

1. 应用转换以重新格式化传感器读数

针对传感器读数数据集，按照以下步骤应用转换。

逆透视数据以合并序列值

由于该数据集在不同列中包含时间序列数据，您必须使用逆透视转换将其合并至单一值列中，使数据能够符合时间序列同步所需的模式，如下所示：

  • 序列 ID： string | TSP 所指的一组时间戳和值对应的序列 ID，必须与 TSP 的序列 ID 相匹配。
  • 时间戳： timestamp 或 long | 测量该数值的时间点。
  • 值： integer、float、double、string | 测量点处该数值的大小。字符串类型表示分类时间序列；每个分类时间序列最多可包含 10,000 个唯一变体。

下方展示的逆透视转换将 altitude 和 speed 的值统一放入同一个 series_value 列中。原始列名将输出至新的 series_name 列，以便父根对象能够识别每个传感器代表的含义（例如 altitude）。

转换后的数据集应预览显示新的 series_value 和 series_id 列：

series_nameseries_valuetimestampflight_idflight_ title
altitude1342023-01-11T23...021fcdbsd...YX4472...
speed114.002023-01-11T23...021fcdbsd...YX4472...

拼接主键以创建序列 ID

现在，您可以使用字符串拼接转换来创建序列 ID（即关联时间序列值的标识符）。利用该转换，将 series_name（每个传感器代表的含义）与各对象的主键结合起来。

flight_sensor_series_idseries_nameseries_valuetimestampflight_idflight_ title
altitude_021fcdbsd...altitude1342023-01-11T23...021fcdbsd...YX4472...
speed_021fcdbsd...speed114.002023-01-11T23...021fcdbsd...YX4472...

移除空值

应用筛选转换，移除所有包含 null 值的行，这些行不会参与我们的时间序列计算。

2. 在数据集中添加单位

手动上传单位数据集

由于原始的示例传感器读数数据集不包含单位，因此你必须添加一个生成的数据集，以将单位映射到序列名称。在 Pipeline Builder 图表的左上角，选择 添加数据，然后选择 手动输入数据。创建一个名为 series_name 的列，再创建一个名为 units 的列。如有必要，你可以针对内部插值重复此步骤（尽管我们的示例中并不需要）。

内部插值用于让 Quiver 推断相邻数据点之间的序列值。内部插值列会为本体提供一个属性，以便为每个传感器对象保存插值设置。Quiver 在可视化时间序列数据时将依赖该属性。查阅我们的插值文档以获取更多信息。

部分传感器读数数据集确实包含单位；在这些情况下，你可以跳过此步骤，直接进行创建支持传感器对象类型的数据集。

创建连接以合并单位和传感器数据集

使用连接面板，创建一个连接以合并来自单位和传感器数据集的数据。请确保设置了以下配置：

  • 添加一个 左连接 并按 series_name 进行匹配。
  • 自动从左侧数据集中选择列。
  • 仅选择 units 列作为右侧列。

数据预览时应显示新的 units 列，如下所示：

第二部分：创建支持传感器对象类型的数据集

现在，你可以开始创建传感器对象类型了。按照以下步骤对连接输出应用转换，从而创建一个最终支持传感器对象类型的数据集。

1. 删除重复项以实现每个传感器仅保留一行

一个传感器对象代表且仅代表一组相关的时间序列数据（例如某个特定航班的 altitude）。由于每个相关时间序列集合都通过 flight_sensor_series_id 进行唯一标识，因此你可以假设，为每个唯一的飞行传感器序列 ID 仅保留一行，即对应一个传感器对象。通过对生成的 flight_sensor_series_id 使用删除重复项转换，你可以为每个传感器对象创建一行。你将在后续步骤中移除其余的序列数据（时间戳和值）。

数据集应如下所示：

flight_sensor_series_idseries_nameseries_valuetimestampflight_idflight_ titleunits
altitude_021fcdbsd...altitude1342023-01-11T23...021fcdbsd...YX4472...ft
speed_021fcdbsd...speed114.002023-01-11T23...021fcdbsd...YX4472...mph

2. 对序列 ID 进行哈希处理以创建主键

由于每个传感器对象都必须可唯一标识，因此你必须使用哈希 sha256 转换为每个对象创建一个主键。你可以重用 series_id，因为它在所有这些传感器对象中应该是唯一的，但进行哈希处理能更直接地表明 series_id 仅应作为唯一标识符使用。

数据集预览应如下所示：

2492af5a2fe62c78ad8d...altitude_021fcdbsd...altitude134.002023-01-11T23...021fcdbsd...YX4472...ft
ec4808668f4b48cc5104...altitude_011794756...altitude0.00002023-01-22T17...011794756...UA1878...ft

3. 拼接字符串以为对象创建标题属性

现在，你应该使用拼接字符串转换为我们的传感器对象创建人类可读的标题属性。即使传感器对象将嵌入到另一个 Flight 对象的对象视图中，为传感器对象创建专用名称也能让用户在平台中构建和分析传感器时更加直观。

数据集预览应如下所示：

4. 删除不必要的列

由于传感器对象仅代表传感器本身，你可以通过删除列转换来移除剩余的 series_value 和 timestamp 列。这些列体现在时间序列属性上，并将基于 flight_sensor_series 值在 Ontology 中与传感器建立关联。此外，由于传感器对象类型不需要 flight_title 列，你也需要将其删除。

你需要保留 flight_id，以便将传感器对象（Flight Sensors）与其根对象（Flights）关联起来。

5. 添加数据集输出以支持传感器对象类型

选择 Add output，然后选择 Add dataset。生成的数据集应与转换中创建的表结构保持一致。后续的任何更改可能需要更新生成数据集中的表结构。

第三部分：创建传感器对象类型时间序列同步

1. 将传感器读数转换为时间序列同步格式

使用选择列转换，仅保留时间序列同步所需的列：series_id、timestamp 和 value。支持数据集将保存所有传感器的值，无论其测量内容是什么（例如 altitude 和 speed）。你可以将生成的 flight_sensor_series_id 用作 series_id，作为特定传感器数据集的唯一标识符。例如，从拉瓜迪亚机场 (LGA) 飞往洛杉矶国际机场 (LAX) 的 A445B 航班将关联一个 speed 传感器对象，用于保存该航班的速度传感器数据。同样，它也会关联一个 altitude 传感器对象。

数据集输出预览结果应如下所示：

flight_sensor_series_idseries_valuetimestamp
altitude_021fcdbsd...134.002023-01-11T23...
altitude_011794756...0.00002023-01-22T17...

第三部分：创建时间序列同步

1. 配置时间序列同步

现在，通过在屏幕右侧的管道输出部分选择 Add 来创建《时间序列同步》。然后，选择 Time series sync。为新的时间序列同步填写必要的数据，并注意以下事项：

  • 为《序列 ID》字段选择 flight_sensor_series_id 列。
  • 在 Time 字段中添加创建的 timestamp 列。
  • 在 Value 字段中添加 series_value。

现在，保存并构建管道。输出结果将创建在管道所在的同一文件夹中。

2. 使用时间序列同步为传感器对象类型添加属性

既然你已经创建了包含时间序列同步的管道，便可以使用该同步为传感器对象类型添加时间序列属性。如需更多指导，请查阅关于《为传感器对象类型添加时间序列属性》的文档。
