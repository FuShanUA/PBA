在 Jupyter® 笔记本中训练模型

模型资产目前不支持 SparkML 库。我们建议切换到单节点机器学习框架，例如 PyTorch、TensorFlow、XGBoost、LightGBM 或 scikit-learn。

可以在《代码工作区》的 Jupyter® 笔记本中训练模型。要训练模型，请完成以下步骤：

  1. 创建 Jupyter® 代码工作区
  2. 导入数据并编写模型训练代码
  3. 添加模型输出并实现模型适配器
  4. 将模型发布到 Foundry
  5. 使用模型

《监督模型训练教程》提供了关于在 Jupyter® 代码工作区中训练模型的更多说明。

创建 Jupyter® 代码工作区

  1. 要为模型训练创建新的 Jupyter® 代码工作区，请导航至项目文件夹并选择 + 新建 > Jupyter® 代码工作区，或者在“代码工作区”应用中选择 + 新建代码工作区。
  2. 在“代码工作区”中，选择 JupyterLab® 作为工作区类型，然后选择右下角的 继续。
  3. 根据你要训练的模型为工作区命名。你可以在代码仓库步骤中选择 高级，根据需要配置计算资源或网络策略等附加设置。为笔记本命名后，选择 继续。
  4. 最后，选择 创建 以创建并启动工作区。

导入数据并编写模型训练代码

建立工作区后，你可以创建新的笔记本来导入数据并开始编写模型训练代码。

安装依赖项

Code Workspaces 允许访问其他 Foundry 代码编写环境（如代码仓库）中可用的包。要添加新包，请打开工作空间左侧边栏的包选项卡，搜索所需的包并选中，然后点击最新版本或其他可用版本，系统将打开终端并运行相应的安装命令。

添加模型输出并实现模型适配器

要在 Code Workspaces 外部提供模型，必须向工作区添加新的模型输出。创建新的模型输出后，Code Workspaces 会自动在现有工作区中创建并保存一个新的 .py 文件，你可以使用该文件来实现模型适配器。模型适配器为 Foundry 中的所有模型提供了标准接口，确保平台的生产应用程序在模型创建后即可立即调用。Foundry 基础设施将负责加载模型、配置其 Python 依赖项、暴露其 API，并启用模型接口。

  1. 要添加输出，请在左侧边栏 Packages 下方的 Models 选项卡中，选择 Add model > Create new model。为模型命名并将其保存到你选择的位置。

  1. 为模型命名并保存后，系统会提示你在工作区左侧面板中发布新模型。通过将代码片段复制到剪贴板并在原始的 .ipynb 笔记本文件中运行，来完成第 1 步：安装 palantir_models。

  2. 成功安装 palantir_models 后，在第 2 步：开发你的模型适配器中创建并开发你的模型适配器。模型适配器必须实现以下方法：

    1. save 和 load：为了能够复用你的模型，你需要定义模型的保存和加载方式。Palantir 提供了默认的序列化（保存）方法，在更复杂的情况下，你可以实现自定义序列化逻辑。
    2. api：定义模型的 API，并告知 Foundry 你的模型需要何种类型的输入数据。
    3. predict：由 Foundry 调用，用于向你的模型提供数据。你可以在此处将输入数据传递给模型并生成推断（预测）。

    有关更多详细信息，请参阅模型适配器 API 参考。

下面的代码示例实现了上述功能，使用 scikit-learn 为线性回归模型开发了一个适配器：



已复制！
1import palantir_models as pm
2from palantir_models.serializers import DillSerializer
3
4class LinearRegressionModelAdapter(pm.ModelAdapter):
5
6    @pm.auto_serialize(
7        model=DillSerializer()
8    )
9    def __init__(self, model):
10        self.model = model
11
12    @classmethod
13    def api(cls):
14        columns = [
15            ('median_income', float),
16            ('housing_median_age', float),
17            ('total_rooms', float),
18        ]
19        return {"df_in": pm.Pandas(columns)}, \
20               {"df_out": pm.Pandas(columns + [('prediction', float)])}
21
22    def predict(self, df_in):
23        df_in['prediction'] = self.model.predict(
24            df_in[['median_income', 'housing_median_age', 'total_rooms']]
25        )
26        return df_in

将模型发布到 Foundry

要将模型发布到 Foundry，请在左侧边栏的步骤 3：发布您的模型下，复制您希望发布的模型对应的可用代码片段，将其粘贴到您的笔记本中并运行该单元格。以下是一个使用上文编写的 LinearRegressionModelAdapter 发布线性回归模型的代码片段示例：



已复制！
1from palantir_models.code_workspaces import ModelOutput
2
3# 模型适配器已在 linear_regression_model_adapter.py 中定义
4from linear_regression_model_adapter import LinearRegressionModelAdapter
5
6# sklearn_model 是在另一个单元格中训练的模型
7linear_regression_model_adapter = LinearRegressionModelAdapter(sklearn_model)
8
9# "linear_regression_model" 是此示例模型的别名
10model_output = ModelOutput("linear_regression_model")
11model_output.publish(linear_regression_model_adapter)

这段代码片段可以直接运行，只需在初始化适配器时正确传入你训练好的模型即可。代码就绪后，你可以运行该单元格，将模型发布到 Foundry。
