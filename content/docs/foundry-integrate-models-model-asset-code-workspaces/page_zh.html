在 Jupyter® 笔记本中训练模型

目前模型资产不支持 SparkML 库。我们建议切换到单节点机器学习框架，如 PyTorch、TensorFlow、XGBoost、LightGBM 或 scikit-learn。

模型 可以在 代码工作区 的 Jupyter® 笔记本中进行训练。要训练一个模型，请完成以下步骤：

  1. 创建 Jupyter® 代码工作区
  2. 导入数据并编写模型训练代码
  3. 添加模型输出并实现模型适配器
  4. 将模型发布到 Foundry
  5. 使用模型

监督模型训练教程 提供了有关在 Jupyter® 代码工作区中进行模型训练的更多说明。

创建 Jupyter® 代码工作区

  1. 要为模型训练创建一个新的 Jupyter® 代码工作区，请导航到您的项目文件夹并选择 + New > Jupyter® Code Workspace，或在 Code Workspaces 应用程序中选择 + New code workspace。
  2. 在 Code Workspaces 中，选择 JupyterLab® 作为工作区类型，然后在右下角选择 继续。
  3. 根据您正在训练的模型来命名您的工作区。您可以通过在 Code Repository 步骤中选择 高级 来选择性地配置其他设置，例如工作区的计算资源或网络策略。在您命名了笔记本后，选择 继续。
  4. 最后，选择 创建 以创建并启动工作区。

导入数据并编写模型训练代码

建立工作区后，您可以创建一个新的笔记本以导入数据并开始编写模型训练代码。

安装依赖项

Code Workspaces（代码工作区）提供对其他Foundry代码创作环境中的包的访问权限，例如Code Repositories（代码仓库）。要添加新的包，请打开工作区左侧边栏上的Packages（包）选项卡，搜索需要的包，然后选择它，点击Latest（最新）或另一个可用版本以打开终端并运行相应的安装命令。

将数据导入工作区

Code Workspaces（代码工作区）应用程序允许用户导入现有的Foundry数据集作为训练数据。在Code Workspaces中使用的训练数据需要一个可读的人类标识符作为其资源标识符。

  1. 要导入数据集，请打开左侧侧边栏顶部的数据选项卡，然后选择添加数据集 > 读取现有数据集。
  2. 选择您要导入到工作区的数据集，输入数据集别名，然后选择+ 添加数据集以完成步骤1。
  3. 代码工作区将在步骤2中生成一个代码片段，您可以从中选择数据集格式，例如pandas DataFrame。要将生成的代码片段复制到您的笔记本中，请在代码片段右上角选择剪贴板图标，然后选择完成。以下是代码工作区生成的代码片段示例：

1from foundry.transforms import Dataset

  1. 将代码片段复制到剪贴板后，通过选择Launcher面板中的Python [用户默认]启动一个笔记本，并将代码片段粘贴到第一个单元格中。
  2. 通过选择操作工具栏中的“播放”图标，或通过菜单栏中的运行 > 运行所选单元格来运行代码以导入数据集。

编写模型训练代码

Code Workspaces 中可用于模型开发的开源工具允许您为各种分析用例（如回归或分类）训练您的模型。以下是使用 scikit-learn 预测中位数家庭收入的示例线性回归模型。

  1. 通过选择左侧侧边栏 数据 下的 包 图标，在您的工作区中安装 scikit-learn 。
  2. 在搜索栏左侧的下拉菜单中选择 Conda 或 PyPi 管理器，然后搜索 scikit-learn 。
  3. 要在终端运行安装命令，请从下拉菜单中选择一个包版本，然后选择版本按钮。或者，您可以一次性从终端使用 maestro env conda install 或 maestro env pip install 命令将多个包安装到 管理环境 中。

添加模型输出并实现模型适配器

要使模型在代码工作区之外可用，您必须向工作区添加一个新的模型输出。在您创建新的模型输出后，代码工作区会自动在您的现有工作区中创建并存储一个新的 ___CODE_BLOCK_13 ___ 文件，您可以使用它来实现 模型适配器。模型适配器为Foundry中的所有模型提供了一个标准接口，确保平台的生产应用程序可以在模型创建后立即使用模型。Foundry基础设施将加载模型，配置其Python依赖项，公开其API，并启用模型接口。

  1. 要添加一个输出，请在下方左侧侧边栏的模型选项卡中打开，然后选择添加模型 > 创建新模型。为模型命名，并将其保存到您选择的位置。

  1. 在您为模型命名并保存后，您将在工作区左侧面板中被提示 发布新模型。通过将代码片段复制到剪贴板并在原始 .ipynb 笔记本文件中运行它来完成 步骤1：安装 palantir_models。

  2. 在成功安装 palantir_models 后，在 步骤2：开发您的模型适配器 中创建和开发您的模型适配器。模型适配器必须实现以下方法：

    1. save 和 load: 为了重用您的模型，您需要定义如何保存和加载您的模型。Palantir 提供了 默认的序列化方法（保存），在更复杂的情况下，您可以 实现自定义序列化 逻辑。
    2. api: 定义了模型的API，并告诉Foundry您的模型需要哪种类型的输入数据。
    3. predict: 由Foundry调用以向您的模型提供数据。这是您可以将输入数据传递给模型并生成推理（预测）的地方。

    有关更多详细信息，请参阅 模型适配器API参考。

下面的代码示例实现了上述函数，以使用 scikit-learn 开发线性回归模型的适配器：

Copied!
1import palantir_models as pm
2from palantir_models.serializers import DillSerializer
3
4class LinearRegressionModelAdapter(pm.ModelAdapter):
5
6    @pm.auto_serialize(
7        model=DillSerializer()
8    )
9    def __init__(self, model):
10        self.model = model
11
12    @classmethod
13    def api(cls):
14        columns = [
15            ('median_income', float),
16            ('housing_median_age', float),
17            ('total_rooms', float),
18        ]
19        return {"df_in": pm.Pandas(columns)}, \
20               {"df_out": pm.Pandas(columns + [('prediction', float)])}
21
22    def predict(self, df_in):
23        df_in['prediction'] = self.model.predict(
24            df_in[['median_income', 'housing_median_age', 'total_rooms']]
25        )
26        return df_in

将模型发布到Foundry

要将模型发布到Foundry，请在左侧边栏的步骤3：发布您的模型下复制您希望发布的模型的可用代码片段，将其粘贴到您的笔记本中并运行该单元格。以下是使用上面编写的LinearRegressionModelAdapter发布线性回归模型的示例代码片段：

Copied!
1from palantir_models.code_workspaces import ModelOutput
2
3# Model adapter has been defined in linear_regression_model_adapter.py
4from linear_regression_model_adapter import LinearRegressionModelAdapter
5
6# sklearn_model is a model trained in another cell
7linear_regression_model_adapter = LinearRegressionModelAdapter(sklearn_model)
8
9# "linear_regression_model" is the alias for this example model
10model_output = ModelOutput("linear_regression_model")
11model_output.publish(linear_regression_model_adapter)

该代码片段可以直接使用，除了需要正确地将您训练的模型传递给适配器初始化。一旦代码准备就绪，您就可以运行该单元格以将模型发布到Foundry。

使用模型

可以通过提交到< a href="/docs/foundry/model-integration/objectives/">建模目标来使用模型

. 模型可以提交到建模目标以进行：

  • 自动模型评估
  • 批量管道
  • 实时托管

也可以使用 模型部署 来消费模型，这代表了超越建模目标的另一种模型托管系统。
