发布管理

发布管理是指在服务于不同目的的独立环境中管理资源的多个版本的过程，也称为环境分离原则。通常，这些环境是资源在发布过程中不同阶段的隔离位置，例如：功能开发、功能测试以及功能部署到生产环境。

例如，您组织中的开发人员可以在开发环境中协作开发新功能，而用户验收测试人员和自动化测试脚本则可以在测试环境中测试尚未经过生产验证的功能，在将其发布到生产环境之前进行测试。在此期间，生产环境中运行的现有功能仍然可供用户使用，并且不会受到其他环境中正在进行的开发和测试的影响。

将资源按环境分隔允许在不影响生产环境中运行的功能的情况下，在受控环境中对更改进行测试和验证。当资源经过测试后，如果准备就绪，就可以将其提升到生产环境，或者根据需要进一步开发。

使用 DevOps 和 Marketplace 是实现发布管理流程的一种方式。在 Palantir 平台中还有其他方法可以实现这一点，例如使用 全局分支。

DevOps 和 Marketplace 中的环境隔离

空间 是 Palantir 平台中一种灵活的基本元素，允许在 Foundry 中实现环境隔离。环境隔离是指为开发和部署的不同阶段（如开发、测试和生产）维护不同的空间。这种隔离确保更改可以在受控环境中进行测试和验证，然后再推广到生产环境，从而减少将错误或问题引入生产工作流的风险。

您可以根据工作流的需要创建任意数量的环境，从而支持自定义的发布管理模型。

一个典型的环境设置包括以下三个环境，每个环境在 Palantir 平台中表示为一个“空间”：

  • 开发：用于开发新功能的环境
  • Foundry -> Foundry Apollo -> Apollo Gotham -> Gotham AIP -> AIP Ontology -> 本体论 ontology -> 本体论 Pipeline -> 管道 pipeline -> 管道 Transform -> 转换 transform -> 转换 Workshop -> Workshop workshop -> Workshop Code Repositories -> 代码仓库 Code Workspaces -> 代码工作区 Data Integration -> 数据集成 Model Integration -> 模型集成 Object -> 对象 object -> 对象 Dataset -> 数据集 dataset -> 数据集 Tenant -> 租户 tenant -> 租户 Marking -> 标记 marking -> 标记 Permission -> 权限 permission -> 权限 Role -> 角色 role -> 角色 Function -> 函数 function -> 函数 Action -> 操作 action -> 操作 Property -> 属性 property -> 属性 Interface -> 接口 interface -> 接口 内容: 测试：功能在前一个环境中开发完成后，用于手动或自动化测试功能的环境。
  • 生产：由最终用户在前一个环境中开发和测试的功能的环境。

通常，随着产品在您的环境中逐步推进，这些产品中的功能将在给定的环境/空间中可用。您可以在不同环境中配置以下差异：

  • 环境特定数据：例如，开发环境可能使用虚拟数据，而测试环境可能使用真实数据的一个子集，而生产环境则使用完整规模的真实数据。
  • 环境特定行为：非生产环境中的行为可能与生产环境中的行为不同。例如，生产环境中发送给相关用户的电子邮件通知可能在其他环境中发送给不同的组。同样，非生产环境中的审批流程可能允许一组不同的用户（如测试人员和开发人员）进行审批，而在生产环境中则由真正的审批人进行审批。
  • 特定环境的集成：例如，开发环境可能会连接到开发级数据库并调用其他开发级系统的API。生产环境通常会连接到生产数据库并调用其他生产级系统。

全局分支与发布管理

全局分支允许在隔离分支上进行工作流更改的开发和测试，非常适合通过允许多个开发者在隔离环境中工作来管理环境内的更改。只有当功能完成且由具有相应权限级别的用户进行时，更改才会合并到主分支。

全局分支非常适合快速迭代，提供了轻松测试新开发功能的能力。然而，由于分支的短暂性和资源类型的覆盖范围有限，它并不总是能满足发布管理的所有需求。

使用DevOps和Marketplace进行发布管理的好处包括能够：

  • 在不同环境中与不同的源系统集成
  • 在不同环境中维护不同的安全策略
  • 回滚到之前的发布版本
  • 配置发布渠道
  • 支持长期运行的环境

使用全局分支进行发布管理的好处包括：

  • 快速迭代新功能，并轻松将其发布给最终用户
  • 轻松创建并发布热修复到某个环境中
  • 由于分支在未使用时会自动删除，从而降低基础设施成本
