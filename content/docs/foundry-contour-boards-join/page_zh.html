关联数据集

Contour 提供了多种不同的看板来执行关联操作。本指南将向您介绍如何使用各个看板进行数据集关联、各看板对应的 SQL 语句，以及性能方面的注意事项。

关联看板

关联看板允许您将当前工作数据集与另一个数据集进行关联，并将匹配的结果合并到您的数据中。以下是概览：

示例

假设您有一张包含客户购买信息的表。

您可能还有第二张表，其中包含有关库存中所有物品的信息：

…

您可以使用关联看板，将购买的每件物品的信息补充到起始数据集（transactions）中。

如果两个数据集中存在同名的列，Contour 会提示您为列名添加前缀。在这个例子中，两个数据集都有一列名为 item_id。我们将对来自传入数据集的列应用前缀“inv”（代表 inventory）。

如果您不想在关联后的结果数据集中包含重复的 item_id 列，可以取消选中传入数据集中的该列。

关联看板的配置

选择要执行的关联类型：左连接（添加列）、内连接（交集）、右连接（切换到数据集）或全连接（合并所有数据，尽可能匹配行）。

选择要将其从另一个数据集添加到当前工作数据集中的列。默认情况下，将返回第一个数据集中的所有列。

接着从每个数据集中选择一个或多个键。如果使用多个关联键，您可以选择匹配任意条件或匹配所有条件。

请注意，对于全连接，将返回两个数据集中的所有行；这意味着任一数据集的关联列都可能出现空值，因为两列没有进行合并处理。

合并看板

使用合并看板可基于另一个数据集来更改当前数据集。您可以追加来自另一个数据集的数据（添加行），过滤数据集以仅保留在另一个数据集中存在的数据（保留行），或者根据在另一个数据集中存在的数据移除数据（移除行）。您可以选择根据列在数据集中的位置或列名进行匹配。

联合面板配置

选择保留行、添加行或移除行，然后选择要与之比较的数据集。

对于保留行和移除行，您可以选择出现在或匹配于。

  • 使用出现在时，您可以选择按位置或按名称进行匹配。
  • 使用匹配于时，您必须指定用于连接的列（每个数据集中各指定一列）。
警告

执行联合操作时，两个数据集必须具有相同的列数。因此，如果模式可能发生变更，在使用联合面板时需格外谨慎。例如，如果数据透视表已切换为透视后的数据，位于其下游的联合面板可能会因模式变更而面临列数的意外变动。

SQL 等效操作

对于熟悉 SQL 的用户而言，通过 SQL 中的等效概念来理解 Contour 连接操作可能会有所帮助。下表展示了哪些 SQL 连接类型受哪些面板支持：

连接 (Join)XXXX
联合 (Union)XX

连接

连接操作等效于以下 SQL 语句：



已复制！
1SELECT [DISTINCT] <Column1, Column2, ...>
2FROM CurrentTable
3<INNER JOIN | LEFT OUTER JOIN | RIGHT OUTER JOIN | FULL OUTER JOIN > OtherTable 
4ON <join condition 1>([AND | OR] <join condition 2> [AND | OR] <join condition 3> ...)

Union

保留匹配行等同于 SQL 中的左半连接：



已复制！
1SELECT L.*
2FROM L INNER JOIN (SELECT DISTINCT <join column> FROM R) AS R_KEY
3ON L.<join column> = R_KEY.<join column>

移除匹配行等价于 SQL 左外连接中连接键不匹配的情况：



已复制！
1SELECT L.*
2FROM L LEFT OUTER JOIN R
3ON L.<join column> = R.<join column>
4WHERE R.<join column> is null

数据类型

提示

检查结果集，确保数据类型符合预期。

使用联合面板时，请注意列的数据类型。兼容的列类型会进行强制转换。举一个具体例子，我们使用两个数据集。

dataset1

ID (int)Name (string)
555Alice
666Bob

dataset2

ID (long)Name (string)
555Alice
999Chloe

以 dataset1 为起点，如果我们从 dataset2 中按位置添加行，结果集如下所示：

ID (long)Name (string)
555Alice
666Bob
555Alice
999Chloe

以 dataset1 为起点，如果我们保留在 dataset2 中按位置出现的行，结果集如下所示：

ID (long)Name (string)
555Alice

请注意，即使初始集包含的 ID 列为 int 类型，在结果集中它也变成了 long 类型。保留在另一数据集中出现的行，此操作使用了 Spark 中的 Intersect 函数。

以 dataset1 为起点，如果我们移除在 dataset2 中按位置出现的行，结果集如下所示：

ID (long)Name (string)
666Bob

再次提醒，即使初始集包含的 ID 列为 int 类型，在结果集中它也变成了 long 类型。移除在另一数据集中出现的行，此操作使用了 Spark 中的 Except 函数。

性能注意事项

  • 为两张表选择连接键时，应尽量使用唯一 ID（如主键）。我们强烈建议不要使用外键连接——这样做会导致 Spark 崩溃。
  • 在执行复杂连接或表达式后，应使用保存为数据集功能，以便在继续操作前“保存”你的工作。由于连接结果已持久化到磁盘，这将提升下游查询的性能。

检查结果

关联数据集后，建议查看一下关联结果集的表格，确认数据是否符合预期。在操作栏中选择 表格 添加一个表格看板，然后浏览新生成的关联结果集。
