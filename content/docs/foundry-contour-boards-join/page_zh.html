连接数据集

Contour 提供了多种不同的板来执行连接。本指南将教您 (1) 如何使用每块板来连接数据集，(2) 每块板的 SQL 等效项，以及 (3) 性能注意事项。

连接板

连接板允许您将当前工作数据集与其他数据集进行连接，并将匹配结果合并到您的数据中。以下是概览：

如果您的数据集有列名相同的列，Contour 会提示您为列名添加前缀。在这种情况下，两个数据集都有一个名为 item_id 的列。我们将为来自传入数据集的列应用前缀“inv”（用于 inventory）。

连接板配置

选择一种连接类型进行操作：左连接（Add columns），内连接（Intersection），右连接（Switch to dataset），或全连接（Incorporate all data, matching rows where possible）。

选择要添加到当前工作集的其他数据集中的列。默认情况下，将返回第一个数据集的所有列。

然后从每个集合中选择一个或多个键。如果您使用多个连接键，可以选择 匹配任意 或 匹配全部 条件。

请注意，在全连接中，两个数据集的所有行都将被返回；这意味着任一数据集的连接列可能显示为 null 值，因为这两个列没有进行合并。

联合板的配置

选择 保留行、添加行 或 删除行，然后选择要比较的集合。

对于 保留行 和 删除行，您可以选择 出现于 或 匹配于。

  • 当使用 出现于 时，您可以选择匹配 按位置或按名称。
  • 当使用匹配时，必须指定要连接的列（每组一个列）。
警告

执行联合操作时，两个数据集必须具有相同数量的列。因此，在使用联合板时，如果模式可能发生变化，需要注意。例如，如果将透视表切换为已透视的数据，则下游的联合板可能会由于模式更改而出现列数的意外变化。

SQL等价关系

对于熟悉 SQL 的用户来说，将其视为 SQL 中的等效操作可能会有所帮助。 下表显示了哪些 SQL 连接类型在哪些面板中受到支持：

板联合左连接右外内
连接XXXX
联合XX

连接

连接操作等同于SQL中的以下内容：

Copied!
1SELECT [DISTINCT] <Column1, Column2, ...>
2FROM CurrentTable
3<INNER JOIN | LEFT OUTER JOIN | RIGHT OUTER JOIN | FULL OUTER JOIN > OtherTable 
4ON <join condition 1>([AND | OR] <join condition 2> [AND | OR] <join condition 3> ...)

联合

保留匹配行 等同于SQL中的左半连接：

Copied!
1SELECT L.*
2FROM L INNER JOIN (SELECT DISTINCT <join column> FROM R) AS R_KEY
3ON L.<join column> = R_KEY.<join column>

删除匹配行 等同于SQL中的左外连接，其中连接键不匹配：

Copied!
1SELECT L.*
2FROM L LEFT OUTER JOIN R
3ON L.<join column> = R.<join column>
4WHERE R.<join column> is null

性能考虑

  • 在选择两个表的连接键时，应尽可能使用唯一ID（如主键）。我们强烈建议不要使用外键连接——这样会导致Spark崩溃。
  • 在进行复杂的连接或表达式后，应使用保存为数据集功能，以便在继续之前“保存”您的工作。这将使下游查询性能更好，因为连接已持久化到磁盘。

检查结果

在合并您的数据集后，查看合并后的数据集的表格是一个好主意，以检查结果是否符合您的预期。在操作选项卡中选择表格以添加一个表格面板，并滚动查看您新的合并数据集。
