apollo-cli cve check

检查产品版本中的安全漏洞和CVE扫描状态

检查产品版本中的安全漏洞和CVE扫描状态。

此命令从一个或多个来源收集产品版本坐标，并通过Apollo API进行检查，以确定：

  • 该版本是否因SECURITY_VULNERABILITY被召回
  • 最后一次CVE扫描的时间
  • 影响该版本的活跃CVE

输入源（至少需要一个）：

  1. 环境（-e）：检查该环境中所有实体的产品版本
  2. YAML文件（-f）：检查提供的YAML文件中列出的Maven坐标
  3. 产品ID（-p）：检查指定产品的所有版本

如果提供了多个源，则在检查前合并版本。

使用 -r/--release-channel 按版本通道筛选结果：

  • 对于环境源：仅包括指定版本通道上的实体
  • 对于产品ID来源：仅包含指定发布渠道上的版本

使用 --rescan 触发过去7天内未扫描的版本的CVE重新扫描。

使用 --details 显示包括每个版本细分的详细CVE信息。

示例YAML文件（coordinates.yaml）：

  • com.palantir.storage:storage-controller:1.1811.0
  • com.palantir.foo:bar:1.0.0

示例用法：

检查YAML文件中版本的CVE状态

apollo-cli cve check -f coordinates.yaml

检查环境中所有版本的CVE状态

apollo-cli cve check -e my-environment

检查特定产品的所有版本的CVE状态

apollo-cli cve check -p my-product

apollo-cli cve 检查 -p com.palantir.foo

合并源并触发重新扫描

apollo-cli cve check -e my-environment -f extra-coords.yaml --rescan

按发布渠道筛选

apollo-cli cve check -e my-environment -r RELEASE

显示详细的CVE信息

apollo-cli cve check -e my-environment --details

标志

标志用途
--details显示详细输出，包括每个版本的CVE信息
-e, --environment环境
-h, --help检查帮助
-f, --maven-coordinates-file包含maven坐标列表的文件（yaml格式）
-p, --product-id形式为'group'的产品ID
-r, --release-channel发布渠道
--rescan对过去7天内未扫描的发布版本触发CVE重新扫描

从父命令继承的标志

标志用途
--apollo-client-id用于生成Bearer Token的客户端ID
--apollo-client-secret用于生成Bearer Token的客户端密钥
--apollo-token用于认证端点的Bearer Token
--apollo-token-provider 指定用于认证Apollo端点调用的Bearer Token的提供方式。有效值为"auto"、"static"、"service-user"或"sso"（默认为"auto"）。如果指定"auto"，则根据配置选择模式："apollo-token"设置时为"static"，"apollo-client-id"和"apollo-client-secret"都设置时为"service-user"，否则为"sso"（交互式浏览器登录）。如果同时设置了bearer token和（client id或client secret），将报错。如果指定"static"，则使用"apollo-token"提供的token。如果指定"service-user"，则使用"apollo-client-id"和"apollo-client-secret"从Multipass生成token。如果指定"sso"，则使用针对Multipass的基于浏览器的SSO登录（支持硬件密钥如YubiKey）；生成的token会按配置文件缓存并静默刷新。当第一次需要token且没有有效的缓存token时，将自动触发登录流程。
--apollo-urlApollo的基础URL，用于推导API端点
--debug启用调试级别日志记录
--http-timeout所有apollo请求的超时时间（分钟）
-k, --insecure-skip-verify跳过服务器证书验证
-o, --output输出格式（json,yaml,pretty）
--profile从配置文件中使用特定的配置文件
--quiet不要将日志输出到标准错误
--space-id用于某些作用域为空间的命令的空间 ID
