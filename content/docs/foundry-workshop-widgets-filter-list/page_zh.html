筛选列表

筛选列表微件用于可视化对象数据的高级摘要，并允许用户在该摘要中进行筛选。配置筛选列表微件的模块构建者可使用以下功能：

  • 直方图和分布图：可视化属性类型中最常见的属性值，并支持按属性值进行筛选。
  • 关键词搜索组件：允许用户输入搜索词，对对象集进行广泛筛选。
  • 单选和多选组件：支持对给定属性类型的所有值进行搜索和筛选。
  • 输出对象集筛选变量：允许将此微件中应用的筛选条件传递至模块内的下游对象集变量。
  • 支持通过输出对象集筛选变量设置默认筛选条件。
  • 提供选项允许用户添加和移除属性类型的筛选器。

筛选列表微件的核心配置选项如下：

  • 输入数据
    • 对象集
      • 此输入变量决定了微件中显示的对象数据。
      • 模块构建者可以复用本模块内其他位置创建的现有对象集变量，也可以内联定义一个新的对象集变量。
      • 下方所示的其他多数配置选项，只有在为此对象集参数赋值后才能进行配置。
  • 筛选器配置
    • 添加筛选器
      • 在此处选择某个属性后，该属性将显示在筛选列表中，供用户进行筛选。
    • 筛选组件
      • 此选项决定了每个属性在筛选列表中的可视化方式。
      • 可选项包括关键词、直方图、单选和多选下拉菜单、分布图、单日期和多日期选择器以及时间轴显示。
    • 允许用户添加和移除筛选器
      • 开启此选项后，用户将在微件中看到添加筛选器按钮，并能自行添加和移除可筛选的属性。
  • 布局配置
    • 垂直布局
      • 此选项将所有筛选器按自上而下的垂直滚动方式排列。
    • 药丸布局
      • 此选项将所有筛选器水平排列在一个交互式药丸组件内。选中后，药丸组件会弹出包含筛选配置界面的浮窗。
  • 输出数据
    • 筛选输出
      • 此变量在筛选列表中具有双重作用。
      • 第一重作用是作为微件的输出变量；这是一个对象集筛选变量，包含微件中当前应用的筛选条件，可用于筛选本模块内的对象集变量。
      • 第二重作用是提供设置筛选列表默认筛选条件的方式；为此变量设置默认值后，当模块初始加载时，这些默认的属性筛选条件便会应用到筛选列表中。
      • 只要属性 ID 匹配，筛选输出可用于筛选不同对象类型的对象集。如需检查两种不同对象类型的属性 ID 是否一致，请导航至 Ontology Manager 并查看属性 ID。
      • 通过对对象集筛选变量设置《筛选值提取》，还可以从微件的筛选输出字段中提取筛选后的值，以便在模块的其他位置使用。

筛选多种对象类型的对象集

可以使用变量存储不同对象类型的多个对象集的合集，并将其传递给筛选列表微件。筛选列表将支持以下筛选选项：

  • 公共属性：不同对象类型共有的属性。这些属性必须具有相同的属性 ID才能匹配在一起。共享属性的对象类型必须使用相同的对象存储版本（V1 或 V2）。
  • 单一属性：仅存在于某一个对象类型上的唯一属性。其他任何对象类型中均不存在此属性。

随后，可以将筛选列表小部件的输出变量用于筛选包含合并对象集的变量，这样所有对象类型的实例都会被筛选。

基于链接属性进行筛选

对象存储 V1 上的对象类型具有 100,000 个对象的链接对象筛选限制。如果您的链接对象类型包含超过 100,000 个对象，建议考虑迁移至对象存储 V2。

若要基于链接对象属性进行筛选，请在添加筛选器...下拉菜单的基于链接进行筛选部分中选择一个链接。

选中后，点击链接配置即可添加筛选部分。您将看到与配置选项部分中所述的筛选器配置选项类似的设置，但会包含一些额外选项。

  • 包含链接
    • 包含链接筛选器是链接对象筛选器所独有的，用于基于链接的存在与否进行筛选。例如：“筛选所有包含指向人员链接的任务。”
  • 显示选项
    • 内联
      • 内联显示选项将与链接无关的筛选器和与链接相关的筛选器并列显示（位于同一分组中）。
    • 分组
      • 分组选项会在视觉上将链接筛选器归入一个独立部分，并添加对象图标和链接对象计数，如下图所示。
      • 默认折叠：启用后，此选项将在模块加载时默认以折叠状态显示链接筛选器分组。

高级筛选

筛选列表小部件通过关键字搜索小部件支持更高级的工作流。将搜索类型切换为高级语法后，您可以将搜索操作相互串联，并通过括号定义操作顺序。如果未定义括号，系统将使用常规布尔逻辑来确定运算符的优先级，顺序依次为：引号、括号、NOT、AND、OR。

可以通过筛选列表小部件访问全新的高级关键字搜索小部件。配置好筛选列表小部件后，任何关键字搜索筛选组件的下拉 UI 中都会提供高级语法选项。
