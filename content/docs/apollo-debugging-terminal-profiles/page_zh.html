终端配置文件

终端配置文件允许你自定义用于调试终端会话的容器镜像。你可以使用自己的镜像，而不是默认的Apollo shell，以满足你的调试工作流程需求。例如包括AWS CLI、数据库客户端或内部诊断二进制文件。

配置文件是在Hub级别创建的，并且在该Hub上的所有环境都可用。每个配置文件从一个工件注册表中拉取其镜像，这可以是Apollo内置的容器注册表，也可以是与Hub链接的外部OCI注册表。

创建配置文件

要创建终端配置文件，请导航到Hub 设置 > 终端配置文件并选择创建新配置文件。

填写以下字段：

  • 名称（必填）： 配置文件的名称。之后无法更改。
  • 描述： 配置文件的用途。
  • 工件注册表（必填）： 从哪个 工件注册表 拉取镜像。您可以选择 Apollo 内置的容器注册表，或与 Hub 关联的外部注册表。您不能选择未安装在 Hub 上的注册表。
  • 容器镜像（必填）： 要使用的镜像。对于 Apollo 的容器注册表，请从下拉菜单中选择。对于外部注册表，请输入完整的镜像路径，例如，docker.io/myorg/my-debug-image。
  • 版本： 要使用的镜像标签，例如，1.2.3。如果使用 Apollo 的容器注册表时留空，则使用最新版本。

使用配置文件

在 Environment 上创建新的终端会话时，从 终端配置文件 下拉菜单中选择配置文件。如果没有选择配置文件，则使用默认的 Apollo shell。

配置文件从一个特定的工件注册表中拉取其镜像，因此它仅在安装了该注册表的 Environment 上可用；否则，该配置文件在下拉菜单中显示为禁用状态。

图像要求

自定义图像必须满足以下要求，否则终端会话将无法启动：

  • 图像必须包含 /bin/bash，Apollo 使用它作为交互式会话的入口点。像 Alpine 这样的最小图像默认不包含它。
  • 图像必须以非 root 用户身份运行，并且具有数字 UID，例如在您的 Dockerfile 中使用 USER 5001。集群策略会拒绝以 root 身份运行的 Pod。
  • 图像必须使用特定版本标签，例如 0.2.0。不允许使用 latest 标签，因为 Apollo 需要一个不可变的引用，以确保环境中的每个终端会话都运行相同的图像。
  • 图像必须与目标集群架构匹配，否则容器无法在节点上执行。大多数环境运行 linux/amd64。在 Apple Silicon 上构建时，向 docker build 传递 --platform linux/amd64。

1FROM ubuntu:24.04
2RUN useradd -m -u 5001 testuser
3USER 5001

为正确的平台构建镜像：



已复制！
1docker build --platform linux/amd64 -t myregistry/my-debug-image:0.1.0 .

构建完成后，将镜像推送到配置文件所引用的注册表：外部注册表或 Hub 上 Apollo 的容器注册表。

推送到外部注册表

将镜像推送到您选择的外部注册表：

已复制！
1docker push myregistry/my-debug-image:0.1.0

推送到您的 Hub 的制品库

在 Hub 设置 > 制品库 下找到仓库 URL。此 URL 是在创建仓库时配置的。对于 Apollo 的容器仓库，通常是 containers-<Hub>.palantircloud.com，但具体的主机名取决于您的 Hub。

使用您的完整电子邮件作为用户名，以及 Apollo 令牌 作为密码登录：



已复制！
1docker login <registry-url>

使用注册表主机名对镜像进行标记并推送：



已复制！
1docker tag myregistry/my-debug-image:0.1.0 <registry-url>/myorg/my-debug-image:0.1.0
2docker push <registry-url>/myorg/my-debug-image:0.1.0

推送需要在 资源库 下的 Hub 中拥有 创建者 权限。如果您没有该权限，请联系您的 Hub 管理员，在 Hub 设置 > 权限 > 资源库 下进行授权。

如果您使用现有标签推送新镜像（例如，覆盖 0.1.0），已经拉取旧镜像的节点将继续使用缓存版本。请使用新标签（例如，0.2.0）以确保拉取更新后的镜像。
