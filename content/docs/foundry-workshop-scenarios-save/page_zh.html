保存场景

保存和加载场景使用户能够在不同会话中重复使用长期场景，并允许用户共享和协作处理场景。

通常，将与存储的场景相关的特定工作流元数据一起存储，可以更轻松地组织和查找相关存储的场景。

为此，我们利用本体论（Ontology）来创建代表存储场景的对象。

然后，用户可以自定义名称、附加属性，甚至场景对象与其他对象之间的关系。

对象场景功能

在我们可以开始保存场景之前，我们需要创建一个实现场景特征的对象。

至少，这需要一个属性来保存场景ID，另一个属性来保存场景名称。

我们强烈建议分别使用对象的主键和标题属性来实现此目的。

您还可以添加其他属性以保存有关场景的额外信息，例如 Created By 或 Description。

在场景对象的属性选项卡中，选择您的ID属性并添加类型类 scenarios:versioned-scenario-rid，如下所示。

然后选择您的名称或标题属性并添加类型类 scenarios:scenario-name。

您的场景对象现在已准备好存储已保存的场景，但在您能够在此对象中保存场景到Workshop之前，您需要设置一个创建此类对象的操作。

场景操作

除了场景对象外，我们还需要有创建和更新场景对象的操作，以便在保存场景时在Workshop中使用。

创建场景操作

我们首先创建一个生成实现场景特性（trait）的对象的操作。

在规则标签页中，添加在创建场景时需要提供的属性。

至少，必须添加场景的Scenario Id和Name属性。

现在我们需要确保Scenario ID和Name属性具有正确的类型类。

在表单标签页中，我们可以从属性列表中移除场景对象，因为我们要创建的新操作会生成一个新的场景。

创建一个名为Scenario Id的新字符串属性。

在详情选项卡中，添加类型类 scenarios:versioned-scenario-rid。

对于设置场景名称的操作属性，如果尚未设置，请添加类型类 scenarios:scenario-name。

在规则选项卡中，现在可以将 Scenario ID 设置为最近创建的字符串值。

更新场景操作

接下来，我们将创建一个操作，用于更新实现场景特性（trait）的对象。

在规则选项卡中，添加一个 Name 属性。

在表单中 选项卡，将类型类 scenarios:scenario-object-locator 添加到决定要修改的场景对象的 Action 属性。

对于更新场景名称的 Action 属性，如果尚未设置，请添加类型类 scenarios:scenario-name。

现在场景对象和操作已准备好在 Workshop 中用于保存和更新存储的场景。

配置一个保存场景按钮

我们将从核心按钮组小部件中的按钮启用保存场景功能。

首先，我们向按钮组中添加了一个新按钮，并选择了“保存场景”事件。

为了保存到一个对象，我们必须选择一个实现了场景特征的对象类型。

选择对象类型后，我们必须选择一个创建该类型场景对象的操作。

如果操作参数的类型类配置正确，则场景ID和名称参数应该自动填写（如下所示）。

其他操作参数可以像Workshop中的其他操作一样进行配置，因此您可以设置默认值或留空，让用户填写。

例如，您可以使用Current user特殊变量来填充created by参数。

在配置了创建和更新操作之后，小部件现在允许模块的用户保存场景。

作为用户保存场景

用户可以使用配置好的保存按钮保存新创建的场景以及更新现有的场景。

此外，在场景管理器中，当创建了一个场景但尚未保存时，该场景会显示“新建”标签，直到它被保存，或者如果该场景之前已保存并进行了修改，则会显示“已更新”。

保存新场景时，将打开“创建场景”操作对话框。

名称和场景ID将自动填写，用户将有机会输入或更改任何其他操作参数的值。

在这种情况下，我们添加了一个描述字段，用户可以填写以与场景一起保存。

点击提交后，场景将被保存，并且将不再显示“新建”指示符。

此时，场景已被保存，但除非进行配置，否则在访问此应用程序时不会默认加载。

参见 加载场景 教程，了解如何配置此行为。

无需编辑器角色保存场景

如果您在Workshop模块中保存场景时遇到问题，这可能是因为您的Foundry权限缺少“编辑器”角色。如果希望在不授予模块编辑权限的情况下在Workshop模块中启用保存场景功能，则这是一个特定的问题。如果您不打算保存场景，则不应出现此问题。

对于此问题，请联系您的Palantir代表来（1）创建一个“查看者并可创建场景”的角色，并（2）更新下图第二张图片中选中的查看者项目。
