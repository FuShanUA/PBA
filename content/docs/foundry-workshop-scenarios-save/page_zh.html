保存场景

保存和加载场景功能支持跨会话复用长期场景，方便用户共享场景并展开协同。

在存储场景的同时保存特定于工作流的元数据，有助于更好地整理和查找已存储的相关场景。

为此，我们利用本体创建对象来表示已存储的场景。

用户可以自定义场景对象的名称及其他附加属性，甚至可以定义场景对象与其他对象之间的关系。

对象场景功能

在开始保存场景之前，我们需要创建一个实现了场景特征的对象。

该对象至少需要一个用于存储场景 ID 的属性，以及另一个用于存储场景名称的属性。

强烈建议分别使用对象的主键和标题属性来实现此目的。

你还可以添加其他属性来保存场景的附加信息，例如创建者或描述。

在场景对象的属性选项卡中，选择你的 ID 属性并添加类型类 scenarios:versioned-scenario-rid，如下图所示。

接着，选择你的名称或标题属性，并添加类型类 scenarios:scenario-name。

至此，你的场景对象已可用于存储已保存的场景。不过，在 Workshop 中将场景保存至该对象之前，还需配置一个用于创建该类型对象的 Action。

场景 Action

除了场景对象外，我们还需要配置用于创建和更新场景对象的 Action，以便在 Workshop 中保存场景时使用。

创建场景 Action

首先，创建一个 Action 来生成实现了场景特征的对象。

在规则选项卡中，添加在创建场景时需提供的属性。

至少需要添加场景的 Scenario Id 和 Name 属性。

现在需确保 Scenario ID 和 Name 属性具有正确的类型类。

在表单选项卡中，由于我们正在创建的 Action 会新建一个场景，因此可以从属性列表中移除场景对象。

创建一个名为 Scenario Id 的新字符串属性。

在详情选项卡中，添加类型类 scenarios:versioned-scenario-rid。

对于设置场景名称的 Action 属性，如果尚未设置，请添加类型类 scenarios:scenario-name。

在规则选项卡中，现在可以将 Scenario ID 设置为刚创建的字符串值。

更新场景 Action

接下来，我们将创建一个 Action 来更新实现了场景特征的对象。

在规则选项卡中，添加 Name 属性。

在表单选项卡中，为用于确定待修改场景对象的 Action 属性添加类型类 scenarios:scenario-object-locator。

对于更新场景名称的 Action 属性，如果尚未设置，请添加类型类 scenarios:scenario-name。

至此，场景对象和 Action 已准备就绪，可在 Workshop 中用于保存和更新已存储的场景。

配置“保存场景”按钮

我们将通过核心按钮组构件中的按钮来启用场景保存功能。

首先，我们在按钮组中新增了一个按钮，并设置了点击时触发的“保存场景”事件。

为了将数据保存至对象中，我们必须选择一个实现了场景特性的对象类型。

选择对象类型后，我们必须选择一个用于创建该类型场景对象的动作。

如果正确配置了动作参数的类型类，那么场景 ID 和名称参数将自动填充（如下图所示）。

其他动作参数的配置方式与 Workshop 中的其他动作相同，你可以设置默认值，也可以留空让用户自行填写。

例如，你可以使用 Current user 特殊变量来填充 created by 参数。

配置好创建和更新动作后，该微件现在允许模块用户保存场景了。

以用户身份保存场景

用户可以使用配置好的保存按钮保存新建的场景，也可以更新现有场景。

此外，在场景管理器中，如果场景已创建但尚未保存，该场景会显示“新建”标签，直到被保存为止；如果场景此前已保存并进行了修改，则会显示“已更新”标签。

保存新场景时，系统会打开创建场景动作对话框。

名称和场景 ID 将自动填充，用户可以在此输入或修改任何附加动作参数的值。

在此示例中，我们添加了一个描述字段，用户可以填写该字段并与场景一起保存。

点击 提交 后，场景将被保存，并且不再显示“新建”标识。

此时场景已保存，但除非另外进行配置，否则在访问该应用时默认不会加载此场景。

请查阅《加载场景》教程，了解如何配置此行为。

在没有编辑者角色的情况下保存场景

如果你在 Workshop 模块中保存场景时遇到问题，可能是因为你的 Foundry 权限中缺少编辑者角色。具体来说，如果你想在不授予模块编辑权限的情况下，在 Workshop 模块中启用场景保存功能，就会出现这个问题。如果你不打算保存场景，则不会遇到此问题。

如需解决此问题，请联系你的 Palantir 代表，以 (1) 创建一个“具备创建场景权限的查看者”角色，并 (2) 更新下第二张图中勾选的查看者项。
