Microsoft SharePoint

Microsoft SharePoint 连接器是 Palantir 提供的驱动程序。

要创建一个新的 Microsoft SharePoint 源，请按照 Palantir 提供的驱动程序的标准设置流程，然后使用下面的章节进行 Microsoft SharePoint 特定的配置和网络设置。有关完整的属性参考，请参阅 官方 Microsoft SharePoint 驱动程序文档 ↗。

配置

以下属性为必填或推荐。

属性必需？说明默认值
Schema必填指定提供程序用于连接到 Microsoft SharePoint 的模式类型。SOAP
URL ↗必填指定要连接的 Microsoft SharePoint 站点的基本 URL。此 URL 用作所有 API 调用的起点。https://teams.contoso.com
AuthScheme ↗推荐指定用于连接到 Microsoft SharePoint 的身份验证方案。AzurePassword
AzureTenant ↗推荐标识用于访问数据的 Microsoft SharePoint 租户。接受租户的域名（例如，contoso.onmicrosoft.com）或其目录（租户）ID。—
Password ↗推荐指定用于对用户进行身份验证的密码。—
SharePointEdition ↗推荐指定要连接的 Microsoft SharePoint 版本。SharePoint Online
User ↗推荐指定用于身份验证的 Microsoft SharePoint 用户帐户。username@domain.onmicrosoft.com

网络

下表列出了源需要能够访问的域名，以便成功运行。

对于每个领域，添加相应的 出站策略。如果源在本地托管且无法直接从 Foundry 访问，则应使用 代理程序出站策略；代理主机本身也必须能够访问列出的领域。有关详细信息，请参阅 使用代理程序作为代理。

领域必需
<URL>始终需要。URL 连接属性
