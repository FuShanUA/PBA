权限

在大多数情况下，权限应保持默认设置。自定义权限往往会导致难以追踪不同用户可使用的平台操作。

创建表单

默认情况下，所有用户都可以创建基于电子表格的表单或无来源的表单。仅部分用户能够创建基于对象类型的表单。

详情

要创建表单，你需要在该表单所属文件系统文件夹中拥有创建资源的权限。

根据表单的后端数据，你还需要以下特定权限：

  • 基于对象的表单：fforms:form-definition:create-phonograph-form
  • 基于电子表格的表单：fforms:form-definition:create-fusion-form
  • 无后端来源创建的表单：fforms:form-definition:create-no-origin-form

以下用户组默认拥有这些权限，可在 Foundry Forms 后端配置中覆盖：

  • fforms:form-definition:create-phonograph-form 授予 "fforms-admins" 和 "Platform Administrators" 用户组。
  • fforms:form-definition:create-fusion-form 授予所有用户。
  • fforms:form-definition:create-no-origin-form 授予所有用户。

创建新条目

默认情况下，用户只需对其填写的表单拥有 Viewer 权限即可创建新条目。在可视化编辑器的设置面板中关闭 Allow creation of new objects without read or write permissions 可限制表单，使仅对后端对象类型或电子表格拥有权限的用户才能填写表单。

详情

用户可能因以下原因无法创建新条目：

  • 表单可能没有已发布且可填写的版本。
  • 如果表单中存在任何附件字段，则用户必须拥有向附件文件夹上传文件的权限。

默认情况下，对表单拥有 compass:view 权限的用户将被授予 fforms:form-definition:view。这可以在 Foundry Forms 后端配置中覆盖。

编辑现有条目

与创建新条目一样，你需要拥有查看表单的权限。此外，你还需要拥有编辑后端来源（无论是对象类型还是电子表格）的权限。

详情

Allow creation of new objects without read or write permissions 选项不影响现有条目的编辑。目前没有针对编辑条目的等效选项。然而，Foundry Forms 先前支持此功能，为向下兼容，部分旧表单可能仍允许在无来源权限的情况下编辑条目。

与创建新条目一样，仍需存在已发布的版本，并且你需要拥有向附件字段中引用的任何文件夹上传文件的权限。

保存对表单定义的更改

要编辑表单，你需要对该表单拥有 Editor 权限。如果想限制谁能让用户看到的表单版本进行更改，可以通过限制谁拥有发布表单的权限来实现。

详情

要编辑表单的未发布版本，你需要对该表单拥有 fforms:form-definition:edit 权限。要编辑表单的已发布版本，你还需要发布该表单版本所需的所有权限。

默认情况下，fforms:form-definition:edit 源于对该表单的 compass:edit 权限。这可以在 Foundry Forms 后端配置中覆盖。

发布表单版本

发布表单需要具备该表单的 Editor 权限。若要将发布表单的权限限定为仅限拥有该表单管理权限的用户，可以在后端配置中将 fforms:form-definition:manage 从 compass:edit 扩展项移至 compass:manage 扩展项。

详情

发布表单版本需要具备该表单的 fforms:form-definition:manage 权限。此外还需要：

  • 管理表单来源的权限
  • 附件字段中使用的任意文件夹的 compass:manage 权限

默认情况下，fforms:form-definition:manage 源自该表单的 compass:edit 权限。这些选项可以在 Foundry Forms 后端配置中进行更改，但对附件文件夹的要求无法更改。

创建表单的新版本

创建表单的新版本需要具备该表单的 Editor 权限。根据具体的使用场景，您还需要加入能够创建对象支持或电子表格支持表单的用户组。

详情

创建表单的新版本需要具备创建该来源类型表单的权限。此外还需要 fforms:form-definition:create-new-version 权限以及管理表单来源的权限。

fforms:form-definition:create-new-version 默认源自 compass:edit 权限，但可以在后端配置中进行更改。

更改表单的响应目标

更改表单的响应目标需要具备以下权限：

  • 创建该来源类型表单的权限
  • 管理表单来源的权限
  • 保存表单定义更改的权限，不过不需要附件字段中出现的文件夹权限。

更改表单版本的名称

更改表单版本的名称需要具备该表单的 Editor 权限。如果该表单是已发布版本，则需要具备管理权限以及管理其来源的权限。

详情

管理已发布的表单需要具备 fforms:form-definition:manage 权限，该权限默认源自 compass:edit。可以在后端配置中覆盖此设置。

删除表单

删除表单需要具备该表单的 Editor 权限。还需要 fforms:form-definition:manage 权限，该权限默认源自 compass:edit。可以在后端配置中进行更改。

取消发布表单

取消发布已发布版本的表单需要具备该表单的 Editor 权限。还需要 fforms:form-definition:manage 权限，该权限默认源自 compass:edit。可以在后端配置中进行更改。

管理表单的来源

此权限是多种操作所必需的。根据表单的数据支持方式，所需的权限也有所不同：

  • fforms:form-definition:create-phonograph-form：表单由对象类型支持
  • fforms:form-origin:manage：表单由电子表格支持

fforms:form-definition:create-phonograph-form 会授予“平台管理员”组的成员。fforms:form-origin:manage 会授予对支持表单的电子表格拥有 fusion:edit-document 权限的用户。这些设置可以在 Foundry Forms 后端配置中进行更改。
