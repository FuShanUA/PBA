建议功能

在安排和资源分配工作流中，一个关键的挑战是知道特定事件可以在哪里以及何时发生或可以移动到哪里。大多数日程都有限制，不是所有的日程选项都同样合适；通过将这些限制和标准纳入逻辑中，工具可以帮助用户快速评估选择。建议功能特性通过视觉指示潜在的日程对象滑块放置的适宜性来引导用户行为，这种适宜性基于您组织定义的逻辑。

每个建议功能都有一个TypeScript函数作为支持。规则逻辑的输出可用于突出显示可以进行分配的区域，或者相反，突出显示无法进行分配的区域。应用构建者可以选择通过Workshop小部件配置中的一个设置来强制执行这些建议。当开启此功能时，该功能会强制将滑块放置到最近的高亮区域。

调度甘特图中的建议功能的结果是静态的。该功能在应用初始加载时运行，之后进行的任何操作均不会受到影响。这对你工作流程是否适合使用此功能有影响。

   建议函数接收一组puck主键以及甘特图的开始/结束时间，并返回一个puck主键到行主键到时间槽数组的映射。

type ISuggestion = (
    scheduleObjectPrimaryKeys: string[],
    domainStart: Timestamp,
    domainEnd: Timestamp,
) => FunctionsMap<string, FunctionsMap<string, Array<ISuggestionSlot>>>

/* 建议类型 */

export interface IDomain {
    start: Long;
    end: Long;
}

/* rating用于确定小部件UI中的高亮颜色。基于-1到1的范围。越接近1，高亮颜色越深的绿色。越接近-1，高亮颜色越红。 
*/

export interface ISuggestionSlot {
    domain: IDomain;
    rating: Float;
}

export type IValidSlots = Array<ISuggestionSlot>;
export type ISlotMappings = FunctionsMap<string, IValidSlots>;
export type ISuggestionResult = FunctionsMap<string, ISlotMappings>;
