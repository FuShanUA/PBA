在代码中评估模型

指标集是为使用 foundry_ml 构建的、基于数据集的模型而设计的，该库自 2025 年 10 月 31 日起已被正式废弃。对于新的实现，我们建议改用《实验》。对于使用 palantir_models 构建的模型，指标集不会显示在模型页面上，尽管它们《可以在建模目标中展示》。

在 Foundry 中，可以通过为一个模型创建一个或多个 metric sets（指标集），在代码中评估该单独模型的性能。本页内容假定您已了解 MetricSet 类的相关知识。

由指标集生成的指标与评估数据集的特定事务相关联，可在《建模目标》应用中进行查看。请注意，您需要在《建模目标设置页面》中切换开启 Only show metrics produced by evaluation configuration（仅显示由评估配置生成的指标）选项，才能启用这些指标。

指标与输入数据集的特定事务相关联；每次更新模型或输入数据集后，您可能需要重新运行生成指标集的代码。

在 Code Workbook 中评估模型

若要在《Code Workbook》应用中评估模型：

  1. 创建一个 Code Workbook 或打开已有的工作簿。
  2. 将 foundry_ml 包导入到您的《Code Workbook 环境》中。foundry_ml_metrics 包将作为 foundry_ml 的一部分提供。
  3. 将模型和评估数据集导入到 Code Workbook 中。
  4. 创建一个《转换》，在 Python 中生成 MetricSet 对象，并将您的模型和评估数据集关联为该 MetricSet 的输入。
    • 请务必《将结果保存为数据集》。
    • 模型的《输入类型》需为 Object，而评估数据集的输入类型需为 TransformsInput。
  5. 在您的转换中将指标添加到 MetricSet。
  6. 返回 MetricSet 作为转换的结果。

Updating metrics

As the above code snippets create transforms, the metric sets are created and computed via Foundry Builds. When a model is updated, or a new input data version becomes available, it is important to rebuild the metric set to update the metrics that are associated with that model.
