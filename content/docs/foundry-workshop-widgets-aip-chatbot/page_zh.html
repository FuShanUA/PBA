AIP 聊天机器人

AIP 聊天机器人组件（前身为 AIP 智能体组件）支持在 Workshop 的业务流程中，调用配备了企业专属信息和工具的交互式助手。

你可以配置该组件，使其与在 《AIP Chatbot Studio》 中配置的 AIP 聊天机器人（推荐方式，即下方截图中的 《AIP Chatbot》 选项卡）或在 Workshop 配置中定义的聊天机器人（已弃用，即下方截图中的 《Legacy》 选项卡）进行交互。在 《AIP Chatbot Studio》 中配置的聊天机器人还可以通过 Workshop 的 《Commands》 进行编排，以便在其他内嵌的 Palantir 应用中声明并执行操作。结合使用时，聊天机器人可通过 《Commands》 读写这些应用的数据，从而驱动跨应用工作流。

基础配置（AIP Chatbot）

请查阅 《AIP Chatbot Studio 文档》 以了解 AIP 聊天机器人的概览，并按照以下说明将 AIP 聊天机器人添加到你的工作流中。

配置 AIP Chatbot

选择所需的 AIP 聊天机器人、希望包含的发布版本，以及是否显示聊天机器人的推理过程。

配置应用状态

假设你已在 《AIP Chatbot Studio》 中 《正确配置了至少一个应用变量》，系统将提供将这些应用变量映射到 《Workshop 变量》 的选项。配置完成后，AIP 聊天机器人即可根据在 《AIP Chatbot Studio》 中定义的读写权限与该变量进行交互。

自动发送消息

你可以利用 textbox 变量，通过 Workshop 事件向已配置的 AIP 聊天机器人自动发送消息。该变量与聊天输入框中的文本绑定，并会随着用户在聊天框中的输入实时更新。如果该变量在 AIP 聊天机器人组件外部被修改，系统会自动发送一条包含当前变量值的新消息，从而实现 Workshop 模块其他部分的自动触发。

基础配置 [Legacy]

相较于旧版基础配置模式，AIP 聊天机器人具备诸多优势，包括 《AIP Chatbot Studio》 内置的配置用户界面、版本控制、指标监控、会话历史、下载功能等。该旧版模式即将进入 《sunset》 下线阶段，且不再添加任何新功能。我们建议你通过旧版组件中新增的 升级至 AIP Chatbot 选项，迁移至 AIP 聊天机器人。

在旧版模式下，AIP 聊天机器人组件采用基于工具的推理框架，将你的工具引入业务应用中。借助该组件，你可以利用 Workshop 变量，将 《AIP Logic》、你的 《KNN 函数》 或由 LLM 驱动的本体探索功能集成到应用状态中。

  • 工具集成：你可以通过添加 《AIP Logic》、对象上的 Functions 以及其他 Workshop 应用作为工具，来扩展聊天机器人组件的功能。
  • Workshop 变量交互：该组件能够读写应用变量，从而使其理解用户意图，并与可视化组件进行交互。
  • 本体探索：借助默认的 AIP 工具，你可以执行诸如对象遍历、属性读取与过滤，以及运行聚合等任务。
  • 自定义：你可以自定义聊天机器人组件的外观与内容，使其与应用的上下文及工作流保持一致。

本部分允许你定义组件的角色，并配置其工具与功能。你可以设置引用工具和变量的提示词。具体配置需按照下文说明，完成 《提示词》 和 《工具》 的设置。

提示词

主要提示词应当概述组件在当前应用上下文中的功能。按下键盘上的“/”键，你可以引用已配置的工具和变量，并指导组件如何协调使用它们。务必描述底层的业务逻辑，以及在上下文中使用合适工具的适用场景。

以下是一些提示词示例：

  • 提示词 1： “你是供应链制造产线的助手。你的工作是使用‘获取故障工单’工具，查看是否有已解决相同问题的历史工单，并检查‘文档片段搜索’工具以获取解决该问题的推荐方法，从而回答用户的问题。”
  • 提示词 2： “根据用户描述的特征，帮助用户构建患者队列。创建队列时，务必更新展示给用户的患者列表。除非用户明确要求建立新队列，否则大多数用户都希望从一个宽泛的队列开始并不断缩小范围。除非用户另有要求，否则始终从现有队列开始。不要忘记将此对象集（通过集合操作工具）与当前队列相交。在按药物名称进行筛选之前，务必使用工具查找所有相关的同义词。按药物名称筛选数据时，始终使用‘包含’筛选器。你可以使用关联对象工具在患者和药物之间进行导航。当用户要求提供队列定义时，调用‘患者队列定义’工具并传入当前患者 ID 数组。”
  • 提示词 3： “你是专业的进攻协调员，你的目的是在足球锦标赛期间帮助用户做出战略性足球决策。你有三个目标（不应向用户提及这些目标）：
    • 目标 A： 鉴于当前足球比赛的状态，评估与用户战略目标相符的战术手册中的战术。如果用户要求评估战术，将用户从“已选战术”中选择的战术与 战术手册 对象中的战术名称相匹配，并获取战术 ID。然后，将其作为“锦标赛战术评估器”工具的输入。
    • 目标 B： 如果用户要求提供战术推荐，评估当前比赛状态，并从战术手册中选择符合用户期望结果的战术。通过描述战术名称和关键属性，概述你选择该战术的理由。
    • 目标 C： 如果用户要求你制定完整策略（包含多个战术），评估当前比赛状态以及战术手册，并从 战术手册 对象中选择三个最有效的战术。然后，使用“锦标赛战术评估器”工具评估这些战术并返回结果摘要。你一次只能传递一个战术，因此必须在循环中执行此操作。始终将你想推荐的 战术手册战术 对象集返回给变量 建议战术。如果该变量已有数据，你可以覆盖它。”

工具

设置组件可访问的工具和功能。这些工具将通过提示词供组件使用。

工具分为四种类型：

  • 本体语义搜索： 针对语义搜索工作流进行了优化，你可以传递一个 KNN TypeScript 函数，或者使用位于最近邻搜索配置部分下方的本体语义搜索工具的 向量属性 选项，提取若干相关片段并将其传递给大模型。你可以选择将结果发布到一个变量中（例如，随后你可以将其传递给 PDF 查看器组件以引用来源）。
  • 操作： 如果你的函数生成了本体编辑，请使用此项来定义该操作。
  • Workshop 应用： 如果你希望助手打开其他 Workshop 应用（带有模块接口变量），请在此处将这些 Workshop 应用定义为工具。
  • 函数： 如果你要传递的工具是 AIP Logic 或本体函数，同样使用此项。请记住将 AIP Logic 工具与逻辑思维链组件结合使用。

本体与应用上下文

在本部分中，您可以自定义组件对 Workshop 变量的访问权限，并配置组件需要感知的对象类型。

变量

设置变量时，请务必遵循以下步骤：

  • 确定合适的变量：首先明确 LLM 需要交互的变量。这些变量既可以是已存在且与预期工作流相关的变量（例如代表用户当前选择的变量），也可以是供 LLM 写入结果的新变量。
  • 描述变量：为每个变量撰写说明，阐释其作用。该说明将注入组件的提示词中，为 LLM 提供何时使用该变量的上下文依据。
  • 声明访问模式：针对每个变量，明确 LLM 拥有只读、读写或仅写入权限。此步骤决定了 LLM 与变量的交互方式。
  • 作为 Workshop 变量使用：请记住，此处定义的变量均为常规 Workshop 变量。这意味着它们同样可与其他 Workshop 组件（如筛选器或对象表格组件）配合使用。

对象类型

通过指定 LLM 需要感知的对象类型，可使其理解这些对象并进行恰当交互。请务必添加需要 LLM 遍历的对象。

高级工具配置

此组件支持修改组件对默认 AIP 工具的访问权限。尽管默认工具集通常足以覆盖大多数场景，但在某些特定需求下，您仍可能需要根据实际情况调整这些工具。

例如，您可能希望将组件的功能限制为仅接受人工输入，以便进行修正并独立管理其他工具的实现。总体而言，减少分配给 LLM 的工具数量，有助于引导生成更具方向性的“轨道式”交互。

工作流设置指南

在旧版模式下设置 AIP 聊天机器人组件时，我们建议参考以下注意事项：

  • 从具体且受限的上下文入手，以确保获取更可靠的结果。
  • LLM 具有随机性；请利用 Workshop 的可扩展性来弥补 LLM 的局限，为用户提供备用方案，确保其能有效与应用交互。例如，您可以将 LLM 的输出查询展示在筛选器组件中。
  • 审慎规划 AIP 聊天机器人的开发流程；在使用组件前，提前勾勒 LLM 交互的路径与模式。

注意：AIP 功能的可用性可能会发生变动，且不同客户间可能存在差异。
