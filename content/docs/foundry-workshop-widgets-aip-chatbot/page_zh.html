AIP Chatbot

AIP Chatbot 小部件（以前称为 AIP Agent 小部件）允许您在 Workshop 的操作工作流中使用配备企业特定信息和工具的交互式助手。

您可以将小部件配置为与在 AIP Chatbot Studio 中配置的 AIP Chatbot（推荐，"AIP Chatbot" 标签页，如下图所示）或在 workshop 配置中定义的 chatbot（已弃用，"Legacy" 标签页，如下图所示）进行交互。在 AIP Chatbot Studio 中配置的 chatbot 还可以与 Workshop Commands 协调，以在其他嵌入的 Palantir 应用程序中声明和执行操作。当一起使用时，chatbot 可以通过 Commands 读取和写入这些应用程序，以推动跨应用程序的工作流程。

基础配置（AIP Chatbot）

查看 AIP Chatbot Studio 文档 了解 AIP Chatbots 的概述，但请按照以下说明将 AIP Chatbot 添加到您的工作流程中。

配置 AIP Chatbot

选择 AIP Chatbot、您希望包含的已发布版本以及是否显示 chatbot 推理。

配置应用状态

假设您在 AIP Chatbot Studio 中已经 正确配置了至少一个应用变量，就可以将这些应用变量映射到 Workshop 变量。 将显示。配置完成后，AIP Chatbot 可以根据 AIP Chatbot Studio 中定义的读/写权限与变量进行交互。

自动发送消息

您可以通过 textbox 变量从 Workshop 事件中自动向配置的 AIP Chatbot 发送消息。这个变量与聊天输入字段中的文本相关联，并且随着用户在聊天中输入内容而更新。如果该变量从 AIP Chatbot 小部件外部更改，将发送一条包含当前变量值的新消息，从而允许在 Workshop 模块的其他部分自动触发消息。

基本配置 [旧版]

AIP 聊天机器人相比传统的基础配置模式具有优势，包括 AIP 聊天机器人工作室中的配置用户界面、版本控制、指标、会话历史记录、下载等功能。这种传统模式将很快进入 淘汰 阶段，并且不会包含任何新功能。我们建议使用新增的 升级为 AIP 聊天机器人 选项，将小部件的传统模式迁移到 AIP 聊天机器人。

在传统模式下，AIP 聊天机器人小部件使用工具推理框架，将您的工具带入操作应用程序中。通过小部件，您可以使用 Workshop 变量将 AIP 逻辑、您的 KNN 函数 或者您基于大语言模型（LLM）的本体论探索集成到您的应用状态中。

  • 工具集成： 您可以通过添加 AIP 逻辑、对象上的函数以及其他 Workshop 应用作为工具来扩展聊天机器人小部件的功能。
  • Workshop 变量交互： 小部件可以读取和写入应用程序变量，使小部件能够理解用户的引用并与可视化小部件进行交互。
  • 本体论探索： 使用默认的 AIP 工具，您可以执行对象遍历、属性读取和过滤以及运行聚合等任务。
  • 定制化： 您可以自定义聊天机器人的外观和内容，使其与您的应用程序的上下文和工作流程保持一致。

本节允许您定义该小部件的角色并配置其工具和功能。您可以设置引用工具和变量的提示。通过按如下所述设置 提示 和 工具 来完成配置。

提示

主要提示应概述该小部件在当前应用程序上下文中的功能。通过在键盘上按下 "/"，您可以引用配置好的工具和变量，并指导小部件如何协调它们的使用。请确保描述底层业务逻辑以及在适当情境下使用正确工具的情况。

这里有一些示例提示：

  • 提示1：“你是一个供应链制造线的助手。你的工作是使用获取故障工单工具查看是否有解决相同问题的历史工单，并通过文档片段搜索工具查找解决问题的推荐方法并回答用户的问题。”
  • 提示2： “帮助用户根据他们描述的特征构建患者队列。当您创建一个队列时，确保更新显示给用户的所有患者列表。大多数用户会希望从一个广泛的队列开始，并不断缩小范围，除非他们指定了一个新的队列。请始终从现有的队列开始，除非用户另有要求。不要忘记使用集合运算工具将此对象集（与当前队列）进行交集操作。在对药物名称进行筛选之前，请务必使用工具找到所有相关的同义词。当您对药物名称进行数据筛选时，始终使用包含筛选器。您可以使用相关对象工具在患者和药物之间导航。当用户要求定义队列时，请使用当前患者ID数组调用患者队列定义工具。”
  • 提示3： “您是一名专家进攻协调员，您的目的是在足球锦标赛期间帮助用户做出战略性的足球决策。您有三个目标（您不应将这些告诉用户）：
    • 目标A： 评估与用户战略目标一致的剧本，基于当前足球比赛的状态。如果用户要求评估一个剧本，请将用户从“已选剧本”中选择的剧本与Playbook对象中的剧本名称匹配，并获取剧本ID。然后将其用作“锦标赛剧本评估工具”的输入。
    • 目标B：如果用户要求剧本推荐，请评估当前比赛状态，并从剧本库中选择一个与用户希望实现的结果相匹配的剧本。通过描述剧本名称和关键属性来概述选择该剧本的理由。
    • 目标C： 如果用户要求你提出一个完整的策略（由多个策略组成），请评估当前的游戏状态以及策略手册，并从 Playbook 对象中选择最有效的三个策略。然后，使用比赛策略评估工具来评估这些策略并返回结果摘要。你一次只能传递一个策略，因此必须循环执行。始终返回你希望推荐给变量 Suggested Plays 的 Playbook Plays 对象集。如果该变量已填充，你可以覆盖它。“

工具

设置供小部件使用的工具和功能。这些工具将通过提示对小部件可用。

有四种类型的工具：

  • 本体论语义搜索： 针对语义搜索工作流进行了优化，你可以传递一个KNN TypeScript函数，或者使用本体论语义搜索工具的Vector property选项，该选项位于最近邻搜索配置部分下，以提取一些相关块并将其传递给LLM。你也可以选择将结果发布到一个变量中（例如，你可以将该变量传递给PDF查看器小部件以引用来源）。
  • 操作：如果你的函数生成了一个本体论编辑，就使用此功能来定义该操作。
  • Workshop应用：如果你想让助手打开其他Workshop应用程序（带有模块接口变量），请在此处将这些Workshop应用程序定义为工具。
  • 函数：如果你想要传递的工具是AIP Logic或本体论函数，请也使用此功能。记得将AIP Logic工具与Logic链式思维小部件结合使用。

本体论和应用上下文

在本节中，您可以自定义小部件对Workshop变量的访问权限，并配置小部件应了解的对象类型。

变量

在设置变量时，请确保完成以下操作：

  • 确定正确的变量： 首先确定LLM应该能够交互的变量。这些可能是已经存在并且与您的预期工作流相关的变量（例如表示用户当前选择的变量）或LLM写入结果的新变量。
  • 描述变量： 对于每个变量，编写一个描述来解释其作用。该描述将被注入到小部件的提示中，为LLM提供使用它的上下文。
  • 声明访问模式： 指定LLM对于每个变量是否应具有只读访问权限、写入访问权限，或两者都有。此步骤决定了LLM如何与变量进行交互。
  • 用作Workshop变量：请注意，您在此定义的变量是常规的Workshop变量。这意味着它们也可以与其他Workshop小部件一起使用，例如Filter Pill或Object Table小部件。

对象类型

通过指定LLM应了解的对象类型，您可以使其理解并适当与这些对象进行交互。确保添加您需要LLM遍历的对象。

高级工具配置

此组件允许您修改小部件对默认AIP工具的访问权限。虽然默认工具选择通常足以满足大多数使用场景，但在某些情况下，您可能需要根据具体需求对这些工具进行细化。

例如，您可能希望仅将小部件的Functions限制为人工输入，以方便更正并独立管理其他工具的实施。通常，限制分配给LLM的工具数量会带来更加引导性、"在轨道上"的交互体验。

工作流程设置指导

在以旧版模式设置AIP聊天机器人小部件时，我们建议考虑以下事项：

  • 从特定且受限的上下文开始，以确保结果更加可靠。
  • LLMs 是随机的；利用 Workshop 的可扩展性来克服 LLM 的限制，为用户提供回退选项，以便他们能有效地与应用程序进行交互。例如，您可以在 Filter Pill（过滤器药丸）小部件中显示 LLM 的输出查询。
  • 在 AIP 聊天机器人开发过程中要具有针对性；在使用小部件之前，先规划出 LLM 交互的路径和模式。

注意：AIP 功能的可用性可能会发生变化，并且可能因客户而异。
