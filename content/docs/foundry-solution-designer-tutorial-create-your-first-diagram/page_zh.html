教程：创建您的第一个图表

在本教程中，您将创建一个表示常见数据集成模式的简单图表。该模式包括连接到外部系统、转换数据、构建本体论（ontology）和创建面向用户的应用程序。

完成本教程后，您将了解如何：

  • 将组件节点添加到您的图表中。
  • 在节点之间创建连接。
  • 将相关节点分组。
  • 保存您的图表。

创建一个新的图表

  1. 导航到您Foundry注册中的Solution Designer。
  2. 从首页选择新图表。
  3. 一个空白画布打开，您可以开始通过添加节点、从Data Lineage加载图，或探索参考示例来构建您的图表。

添加您的第一个节点

按照以下部分的说明，构建一个图表，显示数据从外部系统通过Foundry进入Workshop应用程序。

添加一个外部系统节点

  1. 在左上角工具栏中选择组件（Component）按钮，或按Cmd/Ctrl + B打开组件搜索和选择弹窗。
  2. 在组件搜索和选择弹窗的左侧面板中选择外部系统（External system）。
  3. 选择相关的外部系统（External system）类型，例如数据存储（Data Store）或文件存储（File Store），然后节点会出现在您的画布上。

添加一个数据源节点并将其连接到您的外部系统节点

添加一个数据源节点并将其连接到您的外部系统节点

接下来，添加一个节点以表示Foundry如何连接到外部系统。

  1. 打开组件搜索和选择模态框（Cmd/Ctrl + B）。
  2. 搜索并添加一个数据源节点，例如数据连接器或数据连接代理。
  3. 将鼠标悬停在外部系统节点的右侧连接点上。
  4. 点击并拖动到数据源节点的左侧连接点。

完成数据流

继续通过按顺序添加以下节点来构建图表：

  1. 数据集（Dataset）: 打开组件搜索和选择模态框，搜索并选择数据集（Dataset）。
  2. 管道构建器（Pipeline Builder）: 添加一个管道构建器（Pipeline Builder）节点以表示数据转换。
  3. 数据集（Dataset）: 添加另一个数据集（Dataset）节点以表示转换后的数据。
  4. 对象类型（Object type）: 选择对象类型（Object type） 资源按钮，在顶部左上角的工具栏中，选择新建对象节点，然后从您的本体论中选择一个对象类型。您也可以创建一个与现有对象类型无关的通用对象节点不链接。
  5. Workshop： 添加一个Workshop节点来表示面向用户的应用程序。
  6. 用户组： 添加一个用户组节点来表示一组用户及其需要执行的任务。

您的图表现在显示了一个完整的数据集成和应用程序工作流程。

相关节点组

为了整理您的图表，请将数据转换组件归为一组。

创建一个组

  1. 按住 Cmd（macOS）或 Ctrl（Windows），然后选择 管道构建器 节点。
  2. 继续按住 Cmd/Ctrl，然后在 管道构建器 节点之后立即选择 数据集 节点，该节点表示转换后的数据。
  3. 验证两个节点是否显示高亮边框，表示它们已被选中。
  4. 在图表工具栏中 添加文本节点 左侧选择 组 按钮。
  5. 在两个选定的节点周围创建一个组容器。

为组命名

  1. 在画布上选择新创建的 组。
  2. 选择默认的 组 标签以给组提供一个描述性名称，例如 Data Transformation。
  3. 在组边界外的画布任何位置选择，然后 保存 您的更改。

现在您的图表有了一个逻辑分组，使您更容易理解工作流的架构。

添加描述和上下文

添加节点描述

您可以为单个节点添加注释以记录其用途：

  1. 选择 Pipeline Builder（管道构建器） 节点。
  2. 选择 输入描述... 并添加描述，例如：Cleanses and transforms raw data into business-ready datasets。
  3. 重复此过程，为图表中的其他关键节点进行操作。

添加文本节点用于注释

要为您的图表添加一般性注释：

  1. 在左上角工具栏中选择 添加文本节点（Add Text node） 按钮，文本节点将出现在画布上。
  2. 选择 双击编辑（Double click to edit） 以编辑文本。
  3. 输入您图表的描述，例如：This diagram represents our core data integration pattern for external vendor data。
  4. 选择并拖动文本节点，将其放置在图表的顶部附近。

保存您的图表

现在您的图表已经完成，请将其保存到Foundry文件系统中：

  1. 在解决方案设计器（Solution Designer）标题栏中选择保存。
  2. 如果这是您第一次保存，将弹出一个保存窗口，您可以输入一个文件名，例如First Diagram，并选择保存的位置。
  3. 选择保存以完成保存过程。

您的图表现在已经保存，并且在您进行未来更改时会自动保存。从文件标题中选择草稿已保存，以查看包含有关您的自动保存草稿的更多信息的模态框。

下一步

现在您已经创建了第一个Solution Designer图表，继续探索其功能和特性，方法如下：

  • 探索参考模式： 在屏幕左侧的导航栏中选择参考图表，以浏览预构建的行业和技术模式。
  • 关联资源： 在您的图表中选择节点，并使用节点工具栏中的关联资源按钮，将它们连接到实际的Foundry数据集、管道或应用程序。
  • 使用AIP架构师： 尝试 AIP架构师，根据您的需求生成工作流计划。
  • 审查您的图表： 使用 AIP批评者，获取基于AI的架构反馈。
  • 自定义图表视图： 打开 图表设置，尝试不同的显示选项，例如仅图标模式或边动画。

有关图表功能和特性的更多详细信息，请参阅 图表。

提示

  • 从概念开始： 从高层次组件开始，之后再进行细化。在第一稿中不需要指定具体的 数据集（dataset）或 管道（pipeline）。
  • 大量使用分组： 按功能对节点进行分组（例如，“数据采集（Data Ingestion）”，“处理层（Processing Layer）”，“应用层（Application Layer）”），以使复杂的图表更容易导航。
  • 利用模式： 浏览参考架构库中的经过验证的模式，以匹配您的用例。您可以加载一个模式并对其进行自定义，而不是从零开始。
  • 根据反馈迭代： 使用 AIP Critic 验证您的设计，然后根据建议进行优化。
  • 考虑分层： 从左到右按逻辑分层组织您的图表（数据源 → 采集 → 处理 → 本体论（ontology） → 应用 → 用户），以便于理解。
