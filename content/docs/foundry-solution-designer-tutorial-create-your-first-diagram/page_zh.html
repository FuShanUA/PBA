教程：创建你的第一个图表

在本教程中，你将创建一个简单的图表，用于表示常见的数据集成模式。该模式包括连接外部系统、转换数据、构建本体以及创建面向用户的应用程序。

完成本教程后，你将掌握以下操作：

  • 在图表中添加组件节点。
  • 在节点之间建立连接。
  • 对相关节点进行分组。
  • 保存图表。

创建新图表

  1. 在 Foundry 环境中导航至 Solution Designer。
  2. 从主屏幕选择 New diagram。
  3. 此时会打开一个空白画布，你可以通过添加节点、从《数据血缘关系》加载图形或浏览参考示例来开始构建图表。

添加你的第一批节点

按照以下章节的说明构建图表，展示数据从外部系统流入 Foundry 并进入 Workshop 应用程序的过程。

添加外部系统节点

  1. 选择左上角工具栏中的 Component 按钮，或按 Cmd/Ctrl + B 打开组件搜索与选择弹窗。
  2. 在组件搜索与选择弹窗的左侧面板中选择 External system。
  3. 选择相关的 External system 类型，例如 Data Store 或 File Store，该节点即会出现在画布上。

添加数据采集节点并将其连接到外部系统节点

接下来，添加一个节点以表示 Foundry 如何连接到外部系统。

  1. 打开组件搜索与选择弹窗（Cmd/Ctrl + B）。
  2. 搜索并添加 Data sourcing 节点，例如 Data Connector 或 Data Connection Agent。
  3. 将鼠标悬停在 External system 节点右侧的连接句柄上。
  4. 点击并拖动至 Data sourcing 节点左侧的连接句柄。

完善数据流

继续按顺序添加以下节点以构建图表：

  1. Dataset： 打开组件搜索与选择弹窗，搜索并选择 Dataset。
  2. Pipeline Builder： 添加 Pipeline Builder 节点以表示数据转换。
  3. Dataset： 添加另一个 Dataset 节点以表示转换后的数据。
  4. Object type： 选择左上角工具栏中的 Resource 按钮，选择 New object node，然后从本体中选择一种对象类型。如果愿意，你也可以创建一个未链接到现有对象类型的通用对象节点。
  5. Workshop： 添加 Workshop 节点以表示面向用户的应用程序。
  6. User group： 添加 User Group 节点以表示用户群体及其需要执行的任务。

你的图表现在展示了完整的数据集成与应用程序工作流。

对相关节点进行分组

为了更好地组织你的图表，请将数据转换组件归为一组。

创建分组

  1. 按住 Cmd（macOS）或 Ctrl（Windows）并选择 Pipeline Builder 节点。
  2. 继续按住 Cmd/Ctrl，然后选择紧接在 Pipeline Builder 之后的 Dataset 节点，该节点代表转换后的数据。
  3. 确认两个节点均显示高亮边框，表明它们已被选中。
  4. 选择图表工具栏中 Add Text node 左侧的 Group 按钮。
  5. 此时会围绕选中的两个节点创建一个分组容器。

命名分组

  1. 在画布上选择新创建的 Group。
  2. 选择默认的 Group 标签，为该分组提供一个具有描述性的名称，例如 Data Transformation。
  3. 选择画布上分组边框外的任意位置，然后 Save 你的更改。

你的图表现在具备了逻辑分组，使得工作流的架构更易于理解。

添加描述与上下文

添加节点描述

可以为单个节点添加备注，以此记录其用途：

  1. 选中流水线构建器节点。
  2. 选择输入描述...并添加描述，例如：将原始数据进行清洗与转换，生成可供业务直接使用的数据集。
  3. 对图表中的其他关键节点重复此操作。

添加用于评论的文本节点

若要为图表添加常规评论：

  1. 点击左上方工具栏中的添加文本节点按钮，画布上即会出现一个文本节点。
  2. 选择双击进行编辑以编辑文本内容。
  3. 输入对图表的描述，例如：此图表展示了我们针对外部供应商数据的核心集成模式。
  4. 选中文本节点并将其拖动至图表顶部附近。

保存您的图表

图表构建完成后，将其保存至 Foundry 文件系统：

  1. 在“解决方案设计器”页头处点击保存。
  2. 若为首次保存，系统将弹出保存对话框。您可以在此输入文件名（例如 第一张图表），并选择保存位置。
  3. 点击保存完成保存操作。

图表现已保存，后续的更改将自动保存。点击文件页头处的草稿已保存，即可在弹出的对话框中查看有关自动保存草稿的详细信息。

后续步骤

创建第一张“解决方案设计器”图表后，您可以通过以下方式继续探索其功能与特性：

  • 探索参考模式：点击屏幕左侧导航栏中的参考图表，浏览预置的行业与技术模式。
  • 关联资源：选中图表中的节点，并使用节点工具栏中的关联资源按钮，将其与实际的 Foundry 数据集、数据流水线或应用程序建立连接。
  • 使用 AIP 架构师：尝试使用《AIP 架构师》，根据您的实际需求生成工作流计划。
  • 审查图表：使用《AIP 评审器》获取针对您架构的 AI 驱动反馈。
  • 自定义图表视图：打开《图表设置》以体验不同的显示选项，例如仅显示图标模式或边线动画效果。

有关图表功能与特性的更多详细信息，请参阅《图表》。

提示

  • 从概念入手：先从宏观层面的组件着手，后续再逐步细化。初稿阶段无需明确具体的数据集或数据流水线。
  • 善用分组：按功能对节点进行分组（例如“数据接入层”、“处理层”、“应用层”），可使复杂的图表更易于浏览。
  • 复用模式：在参考架构库中浏览与您用例相匹配的成熟模式。您可以直接加载某个模式并进行定制，无需从零开始构建。
  • 基于反馈迭代：使用《AIP 评审器》验证您的设计，随后根据其建议进行优化。
  • 分层规划：将图表按逻辑层级从左至右进行组织（数据源 -> 接入 -> 处理 -> 本体 -> 应用 -> 用户），以便于直观理解。
