作业分组

在 Pipeline Builder 中，每次成功的部署都会触发一次构建。默认情况下，每个批量流水线的输出都会构建为独立的作业，这些作业将独立成功或失败。在流式流水线中，所有输出会打包成一个在单一 Flink 集群上运行的作业，因此所有输出流会一起成功或失败。

Pipeline Builder 中的作业分组功能允许你将批量流水线中的多个输出打包到一个作业中，或者将流式流水线中的每个输出拆分为独立的作业。你还可以为每个作业分组指定计算配置，从而对输出的构建方式提供精细化的控制。计算资源由作业组内的所有输出共享。

当你希望输出逻辑并行更新时，将输出分组到单个作业中非常有用。在批量流水线中，必须将输出放入同一个作业组中，才能通过检查点高效计算共享逻辑。详细了解 Pipeline Builder 中的检查点。

当你希望输出独立于其他输出运行时，将输出划分为更小的组或单输出作业会很有帮助。请注意，流式流水线中的每个作业组都需要各自的 Flink 集群，这会增加计算成本。

在流式或增量流水线中移动作业组之间的输出被视为破坏性变更，将触发强制重放。详细了解 Pipeline Builder 中的破坏性变更。

分配作业组

要分配作业组，请右键单击 Pipeline Builder 图表中的任何输出节点以打开上下文菜单。然后，将鼠标悬停在分配作业组上。在批量流水线中，输出默认为独立作业。在流式流水线中，输出默认为单个作业组。

选择新建组将输出分配到新的作业组。右侧将打开构建设置面板，并自动为作业组分配一种颜色，以便在图表上轻松识别。

作业分组与标记

在作业组中，来自所有输入的标记将由同一作业组内的所有输出继承，即使这些输出并未直接连接到带有标记的输入。详细了解标记继承。

在以下示例中，Input_A 带有标记，而 Input_B 没有标记。Output_X 和 Output_Y 位于同一作业组中。尽管 Output_Y 并未直接连接到 Input_A，但在部署时它会继承 Input_A 的所有标记。

如果你不希望在 Output_Y 上带有标记，可以预先移除从作业分组继承的标记。
