创建 Helm 图表发布

对于尚未存在于 Apollo 中的产品，您必须先 创建一个新产品 才能创建产品发布。

您可以通过以下两种方式在 Apollo 中创建产品发布：

  • 遵循 产品发布流程
  • 使用 Apollo CLI

产品发布流程

  1. 导航到 Apollo 中的 产品 页面并选择要创建发布的 Helm 图表产品。

  1. 选择 创建新版本。发布创建表单将打开，并且从产品的元数据中预先填充了 组 ID 和 工件 ID 字段。

  1. 在发布版本字段中，输入此发布的版本号。版本必须遵循语义化版本格式（例如，1.2.3 或 1.2.3-rc1）。

  2. 在产品类型下，选择Helm chart。

  3. 在发布类型下，选择该图表是在OCI 仓库还是Helm 仓库中托管。

  4. 完成其余字段：

  • 在Helm 图表名称中，输入图表在仓库中的名称（例如，wordpress）。

  • 在Helm 图表版本中，输入图表在仓库中的版本（例如，13.1.4）。

  • 在Helm 仓库 URL中，输入托管该图表的仓库的 URL：

    • 对于 OCI 仓库，URL 必须以 oci:// 开头（例如，oci://docker.example.com/charts）。
    • 对于 Helm 仓库，URL 必须以 https:// 开头（例如，https://charts.bitnami.com/bitnami）。

  • 可选地，在OCI 艺术品中，添加一个或多个 Apollo 应与该发布相关联的 OCI 艺术品 URI（例如，docker.example.com/charts/my-chart:1.0.0）。

  1. 选择创建发布。

发布创建后，会显示确认消息。选择查看产品发布以导航到新发布的详细页面。

为Apollo打包Helm图表

Apollo将它管理的软件视为产品。为了将现有的Helm图表打包用于Apollo，我们需要执行以下操作：

  • 创建一个Apollo产品tarball
    • 这个产品tarball还将包含元数据，告诉Apollo该应用程序属于哪个组、产品和版本。
  • 使用Apollo发布你的产品tarball

请注意，我们仅会通过Apollo发布关于产品的元数据，这使得编排引擎能够在任何 spoke 环境中为产品的安装生成计划。你需要确保该产品的镜像在你公司的镜像仓库中，并且每个 spoke 都可以访问该仓库以拉取镜像。

在这个例子中，我们将考虑Wordpress，这是一个流行的开源内容管理系统。

创建一个Apollo产品tarball

可以在您的仓库中创建一个Apollo产品tarball，如果您正在部署自己的应用程序，或者可以单独创建，如果您使用的是公共镜像（如我们在这里所做的）。

对于Helm charts，Apollo产品tarball的布局如下：

wordpress-0.0.0-88-g70e29b2
├── deployment            # Apollo所需的元数据
  └── manifest.yml        # 此处包含相关容器镜像的URL

我们的所有文件都包含在一个名为wordpress的目录中。 deployment/manifest.yml 文件包含Apollo平台所需的元数据；此文件应包含以下内容：



已复制！
1manifest-version: "1.0"
2product-group: <组 ID>
3product-name: <产品名称>
4product-version: <版本> # 此版本用于下载工件
5                           # 它可以与 Helm Chart 版本不同，
6                           # 但建议保持一致。
7product-type: helm-chart.v1
8extensions:
9  helm-chart:
10    helm-chart-name: <Helm Chart 仓库中的 Helm Chart 名称>
11    # 将用于在 Apollo 控制中心预设版本
12    helm-chart-version: <Helm Chart 的版本>

组ID、产品名称和版本应仔细选择；它们将用于构建Apache Maven坐标，且product-version字段必须是Apollo可订购的版本（详见产品、版本和发布以获取更多详细信息）。

apollo-cli可用于创建Apollo产品tarball。以下命令将创建一个包含预期清单的TGZ：

product-release helm-chart init --as-tgz --maven-coordinate=[maven-coordinate] --repository-url=[helm-repository-url] --name=[helm-chart-name] --version=[helm-chart-version]

以下是示例调用：



已复制！
1./apollo-cli product-release helm-chart init --as-tgz --maven-coordinate="com.palantir.example:wordpress:13.1.4" --repository-url="https://charts.bitnami.com/bitnami" --name=wordpress --version=13.1.4

如果已存在有效的清单文件，可以使用 product-release init --manifest-file=[manifest-file] 命令创建一个包含清单的制品 tarball 文件。这里是一个调用示例（假设工作目录中存在有效的 manifest.yml 文件）：



已复制！
1./apollo-cli product-release init --manifest-file=manifest.yml

在产品发布中设置依赖项

Apollo 的首要概念 产品依赖 确保只有在所有必需的前置条件（即依赖项）都满足时才会进行升级，并且会维护对消费者的义务。

这使得快速发展的产品团队可以独立发布软件，而无需与其他产品团队协调或顺序进行升级。

产品依赖项嵌入在产品发布版本的清单中，位于扩展部分下的 product-dependencies 中。



已复制！
1extensions:
2  product-dependencies: # 产品依赖项的无序列表
3     # (必需) 依赖项的产品组（根据其清单文件）
4   - product-group: <组 ID>
5     # (必需) 依赖项的产品名称（根据其 SLS 清单文件）
6     product-name: <产品名称>
7     # (必需) 指示此依赖项最低允许版本（包括）的可排序版本
8     # 版本（包括）
9     minimum-version: <最小版本>
10     # (必需) 表示此依赖项最高允许版本（包括）的版本匹配器
11     # 版本（包括）
12     maximum-version: <最大版本>
13     # (可选) 指示此依赖项推荐版本的可排序版本
14     # 如果省略，这将等于最小版本
15     recommended-version: <推荐版本>
16     # (可选) 该依赖项是否为可选。默认为 false。
17     optional: false

用于示例helm-charts产品的已填充清单的示例：



已复制！
1manifest-version: '1.0'
2product-group: com.palantir.example
3product-name: wordpress
4product-version: 13.1.4
5product-type: helm-chart.v1
6extensions:
7  helm-chart:
8    - helm-chart-name: wordpress
9      helm-chart-version: 13.1.4
10  product-dependencies:
11    - product-group: com.palantir.foo
12      product-name: foo
13      minimum-version: 1.3.5
14      maximum-version: 1.x.x
15      optional: false
16    - product-group: com.palantir.bar
17      product-name: bar
18      minimum-version: 2.7.0
19      maximum-version: 2.x.x
20      optional: true

工件清单扩展

如果您在 Apollo Hub 上启用了 漏洞扫描，您必须告知 Apollo 在您的 Helm 图表中使用了哪些容器镜像。您可以使用 artifacts 清单扩展来实现这一点。要了解更多内容，请参阅 容器镜像规范。

访问令牌

您需要一个 Bearer 令牌或客户端 ID 和客户端密钥，才能使用 Apollo CLI 发布产品。

了解有关配置 Apollo CLI 身份验证的更多信息。

将您的 Helm Chart 发布到 Apollo

一旦您的 Helm Chart 已打包，就可以使用 apollo-cli 将其发布到 Apollo；您需要访问令牌进行身份验证。

以下命令使用指定的参数创建并发布 Helm chart 艺术品：

publish --apollo-url=[url] --apollo-token=[token] helm-chart --maven-coordinate=[maven-coordinate] --helm-repository-url=[helm-repository-url] --helm-chart-name=[chart-name] --helm-chart-version=[chart-version]

以下命令使用客户端 ID 和客户端密钥进行身份验证，创建并发布 Helm chart 艺术品：

publish --apollo-url=[url] --apollo-client-id=[client-id] --apollo-client-secret=[client-secret] helm-chart --maven-coordinate=[maven-coordinate] --helm-repository-url=[helm-repository-url] --helm-chart-name=[chart-name] --helm-chart-version=[chart-version]

以下是一个调用示例：



已复制！
1./apollo-cli publish --apollo-url="$APOLLO_URL" --apollo-token="$APOLLO_TOKEN" helm-chart --maven-coordinate="com.palantir.example:wordpress:0.0.0-88-g70e29b2" --helm-repository-url="https://charts.bitnami.com/bitnami" --helm-chart-name=wordpress --helm-chart-version=13.1.4

如果本地已存在 Apollo 艺术品 TGZ，可以使用命令 publish --apollo-url=[url] --apollo-token=[token] artifact --product-tgz=[artifactPath] 进行发布。

以下是一个调用示例：



已复制！
1./apollo-cli publish --apollo-url="$APOLLO_URL" --apollo-token="$APOLLO_TOKEN" artifact --product-tgz=wordpress-13.1.4.helmchart.config.tgz

如果本地存在一个清单文件，可以使用以下命令来发布该清单对应的产品：

publish --apollo-url=[url] --apollo-token=[token] manifest --manifest-file=[manifestPath]

以下是一个调用示例：



已复制！
1./apollo-cli publish --apollo-url="$APOLLO_URL" --apollo-token="$APOLLO_TOKEN" manifest --manifest-file=manifest.yml`

此 CLI 可轻松集成到您的 CI 流程中，以发布 Helm Chart 的后续版本。
