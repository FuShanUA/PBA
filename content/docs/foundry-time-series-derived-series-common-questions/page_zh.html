常见问题解答

派生时间序列逻辑支持哪些 Quiver 卡片？

派生时间序列逻辑支持以下 Quiver 卡片：

  • 对象时间序列属性
  • 过滤时间序列
  • 导数
  • DSP 过滤器
  • 积分
  • 采样
  • 分段统计
  • 时间序列公式
  • 合并时间序列
  • 累计聚合
  • 周期聚合
  • 滚动聚合
  • 相对时间序列
  • 偏移时间序列
  • 偏移日期/时间
  • 数值参数
  • 字符串参数
  • 日期/时间参数
  • 日期/时间范围参数
  • 持续时间单位参数
  • 布尔参数

为什么我会收到关于多个根对象的错误？

如果在 Quiver 创建对话框中收到关于多个根对象的错误，说明您的分析很可能是从传感器对象类型开始的，并使用了两个不同的独立对象进行比较。例如，您可能是从传感器对象类型中提取并比较了 入口压力 和 出口压力 这两个传感器对象，而不是从 机器 1 这个根对象类型中进行提取。要创建派生时间序列，您需要从根对象实例开始操作。

请查阅《设置模板化派生时间序列的逻辑要求》，以了解为什么派生时间序列必须从根对象的角度生成。

能否将相同的派生时间序列模板应用于不同对象？

可以。逻辑和计算已实现模板化，因此可以从任何具有相同对象类型的绑定对象的角度来计算结果。请查阅关于《将派生时间序列存储在本体中》的章节，并注意同一个 codex 模板是如何应用于不同的 机器 对象或传感器对象的。

URL 中的 codex 模板 RID 和派生时间序列 RID 有什么区别？

派生时间序列 RID 用于标识 Quiver 资源，其中包含资源名称、描述和其他元数据。而 派生时间序列管理 页面上列出的 codex 模板 RID 则是对运行时计算派生时间序列数据所用模板逻辑的引用。请查阅关于《将派生时间序列存储在本体中》的章节以获取更多详细信息。

如何在 TSP 中引用特定版本的派生时间序列逻辑？

要引用特定版本的逻辑，您的 TSP 值应如下所示：

{"templateRid":"ri.codex-emu.main.template.8da5f759-4b...","templateVersion":"0.0.x"}

如果输入时间序列发生变化，派生时间序列是否也会相应更新？

由于派生时间序列是在读取时计算的，因此它会显示输入时间序列中所有可用的更新。例如，在 Quiver 中可见的任何针对输入时间序列的更改，也会反映在派生时间序列中。

能否将派生时间序列作为另一个派生时间序列的输入？

可以，派生时间序列支持嵌套，可用于计算另一个派生时间序列。您可以像使用原始时间序列那样使用派生时间序列。请务必遵循《派生时间序列创建要求》，以确保新的派生时间序列能按预期运作。

当我更新派生时间序列的逻辑时，使用该派生时间序列的应用程序是否也会相应更新？

派生时间序列具有即时更新逻辑的优势，且不会消耗管道资源。呈现该派生时间序列的应用程序（通常是嵌入在应用程序中的 Quiver 分析或 Quiver 仪表板）将使用更新后的模板化逻辑来计算派生时间序列。

TSP 能否同时包含衍生序列和非衍生时间序列？

可以！TSP 可以混合包含多种值，这些值既可以是（非衍生）时间序列的序列 ID，也可以是衍生序列的 Codex 模板 RID。这通常用于 传感器对象类型。如果你的 TSP 由多个同步任务支撑且包含衍生序列，那么该衍生序列的 Codex 模板 RID 不应采用限定序列 ID 格式。
