高层转换

在 Java 中，数据转换通常会涉及读取、处理以及写入 DataFrame 对象。回顾一下，在 Java API 中，DataFrame 由 Dataset<Row> 表示。如果你的数据转换依赖于 DataFrame 对象，可以定义一个高层 Transform。高层 Transform 接受 Dataset<Row> 类型的输入，并要求计算函数返回单个 Dataset<Row> 类型的输出。此外，你也可以定义一个更通用的底层 Transform，并显式调用 asDataFrame() 方法来获取包含输入数据集的 Dataset<Row>。

要定义高层 Transform，你需要定义一个计算函数，该函数接受任意数量的 Dataset<Row> 类型输入，并返回单个 Dataset<Row> 类型的输出。

自动注册

以下示例展示了如何在 myproject.datasets 包中创建名为 HighLevelAutoTransform 的类来定义 Transform：

1package myproject.datasets;
2
3import com.palantir.transforms.lang.java.api.Compute;
4import com.palantir.transforms.lang.java.api.Input;
5import com.palantir.transforms.lang.java.api.Output;
6import org.apache.spark.sql.Dataset;
7import org.apache.spark.sql.Row;
8
9/**
10* 这是一个旨在用于自动注册的高级 Transform 示例。
11*/
12public final class HighLevelAutoTransform {
13
14   // 自动注册 Transform 的类包含计算函数以及关于输入/输出数据集的信息。
15   // 
16   // 自动注册需要 "@Input" 和 "@Output" 注解。
17   @Compute
18   @Output("/path/to/output/dataset")
19   public Dataset<Row> myComputeFunction(@Input("/path/to/input/dataset") Dataset<Row> myInput) {
20       // 高级 Transform 的计算函数返回类型为 "Dataset<Row>" 的输出。
21       return myInput.limit(10);
22   }
23}



高层转换支持多输入单输出。因此，每个输入参数必须使用 @Input 注解进行标注（其中包含输入数据集的完整路径），并且计算函数必须使用 @Output 注解进行标注（其中包含输出数据集的完整路径）。


现在，你可以在 PipelineDefiner 实现中调用 Pipeline.autoBindFromPackage() 方法，将此 Transform 添加到项目的 Pipeline 中：

2

4import com.palantir.transforms.lang.java.api.PipelineDefiner;

13  }
14}



手动注册


以下示例展示了如何在 myproject.datasets 包中创建名为 HighLevelManualFunction 的类来定义 Transform：




已复制！
1package myproject.datasets;
2
3import com.palantir.transforms.lang.java.api.Compute;
4import org.apache.spark.sql.Dataset;
5import org.apache.spark.sql.Row;
6
7/**
8* 这是一个用于手动注册的高阶 Transform 的计算函数示例。
9*/
10public final class HighLevelManualFunction {
11
12   // 手动注册的 Transform 类仅包含计算函数。
13   @Compute
14   public Dataset<Row> myComputeFunction(Dataset<Row> myInput) {
15       // 高阶 Transform 的计算函数返回类型为 "Dataset<Row>" 的输出。
16       return myInput.limit(10);
17   }
18}



现在，在 PipelineDefiner 实现类中，你可以通过 HighLevelTransform.builder() 完成 Transform 的定义，随后调用 Pipeline.register() 将该 Transform 注册到项目的 Pipeline 中：




已复制！
1package myproject;
2
3import com.palantir.transforms.lang.java.api.Pipeline;
4import com.palantir.transforms.lang.java.api.PipelineDefiner;
5
6public final class MyPipelineDefiner implements PipelineDefiner {
7
8    @Override
9    public void define(Pipeline pipeline) {
10        // 这是一个高级转换的手动注册示例。
11        HighLevelTransform highLevelManualTransform = HighLevelTransform.builder()
12                // 传入要使用的计算函数。这里，"HighLevelManualFunction" 对应
13                // 高级转换计算函数的类名。
14                .computeFunctionInstance(new HighLevelManualFunction())
15                // 传入要使用的输入数据集。
16                // "myInput" 对应计算函数的输入参数。
17                .putParameterToInputAlias("myInput", "/path/to/input/dataset")
18                // 传入要使用的输出数据集。
19                .returnedAlias("/path/to/output/dataset")
20                .build();
21        pipeline.register(highLevelManualTransform);
22    }
23}



高层转换支持多输入和单输出。计算函数的每个输入数据集应通过 putParameterToInputAlias() 方法提供——该方法要求先传入与计算函数参数对应的输入名称，随后指定输入数据集的完整路径。例如，在上文的示例中，“myInput” 就是 my_compute_function() 中的一个输入参数名。请使用 returnedAlias() 提供输出数据集的完整路径。


底层转换


如果你编写的转换逻辑依赖于 DataFrame 对象或文件，可以使用底层 Transform。


自动注册


假设你正在使用自动注册功能。以下示例展示了如何在 myproject.datasets 包中定义名为 LowLevelAutoTransform 的类来创建 Transform 对象：




Copied!
1package myproject.datasets;
2
3import com.palantir.transforms.lang.java.api.Compute;
4import com.palantir.transforms.lang.java.api.FoundryInput;
5import com.palantir.transforms.lang.java.api.FoundryOutput;
6import com.palantir.transforms.lang.java.api.Input;
7import com.palantir.transforms.lang.java.api.Output;
8import org.apache.spark.sql.Dataset;
9import org.apache.spark.sql.Row;
10
11/**
12* This is an example low-level Transform intended for automatic registration.
13*/
14public final class LowLevelAutoTransform {
15
16   // The class for an automatically registered Transform contains the compute
17   // function and information about the input/output datasets.
18   // Automatic registration requires "@Input" and "@Output" annotations.
19   @Compute
20   public void myComputeFunction(
21           @Input("/path/to/input/dataset") FoundryInput myInput,
22           @Output("/path/to/output/dataset") FoundryOutput myOutput) {
23       Dataset<Row> limited = myInput.asDataFrame().read().limit(10);
24       // The compute function for a low-level Transform writes to the output dataset(s),
25       // instead of returning the output(s).
26       myOutput.getDataFrameWriter(limited).write();
27   }
28}


底层 Transform 支持多个输入和输出数据集。因此，每个输入参数必须使用 `@Input` 进行标注（其中包含输入数据集的完整路径），并且每个输出参数必须使用 `@Output` 进行标注（其中包含输出数据集的完整路径）。

现在，你可以在 `PipelineDefiner` 实现中调用 `Pipeline.autoBindFromPackage()` 方法，将此 `Transform` 添加到项目的 `Pipeline` 中：

手动注册

现在，假设你正在使用手动注册方式。在这种情况下，你需要定义一个仅包含计算函数的类。以下是在 myproject.datasets 包中定义名为 LowLevelManualFunction 类的示例：



已复制！
1package myproject.datasets;
2
3import com.palantir.transforms.lang.java.api.Compute;
4import com.palantir.transforms.lang.java.api.FoundryInput;
5import com.palantir.transforms.lang.java.api.FoundryOutput;
6import org.apache.spark.sql.Dataset;
7import org.apache.spark.sql.Row;
8
9/**
10* 这是一个用于手动注册的低级 Transform 的计算函数示例。
11*/
12public final class LowLevelManualFunction {
13
14    // 手动注册的 Transform 类仅包含计算函数。
15    @Compute
16    public void myComputeFunction(FoundryInput myInput, FoundryOutput myOutput) {
17        Dataset<Row> limited = myInput.asDataFrame().read().limit(10);
18        // 低级 Transform 的计算函数会直接写入输出数据集，
19        // 而非返回输出。
20        myOutput.getDataFrameWriter(limited).write();
21    }
22}
现在，在你的 PipelineDefiner 实现中，使用 LowLevelTransform.builder() 创建实际的 Transform 对象，并通过调用 Pipeline.register() 将这个 Transform 添加到项目的 Pipeline 中：
已复制！
1package myproject; 2 3import com.palantir.transforms.lang.java.api.Pipeline; 4import com.palantir.transforms.lang.java.api.PipelineDefiner; 5 6public final class MyPipelineDefiner implements PipelineDefiner { 7 8 @Override 9 public void define(Pipeline pipeline) { 10 // 这是底层 Transform 的手动注册示例。 11 LowLevelTransform lowLevelManualTransform = LowLevelTransform.builder() 12 // 传入要使用的计算函数。这里，“LowLevelManualFunction”对应 13 // 底层 Transform 计算函数的类名。 14 .computeFunctionInstance(new LowLevelManualFunction()) 15 // 传入要使用的输入数据集。 16 // “myInput”对应计算函数的输入参数。 17 .putParameterToInputAlias("myInput", "/path/to/input/dataset") 18 // 传入要使用的输出数据集。 19 // “myOutput”对应计算函数的输入参数。 20 .putParameterToOutputAlias("myOutput", "/path/to/output/dataset") 21 .build(); 22 pipeline.register(lowLevelManualTransform); 23 } 24}
底层转换支持多个输入和输出数据集。在为计算函数提供输入数据集时，应使用 `putParameterToInputAlias()`；提供输出数据集时，则应使用 `putParameterToOutputAlias()`。这些方法需要接收一个与计算函数参数对应的输入/输出名称，以及输入/输出数据集的完整路径。例如，在上文的示例中，“myInput” 和 “myOutput” 就是 `my_compute_function()` 中的输入参数名称。需要注意的是，底层转换的计算函数会直接向输出数据集写入数据，并不会返回任何值。因此，输入/输出数据集会作为参数传递给该计算函数。
