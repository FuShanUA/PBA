Examples

High-Level Transforms

It’s common for data transformations in Java to read, process, and write DataFrame objects. Recall that in the Java API, a DataFrame is represented by a Dataset<Row>. If your data transformation depends on DataFrame objects, you can define a high-level Transform. A high-level Transform accepts inputs of type Dataset<Row> and expects the compute function to return a single output of type Dataset<Row>. Alternatively, you can define a more general low-level Transform and explicitly call the asDataFrame() method to access a Dataset<Row> containing your input dataset.

To define a high-level Transform, you define a compute function that takes in any number of inputs of type Dataset<Row> and returns a single output of type Dataset<Row>.

Automatic registration

Here is an example for how to define a Transform by creating a class called HighLevelAutoTransform in the myproject.datasets package:

Copied!
1package myproject.datasets;
2
3import com.palantir.transforms.lang.java.api.Compute;
4import com.palantir.transforms.lang.java.api.Input;
5import com.palantir.transforms.lang.java.api.Output;
6import org.apache.spark.sql.Dataset;
7import org.apache.spark.sql.Row;
8
9/**
10* This is an example high-level Transform intended for automatic registration.
11*/
12public final class HighLevelAutoTransform {
13
14   // The class for an automatically registered Transform contains the compute
15   // function and information about the input/output datasets.
16   // Automatic registration requires "@Input" and "@Output" annotations.
17   @Compute
18   @Output("/path/to/output/dataset")
19   public Dataset<Row> myComputeFunction(@Input("/path/to/input/dataset") Dataset<Row> myInput) {
20       // The compute function for a high-level Transform returns an output of type "Dataset<Row>".
21       return myInput.limit(10);
22   }
23}

High-level Transforms support multiple inputs and a single output. Thus, each input parameter must be annotated with @Input (which contains the full path to your input dataset), and the compute function must be annotated with @Output (which contains the full path to your output dataset).

Now, you can add this Transform to your project’s Pipeline by calling the Pipeline.autoBindFromPackage() method in your PipelineDefiner implementation:

Copied!
1package myproject;
2
3import com.palantir.transforms.lang.java.api.Pipeline;
4import com.palantir.transforms.lang.java.api.PipelineDefiner;
5
6public final class MyPipelineDefiner implements PipelineDefiner {
7
8  @Override
9  public void define(Pipeline pipeline) {
10    // Provide the Java package containing any Transforms you want to
11    // automatically register.
12    pipeline.autoBindFromPackage("myproject.datasets");
13  }
14}

Manual registration

Here is an example for how to define a Transform by creating a class called HighLevelManualFunction in the myproject.datasets package:

Copied!
1package myproject.datasets;
2
3import com.palantir.transforms.lang.java.api.Compute;
4import org.apache.spark.sql.Dataset;
5import org.apache.spark.sql.Row;
6
7/**
8* This is an example compute function for a high-level Transform intended for manual registration.
9*/
10public final class HighLevelManualFunction {
11
12   // The class for a manually registered Transform contains just the compute function.
13   @Compute
14   public Dataset<Row> myComputeFunction(Dataset<Row> myInput) {
15       // The compute function for a high-level Transform returns an output of type "Dataset<Row>".
16       return myInput.limit(10);
17   }
18}

Now, in your PipelineDefiner implementation, you finish defining your Transform using HighLevelTransform.builder(), and you add this Transform to your project’s Pipeline by calling Pipeline.register():

Copied!
1package myproject;
2
3import com.palantir.transforms.lang.java.api.Pipeline;
4import com.palantir.transforms.lang.java.api.PipelineDefiner;
5
6public final class MyPipelineDefiner implements PipelineDefiner {
7
8    @Override
9    public void define(Pipeline pipeline) {
10        // This is a sample manual registration for a high-level Transform.
11        HighLevelTransform highLevelManualTransform = HighLevelTransform.builder()
12                // Pass in the compute function to use. Here, "HighLevelManualFunction" corresponds
13                // to the class name for a compute function for a high-level Transform.
14                .computeFunctionInstance(new HighLevelManualFunction())
15                // Pass in the input dataset(s) to use.
16                // "myInput" corresponds to an input parameter for your compute function.
17                .putParameterToInputAlias("myInput", "/path/to/input/dataset")
18                // Pass in the output dataset to use.
19                .returnedAlias("/path/to/output/dataset")
20                .build();
21        pipeline.register(highLevelManualTransform);
22    }
23}

High-level Transforms support multiple inputs and a single output. Each input dataset for your compute function should be provided using putParameterToInputAlias()—this method requires an input name corresponding to a parameter for your compute function followed by the full path to your input dataset. For instance, in the example above, “myInput” is an input parameter name in my_compute_function(). Provide the full path to your output dataset using returnedAlias().

Low-level transforms

A low-level Transform can be used if you’re writing data transformations that depend on DataFrame objects or files.

Automatic registration

Let’s say you’re using automatic registration. Here is an example for how to create a Transform object by defining a class called LowLevelAutoTransform in the myproject.datasets package:

Copied!
1package myproject.datasets;
2
3import com.palantir.transforms.lang.java.api.Compute;
4import com.palantir.transforms.lang.java.api.FoundryInput;
5import com.palantir.transforms.lang.java.api.FoundryOutput;
6import com.palantir.transforms.lang.java.api.Input;
7import com.palantir.transforms.lang.java.api.Output;
8import org.apache.spark.sql.Dataset;
9import org.apache.spark.sql.Row;
10
11/**
12* This is an example low-level Transform intended for automatic registration.
13*/
14public final class LowLevelAutoTransform {
15
16   // The class for an automatically registered Transform contains the compute
17   // function and information about the input/output datasets.
18   // Automatic registration requires "@Input" and "@Output" annotations.
19   @Compute
20   public void myComputeFunction(
21           @Input("/path/to/input/dataset") FoundryInput myInput,
22           @Output("/path/to/output/dataset") FoundryOutput myOutput) {
23       Dataset<Row> limited = myInput.asDataFrame().read().limit(10);
24       // The compute function for a low-level Transform writes to the output dataset(s),
25       // instead of returning the output(s).
26       myOutput.getDataFrameWriter(limited).write();
27   }
28}

Low-level Transforms support a multiple input and output datasets. Thus, each input parameter must be annotated with @Input (which contains the full path to your input dataset), and each output parameter must be annotated with @Output (which contains the full path to your output dataset).

Now, you can add this Transform to your project’s Pipeline by calling the Pipeline.autoBindFromPackage() method in your PipelineDefiner implementation:

Copied!
1package myproject;
2
3import com.palantir.transforms.lang.java.api.Pipeline;
4import com.palantir.transforms.lang.java.api.PipelineDefiner;
5
6public final class MyPipelineDefiner implements PipelineDefiner {
7
8  @Override
9  public void define(Pipeline pipeline) {
10    // Provide the Java package containing any Transforms you want to
11    // automatically register.
12    pipeline.autoBindFromPackage("myproject.datasets");
13  }
14}

Manual registration

Now, let’s say you’re using manual registration. In this case, you would define a class that contains just your compute function. Here is an example for how to define a class called LowLevelManualFunction in the myproject.datasets package:

Copied!
1package myproject.datasets;
2
3import com.palantir.transforms.lang.java.api.Compute;
4import com.palantir.transforms.lang.java.api.FoundryInput;
5import com.palantir.transforms.lang.java.api.FoundryOutput;
6import org.apache.spark.sql.Dataset;
7import org.apache.spark.sql.Row;
8
9/**
10* This is an example compute function for a low-level Transform intended for manual registration.
11*/
12public final class LowLevelManualFunction {
13
14    // The class for a manually registered Transform contains just the compute function.
15    @Compute
16    public void myComputeFunction(FoundryInput myInput, FoundryOutput myOutput) {
17        Dataset<Row> limited = myInput.asDataFrame().read().limit(10);
18        // The compute function for a low-level Transform writes to the output dataset(s),
19        // instead of returning the output(s).
20        myOutput.getDataFrameWriter(limited).write();
21    }
22}

Now, in your PipelineDefiner implementation, you create your actual Transform object using LowLevelTransform.builder(), and you add this Transform to your project’s Pipeline by calling Pipeline.register():

Copied!
1package myproject;
2
3import com.palantir.transforms.lang.java.api.Pipeline;
4import com.palantir.transforms.lang.java.api.PipelineDefiner;
5
6public final class MyPipelineDefiner implements PipelineDefiner {
7
8    @Override
9    public void define(Pipeline pipeline) {
10        // This is a sample manual registration for a low-level Transform.
11        LowLevelTransform lowLevelManualTransform = LowLevelTransform.builder()
12                // Pass in the compute function to use. Here, "LowLevelManualFunction" corresponds
13                // to the class name for a compute function for a low-level Transform.
14                .computeFunctionInstance(new LowLevelManualFunction())
15                // Pass in the input dataset(s) to use.
16                // "myInput" corresponds to an input parameter for your compute function.
17                .putParameterToInputAlias("myInput", "/path/to/input/dataset")
18                // Pass in the output dataset(s) to use.
19                // "myOutput" corresponds to an input parameter for your compute function.
20                .putParameterToOutputAlias("myOutput", "/path/to/output/dataset")
21                .build();
22        pipeline.register(lowLevelManualTransform);
23    }
24}

Low-level Transforms support multiple input and output datasets. Each input dataset for your compute function should be provided using putParameterToInputAlias(), and each output dataset should be provided using putParameterToOutputAlias(). These methods require an input/output name corresponding to a parameter for your compute function as well as the full path to your input/output dataset. For instance, in the example above, “myInput” and “myOutput” are input parameter names in my_compute_function(). Recall that the compute function for a low-level Transform writes to output datasets and does not return a value. This is why your input/output datasets are passed in as parameters to the compute function.
