构建复杂布局

设计应用程序的外观与体验令人兴奋，Slate 赋予了你极大的灵活性，允许你自定义并覆盖组件外观的诸多方面。然而，许多项目在追求与设计稿像素级对齐的过程中偏离了正轨，将其凌驾于工作流功能的简洁、高效实现之上。

换个角度来看，每一次样式覆盖都是在牺牲应用程序的简洁性与可维护性。在选择布局和样式时务必谨慎且深思熟虑，在开展下一轮新功能开发前，务必抽出时间对应用程序进行周期性的重构与清理。

构建复杂布局

使用容器组件来布局你的应用程序。要在“使用足够的容器来合理组织应用”与“无意义的容器嵌套”之间取得平衡。请记住，在构建应用程序时，每个组件最终都会向文档模型 (DOM) ↗ 中增加若干层级，而 DOM 的复杂度与浏览器资源消耗、页面加载速度以及应用的整体响应速度（即你的应用程序有多“重”）直接相关。如果心存疑虑，尽量减少组件数量，并努力避免容器嵌套超过 3 到 4 层。

在几乎所有情况下，你都应该避免使用 CSS 来影响应用程序的布局——远离 position 属性，要么使用组件的静态定位，要么使用弹性容器进行相对定位。在特定场景下会有一些例外，但对于应用程序的常规布局，坚持使用内置定位工具可以避免不必要的复杂性，并提升应用代码的可读性。

对齐组件

Slate 中一个常被忽视的功能是组件间的相互对齐。按住 CTRL 键（Mac 上为 CMD 键）并点击以选择一个或多个组件。选中多个组件后，右侧面板将不再显示组件配置，而是提供多个用于对齐或分布组件的选项。

此外，如果你希望布局“自动”完成，可以使用弹性容器来排列子组件。有关使用弹性容器的更多内容，请参阅下方的响应式布局章节。

显示与隐藏组件

动态布局通常需要根据应用程序的状态来显示或隐藏内容。实现这一目标有多种方法。下文的三种模式介绍了一些选项，并探讨了它们各自的优缺点。

简易方案：使用类隐藏

在最简单的情况下，你可以将组件显示配置中的 Additional Classes 属性模板化为 hidden 来隐藏组件。常规做法是通过函数（或你的状态变量）判断组件是否应显示，然后将该函数的输出结果模板化注入组件中。虽然此法简单，但无法很好地扩展至多个组件；并且在页面加载时，对于“隐藏”的组件，你会看到短暂的闪烁，因为在依赖关系图解析完毕并应用该类之前，它们将保持可见状态。

进阶方案：使用选项卡容器隐藏

更好的方法（尤其是在处理多个组件时）是使用选项卡容器，其中一个选项卡留空，另一个选项卡则包含要显示的组件。此模式的具体操作是应用少量 CSS 来隐藏容器的边框和背景（即使得容器相对于背景“隐形”）：

接下来，对 selectedTabIndex 属性进行模板化处理，以在隐藏和可见状态之间切换。采用这种模式，页面加载时不会出现闪烁，且组件会立即隐藏。

复杂操作：从根元素移除

如果你不仅需要将组件“隐藏”，还需要将其从 DOM 中彻底移除，可以通过在父容器上模板化处理一个附加类，利用根元素选择器来隐藏组件。这种模式最常用于响应式布局中，当你需要隐藏某个组件并且让同级下的其他子组件重新调整位置时。

每个 slate 组件的根元素都有一个 ID 选择器，格式为：widget-[widgetName]。假设我们有一个名为 w_myHTMLWidget 的组件，它是 w_sidebar 容器的子组件。要彻底隐藏该组件，你可以在全局样式中定义一个类：

.hide-myHTMLWidget {
    #widget-myHTMLWidget {
        display: none;
    }
}

然后在 w_sidebar 中，你可以根据条件添加这个类，整个组件就会随之隐藏。关于使用条件 CSS 类的更多模式和示例，请参阅下文的 样式处理 章节。

《“多页”应用程序》

划分应用程序最常用的模式是使用选项卡式容器。每个容器选项卡都可以视为一个独立的页面，但必须注意的是，这些“页面”仅仅是 UI 层面的功能——例如，它不会影响依赖图的解析。因此，即使某个图表位于不同的选项卡上，填充该图表的查询仍会随依赖图的解析而执行。

如果你正在使用选项卡式容器构建多页应用程序，请务必勾选 Lazy rendering enabled（启用延迟渲染）选项——这会将选项卡上组件的渲染时机推迟到选项卡打开时，而不是页面加载时。对于那些在任何时候都有大量组件“隐藏”在其他选项卡上的超大型应用来说，此功能尤为实用，并能显著缩短页面加载时间。

即使您不希望允许用户手动选择特定标签页，您也应当使用标签页容器——例如，当您正在构建一个工作流应用并需要控制何时将用户切换至下一个标签页时。这种情况下，您可以取消勾选 显示标签页标题 选项，随后使用 Handlebars 语句来模板化指定在任意给定时刻应显示哪个标签页（从零开始索引）。一种常见做法是将该标签页作为应用状态的一部分进行追踪（参阅前文的构建有状态应用），这样您只需更新状态变量中当前标签页的值，应用中的标签页便会随之切换。
