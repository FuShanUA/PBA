Iframe

在尝试开发自定义小部件时，我们建议使用 自定义小部件 而不是 Iframe 小部件的双向选项。

Iframe 小部件允许在 Workshop 中嵌入外部全页面应用程序，为构建者提供一种方式，将自定义视图添加到他们的模块中。

嵌入的应用程序也可以通过 Workshop 变量和事件进行双向通信，使嵌入的应用程序能够类似于自定义小部件。有关如何 iframe 嵌入一个可以读取和写入 Workshop 的应用的更多信息，请参阅 双向 配置选项。

请注意，由于 iframe 需要额外的内存和处理资源，可能会影响 Workshop 模块的稳定性和性能。我们不建议在屏幕上嵌入超过一个 iframe 小部件。

为了在您的Foundry环境中嵌入外部资源，您需要配置 内容安全策略 (CSP)。外部资源本身也必须设置 Content-Security-Policy 头的 frame-ancestors 指令 ↗，该指令允许您的Foundry URL嵌入该资源。如果您使用的是与您的Foundry环境外部的URL，并且该URL会请求Foundry API，则您还需要配置 跨域资源共享 (CORS)。

URL

将应用程序的URL作为静态字符串或字符串变量输入。

当嵌入另一个Foundry应用程序时，可以通过添加embedded=true URL查询参数来隐藏Foundry侧边栏。

当粘贴标准的YouTube URL（例如，https://www.youtube.com/watch?v=VIDEO_ID），Workshop将检测URL格式并提供一键选项将其转换为显示在iframe中的正确嵌入格式（https://www.youtube.com/embed/VIDEO_ID）。

Slate

Slate配置选项允许您将Slate应用程序嵌入到您的Workshop模块中。可以使用参数使嵌入的Slate应用程序和Workshop模块相互交互。

以下部分提供了有关如何使用iframe小部件嵌入Slate应用程序的详细信息。

Slate源

Slate 源定义了在您的模块中引用嵌入的 Slate 的方法。您可以选择使用 指南针引用 或 永久链接 作为您的 Slate 源。

  • 指南针引用： 允许您使用 Foundry 资源选择器来选择要嵌入的 Slate 应用程序。
  • 永久链接： 允许您输入 Slate 应用程序的永久链接或 RID 来选择它进行嵌入。您可以在您的 URL 中找到您的 Slate RID：/workspace/slate/documents/\\<slate-permalink>/latest。

URL 参数

URL参数附加到你的Slate网址，并用于设置Slate中的变量。更改URL参数将导致整个页面重新加载，因此我们建议仅对很少需要修改的变量使用URL参数。要在Slate中引用URL参数，请设置一个与URL参数同名的变量，并使用handlebars进行引用：{{username}}. 注意，静态字符串URL参数应进行URI编码，因为它们直接作为URL参数传递。

示例：根据加载时的用户名和ID更改嵌入式Slate应用程序的外观。

输入参数

输入参数从Workshop模块传递到嵌入的Slate应用程序。输入参数由其键和值类型定义。静态字符串、字符串变量、数字和对象集可以传递给嵌入的Slate应用程序。

在你的Slate应用程序中，你可以通过使用事件面板中的Slate.getMessage事件以及下面的代码示例作为参考，从你的Workshop模块检索信息。parameter_key必须与你在Workshop的iframe小部件中定义的键匹配。

Copied!
1const payload = {{slEventValue}}
2return payload["<parameter_key>"]

示例：设置一个Workshop过滤器以调整嵌入式Slate应用程序中的视图。

Copied!
1const payload = {{slEventValue}}
2return payload["flight-alerts"]

输出参数

输出参数从Slate嵌入中传递到Workshop模块。输出参数由其键和值类型定义。对象集、对象集过滤器和字符串变量可以传递到Workshop模块。在您的Workshop模块中，可以通过将Workshop变量分配给参数来从您的Slate应用程序中检索信息。在您的Slate应用程序中，需要使用Slate.sendMessage事件和下面的代码片段来启动信息传输。代码中的参数键必须与您在Workshop小部件中定义的键匹配。

return {
    type: "WORKSHOP//SET_OUTPUT_VALUE",
    outputParameterKey: "<parameter_key>",
    value: {{<data_to_be_sent>}}
}

某些JavaScript原始类型在将值存储到字符串变量时会被映射到适当的Workshop类型。undefined 将被映射到 undefined（实际上将变量恢复为默认值），而 null 将清除字符串变量的值。

示例：使用嵌入式Slate应用程序的选择状态来更改Workshop中的过滤器集状态。

return {
    type: "WORKSHOP//SET_OUTPUT_VALUE",
    outputParameterKey: "selected-objects",
    value: {{f_selection}}
}

可触发事件

您可以通过Slate触发任何 Workshop事件。可触发事件由其键和事件类型定义。您可以触发打开覆盖层、重置变量等事件。

在您的Workshop模块中，定义 event_key 和您想要触发的事件。在您的Slate应用程序中，使用Events面板中的 Slate.sendMessage 事件以及下面的代码片段作为参考。 event_key 必须与iframe小部件中定义的键匹配。

return {
    type: "WORKSHOP//TRIGGER_WORKSHOP_EVENT",
    eventKey: "<event_key>",
}

示例：通过Slate中的按钮点击，在Workshop中切换一个可折叠的部分。

return {
    type: "WORKSHOP//TRIGGER_WORKSHOP_EVENT",
    eventKey: "<event_key>",
}

双向

双向配置选项允许您嵌入一个自定义构建的应用程序，并使用定义的配置字段与Workshop进行双向通信。这使得嵌入的应用程序可以像任何其他Workshop小部件一样运行，能够读取和写入Workshop变量，并执行Workshop事件。为了在您的应用程序中访问本体论，请使用 本体论 SDK（Ontology SDK）。

何时使用

虽然Workshop有 各种可用的小部件 , 有些情况下可能需要超出Workshop内置小部件提供的功能的特定功能。此双向小部件选项允许您创建自己的应用程序，定义配置字段，并像其他任何小部件一样将其嵌入Workshop。

配置双向小部件

为了实现双向通信，您必须在将自定义应用程序作为双向小部件嵌入之前，先对其进行配置。

步骤1：修改您的自定义应用程序以与Workshop进行通信

您的自定义应用程序与Workshop之间的通信是通过npm包 @osdk/workshop-iframe-custom-widget ↗ 进行的。使用以下命令将该包安装到您的自定义应用程序中：

Copied!
1npm i @osdk/workshop-iframe-custom-widget

此包提供通过函数 useWorkshopContext 与 Workshop 进行双向通信的方法，该函数接收您的应用程序希望从 Workshop 接收的变量和事件的定义，并返回一个具有读取 Workshop 变量值、写入 Workshop 变量值以及执行 Workshop 事件接口的上下文对象。请查看 @osdk/workshop-iframe-custom-widget ↗ 的 npm 包页面以获取详细信息和进一步说明。

步骤 2：配置双向小部件

在您的应用程序已部署并能够与 Workshop 进行双向通信后，将一个 Iframe 小部件添加到您的 Workshop 模块中，并选择 双向 选项。

输入应用程序的URL，可以作为静态字符串或字符串变量。小部件的配置面板在等待接收嵌入式应用程序所需的变量和事件定义时，将显示加载状态。

在接收到所需变量和事件的定义后，小部件的配置面板将为每个请求的变量和事件显示变量选择器和事件选择器，允许进行设置。

每次设置的Workshop变量的值发生变化时，该值将被发送到自定义应用程序。设置的事件将确定可以从自定义应用程序中执行哪些事件。

限制

  • 通过iframe应用发送的对象集字段定义是使用单一预设对象类型指定的。在Workshop中配置变量时，变量所受约束的对象类型将在对象集变量的配置面板中显示。如果Workshop对象集变量包含不是预设对象类型对象，则这些对象将在发送到iframe应用之前被过滤掉。

  • 当将对象集变量发送到iframe应用时，最多只能包含10,000个对象。如果您尝试从Workshop向iframe应用传递包含超过10,000个对象的对象集变量，或者尝试从iframe应用内部将包含超过10,000个对象的Workshop变量进行设置，那么第10,000个对象之后的对象将会被截断，并不会发送给接收方。
  • 目前尚不支持结构体（struct）变量。
  • 目前，每次从页面中移除iframe后，再次显示时都会重新加载。为了防止在这种情况下小部件重置，请考虑将双向小部件数据存储在传递给小部件的Workshop变量中。
