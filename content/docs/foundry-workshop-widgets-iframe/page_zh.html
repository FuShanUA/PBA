Iframe

在尝试开发自定义微件时，我们建议使用自定义微件，而不是Iframe微件的双向通信选项。

Iframe 微件支持在 Workshop 中嵌入外部全页应用，为构建者提供了向模块添加自定义视图的方法。

嵌入的应用程序也可以利用 Workshop 变量和事件进行双向通信，使其具备类似自定义微件的功能。请参阅双向通信配置选项，详细了解如何以 Iframe 方式嵌入一个能够读取和写入 Workshop 的应用程序。

请注意，由于 iframe 需要占用额外的内存和处理资源，这可能会影响 Workshop 模块的稳定性和性能。我们不建议在单个屏幕中嵌入多个 iframe 微件。

您需要配置内容安全策略（CSP），才能在您的 Foundry 环境中以 iframe 形式嵌入外部资源。该外部资源本身也必须设置针对 Content-Security-Policy 标头的 frame-ancestors 指令 ↗，以允许您的 Foundry URL 嵌入该资源。如果您使用的 URL 位于 Foundry 环境外部并向 Foundry API 发起请求，则还必须额外配置跨源资源共享（CORS）。

URL

将应用程序的 URL 作为静态字符串或字符串变量输入。

在嵌入其他 Foundry 应用程序时，您可以通过添加 embedded=true URL 查询参数来隐藏 Foundry 侧边栏。

粘贴标准 YouTube URL（例如 https://www.youtube.com/watch?v=VIDEO_ID）时，Workshop 会自动检测该 URL 格式，并提供一键选项将其转换为在 iframe 中显示所需的正确嵌入格式（https://www.youtube.com/embed/VIDEO_ID）。

Slate

通过 Slate 配置选项，您可以将 Slate 应用程序嵌入到 Workshop 模块中。利用参数，可以实现嵌入的 Slate 应用程序与 Workshop 模块之间的交互。

以下章节详细介绍了如何使用 iframe 微件嵌入 Slate 应用程序。

Slate 来源

Slate 来源定义了引用嵌入在模块内的 Slate 的方法。您可以选择使用 Compass 引用或永久链接作为 Slate 来源。

  • Compass 引用：允许您使用 Foundry 资源选择器来选择要嵌入的 Slate 应用程序。
  • 永久链接：允许您输入 Slate 应用程序的永久链接或 RID 来选择并嵌入它。您可以在 URL 中找到您的 Slate RID：/workspace/slate/documents/\<slate-permalink>/latest。

URL 参数

URL 参数会附加到您的 Slate URL 后面，用于在 Slate 中设置变量。更改 URL 参数会导致整个页面重新加载，因此我们建议仅对极少需要修改的变量使用 URL 参数。要在 Slate 中引用 URL 参数，请创建一个与该 URL 参数同名的变量，并使用 Handlebars 语法引用它：{{username}}。请注意，静态字符串 URL 参数应进行 URI 编码，因为它们会直接作为 URL 参数传递。

示例：在加载时根据用户名和 ID 更改嵌入的 Slate 应用程序的外观。

输入参数

输入参数从 Workshop 模块传递到内嵌的 Slate 应用中。输入参数由其键和值类型定义。可以将静态字符串、字符串变量、数字和对象集传递给内嵌的 Slate 应用。

在 Slate 应用内，你可以使用“事件”面板中的 Slate.getMessage 事件并参考下方代码示例，从 Workshop 模块中检索信息。parameter_key 必须与你在 Workshop 的 iframe 小部件中定义的键相匹配。

已复制！
1const payload = {{slEventValue}}
2return payload["<parameter_key>"]

示例：设置 Workshop 筛选器以调整内嵌 Slate 应用中的视图。

已复制！
1const payload = {{slEventValue}}
2return payload["flight-alerts"]

输出参数

输出参数从 Slate 内嵌应用传递到 Workshop 模块中。输出参数由其键和值类型定义。可以将对象集、对象集筛选器和字符串变量传递给 Workshop 模块。在 Workshop 模块内，你可以通过将 Workshop 变量分配给该参数来从 Slate 应用中检索信息。在 Slate 应用内，你需要使用 Slate.sendMessage 事件和下方代码片段来发起信息传递。代码中的参数键必须与你在 Workshop 小部件中定义的键相匹配。

return {
    type: "WORKSHOP//SET_OUTPUT_VALUE",
    outputParameterKey: "<parameter_key>",
    value: {{<data_to_be_sent>}}
}

在将值存储到字符串变量中时，某些 JavaScript 原始类型会被映射为相应的 Workshop 类型。undefined 将映射为 undefined（实际上会将变量重置为其默认值），而 null 将清空该字符串变量的值。

示例：使用内嵌 Slate 应用中的选择状态来更改 Workshop 中的筛选器集合状态。

return {
    type: "WORKSHOP//SET_OUTPUT_VALUE",
    outputParameterKey: "selected-objects",
    value: {{f_selection}}
}

可触发事件

您可以从 Slate 触发任何 Workshop 事件。可触发事件由其键和事件类型定义。您可以触发打开覆盖层、重置变量等事件。

在您的 Workshop 模块中，定义 event_key 以及您想要触发的事件。在您的 Slate 应用中，在事件面板中使用 Slate.sendMessage 事件，并参考以下代码片段。event_key 必须与 iframe 小部件中定义的键相匹配。

return {
    type: "WORKSHOP//TRIGGER_WORKSHOP_EVENT",
    eventKey: "<event_key>",
}

示例：通过 Slate 中的按钮点击来切换 Workshop 中的可折叠部分。

return {
    type: "WORKSHOP//TRIGGER_WORKSHOP_EVENT",
    eventKey: "<event_key>",
}

双向通信

双向通信配置选项允许您嵌入自定义构建的应用，并通过定义的配置字段实现与 Workshop 的双向通信。这使得嵌入的应用能够像任何其他 Workshop 小部件一样运行，具备读取和写入 Workshop 变量以及执行 Workshop 事件的能力。为了在您的应用中访问本体，请使用《本体 SDK》。

何时使用

虽然 Workshop 提供了各种可用的小部件，但在某些情况下，可能需要超出 Workshop 内置小部件所提供功能的特定功能。此双向小部件选项允许您创建自己的应用、定义配置字段，并像使用任何其他小部件一样将其嵌入到 Workshop 中。

配置双向小部件

要启用双向通信，您必须先将自定义应用的源代码配置好，然后才能将其作为双向小部件通过 iframe 嵌入。

步骤 1：修改您的自定义应用以与 Workshop 通信

您的自定义应用与 Workshop 之间的通信是通过 npm 包《@osdk/workshop-iframe-custom-widget ↗》完成的。使用以下命令将该包安装到您的自定义应用中：

已复制！
1npm i @osdk/workshop-iframe-custom-widget

该包通过函数 useWorkshopContext 提供与 Workshop 进行双向通信的方法。该函数接收您的应用希望从 Workshop 接收的变量和事件的定义，并返回一个上下文对象，该对象具有读取 Workshop 变量值、写入 Workshop 变量值以及执行 Workshop 事件的接口。请查看 npm 包页面《@osdk/workshop-iframe-custom-widget ↗》以获取详细信息和进一步的说明。

步骤 2：配置双向小部件

应用程序部署双向通信 Workshop 的更改后，在 Workshop 模块中添加 Iframe 部件并选择 双向 选项。

输入应用程序的 URL，可以输入静态字符串或字符串变量。部件的配置面板将显示加载状态，等待接收嵌入应用程序所需的变量和事件定义。

接收到所需变量和事件的定义后，部件的配置面板会针对请求的每个变量和事件显示变量选择器和事件选择器，允许逐一进行设置。

每次变量的值或加载状态发生改变时，已设置的 Workshop 变量值都会发送到自定义应用程序。已设置的事件将决定可以在自定义应用程序内部执行哪些事件。

限制

  • 嵌入的应用程序发送的对象集字段定义指定了单一预设对象类型。在 Workshop 中配置变量时，对象集变量的配置面板会显示该变量限制的对象类型。如果 Workshop 对象集变量包含非预设对象类型的对象，这些对象会在发送到嵌入的应用程序之前被过滤掉。

  • 对象集变量发送到嵌入的应用程序时最多包含 10,000 个对象。如果尝试将包含超过 10,000 个对象的对象集变量从 Workshop 传递到嵌入的应用程序，或尝试从嵌入的应用程序内部设置包含超过 10,000 个对象的 Workshop 变量，第 10,000 个之后的对象将被截断，不会发送给接收方。
  • 目前不支持结构体变量。
  • 当前，iframe 每次从页面移除并随后再次显示时都会重新加载。为防止部件在这种情况下重置，建议将双向部件数据存储在传递给该部件的 Workshop 变量中。
