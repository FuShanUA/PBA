变更类型 API 参考

每个 CEL 策略评估都会收到一个 type 字段，表示正在评估的变更类型。type 值决定了 referencedResources 和 request 中可用的键。使用 type 来保护策略，使其仅在它设计处理的变更类型上运行。

对于属于模块安装的一部分的实体变更，referencedResources 还包括与相关模块相关的 module 和 moduleRelease 键。

所有变更类型都会用特定于类型的字段填充 request。每个类型的可用字段如下所述。

引用资源字段

referencedResources 中的所有条目都有以下四个字段：

字段类型描述
rid字符串资源标识符 (RID)。将此传递给 require_role() 或 require_operation() 以根据用户对该资源的权限来控制审批。
name字符串资源的显示名称，例如环境名称或产品 ID。
labels映射应用于该资源的标签。键和值都是字符串。如果没有应用标签，则为空。
authorRoles集合更改作者在该资源上的角色。每个条目是一个角色 ID 字符串，如 "entity:operator"。如果作者没有角色，则为空。

大多数条目还有一个 resource 字段，其中包含作为映射序列化的完整资源对象。resource 中的字段因资源类型而异。例如，referencedResources.artifactRegistry.resource.registryType 存储了工件注册表实体的注册表类型。resource 字段对于以下更改类型是空的：

  • CREATE_PIPELINE
  • CREATE_TEAM
  • CREATE_EGRESS_POLICIES
  • DELETE_EGRESS_POLICIES
  • MODIFY_VULNERABILITY_SUPPRESSIONS
  • REMOVE_VULNERABILITY_SUPPRESSIONS
  • MODIFY_SPACE_SETTINGS
  • MODIFY_RESOURCE_OVERRIDE_SPACE_SETTINGS

实体变更

CREATE_ENTITY

当一个实体被添加到环境中时触发。

可用的 referencedResources 键: environment, product（如果该实体分配了产品）

可用的 request 字段:

字段类型描述
apolloEntityId字符串实体的唯一标识符。
apolloEntityType字符串实体类型，例如 "service"、"daemon" 或 "network-security"。
releaseChannel字符串分配给此实体的发布通道。如果实体继承环境默认值，则不存在。
configOverrides映射此实体的产品特定配置。键为版本字符串；值为该版本的配置映射。
entity映射序列化的实体设置。对于在部署配置中声明的实体（服务、守护进程、Helm 图表等）存在；对于跟踪资产为空。参见 entity 子字段。
asset映射序列化的资产设置。对于跟踪资产存在；对于在部署配置中声明的实体为空。参见 asset 子字段。

UPDATE_ENTITY

当实体的应用程序配置被修改时触发。

可用的 referencedResources 键: environment, installation, product（如果该实体分配了产品）

可用的 request 字段:

字段类型描述
apolloEntityId字符串实体的唯一标识符。
apolloEntityType字符串实体类型，例如 "service"、"daemon" 或 "network-security"。
releaseChannel字符串分配给此实体的发布通道。如果实体继承了环境默认值，则不存在。
configOverrides映射更新后此实体的建议产品特定配置。结构与 CREATE_ENTITY 相同。有关 network-security 实体类型的 network-security configOverrides，请参见 network-security configOverrides。
entity映射序列化的实体设置。对于部署配置中声明的实体存在；对于跟踪资产为空。请参见 entity 子字段。
asset映射序列化的资产设置。对于跟踪资产存在；对于部署配置中声明的实体为空。请参见 asset 子字段。

UPDATE_ENTITY_SETTINGS

当实体的操作设置被修改时触发，例如其发布通道。当实体被标记为删除时也会产生。

可用的 referencedResources 键： environment、installation、product（如果实体有分配的产品）

可用的 request 字段：

字段类型描述
apolloEntityId字符串实体的唯一标识符。
apolloEntityType字符串实体类型，例如 "service"、"daemon" 或 "network-security"。
releaseChannel字符串更新后分配给此实体的发布通道。如果实体继承了环境默认值，则不存在。
configOverrides映射此实体的产品特定配置。键为版本字符串；值为该版本的配置映射。
entity映射序列化的实体设置。对于部署配置中声明的实体存在；对于跟踪资产则为空。参见 entity 子字段。
asset映射序列化的资产设置。对于跟踪资产存在；对于在部署配置中声明的实体则为空。参见 asset 子字段。

DELETE_ENTITY

当从部署配置中移除一个实体时触发。

可用的 referencedResources 键： environment、installation、product（如果实体有分配产品）

可用的 request 字段：

字段类型描述
apolloEntityId字符串被删除的实体的唯一标识符。
apolloEntityType字符串实体类型，例如 "service" 或 "daemon"。

entity 子字段

entity 字段是一个包含序列化实体设置的映射。它存在于部署配置中声明的实体（服务、守护进程、Helm 图表等）中，而对跟踪的资产则为空。

字段类型描述
entityLocator对象用于在环境中标识实体类型的定位器。
releaseChannel字符串实体的发布通道。如果实体继承了环境默认值，则不存在。
maintenanceWindows对象在此实体上配置的维护窗口计划。如果没有设置，则不存在。
dependencyStackOverrides映射按产品 ID 键入的每个依赖项的配置覆盖。
ignoredDependencies集合Apollo 应忽略的此实体的依赖项的产品 ID。
rolloutStrategy对象部署策略覆盖。如果使用默认值，则不存在。
tracks集合资产轨道。仅适用于跟踪的资产。
product字符串显式设置在此实体上的产品 ID。如果没有覆盖，则不存在。
targetInfrastructureType字符串基础设施类型覆盖。如果没有设置，则不存在。
markedForDeletion布尔值此实体是否被标记为删除。如果没有标记，则不存在。
markedForUninstallation对象卸载此实体的配置。如果没有设置，则不存在。
helmReleaseName字符串Helm 发布名称。仅适用于 Helm 图表实体。
k8sNamespace字符串Kubernetes 命名空间。仅适用于 Helm 图表实体。
accountId字符串AWS 账户 ID。仅适用于 Terraform 模块实体。
foundrySpace字符串Foundry 空间。仅适用于 Foundry 产品实体。
dependencyGroups集合依赖组。适用于 Helm 图表、Terraform 模块和 Foundry 产品。
dependencyGroupOverrides映射按产品 ID 键入的依赖组覆盖。

asset 子字段

asset 字段是一个包含序列化资产设置的映射。它存在于跟踪的资产中，而在部署配置中声明的实体中为空。

字段类型描述
locator字符串跟踪资产的产品 ID。
dependencyStackOverrides映射按产品 ID 为每个依赖项配置的覆盖项。
ignoredDependencies集合Apollo 应忽略的此资产的依赖项的产品 ID。
markedForDeletion布尔值此资产是否被标记为删除。如果没有被标记，则不存在。
markedForUninstallation对象卸载此资产的配置。如果没有设置，则不存在。

network-security configOverrides

对于 network-security 实体，configOverrides 不是版本键控的映射。它是网络安全规则的序列化声明状态，结构上是一个具有 type 鉴别器字段的联合体。当前唯一的变体是 "v1"。

字段类型描述
type字符串规则变体。目前始终为 "v1"。
v1对象规则内容。请参阅下面的子字段。

v1 子字段：

字段类型描述
description字符串规则的人类可读描述。
ruleDetails对象规则类型和配置。有一个 type 字段标识变体。通过 v1.ruleDetails.<type> 访问变体，例如 v1.ruleDetails.egressRule。
quarantined布尔值如果为 true，则该规则被视为撤回。如果没有设置，则不存在。

ruleDetails 变体：

变体描述
egressRule出站流量规则。包含 dest（一种联合类型，type 为 "cidrs" 或 "domain"；通过 dest.cidrs（CIDR 字符串集合）或 dest.domain（字符串）访问），ports（端口条目集合，每个条目有 type 为 "port" 或 "portRange"；通过 port（整数）或 portRange.start/portRange.end 访问），以及 namespaces（字符串集合）。
httpsIngressRule按域名的 HTTPS 入站规则。包含 domain（字符串）、allowedCidrs（具有 justification（字符串）和 cidrs（字符串集合）的对象集合），以及 allowedCountryCodes（具有 justification（字符串）和 countryCodes（字符串集合）的对象集合）。
httpsPortIngressRule按端口的 HTTPS 入站规则。包含 port（整数）、allowedCidrs 和 allowedCountryCodes。结构与 httpsIngressRule 相同。
tcpIngressRuleTCP 入站规则。包含 ports（整数集合）和 allowedCidrs。
udpIngressRuleUDP 入站规则。包含 ports（整数集合）和 allowedCidrs。

CREATE_EGRESS_POLICIES

当创建出站策略时触发。

可用的 referencedResources 键： egressPolicyRoot

可用的 request 字段：

字段类型描述
data列表正在创建的出站策略。每个条目包含下面表格中描述的字段。

data 中的每个条目都有以下字段：

字段类型描述
address对象目标网络地址。有一个 type 字段（"dns" 或 "cidr"）。对于 DNS 条目，通过 address.dns 访问主机名（字符串）。对于 CIDR 条目，通过 address.cidr 访问块（字符串）。

network对象网络路由类型。有一个 type 字段，如 "direct"、"privateLink"、"secureTunnels"、"agent"、"bucketEndpoint"、"catalogBucket" 或 "bcap"。
description字符串策略的可选描述。如果没有提供则不存在。

MODIFY_VULNERABILITY_SUPPRESSIONS

当添加或修改漏洞抑制时触发。

可用 referencedResources 键： vulnerabilitySuppression

可用 request 字段：

字段类型描述
spaceRid字符串空间的RID。
vulnerabilitySuppressions列表基于CVE的抑制项，正在被添加或修改。每个条目包含以下描述的字段。
scannerSuppressions映射以扫描器名称为键的抑制项。值是抑制条目列表（与vulnerabilitySuppressions条目结构相同）。
imageNameVulnerabilitySuppressions映射以精确镜像名称为键的抑制项。值是抑制条目列表。
imagePrefixVulnerabilitySuppressions映射以镜像名称前缀为键的抑制项。值是抑制条目列表。
productIdVulnerabilitySuppressions映射以产品ID为键的抑制项。值是抑制条目列表。
imageEndOfSupportLifeSuppressions映射以镜像名称为键的生命周期结束抑制项。值是抑制对象（结构与下面的suppression相同，但不包括vulnerabilitiesToSuppress）。

vulnerabilitySuppressions（以及其他抑制列表）中的每个条目都有以下字段：

字段类型描述
suppression对象抑制元数据。包含rationale（字符串）、category（字符串）和validUntil（时间戳）。
vulnerabilitiesToSuppress集合此抑制条目覆盖的CVE ID。每个条目是一个如"CVE-2021-44228"的字符串。

REMOVE_VULNERABILITY_SUPPRESSIONS

当漏洞抑制项被移除时触发。

可用的referencedResources键： vulnerabilitySuppression

可用的request字段：

字段类型描述
spaceRid字符串空间的RID。
vulnerabilitySuppressions集合被移除的CVE ID。每个条目是一个字符串，例如"CVE-2021-44228"。
scannerSuppressions映射被移除的扫描器抑制。键是扫描器名称；值是CVE ID字符串的集合。
imageNameVulnerabilitySuppressions映射被移除的镜像名称抑制。键是镜像名称；值是CVE ID字符串的集合。
imagePrefixVulnerabilitySuppressions映射被移除的镜像前缀抑制。键是镜像名称前缀；值是CVE ID字符串的集合。
productIdVulnerabilitySuppressions映射被移除的产品抑制。键是产品ID；值是CVE ID字符串的集合。
imageEndOfSupportLifeSuppressions集合其生命周期结束抑制被移除的镜像名称。

管理覆盖

CREATE_MANAGEMENT_OVERRIDE

当创建管理覆盖时触发。

可用的referencedResources键：无

可用的request字段：

字段类型描述
name字符串唯一的管理覆盖标识符。
owner字符串将拥有此覆盖的团队的RID。
products集合此覆盖所涵盖的产品ID。
services集合此覆盖所涵盖的服务定位器。每个条目包含stack（字符串）和serviceName（字符串）。

services集合此覆盖所涵盖的已更新服务定位器。每个条目包含 stack（字符串）和 serviceName（字符串）。
