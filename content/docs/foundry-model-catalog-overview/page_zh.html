模型目录

您可以在控制面板中启用模型目录。

模型目录是 Foundry 中的一个 AIP 应用，旨在帮助用户发现并了解 Palantir 提供的所有模型。

模型目录可帮助构建者：

  • 查看 AIP 中可用的模型并发现新模型。
  • 为您的用例选择合适的模型。后续更新将提供更多工具和基准测试，以便进行比较和决策。
  • 通过 Marketplace，利用基础模板或完整的用例模板快速上手工作流。
  • 使用沙盒/演练场测试不同模型。

模型目录目前仅包含 LLM，不包含自定义 ML/AI 模型。您可以在《建模目标》中找到 ML/AI 模型。

模型目录包含两个主要视图：

  • 模型目录主页
  • 模型实体页面

模型目录主页

模型目录主页是一个发现和导航界面，展示了用户在其 Foundry 租户中可用的所有大语言模型。

查看 AIP 中可用模型的完整列表。

在主页上，可以通过以下几种方式对模型进行筛选：

  • 生命周期状态
    • 实验性：如果模型提供商或 Palantir 将某个模型视为实验性模型，则该模型即为实验性模型。这意味着其 API 可能会发生变化，Token 容量可能受限，在 AIP 应用中的行为可能无法获得全面支持，或者可能出现其他不稳定行为。所有模型在首次于 AIP 中可用时，都将处于实验性阶段。实验性模型通常用于探索和测试，不强调长期稳定性和支持。当模型达到 Palantir 的性能和可用性标准后，便会从实验性升级为稳定状态。这些标准包括但不限于：具备足够的吞吐量以在保持可用性的同时处理生产工作负载请求、在所有 AIP 应用中均可用，以及具备广泛的区域可用性。

租户管理员可以在控制面板的 AIP 设置 部分为其租户启用或禁用实验性模型，从而使模型目录仅显示处于稳定生命周期阶段的模型。

  • 稳定：稳定或正式发布（GA）的模型是模型提供商和 Palantir 共同认可的可靠模型。这些模型提供稳健的功能、可靠的支持，并专为长期生产使用而设计。模型的区域可用性因模型提供商提供的产品而异。您可以在《支持的 LLM 文档》中查看各区域模型可用性的完整列表，并在模型选择器中查看每个模型的状态信息。在收到模型提供商规定的停用日期后，Palantir 会将模型从稳定状态转为停用状态，并通过《升级助手》通知用户这一变更。

  • 停用：停用模型将在未来几个月内被废弃，具体由模型提供商决定并发布。虽然停用模型在规定的废弃日期之后无法再支持新的工作流，但仍可支持现有工作流。与稳定模型相比，停用模型将不再获得同等水平的技术支持。在收到模型提供商规定的废弃日期后，Palantir 会将模型从停用状态转为废弃状态，并通过《升级助手》通知用户这一变更。有关停用模型的更多信息，请查阅《模型废弃文档》。

  • 废弃：在停用期结束后，Palantir 会与模型提供商协调，从 Foundry 中移除（即废弃）相关模型。废弃模型无法支持现有工作流，包括新的 API 调用，因此项目必须在模型被废弃之前迁移至稳定模型，以维持功能正常运转。Foundry 会保留废弃模型的历史数据和日志，并确保其可被访问。有关废弃模型的更多信息，请查阅《模型废弃文档》。

  • 类型

    • 补全模型：补全模型旨在通过预测并补全输入文本，生成符合语境的相关文本。这使其非常适合内容生成、自动补全、翻译和问答等任务。例如，GPT-4 Turbo、Mixtral 8x7B 和 Llama2 70B Chat。
    • 嵌入模型：嵌入模型将离散数据（如单词和句子）转换为连续的向量表示。它最常用于语义搜索和其他信息检索用例。例如，text-embedding-ada-002 和 Instructor Large。
    • 视觉模型：视觉模型经过训练，能够分析和理解视觉输入，从而识别对象、对图像进行分类，并支持图像和视频数据的各类计算机视觉任务。例如，GPT-4 Vision 和 Gemini Pro Vision。

  • 模型创建者

    • 模型创建者是负责创建、开发和维护特定 LLM 的组织。例如 OpenAI、Anthropic、Google Gemini 和 Mixtral AI。模型创建者可以直接提供其 LLM，也可以通过与其他组织合作来提供，例如 OpenAI 通过 Azure 提供，Anthropic 通过 AWS 提供。部分模型也可由 Palantir 提供并托管，例如 Llama 和 Mixtral。

如果某个模型不可用或显示为灰色，这意味着您的当前环境尚未启用该模型。如需启用模型，请联系您的平台管理员或 Palantir 代表。

了解有关模型启用的更多信息。

模型实体页面

每个模型都有一个实体页面，包含三个主要部分：

  • 演练场：供构建者试用不同模型的界面。
  • 使用方法：通过创建资源快速入门，该资源已预填开始构建工作流所需的内容。数据目录目前支持函数和转换。
  • 模型描述：基本描述、法律免责声明、模型的上下文窗口（如 Token 限制）、训练数据截止日期等。

模型对比页面

数据目录对比页面允许构建者高效地比较和评估各种 LLM 的性能。该界面支持用户选择两个 LLM，并在相同的补全或视觉任务上进行测试。这有助于做出明智的决策，让构建者能够快速选出最适合其工作流的模型。
