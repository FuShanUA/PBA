发布与版本控制

模块的版本控制让构建者能够安全、迭代地构建 Workshop 应用程序；通过发布稳定的模块版本，可以控制终端用户可访问的版本。这样一来，即便模块在保存时发生了变更，也不会直接暴露给终端用户，从而允许在已上线的 Workshop 应用程序上持续进行开发。

版本历史与设置

在“版本”对话框中，构建者可以查看模块已保存版本的历史记录。每个已保存的版本都会显示时间戳、编辑者，以及描述信息（如果有的话）。针对每个已保存的版本，构建者可以执行以下操作：

  • 发布此版本：发布模块的已保存版本，供查看者访问。
  • 查看此版本：查看该特定版本下的模块状态。查看未发布的版本时，模块顶部会出现一条警告横幅。

  • 回退至此版本：将历史版本保存为模块的最新版本。系统会自动生成一条描述，详细说明此次回退操作。
编辑模块的历史版本

要编辑模块的历史版本，请按照以下步骤操作： * 确保在“版本”对话框中关闭了保存时自动发布开关。 * 回退至您想要编辑的版本。 * 通过文件 > 复制文件，为所选模块版本创建一个副本。 * 将原始模块回退至最新版本，然后对您创建的副本进行所需的任何编辑。

构建者还可以在“版本”对话框中编辑与保存和版本控制操作相关的设置：

  • 保存时自动发布：此开关用于控制保存时是否自动发布新版本。
  • 保存时始终提示添加版本描述：此开关用于控制保存新版本时，是否提示构建者添加可选的描述信息。添加描述有助于记录对模块所做的变更，并积累更详尽的模块历史记录。可以在模块的“版本”对话框中查看、添加和编辑这些描述。

查看版本间的变更

使用《变更日志面板》来直观对比不同模块版本之间的差异。您可以选择：

  • 范围选择：选择起始版本和结束版本，查看这两个版本之间的所有变更。
  • 单选：选择单个版本，将其与前一个版本进行对比。

《变更日志面板》会高亮显示新增、删除、修改、移动的内容以及新近废弃的元素。您可以检查 JSON 差异以查看具体的修改细节，并审查可视化层级结构，以了解这些变更与嵌套组件之间的关系。

分支与变基

在分支上进行开发时，如果自您上次保存以来 main 分支发生了变更，您可能需要先进行变基操作，然后再将 Workshop 的变更合并到 main 分支。变基操作会将您所在分支的变更应用到模块最新的 main 版本上，并提示必须解决的任何合并冲突。

变基用户界面使用《变更日志面板》来可视化变更并突出显示冲突，例如同一个组件在 main 分支和您的分支上都被修改的情况。解决冲突后，保存即可完成变基操作，随后便可以将您的分支合并到 main 中。

如需了解完整流程，请参阅《变基与冲突解决》。
