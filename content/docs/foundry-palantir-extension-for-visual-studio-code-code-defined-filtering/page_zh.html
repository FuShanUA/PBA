代码定义的输入过滤

除了在 预览转换 文档中讨论的 Sampled 和 Full dataset 输入策略配置，VS Code 预览还支持一个 Code-defined filters 选项。这允许您编写自定义的 Python 函数来控制在转换预览期间加载的确切数据。而不是依赖于随机采样或完整数据集加载，您可以定义精确的逻辑，以确定应使用哪些数据子集。

当适用时，这些过滤器利用 下推谓词 ↗ 来确保仅加载最相关数据样本。Visual Studio Code 的 Palantir 扩展会自动发现所有可作为过滤器在您的仓库中使用的函数，并在配置预览时在下拉菜单中显示它们。

为什么使用代码定义的过滤器？

测试特定场景： 专注于对您的开发重要的边缘情况或特定数据条件，而无需浏览无关的数据。

创建可重用逻辑：编写一次过滤器，并在多个预览会话中使用，使您的开发流程更加一致。

优化性能：对于即使完整数据集加载都很慢的大数据集，您可以精确优化被筛选的数据。

组合多个过滤器：以任何顺序选择并链接多个过滤器，以创建复杂的采样策略。

如何使用代码定义的过滤器

为预览选择过滤器

  1. 在预览面板中选择配置输入策略。
  2. 选择Code-defined filters选项。
  3. 从内置过滤器列表中选择，或者选择由您编写的过滤器。
  4. 按照您偏好的顺序排列筛选器；它们将按顺序应用。

该扩展程序会自动发现项目代码库中所有符合条件的筛选器，并在选择下拉菜单中显示它们。

创建筛选函数

要从Python函数创建一个有效的预览筛选器，必须遵循以下规则。

函数要求

函数必须是

  • 在其模块的全局作用域中直接定义的
  • 使用以下签名之一完全类型注解：
    • 对于Spark转换：(pyspark.sql.DataFrame) -> pyspark.sql.DataFrame
    • 对于轻量级转换：(polars.LazyFrame) -> polars.LazyFrame
    • 对于原始文件：(collections.abc.Iterator[transforms.api.FileStatus]) -> collections.abc.Iterator[transforms.api.FileStatus]

函数不应该

  • 嵌套
  • 由 if、with、for 或其他语句保护
  • 是类的一部分
  • 从其他地方导入
  • 是一个赋值给函数的变量
  • 是一个 async 或私有函数（函数名不能以 _ 开头）
  • 应用了任何装饰器

示例过滤函数

以下示例列出了一些有效的预览过滤函数：

Copied!
1from pyspark.sql import DataFrame
2from pyspark.sql import functions as F
3from collections.abc import Iterator
4from transforms.api import FileStatus
5import itertools as it
6import polars as pl
7
8
9def limit_files(files: Iterator[FileStatus]) -> Iterator[FileStatus]:
10    """ Limit the number of files in a file system listing."""
11    return it.islice(files, 10)
12
13def ocean_animals_only(df: DataFrame) -> DataFrame:
14    """ Get only animals living in the Ocean """
15    return df.filter(F.col("Habitat") == "Ocean")
16
17def grassland_animals_only_lightweight(df: pl.LazyFrame) -> pl.LazyFrame:
18    """ Get only animals living on Grassland  """
19    return df.filter(pl.col("Habitat") == "Grassland")

向过滤器添加参数

您可以添加参数，使过滤器更加灵活。这允许您创建通用且可重用的过滤器函数，以适应不同的场景。

如何添加参数

  1. 在函数签名中使用类型注释定义参数。
  2. 通过在参数前添加 * 符号，将其标记为仅关键字参数。
  3. 当您选择过滤器时，扩展将自动检测这些参数并提示输入值。

以下是一个带有参数的示例：

Copied!
1from pyspark.sql import DataFrame
2from pyspark.sql import functions as F
3
4def filter_animals_by_habitat(df: DataFrame, *, habitat: str) -> DataFrame:
5    """ Get only animals living in the Ocean """
6    return df.filter(F.col("Habitat") == habitat)

注意 * 符号在 habitat 参数之前；这将它分类为仅关键字参数。

支持的参数类型

目前支持的参数类型如下：

  • str
  • int
  • float
  • bool

提供默认值

您可以在函数签名中为参数提供默认值。如果提供了默认值，该参数将变为可选参数，并且在配置时如果没有输入值，将使用默认值：

Copied!
1from pyspark.sql import DataFrame
2from pyspark.sql import functions as F
3
4def example_filter_with_params(
5    df: DataFrame, *, int_param: int, float_param: float = 3.3, str_param: str = "Hello world", bool_param: bool = True
6) -> DataFrame:
7    """This is an example function with many parameters of different types"""
8    print(f"int_param: {int_param}, float_param: {float_param}, str_param: {str_param}, bool_param: {bool_param}")
9    return df.limit(int_param)

当您选择一个带有参数的过滤器进行预览时，每个参数都会出现提示：

支持的输入类型

过滤器选择下拉菜单将显示可用的过滤器；然而，根据您的转换上下文，选择特定类型的过滤器可能更合适。

  • 在使用Spark转换处理结构化输入时： 使用 DataFrame 过滤器。
  • 在使用轻量级转换处理结构化输入时： 使用 LazyFrame 过滤器。
  • 当处理 Spark 或轻量级中的非结构化输入（原始文件）时： 使用 Iterator[FileStatus] 过滤器。
