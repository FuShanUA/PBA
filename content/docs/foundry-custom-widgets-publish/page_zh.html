发布一个小部件集

可以通过以下方法之一发布一个小部件集：

  • 通过用户界面手动上传一个 .zip 压缩文件
  • 从 @osdk/cli 执行 CLI 命令
  • 使用 Foundry 代码仓库进行小部件集的 Foundry CI/CD

发布一个小部件集的新版本不会影响小部件集中小部件的现有使用。例如 Workshop 等宿主应用程序不会自动显示新更改，直到应用程序被配置为使用新的已发布小部件集版本。

一个特殊的 .palantir/widgets.config.json 清单文件描述了构建中包含的 widgets，该构建通常从项目代码自动产生，使用 @osdk/widget.vite-plugin ↗ 当从项目模板开始时。

要发布的 widget 集版本包含在此清单文件中。当使用 @osdk/widget.vite-plugin 时，项目中的 foundry.config.json 文件描述了 widgetSet.autoVersion 字段中版本的计算方式。这可能是 package-json，它从 package.json 中读取 version 字段，或者 git-describe，它根据 git 仓库的标签和提交来计算版本。

通过用户界面手动上传 .zip 压缩文件进行发布

在您的本地计算机上，按照 构建项目 的说明进行操作。创建一个包含您的 widget 集生产构建的 .zip 压缩文件，确保没有其他文件夹（例如输出 dist/ 或等效文件夹）。

以下代码片段在项目根目录中使用 bash 终端创建一个 asset.zip 压缩文件，并将生产构建放在 dist/ 文件夹中。您可以使用适用于您操作系统的任何合适的 zip 工具。

已复制！
1
2
cd dist/
zip -r ../asset.zip .

在自定义小部件页面上使用上传发布选项来上传一个小部件集。将 .zip 压缩文件拖放到对话框中，然后选择上传。

通过 @osdk/cli 命令行命令进行发布

NPM 包 @osdk/cli ↗ 可通过一个命令自动完成 .zip 压缩文件的创建和发布步骤，并且可以配置在外部的 CI/CD 流水线中，例如 GitHub Actions 或 CircleCI。

要在本地终端上进行发布，请按照 构建项目 的说明操作。

从终端运行以下 CLI 命令：

npx @osdk/cli@latest widgetset deploy

Publish with Foundry CI/CD

For widget sets using Foundry Code Repositories, Foundry CI/CD is automatically set up to build and publish the widget set on tag builds.

To create a new tag, you can use the platform user interface, or use standard git commands such as the following example:

Copied!
1
2
git tag <x.y.z>
git push origin tag

View the status of the tag build through the Tags tab in VS Code Workspaces or Code Repositories.
