加载有序结果

通过为特定属性指定排序方向，加载一个有序列表Restaurants。通过API调用时，排序条件通过fields数组指定。通过SDK调用时，可以将多个orderBy调用串联在一起。字符串的排序顺序是区分大小写的，这意味着数字会排在大写字母之前，而大写字母又会排在小写字母之前。例如，Cat会排在bat之前。

参数：

  • ordering {ObjectType}Ordering：您要按其排序的属性。使用Java SDK时，这通过{ObjectType}Ordering接口为您提供。
  • 属性名称和方向 ASC| DESC ：您希望排序的方向，可以是升序或降序。使用Java SDK时，这是通过{PROPERTY_NAME}_ASC和{PROPERTY_NAME}_DESC限定符在{ObjectType}Ordering接口上提供的。

示例查询：

Copied!
1Stream<Restaurant> result = client.ontology()
2    .objects()
3    .Restaurant()
4    .orderBy(RestaurantOrdering.RESTAURANT_NAME_ASC)
5    .fetchStream();

过滤

过滤的类型取决于给定对象类型的属性类型。这些过滤器也可以通过布尔表达式组合在一起，以构建更复杂的过滤器。

请注意，此端点利用了用于对象类型的底层对象同步技术。如果Restaurant由对象存储V2支持，则没有请求限制。如果Restaurant由对象存储V1（Phonograph）支持，则结果限制为10,000；如果请求超过10,000个Restaurants，将抛出ObjectsExceededLimit错误。

参数：

  • where Filter（可选）：对特定属性进行过滤。可能的操作取决于属性的类型。这是通过{ObjectType}Filter表示的，每个属性都有调用的方法。
  • orderBy OrderBy（可选）：根据特定属性对结果进行排序。如果使用SDK，可以将.where调用与orderBy调用链接起来以达到相同的效果。这是通过{ObjectType}OrderBy表示的，每个属性都有调用的方法。

示例查询：

Copied!
1List<Restaurant> result = client.ontology()
2    .objects()
3    .Restaurant()
4    .where(RestaurantFilter.restaurantName().isNull(false))
5    .fetchStream()
6    .toList();

聚合

聚合允许您对一组数据计算汇总统计信息。它们对于理解大型数据集中的模式和洞察力非常有用，而无需手动分析每个单独的数据点。您可以将多个聚合操作结合起来，创建更复杂的查询，以深入挖掘数据的洞察力。

请注意，此端点利用了用于对象类型的底层对象同步技术。如果 Restaurant 由对象存储V2支持，则没有请求限制。如果 Restaurant 由对象存储V1（Phonograph）支持，则有10,000个结果的限制；如果请求超过10,000个 Restaurants，将抛出 ObjectsExceededLimit 错误。

在 Restaurants 上执行聚合。

参数：

  • aggregation Aggregation[] (可选)：要执行的聚合函数集合。使用SDK时，可以使用 .where 将聚合计算与其他搜索连接起来。

  • 示例API响应：

    data: [{
        group: {
            "restaurantName": "Restaurant Name"
        },
        metrics: [
            {
                name: "count",
                value: 100
            }
        ]
    }]

    函数

    函数（有时称为“对象上的函数”或FoO）是Palantir平台的一项强大功能，旨在增强数据建模和操作。函数提供了一种在本体论中存储的数据上定义和执行自定义逻辑的方法，使用户能够创建更复杂的数据转换、验证和分析。

    在Java SDK中，用户可以通过生成的查询来执行Foundry函数。

    通过将您的函数添加到应用程序中，您可以生成调用FoO以执行逻辑并获取结果的查询。

    在这个例子中，我们有一个函数 findSimilarRestaurants，它接收一个ID并返回包含所有类似的 Restaurants 的ObjectSet。

    执行函数以查找相似的 Restaurants(findSimilarRestaurants) 的参数

    属性（Property）API 名称（API Name）类型（Type）
    Restaurant IdrestaurantId字符串（String）

    返回值：RestaurantObjectSet

    应用函数

    要应用一个函数，您必须访问并执行一个查询。这是通过访问 .queries() 并通过 .execute(...).getReturnValue() 执行逻辑函数来完成的。

    示例查询：

    Copied!
    1RestaurantObjectSet response = client
2    .ontology()
3    .queries()
4    .findSimilarRestaurants()
5    .execute("restaurantId")
6    .getReturnValue();

    媒体引用属性

    在 Palantir 平台中，媒体引用（media reference） 是对存在于 MediaSet 中的一段媒体的引用。

    在 Java SDK 中，用户可以添加一个媒体引用属性，允许他们 upload 并 download 媒体到支撑媒体集。

    在下面的示例中，我们有一个 restaurantMenu 属性，它表示与特定 Restaurant 对象相关的菜单。

    上传媒体

    要上传媒体文件，首先上传到支撑媒体集本身。然后将已上传的 MediaItem 关联到特定对象。

    每个 ObjectSet 都将有一个方法，用于将媒体上传到特定属性的支撑 MediaSet。这会返回一个表示上传媒体的媒体引用的 Media 实例。在这个例子中，对象有一个生成的 uploadRestaurantMenu() 方法，允许您将媒体上传到支撑 MediaSet：

    Copied!
    1Media media =
2        client.ontology()
3                .objects()
4                .Restaurant()
5                .uploadRestaurantMenu(UploadMediaRequest.builder()
6                        .mediaItemPath(...)
7                        .mediaType(...)
8                        .inputStream(...)
9                        .build());

    成功上传后，您可以使用返回的 Media 对象通过执行适当的操作，将上传的媒体项关联到对应的对象。

    Copied!
    1client.ontology()
2        .actions()
3        .changeMenuAction()
4        .apply(ChangeMenuActionRequest.builder()
5                .restaurant_menu(media)
6                .build());

5                .restaurant_menu(restaurant.restaurantMenu().get())
6                .build());
