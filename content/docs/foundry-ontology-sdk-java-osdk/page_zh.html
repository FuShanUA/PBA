加载 Restaurants 流


加载可收集的对象流。系统会根据用户流式传输的对象数量自动加载多个分页对象。


请注意，该端点利用了对象类型所依赖的底层对象同步技术。若 Restaurant 由对象存储 V2 (Object Storage V2) 提供支持，则没有请求限制。若 Restaurant 由对象存储 V1 (Phonograph) 提供支持，则结果上限为 10,000 条；如果请求的 Restaurants 数量超过 10,000，系统将抛出 ObjectsExceededLimit 错误。

加载有序结果


通过为特定属性指定排序方向，加载有序的 Restaurants 列表。通过 API 调用时，排序条件通过 fields 数组指定。通过 SDK 调用时，可将多个 orderBy 调用串联在一起。字符串的排序规则区分大小写，即数字排在最前，其次是大写字母，最后是小写字母。例如，Cat 会排在 bat 前面。


参数：


    

  • ordering {ObjectType}Ordering：要作为排序依据的属性。在 Java SDK 中，该属性通过 {ObjectType}Ordering 接口提供。
    • 

  • property name and direction ASC| DESC ：排序方向，升序或降序。在 Java SDK 中，该方向通过 {ObjectType}Ordering 接口上的 {PROPERTY_NAME}_ASC 和 {PROPERTY_NAME}_DESC 限定符提供。
    •

已复制！
1Stream<Restaurant> result = client.ontology()

过滤


你可以执行的过滤操作类型取决于给定对象类型上的属性类型。这些过滤器还可以通过布尔表达式组合使用，以构建更复杂的过滤条件。


请注意，该接口利用了对象类型所使用的底层对象同步技术。如果 Restaurant 由对象存储 V2 (Object Storage V2) 提供支持，则没有请求限制。如果 Restaurant 由对象存储 V1 (Object Storage V1, Phonograph) 提供支持，则结果数量限制为 10,000 条；如果请求的 Restaurants 数量超过 10,000，将抛出 ObjectsExceededLimit 错误。


参数：


    

  • where Filter（可选）：对特定属性进行过滤。可用的操作取决于属性的类型。这通过 {ObjectType}Filter 来表示，其中包含针对每个属性调用的方法。
    • 

  • orderBy OrderBy（可选）：根据特定属性对结果进行排序。如果使用 SDK，可以将 .where 调用与 orderBy 调用链式组合以实现相同的结果。这通过 {ObjectType}OrderBy 来表示，其中包含针对每个属性调用的方法。
    •

已复制！

2    .objects()
3    .Restaurant()
4    .where(RestaurantFilter.restaurantName().isNull(false))
5    .fetchStream()
6    .toList();

聚合


聚合允许您对一组数据计算汇总统计信息。它们有助于从大型数据集中理解模式和获取洞察，而无需手动分析每个单独的数据点。您可以组合多个聚合操作来创建更复杂的查询，从而对数据进行更深入的分析。


请注意，此接口利用了对象类型所使用的底层对象同步技术。如果 Restaurant 由 Object Storage V2 提供支持，则没有请求限制。如果 Restaurant 由 Object Storage V1 (Phonograph) 提供支持，则结果数限制为 10,000 条；如果请求的 Restaurants 超过 10,000 条，系统将抛出 ObjectsExceededLimit 错误。


对 Restaurants 执行聚合。


参数：


    

  • aggregation Aggregation[]（可选）：要执行的一组聚合函数。使用 SDK 时，聚合计算可以通过 .where 与进一步的搜索串联在一起。
    • 

  • groupBy GroupBy[]（可选）：为聚合结果创建的一组分组。
    • 

  • where Filter（可选）：对特定属性进行过滤。可用的操作取决于属性的类型。
    • 



查询示例：




已复制！
1AggregationResponse numRestaurants = client.ontology()
2        .objects()
3        .Restaurants()
4        .where(RestaurantFilter.restaurantName().isNull(false))
5        .groupBy(GroupBy.exact(Restaurant.Property.restaurantName()))
6        .count()
7        .compute();



API 响应示例：


{
    excludedItems: 0,
    data: [{
        group: {
            "restaurantName": "Restaurant Name"
