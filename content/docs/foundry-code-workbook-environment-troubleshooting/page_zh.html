我的自定义环境初始化缓慢

初始化缓慢通常表明环境定义过大或过于复杂。初始化时间往往随环境中包的数量呈超线性增长。此外，Foundry 通常会预初始化常用环境，因此，如果您基于默认配置文件创建了自定义环境，变慢的原因很可能是您不再使用预初始化的环境，而必须等待新环境创建完成。有关环境创建过程的更多详细信息，请参阅《环境创建简介》。

请尝试以下操作：

  1. 撤销 - 如果在更改配置后环境开始出现初始化缓慢的情况，请尝试还原更改。在某些情况下，仅添加一个包就会显著增加初始化时间。
  2. 简化 - 从环境定义中移除不需要的包。小型、专用的环境比大型、通用的环境性能好得多。
  3. 固定版本 - 如《选择环境》所述，在“环境配置”面板中为某些包添加版本约束。对于像 python 这样具有许多可能版本的包，以及像 scipy 这样具有复杂依赖关系图的包，这样做最为有效。如果添加特定包导致环境变慢，请尝试为该包固定版本。
    • 注意事项：添加版本固定可能导致环境无法满足依赖。如果环境初始化失败，请尝试放宽版本限制。例如，python=3.6.10 将仅匹配该版本，但 python=3.6 将匹配任何 python 3.6.x 版本，因此更有可能满足依赖要求。
    • 一种方法是先等待环境（未固定版本）解析完成，然后检查选择了哪些版本（参见《查看已解析环境》）。随后您可以为这些特定版本添加固定，直到初始化性能足够快。通常的最佳做法是固定具有许多依赖项的包的版本，而保留所有其他包不固定。

环境初始化失败

如果环境初始化失败，您可以查看环境启动错误日志，以获取有关问题原因的更多信息。

日志的第一行将显示 Execution failed with non-zero exit code:，后跟一个整数错误代码。此错误代码指示特定的失败模式。

代码 1 - 环境解析错误

此错误表明指定的部分包（及其依赖项）不兼容或无法安装。如果“环境配置”面板中最近的更改导致初始化开始失败，请尝试还原更改。以下是常见环境解析错误的列表。

新版 Mamba 错误消息

Palantir 通过提供格式更优的环境初始化错误消息，为开源 Mamba 社区做出了贡献 ↗。自 2023 年 2 月起，Foundry 服务受益于这些新错误消息，它们能更准确地呈现环境失败所破坏的依赖关系树。

有关理解这些新消息、如何解读它们以及如何修复它们的指导，请参阅《新错误消息》。

1Problem: nothing provides B-b.b.b needed by A-a.a.a:

修复 Code 1 错误

要修复环境失败问题，请考虑以下步骤：

  1. 检查环境启动日志中的 STDOUT。
  2. 创建一个失败解析的最小示例。移除包直到环境成功初始化，然后再逐一添加包，直到确定哪些包是阻碍因素。
  3. 尝试通过移除包版本固定来放宽限制条件。
    • 您可以在不同窗口中打开多个工作簿或同一工作簿的多个分支，以加快此过程。
    • 我们建议使用二分法来排查问题。尝试在一个工作簿中移除所有约束条件，在另一个工作簿中移除一半的约束条件，再在第三个工作簿中移除另一半的约束条件，依此类推。

Code 137 - 模块内存不足

此错误通常表明环境初始化所需的内存超过了 Spark 模块上的可用内存。如果是最近在“环境配置”面板中所做的更改导致初始化开始失败，请尝试撤销该更改。否则，请尝试以下一项或多项建议：

  • 从环境中移除不需要的包。内存使用量与环境中的包数量呈超线性增长关系，因此即使移除少量包也可能足以解决问题。
  • 如《环境初始化缓慢》部分第 3 步所述，为部分包添加版本约束。您固定的每个包版本都会降低环境解析的复杂度，进而减少初始化环境所需的内存量。

如果在移除所有不需要的包并将剩余部分包固定到特定版本后，您的环境仍然无法初始化，请联系您的 Palantir 代表，并提供您尝试过的所有调试步骤的详细信息。

其他代码

如果您的环境初始化失败并返回上述未列出的错误代码，且重试后依然失败，请联系您的 Palantir 代表并附上错误日志的完整文本。

默认环境初始化速度不一致

默认配置文件的初始化有时快、有时慢，这属于正常现象。对于后者，您需要等待 Mamba 解析您的环境规格并将其安装到支持您工作簿的 Spark 模块上，这是默认行为。对于前者，要么是您正好获取到了一个预先初始化的模块，要么是您的环境规格与缓存中已解析的环境相匹配，可以直接安装。如果许多用户在同一时间段打开工作簿，您获取到预先初始化模块的概率就会降低，需要等待新模块启动。

同时需要 Conda 包和 JAR 的包

某些包需要同时安装 Conda 包和 JAR 才能使用。一个常见的例子是 graphframes 包。这类包需要特殊的设置流程，如果之前没有为您的实例配置过此包，您在构建期间可能会遇到以下错误：



已复制！
1o257.loadClass.: java.lang.ClassNotFoundException:<Class>

在控制面板的 Conda 环境配置选项卡的高级部分，可以将 JAR 文件添加到配置文件的配置中（参见《配置代码工作簿配置文件》）。如果您在配置此类包时需要协助，请联系您的 Palantir 代表。
