规模和属性限制

为了确保编辑的对象类型可以快速处理编辑并更新用户界面数据而不影响实时应用程序的性能，已经设定了若干限制。超出这些限制的操作将无法成功，并会向用户显示错误信息。

配置限制

Allow multiple values切换允许用户将值列表传递给一个参数。

限制最大值
原始列表参数中的元素数量10,000
对象引用列表参数中的元素数量1,000
在提交条件中使用列表参数时的元素数量1,000

编辑限制

限制最大值
单次操作提交中可以编辑的 对象类型（对象类型） 数量50
单次操作提交中可以编辑的 对象（对象） 数量10,000
单次操作提交中对 对象（对象） 的每次编辑32KB（OSv1），3MB（OSv2）

批量调用限制

一个操作最多可以在一个批次中被调用10,000次。当该操作是函数支撑的，并且该函数未配置为使用 批量执行 时，此限制将降低至20。

在批量操作调用中应用的编辑被视为一个组，当强制执行 编辑限制 时，无论批量中的哪个请求是编辑的原因。

根据调用应用程序的不同，可能会有其他限制。

支持的属性

目前，操作无法用于编辑主键 对象的。修改主键相当于删除一个对象然后添加一个新的对象；而不是通过操作来编辑主键，你可以直接使用 规则 创建或删除对象。

通知接收人

在使用 副作用通知 时，单个操作最多可以通知 500 个接收者。当通知内容是“从函数中渲染”时，此限制会减少到 50 个接收者。有关生成通知时需要考虑的限制的更多信息，请参阅关于 通知的最大接收者限制 的文档。
