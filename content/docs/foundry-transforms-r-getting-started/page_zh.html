入门

本页面解释了使用 Palantir R SDK 在 Foundry 中编写数据转换时可用的结构和配置选项。

仓库结构

每个 R 转换由一个 transforms spec 定义和配置，其中定义以 YAML 格式编写并保存在 .transforms 文件夹中。项目仓库的完整内容将在运行时可用，并可用于转换。

.
├── .transforms
|    └── happiness_ranking.yml
|
├── project
|    └── src
|        └── top_10.R
└── .Rprofile

对于从 Code Workspaces 发布的 R 转换，转换规范文件可以通过 UI 渲染和配置。

编写和配置一个简单的 R 转换

在下面的示例中，我们编写了一个简单的数据转换，它读取一个表格数据集，使用R包dplyr应用过滤器，并生成一个包含过滤结果的输出数据集。

Copied!
1library(foundry)
2library(dplyr)
3
4happiness_2019 <- datasets.read_table("happiness_2019")
5top_10 <- happiness_2019 %>% filter(Overall_rank <= 10)
6datasets.write_table(top_10, "top_10_happiest_countries")

此处是我们对应的转换规范文件，其中包含转换的定义：

将dataset.rid字段替换为输入和输出数据集的RID。请注意，输出数据集必须首先手动创建，因为我们目前不支持在检查时创建新的输出数据集。

您可以通过向inputs或outputs列表中添加新项来添加额外的输入和输出。

Copied!
1inputs:
2- alias: "happiness_2019"
3  properties:
4    type: "dataset"
5    dataset:
6      rid: "<input dataset rid here>"
7outputs:
8- alias: "top_10_happiest_countries"
9  properties:
10    type: "dataset"
11    dataset:
12      rid: "<output dataset rid here>"
13- alias: "second-output"
14  properties:
15    type: "dataset"
16    dataset:
17      rid: "<second output dataset rid here>"
18runtime:
19  type: "rscript"
20  rscript:
21    identifier: "R.4.1.x"
22    filePath: "project/src/top_10.R"

运行时配置：

  • identifier：标识转换的运行时；例如，R.4.1.x表示R的小版本发布4.1。
  • filePath：从项目根目录到将要执行的R脚本文件的相对文件路径。

写入非结构化数据

在本示例中，我们使用 ggplot2 生成带有线性回归线的图表，并将图表保存为 PNG 格式到输出数据集。

Copied!
1library(foundry)
2library(ggplot2)
3
4happiness_2019 <- datasets.read_table("happiness_2019")
5
6png("/tmp/model_plot.png")
7plot <- ggplot(happiness_2019, aes(GDP_per_capita, Overall_rank)) + geom_point() 
8plot + stat_smooth(method = "lm", formula = y ~ x, geom = "smooth")
9dev.off()
10
11datasets.upload_files("/tmp/model_plot.png", "happinessGDP_plot")
Copied!
1inputs:
2- alias: "happiness_2019"
3  properties:
4    type: "dataset"
5    dataset:
6      rid: "<input dataset rid here>"
7outputs:
8- alias: "happinessGDP_plot"
9  properties:
10    type: "dataset"
11    dataset:
12      rid: "<output dataset rid here>"
13runtime:
14  type: "rscript"
15  rscript:
16    identifier: "R.4.1.x"
17    filePath: "project/src/gdp_plot.R"

管理 R 包和环境

默认情况下，R 转换仓库配置了 制品仓库，这些仓库镜像 Posit™ 包管理器和 Bioconductor。我们建议用户使用 renv ↗ 来安装 R 包并为其转换管理 R 环境。

在 代码工作区 RStudio® 工作台 中，renv 默认已安装并可用。用户可以在 包 面板中搜索可用包，并生成相应的 R 代码来安装它们。

R 转换还支持从 renv.lock 恢复。如果在仓库中存在 renv.lock 文件，您可以在转换开始时或在 .Rprofile 中运行 renv.restore() 以恢复 R 环境。

以下示例展示了可以运行的 R 命令，以安装 R 转换所需的包，并生成一个 renv.lock 文件来保存项目库的状态。生成的锁定文件随后可以提交，并用于以后恢复项目。

Copied!
1renv::install("dplyr")
2renv::install("arrow")
3renv::snapshot()
