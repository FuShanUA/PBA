故障排查

本页面提供了在使用 Code Workspaces 时可能遇到的错误的故障排查提示。如果你遇到其他问题，或者通过本指南无法解决你的问题，请联系你的 Palantir 代表。

错误消息及建议的解决方法

错误：未授权 - 会话不再有效。可能已超时。请刷新页面。

如果遇到提示会话可能已超时的“未授权”错误，请按照以下步骤进行修复：

  1. 刷新页面。
  2. 确保你使用的是 Google Chrome 浏览器访问 Code Workspaces。不支持 Safari 浏览器。
  3. 如果之前在此 Code Workspace 中加载了带有安全标记的数据，而你现在失去了对这些标记数据的访问权限，你可能会收到此错误。如果是这种情况，请选择正在运行，然后选择重启工作区，以获取一个不包含数据的新会话。
  4. 如果你是首次使用 Code Workspaces，可能需要联系 IT 管理员，以确保 Code Workspaces 已被正确加入白名单。

错误：内容被阻止。联系站点所有者以解决此问题。

Foundry 实施了严格的内容安全策略，以确保在 Code Workspaces 中渲染的内容得到妥善管控。例如，Code Workspaces 不允许从 CDN 加载任意的 JavaScript，某些 Python 包（如 folium）就是这种情况。

如果因关键工作流收到此错误，请联系你的 Palantir 代表，讨论对该工作流提供支持的可能性。部分工作流已受支持，但可能需要你运行稍有不同的命令：

  • 《构建 Dash 应用》
  • 《构建 Streamlit 应用》
  • 《预览 TensorBoard™》

错误：包或命名空间加载失败

你可能会遇到如下所示的包或命名空间加载失败错误：

> library("hdf5r")
Error: package or namespace load failed for 'hdf5r' in dyn.load(file, DLLpath = DLLpath, ...):
 unable to load shared object '/home/user/libs/r/hdf5r/libs/hdf5r.so':
  libhdf5_serial_hl.so.100: cannot open shared object file: No such file or directory

发生这种情况的原因是，Foundry 限制了服务器上预装的 Linux 包数量，以减少暴露于潜在漏洞的风险。然而，某些 R 库依赖于特定的系统 Linux 库，而这些库可能未在服务器上预装。

请注意，部分包《不受 Code Workspaces 支持》

请联系你的 Palantir 代表，讨论所需 Linux 包的可用性。

错误：安装 pip 包时构建 wheel 失败

Pip 可能期望某些系统依赖项已预装，而 conda 会捆绑 Python 包运行所需的所有要求。一般建议如下：

  1. 如果可能，始终从 conda 而不是 pip 进行安装。
  2. 如果可能，使用 conda 构建 pip 包并将其发布到 Artifacts 仓库。
  3. 否则，尝试使用以下命令从 pip 安装：
已复制！
1maestro env pip install <package>
  1. 如果 pip 包安装失败，报错信息通常会提示缺失了哪个系统依赖项或文件。首先找到提供所需依赖的 conda 包，然后使用以下命令进行安装：
已复制！
1maestro env conda install <package>
  1. 重复步骤 3 和 4，直到安装成功。

错误：尝试将数据同步至代码仓库失败

当正在同步的 git 提交过大时，会出现此错误。如果该提交包含非代码文件（例如输出数据），就可能出现这种情况。代码工作空间内的版本控制不支持大文件。作为替代方案，可以交互式地将数据写回 Foundry，让你能够轻松使用并共享代码工作空间的输出结果。

要查明哪些文件导致了此错误，请在工作空间中打开终端并执行以下命令：

Copied!
1git rev-list --objects --all |
2   git cat-file --batch-check='%(objecttype) %(objectname) %(objectsize) %(rest)' |
3   sed -n 's/^blob //p' |
4   awk '{sum[$3]+=$2} END {for (i in sum) print sum[i], i}' |
5   sort --numeric-sort --key=1 |
6   $(command -v gnumfmt || echo numfmt) --field=1 --to=iec-i --suffix=B --padding=7 --round=nearest

要解决此错误，请从 git 历史记录中移除所有大文件。例如，要删除所有大于 10 MB 的文件，可以运行终端命令：git filter-repo --strip-blobs-bigger-than 10M。或者，要从 git 历史记录中删除特定文件，请使用 git filter-repo --path path/to/file.ext --invert-paths。

为防止再次发生此错误，请使用不应同步的大文件路径或扩展名更新 .gitignore 文件。例如，要防止同步任何输出 CSV 文件，请添加 *.csv。请注意，你可能需要确保在 IDE 中显示隐藏文件，以便查看 .gitignore 文件。

要应用这些更改，请在终端中运行 git push --force。完成这些步骤后，同步更改过程应该就能按预期工作了。

文件下载或上传错误

出于安全原因，Foundry 通常会限制文件下载和上传，以确保这些活动在适当的数据治理框架内进行。这意味着用户无法在 Shiny® 或 Streamlit 应用程序中添加“下载”按钮或类似功能。

要从 Jupyter® 或 RStudio® 工作区中下载数据，请先将数据写入 Foundry 中的输出数据集。随后，可通过 Foundry 的内置流程从该输出数据集下载数据，这一过程能够确保实施适当的数据治理限制。
