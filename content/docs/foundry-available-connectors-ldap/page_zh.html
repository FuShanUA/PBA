LDAP

LDAP 连接器是 Palantir 提供的用于 LDAP 的驱动程序。

要创建新的 LDAP 源，请遵循 Palantir 提供的驱动的标准设置流程，然后使用以下章节进行 LDAP 特定的配置和网络设置。有关完整的属性参考，请参阅官方 LDAP 驱动文档 ↗。

配置

以下属性为必填或推荐属性。

属性是否必填？描述默认值
Server ↗必填指定 LDAP 服务器的域名或 IP 地址。建立连接时需要此属性，无需包含 LDAP:\\ 前缀，仅需提供服务器的域名或 IP 地址。{serverAddress}
UseSSL ↗必填指定是否使用 SSL 保护与 LDAP 服务器的连接。启用此属性可确保客户端与服务器之间的通信经过加密。TRUE
BaseDN ↗推荐指定可辨别名称（DN）的基础部分，以将 LDAP 查询限制在目录中的特定子树内。使用基础 DN 有助于缩小搜索范围并提高查询性能，特别是在大型 LDAP 服务器上。—
Password ↗推荐指定用于向 LDAP 服务器进行身份验证的用户账户的可辨别名称（DN）关联的密码。当服务器强制要求身份验证时，需要此属性。—
Port ↗推荐指定 LDAP 服务器运行所在的端口号。636
User ↗推荐指定用于向 LDAP 服务器进行身份验证的用户账户的可辨别名称（DN）。当服务器强制要求身份验证时，需要此属性。—

对于每个域名，添加相应的出站策略。如果源部署在本地且无法从 Foundry 直接访问，请改用代理出站策略；代理主机本身也必须能够访问所列出的域名。有关详细信息，请参阅将智能体用作代理。
