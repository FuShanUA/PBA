设置您的Apollo中心

Apollo中心是管理您所有连接环境的中央环境。本页列出了您的Palantir代表将与您一起配置的项目。

域名

您的Apollo URL有两种可能的格式：

  • https://<environment-name>.palantircloud.com：当中心专用于一个客户时，将使用此格式。
  • https://<environment-name>.palantirapollo.com：当中心与多个客户共享时，将使用此格式。

在这两种格式中，<environment-name> 将被您定义的环境名称替换。

Kubernetes集群出口

请将您的本地Kubernetes集群出口IP提供给您的Palantir代表。他们将把这些IP添加到您的Apollo中心允许列表中。

联系您的 Palantir 代表以添加更多集群，或在您的 IP 地址发生变化时进行更改。

SSO 提供商集成

Apollo 的权限模型基于 基于角色的访问控制。每项操作和 API 都有一个相关权限，您可以将其授予 Apollo 团队。这些权限可以在 Apollo 中管理，并从 SSO 提供商中同步。

开始使用前，请向您的 Palantir 代表提供以下详细信息：

  • 实体 ID： 一个唯一 ID，用于在 URN 格式（如 urn:uuid:$[UUID]）中标识 Apollo。
  • 断言消费者服务 (ACS) URL： Apollo 端点，用于接收基于断言建立会话的 SAML 响应消息。
  • 单点退出 URL： Apollo 端点，用于接收来自身份提供商的 SAML 单点退出请求。
  • 证书： 用于对发送到身份提供商的 SAML 消息进行签名。

连接您的 SSO 提供商后，您可以配置组、团队和备份设置审批人。

在您的 Hub 设置完成后，您可 开始使用 Apollo。
