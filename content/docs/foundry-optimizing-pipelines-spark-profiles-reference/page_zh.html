Spark profiles reference

本页面提供了 Foundry 中可用的 Spark profiles 的参考。在此处了解更多关于 Spark 和 profiles 的信息：

  • 《Spark concepts》
  • 《Apply Spark profiles》

Driver cores

该系列 profiles 用于配置 spark.driver.cores 的值。

此参数控制分配给 Spark driver 的 CPU 核心数。在实际应用中，除非同一 Spark 模块中存在大量 Spark 作业并发执行的特殊情况，否则通常不需要覆盖此值。

Driver memory

该系列 profiles 用于配置 spark.driver.memory 的值。

此参数控制分配给 Spark driver JVM 的内存大小。在某些情况下可能需要调大此值，例如需要将大量数据拉取回 driver 时，或执行大型广播连接时。

此参数仅控制 JVM 内存，并不控制 Python 进程可用的内存。如果需要在本地拉取大量数据以通过 Pandas 进行转换，则需要使用不同的 profile。

Executor cores

该系列 profiles 用于配置 spark.executor.cores 的值。

此参数控制分配给每个 Spark executor 的 CPU 核心数，进而控制每个 executor 中并发执行的任务数量。在实际的常规转换作业中，通常极少需要覆盖此值。

Executor memory

该系列 profiles 用于配置 spark.executor.memory 及相关设置的值。

此参数控制分配给每个 Spark executor JVM 的内存大小。如果每个 Spark 任务处理的数据量非常大，则可能需要调大此值。

该内存由运行在 executor 上的所有任务共享（由 Executor Cores profiles 控制）。

Executor memory overhead

该系列 profiles 用于配置 spark.executor.memoryOverhead 的值。

此参数控制除 Spark executor JVM 内存之外，分配给每个容器的内存量。如果作业在 JVM 之外需要大量内存，则可能需要调大此值。

Number of executors

该系列 profiles 用于配置 spark.executor.instances 及相关设置的值。

此参数控制请求运行该作业的 executor 数量。增加此值可提升并行运行的任务数，从而提高性能（前提是作业本身的并行度足够），但代价是消耗更多资源。

在实际应用中，仅当大型作业有特定的组织需求且需要极快地运行完成时，才需要覆盖此值。

Dynamic allocation

该系列 profiles 用于配置 spark.dynamicAllocation.enabled、spark.dynamicAllocation.minExecutors 和 spark.dynamicAllocation.maxExecutors 的值。

此参数通过指定 executor 数量范围而非固定数量，来控制请求运行作业的 executor 数量。Spark 会将请求的 executor 数量扩展至 maxExecutors，并在不需要时释放这些 executor。当所需的 executor 数量并非固定不变，或在某些情况下需要加快启动时间时，此机制会有所帮助。系统无法保证模块一定能获得请求的 maxExecutors 数量，且由于 executor 数量可变，不同运行批次间的性能可能会有所差异。

在实际应用中，仅当处理大型作业且充分了解动态分配的优缺点时，才需要覆盖此值。

Adaptive query execution

该系列的配置文件用于启用和禁用自适应查询执行（AQE）。

启用 AQE 后，Spark 会在运行时自动设置分区数，从而有可能加快构建速度。它能避免因分区过少导致的并行度不足，以及因分区过多产生的大量小分区带来的额外开销。

AQE 的目标是使每个分区的输出大小保持在 64 MB 左右以实现均衡。例如，总输出大小为 512 MB 时，大约会产生 8 个分区。

你可以使用该系列中的文件大小配置文件来增加目标大小。如果写入的数据会被频繁读取（例如在 Contour 分析中），建议将分区大小设置为 128 MB 或更大。

如果总输出数据量较小但计算非常耗时（例如由于使用了高开销的 UDF），你可能需要禁用 AQE。在这种情况下，AQE 会降低并行度并拖慢计算速度。

每个任务的内核数

该系列的配置文件用于设置 spark.task.cpus 的值。

此参数控制为每个任务分配的内核数量。在实际操作中，极少需要覆盖此项配置。若需控制作业的并行度，应改用 执行器内核 进行调整。

Arrow

使用这些配置文件可启用或禁用 Arrow，以便在 Pandas 和 PySpark 数据框之间进行转换。若要使用 Arrow，请确保你的 Transform 依赖于 pyarrow 包。

当使用 Pandas 数据框调用 spark.createDataFrame() 或调用 toPandas() 时，Spark 必须对所有行进行序列化，以实现两种格式之间的转换。对于大型数据框而言，这是一个缓慢的过程，可能会成为 Transform 的瓶颈。在使用 Pandas Transform 时，读取和写入数据时都会发生这种序列化。

Arrow 是一种更高效的序列化格式，能够显著加快此类转换速度（详见《Arrow 官方博客》↗）。

Kubernetes

该系列的配置文件用于控制 Spark 作业执行方式的底层细节。

当使用的库对底层机器的 CPU 架构有具体要求时，你可以通过配置文件强制 Spark 作业在特定架构上运行。请注意，某些环境仅支持 AMD 架构的机器；在这些环境中，指定使用 ARM 架构的作业将无法成功运行。

KUBERNETES_OPEN_PORTS_ALL 配置文件可开启 Spark 执行器之间的网络通信，这是进行分布式模型训练的必要条件。

列级统计

该配置文件用于启用每列统计信息的计算，包括 min（最小值）、max（最大值）和 mean（均值）。此配置文件默认设为 false，即仅计算每列的 null 值计数以及数据集级别的指标（如行数）。
