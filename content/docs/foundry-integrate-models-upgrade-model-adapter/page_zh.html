无需重新训练升级模型适配器

本功能仅适用于在代码仓库中训练的模型。

回顾一下，模型由模型产出物和模型适配器组合而成：

  • 模型产出物：保存已训练模型的模型文件、参数、权重、容器或凭证。
  • 模型适配器：Foundry 与模型产出物交互所需的逻辑和环境依赖项，用于加载、初始化模型并执行推理。

有时，在不更改或重新生成模型产出物的情况下更新模型适配器会非常实用。例如：

  • 修复模型适配器中的 Bug，而无需重新训练已保存的模型。
  • 更新模型的 load、api、predict 或 run_inference 方法。
  • 在发现 Python 依赖项存在漏洞或 Bug 时，更新模型的 Python 环境。

然而，更新模型适配器也可能会给模型的推理环境引入破坏性变更。更改模型适配器时必须牢记以下几点：

  • 环境中对任何 Python 依赖项的更改都必须与现有的模型产出物兼容。
  • load、api、predict 或 run_inference 的新实现必须与现有的产出物兼容。
  • 更新后的模型适配器必须位于同一个 Python 模块中，且类名与原始模型适配器保持一致。

强烈建议在更新模型适配器时务必谨慎操作，并先在新的分支上测试更改。

从模型应用升级模型适配器

Foundry 支持在保留已发布模型权重的同时，将模型版本的适配器升级到最新的提交或标签。

要升级模型适配器，首先需在其定义所在的代码仓库中实现并发布对模型适配器类的更改。在代码仓库的模型训练模板中，每次提交后都会发布更改。在模型适配器模板中，创建新标签后才会发布更改。

接下来，在模型应用中，滚动至推理配置卡片。点击升级模型适配器。

点击创建模型版本，系统将使用更新后的模型适配器和之前发布的模型产出物创建一个新的模型版本。

如果先前的模型版本已提交至建模目标，更新后的模型版本不会自动提交，必须重新提交。
