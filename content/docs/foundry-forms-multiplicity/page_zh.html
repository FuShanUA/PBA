多重性

Foundry Forms 的设计目的是尽可能方便地从各种来源获取信息。在大多数情况下，这涉及到配置一个表单，受访者只需填写一次并根据输入的值创建一个新的行（或对象）。这可能更复杂或更简单，但基本的期望是受访者必须与表单的一个实例进行交互。

对于需要同一用户重复填写相同表单的用例，这种期望并不适用。

这种行为在需要记录许多观察结果时会变得繁琐和重复。每次填写表单实例时，都会浪费时间进行相同的选项选择（在这种情况下，Date 和 Primary Category），并提交响应。

这些用例最好通过多条目来解决。用户不需要一次又一次地填写表单，而是指定哪些字段是“共享”的，哪些是每个表单实例的“唯一”的：

请注意以下几点：

  1. 共享字段被提取出来，并在表头中只输入一次。
  2. 然后每个表单实例仅显示唯一的字段。
  3. 所有表单完成后，一次提交使用共享值和唯一值创建行（或对象）。

多条目表单不支持 Default Value 或 UrlParams 配置。

配置一个自由多条目表单

按照以下步骤创建一个在自由格式上下文中接受多个条目的表单：

  1. 在现有字段上方或下方选择+以继续添加更多共享字段。
  2. 在最后一个字段下方选择+，然后选择插入新工作表。
  3. 双击新工作表的标题以打开可视化编辑器。
  4. 切换多重性配置（按钮将显示为蓝色）。
  5. 从下拉菜单中选择自由多重性，并根据需要调整设置。
  6. 选择绿色的保存按钮。
  7. 在新工作表标题下方立即选择+以添加唯一字段。

在上面的例子中，每个Measurement和Measurement Type实例都会在支撑工作表中生成一个新的条目，因为它们位于不同的工作表中。Date和Primary Category存在于工作表之外，已被捕获为从此次提交创建的三个条目的公共值。

配置一个带参数的多条目表单

除了在所有表单实例之间共享公共值外，带参数的多条目表单使用一个共享字段来确定显示多少个表单实例，并在每个实例中设置一个唯一值。

考虑上面的实验观察示例。如果每天需要收集 Pressure、Depth 和 Density 的测量值，我们可以强制这些始终是用于 Measurement Type 的值。

与自由多条目表单不同，带参数的多条目表单需要额外的配置，以将共享字段与捕获其唯一值的字段连接起来：

按照以下步骤创建一个在参数化上下文中处理多个条目的表单：

  1. 在现有字段上方或下方选择 + 以继续添加更多共享字段。
  2. 选择 + 在最后一个字段下方，选择插入新工作表。
  3. 在新工作表标题下方立即选择+以添加将保存值的字段（例如，Text）。
  4. 双击新工作表标题以打开可视化编辑器。
  5. 打开多重性配置（按钮将显示为蓝色）。
  6. 从下拉菜单中选择参数化多重性。
  7. 在参数化 URI字段中，输入工作表中该字段的 URI。
  8. 将下一个下拉菜单从常量更改为标签。
  9. 选择当前标签无字段()。可用字段将以紫色突出显示。
  10. 选择将填充值的共享字段。
  11. 选择绿色的保存按钮。

要使预填值不可编辑，请打开接收字段的可视化编辑器并开启将此字段设为只读。

高级：通过对象进行配置

要为每个对象实例接收一个表单，可以按照以下步骤使用 Objects Provider 字段：

  1. 在现有字段上方或下方选择 + 并添加一个 Objects Provider 字段。
  2. 在最后一个字段下方选择 + 并选择 插入新工作表。
  3. 在新工作表标题下方立即选择 + 并添加一个 Text 字段。
  4. 双击新工作表标题以打开可视化编辑器。
  5. 切换 多重性配置 为开启状态（按钮将显示为蓝色）。
  6. 从下拉菜单中选择 参数化多重性。
  7. 在 参数化 URI 字段中，输入 Text 字段的 URI。
  8. 将下一个下拉菜单从 常量 更改为 标签。
  9. 选择 当前标签无字段()。可用字段将以紫色突出显示。
  10. 选择 Objects Provider 字段。
  11. 选择绿色的 保存 按钮。

Objects Provider 字段可以根据另一个字段的值进行过滤。
