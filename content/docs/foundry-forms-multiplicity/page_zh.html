多重性

Foundry Forms 旨在尽可能简化从各种来源收集信息的过程。在大多数情况下，这需要配置一个表单，由填写者填写一次，并根据输入的值创建单个新行（或对象）。这个过程可能简单也可能复杂，但基本预期是填写者只需与表单的单个实例进行交互。

如果同一表单需要同一用户反复填写，这种预期就不适用了。

然而，如果要记录的观察结果很多，这种行为就会变得繁琐且重复。每次填写表单实例都要进行相同的选择（本例中为 Date 和 Primary Category）并提交回复，纯属浪费时间。

这类用例最适合通过多条目来解决。用户无需每次只填写一个表单，而是可以指定表单的每个实例中哪些字段是“共享”的，哪些是“唯一”的：

请注意以下几点：

  1. 共享字段被提取出来，并在头部一次性输入。
  2. 随后，每个表单实例仅显示唯一的字段。
  3. 所有表单填写完毕后，单次提交即可利用共享值和唯一值创建行（或对象）。

多条目表单不支持 Default Value 或 UrlParams 配置。

配置自由多条目表单

按照以下步骤，在自由场景下创建支持多条目的表单：

  1. 在现有字段的上方或下方选择 +，继续添加更多共享字段。
  2. 在最后一个字段的下方选择 +，然后选择 Insert new sheet。
  3. 双击新表头打开可视化编辑器。
  4. 开启 Multiplicity config（按钮将显示为蓝色）。
  5. 从下拉菜单中选择 Free multiplicity，并根据需要调整设置。
  6. 选择绿色的 Save 按钮。
  7. 在新表头正下方选择 +，添加唯一字段。

在上面的示例中，由于 Measurement 和 Measurement Type 位于不同的工作表中，它们的每个实例都会在底层的电子表格中生成一条新记录。Date 和 Primary Category 位于工作表之外，因此会被捕获为此次提交创建的三个条目中每个条目的公共值。

配置参数化多条目表单

除了在所有表单实例中共享公共值外，参数化多条目表单还使用一个共享字段来确定要显示的表单实例数量，并在每个实例中设置唯一值。

回顾一下上文的实验观察示例。如果每天都需要采集 Pressure、Depth 和 Density 的测量数据，我们可以强制规定这些始终作为 Measurement Type 的值。

与自由多条目表单不同，参数化多条目表单需要进行额外配置，以将共享字段与捕获其唯一值的字段关联起来：

按照以下步骤，在参数化场景下创建支持多条目的表单：

  1. 在现有字段的上方或下方选择 +，继续添加更多共享字段。
  2. 在最后一个字段的下方选择 +，然后选择 Insert new sheet。
  3. 在新表头正下方选择 +，添加用于保存该值的字段（例如 Text）。
  4. 双击新表头打开可视化编辑器。
  5. 开启 Multiplicity config（按钮将显示为蓝色）。
  6. 从下拉菜单中选择 Parameterized multiplicity。
  7. 在 Parameterized uri 字段中，输入工作表中该字段的 URI。
  8. 将下一个下拉菜单从 Constant 更改为 Tag。
  9. 选择当前标签无字段()。可用字段将以紫色高亮显示。
  10. 选择将填充该值的共享字段。
  11. 选择绿色的保存按钮。

如需将预填充值设为不可编辑，请打开接收字段的视觉编辑器，并开启将此字段设为只读选项。

高级：使用对象进行配置

若要为每个对象实例接收一个表单，可以按照以下步骤使用对象提供器字段：

  1. 在现有字段的上方或下方选择+，并添加一个对象提供器字段。
  2. 在最后一个字段下方选择+，然后选择插入新表单。
  3. 在新表单头的正下方选择+，并添加一个文本字段。
  4. 双击新表单头以打开视觉编辑器。
  5. 开启多重性配置（按钮将显示为蓝色）。
  6. 从下拉菜单中选择参数化多重性。
  7. 在参数化 uri字段中，输入该文本字段的 URI。
  8. 将下一个下拉菜单从常量更改为标签。
  9. 选择当前标签无字段()。可用字段将以紫色高亮显示。
  10. 选择对象提供器字段。
  11. 选择绿色的保存按钮。

对象提供器字段可以基于另一个字段的值进行过滤。
