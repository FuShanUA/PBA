流式 Pipeline Builder 管道中的连接操作

借助流式 Pipeline Builder，你可以将流数据与批处理数据集或其他流数据连接起来。考虑到流处理的低延迟特性，其连接操作的实现方式与标准批处理管道有所不同。本页将说明连接操作的工作原理，以及如何在管道中更好地加以运用。

将流数据与批处理数据集连接

Foundry 允许你将低延迟流数据与批处理数据集结合起来，其操作方式类似于在批处理 Pipeline Builder 管道中连接两个批处理数据集。

在 Pipeline Builder 中完成以下步骤，即可将流数据与批处理数据集连接：

  1. 将流数据和批处理数据集添加到你的 Pipeline Builder 图表中。

  2. 在批处理数据集下方，选择下拉菜单并将类型更改为快照。

  3. 选择你要连接的流数据。

  4. 选择连接，并为连接的右侧选择批处理数据集。

  5. 在连接类型下，选择左查找连接。

  6. 输入匹配条件。

架构

流数据与批处理数据集的连接机制是：首先下载批处理数据集，并在流处理集群中建立索引，从而实现低延迟查找。为确保连接的低延迟特性，在同一个 Pipeline Builder 管道中，不允许在与流数据连接之前对该批处理数据集进行转换操作。

当有新的事务写入数据集时，批处理数据集便会更新。向批处理数据集添加新事务后，后台进程会下载新的数据视图并将其转换为可查询的格式。该进程完成后，流数据便会开始与该批次数据集的新视图进行连接。

限制

在与流数据连接之前，无法对批处理数据集进行转换。如需转换批处理数据集，可在上游的 Pipeline Builder 管道中完成。

流式连接需注意以下限制：

  • 与批处理数据集连接时，连接的左侧必须是流数据，或者是读取模式为“流”的批处理数据集。
  • 如果连接的批处理数据集数据量超过 8-10GB，性能可能会下降。
  • 若检测到新的追加事务，批处理数据集最多每五分钟更新一次。
  • 与大型静态数据集进行连接可能会拖慢集群启动速度。

将流数据与其他流数据连接

Foundry 允许你将多个低延迟流数据结合起来，其操作方式类似于在批处理 Pipeline Builder 管道中连接多个批处理数据集。

在 Pipeline Builder 中完成以下步骤，即可连接两个流数据：

  1. 将这两个流数据添加到你的 Pipeline Builder 图表中。
  2. 选择连接，并选择这两个流数据。
  3. 在连接类型下，选择外部缓存连接。
  4. 输入匹配条件。
  5. 指定缓存时间值和单位。缓存时间值和单位决定了数据在用于连接这两个流数据的缓存中的保留时长。

若你想用左连接或右连接替代外连接，可在连接操作下游过滤掉包含空值的记录。对于右连接，过滤掉右侧值为空的记录；对于左连接，同样过滤掉右侧值为空的记录。

架构

由于流数据会持续不断地运行，且连接两侧不断有新记录涌入，因此两个流数据之间的连接操作是对各自的数据缓存进行的，而非针对整个流数据直接进行连接。

多个流之间的关联操作仅限于在数据缓存上进行，以防止状态无界增长，否则会导致流处理集群最终耗尽内存并崩溃。通过为关联操作左右两侧的缓存设置过期时间，存储关联记录所需的状态将受到限制；这能避免流处理集群因内存耗尽而崩溃。

数据按键存储并分布到各个任务管理器上，从而支持更大规模的关联操作。这意味着，若要与更大的流进行关联，可以增加单个任务管理器的内存，或者增加集群中运行的任务管理器数量。

基于关联中指定的键列，流左侧的记录始终会与右侧的最新记录进行关联。仅特定键对应的最新记录会参与关联操作。

局限性

在将流与其他流进行关联时，请考虑以下局限性：

  • 关联操作的左右两侧均需设置缓存过期时间，以防止状态无界增长。
  • 关联操作的每一侧仅存储每个关联键对应的最新值。这意味着该关联操作的行为类似于“外连接”。
  • 如果左侧或右侧流中的记录在另一侧尚未出现匹配记录时就到达，系统将输出一条记录，且该记录中未匹配一侧的字段值为空。
