模块接口

模块接口是一组变量的集合，这些变量在嵌入时可以映射到父模块中的变量，并从 URL 进行初始化。您可以将模块接口视为 Workshop 模块的 API。

模块接口变量不同于本体接口，后者定义了跨对象类型的共享属性和链接。模块接口变量用于定义 Workshop 模块在嵌入和 URL 初始化时的输入与输出。

要将变量添加到模块接口，请导航至该变量的设置面板，添加一个外部 ID，并确保模块接口的开关已启用。您还可以为模块接口变量指定显示名称和描述，这些信息将在模块被嵌入或用于打开 Workshop 模块事件时显示。

嵌入模块接口

嵌入模块时，该模块的模块接口变量将可用，以便将父模块变量映射到子模块接口变量。您可以在嵌入模块文档的接口配置中了解更多相关信息。

利用模块接口变量，可以在父模块与子模块之间，或同级的嵌入模块之间进行通信。这些共享的接口变量可以为共享状态提供支持，例如选定的对象、选定的选项卡，或是否显示覆盖层。嵌入模块可以通过事件修改接口变量的值，从而让引用这些变量的其他位置对更新后的值做出响应。有关更多信息，请参阅跨嵌入模块通信。

当在父模块和嵌入的子模块之间映射接口变量时，Workshop 将使用父模块的变量定义，并忽略嵌入模块自身的接口变量定义。详情请参阅嵌入模块接口配置。

打开 Workshop 模块事件

可以使用打开 Workshop 模块事件来避免手动创建 URL，如下所述。此时将显示所选模块的接口，允许将变量值从当前模块传递到所选模块的接口变量。调用该事件时，URL 会使用当前值打开所选模块。

在编辑模式下，当您通过模块引用打开某个模块时（例如，在其自身的编辑器中打开嵌入的子模块），该模块会使用从源模块传递过来的任何模块接口变量的当前值打开。这允许您在引用该模块的位置保持相同状态，从而对打开的模块进行调试。

使用模块接口变量创建 URL

除打开 Workshop 模块事件外，还可以手动动态生成 URL，以便使用模块接口变量的 URL 查询参数值来共享应用程序状态的自定义链接。为此，请按照以下步骤操作：

  1. 登录您的 Workshop 应用程序。
  2. 从浏览器中复制 URL。确保处于查看模式而非编辑模式。此 URL 是固定的，因此可以与具有正确权限的其他用户共享。
  3. 现在，返回到编辑模式。
  4. 选择位于左侧边栏的变量菜单。
  5. 创建一个新变量 > 字符串 > 静态。
  6. 转到变量的设置选项卡，并添加一个外部 ID。
  7. 在第 2 步的 URL 后追加 ?，然后是外部 ID、= 以及您想要设置的值。例如，?interfaceVariable=123。您可以通过添加与号 &，接着添加额外的外部 ID、等号 = 以及对应的值，来追加其他模块接口变量。

模块在首次加载时，会从 URL 参数初始化模块接口变量。在模块加载完成后更改 URL 不会动态更新变量值。若要在运行时更新变量值，请使用 Workshop 事件，例如设置变量值事件或模块加载时事件。

链接现已就绪，会将模块接口变量设置为 URL 中指定的值。通过使用变量转换可动态生成此链接，其取值可来源于操作表单、函数或其他变量转换。

出于测试目的，您可以将 URL 中的 /latest/ 更改为 /dev/，这样链接将重定向至 Workshop 应用程序上次保存的版本，而非上次发布的版本。

Carbon 导航

请参阅此处关于在 Carbon 导航中使用模块接口变量的说明。请注意，Carbon 支持的变量类型存在若干限制；在为 Carbon 模块标签页配置参数时，外部 ID 必须以 variable. 作为前缀。
