在代码仓库中训练模型

模型可以在 代码仓库 应用程序中通过 Model Training Template 进行训练。

要训练一个模型，请完成以下步骤：

  1. 创建一个模型适配器
  2. 编写一个 Python 转换以训练模型
  3. 构建您的 Python 转换以发布训练后的模型
  4. 使用该模型

1. 创建一个模型适配器

一个模型使用 模型适配器 来确保 Foundry 正确地初始化、序列化、反序列化以及对模型执行推理。您可以通过 Model Adapter Library 模板 或者 Model Training 模板 在代码仓库中创建模型适配器。

模型适配器不能直接在Python转换仓库中编写。要从现有仓库生成一个模型，请使用Model Adapter Library并把库导入到转换仓库中，或者迁移到Model Training模板。

有关何时使用每种类型的模型适配器仓库以及如何创建它们的更多信息，请参阅 模型适配器创建 文档。下面的步骤假定使用的是模型训练模板。

模型适配器实现

模型适配器和模型训练代码应该放在不同的Python模块中，以确保训练后的模型可以在下游转换中使用。在模板中，我们为此目的分别提供了model_adapters和model_training模块。在adapter.py文件中编写你的模型适配器。

模型适配器定义将取决于正在训练的模型。要了解更多，请查阅 ModelAdapter API 参考，查看 sklearn 模型适配器示例，或阅读 监督式机器学习教程 以获取更多信息。

2. 编写一个 Python 转换来训练模型

接下来，在你的代码仓库中创建一个新的 Python 文件来存放你的训练逻辑。

1from transforms.api import transform, Input
2from palantir_models.transforms import ModelOutput
3from model_adapter.example_adapter import ExampleModelAdapter  # This is your custom ModelAdapter
4
5
6def train_model(training_data):
7    '''
8        This function contains your custom training logic.
9    '''
10    pass
11
12@transform.using(
13    training_data=Input("/path/to/training_data"),
14    model_output=ModelOutput("/path/to/model")                 # This is the path to the model
15)
16def compute(training_data, model_output):
17    '''
18        This function contains logic to read and write to Foundry.
19    '''
20    trained_model = train_model(training_data)                 # 1. Train the model in a Python transform
21    wrapped_model = ExampleModelAdapter(trained_model)         # 2. Wrap the trained model in your custom ModelAdapter
22    model_output.publish(                                      # 3. Save the wrapped model to Foundry
23        model_adapter=wrapped_model                            #    Foundry will call ModelAdapter.save to produce a model
24    )

此逻辑会发布到 ModelOutput. 在你提交更改后，Foundry 将自动在提供的路径上创建一个模型资源。你也可以使用 @configure 注解 配置模型训练所需的资源，如 CPU、内存和 GPU 要求。

(可选) 将指标和超参数记录到模型实验中

模型实验 是一个轻量级框架，用于记录模型训练过程中产生的指标和超参数，这些可以随后与模型一起发布并保存在模型页面中。

了解有关创建和编写实验的更多信息。

3. 预览你的 Python 转换以测试你的逻辑

在代码仓库应用中，你可以选择 预览 来测试你的转换逻辑而无需运行完整的构建。请注意，预览将在比你通过 @configure 注解 所配置的资源配置更小的资源配置下运行。

ModelOutput 预览

ModelOutput 预览允许你验证你的模型训练逻辑以及你的模型序列化、反序列化和 API 实现。

ModelInput 预览

ModelInput 预览允许您针对现有模型验证您的推理逻辑。请注意，在代码仓库中进行预览时，每个 ModelInput 有 5GB 的大小限制。

4. 构建您的 Python 转换以发布训练好的模型

在您的代码仓库中，选择 构建 来运行您的转换。Foundry 会在执行您的训练逻辑之前解析 Python 依赖项和模型的依赖项。

调用 ModelOutput.publish() 会将您的模型版本发布到 Foundry。Foundry 将调用 ModelAdapter.save() 函数，并赋予您的 ModelAdapter 序列化执行所需所有字段的能力。

5. 使用模型

提交至建模目标

模型可以发布到建模目标以进行：

  • 自动模型评估
  • 批量管道
  • 实时托管

在 Python 转换中运行推理

您也可以按照以下说明在管道中使用 Palantir 模型。

要在未定义该模型的转换仓库中运行模型，请在use_sidecar = True中设置ModelInput。这将在一个独立环境中运行，以防止依赖冲突，同时自动导入模型适配器及其依赖项。有关更多详细信息，请查看ModelInput类参考。此功能适用于非Spark转换（使用@lightweight或@transform.using装饰器）从palantir_models版本0.2010.0开始。

如果use_sidecar未设置为True，则必须将模型适配器及其依赖项导入或定义在当前代码仓库中。

如果您在与模型创建仓库不同的Python转换仓库中使用模型，则必须将模型适配器Python库添加到您的编写Python环境中

. 这会将加载模型所需的 Python 包引入到消费仓库的环境中。代码编写用户界面会在检测到模型的依赖项不在仓库中时，在悬停在警告上时提供库导入选项。特定模型版本对应的适配器库及其版本可以在 Foundry 的模型页面上找到。

Copied!
1from transforms.api import transform, Input, Output, LightweightInput, LightweightOutput
2from palantir_models.transforms import ModelInput
3from palantir_models import ModelAdapter
4
5@transform.using(
6    inference_input=Input("/path/to/inference_input"),
7    model=ModelInput(
8        "/path/to/model",
9        # use_sidecar=True is recommended for models defined outside the current transform repository
10        ),
11    inference_output=Output("/path/to/inference_output"),
12)
13def compute(
14        inference_input: LightweightInput,
15        model: ModelAdapter,
16        inference_output: LightweightOutput
17    ):
18    inference_results = model.transform(inference_input)
19    # Replace "output_data" with an output specified in the Model Version's API, viewable on the Model Version's web page.
20    # For example, inference_results.output_data is appropriate output for Hugging Face adapters.
21    inference = inference_results.output_data
22    inference_output.write_pandas(inference)

ModelInput 和 ModelOutput API

上面转换中调用的 ModelInput 和 ModelOutput 对象是直接负责与代码仓库中的模型进行交互的对象。其对象概要如下：

ModelInput

Copied!
1# ModelInput can be imported from palantir_models.transforms
2
3class ModelInput:
4    def __init__(self, model_rid_or_path: str, model_version: Optional[str] = None):
5        '''
6            The `ModelInput` retrieves an existing model for use in a transform. It takes up to two arguments:
7            1. A path to a model (or the model's RID).
8            2. (Optional) A version RID of the model to retrieve.
9            If this is not provided, the most recently published model will be used.
10
11            For example: ModelInput("/path/to/model/asset")
12        '''
13        pass

在转换函数中，由 ModelInput 指定的模型将被实例化为与检索到的模型版本相关联的模型适配器的实例。Foundry 将调用 ModelAdapter.load 或使用定义的 @auto_serialize 实例来在启动转换构建之前设置模型。因此，ModelInput 实例可以访问其关联的已加载模型状态以及模型适配器中定义的所有方法。

ModelOutput

Copied!
1# ModelOutput can be imported from palantir_models.transforms
2
3class ModelOutput:
4    def __init__(self, model_rid_or_path: str):
5        '''
6            The `ModelOutput` is used to publish new versions to a model.
7            `ModelOutput` takes one argument, which is the path to a model (or model RID).
8            If the asset does not yet exist, the ModelOutput will create the asset when a user selects Commit or Build and transforms checks (CI) are executed.
9        '''
10        pass

在转换函数中，通过分配一个 ModelOutput 获得的对象是一个 WritableModel，它可以通过使用 publish() 方法发布新的模型版本。此方法以模型适配器作为参数，并创建与其相关联的新模型版本。在 publish() 过程中，平台使用定义的 @auto_serialize 实例或执行实现的 save() 方法。这使得模型适配器能够将模型文件或检查点序列化到 state_writer 对象中。
