在代码仓库中训练模型

模型可以通过 模型训练模板 在 代码仓库 应用中进行训练。

要训练模型，请完成以下步骤：

  1. 编写模型适配器
  2. 编写用于训练模型的 Python 转换
  3. 构建 Python 转换以发布训练好的模型
  4. 使用模型

1. 编写模型适配器

模型通过 模型适配器 确保 Foundry 能够正确对其进行初始化、序列化、反序列化及执行推理。在代码仓库中，你可以使用 模型适配器库 模板 或 模型训练 模板 来编写模型适配器。

无法直接在 Python 转换仓库中编写模型适配器。如果要在现有仓库中生成模型，请使用 模型适配器库 并将该库导入到转换仓库中，或者迁移至 模型训练 模板。

关于何时使用各类模型适配器仓库以及如何创建它们，请参阅 模型适配器创建 文档以了解更多信息。以下步骤假定使用模型训练模板。

模型适配器实现

模型适配器与模型训练代码应放在不同的 Python 模块中，以确保训练好的模型能够在下游转换中使用。为此，模板中专门设置了 model_adapters 和 model_training 独立模块。请在 adapter.py 文件中编写你的模型适配器。

模型适配器的具体定义取决于所训练的模型。欲了解详情，请查阅 ModelAdapter API 参考、查看 sklearn 模型适配器示例，或通读 监督机器学习教程。

3. 预览 Python 转换以测试逻辑

在代码仓库应用中，你可以选择预览来测试转换逻辑，而无需运行完整的构建。请注意，预览所使用的资源配置将低于你通过 @configure 注解配置的资源规格。

ModelOutput 预览

ModelOutput 预览可用于验证模型训练逻辑，以及模型的序列化、反序列化和 API 实现。

ModelInput 预览

ModelInput 预览可用于针对现有模型验证推理逻辑。请注意，在代码仓库中进行预览时，每个 ModelInput 的大小限制为 5GB。

4. 构建 Python 转换以发布训练好的模型

在代码仓库中，选择构建以运行转换。Foundry 会在执行训练逻辑之前，先解析 Python 依赖项和模型依赖项。

调用 ModelOutput.publish() 会将模型的一个版本发布到 Foundry。Foundry 将调用 ModelAdapter.save() 函数，并赋予你的 ModelAdapter 序列化执行所需全部字段的能力。

5. 使用模型

提交至建模目标

可以将模型发布到建模目标，用于：

  • 《自动模型评估》
  • 《批处理管道》
  • 《实时托管》

在 Python 转换中运行推理

你还可以按照下文详述的方式在管道中使用 Palantir 模型。

若要在未定义模型的转换库中运行该模型，请在 ModelInput 中设置 use_sidecar = True。这会在独立环境中自动导入模型适配器及其依赖项并运行它们，从而防止依赖冲突。详情请查阅《ModelInput 类参考》。自 palantir_models 0.2010.0 版本起，此功能适用于非 Spark 转换（使用 @lightweight 或 @transform.using 装饰器）。

如果未将 use_sidecar 设置为 True，则必须将模型适配器及其依赖项导入或定义在当前代码库中。

如果你在另一个 Python 转换库中使用模型，而该模型并非在该库中创建，则必须将模型适配器 Python 库添加到你的《编写 Python 环境》中。这会将加载模型所需的 Python 包引入到使用该模型的库环境中。代码编写界面会检测出代码库中是否缺失模型的依赖项，并在鼠标悬停于警告提示上时，提供执行库导入的选项。在 Foundry 的模型页面中，可以找到与特定模型版本对应的适配器库及其版本信息。

12        '''
13        pass



在转换函数中，ModelInput 指定的模型将实例化为与所检索模型版本关联的模型适配器实例。Foundry 会调用 ModelAdapter.load 或使用已定义的 @auto_serialize 实例来设置模型，随后才启动转换构建。因此，ModelInput 实例能够访问其关联的已加载模型状态以及模型适配器中定义的所有方法。

3class ModelOutput:
4    def __init__(self, model_rid_or_path: str):
5        '''

7            `ModelOutput` 接受一个参数，即模型的路径（或模型 RID）。
8            如果该资产尚不存在，当用户选择提交或构建并执行转换检查（CI）时，ModelOutput 将创建该资产。
9        '''
10        pass


在转换函数中，通过赋值 `ModelOutput` 获取的对象是 `WritableModel`，它能够通过调用 `publish()` 方法发布新的模型版本。该方法接收模型适配器作为参数，并创建与之关联的新模型版本。在执行 `publish()` 时，平台会使用定义的 `@auto_serialize` 实例，或执行已实现的 `save()` 方法。这使得模型适配器能够将模型文件或检查点序列化至 `state_writer` 对象。
