预览转换

Palantir 的 Visual Studio Code 扩展允许您直接从本地 Visual Studio Code 环境或 Palantir 平台中的 VS Code 工作区预览 Python 转换（transform）。预览是一个实时测试环境，您可以在此查看数据转换的效果，然后再将其提交到完整的构建中。编写代码并快速预览更改，以获得即时反馈，而无需构建或切换上下文。

为什么要使用预览？

更快的开发周期： 在不运行完整管道构建的情况下，立即获取对转换的反馈。这在迭代复杂逻辑时可以节省大量时间。

尽早发现错误： 在生产管道中出现问题之前，发现数据问题、逻辑错误或意外结果。

自信地进行实验： 尝试不同的方法并立即看到结果，这在探索数据或测试边缘情况时特别有用。

高效调试： 使用真实数据查看代码的执行情况，使识别和修复问题变得更加容易。

预览您的转换

开始预览

您可以根据最适合您工作流的方式，通过以下四种方式之一开始预览：

  1. 命令面板： 从命令面板中选择 运行预览 命令。

  2. Palantir侧边栏： 从工具栏中选择 预览 图标。

  3. CodeLens： 选择 预览 在转换上方。

  4. 预览面板： 打开预览面板并选择代码文件名旁边的预览按钮。

扩展将使用您输入的真实数据执行您的转换，并立即向您显示结果。

选择您的数据加载策略

转换预览中最强大的功能之一是控制如何加载数据。这尤其重要，因为不同的转换有不同的数据需求；简单的聚合对于任何样本都效果很好，而测试一个狭窄的过滤器或复杂的连接则需要更仔细地选择数据。

要选择策略，请在 预览 面板中选择 配置输入策略，然后在 采样 和 全量 之间进行选择。

采样、完整数据集 或 代码定义的筛选条件 选项。

采样（默认）

采样 策略加载输入数据的随机样本，默认为 1000 行。该子集是从输入中随机选取的，您可以配置行数。

理想用途： 采样足够且不会引入统计偏差的转换。适合快速验证转换逻辑。

限制： 对于某些转换，例如窄过滤或多个输入之间的连接，结果可能具有误导性，因为匹配过滤表达式的值在样本中出现的可能性较低。当多个输入之间进行连接时，问题会变得更加严重。

完整数据集

完整数据集 选项基于您的转换逻辑使用智能数据加载，不会进行预采样。相反，预览引擎依赖于现代数据处理引擎（如 Spark 或 Polars），将谓词 下推 ↗ 到数据源级别，并仅下载最可能匹配查询的数据集块。

理想用途： 在转换代码中任何位置使用的过滤器或其他缩小表达式都可能适用于下推，从而在不增加太多额外计算时间的情况下获得完全准确的预览结果。这对于准确测试过滤器和连接特别有价值。

最佳实践： 某些管道无法充分利用谓词下推（例如，不包含过滤表达式的管道）。在这种情况下，您可以在代码中引入条件过滤表达式，以加快开发过程中的预览运行。谓词下推逻辑的效果高度依赖于所选的数据处理引擎。Spark 和 DuckDB 通常能产生最快的結果，Polars 在流模式下是足够的，而 Pandas 不支持谓词下推。

代码定义的过滤器

编写您自己的自定义过滤函数，以精确控制预览期间加载的数据。此选项允许您直接在您的代码中指定自己的自定义过滤策略。

理想用途：当您需要对用于预览的数据进行细粒度控制、想要创建可重用的过滤逻辑，或需要测试随机采样或完整数据集加载无法有效处理的特定场景时。采样可以有效地加快预览速度，但是这不仅要求过滤器生成一个足够小的输出集，还需要高效实现的过滤逻辑。这是因为代码定义的过滤器将使用整个生产数据集作为其输入。

有关更多详细信息，请查阅我们的 代码定义的输入过滤 文档。

理解增量预览结果

在使用增量转换时，预览 面板会提供有关您的转换执行情况的详细反馈。Python 转换 VS Code 集成使用与生产构建相同的增量解析和评估逻辑，因此如果在相同时间触发，则运行预览将产生与构建相同的输出（忽略抽样）。

如果转换是增量运行的： 在预览面板中会出现一个Ran incrementally标签。

如果转换是快照方式运行的： 会出现一个Ran as a snapshot标签。将鼠标悬停在此标签上，可以查看转换无法进行增量运行的原因（例如对semantic_version参数的更改，或输入中的某些事务类型）。

增量预览功能支持 Spark 和 轻量级转换，包括 v1 和 v2 的增量语义。

如需了解关于增量转换及其评估逻辑的更多背景信息，请查阅 增量转换 文档。

技术细节

本地开发的先决条件

Palantir 扩展程序用于 Visual Studio Code 的本地预览使用 Preview Engine。如果数据具有 Download 权限，此 Preview Engine 会将数据集的部分内容流式传输到您的机器上。有关本地预览的安全性影响和数据存储的更多信息，请参阅我们的文档 本地开发的安全注意事项。

要在本地开发期间使用预览功能，必须由平台管理员从 代码仓库 设置页面在控制面板中。

对于使用 Gradle 的 Java 转换本地预览设置，请查阅我们的文档 在本地开发中预览转换。

当打开 Palantir 仓库时，扩展将配置环境。一旦环境设置完成并检测到转换，您就可以在本地执行预览。

Spark 配置文件

Spark 配置文件允许用户快速定义和使用自定义 Spark 配置值，以指定在预览或构建转换时 Spark 引擎的行为。有关更多详细信息，请参阅 Spark 配置文件 的文档。

VS Code 预览会从预览的转换的 Spark 配置文件中应用一些配置。这包括影响执行引擎运行时行为的配置，通常用于在发生破坏性更改期间保持向后兼容性。无法通过 Spark 配置文件更改预览的资源分配；这可以在 代码工作区设置页面 上单独更改。

外部转换

代码工作区中的外部转换实施严格的导出控制。代码工作区应用程序会保留工作区输入的历史记录，因此包含额外安全标记的先前输入可能会因标记违规而阻止预览。此外，当工作区计算其标记安全检查和导出控制时，该应用程序会考虑所有之前整合的容器标记，以避免标记数据的不当暴露。

如果工作区中存在与外部转换不兼容的标记，请在没有检查点的情况下重新启动工作区以清除跟踪的标记。有关更多信息，请查阅 外部转换文档。
