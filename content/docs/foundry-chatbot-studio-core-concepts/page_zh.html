核心概念

以下核心概念对于理解并充分利用 AIP Chatbot Studio（前身为 AIP Agent Studio）至关重要。你可以在《入门指南》教程中了解有关应用这些概念的更多信息。

AIP Chatbots

AIP Chatbots（前身为 AIP Agents）是配备了企业专属信息和工具的交互式助手。

应用状态

应用状态（前身为“参数”）是提示词中的应用变量，用于自定义和控制大语言模型的行为。它们支持动态输入，并可根据任务需求进行调整。

指令与描述

指令、工具描述和变量描述会被编译为大语言模型的原始系统提示词。这能引导大语言模型如何利用可用上下文来完成任务。

在编写指令时，应将最重要的信息放在开头，例如任务概述。紧接着提供必要的数据，以及关于如何使用应用状态和工具的指引。

在编写每个工具和变量的描述时，需向大语言模型提供明确步骤，说明如何以及何时使用该项特定上下文。请记住，大语言模型只能访问你显式提供的信息。

检索增强生成 (RAG)

检索增强生成利用外部数据源，动态为大语言模型提供相关信息。该方法能够确保大语言模型的回答基于最新且最符合上下文的数据，从而提升其响应质量。

检索上下文

检索上下文是指针对每条消息所检索到的特定信息，用于生成回复。具体流程如下：

  1. 用户发送一条新消息。
  2. 根据用户消息，从已配置的数据源中获取相关内容。
  3. 将相关内容连同用户消息一并发送给大语言模型。

有关更多详细信息，请查阅《检索上下文》文档。

工具

工具是大语言模型可调用的外部功能或 API，用于执行特定操作或获取其基础能力之外的信息。

有关更多详细信息，请查阅《工具》文档。

向量嵌入

嵌入是文本的数值表示形式，能够捕捉语义信息。它们可用于高效地比较和检索相似文本片段。在 AIP Chatbot Studio 中，嵌入有助于识别相关文档和上下文，从而提供准确的回复。

上下文窗口

上下文窗口是指大语言模型单次可处理的文本量（通常以 Token 为计量单位）。在 AIP Chatbot Studio 中，上下文窗口包含系统提示词、对话历史，以及为辅助大语言模型而注入的信息（包括检索上下文、应用状态和工具）。若超出上下文窗口限制将导致报错，系统会提示用户创建新会话以继续交互。

作为函数的聊天机器人

聊天机器人可作为函数发布，使其能够在平台上任何支持执行函数的地方使用。例如，构建者可以将 AIP Chatbots 发布为函数，以便在《AIP Evals》中进行评估、通过《Automate》实现聊天机器人工作流自动化，或在《代码仓库》中使用聊天机器人。有关更多信息，请参阅《作为函数的聊天机器人》文档。
