配置发布提升管道

本页面将介绍如何设置发布提升管道，以开始在Apollo中管理产品升级。

前提条件

在配置发布提升管道之前，您应该 创建任何想要包含在您的提升管道中的自定义发布通道。

定义发布标准

您可以通过从产品主页的操作（Actions）下拉菜单中选择编辑产品设置（Edit product settings）来定义提升要求。

导航到 升级 选项卡以指定 健康要求、发布节奏 和 维护窗口。

发布时的健康要求

为了使不健康阈值对发布和自动召回相关，您的 Helm chart 必须 产生健康状态 对于不产生健康状态的实体，Apollo 仅考虑存活状态，并且该推广将仅基于时间。这意味着所有 Pod 都处于存活状态的时间段将被视为“健康”，而其他时间段将既不被视为“健康”也不被视为“不健康”。因此，推广可以成功或继续评估而无需健康状态，但永远不会失败。

Apollo 在推广评估期间计算两个持续时间：

  • 健康持续时间： 推广被视作健康的持续时间。当所有节点都处于存活状态，并且其健康状态为 HEALTHY 或 DEFERRING 时，这段时间将被视为健康。
  • 不健康持续时间： 推广被视作不健康的持续时间。当至少一个存活节点具有 ERROR 健康状态时，这段时间将被视为不健康。

注意：

  • WARNING、REPAIRING、SUSPENDED 和 TERMINAL 健康状态不会对健康或不健康持续时间产生任何影响。这意味着健康持续时间加上不健康持续时间的总和可能低于总的浸泡时间。
  • 健康和非健康时间段的持续时间不包括周末。

    • 您可以根据健康和非健康时间段定义两个阈值。这些阈值的值必须在零到一之间。

    • 健康阈值：实体必须保持健康的最低百分比，以使Apollo认为发布成功。此阈值的默认值为0.95。
    • 非健康阈值：实体可以非健康的最高百分比，以使Apollo认为发布失败或存在必须在进一步发布前解决的问题。此阈值的默认值为0.05。

    配置推广管道

    要配置推广管道，请导航到产品首页的版本选项卡，并从推广管道框中选择编辑。

    您也可以从操作下拉菜单中选择编辑发布管道。

    这将打开发布管道配置窗口。

    一个发布阶段指的是两个发布通道之间的转换。选择添加阶段以添加管道阶段，或点击阶段右侧的“×”来删除发布阶段。

    对于每个发布阶段，您需要定义以下发布通道：

    • 源通道：Apollo 将从中发布的发布通道。
    • 目标通道：Apollo 将其发布的发布通道。这可以是 Apollo 中创建的任何发布通道，但不能是默认发布通道（RELEASE, RELEASE_CANDIDATE, 和 DEV）。

    两种推广方式可用：

    • 定时推广： 在定义的时间段之后，将发布版本推广到目标通道。
    • 灰度推广： 根据选定的一组测试环境评估的标准，将发布版本推广到目标通道。

    除了满足每个阶段指定的推广要求外，只有在 产品的维护窗口 内才会进行推广。

    对产品推广管道的更改将在下一次评估时生效，评估每分钟发生一次。

    定时推广阶段

    当您选择 定时推广，Apollo 将使用基于时间的推广评估。您可以为基于时间的推广定义以下参数：

    • 持续时间： 发布在被提升到目标发布通道之前，必须在源发布通道上停留的时间。该持续时间可以设置为0秒到31天之间的任何值。
    • 产品发布标签要求：仅当发布满足标签要求时才进行提升。
      • 发布必须具有指定值之一的标签。

    Canary推广阶段

    当你选择Canary推广时，Apollo将使用基于Canary的推广评估。在基于Canary的推广评估期间，Apollo将使用配置的实体过滤器来选择测试环境。它将根据你定义的条件对发布进行评估。当所有条件都满足时，发布将被提升到目标发布通道。

    实体过滤器

    实体过滤器 定义了Apollo将从中选择测试环境的实体池。可用的过滤器包括：

    • 带有标签的环境： 选择具有指定标签的环境中的实体。
      • 该标签必须存在于环境中。
      • 如果没有指定标签值，Apollo将检查环境中是否存在该标签。
      • 如果指定了标签值，Apollo将检查环境中是否存在该标签及其值。
    • 不带标签的环境： 选择不具有指定标签的环境中的实体。
      • 该标签必须不存在于环境中。
      • 如果没有指定标签值，Apollo将检查环境中是否没有该标签。
      • 如果指定了标签值，Apollo 将检查环境上是否存在该标签及其值。
    • 选择环境：从一组离散的环境中选择实体。
    • 排除环境：选择不在一组离散的环境中的实体。

    条件

    条件定义了发布到目标发布通道之前必须满足的要求。

    • 健康时发布：一旦指定数量的实体至少经过指定的时间，并且满足在升级选项卡中定义的推出标准中的 健康阈值，就发布该版本。
      • 实体的数量可以定义为数量或所选实体的百分比。
      • 持续时间可以设置为 0 秒 到 13 天 之间的任何时间。
    • 不健康时召回： 如果指定数量的实体已浸泡至少指定时长，并且满足在升级选项卡中定义的发布标准中的不健康阈值，则需要回忆该发布。
      • 实体数量可以定义为所选实体的数量或百分比。
      • 持续时间可以设置为介于0秒和13天之间的任何时间段。
    • 最小实体数量: 仅当所选实体数量超过指定数量时才进行提升。
    • 需要产品发布标签: 仅当发布满足标签要求时才进行提升。
      • 该标签必须存在于发布中。
      • 如果没有指定标签值，Apollo 将检查发布中是否存在该标签。
      • 如果指定了标签值，Apollo 将检查发布中是否存在该标签及其值。

    编辑完提升管道后选择确认。

    默认推广管道

    Hub管理员可以在Hub级别配置默认推广管道，为新的产品和模块提供一个合理的起点。

    Apollo为每个Hub维护两个默认的推广管道：

    • 产品默认: 应用于新产品。可能包括 定时 和 金丝雀 推广阶段。
    • 模块默认: 应用于新模块。只包括 定时 推广阶段，因为模块推广管道不支持金丝雀阶段。

    当一个新的产品或模块的第一个发布版本被发布时，Apollo会从相关的默认值中填充该产品或模块的推广管道。

    对Hub默认值的更改 不会 可以追溯生效。已经配置了促销管道的产品和模块（包括其管道最初来自默认设置的产品）在默认设置更改时不受影响。要将默认设置传播到现有产品或模块中管道为空的情况，请参见 将默认设置应用到现有产品。

    配置默认的促销管道

    要配置或编辑默认的促销管道，请导航至 发布渠道与推广 选项卡中的 中心设置。选择 编辑 产品默认或模块默认。选择 确认 将生成一个变更请求。

    将默认值应用于现有产品

    Hub管理员可以将默认值应用于Hub中没有配置管道的现有产品或模块。此选项对于在默认值存在之前创建了许多产品的Hub非常有用。

    要批量更新已经配置了管道的产品的推广管道，请使用 apollo-cli product promotion set 命令。
