配置发布提升管道

本页面将介绍如何设置发布提升管道，以开始在Apollo中管理产品升级。

前提条件

在配置发布提升管道之前，您应该创建任何您希望包含在您的提升管道中的自定义发布通道。

定义发布标准

您可以通过从产品主页的“操作”下拉菜单中选择编辑产品设置来定义提升要求。

导航到升级选项卡，以指定健康要求、发布节奏和维护窗口。

发布推广的健康要求

为了使不健康阈值对推广和自动召回相关，您的Helm图表必须 产生健康状态。对于未产生健康状态的实体，Apollo将只考虑活性，并且推广将仅基于时间。这意味着所有Pod都处于活跃状态的推广时间段将被视为“健康”，其他时间段将既不被视为“健康”也不被视为“不健康”。因此，推广可以成功或继续评估而无需健康状态，但永远不会失败。

Apollo在推广评估期间计算两个持续时间：

  • 健康持续时间：推广被认为健康的持续时间。当所有节点都处于活跃状态且其健康状态为HEALTHY或DEFERRING时，这段时间被认为是健康的。
  • 不健康持续时间： 一个推广被认为是不健康的时长。当至少有一个活动节点具有 错误 健康状态时，这段时间被认为是非健康的。

注意：

  • 警告、修复中、暂停 和 终止 健康状态不会对健康或不健康持续时间产生影响。这意味着健康持续时间加上不健康持续时间的总和可能低于总的浸泡时间。
  • 健康和不健康时间段的持续时间都不包括周末。

您可以根据健康和不健康持续时间定义两个阈值。这些阈值的值必须在零到一之间。

  • 健康阈值： Apollo 认为一次发布成功的实体必须保持健康的最小百分比。此阈值的默认值为 0.95。
  • 不健康阈值： Apollo 认为一次发布失败或存在必须在进一步发布前解决的问题的实体可以不健康的最大百分比。此阈值的默认值为 0.05。

配置发布管道

要配置发布管道，请导航到产品首页的发布选项卡，并从发布管道框中选择编辑。

您也可以从操作下拉菜单中选择编辑发布管道。

这将打开发布管道配置窗口。

发布阶段指的是两个发布通道之间的转换。选择添加阶段以添加管道阶段，或点击阶段右侧的“×”来删除一个发布阶段。

对于每个发布阶段，您需要定义以下发布通道：

  • 源通道： Apollo 将从其发布该版本的发布通道。
  • 目标频道： Apollo 将发布内容推广到的发布频道。这可以是 Apollo 中创建的任何发布频道，但不能是默认的发布频道（RELEASE、RELEASE_CANDIDATE 和 DEV）。

有两种类型的推广可用：

  • 定时推广： 发布内容在定义的时间段后推广到目标频道。
  • 灰度推广： 发布内容根据选定的一组测试环境上评估的标准推广到目标频道。

除了满足每个阶段指定的推广要求外，发布内容只能在 产品的维护窗口 内进行推广。

对产品推广管道的更改将在下一次评估时生效，评估每分钟发生一次。

定时推广阶段

当您选择 定时推广 时，Apollo 将使用基于时间的推广评估。您可以为基于时间的推广定义以下参数：

  • 持续时间：发布版本必须在源发布通道上停留的时间，才能被提升到目标发布通道。持续时间可以设置为介于 0 秒 和 31 天 之间的任何值。
  • 产品发布标签要求：只有当发布版本满足标签要求时才进行提升。
    • 标签必须存在于具有指定值之一的发布版本上。

Canary 提升阶段

当你选择 Canary 提升，Apollo 将使用基于 Canary 的提升评估。在基于 Canary 的提升评估期间，Apollo 将使用配置的 实体过滤器 选择测试环境。它将根据你定义的 条件 在这些测试环境中对发布版本进行评估。当所有条件都满足时，发布版本将被提升到目标发布通道。

实体过滤器

实体过滤器 定义了 Apollo 将从中选择测试环境的实体池。可用的过滤器包括：

  • 带有标签的环境： 选择具有指定标签的环境上的实体。
    • 该标签必须存在于环境中。
    • 如果没有指定标签值，Apollo 将检查该标签是否存在于环境中。
    • 如果指定了标签值，Apollo 将检查该标签及其值是否存在于环境中。
  • 不带标签的环境： 选择没有指定标签的环境上的实体。
    • 该标签必须不存在于环境中。
    • 如果没有指定标签值，Apollo 将检查该标签是否不存在于环境中。
    • 如果指定了标签值，Apollo 将检查该标签及其值是否不存在于环境中。
  • 选择环境： 从一组特定的环境中选择实体。
  • 排除环境： 选择不在一组特定环境中的实体。

条件

条件 定义了在发布到目标发布通道之前，发布必须满足的要求。

  • 健康时进行发布： 一旦指定数量的实体至少经过了指定的时间，并且满足在升级选项卡中定义的发布滚动标准中的 健康阈值，就发布该版本。
    • 实体的数量可以定义为一个数量或所选实体的百分比。
    • 持续时间可以设置为介于 0秒 和 13天 之间的任何时间。
  • 不健康时召回： 如果指定数量的实体至少经过了指定的时间，并且满足在 升级 选项卡中定义的发布滚动标准中的 不健康阈值，则召回该版本。
    • 实体的数量可以定义为一个数量或所选实体的百分比。
    • 持续时间可以设置为介于 0秒 和 13天 之间的任何时间。
  • 最小实体数量： 仅当所选实体的数量高于指定数量时才进行发布。
  • 需要产品发布标签: 仅当发布满足该标签要求时才可进行发布。
    • 该标签必须存在于发布中。
    • 如果没有指定标签值，Apollo 将检查该标签是否存在于发布中。
    • 如果指定了标签值，Apollo 将检查该标签及其值是否存在于发布中。

编辑完发布管道后选择 确认。

默认发布管道

Hub 管理员可以在 Hub 层面配置默认发布管道，为新产品和模块提供一个合理的起点。

Apollo 为每个 Hub 维护两个默认发布管道：

  • 产品默认: 应用于新产品。可能包括 定时 和 灰度 发布阶段。
  • 模块默认: 应用于新模块。只可能包括 定时 发布阶段，因为模块发布管道不支持灰度阶段。

当新产品或模块的首个发布版本发布时，Apollo 会从相应的默认发布管道中填充产品或模块的发布管道。

对中心默认值的更改不会追溯生效。已经配置了发布管道的产品和模块——包括那些管道最初来自默认值的产品——在默认值更改时不受影响。要将默认值传播到现有产品或模块中空的管道，请参阅 将默认值应用到现有产品。

配置默认发布管道

要配置或编辑默认发布管道，请导航到 发行渠道与发布 选项卡中的 中心设置。选择 编辑 产品默认值或模块默认值。选择 确认 将生成一个更改请求。

将默认值应用到现有产品

枢纽管理员可以将默认的发布管道应用到枢纽中现有的产品或模块，而无需配置管道。此选项对于在默认管道存在之前创建了多个产品的枢纽非常有用。

要批量更新已配置管道的产品的发布管道，请使用 apollo-cli product promotion set 命令。
