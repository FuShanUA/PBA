apollo-cli entity aggregate [实验性]

该命令是 实验性 的。要 启用此命令，请运行 apollo-cli configure 命令并启用 V2 experimental commands。

将所有环境中的实体聚合到一个统一的实体列表中

从所有 Apollo 环境中获取实体数据，转换实体 ID，并合并到一个统一的实体列表中。

该命令执行以下操作：

  1. 列出 Apollo 空间中的所有环境
  2. 从每个环境中获取实体
  3. 将每个环境的 YAML 文件保存到输出目录
  4. 将实体 ID 转换为使用共同的目标环境
  5. 合并具有相同转换后的 ID 的实体，并合并版本
  6. 在所有合并的实体上设置指定的发布渠道
  7. 将统一的实体列表写入 all_entities.yaml

--generate-versions 标志会生成 product-versions.yaml 文件，其中包含每个产品的所有版本。 --generate-lowest 标志会生成 lowest-versions.yaml 文件，其中包含每个产品的最低版本。

示例用法：

使用 STABLE 发布渠道将实体聚合到输出目录

apollo-cli entity aggregate -r STABLE --output-dir ./aggregate-output

带版本报告的聚合

apollo-cli entity aggregate -r STABLE --output-dir ./aggregate-output --generate-versions --generate-lowest

Copied!
1apollo-cli entity aggregate [flags]

标志

标志用途
--generate-lowest为每个产品生成最低版本的 lowest-versions.yaml 文件
--generate-versions为每个产品生成包含所有版本的 product-versions.yaml 文件
-h, --help聚合帮助
--output-dir工件写入的目录
-r, --release-channel发布渠道

从父命令继承的标志

标志用途
--apollo-client-id用于生成持有者令牌的客户端ID
--apollo-client-secret用于生成持有者令牌的客户端密钥
--apollo-token用于认证端点的持有者令牌
--apollo-token-provider 指定用于认证Apollo端点调用的承载令牌（Bearer Token）的提供方式。有效值为"auto"、"static"、"service-user"或"sso"（默认为"auto"）。如果指定"auto"，则从配置中选择模式：如果设置了"apollo-token"，则为"static"；如果同时设置了"apollo-client-id"和"apollo-client-secret"，则为"service-user"；否则为"sso"（交互式浏览器登录）。如果同时设置了承载令牌和（客户端ID或客户端密钥），将报错。如果指定"static"，则使用"apollo-token"提供的令牌。如果指定"service-user"，则使用"apollo-client-id"和"apollo-client-secret"从Multipass生成令牌。如果指定"sso"，则使用针对Multipass的基于浏览器的SSO登录（支持硬件密钥如YubiKey）；生成的令牌会按配置文件缓存并静默刷新。第一次需要令牌且没有有效的缓存令牌时，将自动触发登录流程。
--apollo-urlApollo的基础URL，用于推导API端点
--debug启用调试级别日志记录
-e, --environment用于作用域为环境的命令的环境ID（包括后缀）
--http-timeout Apollo 请求的超时时间（分钟）
-k, --insecure-skip-verify跳过服务器证书验证
-o, --output输出格式（json,yaml,pretty）
--profile从配置文件中使用特定的配置文件
--quiet不要将日志输出到标准错误
--space-id用于某些空间作用域命令的空间 ID
