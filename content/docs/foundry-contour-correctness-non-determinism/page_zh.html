Contour 中的非确定性

非确定性窗口函数

在窗口函数中使用 ROW_NUMBER、FIRST、LAST、LEAD、LAG、NTILE、ARRAY_AGG 或 ARRAY_AGG_DISTINCT 时，需注意非确定性问题。假设我们按列 A 进行分区，并按列 B 进行排序。若列 A 的同一个值对应多行列 B 的相同值，这些窗口函数的结果可能是不确定的，即面对相同的数据和逻辑输入，可能产生不同的输出结果。

在表达式面板中使用这些表达式时，系统会弹出警告提示，以确保窗口函数中的 ORDER BY 子句具备确定性。

我们通过一组数据示例来具体说明：

nameclassgrade
AaronMath95
BurtMath95
ChrissyMath80
AngelicaScience77
BurtScience81
CharlieScience66

若要按成绩对每个班级的学生进行排名，可添加新列 rank，并使用表达式 ROW_NUMBER() OVER (PARTITION BY "class" ORDER BY "grade" DESC)。

得到的结果如下：

nameclassgraderank
AaronMath951
BurtMath952
ChrissyMath803
AngelicaScience772
BurtScience811
CharlieScience663

但有时，我们会得到如下结果：

nameclassgraderank
AaronMath952
BurtMath951
ChrissyMath803
AngelicaScience772
BurtScience811
CharlieScience663

由于 Aaron 和 Burt 的数学成绩相同，rank 列的结果是非确定的。为使该列结果具备确定性，可在表达式的排序子句中加入 "name" 列：ROW_NUMBER() OVER (PARTITION BY "class" ORDER BY "grade" DESC, "name" ASC)。通过此表达式，利用 name 列对成绩相同的行进行决胜排序，即可确保始终得到以下结果：

nameclassgraderank
AaronMath951
BurtMath952
ChrissyMath803
AngelicaScience772
BurtScience811
CharlieScience663

其他非确定性函数

除上述窗口函数外，CURRENT_DATE、CURRENT_TIMESTAMP、CURRENT_UNIX_TIMESTAMP 和 MONOTONICALLY_INCREASING_ID 函数同样具有非确定性。

对于 CURRENT_DATE、CURRENT_TIMESTAMP 和 CURRENT_UNIX_TIMESTAMP，这些值仅在路径更新时进行计算。例如，若在第 1 天使用 CURRENT_DATE 新建了一列，第 2 天回到该分析中时，该新列显示的仍是昨天的日期。

双精度列的聚合

由于 Spark 计算具有分布式特性，算术运算中操作数的顺序是非确定的（即 1+2 与 2+1 的区别）。当输入类型为 double 时，这种非确定的运算顺序会导致聚合操作产生非确定的输出。这意味着对于双精度浮点数的聚合运算，即便输入数据完全相同，每次计算的结果也可能存在差异，不过这种差异极小，例如 0.000001。

例如，对双精度列求解 mean 或 variance 将生成非确定性列。对非确定性列执行操作（如过滤）的结果同样也是非确定的。

在分析中对双精度列执行 mean、sum、stddev、variance、corr 或 sum_distinct 操作，都会产生非确定性列。

假设有一个双精度列 pickup_latitude。在数据透视表中，我们对双精度列 pickup_latitude 取均值。若切换至透视数据，就会生成一个非确定性列。

随后若对该新建列进行过滤，过滤结果也将是非确定的。例如，在上方截图中，Staten Island 的 pickup_latitude 均值为 40.5830495。如果按该值进行过滤，将看到仅剩一行数据。

然而，若重新计算该路径，过滤后可能不再显示该行，因为均值的微小变化改变了其数值。建议避免对非确定性列使用精确过滤（例如过滤均值 = 40.5830495）。同时，也应避免将非确定性列作为连接键使用。

在 Contour 中对非确定性列执行操作（例如过滤该列）时，执行操作的面板将弹出警告。该警告会指出引发非确定性列的具体聚合操作。

诊断非确定性

分析存在非确定性的一个信号是行数不一致。例如，假设你在一个分析中插入了汇总看板，随后执行了一系列不会改变行数的转换操作，接着又添加了另一个汇总看板。如果这两个汇总看板的行数不匹配，你就应该检查上方的路径中是否存在非确定性操作。注意留意界面中的警告提示，这些提示会在使用非确定性函数或对双精度浮点数进行聚合时发出警告。
