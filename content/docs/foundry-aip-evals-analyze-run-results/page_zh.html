分析运行结果

运行结果展示了您的函数在测试案例和评估标准上的表现。结果视图可在AIP Evals应用程序或AIP Logic和AIP Chatbot Studio中的集成AIP Evals侧边栏中查看。

如果您为评估器配置了< a href="/docs/foundry/aip-evals/create-suite/#add-an-evaluator">通过标准，AIP Evals将自动确定每个测试案例的通过或失败状态。结果页面显示所有测试案例的整体通过百分比。

测试案例调试视图

在某些情况下，您可能需要进一步调查特定测试案例的结果。在这种情况下，可以使用调试视图。此视图提供执行跟踪、输入/输出数据和错误信息，以便您可以了解您的函数输出和评估器结果。

访问调试视图

有多种方法可以为测试用例打开调试视图。你可以从 AIP Evals、AIP Logic 或 AIP Chatbot Studio 中进行操作。

在 AIP Evals 中

  1. 在你的评估套件页面上打开 结果 选项卡。
  2. 选择一个运行并切换到 测试用例。
  3. 将鼠标悬停在测试用例结果上。
  4. 选择测试用例行右侧出现的 打开 选项。

在 AIP Logic 或 AIP Chatbot Studio 中

  1. 在运行结果对话框视图中，将鼠标悬停在测试用例结果上。
  2. 选择测试用例卡片右上角出现的 调试器 选项。
  3. 调试视图将会打开，显示详细的执行信息。

调试视图功能

调试视图提供有关测试函数执行和评估器结果的详细信息。它允许您：

  • 检查测试用例的测试函数输入和输出。
  • TypeScript/Python 函数：访问已进行语法高亮的代码预览。
  • AIP Logic 函数：使用原生 Logic 调试器逐步跟踪函数执行。
  • 评估器：检查输入和输出值，预期与实际评估器结果，以及自定义函数评估器的调试输出。

自定义函数评估器可以返回字符串值以及它们的指标输出。这些字符串会以 调试输出 的形式出现在评估器标签中，提供额外的上下文，例如推理过程、中间值或诊断信息。

由AIP Logic支持的评估函数，如提供的评分标准评分器或包含关键细节的评估器，可以访问原生的Logic调试器。这有助于你了解为什么评估产生了特定结果，这在使用LLM作为评判者的评估器时尤其有帮助。

跨目标函数比较结果

当您的评估套件配置了多个目标函数时，您可以在 AIP Evals 中选择并比较不同目标的运行结果。这对于分析不同函数实现如何在相同测试用例上表现非常有用。

使用 AI FDE 分析结果

您可以使用 AI FDE 来分析失败的测试用例，识别根本原因模式，并获得改进提示的建议。

从 AIP Evals 分析

  1. 打开评估套件页面上的 结果 选项卡。
  2. 选择包含失败测试用例的单个运行，并选择 测试用例 选项卡。
  3. 选择 使用 AI FDE 进行分析。

从 AIP Logic 分析

  1. 从AIP Evals侧边栏打开运行结果对话框视图。
  2. 选择使用AI FDE分析。

AI FDE将在新标签页中打开，并显示您的运行结果上下文。有关AI FDE的更多信息，请参阅AI FDE文档。
