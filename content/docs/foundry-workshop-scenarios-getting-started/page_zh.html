快速入门

在本教程中，我们将逐步介绍如何构建一个基于场景的基础模块。

首先，让我们创建一个全新的 Workshop 模块。

首先，导航至左侧的设置面板，并确保在高级功能中启用了场景功能。

在侧边栏区域，让我们添加一个场景管理器小部件。

这是 Workshop 中专用于场景的小部件之一，用于创建和管理贯穿整个模块的场景。

暂时保留默认配置选项即可；我们稍后再进行详细设置。

在模块的主体区域，让我们添加一个对象表。

你可以使用任何对象集来填充表格，但我们建议从已配置至少一个关联操作的对象类型开始。

现在，让我们添加一些要在表格中显示的属性。

在属性正下方，你会看到在此小部件中启用场景对比的选项（也就是说，对象表是一个感知场景的小部件）。

启用后，你可以选择由场景管理器小部件生成的场景数组变量。这将使表格中的数据反映管理器中场景的所有修改，而非本体中的原始数据。

然而，此时我们尚未对场景应用任何修改，因此数据应该保持不变。

让我们添加一个按钮组小部件，以便配置应用于我们场景的操作。

我们正在选择一个操作，用于修改表格中对应类型的对象。

为了将此操作应用于场景而非真实本体，我们需要启用“应用于场景”选项，并从管理器中选择当前的场景变量。

使用刚刚配置好的操作，让我们尝试将表格中任意对象的属性修改为新值。

不过，在应用操作之前，我们需要通过点击“创建”按钮，在管理器小部件中创建一个新场景。

在本例中，我们正在更新表格中某位客户的余额。

你会看到对象表刷新并显示新数据，其中包含了你刚才所做的修改。

需要注意的是，此操作并未应用于本体，而是仅存在于场景之中。

我们还可以再次点击管理器中的创建按钮，创建另一个场景进行对比。创建完成后，你会在对象表中看到第二个场景的值与第一个场景的值并排显示，但仅限于存在差异的列。由于第二个场景尚未进行任何修改，它显示的应是本体中的值。

某些小部件（如表格）可以接收任意数量的场景并展示其结果。

现在，让我们在表格上方添加一个图表：XY小部件区域。

图表：XY 小部件与表格一样，支持任意数量的场景，并且可以在不同的图层中配置不同的场景。

尝试探索各种图层类型，看看多个场景是如何在其中可视化的。

你还可以配置分组和聚合，它们都会正确遵循场景的值。

我们也可以利用场景中的值来填充指标卡片。 现在让我们添加一个新的指标卡片小部件。

在指标卡配置中，我们正在创建一个新的数值指标，其值由一个新建的对象集聚合变量来定义。

在对象集聚合变量配置面板中，有一个 Scenario config 部分，用于接收一个 Scenario 变量。 若对其进行选择，对象集聚合操作将应用 Scenario 中的修改并执行。

同样地，对象属性变量也包含一个 Scenario 配置部分。 通过这种方式，您可以将来自 Scenario 的变量值配置到原本不支持 Scenario 感知的组件中（例如缺乏独立 Scenario 配置部分的指标卡）。

此处我们选择了 Scenario Selector 的输出变量，从而能够直观查看基于选择而产生的聚合数据变化。

恭喜您，首个 Scenario 教程已顺利结束！ 建议您尝试对本教程所涉及的所有组件进行各种配置与布局的探索。

尽管本教程所展示的布局是构建基于 Scenario 应用的常规方式，但这并非唯一途径。

借助现有工具，您可实现诸多强大的交互效果，特别是在结合 Actions 使用时；持续实践与尝试将带来更优异的成果。
