常见问题解答

如何为时间序列属性（TSP）设置单位和插值？

您可以通过格式化程序为每个 TSP 配置单位和插值。前往 Ontology Manager 中的时间序列对象类型，找到 Capabilities 选项卡下的 Time Series Properties 部分，即可找到该格式化程序。或者，您也可以在 Properties 选项卡中编辑单位和插值，操作方式与任何其他数值格式化程序类似。

什么是度量？应该何时使用它们？

在传感器对象类型开发出来之前，平台使用度量（Measures）作为时间序列模型。目前度量已弃用，您可以在类似的工作流中使用传感器对象类型。有关更多信息，请参阅以下页面：

  • 了解如何从度量迁移至传感器对象类型。

时间序列数据是否建立了索引？

在您的时间序列数据集上创建时间序列同步，会自动在该数据集上创建一个时间序列投影。时间序列投影是数据集的物化副本，能够提供类似于 SQL 数据库索引的优化效果。

在构建时间序列同步时，系统会生成有关其派生来源的时间序列数据集事务的元数据，以此告知 Foundry 的时间序列数据库有哪些可供索引的数据。

在使用时间序列时，您读取的是来自时间序列数据库的已索引数据。时间序列数据库的作用类似于缓存；数据仅在读取时进行水合（hydrate），一旦数据库磁盘空间受限，最近最少水合的序列将优先被驱逐。

什么是时间序列投影？

投影是数据集的物化副本，专门针对特定查询进行优化。对于时间序列而言，投影优化的对象是在读取时间序列数据集以将数据水合至时间序列数据库时所执行的查询。该过程会对时间序列数据集进行过滤，以选定正在进行水合的序列 ID 和时间范围。通过这种方式，投影能够维持良好的分区状态并对时间序列数据进行排序，从而有效地按序列 ID 和时间戳对时间序列数据集建立索引。

为什么我的时间序列加载失败？

如果错误提示“不存在时间序列数据”，可能是您的序列 ID 在时间序列数据集/同步与时间序列对象类型支持数据集之间的映射不正确。各个数据集中的序列 ID 集合应当存在交集；为了让时间序列属性能够正确引用时间序列数据，这些集合最好是完全相同的。此外，请务必检查时间序列对象类型上的序列 ID 属性是否已正确配置。

特别是在处理大规模时间序列时，将数据水合至时间序列数据库的操作可能会彻底失败。这可能是由于缺乏优化或优化失败所致：

  • 如果时间序列数据集上的时间序列投影已过时，系统将从规范数据集读取未投影的事务。这意味着未应用投影优化；数据分区将分散在大量文件中，并且需要扫描更多行才能水合所需数据。系统配置了内置限制来防止这种不受欢迎的访问行为，因为这极有可能导致查询超时并对服务健康状况产生负面影响。请检查时间序列投影的计划任务是否保持稳定且定期运行。您可以从数据集预览页面手动重建投影。
  • 如果时间序列数据集未正确分区和排序，在极端情况下可能会导致类似问题：需要扫描过多行才能索引到所需数据，进而触发内置的服务限制。为避免此类问题，在 Pipeline Builder 中转换数据并将其映射到时间序列同步输出时，系统会自动为您正确格式化时间序列数据集。您也可以通过更新时间序列投影或手动为流水线添加正确的格式化设置来解决此问题。

为什么我的时间序列加载时间过长？

时间序列数据加载缓慢的最常见原因是数据尚未在时间序列数据库中建立索引。索引预 hydrated（加载到内存）通常发生在首次查询特定时间序列（序列 ID）时，或在此后时间序列同步同步任何快照事务之后。同步的快照事务会通知时间序列数据库，将某个序列从完整数据集视图预加载至其索引中。如果查询的时间序列数据已从索引中被剔除，也可能会触发快照预加载；时间序列是根据磁盘空间需求被剔除的，最先被剔除的是最近最少预加载的序列。

为了提高预加载速度，可以在查询中添加时间过滤器，这样只会预加载指定时间范围内的数据点，而不是整个序列。您可以使用 Quiver 中的过滤时间序列卡片或 FoundryTS 中的 time_range 函数来创建时间过滤器。

时间序列完成初次预加载后，查询速度应该会大幅提升。如果您的流水线正在增量添加时间序列数据，那么时间序列数据库会对新数据进行增量预加载，在数据完成首次快照预加载后，您的时间序列应该会迅速完成加载。

我们建议运行增量流水线以提升后续的索引性能。

如果需要进行增量预加载的数据量很大，查询仍可能耗时较长。例如，当增量事务体量庞大，或者由于时间序列不常被查询导致大量增量事务尚未预加载时，加载时间就会增加。

在某些极端情况下，如果未对时间序列数据集应用时间序列重分区或排序，或者相对于数据量写入了过多的分区（因为这可能需要读取大量文件），那么快照和增量预加载都会变得缓慢。此情况仅适用于尚未被时间序列投影处理的事务。在 Pipeline Builder 中转换数据并将其映射到时间序列同步输出时，系统会自动为您正确格式化时间序列数据集。

为什么我的时间序列丢失了数据？

要让所有时间序列数据都在时间序列数据库中建立索引，其时间序列同步必须保持最新状态。这意味着同步操作必须在时间序列数据集构建最新事务之后进行构建，否则无法预加载这些事务中的数据。

如果您的时间序列数据集未以 Soho 格式存储，则任何未投影的数据都不会被预加载到时间序列数据库中。在 Pipeline Builder 中转换数据并将其映射到时间序列同步输出时，支持该同步的物化时间序列数据集会自动为您转换为 Soho 格式。您也可以完成以下任一任务，以获取更多更新后的数据：

  • 将您的时间序列数据集转换为 Soho 格式。这需要通过快照来转换所有数据。
  • 调度时间序列投影，使其在时间序列数据集每次更新时构建。这会为最新数据的查询带来一定延迟。
