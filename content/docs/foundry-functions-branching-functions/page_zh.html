分支函数

您可以在全局分支上开发、发布和使用函数。目前，这仅支持TypeScript v1 函数和AIP Logic 函数。

开发函数

要使用TypeScript v1的全局分支功能，请将您的仓库模板版本升级到0.903.0的functions-typescript子模板或更高版本。请查阅有关仓库升级的文档以获取说明。

您可以开发一个依赖于对全局分支上的资源所做的更改的函数，例如新创建或修改的本体论实体。

当您准备就绪时，使用版本目标发布您的函数 — 当全局分支合并时，发布到 main 的稳定版本。在您分支上的开发过程中，该函数会以不稳定的预览版本发布。

一旦您成功地在分支上发布了函数，就可以在 Workshop、AIP Logic 和基于函数的操作中使用该分支上的新函数版本。此函数版本带有 Branched pre-release 标签。请注意，您分支上发布的函数无法从其他分支（包括 main）访问。

目前尚无法在 TypeScript v1 仓库中依赖查询函数的分支版本。

在您继续在您的分支上开发时，您可以继续将函数版本发布到相同的版本目标。只要您的版本目标保持不变，任何使用您函数的资源都会自动拉取最新发布的版本。这使您可以在每次发布后无需手动更新函数引用的情况下快速迭代。

如果分支上的版本目标发生变化，您必须更新所有依赖于该分支上旧版本函数的依赖项，以使用新的版本目标。这也将显示为一个 检查。

冲突解决和重基函数

在分支上开发时，您不会自动接收到发布到 main 的新函数版本。这可以防止主开发干扰您的分支工作。如果在 main 上有可用的新版本，您将在函数版本选择器和本体论管理器中看到通知。然后您可以将您的函数版本重基，以从 main 中拉取所有新版本。

如果你的版本目标在你开发时被发布到 main，则必须在合并之前选择一个新的版本目标。一旦你这样做，就更新所有依赖项以使用新的版本目标。

合并

当合并你的全局分支时，修改后的函数将在 main 上使用稳定版本目标进行发布。使用你分支上发布的版本的资源将自动开始使用在 main 上发布的新的稳定版本作为合并的一部分。

Foundry 只会将你的版本目标合并到 main，而不会合并分支上发布的其他版本。在你的分支合并后，这些版本将不存在于 main 上。
