设置 Webhook

本教程将指导您如何设置 Webhook，以向外部系统发送 HTTP 请求。

前提条件

本教程将演示如何创建 REST API 数据源及关联的 Webhook。如果您已创建数据源，可直接跳至创建 Webhook步骤。

若您要连接的系统无法从 Foundry 的网络访问，建议使用带有智能体代理出口策略的 Foundry worker 数据源。若该系统只能通过 IP 地址而非主机名引用，请使用智能体代理运行时。

在智能体 worker 上使用数据源配置 Webhook 会增加配置和管理的复杂性。特别建议为运行在智能体 worker 上的 Webhook 专门分配一个独立的智能体。智能体按接收顺序执行任务；若在同一个智能体上同时执行批量同步和 Webhook，短耗时的 Webhook 任务可能会排在长耗时批量同步任务的队列后等待执行。

详细了解智能体 worker 及其配置。

确保您拥有一个项目，用于管理本教程中将要创建的数据源权限。若尚未创建项目，请查阅项目文档了解创建方法。

教程

本教程假设您将针对可通过开放互联网访问的系统使用 REST API。

部分数据源（包括 SAP、旧版 Salesforce 和旧版 magritte-rest-v2）使用更为复杂的任务配置。有关配置 Webhook 任务的更多信息，请参阅Webhook 参考文档。

创建数据源

首先导航至数据连接，选择数据源选项卡。接着选择新建数据源及REST API。

使用数据源编辑器填写您要连接的 REST API 的配置详情。有关 REST API 数据源类型的更多详情，请参阅数据源类型参考。

数据源旨在包含建立连接所需的最少密钥和连接信息集合。使用此数据源配置单个 Webhook 时，您将有机会添加额外的请求详情，包括相对路径、查询参数、标头和正文内容。

其他数据源类型

其他部分数据源类型也支持 Webhook。有关更多详情，请查阅特定数据源类型的参考页面。

创建 Webhook

数据源创建完成后，选择Webhooks选项卡，然后选择新建 Webhook。

按照新建 Webhook 向导中的步骤创建发起外部请求的 Webhook。下例展示了向示例域名的 /api/v1/createItem 端点发送 POST 请求的过程。正文构建为原始 JSON 格式，本例中配置为接受两个字符串输入参数。

复制 Webhook

若要创建与现有 Webhook 配置相似的新 Webhook（例如同一 API 上的两个相似端点），您可以选择复制现有 Webhook。要复制 Webhook，请从数据源概览的Webhooks选项卡中打开操作菜单，或使用 Webhook 页面上的操作按钮，然后选择复制。

配置选项

  1. 输入 Webhook 的名称，并根据需要添加描述。
  2. 可选地，开启函数配置以将 Webhook 发布为函数。为函数提供本体和 API 名称。详见《Webhook 函数参考》。
  3. 选择你在本教程上一步中创建的源。如果你是直接从源页面进入向导的，该源将被预选。
  4. 添加你希望在外部请求中使用的任何输入参数。
  5. 填写请求配置中的必填部分。你可以在路径、查询参数、标头和请求体配置部分引用输入参数。
  6. 可选地，设置你希望应用于 Webhook 的任何限制。有关详细信息，请参阅《限制参考》。
  7. 选择创建以完成 Webhook 的创建。

从 cURL 命令直接导入

另外，你也可以通过直接导入 cURL 命令来创建请求调用。为此，请选择从 cURL 导入选项，并将你的 cURL 命令粘贴到文本区中。如果你的 cURL 命令使用了受支持的选项且关联的域名已存在于源中，选择导入 cURL将创建一个新的调用。

测试 Webhook

保存后，你就可以运行测试请求，以检查配置是否正确。你可以随时通过测试连接侧边栏来执行此操作。

发出测试请求后，你可以使用响应来解析输出参数。有关输出参数的更多信息，请参阅《Webhooks 参考页面》。
