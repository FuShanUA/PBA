影响追踪

Linter 影响追踪界面可用于追踪已处理建议的进展情况及预计节省量。在 Linter 主页上选择影响追踪即可进入该页面。你可以使用筛选器，查看特定时间范围、规则和项目下的各项指标。

影响追踪会展示以下三项指标：

  • 建议数量：显示已解决建议的数量。用户首次从建议表中选中某条建议时，其状态将转为调查中，系统会同时对当前建议进行快照。用户未曾选中的建议将保持待处理状态，并在下一次扫描时消失。此类建议不会被追踪，也不会出现在影响追踪中。
  • 预计影响：显示 Linter 在处理建议前所预测的计算时缩减量。该估算值已归一化为三十天，与建议表中的每月预计节省数值一致。
  • 核实影响：显示处理建议前后每月预计使用量的差值。Linter 会提取修复前一周的使用量，减去修复后一周的使用量，并将差值归一化为三十天的使用量，从而得出核实后的估算值。需注意，修复前后那周的资源使用情况，未必能准确代表受该建议影响的实际资源用量。

核实过程通常需要七天时间，因此某项操作核实后的影响不会立即显示在影响追踪页面上。所有关于节省量的估算均基于 API 提供的可见性和访问权限计算得出。这意味着，由于规则之间存在共享资源，Linter 的各项估算值可能会出现资源重复计算的情况。例如，《计划可能未使用》规则可能会涵盖某个数据集，而该数据集同时也会被《增量追加数据集文件过多》规则提及。在汇总建议数值时，用户应将此因素考虑在内。

影响指标间的差异

预计影响和核实影响均是使用资源管理得出的估算值。两项指标在估算方法上的差异在于：如果在 Linter 提出建议后迅速采取操作，预计节省量将高于核实后的节省量。

举例来说，假设第一天生成了一条计划可能未使用建议：

  1. 在告警生成一天后（即第二天），你暂停了该计划。
  2. 核实影响估算会将修复后七天（正常使用量）的使用量，从修复前七天（六天正常使用量和一天高使用量）的使用量中扣除。
  3. 该差值随后会被归一化为三十天。
  4. 预计影响将远高于核实影响，因为预计影响是基于以下假设计算的：带着该问题运行一个月，然后再应用修复运行一个月所能产生的节省量。
