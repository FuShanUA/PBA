如何在导入和DirectQuery模式中优化连接器的使用？

连接器提供两种连接模式选项，这些选项在设置数据连接时指定。

在导入模式中，所选的表和列被导入到Power BI®中。当你创建或与可视化内容进行交互时，Power BI®会使用已导入的数据。要查看自初始导入或最近刷新以来的底层数据更改，必须手动刷新数据或通过计划刷新来更新。

  • 使用指南: 导入模式适用于希望基于小型至中型Foundry数据集快速响应并具有大量交互性的仪表板。除非您正在处理非常大的数据，而这些数据无法在仪表板消费层之前在Foundry中预筛选，否则导入模式通常更可取。
  • 使用限制 : 由于通过连接器进行数据传输的大小限制阈值，您可能无法导入大型数据集。有关更多详细信息，请参阅 是否对可以传输的数据量有限制？。

在 DirectQuery 模式 下，数据不会被导入或复制到 Power BI® 中。当您创建或与可视化内容进行交互时，Power BI® 会将 Foundry 作为底层数据源进行查询，因此您始终查看的是最新的数据，并且可以将数据转换和筛选推回到 Foundry 层。

  • 使用指南: 对于必须基于非常大的数据集构建的简单仪表板，建议使用 DirectQuery 模式，因为您可以利用 Foundry 的计算引擎动态查询并拉取结果。然而，性能比导入模式要慢，因为您将计算推回到了 Foundry 的分布式计算引擎。因此，我们建议将 DirectQuery 仪表板保持在相对较少的查询数量。
  • 使用限制 : 与导入模式类似，用户只能将最多的结果大小返回到 Power BI®。然而，只要返回的结果在限制范围内，他们可以在 Foundry 中处理更大的数据集。

更广泛的使用建议

我们建议尽可能在导入模式下工作。如果数据量较大，建议在导入前考虑是否可以在 Foundry 中减小数据量，例如通过过滤不必要的行、删除列或尽可能进行预聚合。这种 Foundry 预处理可以通过 Foundry 转换或 Contour 完成，并配置为自动构建。或者，您可以考虑将表导入分成较小的块，这些块在大小限制阈值内，例如按每个日历年度导入一个数据集。

在使用 DirectQuery 模式时，我们建议将仪表板保持为相对较少的查询和简单的交互性。这种方法在 DirectQuery 在处理非常大的数据集方面的强大功能与用户对性能的期望之间取得了平衡。

最后，当处理非常大的数据时，可以考虑使用 复合模型 ↗，以实现“两者兼得”的方法。在复合模型中，报告可以同时包含导入和DirectQuery数据连接。例如，您可以配置一个导入连接，使用预聚合或过滤后的概览和关键统计数据，这样用户打开报告时可以快速加载。在同一报告中，您还可以配置一个DirectQuery连接，以便于按需进行深入分析。

在Foundry和Power BI®之间如何进行访问控制？

通过Foundry Power BI®集成开发的报告将自动尊重报告开发人员在Foundry中配置的访问权限。例如，一位在Power BI® Desktop中构建报告的开发人员只能访问他在Foundry中通常有权访问的数据集或受限视图。

如果使用数据刷新功能发布报告，将使用Power BI®网关上配置的数据源凭据，而不是继承报告消费者的任何凭据。

有关在网关上管理数据源的更多信息，请参阅 管理您的数据源 - Microsoft Power BI® 文档中的导入/计划刷新 ↗。

如何在Power BI®服务中使用Foundry集成？

许多Palantir的客户使用 Power BI®服务 ↗ 在其组织内部更广泛地发布和分发报告。

为了在这些报告中利用Foundry集成，请使用安装了Foundry连接器和ODBC驱动程序的 Power BI®网关 ↗。在网关上使用Foundry连接类型创建连接时，请将 Authentication Method 设置为 Key 并提供一个 令牌 为Foundry中的服务帐户生成。然后网关将能够连接到Foundry，并且可以由Power BI®服务中托管的报表使用。

通过带有Foundry连接器的Power BI®网关访问Foundry时，令牌身份验证是唯一支持的身份验证方法。目前不支持Power BI®服务中的OAuth身份验证，因为此功能尚未提供给第三方开发的Power BI®连接器。如果您有需要OAuth身份验证的用例，请考虑在Power BI® Ideas ↗网站上提交反馈，并与您的Palantir和Microsoft代表分享反馈。

Foundry Power BI®集成与Power BI®网关的功能方式与您从Power BI®桌面中习惯的方式基本相同。请与您的Power BI®管理员合作，以确保已在网关上完成安装。

如何在 Power BI® 服务中使用 Foundry 集成来创建数据流？

Foundry Power BI® 集成与 Power BI® 数据流兼容，使用 Power Query 在线编辑器。

开始之前，请确保您已访问配置了与 Foundry 的现有连接的本地网关。由于 Power BI® 服务存在活动问题，目前在数据流设置过程中不支持创建新的网关连接。

要创建从 Foundry 提取数据的数据流，请按照以下步骤操作：

  1. 在您的 Power BI® 工作区中，选择 新建，然后从下拉菜单中选择 数据流。
  2. 选择 添加新表。
  3. 在 选择数据源 页面上，从连接器列表中选择 Palantir Foundry。
  4. 在 连接到数据源 页面上，指定一个与您本地数据网关上已配置的连接匹配的 基础 URL。例如，https://<subdomain>.palantirfoundry.com/. 可选地，提供一个 数据集 RID 和 分支。
  5. 输入基础 URL后，确保连接下拉菜单显示您本地网关的名称。如果没有出现匹配的网关，请检查基础 URL字段是否与网关连接中配置的URL完全匹配，包括任何结尾的斜杠。
  6. 选择下一步继续。
  7. 在选择数据页面上，选择所需的表格，然后选择转换数据。
  8. 使用Power Query编辑器创建并保存您的数据流。

"意外错误" 当为Dataflows创建新的网关连接时

在Dataflows的连接到数据源页面上，当尝试创建新的本地网关连接时，可能会收到以下错误信息：

Unexpected error (Session ID: <UUID>, Region: <REGION>)

有活跃的 Power BI® 服务问题，这会阻止在 Dataflows 设置过程中创建网关连接。为了解决这个问题，请在创建 Dataflow 之前，在网关上设置一个 Foundry 连接。在 Power BI® 服务中，您可以在 设置 > 管理连接和网关 > 连接 > 新建 中管理网关连接。

一旦您的网关上配置了可工作的连接，当您提供匹配的 基础 URL 时，Dataflows 的 连接到数据源 页面将自动选择您的网关。

如何通过 Power BI® 连接器配置额外的 ODBC 驱动程序设置？

您可以配置 Power BI® 连接器界面中未显示的 ODBC 驱动程序选项，例如代理设置。

当 ODBC 驱动程序安装后，它会自动创建一个名为 FoundrySql 的 Windows 系统 DSN . 此DSN上配置的任何其他设置将应用于Power BI®连接器（服务器名称或身份验证选项除外，这些已由Power BI®连接器指定）。此DSN可以在Windows ODBC数据源管理应用程序中正常配置。

什么是令牌，为什么需要令牌进行基于令牌的身份验证？

你可以将身份验证令牌视为你自己的私有密码，它授权 Power BI® 代表你访问 Foundry 数据。你可以在“账户 → 设置 → 令牌”页面下管理现有令牌并创建新令牌，如 生成令牌的文档 所述。

为什么 OAuth 身份验证不起作用？

如果你在通过 OAuth 进行身份验证时遇到错误消息，这可能意味着此功能尚未启用。你的错误消息可能与下面的截图类似。要使用此功能，Power BI® 必须作为第三方应用被启用。管理第三方应用 的安全文档详细说明了 Foundry 管理员如何启用此集成。

如果 OAuth 身份验证尚未启用，你可以继续通过替代的 基于令牌的身份验证选项 连接。

如何从基于令牌的认证切换为基于OAuth的认证？

如果您之前已通过Power BI®登录到Foundry，现在想主动切换到使用基于令牌的认证，您首先需要从Power BI®中清除现有的凭据。

您可以按照以下说明在无法连接下操作以清除现有凭据并重新进行身份验证。

无法连接

如果您收到“无法连接”消息，很可能您的身份验证凭据已过期。请按照以下说明清除现有凭据并配置新的数据源。

  1. 在Power BI®中，导航至 "文件" > "选项和设置" > "数据源设置"
  2. 在Palantir Foundry连接器上，选择“清除权限”。（如果你有多个Foundry连接，请清除所有连接。）
  3. 配置新的数据源并重新与Foundry进行身份验证。请注意，如果你使用基于令牌的身份验证，你需要生成一个新的令牌。

从Power BI®服务器下载后出现空白报表

可能是你的身份验证凭据存在问题，或者Power BI®可能想要询问你运行查询的权限。在这种情况下，Power BI®可能会静默失败，并显示一个没有“应用更改”选项的空白报表页面。在这种情况下，你应该点击“编辑查询”。它会给你提供继续操作的选项。如果不起作用，请联系你的Palantir代表并分享错误信息。

为什么在表格导航器中看不到我的表格？

通过“获取数据” → “Palantir Foundry”选择数据集时，您可能会注意到某些在Foundry中有访问权限的表格没有在文件夹导航器中显示。这是因为在您有表格访问权限但没有父文件夹结构访问权限的情况下会发生这种情况。

只要您有对表格的Foundry访问权限，您仍然可以将其导入Power BI®进行可视化。使用上一页上的可选对话框来指定“dataset RID”。RID是数据集的唯一标识符，即使您移动了数据集，它也是稳定的。您可以通过导航到所需数据集的“关于”页面，点击“查看更多”并复制RID值来找到正确的RID。

有关查找RID的更详细说明，请参见 指南：在Foundry中识别数据集的RID或文件路径

由于表大小导致的导入缓慢或导入失败

一般来说，表越大，导入处理时间越长。根据您的Power BI®订阅，“Import mode”（导入模式）可能被限制为1GB。Palantir Foundry的Power BI®连接器也对导入大小设置了限制 - 请参见 是否对可以传输的数据量有大小限制？ 了解更多信息。

如果由于表的大小导致您的表导入失败或速度缓慢，请查看 如何在 Import 和 DirectQuery 模式之间优化连接器的使用？ 以获取有关使用大型数据集与连接器的提示。

数据集的数据刷新不再有效

如果您通过 Power BI® 中的表导航器选择一个数据集，您的数据源将依赖于该数据集存在于当前位置。如果在 Foundry 中重命名或移动了数据集，这可能导致数据刷新问题，因为 Power BI® 将无法再找到该数据集。

如果您预计数据集需要移动，建议使用数据集 RID 配置数据源。有关通过 RID 配置数据源的更多细节，请参阅 为什么我在表导航器中看不到我的表？。
