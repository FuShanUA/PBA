生成并导出文档

如果希望他人基于您的模板或字符串变量生成新文档，请使用 Notepad：新建 Notepad 按钮 小部件。您还可以根据需要，选择允许将这些文档导出为 PDF。

Notepad：新建 Notepad 按钮小部件

Notepad：新建 Notepad 按钮 小部件允许您从模板或 Workshop 字符串变量创建并导出 Notepad 文档。

从模板生成文档

首先，将 Notepad：新建 Notepad 按钮 小部件添加到您的 Workshop 模块中。然后，在屏幕右侧的配置选项中，选择 模板 选项，接着点击 + 选择。在文件系统视图中，浏览并找到您的 Notepad 模板文件，将其添加到 文档模板 列表中。 接下来，选中新添加的模板条目以打开其配置，然后选择一个 模板版本。如果没有可用版本，请在新标签页中打开该模板并发布一个模板版本。选择版本后，系统会自动在 模板输入 区域加载该模板所需的输入参数。

您可以根据需要为模板指定 菜单显示名称。如果未设置显示名称，系统将使用 按钮显示 设置中的 自定义文本。

模板输入

根据模板及其版本的不同，您的 Notepad 模板可能会使用多个模板输入。对于其中的每一个输入，请链接一个具有相同数据类型（字符串、数字、日期、时间戳、对象或对象集）的 Workshop 变量，作为该模板的输入。只要选中 Notepad：新建 Notepad 按钮，系统就会使用这些变量的当前值，基于模板生成文档。

如果您的输入参数需要单个对象，请确保将其链接到仅包含一个对象的 Workshop 对象集变量。

从字符串变量创建文档

如果要从 Workshop 字符串变量（而非文档模板）创建 Notepad 文档，请从右侧面板中选择 字符串 配置选项。变量值可以解析为 Markdown 或纯文本，这符合我们现有的 Markdown 功能 规范。

保存选项

您可以为配置在 Notepad：新建 Notepad 按钮小部件中使用的不同文档模板和/或字符串变量设置以下选项：

  • 允许用户选择保存位置： 允许用户在生成文档之前自行选择保存位置和文件名。任何预先定义的 默认保存位置 都将被忽略。
  • 默认保存名称： 为新生成的文档定义默认文件名。当设置了 将生成的 Notepad 导出为 PDF 时，该值也将用作 PDF 文件名。
  • 默认保存位置： 选择新文档的保存位置。可以通过手动选择文件夹或通过 Workshop 字符串变量传入 Compass RID 进行配置。默认情况下，文档将保存在生成该文档的用户的主文件夹中。

点击选项

点击选项允许您修改使用 Notepad：新建 Notepad 按钮 时的某些行为：

  • 将生成的 Notepad 导出为 PDF：文档生成后将自动下载为 PDF 文件。默认情况下，生成的 Notepad 文档不会在 Foundry 中持久化保存，除非它是通过字符串变量创建的。因此，除默认保存名称外，所有其他保存选项均会被忽略。
    • 从 PDF 导出中保存 Notepad：设置此选项后，用于导出为 PDF 的已生成 Notepad 文档也将同步保存至 Compass 中。
  • 在新标签页中打开生成的 Notepad：文档将在新标签页中打开。

此外，你可以指定在文档创建时触发的 Workshop 事件或动作。例如，可通过 Notepad: New Notepad Button 中的 Created document 输出变量，将新生成文档的 RID 作为对象属性进行持久化保存。

在动作中使用生成的 Notepad RID

若要将生成的 RID 传递给动作，相应的参数必须配置类型类。为此，请在 Ontology Manager 中打开该动作参数，并在详情（Details）标签页中添加 notepad:notepad_rid 类型类。
