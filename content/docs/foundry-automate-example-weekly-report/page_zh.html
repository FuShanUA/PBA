示例：发送包含新建问题的每周报告

在本示例中，我们希望每周发送一份 Notepad 报告，其中包含上周所有新建支持工单的列表。如果上周没有新建工单，则不发送报告。我们将使用 Support Ticket 对象类型，这是为本示例创建的自定义对象类型。

触发条件

在自动化创建向导中，我们首先选择 Object added to set 条件。只需选择 Support Ticket 对象类型且不添加额外过滤器，创建新工单时就会触发自动化。

由于我们只需每周发送一次报告，因此需在条件中添加调度计划，并设定自动化在每周一上午 8 点触发。

由于我们在对象添加条件中加入了每周调度计划，自动化会在每周一检查上周是否新建了 Support Ticket 对象。

执行动作

条件满足后，我们希望发送一封带有 PDF 附件的邮件，附件报告需包含上周创建的所有支持工单。为此，我们首先添加一个通知动作。

由于 PDF 报告需要涵盖上周创建的所有支持工单，请保持默认的对象分组选项 Send one notification for all Support Tickets added。在收件人处，添加一个包含目标用户的用户组。如果只想通知被分配了新支持工单的用户，则仅选择由对象属性支持的用户即可。

由于具体信息已包含在附加的 PDF 中，我们无需构建复杂的通知正文：直接使用 Plain notification 类型在界面中配置内容即可。我们还可以在通知中提供指向 Quiver 仪表盘的链接。

将 Notepad 文档附加到通知中

为配置通知附件，我们将使用 Notepad 模板动态生成 Notepad 文档，并将该文档的 PDF 版本附加到通知中。

首先，我们需要创建一个能展示所需信息的 Notepad 模板。我们可以按照 Notepad 文档中概述的步骤进行操作。针对当前场景，我们创建一个以 Support Ticket 对象集作为模板输入的 Notepad 模板。利用该模板输入，我们展示一张图表，并渲染一个包含对象集中各个对象标题和描述的表格。

保存并发布 Notepad 模板后，我们便可在通知动作中使用它。首先，将其选为附件并定义模板版本。然后，如下图所示，将 Notepad 模板公开的 support tickets 模板输入与 New Support Tickets 条件动作输入连接起来。

这样一来，Notepad 模板将以新的支持工单作为输入执行，生成的 Notepad 将作为 PDF 附加到通知中。

摘要

在摘要界面，我们可以确认该自动化将在每周一上午 8 点为所有新支持工单发送一次 Notepad 模板。
