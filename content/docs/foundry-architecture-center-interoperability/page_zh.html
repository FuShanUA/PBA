互操作性

Palantir AIP 和 Foundry 设计用于与各种数据、逻辑、AI、工作流和安全系统进行互操作。

这包括涵盖传统数据、分析、治理和运营领域的工具和技术，包括边缘设备和恶劣环境。

消除集成平台中常见的传统权衡，Palantir 架构的目标是提供连贯且完整的体验，同时支持深度连接现有（或未来）企业软件平台所需的模块化能力。

数据互操作性

Palantir 平台基于开放数据标准构建。所有数据均以原始格式（如 CSV、Iceberg 或 Parquet）存储，并可通过标准接口（如 REST、JDBC 和 S3 兼容访问）进行访问。此外，默认情况下，所有转换后的数据均可在开放格式中访问，例如 Apache Iceberg 和 Apache Parquet。这使得与现有数据平台、记录系统和其他企业数据架构中的服务实现深度连接。

除了原生功能之外，多模态数据平面（MMDP）能够与现有的企业资产实现前所未有的集成。这包括用于利用常见数据平台（如 Databricks、Snowflake 或 BigQuery）中现有数据资产的虚拟表框架，而无需不必要的数据复制。MMDP 还包含完全协调的下推计算，因此像 Pipeline Builder 这样的应用程序可以与现有的计算投资无缝配合使用。

元数据互操作性

Palantir Foundry 和 AIP 提供丰富的元数据集成功能，支持大量强制性元数据（如安全、归属或血缘）以及可选元数据（如标签或增强信息）。元数据服务安全地公开项目、数据集、本体论元素、代理、模型、分析、应用、管道编排、资源健康状况等所有存在的元数据属性。这使得可以与现有的数据目录、元数据管理工具、主数据管理工具和其他服务在现有治理架构中实现深度连接。

语义互操作性

Palantir 本体论超越了传统的语义定义，还包括驱动复杂操作、代理和人工智能驱动自动化的对象、关联、操作和函数的细粒度定义。组织中的本体论所有元素都可以通过 REST API 访问，并通过 JSON 驱动的创作范式进行配置。这使得与现有语义建模工具、数据目录中的本体论以及领域特定建模工具之间能够双向同步。

代码与逻辑互操作性

Palantir 对开放软件标准的承诺适用于数据工程、数据科学以及所有其他代码驱动的创作范式。默认情况下，所有数据转换都使用开放语言（如 Python、Java 或 SparkSQL），这些语言与平台内置的开放运行时（如 Spark、Flink、DataFusion 或 Polars）有绑定关系。此外，所有数据科学工作流都利用开放语言（如 Python 或 R），这些语言也利用相同的开放运行时，并且设计为利用常见的开放格式（如 ONNX）。代码仓库存储在高度可用的 Git 服务中，可以通过 UI 驱动的导出和 API（程序化）交互安全访问。

除了打包的计算运行时和相关语言，Compute Modules 框架使团队能够引入自己的容器化运行时、应用程序、模型和各种可执行文件。这些容器由 Palantir 的底层计算基础设施（Rubix）安全地编排和管理，并可以在数据管道、应用构建、分析和 AI 驱动的工作流的整个范围内被强大地呈现。

分析互操作性

Palantir Foundry 和 AIP 提供一系列完整的分析工具，以赋能用户，并且可以与现有的投资（如 BI 和数据科学工具）无缝互操作。对于 Power BI®、Tableau、Jupyter 和 RStudio® 等常见系统，提供了开箱即用的连接器。这些连接器使广泛的用户能够访问集成数据，同时利用一流的的数据管理、模型管理和治理功能。

除了数据连接器之外，代码工作区 还提供在平台内部原生使用 Jupyter® 和 RStudio® 的无缝体验。

安全互操作性

该平台在所有资源上提供强大且透明的控制。安全服务设计用于利用现有的身份验证系统（例如，通过SAML）进行身份识别，并利用现有的授权系统（如Active Directory）进行权限管理，这些权限可以跨越基于角色、分类和用途的制度。通过本体论SDK，可以灵活地扩展和管理第三方和自定义应用程序开发的权限。通过平台的REST API，可以动态和回顾性地访问所有安全信息。
