多模态数据平面（MMDP）

多模态数据平面（MMDP）是 Palantir 的开放数据和计算架构。MMDP 反映了过去二十年在客户任务前线打造经过实战检验的解决方案的经验，以及在分析系统与操作型系统设计之间常常产生的设计权衡。

具体来说，大多数平台架构都围绕特定的计算运行时或数据存储模式构建。虽然这可以适用于狭窄的用例，但当扩展到需要复杂交互的互联用例时，这些依赖性可能会迅速成为障碍。例如，结构化、非结构化、时间序列、地理空间或几何数据等不同类型的数据必须通过不同的计算形式进行转换和处理，然后才能在无缝的用户体验中整体提供。

在人工智能时代，超越碎片化的数据和计算已不再只是一个“加分项”，而是实现企业自主性的关键。

任何数据：MMDP 的开放数据架构

MMDP 的基础是一个开放的数据架构。这始于对 Apache Iceberg 的承诺，作为 Foundry 和 AIP 的主要表格式。

Iceberg 正在被关键 Palantir 合作伙伴广泛采用，包括 AWS、Google Cloud、Microsoft Azure、Databricks 和 Snowflake。

MMDP 允许在 Palantir 内管理 Iceberg 目录，或者作为从这些（或其他）提供者注册的虚拟目录和虚拟表。这意味着在本体论中利用数据进行操作应用程序和 AI 驱动的自动化时，永远不需要复制数据。

采用 Iceberg 还意味着标准工具如 SQL 应用程序和分析笔记本可以安全地与 Foundry 和 AIP 中的数据进行交互和操作，就像在任何其他环境中一样。

许多组织正在构建一个“数据网格”（或现在称为“AI网格”），其中Palantir是更广泛、更多样化的企业架构中的一个参与者。MMDP的明确目标是通过提供一个“无围墙的花园”，支持此类混合架构，从而实现端到端的结果交付，同时确保架构的每一层都可以与现有的数据湖、湖仓、传统仓库、数据治理工具和其他关键存储仓库进行深度集成。

MMDP：超越表格数据的开放性

除了表格数据，MMDP还将相同的开放性保证扩展到媒体、文档、流数据、地理空间数据以及各种多模态数据类型。

媒体数据可以以最小的假设与底层格式同步；在存在已知格式的地方，分析工具可以立即帮助进行交互式解析；而在集成自定义或特定格式时，可以从其他来源延迟附加元数据。

流式数据和时空数据也是如此，它们可能具有关联的元数据（例如，传感器标签），需要在与主入口管道并行的数据管道中进行处理。

无论数据如何被摄取和转换，所有底层文件和点都可以通过标准REST API以及Python和TypeScript SDK安全访问。

导出作业和基于本体论的Webhook提供了将任何数据（无论其形式如何）同步到外部系统的简单且安全的方法，而基于源的转换范式使开发人员能够利用Foundry工具链的全部功能，根据需要自定义数据出口，粒度为单个工作流步骤。

任意计算：MMDP的开放计算架构

MMDP 的开放计算架构释放了开放数据架构的价值。Foundry 和 AIP 都配备了多种开箱即用的运行时环境，所有这些运行时都使用经过强化、自动扩展的基于 Kubernetes 的计算网格（Palantir Rubix）。

Rubix 基于零信任原则运行，并在其管理的每个运行时和服务中实施严格的网络安全策略；例如，为了防范高级持续性威胁，每个容器都会在 72 小时内被销毁并重新创建。这要求依赖的计算基础设施具有高度可用性，并且通常能够抵御单个节点故障的影响。

不同角色和职能的用户，无论是技术数据工程师、分析型数据科学家还是操作型业务用户，都可以利用平台内的批处理、流式处理和交互式计算引擎（以及通过 API 和 SDK）进行计算。这包括用于批处理计算的自动扩展 Spark；用于流式处理计算的自动扩展 Flink；以及一系列开源的高性能单节点引擎，如 Apache DataFusion、Polars 和 DuckDB。

在“本体论南侧”（从原始数据到本体论数据）进行管理的数据转换，以及在“本体论北侧”（从本体论数据到最终用户）使用的交互式功能，都可以以可靠且受控的方式利用这些运行时和其他运行时。

MMDP：使用任何计算构建

计算模块框架支持“自带计算”（BYO Compute），其中任何容器化资源都可以被导入并通过批处理、流式和交互功能安全地展示。

在操作工作流程中，最重要的可执行文件和模型通常被困在特定领域的工件或不再积极开发的旧包中。计算模块使这些碎片化的计算工件可以被安全释放，然后在AI驱动的工作流程中托管并使用，同时与更现代的逻辑工件（如Python、Java、SQL、Go或Rust程序）一起使用。

Foundry 和 AIP 还可以与位于 Palantir 环境之外的计算资源进行协调，包括现有的模型推理基础设施、Spark 集群、云托管的优化引擎以及在本地运行的高性能计算资源。

MMDP 还支持将计算能力原生且无缝地下推到云原生运行时（如 Databricks 或 Snowflake），使开发人员可以使用 Pipeline Builder 和 Code Workspaces 等工具，结合其现有的计算资源。

任意模型：Palantir 对模型访问的承诺

Palantir 对开放性和选择性的承诺也适用于生成式 AI。MMDP 的“任意模型”理念体现了持续致力于通过 AIP 的 Model Catalog（包括来自 OpenAI、Anthropic、Google、Meta 和 xAI 的模型）提供最新的生成式 AI 模型，并为企业提供一个公平的竞争环境，使其能够注册并同等使用自己的模型。

无论是 Palantir 提供的还是自定义注册的 LLM 和多模态模型，都可以在 Foundry 和 AIP 应用程序中无缝使用，包括 Pipeline Builder、Workshop、AIP Logic 和开发人员工具链。

为了便于管理，可以精确地控制对模型的访问，并在所有使用场景和工作流程中设置令牌限制。同时，资源管理功能在整个通过MMDP连接的数据、计算和AI模型中一致地扩展，无论具体的底层模态、运行时或格式如何。

无处不在：MMDP对开放性的承诺

MMDP通过抵制由单体存储和计算架构提供的简单答案，弥合了分析世界与操作世界之间的差距。

开放数据架构利用Apache Iceberg等开放标准，同时为媒体、文档、流和多模态数据类型提供平等支持。

开放计算架构包含常用运行时，允许团队带来自己的计算资源，并为企业内部现有的计算基础设施提供丰富的接口进行协调。

这种开放性也延伸到了生成式AI，通过模型目录可以访问各种各样的大语言模型（LLM），并且可以轻松注册自定义、微调过的以及现有的企业级模型。

而借助Palantir Apollo，所有这些灵活性以及在各个维度上的能力，都与底层基础设施提供商无关，并且能够持续演进以应对前线最严格的要求。

MMDP的理念是：“任何数据、任何计算、任何模型，无处不在。”
