转换与管道

目前，所有转换默认使用 SNAPSHOT 事务类型。

Transform 描述了如何计算数据集，具体包含以下内容：

  • 输入和输出数据集；
  • 将输入数据集转换为输出数据集所用的代码（我们将其称为计算函数）；
  • 任何额外的配置（例如运行时要使用的自定义 Transforms 配置文件）。

输入和输出数据集以及转换代码都在 Transform 对象中指定，然后注册到 Pipeline 中。定义 Transform 的方式取决于以下两个因素：

  • 你定义的是高级还是低级 Transform；
  • 你是使用自动注册还是手动注册方式将 Transform 添加到项目的 Pipeline 中。

Transform 类型

提示

数据转换可以通过 DataFrame 对象或文件来表达。这些 DataFrame 对象仅指常规的 Spark DataFrame。在 Spark Scala/Java API 中，DataFrame 由 Dataset 表示。因此，作为用户，你会在数据转换代码中直接与 Dataset 对象交互。
有关使用 Spark 的更多信息，你可以查阅在线提供的《Java API for Spark 文档》↗。

对于依赖 DataFrame 对象的转换，你可以：

  • 定义高级 Transform，支持 Dataset<Row> 类型的输入和输出；或者
  • 定义低级 Transform，并显式调用方法来访问包含输入数据集的 Dataset<Row>。

对于依赖文件的转换，你必须定义低级 Transform，然后访问数据集中的文件。

以下是两种 Transform 类型的主要区别总结：

描述高级 Transform低级 Transform
允许依赖 DataFrame 对象进行数据转换✓ *✓
允许依赖文件访问进行数据转换✓
支持多个输入数据集✓✓
支持多个输出数据集✓
计算函数必须返回 DataFrame 值✓
计算函数写入输出，而非返回值✓

* 对于依赖 DataFrame 对象的数据转换，我们推荐使用高级 Transform。

对于这两种 Transform 类型，你都需要创建一个包含计算函数的类。在这个类中，你的计算函数必须是一个带有 @Compute 注解的公共非静态方法。如果没有这个注解，你的数据转换代码将无法正确注册。

注册类型

仓库中的每个 Transforms Java 子项目都会暴露一个单独的 Pipeline 对象。这个 Pipeline 对象用于：

  1. 在 Foundry 中注册数据集，并附带如何构建这些数据集的指令；
  2. 在 Foundry 构建期间，定位并执行负责构建给定数据集的 Transform 对象。

入口点

负责执行 Java 转换的运行时需要能够找到项目的 Pipeline。请注意，Transforms Java 使用标准的 Java 服务加载机制↗。

向流水线添加转换

一旦与项目流水线关联的 Transform 声明某个数据集作为输出，你就可以在 Foundry 中构建该数据集。将 Transform 对象添加到 Pipeline 的两种推荐方式是手动注册和自动注册。

提示

如果你拥有更高级的工作流，并且/或者希望将每个 Transform 对象显式添加到项目的流水线中，可以使用手动注册。例如，如果你想通过元编程将相同的数据转换逻辑应用于多个输入和输出数据集组合，使用手动注册会非常有用。

否则，强烈建议使用自动注册，以确保注册代码简洁且内聚。采用自动注册时，Pipeline.autoBindFromPackage() 会发现包中的所有 Transform 定义（前提是这些对象包含必需的 @Input 和 @Output 注解）。

自动注册

随着项目复杂度增加，手动向 Pipeline 添加 Transform 对象会变得难以维护。因此，Pipeline 对象提供了 autoBindFromPackage() 方法，用于发现 Java 包中的所有 Transform 对象。要使用自动注册，你必须执行以下操作：

  • 定义与 Transform 对应的类。在自动注册中，你定义的类需包含有关输入和输出数据集的信息以及计算函数。
  • 添加足够的 @Input 和 @Output 注解。
  • 调用 Pipeline.autoBindFromPackage() 方法，以注册你提供的 Java 包中的任何 Transform 定义。autoBindFromPackage() 方法仅注册包含必需注解的 Transform 定义。任何不包含必需注解的 Transform 都不会添加到项目的 Pipeline 中，即使这些 Transform 位于你提供给 autoBindFromPackage() 方法的 Java 包内。

手动注册

可以使用 Pipeline.register() 方法手动将 Transform 对象添加到 Pipeline 中。每次调用该方法可注册一个 Transform。要在 Transform 中使用手动注册，你必须执行以下操作：

  • 定义一个类，其中包含 Transform 对象的计算函数。与自动注册不同，在手动注册中，你需要在 PipelineDefiner 实现中提供有关输入和输出数据集的信息
  • 使用 HighLevelTransform.builder() 或 LowLevelTransform.builder() 指定要使用的计算函数，并提供输入和输出数据集。
  • 调用 Pipeline.register() 方法，将你的 Transform 定义显式添加到项目的流水线中。
警告

请注意，@StopProgagating 和 @StopRequiring 等注解仅支持自动注册的 Java 转换。

转换上下文

数据转换过程有时会依赖于输入数据集之外的其他要素。例如，转换逻辑可能需要获取当前的 Spark 会话，或者读取 jobSpec 中的转换参数。针对此类情况，可以将 TransformContext 对象注入到转换流程中。具体操作上，计算函数需接收一个 TransformContext 类型的参数。TransformContext 内部包含了转换模块的 authHeader、Spark 会话、转换参数以及一个 ServiceDiscovery 对象。其中，ServiceDiscovery 类对外暴露了已发现 Foundry 服务的服务 URI。
