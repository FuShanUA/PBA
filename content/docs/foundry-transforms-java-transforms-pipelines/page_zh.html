转换和管道

所有转换目前默认为事务类型 SNAPSHOT。

一个 Transform 是如何计算数据集的描述。它描述了以下内容：

  • 输入和输出数据集，
  • 用于将输入数据集转换为输出数据集的代码（我们将此称为计算函数），以及
  • 任何额外配置（例如在运行时要使用的自定义转换配置文件）。

输入和输出数据集以及转换代码是在一个 Transform 对象中指定的，然后注册到一个 Pipeline 中。你可以根据两个因素来定义一个 Transform：

  • 你是否正在定义 高级或低级转换，以及
  • 你是否使用自动或手动注册来 将你的转换添加到项目的管道中。

转换类型

提示

数据转换也可以用DataFrame对象以及文件来表示。这些DataFrame对象只是普通的Spark DataFrame。在Spark Scala/Java API中，一个DataFrame由一个Dataset表示。因此，作为用户，您在数据转换代码中直接与Dataset对象交互。
有关使用Spark的更多信息，您可以参考在线的 Spark文档的Java API ↗。

对于依赖DataFrame对象的转换，您可以：

  • 定义一个高级转换（Transform），它支持类型为Dataset<Row>的输入和输出，或者
  • 定义一个低级转换，并显式调用方法以访问包含您的输入数据集的Dataset<Row>。

对于依赖文件的转换，您必须定义一个低级转换，然后在您的数据集中访问文件。

以下是两种转换类型之间关键差异的总结：

描述高级转换（High-Level Transform）低级转换（Low-Level Transform）
允许依赖 DataFrame 对象的数据转换✓ *✓
允许依赖文件访问的数据转换✓
支持多个输入数据集（Dataset）✓✓
支持多个输出数据集（Dataset）✓
计算函数必须返回 DataFrame 值✓
计算函数将数据写入输出，而不是返回值✓

* 我们建议对于依赖 DataFrame 对象的数据转换，使用高级转换（High-Level Transform）。

对于 Transform 类型，您需要创建一个包含计算函数的类。在该类中，您的计算函数必须是一个公共的、非静态的方法，并且用 @Compute 进行注解。如果没有此注解，您的数据转换代码将无法正确注册。

注册类型

仓库中的每个 Transforms Java 子项目都会暴露一个单一的 Pipeline 对象（对象）。此 Pipeline 对象用于：

  1. 在 Foundry 中注册数据集，并提供构建它们的说明，以及
  2. 在 Foundry 构建期间定位并执行负责构建给定数据集的 Transform 对象（对象）。

入口点

负责执行 Java 转换的运行时需要能够找到项目的 Pipeline。请注意，Transforms Java 使用 标准 Java 服务加载设施 ↗。

为了定义与您的项目相关联的 Pipeline 对象（对象），您必须实现一个 PipelineDefiner 对象（对象）。在这个 PipelineDefiner 对象（对象）中，您可以将 Transforms 添加到项目的 Pipeline（管道）中。具体来说，每个 Java 子项目必须实现一个单独的 PipelineDefiner 对象（对象）： 

Copied!
1package myproject;
2
3import com.palantir.transforms.lang.java.api.Pipeline;
4import com.palantir.transforms.lang.java.api.PipelineDefiner;
5
6public final class MyPipelineDefiner implements PipelineDefiner {
7
8  @Override
9  public void define(Pipeline pipeline) {
10    // Code here to add Transforms to your project's Pipeline using
11    // automatic or manual registration.
12  }
13}

一旦你创建了 Java 包并实现了 PipelineDefiner 对象，你必须更新 resources/META-INF/services/com.palantir.transforms.lang.java.api.PipelineDefiner 以指向你的 PipelineDefiner 实现：

2// Since each Java subproject implements a single "PipelineDefiner", this file
3// can only contain a single entry.
4myproject.MyPipelineDefiner

MyPipelineDefiner 指的是你 PipelineDefiner 实现的类名。

向管道中添加转换

一旦与你项目中的管道关联的 Transform 声明了一个数据集作为输出，你就可以在 Foundry 中构建这个数据集。向 Pipeline 添加 Transform 对象的两种推荐方式是 手动注册 和 自动注册。

提示

如果你有一个更复杂的流程和/或想要显式地将每个 Transform 对象添加到你项目的管道中，你可以使用手动注册。例如，如果你希望对多个输入和输出数据集组合应用相同的数据转换逻辑，手动注册就很有用。

否则，强烈建议使用自动注册，以确保您的注册代码简洁且集中。通过自动注册，Pipeline.autoBindFromPackage() 会发现包中的任何 Transform 定义（前提是这些对象具有所需的 @Input 和 @Output 注释）。

自动注册

随着项目复杂性的增加，手动将 Transform 对象添加到 Pipeline 可能变得难以管理。因此，Pipeline 对象提供了 autoBindFromPackage() 方法，用于发现 Java 包内的所有 Transform 对象。要使用自动注册，您需要执行以下操作：

  • 定义一个与您的 Transform 相对应的类。通过自动注册，您定义一个包含有关输入和输出数据集以及计算函数的信息的类。
  • 添加足够的 @Input 和 @Output 注释。
  • 调用 Pipeline.autoBindFromPackage() 方法以在您提供的 Java 包中注册任何 Transform 定义。autoBindFromPackage() 方法只会注册具有所需注解的 Transform 定义。任何不具有所需注解的 Transforms 将不会被添加到您项目的 Pipeline 中，即使这些 Transforms 在您提供给 autoBindFromPackage() 方法的 Java 包中。

手动注册

Transform 对象可以使用 Pipeline 方法手动添加到 Pipeline.register()。每次调用此方法可以注册一个 Transform。要使用 Transform 的手动注册，您必须执行以下操作：

  • 定义一个包含您的 Transform 对象的计算函数的类。与自动注册不同，手动注册时，您需要在 PipelineDefiner 实现中提供有关输入和输出数据集的信息
  • 使用 HighLevelTransform.builder() 或 LowLevelTransform.builder() 来指定要使用的计算函数以及提供您的输入和输出数据集。
  • 调用 Pipeline.register() 方法，将您的 Transform 定义显式添加到项目中的管道（Pipeline）。
警告

请注意，像@StopProgagating和@StopRequiring这样的注释仅支持自动注册的Java转换。

转换上下文

有时数据转换可能依赖于除了输入数据集之外的其他因素。例如，转换可能需要访问当前的Spark会话或在jobSpec中访问转换参数。在这种情况下，您可以将一个TransformContext对象注入到转换中。要做到这一点，您的计算函数必须接受一个类型为TransformContext的参数。TransformContext包含Transforms的authHeader、Spark会话、转换参数和一个ServiceDiscovery对象。ServiceDiscovery类公开了发现的Foundry服务的服务URI。

Copied!
1package myproject.datasets;
2
3import com.palantir.transforms.lang.java.api.Compute;
4import com.palantir.transforms.lang.java.api.Input;
5import com.palantir.transforms.lang.java.api.Output;
6import com.palantir.transforms.lang.java.api.TransformContext;
7import org.apache.spark.sql.Dataset;
8import org.apache.spark.sql.Row;
9
10/**
11* This is an example high-level Transform that accesses the TransformContext
12*/
13@Compute
14public Dataset<Row> myComputeFunction(Dataset<Row> myInput, TransformContext context) {
15    int limit = (int) context.parameters().get("limit");
16    return myInput.limit(limit);
17}
