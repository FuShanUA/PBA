平台变更

平台变更是指当平台组件（如后端服务或前端应用程序）被升级或其配置被更改时发生的变更。升级助手会跟踪平台变更，并提供一份< a href="/docs/foundry/upgrade-assistant/impacted-resources/">受影响资源列表，这些资源可能受到即将进行的变更的影响。

平台变更的示例包括：

  • 端点的修改，例如语义、URL、有效负载格式的更改，或完全弃用；
  • 库的修改，例如语义、函数名称或参数的更改，或完全弃用；
  • 重要的错误修复，例如平台安全修复或可能改变作业结果的Spark修复。

这些平台变更可能需要特别关注，因为：

  • 该变更不是向后兼容的（这在端点或库的修改中有时会发生）。
    • 如果不采取适当的步骤，在变更生效前可能会导致平台或工作流问题。
  • Foundry -> Foundry Apollo -> Apollo Gotham -> Gotham AIP -> AIP Ontology -> 本体论 ontology -> 本体论 Pipeline -> 管道 pipeline -> 管道 Transform -> 转换 transform -> 转换 Workshop -> Workshop workshop -> Workshop Code Repositories -> 代码仓库 Code Workspaces -> 代码工作区 Data Integration -> 数据集成 Model Integration -> 模型集成 Object -> 对象 object -> 对象 Dataset -> 数据集 dataset -> 数据集 Tenant -> 租户 tenant -> 租户 Marking -> 标记 marking -> 标记 Permission -> 权限 permission -> 权限 Role -> 角色 role -> 角色 Function -> 函数 function -> 函数 Action -> 操作 action -> 操作 Property -> 属性 property -> 属性 Interface -> 接口 interface -> 接口 内容: 更改可能会影响工作流程、管道或分析结果（例如，随着Spark正确性修复的出现）。
    • 用户应审查可能影响其工作结果的更改的影响。

截止日期

在升级助手中宣布的每个平台更改都有一个截止日期，这是更改生效的预定日期。随着截止日期的临近，升级助手将向被分配了资源的用户发送提醒。

由于部署限制和日程安排，更改可能在其截止日期之后应用。然而，截止日期仍应被视为完成更改准备工作的最终时间。
