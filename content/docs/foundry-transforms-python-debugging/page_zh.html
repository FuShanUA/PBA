调试 Python 转换

本指南概述了可用于调试 Python 转换的技术。有关错误和异常的更多信息，请参阅 Python 文档 ↗。

使用调试器

调试器是识别和解决 Python 转换中问题的有用工具。您可以设置断点来暂停转换执行，检查变量、查看 DataFrame 并了解函数和库。

阅读 Python 跟踪回溯

A traceback 在 Python 中是一个错误信息，包含导致错误的函数调用序列，在其他编程语言中也称为堆栈跟踪。在 Python 中，任何未处理的异常都会导致 traceback，最近的调用会在底部。

大多数 Python 转换运行时失败会以 traceback 的形式出现，因此理解如何阅读它们非常重要。

考虑以下代码示例：

运行此代码会产生以下 traceback：

Copied!
1Traceback (most recent call last):
2  File "test.py", line 26, in <module>
3    main()
4  File "test.py", line 16, in main
5    stats = Statistics()
6NameError: global name 'Statistics' is not defined

与其它编程语言中的堆栈跟踪不同，Python 的 traceback 会最后显示最近的调用。从下到上，traceback 显示如下内容：

  • 异常名称，NameError ↗。有许多 内置 Python 异常类 ↗，但代码也可以定义自己的异常类。
  • 异常消息，global name 'Statistics' is not defined. 此消息包含用于调试的最有用的信息。
  • 引发异常之前的函数调用序列：File "test.py", line 26, in <module>，然后是相关代码行（第 16 行）。

使用此回溯，我们可以看到异常发生在test.py中的main方法的第16行。具体来说，导致错误的代码行是stats = Statistics()，抛出的异常是NameError。由此我们可以推断，名称Statistics不存在。回顾示例代码，似乎应该使用名称Stats而不是Statistics。

日志记录

对于日志记录，您可以使用以下选项：

  • 简单的print语句。这种方法支持标准（轻量级）转换，但不支持Spark转换。
  • 标准的 Python 日志模块 ↗。这种方法支持所有转换类型。请注意，只有INFO级别及以上的日志会被保存。

日志可在 VS Code 的 输出 中找到，并在 构建应用 的 操作 > 查看日志 下找到。

以下代码示例演示了如何输出日志以帮助调试：
