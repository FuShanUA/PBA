SQL 控制台

SQL 控制台是 Foundry 应用程序中提供的嵌入式 SQL 接口，可针对您当前查看的资源提供情境化的 SQL 访问能力。SQL 控制台适用于在特定资源（如数据集或对象类型）情境下执行快速查询。如需专用的全屏 SQL 应用程序，请参阅《SQL Studio》。

访问 SQL 控制台

您可以在受支持应用程序的底部面板中访问 SQL 控制台：

应用程序可查询资源
数据集预览表格数据
数据血缘关系表格数据
本体管理器本体对象类型及底层表格数据

初始查询会根据您当前查看的资源自动填充。

使用 SQL 控制台

SQL 控制台提供核心的 SQL 编辑与执行功能，包括运行查询、表和列的自动补全、内嵌结果面板，以及将查询保存为《SQL 工作表》的功能。SQL 控制台采用 Foundry 的《Spark SQL 方言》。

有关 SQL 功能的完整详情（包括工作表、AI 辅助查询生成和结果可视化），请参阅《SQL Studio 文档》。

模式

在《本体管理器》中，SQL 控制台支持两种模式：

  • 数据模式：使用 Furnace 查询表格数据。
  • 对象模式：使用《本体 SQL》查询本体对象类型。

在本体管理器中查看对象类型时，您可以通过 SQL 控制台工具栏中的模式下拉菜单进行模式切换。在数据模式下，系统会将该对象类型的底层数据集建议为默认资源。

在其他应用程序中，仅支持数据模式。

扩展至 SQL Studio

若要在专用的全屏应用程序中打开当前查询，请选择 SQL 控制台工具栏中的扩展选项。此操作将启动《SQL Studio》，它提供完整的 SQL 应用程序体验，包含资源浏览器、版本历史记录、AI 辅助查询生成以及嵌入式控制台中不具备的其他功能。

在尚未启用 SQL Studio 的环境中，可以临时使用 SQL 控制台的独立全屏视图。该独立视图后续将由 SQL Studio 应用程序取代。

角色与权限

对 SQL 控制台的访问受 Foundry 通用的 SQL 与下载控制角色管理。有关可用角色及其如何与 SQL 访问交互的详情，请参阅《SQL 权限》。
