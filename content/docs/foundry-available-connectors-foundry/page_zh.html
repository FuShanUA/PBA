Foundry

Foundry 连接器允许从一个 Foundry 实例共享数据到另一个 Foundry 实例。此工作流程需要访问两个 Foundry 实例，并指定其中一个实例为“源”（source），另一个为“目标”（destination）。在数据连接过程中，用户将在目标实例上执行大部分功能。

请注意，此连接器目前不兼容 视图（views） 作为输入，也不兼容 受限视图（restricted views）。由目标实例导入的数据集必须首先在其源实例中进行物化（materialized）。

例如，如果用例需要将数据从 red.palantirfoundry.com 传输到 blue.palantirfoundry.com，大多数设置和后续交互将在目标实例 blue.palantirfoundry.com 上进行，这是传输数据最终到达的地方。下面讨论的工作流通过导入（ingest）读取数据，而不是通过导出（export）写入数据。

支持的功能

功能状态
批量导入🟢 通常可用

探索即将推出

设置

  1. 打开 数据连接 应用程序，并在屏幕右上角选择 + 新源。

  2. 从可用的连接器类型中选择 Foundry。

  3. 选择使用 直接连接 通过互联网或通过中间代理 连接。

  4. 输入您源 Foundry 实例的 hostname。在这种情况下，blue.palantirfoundry.com 将从 red.palantirfoundry.com 拉取数据，因此 red.palantirfoundry.com 是源实例。

  5. 选择一种身份验证方式。

  6. 如果您使用直接连接，请为源实例创建出站策略。要将数据从 red.palantirfoundry.com 引入到 blue.palantirfoundry.com，请为 URL https://red.palantirfoundry.com 在 端口 443 上创建出站策略。与传统数据连接不同，您必须将源实例内的所有 IP 地址列入白名单。这是通过控制面板中选择 配置网络入站 选项来完成的。

  7. 按照以下说明通过将目标实例的 IP 地址添加到源实例的 网络入站 扩展中来配置源实例的入站 IP 白名单：

    1. 在目标实例的 网络出站 控制面板扩展中导航以确定适当的 IP 地址。
    2. 在源实例的 网络入站 控制面板扩展中启动。
    3. 查看现有文档以< a href="/docs/foundry/administration/configure-ingress/">配置网络入口。

如果您正在创建一个< a href="/docs/foundry/data-connection/set-up-agent/">基于代理的连接，则必须根据代理的主机提供适当的IP地址。此外，您的代理至少需要使用Java 21，因为使用Foundry连接器的基于代理的连接与旧版本的Java不兼容。 了解有关在配置网络出口时识别IP的更多信息。

如果无法访问< strong>网络入口控制面板扩展，请联系Palantir支持。

了解更多关于< a href="/docs/foundry/data-connection/set-up-source/">在Foundry中设置连接器的信息。

  2. 选择不，我不会使用本体论 SDK，并在选择所属组织后确保启用您的应用。

个人访问令牌（临时）：出于安全原因，我们不允许在生产用例中使用令牌。如果使用有效期超过36小时的令牌运行同步操作，将导致数据导入失败。

身份验证凭据是在目标实例中输入的。在源实例中，您必须创建一个令牌，使目标实例能够读取数据。为此，请导航到以下URL：

网络

Foundry 连接器需要通过 443 端口（HTTPS）访问目标 Foundry 实例。目标实例需要一个与源实例的 URL 对应的出站策略。

要在 Foundry 实例之间建立直接连接，当在 数据连接应用程序 中设置源时，必须添加适当的 出站策略。

使用代理进行连接时不需要出站策略。

从 Foundry 同步数据

要设置从一个 Foundry 到另一个 Foundry 的同步，请在源 概览 屏幕的右上角选择 探索并创建同步。浏览源 Foundry 实例中的可用项目和数据集，然后选择要同步的数据集。准备就绪后，选择 为 x 个数据集创建同步。

增量同步

增量（或追加）同步会记录最近一次同步的状态，并仅从目标数据集导入新数据或更改的数据。通过Foundry连接器，有两种方式可以建立这些数据摄入。

增量选项 1：先摄入所有数据，然后只摄入更新内容

"初始增量状态"可以设置为任意遥远的日期，比如1970年1月1日。在第一次运行数据摄入时，将提取所有数据。从第二次运行开始，每次摄入只会提取最新的可用数据。

增量选项 2：在特定日期之后摄入所有数据

"初始增量状态"可以设置为任意日期，例如2024年1月1日。与上述选项类似，首次运行导入时将提取所有数据。然后后续的导入只会提取最新的可用数据。这是一种更过滤的选项，适用于导入作者知道他们想要排除在特定日期之前写入外部源系统的数据的用例。

创建流同步

您可以从一个Foundry注册流中导入到另一个Foundry注册流。源注册的数据集必须是流。可以通过数据集RID和分支名称建立同步。在指定模式并首次运行后，将在目标注册中创建一个新的数据集。

流偏移选项

同步可以配置为仅摄取新创建的行，或者从摄取流中的所有现有行开始。需要考虑的主要权衡是，如果流数据集足够大，从时间和计算的角度来看，摄取所有历史行可能会很昂贵。

虚拟表

不支持使用虚拟表的数据集。只能在“源”Foundry实例上对< a href="/docs/foundry/iceberg/storage/">管理Iceberg表进行虚拟化。

本节提供了有关在Foundry源中使用< a href="/docs/foundry/data-integration/virtual-tables/">虚拟表的额外详细信息。当同步到Foundry数据集时，本节不适用。

下表突出了适用于Foundry的虚拟表功能。

功能状态

确保“目标”Foundry 实例具有与“源”Foundry 实例以及支撑 Iceberg 表的存储桶位置的网络访问权限。验证此存储桶是否允许来自“目标”Foundry 实例的入站流量。

从代码中调用 Foundry API

除了上述的Foundry到Foundry同步工作流外，Python转换可以使用OAuth2客户端凭证授权直接调用Foundry API。当需要调用未作为同步暴露的Foundry端点时，请使用此模式——例如，列举项目内容、触发构建，或从转换内部读取本体论API。

整体设置（REST API源配置，将client_id/client_secret存储为额外的密钥，以及通用的令牌请求和分页结构）与任何其他OAuth2客户端凭证流程相同。请查看REST API连接器页面上的OAuth客户端凭证授权示例以获取通用模式。

以下部分涵盖了与第三方API不同的Foundry特定细节。

Foundry令牌端点

Foundry的OAuth2令牌端点是：

POST /multipass/api/oauth2/token

端点托管在您调用的Foundry实例上。如果转换在它所调用的同一实例上运行，则REST API源上的hostname即为该实例；如果是不同的实例，则源的hostname为目标实例，您必须按照< a href="/docs/foundry/administration/configure-egress/">出站策略和< a href="/docs/foundry/administration/configure-ingress/">入站允许列表中的描述进行配置。

了解更多关于< a href="/docs/foundry/platform-security-third-party/writing-oauth2-clients/#token-endpoint">令牌端点参数的信息。

Foundry范围

每个Foundry API端点都会记录其所需范围——请参阅< a href="/docs/foundry/api/general/overview/introduction/">API文档中的各端点参考。一些常见示例：

范围授予访问权限
api:datasets-read读取数据集（dataset）
api:datasets-write写入数据集（dataset）
api:ontologies-read读取本体论（ontology）对象和关联类型（link types）

仅请求转换所需的范围。多个范围用空格分隔，例如 api:datasets-read api:datasets-write。

创建客户端

client_id 和 client_secret 在令牌请求中使用，来自于目标 Foundry 实例上注册的 第三方应用程序。按照上面 身份验证 中的步骤创建后端服务应用程序并获取 client_id 和 client_secret，然后将它们作为 附加密钥 存储在您的 REST API 源中。

为客户端创建的服务用户必须被授予访问转换所需的所有项目、数据集或 Ontology 资源的权限。

示例：列出项目子项

The following transform requests an access token against /multipass/api/oauth2/token, then uses the token to list the children of a project via the Foundry /api/v2/filesystem/resources/{rid}/children endpoint. For the generic token-request and pagination scaffolding this example reuses, see the REST API OAuth Client Credentials grant example.

1import logging
2
3import pandas as pd
4from transforms.api import Output, transform_pandas
5from transforms.external.systems import external_systems, Source, ResolvedSource
6
7logger = logging.getLogger(__name__)
8
9
10@external_systems(
11    foundry_api_source=Source("<source_rid>")
12)
13@transform_pandas(
14    Output("<output_dataset_rid>"),
15)
16def compute(foundry_api_source: ResolvedSource) -> pd.DataFrame:
17    base_url = foundry_api_source.get_https_connection().url
18    client = foundry_api_source.get_https_connection().get_client()
19
20    client_id = foundry_api_source.get_secret("additionalSecretClientId")
21    client_secret = foundry_api_source.get_secret("additionalSecretClientSecret")
22
23    token_response = client.post(
24        base_url + "/multipass/api/oauth2/token",
25        data={
26            "grant_type": "client_credentials",
27            "client_id": client_id,
28            "client_secret": client_secret,
29            "scope": "api:datasets-read",
30        },
31        headers={"Content-Type": "application/x-www-form-urlencoded"},
32    )
33    token_response.raise_for_status()
34    access_token = token_response.json()["access_token"]
35
36    auth_headers = {"Authorization": f"Bearer {access_token}"}
37
38    project_rid = "ri.compass.main.folder.xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
39    resources = []
40    page_token = None
41    while True:
42        params = {"pageSize": 100}
43        if page_token:
44            params["pageToken"] = page_token
45
46        response = client.get(
47            base_url + f"/api/v2/filesystem/resources/{project_rid}/children",
48            headers=auth_headers,
49            params=params,
50        )
51        response.raise_for_status()
52        body = response.json()
53
54        for resource in body.get("data", []):
55            resources.append({
56                "rid": resource.get("rid"),
57                "name": resource.get("displayName"),
58                "type": resource.get("type"),
59            })
60
61        page_token = body.get("nextPageToken")
62        if not page_token:
63            break
64
65        logger.info(f"Fetched {len(resources)} resources so far, continuing to next page.")
66
67    return pd.DataFrame(resources)
