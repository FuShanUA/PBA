图示

本页解释了如何与解决方案设计师中的图示进行交互，包括添加节点、创建连接、组织您的图示以及自定义视图。

在您的图示中添加节点

解决方案设计师提供了多种方式来在您的图示中添加表示平台组件、资源或抽象概念的节点。

添加组件节点

组件节点代表特定的Palantir平台应用程序和工具，如 Pipeline Builder（管道构建器）、Workshop（研讨会） 或 Contour（轮廓）。

要添加一个组件节点：

  1. 在图示工具栏中选择 组件 按钮，或按 Cmd/Ctrl + B。
  2. 组件选择器对话框打开，显示所有按类别组织的可用平台组件。
  3. 在左侧浏览类别或使用顶部的搜索栏查找特定组件。
  4. 从列表中选择一个组件以将其添加到画布中。

添加组件节点后，配置其他详细信息，这些信息将呈现在您的图表上，并为用户提供有关该节点在您描绘的工作流程中的目的的更多信息。

添加资源节点

资源节点表示与您的图表相关联的具体Foundry资源，例如数据集（dataset）、对象类型（object types）、函数（functions）或操作（actions）。

要添加资源节点：

  1. 在图表工具栏中选择资源（Resource）按钮。
  2. 出现一个下拉菜单，包含四个选项：
    • 新文件节点： 浏览 Foundry 资源以添加
    • 新对象节点： 从您的本体论中选择一个对象类型
    • 新接口节点： 从您的本体论中选择一个 接口
    • 新操作节点： 从您的本体论中选择一个操作类型
  3. 在弹出的模态窗口中搜索并选择资源，Solution Designer 会将该节点添加到您的画布上，并与所选资源建立链接。

资源节点会自动显示资源名称，并提供一个链接以在原生应用程序中打开该资源。

添加概念节点

概念节点表示抽象的、高层次的工作流概念（如“数据获取”或“分析”），您可以稍后用具体的实现替换这些概念。这些节点主要用于与 AIP Architect 一起规划工作流。

要添加一个概念节点，请选择 概念 在选择您的概念类型（如分析或数据处理）之前，点击图表工具栏中的按钮。

您可以使用AIP Architect将抽象概念节点转换为具体的实现模式。

添加组

组允许您将相关的节点组织到逻辑容器中。

要添加一个空的组，请在图表工具栏中选择组按钮，并拖动节点到组中以将其作为成员添加。

您也可以通过选择多个节点并使用组操作从现有节点创建组。了解有关从现有节点创建组的更多信息。。

添加文本节点

文本节点允许您在图表上直接添加注释、标注或文档。

要添加一个文本节点，请在图表工具栏中选择Add Text node按钮，然后在画布上选择文本节点以编辑其内容。

配置节点

在将节点添加到您的图表后，您可以配置其属性并将其链接到Foundry资源。

节点工具栏

当您选择一个节点时，会在其正上方显示一个小的工具栏。此工具栏是与节点进行交互和配置的主要方式。该工具栏包含几个水平排列的按钮：

  • 主要操作按钮：对于链接到资源的节点，此按钮会打开其原生应用程序中的资源，例如Pipeline Builder或Workshop。对于未链接的节点，此按钮允许您链接资源。
  • 链接资源： 通过其RID将Foundry资源与此节点关联。选择关联资源以打开资源选择器对话框，然后选择一个数据集、管道、应用程序或其他Foundry资源。当关联后，节点会显示资源名称并提供直接导航到该资源的路径。
  • 操作菜单：选择...以访问其他节点操作：
    • 重叠顺序：更改重叠节点的层叠方式。
    • 分组：以所选节点开始创建一个新组。
    • 编辑背景颜色：自定义节点的背景颜色。
    • 移除关联资源：从节点中解除Foundry资源的关联。
    • 编辑标签：添加或删除标签以根据标记、组织或团队对节点进行分类。
    • 删除：从图表中删除该节点。

自定义节点外观

您可以自定义节点在图表中的显示方式：

  • 背景颜色：右键点击节点，选择更改背景颜色，或使用节点工具栏中的操作菜单。
  • 图标：更改自定义节点的图标及其颜色。
  • 自定义标题：当节点被选中时，直接选择它来编辑节点标题。
  • 调整节点大小：将鼠标悬停在节点的角落或边缘，然后拖动调整大小的手柄以调整其大小。

节点文档卡片

每个节点在右上角显示一个文档卡片，展示它所代表的平台组件的信息。卡片显示：

  • 组件名称和图标：标识平台组件。
  • 简要描述：该组件功能的简短概述。
  • 完整描述： 一个可展开的区域，提供有关组件功能和用例的详细信息，您可以通过选择显示更多来查看。
  • 文档链接：在新标签页中打开该组件的官方Foundry文档。
  • 询问助手：如果启用了AIP，将打开一个对话框，用于向AIP助手询问有关该组件的问题。

要最小化文档卡片，请点击卡片右上角的最小化按钮。一个小的信息图标仍然可见，您可以点击它以重新展开卡片。

创建连接

节点之间的连接（或边）表示架构中的数据流、依赖关系或关系。

连接节点

要创建一个连接：

  1. 将鼠标悬停在您想要连接的节点上。在左侧和右侧边缘会出现小的圆形手柄。
  2. 选择手柄，并从源节点的右侧手柄（输出）拖动。
  3. 将光标拖动到目标节点的左侧手柄（输入）。
  4. 释放鼠标按钮以创建连接。

编辑或删除连接

要删除一个连接：

  1. 在画布上选择连接线，选中后会高亮显示。
  2. 按下Backspace或Delete。

或者，右键单击连接并从上下文菜单中选择删除边。

连接标签

您可以为连接添加标签，以描述关系或数据流。

要添加或编辑连接标签：

  1. 在画布上选择连接。
  2. 选择出现在连接上的标签区域。
  3. 在Click to Edit文本输入框中输入您的标签文本。
  4. 在画布上标签区域外的任何位置选择以在选择保存之前注册您的更改。

您可以使用画布工具栏切换所有边标签的可见性。

分组节点

对相关节点进行分组有助于组织复杂的图表，并为您的架构的不同部分创建逻辑边界。

从现有节点创建分组

要对现有节点进行分组：

  1. 按住Cmd/Ctrl并选择每个节点，或者按住Shift并拖动一个选择框围绕这些节点。
  2. 选择节点后，在图表工具栏中选择分组按钮，或右键单击并选择分组。

Solution Designer 将所选节点放入一个容器中。

管理组成员

创建后，请按照以下操作来管理组成员。

  • 添加节点：从画布上选择任意节点并拖动到组中。当节点进入组边界时，释放以将其作为成员添加。
  • 移除节点：选择并拖动节点离开组边界。节点将从组中移除，但会保留在画布上。
  • 移动组：选择并拖动组容器本身。所有成员节点都会随之移动，并保持它们的相对位置。

画布控件

画布工具栏出现在画布的左下角，提供用于浏览和操作你的图表的工具。工具栏包含几个垂直堆叠的图标按钮：

  • 放大: 增加缩放级别以查看更多细节。
  • 缩小: 减少缩放级别以查看更多的图表。
  • 将所有节点适应视图: 自动居中并缩放以显示整个图表。
  • 锁定图表: 启用只读模式以防止意外编辑。锁定时你仍然可以平移和缩放。在演示期间锁定图表很有用。
  • 隐藏/显示背景点: 切换背景点网格的可见性。
  • 自动排列节点: 自动将节点排列成类似网格的结构，间距均匀且交叉最少。对于整理杂乱的图表很有用。
  • 切换边流动画: 在画布上对边流标签进行动画处理。
  • 隐藏/显示边标签: 隐藏或显示所有连接上的标签。
  • 隐藏/显示边有效性: 隐藏或显示边有效性着色。
  • 最小化/最大化节点显示：减少节点上显示的信息。这对于节省大型或复杂图表的空间很有用。

小地图

小地图显示了你整个图表的鸟瞰图以及当前视口的位置。

要启用小地图：

  1. 打开设置面板（参见设置）。
  2. 切换小地图为已启用状态。
  3. 小地图出现在画布的右下角。

选择小地图可以跳转到图表的不同区域。

平移和缩放

要浏览画布：

  • 平移： 选择并拖动空白画布空间，或在设置中启用平移模式以平移而不会意外选择节点。
  • 缩放：使用鼠标滚轮、触控板捏合手势，或画布工具栏中的缩放按钮。
  • 适应视图：选择将所有节点适应到视图中按钮以居中并缩放以显示所有节点。

高级节点操作

侧边菜单

当您在画布上选择一个节点时，右侧会打开一个侧边菜单面板，为该节点提供额外选项。

节点遍历

For all nodes, the side menu includes a Node traversal section that allows you to explore the diagram by navigating through connected nodes:

  • View connected nodes: See a list of all nodes connected to the currently selected node.
  • Jump to connected node: Select any connected node in the list to select and center it on the canvas.
  • Understand data flow: Quickly trace how data moves through your architecture by following node connections.

Node traversal helps you navigate complex diagrams and understand relationships between components without manually searching.

Object traversal

For nodes linked to ontology object types, the side menu includes an Object traversal section that displays the ontology relationships available for that object type:

  • View relationships: See all link types and related object types defined in the ontology.
  • Create traversal: Select specific relationships to automatically generate related object type nodes on the canvas, which mimics the functionality of the Create traversal action in the node toolbar.
  • Explore ontology structure: Understand how this object type connects to other parts of your data model.

Context menu

While the canvas and node toolbars are the primary ways to interact with your diagram, you can also use context menus as an alternative:

  • Right-click on a node: Shows options like Change Bg color, Group with other selected nodes, and Delete.
  • Right-click on canvas: Shows options like Add Comment, Add Empty group, and Layout nodes automatically.

The context menu provides quick access to common actions, but most operations are more conveniently accessed through the dedicated toolbars.
