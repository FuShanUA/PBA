将本体数据添加至 Gaia

您可以根据实际用例，通过多种方法将配置为 Foundry 本体组成部分的地理空间数据添加至 Gotham 的 Gaia 应用程序中。为了让 Gotham 能够访问 Foundry 本体数据，您需要使用 Foundry 的 本体管理器对目标对象类型进行类型映射，从而在两者之间建立统一的本体表示。

注意

如果您的部署环境包含地图渲染服务 (MRS)，则您无需完成类型映射流程即可在 Gaia 中查看 Foundry 对象。您可以创建新的本体对象或将其现有对象添加至 Gotham，而无需进行额外配置。请按照此处的说明检查您的部署环境是否包含 MRS。如果您的部署环境不包含 MRS，则可以按照 Foundry 本体文档中的现有说明完成类型映射流程。

如有关于 MRS 可用性、安装或其在 Gotham 中相关文档的问题，请联系 Palantir 支持团队。

拖放单个对象和已保存的探索

你可以将单个对象或源自已保存探索的对象集（最多可包含 5,000 个对象）从 Foundry 拖放到 Gaia 地图上。

要从 Foundry 将单个对象拖放到你的 Gaia 地图上：

  1. 在 Foundry 中启动对象浏览器。
  2. 搜索一种对象类型。
  3. 从对象浏览器屏幕最右侧的结果面板中选择一个对象。
  4. 选择对象浏览器屏幕左上角的对象图标。
  5. 从 Foundry 拖动对象并将其放到 Gaia 地图上的任意位置 - 你现在可以在地图上和左侧面板中查看该对象，它将作为名为 Dropped Object Layer [Date] [Time] 的默认图层的一部分。

在将对象集从 Foundry 拖放到 Gaia 地图之前，必须先基于现有对象类型创建带版本的对象集。创建带版本对象集的步骤如下：

  1. 在 Foundry 中启动 Object Explorer。
  2. 搜索并选择目标对象类型。
  3. 在 Object Explorer 屏幕顶部功能栏的中间位置，选择 Results。
  4. 使用搜索栏根据对象类型的属性进行筛选，以缩小范围并确定要添加到对象集中的对象。
    • 例如，如果对象类型通过 name 属性定义位置，可以输入 where name is location_1 or where name is location_2 并按下键盘上的 enter 或 return 键，筛选出指定的 location 对象集合。
  5. 点击蓝色的 Save 按钮，将选中的对象作为 Exploration 保存到公共项目中。

也可以在对象类型的 Ontology Explorer 视图中，直接对图表和图形应用对象筛选器来筛选对象。

将带版本的对象集作为 Exploration 创建并保存后，可按照以下步骤将其拖放到 Gaia 地图上：

  1. 在 Object Explorer 视图的顶部功能栏中选择 New exploration。
  2. 根据创建时命名的名称搜索并选择目标对象集。该对象集也可能显示在 Object Explorer 搜索栏下方的 My Explorations & Lists 区域中。
  3. 点击 Object Explorer 屏幕左上角的对象集图标。
  4. 从 Foundry 中拖动对象集，将其放置在 Gaia 地图上的任意位置。对象将逐个在地图上生成，并自动归入左侧面板中以所保存的 Exploration 命名的搜索图层下。

你也可以从 Gotham 内的其他对象视图中拖拽对象到你的 Gaia 地图上。

在 Gaia 中搜索你的 Foundry 对象类型

你可以在 Gaia 左侧面板的 数据源 区域搜索 Foundry 对象类型。此外，在将对象添加到地图之前，你还可以对其应用几何和时间过滤器。要从 Gaia 搜索 Foundry 对象类型：

  1. 在地图左侧的面板中选择 数据源 (Data Sources)。
  2. 根据 Foundry 中的对象名称搜索相应的对象类型，搜索结果会显示在左侧面板底部的 对象类型 (Object types) 下方。
  3. 在你的对象类型旁边选择 搜索对象 (Search for objects)。
  4. 选择想要添加到地图上的对象，并在对象搜索面板顶部的工具栏中应用任何必要的几何过滤器。
    • 全部 (All) 会将所发现对象类型中的所有对象添加到你的地图上。
    • 当前视图 (View) 仅会添加当前地图视图内可见的对象。
    • 绘制 (Draw) 会将位于绘制图形内的对象添加到地图上，你可以从搜索面板中配置并添加该图形。
    • 选择 (Select) 允许你从当前活动地图中选择某个项目。
  5. 应用任何必要的时间过滤器，以加载在 静态范围 (Static range) 或 滚动窗口 (Rolling window) 内活动的对象。
  6. 可选择开启 实时更新搜索结果 (Live update search results)，该功能会根据符合你过滤参数的新建对象，每 30 秒更新一次你的对象或对象集。
  7. 在 搜索标题 (Search title) 中为你的对象搜索输入一个名称，然后选择 搜索 (Search)。
  • 如果在加载搜索结果后无法查看某个对象，你应检查该对象类型是否包含指示其位置的地理空间属性，例如 geopoint 或 geoshape。
  • 如果在 Foundry 中完成前置的类型映射后，仍无法在 Gotham 中搜索到您的对象，则可能是您的 Gotham 授权配置中未启用 Foundry 搜索功能。请联系 Palantir 支持团队以协助启用此功能。

创建搜索模板

您可以在 Gaia 管理应用中创建搜索模板，以便在通过左侧面板的数据源选项卡将 Foundry 对象添加到地图时，预先配置自定义对象搜索界面。

要创建并应用搜索模板，请启动 Gaia 管理应用并按照以下步骤操作：

  1. 找到地理搜索面板并选择显示，以查看您的 Gotham 授权配置中所有处于活动状态的地理搜索。
  2. 滚动到地理搜索列表底部并选择添加。
  3. 为您的搜索命名并填写每个配置输入框。
    • Description 将显示在搜索模板界面的顶部。
    • Tab ID 将启用您的地理搜索与搜索模板之间的映射。
    • Tab icon 将在用户搜索时显示在数据源选项卡中。
    • Template ID 将与 Tab ID 匹配，并启用您的地理搜索与搜索模板之间的映射。
    • Classification，根据您的授权配置，此项可能为可选。
    • ACL Groups 将设置组访问权限。
  4. 根据需要输入其他配置。
  5. 在配置搜索模板之前，选择右上角功能区中的预览并保存，以在 Gaia 管理应用中保存您的地理搜索。

您可以参考以下地理搜索示例，该示例使用户能够搜索法国的城市，并在与搜索模板配对后在 Gaia 地图上显示其地理边界。

  1. 找到搜索模板面板并选择显示，以查看您的 Gotham 授权配置中所有处于活动状态的搜索模板。
  2. 滚动到现有 templates: 的底部，插入包含您的搜索模板配置和地理搜索映射的 YAML 代码。您可以参考下方的示例搜索模板 YAML 代码块。
```yaml - id: french_cities # 匹配地理搜索中的 `Tab ID` 和 `Template ID` 字段。 title: 法国城市 # 将显示在 `Data sources` 面板中的搜索模板名称。 query: queryType: AND # 定义查询类型并插入相关子查询。 subqueries: - inputId: keyword queryType: KEYWORD - field: object_type queryType: OBJECT_TYPE value: - # 对象类型的 RID dataSource: foundry-federation namespaces: - foundry-federation description: 搜索法国城市并显示其边界。 integratedDataSetIds: [] inputs: - hint: 输入关键词... id: keyword inputType: TEXTFIELD # 启用自由文本搜索。 label: 关键词 aggregations: [] ``` 8. 在右上角的功能区中选择 **预览并保存** 以保存您的搜索模板。 您可以返回 Gaia 地图来测试搜索模板。在左侧面板的 **数据源** 标签页中输入搜索模板的名称，或者从 **集成数据源** 子部分中手动选择它。选择您的搜索模板，将启动由地理搜索和 YAML 配置支持的预配置界面。使用 **关键字** 搜索栏或 YAML 代码中设置的任何其他自定义配置来搜索对象，然后在左面板底部选择蓝色的 **搜索** 按钮之前，输入一个 **搜索标题**。 搜索结果随后将显示在您的 Gaia 地图上，供进一步探索。 ### 在 Foundry 中配置实时地理时间序列同步 您可以使用 Foundry 的管道构建器从现有数据集中创建地理时间序列同步，无需额外配置即可将大规模、实时的地理时间数据添加到 Gaia 地图中。 在 Foundry 中创建地理时间序列同步的步骤如下： 1. 导航至您的数据集，并确保其包含启用地理时间序列同步所需的所有前提字段。 2. 选择 **所有操作** > **创建新管道** 以启动管道构建器。 3. 选择您的数据集节点，并在垂直弹出菜单顶部选择 **转换**。 4. 搜索并选择 **构建 GeoPoint 列**，以映射您的 `latitude` 和 `longitude` 字段，并创建一个新的 `geo_point` 列。 5. 通过转换界面左上角的编辑框，将转换重命名为 `Create GeoPoint`。 6. 选择屏幕右上角的绿色箭头图标以 **保存** 您的转换。 7. 在转换界面右上角选择 **返回图形**，导航回管道构建器图形。 8. 选择 `Greate GeoPoint` 转换节点的 **添加输出** 按钮，然后选择 **新建地理时间序列同步**。下图展示了管道构建器中可用的输出类型菜单。

  1. 为新建的输出命名，并指定其目标命名空间，例如 Local System。
  2. 配置所有主字段，确保同步引用其底层数据集中的正确列。
  3. 点击各个属性右侧的 X 图标或清除未映射属性按钮，清除所有未映射的属性。
  4. 在属性下，将观察模式留空——Foundry 会自动将其填充为上述在元数据中为序列同步输出分配的名称。
  5. 检查高级设置下的源系统是否映射到您的管道名称。下图展示了同步输出所需的所有配置。
  1. 点击屏幕右上角的绿色箭头图标以保存您的同步。
  2. 点击保存旁边的锤子图标以部署管道。
  3. 导航至返回图视图。您会在 Create GeoPoint 转换右侧看到一个新建的地理时间序列同步输出节点，节点底部显示绿色的 Deployment up-to-date 文字，确认部署成功。

要将新建的地理时间序列同步添加到您的 Gaia 地图中：

  1. 在 Gaia 左侧面板中打开数据源标签页。
  2. 搜索您的地理时间序列同步名称，并选择轨迹和观察记录搜索。
  3. 在对象搜索面板的顶部功能区中应用所需的几何过滤器。
    • All 会将所发现对象类型中的所有对象添加到您的地图中。
    • View 仅添加当前地图视图中可用的对象。
    • Draw 会将位于您所绘制形状内的对象添加到地图中，您可以从搜索面板中配置并添加该形状。
    • Select 允许您从当前激活的地图中选择一个项目。
  4. 通过选择历史搜索或实时数据流来设定合适的时间窗口。
  5. 选择添加到地图。

一旦你的地理时间序列同步作为 Gaia 地图图层 的一部分加载完成，你便可以选择单个实体（例如某条特定的飞行轨迹）并进行额外配置。

使用由函数支持的企业级地图图层

你可以将数据添加到 Gaia 地图中，作为由 Foundry 中编写并发布至本体（ontology）的函数所支持的企业级地图图层（EML）的一部分。

若要创建一个 TypeScript 函数，从现有对象类型输出对象集以支持 Gaia EML：

  1. 导航至包含你的对象类型支持数据集的项目文件夹。
  2. 选择 新建 > 代码仓库。
  3. 在 你要构建什么？ 下方选择 函数 选项，并选择 TypeScript 函数 作为你的语言。
  4. 为你的仓库命名并将其保存在公共项目文件夹中，以启用 Foundry 本体 API 访问权限，然后在屏幕底部选择 初始化仓库。
  5. 选择屏幕右侧的绿色 前往 index.ts 按钮来编写你的函数。
  6. 遵循草稿 index.ts 文件中嵌入的编写提示，插入你自己的函数；你可以参考下方的示例代码。
```typescript import { Function, Integer } from "@foundry/functions-api"; import { Objects, ObjectSet, ObjectTypeName }; export class MyEmlFunction { @Function() public myFunctionName(): ObjectSet { return Objects.search().ObjectTypeName().filter(location => location.propertyTypeToSearch.exactMatch("match-parameter")); } } ``` 7. 在顶部功能区选择**提交**。 8. 选择**标记版本**，将该函数发布到 Foundry 本体中。 9. 在 Foundry 本体管理器的**函数**选项卡中查看新建的函数。 接下来，启动 Gaia 的管理应用，以创建基于 Foundry 函数的 EML。进入 Gaia 管理应用后，你可以执行以下操作： - 搜索并选择**企业地图图层**页面。 - 在页面底部选择**添加**，创建新的 EML。 - 将 EML 类型设置为 `Foundry function EML`。 - 输入 EML 的**描述**并标明其**功能**。 - 可选操作：输入**标签页图标**。 - 标明 EML 的**分类**。 - 根据需要添加 **ACL 组**以限制 EML 的访问权限。 - 填入你的**函数 RID**——你可以在 Foundry 本体管理器应用的函数“概览”选项卡中将其复制到剪贴板。 - 可选操作：为 EML 设置**刷新间隔（秒）**——若留空，EML 将不会刷新。 - 可选操作：填入**地理筛选属性 ID**，以便在将 EML 作为参考数据图层添加时，在 Gaia 中启用对象的地理筛选功能；你可以使用对象类型中的 `geopoint` 属性 ID。

你可以在本体管理器的对象类型**属性**页面中找到并复制属性 ID；为此，请选择 geopoint 属性右侧的省略号图标，然后选择**复制属性 ID**。

现在，你可以通过左侧面板的**数据源**选项卡搜索名称，将基于 Foundry 函数的 EML 添加到 Gaia 地图中。Gaia 会将 EML 归类为**精选数据源**下的 `Reference data layers`。

你可以使用左侧面板，或在当前视图中选择对象的图标，来选中或隐藏 EML 中的单个对象。

使用基于版本化对象集的企业地图图层

你可以将数据添加到 Gaia 地图中，作为 EML 的一部分，该 EML 由你基于 Foundry 本体中的对象类型所创建的版本化对象集提供支持。

要创建版本化对象集，请按照上文《拖放单个对象和已保存的探索》章节中的说明进行操作。

要在 Gaia 管理应用中建立由版本化对象集支持的 EML，请按照上文《使用函数支持的企业地图图层》章节中的说明进行操作。将 EML 类型设置为 Versioned object set EML，而不是 Foundry function EML；你无需插入 Function rid。

现在，你可以通过左侧面板的 Data sources 标签页搜索名称，将基于版本化对象集的 EML 添加到 Gaia 地图中；Gaia 会将 EML 归类在 Curated data sources 下的 Reference data layers 中。
