将本体论数据添加到Gaia

您可以通过多种方法根据您的使用情况，将配置为Foundry本体论一部分的地理空间数据添加到Gotham的Gaia应用程序中。为了使Foundry本体论数据在Gotham中可用，您需要通过Foundry的本体论管理器对感兴趣的物体类型进行类型映射，以在两者之间建立统一的本体表示。

注意

如果您的注册包含地图渲染服务（MRS），那么您不需要完成类型映射过程即可在Gaia中查看Foundry对象。您可以直接从或添加现有的对象到Gotham而无需额外配置。 按照此处的说明来检查您的注册是否包含MRS。如果您的注册不包含MRS，则可以通过 完成类型映射过程。 按照现有说明，作为Foundry本体论文档的一部分

有关MRS的可用性、安装或Gotham中提供的附加文档的问题，请联系Palantir支持团队。

拖放单个对象和已保存的探索

您可以从Foundry中拖放单个对象或从已保存的探索中派生的对象集（可能包含最多5,000个对象）到Gaia地图上。

要从Foundry拖放单个对象到您的Gaia地图：

  1. 在Foundry中启动对象浏览器。
  2. 搜索一个对象类型。
  3. 在对象浏览器屏幕最右侧的结果面板中选择一个对象。
  4. 在对象浏览器屏幕左上角选择对象的图标。
  5. 从Foundry拖动对象并将其放置在你的Gaia地图的任何位置 - 现在你可以将该对象作为标题为Dropped Object Layer [Date] [Time]的默认图层的一部分，在地图和左侧面板中查看。

在你可以从Foundry拖动和放置对象集到你的Gaia地图之前，你必须首先从现有的对象类型创建一个版本化对象集。要创建版本化对象集：

  1. 在Foundry中启动对象浏览器。
  2. 搜索并选择你的对象类型。
  3. 在对象浏览器屏幕顶部的中间条中选择结果。
  4. 使用搜索栏对要添加到对象集的对象进行筛选，以根据对象类型的属性缩小你的选择范围。
    • 例如，如果你的对象类型捕获由一个name属性定义的位置，你可以通过输入where name is location_1 or where name is location_2并按下键盘上的enter或return来筛选出一组location对象。
  5. 选择蓝色的保存按钮，将所选对象保存为公共项目中的探索。

您还可以通过在对象类型的本体论浏览器视图中的图表和图形中直接应用对象过滤器来筛选对象。

在您创建并以探索的形式保存一个版本化的对象集后，您可以按照以下步骤将其拖放到您的Gaia地图上：

  1. 从对象浏览器视图顶部的选项卡中选择新探索。
  2. 根据创建时给它的名称搜索并选择您的对象集。该对象集也可能出现在对象浏览器搜索栏下方的我的探索与列表下。
  3. 在对象浏览器屏幕左上角选择对象集的图标。
  4. 将对象集从Foundry拖放到你的Gaia地图的任何位置。对象会自动在地图和左侧面板中的搜索层内单独传播，该搜索层会根据你保存的探索自动命名。

你也可以从Gotham内的其他对象视图中将对象拖放到你的Gaia地图上。

在Gaia中搜索你的Foundry对象类型

你可以在Gaia左侧面板的数据源部分中搜索Foundry对象类型。此外，在将对象添加到地图之前，你还可以应用几何和时间相关的过滤器。要在Gaia中搜索Foundry对象类型：

  1. 选择数据源 在地图的左侧面板中。
  2. 在 Foundry 中根据对象类型名称进行搜索，它将出现在左侧面板底部的对象类型下。
  3. 在您的对象类型旁边选择搜索对象。
  4. 选择您想要添加到地图上的对象，并应用对象搜索面板顶部可用的任何必要的几何过滤器。
    • All 将从发现的对象类型中添加所有对象到您的地图。
    • View 仅添加当前地图视图内可用的对象。
    • Draw 将添加位于您可以在搜索面板中配置并添加的绘制形状内的对象。
    • Select 允许您从活动地图中选择一个项目。
  5. 应用必要的时间过滤器以加载在 Static range 或 Rolling window 内处于活动状态的对象。
  6. 可选地打开实时更新搜索结果，这将根据符合您过滤参数的新创建的对象，每30秒更新一次您的对象或对象集。
  7. 在搜索标题中输入您的对象搜索名称，并选择搜索。
  • 如果在加载搜索结果后无法查看对象，则应验证对象类型是否包含表示其位置的地理空间属性，例如geopoint或geoshape。
  • 如果在Foundry中完成必要的类型映射后仍无法在Gotham中搜索对象，则可能未在您的Gotham注册中启用Foundry搜索功能。请联系Palantir支持团队以获取帮助以启用此功能。

创建一个搜索模板

您可以在Gaia管理应用程序中创建一个搜索模板，以便在通过左侧面板的数据源将Foundry对象添加到地图时预配置一个自定义对象搜索界面。 标签。

要创建和实现搜索模板，请启动Gaia管理应用程序并按照以下步骤操作：

  1. 定位地理搜索面板并选择显示，以查看您在Gotham注册中的所有活动地理搜索。
  2. 滚动到地理搜索列表底部并选择添加。
  3. 为您的搜索命名并填写每个配置输入框。
    • Description 将出现在搜索模板界面的顶部。
    • Tab ID 将在您的地理搜索和搜索模板之间建立映射。
    • Tab icon 在用户搜索时将出现在数据源标签中。
    • Template ID 将与Tab ID匹配，并在您的地理搜索和搜索模板之间建立映射。
    • Classification，这可能根据您的注册情况是可选的。
    • ACL Groups 将设置组访问权限。
  4. 根据需要输入其他配置。
  5. 在右上角的选项卡中选择预览并保存，在您配置搜索模板之前将您的地理搜索保存到Gaia管理应用程序中。

在Foundry中配置实时时空序列同步

您可以使用Foundry的Pipeline Builder从现有数据集创建一个时空序列同步，从而在不进行额外配置的情况下，将高规模、实时的时空数据添加到您的Gaia地图中。

在Foundry中创建一个时空序列同步：

  1. 导航到您的数据集并确保它包含所有先决条件字段，以启用时空序列同步。
  2. 选择所有操作 > 新建管道以启动管道构建器。
  3. 选择您的数据集节点，并在垂直弹出菜单的顶部选择转换。
  4. 搜索并选择构建GeoPoint列来映射您的latitude和longitude字段，并创建一个新的geo_point列。
  5. 通过转换顶部左角的编辑框重命名您的转换Create GeoPoint。
  6. 选择屏幕右上角的绿色箭头图标以保存您的转换。
  7. 在转换的右上角选择返回图表，以返回到您的管道构建器图表。
  8. 选择Greate GeoPoint转换节点的添加输出按钮，并选择新的时空序列同步。下图显示了管道构建器中可用的输出类型菜单。

  1. 为您的新输出命名并分配目标命名空间，例如Local System。
  2. 配置所有主字段，以确保同步引用其支撑数据集中的正确列。
  3. 通过选择每个字段右侧的X图标或清除未映射属性按钮，清除所有未映射的属性。
  4. 在属性下留空观察模式 - Foundry会根据您在元数据中为系列同步输出分配的名称自动填充它。
  5. 验证高级设置下的源系统是否映射到您的管道名称。下面的图片展示了您的同步输出所需的全部配置。
  1. 选择屏幕右上角的绿色箭头图标以保存您的同步。
  2. 选择保存旁边的锤子图标以部署管道。
  3. 导航到返回图表。您将在您的Create GeoPoint转换右侧看到一个新的地理时空系列同步输出节点，该节点底部有绿色Deployment up-to-date文本，确认已成功部署。

要将您新创建的地理时空系列同步添加到您的Gaia地图中：

  1. 在Gaia左侧面板中打开数据源选项卡。
  2. 搜索您的地理时空系列同步的名称并选择轨迹和观测搜索。
  3. 在对象搜索面板顶部的工具栏中应用任何必要的几何过滤器。
    • All 将从发现的对象类型中添加所有对象到您的地图。
    • View 仅会添加当前地图视图内的对象。
    • Draw 会将位于您可以在搜索面板中配置并添加的绘制形状内的对象添加到地图上。
    • Select 允许您从活动地图中选择一个项目。
  4. 通过选择历史搜索或实时数据流来选择适当的时段。
  5. 选择添加到地图。

一旦您的时空序列同步作为 Gaia 的地图层的一部分加载，您可以选择单个实体（例如特定的飞行路径）并进行其他配置。

使用基于函数的企业地图层

您可以将数据作为基于函数的企业地图层（EML）的一部分添加到 Gaia 地图中，该函数在 Foundry 中编写并发布到您的本体论中。

要创建一个输出对象集的 TypeScript 函数，以从现有对象类型回填 Gaia EML：

  1. 导航到包含您的对象类型支撑数据集的项目文件夹。
  2. 选择 新建 > 代码仓库。
  3. 选择 What are you building? 下方的 函数 选项，并将 TypeScript 函数 作为您的语言。
  4. 为您的仓库命名，并将其保存在公共项目文件夹中以启用 Foundry 本体论 API 访问权限，然后在屏幕底部选择 初始化仓库。
  5. 选择屏幕右侧的绿色 转到 index.ts 按钮以编写您的函数。
  6. 按照草稿 index.ts 文件中嵌入的写作提示插入您自己的函数；您可以参考下面的示例代码。
Copied!
1import { Function, Integer } from "@foundry/functions-api";
2import { Objects, ObjectSet, ObjectTypeName };
3
4export class MyEmlFunction{
5
6    @Function()
7    public myFunctionName(): ObjectSet<ObjectTypeName> {
8        return Objects.search().ObjectTypeName().filter(location => location.propertyTypeToSearch.exactMatch("match-parameter"));
9
10    }
11}
  1. 在顶部菜单栏中选择 提交。
  2. 选择 标记版本 将函数发布到 Foundry 本体论。
  3. 在 Foundry 本体论管理器的 函数 选项卡中查看您新创建的函数。

接下来，启动 Gaia 的管理应用程序以建立一个由 Foundry 函数支持的 EML。一旦您导航到 Gaia 管理应用，您可以：

  1. 搜索并选择 企业地图图层 页面。
  2. 在页面底部选择添加以创建一个新的 EML。
  3. 将 EML 类型设置为 Foundry function EML。
  4. 输入 EML 的描述并说明其功能。
  5. 可选地输入一个标签图标。
  6. 说明 EML 的分类。
  7. 添加ACL 组以根据需要限制 EML 的访问权限。
  8. 插入你的函数 rid - 你可以从 Foundry 的本体论管理器应用程序的函数的“概览”选项卡中复制此内容到剪贴板。
  9. 可选地设置 EML 的刷新间隔（秒） - 如果留空，EML 将不会刷新。
  10. 可选地插入一个地理过滤属性 ID，以便在将 EML 添加为参考数据层时在 Gaia 中启用对象地理过滤；你可以使用来自你的对象类型的 geopoint 属性 ID。

你可以在你的对象类型 属性 中找到并复制一个属性 ID 在本体论管理器中打开页面；为此，请选择 geopoint 属性右侧的省略号图标，然后选择复制属性ID。

您现在可以通过在左侧面板的数据源选项卡中搜索其名称，将您的Foundry函数支持的EML添加到您的Gaia地图中。Gaia将EML作为 Reference data layers 放在精选数据源下。

您可以使用左侧面板或从当前视图中选择对象的图标来选择或隐藏EML中的单个对象。

使用版本化对象集支持的企业地图层

您可以将数据作为由版本化对象集支持的EML的一部分添加到您的Gaia地图中，该版本化对象集您可从Foundry本体论中的对象类型创建。

要创建一个版本化对象集，请按照上述在 拖放单个对象和已保存的探索 部分中的说明操作。

要在 Gaia 管理应用程序中建立一个由版本化对象集支撑的企业地图层（EML），请按照上述在 使用函数支撑的企业地图层 部分中的说明操作。将 EML 类型设置为 Versioned object set EML 而不是 Foundry function EML；您不需要插入 函数 rid。

现在，您可以通过使用左侧面板中的 数据源 选项卡搜索其名称，将由版本化对象集支撑的 EML 添加到您的 Gaia 地图中；Gaia 将 EML 分组为 Reference data layers，位于 精选数据源 下。
