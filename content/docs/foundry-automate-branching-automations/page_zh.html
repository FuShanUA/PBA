分支自动化

Automate 与全局分支集成，以实现安全、隔离的自动化开发。本文档涵盖在分支上处理自动化的相关内容，包括添加和修改资源、跨应用兼容性、合并要求、变基以及已知限制。

有关全局分支概念和工作流程的一般信息，请参阅 全局分支文档。

添加、删除和修改资源

将自动化添加到分支

要将自动化添加到分支中：

  1. 导航到分支上的自动化文件，或使用页面右上角的分支选择器选择指定的分支。从那里，选择添加到分支。
  2. 进行编辑并保存自动化文件。现在，自动化已成为该分支上的已保存资源。

从分支中移除自动化

要从分支中移除自动化，请使用右下角的侧边栏并选择从分支中移除。从分支中移除自动化会丢弃对分支的所有修改，并停止所有效果的执行。

修改自动化

要在分支上修改自动化，请进行任何更改并保存，就像您在主分支上所做的那样。

在分支上修改自动化中的name和description也会修改main上的这些值。

支持的条件和效果

并非所有 Automate 条件目前都支持在分支上使用。当您在分支上构建自动化时，不支持的条件会在条件选择器中被禁用。

以下条件是< strong >支持在分支上的：

  • 时间: 当达到特定时间时触发；例如，“每周一上午9点。”
  • 对象（对象）: 当一个对象被添加到集合中时触发。
    • 添加到集合中的对象：当在预定义的对象集合中出现新对象时触发。
  • 从集合中移除的对象：当对象离开预定义的对象集合时触发。
  • 集合中修改的对象：当对象在预定义的对象集合中被修改时触发。
  • 对所有对象运行：定期对给定集合中的所有对象执行操作。

以下条件目前在分支中不受支持：

  • 阈值跨越：当指标阈值被跨越时触发并保持在触发状态。
  • 自动化依赖：在另一个自动化完成之后触发。参见 已知限制。
  • 时间序列：当时间序列阈值被跨越时触发。
  • 流 [Beta]：当在 流 中有新记录时触发。
  • 指标变化 [终止]：当聚合对象集合指标增加或减少时触发。

所有 Automate 效果 都在分支上受支持：

  • 操作（Action）: 当条件满足时执行一个操作。
  • 逻辑（Logic）: 当条件满足时执行一个 AIP 逻辑函数，然后对生成的操作进行暂存或应用。
  • 函数（Function）[Beta]: 条件满足时执行一个函数。这些函数不能执行本体论（Ontology）编辑。
  • 通知（Notification）: 向选定的接收者发送通知，可选地附带便签（Notepad）附件。

跨应用程序兼容性

自动化引用平台上的各种资源，包括对象类型、操作（Action）、AIP 逻辑函数（AIP Logic functions） 和 Foundry 函数（Foundry functions） . 在一个分支上，自动化引用了这些资源的分支状态：

  • 分支对象类型和操作类型： 自动化可以引用在同一分支上创建或修改的对象类型和操作类型。
  • 分支函数版本： 自动化可以引用分支函数版本，例如 AIP Logic 或 Foundry 函数的 Branched pre-release 版本。
  • 分支作用域执行： 一个分支自动化在其分支上评估其条件并执行其效果。有关在分支上运行的效果的影响，请参阅 管理分支上的副作用。

合并要求

在从分支部署自动化到 main 之前，必须通过以下检查：

  • 审批已满足： 在分支可以合并之前，自动化所需的所有 审批 必须已完成。请参见下面的 评审者体验。
  • 自动化已基于 main 重新基准化： 在合并之前，如果对自动化 main 分支进行了更改，请将这些更改重新基于您的分支进行变基。
  • 自动化处于有效状态： 自动化在保存时会持续进行有效性检查，因此此检查通常会通过。然而，如果自动化所依赖的资源（例如对象类型、操作类型或函数）被删除，则自动化可能会变得无效。

受保护的自动化

为了确保所有对自动化的编辑都经过分支和审查流程，您可以保护该自动化的 main 分支。

要保护自动化 main 分支，请导航到文件系统中的相关资源并选择 分支保护 > 使用项目策略保护。您也可以从右上角的主要自动化视图中使用 查看详细信息 选项来执行此操作。

当自动化 main 分支被保护时，对 main 的编辑需要用户在保存时通过一个全局分支。

评审员体验

一旦创建了提案，可以通过Global Branching应用程序或在Automate中的Approvals横幅中将评审员添加到自动化中。作为评审员添加的用户会收到一封电子邮件，请求他们进行评审并附有提案的链接。

选择 管理 以添加评审员并查看提案合并前必须满足的审批策略。

当自动化有等待审查的更改时，页面顶部的横幅会显示已满足的批准数量。

从那里，审查者可以通过选择开始审查打开审查对话框，查看main与分支更改的并排比较。然后他们可以通过选择您的审查选项来批准或拒绝更改。

一旦满足所有必要的批准，更改将被批准，并且可以为该资源进行合并。

变基和冲突解决

当main自您的分支创建以来或上次变基后已被修改时，需要进行变基。

在自动化顶部会出现一个横幅，提示您将分支与最新更改进行变基。

要变基您的分支，请在横幅中选择开始变基。 变基自动化对话框会打开，显示主分支和当前分支版本的并排比较。 选择保留主分支或当前分支版本，然后选择完成变基以应用您的选择。

如果需要同时进行重置和批准，则首先显示重置对话框，并且必须在审核人员可以审核更改之前解决。

已知限制

  • 并非所有条件都支持在分支上使用。有关完整列表，请参阅 支持的条件和效果。
  • 重置要求您选择 main 的更改或您分支上的更改。目前尚无法以更细粒度的方式解决差异。
