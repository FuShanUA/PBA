对象集条件

当对象数据在预定义的对象集中发生变化时，可以自动触发自动化效果。

数据源： 变化可能来自 支撑数据集（backing datasets）、操作类型（action types） 或 外部应用程序（external applications）。

评估频率： 数据变化通常在几分钟内被检测到。 了解更多

效果输入参数： 某些对象集条件会暴露效果输入参数。这些变量可以在效果中使用，以访问触发自动化的对象。

可用条件

  • 添加到集合中的对象: 当集合中出现新对象时触发。
  • 从集合中移除的对象: 当一个对象离开集合时触发。
  • 在集合中修改的对象: 当集合中的对象被修改时触发。
  • 对所有对象运行: 定期对给定对象集合中的所有对象执行操作。
  • 阈值已超过: 当指标阈值被超过或函数返回 true 时触发。
  • 指标已更改 [停用]: 当聚合指标增加或减少时触发。

对象集合定义

大多数对象集合条件类型需要指定要监控的对象集合。您可以：

  • 动态定义一个对象集合；
  • 选择一个已保存的对象集合；或者
  • 选择一个返回对象集合的函数。
    • 函数是 Automate 更高级使用的重要构建模块。

在定义好要监控的对象集后，选择预览以验证对象集是否包含预期的对象。

当动态定义一个对象集时，选择您想要的对象类型，然后可选地添加过滤器以缩小范围。例如，您可以监控一个Support Ticket对象类型（如下所示），并将其过滤为“状态 = Open”。当没有选择过滤器时，您将监控该类型的所有对象。对象类型和过滤器的选择决定了对象集是否可以实时监控或按计划监控。了解更多关于评估频率的信息。

函数生成的对象集

下面的示例使用一个函数来返回与用户关联的支持工单。在新增对象条件下使用时，每当新工单进入此集合时，效果就会运行。该函数必须返回一个ObjectSet<T>才能在此处使用。

请注意，在评估期间，此函数会为系统执行一次，然后为每个订阅者执行一次。

Copied!
1/**
2 * This function returns support tickets that are linked to users
3 */
4@Function()
5public getSupportTicketsLinkedToUsers(): ObjectSet<DemoTicket> {
6    return Objects.search()
7        .users()
8        .searchAroundSupportTickets()
9}

条件类型

添加到集合中的对象

添加到集合中的对象条件会在每次有对象被添加到您的预定义对象集合时触发。首先根据上面的对象集合定义部分指定您的对象集合。

如果指定了对象集合过滤器，其他用户新增的对象或对对象的编辑可能会触发你的对象被添加到集合条件中。在下面的示例中，我们监控一个 Support Tickets 的对象集合，其状态属性值为 Open。该条件可以通过两种方式触发：创建一个新的 Support Ticket 对象，其状态为 Open，或者现有的 Support Ticket 对象的状态更改为 Open。

如果你没有对对象集合指定任何过滤器，那么你实际上是在监控该类型的全部对象。这意味着自动化操作只有在完全新的对象被添加到本体论时才会触发。

添加到集合中的对象 条件会将触发该条件的对象作为效果输入暴露出来。这可以是单个 Added object，也可以是 Added objects 的对象集合，或者是 Added objects 的属性。了解更多关于如何使用效果输入。

支持的 评估频率：

  • 实时评估
  • 定时评估

从集合中移除的对象

从集合中移除的对象条件每次对象从您预定义的对象集合中被移除时都会触发。首先根据上面的 对象集合定义部分 指定您的对象集合。

如果您指定了一个对象集合过滤器，用户对对象的编辑可能会触发对象从集合中被移除的条件。例如，如果我们监控一个 Support Tickets 的对象集合，其状态属性值为 Open，那么当 Open 工单的状态更改为任何其他状态时，该条件就会被触发。

请注意，从本体论中删除的对象不会触发。

从集合中移除的对象条件将触发该条件的对象作为效果输入进行暴露。这可以是单个 Removed object 或者 Removed objects 的对象集合。 了解有关如何使用效果输入的更多信息。

支持的 评估频率：

  • 实时评估
  • 计划评估

在集合中修改的对象

在集合中修改的对象 条件每次在您预定义的对象集合中的对象被修改时都会触发。使用此条件类型来监控对象集合中的任何属性值更改。如果您只想监控 特定 值的更改，请改用 添加到集合中的对象 条件。

首先根据上面的 对象集合定义部分 指定您的对象集合。然后，您可以选择性地选择要监视的特定属性。如果没有选择属性，则监控对象集合中的任何属性更改都会触发自动化。

此外，您可以指定 包含新增对象 和 包含移除对象 这些选项控制对已添加到（或从）被监控对象集中进行的修改是否会触发自动化。当两个选项都禁用时，只有那些已经存在于对象集中并在修改后仍保留在对象集中的对象才会触发自动化。

集合中被修改的对象条件会将一个单独的Modified object或一组Modified objects作为效果输入暴露出来。 了解更多关于效果输入的信息。

支持的 评估频率：

  • 实时评估

在下面的示例中，我们正在监控 Support Tickets 集合中具有 USA 位置的对象的修改情况。

  • 包括新增对象：启用。 被添加到对象集中的对象被视为已修改。因此，一个从 UK 改变为 USA 位置的 Support Ticket 会触发自动化，任何具有 USA 位置的新创建工单也会触发自动化。
  • 包括移除对象：禁用 . 状态属性从Open变为Closed，同时位置发生变化（例如，从USA变为UK）的工单不会触发自动化，因为这些对象已经离开了集合。
  • 被删除的Support Tickets不会触发自动化，因为它们不再属于被监控的对象集合。

对所有对象运行

对所有对象运行条件可以定期在给定对象集合中的所有对象上运行效果。此条件需要与一个计划一起配置。

当使用此条件时，可以设置受影响的对象参数以将对象注入到任何效果中。在使用此条件时，会尊重批量大小和并行化设置。此条件允许您在一个大型对象集合上运行一个效果，同时避免函数或操作超时，因为批量处理会得到处理。

支持的 评估频率：

  • 计划评估
  • 自动化依赖评估

阈值超出

当所有定义的检查返回 true 时，阈值超出 条件会触发。每当阈值在任一方向上被超出时，都会触发效果并记录活动。

您可以组合由 Automate定义的逻辑、函数支持的逻辑 或两者组成的条件。

支持的 评估频率：

  • 计划评估

自动化定义的逻辑

要在自动化界面中定义阈值条件，请按照以下步骤操作：

  1. 在条件组合部分，选择添加条件。
  2. 将条件类型选择为条件。
  3. 指定一个已保存的对象集作为指标输入。
  4. 在选择一个指标下拉菜单中，选择您要比较的属性。
  5. 选择一个比较运算符。
  6. 选择一个指标或静态值来与该属性进行比较。

下面显示了一个在自动化界面中定义逻辑的阈值超出条件示例。该条件检查纽约所有空气质量传感器的最大值是否小于0.05。

函数支撑的逻辑

使用函数支撑 逻辑条件不支持接口定义的复杂场景。函数支持的条件通过定义和发布一个返回布尔值（true 或 false）的 函数 来工作。在评估期间调用该函数，当状态发生变化时会记录事件。

了解更多关于 为 Automate 编写函数 的信息。

下面的示例函数计算是否超过 50% 的空气质量传感器不健康。如果满足条件，函数将返回 true；如果不满足条件，函数将返回 false。

Copied!
1/**
2 * This function calculates whether more than 50% of the passed air quality sensors are considered unhealthy.
3 * A sensor is considered unhealthy when the carbon monoxide value passes `0.05`.
4 */
5@Function()
6public async areMoreThanFiftyPercentOfSensorsUnhealthy (airQualitySensors: ObjectSet<automateExampleAirQualitySensor>): Promise<boolean> {
7    const totalSensorCount = await airQualitySensors.count();
8    const unhealthySensorCount = await airQualitySensors.filter(sensor => sensor.carbonMonoxide.range().gt(0.05)).count();
9    if (totalSensorCount === null) {
10        return false;
11    }
12
13    return ((unhealthySensorCount ?? 0) / totalSensorCount) > 0.5;
14}

指标已更改 [日落]

日落

指标已更改 条件处于 日落 在开发阶段，将来可能会被弃用。目前仍提供完整支持，您可以使用此条件编辑现有的自动化。我们建议改用 集合中修改的对象 条件。

指标变化 条件会在预定义的对象集合中对象指标发生变化时触发。示例指标包括集合中的对象总数，或如数值对象属性值的平均值等聚合值。首先根据上面的 对象集合定义部分 指定您的对象集合。

选择要监控的指标 属性：

  • 对象集合中对象的 总数量；或者
  • 在监控的对象集合中，某个数值属性的 聚合值（平均值、最大值、最小值或总和）。

从 变化类型 下拉菜单中，选择指标是 增加 还是 减少。例如（如下所示），当纽约所有空气质量传感器的平均一氧化碳值增加时，可以触发自动化。

支持的 评估频率：

  • 计划评估
