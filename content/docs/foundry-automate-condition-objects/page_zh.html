对象集条件

当预定义对象集内的对象数据发生变化时，系统会自动触发自动化效应。

数据来源：数据变更可能源于底层数据集、操作类型或外部应用。

评估频率：数据变更通常会在几分钟内被捕获。了解更多

效应输入参数：部分对象集条件会公开效应输入参数。这些变量可在效应内部使用，用于访问触发该自动化的对象。

可用条件

  • 对象添加至集合：当集合中出现新对象时触发。
  • 对象从集合中移除：当对象离开集合时触发。
  • 集合中的对象被修改：当集合中的对象发生修改时触发。
  • 对所有对象运行：定期对给定对象集中的所有对象运行效应。
  • 阈值突破：当指标突破阈值或函数返回 true 时触发。
  • 指标变更 [已废弃]：当聚合指标增加或减少时触发。

对象集定义

大多数对象集条件类型都要求指定一个要监控的对象集。你可以：

  • 动态定义对象集；
  • 选择已保存的对象集；或
  • 选择返回对象集的函数。
    • 函数是自动化功能实现更高级用法的强大构建模块。

定义好要监控的对象集后，选择 预览 以验证该对象集是否包含预期的对象。

动态定义对象集时，选择所需的类型，然后视情况添加过滤器以缩小范围。例如，你可以监控 Support Ticket（支持工单）对象类型（如下图所示），并将其过滤为“状态 = Open（开启）”。未选择任何过滤器时，系统将监控该类型的所有对象。对象类型和过滤器的选择决定了该对象集是支持实时监控还是按计划监控。详细了解评估频率。

集合中的对象已修改

集合中的对象已修改条件会在每次对象在您预设的对象集合中发生修改时触发。使用此条件类型可监控对象集合中的任何属性值更改。如果您只想监控对特定值的更改，请改用对象添加到集合条件。

首先，按照上方的对象集合定义部分指定您的对象集合。然后，您可以选择要监视的特定属性。如果未选择任何属性，则受监控对象集合中的任何属性更改都将触发自动化。

此外，您可以指定包含添加的对象和包含移除的对象。这些选项控制对添加到（或从受监控对象集合中移除的）对象的修改是否会触发自动化。当两个选项均禁用时，只有已存在于对象集合中且在修改后仍保留在对象集合中的对象才会触发自动化。

集合中的对象已修改条件会将触发该条件的单个Modified object（已修改对象）或一组Modified objects（已修改对象）作为效果输入公开。了解有关效果输入的更多信息。

支持的评估频率：

  • 实时评估

在下面的示例中，我们正在监控位置为USA的Support Tickets集合中已修改的对象。

  • 包含添加的对象：已启用。添加到对象集合的对象被视为已修改。因此，位置从UK更改为USA的Support Ticket将触发自动化，任何位置为USA的新建工单也会触发自动化。
  • 包含移除的对象：已禁用。状态属性从Open更改为Closed且位置同时更改（例如，从USA更改为UK）的工单不会触发自动化，因为这些对象已离开集合。
  • 已删除的Support Tickets不会触发自动化，因为它们不再处于受监控的对象集合中。

在所有对象上运行

在所有对象上运行条件可以定期对给定对象集合中的所有对象运行效果。此条件需与计划结合配置。

使用此条件时，可以设置受影响对象参数，以将对象注入到任何效果中。使用此条件时，会遵守批处理大小和并行化设置。此条件允许您对大量对象运行效果，同时避免函数或操作超时，因为系统会自动处理批处理。

支持的评估频率：

  • 定时评估
  • 依赖于自动化的评估

阈值跨越

当所有定义的检查都返回true时，阈值跨越条件会触发。无论阈值朝哪个方向跨越，都会触发效果并记录活动。

您可以组合由自动化定义逻辑、函数支持逻辑或两者兼有的条件。

支持的评估频率：

  • 定时评估

自动化定义逻辑

要在自动化界面中定义阈值条件，请按照以下步骤操作：

  1. 在条件组合部分，选择添加条件。
  2. 将条件类型选择为条件。
  3. 指定一个已保存对象集作为指标输入。
  4. 在选择指标下拉菜单中，选择需要进行比较的属性。
  5. 选择比较运算符。
  6. 选择用于与该属性进行比较的指标或静态值。

下方展示了在 Automate 界面中定义逻辑的阈值跨越条件示例。该条件用于监测纽约所有空气质量传感器的最大值是否小于 0.05。

函数支持逻辑

当界面定义的条件无法支持某些复杂场景时，请使用函数支持的逻辑条件。函数支持条件的工作原理是定义并发布一个返回布尔值（true 或 false）的函数。系统在评估期间会调用该函数，并在状态发生变化时记录事件。

有关为 Automate编写函数的更多信息，请参阅相关文档。

下方的示例函数用于计算超过 50% 的空气质量传感器是否处于异常状态。如果满足条件，函数将返回 true；如果不满足条件，函数将返回 false。

1/**
2 * 该函数用于计算传入的空气质量传感器中，不健康状态的比例是否超过 50%。
3 * 当一氧化碳值超过 `0.05` 时，传感器即被视为不健康状态。
4 */
5@Function()
6public async areMoreThanFiftyPercentOfSensorsUnhealthy (airQualitySensors: ObjectSet<automateExampleAirQualitySensor>): Promise<boolean> {
7    const totalSensorCount = await airQualitySensors.count();
8    const unhealthySensorCount = await airQualitySensors.filter(sensor => sensor.carbonMonoxide.range().gt(0.05)).count();
9    if (totalSensorCount === null) {
10        return false;
11    }
12
13    return ((unhealthySensorCount ?? 0) / totalSensorCount) > 0.5;
14}

指标变更 [停止使用]

停止使用

指标变更条件目前处于开发生命周期的停止使用阶段，未来将被弃用。目前仍提供完整支持，您可以继续编辑使用该条件的现有自动化流程。我们建议改用集合内对象已修改条件。

每当预定义对象集合中的对象集指标发生变更时，指标变更条件就会触发。常见的指标示例包括集合中的对象总数，或者数值型对象属性值的平均值等聚合数据。请首先按照上方的对象集定义部分指定您的对象集。

选择要监控的指标属性：

  • 对象集合中对象的总数；或
  • 受监控对象集合中某个数值属性的聚合值（平均值、最大值、最小值或总和）。

在变更类型下拉菜单中，选择指标是增加还是减少。例如（如下图所示），当纽约所有空气质量传感器的一氧化碳平均值上升时，自动化流程就会触发。
