为 4.6C/620/640 安装远程智能体

面向 SAP 应用的 Palantir Foundry Connector 2.0（简称“连接器”）针对 4.6C/620/640 的远程智能体以 SAP Request 的形式发布，需通过 SAP 传输管理系统进行传输。您可以按照以下步骤将请求导入 SAP 系统。

安装步骤

  1. 下载安装包。
  2. 将 4.6c、4.70 (620/640) 的连接器远程智能体解压到一个文件夹中。如果 SAP 系统是 Unicode 系统，则应使用 Unicode 请求；否则应使用非 Unicode 请求。文件如下所示（请求编号可能因要安装的版本而异）：
  • 连接器文件
    • K900xxx.C46
    • R900xxx.C46
  • 远程智能体文件
    • K900xxx.C46
    • R900xxx.C46
  1. 以“R”开头的文件名是数据文件；以“K”开头的文件名是协同文件。将文件相应地复制到 SAP 应用服务器上下文列出的文件夹中：
    /usr/sap/trans/cofiles
    /usr/sap/trans/data
  1. 使用有权限操作 STMS（SAP 传输管理系统）的用户登录 SAP 系统。
  2. 运行 STMS 事务。
  3. 选择传输的目标系统。
  4. 从工具栏菜单中，选择 Extras > Other Requests > Add，然后输入请求编号。（请求编号格式如下：C46K9000xx。前 3 位数字是请求文件的扩展名；其余数字是解压文件中显示的 K 文件名。）
  5. 该请求会列在导入队列中。点击请求编号选中该请求，然后在工具栏上选择 Import。
  6. 转到 Options 选项卡，选择 Leave Transport Request in Queue for Later Import 和 Ignore Invalid Component Version。然后开始传输。
  7. 点击工具栏上的日志按钮，检查导入日志。
  8. 检查是否有任何错误消息。成功的导入过程应无误完成。
  9. 转到 PFCG 事务代码，为以下角色生成授权配置文件：
    • /PALAGT47/SERVICE_USER
    • /PALAGT47/CONTENT_RTABLE_ALL

配置

连接器与连接器远程智能体通过 SAP 远程函数调用（RFC）进行通信。因此，需要建立两个 RFC 连接：一个从连接器指向远程智能体，另一个从远程智能体指向连接器。下一节将详细介绍如何创建这些 RFC 连接。

RFC 配置

RFC 配置包含四个步骤：

  1. 创建一个指向 SAP 系统的 RFC 连接。
  2. 创建一个从连接器指向远程 SAP 系统的 RFC（SOURCE）。
  3. 创建一个从远程 SAP 系统指向连接器的 RFC（TARGET）。
  4. 通过 Web 浏览器测试连接器远程智能体。

创建指向源/目标系统的 RFC 目标连接

本节需要建立两个 RFC 连接：一个从连接器指向远程智能体，另一个从远程智能体指向连接器。请查阅《创建 RFC 连接指南》。

针对从远程 SAP 系统指向连接器的目标 RFC 定义，重复上述相同步骤。与其命名为 SAP\_SOURCE，您可以将其命名为 SAP\_TARGET。（这些名称可自定义。）

配置远程智能体并注册连接器

  1. 登录主连接器系统。
  2. 运行事务 /n/PALANTIR/PARAM_A1。
  3. 输入以下参数值：
    • Agent ID: 智能体标识符（也称为 CONTEXT）
    • Is 4.7 or older: 智能体版本标记
    • Agent Desc: 智能体描述（供参考使用）
  4. 运行事务 /n/PALANTIR/PARAM_A2 并输入以下参数：
    • Agent ID: 智能体 ID（按上一步骤定义）
    • Param ID: 参数 ID（参数的分类）
    • Param Name: 参数名称（供智能体使用）
    • Param Value: 参数值（供智能体使用）
参数 ID参数名称参数值描述
RFCSOURCE从连接器到远程 SAP 系统的 RFC 连接。
RFCTARGET从远程 SAP 系统到连接器的 RFC 连接。
SYSTEMCPU_CHECKTRUE / FALSE启用或禁用 CPU 检查。
SYSTEMMEMORY_CHECKTRUE / FALSE启用或禁用内存检查。
SYSTEMRESOURCE_CHECKTRUE / FALSE启用或禁用资源检查。如果为 FALSE，则全部检查均被禁用；如果为 TRUE，则会检查其他参数（CPU_CHECK 和 MEMORY_CHECK）。
SYSTEMCONTINUOUS_RESOURCE_CHECKTRUE / FALSE对所有请求（初始化及所有分页请求）启用资源检查。如果为 FALSE，则仅对初始化请求执行资源检查。
SYSTEMPROCESS_CHECKTRUE / FALSE启用对最小允许工作进程数的检查；需与 PROCESS_MIN_BG 和 PROCESS_MIN_DIA 配合使用。

局限性

用于 4.6C/620/640 的连接器远程智能体存在以下局限性：

  • 仅支持 table 接口。
  • 仅支持 single 增量类型。
  • 必须在 Foundry 的《数据连接源配置》中设置 useTsvFormat:true。因此，即使 Foundry 中已存在用于主连接器实例的源，也建议为 4.6C/620/640 单独创建一个源。
