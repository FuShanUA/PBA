查看标记变更的影响

你可以使用数据血缘关系来评估数据集标记的变更会对派生数据集产生怎样的影响。这在移除标记时非常实用。

警告

标记模拟依赖于最近的数据集构建结果，且不会考虑尚未最终确定的变更。请确保你当前操作的是最新版本的数据。

进入模拟模式

  1. 打开访问信息侧边栏。
  2. 开启模拟访问要求开关。
  3. 在图表上选择任意数据集。
  4. 点击编辑标记。

模拟标记变更

若要模拟标记的应用，请搜索想要应用的标记，勾选该标记旁边的复选框，然后选择模拟变更按钮。

数据集中已应用的标记将显示为选中状态。若要模拟移除标记，请取消勾选该标记旁边的复选框并点击模拟变更。

你只能移除直接应用于数据集上的标记。无法模拟移除通过数据集血缘关系继承的标记，或是从父项目继承的标记。

分析模拟图表

在模拟模式下，图表的颜色将指示受标记变更影响的数据集。图表颜色已在界面中标注，可表示以下数据集状态：

  • 已应用模拟变更：显示在已应用变更的数据集上。
  • 访问受影响：显示在变更前后标记发生变动的数据集上。
  • 访问不受影响：显示在变更前后标记保持不变的数据集上。
  • 无可见事务：显示在尚未构建的数据集上，或你没有权限查看事务的数据集上。

选择任意数据集后，访问信息侧边栏将展示模拟的访问要求。你可以随时开启或关闭模拟模式来对比差异，且不会丢失任何已设置的模拟变更。

理解变更的提示

在进行变更前，建议查阅《标记文档》，进一步了解标记对用户的影响。

在模拟标记时，请考虑以下几点：

  • 数据集可以通过代码停止传播标记。
    • 在权限配色中，数据血缘关系图上停止传播标记的节点会显示数据访问已通过代码修改。此信息也会显示在节点属性侧边栏的访问信息部分。
    • 在代码助手中，你可以通过检索 stop_propagating 关键词来检查数据集的代码，确认其是否停止了标记传播。
  • 数据集可能会从其他输入传播获得标记；点击数据集节点左侧的箭头即可展开其输入项。
  • 标记可以应用于父项目或文件夹；未开启模拟模式时，标记左侧会显示一个文件夹图标；开启模拟模式时，标记模拟菜单中也会显示对应的文件夹图标。
