快速搜索

Quicksearch 是 Palantir 平台中用于快速、便捷搜索、导航和发现的工具。您可以通过左侧边栏中的 搜索 图标访问 Quicksearch，或者使用 Cmd+J（macOS）或 Ctrl+J（Windows）。Quicksearch 包含两种视图模式：

  1. 跳转模式： 提供一个简短的个性化结果列表，直接引导用户到达可用内容的主要类型：平台应用程序、自定义应用程序、对象、数据集和其他资源。
  2. 完整结果视图： 设计用于帮助用户查找内容并发现平台中存在的内容。用户可以使用高级筛选器搜索 Palantir 应用程序、对象、数据集和其他文件，并找到最相关的结果以及丰富的元数据。

跳转模式

  • 易于访问： 要打开 Quicksearch，请选择 搜索... 在导航侧边栏中，或使用快捷键 Cmd+J(macOS) 或 Ctrl+J (Windows)。
  • 导航聚焦： 初始对话框和下拉菜单是面向导航的。开始输入以获得建议。使用键盘跳转到平台上的任何页面。
  • 仅按标题搜索： 跳转模式仅在应用程序、资源和对象的标题上进行搜索。要搜索其他字段和元数据，请使用完整结果模式。
  • 单击进入完整结果： 从对话框和下拉菜单中，按下 Enter/Return 或通过 应用、对象、数据集 或 文件 过滤器进入高级搜索模式。
  • 个性化结果： 快速搜索中的结果根据您最近访问或收藏的资源进行个性化，并且更多。

完整结果视图

  • 发现聚焦： 专为搜索和发现而设计，用户可以比较多个搜索结果，以找到对他们最有用的资源。
  • 在不同字段中搜索： 可以搜索描述、列名、文件路径等。此外，还可以通过创建者、标签、项目、文件夹等进行筛选。
  • 标签页： 该界面包含一个顶部结果标签页以及四个用于筛选的标签页： 应用，对象，数据集 或 文件。
  • 元数据： 用户可以查看每个结果的信息，以评估该资源是否与他们相关。可以通过关键词高亮、文件路径、浏览次数、最后更新时间、关键安全元数据等进行搜索。
  • 排序： 结果根据包含文本匹配和其他提升参数的算法进行排序。
  • 筛选器： 提供筛选器用于高级搜索。此外，通过从初始搜索下拉菜单中选择筛选标签页，用户可以执行复杂的筛选搜索（例如，“项目X下的所有报告，且由我创建”）。
  • 权限： Quicksearch 会尊重平台中所有现有的权限。具有 Discover 权限的内容将打开 Request access 消息。用户不会看到他们没有访问权限的内容。

Quicksearch 不在平台上搜索所有对象实例。Quicksearch 仅限于搜索 250 种对象类型，优先级按 Active 对象类型，然后是 Prominent，接着是 Normal，最后是 Experimental 状态（已弃用和隐藏的对象类型不会被搜索）。如果用户找不到他们想要的内容，他们会被提示尝试在 对象浏览器 中进行搜索，在那里他们可以调整他们的搜索以仅针对特定的对象类型组，甚至特定的对象类型。

筛选器

您可以通过 Quicksearch 中的筛选器在平台上快速找到所需的资源和数据。通过选择 应用、对象、数据集 或 文件 结果类型来筛选搜索结果。您还可以通过在文件路径、标签或项目中进行搜索进一步筛选。一旦应用了筛选条件，搜索结果视图将只显示您所选择的结果类型。

例如，我们想在我们的注册中搜索“汽车”相关的资源。

应用

要搜索由 Palantir 应用构建界面创建的模块、工作区或其他应用程序，您可以按 应用 进行筛选。

在我们的“汽车”搜索示例中，我们可以看到三个 Workshop 模块、一个 Carbon 工作区和一个 Slate 文档，它们被正确识别为在 Palantir 中构建的应用程序。

对象

要仅查看基于本体论对象类型的结果，请按对象进行筛选。在此搜索结果视图中，您可以从与您的搜索词匹配的对象类型中进行选择，选择编辑或查看对象类型的谱系，并进一步深入到单个对象。

当搜索“汽车”时，我们看到几种对象类型和对象结果，可以进一步筛选到如Award或Auto Parts这样的选定对象类型。

数据集

当按数据集筛选时，结果视图将显示平台中与搜索词在数据集名称或列名称中匹配的可用数据集列表。

使用相同的“汽车”示例，我们可以看到一个包含38个数据集的列表，其中数据集和列名中包含匹配的术语。

文件

在快速搜索中使用文件标签，可以快速找到添加到或在平台上创建的单个资源文件。使用左侧面板中的过滤器来专门搜索特定类型的文件或资源。

当我们搜索“汽车”时，会得到多个文件结果，包括文件夹、图、图像和建模目标。
