可用事件和动作

本页面列出了 Slate 在查询、函数、变量、组件和应用程序级别的所有可用事件和动作。如需了解如何配置和使用这些事件和动作，请参阅我们的《补充文档》。

全局事件

全局事件是指在应用程序级别发生的事件，不特定于单个组件或查询。

slate.ready

页面框架加载完毕且查询开始运行时，将触发 slate.ready 事件。任何希望在页面加载时立即触发的动作（例如，执行某些手动查询）都应在此处进行。

slate.resize

每当浏览器窗口大小发生改变时，将触发 slate.resize 事件。如果某些组件的尺寸基于浏览器窗口大小，它们也会随之调整。不过，部分组件（如图表）具有 redraw 动作，你可以触发该动作，以确保即使浏览器窗口大小发生变化，组件也能重绘为正确的尺寸。

slate.onNavigate[page_name]

当用户导航至添加了该事件的页面时，将触发 slate.onNavigate[page_name]。

slate.onPrint

当用户关闭此前通过 slate.print 动作打开的打印模态框时，将触发 slate.onPrint。

slate.userStorageChanged

slate.userStorageChanged 用于检测用户存储是否已通过 slate.setUserStorage 动作进行更新。如果用户存储是通过其他浏览器标签页或窗口更新的，同样支持触发该事件。

slate.getMessage

slate.getMessage 面向存在于其他应用（如 Workshop）iframe 中的 Slate 应用。当 Slate 从父 iframe 收到如下格式的 postMessage 时，就会触发 slate.getMessage 事件：

已复制！
1{ "target": "slate-parent-iframe-event", "message": <payload> }

如果子窗口或父窗口发送的 postMessage 中包含设置了以下 target 属性的对象，也会触发 slate.getMessage 事件：



已复制！
1{ "target": "SLATE_MESSAGING_SERVICE", ...<payload> }

事件触发后，在该事件对应的 JavaScript 面板中，可以通过 {{slEventValue}} 获取 <payload>，以便合理使用消息中的数据。

发送至 Slate 的消息还能执行特定操作，例如设置变量：



Copied!
1{
2    "slateActionOption": {
3        "type": "SET_VARIABLES",
4        "payload": {
5            "v_variable": 123
6        }
7    }
8}

你也可以执行操作：



已复制！
1{
2    "slateActionOption": {
3        "type": "TRIGGER_ACTION",
4        "payload": <payload for action>
5    }
6}

slate.getBroadcast

每当 Slate 从其已打开的 BroadcastChannel 接收到消息时，就会触发 slate.getBroadcast。若要打开 BroadcastChannel，请使用 slate.createBroadcastChannel 操作。在接收消息之前，需要先打开一个通道。

消息的有效负载将可通过 {{slEventValue}} 获取，其结构如下：



已复制！
1{
2  "channel": "channel_name",
3  "data": <payload>
4}

slate.onCloseCheckpoint

当用户关闭检查点（Checkpoint）时，会触发 slate.onCloseCheckpoint。该事件会提供所关闭检查点的相关信息，可通过 {{slEventValue}} 获取，其结构如下：



已复制！
1{
2    "slateCheckpointId": string,
3    "status": "cancelled" | "no-checkpoint" | "success",
4    "interactionRid": string (可选 - 如果 Checkpoint 成功)；
5}

slateCheckpointId 是用户定义的字符串，用于标识 Checkpoint。在使用 slate.showCheckpoint 操作时会对其进行定义。

您可以在《检查点文档》中阅读有关检查点的更多内容。

slate.onChildWindowClosed

当使用 slate.redirectToUrl 全局操作打开的子窗口关闭时，将触发 slate.onChildWindowClosed。在调用 slate.redirectToUrl 时，必须使用 windowSharingOptions 属性为该窗口分配一个 id，否则此事件将不会被触发。

消息载荷 {{slEventValue}} 具有以下结构，其中 id 即为已关闭窗口的 ID：

Copied!
1{
2    "id": string | number
3}

slate.viewSaved

通过 操作 下拉菜单或 slate.saveView 操作保存 可共享视图 后，将触发 slate.viewSaved。 {{slEventValue}} 包含了新创建视图的视图 ID（字符串）。

此事件可与 slate.loadView 操作配合使用，以构建保存和加载工作流。例如，您可以存储来自 {{slEventValue}} 的视图 ID，随后将其传递给 slate.loadView 以恢复该视图。

查询事件

查询事件是指在 Slate 应用中与查询相关的事件，这些事件在查询完成时，或由于超时、查询格式错误或其他服务器问题导致查询未能成功完成时触发。每个查询都有各自独立的事件实例。

QUERY_NAME.success

每当由 QUERY_NAME 指定的查询成功完成时，就会触发此事件。每个查询都有各自独立的事件实例。

QUERY_NAME.failure

每当由 QUERY_NAME 指定的查询未能成功完成时，就会触发此事件，原因可能包括超时、查询格式错误或其他服务器问题。每个查询都有各自独立的事件实例。

函数事件

函数事件是指与 Slate 函数执行相关的事件。每个函数都有各自独立的事件实例。

FUNCTION_NAME.ran

每当由 FUNCTION_NAME 指定的函数运行结束时，就会触发此事件。每个函数都有各自独立的事件实例。

变量事件

变量事件是指与 Slate 变量交互相关的事件。每个变量都有各自独立的事件实例。

VARIABLE_NAME.changed

每当由 VARIABLE_NAME 指定的变量发生更改时，就会触发此事件。更改后的值将通过 {{slEventValue}} 获取。每个变量都有各自独立的事件实例。

组件事件

组件事件是用于触发 Slate 应用内操作或活动的触发器。它们可用于在应用内创建交互并自动化流程。每个组件都有其独立的事件实例。

WIDGET_ID.cssClassesUpdated

当指定组件 WIDGET_ID 的“附加 CSS 类”属性发生改变时，便会触发此事件。该事件在 DOM 更新之后、但在渲染之前触发；这意味着，如果你正在使用 CSS 类来更改不同组件的尺寸，利用此事件触发这些组件的 redraw 操作将达到预期效果。每个组件都有其独立的事件实例。

WIDGET_ID.transitionend

只要在 WIDGET_ID 指定的组件内部直接发生 JavaScript transitioned 事件，就会触发此事件。这里的“直接”是指并非源自子组件；每次 transitioned 仅触发一次事件。具体而言，你可以使用 CSS 过渡效果为页面上的元素设置动画，例如让侧边栏滑入和滑出，而不是直接弹出和消失。每当这类过渡效果完成时，JavaScript 就会触发一个 transitioned 事件，Slate 会捕获并将其传递下去。如此一来，你就可以在过渡效果完成时触发页面上的其他更改，或者对尺寸可能发生变化的组件执行 resize 操作。

每当触发此事件时，{{slEventValue}} 都是一个包含以下属性的 JavaScript 对象：

  • elapsedTime：过渡效果花费的时间
  • propertyName：正在执行动画效果的属性名称
  • targetId：如果正在执行动画效果的 DOM 元素具有 ID 属性，则可在此获取该 ID

这些属性有助于将特定的过渡效果与其他效果区分开来；请确保你的代码响应的是正确的事件。每个组件都有其独立的事件实例。

WIDGET_ID.INTERACTION_PROPERTY.changed

当由 WIDGET_ID 指定的组件所拥有的 INTERACTION_PROPERTY 属性发生变更时，即会触发此事件。变更后的值可通过 {{slEventValue}} 获取。请注意，此事件并非适用于每个组件的所有属性，而仅适用于“交互属性”（即指组件中可由用户交互改变的属性，例如 selectedValue）。每个组件的交互属性都有其独立的事件实例。

全局操作

全局操作用于控制应用层级的行为变更，例如页面或窗口的修改、数据导出以及跨应用的消息传递。

slate.navigateTo[page_name]

slate.navigateTo[page_name] 操作会将当前作用域内的页面切换为 page_name，同时更新 URL 路由以指向目标页面。共享变量在页面切换过程中将保持其状态。

slate.scrollToId

slate.scrollToId 操作会使页面（及页面内的其他可滚动容器）滚动至带有指定 ID 的 DOM 元素（该 ID 为事件面板中 JavaScript 代码所返回的 ID）。此操作可用于通过编程方式将页面滚动至特定元素，或返回至可滚动表格或列表中之前记住的位置。

slate.setWindowTitle

slate.setWindowTitle 操作支持重命名当前浏览器窗口的标题并更新标签页名称。调用此操作并返回一个字符串即可设置标签页名称。若在页面加载时未调用此操作，Slate 会默认将标签页标题设为文档名称。

slate.print

slate.print 操作会打开浏览器的打印对话框，以便用户在打印前选择打印机选项。

slate.downloadBlob

slate.downloadBlob 操作允许您在遵循 Slate 导出限制的前提下，下载任意数据块。如需使用该操作，请添加如下返回语句：



已复制！
1return {
2    fileName: "my_file.txt",
3    blob: new Blob(["Hello, world!"], { type: "text/plain;charset=utf-8" }),
4};

一个示例用例是在“函数”选项卡中使用类似 ExcelJS 的库生成带有条件格式的 Excel 文件，随后将其传递给 slate.downloadBlob 供用户下载。有关 Blob 的更多信息，请参阅 MDN Web 文档 ↗。

slate.exportWidget

slate.exportWidget 动作支持将组件导出为 PNG 或 PDF 文件。若要使用此动作，请添加如下 return 语句：



已复制！
1return {
2    widgetId: "my_chart",
3    exportType: "download",
4    outputType: "png",
5    fileName: "exported_chart",
6};

使用此事件将触发 slate.onExportWidget 事件，该事件包含的 slEventValue 内容如下：

  • widgetId：已导出组件的 ID。
  • base64：导出组件（PNG 或 PDF 格式）的 base64 编码字符串。

将 exportType 设置为 eventOnly，即可触发 slate.onExportWidget 事件，而无需复制或下载文件。

slate.sendMessage

slate.sendMessage 动作专用于嵌套在其他应用 iframe 中的 Slate 应用。具体而言，该动作会将传入的值（由面板中的 JavaScript 代码返回）作为以下格式的 postMessage 发送至父级 iframe：

已复制！
1{"source": "slate-parent-iframe-action", "message": <payload>}

<payload> 是面板中 JavaScript 代码返回的值。

slate.sendMessage 动作也可用于向其他窗口发送消息，即使 Slate 并未嵌套在 iframe 中。

例如，你可以使用 slate.redirectToUrl 在新标签页或浏览器窗口中打开子窗口，并为该窗口设置 ID。

随后，你可以使用 slate.sendMessage 动作向特定窗口发送消息：



已复制！
1return {
2    "targetOptions": {
3        "targetWindow": "child" | "opener" | "parent",
4        "children": ["child-id1", "child-id2"] // （可选）
5    },
6    "message": <payload>
7}

如果目标窗口设置为 child，你可以提供一个子窗口 ID 数组，这些 ID 是在使用 slate.redirectToUrl 时设置的。如果未提供该参数，消息将发送给所有子窗口。

针对 opener 和 parent 的 targetWindow 将分别向 window.opener 和 window.parent 执行 postMessage 操作。

slate.createBroadcastChannel

slate.createBroadcastChannel 操作将创建一个具有指定名称的 BroadcastChannel。此通道可用于向所有具有相同通道名称的广播通道发送消息，或从中接收消息。

要使用此操作，请添加如下返回语句：

已复制！
1return {
2    channel: "my_new_channel",
3};

有关 BroadcastChannel 的更多信息，请参阅《MDN Web 文档 ↗》。

slate.closeBroadcastChannel

slate.closeBroadcastChannel 动作将关闭并删除具有指定名称的 BroadcastChannel。通道关闭后，将无法接收或发送任何消息。

若要使用此动作，请添加如下返回语句：

Copied!
1return {
2    channel: "channel_to_close",
3};

slate.broadcastMessage

slate.broadcastMessage 动作将向具有指定名称的广播通道发布消息。在发送消息之前，需要先使用 slate.createBroadcastChannel 动作打开通道。

若要使用此动作，请添加如下返回语句：



已复制！
1return {
2    channel: "my_channel",
3    message: <payload>
4};

slate.showCheckpoint

slate.showCheckpoint 操作将打开指定的检查点（Checkpoint）模态框。你可以在《检查点文档》中阅读更多关于检查点的内容。

要使用此操作，请添加如下返回语句：



已复制！
1return {
2    slateCheckpointId: string;
3    checkpointType: CheckpointTypeV1;
4 primaryCheckpointedItemId?: {
5        id: string;
6        type: CheckpointedItemIdType;
7    };
8 additionalCheckpointedItemIds?: {
9        id: string;
10        type: CheckpointedItemIdType;
11    }[];
12};

slateCheckpointId 是一个用户自定义字符串，用于标识检查点（Checkpoint），并会在 onCloseCheckpoint 事件的响应中返回。

checkpointType 应为所配置检查点的类型。查看检查点类型的详细说明。

导出配置

属性类型描述
widgetIdstring要导出的微件（widget）ID（如微件列表所示）。
exportType"copy" | "download"将微件下载为文件或保存至剪贴板。
outputType"png" | "pdf"将微件导出为 PNG 或 PDF 文件。
fileNamestring (可选)当 exportType 为 "download" 时，下载文件的名称。

不支持的微件

iframe、code sandbox 和 multiselect box 微件，以及包含上述任意微件的容器，均无法导出。

slate.openAppsPortal

slate.openAppsPortal 操作将打开应用门户。你可以通过传入可选参数来指定要导航至应用门户的具体位置。该参数必须采用以下形式：

应用门户中的位置参数
无指定无参数
所有应用{ type: "allResultsNavItem" }
平台应用{ type: "platformAppsNavItem" }
推广类应用{ type: "promotedAppsNavItem" }
指定类别的平台应用{ type: "platformAppsCategoryNavItem", title: string }
指定集合的推广类应用{ type: "promotedAppsCollectionNavItem", title: string, rid: string }

slate.askAIPAssist

slate.askAIPAssist 操作将打开 AIP Assist 支持工具。你可以指定一个可选的 prompt 字符串参数，该参数会作为用户消息发送至 AIP Assist 聊天中。prompt 可以从应用中当前用户的状态派生，以便向 AIP Assist 提出更具针对性的问题。



已复制！
1return {
2  "prompt": "How do I interact with widgets in Slate?"
3};

slate.logout

slate.logout 操作会弹出一个浏览器对话框，让用户确认是否要退出登录。如果用户确认，便会退出 Foundry 系统。

slate.copyToClipboard

slate.copyToClipboard 操作会将面板中 JavaScript 代码返回的值复制到用户的剪贴板。

slate.redirectToUrl

slate.redirectToUrl 操作会将用户重定向至传递给该操作的 URL。

不建议将此 Action 与 slate.ready 事件配合使用，因为这样会在页面加载时立即重定向用户，导致编辑困难。如果使用场景确实需要在 slate.ready 事件中使用 slate.redirectToUrl action，建议添加查询参数来禁用该 Action。

配置

属性类型描述
urlstring重定向的目标 URL。
targetstring (可选)加载资源的浏览上下文名称。相当于 window.open 中的 target 参数。
popupboolean (可选)指定是否在弹出窗口中打开 url。
windowSharingOptions{ id: string; } (可选)允许在打开的子窗口中发送或接收 postMessages。id 属性用于为子窗口指定唯一标识。随后可以通过 slate.sendMessage action 与该窗口进行通信。

例如，以下代码将为指定的网站打开一个新标签页：


```json
return {
  "url": "https://www.palantir.com/",
  "target": "_blank"
};
```

返回多个 URL 同样可行，借此可打开多个标签页或页面。只需返回一组配置对象列表即可实现。例如，以下代码将分别为这两个网站打开新标签页：


已复制！
1return [{
2  "url": "https://www.palantir.com/",
3  "target": "_blank"
4}, {
5  "url": "https://www.palantir.com/uk",
6  "target": "_blank"
7}];

slate.saveView

slate.saveView 动作会将当前应用程序状态保存为新的可共享视图。无需任何参数。

视图保存完成后，会触发 slate.viewSaved 事件，新的视图 ID 会通过 {{slEventValue}} 获取。

slate.loadView

slate.loadView 动作会根据 ID 加载之前保存的可共享视图。从事件的 JavaScript 代码中返回视图 ID 字符串，即可指定要加载的视图。

Copied!
1return "view-id-string";

视图加载后，应用程序将导航并显示指定视图，同时应用已保存的组件交互属性和变量值。

查询动作

查询动作作用于特定查询或其结果集。通过这些动作，应用构建者可以在触发特定事件时执行查询或导出查询结果集。

QUERY_NAME.run

此操作将立即运行由 QUERY_NAME 指定的查询。无论查询是否为手动，此操作均有效。每个查询都有其各自的此操作实例。

QUERY_NAME.export, QUERY_NAME.exportXlsx, QUERY_NAME.exportCsv

QUERY_NAME.export 会将 QUERY_NAME 指定的查询结果导出为 .xlsx 文件。它将在服务器上运行查询，生成文件，并将其下载到用户计算机。每个查询都有其各自的此操作实例。

在 Queries 标签页的 Export on 下拉菜单中，可将 QUERY_NAME.export 操作添加为事件的触发器。

默认情况下，文件将被命名为 QUERY_NAME.xlsx。若要更改此名称，请转至 Events 标签页并选择 QUERY_NAME.export。随后，您可以添加 return 语句来配置文件名。例如，return "january_data" 或 return {fileName: "january_data"} 将下载名为 january_data.xlsx 的 .xlsx 文件。有效的文件名不能为空，且只能包含字母、数字、下划线或空格。如果 return 语句中的文件名无效，将使用查询名称作为文件名。

您可以在 Events 标签页的 Actions 下拉菜单中选择 QUERY_NAME.exportXlsx 或 QUERY_NAME.exportCsv 操作。然后，您可以添加 return 语句，将文件名更改为有效的文件名。

QUERY_NAME.exportCsv, QUERY_NAME.exportXlsx

这些操作将 QUERY_NAME 指定的查询输出下载到用户的本地计算机。结果可以下载为 CSV 或 Excel 文件。默认情况下，文件会采用查询名称，不过您可以通过在事件中返回 { fileName: <<custom_string>> } 对象来自定义名称。

函数操作

函数操作作用于特定函数或其结果集。它们使应用构建者能够在触发特定事件时运行函数或导出函数结果集。

FUNCTION_NAME.run

该操作将立即运行由 FUNCTION_NAME 指定的函数。每个函数都有各自的此操作实例。

FUNCTION_NAME.export, FUNCTION_NAME.exportXlsx, FUNCTION_NAME.exportCsv

FUNCTION_NAME.export 会将 FUNCTION_NAME 指定函数的结果导出为 .xlsx 文件。该操作会运行函数，将结果发送至服务器，生成文件，并将其下载到用户的计算机上。每个函数都有各自的此操作实例。

可以从函数选项卡中的导出到下拉菜单内，将 FUNCTION_NAME.export 操作添加为事件的触发器。

默认情况下，文件将被命名为 FUNCTION_NAME.xlsx。如需更改此名称，请导航至事件选项卡并选择 FUNCTION_NAME.export。随后，您可以添加一条 return 语句来配置文件名。例如，return "january_data" 或 return {fileName: "january_data"} 将下载一个名为 january_data.xlsx 的 .xlsx 文件。有效的文件名不能为空，且只能包含字母、数字、下划线或空格。如果 return 语句中的文件名无效，系统将使用函数名作为文件名。

您可以从事件选项卡中的操作下拉菜单内选择 FUNCTION_NAME.exportXlsx 或 FUNCTION_NAME.exportCsv 操作。然后，您可以添加一条 return 语句，将文件名更改为有效的文件名。

要使函数导出正常工作，该函数必须返回一个对象，其中每个属性代表一个列名，且每个属性值必须是长度相等的数组，用于代表该列中每个单元格的值。这意味着，如果您正在处理查询结果，必须从查询中删除 ._results 属性；该属性由 Slate 添加，用于提供有关查询是否成功运行的信息，但它会阻止从函数中进行导出。

变量操作

变量动作作用于特定变量。当特定事件触发时，应用构建者可借此修改变量的值。

VARIABLE_NAME.set

此动作会将 VARIABLE_NAME 指定的变量设置为该动作传入的值（即动作面板中 JavaScript 代码的返回值）。注意，变量原有的/当前的值仍可通过 {{VARIABLE_NAME}} 获取，因此可利用旧值来决定新值。每个变量都有各自的此动作实例。
