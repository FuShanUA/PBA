Microsoft Power BI® XMLA

以下页面讨论了用于数据集成的 Microsoft Power BI® XMLA 连接器。如果你正在寻找有关 Power BI® 连接器从 Power Query 接口访问 Foundry 资源的信息，请查阅我们的《分析连接文档》。

Microsoft Power BI® XMLA 连接器是由 Palantir 提供的、专门用于 Microsoft Power BI® XMLA 的驱动程序。

要创建新的 Microsoft Power BI® XMLA 数据源，请遵循 Palantir 提供驱动程序的标准设置流程，然后参考以下章节进行 Microsoft Power BI® XMLA 特定的配置和网络设置。有关完整的属性参考，请查阅《Microsoft Power BI® XMLA 官方驱动文档 ↗》。

配置

以下属性为必填或推荐项。

属性是否必填？描述默认值
AuthScheme ↗推荐连接到 Microsoft Power BI XMLA 时使用的身份验证类型。AzureServicePrincipal
AzureTenant ↗推荐标识用于访问数据的 Microsoft Power BI XMLA 租户。接受租户的域名（例如 contoso.onmicrosoft.com）或其目录（租户）ID。—
InitiateOAuth ↗推荐指定获取或刷新 OAuth 访问令牌的流程，以便在经过身份验证和授权的用户工作时维持其访问权限。GETANDREFRESH
OAuthClientId ↗推荐指定分配给自定义 OAuth 应用程序的客户端 ID（也称为使用者密钥）。在身份验证期间，需要此 ID 向 OAuth 授权服务器标识该应用程序。—
OAuthClientSecret ↗推荐指定分配给自定义 OAuth 应用程序的客户端密钥。此机密值用于向 OAuth 授权服务器验证应用程序的身份。（仅限自定义 OAuth 应用程序。）—
PowerBIEnvironment ↗推荐指定用于 API 请求的 PowerBI 环境。NONE
Workspace ↗推荐使用以逗号分隔的工作区名称列表，指定要连接的 Premium Power BI 工作区。—
