开发环境

本页将引导您了解使用平台内置 VS Code 工作区构建 OSDK React 应用的开发工作流。

开始使用

要在代码环境中启用开发功能，您需要设置开发者控制台。在设置过程中，您必须选择创建面向客户端的应用，以便能够从开发者控制台创建代码仓库。

要使用 VS Code 工作区，请打开开发者控制台，并在左侧面板的代码仓库部分下选择创建代码仓库。此操作将使用默认的 React 模板初始化您的仓库。随后，您可以选择在 VS Code 中打开，以启动平台内置的工作区。

该仓库模板将包含一个基础 React 应用，并集成了授权功能和 Ontology SDK。

开发生命周期

VS Code 工作区启动后，您会在左侧看到代码编辑器，右侧则是应用的实时预览。您可以像在本地环境中一样开始编辑代码。

编辑

现在您可以开始构建 React 应用了。编写代码后，请务必进行编辑和测试，然后使用 git 命令或 VS Code 界面提交并推送更改，使项目中的其他开发者能够看到这些更改。在终端中运行 npm run lint、npm run test 和 npm run build，以确保检查能够顺利通过。

您还可以查看左侧面板的 Ontology 选项卡，获取 OSDK 的生成文档和代码片段，并返回开发者控制台。

部署

要部署 React 应用，您必须首先创建一个发布版本。这通常通过 git tag 命令完成，您可以设置版本号，然后将其推送到仓库：

Copied!
1git tag <x.y.z>
2git push origin tag

或者，您可以从代码仓库界面创建发布版本，方法是在屏幕右上角选择在代码仓库中打开。导航到版本控制选项卡，然后打开标签与发布部分，查看之前的发布版本并创建新版本。

发布版本通过检查后，您将能够在开发者控制台中查看您的应用。

内容安全策略

默认情况下，我们会应用严格的内容安全策略（CSP）。这意味着对非 Foundry URI 的请求将失败，必须显式允许此类请求。

要检测 CSP 错误，请选择 <F12> 并查看控制台内的日志。如果出现 CSP 失败，您会看到类似于下图的错误：

您可以对 VS Code 工作区应用临时 CSP。出于安全考虑，此 CSP 不会应用于此工作区的其他用户，并且会在工作区暂停或重启时过期。

在 VS Code 工作区中测试应用程序后，请确保为您的开发者控制台应用程序更新 CSP。

开发提示

待办事项应用程序教程

如果您刚开始使用开发者控制台和 OSDK，我们建议您遵循我们的教程之一来入门。导航至您注册环境中的 使用 AIP 构建 应用程序，并安装我们的示例工作流之一。

渐进式 Web 应用程序 (PWA)

为了获得优化的编辑体验，您可以选择将您的工作区安装为渐进式 Web 应用程序 (PWA)。PWA 将接受一些常用的快捷键，例如 Cmd+W (macOS) 来关闭标签页。

沉浸模式

为了获得近乎原生的 VS Code 体验，我们建议使用沉浸模式。选择 视图 > 激活沉浸模式 即可启用。沉浸模式将隐藏平台界面，并让 VS Code 占据整个浏览器窗口。要退出沉浸模式，请将鼠标悬停在屏幕顶部中央的三个点 ... 上。

本地开发

除了在平台内部工作外，您还可以通过以下方式在本地计算机上进行开发：

  • 导航至代码仓库应用程序中的您的仓库，然后选择屏幕右上角的 本地工作 以克隆您的仓库，并将 FOUNDRY_TOKEN 导出至您的本地环境。



有关更多信息，请参阅仓库中的 readme.md 文件。

您可以将仓库存储在 Palantir 平台之外，同时仍使用开发者控制台来部署应用程序。请参阅我们的文档《在 Foundry 上部署 Ontology SDK 应用程序》以了解更多信息。

PR 预览 [Beta]

React 应用程序的 PR 预览处于《开发生命周期》文档所述的 Beta 阶段，您的注册环境中可能无法使用此功能。在活跃开发期间，功能可能会发生变化。

为了提升项目的效率与协作便利性，在推送更新前，你可以先查看 React 应用程序变更的 PR 预览。PR（拉取请求）预览会根据你拉取请求中提交的代码，提供一个可运行的 React 应用程序版本。在变更合并到主分支和生产环境之前，你可以预览应用程序的任何拟定变更，从而便于排查不良结果，并在变更进入生产环境前验证面向用户的工作流。以往，构建人员、设计师及应用开发团队的其他成员需要搭建开发环境才能查看变更。PR 预览功能免去了这一要求，能够更快地验证设计和功能方面的生产变更，使协作流程更高效。

任何使用开发者控制台的 网站托管功能托管在 Foundry 代码仓库中的 OSDK React 应用程序，均可使用 PR 预览功能。

请按照以下步骤开始使用 PR 预览：

  1. 首先，在开发者控制台中生成一个应用程序。

  2. 然后，为该应用程序配置网站托管。



  3. 选择使用 VS Code 工作区，以充分利用 Palantir Visual Studio Code 扩展提供的开发者功能。你也可以在 代码仓库中打开你的应用程序代码。在这两种环境中，你都可以创建仓库分支、通过提交实现代码变更，并推送更新。你可以使用以下终端命令或环境界面来进行更改：

    1. 创建新分支：git checkout -b myFeatureBranch main
    2. 实现代码变更并提交工作：git commit -am "implement UI change to show PR Preview"
    3. 将变更推送到代码仓库：git push —set-upstream origin myFeatureBranch

  4. 完成所需变更且所有检查通过后，如果你之前在 VS Code 工作区操作，请返回代码仓库。

  5. 转到屏幕顶部的版本控制选项卡，选择拉取请求。

  6. 选择你想预览的拉取请求，然后点击右下角的预览，即可打开 React 应用程序的预览版本。



你也可以通过拉取请求的提交选项卡，找到分支上每次提交对应的 PR 预览链接。请注意，PR 预览仅在拉取请求创建后的七天内可用。超过七天后，预览将过期失效。

预览权限

要使用 React 应用程序的 PR 预览功能，用户必须在开发者控制台中拥有该应用程序的相应权限。要管理这些权限，请在开发者控制台中打开你的应用程序，然后转到左侧边栏的共享。

要进行更改、为应用程序代码创建拉取请求，以及与其他用户共享 PR 预览，你必须具备 Owner 或 Editor 权限。如果在不授予用户编辑应用程序代码或配置权限的情况下与其共享 PR 预览，请在共享托管网站下将其添加为 Viewer。
