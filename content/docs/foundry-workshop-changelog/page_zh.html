Workshop 中的变更日志面板

Workshop 的变更日志面板允许构建者直观查看模块版本之间的差异。这有助于跟踪随时间推移所做的修改，并在调试模块问题时定位可能引发异常的具体变因。

理解变更日志

若所选模块版本存在变更，面板将填充变更日志节点。变更共分为 5 种类型：

  • 已更改：节点已被修改（例如按钮上的文本发生变化）。
  • 新增：向模块中添加了节点。
  • 已删除：从模块中移除了节点。
  • 已移动：节点位置发生了移动（例如从页面 1 移至页面 2）。
  • 已弃用：部件被删除但未从模块中移除，而是移入 unused widgets。

模块选择

选择模块版本有两种方式：

  • 范围选择：指定起始版本与结束版本，以查看两者之间的差异。例如，选择 0.1.0 和 0.4.0 即可查看版本 0.1.0 至 0.4.0 之间的变更。

  • 单版本选择：单版本选择允许您查看特定模块版本相较于上一版本的变更。例如，若选择版本 0.5.0，变更日志将展示 0.4.0 与 0.5.0 之间的差异。

在变基过程中使用变更日志面板

当需要将分支变更合并至 main 前进行变基时，变更日志面板会显示视觉通知圆点，并提供启动变基的选项。

在变基期间，变更日志面板会描绘从您的分支应用到模块最新 main 版本的变更，并突出显示合并冲突。若某项变更在 main 和您的分支上同时被修改，则该变更会被标记为冲突。常见情况包括：

  • 部件或变量在 main 和您的分支上均发生了修改。
  • 某部分在 main 上被删除，但在分支上被编辑。
  • 部件在 main 上从位置 A 移至 B，而在您的分支上从 A 移至 C。

在解决冲突时，您可以在三种状态间切换模块，以便实时评估结果：

  • Main：在 main 上的修改呈现。
  • Branch：在您的分支上的修改呈现。
  • Modification：启动变基后为协调差异而所做的更改。

冲突解决完毕且您对模块满意后，即可保存以完成变基。随后，您可以安全地将分支合并至 main。

有关端到端的操作指引，请参阅《变基与冲突解决》。
