设置

  1. 打开 数据连接 应用程序，并在屏幕右上角选择 + 新源。
  2. 从可用的连接器类型中选择 Databricks。
  3. 按照后续的配置提示继续完成连接器的设置，使用以下部分的信息。

网络连接

对于Databricks连接，在设置源时请添加适当的< a href="/docs/foundry/administration/configure-egress/">出站策略，在< a href="/docs/foundry/data-connection/overview/">数据连接应用中。

Databricks连接通常同时打开大量连接。当使用< a href="/docs/foundry/administration/configure-egress/#agent-proxy-egress-policies"> 代理出站策略，您可能会耗尽代理的连接池。如果您遇到连接池错误，请在 代理配置 中增加 maxConnections 和 coreConnections 设置。

Databricks 连接器需要通过端口 443 访问 Hostname 提供的 配置选项 中的网络。这允许 Foundry 连接到 Databricks 工作区和 Unity 目录 REST API。

云获取

云获取是 Databricks JDBC 驱动程序的一项功能。云获取通过云存储实现了从 Databricks 到 Foundry 的并行数据提取，相比传统的单线程传输，性能可提高多达 10 倍。

启用后，可能需要额外的网络策略以允许出站连接到云存储服务（AWS S3、Azure Data Lake Storage 或 Google Cloud Storage），Databricks 会临时将查询结果存储在此处。如果您使用的是 Foundry 工作器连接，则需要为工作区存储桶创建出站策略。这是 Cloud Fetch 所使用的云存储位置。

查看 Databricks 官方文檔 ↗ 以获取详细信息。

对外存储位置的访问（仅虚拟表）

本文档的 虚拟表 部分提供了有关 Unity Catalog 中 外部访问 及其启用功能的详细信息。外部访问需要与表的存储位置（管理或外部）建立网络连接。为了享受外部访问所启用的功能，需要为每个存储位置创建出站策略。

有关外部访问的更多信息以及如何确定表的存储位置，请参阅 官方 Databricks 文档 ↗。

当您为存储位置配置出站策略时，这允许网络流量从Foundry出站到该存储位置。此外，您还应确保存储位置上的任何网络控制都允许来自Foundry的网络流量。这些网络控制会根据云提供商而有所不同。 了解更多关于识别Foundry流量来源的IP信息。

更多选项：SSL 和主机名验证

您可能还需要传入一个 JDBC 属性以允许自签名证书。

如何识别是否需要此属性：

  • SSL 连接验证服务器证书。通常，SSL 验证通过证书链进行。默认情况下，代理和 Foundry 工作器信任大多数行业标准证书链。
  • 如果要连接的服务器具有自签名证书，或者防火墙在连接上执行 TLS 拦截，连接器必须信任该证书。了解更多关于 在基于代理的连接中使用证书 的信息。
  • 如果您正在创建一个 Foundry 工作器连接并使用自签名证书，您需要为 AllowSelfSignedCerts=1 属性添加一个 JDBC 属性。

如何添加允许自签名证书的属性：

  • 在连接详情页面下连接设置部分的底部，选择更多选项然后选择JDBC 属性。
  • 在JDBC 属性下 配置，选择添加属性然后选择新属性，然后将AllowSelfSignedCerts作为键，1作为值输入。

当AllowSelfSignedCerts属性设置为1时，SSL验证被禁用。在这种情况下，连接器不会将服务器证书与信任库进行验证，也不会验证服务器的主机名是否与服务器证书中的通用名称或主题备用名称匹配。

这个JDBC属性和其他属性在 Databricks驱动程序文档 ↗ 中有详细说明。本文档中列出的JDBC属性是特定于Databricks驱动程序的，并与其他源类型不同。

为了使SSL验证正常工作，服务器必须提供完整的证书链。Databricks服务器的证书链可以通过运行命令openssl s_client -connect {hostname}:{port} -showcerts获取。要验证证书链，请使用OpenSSL命令行工具或其他可用工具。

虚拟表

虚拟表 允许您连接到 Databricks Unity Catalog 中注册的数据。这使您可以在 Foundry 中读取和写入 Databricks 中的表，同时也可以从 Foundry 的管道中将计算下推到 Databricks。本节提供了有关使用虚拟表与 Databricks 的更多详细信息。当同步到 Foundry 数据集时，此部分不适用。

Databricks 连接器现在在使用虚拟表来公开 Delta Lake 和 Apache Iceberg 的功能时提供了增强的功能。此功能需要 外部访问 在Unity Catalog中启用。启用后，外部访问允许Foundry通过Unity REST API和Iceberg REST目录访问表，并读写底层存储位置的数据。使用Unity Catalog凭据发放功能来确保对云对象存储的安全访问。除了增强的功能外，这还可以提高对这些表的读写性能。



如果满足以下条件，Databricks连接器会自动公开Delta Lake和Apache Iceberg的功能：

  1. 在Unity Catalog中启用外部访问。
  2. 配置允许Foundry与表的存储位置之间连通的网络出站策略。
  3. 在源端配置可以从Unity Catalog获取发放凭据的凭据。

用于读取或写入表的连接将直接使用Delta Lake或Apache Iceberg客户端连接到存储位置。不会使用Databricks计算来读取或写入表。对于某些元数据操作（例如确定正在访问的表类型），将使用Unity Catalog REST API。

参见官方Databricks文档 ↗ 对于外部访问的更多信息以及如何确定表的存储位置，请参阅本文档的 网络 部分，了解启用对存储位置的网络访问的详细信息。

使用外部访问需要 Unity Catalog 凭据发放。并非所有表类型和表功能都支持凭据发放。例如，视图或具有行过滤器的表不支持凭据发放。有关凭据发放和要求的更多信息，请参阅 官方 Databricks 文档 ↗。

如果上述任何要求未得到满足，将通过 JDBC 连接到 Databricks。JDBC 是用于 同步 的相同机制。有关 Databricks 的 JDBC 连接的更多信息，请参阅 官方 Databricks 文档 ↗。

源配置要求

使用 虚拟表 时，请记住以下源配置要求：

  • 您必须使用 Foundry 工作进程 源。虚拟表不支持使用 代理工作进程 连接。
  • 确保已建立双向连接和允许列表，如 本手册的网络部分 所述，包括推荐的到存储位置的网络连接。
  • 如果在代码仓库中使用虚拟表，请参考 虚拟表文档 以获取额外源配置的详细信息。
  • 您必须在连接详细信息中使用 ___CODE_BLOCK_45 ___ 字段指定仓库或计算集群。参见 Databricks 官方文档 ↗，了解获取 Databricks 计算资源连接详情的方法。

有关更多详细信息，请参阅上面的 连接详情 部分。

计算下推

Foundry 提供了在使用虚拟表时将计算下推到 Databricks 的能力。当使用注册到相同源的 Databricks 虚拟表作为管道的输入和输出时，可以将计算完全联邦到 Databricks。有关如何在 Python Transforms 中将计算下推到 Databricks 的详细信息，请参阅 Python 文档。要在 Pipeline Builder 中将计算下推到 Databricks，请查阅 外部管道文档。
