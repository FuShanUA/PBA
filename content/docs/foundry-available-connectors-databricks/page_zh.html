设置

  1. 打开《数据连接》应用，并在屏幕右上角选择+ 新建源。
  2. 从可用的连接器类型中选择Databricks 平台。
  3. 按照后续的配置提示，使用以下各节中的信息继续设置连接器。

网络

对于 Databricks 平台连接，在《数据连接应用》中设置源时，请添加相应的《出口策略》。

Databricks 平台连接通常会同时打开大量连接。使用《代理出口策略》时，可能会耗尽代理上的连接池。如果遇到连接池错误，请在您的《代理配置》中增加 maxConnections 和 coreConnections 设置。

Databricks 平台连接器需要通过 443 端口访问《配置选项》中提供的主机名。这允许 Foundry 连接到 Databricks 平台工作区和 Unity Catalog REST API。

云端获取

Cloud Fetch 是 Databricks 平台 JDBC 驱动的一项功能。它通过云存储，实现从 Databricks 平台到 Foundry 的并行数据提取，性能较传统单线程传输提升高达 10 倍。

启用该功能后，可能需要配置额外的网络策略，以允许出站连接至 Databricks 平台临时存储查询结果的云存储服务（AWS S3、Azure Data Lake Storage 或 Google Cloud Storage）。若使用 Foundry worker 连接，需为工作区存储桶创建出站策略。该存储桶即为 Cloud Fetch 使用的云存储位置。

查阅 Databricks 官方文档 ↗了解详情。

对存储位置的外部访问（仅限虚拟表）

本文档的虚拟表部分详细介绍了 Unity Catalog 数据目录中的外部访问及其启用的功能。外部访问要求网络能够连接至表的存储位置（托管表或外部表）。需为每个存储位置创建出站策略，才能受益于外部访问启用的各项功能。

有关外部访问及如何确定表存储位置的更多信息，请参阅Databricks 平台官方文档 ↗。

为存储位置配置出站策略后，网络流量即可从 Foundry 流向该存储位置。此外，还需确保存储位置上的所有网络控制策略均允许来自 Foundry 的网络流量。这些网络控制策略因云服务提供商而异。详细了解如何识别 Foundry 流量的源 IP。

更多选项：SSL 与主机名验证

此外，您可能需要传入 JDBC 属性以允许自签名证书。

如何判断是否需要此属性：

  • SSL 连接会验证服务器证书。通常，SSL 验证通过证书链完成。默认情况下，智能体和 Foundry worker 均信任多数行业标准证书链。
  • 若所连接的服务器使用自签名证书，或防火墙对该连接执行 TLS 拦截，连接器必须信任该证书。详细了解在基于智能体的连接中使用证书。
  • 若正在创建 Foundry worker 连接且使用自签名证书，则需添加 AllowSelfSignedCerts=1 的 JDBC 属性。

如何添加允许自签名证书的属性：

  • 在连接详情页面底部的连接设置下，选择更多选项，然后选择JDBC 属性。
  • 在JDBC 属性配置下，选择添加属性，然后选择新建属性，输入 AllowSelfSignedCerts 作为键，输入 1 作为值。

当 AllowSelfSignedCerts 属性设置为 1 时，SSL 验证将被禁用。此时，连接器既不会针对信任库验证服务器证书，也不会验证服务器主机名是否与服务器证书中的通用名或主体备用名称相匹配。

此 JDBC 属性及其他属性在《Databricks 驱动程序文档 ↗》中有所说明。本文档中所述的 JDBC 属性为 Databricks 驱动程序所特有，与其他数据源类型有所不同。

服务端必须提供完整的证书链，SSL 验证才能生效。运行命令 openssl s_client -connect {hostname}:{port} -showcerts 即可获取 Databricks 平台服务器的证书链。如需验证证书链，请使用 OpenSSL 命令行工具或其他可用工具。

虚拟表

虚拟表允许您连接至在 Databricks 平台 Unity Catalog 中注册的数据。借此，您不仅可以在 Foundry 中对 Databricks 平台的表进行读写操作，还能将计算任务从 Foundry 的流水线下推至 Databricks 平台执行。本节将详细介绍虚拟表在 Databricks 平台中的使用方法。此部分内容不适用于同步至 Foundry 数据集的场景。

在使用虚拟表展现 Delta Lake 和 Apache Iceberg 的特性时，Databricks 平台连接器现提供增强功能。如需使用此功能，必须在 Unity Catalog 中启用外部访问。启用后，外部访问允许 Foundry 通过 Unity REST API 和 Iceberg REST 数据目录访问表，并在底层存储位置读写数据。系统将利用 Unity Catalog 凭据分发机制来保障云对象存储的访问安全。除功能增强外，此举还能提升针对这些表的读写性能。



若满足以下条件，Databricks 平台连接器将自动为您暴露 Delta Lake 和 Apache Iceberg 的功能：

  1. 在 Unity Catalog 中启用外部访问。
  2. 配置允许 Foundry 连接至表存储位置的网络出口策略。
  3. 在数据源上配置可从 Unity Catalog 获取分发凭据的凭证。

系统将使用 Delta Lake 或 Apache Iceberg 客户端，直接连接至存储位置进行表的读写操作。此过程不会调用 Databricks 平台的算力进行表的读写。针对某些元数据操作（例如确定所访问表的类型），系统将使用 Unity Catalog REST API。

有关外部访问及如何确定表存储位置的更多信息，请参阅《Databricks 平台官方文档 ↗》。如需了解如何启用对存储位置的网络访问详情，请参阅本文档的网络章节。

要使用外部访问，必须开启 Unity Catalog 凭据分发功能。并非所有表类型和表特性都支持凭据分发。例如，视图或带有行过滤器的表不支持凭据分发。有关凭据分发及其要求的更多信息，请参阅《Databricks 平台官方文档 ↗》。

若上述任何要求未能满足，系统将退而使用 JDBC 建立与 Databricks 平台的连接。JDBC 也是用于同步的同一套机制。有关连接 Databricks 平台的 JDBC 更多信息，请参阅《Databricks 平台官方文档 ↗》。

源凭据权限

为了获得完整的功能支持，我们建议为源连接提供的凭据授予以下权限。根据所需的继承模型，这些权限应应用在数据目录、架构或表级别。

类别权限说明
前提条件USE CATALOG, USE SCHEMA必须在将在 Foundry 中使用的 Databricks 平台数据目录和架构上授予。
元数据BROWSE浏览源和注册表所需。
读取SELECT在使用同步或虚拟表输入时读取 Databricks 平台表所需。
编辑MODIFY在使用虚拟表输出时修改 Databricks 平台表所需。
创建CREATE SCHEMA, CREATE TABLE在使用虚拟表输出时创建 Databricks 平台表所需。
其他EXTERNAL USE SCHEMA启用对存储位置的外部访问。有关更多详细信息，请参阅外部访问。

使用外部表时，我们建议在所使用的外部位置上授予 BROWSE、CREATE EXTERNAL TABLE 和 EXTERNAL USE LOCATION 权限。在使用虚拟表输出创建外部表时，这些权限是必需的。

此外，提供的凭据必须对源配置中提供的仓库或计算集群具有使用权限。

有关在 Unity Catalog 中管理权限的更多信息，请参阅Databricks 平台官方文档 ↗。

源配置要求

使用虚拟表时，请注意以下源配置要求：

  • 必须使用Foundry worker源。虚拟表不支持使用智能体 worker连接。
  • 确保按照本文档的网络部分所述建立双向连接和允许列表，包括到存储位置的建议网络配置。
  • 如果在代码仓库中使用虚拟表，请参阅《虚拟表文档》，了解所需的额外源配置详细信息。
  • 必须在连接详细信息中使用 HTTP path 字段指定仓库或计算集群。有关获取 Databricks 平台计算资源连接详细信息的说明，请参阅Databricks 平台官方文档 ↗。

有关更多详细信息，请参阅上方的连接详细信息部分。

计算下推

使用虚拟表时，Foundry 支持将计算下推到 Databricks 平台。当使用注册到同一源的 Databricks 平台虚拟表作为流水线的输入和输出时，可以将计算完全下放给 Databricks 平台处理。有关如何在 Python Transforms 中将计算下推到 Databricks 平台的详细信息，请参阅《Python 文档》。若要在 Pipeline Builder 中将计算下推到 Databricks 平台，请查阅《外部流水线文档》。
