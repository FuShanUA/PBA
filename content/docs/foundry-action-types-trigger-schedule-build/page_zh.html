触发计划构建

一个 计划 定义了一组 Foundry 在进行 构建 时重新计算的资源。通过在操作类型上配置一个 计划规则，您可以在操作被应用时触发该计划的构建。这使得 Ontology 中的最终用户工作流可以重新计算数据集，而无需用户前往 数据血缘 或 构建应用程序。

当操作类型包含一个计划规则时，操作的 Ontology 编辑会在 构建开始后 应用。编辑不会等待构建完成。相反，该操作会触发构建，捕获计划运行的 RID，并立即应用其余规则，包括 Ontology 编辑。

配置计划规则

将一个计划规则添加到一个操作类型并选择一个计划。该计划必须处于 项目范围模式。

如果所选的调度是 参数化 的，您必须为每个调度参数提供一个值。当操作被应用时，解析后的参数值将传递给调度，并转发到构建内部的底层参数化转换。

调度规则在与 并行参数化调度 配对使用时特别有用。请查看 参数化文档，了解有关在本体论中使用操作进行并行调度的更多信息。

权限

该操作的 提交条件 管理通过操作触发调度所需的权限。如果用户满足操作提交条件，他们可以在没有调度直接权限的情况下运行调度。

Foundry 在第一次引用调度时以及每次编辑调度规则时，都会检查用户是否有运行调度的权限。从操作类型中引用调度会将运行调度的控制权从调度转移到操作类型。任何能够管理操作类型的用户都将控制谁可以触发该调度。

跟踪构建进度

当调度规则被触发时，产生的调度运行会通过一个调度运行RID来标识。这个RID作为可从操作类型本体论编辑规则中引用的值，允许你将其写入已编辑对象的字符串属性中。这在你想在对象上记录由操作触发的构建时非常有用。

要捕获计划运行的RID，请在相同的操作类型上配置一个修改对象或创建对象规则，并将目标对象的字符串属性映射到由计划规则提供的计划运行RID值。

要将存储的RID显示为实时构建状态指示器，请对属性应用资源RID格式化。启用格式化后，Foundry会将RID值显示为带有图标和文本的链接，文本反映了当前构建的状态：运行中、已忽略、失败 或 成功。
