使用容器模型上传镜像

本页将指导您完成使用容器支持模型上传镜像的流程。在此工作流中，您将创建一个模型，将镜像推送到模型资源中，并配置模型版本。

1. 创建模型

要创建容器支持的模型，首先需要创建镜像将要推送至的资源。该资源将控制模型权限；对该资源拥有 Viewer 权限的用户将能够从 Foundry 中拉取镜像。

导航至任意建模目标页面，选择 +添加模型，然后选择 导入容器化模型，即可创建新模型。

2. 将镜像推送到模型

您推送的镜像必须符合 Foundry 的《镜像要求》。

在 Palantir 中创建模型资源后，请按照说明将镜像推送到该资产中。

选择 生成令牌，复制默认命令块并进行编辑，将 example-package:<TAG> 替换为本地 Docker 仓库中存在的镜像名称和标签（或摘要）。请注意，根据《镜像要求》，标签不允许使用“latest”。执行这些命令会将相关镜像推送到模型中，拥有 Viewer 权限的用户可以访问该镜像。

镜像成功推送并出现在容器镜像列表中后，选择 创建模型版本 开始配置您的版本。为此，您必须首先编写一个使用《模型适配器库模板》创建的《容器模型适配器》。

镜像大小不同，容器镜像可能需要几分钟时间才能在模型中可用。

要在发布前对模型适配器实现进行迭代，请使用 Sidecar 装饰器测试逻辑如何与模型中已有的容器配合工作。按照《Sidecar 转换装饰器文档》中的步骤创建一个启用该装饰器的新转换存储库，并使用您刚创建的模型作为支持 Artifacts 仓库。然后，在转换存储库中定义一个实现该适配器的本地 Python 文件，并按如下所示手动实例化它：

1from transforms.api import transform, Input, Output
2from transforms.sidecar import sidecar, Volume
3# 这假设你在 `myproject` 文件夹下创建了一个 Python 类 testAdapter.py，并定义了一个 TestModelAdapter 类来实现 palantir_models 的 ModelAdapter 类。
4from myproject.testAdapter import TestModelAdapter
5import palantir_models.transforms
6
7
8@sidecar(image='image-name', tag='1.0.0', volumes=[Volume("shared")])
9@transform(
10    output=Output("ri.foundry.main.dataset.<RID>"),
11    source_df=Input("ri.foundry.main.dataset.<RID>"),
12)
13def compute(source_df, output):
14    testingEntrypoint = TestModelAdapter(
15       "/opt/palantir/sidecars/shared-volumes/shared",
16       # 注意，此处的端口应反映你的特定镜像正在监听的端口。
17       "localhost:PORT")
18    testingEntrypoint.transform_write(source_df, output)

3. 配置模型版本

在配置模型版本页面，选择要包含在特定模型版本中的一个或多个容器镜像。选择下一步继续进行模型版本配置。

在配置模型对话框的模型详情页面，配置要进行的语义版本变更类型。接着，添加可选的描述和配置文件，并选择您创建的适配器。在模型适配器配置区域选择您的模型适配器仓库，选择模型适配器下拉菜单将自动填充该仓库中定义的所有有效适配器的最新版本。

最后，为每个镜像配置可选的运行时信息，包括内存限制、参数、任何环境变量或新的执行命令。请注意，如果您的适配器使用 requests 库，指定镜像监听的端口至关重要。您可以为选入模型版本的每个镜像配置运行时信息。

可将模型配置为生成遥测日志。如果您的容器支持遥测，您可以在下方截图所示的步骤中为模型版本启用该功能。

4. 查看模型版本

模型版本创建完成后，其元数据将显示在模型页面的模型概览选项卡中。在此处，使用左侧面板可查看并导航至历史版本。

在此页面，您可以点击右上角的+图标或操作下拉菜单，提交至建模目标。您也可以将模型用作Python 转换的输入。请注意，如果模型由容器提供支持，您需要在 ModelInput 类中指定第二个可选参数，选择要在转换中使用的版本 RID。您还必须在转换仓库中添加对模型适配器的依赖。相关示例如下所示：

1from transforms.api import transform, Input, Output
2from palantir_models.transforms import ModelInput
3
4
5@transform(
6    output=Output("/path/to/output/output_dataset"),
7    model=ModelInput(
8        "ri.models.main.model.<RID>",
9        "ri.models.main.model-version.<RID>"),
10    source_df=Input("/path/to/input/input_dataset"),
11)
12def compute(model, source_df, output):
13    model.transform_write(source_df, output)
