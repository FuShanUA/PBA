问题应用程序

问题应用程序是一个支持系统，使用户能够从 Palantir 平台内部获得帮助。

使用问题应用程序，您可以：

  • 就应用程序或平台本身提出问题。
  • 透明地进行问题、请求和问题的分类和解决。
  • 从平台内的任何位置报告资源上的问题，并查看哪些资源有未解决的问题。
  • 在平台上提出更多数据集成的请求。

访问问题应用程序

从平台主页：

  1. 在左侧导航栏中，选择 查看所有，位于 平台应用 部分右侧。
  2. 从 支持 下方选择 问题。

搜索问题

从“问题”主页，您可以筛选和搜索特定的问题以缩小到您感兴趣的问题。默认情况下，您将看到打开的问题概览页面。您也可以选择 已关闭 或 全部 以查看相应的视图。

  1. 侧边栏筛选：左侧的侧边栏提供以下筛选选项：

    • 优先级
    • 负责人
    • 报告人
    • 提及的人
    • 标签
    • 截止日期
    • 报告日期
    • 最后更新日期

  2. 搜索问题：此搜索栏允许您搜索所有用户输入的字段，包括标题和评论。

  3. 选择筛选条件：选择筛选选项允许您根据与您相关的问题进行筛选（例如，分配给您的问题、由您报告的问题或提及您的问题）。在“问题”应用程序中，有多种方式可以对显示的问题进行排序，包括：

    • 最佳匹配
    • 最近更新（默认）
    • 最久未更新
    • 最近创建
    • 最早创建
    • 最高优先级
    • 最低优先级
    • 最早截止日期
    • 最新截止日期

报告问题

要报告问题，请导航到工作区侧边栏中的支持部分，然后选择报告问题或联系支持。通过选择查看支持工单来查看现有问题。

一旦您的问题被创建，您将返回到问题概览页面，最近的问题会在此列出，这些问题将由相关负责人进行审核。选择它进入特定问题页面并添加评论以提供进一步更新，或者编辑其状态、优先级、截止日期、负责人、查询类型、（相关）应用程序或标签。否则，如果已达成解决方案，请选择关闭问题。

在创建问题时：

  1. 选择最能反映您需求的支持类型。
  1. 尽可能提供更多信息，包括标题、描述、优先级，以及任何其他相关信息，例如相关资源或附件。然后选择创建问题。

只有在问题创建之后才能修改负责人，除非所选的支持类型未配置初始负责人，在这种情况下，报告人需要选择适当的用户或组来解决其问题。

问题权限

问题通常对组织中的所有其他用户都可见。但是，如果一个问题与数据集或资源有关，则可以通过在数据集或资源所在项目的 角色 上进行控制。您可以根据用户在数据集或资源上的文件系统权限来控制用户是否能看到该问题。

注意: 在数据集或资源上创建问题不会授予其他用户访问该数据集或资源的权限。

跨组织协作

对一个问题的访问权限取决于与之相关的组织。默认情况下，所有问题都与报告者所属的组织相关联。当将一个问题分配给此组织之外的用户或组时，更新问题的用户可以将与被指派者相关的组织添加到问题中。只有当问题与被指派者所属的组织相关联时（例如，他们的组织或他们被授予了访客成员资格的组织），被指派者才能查看该问题。重新分配问题时也是如此。

当将问题分配给组时，并没有严格的保证，任何被分配组的个人用户都能查看该问题。这是因为被分配组的个别成员可能无法访问与该问题相关的任何组织。

The association of a group with an Organization does not grant all members of that group access to the Organization. This can only be done through granting guest membership to an Organization. When assigning groups to an issue, you should investigate the permissions of group members to confirm which Organizations (if any) need to be added to the issue to ensure that members have access to respond to the issue.

Comment tags

When viewing comments on an issue, you may see colored tags displayed next to the commenter's name. These tags help identify the role or relationship of the commenter to the issue, making it easier to understand who is responding and their context.

Available tags

TagDescription
AssigneeDisplayed for users who are assigned to the issue, either directly or through a group assignment.
ReporterDisplayed for the user who originally created the issue.
AutomatedDisplayed for auto-generated comments from the system.

Tag display rules

  • A single comment can have multiple tags if the commenter has multiple roles. For example, a user who reported an issue and is also assigned to it will see both the Reporter and Assignee tags.
  • Tags are determined automatically based on the commenter's relationship to the issue and cannot be manually assigned or removed.
