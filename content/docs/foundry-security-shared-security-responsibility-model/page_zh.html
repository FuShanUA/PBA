共担安全责任模型

Foundry 的安全由 Palantir 与 Palantir 的客户共同负责。虽然 Palantir 负责我们控制范围内各个服务层级的安全，但客户在确保其数据得到保护方面也发挥着至关重要的作用。

共担责任模型是众多大型基于云的软件公司采用的一种框架，用于界定客户与云提供商之间的安全责任。作为云提供商，Palantir 负责云本身的安全，而客户则负责云内部的安全。

在实践中，这意味着 Palantir 和底层基础设施即服务（IaaS）提供商负责通过管理物理安全、主机基础设施、网络控制和应用安全来保障 Foundry 的安全。同时，客户需对其选择在 Foundry 中托管的内容负责：具体而言，即客户数据和用户身份/访问配置。共担责任模型使客户能够掌控自己的数据、数据的使用方式以及访问权限，同时免除客户对底层基础设施安全的担忧。

共担责任模型可以划分为双方各自的责任：

Palantir ↘客户 ↘
Foundry 服务客户数据
传输中和静态加密客户构建的应用程序
网络流量保护身份与访问管理 (IAM)
操作系统、网络和防火墙配置资源权限 (RBAC)
基础设施级监控与告警应用级监控与告警
持续交付、自动升级与补丁管理
基础服务：

作为客户，您在保护数据方面的一些责任可能因环境和架构而异，但您始终需对以下事项负责：

  • 通过单点登录 (SSO)管理用户的访问和身份。这包括强制执行多因素身份验证。如果您没有单点登录提供商，我们或许可以为您提供。
  • 确保上传到环境中的任何数据是适当的，并符合您自身的政策和义务。
  • 确保您的用户应用适当的自主访问控制（角色）和强制访问控制（标记）。
  • 监控应用级安全审计日志，以确保用户对平台的使用是适当的，并符合您自身的政策和义务。

如果您对共担责任模型有任何疑问或担忧，请联系您的 Palantir 代表。
