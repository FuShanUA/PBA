本体论 SDK 支持

Python 和 TypeScript v2 函数支持 本体论 SDK（OSDK）。OSDK 允许您直接从开发环境中利用本体论，并提供 优势，例如与开发者控制台的兼容性和 OSDK 版本管理。我们建议使用 Python 或 TypeScript v2 来在您的函数仓库中享受这些优势。

TypeScript v1 与 TypeScript v2

TypeScript v1 和 TypeScript v2 都允许用户利用 TypeScript 的核心语言特性，但支持的平台功能有所不同，如上表所示。我们建议使用 TypeScript v2 函数来构建工作流，以利用相对于 TypeScript v1 的多项关键改进：

  • 在完整的 Node.js 运行时中无服务器执行： TypeScript v2 函数在 Node.js 环境中运行，支持核心模块如 fs、child_process 和 crypto。这使得与文件系统交互、执行并行的 CPU 密集型任务或需要其他系统级操作的 NPM 库具有更高的兼容性。
  • 一流的OSDK支持： OSDK现在可以在TypeScript v2函数中无缝使用，使代码在平台内外的重用变得容易。它还为处理大规模本体论数据提供了更高效的API。
  • 可配置的资源请求： TypeScript v2函数允许您请求最多8个vCPU和5GB内存，从而对性能和可扩展性有更大的控制。

无服务器函数

如果您的注册启用了无服务器函数，则新仓库将默认使用无服务器函数。我们建议大多数情况下使用无服务器函数而不是已部署的函数。通过无服务器函数，您可以按需提供单个函数的多个版本，使升级更加安全。

当使用Python或TypeScript v2函数时，也存在一些情况需要优先使用或必须使用已部署的函数而不是无服务器函数，但这些情况并不常见。如果可用，出于以下几个原因，无服务器函数是首选：

  • 无服务器函数可按需执行单个函数的不同版本，使升级更安全。已部署的函数只能同时运行一个函数版本。
  • 无服务器函数仅在执行时产生费用，而已部署的函数则在部署运行期间持续产生费用。
  • 无服务器函数需要较少的前期设置和长期维护，因为基础设施是自动管理的。
