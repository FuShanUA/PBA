# 提交支持工单 如果在查阅[可用资源](/docs/foundry/getting-help/overview/#2-search-the-issues-application-and-the-developer-community)后仍未能找到针对您问题的合适解决方案，接下来就该提交支持工单了。 提交工单能确保您的问题或请求得到跟踪、妥善安排优先级，并在必要时进行升级处理。 为提高支持处理效率，请按照以下步骤整理有价值的信息，以便提交支持工单： - [创建示例](#1-create-an-example) - [共享相关资产](#2-share-relevant-assets) - [报告问题](#3-report-the-issue)
前提条件

在继续遵循下文说明前，请确保您已通过[获取帮助页面](/docs/foundry/getting-help/overview/)提供的步骤诊断了问题，并查阅了[使用 Chrome™ DevTools 调试](/docs/foundry/getting-help/debug-use-devtools/)指南以及 [HTTP 错误代码](/docs/foundry/getting-help/http-error-codes/)。

## 1. 创建示例 在就遇到的问题提问时，提供可供我们复现该问题的清晰步骤，将有助于快速解决。这些步骤应具备以下特点： - **最小化：** 尽量使用最少的步骤，同时仍能触发该问题。 - **完整性：** 提供复现该问题所需的所有组成部分。 - **可验证性：** 测试您即将提供的操作流程，确保其能够复现该问题。 一个最小化、完整且可验证的示例，能帮助我们在您的环境之外复现问题，进而更容易找到问题的根本原因并更快解决。 在 Palantir 的语境下，此类示例可以是一系列复现该行为的步骤，也可以是一个复现该问题的最小资源。通常，在示例中包含预期行为与实际观察到的行为的对比会非常有帮助。 ### 提示 以下是为您创建最小化、完整且可验证的示例提供的一些提示： - 首先确定复现该问题所需的步骤。这些步骤越具体、越详细，他人复现该问题就越容易。 - 确保您的示例是最小化的，即仅包含复现该问题所需的最基本信息。这有助于缩小问题的潜在原因范围，并使修复更容易。 - 包含您所提供步骤的预期结果以及实际结果。这将有助于他人理解问题，并判断他们是否遇到了相同的问题。 - 确保您的示例是完整的，即包含复现该问题所需的所有信息。如果遗漏了任何重要细节，他人将很难复现该问题并帮助您解决。 - 最后，确保您的示例是可验证的，即他人能够使用您提供的信息轻松复现该问题。如果示例不可验证，他人将很难帮助您修复问题。 ### 示例 以下是一个历史示例，展示了一个已解决的最小化、完整且可验证的问题。 当时复现该问题的步骤： 1. 打开“代码仓库”应用程序。 2. 通过侧边栏添加 Graphframes 库。 3. 对仓库运行检查。 预期行为：该库应被添加到“代码仓库”中且不出现任何错误。 实际出现问题的行为：显示一条错误消息，提示“o257.loadClass.: java.lang.ClassNotFoundException:” 补充信息： - 以下是我配置文件中的其他包：Python 3.12.* 等。 - 我已经尝试过执行 x、y 和 z 操作。 - 除了添加此包之外，此“代码仓库”近期未做任何其他更改。 - 此问题似乎会影响该包的用户。

此示例仅包含复现问题所需的最少步骤，且不依赖任何专有数据。由于该示例涵盖了所有必要信息，如预期行为与实际行为，以及先前已进行的排查等附加上下文，因此被视为完整的示例。由于通过相同步骤即可复现错误信息，该示例具备可验证性。提供这些信息后，支持团队便能迅速复现问题并着手解决。

若您对该问题的修复方案感兴趣，请查阅《查看同时需要 Conda 包和 jar 包的包》。

2. 分享相关资产

在文本中提供可验证的示例后，请为支持团队整理以下附加资产：

  • 包含您在步骤 1 中自行调查的任何发现。
  • 提供所显示的任何构建错误或错误信息的完整文本。
  • 复制并粘贴任何 errorID/errorInstanceID 文本，以便我们进行后端审查。
  • 在可能且适当的情况下，与 Palantir 支持团队分享所有相关资源。
    • 相关资源（Slate 仪表板、分析、代码仓库）。
    • 这些资源所依赖的数据集。
    • 这些资源依赖的任何上游数据集。

3. 报告问题

若要在指定应用程序中报告问题，请导航至 支持 > 报告问题（或 支持 > 联系支持），并按照向导操作。

作为替代方案，您也可以在Palantir 开发者社区 ↗中向其他用户寻求支持。

在 Palantir 开发者社区等公共论坛发帖时，请务必在发布前清除所有敏感数据。
