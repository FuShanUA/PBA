核心概念

本页介绍与 Pipeline Builder 相关的 Foundry 数据集成核心概念。

构建

数据集、分支、转换和输出的概念是 Pipeline Builder 的基础。我们建议在构建首个管道前，以及在转换数据并集成为管道输出的过程中，仔细阅读这些内容。

数据集

数据集是管道的构建块。在数据集成过程中，从数据进入 Foundry 到映射为 本体对象模型期间，数据都以 Foundry 数据集的形式存在。

从根本上讲，Foundry 数据集是对存储在底层文件系统中的一组文件的封装。Pipeline Builder 主要面向结构化数据，但也支持半结构化数据。

详细了解 Pipeline Builder 中的输入数据集。

分支

版本控制对于维护健康的管道工作流至关重要。在 Pipeline Builder 中，版本控制通过管道分支实现，其运作方式与 Git 版本控制中的代码分支类似。

管道分支是管道的副本，用户可在其上进行迭代，而无需回写至主管道，这与 Git 中的代码分支类似。用户可以在自己的分支上进行修改、预览、保存和构建。对修改满意后，可以提议合并回 主分支，类似于合并 Git 拉取请求。

详细了解 Pipeline Builder 中的分支。

转换

可以将转换视为一个函数定义；即转换接收一组输入（如数据集）并产生一组输出。管道是由数据集、数据期望和目标数据输出通过转换连接而成的整体。

详细了解 Pipeline Builder 中的转换。

管道输出

Pipeline Builder 中的输出是管道中执行转换后的结果，可以是数据集、虚拟表，或本体组件（如对象类型、对象链接类型或时间序列）。这些输出可用于 Foundry 的其他应用程序，如 Quiver 或代码工作簿。

详细了解 Pipeline Builder 中的管道输出。

管理

调度和数据期望的概念有助于维护健康、稳定的管道。我们建议在构建首个管道后，进一步学习这些内容。

调度

调度用于定期运行数据集构建，以确保数据在 Foundry 中持续流转。在 Pipeline Builder 中，可以设定在特定时间、以特定频率，或基于上游资源的状态进行调度构建；例如，您可以设定在上游数据集更新时触发构建。

详细了解 Pipeline Builder 中的调度。

数据期望

Pipeline Builder 通过单元测试支持对输出和中间转换设置数据期望。数据期望是可以应用于数据集输出的要求。这些要求（即“期望”）可用于创建检查，以提升数据管道的稳定性。

可以为每个管道输出设置数据期望，从而对最终输出设定预期。Pipeline Builder 目前支持两种数据期望类型：主键和行数。

如果任何期望未通过，构建将会失败。作业期望面板会显示哪些数据期望通过，哪些失败。
