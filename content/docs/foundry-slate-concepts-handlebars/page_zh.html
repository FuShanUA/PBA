通过 Handlebars 访问值

Slate 使用 Handlebars ↗ 在查询、函数和小部件中进行模板处理。Handlebars 模板被包裹在 {{ }} 中，可以访问 Slate 对象，如小部件、变量和查询/函数结果。例如，如果您有一个下拉小部件 w1，您可以使用模板 {{w1.selectedValue}} 访问所选值。Slate 会自动评估模板并将其替换为计算后的值。

Handlebars 模板也可以调用辅助函数。有关更多信息，请参阅 辅助函数。

可以引用哪些 Slate 对象？

变量

您可以访问 变量 在变量编辑器中定义的 {{variableName}}。例如，如果您有一个变量 var1 = {"a": [1,2,3]}，那么模板 {{var1.a.[0]}} 的值为 1。 您还可以访问 Slate 环境变量，包括当前应用程序用户或用户专用存储的信息。例如，使用模板 {{$global.user.firstName}} 将显示用户的名。

查询

您可以访问 查询结果。例如，{{myQuery}} 包含 SQL 或 HttpJson 查询的结果。还有一个隐藏的 _response 字段，其中包含一些元数据。

Copied!
1{"name": ["Ceres","Pallas"],"earthmoid": [1.59376,1.23071],"pha": [false,false],"datefirstobs": ["1830-04-18","1825-03-23"]"_response": {"hasRun": true,"message": "Unable to parse query.","success": false}}

函数

您可以通过将函数名称放在 handlebars 中来访问 函数 的返回值。例如，{{myFunction}}。

小部件属性

您可以访问某些小部件属性。具体来说，模板可以访问叶属性（包括数组及其子属性）。例如，假设我们有一个图表小部件。

这是小部件的原始JSON标签中列出的属性：

Handlebars可以在哪里使用？

查询编辑器

在查询编辑器中的Handlebars模板始终会被替换为字符串值。如果模板未计算为字符串，则该值会首先转换为字符串。这是因为查询本身始终是一个字符串。例如：

Copied!
1SELECT {{column w8.selectedValues}} FROM allNamed WHERE name = {{param w12.selection.xValues}};

非字符串值使用 toString 方法进行转换。这通常对整数和布尔值有效，但可能会在更复杂的对象上产生意外结果。例如，在对象上运行 toString 会返回 [object Object]，这可能不是你想要的。你可以使用像 {{jsonStringify myObject}} 这样的辅助函数来解决这个问题。

函数编辑器

与 Queries 不同，函数编辑器中的 Handlebars 模板始终会被其计算后的值替换。例如：

Copied!
1var a = {{query1}};  // query1 is an object
2var b = a.results[0];  // You can access its properties.

不能在函数编辑器中使用辅助函数，你必须直接引用 Slate 对象。此外，你不能设置变量或属性值，因为你不能更改函数外部的任何状态。例如：

Copied!
1var c = {{jsonStringify query1}}; //invalid - cannot use helpers
2{{var1}} = [1,2,3];  // invalid - cannot set global variables

小部件属性编辑器

当应用程序处于 编辑模式。在小部件中，Handlebars 模板必须始终用双引号 "" 包裹，因为大括号不是有效的 JSON 语法。

Copied!
1{
2  "property1": "{{variable1}}",
3  "property2": "Hello, my name is {{var1}}",
4  "property3": [1, 2, "{{var3}}"],
5  "property4": {{variable1}}  //Invalid: JSON parse error
6}

一般来说，小部件中的模板会被替换为字符串。如果计算后的值不是字符串，Slate 将对值运行 toString。对此规则有少数例外情况。当以下条件成立时，Slate 会直接将模板替换为计算后的值：

  1. 只有一组括号 {{}} 并且
  2. 括号外没有其他字符（包括空格）。

让我们看几个例子：

Copied!
1{
2  // value replacement examples
3  "property1": "{{var1}}",
4  "property2": "{{ var1 }}",  // whitespace is inside the handlebars
5  "property3": "{{ add 2 (add 1 var1) }}",  // helpers, can be nested
6  "property4": ["{{var1}}", "{{w1.property.[0]}"],
7
8  // string replacement examples
9  "property5": "Hello {{var1}}!",
10  "property6": " {{var1}} ",  // whitespace outside of brackets
11  "property7": "{{#if 0}}3{{else}}4{{/if}}"  // more than one set of brackets
12}

Handlebars 不能在哪里使用？

  • Handlebars 不能在变量编辑器中使用。
  • Handlebars 不能在样式编辑器中使用。
  • Handlebars 不能在应用程序的查看模式中使用。如果您在小部件或文本字段中输入 handlebars，表达式既不会被计算也不会被替换。

教程：关联小部件

本教程将教您如何创建一个按钮，使用户可以选择了解有关特定航班的飞机的更多信息。这需要我们在两个小部件之间建立关联。

要关联小部件，您必须知道小部件的名称以及您想要关联的属性。在我们的案例中，我们想获取原始表格中所选行的 tail_num 值，但我们配置了表格中的选择以使用 flight_id 作为主键。

这是使用另一个辅助函数的理想时机。因为获取选定表格行的列值是一个常见模式，我们可以使用一个函数来实现它：

Copied!
1function getRowData(s){
2    var returnObj = {};
3    var index = data[selectionColumn].indexOf(s);
4
5    //this iterates through each array in the object data
6    //value is the actual array, and key is the key for that array
7    _.forOwn(data, function(value, key){
8        returnObj[key] = value[index];
9    });
10
11    return returnObj;
12}
13
14// The same data as is fed into the table
15var data = {{f_data}};
16
17// The current selection in the table
18var selection = {{w_mainTable.selectedRowKeys}};
19
20// The column configured as the table primary key
21var selectionColumn = {{w_mainTable.selectionIdentityColumnId}}
22
23var rowData = _.map(selection, getRowData);
24
25return rowData;

在上面的代码中添加一个名为 f_selectedRowValues 的新函数。更改 var data 值的 Handlebar 模板，使其指向 {{q_allFlights}}，并将 selectionColumn 和 selection 变量指向我们的 w_flightTable。

要了解selectedRowKeys是什么，请回顾表格小部件的代码。我们将"selectionIdentityColumnId"设置为"flight_id"。这意味着列flight_id被用作给定行的唯一键。一旦用户在表格中选择一行，该唯一键（航班ID号）就会被添加到selectedRowKeys中。

selectedRowKeys是表格小部件的一个属性，只能通过用户交互进行修改。在编辑小部件时，您可以在JSON编辑器下方看到只读的用户交互属性值。

现在，我们可以通过选择Test来在表格中进行选择，以验证功能行为。

在配置好我们的选择功能后，我们可以添加一个文本小部件，将其命名为w_linkingButton，并将其移动到表格上方。在小部件的属性编辑器中，切换到HTML选项卡（位于内容下），并输入以下内容：

Copied!
1<a
2     class = 'pt-button pt-intent-primary pt-icon-document-open'
3     target = '_blank'
4     href = 'https://registry.faa.gov/AircraftInquiry/Search/NNumberResult?nNumberTxt={{f_selectedRowValues.[0].tail_num}}'
5     role='button'>
6        Learn More About Aircraft {{f_selectedRowValues.[0].tail_num}}
7</a>

上面的代码具有两种功能。添加 pt-button 会使小部件看起来像一个按钮，而添加 pt-intent-primary 会将背景变为蓝色。添加 "pt-icon-document-open" 会在按钮上添加一个蓝图图标。你可以从 Blueprint 文档 ↗ 中查看完整的图标列表。

接下来，我们通过将用户输入添加到基础 URL 来构建一个关联。然后，我们设置显示文本。

此外，我们可以添加一段样式来右对齐小部件，以便于对齐：

sl-text {text-align: right}

文本和关联都依赖于上面配置的选择函数：{{f_selectedRowValues.[0].tail_num}}. f_selectedRowValues 指的是从我们在教程中之前创建的表格选择中读取的函数。

当没有选择任何内容时，文本小部件将显示 了解有关飞机的更多信息。此时选择它将跳转到一个错误链接。

一旦选择了表格中的一行，按钮文本将更新以反映所选内容的名称；选择该按钮将带您到正确的网页。

确保保存您的应用程序，并从右上角选择 X 以进入查看模式。
