使用 Handlebars 访问值

Slate 在查询、函数和组件中采用 Handlebars ↗ 进行模板渲染。Handlebars 模板包含在 {{ }} 中，可访问 Slate 对象，例如组件、变量以及查询或函数的结果。举例而言，若存在下拉组件 w1，即可通过模板 {{w1.selectedValue}} 获取所选值。Slate 会自动对模板进行求值，并将其替换为计算后的结果。

Handlebars 模板还能调用辅助函数。有关更多信息，请参阅《辅助函数》。

哪些 Slate 对象支持引用？

变量

可通过 {{variableName}} 访问在变量编辑器中定义的变量。例如，若存在变量 var1 = {"a": [1,2,3]}，那么模板 {{var1.a.[0]}} 的计算结果即为 1。 此外，还能访问 Slate 环境变量，其中包含当前应用用户的信息或用户专属存储的数据。例如，使用模板 {{$global.user.firstName}} 将展示用户的名字。

查询

支持访问查询结果。例如，{{myQuery}} 包含了来自 SQL 或 HttpJson 查询的结果。此外还存在一个隐藏的 _response 字段，其中存放了部分元数据。

1{"name": ["Ceres","Pallas"],"earthmoid": [1.59376,1.23071],"pha": [false,false],"datefirstobs": ["1830-04-18","1825-03-23"]"_response": {"hasRun": true,"message": "无法解析查询。","success": false}}

函数

你可以通过将函数名包裹在双花括号中来访问其返回值。例如，{{myFunction}}。

组件属性

你可以访问特定的组件属性。具体而言，模板可以访问叶子属性（包括数组及其子属性）。例如，假设我们有一个图表组件。

哪里可以使用 Handlebars？

查询编辑器

在查询编辑器中，Handlebars 模板始终会被替换为字符串值。如果模板求值后的结果不是字符串，则会先将该值转换为字符串。这是因为查询语句本身始终是一个字符串。例如：

已复制！
1SELECT {{column w8.selectedValues}} FROM allNamed WHERE name = {{param w12.selection.xValues}};

非字符串值会通过 toString 方法进行转换。这种方法对于整数和布尔值通常效果良好，但在处理更复杂的对象时可能会产生意外输出。例如，对对象执行 toString 会得到 [object Object]，这多半不是你想要的结果。你可以使用 {{jsonStringify myObject}} 等辅助函数来解决此问题。

函数编辑器

与查询不同，函数编辑器中的 Handlebars 模板始终会被其计算后的值替换。例如：

已复制！
1var a = {{query1}};  // query1 是一个对象
2var b = a.results[0];  // 你可以访问其属性。

在函数编辑器中无法使用辅助函数，必须直接引用 Slate 对象。此外，由于无法更改函数外部的任何状态，因此不能设置变量或属性值。例如：

Copied!
1var c = {{jsonStringify query1}}; //invalid - cannot use helpers
2{{var1}} = [1,2,3];  // invalid - cannot set global variables

当应用处于编辑模式时，您可以在组件的属性编辑器中使用 Handlebars。在组件中，Handlebars 模板必须始终包含在双引号 "" 中，因为花括号不是合法的 JSON 语法。


```json
{
  "property1": "{{variable1}}",
  "property2": "Hello, my name is {{var1}}",
  "property3": [1, 2, "{{var3}}"],
  "property4": {{variable1}}  //无效：JSON 解析错误
}
```


通常情况下，组件中的模板会被替换为字符串。如果计算后的值不是字符串，Slate 将对该值执行 toString 操作。此规则存在少数例外。当满足以下条件时，Slate 会直接用计算后的值替换模板：


    

  1. 仅存在一对括号 {{}}
且
    2. 

  2. 括号外没有其他任何字符（包括空格）。
    3.

已复制！
1{
2  // 值替换示例
3  "property1": "{{var1}}",
4  "property2": "{{ var1 }}",  // 空格在双花括号内
5  "property3": "{{ add 2 (add 1 var1) }}",  // 助手函数，支持嵌套
6  "property4": ["{{var1}}", "{{w1.property.[0]}"],

哪里不能使用 Handlebars？

  • 不能在变量编辑器中使用 Handlebars。
  • 不能在样式编辑器中使用 Handlebars。
  • 不能在应用的查看模式下使用 Handlebars。如果将 Handlebars 输入到组件或文本字段中，表达式既不会被求值，也不会被替换。

教程：关联组件

以下教程将指导您如何创建一个按钮，让用户可以选择了解特定航班某架飞机的更多信息。这需要我们在两个组件之间建立关联。

要关联组件，您必须知道组件的名称以及想要关联的属性。在我们的示例中，我们需要获取原始表格中所选行的 tail_num 值，但我们之前配置表格选择时，将 flight_id 设置为了主键。

这正是使用另一个辅助函数的好时机。由于获取所选表格行的列值是一种常见模式，我们可以使用一个函数来实现这一操作：



已复制！
1function getRowData(s){
2    var returnObj = {};
3    var index = data[selectionColumn].indexOf(s);
4
5    //此处遍历数据对象中的每个数组
6    //value 为实际数组，key 为该数组的键名
7    _.forOwn(data, function(value, key){
8        returnObj[key] = value[index];
9    });
10
11    return returnObj;
12}
13
14// 输入表格的同一份数据
15var data = {{f_data}};
16
17// 表格中的当前选中项
18var selection = {{w_mainTable.selectedRowKeys}};
19
20// 配置为表格主键的列
21var selectionColumn = {{w_mainTable.selectionIdentityColumnId}}
22
23var rowData = _.map(selection, getRowData);
24
25return rowData;

在上面的代码中添加一个名为 f_selectedRowValues 的新函数。将 var data 的 Handlebar 模板值指向 {{q_allFlights}}，并将 selectionColumn 和 selection 变量指向我们的 w_flightTable。

要理解 selectedRowKeys 的含义，可以回顾一下表格小部件的代码。我们将 "selectionIdentityColumnId" 设置为 "flight_id"。这意味着 flight_id 列将作为特定行的唯一键。当用户在表格中选中某一行时，该唯一键（即航班 ID 号）就会被添加到 selectedRowKeys 中。

selectedRowKeys 是表格小部件的一个属性，只能通过用户交互来修改。在编辑小部件时，可以在 JSON 编辑器下方看到用户交互属性的只读值。

现在，我们可以通过点击 Test 在表格中进行选择，以验证函数的行为。

配置好选择函数后，我们可以添加一个文本小部件，将其命名为 w_linkingButton，并将其移动到表格上方。在该小部件的属性编辑器中，切换到 Content 下的 HTML 标签页，并输入以下内容：



已复制！
1<a
2     class = 'pt-button pt-intent-primary pt-icon-document-open'
3     target = '_blank'
4     href = 'https://registry.faa.gov/AircraftInquiry/Search/NNumberResult?nNumberTxt={{f_selectedRowValues.[0].tail_num}}'
5     role='button'>
6        了解更多关于飞机 {{f_selectedRowValues.[0].tail_num}} 的信息

上述代码实现了两项功能。添加 pt-button 使该组件呈现按钮外观，而添加 pt-intent-primary 则将背景色设为蓝色。添加 "pt-icon-document-open" 会为按钮增加一个 Blueprint 图标。你可以在《Blueprint 文档 ↗》中查阅完整的图标列表。

接下来，我们通过将用户输入追加到基础 URL 来构建一个链接。随后，我们设置显示文本。

此外，我们可以添加一小段样式代码，使该组件右对齐，以便更轻松地进行布局：

sl-text {text-align: right}

文本和链接都依赖于上文配置的选择函数：{{f_selectedRowValues.[0].tail_num}}。f_selectedRowValues 指代的是我们此前在教程中创建的、用于读取表格选择内容的函数。

未选中任何内容时，文本组件将显示 了解更多飞机信息。此时点击该组件会跳转至一个无效链接。

选中表格中的某一行后，按钮文本会随之更新以反映所选内容的名称；此时点击按钮即可跳转至正确的网页。
