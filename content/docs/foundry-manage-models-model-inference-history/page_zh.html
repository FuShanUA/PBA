模型推理历史

模型推理历史是 Foundry 中的一个数据集，用于捕获发送至建模目标（Modeling Objective）实时部署的所有推理请求（输入）和推理结果（输出）。这可以支持多种工作流，例如：

  • 漂移检测：模型推理历史可帮助您检测模型性能随时间发生的变化，这可能表明底层数据分布发生了漂移。检查推理历史能够为模型准确率的演变提供有价值的洞察。
  • 持续重训练：模型推理历史可用于持续重训练场景，在此场景中，模型会基于先前的预测结果，使用新数据进行迭代训练。
  • 模型性能评估：推理输入和输出的历史记录可用于评估模型随时间变化的性能表现。推理历史有助于识别模型性能中的模式和趋势。
  • 使用分析：推理历史可帮助您了解模型被调用的位置及频率。这些对模型使用模式的洞察，可以为资源分配与优化决策提供依据。

模型推理历史的使用

要创建模型推理历史，请导航至建模目标的“部署”页面，并选择您想要添加模型推理历史的实时部署。在“模型推理历史”卡片下，点击“创建数据集”。这将打开“创建新模型推理历史”对话框。

在“创建新模型推理历史”对话框中，输入模型推理历史的数据集名称和保存位置。我们强烈建议添加《安全标记》，因为输入和输出可能包含敏感信息，仅应允许具有相应安全权限的人员访问。

创建模型推理历史后，数据集中将记录以下信息：

  • timestamp：输入请求的时间戳
  • user_id：发送请求的 Foundry 用户 ID
  • request_uuid：请求的唯一标识符
  • live_rid：建模实时部署资源标识符
  • objective_rid：建模目标资源标识符
  • model_rid：模型资产资源标识符
  • model_version_rid：模型资产版本资源标识符
  • input：发送给模型的输入的 JSON 表示形式
  • output：模型输出的 JSON 表示形式
  • error：模型在对提供的输入进行推理时遇到的详细错误消息或堆栈跟踪

要临时启用或禁用模型推理历史，请点击“模型推理历史”卡片右上角标有“启用记录”的开关。

要永久禁用模型推理历史，请在“模型推理历史”卡片中数据集右侧选择“移除”。

移除模型推理历史是永久性操作，无法撤销。一旦数据集作为模型推理历史被移除，该数据集仍会存在，但无法重新添加回部署中。

常见问题解答 (FAQ)

在我的实时部署上设置模型推理历史需要哪些权限？

要启用模型推理历史，用户需要拥有网关权限 foundry-ml-live:edit-inference-ledger。通常，授予建模目标上的“数据主人”角色即可获得此权限。

我应该将模型推理历史数据集保存在哪里？

由于输入和输出可能包含敏感信息，您需要将该数据集保存至与其父级建模目标相同的项目中。

我可以将模型推理历史的输入和输出写入现有数据集吗？

为确保模型推理历史与指定模式相匹配，您不能为模型推理历史选择现有的数据集。

如果移除模型推理历史会怎样？

如果移除模型推理历史，数据集不会被删除，但 Foundry 会停止向其中写入新记录。

如何临时启用或禁用模型推理历史？

可以使用录制开关来临时启用或禁用模型推理历史。

能否保证所有推理输入都会记录到模型推理历史中？

并非所有输入和输出都必然会出现在推理历史中。为了维持平台性能，尝试写入模型推理历史失败的请求不会进行重试。
