转换

Foundry Transforms 是数据处理管道的核心构建块。它们定义了数据从输入到输出的流动方式，将计算逻辑和资源需求封装在一个单一的声明式单元中。

转换被注册在 Pipelines 中，这些可以由多个转换定义组成。

定义转换

每个转换包含三个关键组件：

  1. 装饰器： 定义输入、输出和配置
  2. 函数签名： 与装饰器中声明的输入和输出匹配
  3. 计算逻辑： 实际的数据处理代码

以下是定义转换的基本模式：

Copied!
1from transforms.api import transform, Input, Output
2
3
4@transform.using( # decorator
5    hair_eye_color=Input('/examples/students_hair_eye_color'), # inputs
6    processed=Output('/examples/hair_eye_color_processed') # outputs
7)
8def filter_hair_color(hair_eye_color, processed):  # function signature matches input/output
9    ... # compute logic

Python 转换可以配置为单节点（轻量级）或多节点（Spark）计算引擎 . 默认情况下，转换在单节点上运行，您可以将数据作为 Polars ↗ 或 pandas ↗ DataFrame 加载。Polars 是一个用于转换表格数据的 DataFrame 库。它以性能、稳定性和易用性而闻名。Pandas 是一个广泛采用且易于使用的数据分析工具。

要获取 DataFrame，请在您的输入上调用相应的方法，如下所示：

Copied!
1import polars as pl
2from transforms.api import transform, Input, Output
3
4
5@transform.using(
6    hair_eye_color=Input('/examples/students_hair_eye_color'),
7    processed=Output('/examples/hair_eye_color_processed')
8)
9def filter_hair_color(hair_eye_color, processed):
10    # Load data as Polars DataFrame
11    df = hair_eye_color.polars()
12
13    # Apply filtering logic
14    filtered = df.filter(pl.col('hair') == 'Brown')
15
16    # Write the result
17    processed.write_table(filtered)

处理多种数据格式

轻量级的转换 API 支持以各种格式加载数据，如下面的示例所示：

Copied!
1@transform.using(output=Output('/Project/folder/output'), dataset=Input('/Project/folder/input'))
2def compute(ctx, output, dataset):
3    polars_df = dataset.polars()         # polars_df is a polars.DataFrame object
4    lazy_df = dataset.polars(lazy=True)  # Activate streaming mode, lazy_df is a polars.LazyFrame
5    pandas_df = dataset.pandas()         # pandas_df is a pandas.DataFrame
6    arrow_table = dataset.arrow()        # arrow_table is a pyarrow.Table
7
8    # DuckDB connection with pre-registered input datasets
9    conn = ctx.duckdb().conn
10    df_from_duckdb = conn.sql("SELECT * FROM dataset").fetchdf()  # dataset is automatically registered
11
12    output.write_table(lazy_df)          # Any of the formats above can be passed to write_table

请注意，调用 dataset.pandas() 需要您的环境中安装 pandas。同样，dataset.polars(...) 需要 Polars 可用。

配置计算资源

您可以使用 .with_resources() 方法控制分配给转换的计算资源。

Copied!
1from transforms.api import transform, Input, Output
2
3
4@transform.using(
5    large_dataset=Input('/datasets/large_file'),
6    processed=Output('/datasets/processed')
7).with_resources(
8    cpu_cores=8,          # Use 8 CPU cores
9    memory_gb=16,         # Allocate 16GB memory
10)
11def process_large_dataset(large_dataset, processed):
12    ...
  • cpu_cores: CPU 核心数量。可以是小数，例如 0.5，默认为 2。
  • memory_mb 或 memory_gb: 内存分配；只使用其中一个，不要同时使用两个。

转换上下文

有时数据转换可能依赖于输入数据集以外的其他因素。例如，转换可能需要访问当前的 Spark 会话或联系外部服务。在这种情况下，您可以将一个 TransformContext 对象注入到转换中。

要注入一个 TransformContext 对象，您的计算函数必须接受一个名为 ctx 的参数。请注意，这意味着输入或输出的名称不能为 ctx。有关 TransformContext 的完整功能，请参阅 API 参考 ↗。

Copied!
1from transforms.api import transform, Output
2
3
4@transform.using(
5    out=Output('/examples/context')
6)
7def generate_dataframe(ctx, out):
8    ...

Pyspark 转换

默认情况下，转换在单个计算节点上运行，并利用 pandas 或 Polars 等数据处理库。对于大数据计算工作流，可以使用 PySpark ↗ 在 Python 转换中。

PySpark 是一种包装语言，允许您与 Apache Spark 后端进行交互以快速处理数据。Spark 可以在分布式服务器网络上处理非常大的数据集，这在正确使用时提供了扩展性和可靠性优势。然而，它也伴随着更高的开销和资源使用，因此对于中小规模的数据来说不是一个好选择。

虽然这里给出了 PySpark 转换 API 的概述，但请参考 Python 转换（Spark） 以获取更多详细信息。

定义 PySpark 转换

PySpark 转换的定义语法与标准转换类似。

Copied!
1from transforms.api import transform, Input, Output
2
3
4@transform.spark.using(
5    hair_eye_color=Input('/examples/students_hair_eye_color'),
6    processed=Output('/examples/hair_eye_color_processed')
7)
8def filter_hair_color(hair_eye_color, processed):
9    ... # transform logic goes here

与 Polars 或 pandas 转换类似，可以从转换输入中检索 DataFrame 对象（对象）。

Copied!
1from transforms.api import transform, Input, Output
2
3
4@transform.spark.using(
5    hair_eye_color=Input('/examples/students_hair_eye_color'),
6    processed=Output('/examples/hair_eye_color_processed')
7)
8def filter_hair_color(hair_eye_color, processed):
9    filtered_df = hair_eye_color.dataframe().filter(hair_eye_color.dataframe().hair == 'Brown')
10    processed.write_dataframe(filtered_df)

转换输入返回的 DataFrame ↗ 对象（对象）是 PySpark DataFrame（数据集）。有关使用 PySpark 的更多信息，请参阅 Spark Python API 文档 ↗。

PySpark 转换也可以接受多个输入和输出：

Copied!

8    females=Output('/examples/hair_eye_color_females'),
9)
10def filter_hair_color(hair_color, eye_color, males, females):
11    ...

由于 PySpark 转换仅支持一种数据处理库，可以使用 transform_df 装饰器直接访问 DataFrame ↗ 以方便操作。

Copied!
1from transforms.api import transform_df, Input, Output
2
3
4@transform_df(
5    Output('/examples/hair_eye_color_processed'),
6    hair_eye_color=Input('/examples/students_hair_eye_color')
7)
8def filter_hair_color(hair_eye_color):
9    return hair_eye_color.filter(hair_eye_color.hair == 'Brown')

配置 PySpark 转换

在 PySpark 转换中，资源通过 spark profiles 进行配置。这些允许对大规模计算使用进行严格的访问控制。

转换逻辑层级版本控制

警告

为了使 TLLV 正常工作，您的代码必须在模块级别声明所有导入，并且不应尝试修补或以其他方式修改另一个模块中的对象。如果您的项目不符合此要求，请 您必须禁用 TLLV。有关更多信息，请参阅下面的代码示例。

TLLV 默认是启用的。 要禁用 TLLV，请将 transformsPython 配置中的 tllv 设置为 false。此配置位于您的 Python transforms 子项目中的 build.gradle 文件中。

transformsPython {
    tllv false
}

转换的版本是一个字符串，用于在考虑逻辑过时时比较两个转换版本。如果转换的输入未更改且转换的版本未更改，则转换的输出是最新的。如果版本发生变化，转换的输出将被标记为无效并重新计算。

默认情况下，转换的版本包括以下内容：

  • 转换定义的模块
  • 转换所依赖的所有模块
  • 任何项目依赖项

如果这些中的任何一个发生变化，版本字符串将被更改。 如果您希望在未列出的部分中发生更改时使输出失效，请在 tllvFiles 配置中设置 transformsPython。一个使用案例是，当您读取文件的配置并希望在配置更改时使输出失效时。

事务性

转换操作将在事务上下文中执行，该事务在输出上打开，并在作业成功时提交。如果未在输出或写入期间定义逻辑，或者错误被捕获并吞没，则成功的转换将在这些输出上产生一个空快照。
