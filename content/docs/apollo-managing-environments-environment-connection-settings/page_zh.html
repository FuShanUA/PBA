配置环境连接设置

连接设置决定了Apollo如何处理环境的< a href="/docs/apollo/core/agents/">代理连接。环境中的代理可以直接连接到当前的Hub或者另一个Hub。代理连接控制着在Apollo中显示的健康和活动信息以及是否发布计划。

连接设置选项

未知

在环境可以操作使用之前，您必须手动配置连接设置。当代理首次连接或连接状态被导入时，Apollo也可以自动将连接设置转换为另一种状态。

您不能返回到此状态。

代理已连接

代理直接连接到当前的中心，并将它们的状态报告给该中心。

特点：

  • 代理可以直接连接到当前的中心。
  • 会发布计划。
  • 实时显示活动和健康信息。
  • 不允许通过捆绑包导入报告的状态。
  • 支持版本通道提升。

不期望连接

此环境被显式配置为在此中心不期望代理连接或健康状态。 此设置适用于导出并连接到另一个中心的环境，且报告状态无法导入。

特点：

  • 代理不能连接到此中心。
  • 不会发布计划。
  • 不允许通过捆绑包导入报告的状态。
  • 不能参与版本通道提升。

从其他中心中继

代理状态通过捆绑包报告到另一个中心，并导入到此中心以供查看。 此设置适用于创建连接到其他中心的环境的只读视图。

特点：

  • 代理不能连接到此中心。
  • 不能从此中心发布计划。
  • 健康状况和报告的状态信息通过捆绑包导入。
  • 版本和健康信息按最后一次导入的捆绑包显示。
  • 支持发布渠道提升。

管理连接设置

查看当前连接设置

环境的当前连接设置显示在环境标题中。

编辑连接设置

  1. 导航到您的环境。
  2. 选择 编辑环境连接 从 操作 下拉菜单。
  3. 选择新的连接设置：
    • 代理已连接: 启用直接代理连接到此中心。
    • 不期望有连接: 禁用代理连接并隐藏操作标签页。
    • 从其他中心中继: 从捆绑包导入代理状态。
  4. 确认并应用更改。

更改连接设置时有三个重要注意事项：

  • 从 代理已连接 更改将断开现有的代理并取消活动的计划。
  • 切换到 不期望有连接 将隐藏操作信息。
  • 启用 从其他中心中继 需要从其他中心导入一个捆绑包。
