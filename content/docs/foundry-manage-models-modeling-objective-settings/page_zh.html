建模目标设置

建模目标有多个配置选项，如下所述。您可以通过点击界面右上角的建模目标设置，从建模目标主页导航到建模目标设置页面。

检查

目标检查是一种确保模型在投入运行前通过预定义的质量检查的方法。有关目标检查的完整详细信息，请参阅文档中关于< a href="/docs/foundry/manage-models/set-up-checks/">如何为所有提交设置检查的部分。

部署

部署设置页面定义了在Python模型提交进行批量或实时部署之前，模型提交部署配置的要求。

模型元数据

提交到建模目标的模型可以在建模目标内包含特定于模型提交的元数据。可以为某个目标配置该元数据，以确保跟踪有关模型的相关信息。有关完整配置详细信息，请参阅关于 如何配置目标元数据 的文档。

外观

通过外观设置，您可以配置单个建模目标是显示为浅色模式还是深色模式。

高级

高级设置页面包含非标准的配置选项。通常，建模目标的高级设置不应进行调整。这些设置是为了允许与旧功能的向后兼容性。

评估设置

评估仪表板有两个高级配置选项： 仅显示由评估配置生成的指标 和 在评估仪表板中显示固定标签。

仅显示由评估配置生成的指标

当建模目标设置为仅显示由评估配置生成的指标时，则只显示通过 评估配置 创建的指标 将显示在 评估仪表板 上。我们建议启用此设置，以确保您的建模目标中模型的标准化评估过程。如果未启用此设置，则您的建模目标无法知道显示的模型指标是否是最新的。

通过旧版管道管理配置的指标管道除非关闭 仅显示由评估配置生成的指标，否则不会显示。

在某些情况下，例如对于在评估配置发布之前配置的建模目标，或者具有 在代码中定义的指标 的非生产建模目标，可能更希望从建模目标评估配置之外显示指标，可以通过此设置启用。

在评估仪表板中显示固定标签

可选地，可以配置建模目标上的评估仪表板以显示 自定义指标视图 以及评估仪表板上的子集标签。

推断和指标数据集命名

由旧版指标管道管理生成的数据集可能会在多个建模目标在同一输出文件夹中创建推断和指标数据集时产生命名冲突，从而导致意外行为。此问题已在更新后的评估仪表板和配置中解决。

为了保持向后兼容性，仍可以使用旧版的推断和指标数据集命名方案。

更新的命名约定：

  • 推断数据集: inference_<Model Submission Name>_<Evaluation Dataset Name>-<Generated Hash>
  • 指标数据集: metrics_<Model Submission Name>_<Evaluation Dataset Name>_<Evaluation Library Name>-<Generated Hash>

旧版命名约定：

  • 推断数据集: <Model Submission Name> inference
  • 指标数据集: <Model Submission Name> metrics
