二分类评估器

建模目标中的默认评估库之一是二分类评估器。该库提供了一套核心的常用指标，用于评估二分类模型。

包含的指标

以下指标会针对评估仪表盘中配置的每个子集分桶生成。例如，如果生成的子集在模型预测列或标签列中未同时包含两个类别，则可能无法在该子集上生成所有指标。

默认的二分类评估器会生成以下数值指标：

  • 行数：评估数据集中的记录数量。
  • 先验率：评估数据集中正类结果的比例。请注意，先验率并非衡量模型性能的直接指标，但有助于为其他指标提供背景参考。
  • 准确率：被正确分类的记录比例。模型的准确率介于 0 和 1 之间，准确率为 1 表示模型正确分类了评估数据集中的每一条记录。
  • 精确率：真正例数除以真正例数与假正例数之和。该值介于 0 和 1 之间，1 表示模型精确率完美。
  • 召回率：真正例总数除以真正例数与假负例数之和。该值介于 0 和 1 之间，1 表示模型召回率完美。也称为灵敏度。
  • F1 分数：精确率和召回率的调和平均数，计算方式为精确率与召回率的乘积除以二者之和。模型的 F1 分数介于 0 和 1 之间，1 表示模型正确分类了评估数据集中的每一条记录。
  • 马修斯相关系数 (MCC)：一种衡量模型性能的数值指标，它同等权重地结合了两个分类类别下标签值与预测值之间的数值相关性。因此，只有当分类器在两个类别上均表现良好时，MCC 才会较高。MCC 介于 -1 和 1 之间，分数为 1 表示每条记录都被正确分类。
  • ROC AUC 分数（需要概率值）：ROC（受试者工作特征）曲线下的面积；AUC 代表“曲线下面积”。取值范围从 0 到 1，其中分数为 1 代表完美模型，分数为 0.5 是随机猜测模型的预期分数。

默认的二分类评估器会生成以下图表：

  • 混淆矩阵：展示评估数据集中预测的真正例、真负例、假正例和假负例的数量。
  • 分数分布：模型在评估数据集上预测结果的条形图。
  • 概率分布（需要概率值）：模型在评估数据集上输出概率的条形图。分桶宽度为 0.05，因此在 [0.0, 1.0] 范围内总共有 20 个桶。
  • ROC 曲线（需要概率值）：绘制不同概率阈值下的真正例率与假正例率。曲线越陡峭，通常表明模型性能越好。
  • 真正例率 (TPR) 曲线（需要概率值）：TPR 曲线展示了真正例率与模型预测概率之间的关系。
  • 假正例率 (FPR) 曲线（需要概率值）：FPR 曲线展示了假正例率与模型预测概率之间的关系。
  • 提升曲线（需要概率值）：显示模型在每个概率下的提升度；提升度是衡量模型在正类上相对于随机猜测表现的指标。

配置

有关完整的配置说明，请参阅如何配置模型评估库的文档。

必填字段

二分类评估器需要以下必填字段。这些列的预期值类型为整数。

  • inference_field：代表模型预测分类的列。该评估器假设 1 为正类，0 为负类。
  • actual_field：包含模型预测比较基准值的列。该评估器假设 1 为正类，0 为负类。

可选字段

  • probability_field: 这是一个可选字段，表示预测为正类的概率。提供 probability_field 时，默认的二分类评估库将生成以下指标：

    • ROC AUC 分数
    • ROC 曲线
    • TPR 曲线
    • FPR 曲线
    • 提升度曲线

  • max_samples_for_roc 生成模型 ROC 曲线所用的最大样本数。若未提供，默认值为 200。
