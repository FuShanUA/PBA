卡片输入和输出类型

通过一系列可以相互依赖的卡片构建Quiver分析，形成一个分析图。Quiver中的每个卡片都可以有零个或多个输入，并生成特定类型的输出。

例如：

  • 一个 数值聚合 卡片将 object set 作为输入，并生成 number 作为输出。
  • 一个 滚动聚合 图表将 time series 作为输入，并生成 time series 作为输出。
  • 一个 数字转日期 卡片将 number 作为输入，并生成 time 作为输出。

查看完整的输入/输出类型列表。

如果另一张卡片的输出类型等于下游卡片的输入类型，那么一张卡片才能被添加为另一张卡片的输入。有时，如果两个卡片的输入/输出类型不匹配，则必须在它们之间添加一个转换卡片。

例如，过滤对象集不能直接作为输入传递给 对象属性，因为过滤对象集的输出是一个对象集，而对象属性需要单个对象作为输入。为了解决这个问题，必须在两张卡片之间放置一个 对象选择器，因为对象选择器以对象集作为输入并输出单个对象。

理解一张卡片的输出类型以及它可以用于哪些下游卡片是 Quiver 中一个非常重要的概念。幸运的是，Quiver 的 下一步操作菜单（当悬停在卡片上时显示）有助于简化这个问题。它只显示能够将您当前卡片的输出类型作为输入的卡片。下一步操作菜单所使用的类型显示在菜单的右上角。

此外，当在编辑器中配置 Quiver 卡片的变量输入时，只显示具有兼容输出类型的卡片以供选择。例如，在下面的图片中，当配置 数字到日期 卡片的输入数字时，我们可以将数值参数作为输入（$I），但不能将字符串参数作为输入（$H）。
