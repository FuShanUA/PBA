卡片输入与输出类型

Quiver 中的分析由若干个相互依赖的卡片构建而成，进而形成一张分析图。Quiver 中的每张卡片都可以接收零个或多个输入，并产生特定类型的输出。

例如：

  • 数值聚合卡片接收 对象集 作为输入，并输出一个 数字。
  • 滚动聚合图接收 时间序列 作为输入，并输出一个 时间序列。
  • 数字转日期卡片接收 数字 作为输入，并输出一个 时间。

查看输入/输出类型的完整列表。

只有当某张卡片的输出类型与下游卡片的输入类型相匹配时，该卡片才能作为下游卡片的输入。有时，如果两张卡片之间的输入/输出类型不匹配，就必须在它们之间插入一张转换卡片。

例如，过滤对象集无法直接作为对象属性的输入，因为过滤对象集输出的是一个对象集，而对象属性接收的是单个对象。要解决这一问题，必须在两张卡片之间放置一个对象选择器，因为对象选择器接收对象集作为输入，并输出单个对象。

在 Quiver 中，理解卡片的输出类型以及该输出能用于哪些下游卡片，是一个非常重要的概念。幸运的是，当鼠标悬停在卡片上时，Quiver 的后续操作菜单能帮助简化这一问题。该菜单仅显示能够接收当前卡片输出类型作为输入的卡片。后续操作菜单所使用的类型会显示在菜单的右上角。

此外，在编辑器中为 Quiver 卡片配置变量输入时，系统仅显示输出类型兼容的卡片供选择。例如，在下图中，当为数字转日期卡片配置输入数字时，我们可以选择数值参数作为输入（$I），但无法选择字符串参数（$H）。

输入与输出类型列表

下方提供了 Quiver 支持的输入与输出类型的完整列表。

类型描述
对象集由本体支撑的一组对象。适用于中等规模的简单、响应式分析。
单个对象本体中的单个对象。
分类图表由 (字符串, 数字) 或 (字符串, 字符串, 数字) 值构成的图表。
对象选择支持通过交互选择对象的卡片。
透视表通过透视表聚合产生的表格数据。
本体 SQL查询本体对象数据所产生的 SQL 查询结果。可转换为转换表以进行进一步分析。
转换表用于灵活、小规模分析的本地表。可用于在不同数据类型之间进行转换、编辑或变换。
物化由数据集支撑的对象物化形式，适用于灵活的大规模分析。
时间序列由 (值, 时间戳) “数据点”组成的时间序列。适用于高频、基于时间的分析。
时间序列图一种交互式的基于时间的图表，能够可视化时间序列、时间范围、事件集和时间点。
时间序列组一组可以一起可视化或转换的时间序列。
限定时间序列由上限时间序列和下限时间序列界定的区域。
事件集包含开始时间和结束时间的事件集合。
时间散点图从时间序列散点图返回的数据。
字符串单个字符串值。
数字单个数值。
时间单个日期/时间值。
布尔值单个布尔值。
持续时间单位时间单位（毫秒、秒、分钟、小时、天、周）。
字符串数组字符串值的数组。
数字数组数值的数组。
时间数组日期/时间值的数组。
布尔数组布尔值的数组。
数值范围由起始数值和结束数值构成的范围。
时间范围由起始日期/时间值和结束日期/时间值构成的范围。
X/Y 范围由两个范围组成的集合，用于在图表上创建一个“方框”。
流起始指示卡片不接收任何输入。
流结束指示卡片不产生任何输出类型。
