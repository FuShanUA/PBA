SQL 语法

Foundry SQL 通常为 Spark SQL ↗ 的一个子集，并符合 ANSI 标准。

关键语法说明

  • 子查询 必须 被括号包裹。
  • 默认创建的表是Iceberg，但可以使用 USING <format> 语法覆盖。
  • 只有Iceberg表支持 INSERT, UPDATE, 和 DELETE。

表创建

CREATE [ OR REPLACE ] TABLE table_identifier [ ( col_name1 col_type1, ... ) ] [ USING table_format ] [ PARTITIONED BY ( expr [, ...] ) ] [ TBLPROPERTIES ( 'key'='value' [, ...] ) ] [ AS select_statement ]

PARTITIONED BY 仅支持 Iceberg 表。 TBLPROPERTIES 设置 Iceberg 表属性 ↗。

如果没有指定，表格式将是 Iceberg。有效的表格式包括：

  • Iceberg: iceberg
  • Parquet: parquet
  • Avro: avro

3SELECT *
4FROM `/path/to/table`;
5
6CREATE TABLE `/path/to/table` USING parquet AS
7SELECT *
8FROM `/path/to/table`;

事务块

事务块将多个语句组合在一起，作为一个单一的原子作业执行。

BEGIN;
statement1;
statement2;
...
COMMIT;

规则：

  • BEGIN/COMMIT 内的所有语句在一个作业中一起执行。
  • 事务块不能嵌套。
  • 事务块必须至少包含一个写入语句。
  • BEGIN 未包含 COMMIT 或 COMMIT 未包含 BEGIN 是解析错误。
Copied!
1-- Create and populate a table atomically
2BEGIN;
3CREATE OR REPLACE TABLE `/path/to/table` (id INT, name STRING);
4INSERT INTO `/path/to/table` VALUES (1, 'foo'), (2, 'bar');
5COMMIT;
6
7-- Multiple inserts into separate tables as a single job
8BEGIN;
9INSERT INTO `/path/to/table1` SELECT * FROM `/path/to/source`;
10INSERT INTO `/path/to/table2` SELECT * FROM `/path/to/source`;
11COMMIT;

SELECT 语句

其中 select_statement 为：

SELECT [ ALL | DISTINCT ] { [ named_expression | regex_column_names | * ] [, ...] }
  FROM { from_item [, ...] }
  [ WHERE boolean_expression ]
  [ GROUP BY { expression | ROLLUP(...) | CUBE(...) | GROUPING SETS(...) } [ , ... ] ]
  [ HAVING boolean_expression ]
  [ LIMIT { ALL | expression } [ OFFSET expression ] ]

Copied!
1-- Examples
2SELECT * FROM `/path/to/table` VERSION AS OF `ri.foundry.main.transaction.1234`;
3
4SELECT * FROM `/path/to/table` VERSION AS OF 1234567;

对于 Foundry 数据集，您可以使用 VERSION BETWEEN ... AND ... 查询在事务范围内的新增行。该查询返回严格在起始事务之后且包括结束事务的新增行。起始边界是不包含的，而结束边界是包含的。两个边界必须是事务 RID，而不是快照 ID。

Copied!
1-- Rows appended after txn.1234, up to and including txn.5678
2SELECT * FROM `/path/to/table`
3    VERSION BETWEEN `ri.foundry.main.transaction.1234` AND `ri.foundry.main.transaction.5678`;

VERSION BETWEEN 不适用于 Iceberg 表。其他时间旅行语法目前不被支持。

完整查询示例

Copied!
1-- Complex query with joins and filtering
2SELECT DISTINCT o.order_id,
3                c.name,
4                o.total_amount,
5                p.category
6FROM `/data/orders` o
7         LEFT JOIN `/data/customers` c ON o.customer_id = c.id
8         CROSS JOIN (SELECT product_id, category
9                     FROM `/data/products`
10                     WHERE category = 'Electronics') p ON o.product_id = p.product_id
11WHERE o.order_date > DATE '2024-01-01'
12  AND o.total_amount > 1000.0
13ORDER BY o.total_amount DESC
14LIMIT 100;
15
16-- Aggregation with HAVING clause
17SELECT c.name,
18       COUNT(o.order_id) as order_count,
19       SUM(o.total_amount) as total_spent
20FROM `/data/customers` c
21       LEFT JOIN `/data/orders` o ON c.id = o.customer_id
22GROUP BY c.name
23HAVING COUNT(o.order_id) > 0;
24
25-- UNION query with multiple joins
26SELECT emp_id, name, dept, 'Active' AS status
27FROM `/hr/employees` e
28         LEFT OUTER JOIN `/hr/departments` d ON e.dept_id = d.id
29WHERE e.active = true
30UNION
31SELECT emp_id, name, dept, 'Inactive' AS status
32FROM (SELECT *
33      FROM `/hr/employees`
34      WHERE termination_date < current_timestamp()) e
35         RIGHT JOIN `/hr/departments` d ON e.dept_id = d.id
36WHERE e.active = false
37ORDER BY dept ASC, status DESC NULLS LAST;

SQL 提示

SQL 提示为查询引擎提供额外的指令。提示使用以下语法指定：

/*+ hint1, hint2(opts...) */

目前支持以下提示：

  • adhoc - 以临时查询的方式运行写入查询。新查询将作为单独的目标添加到表中已有的目标旁边，同时保留现有的默认目标。直接重新构建表仍将使用原始的默认目标，而不是临时目标。
Copied!
1/*+ adhoc */
2CREATE OR REPLACE TABLE `/path/to/table` AS SELECT * FROM `/path/to/source`;

Introspection

使用 SHOW FUNCTIONS 来列出 Foundry SQL 支持的函数。该语句返回三列：name, kind（可以是 scalar, aggregate 或 window 中的一个），以及 signature。一个可选的 LIKE 模式用于按函数名称进行过滤，其中 % 匹配任意字符序列，而 _ 匹配单个字符。

Copied!
1SHOW FUNCTIONS;
2SHOW FUNCTIONS LIKE 'abs%';
