为公共应用上传数据

启用数据上传

通过公共应用上传数据，需要通过查询提供两个安全上传数据源：

  1. 用于生成安全通道和令牌的数据源。可以是以下其中之一：
    • 名为 secure-upload 的 SERVICE API 数据源
    • 名为 secure-upload-raw-oauth 的 HTTP JSON 数据源
  2. 用于通过生成的安全令牌上传 JSON blob 的数据源。通常命名为 secure-upload-raw。

你可以通过创建一个公共应用并新建查询来检查它们是否可用。如果安全上传源不可用，请联系 Palantir 支持团队。

公共应用可进行配置，允许用户向 Foundry 上传数据。上传数据的用户无需拥有 Foundry 的访问权限。然而，只有具备相应资源权限的 Foundry 用户才能访问已上传的数据。

要在你的公共应用中设置数据上传，请按照以下说明操作：

1. 创建空数据集

在你选择的 Project 或文件夹中创建一个新的数据集资源。

2. 创建通道

在你的公共 Slate 应用中，基于 secure-upload 或 secure-upload-raw-oauth 源创建一个新查询。勾选 Test 旁下拉菜单中的复选框，将查询设置为仅手动运行。

为你的通道选择一个唯一的 ID，例如 slate-app-name-data-upload。channelId 不能包含空格。请求体需要你之前创建的空数据集的 RID。你可以从数据集的 URL 中获取该数据集的 RID，例如 ri.foundry.main.dataset.abcd1234-1234-5678-9000-1234abcd1234。

使用 secure-upload 创建通道

在 Available 服务中，选择 Channel Service，然后从 Endpoints 下拉菜单中选择 Put channel。

在 channelId 字段中，输入频道的唯一 ID。

在请求体中输入以下 JSON：


```json
{
  "configuration": {
    "dataset": "<在此填入您的数据集RID>",
    "maxBlobSize": <可选：单个二进制大对象的大小上限（以字节为单位）>,
    "options": {
      "type": "json",
      "json": {}
    }
  }
}
```


使用 secure-upload-raw-oauth 创建频道


在查询编辑器中输入以下 JSON：




已复制！
1{
2    "path": "channels/{channelId}",
3    "method": "PUT",
4    "extractors": {
5        "result": "$"
6    },
7    "pathParams": {
8        "channelId": "<在此处填写您的频道名称>"
9    },
10    "queryParams": {},
11    "bodyJson": {
12        "configuration": {
13            "dataset": "<在此处填写您的数据集RID>",
14            "maxBlobSize": <单个二进制大对象的可选大小限制（以字节为单位）>,
15            "options": {
16                "type": "json",
17                "json": {}
18            }
19        }
20    }
21}



基于 secure-upload 或 secure-upload-raw-oauth 源创建查询后，选择测试。响应将返回与您在字段中输入的值相匹配的 channelId。请妥善保存该 channelId，后续步骤将会用到。


用户上传数据时无需再次使用此查询，除非您需要创建更多通道。您可以放心删除该查询。


3. 创建令牌


创建通道后，您需要创建一个令牌。提交安全上传查询以及将数据上传至数据集均需使用此令牌。您需要指定一个过期日期。最长有效期限取决于您注册项的安全上传配置。基于 secure-upload 或 secure-upload-raw-oauth 源创建一个新查询，并将该查询设置为仅限手动运行。


使用 secure-upload 创建令牌


在可用服务中，选择令牌服务，然后从端点下拉菜单中选择创建令牌。


请求体格式如下：




已复制！
1{
2  "type": "blobUploadToDataset",
3  "blobUploadToDataset": {
4    "channel": "<在步骤 2 中定义的 channelId>",
5    "expiry":"<ISO-8601 格式的过期日期：例如 2021-06-30T00:00:00Z>",
6    "count": 1
7  }
8}



使用 secure-upload-raw-oauth 创建令牌


在查询编辑器中输入以下 JSON：




已复制！
1{
2    "path": "tokens",
3    "method": "POST",
4    "extractors": {
5        "result": "$"
6    },
7    "pathParams": {},
8    "queryParams": {},
9    "bodyJson": {
10        "type": "blobUploadToDataset",
11        "blobUploadToDataset": {
12            "channel": "<步骤 2 中定义的 channelId>",
13            "expiry":"<ISO-8601 格式的过期日期：例如 2021-06-30T00:00:00Z>",
14            "count": 1
15        }
16    }
17}



基于 secure-upload 或 secure-upload-raw-oauth 源创建查询后，选择 Test 提交查询。响应中包含一个令牌，请务必妥善保存，因为该令牌仅显示一次。如有需要，可重新运行查询以获取新令牌。用户上传数据时不再需要此查询，除非需要创建更多令牌。可安全删除此查询。


4. 为数据上传创建查询


为允许用户提交查询，还需创建一个查询：


    

  1. 选择 secure-upload-raw 作为源。
    2. 

  2. 打开 Test 旁边的下拉菜单，选择数据提交的时机（点击按钮时或其他条件，与常规查询类似）。请求体定义了需要捕获并上传至先前创建数据集的字段。即使初始提交后也可编辑这些字段。修改前已录入的条目，在新添加的字段下将不显示任何值。
    3. 



每个字段都需映射到一个值，且值必须为字符串。可使用 jsonStringify handlebar 助手，或通过函数将值转换为字符串。


请求体示例如下：




已复制！
1{
2    "path": "blobs",
3    "method": "POST",
4    "extractors": {
5        "result": "$"
6    },
7    "headers": {
8        "authorization": "Bearer <token_from_step_3>"
9    },
10    "bodyJson": {
11        "content": {
12            // 在此处放置您所需的任何 JSON 格式内容
13            // 例如：
14            "field1": "value1",
15            "field2": {{jsonStringify w_input1.text}},
16            "field3": [
17                "item1",
18                {{jsonStringify w_input2.text}},
19                {{jsonStringify f_function1}}
20            ]
21        }
22    }
23}



运行此查询时，数据将落入一个附加了 _buffer_v2 后缀的数据集中，并与原始数据集并列存放。数据的结构将严格按照此步骤中的定义进行组织。例如，使用上述示例中的主体执行查询时，将在“content”列中生成如下数据条目（该数据集的其他列还将包含额外的元数据）：


{
  content: {
    "field1": "value1",
    "field2": "<<value of jsonStringify w_input1.text>>",
    "field3": ["item1", "<<value of jsonStringify w_input2.text>>", "<<value of jsonStringify f_function1>>"]
  }
}



停止数据上传


若要停止向数据集上传新条目，您需要同时取消发布应用程序，并删除在上一个查询中用作身份验证标头的令牌。


尽管取消发布可以在未通过身份验证的用户面前隐藏应用程序，但这些用户仍可能调用公开的安全上传数据源。因此，删除令牌对于维护应用程序的安全至关重要。


请务必将生成的令牌保存在安全的位置，因为系统不提供任何恢复选项。


要删除令牌，请按照以下步骤操作：


    

  1. 在 Slate 中创建一个查询，将源设置为 secure-upload 或 secure-upload-raw-oauth。
    2. 

  2. 打开 Test 附近的下拉菜单，勾选 manually 复选框以手动运行此查询。
    3. 

  3. 配置并运行查询：

      

    • 如果使用 secure-upload：

        

      1. 从 Available Services 中选择 Token Service。
        2. 

      2. 从 Endpoints 中选择 Delete Token。
        3. 

      3. 以字符串形式输入要删除的令牌，例如 1abcdefgha2bcdef3ghabcdefg。
        4. 

      

      • 

    • 如果使用 secure-upload-raw-oauth：

        

      1. 使用您要删除的令牌（例如 1abcdefgha2bcdef3ghabcdefg），并在查询编辑器中输入以下 JSON：
        2. 

      

```json
{
    "path": "tokens/{token}",
    "method": "DELETE",
    "extractors": {
        "result": "$"
    },
    "headers": {},
    "pathParams": {
        "token": "1abcdefgha2bcdef3ghabcdefg"
    },
    "queryParams": {}
}
```


      • 

    

    4. 

  4. 要验证令牌是否已被删除，可通过提交数据进行检查。你将收到以下 InvalidToken 响应：
