为公共应用程序上传数据

启用数据上传

通过公共应用程序上传数据的能力需要两个安全上传数据源通过查询可用：

  1. 用于生成安全通道和令牌的数据源。它包括以下之一：
    • 一个名为secure-upload的服务API数据源
    • 一个名为secure-upload-raw-oauth的HTTP JSON数据源
  2. 使用生成的安全令牌上传JSON blob的数据源。它通常命名为secure-upload-raw。

您可以通过创建一个公共应用程序并创建新查询来检查它们的可用性。如果无法获得安全上传源，请联系Palantir支持。

公共应用程序可以配置为允许用户将数据上传到Foundry。上传数据的用户不需要访问Foundry。但是，只有具有适当资源权限的Foundry用户才能访问上传的数据。

要在您的公共应用程序中设置数据上传，请按照以下说明操作：

1. 创建一个空数据集

在您选择的项目或文件夹中创建一个新的数据集资源。

2. 创建一个通道

在您的公共Slate应用程序中，根据secure-upload或secure-upload-raw-oauth源创建一个新的查询。通过在“测试（Test）”旁边的下拉菜单中勾选复选框，仅手动运行该查询。

为您的频道选择一个唯一的ID，例如slate-app-name-data-upload。channelId不能包含空格。请求体需要您之前创建的空数据集的RID。您可以从数据集的URL中获取数据集RID，例如ri.foundry.main.dataset.abcd1234-1234-5678-9000-1234abcd1234。

使用secure-upload创建频道

从可用服务中，选择频道服务，然后从端点下拉菜单中选择放置频道。

在channelId字段中，输入您的频道的唯一ID。

在请求正文中输入以下JSON：

Copied!
1{
2  "configuration": {
3    "dataset": "<your_dataset_rid_here>",
4    "maxBlobSize": <optional size limit for individual blob size (in bytes)>,
5    "options": {
6      "type": "json",
7      "json": {}
8    }
9  }
10}

使用secure-upload-raw-oauth创建频道

在查询编辑器中输入以下JSON：

Copied!
1{
2    "path": "channels/{channelId}",
3    "method": "PUT",
4    "extractors": {
5        "result": "$"
6    },
7    "pathParams": {
8        "channelId": "<your_channel_name_here>"
9    },
10    "queryParams": {},
11    "bodyJson": {
12        "configuration": {
13            "dataset": "<your_dataset_rid_here>",
14            "maxBlobSize": <optional size limit for individual blob size (in bytes)>,
15            "options": {
16                "type": "json",
17                "json": {}
18            }
19        }
20    }
21}

在基于secure-upload或secure-upload-raw-oauth源创建查询后，选择测试 . 响应将返回一个 channelId，与您在字段中输入的匹配。请保存 channelId，因为它将在下一步中使用。

此查询将不再需要用户上传数据，除非您想创建更多通道。此查询可以安全地删除。

3. 创建令牌

创建通道后，您需要创建一个令牌。该令牌用于提交安全上传查询并将数据上传到数据集。您需要指定过期日期。最大过期日期取决于您的注册的安全上传配置。根据 secure-upload 或 secure-upload-raw-oauth 源创建新查询。将查询设置为仅手动运行。

使用 secure-upload 创建令牌

从可用服务中选择 令牌服务（Token Service） ，然后从端点下拉菜单中选择创建令牌。

请求中的正文格式如下：

Copied!
1{
2  "type": "blobUploadToDataset",
3  "blobUploadToDataset": {
4    "channel": "<channelId defined in step 2>",
5    "expiry":"<expiration date in ISO-8601: e.g. 2021-06-30T00:00:00Z>",
6    "count": 1
7  }
8}

使用 secure-upload-raw-oauth 创建一个令牌

在查询编辑器中输入以下JSON：

Copied!
1{
2    "path": "tokens",
3    "method": "POST",
4    "extractors": {
5        "result": "$"
6    },
7    "pathParams": {},
8    "queryParams": {},
9    "bodyJson": {
10        "type": "blobUploadToDataset",
11        "blobUploadToDataset": {
12            "channel": "<channelId defined in step 2>",
13            "expiry":"<expiration date in ISO-8601: e.g. 2021-06-30T00:00:00Z>",
14            "count": 1
15        }
16    }
17}

在基于 secure-upload 或 secure-upload-raw-oauth 源创建查询后，选择测试以提交查询。响应中包含一个令牌，你需要保存它，因为它只会显示一次。如果需要，可以重新运行查询以获取另一个令牌。对于用户上传数据，此查询将不再需要，除非你想要创建更多令牌。该查询可以安全地删除。

4. 创建用于数据上传的查询

为了让用户提交查询，你需要创建另一个查询：

  1. 选择secure-upload-raw 作为数据源。
  2. 打开靠近测试的下拉菜单，并选择您希望数据提交的时间（在按钮点击时或根据其他条件，就像普通查询一样）。请求正文定义了要捕获并上传到之前创建的数据集中的字段。您可以在初始提交后编辑这些字段。在修改之前创建的条目在新添加的字段下将没有值。

每个字段都需要映射到一个值。这些值必须是字符串。您可以使用jsonStringify handlebar辅助函数，或者使用函数将值转换为字符串。

请求正文的示例如下：

Copied!
1{
2    "path": "blobs",
3    "method": "POST",
4    "extractors": {
5        "result": "$"
6    },
7    "headers": {
8        "authorization": "Bearer <token_from_step_3>"
9    },
10    "bodyJson": {
11        "content": {
12            // whatever json-shaped content you want here
13            // for example:
14            "field1": "value1",
15            "field2": {{jsonStringify w_input1.text}},
16            "field3": [
17                "item1",
18                {{jsonStringify w_input2.text}},
19                {{jsonStringify f_function1}}
20            ]
21        }
22    }
23}

当运行此查询时，数据将被存储在一个带有后缀_buffer_v2的数据集里，并与原始数据集一起存放。数据的结构将严格按照本步骤中定义的方式进行。例如，使用上面示例中的正文进行查询将导致“内容”列中出现以下数据条目（数据集的其他列中将包含额外的元数据）：

{
  content: {
    "field1": "value1",
    "field2": "<<value of jsonStringify w_input1.text>>",
    "field3": ["item1", "<<value of jsonStringify w_input2.text>>", "<<value of jsonStringify f_function1>>"]
  }
}

停止数据上传

要阻止新条目被上传到数据集，您需要同时 取消发布应用程序 并删除在上一个查询中用作身份验证头的令牌。

虽然取消发布会将应用程序从未认证用户中隐藏，但未认证用户仍可以调用公共安全上传数据源。删除令牌对于维护应用程序的安全性非常重要。

请记得将生成的令牌存储在一个安全的位置，因为没有恢复选项。

要删除令牌，请按照以下步骤操作：

  1. 在Slate中，使用 secure-upload 或 secure-upload-raw-oauth 作为来源创建一个查询。
  2. 打开靠近 测试 的下拉菜单，并勾选 手动 复选框以手动运行此查询。
  3. 配置并运行查询：
    • 如果使用 secure-upload：
      1. 从 可用服务（Available Services） 中选择 令牌服务（Token Service）。
      2. 从 端点（Endpoints） 中选择 删除令牌（Delete Token）。
      3. 将您要删除的令牌作为字符串输入，例如 1abcdefgha2bcdef3ghabcdefg。
    • 如果使用 secure-upload-raw-oauth：
      1. 使用您要删除的令牌，例如 1abcdefgha2bcdef3ghabcdefg，并在查询编辑器中输入以下 JSON：
      Copied!
      1{
2    "path": "tokens/{token}",
3    "method": "DELETE",
4    "extractors": {
5        "result": "$"
6    },
7    "headers": {},
8    "pathParams": {
9        "token": "1abcdefgha2bcdef3ghabcdefg"
10    },
11    "queryParams": {}
12}
  4. 要验证令牌是否已被删除，请通过提交数据进行检查。您应该会得到以下 InvalidToken 响应：
{
  "errorCode":"PERMISSION_DENIED",
  "errorName":"TokenSecurity:InvalidToken",
  "errorInstanceId":"13abec1b-3c0f-467d-ad14-6ae702b52b33",
  "parameters":{}
 }
