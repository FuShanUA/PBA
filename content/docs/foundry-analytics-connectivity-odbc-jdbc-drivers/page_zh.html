Foundry 数据集的 ODBC 和 JDBC 驱动

Foundry 数据集的 ODBC 和 JDBC 驱动提供了一个基于 SQL 的只读接口，供客户端应用程序（如 BI 工具和 ETL 工具）访问数据集。用户可以在 Foundry 中浏览项目和数据集，并执行 SQL 查询以访问表格数据。这些驱动在服务器端利用 Foundry SQL 服务器来处理和执行 SQL 查询。

选 JDBC 还是 ODBC？

如果客户端应用程序同时支持这两种协议，我们建议使用 JDBC 而非 ODBC。JDBC 驱动不仅安装和配置更简单，数据加载性能也更为出色。

Foundry SQL 服务器

这些驱动程序依赖 Foundry SQL 服务器来处理并执行针对 Foundry 数据集的 SQL 查询。请查阅 Foundry SQL 服务器架构文档，详细了解其架构设计及查询性能优化方法。

安装指南

ODBC

第 1 部分：安装 ODBC 驱动程序

请按照《下载：ODBC 驱动程序》中的说明运行 ODBC 驱动程序安装程序。

第 2 部分：配置数据源名称（DSN）

要在客户端应用程序中使用该驱动程序，请先为您的 Foundry 环境配置数据源名称（DSN）：

  1. 在开始菜单中搜索ODBC，然后打开ODBC 数据源工具。请选择 64 位版本。
  2. 在用户 DSN或系统 DSN选项卡中，点击添加...。
  3. 从驱动程序列表中选择FoundrySqlDriver。
  4. 输入以下必填参数：
    1. 数据源名称：您的计算机上该数据源的名称。
    2. 服务器：您的 Foundry 环境 URL；例如 https://<SUBDOMAIN>.palantirfoundry.com。
    3. 令牌：在 Foundry 内部的设置页面生成的安全令牌。有关如何获取令牌的说明，请参阅《用户生成的令牌》文档。如需了解其他身份验证选项，请查看《使用 OAuth 进行身份验证》指南。
  5. 点击测试...以验证连接是否成功，然后点击确定保存 DSN 配置。

有关其他配置参数以及构建连接 URL 的说明，请参阅《配置参数》。

创建 DSN 后，您就可以在支持 ODBC 数据源的客户端应用程序中引用它。有关 ODBC 数据源的详细信息，请参阅客户端应用程序文档。以下是与 Foundry 配合使用的常见应用程序设置指南：

JDBC

第 1 部分：安装 JDBC 驱动程序

请前往《下载：JDBC 驱动程序》页面下载 JDBC 驱动程序（.jar 文件）。下载完成后，请将该文件放置在客户端应用程序文档中指定的、用于配置 JDBC 连接的相应位置。

第 2 部分：构建 JDBC 连接字符串

JDBC 连接字符串的格式如下：

jdbc:foundrysql://<FOUNDRY_HOSTNAME>?Password=<TOKEN>
  • FOUNDRY_HOSTNAME 是你的 Foundry 环境的主机名（例如 subdomain.palantirfoundry.com）。
  • TOKEN 是从 Foundry 内部的设置页面生成的安全令牌。有关如何获取令牌的说明，请参阅《用户生成的令牌》文档。如需了解其他身份验证选项，请查阅《使用 OAuth 进行身份验证》指南。

可以通过在连接字符串后追加 &OptionalParam=<VALUE> 来指定可选参数。有关可用参数的完整列表，请参阅《配置参数》。

如果 JDBC 客户端要求显式指定驱动程序类，请指定 com.palantir.foundry.sql.jdbc.FoundryJdbcDriver。

（可选）第 3 部分：执行 SQL 查询

使用指南

使用 SQL 查询 Foundry 数据集

在 SQL 查询中，可以通过路径或 RID 引用数据集。SQL 语法取决于为连接设置的方言（参见配置参数）。

SPARK 方言

-- 基本 SELECT
SELECT * FROM `/Path/To/Dataset`

-- 使用 WHERE 子句过滤
SELECT * FROM `/Path/To/Dataset`
WHERE years < 13 AND category = 'Z';

-- 使用 JOIN
SELECT *
FROM `/Path/To/Dataset_A` a
JOIN `/Path/To/Dataset_B` b
    ON a.id = b.fk_id;

ODBC 与 ANSI 方言

-- 基本 SELECT 语句
SELECT * FROM "/Path/To/Dataset";

-- 使用 WHERE 子句进行过滤
SELECT * FROM "/Path/To/Dataset"
WHERE years < 13 AND category = 'Z';

-- 使用 JOIN
SELECT *
FROM "/Path/To/Dataset_A" a
JOIN "/Path/To/Dataset_B" b
    ON a.id = b.fk_id;

有关使用数据集标识符的详细说明，请参阅《指南：识别数据集的 RID 或文件路径》。

使用 OAuth 进行身份验证

ODBC 和 JDBC 驱动程序支持 OAuth 2.0 流程，提供了更多的身份验证选项，无需手动生成绑定到单个 Foundry 账户的身份验证令牌：

  • 个人用户：驱动程序会在浏览器中打开登录提示，供您向 Foundry 进行身份验证。
  • 服务用户：驱动程序以服务用户身份连接到 Foundry，该服务用户与在 Foundry 中注册的第三方应用程序相关联。
  • 外部应用程序（面向开发者）：第三方应用程序与 Foundry 的 OAuth 系统集成，代表其用户获取令牌。

在条件允许的情况下，我们建议优先使用这些基于 OAuth 的选项，而非生成令牌。因为这些方式安全性更高，并且允许共享内嵌的连接字符串，无需共享个人用户的令牌。

个人用户（仅限 Windows 上的 ODBC）

在 Windows 上运行时，ODBC 驱动程序支持自动的 OAuth 登录和授权流程。此流程仅支持安装在您计算机上的桌面应用程序，不适用于通过 Web 浏览器访问的应用程序。

请按照以下步骤设置此流程：

  1. （由 Foundry 管理员完成）在 Foundry 中注册第三方应用程序，并指定以下配置选项：
    1. 客户端类型：在大多数情况下，建议选择公共客户端；如果您的使用场景有需求，也支持机密客户端。
    2. 授权许可类型：启用授权码许可，并将重定向 URL 设置为 http://127.0.0.1/foundrydriver/oauthredirect。
    3. 确保该应用程序已注册并启用。
    4. 从应用程序详情页面复制该应用程序的客户端 ID，并与任何要在其计算机上设置 ODBC 连接的用户共享。
  2. 从 Foundry 管理员处获取客户端 ID后，请设置以下 ODBC 连接参数：
    1. AuthMethod = OauthFlow
    2. OauthClientId = <YOUR_CLIENT_ID>
      或者，也可以将客户端 ID填入应用程序的用户名字段中，替代 OauthClientId，以便在应用内完成凭据配置。但仍需设置 AuthMethod。
  3. （可选）如果在 Windows ODBC 数据源管理器中配置这些设置，可以选择测试选项来触发登录提示，验证登录是否正常。

下次在客户端应用程序中使用该驱动时，系统会提示你在浏览器中登录 Foundry。此后，无需每次使用驱动时都进行登录，但偶尔可能会再次提示你登录。

服务用户

对于不与特定用户绑定的操作流程（例如按计划刷新仪表板数据），我们建议使用基于 OAuth 的服务用户，并借助采用 OAuth 客户端凭据授权类型的第三方应用来实现。驱动程序将使用长效的客户端 ID/密钥对向 Foundry 进行身份验证，这种方式比手动创建服务账户并生成令牌更易于管理。

请按照以下步骤为服务用户配置 OAuth：

  1. （由 Foundry 管理员完成。） 在 Foundry 中注册第三方应用：
    1. 客户端类型：选择机密客户端。
    2. 授权许可类型：启用客户端凭据授权，并务必将客户端 ID和客户端密钥对保存在安全位置。
    3. 确保应用已注册且启用。
    4. 确保为生成的服务用户授予了通过 ODBC 或 JDBC 驱动访问所需数据集的权限。为该应用生成的服务用户显示在客户端凭据授权详情面板中。
  2. 配置驱动时，请设置以下配置参数：
    1. AuthMethod = ClientCredentials
    2. OauthClientId = <YOUR_CLIENT_ID>
    3. OauthClientSecret = <YOUR_CLIENT_SECRET>
      或者，也可以将客户端 ID和客户端密钥填入应用程序的用户名和密码字段中，替代 OauthClientId 和 OauthClientSecret，以便在应用内完成凭据配置。但仍需设置 AuthMethod。

现在，驱动程序将以该 OAuth 应用的服务用户身份连接到 Foundry。

外部应用（面向开发者）

应用开发者可将 ODBC 和 JDBC 驱动集成到应用中，由该应用运行自身的 OAuth 客户端来管理与第三方应用之间的 OAuth 登录流程。此选项支持对登录流程的完全掌控，包括将用户重定向至 Foundry 进行身份验证，以及处理用户完成认证和授权后的授权响应。请参阅《为 Foundry 编写 OAuth2 客户端》文档，了解如何代表 Foundry 用户获取访问令牌。

一旦应用代表用户获取了访问令牌，即可通过标准的密码属性将其传递给驱动程序：

  • ODBC Pwd / JDBC Password = <从令牌端点获取的访问令牌>

在 ODBC 驱动中启用日志记录

若需启用驱动程序日志记录以排查问题，请按照以下步骤操作。请注意，这些步骤可能需要管理员权限。

  1. 创建一个用于保存日志的文件夹，例如 My Documents\Foundry Driver logs。
  2. 打开 Windows 的 ODBC 数据源 工具。可以在 Windows 搜索栏中搜索“ODBC”来找到该工具。请选择 64 位版本。
  3. 打开 系统 DSN 选项卡。选择 FoundrySql 数据源，然后选择 配置。请注意，选择哪个 Foundry 数据源并不影响结果，因为日志记录设置将应用于所有使用 Foundry 驱动的数据源。
  4. 在配置窗口中选择 日志记录选项。
  5. 将日志级别设置为 DEBUG。将日志路径设置为您先前创建的文件夹。
  6. 选择 确定 以保存设置。

重启客户端应用程序并执行需要排查的操作。日志应当会出现在您所选的文件夹中。如果需要 Palantir 或其他支持团队提供协助，可以将该文件夹压缩为 zip 文件进行共享。

排查完成后，请返回 ODBC 数据源 工具，将日志级别设置为 OFF 以禁用日志记录。建议执行此步骤以提升性能。

在 JDBC 驱动中启用日志记录

JDBC 驱动会自动发现 Java 应用程序在 classpath 上提供的任何 SLF4J 记录器。具体而言，应用程序需提供 org.slf4j.impl.StaticLoggerBinder 和 org.slf4j.impl.StaticMDCBinder 类的实现。通过将 slf4j-simple（1.X 版本）添加为项目依赖，即可使用默认实现。

若未配置 SLF4J 记录器，驱动首次加载时将输出以下提示信息：

SLF4J: Failed to load class "org.slf4j.impl.StaticLoggerBinder".
SLF4J: Defaulting to no-operation (NOP) logger implementation
SLF4J: See http://www.slf4j.org/codes.html#StaticLoggerBinder for further details.
SLF4J: Failed to load class "org.slf4j.impl.StaticMDCBinder".
SLF4J: Defaulting to no-operation MDCAdapter implementation.
SLF4J: See http://www.slf4j.org/codes.html#no_static_mdc_binder for further details.

使用 foundry.ini 配置文件配置 JDBC 驱动

除了使用连接字符串外，还可以通过配置文件来配置 JDBC 驱动。为此，需在 JDBC .jar 文件所在目录下创建一个名为 foundry.ini 的文件。

该 .ini 文件分为两个部分：low-priority 和 high-priority。在 low-priority 部分指定的属性，其优先级低于连接字符串中的属性。这意味着，若同一属性同时在 low-priority 部分和连接字符串中指定，将以连接字符串中的值为准。相反，high-priority 部分指定的属性优先级高于连接字符串中的属性。这在将报表从开发机发布到服务器的场景中非常实用，可确保服务器上的属性配置优先于开发机上的配置。

foundry.ini 文件示例：

[high-priority]
proxyHost=myproxy.abc
proxyPort=1234

[low-priority]
branch=production-branch

在 Java 16 及以上版本中使用 JDBC 驱动

JDBC 驱动默认支持的最高 Java 版本为 Java 15。若要在 Java 16 及以上版本中使用该驱动，需在应用程序中将 --add-opens Java 运行时选项设置为 java.base/java.nio=org.apache.arrow.memory.core,ALL-UNNAMED。
