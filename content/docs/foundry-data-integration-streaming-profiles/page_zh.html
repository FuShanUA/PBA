流式处理配置

使用 Palantir 平台进行流式处理时，系统会自动为您管理 Flink 运行时环境的各项配置，包括为 Flink 作业配置提供合理且高性价比的默认值。在大多数情况下，您无需手动配置任何参数，即可在 Flink 计算引擎上高效运行作业。

然而，有时您的作业所需的资源会超出 Foundry 流式处理默认提供的额度。这类资源可能包括额外的并行度（例如，当数据流吞吐量极高时），或者在状态数据或单条记录非常大的情况下，JobManagers 或 TaskManagers 所需的额外资源。

针对作业需要额外资源的情况，Palantir 提供了一系列作业配置文件，可用于调整特定的配置项。需要注意的是，Palantir 流式处理平台已为您代管了大部分配置，因此我们仅开放了部分 Flink 配置供您使用。

与《代码仓库中的 Spark 配置》类似，每个流式处理作业配置文件都专门用于管理流式作业资源需求的特定部分，并且可以将多个流式处理作业配置文件组合使用，以满足您的实际业务需求。

何时使用流式处理配置

通常情况下，您完全无需在平台中使用流式处理配置，因为我们的默认流式配置已经针对绝大多数应用场景进行了性能和成本优化。但是，如果您的流式作业遇到以下特定问题，我们建议您使用流式处理配置：

  • 极高的吞吐量
  • 极大的记录数据
  • 极大的状态需求

您可以结合作业需求、JobTracker 界面中提供的日志与错误信息，以及对 Flink 的基本了解，来判断需要添加哪些额外的配置文件。

管理流式处理配置

由于流式处理配置可用于决定分配给长期运行的流式作业的资源总量，因此平台对这些配置实施了一些管理控制措施，以便管理流式处理成本。

项目引用

在 Palantir 平台中，项目（Projects）界定了相关工作的概念边界，同时也是应用和管理访问权限的安全边界。安全与管理控制的默认应用层级位于项目级别。通常，使用数据或资源的前提是它们必须存在于您当前的项目中，或已被导入到当前项目，从而获得相应的《项目引用》。您可以在项目工作区侧边栏的引用部分查看项目中的所有引用。

对流式处理配置实施的主要管理控制措施是：必须将其作为项目引用，添加到与相关流式处理管道或应用程序相同的项目中。如果您尝试使用未导入到项目中的流式处理配置，作业将会失败，并提示缺少该必要项的错误信息。

通过将 Project 引用作为管理控制手段，可以授权高级用户代表其他用户将配置文件导入 Project。这样，管理员能够细粒度地控制流式配置文件的使用位置，同时允许偏向操作层面的用户使用已获授权的配置文件。

将配置文件导入为 Project 引用

通常，系统会利用配置文件选择器组件，将配置文件自动导入相关应用内的 Project 中。例如，Pipeline Builder 中的配置文件选择器就会自动导入选中的配置文件。

只要用户拥有足够的资源导入权限，所有用户都能将大多数配置文件作为 Project 引用导入各自的 Project。这通常意味着用户已被分配了某项角色，且该角色在对应 Project 上授予了 compass:import-resource-to 权限。你可以进入用户设置中的角色选项卡查看角色配置。在页面上使用 Filter operations... 搜索工具即可查找该权限。

要查看所有可用的流式配置文件列表，请访问控制面板中“注册设置”部分的“流式配置文件”选项卡。在这里，你可以选取任意具体的配置文件，查看已将该配置文件添加为 Project 引用的所有 Project（前提是你具备访问权限），并将其导入新的 Project。你也可以从已导入配置文件的某个 Project 中移除对该配置文件的引用；但请注意，这会导致该 Project 中所有依赖该配置文件的流式作业中断，因为系统要求配置文件必须与流式作业位于同一个 Project 内才能正常使用。

使用大型流式配置文件

对于授予大量资源的配置文件，必须通过控制面板中“注册设置”部分的流式配置文件选项卡来创建 Project 引用。此设置仅对被指定为 Enrollment Resource Administrators 的用户开放。可以在“注册权限”选项卡中指派这些管理员，他们能将受限配置文件导入其有权访问的任何 Project。

管理员将配置文件导入某个 Project 后，任何有权访问该 Project 的用户都可以使用该配置文件。
