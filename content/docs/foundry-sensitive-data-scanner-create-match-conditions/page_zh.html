创建匹配条件

敏感数据扫描器附带一组 内置匹配条件；您也可以定义自己的 自定义匹配条件 用于敏感数据扫描器。

内置匹配条件

敏感数据扫描器提供了一系列 内置的 匹配条件，用于检测常见的个人身份信息（PII）类型，例如社会安全号码、电子邮件地址或电话号码。您可以通过选择箭头展开右侧边栏中的 内置匹配条件 部分来找到这些条件：

内置匹配条件旨在检测常见的个人数据类型。然而，它们的有效性会因您的具体数据结构和格式而异。请确保这些条件符合您的独特数据标准，如有需要，可以创建自定义条件。如果您有进一步的问题，请咨询您的数据保护官员。

创建自定义匹配条件

要为您的 空间 创建一个 自定义 匹配条件，有以下两种方式可以开始：

  • 从敏感数据扫描仪的首页。
  • 在创建敏感数据扫描时。

从首页上，选择 添加，在匹配条件侧边栏中列出的 自定义匹配条件 上方。

在创建敏感数据扫描时，在 选择匹配条件 页面上，您也可以创建一个新的匹配条件并立即在扫描中使用它，通过选择 创建新匹配条件。

这两种起始点都会打开相同的匹配条件创建过程。从那里，您可以选择是创建一个 正则表达式 (Regular expression) 匹配条件，还是一个 重叠 (Value overlap) 匹配条件。

创建正则匹配条件

在创建正则表达式（regex）匹配条件时，您可以指定两种类型的正则表达式选项；内容正则表达式和列名正则表达式。

  • 内容正则表达式: Sensitive Data Scanner 将用于检查数据集内容的正则表达式（不是数据集的列名）。
  • 列名正则表达式: Sensitive Data Scanner 将用于检查数据集列名的正则表达式（不是内容本身）。

Sensitive Data Scanner 允许您结合这两种正则表达式选项以达到最大精确度：

  • 如果内容正则表达式匹配，则突出显示数据集。
  • 如果列名正则表达式匹配，则突出显示数据集。
  • 如果 同时 内容和列名正则表达式都匹配，则突出显示数据集。
  • 如果 任一 内容或列名正则表达式匹配，则突出显示数据集。

The content regex contains an optional content threshold field in which you can specify a number greater than 0 and less than or equal to 100; this content threshold is the percentage of the cells in a particular column of a given dataset that must match the content regex in order for that dataset to be highlighted as a match. The content threshold field is optional. If no value is specified, Sensitive Data Scanner will highlight a dataset as a match if there is at least one match of the content regex.

创建重叠匹配条件

重叠匹配条件在寻找无法用正则表达式表示的敏感数据时非常有用。例如，很难创建一个内容正则表达式来匹配姓名，但在某些情况下，创建一个列名正则表达式就足够了。然而，如果你已经有一个包含详尽敏感数据列表的数据集，并且想要扫描这些数据，那么重叠匹配条件将非常有用。
