概述

召回 是你向 Apollo 通知某个或多个版本存在有问题的方式。你可以使用召回将实体从有问题的版本中移除，并防止 Apollo 将模块或实体升级到这些版本。

版本可以通过 手动 方式进行召回，或者由于 在发布管道中测量到的不稳定性 自动召回，也可以通过外部服务（如容器漏洞扫描）的 API 进行召回。

在一个或多个版本被召回后，Apollo 会通过阻止该版本进一步发布，并根据召回的 下线策略，优先将其他未被召回的版本部署到受影响的实体来自动修复问题。

Apollo 还可以配置为直接将实体升级到一个因 CVE 被召回的版本，这样可以提高其安全态势。参见 将实体升级到因 CVE 被召回的版本。

你可以在产品的 Recalls 选项卡中查看召回列表及其下线状态，以及已完成和已撤销的召回。

您可以通过导航到 版本 标签页，并将鼠标悬停在该版本的 状态 列上来查看影响特定版本的召回信息。

当一个版本受到活跃召回影响时，其主页上会显示一个横幅。选择 详细信息 将打开有关每个活跃召回的详细信息，例如召回范围、谁发起了召回以及召回的原因。

您也可以通过从表格中选择特定版本，然后从版本主页上的横幅中选择 详细信息 来查看该版本的召回信息。

我们建议在发现问题后尽快召回版本，以防止问题被部署到更多的环境。一旦对问题进行了调查，您可以 编辑 或 撤销 召回操作。
