Amazon DynamoDB

Amazon DynamoDB 连接器是用于 Amazon DynamoDB 的 Palantir 提供的驱动程序。

要创建新的 Amazon DynamoDB 源，请遵循 Palantir 提供的驱动程序的标准设置流程，然后使用以下章节进行 Amazon DynamoDB 特定的配置和网络设置。有关完整的属性参考，请参阅 Amazon DynamoDB 官方驱动程序文档 ↗。

配置

以下属性为必填或建议填写。

属性是否必填？描述默认值
AuthScheme ↗必填指定连接到 Amazon DynamoDB 时使用的身份验证类型。如果此属性留空，则使用默认身份验证。AwsRootKeys
AWSAccessKey ↗建议指定您的 AWS 账户访问密钥。此值可在您的 AWS 安全凭证页面获取。—
AWSRegion ↗建议您的 Amazon Web Services 的托管区域。NORTHERNVIRGINIA
AWSSecretKey ↗建议您的 AWS 账户秘密密钥。此值可在您的 AWS 安全凭证页面获取。—
Domain ↗建议指定您的 AWS 域名。如果您的组织已将自定义域名关联到 AWS，请使用此属性设置该域名。amazonaws.com

网络配置

下表列出了为成功运行而源必须能够访问的域名。

针对每个域名，添加相应的出站策略。如果源托管在本地且无法从 Foundry 直接访问，请改用代理出站策略；代理主机本身也必须能够访问所列出的域名。有关详细信息，请参阅使用代理作为代理。

域名是否需要
dynamodb.<AWSRegion>.<domain>始终需要。AWSRegion 映射
sts.<Region>.amazonaws.<TLD>如果 AuthScheme=AwsIAMRoles,AwsMFA,TemporaryCredentials
cognito-idp.<AWSCognitoRegion>.amazonaws.<TLD>如果 AuthScheme=AwsCognitoBasic,AwsCognitoSrp
cognito-identity.<AWSCognitoRegion>.amazonaws.<TLD>如果 AuthScheme=AwsCognitoBasic,AwsCognitoSrp
<SSOLoginURL>如果 AuthScheme=Okta,ADFS,PingFederate, 使用 SSOLoginURL 属性
<Resource>如果 AuthScheme=AzureAD, 在 SSOProperties 中设置 Resource
<SSOExchangeURL>如果 AuthScheme=Okta
