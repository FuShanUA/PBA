标量类型

标量类型表示单个值，通常用于存储文本、数值或时间数据。

在 JavaScript 和 TypeScript 中，有一种 number 类型常用于表示整数和浮点数值。为了提供进一步的类型验证和结构，我们仅支持从 @foundry/functions-api 包导出的数值别名用于 TypeScript v1 函数，以及 @osdk/functions 包导出的数值别名用于 TypeScript v2 函数。同样地，当在 Python 函数中使用数值类型时，我们建议使用从 functions.api 模块导出的类型别名。

强制标记

标记是强制控制措施，通过要求用户必须具有特定标记才能访问数据来限制访问。

Copied!
1import { OntologyEditFunction, MandatoryMarking } from "@foundry/functions-api";
2import { Employee, Objects } from "@foundry/ontology-api";
3
4export class MyFunctions {
5    @Edits(Employee)
6    @OntologyEditFunction()
7    public async editMandatoryMarkings(markings: MandatoryMarking[]): Promise<void> {
8        const employeeOne = Objects.search().employee().filter(e => e.id.exactMatch(1)).all()[0];
9        employeeOne.markingsProperty = markings;
10    }
11}
Copied!

  • TypeScript 函数中，可选参数声明为 varName?: T 或 varName: T | undefined。例如，一个具有名为 value 的可选整数参数的函数可以声明为 value?: Integer 或 value: Integer | undefined。TypeScript 函数还可以通过指定 T | undefined 类型来声明可选返回类型。例如，一个可能返回 Integer 或无值的函数将具有 Integer | undefined 返回类型。

结构体/自定义类型

自定义类型由其他受支持的类型（包括其他自定义类型）组成，可以在函数签名中使用。

  • 在TypeScript函数中，自定义类型是通过使用interface关键字定义的用户自定义TypeScript接口。

    • 可以通过?可选标记或与undefined的联合来支持可选字段。

  • 在Python函数中，自定义类型是用户定义的Python类。

    • 要成为有效的自定义类型，该类必须遵循以下要求：
      • 该类的所有字段都必须有类型注释。
      • 字段类型必须是支持的类型；可以使用原始API类型或原生Python类型（如上表所示）。
      • __init__ 方法只能接受与字段名称和类型注释相同的命名参数。
    • 可以使用 dataclasses.dataclass ↗ 装饰器来自动生成一个符合这些要求的 __init__ 方法。

聚合类型

聚合类型可以从函数中返回，供平台的其他部分使用，例如 Workshop 中的图表。

有两种支持的聚合类型：

  • 一个 二维聚合 从单个桶键映射到数值。例如，这可以用于表示如特定职位的员工数量等聚合。
  • 一个 三维聚合 从两个桶键映射到数值。这可以用于表示如按每个员工的职位和办公地点统计的员工数量等聚合。

聚合可以使用多种类型进行键控：

  • 布尔型 桶代表值为 true 或 false 的情况。
  • 字符串 桶可用于表示分类值。
  • 范围 桶表示桶键是一个值范围的聚合。这些可以用于表示图表中的直方图或日期轴。
    • 数值范围，包括 整数（整数） 和 双精度浮点数（双精度浮点） , 表示对数值的分桶聚合。
    • 日期和时间范围，包括在 Date 和 Timestamp 上，表示对日期范围的分桶聚合。

对象集

有两种方法可以将一组对象传入和传出函数：具体的对象集合（如数组）或对象集。

将对象数组传递给函数，可以在不预先将对象加载到函数执行环境的情况下，对具体的对象列表执行逻辑。然而，对象集允许您执行过滤、搜索和聚合操作，并且仅在请求时才加载最终结果。

我们建议使用对象集而不是数组，因为对象集通常能实现更好的性能，并且可以传递超过10,000个对象到函数中。

下面的示例展示了如何在不将对象加载到内存中的情况下对对象集进行过滤，使您可以将过滤后的对象集返回到应用程序的其他部分。

1import { Function } from "@foundry/functions-api";

4export class MyFunctions {
5    @Function()
6    public filterAircraft(aircraft: ObjectSet<Airplane>): ObjectSet<Airplane> {
7        return aircraft.filter(a => a.capacity.range().gt(200));
8    }
9}

9

通知

通知类型可以从函数中返回，以灵活配置应在平台上发送的通知。例如，您可以编写一个函数，该函数接受诸如 User 和对象类型等参数，并返回一个配置了消息的 Notification。

  • 一个 Notification 包含两个字段：一个 ShortNotification 和 EmailNotificationContent。
  • 一个 ShortNotification 代表通知的摘要版本，将在 Foundry 平台内显示。它包括一个简短的 heading、content 和一组 Link。
  • EmailNotificationContent 代表通知的丰富版本，可以通过电子邮件外部发送。它包括一个 subject，一个由无头 HTML 组成的 body，以及一组 Link。
  • 一个 Link 具有一个面向用户的 label 和一个 linkTarget。LinkTarget 可以是 URL、一个 OntologyObject 或者 Foundry 内任何资源的 rid。

有关如何使用 Notifications API 的示例，请查看 我们的指南。

Copied!

15                .build())
16            .emailNotificationContent(EmailNotificationContent.builder()
17                .subject("New issue")
18                .body("hello")

21    }
22}

媒体类型

媒体

函数可以接受并返回媒体项。在 TypeScript v1 中，使用 MediaItem 类型。在 TypeScript v2 和 Python 中，使用 Media 作为输入和输出类型。调用者可以传递一个现有的 MediaReference，例如对象上的媒体属性，传入函数中。下游消费者可以使用返回值来获取内容、获取元数据或将其附加到另一个对象上。有关更多信息，请查看 媒体 指南。

Copied!
1import { Function, MediaItem } from "@foundry/functions-api";
2
3export class MyFunctions {
4    @Function()
5    public async echoMedia(media: MediaItem): Promise<string | undefined> {
6        const mimeType: string = media.mimeType;
7        // Fetch type-specific metadata (page count, dimensions, duration, and so on)
8        const metadata = await media.getMetadataAsync();
9        // Read the binary contents as a Blob
10        const contents: Blob = await media.readAsync();
11        // Narrow to a specialized subtype to call type-specific methods
12        if (MediaItem.isDocument(media)) {
13            const text = await media.extractTextAsync({ startPage: 0, endPage: 1 });
14        } else if (MediaItem.isAudio(media)) {
15            const transcript = await media.transcribeAsync();
16        }
17        return undefined;
18    }
19}
Copied!
1import type { Media } from "@osdk/client";
2
3export default async function echoMedia(media: Media): Promise<Media> {
4    // Get the underlying MediaReference
5    const mediaReference = media.getMediaReference();
6    // Fetch slim metadata: path, sizeBytes, mediaType
7    const metadata = await media.fetchMetadata();
8    // Fetch contents as a Response; call .blob() or .arrayBuffer() for bytes
9    const response = await media.fetchContents();
10    const contents = await response.blob();
11    return media;
12}

用户、组和主体

主体（Principal）代表一个 Foundry 用户账户或组。这些类型可以传递给函数，以允许访问与用户或组相关的信息，例如组的名称或用户的名、姓或电子邮件地址。所有主体类型都从 @foundry/functions-api 包中导出。

  • User 始终具有用户名，并且可能具有 firstName、lastName 或 email。它还包括与 Principal 相关的所有字段。
  • Group 具有名称。它还包括与 Principal 相关的所有字段。
  • Principal 可以是 User 或 Group。您可以检查 type 字段来确定给定的 Principal 是 User 还是 Group。除了 User 和 Group 特定的字段外，Principal 还具有 id、realm 和 attributes 的字典。

几何类型

几何类型表示函数中的空间数据和地理形状。支持两种几何类型：

  • GeoPoint（地理点位）表示一个带有纬度和经度坐标的单个地理点。
  • GeoShape（地理形状）表示任何有效的GeoJSON几何图形，包括点、多边形、线串和其他形状。

这些类型遵循GeoJSON规范 ↗，可用于空间操作、地图绘制和地理分析。请注意，根据GeoJSON规范，位置参数的顺序是经度、纬度。
