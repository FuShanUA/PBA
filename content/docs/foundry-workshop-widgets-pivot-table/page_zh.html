透视表

透视表 小部件可以动态地对对象数据进行分组和聚合，然后以表格形式显示这些聚合后的数据。配置透视表小部件的模块构建者可以使用以下功能：

  • 按一个或多个属性类型对数据进行行级分组。
  • 最多按一个属性类型对数据进行列级分组。
  • 对日期和时间戳属性类型按日期、周、月和年进行动态分组。
  • 根据行级和列级分组进行排序。
  • 通过计数、基数、平均值、最大值、最小值和总和进行聚合。
  • 单元格、行和列级别的选择，以便在所选分组桶上进行下游过滤。

配置选项

在配置透视表时，构建者可以从对象或函数输出中推导数据。

基于对象的透视表

下面的示例显示了配置之前的基于对象的透视表的初始状态。小部件的配置面板显示初始输入 基础对象集 设置为 Flight Alert: All。

透视表小部件具有以下核心配置选项：

  • 基础对象集： 此参数决定了传递到透视表的对象数据，并接受一个对象集变量作为输入。请注意，透视表仅支持单一对象类型的对象集变量。
  • 行分组： 以下选项允许添加一个或多个行级别的分组。
    • 添加行分组： 添加按所选属性类型进行行分组。
    • 列宽： 在每个行分组中，调整该分组属性类型的默认列宽。仪表板查看者还可以通过拖动列边界在运行时调整行分组列的宽度。
    • 时间间隔： 对于日期或时间戳属性类型，在每个行分组中配置分组的时间间隔（如精确日期/时间、周、月）。
    • 显示总计： 在表格底部添加一个总计行分组。当使用多个聚合来计算列值时，总计值是将相同的聚合应用于该属性原始值总和的结果。
      • 免责声明： 在Total行中出现的值将是针对所有对象的原始值进行多步骤聚合后的结果，在每次聚合之前。
    • 排序行： 启用对配置的行分组属性中的一个或多个进行排序。
  • 列分组： 以下选项允许添加最多一个列级分组。
    • 选择一个属性： 按所选属性类型添加列分组。
    • 时间间隔： 在日期或时间戳属性类型的每个列分组中，配置分组的时间间隔（例如精确日期/时间、周、月）。
    • 显示总计？： 在表格的右侧添加一个总计列分组。
    • 排序值： 切换按升序或降序对列分组值进行排序。
  • 聚合： 以下选项控制表格单元格中显示的聚合。
    • 添加聚合： 允许在属性类型或整体对象计数上添加新的聚合。
    • 聚合位置： 切换聚合是放置在列还是行上。默认行为是将聚合放在列上。
    • 聚合标题： 点击聚合的标题允许编辑该标题。每个指标选择的标题将在表格中作为列标题显示。
    • 聚合指标： 控制给定聚合的计算方式。选项包括平均值、最大值、最小值、总和、计数或基数。
    • 列宽： 可调整给定聚合的列宽。
  • 选定的筛选条件： 此输出对象集筛选变量捕获用户选定单元格的分组标准，并可用于筛选下游小部件和对象集变量。用户可以选择单个单元格、一组单元格或整个行或列。
  • 在选择时： 允许当用户在表格中选择内容时触发Workshop 事件（例如，在当前模块内打开抽屉）。有关更多详细信息，请参阅 Workshop 事件文档。

基于函数的透视表

基于函数的透视表的数据来源于 函数 的输出。

这种方法适用于以下使用场景：

  • 在显示数据之前对数据进行转换或应用自定义聚合。
  • 结合来自多个来源的数据。
  • 将复杂的业务逻辑应用于数据。

前提条件

  • 您的函数必须输出一个包含 结构体（struct） 的数组。
  • 每个结构体必须包含一个名为 values 的字段，该字段保存透视表的值。

基本结构

以下是可用于函数支持的透视表的TypeScript接口示例。

在此接口中：

  • region、productType、productName 和 year 是用于分组的字段。
  • totalSales 和 estimatedSales 是在透视表单元格中显示的值。

配置

在下拉菜单中选择一个函数后，构建者可以进行以下选择：

  1. 分组字段（Group-by fields）: 这些决定了数据在行和列中的组织方式。
  2. 值字段： 这些决定了单元格中显示哪些指标。
  3. 可展开行： 可以展开以显示更详细数据的字段。

配置完成后，透视表将根据您的函数返回的数据进行渲染：

总计

基于函数的透视表支持显示总计。要显示总计，请在您的列表中返回一个符合以下指南的结构体。对于下面的示例，假设region和productType是行分组字段，year是列分组字段。

行总计： 要表示所有行的总和（行总计），在数据点中省略行分组字段。

示例： 代表2021年总计的数据点：

Copied!
1{
2    year: "2021";
3    values: {
4        totalSales: 622000;
5        estimatedSales: 57000;
6    }
7};

列总计： 要表示所有列的总和（列总和），在数据点中省略列分组字段。

示例：一个表示EU和Clothing总和的数据点：

Copied!
1{
2    region: "EU";
3    productType: "Clothing";
4    values: {
5        totalSales: 57000;
6        estimatedSales: 57000;
7    }
8}

总计：要表示总计，省略所有分组字段。

Copied!
1{
2    values: {
3        totalSales: 3147000;
4        estimatedSales: 3160000;
5    }
6}

空桶

空桶对于表示缺失或未定义的数据很有用。

要创建一个空桶：

  1. 为桶的值返回undefined。
  2. 确保你的接口支持未定义的字段。

下面是示例：

Copied!
1interface SiteData {
2    site?: string;
3    part?: string | undefined;  // Note the explicit undefined type
4    values: {
5        quantity: Double;
6    }
7}
Copied!
1{
2    "part": undefined,
3    "source": "SourceA",
4    "values": {
5        "quantity": 100
6    }
7}

省略一个字段与传递undefined是不同的。省略一个字段会创建一个总计，而undefined会创建一个空桶。

选择

一个基于函数的透视表的输出选择可以写入到 结构变量 中 . 结构字段来源于函数的输出。

显示与样式

透视表小部件具有以下显示和样式选项：

  • 布局：根据用户的偏好配置透视表的视图。
    • 表格：默认的透视表视图。
    • 堆叠：通过将所有行分组合并到一个列中，提供更紧凑的视图。
      • 自定义堆叠分组标签：启用以重命名分组列。
  • 填充宽度：启用后，透视表会扩展以填满其父容器内的可用空间，并在容器尺寸变化时自动调整。
  • 表格样式：为透视表单元格和边框样式提供三种选项。
    • 基本： 默认的透视表样式。
    • 轮廓：在每个顶级行分组的上方和下方添加较深的轮廓。
    • 条纹行：在每个顶级行分组的上方和下方添加较深的轮廓，并且在表格中的每个交替行上添加浅灰色背景。
  • 颜色：自定义透视表标题的颜色。如果选择了基础表格样式，所选颜色的较浅色调将应用于行分组列的背景。
    • 简洁：在透视表标题中显示所选颜色的较浅色调。
    • 突出：在透视表标题中直接显示所选颜色。
