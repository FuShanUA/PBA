数据透视表

数据透视表微件支持对对象数据进行动态分组和聚合，并以表格形式展示聚合后的数据。配置数据透视表微件的模块构建者可使用以下功能：

  • 按一个或多个属性类型对数据进行行级分组。
  • 按至多一个属性类型对数据进行列级分组。
  • 按日期、周、月和年对日期及时间戳属性类型进行动态分组。
  • 按行级和列级分组进行排序。
  • 按计数、基数、平均值、最大值、最小值和求和进行聚合。
  • 支持单元格、行和列级选择，以便对选定的分组桶进行下游过滤。

配置选项

在配置数据透视表时，构建者可以从对象或函数输出中获取数据。

基于对象的数据透视表

下方的示例展示了一个基于对象的数据透视表在配置前的初始状态。微件的配置面板显示，初始输入的 Base object set 被设置为 Flight Alert: All。

数据透视表微件提供以下核心配置选项：

  • Base object set: 此参数决定了传入数据透视表的对象数据，并接受一个对象集变量作为输入。请注意，数据透视表仅支持单一对象类型的对象集变量。
  • Row grouping(s): 以下选项允许添加一个或多个行级分组。
    • Add row grouping: 按所选属性类型添加行分组。
    • Column width: 在每个行分组内，调整该分组属性类型的默认列宽。看板查看者也可以在运行时通过拖动列边框来调整行分组列的宽度。
    • 时间间隔：在针对日期或时间戳属性类型的每个行分组内，配置分桶的时间间隔（如精确的日期/时间、周、月）。
    • 显示总计：在表格底部添加一个总计行分组。当使用多种聚合方式计算列值时，总计值是对属性的原始值求和后，再应用相同聚合方式得出的结果。
      • 免责声明：在Total行中显示的值，是在每次聚合之前对所有对象的原始值执行多步聚合的结果。
    • 行排序：支持对一个或多个已配置的行分组属性进行排序。
  • 列分组：以下选项支持添加最多一个列级别的分组。
    • 选择属性：按所选属性类型添加列分组。
    • 时间间隔：在针对日期或时间戳属性类型的每个列分组内，配置分桶的时间间隔（如精确的日期/时间、周、月）。
    • 显示总计？：在表格右侧添加一个总计列分组。
    • 值排序：切换列分组值的升序或降序排列。
  • 聚合：以下选项用于控制表格单元格中显示的聚合结果。
    • 添加聚合：支持对属性类型或整体对象计数添加新的聚合操作。
    • 聚合位置：切换聚合结果放置在列还是行上。默认行为是将聚合结果放置在列上。
    • 聚合标题：点击聚合标题即可进行编辑。为每个指标设定的标题将作为列标题显示在表格中。
    • 聚合指标：控制特定聚合的计算方式。选项包括平均值、最大值、最小值、求和、计数或基数。
    • 列宽：调整特定聚合的列宽。
  • 已选筛选器：此输出对象集筛选器变量用于捕获用户所选单元格的分组条件，可用于筛选下游的小部件和对象集变量。用户可以选择单个单元格、多个单元格组合，或整行、整列。
  • 选中时：允许在用户选中表格内容时触发Workshop 事件（例如在当前模块中打开抽屉）。有关更多详情，请参阅 Workshop 事件文档。

函数驱动的数据透视表

函数驱动的数据透视表从函数的输出中获取数据。

该方法适用于以下用例：

  • 在展示数据前对其进行转换或应用自定义聚合。
  • 合并来自多个数据源的数据。
  • 对数据应用复杂的业务逻辑。

前提条件

  • 函数必须输出一个结构体数组。
  • 每个结构体必须包含一个名为 values 的字段，用于存放数据透视表的值。

配置

在下拉菜单中选择函数后，构建者可以选择：

  1. 分组字段：用于决定数据在行和列中的组织方式。
  2. 值字段：用于决定单元格中显示的指标。
  3. 可展开行：可展开以显示更详细数据的字段。

配置完成后，透视表将根据函数返回的数据进行渲染：

总计

基于函数的透视表支持显示总计。若要渲染总计值，请在列表中返回一个遵循以下准则的结构体。在以下示例中，假设 region 和 productType 为行分组字段，year 为列分组字段。

行总计：若要表示所有行的总和（行总计），请省略数据点中的行分组字段。

示例：表示 2021 年总计的数据点：



已复制！
1{
2    year: "2021";
3    values: {
4        totalSales: 622000;
5        estimatedSales: 57000;
6    }
7};

列总计：要表示所有列的总和（列总计），请省略数据点中的列分组字段。

示例： 表示 EU 和 Clothing 总计的数据点：



已复制！
1{
2    region: "EU";
3    productType: "Clothing";
4    values: {
5        totalSales: 57000;
6        estimatedSales: 57000;
7    }
8}

总计：要表示总计，请省略所有分组字段。

空桶

空桶可用于表示缺失或未定义的数据。

创建空桶的方法如下：

  1. 为桶的值返回 undefined。
  2. 确保你的接口支持 undefined 字段。

以下是一个示例：



已复制！
1interface SiteData {
2    site?: string;
3    part?: string | undefined;  // 注意这里显式指定的 undefined 类型
4    values: {
5        quantity: Double;
6    }
7}


已复制！
1{
2    "part": undefined,
3    "source": "SourceA",
4    "values": {
5        "quantity": 100
6    }
7}

省略字段与传入 undefined 不同。省略字段会产生总计，而 undefined 会产生空桶。

可展开行

可展开行允许用户下钻到更详细的数据中。

实现可展开行的步骤如下：

  1. 在配置选项的可展开行区域添加行字段。
  2. 调整数据结构以支持不同层级的展开。

针对以下接口，我们会在配置选项中选择 productName 和 productType 作为可展开行：



已复制！
1interface SalesData {
2    region: string;
3    year: string;
4    productType?: string;
5    productName?: string;
6    values: {
7        totalSales: Integer;
8        estimatedSales: Integer;
9    }
10};

以下是三个扩展级别的示例：

  1. 无扩展：仅 region 级别。
[ { "region": "NA", "year": "2021", "values": { "totalSales": 30000 } } ]
  1. 一级展开：region 和 productType。
```json [ { "region": "NA", "year": "2021", "productType": "服装", "values": { "totalSales": 90000 } }, { "region": "NA", "year": "2021", "productType": "电子产品", "values": { "totalSales": 150000 } }, { "region": "NA", "year": "2021", "productType": "家具", "values": { "totalSales": 60000 } } ] ``` 3. **二级展开：** `region`、`productType` 和 `productName`。 ```json [ { "region": "NA", "year": "2021", "productType": "Electronics", "productName": "ProductA", "values": { "totalSales": 5000 } }, { "region": "NA", "year": "2021", "productType": "Electronics", "productName": "ProductB", "values": { "totalSales": 5000 } }, { "region": "NA", "year": "2021", "productType": "Electronics", "productName": "ProductC", "values": { "totalSales": 5000 } } ] ```

选择

基于函数的数据透视表输出选择可写入结构体变量。结构体字段由函数输出派生而来。

显示与样式

数据透视表小组件提供以下显示与样式选项：

  • 布局：根据用户偏好配置数据透视表的视图。
    • 表格：默认数据透视图视图。
    • 堆叠：将所有行分组合并至单列，提供更紧凑的视图。
      • 自定义堆叠分组标签：启用后可重命名分组列。
  • 宽度填充：启用后，数据透视表将扩展以填满其父容器中的可用空间，并在容器尺寸改变时自动调整。
  • 表格样式：提供三种数据透视表单元格与边框样式选项。
    • 基础：默认数据透视表样式。
    • 轮廓：在每个顶层行分组的上下方添加较深的轮廓线。
    • 条纹行：在每个顶层行分组的上下方添加较深的轮廓线，并为表中交替排列的行额外添加浅灰色背景。
  • 颜色：自定义数据透视表表头颜色。若选择基础表格样式，所选颜色的较浅色调将应用于行分组列的背景。
    • 极简：在数据透视表表头显示所选颜色的较浅色调。
    • 醒目：在数据透视表表头按原色显示所选颜色。
