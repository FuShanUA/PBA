停止和启动实体

在 Kubernetes 中，活的 Pod 不能以传统方式暂停和重启。然而，可以将副本数设置为 0 来停止所有实例。然后，将副本数增加到非零值会“重启”服务。

您可以通过在 Apollo 的配置覆盖中将副本数设置为 0，在给定环境中停止和重新启动实体。要重新启动已停止的服务，删除将副本数设置为 0 的配置，以将副本数恢复到产品的默认值。

下面提供了一个将副本数设置为 0 的示例覆盖。此示例覆盖将该实体的期望副本数设置为 0（具体来说，是所有大于或等于 2.2.0 的 2.x.x 版本）。



已复制！
1"2.2.0":
2  live-reloaded-config:
3    replication:
4      desired: 0
