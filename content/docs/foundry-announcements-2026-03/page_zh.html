AIP 分析师现已普遍可用

发布日期：2026-03-31

AIP Analyst 将于 2026 年 4 月 13 日那一周对启用了 AIP 的用户正式发布。AIP Analyst 是一个对话式 AI 界面，用于探索您的本体论，使技术人员和非技术人员都能从本体论数据中探索并生成洞察。通过 AIP Analyst，您可以提出问题、可视化结果，并完全透明地了解分析的每一步。

关键特性

AIP Analyst 可以使用各种工具来遍历和研究您的本体论数据：

  • 搜索和理解您的本体论。
  • 使用搜索周边和语义搜索来过滤对象集。
  • 执行分组聚合以进行细分、分析和报告。
  • 针对对象集和数据集编写 SQL 查询。
  • 使用 Vega 图表和地图创建可视化效果。
  • 在对话中直接执行 Foundry 操作和函数。
  • 分析手动添加的文件，如图片、PDF 和电子表格。

AIP Analyst 设计用于为用户提供透明度和控制权。您可以审查分析的每一步，验证逻辑，在需要时进行手动调整，并以交互式图表视图查看分析的谱系。

示例用例

例如，运营经理可以向 AIP Analyst 提问：“我设施中的哪些设备需要关注？”为了回答这个问题，AIP Analyst 将自动遍历本体论，根据相关对象（如检查日志和维护记录）之间的关系生成一个需要采取行动的项目优先级摘要。

在探索本体论数据的过程中，AIP Analyst 还可以揭示并呈现数据质量问题，比如过时的记录。这有助于团队在不需要等待数据完全清洁的情况下识别差距。从那里，AIP Analyst 可以帮助制定改进本体论的计划，然后由 AI FDE 执行。

具有完全透明性的智能分析

AIP 分析师在回答您的问题时会“展示其工作过程”，以便为您提供完全的透明度。

  • 每次分析都会创建一个交互式依赖图，显示从问题到答案的流程。
  • 用户可以清楚地看到代理是如何处理他们的请求的，检查中间结果，并手动调整步骤。
  • 最终摘要消息包括内联引用，这些引用链接到每个响应部分所依据的具体工具结果，这些被引用的摘要直接出现在分析图中。
  • 摘要中的代码块也可以独立复制。

上面的例子展示了 AIP Analyst 探索有关机场的数据，以了解哪些机场最繁忙。

Workshop 集成

AIP 分析师可以作为 Workshop 小部件嵌入，以在 Workshop 模块中直接提供 AI 驱动的分析功能。Workshop 小部件支持在发送消息时触发操作以进行日志记录和跟踪用户交互，在用户切换标签页或关闭部分时保留对话状态，并提供广泛的配置选项来控制数据访问、工具可用性和用户界面自定义。

现在，您可以在Pipeline Builder中的提案内的各个节点上直接添加评论。以前，评论仅限于管道级别。

所有注册用户均可使用此功能，节点级别的评论让您：

  • 在节点级别添加、查看和管理评论： 将您的反馈、问题或标记的问题准确地定位到相关节点上。

  • 使用丰富的格式： 所有节点注释都支持 Markdown：添加 粗体, 斜体, 链接, 以及 code blocks 以实现更清晰的沟通。

  • 选择一个节点并使用 评论图标 添加您的反馈。您也可以双击进入节点并在其中添加评论。

在“更改”选项卡中的管道图中选择特定节点时，使用评论图标来添加与该节点相关的评论。

使用Insight分析和调查您的本体论

发布日期：2026-03-27

Insight是一款无需编写代码即可探索和分析本体论数据的点击式应用程序，从3月30日的那一周开始将对所有注册用户正式推出。

显示一个 Aircraft 对象类型的分析路径的 Insight 工作簿。

Insight 允许您逐步构建分析路径以过滤对象、跟踪对象链接、聚合结果并创建可视化效果。您的工作将保存在 工作区（workbook） 中。 ，用于在一个地方存储分析路径、布局和数据。工作簿可以保存、与其他用户共享，并用于创建可在平台上重复使用的已发布对象集。

Insight 专为需要探索本体论数据、深入研究对象类型之间的关系并在不需要深厚技术专业知识的情况下对结果采取操作的运营用户和分析师而设计。

关键功能

分析路径：通过在分析面板中连接过滤器、关联和转换卡片来构建多步骤分析。每张卡片会跟踪该步骤的对象数量，且单个步骤可以切换开启或关闭，而不必将其从路径中移除。

对象类型关联：使用关联 通过卡片遍历对象关系，并将您的分析转向连接的对象类型。例如，从过滤后的 Aircraft 开始，跟随关联到 Flights，然后再次跟随关联到 Airports，以仅查看与您过滤结果相关的机场。

聚合和SQL： 使用 按组分组 操作对对象进行分组并计算指标，或者在 SQL沙盒中直接编写 SQL 查询 进行高级分析。SQL 作用于单一对象类型，并支持与其他对象类型的连接，而无需依赖本体论关联类型。

Insight 中的 SQL 查询，用于搜索城市属性为 City 的对象类型数据。

可视化和地图： 在 图表 选项卡中将属性可视化为分布、直方图和单个统计指标。如果对象类型包含地理属性，则 地图 选项卡可用，可通过绘制的形状和多边形来空间筛选对象。

在图表选项卡中，显示了 Aircraft 对象类型分析的数据可视化。

数据回写： 通过操作直接从分析结果创建、更新和删除对象，无需离开Insight。

共享与复用： 将任何分析路径发布为对象集，使其在其他应用中可被发现并使用，包括 Quiver 和 Vertex。与特定用户共享工作簿，或通过链接共享并配置访问角色。

要开始在Insight中构建分析，请 查看我们的文档。

一个示例 Palantir CRM 代理使用 Copilot Studio 通过 Microsoft 自适应卡片显示数据。

开始

首先，安装 Microsoft 365 Agents Toolkit 和 Adaptive Card Previewer VS Code 扩展。然后，在 VS Code 中配置你的本体论 MCP 服务器，并使用 Microsoft 365 Agents Toolkit 创建一个自适应代理。在代理的工具定义中添加一个功能块 response_semantics，以将本体论 MCP 工具输出映射到 Adaptive Card 模板。

Autopilot 工作区的看板视图。

功能

  • 交互式流程图： 可视化您的自动化、逻辑函数和操作如何作为 依赖关系图 连接。展开节点以查看回退效果和历史对象路径，并使用实时活跃指示器监控活动执行。
  • 对象级追踪： 跟踪任何 对象在工作流中的完整旅程，包括哪些自动化运行了，做了哪些更改，以及在哪里出现了故障。
  • 实时调试： 使用 对象执行 选项卡 实时监控自动化事件和故障，直接在 Autopilot 中显示跟踪日志、错误信息和时间细节。
  • 看板式监控：观察对象 在工作流状态中移动，具有实时更新和对对象详细信息的即时访问。
  • 统一资源视图： 从一个工作区浏览和管理所有自动化资源，支持 Workshop、操作和专业代码代理。

Autopilot中的依赖关系图视图。

    介绍Pilot，Foundry的AI驱动工具，用于构建React OSDK应用程序
    发布日期：2026-03-05
    Pilot是一个AI驱动的应用程序构建器，它允许您使用自然语言提示在您的本体论基础上创建全栈应用程序。Pilot将在AIP启用的注册用户中于3月9日那一周开始提供测试版。要使用Pilot，请描述您想要构建的应用程序，Pilot将生成本体论、设计和前端代码，在一个隔离的工作区中，无需手动数据连接或UI编码。
    

使用 Pilot，构建一个基于本体论的应用程序从一个提示开始。而不是分别定义对象类型、编写操作类型、设计 UI 并连接 OSDK 钩子，Pilot 会处理从描述到可部署应用程序的开发生命周期，使您能够专注于您想要构建的内容，而不是如何构建它。
    Pilot 首页，您可以在此描述您想要构建的应用程序。
    基于提示的本体论和设计生成
    当您描述您的应用程序时，Pilot 将启动一个隔离的容器，并将工作分解为结构化任务。首先，本体论构建者代理将为您创建数据模型，包括对象类型、操作类型和关系。您可以在 本体论 标签页中查看生成的本体论，并通过聊天面板中的对话式后续步骤进行优化。
    


    Pilot 根据您的应用程序描述生成对象类型、操作类型和关系。
    接下来，设计师代理会读取本体论和您的需求，以生成涵盖配色方案、排版、布局、交互模式和表单的详细设计规范。此规范确保从一开始生成的前端就经过打磨并可直接用于生产。
    前端生成与实时预览
    应用构建器代理使用本体论和设计规范来实现用户界面。它利用OSDK钩子、功能表单、状态管理和筛选器构建一个具有实时数据加载的React应用程序；所有功能都直接连接到您的本体论操作。生成完成后，将在Pilot工作区显示您应用程序的实时预览，让您立即查看结果。
    

你可以通过与Pilot聊天继续对应用程序的任何方面进行迭代。例如，你可以让Pilot向本体论中添加新字段、更改布局或引入额外功能。Pilot会将每次更改记录为结构化任务，使你能够轻松地跟踪应用程序的演变。
    由Pilot生成的应用程序，包含实时预览和迭代聊天优化。
    使用种子数据的安全测试
    Pilot可以在容器内生成真实的种子数据，让你在不暴露真实数据集的情况下测试你的应用程序。因为种子数据存储在容器的本地数据存储中，因此你可以安全地对应用程序进行迭代而不会影响生产数据。如果出现任何导入问题，Pilot会自动显示并解决这些问题。
    

引导部署到生产环境
    当您的应用程序准备就绪时，Pilot 将提供一个引导式部署工作流，帮助您通过 Foundry 分支进行本体论更改、配置开发人员控制台应用程序、运行 CI 检查并标记发布版本。结果是一个托管在自定义子域名上的生产应用程序，具有 OSDK 支持的本体操作，并且无需手动 API 编程。

    使用Quiver中的专用工作区进行时间序列分析

在 Quiver 中使用专用工作区进行时间序列分析
    发布日期：2026-03-03
    Quiver 现在包括重新设计的分析创建体验，使您更容易为您的任务选择合适的工作区。创建新分析时，现在可以从三种分析类型中选择：Quiver 分析、时间序列分析和对象集路径分析。
    更新后的新建分析页面在 Quiver 中，允许您从三种可用的分析工作区中进行选择。
    作为此更新的一部分，我们推出了 时间序列分析 工作区

，一个专为临时时间序列分析设计的界面。它提供了一个简化的环境，用于可视化和比较时间序列数据，而无需承担Quiver分析的全部复杂性，从而使更多用户能够使用。当需要更高级的工作区时，可以将时间序列分析直接在Quiver中打开。
    主要功能
    • 添加和探索时间序列数据：通过熟悉的搜索体验从本体论中添加时间序列，然后从详细信息面板查看和配置图表属性，例如轴分配、单位、根对象、插值和统计信息。
    • 执行时间序列操作：在不离开分析上下文的情况下，直接对图表应用滚动聚合、公式、过滤器和事件统计等操作。
    • 可视化事件集：在图表上叠加来自关联对象或可配置条件的事件，与时间序列数据一起显示。
    • 在多个画布上进行组织：

在多个图表画布上使用同步或独立的x轴进行操作。根据需要重新排序、移动和隐藏图表，并按单位自动分组轴。
    • 保存和共享分析：将分析保存为Foundry资源以便以后使用，并在专用资源查看器中打开它们，或通过分析RID将它们加载到Workshop小部件中。
    在Workshop中构建引导式时间序列分析体验
    时间序列分析小部件将相同的界面和工具带入Workshop，使应用构建者可以直接在其应用中嵌入时间序列分析。该小部件提供了细粒度的配置选项，以针对操作用户定制体验：
    • 过滤哪些本体论（Ontology）系列可供用户添加到他们的分析中
    • 使用对象集（object set）变量控制初始时间序列和事件集
    • 自定义可用的图表类型和事件集类型
    • 

Set chart display options including default view range, tooltip display, and x-axis syncing behavior across canvases
    • Configure how analyses are saved and loaded, including default save location and autoloading
    Users can also open their analysis in Quiver directly from the widget for more advanced workflows. Note that changes made in Quiver are not reflected back in the Workshop widget.
    Getting started
    To create a new time series analysis, navigate to the New Analysis button on the Quiver splash page or Foundry side panel. Choose a name and location for your file and select Time Series Analysis before saving.

    For more information, review the Quiver analysis types documentation

 and the Time Series Analysis widget documentation.

    Expanded Workflow Lineage access across the Palantir platform
    Date published: 2026-03-03
    You can now open Workflow Lineage graphs from more locations across the platform using the Cmd+i (macOS) or Ctrl+i (Windows) shortcut, as well as a dedicated navigation option available on various resource types.

    Access Workflow Lineage
    • Use the Cmd+i (macOS) or Ctrl+i (Windows) keyboard shortcuts to open Workflow Lineage, or select the Open in Workflow Lineage option on a resource where available.
    • You will be redirected to the resource's Workflow Lineage graph displaying the selected node, plus any direct upstream and downstream nodes. Note that some resources may not display actions, functions, or objects if those concepts do not apply.
