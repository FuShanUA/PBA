AIP Analyst 一般可用

发布日期：2026-03-31

AIP Analyst 将于 2026 年 4 月 13 日这一周对启用了 AIP 的用户正式发布。AIP Analyst 是一个对话式 AI 接口，用于探索您的本体论（Ontology），使技术人员和非技术人员都能从本体论数据中探索并生成见解。通过 AIP Analyst，您可以提出问题、可视化结果，并完全透明地了解分析的每一步。

关键特性

AIP Analyst 可以使用各种工具来遍历和研究您的本体论（Ontology）数据：

  • 搜索并理解您的本体论（Ontology）。
  • 使用搜索范围和语义搜索来过滤对象集。
  • 执行分组聚合操作以进行细分、分析和报告。
  • 针对对象集和数据集编写 SQL 查询。
  • 使用 Vega 图表和地图创建可视化效果。
  • 在对话中直接执行 Foundry 操作和函数。
  • 分析手动添加的文件，如图片、PDF 和电子表格。

AIP Analyst 设计用于为用户提供透明度和控制权。您可以审查分析的每一步，验证逻辑，在需要时进行手动调整，并以交互式图表视图查看分析的谱系。

示例用例

例如，运营经理可以向 AIP Analyst 提问：“我设施中的哪些设备需要关注？”为了回答这个问题，AIP Analyst 将自动遍历本体论，根据相关对象（如检查日志和维护记录）之间的关系生成一个需要采取行动的项目优先级摘要。

在探索本体论数据的过程中，AIP Analyst 还可以揭示并呈现数据质量问题，比如过时的记录。这有助于团队在不需要等待完美清洁数据之前就开始识别差距。从那里，AIP Analyst 可以帮助制定改进本体论的计划，然后由 AI FDE 执行。

具有完全透明度的智能分析

AIP Analyst 在回答您的问题时会“展示其工作过程”，以便为您提供完全的透明度。

  • 每次分析都会创建一个交互式依赖图，显示从问题到答案的流程。
  • 用户可以查看代理是如何推理他们的请求的，检查中间结果，并手动调整步骤。
  • 最终摘要信息包括内联引用，这些引用链接到每个部分响应所依据的具体工具结果，并且这些引用的摘要直接出现在分析图中。
  • 摘要中的代码块也可以独立复制。

在Pipeline Builder中的提案节点上添加评论

发布日期：2026-03-31

现在，您可以在Pipeline Builder中的提案内的各个节点上直接添加评论。以前，评论仅限于管道级别。

所有注册用户均可使用，节点级别的评论功能使您可以：

  • 在节点级别添加、查看和管理评论： 将您的反馈、问题或标记的问题准确地定位到相关节点上。
  • 解决和重新打开评论： 将反馈标记为已处理，或根据需要重新开启讨论。从“评论”面板切换已解决评论的可见性，以专注于当前的讨论。
  • 使用@提及同事： 将正确的人引入对话中，确保他们在需要输入的地方参与进来。
  • 使用丰富的格式： 所有节点注释都支持 Markdown：添加 粗体, 斜体, 链接, 以及 code blocks 以实现更清晰的沟通。

从注释文档中了解更多内容。

“注释”选项卡位于“更改”部分内。左侧面板显示所有与提案相关的注释，而右侧窗口显示特定节点上的注释。

提案概览页面，底部有一个用于新输入的注释框，顶部有一个“隐藏已解决的注释”切换按钮，用于过滤注释。

如何使用

节点级评论使协作更加高效和有条理，确保反馈始终具有可操作性和上下文相关性。要开始使用，请按照以下步骤：

  • 在Pipeline Builder中打开您的提案。
  • 转到 更改 选项卡。
  • 选择一个节点并使用 评论图标 添加您的反馈。您也可以双击进入节点并在其中添加评论。

在“更改”选项卡中的管道图中选择特定节点时，使用评论图标来添加与该节点相关的评论。

使用Insight分析和调查您的本体论

发布日期：2026-03-27

Insight，一个无需编写代码即可探索和分析本体论数据的点按式应用程序，将于3月30日那一周开始对所有注册用户全面推出。

显示一个 Aircraft 对象类型的分析路径的Insight工作簿。

Insight让您逐步构建分析路径以过滤对象、跟踪对象链接、聚合结果并创建可视化效果。您的工作将保存在工作区中 ，用于在一个地方存储分析路径、布局和数据。工作簿可以保存、与其他用户共享，并用于创建可在平台上重复使用的已发布对象集。

Insight 专为操作用户和分析师设计，他们需要探索本体论数据，深入研究对象类型之间的关系，并在不需要深入技术专业知识的情况下对结果采取操作。

关键功能

分析路径：通过在分析面板中链接 筛选器、关联 和 转换 卡片来构建多步骤分析。每张卡片跟踪该步骤的对象数量，单个步骤可以切换开启或关闭，而无需从路径中移除它们。

对象类型关联： 使用 关联 用于遍历对象关系并将分析转向相关对象类型的操作卡。例如，从过滤后的Aircraft集合开始，跟随关联到Flights，然后再次跟随关联到Airports以仅查看与您过滤结果相关的机场。

聚合和SQL：使用按组进行操作对对象进行分组并计算指标，或者在SQL沙箱中直接编写SQL查询进行高级分析。SQL针对单一对象类型，并支持与其他对象类型进行连接，而无需依赖本体论关联类型。

Insight中的SQL查询，用于搜索城市属性为埃尔帕索的对象类型数据。 City 属性是埃尔帕索。

可视化和地图：在图表选项卡中将属性可视化为分布、直方图和单个统计指标。如果对象类型包含地理属性，则可以通过绘制的形状和多边形来空间过滤对象，此时会提供地图选项卡。

在图表选项卡中，显示了 Aircraft 对象类型分析的数据可视化。

数据回写： 使用操作直接从分析结果创建、更新和删除对象，无需离开Insight。

共享与复用： 将任何分析路径发布为一个对象集，使其在包括Quiver和Vertex在内的其他应用程序中可发现和可用。与特定用户共享工作簿，或通过链接共享并配置访问角色。

要开始在Insight中构建分析，请查看我们的文档。

Ontology MCP 与 Microsoft Copilot Studio 集成

发布日期：2026-03-26

Ontology MCP 现在支持与 Microsoft Copilot Studio 集成，使您可以将 Foundry 的本体论连接到 Microsoft 的 AI 代理平台，并直接在 Teams、Copilot 和 Outlook 中显示结构化的本体数据。

在设置此集成时，请将您的开发者控制台应用程序创建为一个 Backend 服务，并具有 User's 权限。Copilot Studio 仅支持保密客户端中的授权码授予，这需要一个服务用户代表您的用户颁发令牌。

用于丰富数据展示的自适应卡片

通过此集成，您可以使用 Microsoft 自适应卡片将本体论数据呈现为格式化、交互式的卡片，而不是纯文本。例如，代理可以查询您的本体论以获取客户投资组合，并以包含风险状态、续订日期和操作按钮的结构化卡片显示结果。

一个示例 Palantir CRM 代理使用 Copilot Studio 通过 Microsoft 自适应卡片显示数据。

您可以使用 Action.OpenUrlDialog 在 Adaptive Cards 中打开 Teams 或 Copilot 中的自定义对话框，使用户能够内联与 Ontology MCP 数据进行交互。

开始

首先安装 Microsoft 365 Agents Toolkit 和 Adaptive Card Previewer VS Code 扩展。然后在 VS Code 中配置您的 Ontology MCP 服务器，并使用 Microsoft 365 Agents Toolkit 创建一个自适应代理。将包含 response_semantics 的功能块添加到代理的工具定义中，以将 Ontology MCP 工具输出映射到 Adaptive Card 模板。

如需逐步操作，请参阅 MCP 工具和代理配置文档。

使用 Autopilot 可视化、监控和调试您的自动化工作流程

发布日期：2026-03-19

随着团队在 Foundry 中连接自动化、逻辑函数和操作，理解这些系统的行为变得困难。 Autopilot 现已推出测试版，提供所需的可见性以了解自动化如何连接，追踪工作流中的对象，并在一个地方调试故障。作为测试产品，功能和外观可能在开发过程中发生变化。

Autopilot 工作区的看板视图。

功能

  • 交互式流程图：可视化您的自动化、逻辑函数和操作如何作为< a href="/docs/foundry/autopilot/workbench-graph/">依赖图连接。展开节点以查看回退效果和历史对象路径，并通过实时存活指示器监控活动执行。
  • 对象级追踪：追踪任何 对象在工作流中的完整旅程，包括哪些自动化运行了、进行了哪些更改以及故障发生在哪里。
  • 实时调试：使用 对象执行 选项卡，实时监控自动化事件和故障，直接在 Autopilot 中显示跟踪日志、错误信息和时间细节。
  • Kanban 风格的监控：观察对象 在工作流状态中移动，具有实时更新和对对象详细信息的即时访问。
  • 统一资源视图：从一个工作区浏览和管理所有自动化资源，支持 Workshop、操作和专业代码代理。

Autopilot中的依赖关系图视图。

入门

从应用门户打开Autopilot，或在任何自动化概览页面的操作菜单中选择在Autopilot中打开。

  • 选择您的起点：选择一个对象类型或自动化。Autopilot会生成一个初始工作区，并提醒您相关但尚未包含的自动化。
  • 审查并添加自动化：按照提示构建完整的系统视图。
  • 自定义状态：在状态侧边栏中添加、重新排序或手动定义状态。
  • 启用项目范围模式：允许所有工作区用户查看执行事件并发现自动化依赖项。请注意，这将扩展项目的权限。
  • 启用编辑历史跟踪：要查看对象的历史记录并追踪对象在您的工作流中的路径，这是必需的。

一旦你的工作台配置完成，就可以探索流程图以可视化你的自动化系统，并选择任意对象来追踪它在工作流中的路径。

通过 AIP 文档智能中的分块和嵌入提高 RAG 准确率

发布日期：2026-03-17

AIP 文档智能现在支持所有注册中的文本分块和嵌入。除了基于视觉语言模型和 OCR 的文档提取功能外，您现在可以在平台上直接端到端处理文档。分块是文档为中心的工作流程中的关键步骤 —— 它决定了传递给 RAG 系统模型的文本粒度，直接影响检索准确性和下游生成质量。

新的分块策略针对 Markdown 进行了优化，并能处理复杂的结构，如项目符号和表格，优于 Pipeline Builder 和 AIP Logic 中现有的原始文本分块。您可以通过 AIP 文档智能的文本提取工作流访问此功能，或者通过 Python 转换进行部署，并提供生成嵌入选项以支持基于 RAG 的工作流。

关键优势

  • 更高品质的嵌入: 将大文档分解为语义连贯的块，以适应嵌入模型的上下文窗口。每个块都会生成向量嵌入。

在Pipeline Builder中强制增量执行

发布日期：2026-03-17

您现在可以通过Pipeline Builder中的强制增量执行设置来确保管道的增量执行。启用此设置后，如果无法进行增量执行，配置为增量运行的任务将自动失败。这有助于防止意外快照输入、因输出模式变更导致的强制快照以及其他意外快照场景。

强制增量执行设置在构建设置菜单中。

启用强制增量执行

要配置强制增量执行，请打开 构建设置 通过选择管道中部署按钮右侧的配置图标。向下滚动至高级配置，并将需要增量执行设置为是。此设置的默认值为无值。

在为您的管道启用此设置时，请注意以下事项：

  • 当启用后，如果无法进行增量运行，则给定管道中的所有增量输出都会失败，无论作业分组如何。
  • 只有当您的管道至少有一个增量输入或输出时，才能将需要增量执行设置为是。如果没有，则启用此选项会导致部署错误。
  • 仅有一种情况，配置为需要增量执行的管道可以作为快照运行，即当管道的更改需要进行状态中断时。您将在运行前被要求确认。

此功能之前仅在PySpark和轻量级增量转换中可用

通过在 require_incremental=True 装饰器中设置 @incremental，现在已经可以在管道构建器（Pipeline Builder）中使用，以弥合低代码和专业代码工作流之间的差距。

    介绍 Pilot，Foundry 的 AI 驱动工具，用于构建 React OSDK 应用程序

    发布日期：2026-03-05

    Pilot 是一个 AI 驱动的应用程序构建器，它允许您使用自然语言提示在您的本体论（ontology）之上创建全栈应用程序。Pilot 将在 AIP 启用的注册用户中于 3 月 9 日的那一周开始提供测试版。要使用 Pilot，请描述您想要构建的应用程序，Pilot 将在隔离的工作区中生成本体论（ontology）、设计和前端代码，无需手动的数据连接或 UI 编程。

    使用 Pilot，构建一个基于本体论的应用程序从一个简单的提示开始。而不是分别定义对象类型、编写操作类型、设计用户界面并连接 OSDK 钩子，Pilot 会从描述到可部署的应用程序处理开发生命周期，使您能够专注于您想要构建的内容，而不是如何构建它。

    Pilot 首页，您可以在此描述想要构建的应用程序。

    基于提示的本体论和设计生成

    当您描述您的应用程序时，Pilot 将启动一个隔离的容器，并将工作分解为结构化任务。首先，本体论构建者代理会为您的应用程序创建数据模型，包括对象类型、操作类型和关系。您可以在 本体论 选项卡中查看生成的本体论，并通过聊天面板中的对话式后续步骤进行优化。

    Pilot根据您的应用程序描述生成对象类型、操作类型和关系。

    接下来，Designer代理读取本体论和您的需求，以生成涵盖配色方案、排版、布局、交互模式和表单的详细设计规范。此规范确保生成的前端从一开始就经过打磨并可投入生产使用。

    前端生成与实时预览

    App builder代理使用本体论和设计规范实现用户界面。它使用OSDK钩子、功能表单、状态管理和过滤器构建一个具有实时数据加载的React应用；所有内容直接连接到您的本体论操作。当生成完成后，将在Pilot工作区显示您应用程序的实时预览，使您可以立即查看结果。

    你可以通过与Pilot聊天继续对应用程序的任何方面进行迭代。例如，你可以让Pilot向本体论中添加新字段、更改布局或引入额外功能。Pilot会将每次更改记录为结构化任务，使你能够轻松地跟踪应用程序的演变。

    由Pilot生成的应用程序，包含实时预览和迭代聊天优化。

    使用种子数据的安全测试

    Pilot可以在容器内生成真实的种子数据，让你在不暴露真实数据集的情况下测试你的应用程序。因为种子数据存储在容器的本地数据存储中，因此你可以安全地对应用程序进行迭代而不会影响生产数据。如果出现任何导入问题，Pilot会自动显示并解决这些问题。

    引导式部署到生产环境

    当您的应用程序准备就绪时，Pilot将提供一个引导式部署工作流，指导您通过Foundry分支来推广本体论更改，配置开发者控制台应用，运行CI检查，并标记版本。结果是一个托管在自定义子域名上的生产应用，具有由OSDK支持的本体论操作，无需手动API连接。

    作为此次更新的一部分，我们推出了 时间序列分析 工作区 ，一个专为临时时间序列分析设计的界面。它提供了一个简化的环境，用于可视化和比较时间序列数据，而无需承担Quiver分析的全部复杂性，从而使更多用户能够使用。当需要更高级的工作区时，可以将时间序列分析直接在Quiver中打开。

    关键特性

    • 添加并探索时间序列数据：通过熟悉的搜索体验从本体论中添加时间序列，然后从详细信息面板查看和配置图表属性，例如轴分配、单位、根对象、插值和统计信息。
    • 执行时间序列操作：在不离开分析上下文的情况下，直接对图表应用滚动聚合、公式、过滤器和事件统计等操作。
    • 可视化事件集：在图表上叠加来自关联对象或可配置条件的事件，与时间序列数据一起显示。
    • 在多个画布上进行组织： 在多个图表画布上使用同步或独立的x轴进行操作。根据需要重新排序、移动和隐藏图表，并按单位自动分组轴。
    • 保存和共享分析：将分析保存为Foundry资源以供以后使用，并在专用资源查看器中打开它们，或使用分析RID将其加载到Workshop小部件中。

    在Workshop中构建引导式时间序列分析体验

    时间序列分析小部件将相同的界面和工具带入Workshop，使应用构建者可以直接在其应用中嵌入时间序列分析。该小部件提供了细粒度的配置选项，以针对操作用户定制体验：

    • 过滤哪些本体论系列可供用户添加到他们的分析中
    • 使用对象集变量控制初始时间序列和事件集
    • 自定义可用的图表类型和事件集类型
    • 设置图表显示选项，包括默认视图范围、提示信息显示和跨画布的x轴同步行为
    • 配置分析的保存和加载方式，包括默认保存位置和自动加载

    用户还可以直接从小部件在Quiver中打开他们的分析，以进行更高级的工作流程。请注意，在Quiver中所做的更改不会反映回Workshop小部件中。

    开始使用

    要创建一个新的时间序列分析，请导航到Quiver启动页面或Foundry侧边栏上的新分析按钮。为您的文件选择一个名称和位置，并在保存之前选择时间序列分析。
