OpenID Connect (OIDC) 身份验证

OpenID Connect (OIDC) ↗（简称 OIDC）是一种开放身份验证协议，支持在不使用静态凭证的情况下对外部源资源进行身份验证。

采用 OIDC 时，无需在 Foundry 中为源系统配置凭证。如此一来，可避免将源系统凭证作为长期密钥复制到 Foundry 中。取而代之的是，将在 Foundry 与源系统之间配置信任关系。Foundry 充当 OIDC 身份提供商；每当 Foundry 中的工作流需要与源系统进行身份验证时（例如数据连接同步），Foundry 都会颁发一个 OIDC 令牌，该令牌包含用于标识正在使用的数据连接源的声明。源系统能够验证这些声明，并提供一个短期访问令牌，用于后续与源系统的交互。该访问令牌的范围（例如允许访问的资源）完全在源系统中利用原生的身份验证与授权工具进行管理。在配置信任关系时，可以添加条件来过滤传入请求。不受信任的 Foundry 源无法请求获取其不应访问的源系统资源的访问令牌。

支持的来源

以下来源支持 OIDC 身份验证。有关如何设置 OIDC 与 Palantir 之间信任关系的更多详细信息，请参阅各个来源的文档。

  • OneLake 和 Azure Data Lake Storage Gen2（Azure Blob 存储）
  • BigQuery
  • Databricks 平台
  • Google Cloud Storage
  • S3
  • Snowflake 平台

OIDC 身份令牌

以下是由 Foundry 生成的 OIDC 令牌示例：

{
  "iss": "https://pltroidcpublicexample.blob.store.com/foundry",
  "sub": "ri.magritte..source.7f3b8e21-4d9a-6c2e-1b7d-8a5f3c9e0b4f",
  "aud": "your-source-system-audience",
  "iat": <issued-at>,
  "nbf": <not-before>,
  "exp": <expiry>,
  "jti": "<token-unique-identifier>",
  "scp": "<additonal-scope>",
}
声明声明类型描述
iss颁发者 URL将 Foundry 标识为 OIDC 身份提供商的 URL。
sub主体正在连接到您的源系统的 Foundry 源的源 RID。
aud受众用于标识您的源系统的已配置受众。

应使用 source-rid 过滤传入请求，以确保不受信任的 Foundry 源无法访问您的资源。

由 Foundry 生成的 OIDC 令牌将在一小时后过期。
