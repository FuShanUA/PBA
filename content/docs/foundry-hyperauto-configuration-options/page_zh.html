HyperAuto V2 配置选项

本页面介绍 HyperAuto V2 的配置选项。HyperAuto V2 的配置流程包含以下步骤：

  • 名称与位置
  • 源配置
  • 输入配置
  • 管道配置

有关 HyperAuto V1 配置参考，请查阅旧版文档。

名称与位置

在 HyperAuto V2 配置向导的第一步中，需要指定新管道的名称以及 Foundry 文件系统中的目标文件夹位置。HyperAuto 管道资源及关联的输出数据集将在此文件夹内创建。

源配置

HyperAuto V2 的源配置页面可帮助您选择源系统和数据接入方式。

源系统

对于包含多个子系统且需要用户进行选择的源（例如 SAP 内部的“上下文”），可使用此选项。子系统被定义为源内部的一种配置，该配置会生成自身专属的可用数据表和元数据。因此，切换子系统将彻底改变其他可用配置，例如所支持的管道模式（批处理或流式处理），以及在输入配置页面上可供选择的数据表和现有同步任务。

SAP 源系统

将 Foundry 连接到 SAP 系统主要有三种架构模式：

  • 直连模式：连接器安装在 ERP 系统自身的应用服务器上，提供对数据表的直接访问。
  • SLT 模式：连接器安装在 SAP SLT 复制服务器上，由其连接底层的 ERP 系统。若要使用流式管道模式，必须采用 SLT 模式。
  • 远程模式：连接器安装在连接底层 ERP 系统的“网关”应用服务器上。当 SAP 源无法满足连接器前置条件时，通常使用此模式。 在采用 SLT 或远程连接的情况下，用户必须选择一个上下文，用于标识要连接的 SAP 子系统。

管道模式

HyperAuto 支持两种同步和数据转换模式。您可以在源配置页面的初始 HyperAuto 管道设置中，选择流式处理或批处理模式。

  • 批处理：每次运行管道都会重新处理所有输入，并覆盖所有现有输出。这是默认模式，支持最广泛的功能，包括数据聚合和去重。建议大多数用例采用此模式。
  • 流式处理：持续轮询源系统中尚未处理的数据。数据在源系统上一旦可用便立即处理，从而将同步至本体（Ontology）的延迟降至准实时。这对于支持依赖本体对流式数据进行去重的实时应用尤为关键。

流式处理需要始终保持计算状态以实时处理数据，因此可能会增加源系统和 Foundry 内部的负载。

输入配置

在输入配置页面，用户可选择特定 HyperAuto 管道需要处理的具体输入内容。

为提升易用性，输入选择界面支持多种方式来浏览和发现相关的源数据表。对于 SAP，支持的方式包括：

  • 模块：对源中的数据表进行预设分类，提供分层视图，用户可据此进行探索并批量添加。相关数据表可能存在于多个模块中，但同一数据表不能被重复选择。
  • 工作流：另一种形式的表分类方式，侧重于数据源的特定常见用例（例如 SAP 数据源的供应链管理）。同样，用户可以利用工作流进行探索并按需批量添加，还能在工作流与模块之间切换，而不会丢失进度或意外重复选择。

在输入配置页面也可以创建同步，允许用户为任何尚无同步的输入创建新同步。这样，用户只需点击几下，即可从全新的数据源开始，构建出完全配置好的 HyperAuto 管线，无需费力研究每个同步的具体配置方式。

Beta

同步创建功能目前处于 Beta 开发阶段，您的实例可能尚未提供此功能。在活跃开发期间，功能可能会发生变化。请联系 Palantir 支持团队申请获取同步创建权限。

您的 Foundry 实例可能已在 Suggest（建议）选项卡中启用了 AIP 功能；更多信息请参阅 《AIP 文档》。

管线配置

通过管线配置页面，您可以搭建满足自身需求的管线，可用选项包括：

  • 语言选择
  • 配置选项
    • 自动连接
    • 易读列名
    • 易读输出数据集名称
    • 生成主键
    • 生成外键
    • 行去重
    • 数据清洗
    • 增量
  • 批处理计算设置

语言选择

对于包含多语言数据表的源，HyperAuto 提供了语言过滤步骤，以避免在输出中为每种可能的语言生成多行数据。此处选定的语言将在应用其他转换操作（例如与其他表进行连接）之前，作为筛选条件应用于相关表。

配置选项

您可以通过管线配置选项，决定希望对源输入数据自动应用多少处理量。默认情况下，所有配置选项均处于启用状态，但可根据需要禁用（例如，在功能与管线性能之间取得平衡）。

自动连接

HyperAuto 通过数据源的元数据接收表分类信息，并将其拆分为对象表或扩展表。在此定义中，扩展表本身不具备独立的内在价值，而是作为关联对象表的扩展或查找表（例如文本描述表）。

通过这种方式，HyperAuto 能够从数据源中查询 object（对象）与 enrichment（扩展）表之间的关系，并生成从扩展表到对象表的相应左连接。这样一来，每个对象都会生成一个丰富且全面的非规范化数据集，无需再与其他表进行连接即可进行广泛的审查。

这在构建 Foundry 本体时尤为实用，因为标准做法就是使用面向语义的非规范化数据模型。

SAP 中的自动连接

对于 SAP 系统，"TEXT" 表在 HyperAuto 的处理过程中会被归类为 Enrichment（扩展）表。例如，MAKT（物料描述表）可以连接到 MARA（常规物料数据表）上。

流处理

被归类为 Enrichment（扩展）的表将作为批量输入而非数据流被消费。这使得管线能够从这些表创建“查找”左连接并应用到核心数据流上，从而增强流数据，避免尝试同时连接两个实时数据流所带来的问题。

只有在模式符合 Foundry 流处理及底层所使用的 Avro 文件格式要求时，配置相关输入才会提供流处理模式下 Enrichment 表的现有同步选项。

提示：对于 SAP 同步，将配置选项 cleanFieldNamesForAvro 设置为 true 可确保模式符合 Avro（流处理）规范。HyperAuto 创建的同步默认将启用此选项。

人类可读的列名

HyperAuto 可以利用数据源提供的列元数据，将源定义的列名重命名为含义清晰且便于不熟悉源模式用户使用的名称。

具体做法是将列的人类可读名称拼接到原始列名前，格式为 Human readable_|_original，从而在交互操作数据时提供对两种格式的同时访问，实现最佳易用性。

人类可读的输出数据集名称

启用该选项后，HyperAuto 会根据源表元数据，为新建的输出数据集生成描述性名称。这样无需查阅原始源表名，即可更轻松地识别和导航管道输出结果。对于现有管道，开启此选项不会重命名已存在的输出数据集。

生成主键

如果数据源没有单列主键，HyperAuto 可以动态生成主键。数据源的元数据包含了指示表中哪些列共同组成主键的信息，HyperAuto 会据此构建拼接逻辑，从而创建一个 primary_key 列。

这些值会使用 _|_ 分隔符进行拼接。

若要将输出结果用作本体对象的支持数据集，必须拥有单列主键。

生成外键

HyperAuto 还可以访问源数据模型元数据中定义的对象间关系。利用这些元数据，可以在管道中创建逻辑，为每个关系生成一个外键列（通过拼接相关列实现，类似于主键逻辑），该列可用于表连接或构建本体链接。

外键的命名格式为 column1_column2_|_foreign_key_tableA，其含义如下：

  • 列值由 column1 和 column2 拼接而成，分隔符为 _|_，并且
  • 存在一个外键关系，使得用户可以通过此列将本表与 tableA 的 primary_key 进行连接。

要生成本体中对象间的关系，必须存在外键。

当启用了自动连接配置选项时，对于对象到扩充表的关系，系统不会创建外键。

行去重

HyperAuto 提供了自动对包含重复行的表进行去重的逻辑。这在变更数据捕获（CDC）等每次发生变更都会追加新行的场景中非常实用。HyperAuto 会执行去重操作，为每个主键选取最新且有效的数据行。

流处理

在流处理模式下，去重逻辑的处理方式有所不同。系统会创建两个流输出。主要输出在被批处理或增量管道读取时，将解析为去重后的数据集。变更日志输出在被批处理或增量管道读取时，将按需提供未去重的数据集。这两个输出均可以正常被其他流消费。

如果组成表主键的列不属于以下类型，系统会将其转换为字符串类型，以确保去重操作能够正常执行：

  • String
  • Timestamp
  • Boolean
  • Binary
  • Integer
  • Byte
  • Short
  • Float
  • Long
  • Double

数据清洗

数据清洗配置选项可消除所有表中常见的数据质量隐患。有关所解决问题的类型详情，请参阅下文。

  • 空字符串处理：将 "" 空字符串转换为 null（这是 Foundry 中数据处理的的标准实践）。
  • DECIMAL 类型转换：将 DECIMAL 数据类型转换为 DOUBLE，这能带来全平台层面的优势（包括支持将其作为本体属性）。

增量

实验性功能

HyperAuto 管道的增量处理目前处于实验性开发阶段，您的租户环境中可能尚未提供此功能。在活跃开发期间，相关功能可能会发生变更。

对于批量 SAP 管道，您可以启用增量处理。启用后，管道仅处理自上次构建以来追加到输入数据集中的新数据，而无需在每次运行时重新处理所有输入。这能有效降低计算资源消耗并缩短管道的构建时间，尤其适用于拥有庞大数据量且频繁更新的输入数据集的管道。

增量处理仅适用于采用直接 SAP 连接的批量管道。有关增量计算的更多信息，请参阅批量输入数据集的计算模式。

批量计算设置

测试版

HyperAuto 管道的批量计算设置目前处于测试版开发阶段。在活跃开发期间，相关功能可能会发生变更。

对于批量管道，您可以在管道配置页面的批量计算设置部分选择计算后端。此设置决定了用于处理管道的计算方式。

可用的计算后端选项如下：

  • 标准：由 Spark ↗ 驱动的默认后端。此选项支持管道构建器中可用的全部表达式和转换操作。
  • 更快：由 DataFusion ↗ 驱动的替代后端，DataFusion 是一个采用 Rust 编写的开源查询引擎。该后端专为优化构建时间并高效执行低延迟操作而设计。特别是对于运行时间在 15 分钟以内的管道，使用该配置将获得最大的性能提升。

您可以在创建新的 HyperAuto 管道时选择“更快”后端，也可以通过编辑现有管道的配置来切换其后端。
