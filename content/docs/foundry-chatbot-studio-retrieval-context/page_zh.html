检索上下文

AIP 聊天机器人可以配置检索上下文。检索上下文会在每条新用户消息触发时确定性运行，并将检索到的信息传递给 LLM。

您可以为聊天机器人配置任意数量的以下检索上下文类型：

  • 本体上下文
  • 文档上下文
  • 函数支持的上下文

本体上下文

本体上下文为您的聊天机器人提供本体中对象的上下文信息。您可以提供一组固定的 N 个对象，或者执行语义搜索以识别与用户查询最相关的 K 个对象，前提是您的对象类型具有向量嵌入属性。

在配置本体上下文时，您可以选择起始对象集为静态输入（包含完整的对象类型）或变量输入（可能包含作为应用程序状态变量传入的已筛选对象集）。

您还可以配置对象属性列表，决定哪些属性会被打印并作为每个检索对象的上下文传递给 LLM。默认情况下，所有属性都会被选中，但不包括无法打印的属性（如媒体引用或向量嵌入）。

此外，您可以通过配置对象集输出和引用变量输出的变量，将本体上下文与应用程序状态集成。有关更多信息，请参阅应用程序状态和引用变量更新部分。

文档上下文

文档上下文允许用户将文档中的相关文本包含在发送给 LLM 的每条消息中。选择文档并将其包含在 AIP 聊天机器人配置中的方式，与在 AIP Threads 中将文档添加到对话的方式相同。

提供文档上下文有两种模式：

  1. 完整文档文本模式：此模式将文档的全部文本内容提供给 LLM 作为上下文。
  2. 相关分块模式：此模式对文档执行语义搜索，将与上下文最相关的 K 个分块返回给 LLM。
Beta

相关分块模式目前处于测试阶段，您的部署环境中可能无法使用。功能可能会在活跃开发期间发生变化。请联系您的 Palantir 代表或 Palantir 支持团队以申请访问此功能。

函数支持的上下文

函数支持的上下文使用户能够对每个查询执行自定义检索。当本体上下文或文档上下文提供的开箱即用检索方法无法满足特定用例时，此功能非常理想。例如，如果用户想要组合不同的检索方法（如关键字搜索和语义搜索），则需要编写一个函数来实现，因为聊天机器人工作室目前尚不支持此功能。

用户可以在代码仓库中使用 TypeScript 编写这些函数。为此，请导航至 TypeScript 仓库并导入 AipAgentsContextRetrieval 函数接口。

然后，编写满足该接口的函数，如下所示。请注意，为了满足契约要求，该函数必须将 messages 作为唯一必需的输入。

3    let combinedText: string[] = [];
4    messages.forEach((message) => {
5        ...
6    })
7    return {
8        retrievedPrompt: "..."
9    }
10}

检索函数必须输出一个 retrievedPrompt 字符串，AIP 聊天机器人会将其粘贴到 LLM 系统提示词中，用于回答用户查询。

发布函数后，在 Chatbot Studio 的 检索上下文 面板下选择 基于函数的上下文，即可选择用于检索的函数。

检索函数中的应用程序变量

检索函数还可以将聊天机器人上的应用程序变量的值作为输入。要进行配置，请在函数定义中添加可选参数。目前仅支持字符串和对象集合类型的应用程序变量，因此函数输入必须是其中一种类型。

3    ...

请为对象类型使用 API 名称。该名称可在 Ontology Manager 中找到。随后，你可以配置聊天机器人上的应用变量与函数输入之间的映射，确保双方对应类型保持一致。

要创建应用变量，请前往 Chatbot Studio 中的 Application variables（应用变量）面板。

在 AIP Logic 中编写检索函数

用户很快就能在 AIP Logic 中编写此类函数了，该平台提供了一个开箱即用的界面，专门用于开发无代码且由大语言模型驱动的函数。在此期间若要使用检索函数，我们建议编写一个符合接口要求的 TypeScript 函数，并在底层调用 Logic 函数。

自定义引用

如果大语言模型以特定的 XML 格式回复引用内容，AIP Chatbot 界面将渲染出引用气泡。借助基于函数的上下文，用户可以让函数返回一段字符串，提示大语言模型按照该指定格式撰写引用，从而渲染出这些引用。请参阅《引用格式》了解所提供的格式列表，并查阅《引用变量更新》获取有关如何在每次选中对象引用时更新 Workshop 变量的信息。
