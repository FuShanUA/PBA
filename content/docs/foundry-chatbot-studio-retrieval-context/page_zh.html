检索上下文

AIP 聊天机器人可以配置检索上下文。检索上下文与 每个 新用户消息一起确定性运行，并将检索到的信息传递给 LLM。

您可以使用以下任何数量的检索上下文类型来配置您的聊天机器人：

  • 本体论上下文
  • 文档上下文
  • 函数支撑的上下文

本体论上下文

本体论上下文为您的聊天机器人提供来自本体论中的对象的上下文。您可以提供一组固定的 N 个对象，或者执行语义搜索以识别 K 与用户查询最相关的对象，前提是你的对象类型具有向量嵌入属性。

在配置本体论上下文时，您可以选择起始对象集为静态输入，这包括完整的对象类型，或者为变量输入，这可能由作为应用程序状态变量传入的过滤后的对象集组成。

您还可以配置一个对象属性列表，该列表确定哪些属性会被打印并作为上下文传递给LLM以供每个检索到的对象使用。默认情况下，所有属性都会被选中，但不包括无法打印的属性（例如媒体引用或向量嵌入）。

此外，您可以通过为对象集输出和引用变量输出配置变量，将本体论上下文与您的应用状态集成。有关更多信息，请参阅关于应用状态和 引用变量更新。

文档上下文

文档上下文允许用户在将消息发送到LLM时包含来自文档的相关文本。文档可以以与在AIP Threads中添加到对话相同的方式，在AIP聊天机器人的配置中选择并包含。

提供文档上下文有两种模式：

  1. 完整文档文本模式：这种模式将文档的全部文本内容提供给LLM作为上下文。
  2. 相关片段模式：这种模式对文档进行语义搜索，将最相关的个片段返回给LLM作为上下文。
测试版

相关片段模式在测试版中 开发阶段，可能无法在您的注册中使用。功能可能会在积极开发期间发生变化。请联系您的Palantir代表或Palantir支持以请求访问此功能。

函数支撑的上下文

函数支撑的上下文允许用户在每次查询中执行自己的检索。这适用于通过本体论（本体论）上下文或文档上下文提供的内置检索方法无法满足特定用例的情况。例如，如果用户想要结合不同的检索方法，比如关键词搜索和语义搜索，那么他们将编写一个函数来实现这一点，因为目前Chatbot Studio不支持该功能。

用户可以在< a href="/docs/foundry/code-repositories/overview/">代码仓库中的TypeScript中编写这些函数。为此，请导航到一个TypeScript仓库并导入AipAgentsContextRetrieval函数接口。

然后，按照下面所示编写一个满足接口的函数。请注意，为了满足合同，该函数必须将 messages 作为唯一必需的输入。

1@AipAgentsContextRetrieval()
2public exampleRetrievalFunction(messages: MessageList): RetrievedContext {
3    let combinedText: string[] = [];
4    messages.forEach((message) => {
5        ...
6    })
7    return {
8        retrievedPrompt: "..."
9    }
10}

检索函数必须输出一个 retrievedPrompt 字符串，AIP Chatbots会将其粘贴到LLM系统提示中以回答用户查询。

发布您的函数后，在Chatbot Studio的检索上下文面板下选择基于函数的上下文，以选择用于检索的函数。

检索函数中的应用变量

检索函数还可以接受 应用变量 的值 将聊天机器人作为输入。要进行此配置，请向函数定义中添加可选参数。目前，仅支持字符串和对象类型的设置应用程序变量，因此函数输入必须是这些类型之一。

Copied!
1@AipAgentsContextRetrieval()
2public movieRetrievalFunction(messages: MessageList, movieTitle?: string, movieSet?: ObjectSet<Movie>): RetrievedContext {
3    ...
4}

使用对象类型的API名称。这可以在本体论管理器中找到。然后可以配置聊天机器人上的应用程序变量与匹配其各自类型的函数输入之间的映射。

要创建应用程序变量，请导航到聊天机器人工作室中的应用程序变量面板。

在AIP Logic中编写检索函数

用户很快就可以在AIP Logic中编写这些函数，它提供了一个易于使用的界面来开发无需编码的LLM驱动函数。为了在此期间利用检索函数，我们建议编写一个满足接口并调用Logic函数的TypeScript函数。
