apollo-cli product-release get

获取一个或多个产品发布

通过 Maven 坐标获取一个或多个产品发布。

当提供单个坐标时，以 YAML 格式打印发布信息。 当提供多个坐标时，结果以表格形式显示。

示例： apollo-cli product-release get com.palantir.product:foo:1.0.0 apollo-cli product-release get com.palantir.product:foo:1.0.0 com.palantir.product:bar:2.0.0

如果您想获取您不是成员的注册机构的主要组织中的产品发布， 请使用全局 '--space-id' 标志设置 spaceID。

从父命令继承的标志

标志用途
--apollo-client-id用于生成 Bearer Token 的客户端 ID（Client ID）
--apollo-client-secret用于生成 Bearer Token 的客户端密钥（Client secret）
--apollo-token用于认证端点的 Bearer Token（Bearer Token）
--apollo-token-provider 指定用于认证Apollo端点调用的持有者令牌的提供方式。有效值为"auto"、"static"、"service-user"或"sso"（默认为"auto"）。如果指定"auto"，则从配置中选择模式："apollo-token"设置时为"static"，"apollo-client-id"和"apollo-client-secret"都设置时为"service-user"，否则为"sso"（交互式浏览器登录）。如果同时设置了持有者令牌和（客户端ID或客户端密钥），将报错。如果指定"static"，则使用"apollo-token"提供的令牌。如果指定"service-user"，则使用"apollo-client-id"和"apollo-client-secret"从Multipass生成令牌。如果指定"sso"，则使用针对Multipass的基于浏览器的SSO登录（支持硬件密钥如YubiKey）；生成的令牌按配置文件缓存并静默刷新。当首次需要令牌且没有有效的缓存令牌时，将自动触发登录流程。
--apollo-urlApollo的基础URL，用于推导API端点
--debug启用调试级别日志记录
-e, --environment用于环境作用域命令的环境ID（包括后缀）
--http-timeout 超时时间（分钟）用于所有Apollo请求
-k, --insecure-skip-verify跳过服务器证书验证
-o, --output输出格式 (json,yaml,pretty)
--profile从配置文件中使用特定的配置文件
--quiet不要将日志输出到标准错误
--space-id用于某些作用域为空间的命令的空间ID
