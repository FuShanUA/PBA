了解 Spark 详细信息

Apache Spark ↗ 是 Foundry 数据集成层中最常用的执行引擎。为了了解您的管道性能特征并找到优化管道的方法，理解您的代码在 Spark 中的执行细节非常重要。Foundry 提供了集成工具，帮助您查看和理解您的作业在 Spark 中的性能。本页面概述了可用的 Spark 详细信息，并提供了这些详细信息含义的指导。

获取 Spark 详细信息

对于任何在 Foundry 中构建的数据集，请按照以下步骤查看 Spark 详细信息：

  1. 查看构建报告。
    • 从 数据集预览 或数据血缘关系中，选择 历史记录 选项卡，从列表中选择一个单独的构建，然后选择查看构建报告。
    • 从 所有构建 的视图中，只需在列表中选择一个构建。
  2. 选择一个作业。一个构建包含一个或多个作业，在甘特图下方的列表中可见。从列表中选择一个作业，然后选择 Spark 详细信息 按钮：

Spark 详细信息页面提供有关在 Spark 中执行作业的信息。对于每个作业，Spark 详细信息页面会显示如下所示的各种类别的信息：

概览选项卡

概览选项卡提供以下关于作业的信息：

  • 高级作业指标
  • 阶段执行时间线和阶段间依赖关系
  • 任务并发图
  • 阶段详情

高级作业指标

  • 所有任务的总运行时间： 所有阶段中所有任务运行时间的总和
  • 作业持续时间： Spark 计算的持续时间（第一个阶段开始到最后一个阶段完成之间的时间）

通过这两个指标，您可以计算并行比率为

Total runtime across all tasks / Job duration

接近1的比率表示并行性较差。

  • 磁盘溢出： 所有阶段中从执行器的RAM移动到磁盘的数据大小。

    • 当数据无法放入执行器的内存中时就会发生这种情况。向磁盘写入和读取是缓慢的操作，因此如果您的作业发生溢出，其运行速度将显著变慢。偶尔，根据正在进行的计算类型，溢出可能导致执行器内存不足，从而使作业失败。
    • 请注意，对于大型数据集，磁盘溢出是预期的。

  • Shuffle 写入： 作业期间在所有阶段中已进行 shuffle 的数据量。

    • Shuffling 是数据在 Spark 阶段之间以及分区之间移动的过程；例如，为了计算连接（当没有表被广播时）、执行聚合或应用重新分区。
    • 由于 shuffle 会导致网络 I/O 和磁盘 I/O，它可能占作业运行时间的很大一部分。
    • 因此，编写高性能Spark作业的关键目标之一是尽量减少数据混洗；例如，通过确保可以广播的连接实际上被广播，利用分桶技术对可能在下游作业中经常基于相同键进行连接或聚合的数据集（以避免该数据集的下游数据混洗），或者避免不必要的重新分区步骤。

执行器标签页

执行器 标签页会从 Spark 作业的驱动程序或执行器中捕获某些指标，包括堆栈跟踪和内存直方图。当调试 Spark 作业性能问题时，这些指标很有用。

选择 快照 按钮会从正在运行的作业中捕获 Java 堆栈跟踪或仅限于驱动程序的内存直方图。作业必须处于运行状态（如果作业已完成，这些指标将不再可用）。

堆栈跟踪是查看您的 Spark 任务每个线程当前正在执行的内容的一种方式。例如，如果一个任务似乎卡住了（即没有按预期推进），获取堆栈跟踪可能会揭示当时正在执行的内容。

内存直方图显示了当前在堆上的 Java 对象数量及其在内存中的大小（以字节为单位）。这有助于理解内存的使用情况以及在调试与内存相关的问题时有所帮助。

请注意，获取指标可能会影响运行中任务的性能。收集这些指标是 JVM 需要完成的额外工作。例如，获取内存直方图会触发垃圾回收。
