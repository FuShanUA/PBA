了解 Spark 详情

Apache Spark ↗ 是 Foundry 数据集成层中最常用的执行引擎。要了解流水线的性能特征并寻找优化途径，关键在于弄清楚代码在 Spark 中的具体执行细节。Foundry 提供了集成工具，帮助用户查看和掌握作业在 Spark 中的性能表现。本页概述了可用的 Spark 详情，并针对这些指标的含义提供了解读指南。

查看 Spark 详情

对于在 Foundry 中构建的任何数据集，请按照以下步骤查看 Spark 详情：

  1. 查看构建报告。
    • 在《数据集预览》或数据血缘关系中，选择“历史记录”选项卡，在列表中选中某个具体的构建，然后选择“查看构建报告”。
    • 在“所有构建”视图中，直接在列表中选择一个构建即可。
  2. 选择作业。一个构建包含一个或多个作业，这些作业显示在甘特图下方的列表中。从列表中选择一个作业，然后点击“Spark 详情”按钮：

概览选项卡

“概览”选项卡提供有关作业的以下信息：

  • 作业宏观指标
  • 阶段执行时间线与阶段间依赖关系
  • 任务并发图表
  • 阶段详情

作业宏观指标

  • 所有任务的总运行时间： 所有阶段中全部任务运行时间之和
  • 作业耗时： Spark 计算的持续时间（即从第一个阶段开始到最后一个阶段完成的时间）

利用这两个指标，可以按以下公式计算并行度：

所有任务的总运行时间 / 作业耗时

该比值接近 1 说明并行度较差。

  • 磁盘溢写： 所有阶段中，从执行器内存写入磁盘的数据量大小。

    • 当数据量超出执行器内存容量时就会发生这种情况。磁盘的读写操作非常耗时，因此一旦作业发生溢写，运行速度会显著下降。在某些情况下（具体取决于计算类型），溢写还可能导致执行器内存耗尽，进而导致作业失败。
    • 需要注意的是，对于大型数据集，出现磁盘溢写属于正常现象。

  • Shuffle 写入量： 作业在所有阶段中进行 Shuffle 的数据量。

    • Shuffle 是指数据在 Spark 各阶段及各分区之间传输的过程；例如，为了计算关联（在没有对表进行广播的情况下）、执行聚合操作或进行重新分区，都会触发 Shuffle。
    • 由于 Shuffle 会引发网络 I/O 和磁盘 I/O，因此它可能占据作业运行时间的很大一部分。
    • 正因如此，编写高效 Spark 作业的核心目标之一就是尽可能减少 Shuffle；例如，确保可广播的关联操作确实进行了广播；针对在下游作业中经常按相同键进行关联或聚合的数据集利用分桶机制（从而避免该数据集在下游发生 Shuffle）；或者避免不必要的重新分区步骤。

任务并发图表

任务并发图表有助于了解资源的使用情况。它绘制了阶段并发度随时间变化的曲线。与作业并发度类似，阶段并发度的计算公式为：

阶段内任务的总运行时间 / 阶段持续时间

执行器选项卡

执行器选项卡会从 Spark 作业的驱动器或执行器中捕获某些指标，包括堆栈跟踪和内存直方图。在调试 Spark 作业的性能问题时，这些指标非常有用。

选择快照按钮可以从正在运行的作业中捕获 Java 堆栈跟踪或仅限驱动器的内存直方图。作业必须处于运行状态（如果作业已完成，则无法再收集这些指标）。

堆栈跟踪是一种查看您的 Spark 作业中每个线程在那一刻正在执行什么操作的方法。例如，如果作业似乎卡住了（即在预期内没有取得进展），获取堆栈跟踪可能会揭示当时正在执行的内容。

内存直方图显示了当前堆上 Java 对象的数量及其在内存中的大小（以字节为单位）。这有助于了解内存的使用情况，并在调试与内存相关的问题时发挥作用。

需要注意的是，采集指标可能会影响正在运行的任务的性能。收集这些指标是 JVM 需要执行的额外工作。例如，获取内存直方图会触发垃圾回收。
