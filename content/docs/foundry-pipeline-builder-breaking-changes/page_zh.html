破坏性变更

在流式或增量管道中修改有状态函数时，会引发破坏性变更。数据转换分为行级转换和有状态转换两类。

  • 行级转换：仅需单行数据即可生成结果，例如 数字相乘 或 过滤。
  • 有状态函数：需要跨多行数据才能生成结果的转换操作。

主要有四种有状态函数：

  • 聚合（流式处理中的窗口聚合）
  • 外缓存连接（仅限流式处理）
  • 心跳检测（仅限流式处理）
  • 限时去重（仅限流式处理）
  • 限时事件时间排序（仅限流式处理）

修改有状态函数后，先前输出的结果可能不再准确。举例来说，假设你先过滤出偶数，然后计算其总和。如果此时将过滤条件改为奇数，现有状态仍是偶数之和，但所有新过滤出的值都是奇数。因此，求和结果的含义现在变得不明确，实际上是偶数集合与奇数集合相加的总和。要刷新状态，可以执行重放。

重放分为两种类型：

  • 从输入数据起点重放：从数据起点重新运行管道。具体起点取决于输入类型是流还是增量数据集，前者为流的起点，后者为输入数据集上的首个事务。

  • 从指定时间前重放（仅限流式处理）：使用从指定时间点开始的上游数据重新运行管道。细粒度重放将包含在指定时间之前提交的首个事务及之后的所有数据，此前的数据将不再处理。这意味着你可能会获取到指定时间之前某一个事务的数据量。

重放可以是可选的，也可以是必需的；遇到破坏性变更时，管道构建器会自动检测到该变更，并要求在部署时进行重放。下图展示了增量管道中的强制重放场景。

对管道进行重放可能导致较长的停机时间，有时甚至长达数天。执行重放时，流历史记录将会丢失，且所有下游管道消费者也必须同步进行重放。

状态中断的常见原因

以下变更通常会导致状态中断，进而需要进行全量重放。了解这些场景有助于你更好地规划管道修改，避免意料之外的停机。

有状态转换的修改

移除或修改有状态转换（如聚合、外缓存连接或心跳检测）需要进行全量重放。这包括移除输入到有状态转换的数据源，因为缺失了这些数据后，有状态函数将无法再产出一致的结果。

影响有状态转换的输入变更

添加或移除输入会触发状态中断，你可以直接确认该中断而无需进行全量重放。但是，对输入到有状态转换的数据源进行修改，可能会破坏其与历史数据的逻辑一致性。例如，如果某个有状态聚合依赖特定的输入，而该输入被移除或替换，那么现有状态将无法再准确反映管道正在处理的数据。

即使管道构建器允许你在不重放的情况下确认输入变更，也应评估该变更是否会影响管道后续的任何有状态转换。如果确有影响，仍需执行重放以维持数据一致性。

输出模式变更

向输出模式中添加新列无需重放。然而，从输出模式中移除列属于一种状态中断，需要进行重放，因为已有的输出数据包含的列已与更新后的模式不再匹配。

源管道变更

对向当前管道进行数据输入的上游管道进行重放，可能也会要求你对当前管道进行重放。当源数据被重新处理时，到达管道输入端的数据可能与管道最初处理的数据有所不同，这会使有状态转换的当前状态失效。

保留状态的修改

管道构建器包含允许在不重放的情况下进行某些管道修改的功能。这些功能使您能够从中断处继续处理，保留流历史记录，并避免对下游消费者产生影响。

输入与输出修改

您可以在部署后修改管道的输入和输出。其行为取决于变更的类型：

  • 添加输入：新输入将从头开始读取，而现有输入则从其上次处理的位置继续读取。
  • 移除输入或输出：与被移除的输入或输出相关联的状态将从处理集群中丢弃，无需进行重放。
  • 添加输出：添加输出会导致状态中断，需要进行重放，重放范围可以是过去一段时间的数据，也可以是从输入数据的起始点开始。

当管道构建器检测到输入或输出变更时，状态中断模块会提示您确认该变更。此确认操作会告知系统从中断处继续处理，而不是要求进行重放。

如果您移除的输入或输出位于作业组内，则确认选项将不可用，并需要进行重放，重放范围可以是过去一段时间的数据，也可以是从输入数据的起始点开始。请评估对输入或输出的任何更改是否会影响管道后续的有状态转换，因为为了保持数据一致性，可能仍需要进行重放。

模式变更

部署管道时，输入模式会被锁定。如果输入模式发生变更，管道将继续使用先前的模式读取数据，直到您手动重新部署。

对于输出模式，添加新列无需重放。然而，从输出模式中移除列属于一种状态中断，需要进行重放。

选择性数据重新摄入

您可以从特定时间点重新摄入数据，而无需重置输出视图。当您选择重新摄入时，重新摄入时输出中存在的所有数据都将被保留，这允许您在重新处理历史数据的同时保持现有的输出状态。

要配置此行为，请在部署面板中展开高级部分，并在重放管道时禁用重放时重置输出选项。

强制输出的增量行为

您可以使用管道构建器中的要求增量执行设置，在管道中强制执行增量执行。

此设置确保配置为增量运行的作业在无法进行增量执行时会自动失败。这有助于防止意外的快照场景，例如：

  • 意外快照的输入
  • 由于输出模式变更导致的强制快照
  • 其他意外的全量刷新

请按照以下步骤为您的管道配置强制增量执行：

  1. 在管道构建器中打开您的管道。
  2. 选择“部署”按钮右侧或右侧工具栏中的构建设置滑块图标。
  3. 向下滚动至高级配置。
  4. 将要求增量执行设置为True。此设置默认禁用（设置为无值）。

关键注意事项

  • 启用该功能后，如果管道中的所有增量输出无法以增量方式运行，无论作业如何分组，这些输出都会失败。
  • 仅当管道至少包含一个增量输入或输出时，才能将要求增量执行设置设为 True。否则，启用此选项将导致部署错误。
  • 配置为要求增量执行的管道，只有在管道变更需要状态中断时，才能以快照方式运行。运行前系统会提示您确认此操作。

在 PySpark 增量转换中，通过在 @incremental 装饰器中设置 require_incremental=True，也可使用此功能。
