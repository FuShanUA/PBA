破坏性变更

当流式或增量管道中的有状态函数被修改时，会发生破坏性变更。转换可以是行级的或有状态的。

  • 行级转换： 只需要单行数据来生成结果，例如 Multiply numbers 或 Filter。
  • 有状态函数： 一种需要跨多行数据才能生成结果的转换。

主要有四种有状态函数：

  • 聚合（在流处理中为窗口聚合）
  • 外缓存连接（仅在流处理中）
  • 心跳检测（仅在流处理中）
  • 时间限制去重（仅在流处理中）
  • 时间限制事件时间排序（仅在流处理中）

当有状态函数被修改时，之前的输出可能不再准确。例如，假设你正在过滤偶数并计算该集合的总和。如果你将过滤器更改为所有奇数，现有的状态将是偶数的总和，但所有新的过滤值都将是奇数。因此，总和所代表的内容现在变得模糊，它可能是偶数集合的总和加上奇数集合的总和。为了刷新状态，你可以运行一个重放。

有两种类型的重放：

  • 从输入数据开始处重放：从数据的开始处重新执行你的管道，无论是流的开始还是输入数据集的第一个事务（根据输入是流还是增量数据集来确定）。

  • 从一定时间之前重放（仅适用于流）： 使用上游数据从指定时间开始重放管道。粒度重放将包括在指定时间之前提交的第一个事务开始的所有数据，该时间之前的数据将不会被处理。这意味着您可能会获得指定时间之前的一个事务的数据。

重放可以是可选的或必需的；在存在破坏性更改的情况下，Pipeline Builder 会自动检测此更改并在部署时要求进行重放。下图显示了增量管道中的强制重放。

重放您的管道可能导致长时间的停机，可能长达数天。当您重放管道时，您的流历史记录将会丢失，所有下游管道消费者都必须重新重放。

状态中断的常见原因

以下更改通常会导致需要完全重新播放的状态中断。了解这些情况可以帮助您规划管道修改并避免意外停机。

有状态转换修改

删除或修改有状态转换（如聚合、外部缓存连接或心跳检测）需要完全重新播放。这包括删除输入数据，因为没有这些数据，有状态函数无法再产生一致的结果。

影响有状态转换的输入更改

添加或删除输入会触发一个状态中断，您可以无需完整重放即可确认。然而，对进入有状态转换的输入进行更改可能会与历史数据破坏逻辑一致性。例如，如果一个有状态的聚合依赖于特定的输入，而该输入被删除或替换，则现有状态将不再准确反映管道处理的数据。

即使Pipeline Builder允许您在不重放的情况下确认输入更改，也应评估该更改是否会影响管道后续的任何有状态转换。如果会，可能仍需要重放以保持数据一致性。

输出模式变更

向输出模式中添加新列不需要重放。但是从输出模式中删除列会导致状态中断，需要重放，因为现有输出数据包含的列不再与更新后的模式匹配。

源管道更改

重新播放一个为当前管道提供数据的管道可能需要你重新播放你的管道。当源数据被重新处理时，到达你的管道输入的数据可能与你的管道最初处理的数据不同，这可能导致有状态转换的当前状态无效。

保持状态的修改

Pipeline Builder 包含一些功能，允许在不重新播放的情况下进行某些管道修改。这些功能使你可以从你离开的地方继续处理，保留你的流历史记录，并避免对下游消费者造成影响。

输入和输出修改

您可以 在部署后修改管道的输入和输出。行为取决于更改的类型：

  • 添加输入： 新输入将从开始读取，而现有输入将继续从其上次处理的位置继续。
  • 删除输入或输出： 与删除的输入或输出相关联的状态会从处理集群中删除，而无需重新播放。
  • 添加输出： 添加输出会导致状态中断，需要重新播放，可以从一段时间前或输入数据的开始处重新播放。

当Pipeline Builder检测到输入或输出更改时，状态中断模块会提示您确认更改。此确认告诉系统从它停止的地方继续处理，而不是要求重新播放。

如果你在一个作业组内删除了输入或输出，那么“确认”选项将不可用，并且需要重新播放数据，可以是从一段时间前开始，也可以是从输入数据的开始。评估对输入或输出所做的任何更改是否会影响管道中后续的有状态转换，因为可能仍需要重新播放以保持数据一致性。

模式变更

当您部署管道时，输入模式会被固定。如果输入模式发生变更，管道将继续使用之前的模式读取数据，直到您手动重新部署。

对于输出模式，添加新列不需要重新播放。然而，从输出模式中删除列会导致状态中断，需要重新播放。

选择性数据重新摄入

您可以从特定时间点重新摄入数据，而无需重置输出视图。当您选择重新摄入时，重新摄入时输出中的所有数据都会被保留，这允许您重新处理历史数据，同时保持您现有的输出状态。

要配置此行为，请在部署面板中展开高级部分，并在重新播放您的管道时禁用重新播放时重置输出选项。

强制输出的增量行为

您可以使用Pipeline Builder中的要求增量执行设置来在您的管道中强制增量执行。

此设置确保配置为按增量运行的任务如果无法进行增量执行将自动失败。这有助于防止意外快照场景，例如：

  • 意外快照的输入
  • 由于输出模式更改导致的强制快照
  • 其他意外的完全刷新

按照以下步骤为您的管道配置强制增量执行：

  1. 在Pipeline Builder中打开您的管道。
  2. 点击部署按钮右侧或右侧工具栏中的构建设置滑块图标。
  3. 向下滚动到高级配置。
  4. 将要求增量执行设置为True. 此设置默认情况下是禁用的（设置为No value）。

关键注意事项

  • 启用后，如果无法进行增量运行，您管道中的所有增量输出都将失败，无论作业分组如何。
  • 要求增量执行 设置只能在您的管道至少有一个增量输入或输出时设置为True。如果没有，启用此选项将导致部署错误。
  • 只有当管道的更改需要状态中断时，配置为需要增量执行的管道才能作为快照运行。在运行之前您将被要求确认这一点。

通过在require_incremental=True装饰器中设置@incremental，此功能也可在 PySpark 增量转换中使用。
