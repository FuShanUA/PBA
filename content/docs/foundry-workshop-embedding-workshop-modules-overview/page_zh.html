嵌入 Workshop 模块

Workshop 提供了两项支持嵌入其他 Workshop 模块的功能：Embedded Module 小部件和 Loop 节区布局（“循环布局”）。

利用这些功能，可以构建能够嵌入到其他 Workshop 模块内部的 Workshop 模块。嵌入模块支持 Workshop 提供的所有功能，但在嵌入模块与被嵌入模块之间的交互上存在一些限制。本文档在表述时，会将嵌入其他模块的模块称为“父”模块，将被嵌入的模块称为“子”模块。除另有说明外，本文档中提及的“嵌入模块”同时适用于 Embedded Module 小部件以及嵌入在 Loop 布局中的模块。

使用场景

嵌入模块有多种使用场景，包括：

  • 跨多个模块复用
  • 在单个模块内复用
  • 可维护性
  • Marketplace 定制
  • 原地替换 Workshop 模块

跨多个模块复用

嵌入模块允许对单个子模块进行配置，并在多个父模块中复用。可构建的嵌入模块类型示例包括：

  • 已配置的对象视图
  • 具有复杂配置的单个小部件
  • 由复杂变量逻辑支撑的一组小部件
  • 完全不包含小部件的复杂变量逻辑

在单个模块内复用

通过循环布局或单独的嵌入模块小部件，嵌入模块可以在单个父模块中被多次使用。在同一个父模块中会被多次嵌入的子模块示例包括：

  • 基于提供的变量派生的自定义卡片视图
  • 在每个标签页或页面上各显示一次的一组小部件

可维护性

利用嵌入模块，可以将较大的父模块拆分为多个子模块，从而提升可维护性。这使得每个子模块都能拥有独立的变量作用域，并且不同的 Workshop 构建者可以同时编辑各个子模块。

Marketplace 定制

在通过 Marketplace 打包 Workshop 模块时，嵌入模块可用于提供定制区域。嵌入的子模块将作为依赖项打包，在安装时，Marketplace 用户可以选择具有相同模块接口变量类型签名的模块，以定制已安装父模块的一部分。

原地替换 Workshop 模块

无需更改用户正在使用的 Workshop 资源，嵌入模块即可用于部署新的 Workshop 模块实现。实现方式是将嵌入模块作为该模块中的唯一小部件。这样一来，即便对模块进行了大幅改动，用户也无需迁移到新的 Workshop 资源。

跨嵌入模块通信

使用模块接口变量在父模块与子模块之间或同级模块之间进行通信。例如，这些共享接口变量可以支撑共享状态，如选中的对象、选中的标签页或是否显示覆盖层。嵌入模块可以通过事件修改接口变量的值，从而让引用这些变量的其他位置根据更新后的值做出响应。

Workshop 始终使用父模块的变量定义，并忽略嵌入模块的接口变量定义。嵌入模块接口配置文档提供了有关此行为的更多信息。

在编辑模式下打开嵌入的子模块时（例如，通过嵌入式模块组件中的打开引用模块，或通过循环实例），子模块编辑器会打开并应用所有已映射模块接口变量的当前值。这与查看模式下的行为一致，便于在与父模块相同的状态下调试嵌入式模块。

限制

子模块不继承模块设置

子模块在模块级别的配置（如路由、状态保存和自动刷新）不会被使用。这意味着必须为每个打算使用自动刷新的模块显式配置自动刷新，且自动刷新不会在嵌入式模块的上下文中单独生效。要在不自动刷新的父模块中嵌入一个会自动刷新的模块，可以使用iFrame 组件来嵌入一个作为沙盒环境且会自动刷新的独立模块。你可以设置 URL 查询参数 embedded=true 来隐藏 Foundry 侧边栏。

如果工作流有此要求，可以在父模块级别配置这些设置，并通过模块接口将变量传递给子模块。

事件传递

嵌入式模块不支持事件传递的概念。事件传递允许构建者将事件配置从父模块传递给子模块，使子模块能够调用引用父模块配置的事件（例如，子模块调用一个在父模块中打开覆盖层的事件）。

要跨嵌入式模块通信，父模块可以将支持布局状态的变量传递给子模块的模块接口变量，作为子模块修改父模块布局状态的一种方式。

嵌入式模块溯源

与 Workshop 中的其他“Foundry 应用”组件类似，数据血缘关系不会报告其他 Workshop 模块所使用的嵌入式模块的溯源。

自引用嵌入式模块

模块不能直接或通过子模块链嵌入自身。如果配置了自引用，模块会向构建者显示警告，并且不会向查看者渲染任何内容，以防止可能出现的嵌入式模块无限链。如果您的用例需要递归或自引用嵌入式模块，请联系您的 Palantir 代表。

嵌入式模块的权限

嵌入式模块是拥有自身权限设置的独立资源。如果用户有权查看嵌入了子模块的父模块，但缺乏查看该子模块的权限，用户将看到“模块加载失败”错误。

性能

嵌入式模块的性能特征应与普通 Workshop 模块相似。一个显著的区别是，嵌入式模块的初始化会延迟到该模块在视图中显示时才进行。一旦初始化，嵌入式模块的运行效果就与整个模块在同一位置配置好一样。

请注意，子模块配置会按实例初始化一次，这伴随着初始化其变量的开销；但如果在多个实例中重用，子模块配置只需加载一次。

使用嵌入式模块和循环布局，让构建者配置非常庞大、复杂的模块变得更加容易。构建者应注意同时显示的组件和变量的总数，特别是那些加载开销较大的组件和变量。

移动端模式

当模块处于移动端模式时，嵌入式模块组件和循环布局只能嵌入其他移动端模块。
