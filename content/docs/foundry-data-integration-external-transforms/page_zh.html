外部转换

《数据连接源》现在可以直接导入代码仓库，这是与外部系统交互的首选方法，已取代《基于凭据的旧版外部转换》。

外部转换允许从《Python 转换》仓库连接到外部系统。

外部转换不支持使用《出站应用》进行身份验证的源。

当满足以下任一条件时，通常会使用外部转换来执行《批处理同步》、《导出》和《媒体同步》工作流：

  • 不存在可用的现有数据连接源类型。
  • 目标源类型不具备所需的功能。
  • 数据连接用户界面提供的功能不具备所需特性。

针对这些情况的解决方案可能包括以下几种：

  • 连接 REST API，包括通过互联网和私有网络进行连接。
  • 连接数据库，以安排数据连接用户界面当前无法实现的自定义查询逻辑。
  • 在同步或导出过程中根据需要转换数据。这可能包括在写入 Foundry 之前将文件打包在一起、在传输过程中处理数据的自定义加密/解密等操作。

任何使用《虚拟表》的转换也会被视为外部转换，因为转换作业必须能够访问包含虚拟化数据的外部系统。要在 Python 转换中使用《虚拟表》，请按照下文说明了解如何设置源。

本设置指南中使用的词典 API 与 Palantir 无关，且可能随时发生变更。本教程并非认可、推荐或建议将该 API 用于生产环境用例。

示例：从 Dictionary API 导入数据

以下示例展示了一个完整的转换过程，该过程遍历一系列单词，并从 Dictionary API 获取其注音符号。


```python
from pandas import DataFrame
from transforms.api import transform
from transforms.api import Output, Input, TransformContext, transform_pandas
from transforms.external.systems import external_systems, Source
import pandas as pd
import logging

logger = logging.getLogger(__name__)


@external_systems(
    dictionary_api_source=Source(
        ""
    )
)
@transform_pandas(Output(""))
def compute(dictionary_api_source) -> DataFrame:
    dictionary_api_url = dictionary_api_source.get_https_connection().url
    dictionary_api_client = dictionary_api_source.get_https_connection().get_client()

    words = ["apple", "dog", "cat"]

    phonetics = []

    for word in words:
        logger.info("Fetching word from api.dictionaryapi.dev : " + word)

        response = dictionary_api_client.get(
            dictionary_api_url + "/api/v2/entries/en/" + word
        ).json()

        phonetics += [{"word": word, "phonetic": response[0]["phonetic"]}]

    return pd.DataFrame(phonetics)
```


在外部转换中使用 Foundry 输入


外部转换通常需要使用 Foundry 的输入数据。例如，你可能需要查询 API 来为表格数据集中的每一行收集额外的元数据。或者，你的工作流可能需要将 Foundry 数据导出到外部软件系统中。


此类情况被视为受出口管控的工作流，因为它们有可能将安全的 Foundry 数据导出到安全保证未知且数据血缘关系断裂的系统中。在配置源连接时，数据主人必须明确是否允许导出 Foundry 数据，并提供可导出的安全标记集合和组织列表。Foundry 提供了数据治理管控措施，确保开发者能够清晰编码安全意图，同时信息安全官可以审计与外部系统交互的工作流的作用范围和意图。


在数据源上配置出口管控


导出操作通过安全标记进行管控。配置数据源时，导出配置用于指定哪些安全标记和组织可以安全地导出到外部系统。此操作需在数据连接应用中打开相应的数据源，然后进入连接设置 > 导出配置选项卡。接着，你需要开启启用导出到此数据源选项，并选择可能需要导出的标记集合和组织。


执行此操作需要具备移除相关数据和组织标记的权限，因为在 Foundry 中，导出操作等同于移除数据上的标记。


必须开启启用导出到此数据源设置，才能允许以下操作：


    

  • 在导入此数据源的 Python 转换代码中，将数据集、媒体集和流作为输入使用。
    • 

  • 在 Python 转换中使用在此数据源上注册的虚拟表。
    •

外部转换与传统外部转换的对比


外部转换与传统外部转换在以下几个方面存在关键的工作流差异：


    

  • 对于外部转换，导入数据源的选项卡会始终自动显示。此前，只有在信息安全官在仓库设置中开启使用外部系统的功能后，才会显示添加出口策略和凭证的选项卡。
    • 

  • 允许外部连接和使用输入的设置不再位于仓库设置中。相反，这些设置由每个单独的数据源控制。
    • 

  • 凭证、出口策略和可导出标记不再在代码中指定。相反，这些设置取自导入到转换中的数据源，并自动应用于作业。

      

    • 如果此配置在数据源级别发生更改，导入该数据源的转换会自动获取更改，无需任何代码修改或版本升级。这实现了凭证、出口和可导出标记的集中式治理，并能立即传递到下游工作流。
      • 

    • 更改将在构建开始时生效，不会影响正在运行的构建。
      • 

    

    • 

  • 装饰器已从 @use_external_systems() 更改为 @external_systems()。
    • 



外部转换的主要优势包括：


    

  • 支持连接无法从互联网访问的系统
    • 

  • 支持轮换或更新凭证而无需修改代码
    • 

  • 支持在多个仓库间共享连接配置
    • 

  • 为选定的数据源类型提供开箱即用的 Python 客户端
    • 

  • 改进并简化了用于启用和管理外部转换仓库的治理工作流
    • 

  • 在《数据血缘关系》中可视化连接到外部数据源的外部转换
    • 



迁移至外部转换


目前没有将传统外部转换更新为外部转换的自动迁移路径。不过，对于大多数工作流而言，所需的手动操作预计极少。


手动迁移至外部转换的主要步骤如下：


    

  1. 确定现有传统外部转换代码中使用的凭证、出口策略和出口控制标记集合。
    2. 

  2. 识别或配置连接到您希望从外部转换连接的系统的数据连接数据源。确保这些数据源已配置为允许导入代码。
    3. 

  3. 将步骤 2 中的相关数据源导入到现有的 Python 转换仓库中。
    4. 

  4. 修改代码，导入并使用带有数据源引用的新 @external_systems() 装饰器，然后移除所有 @use_external_systems() 装饰器实例。这可能涉及更新转换逻辑中对凭证的引用，改为引用从当前导入的数据源中检索到的凭证。
    5. 

  5. 在分支上测试更改，确保转换能够继续成功构建。
    6. 

  6. 合并更新后的转换代码后，您现在可以取消切换仓库设置。
    7. 



转换不能同时包含外部转换及其传统版本。为解决此问题，您可以将所有传统外部转换迁移为使用基于数据源的外部转换（首选），或者将转换拆分为多个转换。转换可以拆分为一个使用 use_external_systems 装饰器的转换，和另一个使用 external_systems 装饰器的转换。

高级模式


通过感知内存的缓冲机制将数据写入 Parquet 文件


当外部转换从外部源获取大量记录时，在写入前将所有记录保留在内存中可能会导致内存不足错误。下面的 BufferedParquetWriter 类会在超过可配置的内存阈值后，定期将累积的行数据刷新到 Parquet 文件中。该模式适用于任何外部数据源，并使用 filesystem().open() 写入原始文件，使用 put_metadata() 完成模式推断。



```python
import gc
import logging
from datetime import datetime

import pyarrow as pa
import pyarrow.parquet as pq
from pympler import asizeof

from transforms.api import Output, transform, LightweightOutput
from transforms.external.systems import external_systems, Source, ResolvedSource

logger = logging.getLogger(__name__)


class BufferedParquetWriter:
    """在内存中缓冲数据行，当缓冲区超过可配置的大小阈值时，将其刷新到 Parquet 文件。"""

    def __init__(self, output: LightweightOutput, schema: pa.Schema, flush_threshold_mb: int = 64):
        self.output = output
        self.schema = schema
        self.flush_threshold_bytes = flush_threshold_mb * 1024 * 1024
        self.buffer = []
        self.buffer_size_bytes = 0
        self.file_index = 0
        self.row_size_estimate = None

    def append(self, row: dict):
        """将一行数据添加到缓冲区。如果缓冲区超过阈值，则刷新到磁盘。"""
        self.buffer.append(row)

        # 从第一行估算行大小，然后后续均使用该估算值
        if self.row_size_estimate is None:
            self.row_size_estimate = asizeof.asizeof(row)
        self.buffer_size_bytes += self.row_size_estimate

        if self.buffer_size_bytes >= self.flush_threshold_bytes:
            self.flush()

    def flush(self):
        """将缓冲的数据行写入 Parquet 文件并清空缓冲区。"""
        if not self.buffer:
            return
        self.file_index += 1
        table = pa.Table.from_pylist(self.buffer, schema=self.schema)
        timestamp = datetime.now().strftime("%Y%m%d_%H%M%S")
        filename = f"output_{timestamp}_{self.file_index}.parquet"
        with self.output.filesystem().open(filename, "wb") as f:
            pq.write_table(table, f)
        logger.info(f"已将 {len(self.buffer)} 行刷新到 {filename}")
        self.buffer = []
        self.buffer_size_bytes = 0
        gc.collect()

    def finalize(self):
        """刷新所有剩余的数据行并触发模式推断。"""
        self.flush()
        if self.file_index > 0:
            self.output.put_metadata()


# 定义输出记录的模式
OUTPUT_SCHEMA = pa.schema([
    ("id", pa.string()),
    ("name", pa.string()),
    ("value", pa.float64()),
])


@external_systems(
    my_source=Source("")
)
@transform.using(
    output=Output(""),
).with_resources(
    memory_gb=4,
)
def compute(my_source: ResolvedSource, output: LightweightOutput):
    output.set_mode("replace")
    writer = BufferedParquetWriter(output, OUTPUT_SCHEMA, flush_threshold_mb=64)

    # 将下方的循环替换为您的数据获取逻辑。
    # 例如，遍历 API 的分页数据、数据库游标的记录，
    # 或消息队列中的消息。
    # 每次调用 append() 都会向写入器添加数据，
    # 当缓冲区超过阈值时，会自动刷新到 Parquet 文件。
    for record in fetch_records_from_source(my_source):
        writer.append({
            "id": record["id"],
            "name": record["name"],
            "value": record["value"],
        })

    # 写入所有剩余的缓冲数据行并完成模式推断。
    writer.finalize()
```
