外部转换

数据连接源现在可以直接导入到代码仓库中，并且是与外部系统交互的首选方法，取代了 基于凭证的旧版外部转换

外部转换允许从 Python 转换 仓库连接到外部系统。

外部转换不支持使用 出站应用程序 进行身份验证的数据连接源。

当以下情况之一成立时，外部转换主要用于执行 批量同步、导出 和 媒体同步 工作流：

  • 现有的数据连接源类型不可用。
  • 目标源类型不支持所需的功能。
  • 通过数据连接用户界面提供的功能没有所需的特性。

这些情况的解决方案可能包括以下内容：

  • 连接到REST API，包括互联网和私有网络中的API。
  • 连接到数据库以安排在数据连接用户界面中目前无法实现的自定义查询逻辑。
  • 在同步或导出过程中按需转换数据。这可能包括在写入Foundry之前将文件分批处理、在传输过程中处理自定义的加密/解密数据等。

任何使用虚拟表的转换也被视为外部转换，因为转换任务必须能够访问包含虚拟化数据的外部系统。要在Python转换中使用虚拟表，请按照下面的说明了解如何设置源。

在外部转换中使用 Foundry 输入

外部转换通常需要使用Foundry输入数据。例如，您可能需要查询API以收集表格数据集中每行的附加元数据。或者，您可能有一个工作流，需要将Foundry数据导出到外部软件系统。

这些情况被视为出口管制的工作流，因为它们有可能将安全的Foundry数据导出到具有未知安全保证和断开的数据溯源的另一个系统。在配置源连接时，源所有者必须指定是否可以将来自Foundry的数据导出，并提供可以导出的安全标记和组织。Foundry提供了治理控制，以确保开发人员可以明确地编码安全意图，信息安全部门可以审核与外部系统交互的工作流的范围和意图。

在源上配置导出控制

导出通过 安全标记 进行控制 当配置数据源时，导出配置用于指定哪些安全标记和组织可以安全地导出到外部系统。这是通过在数据连接应用程序中导航到该数据源，然后导航到连接设置>导出配置选项卡来完成的。然后应打开启用对此数据源的导出选项，并选择可能被导出的标记和组织集合。

这样做需要对相关数据和组织移除标记的权限，因为将数据导出到外部系统被视为在Foundry中移除数据标记的行为。

必须打开启用对此数据源的导出设置，以允许以下操作：

  • 使用数据集、媒体集和流作为导入此数据源的Python转换代码的输入。
  • 在Python转换中使用在此数据源上注册的虚拟表。

9
10
11@external_systems(
12    dictionary_api_source=Source(

14    )
15)
16@transform_pandas(
17    Output("<output_dataset_rid>"),
18    words_df=Input("<input_dataset_rid>"),

20def compute(dictionary_api_source: ResolvedSource, words_df: DataFrame) -> DataFrame:
21    dictionary_api_url = dictionary_api_source.get_https_connection().url

23
24    words = words_df["word"].tolist()
25
26    phonetics= []
27
28    for word in words:
29        logger.info("Fetching word from api.dictionaryapi.dev: " + word)
30
31        response = dictionary_api_client.get(
32            dictionary_api_url + "/api/v2/entries/en/" + word
33        )
34
35        if response.status_code == 200:
36            data = response.json()[0]
37
38            if "phonetic" in data:
39                phonetic_transcription = data["phonetic"]
40            else:
41                logger.warning(f"No phonetic transcription found for {word}.")

43        else:

46
47        phonetics += [{"word": word, "phonetic": phonetic_transcription}]
48
49    return pd.DataFrame(phonetics)

外部转换与旧版外部转换的比较

以下是外部转换和旧版外部转换之间的一些关键工作流程差异：

  • 导入源的标签页对于外部转换始终会自动显示。以前，添加出口策略和凭据的标签页只有在信息安全部门人员在仓库设置中开启使用外部系统的功能后才会显示。
  • 允许外部连接和使用输入的设置不再位于仓库设置中。相反，这些设置由每个单独的源进行控制。
  • 凭据、出口策略和可导出的标记不再在代码中指定。相反，这些设置从导入到转换中的源中获取，并自动应用于作业。
    • 如果此配置在源级别发生更改，将被导入该源的转换自动识别，无需任何代码更改或版本提升。这允许对凭据、出站流量和可导出的标记进行集中管理，这些更改会立即传播到下游工作流。
    • 更改将在构建开始时生效，并且不会影响正在运行的构建。
  • 装饰器已从@use_external_systems()更改为@external_systems()。

外部转换的关键优势包括以下几点：

  • 支持连接到无法从互联网访问的系统
  • 支持旋转/更新凭据而不需要代码更改
  • 支持在多个仓库之间共享连接配置
  • 为选定的源类型提供开箱即用的 Python 客户端
  • 改进并简化了启用和管理外部转换仓库的治理工作流程
  • 在 数据谱系 中可视化连接到外部源的外部转换

迁移到外部转换

目前没有自动迁移路径来将外部转换更新为外部转换。但是，大多数工作流程所需的手动操作预计将非常少。

以下是要手动迁移到外部转换的主要步骤：

  1. 确定您现有的旧外部转换代码中使用的凭证、出口策略和出口控制标记的集合。
  2. 确定或配置连接到您希望从外部转换连接到的系统的数据连接源。确保这些源已配置为允许导入到代码中。
  3. 将步骤2中的相关源导入到您现有的Python转换仓库中。
  4. 更改您的代码以导入并使用带有源引用的新@external_systems()装饰器，然后删除任何@use_external_systems()装饰器的实例。这可能涉及将转换逻辑中对凭证的任何引用更新为引用现在导入的源中检索到的凭证。
  5. 在分支上测试您的更改，以确保您的转换继续成功构建。
  6. 合并更新后的转换代码后，您可以现在取消切换仓库设置。

转换不能同时包含外部转换和其旧版本。为解决此问题，您可以将所有旧的外部转换迁移到使用基于源的外部转换（推荐），或者将您的转换拆分为多个转换。转换可以拆分为一个使用use_external_systems装饰器的转换和另一个使用external_systems装饰器的转换。

使用内存感知的缓冲区将数据写入Parquet文件

当外部转换从外部源获取许多记录时，在写入之前将所有记录保留在内存中可能导致内存不足错误。下面的BufferedParquetWriter类在超过可配置的内存阈值后，会定期将累积的行刷新到Parquet文件中。该模式适用于任何外部数据源，并使用filesystem().open()来写入原始文件，以及使用put_metadata()来最终推断模式。

12logger = logging.getLogger(__name__)

15class BufferedParquetWriter:
16    """Buffers rows in memory and flushes them to Parquet files when the buffer
17    exceeds a configurable size threshold."""
18
19    def __init__(self, output: LightweightOutput, schema: pa.Schema, flush_threshold_mb: int = 64):
20        self.output = output
21        self.schema = schema
22        self.flush_threshold_bytes = flush_threshold_mb * 1024 * 1024
23        self.buffer = []
24        self.buffer_size_bytes = 0
25        self.file_index = 0
26        self.row_size_estimate = None
27
28    def append(self, row: dict):
29        """Add a row to the buffer. If the buffer exceeds the threshold, flush to disk."""
30        self.buffer.append(row)
31
32        # Estimate row size from the first row, then use that estimate going forward
33        if self.row_size_estimate is None:
34            self.row_size_estimate = asizeof.asizeof(row)
35        self.buffer_size_bytes += self.row_size_estimate
36
37        if self.buffer_size_bytes >= self.flush_threshold_bytes:
38            self.flush()
39
40    def flush(self):
41        """Write buffered rows to a Parquet file and clear the buffer."""
42        if not self.buffer:
43            return
44        self.file_index += 1
45        table = pa.Table.from_pylist(self.buffer, schema=self.schema)
46        timestamp = datetime.now().strftime("%Y%m%d_%H%M%S")
47        filename = f"output_{timestamp}_{self.file_index}.parquet"
48        with self.output.filesystem().open(filename, "wb") as f:
49            pq.write_table(table, f)
50        logger.info(f"Flushed {len(self.buffer)} rows to {filename}")
51        self.buffer = []
52        self.buffer_size_bytes = 0
53        gc.collect()
54
55    def finalize(self):
56        """Flush any remaining rows and trigger schema inference."""
57        self.flush()
58        if self.file_index > 0:
59            self.output.put_metadata()
60
