时间序列属性 (TSPs)

时间序列属性 (TSP) 是一种能够在 Foundry 应用中启用时间序列数据使用功能的属性。请查阅如何使用时间序列以获取更多详细信息。

查看时间序列属性时，其值将显示为关联时间序列值的图表。例如，下例展示了一个 temperature_id 时间序列属性，它是使用 Quiver 中的底层时间序列数据可视化的。

时间序列属性设置

如需以下步骤的引导式操作指南，请使用平台中的时间序列设置助手。直接访问 https://<domain>/workspace/ontology/home/overview/time-series-setup 即可启动设置助手；或者导航至包含时间戳列的数据集预览，并从分析数据操作菜单中选择设置时间序列。

有关设置时间序列对象类型的更多指导，请参阅关于如何创建或选择时间序列对象类型的文档。

时间序列属性在 Ontology Manager 应用的功能选项卡中进行配置。如果您正跟随设置助手的指引，在创建新对象类型或选择现有对象类型后，系统会自动将您引导至功能选项卡。如果该对象类型已配置了时间序列属性，您将在表视图中看到它们，您也可以在此添加或删除时间序列属性。

如果该对象类型上没有现有时间序列属性，系统将引导您完成添加操作。

要添加时间序列属性，请选择开始使用。这将启动一个对话框，引导您完成以下步骤：

  1. 选择属性：选择一个包含序列 ID 的 string 属性，然后选择下一步。

对象类型的主键属性不能被选为时间序列属性。

  1. 选择时间序列同步：如果已有时间序列同步，请选择一个；或者按照说明创建新的时间序列同步。您可以继续选择+选择文件来选择多个时间序列同步。如果您的 TSP 由多个时间序列同步提供支持，您将需要使用限定序列 ID。

  1. 确定对象类型：决定此对象类型是否为传感器对象类型。

如果您想添加或修改时间序列属性但已关闭设置助手，您可以通过以下任一方式恢复进度：

  • 在 Ontology Manager 中导航至您的对象类型，然后前往左侧面板的功能选项卡。
  • 直接访问 https://<domain>/workspace/ontology/home/overview/time-series-setup 重新启动设置助手，或者导航至包含 timestamp 列的数据集预览，并从分析数据操作菜单中选择设置时间序列。

时间序列格式化

时间序列格式化允许设置所需的内部插值和时间序列单位。像 Quiver 这样的应用将遵循所提供的插值和单位值。

单位和插值格式化可以指向该对象类型上的其他 string 属性，以实现更精细的控制（例如，如果时间序列属性中包含的每个时间序列具有不同的单位和/或插值）。如果不需要精细控制，插值和单位都有一组标准值可供选择。

传感器对象类型的内部插值和单位在传感器对象配置部分中进行配置。

时间序列格式化器中提供的单位主要用于可视化展示。例如，在 Quiver 中作为图表的坐标轴标签。

默认时间序列属性

一个对象类型可以指定一个时间序列属性作为默认时间序列属性。为对象类型配置首个时间序列属性时，该属性将被设为默认时间序列属性。

在部分应用中，默认时间序列属性会在无需用户额外干预的情况下展示。例如在 Quiver 中，对象属性时间序列卡片默认指向该默认时间序列属性，除非另行指定。

传感器对象类型的唯一时间序列属性必须是默认时间序列属性。请查阅如何设置传感器对象类型以了解详情。
