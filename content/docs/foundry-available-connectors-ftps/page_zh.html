FTP/FTPS

将Foundry连接到FTP和FTPS服务器，以在文件夹和Foundry数据集之间同步数据。

支持的功能

功能状态
探索🟢 通常可用
批量导入🟢 通常可用
增量🟢 通常可用

数据模型

连接器可以将任何类型的文件传输到Foundry数据集。在传输过程中或之后，文件格式将被保留，并且不会应用任何模式。对输出数据集应用任何必要的模式，或者编写一个下游转换以访问数据。

性能和限制

可传输文件的大小没有限制。然而，网络问题可能导致大规模传输失败。特别是，在Foundry工作进程中运行时间超过两天的同步操作将被中断。为了避免网络问题，我们建议使用较小的文件大小并限制每次同步执行中摄入的文件数量。同步可以 安排 为频繁运行。

设置

  1. 打开 数据连接 应用程序并选择 + 新数据源位于屏幕右上角。
  2. 从可用的连接器类型中选择FTP/FTPS。
  3. 按照后续配置提示继续使用以下部分中的信息设置您的连接器。

要访问本地 FTP 服务器，我们建议您 通过代理策略连接。

了解有关 设置连接器 的更多信息。

认证

FTP/FTPS 认证使用用户名和密码完成。如果 FTP 用户没有权限访问连接配置的根目录，FTP/FTPS 连接可能会失败。有关根目录的更多详情，请参阅 配置选项。

选项是否必需?描述
Username是 FTP 登录用户名。
Password是FTP 登录密码。此字段可以留空以进行匿名登录。有关更多信息，请联系您的服务器管理员。

网络

如果您在 Foundry 中运行 FTP/FTPS 连接，您必须添加一个 网络出站策略 以允许连接。

如果通过域名连接，则应在控制端口（通常为端口 21）和数据端口上为 FTP 服务器域名创建出站策略。我们建议创建两个网络策略：一个用于控制端口的单个端口策略，以及一个用于数据端口的端口范围策略。数据端口由 FTP 服务器的管理员确定。如果在正确配置出站策略后错误仍然存在，请 提交问题，并引用应用的策略列表。

如果服务器的域名解析到多个域名和/或服务器，所有相关的域名及其相关IP都需要加入白名单。要验证服务器是否解析到多个域名和/或服务器，请从终端运行 dig <domain> 命令，针对您尝试连接的服务器，并查看答案部分。

证书和私钥

使用以下指导配置额外的客户端或服务器证书和私钥，以正确设置您的连接器。

SSL 和主机名验证

SSL连接验证服务器证书。通常，SSL验证通过证书链进行；默认情况下，代理和Foundry工作者信任大多数行业标准的证书链。如果您连接的服务器具有自签名证书，或者防火墙对连接执行TLS拦截，则连接器必须信任该证书。了解有关< a href="/docs/foundry/data-connection/set-up-source/#optional-add-certificates">在数据连接中使用证书的更多信息。

为了使SSL验证正常工作，服务器必须提供完整的证书链。FTP服务器的证书链可以通过运行命令openssl s_client -connect {hostname}:{port} -showcerts -starttls ftp获取。要验证证书链，请使用OpenSSL命令行工具或任何其他可用工具。

如果使用FTPS，请确保已将FTPS服务器的证书添加到代理的信任存储中。

Foundry 会尝试对所有出站路由进行验证。然而，FTP 无法被检查，这会导致连接挂起和/或超时错误。如果在正确配置出站策略的情况下仍出现错误，请报告一个问题，并提供您希望禁用主机名验证的策略列表。

隐式/显式 SSL

FTP 服务器可以配置为支持显式或隐式 SSL。运行在端口 990 上的服务器通常使用隐式 SSL。

请与您的服务器管理员确认您的服务器设置。默认情况下，连接器假设是显式 SSL；您可能需要根据您的环境更改此设置。

连接要求

FTP 需要CONTROL 和 DATA 连接类型。DATA 连接必须配置为 ACTIVE 或 PASSIVE 模式。

  • 控制（CONTROL）: 客户端到服务器
  • 数据（DATA）: 从某个范围（例如，1024–1123）中选择的数据
    • 被动模式：客户端到服务器
    • 主动模式：服务器到客户端
      • 仅适用于代理连接

默认的FTP/FTPS连接端口：

  • FTP：21
  • FTPS 显式：21
  • FTPS 隐式：990

主动模式和被动模式

我们建议使用被动模式的网络连接。在被动模式下，所有连接都由客户端发起。使用被动模式时，请确保控制端口（通常为21）以及数据传输的端口范围（例如1024–1123）已被允许访问。请联系您的FTP/FTPS服务器管理员以获取连接详细信息。

主动模式是一种较旧的文件传输方法。在主动模式下，客户端连接到服务器，同时服务器也连接到客户端。服务器和客户端相互依赖，并且需要双向 网络连接。这种网络方法在大多数安全环境中通常难以实现，只有在使用代理连接时才有可能。

故障排除

代理连接

  • 您在设置代理连接时遇到问题了吗？安装一个FTP/S客户端，并尝试使用与源相同的配置连接到服务器。如果此连接失败，则不是连接器的错误。在提交问题之前，请检查网络连接、认证和FTP服务器配置。

  • 您是否使用了出站代理负载均衡器？FTP是一个有状态的协议，因此如果顺序请求不是来自同一IP地址，使用负载均衡器可能会导致同步失败（非确定性地）。

SSL 和 FTPS

  • 您的服务器是否使用自签名证书？您是否已将其添加到源信任库中？请参阅上面的 SSL 和主机名验证 部分。

  • 您的 FTP 服务器是否仅支持旧版 TLS 版本（例如，TLS 1.1）？如果是，连接器运行时可能不会接受服务器提供的任何密码套件。请提交一个问题，与 Palantir 代表探讨其他方案。
