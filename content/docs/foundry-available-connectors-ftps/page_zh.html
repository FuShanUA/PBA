FTP/FTPS

将 Foundry 连接到 FTP 和 FTPS 服务器，以在文件夹和 Foundry 数据集之间同步数据。

支持的功能

功能 状态
探索 🟢 正式可用
批量导入 🟢 正式可用
增量同步 🟢 正式可用

数据模型

该连接器可以将任何类型的文件传输到 Foundry 数据集中。传输过程中及传输完成后，文件格式将保持不变，且不应用任何模式。请对输出数据集应用必要的模式，或编写 下游转换 来访问数据。

性能与限制

可传输的文件大小没有限制。但网络问题可能导致大规模传输失败。特别是，在 Foundry worker 上运行超过两天的同步任务将被中断。为避免网络问题，建议使用较小的文件体积，并限制每次同步执行时摄入的文件数量。可以 设置定时计划，让同步任务更频繁地运行。

设置

  1. 打开 数据连接 应用，并在屏幕右上角选择 + 新建源。
  2. 从可用的连接器类型中选择 FTP/FTPS。
  3. 按照后续的配置提示，使用以下各节中的信息继续设置连接器。

要访问本地 FTP 服务器，建议您 采用智能体代理策略进行连接。

详细了解如何在 Foundry 中 设置连接器。

身份验证

FTP/FTPS 的身份验证通过用户名和密码完成。如果 FTP 用户对为连接配置的根目录没有访问权限，FTP/FTPS 连接可能会失败。有关根目录的更多详细信息，请参阅 配置选项。

选项 是否必填？ 描述
用户名 是 FTP 登录用户名。
密码 是 FTP 登录密码。对于匿名登录，此字段可留空。如需更多信息，请联系您的服务器管理员。

网络配置

如果您的 FTP/FTPS 连接 在 Foundry 中运行，则必须添加 网络出站策略 以将连接加入白名单。

如果通过域名连接，应在控制端口（通常是 21 端口）和数据端口上为 FTP 服务器域创建出站策略。建议创建两条网络策略：一条用于控制端口的单端口策略，以及一条用于数据端口的端口范围策略。数据端口由 FTP 服务器的管理员指定。如果已正确配置出站策略但仍频繁报错，请 提交工单 并附上已应用的策略列表。

如果服务器域名解析到多个域名和/或多台服务器，则需要将所有关联的域名及其相关 IP 加入白名单。要验证服务器是否解析到多个域名和/或服务器，请在终端中对尝试连接的服务器运行 dig <domain> 命令并查看应答部分。

证书与私钥

请参照以下指引，配置附加的客户端或服务器证书及私钥，以正确设置您的连接器。

SSL 与主机名验证

SSL 连接会验证服务器证书。通常，SSL 验证通过证书链完成；默认情况下，智能体和 Foundry worker 均信任大多数行业标准证书链。如果您要连接的服务器使用的是自签名证书，或者防火墙对该连接执行了 TLS 拦截，则连接器必须信任该证书。详细了解 在数据连接中使用证书。

服务器必须提供完整的证书链，SSL 验证才能正常工作。可以通过运行命令 openssl s_client -connect {hostname}:{port} -showcerts -starttls ftp 来获取 FTP 服务器的证书链。要验证证书链，请使用 OpenSSL 命令行工具或其他任何可用工具。

如果使用 FTPS，请确保已将 FTPS 服务器的证书添加到智能体的信任库中。

Foundry 会尝试对所有出口路由进行验证。但是，FTP 无法被检查，这会导致连接挂起和/或超时错误。如果尽管已正确配置出口策略，但错误仍然持续出现，请提交问题报告，并附带您希望禁用主机名验证的策略列表。

隐式/显式 SSL

FTP 服务器可以配置为支持显式或隐式 SSL。运行在 990 端口的服务器通常使用隐式 SSL。

请与您的服务器管理员确认服务器的设置。默认情况下，连接器假定使用显式 SSL；您可能需要针对您的环境更改此设置。

连接要求

FTP 需要 CONTROL 和 DATA 连接类型。DATA 连接必须配置为 ACTIVE 或 PASSIVE 模式。

  • CONTROL：客户端到服务器
  • DATA：从范围内选择的数据（例如 1024–1123）
    • PASSIVE：客户端到服务器
    • ACTIVE：服务器到客户端
      • 仅适用于智能体连接

默认的 FTP/FTPS 连接器端口：

  • FTP：21
  • FTPS 显式：21
  • FTPS 隐式：990

主动和被动模式

我们建议使用被动模式网络连接。在被动模式下，所有连接均由客户端发起。使用被动模式时，请确保控制端口（通常为 21）和数据传输的端口范围（例如 1024–1123）已加入白名单。请联系您的 FTP/FTPS 服务器管理员获取连接详情。

主动模式是一种较旧的建立文件传输的方法。在主动模式下，客户端连接到服务器，同时服务器也连接到客户端。服务器和客户端相互依赖，需要双向网络连接。这种网络方法在大多数安全环境中通常难以实现，并且仅在使用智能体连接时才可行。

故障排查

智能体连接

  • 在设置智能体连接时遇到问题了吗？请安装一个 FTP/S 客户端，并尝试使用与数据源相同的配置连接到服务器。如果此连接失败，则该问题并非连接器漏洞。在提交问题之前，请先排查网络连通性、身份验证和 FTP 服务器配置。

  • 是否在使用出口代理负载均衡器？FTP 是一种有状态协议，如果连续请求不是来自同一 IP，使用负载均衡器可能会导致同步失败（非确定性失败）。

SSL 和 FTPS

  • 服务器是否使用自签名证书？是否已将其添加到源信任库中？请参阅上文的《SSL 和主机名验证》部分。

  • FTP 服务器是否仅支持旧版 TLS（例如 TLS 1.1）？如果是，连接器运行时可能无法接受服务器提供的任何密码套件。请提交问题，以便与 Palantir 代表共同探讨替代方案。
