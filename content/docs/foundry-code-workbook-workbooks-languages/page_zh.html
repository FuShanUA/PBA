语言

支持的语言和版本

Code Workbook 目前支持三种语言：Python、R 和 SQL。

Code Workbook 当前支持的 Python 版本包括 Python 3.10 和 Python 3.12。较低版本的 Python 不受支持，使用这些版本的环境将无法解析。我们强烈建议使用最新可用的 Python 版本，因为 Palantir Foundry 会根据 Python 开发者文档的说明，停止对被认为已结束生命周期（End-of-Life）的 Python 版本的支持。

当前支持的 R 版本包括 R 3.6、R 4.0、R 4.1 和 R 4.2。R 3.3、R 3.4 和 R 3.5 版本不受支持，它们各自的环境将无法初始化。

Code Workbook 支持的 SQL 变体是 Spark SQL ↗。

要在一个Code Workbook配置文件中启用特定语言，请参阅配置Code Workbook配置文件文档中的Conda环境部分。下面提供了每种支持语言的示例，分别在Python（Python）、R（R）和SQL（SQL）的相应介绍中。

在工作簿中启用语言

为了使支持的语言正常运行，需要进行特定配置，如下面各节所述。

启用 R

R 尚未提供自助服务功能。

要在 Code Workbook 中创建 R 转换，必须满足以下两个条件：

  • R语言必须在您的注册中特别启用。请联系您的Palantir代表以获取启用帮助。
  • 当前工作簿使用的环境中必须包含包vector-spark-module-r。可以通过以下任一方式实现：
    • 在控制面板的配置中切换R复选框。这会自动将vector-spark-module-r包添加到配置的环境中。
    • 使用添加包下拉菜单手动将vector-spark-module-r添加到环境中。

有关更多信息，请参阅配置代码工作区配置文件。

启用Python

当前工作簿使用的环境中必须包含包vector-spark-module-py。可以通过以下任一方式实现：

  • 在控制面板的配置中切换Python复选框。这会自动将vector-spark-module-py包添加到环境中。
  • 使用添加包下拉菜单手动将vector-spark-module-py添加到环境中。

有关更多信息，请参阅 配置代码工作簿配置文件。

启用 SQL

SQL 转换不需要任何额外的包即可运行。因此，对于任何给定的配置文件，SQL 转换默认始终可用。

如果您不打算在某个配置文件中使用 Python 或 R，请考虑移除相关的 vector-spark-module 包以减小您的环境。您可以在需要时随时重新添加它们。

Python 简介

Python 转换

Python转换被定义为一个Python函数，可以有任意数量的输入，最多一个输出，并且可选地有一个或多个可视化。通过将转换的别名作为函数参数进行引用，Code Workbook会自动将提到的别名的输出作为转换的输入传递。有关Code Workbook中转换的更多信息，请参阅 转换概述 文档。

一个简单的Python转换示例可能包括一个PySpark DataFrame作为输入，使用PySpark语法转换数据，并以转换后的Spark DataFrame作为输出。

Copied!
1def child(input_spark_dataframe):
2    from pyspark.sql import functions as F
3    return input_spark_dataframe.filter(F.col('A') == 'value').withColumn('new_column', F.lit(2))

Python中的Spark和Pandas之间的转换

在Python转换中，Spark和Pandas数据帧之间的转换非常简单。

Copied!
1# Convert to PySpark
2spark_df = spark.createDataFrame(pandas_df)
3# Convert to pandas
4pandas_df = spark_df.toPandas()

将数据转换为pandas意味着将数据收集到驱动程序中。因此，数据的大小受到Spark模块上可用驱动程序内存的限制。如果您处理的是大型数据集，可能希望先使用Spark对数据进行过滤和聚合，然后将其收集到pandas DataFrame中。

Copied!
1from pyspark.sql import functions as F
2def filtering_before_pandas(input_spark_dataframe):
3    # Use PySpark to filter the data
4    filtered_spark_df = input_spark_dataframe.select("name", "age").filter(F.col("age") <= 18)
5
6    # Convert to a pandas df, which collects the data to the driver
7    pandas_df = filtered_spark_df.toPandas()
8
9    # Perform pandas operations
10    mean_age = pandas_df["age"].mean()
11    pandas_df["age_difference_to_mean"] = pandas_df["age"] - mean_age
12
13    # Output the resulting DataFrame
14    return pandas_df

为了在保存后保持排序后的pandas DataFrame的顺序，请将其保存为单个分区的Spark DataFrame：

Copied!
1import pyspark.pandas as p
2
3return p.from_pandas(df).to_spark().coalesce(1)

R简介

R 转换

一个R转换被定义为一个R函数，可以有任意数量的输入，最多一个输出，并且可以选择性地有一个或多个可视化。通过将转换的别名作为函数参数进行引用，Code Workbook会自动将提到的别名的输出作为转换的输入传递。有关Code Workbook中转换的更多信息，请参阅 转换概述 文档。

R转换的一个简单示例可能包括一个父级R数据框，使用R对数据进行转换，并以一个R数据框作为输出。

Copied!
1child <- def(r_dataframe) {
2    library(tidyverse)
3    new_df <- r_dataframe %>% dplyr::select(col_A, col_B) %>% dplyr::filter(col_A == true) %>% dplyr::mutate(new_column=1000)
4    return(new_df)
5}

Spark与R数据框之间的转换

在R转换中，将Spark DataFrames和R data.frame相互转换非常直接：

Copied!
1# Convert from Spark DataFrame to R data.frame
2new_r_df <- SparkR::collect(spark_df)
3
4# Convert from R data.frame to Spark DataFrame
5 spark_df <- SparkR::as.DataFrame(r_df)

请注意，将数据转换为 R 的 data.frame 意味着将数据收集到驱动程序中。因此，数据的大小受到 Spark 模块上可用驱动程序内存的限制。如果您处理的是大型数据集，可能需要先使用 Spark 过滤和聚合数据，然后将其收集到 R 的 data.frame 中。

Copied!
1output_dataset <- function(spark_df) {
2    library(tidyverse)
3
4    # Use SparkR to filter the data
5    input_dataset_filtered <- SparkR::select(spark_df, 'column_A', 'column_B')
6
7    # Convert to R data.frame
8    local_df <- SparkR::collect(input_dataset_filtered)
9
10    # Use tidyverse functions to transform your data
11    local_df <- local_df %>% dplyr::filter(column_A == true) %>% dplyr::mutate(new_column = 1000)
12
13    # Output an R data.frame
14    return(local_df)
15}

R 故障排除

当 Code Workbook 中导入的数据集被读取为 R data.frame 时，该数据集通过收集到驱动程序中从 Spark DataFrame 转换为 R data.frame。

  • 您无法将任意大的数据作为R数据框进行读取。数据的大小受Spark模块上的驱动内存限制。要处理大型数据，请先将数据集作为Spark DataFrame读入，使用SparkR将数据转换为更小的形式，然后调用SparkR::collect()将其转换为R数据框。或者，可以使用Python或SQL在使用R之前将数据转换为更小的形式。
  • 当将某些数据类型收集到R数据框时，存在一些已知问题。在大多数情况下，我们使用一个名为r-arrow ↗的库来进行收集，这加快了序列化和反序列化过程。特别是，当使用r-arrow时，Long、Array、Map、Struct和Datetime类型不可转换。请考虑删除这些列或将它们转换为其他数据类型（如String）。当尝试将这些类型的输入作为R数据框读取时，界面会发出警告。

R 在代码工作区中是单线程的，这意味着在同一 Spark 模块上同时只能运行一个 R 作业。如果您同时启动多个 R 作业，它们将按顺序运行；排队的作业会显示为“正在代码工作区中排队”。

  • 如果您有长时间运行的作业或者转换操作已保存为数据集的作业，我们建议您运行批量构建。批量构建将在其自己的 Spark 模块上运行，使您可以在同一工作区或其他共享 Spark 模块的工作区中继续迭代。

SQL 简介

代码工作区中支持的 SQL 变体是 Spark SQL ↗。唯一支持的输入和输出类型是 Spark DataFrames。

一个简单的 SQL 转换示例可能是在连接键上连接两个输入 DataFrame。

Copied!
1SELECT table_b.col_A, table_b.col_B, table_a.*
2FROM table_a
3JOIN table_b ON table_a.col_C == table_B.col_C

要为 SQL 节点添加父节点，仅在代码中引用别名是不够的。您必须通过选择输入栏使用 UI，或者使用 + 按钮创建子节点。
