应用状态

AIP 聊天机器人的应用状态此前被称为参数。

你可以在 AIP 聊天机器人上配置多个字符串或对象集合类型的应用变量，以此设定应用状态。当包含应用变量的 AIP 聊天机器人被嵌入到 Workshop 中的 AIP 聊天机器人组件时，变量列表会显示出来。接着，你可以将每个应用变量映射到对应类型的 Workshop 变量，从而在其他组件中展示输出结果。

在 AIP Chatbot Studio 中配置应用状态

在设置 AIP 聊天机器人的应用状态时，请进行以下配置：

  • 确定应用变量：明确大模型需要与哪些对象集合或字符串变量进行交互。这些变量可以是供大模型写入结果的新变量，也可以是工作流中已有的变量（例如代表用户当前选择的变量）。
  • 命名并描述变量：为每个变量编写说明其作用的描述。该描述会被注入到聊天机器人的提示词中，为大模型提供何时使用特定变量的上下文依据。
  • 设置值可见性：你可以通过设置值可见性来决定聊天机器人是否能查看变量值。建议仅在必要时才允许大模型读取变量值，以减少干扰。例如，如果你将对象集合变量用作本体上下文的输入，则该变量无需对大模型可见；对象集值的相关内容会在每一轮循环中包含在系统提示词中。但是，如果字符串变量包含系统提示词中的动态部分，则必须将该变量设置为可见，以便大模型能够读取其内容。
  • 将变量添加为输入：你可以选择将变量配置为本体检索上下文的输入，从而为语义搜索提供确定性输入，或向大模型提供完整的对象集。此外，基于函数的上下文可以接收多个输入变量，以便在运行函数时包含状态信息。对象查询工具还可以为每种对象类型接收一个初始对象集变量，为大模型应用额外的筛选或聚合操作提供起点。

你还可以在用户自定义的系统提示词中，通过键盘输入斜杠命令 / 来引用应用状态。

你可以使用调试应用状态部分来测试应用状态。支持手动覆盖每个变量的值，且当聊天机器人更新变量值时，调试部分会提供可视化的反馈。

通过聊天机器人更新应用变量

你所配置的应用变量可以通过聊天机器人工具或添加到应用状态配置中的检索上下文进行修改。变量既可以从工具或上下文的输出中确定性地更新，也可以通过更新应用变量工具非确定性地更新。

自动变量更新

你可以将变量配置为根据对象查询工具、基于函数的上下文或本体上下文的输出进行确定性更新。在每次执行上下文或工具后，Chatbot Studio 会记录最新的输出，并在大模型完成流式传输时，将映射的应用变量更新为该输出值。建议采用确定性更新而非非确定性更新，以避免大模型产生混淆。在某些情况下，你可能希望对大模型完全隐藏变量值。

确定性工具输入

**Action** 和 **Function** 工具可以配置为使用来自应用变量的输入，而不是通过 LLM 动态生成输入。此功能允许您将预设值直接传递给工具，从而提高一致性并减少 Token 消耗。

请注意，此功能仅支持字符串和对象集输入类型。传递的值会固定为推理循环开始时变量的初始值，这意味着同一查询中先前工具调用的更新不会反映在作为确定性输入传递的值中。

使用 LLM 更新变量

如果您希望由 LLM 配置要更新的新变量，或者根据当前用户查询有条件地应用更新，请添加 **更新应用变量** 工具。此工具支持 LLM 可更新的一系列变量列表。

将变量作为引用

您还可以配置对象集变量，使其在用户选择引用时进行更新。配置的对象集变量将更新为包含所引用对象的静态对象集。

在 Workshop 中进行配置

如果 AIP 聊天机器人已配置应用状态，您可以在 Workshop 中配置该聊天机器人的应用变量。请查阅 AIP 聊天机器人小部件文档以获取更多信息。
