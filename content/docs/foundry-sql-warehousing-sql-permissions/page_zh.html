SQL 权限

此页面描述了控制对 Foundry 中 SQL 功能访问的 角色，包括 SQL Studio、嵌入式 SQL 控制台，以及通过 Arrow Flight SQL 或 SQL REST API 连接的外部 SQL 客户端。

此处描述的角色属于 Foundry SQL Server 和 Download 角色集类别。

相关操作

以下操作控制 SQL 访问。用户必须在资源上拥有 foundry-sql-server:preview 或 foundry-sql-server:read 权限之一，才能对其执行任何 SQL 操作。

操作Foundry 行为外部 API 行为
预览: foundry-sql-server:preview结果预览返回查询结果的前 1,000 行。-
查询: foundry-sql-server:read结果预览默认显示 1,000 行。在 SQL Studio 中，用户可以从设置菜单中将预览限制扩展到 10,000 行。返回完整的查询结果，没有行数限制。
下载: foundry-sql-server:frontend-download在结果面板中的下载操作中需要。下载结果预览中显示的行（最多 1,000 行）。—
工作表读取: foundry-sql-server:read-worksheet打开并查看保存的 SQL 工作表。—
工作表写入: foundry-sql-server:write-worksheet创建、编辑和保存 SQL 工作表。—

这些操作可以作为默认角色集的一部分，或者通过< a href="/docs/foundry/platform-security-management/manage-roles/#role-sets">自定义角色集中的自定义角色进行授予。

通过< a href="/docs/foundry/sql-warehousing/ontology-sql/">本体论 SQL查询本体论不需要额外的角色。访问遵循所查询对象类型的标准本体论角色。

自定义角色配置

默认角色可以通过< a href="/docs/foundry/platform-security-management/manage-roles/#role-sets">自定义角色集进行自定义。常见的配置包括：

  • 限制查询：一些组织会限制使用 SQL 查询数据集，以防止用户通过 SQL API 运行无限制的查询。在这种情况下，可以授予在 Foundry 中使用 SQL 预览数据集，以允许用户在 Foundry 内运行有限制的查询。
  • 将下载与查询分离：一些组织会限制在 Foundry 中下载 SQL 结果，以防止用户通过 UI 下载按钮下载结果，即使他们被授予了预览权限。

AI辅助查询生成

AI辅助查询生成 功能由 AIP 的启用状态决定，而不是基于角色权限。

AIP 必须为用户的组织和包含所查询资源的项目启用。有关详细信息，请参阅 AIP 权限。
