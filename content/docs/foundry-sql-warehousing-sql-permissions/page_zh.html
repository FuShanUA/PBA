SQL 权限

本页描述了控制对 Foundry 中 SQL 功能访问的 角色（role），包括 SQL Studio、嵌入式 SQL 控制台（console），以及通过 Arrow Flight SQL 或 SQL REST API 连接的外部 SQL 客户端。

此处描述的角色属于 Foundry SQL Server 和 Download 角色集类别。

相关操作

以下操作控制 SQL 访问。用户必须在资源上拥有 foundry-sql-server:preview 或 foundry-sql-server:read 中至少一个权限才能对其进行任何 SQL 操作。

操作（Operation）Foundry 行为（Foundry behavior）外部 API 行为（External API behavior）
预览：foundry-sql-server:preview 结果预览返回查询结果的前1000行。-
查询：foundry-sql-server:read结果预览默认为1000行。在SQL Studio中，用户可以从设置菜单将预览限制扩展到10000行。返回完整的查询结果，没有行数限制。
下载：foundry-sql-server:frontend-download在结果面板中的下载操作中是必需的。下载结果预览中显示的行（最多1000行）。—
工作表读取：foundry-sql-server:read-worksheet打开并查看保存的SQL工作表。—
工作表写入：foundry-sql-server:write-worksheet创建、编辑并保存SQL工作表。—

这些操作可以作为默认角色集的一部分，或者通过自定义角色内的< a href="/docs/foundry/platform-security-management/manage-roles/#role-sets">自定义角色集进行授予。

通过< a href="/docs/foundry/sql-warehousing/ontology-sql/">本体论SQL查询本体论不需要额外的角色。访问遵循被查询对象类型的标准本体论角色。

自定义角色配置

默认角色可以通过 自定义角色集 进行自定义。常见的配置包括：

  • 限制查询： 一些组织限制 Query datasets using SQL 以防止用户通过 SQL API 运行无限制的查询。在这种情况下，Preview datasets using SQL 仍可被授予，以允许用户在 Foundry 内运行有上限的查询。
  • 区分下载和查询： 一些组织限制 Download SQL results in Foundry 以防止用户通过 UI 下载按钮下载结果，即使他们被授予了预览权限。

AI辅助查询生成

AI辅助查询生成 功能由 AIP 启用，而不是角色集权限。

AIP 必须为用户的组织以及包含所查询资源的项目启用。有关详细信息，请参阅 AIP 权限。
