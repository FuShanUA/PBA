创建一个新的工件注册表

要创建一个工件注册表，请导航到您的Hub设置中的工件注册表部分。

这是管理您Apollo Hub所有工件注册表的位置。

步骤1：输入你的OCI注册表的信息

选择新建注册表以创建一个工件注册表。

在注册表URL字段中，输入你OCI注册表的URL。这应该采用 registry.domain.com:port 格式，其中 port 是你的注册表可访问的可选端口。如果省略，则默认为 :443。

显示名称和描述是可选字段，可用于将Artifact Registry标识给其他Apollo用户。我们建议输入一个具有描述性的名称，这将有助于用户理解Artifact Registry的用途。

有关如何使用镜像注册表的更多信息，请参见 镜像Artifact注册表。通常情况下，您不需要设置此字段，除非在特定情况下。

注册表URL在Apollo中必须唯一。不能有两个具有相同URL的Artifact注册表。

接下来，选择Apollo用于访问OCI注册表的凭据类型。有两种选项：

  • 直接使用基本认证或无需认证的OCI认证。
  • 使用Amazon Elastic Container Registry (ECR)认证。

OCI凭据

OCI凭据有两种选项：

  • 基本认证：您将输入用户名和密码组合。
  • 无认证：Apollo在尝试访问注册表时不会使用凭据。

一旦创建，您将无法再次查看输入的 OCI 注册表凭证，请在创建之前确保已正确输入。

您仅应在部署包含图像在公共 OCI 注册表（如 docker.io 或 quay.io）中的产品时选择 无身份验证 方法。

亚马逊弹性容器注册表 (ECR)

要使用亚马逊 ECR，您需要从预置的 AWS IAM 用户获取以下信息：

  • 访问密钥 ID
  • 秘密访问密钥
  • FIPS 模式：为 ECR 使用 AWS FIPS ↗ 端点。

步骤 2：设置角色

接下来，您必须为新创建的工件注册表分配一些角色。在创建工件注册表后的五分钟内，您必须设置初始权限。我们建议将“工件注册表管理员”角色分配给您所属的一个团队，以便您可以进行进一步更改。从 权限 选项卡 进行的任何全局角色分配都将被继承，并在此步骤中显示。

在 添加更多团队... 文本框中输入一个团队名称以选择一个团队，然后使用下拉菜单配置他们获得的角色。

我们建议将“工件注册表分配者”角色分配给负责管理环境的任何团队，这样他们就可以将任何创建的工件注册表添加到他们的环境中。

第3步：更新工件注册表

此时，您可以更新工件注册表的某些部分：

  • 显示名称
  • 描述
  • 镜像注册表
  • 凭据（必须替换，不能编辑）

更新将创建一个新的修订版本。您可以将该版本与环境设置中显示的版本进行核对，以确保更改已应用到环境中并可供使用。
