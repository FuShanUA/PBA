设置数据源

Foundry 数据源代表 Foundry 与外部系统之间的连接。Foundry 数据源类型的示例包括 Postgres 数据库、S3 存储桶、Linux 服务器上的文件系统、SAP 实例或互联网上的 REST API。

总体而言，将数据源连接到 Foundry 需要以下步骤。请注意，步骤 1 到步骤 3 可能需要您更改或验证现有架构中的配置：

  1. 确保数据源与 Foundry 之间存在有效的网络连接。
  2. 配置有效的凭证，以便 Foundry 对数据源进行身份验证。
  3. 仅适用于传统 agent worker 数据源：确保 agent 拥有访问外部系统的适当驱动程序。
  4. 最后，在数据连接中创建数据源。

建立此数据源连接后，您可以配置同步，将特定数据集引入 Foundry。同步完全可以通过数据连接 UI 进行配置，因此数据源设置是最后一项可能需要更新组织环境配置才能在 Foundry 中访问数据的任务。

配置网络访问

要将 Foundry 连接到外部系统，首先请验证网络路径：

  • 对于与 Foundry 托管在同一网络中并接受 Foundry 入站连接的外部系统，仅需建立一条从 Foundry 到该外部系统的有效网络连接。对于云托管的 Foundry 实例，这通常是基于云的系统或 SaaS 服务的情况。
  • 对于托管在与 Foundry 不同网络中的外部系统，必须使用带有两条有效网络路径的agent：(1) 从 agent 主机到外部系统，以及 (2) 从 agent 主机到 Foundry。对于云托管的 Foundry 实例，独立网络通常意味着本地部署网络。

（可选）设置 agent 并配置 agent 连接

如果您要连接的外部系统托管在与 Foundry 网络隔离的网络上，则需要设置 agent。

在 Foundry worker 和 agent 代理策略下，agent 仅用作网络代理，计算在 Foundry 中执行。传统的 agent worker 数据源同时使用 agent 进行网络连接和计算；请参阅 Foundry worker 与 agent worker 对比。

请确保您网络中设置的 agent 与外部系统之间存在连接。该 agent 作为您网络访问 Foundry 的单一验证入口，将处理读取数据并将其发送到 Foundry 实例的过程。对于每个新系统，您只需确认 agent 与该新系统之间存在有效的连接即可。

您无需建立从外部系统到 Foundry 的直接网络出口，因为流量仅在 agent 到 Foundry 以及 agent 到外部系统之间流动。了解有关数据连接架构的更多信息。

建立此连接所需的具体步骤因组织的网络设置而异。无论采用何种具体设置，目标都是使 agent 能够连接到外部系统。这可能涉及配置 agent 主机上的出口设置、外部系统上的入口设置、跨网络的防火墙规则、agent 上的代理设置、将源系统证书添加到 agent 信任库等操作。

如果需要为 agent 配置代理设置以使其能够访问外部系统，可以通过数据连接完成此操作。

配置网络策略

您必须在 enrollment 上拥有 Information security officer 角色才能配置网络出口。如果您没有配置出口的权限，请联系您的 Palantir 管理员申请访问权限。

您可以在控制面板的 Enrollment permissions 部分找到 Information security officer 角色。用户必须拥有 Enrollment administrator 角色才能查看此部分。

Foundry worker 数据源还需要网络出口策略来路由流量。

要配置网络策略，请前往控制面板中的网络出口部分。如果连接的外部系统与 Foundry 托管在同一网络中，请选择直接连接策略。如果连接的源位于与 Foundry 不同的网络中，请选择智能体代理出口策略。

如果无法查看网络出口部分，请联系 Palantir 管理员设置网络策略。

配置凭证

在大多数情况下，Foundry 需要经过授权的凭证（如用户名和密码）才能访问外部系统。建议使用专属服务账户，其凭证范围专门针对 Foundry 中的所需访问权限设定。

请遵循贵组织内部关于建立服务账户的指南和流程，为源配置服务账户。在继续下一步之前，请记录下相关凭证。

在数据连接中创建源

完成上述步骤后，即可在数据连接中创建源：

  • 登录后，通过侧边栏导航至数据连接。
  • 选择源选项卡。
  • 在右上角选择新建源。
  • 选择与您的数据源相对应的源类型。
  • 在右下角选择继续。该源配置为在Foundry 工作节点上运行。对于现有的智能体工作节点源，请参阅智能体工作节点配置参考。

将源保存在项目中

接下来，为源命名并选择一个项目来放置它。通常建议为每个源创建一个新项目，因为这是为源自的数据集配置权限最自然的方式。

您可以阅读有关源权限最佳实践的更多内容，或参阅《在 Foundry 中构建端到端数据管道的完整指南》。

在右下角选择创建源并继续。

选择网络策略 [Foundry 工作节点]

在下一个设置页面，通过选择使用现有策略并搜索策略名称，选择您之前配置的网络策略。

配置源

添加有关如何连接到源的详细信息。这些信息取决于您所使用的源类型，通常包含基本凭证，如连接 URL、云提供商区域等。

（可选）添加 JDBC 驱动

JDBC 源可能需要您上传JDBC 驱动，然后指定要使用该驱动中的哪个 Java 类。

（可选）添加证书

外部系统可能需要证书以确保连接可信。这适用于以下情况：

  • 使用带有自签名证书的 TLS 的系统，需要为其添加服务器证书。
  • 使用 mTLS 的系统，要求 Foundry 客户端通过客户端证书证明其身份。

要了解是添加服务器证书还是客户端证书，请参阅《服务器和客户端证书》。

对于 Foundry 工作节点连接，请按照以下步骤将证书添加到源本身：

  1. 展开证书卡片，查看源上选择的证书。
  2. 选择添加证书（如果尚未配置证书，请先选择创建新证书）。
  3. 选择要附加到源的客户端证书和服务器证书包。

请查阅《出口证书配置文档》以获取更多信息。

在 连接详情 页面底部的 更多选项 下，选择 配置服务器证书（旧版） 或 配置客户端证书和私钥（旧版），以为使用旧版证书创建的数据源进行配置。

了解在使用智能体工作线程连接时，如何配置智能体上的证书。。

添加凭证

添加您先前预配的凭证，以允许数据源连接到您的数据。

保存并继续

在右下角选择 保存 以完成数据源设置。数据源完全设置好后，您可以继续设置同步，将数据引入 Foundry。

故障排除

要确认连接是否已建立，请在数据源页面的右侧面板中选择 预览。

如果数据源未按预期工作，请参阅《故障排除》以获取可用的调试工具——包括数据源终端和网络出口日志——以及关于连接和证书问题的指导。对于同步特定的问题，请参阅《同步故障排除参考》；如果您正在使用智能体工作线程，请参阅《智能体故障排除参考》。
