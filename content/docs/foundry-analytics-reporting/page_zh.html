报告

在 Foundry 中，我们将“报告”定义为创建带有标注的时间点内容，以便在平台内分享或下载。如果您想创建供他人直接交互的互动内容，我们建议使用 Foundry 的仪表盘产品。

Notepad

Notepad 是 Foundry 的报告工具。借助 Notepad，您可以创建实时更新和静态（“时间点”）文档、为文档添加标注、在平台内分享文档，以及将文档导出为 PDF。

Notepad 的主要功能包括：

  • 简便易用的文档创建和编辑体验。
  • 能够添加来自各种 Foundry 源的内容，包括 Quiver、Contour、Code Workbook 和 Object Explorer。
  • 支持通过动态输入进行文档模板化。
  • 可自定义导出 PDF。

了解如何开始使用 Notepad。

报告功能处于开发生命周期的淘汰阶段，未来将被废弃。目前仍提供全面支持。先前保存的报告仍以只读形式提供，除非需要编辑，否则无需将其转换或迁移至新格式。我们建议将您的工作流迁移至其他应用和工具，以满足您的用例需求。

传统的 Foundry 报告应用同时支持实时仪表盘和静态报告用例。然而，其他 Foundry 工具为这些工作流提供了更好的支持和更丰富的功能集。对于大多数仪表盘和报告用例，我们推荐以下产品：

  • Contour 仪表盘：适用于由 Contour 分析生成的内容。Contour 仪表盘相比报告提供了更好的交互性，具有图表联动筛选等功能，并提供了简便的拖拽界面，可在迭代分析的同时构建仪表盘。详细了解如何开始使用 Contour 仪表盘。
  • Quiver 仪表盘：适用于由 Quiver 分析驱动的内容。报告不支持嵌入大多数 Quiver 可视化类型。详细了解如何开始使用 Quiver 仪表盘。
  • Notepad：适用于跨产品内容和大多数 Foundry 报告用例，如上文所述。详细了解如何开始使用 Notepad。

如果您正在创建包含来自多个源（例如 Contour 分析和 Code Workbook，或两个 Contour 分析）内容的仪表盘，您可能仍需使用报告。如果是这样，您可以详细了解如何开始使用报告。
