使用 Chrome™ DevTools 进行调试

在本指南中，您将学习如何在 Chrome™ 浏览器中使用开发工具（DevTools）来帮助识别意外行为或收集必要的浏览器日志以提供给 Palantir 支持。

在调试模糊错误时，最重要的工具之一是 Chrome™ DevTools ↗。这提供了大量用于调试问题的信息，并允许您查看网页背后发生的事情的一些内部情况。将其想象成打开汽车的引擎盖——你可以看到里面到底发生了什么。

使用 Chrome™ DevTools

您可以按照以下步骤访问控制台日志：

  1. 在您打开的网页上右键点击一个元素。
  2. 选择检查。这将在网页的一侧打开一个屏幕。
  3. 在顶部，导航到控制台选项卡，大多数错误都会出现在这里。
  4. 失败的请求会显示为红色。找到看起来与页面上出现的问题相关的红色请求，或者支持团队指示您提供的特定请求。在执行相关操作时，您可能需要保持网络窗口打开，以便错误信息能够显示出来。
  5. 点击左侧的三角形以展开请求。通常这会包含其他有用的信息，例如错误消息和 errorInstanceID。

目前最重要的是顶部的工具栏。对于大多数故障排除，控制台和网络选项卡将包含相关错误信息，并且对支持团队最有帮助。然而，元素选项卡也可以用于识别问题，因此我们也将对其进行介绍。

控制台标签

Chrome™ 控制台，如下面的图片所示，并可通过工具栏中的 控制台 标签访问，有两个主要用途。第一个是供网络应用程序本身使用。网络应用程序可以随意在控制台中打印任何内容 - 调试信息、错误日志、信息消息等等。

第二个用途是让用户运行代码片段并检查结果。然而，这种用法超出了本指南的范围。

当你遇到错误时，打开控制台并向上滚动历史记录通常很有用，特别是要查找错误（通常以红色突出显示）。这些错误将有助于报告问题，并且通常包含更多关于出错原因的信息。

网络标签页

当您在 Palantir 应用程序中选择一个按钮时，您的浏览器会向服务器发送请求，然后由服务器处理后再发送响应。一个网页可能会发出许多请求，这些请求不会导致页面重新加载，但可能会更新您看到的数据或向服务器发送新数据。网络 选项卡可让您检查浏览器发出的所有请求，以及查看接收到的响应。

要在特定页面上访问网络日志：

  1. 右键单击您打开的页面上的一个元素，然后选择 检查。这会在网页的一侧打开一个屏幕。
  2. 导航到页面顶部附近的 网络 选项卡。
  3. 失败的请求会显示为红色。找到看起来与页面上出现的问题相关的红色请求。
  4. 选择左侧的三角形以展开请求。通常会包含更多详细信息，例如错误消息和 errorInstanceID。

以下是使用此视图获取更多有关错误信息的一些最佳实践。首先，查看侧边栏左上角的两个按钮：

红色的第一个图标表示当前正在记录网络标签页。这意味着当有新的请求发出时，它将被记录在下面的视图中。如果此功能不可用，您应切换 记录 功能开启。同样，如果您发现请求日志的填充速度比您能检查的速度快，您可以选择它再次关闭记录。

The second symbol (the circle with a line through) lets you clear the log. This is useful to do right before you enact the minimum steps to reproduce. This action clears the log of all currently recorded requests so that you can see just those new requests that come in.

The best way to use this log is to:

  1. Perform the minimum steps to reproduce the error, but right up until just before the error.
  2. Clear the log.
  3. Perform the final step that causes the error.
  4. Examine new requests, particularly looking for any HTTP error codes. You may wish to attach a screenshot of the requests to your report.

需要注意的第一件事是请求所使用的URL。在这种情况下，redirect?code=... 是我们联系的URL。这对于识别导致错误的服务很有用。例如，如果URL包含 foundry-metadata，那么 foundry-metadata 服务就是问题所在——这当然对在错误报告中包含这些信息非常有帮助。

该请求的cURL等效项也有助于报告。您可以通过右键单击请求本身并从菜单中选择复制 > 作为cURL复制来将其复制到报告中。

cURL 是一个命令行工具，它允许您从命令行执行与请求完全相同的操作。此按钮会将发出的请求复制到剪贴板，并让调试器在任何地方和以任何方式运行 cURL 请求，从而可以检查请求以确定哪里可能出错。这在 4xx 错误时特别有用。如果有任何错误 ID，请同时记录下来。您可能需要将此副本保存为文本文件以发送给 Palantir 支持部门，但请注意：

  • 出于安全原因，从日志中删除任何现有令牌，否则您的 Palantir 团队会撤销它们。您可以通过搜索 "token" 和 "bearer" 来检查文件中是否包含令牌，并删除其后的字母字符串。
  • 在共享前删除敏感信息。

报告中最后一个重要部分是响应本身，您可以通过同一复制菜单中的复制响应来访问它。

要访问特定页面上的网络选项卡：

  1. 在您打开的页面上右键选择一个元素，然后选择检查。这将在网页的一侧打开一个屏幕。
  2. 导航到页面顶部附近的网络选项卡。
  3. 有时在打开网络选项卡时它可能没有数据。通过刷新页面或重新触发导致失败的行为来填充该选项卡。
  4. 网络选项卡中的第二列是“状态”。选择状态直到它们按降序排序。这将把失败的请求放在最上面，因为它们会有更高的状态代码。失败的请求还会显示为红色。
  5. 找到与您页面上出现的问题相关的请求。选择该请求以获取更多信息，例如错误消息或错误InstanceID。

在网络选项卡中查找追踪ID

TraceIDs 是请求的唯一标识符，使浏览器中发生的进程可以与日志中存储的记录相匹配。

To find traceIDs, open the Network Tab as described in the previous section. Different requests may be worth investigating depending on the situation; in general, red-colored (indicating failure) requests are often useful when an error is displayed in the browser.

Elements tab

The elements tab shows you the "DOM" (Document Object Model) of the page you are viewing. This constitutes all the visual content you are seeing on the page represented in its underlying HTML form:

This tree of HTML data can be explored by using the Inspect Element tool. There are two ways to use this. First, we can use the pointer version, accessible by this button in the DevTools toolbar:

After we select this button, the pointer becomes "active" and lets us choose Elements on the webpage and view them in the DOM.

The second way to use this is to right-select on something in the page and choose Inspect from the dropdown. This has the same effect as using the pointer version.

If necessary, this can provide your support team with additional information to aid in troubleshooting.
