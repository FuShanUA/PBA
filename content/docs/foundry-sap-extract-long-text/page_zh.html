从 SAP 提取长文本

长文本（也称为 SAPscript 文本或文本对象）是附加在 SAP ERP 对象上的容器，用于在 SAP 系统中容纳长文本。用户可以添加自由文本甚至应用格式，而不会受到常见数据库或应用程序限制的阻碍。用户可以将长文本添加到常见的 SAP 对象，如销售订单、物料或通知中。

长文本以压缩格式存储在 STXL 表中。此表保存所有 SAP 对象的长文本。为了在 Foundry 中可读，需要对其进行解压缩。

前提条件

  • Palantir Foundry 连接器 2.0 for SAP Applications SP16 或更高版本

提取长文本

Foundry SAP 连接器具有在将 STXL 表发送到 Foundry 之前解压缩长文本的功能。需要向配置表中添加一条新记录以启用此功能。要添加新记录：

  1. 运行事务 /n/PALANTIR/DECOMPRESS

  2. 在配置表中，填写以下连接器参数：

    1. OBJECT TYPE: SLT, TABLE 或 REMOTETABLE（根据您的设置而定）
    2. OBJECT: STXL
    3. FIELD: CLUSTD
    4. ITEM NO: 1
    5. INTERFACE COMPONENT: DECOMPRESSION_LRAW

  3. 创建一个新的同步并导入长文本表。由于 STXL 表通常非常大，最好按对象名称筛选该表。

    同步配置的格式如下：

    Copied!
    1type: magritte-sap-source-adapter
2sapType: <slt>/<table>/<remotetable>
3obj: STXL
4context: <SLT_Context>/<Remote_Agent_Context>
5filter: <filters>

    一个同步示例：

    Copied!
    1type: magritte-sap-source-adapter
2sapType: table
3obj: STXL
4filter: TDOBJECT=QMEL

    您可以根据对象名称和文本 ID 筛选 STXL 表。例如：

    • 通知对象: TDOBJECT=QMEL
    • 通知对象标题长文本: TDOBJECT=QMEL;TDID=LTXT
    • 采购订单标题文本: TDOBJECT=EKKO
    • 采购订单条目文本：TDOBJECT=EKPO
