可用的音频模型

录音和同意是您的责任

在部署任何记录或转录人类语音的应用程序之前，请确保参与者已被告知，并且在您运营的司法管辖区要求的情况下，已设有同意流程。这可能适用于参与者是应用程序用户、通话中的第三方，或者任何被录音的人。参见 录音、转录和同意。

Palantir AIP 支持两种类型的音频模型：用于构建对话式语音应用的实时语音到语音模型，以及用于将音频转换为文本的转录模型。要了解如何使用浏览器应用程序中以 Foundry 用户身份认证的音频模型，请参阅 构建一个具有语音功能的 OSDK 应用程序。有关 Foundry 上音频的概览，请参见 实时音频。

模式 列表示每个模型支持的使用路径：

  • 实时: 双向流 WebSocket。该模型处理语音活动检测、轮流说话和流式输出；应用程序在采集时发送原始音频。
  • 静态: 请求-响应 HTTP。应用程序发送一个有限的音频负载并接收转录文本返回。适合转录之前录制的音频文件。如果应用程序自行进行音频分块处理，也可以用于实时转录——这是一种以增加应用复杂性为代价来降低每分钟成本并使其更可预测的权衡。当你的注册中没有实时模型时，这种方法也有帮助。延迟取决于分块大小和模型，因此请根据您的使用场景进行评估。

实时语音到语音

实时语音到语音模型接受流式音频输入，在对话过程中进行工具调用，并通过 WebSocket 连接返回流式音频输出。

转录模型将音频转换为文本。大多数支持实时（流式 WebSocket）和静态（请求-响应 HTTP）两种模式；Whisper_large_v3 是 Palantir 托管的，仅支持静态模式。

LLM 可用性前提条件

AIP 是模型无关的，并支持多种模型用于 LLM 驱动的用例；更多信息，请参见 AIP 中所有可用模型的列表。

然而，LLM 的选择和可用性会根据特定模型在注册中可用的某些先决条件而有所不同。这些先决条件决定了模型家族在控制面板中显示为 已启用、已禁用 或 被禁止。了解更多关于 模型启用界面中的模型状态 的信息。

以下列出了相关标准：

  • 模型已与 AIP 集成： Palantir 努力支持行业最新的旗舰模型，并根据模型的发布和更新积极开发支持。
  • 在需要时已提供法律确认： 注册管理员必须接受供应商的法律要求和使用条款，才能启用某些模型。这通常可以在控制面板的 模型启用 选项卡下完成。等待法律确认的模型处于 已禁用 状态。
  • 已在注册中启用 AIP 以使用 LLM： 在 AIP Logic、Transforms、Functions 和 Pipeline Builder 等产品中使用时，必须为指定用户组启用 AIP 功能的权限。
  • 区域可用性兼容性（针对外部供应商模型）： 对于GPT、Claude和Gemini等模型，如果您处于地理限制的注册状态，则可能需要考虑区域可用性。例如，在各自模型供应商发布时，GPT4o和Claude 3 Sonnet仅在美国可用，之后才在欧盟、英国和其他地区推出。请查看模型地理限制部分以获取详细信息。由于地理限制不可用的模型会显示为不允许状态。
  • 已进行额外审查（针对某些Palantir提供的模型）： 开源模型如Llama和Mixtral可能需要额外的Palantir工程审查以支持您的环境。需要这些审查的模型会显示为不允许状态。
  • 与特定AIP前端产品集成所需的时间： 新的LLM需要时间才能在所有AIP产品中得到完全支持（例如，在AIP Logic和Pipeline Builder的使用LLM节点功能中）。
  • 风险考虑（针对实验性模型）： 由于实验性模型可能会出现故障或需要手动迁移到更新的模型，我们限制其发布，客户在使用前可能需要确认该情况。"实验性"一词由模型提供方定义，并不适用于生产环境使用。

按地理位置的 LLM 可用性

由于地理限制（简称地缘限制），某些注册可能只能访问有限的一组模型；特定区域的地缘限制意味着任何 AIP 请求到 LLM 都会在该区域边界内进行。例如，如果一个注册被定义为欧盟地缘限制，则所有 LLM 请求都会在欧盟处理。非地缘限制的注册可以访问 Palantir 支持的所有模型。
