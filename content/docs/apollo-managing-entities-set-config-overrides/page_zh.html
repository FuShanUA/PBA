设置配置覆盖

实体通常需要一定程度的环境定制。当你的发布版本中包含默认配置时，Apollo 的 Config 功能可用于覆盖默认配置的部分内容。

配置覆盖可以从 Config 选项卡中的实体进行指定。这些覆盖可能根据实体类型的不同而被解释为不同的内容。配置更改需通过变更请求，并必须经过 实体操作员 的批准。这创建了一个强大的配置更改历史记录，为生产环境提供了清晰的审计和合规路径，并提供了验证的机会。

配置覆盖应用于产品的版本范围。Apollo 将 永远 不会部署没有为该版本定义配置覆盖的产品发布。这意味着如果希望部署某个产品，至少必须提供一个声明支持该产品主要版本的空配置覆盖块。

配置的顶级键与该配置适用的主要版本范围中的起始版本相关。下面的示例声明了对主要版本2的任何产品发布的基本支持。请注意，主要版本1或3，或其他任何版本的发布如果没有为这些主要版本特别指定覆盖块，则无法部署。

可以为产品的不同版本范围提供不同的覆盖配置。当部署一个版本时，会选择与该版本最具体的覆盖范围。此配置解析过程与升级计划的推荐同时发生，因此您可以确保在部署版本的同时部署最合适的配置，以避免因配置值或功能标志对不同支持情况而产生的竞争条件和边缘情况。

在此示例中，声明了对产品主要版本1的所有版本（大于等于1.2.0）的支持。也声明了对主要版本2的所有版本的支持。对于主要版本2中2.6.0或更新的任何版本，还将为实体提供一些额外的配置覆盖值。
