高级

高级组件类别包含以下组件：

  • 代码沙盒
  • 文件导入
  • Iframe

代码沙盒

代码沙盒组件提供了一个安全的沙盒环境，让您可以实现自定义组件以扩展 Slate 的功能。您可以构建自定义可视化效果、利用第三方 JavaScript 库，或构建高级工作流交互。您可以定义渲染、组件模型和事件交互，使您的组件与应用程序的其余部分集成。外部 JavaScript 库可以加载到您的项目文件夹中，并在组件中引用。

警告

代码沙盒组件在您的 Slate 应用程序中解锁了高级自定义开发功能。通过代码沙盒组件实现自定义功能时，应充分理解其中涉及的技术复杂性以及成功开发所需的长期维护工作。组件开发由您自行决定并承担风险；不提供针对自定义代码调试的支持。

交互摘要

下面是 Slate 的交互摘要以及可用的功能：

JavaScript 选项卡

此选项卡包含 JavaScript 的字段，以及您可能希望加载的任何 JavaScript 库。

JavaScript

这是一段 JavaScript 代码，会在组件加载时执行。对该 JavaScript 代码进行任何修改都会导致整个组件重新加载。此字段中的 JavaScript 代码会在库字段中的 JavaScript 代码之后执行。有关加载第三方库的说明，请参见 JavaScript 库章节。

请勿在此处使用 Handlebars。我们建议您通过 JavaScript 选项卡使用 SlateFunctions 传入数据，以便与交互选项卡中的状态进行交互。

警告

在 JavaScript 的代码沙箱、JavaScript 库或 HTML 中，不支持网络请求和引用（例如使用 fetch）。任何网络请求都必须通过 SlateFunctions 代理到已配置的查询。

可用函数

以下原生函数可被调用，以与 Slate 的特定功能进行交互。这些函数允许您通过事件、动作和状态更改来与 Slate 应用程序的其余部分进行交互。这些函数会暴露给在组件内运行的 JavaScript。更多信息及示例如下所示。

  • SlateFunctions.onAction
  • SlateFunctions.watch
  • SlateFunctions.getState
  • SlateFunctions.setState
  • SlateFunctions.triggerEvent

JavaScript 库

这是一个项目路径数组，会被下载并按顺序在组件内执行；或者是一个同时被 CORS 和 CSP 策略允许的 URL 数组（可以在其中混用）。

下载过程会使用浏览器，因此该 URL 必须被 CORS 和 CSP 策略允许。如果您希望在 Foundry 中托管库，可以将其托管在 Blobster 中并使用 cookie 认证的 API，或者将脚本放在资产服务器上。对此字段进行修改会导致整个组件重新加载并刷新。（库本身可以调用 SlateFunctions 来与 Slate 的特定功能进行交互）。

导入的 JavaScript 库必须将其功能分配给全局可用的作用域，以便通过 JavaScript 选项卡进行引用（例如打包为 UMD 模块，或显式将功能分配给 window）。

示例

要使用 d3.min.js 库，请从 https://d3js.org/d3.v5.min.js 下载，并通过拖放将其保存到 Foundry 目录中。 文件上传后，复制其 项目路径并将其粘贴到库数组中。

["/Users/admin/d3.min.js"]

或者，您可以使用该资源的 RID。例如：

["ri.blobster.main.code.7a4a12a8-e9f5-46ef-8008-5c3f4bbd4abc"]

您可以通过查看资源的文件目录来获取项目的路径/RID。或者，在文件夹中选中该文件后右键单击，复制其位置/RID。

HTML/CSS 选项卡

HTML

这是一个 HTML 字符串，将在组件加载时渲染。

请勿在此处使用 Handlebars。HTML 标签 <script> 将不起作用，您需要将所有 JavaScript 提取到 JavaScript 选项卡中。

CSS

这是一个 CSS 字符串，将在组件加载时渲染。

请勿在此处使用 Handlebars。指定的 CSS 允许您覆盖在 iframe 边框中渲染的 CSS，而不是在框架内渲染。同样的边框样式也可以通过组件顶级样式选项卡中的附加 CSS 类和自定义样式来应用。

CSS 库

CSS 库功能允许您加载 CSS，以便使用 CSS 样式（例如 Blueprint）来创建自定义小部件。您可以在小部件的 HTML/CSS 标签页中访问该功能。

CSS 库的工作方式类似于代码沙盒的 JavaScript 库。CSS 库接受一个项目路径数组，这些路径对应的文件将被下载并按顺序在小部件中渲染；或者接受一个同时被 CORS 和 CSP 策略允许的 URL 数组（您可以混合使用）。下载过程将通过浏览器进行，因此 URL 必须获得 CORS 和 CSP 策略的许可。如果希望在 Foundry 内部托管 CSS 库，您可以将其托管在 Blobster 中并使用带有 Cookie 身份验证的 API，或者将脚本放置在资产服务器上。修改此字段会导致整个小部件重新加载并刷新。

对于 CSS 库文件的内容，使用纯 CSS 即可；请确保不要在 CSS 样式外包裹 HTML <style> 标签。

交互标签页

该控件允许小部件与 Slate 范式进行交互。代码沙盒小部件与 Slate 其余部分之间的所有交互，都应通过状态、事件或操作之一来传递。

我们建议您仅通过此交互标签页传入 Handlebars。直接在 JavaScript 框中使用 Handlebars 仍然有效（例如，在不使用 SlateFunctions.watch 或 SlateFunctions.getState 的情况下传入某些状态），但不建议这样做，因为这会导致每次操作时整个小部件都重新加载并刷新。

状态

这是一个 JSON 数据块，表示当前配置的组件状态。这类似于 Slate 中其他组件使用的状态，不同之处在于它嵌套在此字段下，以支持其他元数据字段。该状态既可以在组件内部修改，也可以通过常见的在状态中使用 Handlebars 的方式来修改。

要在组件内部修改状态，请在 JavaScript 中使用 SlateFunctions.setState。其他用于将状态传递给组件的 Slate 函数还有 SlateFunctions.watch 和 SlateFunctions.getState（见下方示例）。

可以通过某些方式利用此字段将额外的运行时代码加载到 Code Sandbox 组件中（例如，将 JavaScript 传入状态并在稍后加载）。但是，不推荐这种做法，因为该字段的用途是表示组件的状态。

事件

这是一个字符串数组，包含此组件能够触发的事件名称。这些触发器必须使用下方提供的函数，从组件内部的 JavaScript 中显式调用。在事件选项卡中显示时，事件将被命名为 custom.{event_name}。在 triggerEvent 参数中无需单独输入事件名称。

要触发事件，请在 JavaScript 中使用 SlateFunctions.triggerEvent(“event”)，如下方示例所示。

操作

这是一个字符串数组，包含此组件允许其他组件触发的操作名称。组件内的 JavaScript 应使用下方提供的函数监听该操作。在事件选项卡中显示时，操作将被命名为 custom.{action_name}。在 onAction 参数中无需单独输入操作名称。

要让组件响应在 Slate 上下文中创建的操作，请在 JavaScript 中使用 SlateFunctions.onAction(“action_name”,(value)=>{put JavaScript here})，如下方示例所示。

Slate 函数

这些原生函数允许您与 Slate 特有的功能进行交互，并向组件内运行的 JavaScript 暴露这些接口。通过事件、Action 和状态变更，您可以利用这些函数与 Slate 应用的其余部分进行交互。

onAction

数据流向：Slate 上下文 → 组件

SlateFunctions.onAction 允许您注册一个回调函数，当组件上触发 Action 时执行该回调。其参数为 Action 的名称以及接收到该 Action 后要执行的函数。调用该函数时，唯一传入的参数是随 Action 一同传递的 'body'。

随后，您需要在下方的“交互”选项卡中列出您的 Action。

示例

  • 在 Slate 的事件面板中，已注册该 Slate 事件以触发代码沙盒组件中的 Action (custom.checkbox)。
  • 事件面板能够检测到此 Action，因为该 Action 已在代码沙盒组件的“交互”Action 框中注册。

监听

数据流向：Slate 上下文 → 部件（持续监听）

SlateFunctions.watch 旨在帮助你检测状态字段的变更，其机制类似于 AngularJS 中的监听功能。SlateFunctions.watch 接受一个字符串和一个函数作为参数。当该字符串所代表的状态部分发生变化时，传入的函数将被调用，并以新状态和旧状态作为参数。这可能是最实用的函数，你可以通过在“交互”选项卡中传入 Handlebars 来将数据传递给部件（例如 "First Argument": "{{handlebars}}"）。

例如，假设你的状态包含两个字段——高度和宽度——且你希望在高度发生变化时调用某个函数，你可以调用 SlateFunctions.watch("height", <在此处插入函数>)。如果初始字符串为空，则任何状态变更都会触发该函数。

示例

本示例使用了 Slate 的 Input 部件，它将状态传递给 Code Sandbox 部件并显示如下：

  • Slate Input 部件生成文本状态作为数据输出 {{w_widget2.text}}
  • 该状态已被传入 Code Sandbox 部件的交互 状态 框，因此现在你可以在 Code Sandbox 部件中使用该状态。你可以使用 SlateFunctions.watch("State Name", (data) =>... 在部件中调用该状态/数据。

setState

数据流向：组件 → Slate 上下文。

SlateFunctions.setState 用于修改组件状态。这会同时影响通过 Handlebars 引用该组件状态的外部组件，以及后续的 getState 调用。它接受两个参数：一个表示待修改状态部分的字符串，以及一个表示该部分新值的 JSON 数据块。

例如，若想将组件的视图高度设为 4 并保持其他所有属性不变，可调用 SlateFunctions.setState("view.height", 4)。若要覆盖整个状态（而非仅修改高度），可将第一个参数传为空字符串 ""，替代 "view.height"。

示例

本示例在 Code Sandbox 组件中使用了一个生成的按钮，用于将 Slate 的“HTML 组件”状态从“初始组件状态”更新为“更新后的组件状态”。

  • 点击按钮时，与 Code Sandbox 组件的交互会触发 SlateFunctions.setState。这会将“outval”的状态从“初始组件状态”更新为“更改后的组件状态”。
  • Slate 能够检测到更新后的状态，因为该状态已在 Code Sandbox 组件的 Interaction 的“state”中注册。

JavaScript：


```javascript
var button1 = document.createElement("button");
button1.innerHTML = "button1";

button1.onclick = () => {
    SlateFunctions.setState("outval", "已更新组件状态");
};

document.body.appendChild(button1);
```

加载顺序


Code Sandbox 组件按以下顺序加载：


    

  1. Slate（包含该组件的父框架上下文）先加载，随后组件 iframe 加载。
    2. 

  2. Slate 将组件的“状态”发送至组件（详情见下文的“交互”），接着发送 CSS、HTML、库和 JavaScript。
    3. 

  3. iframe 接收并执行以下设置：

      

    • 追加 CSS 和 HTML。
      • 

    • 执行库
      • 

    • 执行 JavaScript
      • 

    • 触发相应的“SlateFunctions”（输入）。
      • 

    

    4. 

  4. 用户与组件框架交互（更新状态、触发事件、创建动作等）。这些操作进而可以修改或生成“SlateFunctions”，并由组件发送回父级 Slate 框架。如果这些组件输出被指定为交互（输出），它们便可在 Slate 的其余部分中使用。
    5. 




安全


该组件的安全模型与 Slate 函数非常相似。代码将在沙盒化的 iframe 中执行，输入输出通过 postMessage 传输。借此，我们可以安全地执行不受信任的 JavaScript 代码。唯一的改动在于该 iframe 在页面上可见，但这并未改变安全模型。


调试技巧


在 Chrome 控制台中，你可以选择默认为顶部的下拉菜单，并选中 codeSandboxIframe.html。这样一来，你在控制台输入的任何 JavaScript 都会在组件环境中执行。由于对 JavaScript 字段的修改会强制重载组件，这在排查如何让组件正常运行时非常实用。
