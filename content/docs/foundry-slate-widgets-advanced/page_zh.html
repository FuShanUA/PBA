代码沙盒

代码沙盒小部件是一个安全的沙盒，您可以在此实现自己的自定义小部件以扩展Slate的功能。您可以构建自己的自定义可视化效果，利用第三方JavaScript库，或构建高级工作流交互。您可以定义渲染、小部件模型和事件交互，从而使您的小部件与您应用程序的其余部分集成。外部JavaScript库可以加载到您的项目中的一个文件夹中，并在您的小部件中引用。

警告

代码沙盒小部件在你的Slate应用中解锁了高级的、自定义的开发功能。通过代码沙盒小部件实现自定义功能，应清楚了解涉及的技术复杂性以及成功开发所需的长期维护。小部件开发由您自行决定并承担风险；不提供对自定义代码的调试支持。

这是一个在小部件加载时将执行的JavaScript字符串。对此JavaScript的任何更改都会导致整个小部件重新加载。此字段中的JavaScript将在库字段中的JavaScript之后执行。有关加载第三方库的说明，请参阅 JavaScript库 部分。

JavaScript库

这是一个项目路径数组，这些路径将被下载并在小部件中按顺序执行，或者是一个由CORS和CSP策略允许的URL数组（您可以混合使用）。

下载将使用浏览器，因此URL必须符合CORS和CSP策略。如果您希望在Foundry中托管库，可以将其托管在Blobster中并使用cookie认证的API，或者将脚本放入资产服务器。更改此字段将导致整个小部件重新加载和刷新。(这些库本身可以调用SlateFunctions以允许它们与Slate特定的功能进行交互)。

导入的JavaScript库必须将其功能分配到全局可用的作用域中，以便通过JavaScript选项卡引用（例如作为UMD模块打包，或显式地将功能分配给window）。

示例

要使用d3.min.js库，请从https://d3js.org/d3.v5.min.js下载它，并通过拖放文件保存到Foundry目录中。文件上传后，复制其项目路径并粘贴到库数组中。

["/Users/admin/d3.min.js"]

你可以使用资源的RID。例如：

["ri.blobster.main.code.7a4a12a8-e9f5-46ef-8008-5c3f4bbd4abc"]

你可以通过查看文件的目录来获取资源的项目路径/RID。或者，你可以在其文件夹中右键点击文件并复制位置 / RID。

CSS库的工作方式类似于Code Sandbox的JavaScript库。CSS库接受一个项目路径数组，这些路径将被下载并在小部件中按顺序渲染，或者是一个由CORS和CSP策略允许的URL数组（您可以混合使用）。下载将使用浏览器，因此URL必须符合CORS和CSP策略。如果您希望在Foundry中托管CSS库，可以将其托管在Blobster中并使用cookie认证的API，或者将脚本放在资产服务器中。对这个字段进行更改会导致整个小部件重新加载和刷新。

交互选项卡

这是使小部件与Slate范式交互的控制项。Code Sandbox小部件与Slate其余部分的所有交互都应通过状态（State）、事件（Events）或操作（Actions）之一传递。

我们建议您仅通过此交互标签页传递Handlebars。在JavaScript框中直接使用Handlebars仍然有效（例如，要在不使用SlateFunctions.watch或SlateFunctions.getState的情况下传递某些状态），但不推荐这样做，因为每次都会重新加载和刷新整个小部件。

状态

这是一个表示该小部件当前配置的JSON块。这与其他Slate中的小部件使用的状态类似，只是它嵌套在该字段下，以允许其他元字段。该状态可以从小部件内部或通过通常将Handlebars放入状态中的方法进行修改。

要从小部件中修改状态，请在JavaScript中使用SlateFunctions.setState。传递状态到小部件的其他Slate函数是SlateFunctions.watch和SlateFunctions.getState（请参见下面的示例）。

可以通过此字段以某些方式将额外的运行时代码加载到代码沙盒小部件中（例如，将JavaScript传入状态并在稍后加载）。然而，这些方法不被推荐，因为该字段的目的是表示小部件的状态。

事件

这是一个字符串数组，表示该小部件可以触发的事件名称。这些触发器必须在小部件内的 JavaScript 中使用下面提供的函数显式调用。当在事件选项卡中显示时，事件名称将被命名为custom.{event_name}。事件名称不需要在 triggerEvent 参数中单独输入。

要触发一个事件，请在您的 JavaScript 中使用 SlateFunctions.triggerEvent(“event”)，如下面的示例所示。

操作

这是一个字符串数组，表示该小部件允许由其他小部件触发的操作名称。小部件内的 JavaScript 应该使用下面提供的函数来监听操作。当在事件选项卡中显示时，操作名称将被命名为custom.{action_name}。操作名称不需要在 onAction 参数中单独输入。

要让一个小部件响应在 Slate 上下文中创建的操作，请在您的 JavaScript 中使用 SlateFunctions.onAction(“action_name”,(value)=>{put JavaScript here})，如下面的示例所示。

Slate 函数

这些原生函数允许您与 Slate 特定的功能进行交互，并暴露给运行在 widget 中的 JavaScript。这些函数将允许您通过使用事件、操作（Action）和状态更改来与整个 Slate 应用程序进行交互。

onAction

数据方向：Slate 上下文 → widget

SlateFunctions.onAction 允许您注册一个回调，当您的 widget 上调用操作（Action）时会触发该回调。其参数是操作（Action）的名称以及当接收到操作（Action）时要调用的函数。该函数将使用传入操作（Action）的 'body' 作为唯一参数进行调用。

然后您需要在下面的 Interaction 标签页中列出您的操作（Action）。

示例

此示例使用Slate复选框小部件，当点击复选框小部件时，会触发代码沙盒小部件更新：

  • 当复选框小部件被点击时，会从复选框小部件发出一个事件。
  • 在Slate的事件面板中，这个Slate事件已被注册为代码沙盒小部件中的一个操作（custom.checkbox）。
  • 事件面板能够检测到这个操作，因为该操作已在代码沙盒小部件的交互 操作 框中注册。

2    var checkbox = document.createElement("div")
3    checkbox.innerHTML = value;
4    document.body.appendChild(checkbox);
5});

watch

数据方向：Slate上下文 → 小部件（持续监控）

SlateFunctions.watch 的作用是帮助您以类似于 AngularJS 中的 watch 方式检测状态字段的变化。SlateFunctions.watch 接收一个字符串和一个函数作为参数。当由该字符串表示的状态部分发生变化时，将使用新的状态和旧的状态作为参数调用提供的函数。这可能是最有用的函数，您可以借此通过传入到 Interaction 选项卡中的 Handlebars 将数据传递到小部件（例如 "First Argument": "{{handlebars}}"）。

例如，如果您的状态有两个字段 - height 和 width - 并且您希望在 height 变化时调用一个函数，您将调用 SlateFunctions.watch("height", <insert function here>)。如果初始字符串为空，则会在任何状态变化时调用该函数。

示例

此示例使用了 Slate 的 Input 小部件，它将状态传递给 Code Sandbox 小部件并显示它：

  • Slate 的 Input 小部件生成文本状态作为数据输出 {{w_widget2.text}}
  • 此状态已被输入到 Code Sandbox 小部件的 Interaction state 盒子，现在你可以在代码沙盒小部件中使用状态。你使用SlateFunctions.watch("State Name", (data) =>...来在小部件中使用状态/数据。

JavaScript:

Copied!
1var el =  document.createElement("div")
2el.id = "display"
3document.body.appendChild(el);
4
5SlateFunctions.watch("Input State", (data) => {
6    el.innerHTML = "Code Sandbox widget: " + data;
7});

交互操作：

Copied!
1{
2  "Input State": "{{w_widget2.text}}"
3}

getState

数据方向：Slate上下文 → 小部件（一次性获取）

此函数返回由状态字段初始填充的小部件的当前状态。

SlateFunctions.getState 返回由状态字段初始填充的小部件的当前状态的JSON对象。你可以访问该对象的不同属性（即不同的“状态”）。

SlateFunctions.getState 将返回你的状态字段的JSON对象，你可以访问该对象的不同属性（即不同的“状态”）。

SlateFunctions.getState 只有在被调用时才会获取状态。相比之下，SlateFunctions.watch 会持续“监视”状态字段的更新。你可以使用这个 SlateFunctions.getState 函数在 SlateFunctions.watch 中获取完整状态对象的特定属性。

例如，如果你的状态是：

setState

数据方向：组件 → Slate 上下文。

SlateFunctions.setState 既修改了组件的状态，也适用于那些使用 Handlebars 引用此组件状态的外部组件，以及未来的 getState 调用。它接受两个参数：表示要修改的状态部分的字符串和表示该部分新值的 JSON 数据块。

例如，如果您希望将小部件的视图高度设置为 4，但保留所有其他属性，您将调用 SlateFunctions.setState("view.height", 4). 如果您想覆盖整个状态（而不是仅高度），您可以将第一个参数替换为 "" 而不是 "view.height"。

示例

此示例使用代码沙盒小部件中的生成按钮来更新 Slate "HTML 小部件" 的状态，从 "初始小部件状态" 到 "已更新的小部件状态"。

  • 当点击按钮时，与代码沙盒小部件的交互会触发 SlateFunctions.setState。这会将 "outval" 的状态从 "初始小部件状态" 更改为 "已更改的小部件状态"。
  • 由于状态已在代码沙盒小部件的交互 "state" 中注册，因此可以在 Slate 中检测到此更新状态。

JavaScript:

Copied!
1var button1 =  document.createElement("button");
2button1.innerHTML = "button1";
3
4button1.onclick = () => {
5    SlateFunctions.setState("outval", "Updated widget state");
6};
7
8document.body.appendChild(button1);

交互状态：

Copied!
1{
2  "outval": "Initial Widget State"
3}

加载顺序

代码沙盒小部件按以下顺序加载：

  1. Slate（包含该小部件的父框架上下文）加载，然后加载小部件的iframe。
  2. Slate将“状态”发送给小部件（有关更多详细信息，请参见下面的交互部分），然后是CSS、HTML、库和JavaScript。
  3. iframe接收并设置以下内容：
    • CSS和HTML被追加。
    • 库被执行
    • JavaScript被执行
    • 触发适当的“SlateFunctions”（输入）。
  4. 用户与小部件框架进行交互（更新状态，触发事件，创建操作等）。这些反过来可以修改或生成“SlateFunctions”，这些函数从子小部件发送到父Slate框架。如果这些小部件输出被指定为交互（输出），则可以在Slate的其余部分中使用它们。

安全

此小部件的安全模型与Slate函数非常相似。代码将在沙盒iframe中执行，输入和输出通过postMessage进行传输。这使我们能够安全地执行不可信的JavaScript代码。唯一的变化是iframe在页面上可见，但这不会改变安全模型。

调试提示

在Chrome控制台中，您可以选择默认为顶部的下拉菜单并选择codeSandboxIframe.html。这将导致您在控制台中输入的任何JavaScript代码在小部件的环境中执行。当您试图弄清楚如何让您的小部件工作时，这会很有帮助，因为对JavaScript字段的更改会强制重新加载小部件。

第三方代码

我们建议在使用第三方代码时遵循以下一般方法：

  • 检查许可证，以确保您可以使用它。
  • 通过找到尽可能少的独立JavaScript代码块示例来最小化代码复杂性。
  • 如果JavaScript使用了任何库，请转到下载源文件中的.js文件，并将上传文件的Foundry项目路径放入库部分。您也可以直接链接URL并在之后进行测试，以确保它不会与CSP或CORS策略冲突。
  • 确保HTML中的<script></script>标签内的所有JavaScript都被重构并放置到JavaScript部分。
  • 使用Slate函数进行交互：
    • 使用SlateFunctions.getState从Slate的自定义代码沙箱中传递您需要的任何数据。
    • 要将小部件内的交互传达给Slate的其余部分，您可以将SlateFunctions.triggerEvent标记到该函数上。
    • 要让小部件响应来自Slate其余部分的操作，您可以将SlateFunctions.onAction标记到该函数上。
