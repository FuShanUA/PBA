创建交互式音频转录应用程序

本指南将介绍如何构建一个工作流来转录音频，并使用媒体集创建用于查看转录内容的交互式应用程序。

第一部分：在 Foundry 中将音频文件导入为媒体集

首先，您需要将音频文件导入为媒体集。有两种方式可以实现此操作：

  • 直接上传
  • 数据连接

导入完成后，您便可以查看您的音频媒体集。

第二部分：通过管道构建器转录音频媒体集

  1. 在管道构建器中创建新管道，并将您的音频媒体集添加到该管道中。详细步骤请参阅管道构建器文档中的初始设置部分。

    您导入的音频媒体集应如下所示：



  2. 接下来，使用转换选择转录音频转换操作。



  3. 为转录音频转换指定输入项并选择应用。



    使用媒体集输入中的 media_reference 列，并选择所需的语言。如果未提供语言，系统将从音频的前 30 秒内容中进行推断。此外还有多个配置选项可用。在此示例中，我们希望输出结果包含说话人分离信息，因此我们将选择更高性能模式、分段详情输出类型，并开启说话人识别。

  4. 您可以在表格中预览转录的输出结果。



    如果您不打算使用转录组件，可以根据需要继续对音频转录的字符串输出进行转换。

    按照需求完成转录的转换后，您可以将其输出为数据集，或通过选择对象类型输出将结果本体化。如果执行了此操作，您可以在此停止跟随本指南。

    如果您希望使用转录组件，请继续阅读下文的第三部分，为该组件创建转录分段。

第三部分：将转录分段本体化以用于 Workshop 组件

在本节中，我们将从转录输出中创建分段对象，这些对象包含在转录组件中显示所需的属性。

  1. 使用展开数组转换，将分段结构体数组转换为单独的行。

  2. 然后，应用提取多个结构体字段来提取组件所需的字段。我们将选择以下字段：id、begin、end、contents 和 speaker_id。由于我们已经知道了说话人的姓名，将使用值映射将说话人 ID 转换为说话人姓名。



  3. 通过选择对象类型输出将结果本体化。了解有关如何保存管道输出的更多信息。确保将 Id 属性设置为主键，以便每个分段对象都具有唯一的键。



第四部分：在 Workshop 中显示转录内容

  1. 在 Workshop 中，添加音频与转录显示组件。



  2. 使用在第三部分中创建的对象类型作为分段对象集来配置该组件。有关配置选项的完整枚举，请参阅《音频与转录显示组件》文档。

  3. 配置要在组件中使用的操作。

    你可以在片段对象集上创建 操作类型，并在组件中将其展示出来。例如，你可能希望允许用户编辑片段内容或更正时间戳。

    在本示例中，我们将配置一个简单的操作，允许用户编辑片段的说话人属性。

  4. 定义好操作类型后，在 Workshop 中配置该操作。勾选 启用操作，并选择你在对象类型上创建的操作。你可以配置图标和名称。当鼠标悬停在某个片段上时，这些配置将显示在片段的工具栏中。

    你还可以配置 参数默认值，以便用悬停片段中的数据填充默认值。我们将使用 选定的片段 作为操作要编辑对象的参数默认值。



    在这条示例工作流中，我们现在可以在组件中检查片段时，更正其说话人属性。
