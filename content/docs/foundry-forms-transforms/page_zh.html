转换

Foundry 表单（Foundry Forms）不再是 Foundry 上数据输入或写回工作流的推荐方法。相反，应使用 Foundry 本体论（Foundry Ontology）构建用户输入工作流，将相关数据结构表示为对象类型，并配置与操作（Actions）的写回交互。在 表单概述 文档中了解更多信息。

转换（Transforms）是一组工具，允许用户在其表单中创建动态依赖关系。通过指定字段的条件修改，用户可以更高效、准确地引导受访者的数据。

在创建并配置新字段后，可以访问转换（Transforms）。有五种类型的转换，所有这些都可以组合起来，使受访者在填写表单时产生不同的结果。

隐藏字段

hide field 转换允许用户根据其他字段的值从受访者中隐藏一个字段。通过减少不相关字段的杂乱，此转换可以加快受访者填写表单的速度。

示例

Hide Field B if Field A has value X.

显示字段

show field 转换允许用户根据其他字段的值向受访者显示一个字段。类似于 hide field，此转换可以加快受访者填写表单的速度，只提供相关的字段。

示例

Show Field B if Field A has value X.

禁用字段

disable field 转换允许用户根据其他字段的值来禁用一个字段。与 hide field 转换不同，使用 disable field 转换的字段仍然可见；这些字段可以配置为显示对受访者重要的信息，引导他们到未回答的字段或以某种方式现在被阻止输入值的字段。

示例

Disable Field B if Field A has value X.

更新配置

update configuration转换允许用户根据其他字段的值来更新字段的配置。几乎所有选项都可以更改，排除那些必须保持不变的选项，包括tag、uri、defaultValue、transforms本身以及在附件（Attachments）字段中可用的选项。

示例

Add [required validator] to Field B if Field A has value X.

更改字段类型

change field type转换允许用户根据其他字段的值来更改字段的类型。

示例

Change type of Field B to [Text Area] if Field A has value X.

添加转换

要向字段添加转换，请完成以下步骤：

  1. 首先，双击一个字段以在右侧打开可视化编辑器。

  2. 从转换选项卡中，选择添加新转换，然后选择一种类型。

  3. 选择转换所依赖的字段（可用选项将以紫色突出显示）。

  4. 配置条件；例如，Has no value。

  5. 如果使用更新配置或更改字段类型转换，请定义结果。

  6. 选择绿色的保存按钮。

创建复杂条件

在添加具有简单条件的转换后，可以按照以下方式创建更复杂的条件：

  1. 选择视觉编辑器的“Transforms”选项卡顶部右上角的铅笔图标。
  2. 在现有转换条件下方选择+，然后选择配置新条件。
  3. 使用Is/Not和And/Or下拉菜单创建复杂的条件树。

将鼠标悬停在Is/Not下拉菜单上，以了解哪些条件通过And/Or分组在一起。

示例

Show Field C if Field A has value X and Field B does not have value Y.

定义多个转换

可以通过点击添加新转换来为单个字段添加多个转换 面板底部的按钮。转换是按照定义的顺序应用的；如果有任何冲突，最后的转换将具有优先权。

示例

Change [Label] of Field B to "Zip Code" if Field A has value X. Add 5-digit [regex validator] to Field B if Field A has value X.

一个字段不应该同时具有 show 和 hide 转换（转换）。如果您希望该字段默认隐藏，可以合并为一个 show 转换（转换），否则可以合并为一个 hide 转换（转换）。
