将安全信息添加到发布评估中

您可以添加两个标签作为发布通道的要求，以根据漏洞扫描来限制发布：

  • 根据正在运行的扫描来限制发布
  • 根据扫描结果来限制发布

根据正在运行的扫描来限制发布

如果您希望防止所有环境中的漏洞，并且已启用基于漏洞的自动召回，我们建议选择此选项。这将确保在发布安装到您的环境中之前，有足够的时间进行扫描。

Apollo 会向所有经过漏洞扫描的产品发布添加一个名为 产品发布标签 的 vulnerability-scanner.palantir.build/security-scanned 标签。此标签可以具有值 true 和 false。

要基于安全扫描运行来限制发布升级：

  1. 创建一个额外的发布通道，例如 SECURE。
  2. 为此发布通道添加一个 标签要求，使得所有发布都必须具有 vulnerability-scanner.palantir.build/security-scanned 标签。打开 需要值 并输入 true。

根据扫描结果限制发布

如果您希望对某些环境中的漏洞更加宽松，并且尚未想基于漏洞自动召回，请选择此选项。

在扫描发布后，Apollo 会添加一个名为 产品发布标签 的 vulnerability-scanner.palantir.build/security-scan-outcome 标签，用于指示扫描结果。此标签有两个可能的值：

  • pass: 未发现活动漏洞。发布可能包含处于宽限期 SLA 内的漏洞或已被抑制的漏洞。
  • fail：该版本中存在需要修复或抑制的活跃漏洞。

要根据安全扫描结果来限制版本发布：

  1. 确定每个版本通道（Release Channel）对漏洞的容忍度。例如，您可能希望仅允许在 DEV 版本通道中存在漏洞。
  2. 为任何不应接受具有活跃漏洞的版本的版本通道添加一个 标签要求。例如，您可能不会为 DEV 版本通道添加此标签要求，但可能会为 STAGING 和 PRODUCTION 添加。输入标签名称 vulnerability-scanner.palantir.build/security-scan-outcome，然后打开 需要值 并输入 pass。
