在本地设置Kubernetes集群

本指南将指导您在本地计算机上设置一个本地Kubernetes集群。

这些步骤仅适用于开发环境。对于生产环境，您应该使用一个 经过CNCF认证的Kubernetes发行版 ↗。

  1. 在您的本地机器上安装 Docker ↗。

    • 对于macOS，您可以运行以下命令：
    Copied!
    1brew install docker
    • 了解有关 安装Docker的不同方法 ↗ 的更多信息。

  2. 在您的本地机器上安装 kind ↗。您应该在机器上的安全文件夹中安装kind，例如 My Documents。

    • 对于macOS，您可以运行以下命令：
    Copied!
    1brew install kind
    • 了解有关 安装kind ↗ 的更多信息。

  3. 如果您使用的是 Docker 桌面版，请运行以下命令以开始：

    Copied!
    1docker run -d -p 80:80 docker/getting-started

  4. 在您的桌面创建一个空目录，用于存储在 Apollo 中注册集群所需的文件。注意该目录的位置。

  5. 在上一步中创建的目录中，创建一个名为 cluster.yml 的文件，并添加以下配置：

    Copied!
    1kind: Cluster
2apiVersion: kind.x-k8s.io/v1alpha4
3# Apollo requires a multi-node cluster
4nodes:
5- role: control-plane
6- role: control-plane
7- role: control-plane
8- role: worker
9- role: worker
10- role: worker

  6. 在您上面创建的目录中，运行以下命令，使用上一步的配置创建一个新的 Kubernetes 集群。

    Copied!
    1kind create cluster --config cluster.yml

  7. 等待集群安装并启动。

  8. 当完成时，您可以运行 kubectl get nodes 以查看所有已创建的节点。此命令应返回以下输出。请注意，您的 AGE 和 VERSION 信息可能与下面的示例不同。

    Copied!
    1$ kubectl get nodes
2NAME                    STATUS      ROLES               AGE     VERSION
3kind-control-plane      Ready       control-plane       1h      v1.24.0
4kind-control-plane2     Ready       control-plane       1h      v1.24.0
5kind-control-plane3     Ready       control-plane       1h      v1.24.0
6kind-worker             Ready       <none>              1h      v1.24.0
7kind-worker2            Ready       <none>              1h      v1.24.0
8kind-worker3            Ready       <none>              1h      v1.24.0

  9. 现在您可以使用此集群来 在 Apollo 中设置 Spoke 环境。导航到 Apollo 中的 创建环境 工作流以继续设置您的 Spoke 环境。
