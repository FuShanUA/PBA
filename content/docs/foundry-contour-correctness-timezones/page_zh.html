Contour 中的时区

本页介绍了 Contour 可用的时区配置，以及 Contour 的 DateTime 类型。

时区配置

Contour 提供两种不同的时区配置，直接影响日期时间的显示方式：

  1. 分桶时区（看板级设置）： 允许用户按时间对数据进行分组的看板（如图表、直方图或时间序列），要求用户为时间列选择时区。这决定了数据的分组方式。例如，若选择 UTC 时区，数据将按 UTC 午夜起止进行按日分组。为确保所有用户查看到的看板数据保持一致，此设置不可默认采用用户本地时区，必须设定为固定时区。下方展示了图表看板中的配置示例。由于为分析配置了可视化时区，图中说明数据将以 UTC 显示。



  2. 可视化时区（分析级设置）： 该时区用于在用户界面中呈现日期时间。此设置会影响分析中每个看板的数据显示方式。例如，如果看板配置为按 UTC 进行按日分桶，而分析配置为以 UTC+5 渲染时区，那么看板显示的一天将看似从 UTC+5 的早上 5:00 开始并结束。该设置位于设置侧边栏的数据设置部分，可设为默认采用用户本地时区，使每位查看该分析的用户都能在自己的时区下查看所有日期时间。此设置可在设置侧边栏的数据设置部分找到。



    以上述图表看板为例，数据按 EDT 分桶，而分析配置为以 UTC 显示数据。因此，柱状图按 EDT 分桶，却以 UTC 显示，导致分桶时间跨度从晚上 8:00 延伸至次日的晚上 8:00。如果此分析在可视化时区设置中使用本地时区，那么该图表在不同时区用户看来会略有差异，但各分桶所代表的数据完全一致。



    部分较早的分析可能未配置可视化时区设置。在这种情况下，不同看板在显示日期时间所采用的时区行为上会有所不同。

Contour 中的 DateTime 类型

你可以使用 Mesa 语言（一种基于 Java 的专用 DSL）来创建 DateTime 类型。在 Contour 中使用此类型时，DateTime 类型会自动转换为 timestamp 列，并额外生成一列用于标记原始时区。

例如，如果原始数据集中包含 landing_datetime 列，在 Contour 中你会看到两列：landing_datetime_timestampinUTC（将原始 datetime 列表示为 Timestamp 类型）和 landing_datetime_originalTimezone（以 String 类型存储该列的原始时区）。
