请注意，此端点利用了用于对象类型的底层对象同步技术。如果 Restaurant 由对象存储 V2 支持，则没有请求限制。如果 Restaurant 由对象存储 V1（Phonograph）支持，则最多有 10,000 个结果；如果请求超过 10,000 个 Restaurants，将抛出 ObjectsExceededLimit 错误。

聚合

聚合功能允许您对一组数据计算汇总统计信息。它们对于理解大型数据集中的模式和洞察力非常有用，而无需手动分析每个单独的数据点。您可以结合多个聚合操作来创建更复杂的查询，以深入挖掘数据中的信息。

请注意，此端点使用了对象类型的基础对象同步技术。如果Restaurant由对象存储V2支持，则没有请求限制。如果Restaurant由对象存储V1（Phonograph）支持，则结果限制为10,000；如果请求的Restaurants超过10,000，则会抛出ObjectsExceededLimit错误。

在Restaurants上执行聚合。

参数：

  • $select：要执行的一组聚合函数。
  • $groupBy（可选）：用于创建聚合结果的一组分组
  • where（可选）：对特定属性进行过滤。可能的操作取决于属性的类型。

示例查询：

Copied!
1const numRestaurants = await client(Restaurant)
2  .where({ restaurantName: { $isNull: false } })
3  .aggregate({
4    $select: { $count: "unordered" },
5    $groupBy: { restaurantName: "exact" },
6  });

示例API响应：

{

    data: [{
        group: {
            "restaurantName": "Restaurant Name"

                name: "count",

函数

函数（有时称为“对象上的函数”或“FOO”）在Palantir平台上是一个强大的功能，旨在增强数据建模和操作。函数提供了一种方式，在Ontology中存储的数据上定义和执行自定义逻辑，使用户能够创建更复杂的数据转换、验证和分析。

在TypeScript SDK中，用户可以通过生成的函数定义来执行Foundry函数。

通过将您的函数添加到应用程序中，您可以生成调用对象上的函数以执行逻辑并获取结果的代码。

在这个例子中，我们有一个函数 findSimilarRestaurants，它接受一个ID并返回包含所有类似的 Restaurants 的对象集。
