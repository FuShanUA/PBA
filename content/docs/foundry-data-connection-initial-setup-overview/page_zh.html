初始设置概述

本指南将引导您完成将组织数据连接到 Foundry 的全过程。

在开始之前，务必明确一点：将组织数据接入 Foundry 的这第一步，本质上属于网络概念。初始设置最好由熟悉网络工程并了解组织网络拓扑和配置（如防火墙规则）的人员来完成。

概念概述

要将数据接入 Foundry，需按以下顺序配置三个组件：

1. 配置网络连通性

首先，必须确保 Foundry 与外部系统之间存在有效的网络路径。

对于部署在与 Foundry 相同网络中的外部系统（对于云端托管的 Foundry 实例，通常指可通过互联网访问的系统），需定义直接连接出站策略来路由流量。确保外部系统允许来自 Foundry 的入站连接。

对于部署在与 Foundry 不同网络中的外部系统（对于云端托管的 Foundry 实例，通常指本地部署系统），需要使用智能体。智能体是 Palantir 提供的软件，运行在您组织的网络内部，作为组织数据源与 Foundry 实例之间的安全中介。请确保外部系统允许来自智能体的入站连接，同时智能体也能与外部系统及 Foundry 建立出站连接。

随后，在使用 Foundry worker 时，可通过该智能体定义智能体代理出站策略，以通过智能体路由流量。

智能体也可用作传统的智能体 worker，各项功能将在智能体宿主机上执行。新增数据源应使用 Foundry worker；详情请参阅 Foundry worker 与智能体 worker 对比。

智能体 worker 数据源和使用智能体代理出站策略的数据源可以共享同一个智能体，但我们建议始终为特定数据源分配多个智能体，以最大程度保障可用性。

进一步了解各类架构模式。

2. 数据源配置

在执行任何功能之前，必须配置数据源（或连接），将外部系统接入 Foundry。外部系统可以是 Postgres 数据库、S3 存储桶、Linux 服务器上的文件系统、SAP 实例，或者可通过互联网访问的 REST API。

3. 功能配置

配置好用于连接外部系统的数据源后，必须配置要在该数据源上执行的功能。这些功能包括数据批量同步、流式同步、Webhook、数据导出等。

以批量同步为例，该功能从外部系统读取特定数据并将其摄取到 Foundry 中。假设您有一个包含多个表的 PostgreSQL 数据库，您可以配置同步任务，将其中某个特定的表摄取到 Foundry。同步任务成功执行后，Foundry 中将生成一个数据集，供所有 Foundry 数据管道、模型开发及分析工具使用。

角色与工作流

大多数 Foundry 用户无需自行设置新智能体。智能体设置需要具备 IT 相关技能，但同一个智能体可被重复用于支持多个数据源和同步任务。部分组织仅凭 Foundry 部署首周设置的智能体即可长期稳定运行。只有需要访问现有智能体无法触及的数据（例如由于网络隔离或数据规模限制），或者需要增设智能体以实现高可用性时，才需要部署新智能体。

高可用

建议通过配置冗余硬件来构建高可用（HA）架构。高可用能够提升系统容错能力，并支持在工作时间内进行零停机维护。

Foundry 在数据源级别提供高可用支持，即如果将某个数据源分配给多个智能体，Foundry 会将数据摄取任务分发给其中运行状态正常的智能体。强烈建议在创建数据源之初就配置高可用环境；若要在已创建的数据源中新增智能体，则需要重新输入该数据源的凭证。

搭建高可用环境时，建议遵循以下最佳实践：

  • 始终将智能体成对部署在硬件配置相似的设备上。
  • 为成对的智能体赋予相似的名称，例如 agent-1 和 agent-2。
  • 系统性地将成对的两个智能体分配给每一个数据源。
  • 为成对的两个智能体配置互不重叠的升级时间窗。升级时间窗应设在工作日，并预留充足的预热观察时间。这样做可确保更新过程中出现的任何意外问题仅局限于单个智能体，便于操作人员或管理员及时发现。
