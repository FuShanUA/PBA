Couchbase

Couchbase 连接器是 Palantir 针对 Couchbase 提供的驱动程序。

要创建新的 Couchbase 源，请按照《Palantir 提供驱动程序的标准设置流程》操作，然后使用以下章节进行 Couchbase 特定的配置和网络设置。有关完整的属性参考，请查阅《官方 Couchbase 驱动程序文档 ↗》。

配置

以下属性为必填或推荐配置。

属性是否必填?描述默认值
AuthScheme ↗必填连接 Couchbase 时使用的身份验证类型。Basic
ConnectionMode ↗必填决定如何连接到 Couchbase 服务器。必须为 Direct 或 Cloud。Cloud
CouchbaseService ↗必填决定要连接的 Couchbase 服务。默认为 N1QL。可用选项为 N1QL 和 Analytics。Analytics
DNSServer ↗必填决定检索 Couchbase Capella 信息时使用的 DNS 服务器。8.8.8.8
Server ↗必填要连接的 Couchbase 服务器地址。https://couchbase.server
UseSSL ↗必填连接 Couchbase 服务器时是否协商 TLS/SSL。TRUE

网络连接

下表列出了源为成功运行而需要能够访问的域名。

对于每个域名，添加相应的《流出策略》。如果源托管在本地并且无法从 Foundry 直接访问，请改用《智能体代理流出策略》；智能体主机本身也必须能够访问列出的域名。有关详细信息，请参阅《使用智能体作为代理》。

域名是否必需
<Server>如果 ConnectionMode=Direct（默认）；服务器连接属性。可以是 IP 地址或 HTTP/S URL。可接受多个 URL
<DNSServer>如果 ConnectionMode=Cloud，DNS 服务器用于查找服务器地址（默认端口 53，端口可以通过 <Server>:<Port> 传递，但非必填）
<N1QLPort>如果 ConnectionMode=Direct 且 CouchbaseServer=N1QL，使用 SSL 时端口默认为 18093，不使用 SSL 时为 8093
<AnalyticsPort>如果 ConnectionMode=Direct 且 CouchbaseServer=Analytics，使用 SSL 时端口默认为 18095，不使用 SSL 时为 8095
<WebConsolePort>如果 ConnectionMode=Direct，使用 SSL 时端口默认为 18091，不使用 SSL 时为 8091
