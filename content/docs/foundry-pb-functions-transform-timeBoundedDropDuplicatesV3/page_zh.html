时间范围去重

支持：流处理

从输入中删除给定列子集的重复行，看到的行将在配置的事件时间后过期。比配置的事件时间晚到达的行将始终被删除。按指定的键进行分区。每个去重操作将针对不同的键列值单独计算。

转换类别: 其他

声明的参数

  • 数据集: 需要去重行的数据集。
    表
  • 键过期时间单位: 用于等待数据去重的时间量的单位。
    枚举<天, 小时, 毫秒, 分钟, 秒>
  • 键过期时间值: 用于等待数据去重的时间量的值。
    字面量<长整型>
  • 可选 列子集: 如果指定了任何列，则仅使用这些列来确定唯一性，否则将隐式使用流所基于的键子集来确定唯一性。
    集合<列<任意类型>>
  • 可选 驱逐窗口滑动: 驱逐的滚动窗口的值，表示过期状态将被驱逐的频率。当事件时间超过指定的超时时间时，状态被视为过期。在（key_expiry : key_expiry + eviction_slide] 之间将删除重复项，因为最后一次看到重复项。更改此值被认为是状态中断，并需要重新播放。
    Tuple<Literal<Long>, Enum<Days, Hours, Milliseconds, Minutes, Seconds, Weeks>>
  • 可选 按列分组: 用于按键对输入进行分区的列。每个键值的重复项将分别并行计算。
    Set<Column<Binary | Boolean | Byte | Double | Float | Integer | Long | Short | String | Timestamp>>

示例

示例 1：基本用法

描述: 在 00:00:00 的第一条记录被发出，其状态在由驱逐窗口滑动（默认为 1 分钟）确定的下一个滚动窗口边界处安排驱逐。尽管配置的超时时间为 10 秒，但 00:00:09、00:00:18 和 00:00:28 的后续记录都被作为重复项丢弃，因为水印没有前进到足以触发驱逐计时器的程度。在键过期和键过期加上驱逐窗口滑动之间丢弃重复项。

参数值：

  • 数据集： ri.foundry.main.dataset.test
  • 键过期时间单位： SECONDS
  • 键过期时间值： 10
  • 列子集： null
  • 驱逐窗口滑动： null
  • 按列分组： null

输入：

row_orderdaytemperaturemeasurement_timestamp
4Monday10.42024-09-30T00:00:28
3Monday10.32024-09-30T00:00:18
2Monday10.22024-09-30T00:00:09
1Monday10.12024-09-30T00:00:00

输出：

daytemperaturemeasurement_timestamp
Monday10.12024-09-30T00:00:00

示例 2：基本情况

描述: 通过按天列进行去重，每个键保持独立状态。周一00:00:20的第一条记录被发出并推进水位线。周二00:00:05的记录被丢弃，因为它到达太晚：其事件时间加上10秒超时（00:00:15）落后于水位线（约00:00:20）。即使周二没有之前的记录也会发生这种情况。周三00:00:25的记录不晚，并作为该键的第一条记录发出。

参数值：

  • 数据集： ri.foundry.main.dataset.test
  • 键过期时间单位： SECONDS
  • 键过期时间值： 10
  • 列子集： null
  • 驱逐窗口滑动： null
  • 按列分组： {day}

输入：

row_orderdaytemperaturemeasurement_timestamp
3Wednesday22.12024-09-30T00:00:25
2Tuesday18.32024-09-30T00:00:05
1Monday20.52024-09-30T00:00:20

输出：

daytemperaturemeasurement_timestamp
Monday20.52024-09-30T00:00:20
Wednesday22.12024-09-30T00:00:25
