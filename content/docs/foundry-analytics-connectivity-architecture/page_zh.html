架构

本页面记录了原始 Spark 和直接读取 SQL 引擎的架构。Foundry 正在向 Furnace 过渡，这是一款具备更强性能、更高灵活性和更好兼容性的下一代 SQL 引擎。此过渡过程将自动完成，无需对现有工作流进行任何更改。

本节介绍的外部 SQL 连接及 BI 工具集成功能，均由名为 Foundry SQL Server 的服务提供支持。该服务为针对 Foundry 数据集的只读查询提供了轻量级的 SQL 会话与语句管理能力。Palantir 提供了 JDBC 和 ODBC 驱动，以便客户端利用开放标准与此服务进行交互；同时，针对部分依赖这些驱动的第三方平台，也提供了相应的插件实现。

支持的 SQL 方言

支持的 SQL 方言包括 ANSI、ODBC 和 SparkSQL。

请注意，对这些方言的支持仅限于只读功能。

执行引擎

Foundry SQL Server 会根据查询的复杂度自动选择执行引擎。每种执行引擎在整体性能、结果大小限制以及支持的查询复杂度方面都有所权衡。

Spark 引擎

默认的查询执行引擎利用了 Spark SQL 功能。该引擎支持完整的 SQL 计算功能，例如聚合、连接、排序和过滤等。需要使用此执行引擎的查询将受到数据规模的限制，因为在将结果传递给客户端应用程序之前，必须先将结果收集到 Spark 驱动程序的内存中。这些限制取决于计算结果中的行数和字节数。

直接读取引擎

在条件允许时，Foundry SQL Server 将使用直接读取引擎来执行查询。如果查询不需要 SQL 计算，Foundry SQL Server 将绕过 Spark SQL，直接从数据集的底层文件中流式读取记录。直接读取查询不受与需要完整 SQL 计算的查询相同的规模限制。

在以下情况下，查询符合直接读取条件：

  1. 在数据集上执行。目前不支持 视图。
  2. 数据集文件需为受支持的格式。直接读取目前支持 Parquet、CSV、Avro 和 Soho 格式。
  3. 查询不需要 SQL 计算。包含聚合、过滤、连接和排序谓词的查询不符合直接读取条件。
  4. 查询未从不符合直接读取条件的类型列中进行选择。array、map 和 struct 不符合直接读取条件。

直接读取查询区分大小写。

注意事项

  • 此功能旨在支持 Foundry 平台外部的客户端，例如 PowerBI、Tableau 或其他下游应用程序。有关 Foundry 平台内基于 SQL 的转换，请参阅 SQL 转换。
  • Foundry SQL Server 的架构已经针对中等数据规模的临时交互式查询进行了优化。
