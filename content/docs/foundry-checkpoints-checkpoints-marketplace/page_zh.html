将检查点配置添加到市场产品

使用 Foundry DevOps 将您的检查点配置包含在 市场产品 中，以便其他用户安装和重用。 了解如何创建市场产品 以开始。

支持的功能

现有的检查点配置通常可以包含在市场产品中，但有一些限制：

  • 当安装市场产品时，由安装创建的任何检查点配置将基于安装位置具有空间范围。检查点配置还将具有一个位置匹配器，这会将检查点限制为安装的输入、安装中的另一个资源或安装项目。因此，只有支持空间范围和位置匹配器的检查点类型才能包含在市场产品中。
  • Foundry -> Foundry Apollo -> Apollo Gotham -> Gotham AIP -> AIP Ontology -> 本体论 ontology -> 本体论 Pipeline -> 管道 pipeline -> 管道 Transform -> 转换 transform -> 转换 Workshop -> Workshop workshop -> Workshop Contour -> Contour Quiver -> Quiver Slate -> Slate Code Workbook -> Code Workbook Code Repositories -> 代码仓库 Code Workspaces -> 代码工作区 Data Integration -> 数据集成 Model Integration -> 模型集成 Object -> 对象 object -> 对象 Dataset -> 数据集 dataset -> 数据集 Tenant -> 租户 tenant -> 租户 Marking -> 标记 marking -> 标记 Permission -> 权限 permission -> 权限 Role -> 角色 role -> 角色 Function -> 函数 function -> 函数 Action -> 操作 action -> 操作 Backing -> 支撑 backing -> 支撑 Link -> 关联 link -> 关联 Property -> 属性 property -> 属性 Interface -> 接口 interface -> 接口 内容: 有各种条件匹配器未启用用于 Marketplace 产品。您可以在 检查点条件文档 中了解更多有关这些条件的信息。
    • 包含显式限制检查点类型只能归特定用户的检查点配置不能包含在 Marketplace 产品中。这包括选择了 提交检查点的用户 并且只指定了一个用户而不是组的情况。
    • 包含标记匹配器的检查点配置不能包含在 Marketplace 产品中。
    • 包含指定空间的位置匹配器的检查点配置不能包含在 Marketplace 产品中。
  • 只有包含单一检查点类型的检查点配置可以包含在 Marketplace 产品中。目前尚无法将包含多个检查点类型的检查点配置包含在 Marketplace 产品中。

包检查点配置

  1. 导航到 Marketplace 应用程序。
  2. 在页面底部选择 DevOps，然后在 DevOps 页面上选择 选择商店。在 选择您的商店 对话框中选择您的商店。
    • 如果您已经选择了存储库，可以通过右上角选择存储库名称以打开选择存储库对话框，以根据需要选择其他存储库。
  3. 要向产品添加检查点类型，您可以创建一个新产品或向现有产品添加检查点类型。
    • 要创建一个新产品，请选择+ 新建产品。有关更多指导，请参阅产品创建教程。
    • 要向现有产品添加检查点配置，请选择该产品，然后在右上角选择开始新版本。
  4. 使用添加输出功能，并选择检查点配置选项。
  5. 选择适当的范围以查看可用的检查点配置。

  1. 您可以在命名空间或组织范围下选择现有的检查点配置。
    • 使用右侧列出的检查点配置或配置详细信息面板右上角的选择按钮，选择感兴趣的检查点配置。

  1. 当您选择所有相关的检查点配置后，在右下角确认您的选择。
  2. 在编辑完Marketplace产品后，打开审核和发布选项卡并选择发布。
