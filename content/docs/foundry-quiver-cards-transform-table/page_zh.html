转换表

转换表是一种容器，用于对对象或从对象派生的表格数据执行批量分析。转换表允许用户根据输入对象（包括时间序列）的属性派生列，并提供对象数据的连接和分组功能。

由于这些操作的计算强度较大，转换表中最多允许 50,000 行。

转换表允许您：

  • 连接和聚合对象；
  • 从对象属性中派生新列；
  • 通过创建或编辑值来丰富数据；
  • 对时间序列列批量执行转换；以及
  • 以多种方式绘制操作输出。

添加转换

在转换表中，可通过添加转换按钮访问转换表搜索窗口。该窗口按转换操作类型（如编辑列、过滤、时间序列操作或空值/错误处理）对所有可用转换进行分组显示。大多数转换会添加各自类型的新列，但有些转换会改变表格的结构（例如，在连接其他对象类型或将表格按类别分组以聚合行时）。

还可以通过选择转换类别和具体的转换操作，直接从“后续操作”菜单添加转换表转换。此操作会向分析中添加一张转换表卡片，并将所选转换应用于该表。

也可以在转换表之外，对单个受支持的输入应用转换，此时仅对该值执行一次操作，而不是在表中执行批量操作。

要向分析中添加转换卡片，只需使用顶部菜单栏中的搜索窗口进行搜索即可。

转换表输入

转换表可以接收多种类型的输入，包括对象集、分类图表、时间序列图表、时间序列图、物化视图、数据透视表和其他转换表。用户还可以手动输入数据来构建转换表。

要将转换表添加到分析中，请在顶部的表菜单中选择转换表。这将打开编辑器面板，您可以在其中选择符合分析条件的现有输入。

输入：对象集

对象集是转换表最常见的输入。

您可以通过后续操作菜单，选择转换 > 转换表，从分析中的任何对象集创建转换表。

请注意，每个转换表限制 50,000 行（无论是初始数据还是连接产生的数据），因此您只能创建对象少于 50,000 个的转换表。

从对象集创建转换表时，集合中的每个对象将占一行，并包含一个表示每个对象唯一 ID 的主键列，以及对象每个突出属性（在本体中定义）对应的列；如果没有突出属性，则显示对象的前几个属性（最多 20 个）。

点击表格右下角的属性按钮，可添加或移除属性列，以及对象集关联的任何传感器对象。拖动列可重新排列其在表中的顺序。

您可以为转换表的每个转换步骤设置不同的列配置。有关转换表格式的更多信息，请参阅下方的章节。

输入：分类图表

条形图、折线图和散点图均可作为转换表的输入源。使用这些输入项会在图表中展示对应的类别和值，但不会提取底层数据。若要提取底层数据，您应当使用创建该图表时所用的对象集。您可以基于分类图表创建转换表，方法是选中该图表，点击后续操作菜单，然后选择表格 > 转换表。

输入：时序图

时序图可作为表格的输入源，将时间序列数据转换为表格格式，以便进行操作、编辑或丰富数据内容。

若要基于时序图创建转换表，请从图表中选中特定的时序图。然后，在时序图底部的后续操作菜单中，选择表格 > 转换表。

接着，在转换编辑器面板中，从可用的范围选项中进行选择：

  • 使用输入时间序列的完整时间范围（默认选项）。
  • 指定要导入的时间范围，可以通过手动输入起始和结束时间戳，或通过引用范围参数来实现。

出于性能考虑，转换表的行数限制为 50,000 行；转换表会将数据拉取到前端进行处理，而非将其推送到后端服务。因此，时间序列数据可能需要进行采样（分桶）以适应此限制。

您可以在数据选项中选择将时间序列数据转换为表格格式的方式：

  • 在“已采样”设置下，表格将显示每个分桶的边界时间戳和值（最早和最晚时间戳，以及最小和最大值）、平均值和点数。您可以调整采样分桶的数量。
  • 在“未采样”设置下，表格将显示原始的未采样数据（“逐笔数据”），并包含三列：主键、时间戳和值。

时间戳序列数据默认以 UTC（协调世界时）显示。在转换表编辑器的列设置中，可以将时间戳时区更改为用户所在时区或其他静态时区。

输入：时序图表

时序图表可作为表格的输入源，将图表中的每个时序图作为表格中的一行展开。

若要基于时序图表创建转换表，请选中时序图表（而非特定的时序图）。然后，在时序图表底部的后续操作菜单中，选择表格 > 基于图表时间序列的表格。

输入：透视表及其他转换表

透视后的数据可作为转换表的输入源，使您能够使用公式对透视表的列进行操作。与图表类似，基于透视表创建转换表时，引入的不是其输入数据，而是透视后的数据本身。 转换表同样可以作为另一个转换表的输入源。这在您希望将转换逻辑拆分为逻辑步骤的“区块”，或者将数据转换与数据格式化分离时非常有用。 您可以通过在表格卡片底部的后续操作菜单中选择表格，来基于另一个表格创建转换表。

输入：手动录入转换表

用户最多可手动录入 5,000 行数据来创建新的转换表。手动录入转换表卡片采用类似电子表格的用户界面，支持五种数据类型：字符串、数字、布尔值、时间和时间序列。用户可以像操作其他转换表一样，对手动录入转换表进行交互和操作。

若要向 Quiver 画布添加新的手动录入转换表，请在分析标题栏中选择手动录入，或者在搜索卡片对话框中搜索该表格。

手动输入表工作流示例

  • 使用表值参数化分析： 将手动输入转换表与行和列选择器结合使用，以在不同行之间切换。表中所选行的值可用作下游分析中的动态参数。

  • 从外部源摄取少量数据以补充分析和本体： 用户可以将电子表格中的元数据复制并粘贴到新表中，然后通过在对象集下方的“后续操作”菜单中搜索“联接到转换表”，将其联接到现有对象集。

在联接的对象集和手动输入转换表中，配置“联接到转换表”转换以指定联接条件。

  • 分析少量时间序列数据而无需设置时间序列同步： 复制并粘贴表格时间序列数据，并使用“表格时间序列”卡片将表转换为时间序列图，以便利用 Quiver 完整的时间序列操作继续分析。

默认情况下，手动输入转换表会为每一行生成一个随机的唯一主键，但用户可以选择将某一列设置为表的主键。主键会向下游传播，并作为唯一标识符在其他卡片（如转换表行选择器）中使用。

对于超过 5,000 行的时间序列数据，或者要在 Foundry 中重复使用时间序列数据，请设置时间序列同步，以提升性能和可重用性。

输入：本体 SQL

本体 SQL结果可用作转换表的输入，将 SQL 查询结果转换为表格格式，以便进一步操作、丰富或可视化。

要从本体 SQL 卡片创建转换表，请选择该本体 SQL 卡片，然后在卡片底部的“后续操作”菜单中选择“转换 > 转换表”。

当本体 SQL 卡片转换为转换表时，各列将与 SQL 查询返回的列相对应。随后，您可以对结果数据应用任何转换表操作，如筛选、分组、派生新列或绘图。

转换表输出

转换表输出主要有四种类别：时间序列、图表、表格和值。 可以通过表底部的“后续操作”菜单添加转换表输出。

输出：时间序列

从转换表输出时间序列数据有几种方式：

  • 当鼠标悬停在时间序列表单元格上时，会出现一个“弹出”按钮。点击该按钮即可在新的时间序列图中绘制时间序列。
  • 在表格底部的下一步操作菜单中，选择时间序列 > 分组时间序列图，即可在单一时间序列图表上绘制时间序列列。从表中选择输入列并设置页面大小，以配置需要叠加显示的时间序列数量。
  • 将表格格式的时间序列数据（时间戳和数值）转换为时间序列图，以使用 Quiver 中提供的全部时间序列可视化和转换功能。在表格底部的下一步操作菜单中，选择时间序列 > 表格时间序列。从转换表中选择时间戳（日期/时间）和数值（数字）列，并可选择设置单位。请注意，如果表格数据中每个时间戳包含多个数值点，则仅绘制该时间戳对应的第一个数据点。

输出：图表

您可以根据转换表数据创建折线图、分类散点图、条形图或 Vega 图表。在 Quiver 中，这些图表可用于任何将图表作为输入的功能，例如分类公式图或叠加图表。

输出：表

您可以基于现有的转换表创建新转换表，从而提供一个可以独立于底层数据进行格式化和组织的视图。

输出：透视转换表

您可以基于转换表创建透视转换表来展示聚合数据。

输出：值

您可以对转换表的列进行聚合，并将其作为数字或数组输出到指标卡片上。在 Quiver 中，这些指标可用于任何将数字或数组作为输入的功能。

计算新列

要添加计算列，请选择添加转换，然后选择与您要创建的数据类型相对应的选项卡：数值、日期/时间、字符串、布尔值、数组或时间序列。

有关创建列的可用转换完整列表，请参阅转换表转换索引。

连接

连接到关联对象

当转换表将对象集作为其输入时，您可以使用本体将关联对象连接到表中。此转换称为连接到关联对象，可以在转换菜单的表选项卡下找到。

在将对象集与其他关联对象进行连接时，系统会提示您选择链接类型，即起始对象与传入对象之间的连接关系。生成的表将具有一个新的联合主键；该主键是起始集主键与传入集主键的组合。新的联合主键会将传入对象的属性添加到每一现有行中，从而增加列数。

如果您的起始对象仅连接到一个传入对象（即“一对一”或“多对一”的链接类型），则行数不会增加。如果每个起始对象对应多个传入对象（换言之，即“一对多”的链接类型），表中的行数将会增加。

在下面的示例中，我们从包含“公司”对象集的转换表开始。该表最初有 505 行（每个对象 1 行）。随后，我们添加“连接至关联对象”转换，以添加关联的“股票事件”对象。现在，该表包含 11,149 行，主键是两个对象主键的组合，并且已将“股票事件”对象的列添加到表中。

连接

可以在转换菜单的表选项卡下找到连接转换。与 SQL 连接操作类似，它允许您合并两个转换表，同时指定匹配条件（如下图绿框所示）。

执行连接操作：

  1. 选择要连接的目标表。
  2. 设置连接类型（左连接、内连接或全连接）。
  3. 配置匹配条件：匹配模式（全部匹配或任意匹配），以及用于比较的输入表与连接表中的一对或多对列。
  4. 选择输入表中需要保留的列。
  5. 选择连接表中需要添加到输入表的列。如果输入表中已经包含与待添加列同名的列，系统将自动添加连接列前缀。

交叉连接

交叉连接转换通过执行笛卡尔积 ↗连接来合并两个转换表。交叉连接会为第一个表中的行与第二个表中的行的每一种配对组合生成一行数据。它不基于特定列进行连接。

分组

分组是指在预设的窗口或分桶内对一系列值进行聚合的操作。在转换表中，可以通过以下两种方式进行分组：

  • 分组
  • 分组后接连接（也称为联合分组）

分组

在分组操作中，您将为分组列中的每个属性创建一个类别。对于每个类别，所有关联的基于时间的列、数值列和时间序列列都会转换为值数组（也称为组）。可以对这些数组（组）执行数组转换和聚合操作。

例如，下方的转换表展示了一组具有不同海拔值的全球气象站。我们可以按照气象站所在的国家对其进行分组，从而创建按国家划分的海拔值数组。

如果我们想对这些海拔值进行聚合操作，例如计算各个国家的平均海拔，我们可以使用数值组聚合转换。该转换会在表中添加一列，用于返回所选输入列（此处为 elevation_array）的平均值。

联合分组

当您希望计算某些窗口化聚合量（如上文中每个类别的平均屋顶高度），同时又希望保持表中的行数不变（每栋建筑一行），并为聚合指标添加一个新列时，联合分组会非常有用。

执行联合分组转换时，对于不属于对象所属分组类别的每个属性，系统都会添加一个值数组列（如下图绿框所示）。随后，您需要添加聚合转换以计算这些数组的值（如下图红框所示）。

时间序列

转换表旨在支持时间序列的批量分析。这意味着，诸如筛选、求导或累计聚合等转换操作，现可一次性应用于多条时间序列。有关时间序列批量分析的详尽指南，请参阅《批量分析时间序列》。

有关其中包含的各项时间序列转换操作的更多信息，请参阅《转换表转换操作索引》。

根据待转换数据的类别，向转换表中添加时间序列列的方法共有三种，分别如下：

  • 从时间序列对象集合中显示时间序列折线图（迷你图）
  • 从根对象添加传感器对象
  • 从一组时间戳和数值创建时间序列

从时间序列对象集合中显示时间序列折线图（迷你图）

默认情况下（且出于性能考量），除非明确添加，否则不会显示时间序列数据的迷你图。若已创建包含时间序列或传感器对象的转换表，可通过添加“时间序列对象”转换并选择该时间序列对象的主键来将其纳入。

从根对象添加传感器对象

对于已标记为根对象，且在本体中连接了时间序列对象的对象，只需添加“关联传感器”列即可显示其关联序列。操作时，请选择“关联时间序列传感器”，并选择父对象的主键。

从一组时间戳和数值创建时间序列

转换表可将一组日期或时间转换为时间序列，如同在转换表之外将对象的条形图或折线图转换为时间序列一样。操作时，首先选择“分组依据”以创建日期分组和数值分组。接着，选择“分组为时间序列”，并将日期/时间分组选为“日期分组属性”，将数值分组选为“数值分组属性”。

例如，下方的转换表展示了一组具有不同股息值的股票股息事件。若要利用这些股息值创建时间序列，可按以下步骤操作：

  1. 首先，按事件对应的股票代码进行分组，以生成每个事件的股息值数组以及每个事件发生日期的数组。
  2. 随后，使用“分组为时间序列”转换在表中添加一列，展示以数值分组属性（本例中为 amount_array）对应日期分组属性（start_ts_array）绘制的时间序列图。

格式化转换表

转换表的列及其内部的具体值均可通过自定义宽高进行格式化。所有格式化操作均可在转换表的“显示”选项卡中完成。

列格式化

所有列（无论是数值列还是其他类型）均提供了格式化选项，支持静态设置列宽和最小高度。

可通过在转换菜单中添加“重命名列”转换操作来对列进行重命名。

值的条件格式化

此外，数值列还提供以下值和条件格式化控制选项：

  1. D3/视觉格式：这些数字格式化选项类似于 Quiver 中用于格式化指标卡片或其他数值数据的选项。了解有关 D3 格式化的更多信息。 ↗
  2. 对齐方式：这些选项控制数字在单元格内的位置；支持左对齐、居中对齐和右对齐设置。
  3. 条件着色：这些选项支持根据阈值对数值进行条件着色。阈值既可以静态设置，也可以配置为引入 Quiver 分析中的任意其他数值。你可以选择对单元格的文本或背景进行着色。

格式化时间序列列

转换表中的时间序列列会渲染关联时间序列属性的迷你图。默认情况下，迷你图会显示该序列中的全部数据。你可以在编辑器的显示选项卡中修改此项设置：

  1. 在目标时间序列列的列标题处选择向下箭头图标，这将打开列操作菜单。
  2. 选择配置显示设置选项。这是一个快捷操作，会将转换表编辑器定位到显示选项卡中的相关配置区域。
  3. 从菜单中选择相应选项，即可更改迷你图的视图范围。

你可以按照相同的步骤，为对象集表格中的迷你图配置视图范围。

视图范围选项

设置迷你图视图范围的方式共有四种：

  • 完整数据范围：默认选项。显示时间序列中的所有数据。对于庞大数据量的时间序列，这可能会对性能产生负面影响。
  • 固定范围：将视图范围设置为静态的起止日期，例如 2025-5-21 至 2025-06-04。
  • 相对范围：将视图范围设置为相对于今天的范围。例如，从 2 周前 到 现在。默认情况下，范围的结束日期会设为 现在，但你可以通过选择右上角的 设为相对时间 切换开关来输入特定时间。
  • 初始默认时间轴范围：将视图范围设置为用于初始化默认时间轴的范围。该值由 全局设置 菜单控制。

常见问题

能否使用转换表创建分组时间序列？

可以。转换表支持分组时间序列（即从条形图创建时间序列，随后将其细分为多个时间序列的功能）。你可以利用“分组至时间序列”功能，结合一组日期或时间以及一组数值来创建分组时间序列。

如需了解更多信息，请参阅转换表中的时间序列。

能否对转换表进行格式化？

可以，你可以使用参数或手动输入数值，从而按照用户定义的逻辑规则对表格单元格进行格式化。

如需了解更多信息，请参阅有关格式化转换表的文档。

转换表的数据规模限制是多少？

出于性能考虑，转换表存在 50,000 行的数据限制，因为转换表是将数据拉取至前端进行运算，而非将数据推送至后端服务处理。

能否对转换表中的数值进行绘图？

可以。转换表共有四种输出形式：时间序列、表格、图表和数值（指标卡片）。 如需了解更多信息，请参阅有关转换表输出的文档。

转换表是否仅允许对象集作为输入？

并非如此。虽然在转换表中使用对象是常见做法，但你同样可以利用转换表来处理时序数据、图表数据，或是处理透视表及其他转换表等表格类型。 需要注意的是，在处理图表数据或透视表时，数据行将失去与底层对象的关联。此时你只能对聚合数据进行操作，而无法回溯至底层数据。这一机制使得在透视表中执行按列操作成为可能，同时也允许以表格形式创建并处理绘图数据。

详情请参阅有关转换表输入的文档。
