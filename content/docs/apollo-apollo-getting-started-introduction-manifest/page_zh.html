通过附加详细信息和约束创建产品版本

假设你需要创建一个带有产品依赖项的新Helm图表版本。以下步骤描述了如何使用Apollo CLI通过从产品版本清单文件中创建产品版本来声明这种依赖关系。

配置Apollo CLI

Apollo CLI可以将你的Apollo URL和Bearer Token存储为变量，这样你就无需每次传递它们。在之前的例子中，Apollo UI会为你提供这些变量。由于我们使用的是不同的Apollo CLI命令，我们将需要生成令牌并将其添加到CLI中。

创建承载令牌

  1. 通过以下方式更改您的URL来创建承载令牌：

    https://<yourHubUrl>/workspace/autopilot/dashboard → https://<yourHubUrl>/multipass/app/tokens

  2. 选择创建令牌。给它一个唯一的名称，例如“apollo-cli”，并设置几天后的过期日期。

  3. 选择生成，并将令牌保存到安全的位置以备将来使用。

存储您的Apollo URL

  1. 打开您的命令行或终端窗口，并导航到包含可执行文件 apollo-cli 的目录。
  2. 运行 ./apollo-cli configure（在PC上为 apollo-cli.exe configure）。
  3. 输入您指定的Apollo Hub的URL并按回车键。
  4. 粘贴之前步骤中保存的承载令牌。如果您找不到令牌，请按照上述说明创建一个新的令牌。
  5. 如果你被提示提供任何其他字段，请按回车键跳过它们。
  6. 通过运行 ./apollo-cli list-environments 来测试你的配置，以生成一个包含所有连接到你的中心（Hub）的 spoke 环境名称的 JSON 对象。如果你收到错误信息，请重新检查上述说明，确保你没有遗漏任何配置步骤。

创建一个带有依赖项的新版本

  1. 在你的命令行工具中运行以下命令，创建你的 Helm 图表（或开源 Nginx 图表）的压缩 TAR 文件。请记得将版本号增加一个补丁版本。下面的步骤引用了 Nginx，因此在适当的地方，用你的 Helm 图表名称和 Nginx 版本替换 Nginx。
Copied!
1./apollo-cli product-release helm-chart init --as-tgz --name nginx --version 13.1.6 --repository-url https://charts.bitnami.com/bitnami --maven-coordinate com.intro.helmcharts.bitnami:nginx-${yourLastName}:13.1.6

在这个命令中，你将版本号增加了一个补丁版本，并生成了一个名为 nginx-${yourLastName}-13.1.6.helmchart.config.tgz 的文件。

  1. 通过在命令行中运行 tar -xvf nginx-${yourLastName}-13.1.6.helmchart.config.tgz 来解压归档文件。这将创建一个名为 /nginx-${yourLastName}-13.1.6 的目录，其中包含一个名为 /deployment 的嵌套目录。

  2. 使用文本编辑器（如 vim）打开该文件，例如 vim nginx-13.1.6/deployment/manifest.yml。此文件包含 Helm 图表的元数据，包括图表名称、版本和容器。我们还可以在清单中添加产品依赖项，并使用它在 Apollo Hub 上创建一个新的发布版本。我们将在 extensions 块下添加对 WordPress 15.x.x 的简单依赖。

  3. 编辑文件并在 extensions 标题的末尾添加以下块：

    Copied!
    1extensions:
2  product-dependencies:
3   - product-group: com.bitnami.repos
4     product-name: wordpress
5     minimum-version: 15.0.0
6     maximum-version: 15.x.x
7  optional: false

    此时文件应如下所示：

    Copied!
    1manifest-version: "1.0"
2product-group: com.intro.helmcharts.bitnami
3product-name: nginx-${yourLastName}
4product-version: 13.1.6
5product-type: helm-chart.v1
6extensions:
7  artifacts:
8  - type: oci
9    uri: docker.io/bitnami/nginx:1.23.1-debian-11-r4
10  helm-chart:
11    helm-chart-name: nginx
12    helm-chart-version: 13.1.6
13    helm-repository-url: https://charts.bitnami.com/bitnami
14  product-dependencies:
15  - product-group: com.bitnami.repos
16    product-name: wordpress
17    minimum-version: 15.0.0
18    maximum-version: 15.x.x
19    optional: false

  4. 通过运行以下命令，将带有依赖项的新 Nginx 产品发布（13.1.6）注册到系统中：

    Copied!
    1./apollo-cli product-release create --manifest nginx-${yourLastName}-13.1.6/deployment/manifest.yml

  5. 产品发布完成后，导航至 Nginx 的 发布版本 选项卡，现在其中包括你的新发布版本 13.1.6。

  6. 选择 13.1.6 发布版本。在屏幕右侧，注意已声明的 WordPress 依赖项现在出现在 依赖项 部分。

如果您的环境没有自动升级，可能是因为您尚未在您的环境中安装WordPress。由于现在这是您新版本的声明依赖项，您需要先进行此操作。您可以通过以下方式在您的环境中安装WordPress：

  1. 在Apollo中创建新的Helm chart版本
  2. 在您的环境中安装WordPress产品

在这个例子中，您添加了一个声明依赖于WordPress 15.x.x的产品版本。Apollo直到您安装了该依赖项后才允许进行升级。
