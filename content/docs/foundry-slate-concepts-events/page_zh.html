配置事件与操作

Slate 中的事件面板允许应用构建者配置在特定事件发生时执行的操作。

操作是对应用行为的更改，包括打开或关闭对话框/提示框、运行查询或设置变量值。

事件是触发器，例如用户与应用的交互（用户点击、值更改或对话框的打开与关闭）或数据加载状态（查询运行完成），这些触发器会执行特定操作。

事件面板会显示 Slate 应用中配置的所有事件/操作对。你还可以通过 Handlebar 引用和 JavaScript 为事件定义自定义逻辑，以控制传递给触发操作的值。正如函数一样，事件 JavaScript 无法访问 DOM 或 Slate 空间，且不会保存任何状态。

相较于仅使用 Handlebar 引用，事件与操作能够实现更为强大的应用行为控制能力。

事件编辑器不接受任何类型的辅助函数。

示例 2：将变量设置为下拉菜单值的两倍

此事件会将一个变量设置为下拉菜单中所选数值的两倍。注意：此示例仅供演示说明；通常此类操作应使用 Slate 函数来完成。

假设存在一个名为 w_selectionDropdown 的下拉组件，用户可从中选择不同的数字；同时存在一个变量 v_doubleSelection，用于包含所选值的两倍。为了实现此目标，请打开事件面板并创建一个新事件。将 w_selectionDropdown.selectedValue.changed 选作事件，将 v_doubleSelection.set 选作操作。然后在事件面板中按如下方式填写代码：

1var dropdownSelectionValue = {{slEventValue}};
2return 2 * dropdownSelectionValue;

{{slEventValue}} 是事件面板中的一个特殊 Handlebars 变量。如果触发操作带有相关联的值，那么 {{slEventValue}} 就会是那个关联的值。对于 .changed 操作而言，{{slEventValue}} 是关联组件属性已改变至的值。

因此，每当下拉菜单发生改变时，代码会获取下拉菜单改变后的值，将其翻倍并返回。该返回值会被 .set 操作用于将变量设置为该返回值。结果就是，每当下拉菜单发生改变时，代码都会获取下拉菜单的新值，将其翻倍，并将变量设置为此翻倍后的值。

示例 3：在变量中持续记录随时间累积的对象列表

此事件将记录用户在下拉菜单中选择过的所有值的历史，以便在应用程序的其他地方以某种方式使用此值列表。

假设有一个名为 w_selectionDropdown 的下拉菜单组件，用户可以从中进行选择；还有一个变量 v_selectionHistory，用于包含选择历史。为了实现这一点，请打开事件面板并创建一个新事件。将事件选为 w_selectionDropdown.selectedValue.changed，操作选为 v_selectionHistory.set。然后在事件面板中按如下方式填写代码：

1var history = {{v_selectionHistory}};
2var newValue = {{slEventValue}};
3history.push(newValue);
4return history;

现在，每当用户从下拉菜单中选择内容时，都会触发该事件。代码会获取 v_selectionHistory 中的当前值，将通过 {{slEventValue}} 传入的新选中值追加到列表末尾，并返回更新后的历史记录。由于该动作为 v_selectionHistory.set，变量 v_selectionHistory 将被设置为此新数组，从而将新选中的项添加到累积的历史记录中。

示例 4：在按钮点击时有条件地触发手动查询

此事件将在点击按钮时运行查询——但前提是用户已在所需的输入组件 w_input1 和 w_input2 中输入了数据。

假设有两个输入组件（用户可输入的文本框），分别名为 w_input1 和 w_input2。此外，假设有一个名为 w_buttonConditional 的按钮和一个手动查询 q_queryConditional。

进一步假设该查询依赖于用户在输入框中输入的值，即 w_input1.text 和 w_input2.text，如果这两个值缺失，运行查询将毫无意义——因此，即使点击了按钮，如果这些输入为空，我们也不希望运行查询。我们可以通过以下方式实现此逻辑。创建一个新事件，然后将事件选择为 w_buttonConditional.click，将动作选择为 q_queryConditional.run。接着，在事件面板中按如下方式填写代码：

1var input1Text = {{w_input1.text}};
2var input2Text = {{w_input2.text}};
3if (input1Text === "" || input2Text === "") {
4  return {{slDisableAction}};
5}
