配置事件和操作

在Slate中的事件面板允许应用构建者配置当特定的事件发生时执行的操作。

操作（Actions）是应用程序行为的变化，包括打开或关闭对话框/提示信息、运行查询或设置变量值。

事件（Events）是触发器，例如用户与应用程序的交互（用户点击、值更改或对话框的打开和关闭），或者数据加载状态（查询完成）来执行一个操作。

事件面板显示了Slate应用程序中所有配置的事件/操作对。你也可以使用Handlebar引用和JavaScript定义自定义逻辑来控制哪些值被发送到触发的操作。就像使用函数（functions）一样，事件JavaScript无法访问DOM或Slate的空间（space），也不会保存任何状态。

事件和操作允许比单独使用Handlebar引用更强大的应用程序行为控制。

事件编辑器不接受任何类型的帮助函数。

示例2：将变量设置为下拉菜单值的两倍

此事件会将变量设置为下拉菜单中所选数值的两倍。注意：此示例仅用于说明；通常这种操作应使用slate函数完成。

假设有一个名为 w_selectionDropdown 的下拉小部件，其中包含用户可以选择的多个数字，还有一个变量 v_doubleSelection，其目的是包含所选值的两倍。然后为了创建这个功能，请打开事件面板并创建一个新事件。选择 w_selectionDropdown.selectedValue.changed 作为事件，v_doubleSelection.set 作为操作。然后在事件面板中按照以下方式填写代码：

Copied!
1var dropdownSelectionValue = {{slEventValue}};
2return 2 * dropdownSelectionValue;

{{slEventValue}} 是事件面板中的一个特殊 handlebars 值。如果触发操作有相关联的值，则 {{slEventValue}} 将是该相关联的值。在 .changed 操作的情况下，{{slEventValue}} 是关联的小部件属性改变后的值 到 什么。

因此，每当下拉框发生变化时，代码会获取下拉框变化后的值，将其乘以二并返回。返回的值被.set操作使用，用于将变量设置为返回的值。结果是，每当下拉框发生变化时，代码会获取下拉框的新值，将其乘以二，并将变量设置为这个乘以二后的值。

示例 3：在变量中跟踪随时间积累的对象列表

此事件将保存用户在下拉框中选择的所有值的历史记录，以便该值列表可以在应用程序的其他地方以某种方式使用。

假设有一个名为 w_selectionDropdown 的下拉小部件，它有用户可以选择的选项，还有一个变量 v_selectionHistory，该变量用于包含选择历史。然后为了创建这个功能，打开事件面板并创建一个新事件。为该事件选择 w_selectionDropdown.selectedValue.changed，并为操作选择 v_selectionHistory.set。然后按照以下方式在事件面板中填写代码：

Copied!
1var history = {{v_selectionHistory}};
2var newValue = {{slEventValue}};
3history.push(newValue);
4return history;

现在，每当用户从下拉菜单中选择某项时，就会触发该事件。代码会获取 v_selectionHistory 中的当前值，并将新选择的值（由 {{slEventValue}} 传入）推送到列表末尾，然后返回新的增强历史记录。因为操作是 v_selectionHistory.set，变量 v_selectionHistory 将被设置为这个新数组，将新选择的项目添加到累积的历史记录中。

示例4：在按钮点击时触发手动查询，条件性地

此事件将在按钮被点击时运行查询——但仅当用户已输入到所需的输入控件w_input1和w_input2中。

假设有两个输入控件（用户可输入的文本字段），分别命名为w_input1和w_input2。此外，假设有一个名为w_buttonConditional的按钮和一个手动查询q_queryConditional。

进一步假设该查询依赖于用户输入到输入框中的值，即w_input1.text和w_input2.text，如果没有这两个值，运行该查询将毫无意义——因此即使点击了按钮，我们也不希望运行该查询。我们可以按以下方式实现。创建一个新事件，然后为事件选择w_buttonConditional.click，为操作选择q_queryConditional.run。然后在事件面板中填写如下代码：

Copied!
1var input1Text = {{w_input1.text}};
2var input2Text = {{w_input2.text}};
3if (input1Text === "" || input2Text === "") {
4  return {{slDisableAction}};
5}

{{slDisableAction}} 是事件面板中的一个特殊 handlebars 值。如果从面板中的代码返回此值，将禁用触发的操作。换句话说——如果 w_input1 或 w_input2 为空，则会返回此特殊值，查询 q_query 将不会执行。然而，如果这两个输入都具有值，则不会返回任何内容，操作将正常进行。
