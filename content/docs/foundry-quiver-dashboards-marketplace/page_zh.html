将 Quiver 仪表板添加到 Marketplace 产品中

使用 Foundry DevOps，您可以将 Quiver 仪表板包含在 Marketplace 产品中，供其他用户安装和复用。了解如何创建 Marketplace 产品。

支持的功能

支持包含以下功能的 Quiver 仪表板：

  • 对象分析卡片，例如过滤对象集、集合运算和链接对象集。
  • 基于对象的可视化图表，例如条形图、折线图和热力网格。只要底层数据转换表受支持，Vega 图表也可以打包。
  • 时间序列图表和图形，包括事件图和分组图。
  • 表格，包括大部分透视表，以及带有分组、连接和过滤等转换操作的转换表。
  • 基础可视化，支持数字卡片、字符串卡片、文本卡片和日期/时间卡片。

如何打包仪表板

  1. 在仪表板设置面板右侧，选择启用 Marketplace 模板化选项。
  2. 在顶部导航栏选择运行校验按钮，以识别潜在的 Marketplace 模板化问题（如果有）。
  3. 如果校验器未显示任何错误，仪表板的发布或重新发布按钮也会同时保存一份适用于 Marketplace 的仪表板版本。
  4. 在新的或现有的 Marketplace 产品中使用该仪表板版本。

代码检查器错误

代码检查器最多可能提示三类主要错误：

  • 不支持的卡片：该卡片当前不支持进行 Marketplace 模板化。
  • 不支持的配置：该卡片受支持，但其配置不受支持（如下面的透视表示例所示）。请查看提供的错误信息以了解更多详情。
  • 意外错误：在大多数情况下，解决上述两项问题即可消除所有意外错误。如果仍然存在意外错误，请联系 Palantir 支持团队。

您可以无需逐一检查各个仪表板，而是为整个分析启用 Marketplace 代码检查功能，从而在您工作时自动运行校验。启用全分析范围的代码检查后，保存按钮旁边会出现一个 Marketplace 校验指示器，显示当前校验状态以及发现的错误和警告数量。选择该指示器可查看校验详情。

在全局设置面板中，选择启用 marketplace 代码检查选项，即可为分析中的所有仪表板开启 Marketplace 模板化，并启用全分析范围的代码检查器。您依然可以在单个仪表板的设置中禁用其模板化功能。

Marketplace 安装所需的输入项

如果校验成功，顶部导航栏会出现查看输入项选项。选择该选项将显示仪表板打包至 Marketplace 时所需的对象类型、对象实例、函数和链接类型列表。对于对象类型输入项，将鼠标悬停在每个输入项右侧的属性文字上，即可查看需要进行映射的属性类型列表。

从已安装的仪表板创建分析

虽然底层的 Quiver 分析不会随仪表板一起打包，但您可以通过选择从仪表板创建分析，基于已安装的仪表板创建一个新的 Quiver 分析。

创建的分析将包含此仪表板的副本、仪表板中所有可见的卡片及其上游依赖项。

创建的分析及其仪表板将与已安装的仪表板断开连接。对分析所做的修改不会更新已安装的仪表板，对已安装仪表板所做的更改也不会自动合并到分析中。

功能使用注意事项

该功能仍处于积极开发阶段。请注意以下重要的使用事项。

不支持的卡片与配置

部分配置衍生自底层数据的卡片，在为 Marketplace 打包时支持有限。详情请参阅下文的透视表和物化章节。

时间序列

Foundry 的大部分时间序列功能已在 Marketplace 产品中支持。不过，时间序列度量值目前尚不支持在 Marketplace 中使用。

为确保与 Marketplace 兼容，您的仪表板必须满足以下条件：

  • 不得在时间序列功能和转换中使用度量值功能。
  • 不得在转换表或对象集视图中添加度量列。可在转换表的显示设置中移除这些列。

若代码检查器检测到使用了任何度量值，将抛出不支持配置错误，并详细指出存在问题的卡片和/或度量值。

作为使用度量值功能的替代方案，您可以直接从传感器对象导入序列，这是受支持的方法。

透视表

透视表卡片可用，但当转换表处于下游时存在限制，这会影响列重命名和行操作。请确保您的卡片符合以下要求：

  • 不包含任何列属性，因为这些列本应基于底层数据生成。
  • 在输入透视表卡片中为所有聚合配置标签，以确保列名稳定且唯一。

此外，如果在透视表下游存在已选对象集卡片，在打包前会清除其已选数据。

物化

使用物化数据类型的卡片可以进行打包，但存在限制。在打包和安装过程中，列名保持不变，因此不会根据已安装的对象类型进行更新。如果仪表板安装了与打包时不同的对象类型，仪表板中的物化卡片（及其下游卡片）可能无法加载。具体而言，取决于是否启用了将列名重命名为 API 名称，属性类型 API 名称或底层数据集列名必须与原始对象类型中的对应名称匹配。
