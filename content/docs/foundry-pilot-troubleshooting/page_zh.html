本页面解决使用 Pilot 时出现的常见问题和疑问。

种子数据和实时数据

Pilot 根据视图使用两种数据源：

  • 编辑器视图 始终使用在 Pilot 容器内生成的种子数据。AI 代理从不访问真实的企业数据。这可以防止 Pilot 将生产值硬编码到您的应用程序代码中。
  • 部署视图 将您的应用程序连接到真实的本体论（ontology）数据。在部署分支上，本体论模式更改（对象类型、操作类型、链接类型）作用于该分支，并且不会影响 Main，直到您合并。在 Main 上，您在应用程序中的任何操作都是即时生效的本体论编辑。

部署期间的权限错误

部署需要在项目、组织和注册级别具有权限。如果您遇到权限错误，请验证以下内容：

  • 您在 Pilot 部署所在的 Foundry 项目中至少拥有 Editor 角色（角色）。
  • 您在 Control Panel 中拥有 Third-party application administrator 角色（角色），这是创建 Developer Console 应用程序及其关联 OAuth 客户端所必需的。
  • 注册管理员已在 Control Panel > 审批 中批准了子域名请求。
  • 该项目没有阻止数据集创建的标记（标记）限制。

有关 Developer Console 权限的完整说明，请参阅 Developer Console 权限。如果权限看起来正确但错误仍然存在，请检查浏览器控制台中的详细错误信息，并与您的管理员分享。

访问 Pilot 日志

要访问 Pilot 系统日志：

  1. 选择内容标题栏中标签栏旁边的三个点菜单（...）。
  2. 选择Pilot 日志。

Pilot 日志标签页显示实时容器日志。使用这些日志来诊断构建错误、依赖项安装问题和运行时问题。

预览标签中的构建错误

如果预览标签页显示的是构建错误而不是你的应用程序：

  1. 在预览标签页中阅读错误信息。常见的错误包括缺失的导入、类型不匹配和语法错误。
  2. 选择通过 Pilot 修复，以提示 Pilot 自动诊断并解决该问题。
  3. 如果自动修复无法解决问题，请在聊天面板中描述错误，并让 Pilot 进行调查。

你也可以在Pilot 日志标签页中查看开发服务器的详细构建输出。

容器启动问题

创建新应用后，Pilot 会分配一个容器。这个过程通常需要 30 到 60 秒。如果容器未启动，请检查以下内容：

  1. 通过选择右上角的状态指示器来查看状态监控弹窗。Pilot 状态 字段表示连接是否正常。
  2. 如果状态显示为 断开连接，请尝试刷新页面。
  3. 如果刷新后容器仍未启动，请在新的会话中创建一个新的 Pilot 应用。

部署视图中的意外编辑

在 Main 的部署视图中预览您的应用时，您执行的任何操作（如创建、编辑或删除对象）都会立即对生产数据产生影响，这是预期的行为，不是错误。

如果你希望在不影响生产环境的情况下测试操作，请在部署分支上使用部署视图预览你的应用程序。在分支上进行的本体论编辑仅限于该分支，并且不会传播到 Main 直到你合并。

操作引用跨环境失效

当你从 Pilot 分支部署本体论实体到 Main，操作类型标识符（RIDs）可能在分支和 Main 之间有所不同。如果你的应用程序代码引用了分支特定的 RIDs，这可能导致引用错误。

为了解决这个问题：

  1. 在合并你的分支后，根据提示切换回 Pilot 中的 Main。
  2. 如果错误仍然存在，请让 Pilot 将操作引用更新为使用 Main 分支的标识符。

分隔元素控制台错误

你可能会在浏览器控制台中看到引用“divider”元素且没有高度的错误信息。这些错误是外观问题，不会影响应用程序的功能。你可以安全地忽略它们。

依赖安装失败

如果状态监控弹出窗口显示npm依赖项安装失败，请尝试以下步骤：

  1. 在状态弹出窗口中选择使用 Pilot 解决以尝试自动解决。
  2. 如果问题仍然存在，请查看Pilot 日志标签页以获取具体的错误信息。
  3. 在聊天面板中输入Reinstall npm dependencies，让 Pilot 重新安装依赖项。

应用无法部署

如果部署面板显示你的应用无法部署，请按照以下步骤操作：

  1. 打开状态监控弹出窗口并检查可部署性状态以获取详细信息。
  2. 常见原因包括无效的本体论实体、缺少必需的属性或未解决的构建错误。
  3. 选择通过Pilot修复以自动解决阻塞问题。
  4. 在修复问题后，从部署面板重新尝试部署。
