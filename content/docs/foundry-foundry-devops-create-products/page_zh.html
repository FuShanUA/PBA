创建产品

本页面包含在 Foundry DevOps 中创建新产品的说明。用户可以通过 Marketplace 门户浏览并安装可用产品。阅读更多关于您可能希望为其创建产品的用例。

有关同时打包多个产品的说明，请参阅打包多个产品。

选择商店

要创建新产品，请选择一个用于发布产品的商店。若要选择现有商店或创建新商店，请在右上角选择更改商店选项。

新商店会保存到项目中，并继承该项目的权限。具体而言，对商店所属项目拥有编辑权限的任何人都可以在该商店中创建新产品和编辑现有产品，而对商店所属项目拥有查看或编辑权限的任何人都可以从该商店安装产品。

选择商店后，选择新产品或创建新组开始创建您的产品。

配置新产品草稿

在左上角的设置产品标题...输入框中为您的草稿产品提供一个名称。

启动产品草稿时，您可以：

  • 添加资源：从项目、文件夹或本体中手动选择资源，以将它们打包为输出。
  • 追踪文件夹：产品将追踪源文件夹。新产品版本将自动从该文件夹引入新输出。

在下面的示例中，我们选择手动添加资源。草稿随时可以迁移到文件夹追踪。

添加输出

选择添加输出以选择要包含在草稿产品中的输出；这些输出是用户安装产品时 Marketplace 会重新创建的资源。您可以选择添加文件来选择 Compass 文件系统中的大多数资源类型。如果无法通过 Compass 选择某项资源，DevOps 会提供自身的选取选项，例如添加本体资源。

DevOps 会自动识别资源依赖项，因此您应该首先添加最下游的资源。例如，如果您想打包一个 Workshop 应用程序和四个对象类型，您只需添加该 Workshop 应用程序即可。

DevOps 还允许您批量添加输出。对于一次性批量添加输出，请选择从文件夹添加所有文件。您还可以使用从数据血缘关系添加和从工作流血缘关系添加，从图形化依赖视图中添加资源。如需采用更自动化的方式追踪输出并随着产品演进将变更与源文件夹同步，请考虑追踪源文件夹。

将输出添加到草稿产品后，选择一个资源以在屏幕右侧启动其详情面板。

选择依赖项选项卡以查看输出的依赖项，DevOps 也会将这些依赖项显示为输入。

添加输入

在您将输出添加到草稿产品时，DevOps 会自动将输出依赖项显示为输入。安装您已完成产品的用户必须提供资源以满足每个输入项。

在设计产品时，您可以将输入提升为输出。在输入行上选择省略号图标，然后选择移至输出，将其提升为输出。

你还可以通过选择多个输入，并从屏幕底部的弹出菜单中选择 移动 {N} 项至输出，批量将输入移至输出。

通常情况下，如果希望安装者自行提供某项资源（例如他们自己的数据集或对象类型），应将该需求列为输入。如果希望由你的产品为安装者提供某项资源，则应将该输入提升为输出。

如果你逐步将所有输入提升为输出，那么安装该产品的用户在安装过程中将无需进行任何映射操作。

某些输入类型（例如参数或用户组）无法移至输出，因为 DevOps 在安装时需要进行这些配置。

要查看特定输入的更多信息，请选择该输入以启动 详情 面板。依赖项 选项卡包含了哪些内容资源需要此输入的信息。在这里，你还可以为输入配置预设，从而限制输入的选项并为安装者提供默认值。

管理输入和输出

DevOps 提供了多种选项，帮助你管理包含大量输出的产品。如果你启用了文件夹结构以便在安装时复制打包资源的组织结构，可以在 输出 面板中使用 按文件夹分组 选项，在其目标文件夹中预览输出。

你还可以在 输入 和 输出 面板的功能区中选择筛选图标，对显示的资源进行筛选。例如，借此可以仅显示附带错误信息的资源或特定类型的资源。

预览关联产品

从产品左侧面板中选择 关联产品 选项卡，预览所有可用的关联产品，这些产品可以为你的产品提供输入。

你还可以在 输入 表中使用 按关联产品分组 选项，预览每个上游关联产品提供的输入。

添加产品文档

从产品左侧面板中选择 文档 选项卡，为在 storefront 中浏览产品的用户添加可见信息。在添加用作 缩略图 的图片之前，请使用 描述 部分撰写产品的简短描述。

你可以添加额外的 图片 作为产品内容的预览。可选地，使用 Markdown ↗ 语法输入较长的 产品描述，其中包含详细的产品使用说明。

配置产品设置

选择 设置 选项卡以配置其他产品设置。

文件夹结构

通过开启 文件夹结构 开关来启用该功能，使 DevOps 在安装过程中创建与原始组织结构相匹配的文件夹。

DevOps 会将文件夹打包至最低公共祖先层级，该层级在安装时将被安装项目所取代。

如果您的产品资源组织在专用文件夹或项目中，建议考虑跟踪源文件夹，以便在产品迭代时实现自动化的输出管理。

安装模式

使用安装模式可为产品的所有安装设置默认配置。这些设置可在每次安装时单独调整。

  • 生产：适用于需要集中管理所有安装并使用自动升级等功能的场景。默认情况下，安装项会禁用自动升级。安装者可选择锁定其项目，以确保安全升级。
  • 单例：与生产模式设置相同，但限制每个空间和本体中只能安装一个产品实例。
  • 引导：适用于安装者将您的产品作为自定义开发起点的场景。默认情况下，安装项会禁用自动升级，安装者可将其安装在拥有编辑权限的任何位置。

构建设置

使用构建设置可决定 DevOps 是否在应用商店安装任务期间自动构建数据集和模型。这能确保 DevOps 在任务完成后，将所有数据集填充至新部署的资源中。

审核与发布

确认产品无误后，选择发布将产品上架至应用商店。系统会弹出审核变更并发布对话框，您可以在此审核产品变更并添加更新日志描述。发布过程可能需要几分钟，具体取决于产品大小。
