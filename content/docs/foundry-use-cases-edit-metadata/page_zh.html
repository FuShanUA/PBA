编辑元数据

通过现有或新建的后备项目<创建用例>后，你可以修改用于定义该用例的元数据。这些元数据位于用例概览页面，包含用例名称、描述、数据主人和状态。

名称

点击页面顶部的用例名称并在文本框中编辑，即可更改用例名称。

描述

点击页面顶部的描述并在文本框中编辑，即可更改用例描述。 作为构建者，你可以利用此字段记录用例，说明其用途及任何有助于后续维护的相关信息。

你也可以进入应用页面编辑描述字段，为用例中的每个应用单独添加描述。

文档

你可以使用 Markdown 语言查看和编辑用例文档。选择显示文档 > 编辑以展开文档部分。

在 Markdown 编辑器中，使用选项卡输入或更新用例文档，并使用<预览>选项卡在保存前查看更改。

要编辑用例文档，你必须拥有<用例编辑者或数据主人角色>，并具备项目<封面页>的编辑权限，因为用例文档存储在后备项目的封面页上。

数据主人

你可以为用例分配两种类型的数据主人：技术型和业务型。两者均可分配给单个用户或群组。

  • 技术数据主人：技术数据主人通常是负责开发该用例的应用构建者。

    • 创建用例的用户默认为该用例的技术数据主人。

  • 业务数据主人：业务数据主人通常是跟踪用例工作流进度和财务细节的用户。

    • 用例默认不会分配任何业务数据主人。

要更改用例的数据主人，请将鼠标悬停在名称或空白字段上并选择铅笔图标。系统会弹出一个搜索下拉框，你可以滚动浏览或搜索可用的用户及群组。

状态

你可以为用例分配与其运行情况相关的状态：

  • 活跃：已发布并在你的 Foundry 实例中运行。
  • 实验性：开发中且未正式发布。
  • 已弃用：不再运行。

要将用例状态从实验性更改为活跃，请选择状态下拉框，选择活跃，然后点击更改状态。

如果用例的对象类型或操作类型处于实验性或已弃用状态，而你却将用例状态更改为活跃，状态下拉框附近和左上角侧边栏将出现警告标志，提示资源状态与用例状态不匹配。通常不建议制造此类差异。
