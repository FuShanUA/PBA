编辑元数据

在 创建您的使用案例 后，无论是使用现有的或新的支撑项目，您都可以修改有助于定义您的使用案例的元数据。此元数据可以在您的使用案例概览页面中找到，包括使用案例名称、描述、负责人和状态。

名称

通过点击页面顶部的使用案例名称并在文本字段中进行编辑来更改您的使用案例名称。

描述

通过点击页面顶部的描述并编辑文本字段，可以更改用例的描述。

您还可以通过点击应用程序页面并编辑描述字段，为用例中的每个应用程序添加描述。

文档

您可以使用Markdown语言查看和编辑用例文档。通过选择显示文档 > 编辑来展开文档部分。

在Markdown编辑器中，使用撰写 在选项卡中输入或更新用例文档，以及 预览 选项卡来在保存之前查看您的更改。

要编辑用例文档，您必须拥有 用例编辑器或所有者角色 并且在项目 封面页 上有编辑权限，因为用例文档存储在支撑项目的封面页上。

所有者

您可以为用例分配两种类型的拥有者： 技术 和 业务。两者都可以分配给单个用户或组。

  • 技术拥有者： 技术拥有者通常是开发该用例的应用程序构建者。

    • 创建该用例的用户是该用例的默认技术负责人。

  • 业务负责人：业务负责人通常是跟踪用例的工作流程进度和财务细节的用户。

    • 您的用例没有默认分配业务负责人。

要更改您的用例的负责人，请将鼠标悬停在名称或空白字段上，然后选择铅笔图标。这将打开一个搜索下拉菜单，您可以在其中滚动或搜索可用的用户或组。

状态

您可以为用例分配与其操作状态相关的状态：

  • 活跃： 在您的Foundry实例中发布并运行。
  • 实验性：正在进行中，未被积极发布。
  • 已弃用：不再运行。

要将我们的用例状态从Experimental更改为Active，请选择状态下拉菜单，选择Active，然后点击更改状态。

如果您在用例具有对象类型或操作类型处于Experimental或Deprecated状态时将其状态更改为Active，则会在状态下拉菜单附近以及顶部左侧侧边栏出现警告标志，表示资源和用例状态之间存在差异。我们通常不建议创建这种差异。
