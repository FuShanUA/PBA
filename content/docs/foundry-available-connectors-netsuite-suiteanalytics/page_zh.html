Oracle NetSuite SuiteAnalytics

通过 SuiteAnalytics Connect 将 Foundry 连接到 Oracle NetSuite，将 NetSuite ERP 中的数据同步至 Foundry。

必须在 NetSuite 实例中启用 SuiteAnalytics。请参阅《NetSuite 文档》↗以启用该功能。

设置

  1. 打开《数据连接》应用，并在屏幕右上角选择 + 新建源。
  2. 从可用的连接器类型中选择 JDBC。
  3. 按照后续配置提示，结合下方各节的信息继续完成连接器设置。

进一步了解如何在 Foundry 中《设置连接器》。

身份验证

可以通过用户名/密码组合对 SuiteAnalytics 进行身份验证。建议使用服务用户凭据，而非个人用户凭据。

在 NetSuite 中配置用户角色和权限

在 NetSuite 中，为了控制访问权限，每个用户会被分配一个或多个角色，每个角色包含一组权限，定义了用户可执行的操作及可访问的数据。对于将要连接到 Foundry 的用户，建议进行如下配置：

  1. 创建具有适当权限的专用角色。
    1. 在 NetSuite 工具栏中，选择 设置 > 用户/角色 > 管理角色 > 新建，并为该角色指定明确的名称。建议使用 foundry-role。
    2. 导航至角色页面底部，选择 权限 > 设置，为该角色添加系统级权限。选择 SuiteAnalytics Connect，点击 添加，然后点击 保存。
      • 注意：NetSuite 文档建议添加 SuiteAnalytics Connect: 读取全部 权限，但该权限对 NetSuite2.com 数据源无效（详见此处 ↗）。添加此权限不会产生任何效果。
    3. 导航至角色页面底部，选择 权限 > 列表，为希望在 Foundry 中查询的表添加相应的表权限。选择所需的表，点击 添加，然后点击 保存。
  2. 将新角色分配给用户。
    1. 在 NetSuite 工具栏中，选择 设置 > 用户/角色 > 管理用户。选择要用于连接 Foundry 的用户，然后点击 编辑。
    2. 导航至 访问 选项卡，确保已勾选 授予访问权限 复选框。
    3. 在 角色 选项卡中，从下拉列表中选择新创建的角色 (foundry-role)，点击 添加，然后点击 保存。
      • 注意：NetSuite 文档或者建议使用 数据仓库集成器 角色来替代自定义角色。然而，该角色要求使用基于令牌的身份验证进行访问（详见此处 ↗），而 Foundry 暂不支持此方式。

要验证添加的权限是否正确，请以分配了新角色的用户身份登录，并检查能否查看所有预期的数据。

网络配置

SuiteAnalytics 连接器需要对目标 NetSuite Connect 实例的网络访问权限。

Foundry worker 连接

如果连接运行在《Foundry worker》上，则必须在设置源时添加相应的《出口策略》。

允许访问的服务主机和端口可在 NetSuite 配置主页的 https://<YOUR_ACCOUNT_ID>.app.netsuite.com/app/external/odbc/suiteAnalyticsConnectDownload.nl 找到。在无 NetSuite 账户 ID 的情况下访问此页面：

  1. 登录 NetSuite 账户主页。
  2. 找到左下角的 设置 面板，选择 设置 SuiteAnalytics Connect。

服务主机通常采用 <ACCOUNT_ID>.connect.api.netsuite.com 格式，端口为 1708。

如果不存在此类出口策略，可以《申请新策略》；否则可以《添加策略》。

由于此处采用了非 HTTPS 协议，您需要同时添加以下两项策略：

  • 通过名称引用您的服务主机的 DNS 策略，以及
  • 显式引用 IP 范围的 CIDR 策略。您可以在终端中运行 nslookup you-service-host 来获取 NetSuite 实例的 IP 范围。NetSuite 服务的 IP 地址可能会随时变更，恕不另行通知。

URL是格式为 jdbc:ns://<SERVICE_HOST>:<SERVICE_PORT>，其中 SERVICE_HOST 和 SERVICE_PORT 可从 NetSuite 的配置主页 获取。通常格式为 jdbc:ns://<ACCOUNT_ID>.connect.api.netsuite.com:1708

Drivers是对于 Foundry worker，请从 NetSuite 的配置主页 上传最新的 JDBC 驱动。对于传统的 agent worker 连接，在智能体上上传前必须对同一 JDBC 驱动进行正确签名——请联系 Palantir 技术支持。有关更多详细信息，请参阅如何 向智能体添加驱动。
Credentials是用于连接至 Foundry 的用户的用户名和密码。
JDBC properties是可用属性的完整列表请见 此处 ↗。以下属性为必填项：

  - CustomProperties : (AccountID=<ACCOUNT_ID>;RoleID=<ROLE_ID>)
ROLE_ID 是与您分配给用户（foundry-role）的角色关联的内部 ID。您可以在 Setup > Users/Roles > Manage Roles 页面上角色名称旁边找到此值。如果未显示内部 ID，请参阅 如何开启此功能 ↗。

  - NegotiateSSLClose : false

  - ServerDataSource : NetSuite2.com
自 2021 年 11 月 8 日起，新的 Connect 用户只能使用 NetSuite2.com 数据源访问 Connect Service。有关更多详细信息，请参阅 Oracle NetSuite 文档 ↗。

  - encrypted : 1

其他连接参数与任何 JDBC 源 相同。
