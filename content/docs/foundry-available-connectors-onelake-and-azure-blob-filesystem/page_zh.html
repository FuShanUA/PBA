OneLake 和 Azure Blob 文件系统（ABFS）

使用 Azure Blob 文件系统（ABFS） ↗ 将 Foundry 连接到 OneLake ↗、Azure 数据湖存储 Gen2（ADLS Gen2）和其他符合条件的 Azure 产品。ABFS 连接器允许将文件读入 Foundry 并从 Foundry 写入 Azure。

不支持连接到 Azure 数据湖存储 Gen1。

支持的功能

功能状态
探索🟢 一般可用
批量导入🟢 一般可用
增量🟢 一般可用
媒体集 🟢 通常可用
虚拟表🟢 通常可用
导出任务🟡 逐渐淘汰
文件导出🟢 通常可用

数据模型

连接器可以将任何类型的文件传输到Foundry数据集。文件格式会被保留，并且在传输过程中或之后不会应用模式。对输出数据集应用必要的模式，或者编写一个 下游转换 来访问数据。

性能和限制

可传输文件的大小没有限制。然而，网络问题可能导致大规模传输失败。特别是，在 Foundry worker 上运行的同步 如果运行时间超过两天，将被中断。为避免网络问题，我们建议使用较小的文件大小，并限制每次同步执行时摄入的文件数量。同步可以 安排 以频繁运行。

设置

  1. 打开 数据连接 应用程序，并在屏幕右上角选择 + 新数据源。
  2. 从可用的连接器类型中选择 ABFS—Azure 数据湖存储 Gen2。
  3. 按照后续配置提示继续使用以下部分中的信息设置您的连接器。

我们建议使用一个 具有直接连接策略的 Foundry 工作人员 与 ABFS 连接器一起使用，以简化设置和配置。

了解更多关于 设置连接器 的信息。

认证

要导入数据，连接器必须 list 文件系统内容并 read Azure 中的文件内容。这种行为可以通过与提供连接凭据相关的主体的授权模式实现。

  • 在基于角色的访问控制（RBAC）中，这是一种粗粒度的访问控制形式，主体需要在数据所在容器上拥有 Storage Blob Data Reader ↗ 角色。
  • 对于访问控制列表（ACL），这是一种细粒度的访问控制形式，主体需要在摄入文件上拥有 Read ACL，并且在从容器根目录到文件位置的所有路径目录上拥有 Execute ACL。

了解有关 ADLS Gen2 中的访问控制 ↗ 的更多信息。

ABFS 连接器支持以下认证配置选项，按推荐程度从高到低排列：

选项是否必需？描述

连接器需要一个静态的SAS令牌，并将存储一个长期有效的令牌以用于与存储帐户进行身份验证。

网络

在Foundry中运行Azure连接时，需要为Foundry设置网络访问权限以与源进行通信。这是一个两步过程：

  1. 从Foundry设置网络访问权限，以便出口到源。 您可以通过在数据连接应用程序中的源上应用适当的出站策略来实现这一点。Azure连接器需要对存储账户域名的网络访问权限port 443。例如，如果您正在连接到abfss://file_system@account_name.dfs.core.windows.net，则存储账户的域是account_name.dfs.core.windows.net. 数据连接应用程序会根据在源中提供的连接详细信息建议适当的出站策略。
  2. 将Foundry加入您的Azure存储容器的白名单。 你可以通过在Azure存储帐户中允许你的Foundry IP，如Azure的文档 ↗中所述。你的Foundry IP详细信息可以在控制面板应用程序中的网络出站下找到。

如果你正在设置访问与Foundry注册相同Azure区域内的Azure存储桶的权限，你需要通过PrivateLink设置不同于上述的网络配置。如果是这种情况，请联系你的Palantir管理员以获得帮助。

导出数据到 Azure Data Lake/Blob Storage

要导出到 ABFS，请先 启用导出 您的 ABFS 连接器。然后，创建一个新的导出。

导出任务（旧版）

导出任务 是一个旧版功能，我们不推荐用于新实现。对于新的导出，请使用 推荐的导出工作流 . 本文件是为仍在使用现有导出任务的用户提供的。

要开始导出数据，您必须配置一个 导出任务。导航到包含您要导出的连接器的项目文件夹。右键单击连接器名称，然后选择 Create Data Connection Task。

在数据连接视图的左侧面板中：

  1. 确认 Source 名称与您要使用的连接器匹配。
  2. 添加一个名为 inputDataset 的 Input。 输入数据集 是正在导出的 Foundry 数据集。
  3. 添加一个名为 outputDataset 的 Output。 输出数据集 用于运行、计划和监控任务。
  4. 最后，在文本字段中添加一个 YAML 块来定义任务配置。

在左侧面板中显示的连接器和输入数据集的标签不反映 YAMl 中定义的名称。

创建导出任务 YAML 时，请使用以下选项：

选项 是否必需？描述
directoryPath是文件将被写入的目录。路径必须以尾随的 / 结尾。
excludePaths否正则表达式列表；与这些表达式匹配的文件名不会被导出。
rewritePaths否请参阅下方 部分 了解更多信息。
uploadConfirmation否当值为 exportedFiles 时，输出数据集将包含已导出的文件列表。
createTransactionFolders否启用后，数据将写入指定的 directoryPath 中的一个子文件夹。每个子文件夹在 Foundry 中每次导出事务时都会有一个唯一的名称，并基于事务在 Foundry 中提交的时间。
incrementalType否对于逐步构建的数据集，设置为 incremental 以仅导出自上次导出以来发生的事务。
flagFile否请参阅下方 部分 了解更多信息。
spanMultipleViews否如果 true，Foundry 中的多个事务将一次性导出。如果 false，一个构建将一次只导出一个事务。如果启用了增量，则会首先导出最早事务中的文件。

rewritePaths

如果第一个键与文件名匹配，键中的捕获组将被替换为值。值本身可以包含额外的部分，以向文件名添加元数据。

如果值包含：

  • ${dt:javaDateExpression}：该部分的值将被正在导出文件的时间戳替换。javaDateExpression 遵循 DateTimeFormatter ↗ 模式。
  • ${transaction}：该部分的值将被包含此文件的事务的 Foundry 事务 ID 替换。
  • ${dataset}: 这部分的值将被替换为包含此文件的Foundry数据集ID。

示例：

考虑一个名为"spark/file_name"的Foundry数据集中的文件，在事务ID transaction_id 和数据集ID dataset_id 的情况下。如果您使用表达式 fi.*ame 作为键，file_${dt:DD-MM-YYYY}-${transaction}-${dataset}_end 作为值，当文件写入Azure时，它将被存储为 spark/file_79-03-2023-transaction_id-dataset_id_end。

标记文件

连接器可以在给定构建的所有数据复制完成后，向 Azure 存储写入一个空标志文件。该空文件表示内容已准备好供使用，并且将不再被修改。标志文件将被写入到 directoryPath。然而，如果 createTransactionFolders 被启用，则会为每个写入内容的文件夹生成一个标志文件。如果启用了标志文件，并且标志文件名为 confirmation.txt，则在构建中导出的文件写入后，所有标志文件将一次性写入。

标志文件是在构建结束时写入的，而不是在子文件夹导出后写入。

如果 Azure 中的文件比标志文件更新，这通常表明之前的导出未成功，或者该文件夹正在进行导出。

配置导出任务后，在右上角选择 保存。

源配置要求

使用虚拟表时，请记住以下源配置要求：

  • 您必须使用Foundry worker源。虚拟表不支持使用agent worker连接。
  • 确保建立双向连通性和允许列表，如本文件的“网络”部分所述。
  • 如果在代码仓库中使用虚拟表，请参考 虚拟表文档以了解所需的附加源配置详情。
  • 在设置源凭据时，您必须使用以下之一client credentials、username/password 或 workload identity federation。使用虚拟表时，不支持其他凭据选项。

Delta

为了在由虚拟表支撑的管道中启用增量支持，请确保在源 Delta 表上启用了 变更数据流 ↗。在 Python 转换 中支持 current 和 added 读取模式。_change_type、_commit_version 和 _commit_timestamp 列将在 Python 转换中可用。

Iceberg

要加载由 Apache Iceberg 表支撑的虚拟表，需要一个 Iceberg 目录。有关 Iceberg 目录的更多信息，请参阅 Apache Iceberg 文档 ↗ . 所有在源上注册的Iceberg表必须使用相同的Iceberg目录。

默认情况下，将使用S3中的Iceberg元数据文件创建表。注册表时必须提供一个warehousePath来指示这些元数据文件的位置。

Unity Catalog ↗可以在Databricks中使用Delta Universal Format（UniForm）时作为Iceberg目录。有关此集成的更多信息，请参阅 Databricks文档 ↗。目录可以在源的连接详细信息选项卡中进行配置。您需要提供端点和一个 个人访问令牌 以连接到Unity Catalog。应使用 catalog_name.schema_name.table_name 命名模式注册表。

增量支持依赖于Iceberg 增量读取 ↗，目前仅支持追加操作。在 Python Transforms 中的 current 和 added 读取模式是受支持的。

Parquet

使用 Parquet 的虚拟表依赖于模式推断。最多将使用 100 个文件来确定模式。
