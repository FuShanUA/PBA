Spark sidecar 转换

前提条件

以下文档假设您已具备以下前置工作知识：

  1. 容器化基础设施及容器镜像等概念 ↗
  2. PySpark 转换

Spark sidecar 转换允许您部署容器化代码，同时利用 Spark 和转换提供的现有基础设施。

将代码容器化后，您可以将任何代码和依赖项打包，以便在 Foundry 中运行。容器化工作流与转换功能集成，这意味着调度、分支和数据健康状态均已无缝整合。由于容器化逻辑与 Spark 执行器并行运行，您可以根据输入数据规模对容器化逻辑进行扩展。

简而言之，任何能在容器中运行的逻辑，都可用于在 Foundry 中处理、生成或消费数据。

如果您已熟悉容器化概念，请参阅以下章节以进一步了解如何使用 Spark sidecar 转换：

  • 架构
  • 构建镜像
  • 推送镜像
  • 编写 Spark sidecar 转换

进一步了解 Foundry 中的容器化。

架构

在 Foundry 中，转换功能可以利用 Spark 驱动程序在多个执行器之间分发处理任务，从而实现数据在数据集之间的输入输出，如下图所示：

使用 @sidecar 装饰器（由 transforms-sidecar 库提供）为转换添加注解，您可以精确指定一个容器，该容器会在 PySpark 转换中与每个执行器一起启动。这个由用户提供、包含自定义逻辑并与每个执行器同步运行的容器，称为 sidecar 容器。

每个执行器与 sidecar 容器之间的接口是一个共享卷或目录，用于传递以下信息：

  • 何时开始执行容器化逻辑
  • 在容器中需要处理哪些输入数据
  • 需要从容器中提取哪些输出数据
  • 何时结束容器化逻辑的执行

这些共享卷通过 @sidecar 装饰器的 Volume 参数指定，它们将作为 /opt/palantir/sidecars/shared-volumes/ 路径下的子文件夹存在。

接下来的章节将引导您准备并编写 Spark sidecar 转换。

构建镜像

要构建兼容 Spark sidecar 转换的镜像，该镜像必须满足镜像要求。此外，镜像还必须包含下文所述的关键组件，并且这些组件已包含在示例 Docker 镜像中。要构建此示例镜像，您需要用到 Python 脚本entrypoint.py。

您需要在本地计算机上安装 Docker，并且能够使用 docker CLI 命令（官方文档 ↗）。

推送镜像

要推送镜像，请创建一个新的 Artifacts 仓库，并按照说明为您的镜像打标签，然后将其推送到相应的 Docker 仓库。

  1. 创建一个 Artifacts 仓库。

  1. 将类型更改为 Docker。

  1. 按照屏幕上的说明生成令牌。
  2. 使用以下命令模式构建示例镜像：docker build . --tag <container_registry>/<image_name>:<image_tag> --platform linux/amd64，其中：
    • container_registry 表示您的 Foundry 实例容器注册表的地址，您可以在将 Docker 镜像推送到 Artifact 仓库的说明中，作为最后一条命令的一部分找到该地址。
    • image_name 和 image_tag 由您自行决定。本示例使用 simple_example:0.0.1。
  3. 复制 Artifacts 仓库中的说明，推送本地构建的镜像。请确保将最后一条命令中的 `<image_name>:<image_version>` 替换为上一步镜像构建过程中使用的 `image_name` 和 `image_version`。

编写 Spark sidecar 转换

  1. 在代码仓库应用中创建 Python 数据转换仓库。
  2. 在左侧的 Libraries 标签页中，添加 transforms-sidecar 并提交更改。
  3. 在 Settings > Libraries 下，添加你的 Artifact 仓库。
  4. 编写转换逻辑。

自定义 Dockerfile

您可以如上所示构建自己的 Dockerfile，但请务必涵盖以下内容：

  • 在第 10 行指定一个非 root 数字用户。这是镜像要求之一，有助于维持适当的安全态势，避免容器获得特权执行权限。

  • 在第 6 至 8 行设置共享卷的创建。正如前文架构部分所述，作为 /opt/palantir/sidecars/shared-volumes/ 子目录的共享卷，是将输入数据和输出数据从 PySpark 转换共享到 Sidecar 容器的主要方法。

    • 第 6 行创建目录。
    • 第 7 行确保将该目录的权限分配给所创建的用户。
    • 第 8 行将此共享目录的路径存储为环境变量，以供其他地方引用。

  • 在第 3 行向容器添加一个简单的 entrypoint 脚本，并在第 12 行将其设为 ENTRYPOINT。此步骤至关重要，因为 Spark Sidecar 转换本身不会指示 Sidecar 容器在启动前等待输入数据就绪。此外，Sidecar 转换也不会通知容器保持存活以等待输出数据被复制出去。提供的 entrypoint 脚本利用 Python 指示容器在执行指定逻辑之前，等待共享卷中写入 start_flag 文件。当指定逻辑执行完毕后，它会在同一目录中写入一个 done_flag。容器会等待共享卷中写入 close_flag，随后才会自行停止并自动清理。

如上例所示，容器化逻辑使用 POSIX Disk Dump (dd) 实用程序将 CSV 文件从共享目录复制并输入到存储在同一目录下的输出文件中。这个传递给 entrypoint 脚本的“命令”，可以是任何能够在容器中执行的逻辑。

Entrypoint

在与 Dockerfile 相同的本地文件夹中，将以下代码片段复制到名为 entrypoint.py 的文件中。


```python
#!/usr/bin/env python3

import os
import time
import subprocess
from datetime import datetime

import argparse

parser = argparse.ArgumentParser()
parser.add_argument("-c", "--command", type=str, help="要执行的模型命令")
args = parser.parse_args()
the_command = args.command.split(" ")


def run_process(exe):
    "定义一个函数用于运行命令并逐行捕获标准输出"
    p = subprocess.Popen(exe, stdout=subprocess.PIPE, stderr=subprocess.STDOUT)
    return iter(p.stdout.readline, b"")


start_flag_fname = "/opt/palantir/sidecars/shared-volumes/shared/start_flag"
done_flag_fname = "/opt/palantir/sidecars/shared-volumes/shared/done_flag"
close_flag_fname = "/opt/palantir/sidecars/shared-volumes/shared/close_flag"

# 等待启动标志
print(f"{datetime.utcnow().isoformat()}: 正在等待启动标志")
while not os.path.exists(start_flag_fname):
    time.sleep(1)
print(f"{datetime.utcnow().isoformat()}: 检测到启动标志")

# 执行模型，并将输出记录到文件中
with open("/opt/palantir/sidecars/shared-volumes/shared/logfile", "w") as logfile:
    for item in run_process(the_command):
        my_string = f"{datetime.utcnow().isoformat()}: {item}"
        print(my_string)
        logfile.write(my_string)
        logfile.flush()
print(f"{datetime.utcnow().isoformat()}: 执行完毕，正在写入输出文件")

# 写入完成标志
open(done_flag_fname, "w")
print(f"{datetime.utcnow().isoformat()}: 完成标志文件已写入")

# 在允许脚本结束之前等待关闭标志
while not os.path.exists(close_flag_fname):
    time.sleep(1)
print(f"{datetime.utcnow().isoformat()}: 检测到关闭标志。正在关闭")
```

示例 1：单次执行


下面的转换代码从 transforms-sidecar 库中导入了 @sidecar 装饰器和 Volume 原语。
该转换在注解中使用了这两个对象，以便每个执行器启动一个 simple-example:0.0.1 容器实例。每对执行器/sidecar 都将在 /opt/palantir/sidecars/shared-volumes/shared 路径下拥有一个共享卷。


第一个示例在一个执行器上启动一个容器实例，遵循下图所示的架构：




随后，该转换使用实用工具函数 lanch_udf_once 来启动一个 user_defined_function 实例。该用户定义函数将在一个执行器上运行，并与一个 sidecar 容器实例进行通信。用户定义函数将调用导入的实用工具函数来执行以下操作：


    

  • 将输入文件复制到共享目录，以便 sidecar 容器能够访问它们。
    • 

  • 复制启动标志，以便 sidecar 容器知道开始执行。
    • 

  • 等待容器化逻辑执行完毕。
    • 

  • 将容器化逻辑创建的文件复制出来。
    • 

  • 复制关闭标志，以便容器能够停止并得到清理。
    • 



```python
from transforms.api import transform, Input, Output
from transforms.sidecar import sidecar, Volume
from myproject.datasets.utils import copy_files_to_shared_directory, copy_start_flag, wait_for_done_flag
from myproject.datasets.utils import copy_output_files, copy_close_flag, launch_udf_once


@sidecar(image='simple-example', tag='0.0.1', volumes=[Volume("shared")])
@transform(
    output=Output(""),
    source=Input(""),
)
def compute(output, source, ctx):
    def user_defined_function(row):
        # 将文件从源目录复制到共享目录。
        copy_files_to_shared_directory(source)
        # 发送开始标志，让容器知道已收到所有输入文件
        copy_start_flag()
        # 持续轮询，直到写入停止标志或达到最大时间限制
        wait_for_done_flag()
        # 将输出文件从容器复制到输出数据集中
        output_fnames = [
            "start_flag",
            "outfile.csv",
            "logfile",
            "done_flag",
        ]
        copy_output_files(output, output_fnames)
        # 写入关闭标志，让容器知道你已提取数据
        copy_close_flag()
        # 用户定义函数必须返回内容
        return (row.ExecutionID, "success")
    # 这会生成一个任务，该任务映射到一个执行器，并启动一个“sidecar 容器”
    launch_udf_once(ctx, user_defined_function)
```


示例 2：并行执行


本示例会启动多个 sidecar 容器实例，每个实例处理一部分输入数据。随后，系统会收集这些信息并保存到输出数据集中。本示例更贴近下方所示的架构：




以下转换逻辑使用不同的实用工具函数对输入数据进行分区，并将各个文件发送至每个容器，从而在不同的输入数据块上执行相同的操作。这些实用工具函数经过编写，既能将输出文件保存为独立文件，也能将其保存为表格形式的输出数据集。


你会看到，@sidecar 装饰器和 Volume 规范所配置的参数与示例 1 中的保持一致。


此处设置了 @configure 标志，以确保每个执行器仅启动一个任务，且总共只能启动四个执行器。结合输入数据集恰好有四行数据且输入重分区数设为 4 这一条件，该配置意味着用户自定义函数的四个实例将在四个执行器上启动。因此，sidecar 容器的四个实例将恰好被启动，并分别处理各自的输入数据段。


请确保你的代码仓库已在 Settings > Spark 下导入了两个 Spark profiles。

1from transforms.api import transform, Input, Output, configure
2from transforms.sidecar import sidecar, Volume
3import uuid
4from myproject.datasets.utils import copy_start_flag, wait_for_done_flag, copy_close_flag
5from myproject.datasets.utils import write_this_row_as_a_csv_with_one_row
6from myproject.datasets.utils import copy_output_files_with_prefix, copy_out_a_row_from_the_output_csv
7
8
9@configure(["EXECUTOR_CORES_EXTRA_SMALL", "NUM_EXECUTORS_4"])
10@sidecar(image='simple-example', tag='0.0.1', volumes=[Volume("shared")])
11@transform(
12    output=Output("<first output dataset rid>"),
13    output_rows=Output("<second output dataset rid>"),
14    source=Input("<input dataset rid>"),
15)
16def compute(output, output_rows, source, ctx):
17
18    def user_defined_function(row):
19        # Copy files from source to shared directory
20        write_this_row_as_a_csv_with_one_row(row)
21
22        # Send the start flag so the container knows it has all the input files.
23        copy_start_flag()
24
25        # Iterate until the stop flag is written or you hit the maximum time limit.
26        wait_for_done_flag()
27
28        # Copy output files from the container to the output datasets
29        output_fnames = [
30            "start_flag",
31            "infile.csv",
32            "outfile.csv",
33            "logfile",
34            "done_flag",
35        ]
36        random_unique_prefix = f'{uuid.uuid4()}'[:8]
37        copy_output_files_with_prefix(output, output_fnames, random_unique_prefix)
38
39        outdata1, outdata2, outdata3 = copy_out_a_row_from_the_output_csv()
40
41        # Write the close flag so the container knows you have extracted the data.
42        copy_close_flag()
43
44        # The user-defined function must return something.
45        return (row.data1, row.data2, row.data3, "success", outdata1, outdata2, outdata3)
46
47    results = source.dataframe().repartition(4).rdd.map(user_defined_function)
48    columns = ["data1", "data2", "data3", "success", "outdata1", "outdata2", "outdata3"]
49    output_rows.write_dataframe(results.toDF(columns))

29def copy_output_files_with_prefix(output, output_fnames, prefix):

41def copy_files_to_shared_directory(source):
42    source_fs = source.filesystem()

80def launch_udf_once(ctx, user_defined_function):
81    # Using a dataframe with a single row, launch user_defined_function once on that row
82    schema = T.StructType([T.StructField("ExecutionID", T.IntegerType())])
83    ctx.spark_session.createDataFrame([{"ExecutionID": 1}], schema=schema).rdd.foreach(user_defined_function)
