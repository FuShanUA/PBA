什么是用例？

用例应用程序允许构建者在单一操作界面中组织其工作。通过将文件系统视图与本体管理视图相结合，开发者可以访问一个经过筛选的视图，该视图专注于他们负责的工作。

用例应用程序可以将任何项目作为其支撑项目，但如果以推荐的管道项目结构中的工作流项目为支撑，效果最佳。工作流项目用于存储应用程序，例如那些基于本体资源的 Workshop 和 Quiver 应用程序。

用例概览页

功能特性

用例应用程序的核心功能允许用户：

  • 查看所有用例，并在它们之间进行导航。
  • 基于新项目或现有项目创建新用例。
  • 直接在界面内的用例中创建新应用程序。
  • 在单一视图中查看用例中的所有应用程序。
  • 查看本体资源，包括添加到用例中的资源或该用例中任何应用程序所使用的资源。
  • 在用例及其连接的应用程序之间快速导航。

用户

对于使用 Foundry 本体来创建真实业务运营应用程序的工作流构建者而言，用例应用程序是理想之选。对于需要在不同项目之间切换以获取多种类型应用程序所需资源的构建者来说，它尤为实用。

如果您是首次使用用例应用程序，我们建议您阅读以下文档，以详细了解《空间导航》以及如何《开始使用》您的第一个用例。

您也可以选择从此处启动新用例。选择屏幕右侧的+ 创建新项，即可打开+ 创建用例弹窗。

概览

在用例概览中，您可以找到与所创建用例相关的 Foundry 资源摘要。利用这些资源，您可以创建基于本体实体或应用程序输出的工作流。

用例概览还提供了用例的快速元数据，包括描述、状态和数据主人。

详细了解如何为您的用例《编辑元数据》。

应用程序

在“应用程序”部分，您可以找到最近编辑过的三个应用程序，以及指向列出所有已连接应用程序页面的链接。

在概览页面该部分的右上角，点击创建新应用程序即可打开创建新应用程序弹窗。

详细了解如何将应用程序添加到您的用例中。

您还可以点击侧边栏中的应用程序以打开应用程序页面，查看已连接应用程序的完整列表。

本体组件

用例概览的“本体”部分包含两个子部分：对象类型和操作类型。

对象类型：用例视图中的对象类型，要么是用例中各应用程序正在使用的，要么是手动添加到用例中的。

操作类型：用例视图中的操作类型，要么是用例中各应用程序正在使用的，要么是手动添加到用例中的。

在侧边栏中选择对象类型，即可打开列表，查看用例中应用程序所使用的或手动添加的所有对象类型。

在侧边栏中点击操作类型，即可查看用例中应用程序所使用的或手动添加到用例中的操作类型列表。

底层数据

从侧边栏中选择底层数据页面，按对象类型查看数据源的完整列表。对于每种对象类型，您可以查看输入数据源列表以及回写数据集（如果存在）。

其他资源

在左侧侧边栏中选择其他资源，可查看底层项目中存在的所有其他资源列表。
