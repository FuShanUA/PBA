排期故障排查

调度器指标页面

开始排查排期问题的最佳方法之一是查看 调度器指标页面。该指标页面可以告诉你失败的原因，包括常见的失败模式，例如：

  • 计划构建失败。你将在 Run History 标签页中看到失败构建的证据，点击这些构建将导航到构建应用中的构建报告以获取完整日志。
  • 计划构建被忽略。Run History 标签页将显示任何已触发但未构建的构建在状态列中为 Ignored。
  • 排期未被触发 在预期的时间或频率下。在这种情况下，Run History 选项卡可能不会显示您期望的构建已被触发。

Versions 选项卡显示过去的计划版本和编辑记录，如果您发现计划突然行为与预期不同，这可能会有所帮助。检查与此次变化相符合的计划版本更改，并考虑将您的计划恢复到之前正常工作的状态。

计划构建失败

您可以通过查看 调度程序指标页面 上的 Run History 选项卡来验证计划是否在预期时间被触发。

如果计划被触发，但随后构建失败，您可以按照 调试指导 来调试此构建。

如果未设置适当的权限，计划也将无法构建。计划的权限取决于计划所处的令牌模式。请参见 项目范围的计划 有关更多信息。

计划构建被忽略

您可以通过查看 调度程序指标页面 上的 Run History 标签页来验证计划是否在预期时间触发。这通常也会给出计划被忽略的原因。

所有数据集都是最新的

如果所有目标数据集都是最新的，即它们的输入自上次对该数据集的构建以来没有更新，则计划运行将被忽略。如果是这种情况，您将在 Run History 标签页中看到此原因。在 计划编辑器 中，导航到计划列表。然后您可以选择按 Out-of-date 对数据谱系图进行着色，这将使您了解哪些作业规范被认为是过时的。

此行为可以使用 高级设置 中的 Force Build 选项在特殊情况下进行覆盖，但在此类情况之外，这会浪费计算资源。如果目标数据集中的任何数据集由 Phonograph 同步、通过 API 调用或数据连接同步进行转换，则它们可能不会显示为过时，并且可能需要启用 Force Build 选项以使计划运行。

计划仅构建部分数据集

如果计划仅触发部分数据集，您将在 调度程序指标页面 的 Run History 选项卡中看到相关证据。

这种情况的一个原因是只有部分数据集是过时的。调度程序只会构建过时的数据集，在构建过程中，那些最新的数据集将被忽略。有关更多故障排除信息，请参见 所有数据集都是最新的。当所有这些数据集都是最新的时，就会发生构建为 Ignored 的情况。

另一个原因可能是该数据集未包含在构建的数据集图中。在 计划编辑器 中，当选择某个计划时，要在构建的数据集会在数据血缘图中高亮显示。数据集的选择取决于 构建类型。如果使用 Connecting build，应特别注意验证对于使用相同数据集的多个分支的计划，是否有连接数据集存在。

计划未被触发

您可以通过查看 调度程序指标页面 上的 Run History 选项卡来验证计划是否在预期时间被触发。这里的一些常见调试步骤是：

  • 检查 计划是否未暂停。暂停的计划不会触发，直到取消暂停。
  • 检查 计划触发配置 . 如果之前成功过，请查看计划历史记录，看看最近是否对触发器进行了更改。
  • 如果计划使用的是事件触发器，请确认预期的事件确实发生了。例如，如果构建应在输入更新时触发，请检查输入上的最后一次构建是否成功运行，并且此构建上的事务是否在 数据集预览历史视图 中成功提交。

计划重试次数与配置不同

请注意，并非所有类型的失败都可以重试。当计划运行时，重试次数将被限制为管理员配置的最大值。有关更多信息，请参见 高级设置。

计划失败，出现 jobspec.inputs.trashed 或 jobspec.outputs.trashed 或数据血缘警告某些数据集已被删除的情况

计划失败，提示JobSpecInputsTrashed或JobSpecOutputsTrashed，或者数据血缘警告称某些数据集已被删除

这意味着该计划包含或读取了已删除的资源。您可以通过以下方式之一进行解决：

  • 从垃圾箱中恢复已删除的数据集。
  • 将已删除的数据集从计划中排除。如果此数据集在计划中用作另一个下游数据集的输入，则还需要执行以下操作之一：
    • 同时排除下游数据集和已删除的数据集。
    • 修改下游数据集的逻辑，使其不再以已删除的数据集作为输入。

调度程序权限

如果您遇到无法编辑计划的问题，项目范围权限可能是根本原因。

要在 项目范围模式 中编辑一个计划 ，用户必须在目标数据集上拥有Edit权限，触发数据集上拥有View权限，并且在该计划所作用的项目上拥有Edit权限。如果您失去了某个数据集的权限，请在保存更改之前将该数据集从计划中移除。

要编辑、删除或暂停一个计划，用户需要在目标数据集上拥有Edit权限，并且在该计划所作用的项目上拥有Edit权限。要查看一个计划，用户需要在目标数据集上拥有View权限。
