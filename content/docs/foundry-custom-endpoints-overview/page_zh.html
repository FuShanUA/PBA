自定义端点

测试版

自定义端点应用程序目前处于 测试 阶段，可能在您的注册中不可用。在积极开发过程中，功能可能会发生变化。如需访问自定义端点，请联系 Palantir 支持。

自定义端点 应用程序使开发者能够配置和部署用户定义的 API 端点，并使用自己的 URL 模式。用户还可以配置请求和响应格式以及端点规范，同时利用 Foundry 的后端功能。自定义端点通过动作和函数由本体论支撑。

自定义端点应用程序为将Foundry作为后端服务提供了托管基础设施。开发人员可以定义描述HTTP请求如何映射到本体论操作的端点元数据，从而无需外部中间件服务。这使组织能够通过符合其现有企业标准和规范的API来公开Foundry数据。

以下是标准Foundry API调用的示例：

Copied!
1POST https://{your enrollment}.palantirfoundry.com/api/v2/ontologies/{ontology}/queries/{queryApiName}/execute
2Body: {"parameters": {"form_id": 62536, "section_id": 5}}
3Response: {"code": 200, "data": {"value": ["Val1", "Val2", "Val3"]}}

以下是相同的API调用，已定制以适应现有的组织标准，并重新映射为一个 GET 请求：

Copied!
1GET https://subdomain.domain.com/myApi/form/{form_id}/section/{section_id}
2Response: {"code": 200, "data": {"section1": "Val1", "section2": "Val2", "section3": "Val3"}}

自定义端点的使用案例包括创建一个将Foundry数据与第三方服务结合的统一API，或是一个与现有企业URL模式和响应格式相匹配的兼容旧系统的端点。
