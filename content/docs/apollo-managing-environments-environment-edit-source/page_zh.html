配置环境编辑源

环境的编辑源决定了可以修改该环境设置的位置，例如其实体、配置或工件注册表。

环境编辑器可以在任何时候切换编辑源。

编辑源选项

在此中心进行编辑

您可以在当前中心直接编辑环境的设置。此中心是环境配置的权威来源。

当创建一个新环境时，默认会设置为“在此中心进行编辑”。这样可确保在从其他中心导入包时，不会意外覆盖环境的设置。

特点：

  • 可以在该中心修改环境的设置。
  • 该环境可以连同其设置一起导出到另一个中心。
  • 如果导入的包会覆盖此环境的设置，将因错误 InvalidTargetDeclaredStateSource 或 CannotImportInEnvironmentWithAuthoritativeDeclaredStateSource 被拒绝。如果您仍决定导入该包，则必须首先 切换编辑源 为“在另一个 Hub 上进行编辑”，然后重新导入该包。

在另一个 Hub 上编辑

环境的设置由另一个 Hub 管理。此 Hub 不是环境配置的权威来源。

要编辑环境的设置，您必须前往上游 Hub，在那里编辑环境设置，然后通过包将更改导入到此 Hub。

当环境首次导入到 Hub 时，它会根据所选的导出选项设置为“在另一个 Hub 上进行编辑”。

特征：

  • 此 Hub 无法修改环境的设置。
  • 环境设置通过包从另一个 Hub 导入。
  • 如果上游 Hub 允许，某些实体配置可能在此 Hub 上进行修改。
  • 可以在此中心修改单个实体配置，以应对紧急情况。
  • 环境可以连同其设置一起导出到另一个中心。

另一个中心的副本

该环境的设置是另一个上游中心的副本。此中心不是该环境配置的权威来源。 当一个环境首次导入到一个中心时，它会根据所选的导出选项被设置为“另一个中心的副本”。 此选项适合创建与其他中心连接的环境的只读视图。 如果其他中心发生中断，您可以将该环境重新连接到此中心，并在保持其设置的同时将编辑源切换为“在此中心编辑”。

特点：

  • 不能在此中心修改该环境的设置。
  • 环境设置通过包从另一个中心导入。
  • 即使上游中心允许，也不能在此中心修改实体配置。
  • 对于紧急情况，不能在此中心修改单个实体配置。
  • 环境可以连同其设置一起导出到另一个中心。

管理编辑源

查看当前编辑源

当前的编辑源显示在环境标题中。

切换编辑源

  1. 导航到您的环境。
  2. 从操作下拉菜单中选择切换编辑源。
  3. 选择新的连接设置：
    • 在此中心进行编辑: 在此中心启用对环境设置的编辑。
    • 在其他中心进行编辑: 禁用在此中心对环境设置的编辑。设置将从其他中心导入。
    • 另一个中心的副本：禁用此中心中环境的设置编辑。设置将从另一个中心导入，即使在紧急情况（break-glass）下也无法修改。
  4. 确认并应用更改。
