授权第三方应用程序

一旦第三方应用程序已通过 注册 并 启用，授权该第三方应用程序的过程就非常简单。

如果所需的第三方应用程序尚未注册和启用，请先进行注册和启用，然后再继续授权流程，或联系您的 Palantir 代表以请求注册和/或启用。

在这个例子中，我们将使用一个简单的测试应用程序来演示为 Gotham 访问授权已注册和启用的第三方应用程序的工作流程。第三方应用程序可能会提供某种 连接 选项，如下所示。

Foundry -> Foundry Apollo -> Apollo Gotham -> Gotham AIP -> AIP Ontology -> 本体论 ontology -> 本体论 Pipeline -> 管道 pipeline -> 管道 Transform -> 转换 transform -> 转换 Workshop -> Workshop workshop -> Workshop Code Repositories -> 代码仓库 Code Workspaces -> 代码工作区 Data Integration -> 数据集成 Model Integration -> 模型集成 Object -> 对象 object -> 对象 Dataset -> 数据集 dataset -> 数据集 Tenant -> 租户 tenant -> 租户 Marking -> 标记 marking -> 标记 Permission -> 权限 permission -> 权限 Role -> 角色 role -> 角色 Function -> 函数 function -> 函数 Action -> 操作 action -> 操作 Property -> 属性 property -> 属性 Interface -> 接口 interface -> 接口 内容： 尝试连接或授权已注册并启用的第三方应用程序时，会将您引导至平台并打开一个确认屏幕，如下所示，在此您可以选择 允许 或 不允许 访问。此确认屏幕将显示第三方应用程序请求权限的操作集；该操作集由第三方应用程序连接器的作者决定。

在允许访问后，您应该会被重定向回第三方应用程序，并收到访问权限的确认信息。

管理已授权的应用程序

用户设置 > 已授权的应用程序 选项卡显示了已被批准访问的第三方应用程序。

点击 操作 下拉菜单会显示以下选项：详情 和 撤销。

选择 详情 会显示有关第三方应用程序可以访问的数据的信息。

Selecting Revoke brings up a confirmation screen; revoking access removes the application’s ability to access information in your account.

After revoking access, we see that no applications have been authorized to access your account. An application whose access has been revoked can be added again by repeating this workflow.
