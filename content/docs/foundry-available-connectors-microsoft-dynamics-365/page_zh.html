Microsoft Dynamics 365

Microsoft Dynamics 365 连接器是 Palantir 提供的用于 Microsoft Dynamics 365 的驱动程序。

要创建新的 Microsoft Dynamics 365 数据源，请遵循 Palantir 提供的驱动程序的标准设置流程，然后使用以下章节进行 Microsoft Dynamics 365 的特定配置和网络设置。有关完整的属性参考，请参阅《官方 Microsoft Dynamics 365 驱动程序文档 ↗》。

配置

以下属性为必填或推荐配置。

属性是否必填？描述默认值
AuthScheme ↗必填指定连接到 Microsoft Dynamics 365 时使用的身份验证类型。如果将此属性留空，则使用默认身份验证。AzureAD
Edition ↗必填指定正在使用的 Microsoft Dynamics 365 版本。如果未指定，则默认为 (FinOpsOnline)。Sales
OrganizationURL ↗必填您的 Dynamics 365 组织的 URL，有时也称为资源。要找到您的 Microsoft Dynamics 365 实例的正确 URL，请搜索 Web 服务页面。https://myaccount.operations.dynamics.com/
AzureTenant ↗推荐通过名称（例如，contoso.onmicrosoft.com）或 ID 标识用于访问数据的 Microsoft Dynamics 365 租户。（视情况而定）。—
InitiateOAuth ↗推荐指定获取或刷新 OAuth 访问令牌的流程，以便在经过身份验证和授权的用户工作时维持其访问权限。GETANDREFRESH
OAuthClientId ↗推荐指定创建自定义 OAuth 应用程序时分配的客户端 ID。（也称为使用者密钥）。此 ID 用于向 OAuth 授权服务器注册自定义应用程序。—
OAuthClientSecret ↗推荐指定创建自定义 OAuth 应用程序时分配的客户端密钥。（也称为使用者密钥）。此密钥用于向 OAuth 授权服务器注册自定义应用程序。—
OAuthGrantType ↗推荐指定所选 OAuth 流程的授权类型。此值应与创建 OAuth 自定义应用程序期间设置的 grant_type 相同。CLIENT

网络连接

下表列出了数据源为了成功运行而需要能够访问的域。

对于每个域，添加相应的出口策略。如果数据源托管在本地并且无法从 Foundry 直接访问，请改用智能体代理出口策略；智能体主机本身也必须能够访问所列出的域。有关详细信息，请参阅将智能体用作代理。

域是否需要
