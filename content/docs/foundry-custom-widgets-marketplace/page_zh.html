将小部件集添加到 Marketplace 产品

使用 Foundry DevOps 将您的小部件集包含在 Marketplace 产品 中，以便其他用户安装和重用。 了解如何创建您的第一个产品。

支持的功能

所有自定义小部件功能都由 Marketplace 产品支持，但以下功能除外：

  • 在 Foundry 代码仓库中修改已安装的源代码以开发和发布新版本。已安装的 Foundry 代码仓库中的源代码仅可用于调试目的，不适用于开发。
  • 在安装过程中自动启用本体论 API。
  • 具有不同 API 名称的本体论资源。所有本体论资源必须使用与源相同的 API 名称，要么不带前缀进行安装，要么将输入映射到现有的本体论资源。
  • 使用版本化的函数与本体论 SDK。仅支持函数的最新版本。

将自定义小部件和小部件集添加到产品中

要将自定义小部件小部件集添加到产品中，首先 创建一个产品，然后 添加输出。选择 添加文件 选项，从 Compass 文件系统中导航到小部件集，并将其添加到您的产品中。

或者，如果您有一个嵌入了小部件集的 Workshop 应用程序，可以将 Workshop 模块添加到产品中，小部件集将自动包含在内。

需要 @osdk/widget.vite-plugin ↗ 的最低版本为 3.1.0。如果先前发布的某个小部件集版本使用的是较旧版本的 vite 插件，请先在源代码项目中更新 vite 插件版本，然后在 DevOps 中包含之前重新发布一个新的小部件集版本。

安装后手动启用本体论 API

Widget 使用本体论 SDK（OSDK）需要在安装后进行一次手动配置，以启用本体论 API。此配置在产品升级后仍然有效。有关详细说明，请参阅 在 Widget 集中使用本体论 SDK（OSDK）。
