将 Workshop 应用程序添加到 Marketplace 产品中

使用 Foundry DevOps 将您的 Workshop 应用程序纳入 Marketplace 产品，以便其他用户安装和复用。了解如何创建 Marketplace 产品。。

支持的功能

所有正式发布的 Workshop 功能均支持打包，但不支持静态/对象支持的场景。

将 Workshop 应用程序添加到产品中

如需将 Workshop 应用程序添加到产品中，请先创建产品，然后添加输出。选择添加文件选项，在 Compass 文件系统中导航至该 Workshop 应用程序并将其添加到您的产品中。

选择应用程序后，如果收到任何错误消息，请访问打包错误检查器获取更多信息。我们建议在尝试打包应用程序前先启用该检查器。

安装配置

如果您已创建希望作为 Marketplace 产品组成部分的 Workshop 应用程序，请在 Workshop 设置中启用安装配置，如下所示。此功能可在创建产品时提供调试打包错误的附加功能，并支持创建在安装期间公开的安装参数。

打包错误检查器

要成功打包 Workshop 应用程序，DevOps 必须能够成功识别所有应用程序依赖项，例如应用程序中使用的对象类型。启用安装配置后，任何错误都会显示在应用程序右上角的警告旁边。

请注意，根据应用程序大小，错误出现可能会有约 5-10 秒的延迟。如果没有错误，则不会显示任何内容。

在尝试于 DevOps 中打包应用程序之前启用检查器并检查错误通常是最佳实践。如果在启用检查器之前尝试打包，且存在任何未解决的问题，系统会提示您使用该检查器。

检查器显示的常见错误包括：

  • 应用程序引用了对象类型变量的属性，由于尚未与 Marketplace 完全集成的新功能，该属性无法解析。要解决此错误，您可以使用变量的模块接口设置手动指定对象类型。

  • 应用程序引用了已被删除或主键已重命名的属性或对象类型。要解决此问题，请用新的属性或对象类型替换原有的引用，或者在不再需要时删除该引用。

  • 应用程序使用了尚未配置打包的自定义微件。请联系您的 Palantir 代表获取进一步帮助。

  • 应用程序使用资源列表微件引用 Compass 资源。资源列表微件不支持自动检测 Compass 资源作为 Marketplace 打包的输入。使用此微件的应用程序在部署期间需要手动配置依赖项。

安装参数

启用安装配置后，设置下方会出现一个新面板。可在此面板中配置两种类型的安装参数：string 参数和 boolean 参数。

使用 string 参数允许在安装时自定义内容，如应用程序标题。例如，您可以使用 string 参数允许安装者在安装您的产品时使用其组织名称自定义应用程序。

使用 boolean 参数根据安装者偏好显示/隐藏内容。例如，您可以使用 boolean 参数允许安装者在相关时在应用程序中显示特定图表。

创建安装参数后，您可以将参数连接到 Workshop 变量。

其他用户安装您的产品时，这些参数将作为输入显示。
