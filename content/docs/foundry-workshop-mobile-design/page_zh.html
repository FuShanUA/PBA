设计最佳实践

针对移动设备上的交互设计，与桌面设备相比有着截然不同的考量维度。移动端的屏幕视口尺寸不同，用户通常通过触摸屏与组件交互，且天然对移动端的交互方式抱有特定预期。此外，还需留意应用在 iOS 和 Android 等不同操作系统上的跨平台使用情况。

尽管 Workshop 会尽可能调整移动应用中的组件，以提供对移动端友好的用户体验，但打造流畅体验的最终责任仍在于您。本文将介绍一系列移动应用设计的最佳实践，并详细阐述如何利用 Workshop 中的各项概念（包括正确使用布局和组件）来实现高质量的用户体验。

我们推荐以下移动端设计最佳实践：

  • 避免使用折叠区块，优先让用户滚动浏览内容
  • 使用大尺寸按钮，确保触控目标适配移动端
  • 使用抽屉进行复杂数据录入
  • 导航至子页面以实现多步交互
  • 使用导航栏扩展应用，或将其拆分为多个应用

优先支持滚动内容

桌面应用中常见的模式是在左右两侧使用可折叠面板（或侧边栏）。这些面板通常用于展示可配置的筛选器，或表格、列表中某一项的详细信息。

在移动设备上，折叠区块存在一些问题。首先，查找并点击控制区块开关的按钮并不容易；其次，移动端往往没有足够的屏幕空间来容纳折叠区块，难以提供自然的用户体验。

建议改用流式布局来展示一系列组件，供用户滚动浏览。垂直滚动是在移动设备上展示大量内容的自然方式，用户可以通过滚动浏览各个组件，轻松找到所需信息。

若需在屏幕上始终保留某些摘要信息或操作按钮，请使用包含流式布局的工具栏或行布局。这样既能保证特定组件始终固定在屏幕顶部或底部，又能让用户滚动浏览主体内容，如下方示例所示。

确保触控目标适配移动端

桌面设备用户通常通过鼠标和键盘与应用交互。鼠标让用户能够精准点击按钮及其他控件。然而，在触摸屏移动设备上，用户可能难以操作那些原本为桌面端设计的按钮尺寸。

在移动应用中添加按钮组组件时，我们建议在多数情况下选择大样式选项，以确保按钮在触摸屏上易于选中。在合适的情况下，还建议启用填充可用水平空间选项，以进一步扩大控件的触控范围。

需向最终用户展示多个按钮选项时，请将按钮组配置为菜单按钮类型。在移动应用中，该配置会显示一个顶层按钮，点击后会打开底部抽屉，供用户选择已配置的嵌套按钮。

使用抽屉进行数据录入

桌面应用通常有充足的屏幕空间来展示多个输入组件，如文本输入框、数值输入框或筛选列表等组件。有时，为了保持界面的整洁和条理性，这些组件会被嵌套在折叠区块中。

在移动应用中，我们建议使用抽屉式布局来提供专注的数据录入体验。这符合 iOS 设计规范中采用模态特性的要求，有助于用户专注于特定范围内或复杂的任务。

基于此，移动端 Workshop 应用会自动在抽屉中显示 Action 表单。当您添加按钮组微件时，选中后它将在全屏抽屉中展示。

除 Action 表单外，当用户需要输入多项信息时，您也应考虑在 Workshop 中添加抽屉。常见场景是用户需要为当前查看的一组对象自定义多个过滤器。您可以创建一个包含过滤器列表微件的抽屉供用户配置过滤器，随后添加一个按钮组微件来打开该抽屉。

有关设置抽屉的分步指南，请参阅教程部分中关于添加过滤器抽屉的内容。

多步交互使用子页面

在上述最佳实践的基础上，当您需要用户查看特定对象的详情或执行操作时，应创建一个展示该对象完整详情的新页面。在移动应用启动器中打开的 Workshop 应用支持浏览器级别的前进和后退交互，允许用户进入和退出子页面，从而在 Android 和 iOS 平台间提供一致的体验。

要设置此模式，请在 Workshop 应用中创建页面，添加用于展示相关信息的微件，并使用事件导航至该页面。随后，最终用户将能够通过 Web 浏览器导航回初始页面，如下所示。

使用导航栏扩展应用规模

在桌面应用中，通常使用选项卡式布局将多种不同功能集成在单一界面中。选项卡与可折叠区域的组合可用于构建复杂且数据丰富的界面，实现开放式分析并在屏幕上展示大量信息。

相较于桌面应用，移动应用通常需要更加贴合目标用户群体的特定需求。因此，移动应用中的每个顶级页面应保持相对简单，通常只需允许用户浏览和筛选一组对象，随后导航至其他页面查看详情并执行操作。

随着应用复杂度增加，且包含面向用户群体的多个工作流时，您应使用移动导航栏来呈现用户可访问的多个页面。每个导航项可引导用户进入不同的顶级页面，通常用于展示不同类型的对象，或提供针对同一组对象的多种检索方式。

例如，在用于展示技术员任务的应用中，名为“我的任务”的选项卡可显示当前用户的所有活动任务，名为“任务搜索”的选项卡可支持跨所有技术员搜索任务，而名为“历史记录”的最后一个选项卡则可展示用户先前已完成任务的记录。

有关配置导航栏的详细信息，请参阅导航栏微件页面。

导航栏支持在屏幕底部最多显示四个选项卡，您可以通过子菜单展示更多导航项。然而，若应用已扩展至具备多个选项卡，您应考虑将相关功能拆分至多个应用中。用户可通过应用启动器在各应用间导航，为每个应用设定单一明确的目标有助于保持功能简洁，并确保应用维持良好性能。
