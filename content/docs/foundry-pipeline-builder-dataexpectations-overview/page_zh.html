数据期望

数据期望是可以应用于数据集输出的要求。这些要求（称为“期望”）可用于创建检查，从而提升数据管道的稳定性。

可以为每个管道输出设置数据期望，以对最终的输出结果定义期望。管道构建器目前支持两种数据期望：主键和行数。

如果任何期望检查未通过，构建将会失败。作业期望面板将显示哪些数据期望通过，哪些失败。

主键数据期望

主键期望需提供一个或多个列名，并验证以下内容：

  • 每列均无空值。
  • 列的组合具有唯一性。

如果执行增量构建，且主键检查耗时较长，可以尝试为主键列添加投影。

主键数据期望示例

对于选定的特定列，我们会检查其下的每个条目是否唯一。

如果选择了两列，我们会检查这两列的组合是否唯一。

在示例中，我们将使用数据集中存在的 id 和 time 两列。

示例数据集：

idtime
18pm
19pm
28pm
38pm

上述示例将通过检查。这是因为，尽管 1 和 8pm 各自重复出现，但 id 和 time 的组合仍然保持唯一。

相反，以下情况将无法通过：

idtime
18pm
29pm
18pm

该表将无法通过检查，因为 1 和 8pm 的组合重复出现了。

行数数据期望

行数期望需提供最小和/或最大行数。

如果提供了最小行数，期望将验证数据行数是否至少达到指定的数量。

如果提供了最大行数，期望将验证数据行数是否最多不超过该数量。
