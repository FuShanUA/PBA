对象列表

对象列表微件用于以列表或网格形式展示对象数据。配置对象列表微件的模块构建者可以：

  • 展示一种或多种对象类型的数据。
  • 选择要展示的属性类型和关联对象及其显示格式。
  • 可选地选择一个媒体属性，为每个对象展示图像或图标。
  • 设置初始排序条件。
  • 展示在 Ontology Manager 中配置的条件格式和数字格式选项。
  • 按对象隐藏空值属性，以实现更紧凑的显示效果。
  • 允许在列表中进行单选或多选。
  • 在列表中选中对象时触发 Workshop 事件。

配置选项

下图展示了新添加的对象列表微件的初始状态及其初始配置面板：

对于对象列表微件，核心配置选项如下：

  • 输入数据
    • 对象集：此输入变量决定了微件中将要展示的数据。模块构建者可以复用本模块中其他位置创建的现有对象集变量，也可以直接在此内联定义一个新的对象集变量。在填充此对象集参数之前，下方所示的许多其他配置选项将无法进行配置。
  • 属性配置
    • 添加媒体：您可以选择在对象信息旁展示图像。有关媒体配置的更多信息，请参阅下文。
    • 添加属性：在此处选择某个属性后，该属性将显示在列表中。
    • 默认排序：此设置允许在列表中应用最多一个默认排序。模块构建者既可以对列表中显示的可见属性类型进行排序，也可以对未显示的隐藏属性类型进行排序。
    • 启用用户排序：启用后，列表上方将显示一个子标题，用户可通过它更改列表的排序条件。用户既可以对列表中显示的可见属性类型进行排序，也可以对未显示的隐藏属性类型进行排序。
    • 启用重新排序：启用并配置后，允许用户在微件中拖放项目来重新排序。项目仅可通过对象卡片右侧出现的拖拽柄进行拖动。有关重新排序配置的更多信息，请参阅下文。
    • 隐藏空值属性：启用后，将按对象在列表中隐藏空值属性。
  • 选择
    • 活动对象：这是此微件的两个输出变量中的第一个，用于输出当前活动/高亮显示行的单例对象集。随后，此对象集可在当前模块的下游微件中使用。
    • 启用活动对象自动选择：默认情况下，列表中的第一行会在加载时自动设为活动对象。禁用此设置可阻止此行为，导致加载时活动对象为空。
    • 选中对象：这是此微件的两个输出变量中的第二个，用于输出当前勾选/选中行的对象集。随后，此对象集可在当前模块的下游微件中使用。请注意，仅当启用多选开关设为 true 时，此输出变量才会被使用并填充。
    • 关于活动对象选择：此选项允许模块构建者配置在列表中选中某一行时触发的 Workshop 事件，例如弹出一个包含更详细对象视图的抽屉。
    • 启用多选：启用后，此开关允许用户在列表中勾选/选择多个对象。随后，此选中的对象集将通过选中对象对象集输出，供当前模块中的下游微件使用。
  • 显示与格式化
    • 标题：若已填写，列表上方将显示此处输入的字符串标题作为副标题。
    • 显示安全标记：开启后，属性安全标记将渲染为紧凑的灰色胶囊状标签，选中时会展开窗口视图。
    • 显示对象计数：开启后，列表上方将显示副标题，其中包含输入对象集变量所含对象的计数。
    • 微件显示方式：默认以列表形式显示对象，也可选择网格形式显示，并支持指定行数、网格中每个对象卡片的大小以及卡片间的间距。
    • 属性样式：默认情况下，属性与其值内联显示，也可设置为对齐显示。
    • 空状态消息：当此微件没有可显示的对象时，可自定义空状态消息。

媒体配置

配置选项

对于对象列表微件，媒体配置选项如下：

  • 属性：这是对象集中包含待显示图像的属性。目前支持图像 URL、图像附件和媒体引用属性。
  • 媒体显示：所选媒体可作为大图显示，或作为图标替换对象标题旁的本体图标。
  • 启用展开图像预览：开启后，可在模态窗口内展开并全屏查看图像。
  • 悬停时显示属性：开启后，鼠标悬停时将在图像上方的覆盖层中显示额外的属性。
  • 图像位置：图像可在多种位置显示，并可将其高度或宽度控制为固定尺寸。
  • 缩放：图像可裁剪以填满空间，也可适应容器以显示无裁剪的完整图像。

重新排序配置

构建者可允许用户在微件中拖放对象卡片，从而将对象列表中的项目配置为可重新排序。

限制条件

  • 目前仅支持在单个对象列表微件内进行重新排序。
  • 目前不支持在网格布局中重新排序项目。
  • 开启重新排序功能后，对象列表微件将仅显示前 500 个对象。

配置选项

  • 条件启用：设置布尔变量以确定微件内是否开启重新排序功能。
  • 主键数组：设置包含所显示对象主键的字符串数组。数组中主键的顺序将决定这些对象在微件已排序区域中的显示顺序。主键不在数组中的对象将显示在已排序对象下方的未排序列表区域中，这些对象可以不排序显示，或者在微件中配置了默认排序的情况下应用该排序。
    • 若将项目从未排序列表区域拖放至已排序区域，该对象的主键将被添加至主键数组。同理，若将项目拖放至微件的未排序列表区域，该项目的主键将从主键数组中移除。
    • 若主键数组为空，或所显示对象的主键与数组内容无交集，微件顶部将显示提示信息，引导用户“将未排序项目拖放至上方进行排序”。
    • 如果主键数组包含当前未在微件中显示的对象主键（例如被过滤出视图的对象），这些对象的顺序仍会保留。例如，假设数组顺序为 [A, B, C, D]，但 B 当前在微件中被过滤出视图，可见列表将显示为 [A, C, D]。如果用户将 D 重新拖拽到 A 和 C 之间，更新后的数组将变为 [A, B, D, C]。唯一的例外是用户将某个项拖动至最后一个可见对象下方的情况。此时，该项将直接排在该最后可见对象之后。
  • 关于重新排序：开发者可以选择更新主键数组，也可以选择在微件中的项重新排序时触发一个动作。
    • 更新数组变量：当用户在微件中重新排序某个项时，主键数组将随之更新以保持一致。
      • 请注意，主键数组的更新不会自动保存或存储，这意味着如果用户刷新或离开模块，主键数组的任何更改都将丢失。开发者需要配置一个动作来存储主键数组的新值，以便保存对数组的更新。如果您不想在微件中每次重新排序项时都触发动作，将 更新数组变量 选项与单独配置的动作结合使用（例如在模块其他位置的按钮组微件中）是一个不错的方案。
    • 提交动作：配置一个 重新排序动作，该动作会在用户每次于微件中重新排序项时触发。
      • 可以使用一个特殊的 已重排序数组 参数作为动作参数来保存对数组的更新，该参数表示重新排序时更新后的主键数组。
