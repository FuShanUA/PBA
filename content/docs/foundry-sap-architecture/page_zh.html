架构

将 Foundry 连接到 SAP 系统主要有三种架构模式。这三种模式均遵循标准的《数据连接架构》，并使用数据连接智能体。

直接连接到 SAP ERP 或 BW 系统

在此场景下，面向 SAP 应用的 Palantir Foundry 连接器 2.0（简称“连接器”）会直接安装在包含待读取数据或托管待执行函数/查询的 SAP ERP 或 Business Warehouse (BW) 系统上。

通过 SAP SLT 复制服务器连接到 SAP ERP 系统

在此场景下，连接器安装在 SAP SLT 复制服务器上，并在该服务器与源 ERP 系统之间建立远程函数调用 (RFC) 连接。系统基于数据库触发器，将数据从每个源 ERP 表复制到 SLT 系统中对应的运营增量队列 (ODQ) 中。

通过网关连接到远程 SAP ERP 系统

当源 ERP 系统不满足安装连接器的最低要求（SAP_BASIS，即适用于 SAP NetWeaver 和 SAP S/4HANA 系统的技术组件版本，要求 7.4 SP5 或更高版本）时，适用此场景。此时，连接器的主要组件安装在满足最低要求的“网关”应用服务器上，并在该服务器与源 ERP 系统之间建立 RFC 连接。连接器的远程智能体组件安装在源 ERP 系统上，用于协助读取数据或执行函数/查询。
