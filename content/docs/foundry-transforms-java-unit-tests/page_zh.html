单元测试

Java Transforms 支持将单元测试作为 CI 检查的一部分运行。

Java Transforms 目前不支持交互式单元测试或调试。如果您的使用场景需要此功能，请考虑改用 《Python Transforms》。

本页所述的 Java 单元测试仅适用于批处理管道，不支持流式管道。

当您将 com.palantir.transforms.lang.java-defaults 插件应用于 Java transforms 子项目时，默认会设置好单元测试环境。该插件会向子项目的 testCompile 配置中添加主流的 Java 单元测试库 《JUnit》 ↗、《Mockito》 ↗ 和 《AssertJ》 ↗。 您需要将测试文件放置在 src/test/java 目录下，以便 Gradle 自动识别它们。有关如何向环境中添加插件的更多细节，请参阅 《Python Tests》 中的步骤。

使用 Spark 进行测试

我们提供了 JUnit5 扩展和 JUnit4 规则，帮助您轻松访问受管理的 SparkSession。

如果您使用的是 JUnit5，可以像声明任何常规扩展一样声明该扩展：

已复制！
1import com.palantir.transforms.lang.java.testing.api.SparkSessionExtension;
2import org.junit.jupiter.api.extension.RegisterExtension;
3
4@RegisterExtension
5public static SparkSessionExtension sparkSession = new SparkSessionExtension();

如果您使用的是 JUnit4（现已弃用），可以在测试文件中将 SparkSession 声明为 ClassRule 或 Rule。例如：



已复制！
1import com.palantir.transforms.lang.java.testing.api.SparkSessionResource;
2import org.junit.ClassRule;
3
4@ClassRule
5public static SparkSessionResource sparkSession = new SparkSessionResource();


Copied!
1import com.palantir.transforms.lang.java.testing.api.SparkSessionResource;
2import org.junit.Rule;
3
4@Rule
5public static SparkSessionResource sparkSession = new SparkSessionResource();

在这两种情况下，您随后都可以在测试中正常使用该扩展或规则，如下方代码片段所示：

Copied!
1@Test
2public void myUnitTest() {
3    SparkSession sparkSession = sparkSession.get();
4}

需要注意的是，在 JUnit4 中 ClassRule 和 Rule 是不同的：如果将 SparkSession 声明为 ClassRule，同一个 SparkSession 将被测试类中的所有测试共享。这能节省重建 SparkSession 的时间，但类路径会继承自先前运行的测试。 相比之下，如果将 SparkSession 声明为 Rule，每个独立的测试都会获得一个全新且隔离的 SparkSession。如果在一个类中有大量使用 SparkSession 的测试，这会导致显著的性能下降。

警告

如果一个类中有大量使用 SparkSession 的测试，将 SparkSession 声明为 Rule 可能会导致显著的性能下降。因此，我们强烈建议尽可能使用 ClassRule，或者使用 JUnit5 的 Extension 来代替 JUnit4 的规则。
