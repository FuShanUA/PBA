单元测试

Java 转换支持在 CI 检查过程中运行单元测试。

Java Transforms 目前不支持交互式单元测试或调试。如果您的使用场景需要此功能，请考虑改用 Python Transforms。

本页面上描述的 Java 单元测试仅适用于批处理管道，不支持流式管道。

默认情况下，当您将 com.palantir.transforms.lang.java-defaults 插件应用于 Java 转换子项目时，会设置单元测试环境。它会将流行的 Java 单元测试库 JUnit ↗、Mockito ↗ 和 AssertJ ↗ 添加到子项目的 testCompile 配置中。 您应将测试文件放在 src/test/java 下，以便 Gradle 自动识别它们。请参见步骤 in Python 测试 以了解有关如何将插件添加到您的环境的更多详细信息。

使用 Spark 进行测试

我们提供一个 JUnit5 扩展和一个 JUnit4 规则，以帮助您轻松访问受管理的 SparkSession。

如果您使用的是 JUnit5，请像使用任何常规扩展一样声明该扩展：

Copied!
1import com.palantir.transforms.lang.java.testing.api.SparkSessionExtension;
2import org.junit.jupiter.api.extension.RegisterExtension;
3
4@RegisterExtension
5public static SparkSessionExtension sparkSession = new SparkSessionExtension();

如果您使用的是 JUnit4（现已弃用），可以将 SparkSession 声明在测试文件中，作为 ClassRule 或 Rule。例如：

Copied!
1import com.palantir.transforms.lang.java.testing.api.SparkSessionResource;
2import org.junit.ClassRule;
3
4@ClassRule
5public static SparkSessionResource sparkSession = new SparkSessionResource();
Copied!
1import com.palantir.transforms.lang.java.testing.api.SparkSessionResource;
2import org.junit.Rule;
3
4@Rule
5public static SparkSessionResource sparkSession = new SparkSessionResource();

在两种情况下，您都可以像下面的代码片段所示那样正常使用该扩展或规则：

Copied!
1@Test
2public void myUnitTest() {
3    SparkSession sparkSession = sparkSession.get();
4}

请注意，在 JUnit4 中 ClassRule 和 Rule 是不同的：如果你将 SparkSession 声明为 ClassRule，那么同一个 SparkSession 将在测试类内的所有测试中共享。这可以节省一些重新创建 SparkSession, 的时间，但类路径会从之前运行的测试中继承。 相比之下，如果你将 SparkSession 声明为 Rule，你将为每个单独的测试获得一个新的隔离的 SparkSession。如果你在一个类中有许多使用 SparkSession 的测试，这可能导致显著的性能下降。

警告

如果在一个类中有很多测试使用 SparkSession，那么将 Rule 声明为 SparkSession 可能会导致显著的性能下降。因此我们强烈建议尽可能使用 ClassRule 或者使用 JUnit5 的 Extension 而不是 JUnit4 的规则。
