添加一个地理时间序列输出

您可以将地理时间序列同步输出添加到管道中，以便在下游应用程序（如地图应用程序）中使用 Foundry 中的批量或流式地理时间数据。Foundry 的地理时间观测结果写入 Pipeline Builder。了解更多关于 不同的输出类型 和 地理时间序列 的信息。

前提条件

要在 Pipeline Builder 中使用地理时间序列，它们必须为您的注册启用。有关启用此功能的更多信息，请联系您的 Palantir 代表。

创建一个地理时间序列输出

打开 管道输出 面板位于Pipeline Builder界面的右侧。然后，选择添加>时空序列同步>添加以创建一个新的时空序列。

如果您在Pipeline Builder中看不到时空序列输出类型，请检查您的注册是否符合上述先决条件。

映射主字段

Pipeline Builder中的所有时空序列输出都有主字段（或称为固有字段），这些字段必须作为您管道上游转换逻辑的一部分进行生成。这些字段控制从管道中的行到时空观测的转换。有关主字段的更多信息，请参阅 时空序列数据建模。这些主字段包括：

  • 系列ID：一个字符串值，用作集成中每个轨迹的唯一标识符，例如用于区分不同航班的航班标识符。请参见 地理时间序列数据建模 了解有关选择序列 ID 的更多示例。
  • 时间戳： 表示给定观察时间的值。
  • 位置： 一个 GeoPoint 值，表示观察时的位置。在 Pipeline Builder 中，可以使用 Construct geopoint column 表达式，在上游转换逻辑中通过纬度和经度坐标创建 GeoPoint 值。
  • 海拔： 可选的 Double 类型的值，表示以米为单位的地球表面高度。

映射其他字段

除了上述必需的字段外，您还可以将自定义字段映射到输出观察结果。为此，请导航到地理时间序列输出面板的 属性 部分，并选择 添加属性 . 选择上游转换逻辑输出列，以便在下游观察中包含相应的值。您必须指定名称、映射是否为实时或静态，以及可选的说明，说明该字段代表什么。实时属性会随着每次观察而变化，例如飞机的航向。静态属性在整个系列中是恒定的，用于去规范化以实现搜索和过滤功能，例如飞机类型。

除了Pipeline Builder中的标准数据类型外，时空序列输出还支持将特殊的几何类型映射到时空序列数据模型中的等效类型。有关这些类型的更多信息，请参阅Pipeline Builder 的地理空间文档：

  • GeoPoint（地理点位）: 一个单独的点，可以在下游地图上渲染，并且可以与“Position（位置）”主字段中设置的点不同。
  • Geometry（几何）: 一个GeoJSON对象，可以在下游地图上渲染。

高级设置

除了基本的输出映射，时空序列输出支持以下高级和可选设置：

  • 存活时间： 一个 long 类型的值，表示数据在多少毫秒后将被视为过期或失效。过期的数据不会在下游应用程序中渲染。请注意，存活时间值不会影响索引到 dataset archive 的数据。
  • 索引设置： 允许用户指定他们希望如何对时空序列观测进行索引。有两种支持的数据存储选项：实时流和数据集存档。实时流可以实现实时渲染点和轨迹，而数据集存档会将所有系列数据保留在永久存储中，并为时空搜索进行索引。
  • 观测类型： 一个 string 类型的值，表示用于在下游应用程序中过滤数据的逻辑分类，例如“GPS ping”或“手动报告”。
  • 样式配置： 允许用户在下游应用程序中配置观察的外观。支持的属性包括图标大小、不透明度、旋转、符号、标题、几何宽度、几何颜色、几何填充、几何不透明度和轨迹颜色。
  • 观察模式：您可以选择一个现有的 观察模式 作为您时空序列的一部分。这样会预先填充与该模式相关的所有字段作为要映射的属性。为了支持向后兼容性，您将无法删除、更改或在实时和静态之间切换现有字段。如果您没有选择现有的模式，在管道输出首次成功部署时，将会为您创建一个新的模式。
  • 源系统： 源系统是不同时空序列同步的分组，您可以选择性地重复使用。如果您选择一个现有的源系统，那么在成功部署后，生成的时空序列集成将被上载到指定的源系统中。源系统中的现有集成不会被修改。如果您不选择现有的源系统，则会在管道输出首次成功部署时创建一个新的源系统。默认情况下，给定管道内的所有时空序列输出都将被分组到与该管道对应的源系统中。
  • 目标命名空间：如果用户有多个具有配置好的时空输出序列的命名空间，可以指定管道要导出观测值的目标命名空间。更多详情请参见 下方。有关配置额外目标的指导，请联系您的帕兰提尔代表。

安全

下游时空序列观测的安全性来源于您的管道输入数据：

  • 强制权限：
    • 下游观察结果将由Pipeline Builder中地理时空系列输出数据集的最新输出事务或流视图进行保护。此资源会将从输入数据集继承的强制性或分类标记传播到管道。
    • 如果在地理时空系列输出的构建之间上游输入数据标记发生变化，则后续构建将失败，以防止先前集成的数据标记发生意外更改。在这种情况下，您必须删除并重新创建带有适当标记的地理时空系列输出。
    • 所有管道输入都必须具有组织标记，并且必须为相应的地理时空系列输出配置目标命名空间的访问权限。地理时空系列输出上存在的组织标记必须是具有输出命名空间访问权限的组织的超集。
  • 可选权限：
    • 当首次部署地理时空系列输出时，父项目上具有读取权限及以上的所有组都将被授予对下游地理时空系列输出的查看权限。父项目的可选权限的后续更改不会传播到下游观察结果。如果您需要更细粒度的可选权限，请联系您的Palantir代表。

故障排除

在管道部署并触发构建后，观察结果应很快流入您的时空序列。如果数据未出现在下游应用程序中，您可以按照以下步骤进行故障排除：

  • 通过导航到右侧的输出面板并选择您输出旁边的三个点，然后选择查看支撑数据集，以验证数据是否出现在Pipeline Builder的观测输出数据集中。
  • 通过在上述输出面板中选择查看错误数据集来检查错误数据集中的错误。
  • 检查错误日志，查看当前运行的管道构建中是否存在运行时失败。这可以通过 Pipeline Builder 中的 历史记录 标签页访问。
  • 通过确保下游资源包含必要的分类标记（Marking）、强制标记（Mandatory Marking）和可选组，验证下游应用程序中没有安全配置错误，以便查看底层时空序列。
