添加地理时间序列输出

您可以将地理时间序列同步输出添加到管道中，以便在 Map 应用等下游应用中使用 Foundry 中的批处理或流式地理时间数据。来自 Foundry 的地理时间观测数据通过 Pipeline Builder 写入。了解更多关于《不同输出类型》和《地理时间序列》的信息。

前提条件

要在 Pipeline Builder 中使用地理时间序列，必须为您的租户启用该功能。请联系您的 Palantir 代表，了解有关启用此功能的更多信息。

创建地理时间序列输出

打开位于 Pipeline Builder 界面右侧的管道输出面板。然后，选择添加 > 地理时间序列同步 > 添加来创建新的地理时间序列。

如果在 Pipeline Builder 中看不到地理时间序列输出类型，请检查您的租户是否满足上述前提条件。

映射主要字段

Pipeline Builder 中的所有地理时间序列输出都包含主要字段（或固有字段），这些字段必须作为管道上游转换逻辑的一部分生成。它们控制着从管道中的行到地理时间观测数据的转换。有关主要字段的更多信息，请参阅《地理时间序列数据建模》。这些主要字段包括：

  • 序列 ID：字符串值，用作集成中每个轨迹的唯一标识符，例如用于区分不同航班的飞行标识符。有关选择序列 ID 的更多示例，请参阅《地理时间序列数据建模》。
  • 时间戳：表示给定观测时间的值。
  • 位置：GeoPoint 值，表示观测时的位置。在 Pipeline Builder 中，可以在上游转换逻辑中使用 构造地理点列 表达式，通过经纬度坐标创建 GeoPoint 值。
  • 海拔：可选的 Double 类型值，表示高于地球表面的高度（以米为单位）。

映射附加字段

除了上述必填字段外，您还可以将自定义字段映射到输出观测数据。为此，请导航至地理时间序列输出面板的属性部分，并选择添加属性。选择上游转换逻辑的输出列，使相应的值出现在下游观测数据中。您必须指定名称、映射方式应为实时还是静态，以及该字段所代表含义的文档说明（可选）。实时属性会随每次观测而变化，例如飞机的航向。静态属性在整个序列中保持不变，用于反规范化以支持搜索和过滤功能，例如飞机类型。

除了 Pipeline Builder 中的标准数据类型外，地理时间序列输出还支持将特殊的几何类型映射到地理时间序列数据模型中的对应类型。有关这些类型的详细信息，请参阅《Pipeline Builder 的地理空间文档》：

  • GeoPoint：单个点，可以在下游地图上渲染，并且可以与“位置”主要字段中设置的点不同。
  • 几何图形：GeoJSON 对象，可以在下游地图上渲染。

高级设置

除了基本输出映射外，地理时间序列输出还支持以下高级可选设置：

  • 生存时间：long 类型值，表示以毫秒为单位的时间长度，超过该时间后数据将被视为陈旧或过期。陈旧数据不会在下游应用中渲染。请注意，生存时间值不会影响索引到 数据集归档 的数据。
  • 索引设置：允许用户指定如何索引地理时间序列观测数据。支持两种数据存储选项：实时流和数据集归档。实时流支持实时渲染点和轨迹，而数据集归档会将所有序列数据保留在永久存储中，并建立索引以便进行地理时间搜索。
  • 观测类型：一个 string 类型值，代表用于在下游应用中过滤数据的逻辑分类，例如“GPS ping”或“人工报告”。
  • 样式配置：允许用户配置观测数据在下游应用中的显示外观。支持的属性包括图标大小、不透明度、旋转、符号化、标题、几何宽度、几何颜色、几何填充、几何不透明度以及轨迹颜色。
  • 观测模式：您可以选择将现有的观测模式作为地理时间序列的一部分。这样做会预填充与该模式关联的所有字段作为要映射的属性。为了支持向后兼容，您将无法在实时和静态之间删除、更改或切换现有字段。如果您未选择现有模式，则在首次成功部署管道输出时，将为您的集成创建一个新模式。
  • 源系统：源系统是不同地理时间序列同步的分组，您可以选择重复使用。如果您选择现有源系统，则在成功部署后，生成的地理时间序列集成将被 upsert 到指定的源系统中。源系统中的现有集成不会被修改。如果您未选择现有源系统，则在首次成功部署管道输出时将创建一个新系统。默认情况下，给定管道内的所有地理时间序列输出将被分组到与该管道对应的源系统中。
  • 目标命名空间：如果用户有权访问配置了地理时间输出序列的多个命名空间，则可以指定管道将观测数据导出到的目标命名空间。更多详细信息请参阅下文。请联系您的 Palantir 代表以获取配置其他目标的指导。

安全

下游地理时间序列观测数据的安全继承自您的管道输入数据：

  • 强制权限：
    • 下游观测数据将通过管道构建器中地理时间序列输出数据集上的最新输出事务或流式视图进行保护。此资源会将输入数据集继承的强制或分类标记传播到管道。
    • 如果上游输入数据标记在地理时间序列输出的不同构建版本之间发生变化，随后的构建将失败，以防止先前集成数据标记发生意外更改。在这种情况下，您必须删除并使用适当的标记重新创建地理时间序列输出。
    • 所有管道输入必须具有组织标记，并具有对为相应地理时间序列输出配置的目标命名空间的访问权限。地理时间序列输出上显示的组织标记必须是具有对该输出命名空间访问权限的组织集合的超集。
  • 自主权限：
    • 首次部署地理时间序列输出时，对父项目具有读取权限及以上的所有组将被授予对下游地理时间序列输出的查看访问权限。随后对父项目自主权限的更改不会传播到下游观测数据。如果您的地理时间序列工作流需要更精细的自主权限，请联系您的 Palantir 代表。

故障排除

在部署管道并触发构建后，观测数据应很快流入您的地理时间序列。如果数据未在下游应用中显示，您可以使用以下步骤进行故障排除：

  • 导航至右侧的 Outputs 面板，选择输出旁边的三个点，然后选择 View backing dataset.，验证数据是否出现在管道构建器的观测输出数据集中。
  • 通过在上述提到的 Outputs 面板中选择 View errors dataset，检查错误数据集中的错误。
  • 检查指示当前运行的管道构建中出现运行时失败的错误日志。这可以通过管道构建器中的 History 选项卡访问。
  • 通过确保下游资源包含必要的分类标记、强制标记和自主组以查看底层地理时间序列，验证下游应用中没有安全配置错误。
