选择导入数据集的分支

Code Workbook 针对导入数据集实现了分支回退机制，这意味着系统将根据工作簿中的分支层级结构，决定从何处拉取导入的数据集。

举例而言，假设您的工作簿包含 master 和 develop 两个分支，而输入数据集 titanic 同样具备这两个分支。当您处于 develop 分支时，导入的数据集默认会从 develop 拉取。若存在另一个仅在 master 而未在 develop 中存在的数据集，则输入数据将从 master 拉取。

这种回退结构能够支持实用的工作流。例如，若您已从其他工作簿或代码库导入了数据集，并希望使用该工作簿或代码库的分支版本来测试您的转换操作，只需在您的工作簿中新建一个同名分支，导入的数据集便会自动回退至相应的分支。
