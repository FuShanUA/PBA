拖放

拖放功能允许用户通过将数据从一个应用拖拽到另一个应用，轻松实现跨应用数据移动。将图片拖拽到文件夹中就是一个常见的使用场景。我们将可以从中拖入或拖出的组件称为拖放集成点。只需添加一个拖放集成点，您就可以将应用与其他支持拖放功能的应用连接起来，从而构建新的工作流。

拖放是一项被广泛接受的 Web 标准，可实现应用间的无缝集成。本文档涵盖了 Palantir 专有的拖放建议与指南，支持将数据拖入和拖出 Palantir 平台。这包括在 Gotham、Foundry 以及用户自主开发的应用之间进行拖放操作。有关 Palantir 专有拖放功能的更多信息，请参阅 Palantir 媒体类型。

核心概念

要实现拖放功能，您需要先熟悉以下四个主要概念：

  1. DataTransfer 对象
  2. 媒体类型
  3. 拖拽区
  4. 放置区

DataTransfer 对象

DataTransfer 对象就像一个容器，在计算机上将内容从一个地方移动或复制到另一个地方时，用于保存信息。DataTransfer 对象（也称为“DataTransfer”）在拖放操作中使用，例如将图片从一个文件夹移动到另一个文件夹；也用于复制和粘贴操作，比如复制文本并将其粘贴到文档中。在拖放场景下，DataTransfer 对象也被称为拖放载荷。

DataTransfer 可以保存不同类型的信息，如文本或图像。DataTransfer 对象最初专门为拖放而设计，但如今也用于其他功能。请参阅 《MDN 关于 DataTransfer 的文档》 ↗ 以了解更多信息。

媒体类型

媒体类型，前称 MIME 类型，表示所传输数据的性质。DataTransfer 对象利用媒体类型来标识正在传输的数据类别。例如，包含文本“Hello”的 DataTransfer 可能具有 text/plain 媒体类型，这样接受文本作为有效媒体类型的组件就能访问该文本。请参阅 《MDN 关于媒体类型的文档》 ↗ 以了解更多信息。

《MDN 接受的媒体类型完整列表》 ↗ 提供了适用于通用拖放功能的类型清单。有关 Palantir 特定功能的更多上下文，请参阅下文的 Palantir 媒体类型。

Palantir 媒体类型

Palantir 媒体类型代表 Palantir 平台专有的概念。为应用添加 Palantir 专有的拖放支持后，用户即可在您的应用与 Palantir 平台之间移动数据。我们已有支持标准媒体类型的拖放区，但建议在构建拖放工作流时，将 Palantir 媒体类型添加到拖放载荷中，以提升互操作性。

Palantir 平台中的拖放区会将 Palantir 媒体类型添加到拖放载荷中，或者接受包含这些媒体类型的拖放载荷。例如，代表 Foundry 对象资源标识符 (RID) 数据的 对象 媒体类型，并不像 image/jpeg 那样是通用媒体类型。相反，该对象媒体类型代表了 Palantir 专有的概念，若要在应用间移动此类数据，需要 Palantir 媒体支持。

目前支持的 Palantir 媒体类型包括 对象 和 对象集。

拖拽区

拖拽区是 文档对象模型 (DOM) ↗ 上的交互元素，允许用户“抓取”数据并将其转移到其他地方。用户发起拖拽时，会触发 dragstart 事件。拖拽区的工作原理是监听 DOM 元素上的 dragstart 事件，并向 DataTransfer 添加额外数据。

拖放区

拖放区是 DOM 上的交互元素，允许用户通过将数据“拖放”到元素上来传递数据。其工作原理是监听 DOM 元素上的 ondrop 事件，并在事件触发时从该事件的 DataTransfer 中获取信息。为了提供最佳用户体验，应设置拖放区的样式，使其在拖放操作过程中提供视觉反馈。拖放区还可以选择仅接受部分媒体类型，因此，针对媒体类型对当前拖放区是否有效提供视觉反馈，能让拖放功能更加直观、易用。

拖放工作原理

综合上述概念，拖放交互是一项被广泛接受的 Web 标准，由 DataTransfer 提供底层支持，用于保存用户拖动的数据。当最初拖动拖动区时，数据会连同与所传递数据类型相对应的媒体类型一起添加到 DataTransfer 中。随后，这些数据可被拖动到能够访问拖拽负载的拖放区上。拖放区可以根据拖拽负载的媒体类型选择性地接收数据，这使您能够为用户定制专属的拖放交互。

要了解拖放功能背后的技术细节，请按照《实现拖放集成点》的教程进行操作，并查阅 Workshop 的《拖放文档》。
