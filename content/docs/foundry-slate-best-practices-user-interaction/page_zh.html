启用用户交互

几乎每个应用程序都需要用户交互，Slate 提供了三种主要功能来实现交互性：

  • 输入控件： 提供常见 UX 元素的简单控件，用于捕获输入，例如下拉列表、文本字段、单选按钮等。
  • 其他控件内的选择状态： 大多数其他控件类型都支持某种类型的选择。每个控件中的选择方式略有不同，因此你需要进行尝试以了解可能的选择类型。
  • 事件触发器： 所有控件都有一组事件触发器，当控件状态发生变化或用户执行操作时会广播这些事件。有关如何将事件集成到您的应用程序中的更多信息，请参阅 事件 文档。

捕获用户输入的最简单模式是一个静态表单：添加输入控件并提供静态选项，然后可以在查询和函数中引用用户的选择以提供动态视图。然而，通常会有一些“链式”输入的复杂性，其中一个或多个输入中的选择会影响下一组输入中可用选项的集合。保持这些链路简短且易于理解非常重要——设置太多参数可能导致不直观且性能不佳的应用程序。请参见下面的“开放探索”反模式。

在这一更复杂的依赖输入配置中，最佳做法是将配置过滤器的工作流程与分析结果数据的工作流程分开。简而言之，这意味着您应该将任何依赖于这些用户输入的查询设置为 manual，并为用户提供一个按钮控件来 更新数据。这种模式可以确保您的应用程序不会因为每次过滤器更改都重新运行所有查询而浪费资源（以及用户时间）。如果您有任何类型的自由文本输入——文本字段 或 文本区域——这一点尤其关键。 类型小部件 - 否则下游依赖项如查询将随着每次用户输入而重新计算 每一次用户按键。

写回工作流

任何时候你的应用程序正在捕获用户输入以写回数据时，你必须配置查询手动运行，并在明确的用户操作上触发查询。否则你的数据持久化查询将随着每次输入变化运行，包括文本输入小部件中的每次按键，导致高度不可预期的行为。

在会话之间保持用户选择

如果你的应用程序需要配置多个不同的输入才能获得有用视图，用户可能希望有一种方式来“保存”他们的配置以便分享或以后返回。

可以 构建这种功能的自定义版本，但使用内置的 可共享视图 功能更容易。

重置选择并管理默认值

另一种常见模式是为用户提供一个操作，以将所有字段重置为一组默认值。这里最简单的模式是定义一个 v_defaults 变量：

Copied!
1{
2    "w_multiselectWidget_raw": ["a", "b"],
3    "w_multiselectWidget_display": ["Alpha", "Beta"],
4    "w_textInput": "default",
5    ...
6}

然后，在每个小部件的配置中，在 JSON 定义（在 </> 图标下）中，您可以对选定值属性的特定版本进行模板化。

对于除了原始值之外还有显示值的小部件，请确保对 两个 selectedValues 和 selectedDisplayValues 进行模板化：

Copied!
1{
2    ...
3    selectedValues: "{{v_defaults.w_multiselectWidget_raw}}",
4    selectedDisplayValues: "{{v_defaults.w_multiselectWidget_display}}",
5    ...
6}

最后一步是配置一个事件，以触发对 v_defaults 变量的更新，这将导致依赖图更新所有下游节点，其中包括所有具有模板化选择值的输入小部件，而选择将恢复为默认值。

看起来只需设置这样的事件就足够了： w_resetWidgetButton.click → v_defaults.set

Copied!
1// Set v_defaults to itself
2return {{v_defaults}}

如上文 Slate 中事物发生的原因 所述，这里有一个细微差别：如果节点的解析值没有发生变化，那么需要“强制”依赖图重新评估。因此我们需要添加一些“熵”，以便 Slate 知道这是一个新值。这很简单：

Copied!
1// Get the default values
2const defaults = {{v_defaults}}
3
4// Generate some randomness
5defaults.entropy = Math.random()
6
7// Set the value
8return defaults

使用这种模式，你可以轻松地为用户提供一个点击按钮来重置默认值，或在提交查询后重置它们。

你还可以通过 URL 参数设置变量值来进一步处理默认值。例如，你可能希望从另一个应用程序链接过来的用户有不同于直接进入应用的用户的默认值，而再次对于在其他工具内部 iframe 中查看的用户，又设置另一组不同的默认值。

要将此集成到上述模式中，而不是使用一个具有多个属性的复杂变量，你会“展开”该变量，使每个默认值都有一个单独的变量，这样你就可以在 URL 中使用变量名 —— 请参见 变量 关于其工作原理的更多细节）。另一个变量将用作“熵”，你可以将它们全部组合在一个返回对象的函数中，就像我们最初在单个变量中拥有的那个一样：

Copied!
1const defaults = {
2    "w_multiselectWidget_raw": {{v_multiSelect_raw}},
3    "w_multiselectWidget_display": {{v_multiSelect_raw}},
4    "w_textInput": {{v_textInput}},
5    "entropy": {{v_entropy}}
6    ...
7}

当你想要回到默认值时，只需将事件重置为一个随机值，默认值就会被重置。

“动态输入”

有时你需要捕获无法放入静态输入控件中的输入。在许多情况下，你可以换个角度思考问题并找到一个简单的解决方案，但假设你需要允许用户构建更复杂的过滤器，或者你的使用场景似乎无法通过任何创意性的静态输入实现。

你可能会被吸引使用重复容器控件来处理这个流程，但是这些控件仅限于显示。你可以在重复容器中使用输入控件，但它仅作用于该容器，并且你将无法从容器内的另一个控件以外的实例引用该输入控件。

相反，您可以拥有一个输入实例，但允许用户通过“保存”他们的选择来多次进行选择。您可以在 HTML 小部件中显示累积的选择，并甚至可以内置功能以删除之前的选择。

要实现此模式，请使用一个变量 v_userSelections，将其设置为空数组（[]）。然后每次用户点击按钮保存他们的选择时，您可以更新这个变量：

w_saveSelection.click → v_userSelections.set

Copied!
1// Get the existing selections
2const userSelections = {{v_userSelections}}
3
4// Get the new selection from the current state of the input widgets
5const newSelection = {
6    primaryCategory = {{w_primaryCategory.selectedValue}},
7    secondaryCategories = {{w_secondaryCategory.selectedValues}}
8    ...
9}
10
11// Combine the new selections with the prior selections
12const userSelections.push(newSelection);
13
14// Update the value of the variable
15return userSelections;

然后您可以实现其他事件，让用户操作他们已有的选择，最常见的是为每个选择添加“删除”操作，以便它们可以被删除。

使用此模式，您可以允许用户构建任意复杂的输入条件集。请注意复杂性增长，并进一步阅读上述事件部分中关于事件密集型模式的注意事项。

可共享视图

可共享视图允许用户捕获Slate应用的当前状态，并生成一个唯一的URL，可以与他人分享。当其他用户打开该URL时，应用会以创建视图时存在的相同选择和配置加载。

一个可共享视图存储以下内容：

  • 所有部件交互属性（interaction properties）的当前值；这些是可以通过用户交互更改的属性，例如selectedValue。
  • 所有本地变量（variables）的当前值。

当加载可共享视图URL时，保存的值将覆盖小部件和本地变量的默认值，并相应地初始化下游依赖项。

创建可共享视图

要手动创建一个可共享视图，请在查看（View）模式下打开应用程序，然后从操作（Actions）下拉菜单中选择获取可共享视图。这会生成一个带有视图ID的独特URL，您可以与他人共享。

Slate应用构建者可以使用 slate.saveView 和 slate.loadView 操作以编程方式创建并加载可共享视图。在视图被保存后，slate.viewSaved 事件会触发，提供可传递给 slate.loadView 的视图ID。

限制

可共享视图有以下限制：

  • 仅限首页： 可共享视图目前仅保存和恢复Slate应用程序的主页状态。仅包含本地变量；不保存或恢复共享变量。
  • 小部件和变量名称：可共享视图与小部件和变量名称相关联。如果在创建视图后重命名了小部件或变量，则在加载视图时将不会恢复该小部件或变量的已保存状态。
  • 小部件类型更改：如果在保存视图和打开视图之间更改了小部件的类型，则不会应用该小部件的已保存状态。
  • 与版本无关：可共享视图可在任何版本的应用程序中使用。只要引用的小部件和变量名称仍然存在，就可以在任何其他版本的应用程序中加载在某一版本应用程序中创建的视图。
