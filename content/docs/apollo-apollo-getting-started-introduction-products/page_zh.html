创建新产品和产品发布

现在您已将Kubernetes集群连接到Apollo，可以在产品目录中创建新产品。当将Apollo作为企业DevSecOps流程的一部分时，可以将一个 Apollo CLI 命令添加为仓库的 CI文件 中的一个阶段。

创建新产品

在此任务中，您将 将新产品添加到您的中心的产品目录 中。如果您没有自己的Helm图表，可以根据说明使用开源的Nginx Helm图表。

从产品目录中，从 操作 下拉菜单中选择 新建产品。

按照说明操作，在 产品ID（Product ID） 字段中提供以下值：

  • 组ID（Group ID）:
    • 如果安装 Nginx，使用 com.intro.helmcharts.bitnami
    • 如果使用自己的 helm chart，使用 com.intro.helmcharts
  • 工件ID（Artifact ID）: 使用你的 helm chart 名称或 nginx-${yourLastName}，如果没有的话，例如，nginx-smith

选择 创建新产品（Create a new Product） 以继续。

接下来，你将为你的产品设置权限角色。你应该已经是某个 Team 的成员，搜索 Team 的名称并为其分配 Product Release Creator 角色。

选择 设置初始角色（Set initial roles）。

创建新产品发布

在此任务中，你将在本地执行 Apollo CLI publish 命令，以创建在上一步中创建的产品的发布版本。

从 Nginx 产品页面或您的 Helm 图表中，从 操作（Actions） 下拉菜单中选择 创建新版本（Create new release）。

根据产品，将为您自动填充工件 ID 和组 ID。对于 Helm 图表版本：

  • 如果您使用自己的 Helm 图表，请使用您拥有的任何 Helm 图表版本。如果您有多个版本，建议使用较早的版本以方便下面的一些工作流程。
  • 对于 Nginx，请使用 13.1.5。

复制、粘贴并运行生成的命令到您的命令行工具中。

现在，该版本已在 Apollo 产品目录中可用。

使用附加详细信息和约束创建产品版本

除了产品名称和发布版本外，您在创建产品发布时可能还想声明其他值。例如，您可以为 Apollo 设置产品依赖项，以便在决定是否将发布版安装到环境中时进行评估。您可以通过编辑和创建产品发布清单来使用 Apollo CLI 完成此操作。尽管这不是 Apollo 入门指南中的核心工作流程，但如果您需要，可以 尝试该功能。
