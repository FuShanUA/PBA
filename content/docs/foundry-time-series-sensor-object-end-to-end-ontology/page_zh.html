使用 Ontology Manager 创建传感器对象类型

本指南介绍了如何使用 Ontology Manager 创建传感器对象类型，并将其链接至根对象类型。完成以下步骤后，您即可在平台中操作该传感器对象类型。在此示例中，您将创建一个 Flight Sensor 对象类型，并将其链接至 Flight 根对象类型。

第一部分：创建传感器对象类型

  1. 进入 Ontology Manager，从左侧面板中选择 Object types。
  2. 在屏幕右上角，选择 New object type。
  3. 在弹出的配置对话框中，配置您的对象类型元数据。
  4. 在 Properties 步骤中，选择使用 [Example] Sensors 数据集作为该对象类型的数据支撑，该数据集是您先前通过传感器数据管道创建的。
  5. 随后，在 Property 字段中，选择字符串类型的 unique_sensor_flight_id 属性作为主键。如果有对应选项可用，您也可以选择 Sync all columns from datasource 或 Map all columns。
  6. 选择 Title 列作为标题属性，这能让 Flight Sensor 对象类型在各应用中显示为可读名称。

  1. 视平台版本而定，您可能会看到为 Flight Sensor 对象类型配置权限和操作的选项。本示例无需额外的权限或操作配置。

  2. 完成对话框中的所有步骤后，选择 Create。

第二部分：为传感器对象类型配置时间序列属性

  1. 在 Ontology Manager 中进入 [Example] Flight Sensor 对象类型，从左侧面板选择 Capabilities 标签页。
  2. 在 Time series property 区域，选择 + Add。
  3. 选择现有的 Flight Sensor Series Id 属性作为时间序列属性，随后选择 Set as default time series property，使其能在 Quiver 中自动显示。

  1. 选择您在 Pipeline Builder 中创建的时间序列同步。本示例中名为 [Example] Time Series Sync | Sensor Readings。

  1. 选择 Add property 以保存时间序列属性配置。

第三部分：将传感器对象类型链接至根对象类型

按以下步骤在 [Example] Flight Sensor 与 [Example] Flight 对象类型间建立链接。

  1. 在 Ontology Manager 的 [Example] Flight Sensor 视图中，选择 New 下拉菜单并点击 Link type
  2. 在链接左侧选择 [Example] Flight Sensor 对象类型，在右侧选择 [Example] Flight 对象类型。
  3. 将左侧 Flight Sensor 对象类型的基数设为 Many，右侧 Flight 对象类型设为 One，这表示 Flight 对象类型与 Flight Sensor 对象类型之间是一对多关系。
  4. 将 flight_id 列设为 Flight Sensor 对象类型的外键，这会将 flight_id 设为 Flight 对象类型的主键。

了解更多关于链接类型的信息。

第四部分：配置传感器对象类型

在时间序列区域，确保传感器对象类型的开关已开启。设置 Sensor link 以使用刚刚创建的从 Flight 至 Flight Sensor 的链接。

  1. 将链接名称设为 Series Name 列。应用程序将以该序列名称呈现传感器对象数据。

  2. 在传感器对象类型设置中，通过选择 Units 下拉菜单来配置单位。

对于本用例，Is categorical 和 Internal interpolation 并非必选项，但可以通过传感器对象类型的属性进行推断。只有当需要明确区分分类时间序列值与数值型时间序列值时，才需要设置 Is categorical。

Internal interpolation 用于支持 Quiver 等应用程序推断相邻数据点之间的序列值。如需了解更多信息，请查阅《Quiver 插值文档》。

  1. 从屏幕右上角选择保存对对象类型的编辑，即可在本体及整个平台中查看更改。

至此，您已准备好在业务场景中使用 Flight Sensor 和 Flight 对象类型。请继续阅读文档，了解如何《在 Workshop 和 Quiver 中使用传感器对象类型时间序列数据》。

在我们的示例用例中，无需配置 Is enum?。Is deprecated 和 Sparkline preview 属性可忽略不计。
