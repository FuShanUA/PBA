添加、编辑和删除密钥

只有 环境编辑器 可以为环境中的实体添加、编辑和删除密钥。任何可以查看实体的人都可以查看其密钥的名称、描述和键。

密钥值在提交后立即加密，只能从环境内部解密。无论权限如何，您都无法从 Apollo Hub 中读取密钥值。

您可以通过导航到实体页面并从 操作 下拉菜单中选择 管理密钥 来添加、编辑和删除实体的密钥。

添加用户定义的密钥

在管理密钥菜单的右上角选择+ 添加密钥来创建一个新的用户定义的密钥。

填写表单以为一个实体创建一个新的密钥。

配置项：

  • 密钥名称： Apollo 中的密钥标识符。
  • 描述： 密钥的简要描述。
  • 键值： 密钥的键值对（s）。请注意，只有值被认为是敏感的。

您可以选择添加一对以向密钥添加另一个键值对。

填写完表单后选择提交。Apollo 将生成一个计划来创建该密钥。

Apollo 将使用您提供的信息在实体的环境（Environment）中创建一个 Kubernetes secret。该 secret 将与实体的 Helm chart 在同一个 Kubernetes 命名空间中创建。Kubernetes secret 的名称将是实体名称后跟您提供的 Secret name，用 - 分隔。例如，在上面的例子中，Kubernetes secret 的名称将是 example-entity-example-secret，例如 example-entity 实体。

为了允许立即加密 secret 值，Hub 需要知道 Agent 的公钥。如果在加密时，Hub 不知道最终执行 Plan 的 Agent 的公钥，用户添加或编辑 secret 值的请求将会失败。

编辑用户定义的 secret

要编辑用户定义的 secret，请选择 secret 名称右侧的铅笔图标。

这将打开编辑密钥表单。输入所需的更新并选择更新密钥。

此表单只会显示现有密钥。Apollo 无法检索密钥值，但只要键值对未被删除，则值将被保留。如果意外删除了键值对，您只能在选择取消后撤销该操作，然后再选择更新密钥。之后，您仍可以通过重新添加键值对来有效地撤销删除，但必须再次提供要设置的值；请注意，直到重新添加它的计划成功之前，该键值对在 Environment 中将不可用。

您只能编辑与管理实体相关的密钥。

删除用户定义的密钥

要删除用户定义的密钥，请在管理密钥菜单中选择密钥名称右侧的垃圾桶图标。

如果当前配置正在使用用户定义的密钥，删除它将破坏服务。无法通过当前配置模式发现哪些用户定义的密钥正在被使用。

密钥计划

Apollo 使用一个计划来执行所有密钥操作，这样更改只会在适当的维护窗口期间发生。您可以在实体主页的 计划 选项卡中查看密钥操作的进度。

密钥操作也是作为 Apollo 命令 的一部分执行的。
