添加、编辑和删除密钥

只有 环境编辑者 可以为环境中的实体添加、编辑和删除密钥。任何可以查看实体的人都可以看到其密钥的名称、描述和键。

密钥值在提交时立即加密，只能从环境中解密。无论权限如何，您都无法从您的 Apollo Hub 读取密钥值。

您可以通过导航到实体页面并从操作下拉菜单中选择管理密钥来添加、编辑和删除实体的密钥。

添加用户定义的密钥

在 管理密钥 菜单右上角选择 + 添加密钥 以创建一个新的用户定义的密钥。

填写表单以为一个实体创建一个新的密钥。

配置项：

  • 密钥名称： Apollo 中该密钥的标识符。
  • 描述： 该密钥的简要描述。
  • 键值对： 密钥的键值对。请注意，只有值被认为是敏感的。

您可以选择 添加键值对 来为密钥添加另一个键值对。

在完成表单后选择 提交。Apollo 将生成一个 计划 来创建该密钥。

Apollo 将使用您提供的信息在实体的环境中的 Kubernetes 中创建一个密钥。该密钥将与实体的 Helm 图表在同一 Kubernetes 命名空间中创建。Kubernetes 密钥的名称将是实体名称后接您提供的 密钥名称，用 - 分隔。在上面的例子中，对于 example-entity 实体，Kubernetes 密钥名称将是 example-entity-example-secret。

为了立即对秘密值进行加密，Hub需要知道代理的公钥。如果在加密时，Hub不知道最终执行计划的代理的公钥，则用户添加或编辑秘密值的请求将失败。

编辑用户定义的秘密

要编辑一个用户定义的秘密，请点击该秘密名称右侧的铅笔图标。

这将打开编辑秘密表单。输入所需的更新并选择更新秘密。

此表单将仅显示现有密钥。Apollo 无法检索密钥的值，但只要键值对未被删除，则值将被保留。如果意外删除了键值对，您只能在选择 取消 之前选择 更新密钥 来撤销操作。之后，您仍可以通过重新添加键值对来有效撤销删除，但您必须再次提供要设置的值；请注意，在重新添加该键值对的计划成功之前，该键值对将在环境中不可用。

您只能编辑与 受管理实体 相关的密钥。

删除用户定义的密钥

要删除用户定义的密钥，请在 管理密钥 菜单中选择密钥名称右侧的垃圾桶图标。

删除当前配置中正在使用的用户定义的密钥将导致服务中断。无法发现当前配置模式中正在使用的用户定义的密钥。

密钥计划

所有密钥操作都通过Apollo使用计划执行，这样更改只会在适当的维护窗口期间发生。您可以在实体主页的计划选项卡中查看密钥操作的进度。

密钥操作也可以作为Apollo命令的一部分执行。
