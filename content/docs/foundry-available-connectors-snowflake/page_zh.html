设置

  1. 打开 数据连接 应用程序，并在屏幕右上角选择 + 新源。
  2. 从可用的连接器类型中选择 Snowflake。
  3. 按照后续的配置提示继续使用下面部分的信息来设置您的连接器。

网络

对于在 Foundry 工作器上运行的连接，在 数据连接应用 中设置源时，必须添加适当的 出站策略。

要识别你的 Snowflake 账户的主机名和端口号以进行白名单设置，可以在 Snowflake 控制台中运行以下命令。确保至少将 SNOWFLAKE_DEPLOYMENT 和 STAGE 的条目作为出站策略添加到 Foundry 中。

Copied!
1SELECT t.VALUE:type::VARCHAR as type,
2       t.VALUE:host::VARCHAR as host,
3       t.VALUE:port as port
4FROM TABLE(FLATTEN(input => PARSE_JSON(SYSTEM$ALLOWLIST()))) AS t;

有关识别要白名单的主机名和端口号的更多信息，请参阅 Snowflake 的 官方文档 ↗。

从 Foundry 到 Snowflake 的连接通常来自你环境的默认公共网关 IP。然而，同一云服务提供商内部（例如 AWS-AWS 或 Azure-Azure）的流量可能会使用不同的路由，并需要通过 PrivateLink 建立连接。请参见下面每个云服务提供商所需的额外设置，或联系你的 Palantir 代表获取更多指导。

托管在S3上的Snowflake实例

如果你的Snowflake实例配置为通过 VPCE ↗ 将内部S3阶段流量路由，那么Snowflake JDBC驱动程序必须手动配置为 不 使用自定义的VPCE域名。否则，驱动程序将被路由到自定义的VPCE域名（从Foundry的VPC中无法访问），并且会失败连接到格式为 <bucketname>.bucket.vpce-<vpceid>.s3.<region>.vpce.amazonaws.com 的URL。

你可以通过在实例的 连接详细信息 中添加一个JDBC连接属性来手动配置此设置，键为 S3_STAGE_VPCE_DNS_NAME，值字段为空（相当于将其设置为 null）。 然后，S3阶段流量将通过AWS S3网关端点（<bucketname>.bucket.s3.<region>.vpce.amazonaws.com）进行路由，这保持了私有连接，因此流量不会通过公共互联网进行路由。

有关更多信息，请查阅我们的 PrivateLink出站文档 ↗。

对于依赖于与您的Foundry实例位于同一区域的S3存储桶的出站策略，请确保已完成我们在Amazon S3存储桶策略文档中详细说明的额外配置步骤，适用于受影响的存储桶。

Snowflake 实例托管在 Azure 上

用于 Foundry Snowflake 连接器的 Snowflake JDBC 驱动程序在获取数据时可能会尝试直接连接到底层“内部阶段”存储桶。对于托管在 Azure 上的 Snowflake，因为 Azure 托管的 Foundry 注册通过 Azure 服务端点路由流量，因此必须按照以下说明显式地允许从 Foundry 到底层阶段存储桶的网络连接。

收集有关您的 Snowflake 仓库的信息

为了建立与 Foundry 的网络连接，您需要以下关于 Azure 托管的 Snowflake 仓库的信息：

  • < a href="#system-allowlist-domains">系统允许列表域名的完整列表。
  • 您的 Azure 存储帐户标识符，从 Snowflake 获取。
系统允许列表域名

使用 SYSTEM$ALLOWLIST 命令获取可能需要成功连接的域名完整列表。

  • 注意：这是与上面用于定义出站策略相同的命令，并在数据连接中的网络面板提示中进行了说明。
  • 此列表将包括用作您的 Snowflake 仓库阶段的 Azure 存储桶的域名。
Azure 存储帐户标识符

对于从 SYSTEM$ALLOWLIST 命令返回的 Azure 存储桶，您还需要检索存储帐户标识符。

  • 如果你使用的是 Snowflake 标准版或企业版，你需要向 Snowflake 支持团队提交工单以请求存储帐户标识符。
  • 如果你使用的是 Snowflake 商业关键版 ↗，你可以按照以下步骤获取存储帐户标识符：
    • 将 ENABLE_INTERNAL_STAGES_PRIVATELINK ↗ 参数设置为 TRUE 以用于该帐户。
    • 然后，调用 SYSTEM$GET_PRIVATELINK_CONFIG() ↗ 函数，该函数返回一个名为 privatelink-internal-stage 的字段，其中包含 Azure 存储帐户资源标识符。
      • 请注意，即使你不是通过 PrivateLink 连接，仍然需要检索并提供存储帐户资源标识符。

完整的 Azure 存储帐户资源标识符的格式如下：

/subscriptions/{subscriptionId}/resourceGroups/{resourceGroupName}/providers/Microsoft.Storage/storageAccounts/{storageAccountName}

有关如何在 Azure 控制台中直接查找 Azure 存储帐户资源 ID 的更多信息，请参阅 Azure 文档 ↗。使用 VNET 规则和存储帐户标识符来限制跨帐户网络流量符合 Microsoft 发布的最佳实践 ↗ ，并且应用于从Azure计算实例内部连接到Azure托管的Snowflake仓库的所有连接。

允许从Foundry到与您的Snowflake仓库关联的Azure存储帐户的出站流量

现在您已经收集了关于您的Snowflake仓库所需的信息，您可以创建所需的策略以启用Foundry访问您的Snowflake数据。

  1. 为Azure存储内部阶段创建一个标准出站策略，并将其附加到您的Snowflake源。

    • 请注意，您需要添加SYSTEM$ALLOWLIST命令返回的所有内容的策略，而不仅仅是存储桶域名。

  2. 创建一个Azure存储策略，粘贴存储帐户资源标识符。

返回到Data Connection中的您的Snowflake源，确认您可以浏览该源并运行同步操作。

Iceberg 表（仅限虚拟表）

本文档中的 虚拟表 部分提供了有关集成 Snowflake Horizon 目录中注册的 Iceberg 表的详细信息。此功能需要与存储 Iceberg 表的外部卷的网络连接。您必须为每个外部卷创建 网络出站策略，因为 Foundry 直接读写存储位置，而不是使用 Snowflake 计算仓库查询该表。

有关外部卷以及如何确定表存储位置的更多信息，请参阅 Snowflake 官方文档 ↗。

为外部卷配置出站策略可允许网络流量从 Foundry 出发到该存储位置。由于不同云提供商的网络控制可能有所不同，因此您应确保存储位置上的任何网络控制都允许来自 Foundry 的网络流量。 了解有关识别 Foundry 流量来源的 IP 地址的更多信息。.

此外，请参考此网络文档的其他部分，以确保正确设置与您Foundry实例所在同一云区域的外部卷的连接。

虚拟表

本节提供有关使用虚拟表与Snowflake源的相关详细信息。当同步到Foundry数据集时，本节不适用。

Snowflake连接器现在在使用虚拟表访问Horizon目录中注册的Iceberg表时提供了增强功能。Foundry使用Horizon目录中暴露的Iceberg REST API来访问表以及读取和写入底层存储位置的数据，这些存储位置在Snowflake中配置为外部卷。此外，Foundry使用Horizon目录凭证发放功能来确保对云对象存储的安全访问。这也可以提高对这些表的读写性能。


如果满足以下条件，连接器将自动暴露Iceberg功能：

  1. 配置网络出站策略，允许从Foundry到存储表的外部卷的连接。
  2. 在源上配置具有从Horizon目录获取已发行凭证权限的凭据。



Foundry 使用 Iceberg 客户端直接建立连接，以读取或写入存储位置中的表 不 使用 Snowflake 计算。但是，Foundry 仍然使用源上配置的仓库进行某些元数据查询，例如确定正在访问的表类型。

有关通过 Snowflake Horizon 目录使用外部引擎查询 Iceberg 表的更多信息，请参阅 Snowflake 官方文档 ↗。对 Horizon 目录的写入目前是 Snowflake 的 私有预览 功能。有关启用对外部卷的网络访问的详细信息，请参阅本文档的 网络 部分。

Foundry 如果未满足上述任何要求，会将 Iceberg 表视为常规 Snowflake 表。与 Snowflake 的连接方式与其他 Snowflake 数据（如表、视图或物化视图）相同，并依赖于 Snowflake 计算仓库来读写表。

源配置要求

使用 虚拟表 时，请记住以下源配置要求：

  • 您必须使用 Foundry 工作进程 源。虚拟表不支持使用 代理工作进程 连接。
  • 确保已按照 本文档的网络部分 中所述建立双向连接和允许列表。

  • 提供的凭据必须具有对数据仓库的使用权限。

计算下推

Foundry 在使用虚拟表时，可以将计算下推到 Snowflake。虚拟表输入利用 Snowflake Spark 连接器 ↗，该连接器内置了谓词下推支持。

当使用注册到相同源作为管道的输入和输出的 Snowflake 虚拟表时，可以完全将计算联邦到 Snowflake。要将计算下推到 Snowflake，请查阅 Python 文档 获取详细信息。要在 Pipeline Builder 中将计算下推到 Snowflake，请查阅 外部管道文档。

在代码中使用 Snowflake 源

您可以使用 编程替代方案 来连接到 Snowflake 源以处理更复杂的场景。

下面的示例演示如何使用 Snowflake Python 连接器 ↗（snowflake-connector-python）在 外部转换 中连接到 Snowflake 源。认证通过 OIDC 处理。 , 无需静态Snowflake凭据即可提供短期OAuth令牌。

通过外部转换从Snowflake读取

此示例使用基于OIDC的OAuth凭证从Snowflake表中读取数据。

Copied!
1import polars as pl
2from transforms.api import transform, Output, LightweightOutput
3from transforms.external.systems import ResolvedSource, external_systems, Source
4import snowflake.connector
5
6
7@external_systems(
8    snowflake_source=Source("<source_rid>")
9)
10@transform.using(
11    output=Output("<output_dataset_rid>")
12)
13def read_from_snowflake(output: LightweightOutput, snowflake_source: ResolvedSource):
14    conn = snowflake.connector.connect(
15        authenticator="oauth",
16        token=snowflake_source.get_session_credentials().get().access_token,
17        account="<account_identifier>",
18        warehouse="<warehouse_name>",
19        database="<database_name>",
20    )
21
22    cursor = conn.cursor()
23    try:
24        cursor.execute("SELECT * FROM my_table LIMIT 100")
25        columns = [col[0] for col in cursor.description]
26        rows = cursor.fetchall()
27    finally:
28        cursor.close()
29        conn.close()
30
31    df = pl.DataFrame(rows, schema=columns)
32    output.write_table(df)

通过外部转换写入Snowflake

此示例将数据从Foundry数据集导出到Snowflake表。

Copied!
1import polars as pl
2from transforms.api import transform, Input, Output, LightweightOutput
3from transforms.external.systems import ResolvedSource, external_systems, Source
4import snowflake.connector
5
6
7@external_systems(
8    snowflake_source=Source("<source_rid>")
9)
10@transform.using(
11    output=Output("<output_dataset_rid>"),
12    source_data=Input("<input_dataset_rid>"),
13)
14def write_to_snowflake(output: LightweightOutput, snowflake_source: ResolvedSource, source_data):
15    conn = snowflake.connector.connect(
16        authenticator="oauth",
17        token=snowflake_source.get_session_credentials().get().access_token,
18        account="<account_identifier>",
19        warehouse="<warehouse_name>",
20        database="<database_name>",
21    )
22
23    cursor = conn.cursor()
24    try:
25        df = source_data.dataframe()
26        for row in df.iter_rows(named=True):
27            cursor.execute(
28                "INSERT INTO my_table (col1, col2) VALUES (%s, %s)",
29                (row["col1"], row["col2"]),
30            )
31    finally:
32        cursor.close()
33        conn.close()
34
35    output.write_table(df)

有关使用带有OIDC功能的数据源的会话凭据的更多详情，请参阅 Python中的数据源 文档。

数据模型

请注意，类型为 数组（数组） 的列 array ↗, object ↗, 以及 variant ↗ 将由 Foundry 解析为类型 string. 这是由于源数据的可变类型所致。

例如，Snowflake 数组 [ 1, 2, 3 ] 将被 Foundry 解释为字符串 "[1,2,3]"。

有关更多详细信息，请参阅 Snowflake 的 官方文档 ↗。
