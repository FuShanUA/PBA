创建地理空间转换

借助 Pipeline Builder，您可以加载、转换并运用地理空间数据。若 Pipeline Builder 的现有功能暂不支持您的地理空间工作流，请查阅 Foundry 的《遗留地理空间文档》以在代码仓库中转换数据。

地理空间数据建模

逻辑类型

Pipeline Builder 在内部使用逻辑类型这一概念对地理空间数据进行建模。逻辑类型本质上是一种基础类型（字符串、整数、布尔值、数组、结构体），但对其所表示的数据施加了额外限制。例如，Geometry 类型被定义为必须是有效 GeoJSON 的字符串，而 GeoPoint 必须是一个结构体，其经度介于 -180 和 180 之间，纬度介于 -90 和 90 之间，包含边界值。受支持类型的完整列表请见下文。

Pipeline Builder 中的所有逻辑类型均继承自其基础类型。例如，几何图形可用作期望输入为字符串类型的表达式的输入，反之则不可行。要从基础类型强制转换为扩展该基础类型的特定逻辑类型，可以使用“逻辑类型转换”表达式。该表达式会将该逻辑类型关联的限制条件应用于数据，并将未通过校验的值置空。表达式支持将逻辑类型指定为输入和输出，这确保了当地理空间专用表达式需要 GeoJSON 字符串时，接收到的必然是 GeoJSON 字符串。

支持的地理空间类型

Pipeline Builder 目前支持以下地理空间类型：

  • GeoPoint： 一个包含 longitude 和 latitude 的结构体，其中 longitude 为介于 -180 和 180 之间的双精度浮点数，latitude 为介于 -90 和 90 之间的双精度浮点数，均包含边界值。根据 WGS:84 或 EPSG:4326 坐标参考系统 (CRS)，GeoPoint 必须是有效的 (x, y) 坐标。
  • Geometry： 符合 GeoJSON 规范的字符串化 JSON 数据块。与 GeoPoint 类型一样，单个坐标应采用 WGS:84/EPSG:4326 格式。
  • H3Index： 表示有效 H3 六边形索引的字符串。
  • LatLonBoundingBox： 一个边界框，由 minLat、minLon、maxLat、maxLon 结构体表示，其中每个条目都是有效的 GeoPoint，且 maxLat > minLat，maxLon > minLon。
  • Ontology GeoPoint： 与本体 的 GeoPoint 属性类型兼容的字符串，需满足 {lat},{lon} 格式，其中 -90 <= lat <= 90 且 -180 <= lon <= 180。
  • MGRS： 表示有效 MGRS（军用格网参考系统）坐标的字符串。

加载地理空间数据

Pipeline Builder 支持针对地理空间数据的多种转换和表达式。

  • GeoPoint：
    • 构建 GeoPoint 列： 接收一对 lat,lon，校验上述边界范围，并将其转换为 GeoPoint 表示形式。
    • 从坐标系统 (CRS) 创建 GeoPoint： 接收一对 x,y 和一个坐标参考系统，将该 (x,y) 投影至 WGS:84，随后构建 GeoPoint 表示形式。支持从 EPSG 数据库中的大多数坐标系统进行转换，包括所有 UTM 分区。
  • Geometry：
    • 解析知名文本 (WKT)： 将知名文本 (WKT) 字符串转换为几何逻辑类型。可选择提供源坐标系统标识符，以便在 WKT 尚未采用 WGS:84 时，从源 CRS 转换至 WGS:84。
    • 标准化几何： 给定 WGS:84 格式的 GeoJSON 字符串，对以下属性进行标准化：正确顺序（右手定则）、闭合环、去重以及点的维度一致性。
    • 从 Shapefile 中提取行： 给定一个原始 shapefile 数据集，将每个 shapefile 解析为包含每个条目几何图形和属性的行。输出数据集将包含一个几何列，以及用户列出的每个属性对应的列。可针对非 WGS:84 数据集指定坐标参考系统。
    • 从 GeoJSON 中提取行： 给定一个原始 GeoJSON 文件数据集，将每个 shapefile 解析为包含每个条目几何图形和属性的行。输出数据集将包含一个几何列，以及用户列出的每个属性对应的列。可针对非 WGS:84 数据集指定坐标参考系统。

此外，还有其他表达式可用于在上述两种类型之间进行转换，以及将它们转换为 H3 索引、MGRS、边界框和本体 GeoPoint 格式。

在有关坐标参考系与投影的文档中了解有关地理空间数据格式的更多信息。

转换地理空间数据

在 Pipeline Builder 中填充地理空间类型的列之后，便可以使用专门针对地理空间数据进行操作的转换功能。除了地理连接之外，大多数转换操作目前同时支持流处理和批处理工作流。以下列出了部分核心功能。

几何比较

  • 交集
  • 差集
  • 对称差集
  • 并集（按列合并与聚合）

球面几何

  • 两点之间的半正矢/大圆距离
  • 逆向半正矢距离（给定起点、距离和方位角，计算终点）
  • 几何图形的面积/质心/长度

H3

  • 获取特定分辨率下 H3 六边形的相邻六边形
  • 使用特定分辨率的 H3 六边形覆盖多边形

复杂形状近似

  • 椭圆/圆形
  • 扇形区域（环形扇区）
  • 给定几何图形的凸包

地理空间连接

Pipeline Builder 支持以下地理空间连接：

  • 几何交集连接
  • 几何距离连接
  • 几何最近邻连接

几何交集连接

Pipeline Builder 的几何交集连接需要两个数据集，且每个数据集都必须包含一个几何类型的列。几何交集连接不接受 Ontology GeoPoint 或 GeoPoint 作为输入类型。在应用连接之前，建议对几何列进行规范化处理，并在输出结果不需要 null 值时将其显式过滤掉。如果管道中存在非确定性操作或其他连接，建议在地理连接之前添加检查点。

假设输出行数增加两倍，Pipeline Builder 能够连接中型几何图形（最多约 34 个点）的数据集，且两侧的规模最多可达 100 万行。对于倾斜数据，该连接可以在一侧最多 2.5 亿行、另一侧 1600 行的情况下提供支持。随着几何图形尺寸的增加，稳定性可能会下降。该连接能够稳定支持将包含单个巨型几何图形（约 4 万个点）的数据集与最多 50 万行进行连接。更大规模的连接或许偶尔能够成功，但不在官方支持范围内。

如果几何交集连接的输出行数与交叉连接的行数相当，则可能导致连接稳定性下降。

作为几何交集连接的替代方案，配置了“几何图形存在交集”过滤器的交叉连接可能会提供更稳定的内存使用。然而，这种方法可能会导致构建时间急剧增加。

几何距离连接

Pipeline Builder 的几何距离连接需要两个数据集，且每个数据集都必须包含一个几何类型的列、一个大于零的距离值，以及一个用于确定所提供距离单位的坐标参考系字符串。例如，如果坐标参考系提供的是“epsg:4326”，则会假定距离以度为单位。与交集连接类似，建议对几何列进行规范化处理，并在输出结果不需要 null 值时将其显式过滤掉。如果管道中存在其他连接或非确定性操作，请在连接之前添加检查点。

假设连接操作导致行数增加两倍，Pipeline Builder 能够将小型几何图形（每个约包含 8 个点）的数据集进行连接，且两端数据规模最多可达 100 万行。若输出行数接近交叉连接的量级，系统稳定性可能会下降。

作为几何距离连接的替代方案，配置了几何缓冲区并带有“几何图形存在交集”过滤器的交叉连接，在行数大幅增加时能提供更稳定的内存占用。然而，在大多数情况下，这种方式可能会显著增加构建时间。

几何 K 近邻 (KNN) 连接

Pipeline Builder 的几何最近邻连接需要两个数据集：一个包含几何图形的 base 数据集和一个包含点的 neighbors 数据集。k 整数参数用于配置为每个基础几何图形查找的最近邻数量。系统需要坐标系来确定如何计算和比较基础几何图形与邻近点之间的距离。输出结果将为组合后的行集，每一行都包含一个 GeoPoint，该点是距离基础几何图形最近的 k 个点之一。平局情况将随机处理，返回结果不按特定顺序排列。

请注意，此连接操作有两个要求：

  • neighbors 数据集中的所有 GeoPoint 必须能够装入执行器和驱动程序的内存。这是目前的硬性要求，限制了连接操作的可扩展性。如果您的使用场景需要分布式处理 neighbors 数据集，请联系您的 Palantir 代表。

  • 为限制内存消耗，Foundry 目前在 neighbors 数据集中仅支持 GeoPoint 逻辑类型。如果在连接的 neighbors 端需要非点几何图形，请联系您的 Palantir 代表。

在实际应用中，Pipeline Builder 支持适中的 k 值（< 5），此时 neighbors 数据集最多可达数十万行，base 数据集可达 100 万个几何图形。当两个数据集都包含数十万行时，Pipeline Builder 能够支持更大的 k 值。在此类场景下，查找多达数百个最近邻的操作也能迅速完成。将输入规模超出此限度可能偶尔会成功，但目前通常不予支持。

故障排除

如果您的连接操作遇到稳定性问题，请按照以下步骤进行修复：

  1. 在连接前删除不必要的列。
  2. 简化输入的几何图形（例如，对于大型几何图形，是否可以使用更粗的粒度？）
  3. 垂直扩容；手动为驱动程序和执行器选择内存更大的计算配置。
  4. 将最大的输入数据集拆分为约 2500 万行的子集，然后在单独的构建中将结果合并。
  5. 减少输出中的行数（即左右几何图形之间的交集数量）。

预览转换结果

在 Pipeline Builder 中完成数据转换后，您可以在地图上直观地验证这些转换的结果。在常规预览窗格中，选择您希望在地图上预览的单元格（这些单元格必须属于上述某种地理空间类型的列）。右键单击并选择 打开地理预览。

此时将出现一个新的预览标签页，显示所选单元格在地图上的绘制情况。

将地理空间数据与本体结合使用

Pipeline Builder 的地理空间功能旨在与平台上的下游数据实现无缝集成。

  • 本体
    • Builder 的几何列类型与本体的地理形状类型兼容，但在 Builder 中将列映射到对象之前，请确保应用“标准化几何图形”表达式。这可以确保地理形状数据在将数据索引到本体时能够通过验证。
    • 虽然当前的 GeoPoint 逻辑类型无法直接在本体中使用，但在索引之前，可以轻松地将点转换为“本体 GeoPoint”类型（格式为“{lat},{lon}”的字符串）。
  • 数据集
    • 地理空间类型数据会持久化存储在 Builder 管线的输出数据集上，因此，如果您从该数据集创建下游 Builder 管线，您的数据仍将保留其正确的逻辑/地理空间类型。
  • 地理时间序列同步
    • 您可以将 GeoPoint 和几何列映射到地理时间序列同步输出，以便在下游应用（如地图应用）中渲染点和几何图形。所有地理时间序列同步输出都必须包含一个 GeoPoint 列，用于指示与每行对应的观测位置。有关如何在集成中配置地理空间属性的更多详细信息，请参阅地理时间序列输出文档。
