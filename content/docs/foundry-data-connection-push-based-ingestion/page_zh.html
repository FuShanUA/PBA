将数据推送到流中

Foundry 中的大部分数据接入工作，主要是从源系统抽取数据，并将其同步至数据集或流中。

借助流，Foundry 支持基于推送的接入方式，以满足事件驱动的业务流需求。

Foundry 中基于推送的记录接入方式，与常规 REST 服务遵循相同的原则。我们提供了一系列 REST 端点，暴露出基于推送的 API，用于接收记录并将其写入流和数据集。向流中推送数据需要以下基本信息：

  • 流的数据集资源标识符。
  • 分支名称。
  • 用于对请求进行身份验证的令牌。

如果已经为数据配置了 Foundry 源，建议直接连接该源。了解如何使用现有源设置流同步，或了解如何设置 Kafka 源。

如果需要从无法通过 Foundry 正确进行身份验证或不符合标准流模式的系统接收入站 Webhook，请考虑使用监听器。

接下来，我们将探讨将数据推送到流中所需的步骤：

  1. 设置新流。
  2. 将记录推送至流中。
  3. 共享该流。
  4. 测试该流。

第一部分：初始设置

首先，启动创建流的工作流。

  1. 登录 Foundry。
  2. 导航至项目。
  3. 在右上角选择 + 新建。
  4. 向下滚动并选择 流。

接下来，必须定义流的模式、吞吐量和键。

  • 模式定义了流数据的结构和类型。
  • 吞吐量代表数据处理速率，会影响流中使用的分区数量。详情请参阅关于吞吐量和分区的文档。
  • 键用于在使用多个分区时保证唯一 ID 的顺序。详情请参阅关于流键的文档。

在本教程中，我们将创建一个简单的单分区流。

  • 将模式设置为 sensor_id: String、temperature: Double 和 created_at: Timestamp。
  • 将吞吐量设置为 Normal。

然后，对流进行配置。

  • 选择 从 JSON 样本生成... 按钮并粘贴现有的 JSON 文本，即可从现有的 JSON 数据块中自动生成模式。
  • 如果要从关系型数据库流式传输实时更新，请设置变更数据捕获。
  • 根据流的并行处理需求，评估是否需要更高的吞吐量设置。有关更多信息，请查阅关于分区的文档。
  • 在使用多个分区时，考虑设置键以保证顺序。

在选择创建流之前，必须解决所有验证错误。将鼠标悬停在页面底部的工具提示上，可查看有关错误的更多详细信息。

选择右下角的 创建流 按钮，进入连接页面。在此页面，可以指定连接流数据的方式。

第二部分：将记录推送到流中

现在可以开始连接流了。

  1. 对于基于推送的接入，请在通过 API 连接部分下方选择任一选项。

    任何能够发起 HTTP 请求的语言或技术均可用于推送记录。我们提供了 cURL、Python、Node 环境下的 JavaScript 以及 Java 的示例。如果想通过 Foundry 同步接入数据，而不是将数据推送到 Foundry，请查阅如何设置流同步。

    本示例中，我们将选择 cURL。

    随后将进入推送数据工作流页面。

  2. 接下来，选择身份验证机制。我们支持两种请求身份验证方式：

    • 使用第三方应用程序推送（推荐）：此方法使用 OAuth2 工作流创建安全令牌，用于将记录推送到流中。
    • 使用个人令牌推送：此方法使用用户生成的令牌，仅供测试使用。

  3. 在本教程中，请选择使用第三方应用程序推送方法。按照屏幕上的步骤设置第三方应用程序，并创建您的客户端密钥。配置应用程序的操作限制时，请确保包含 api:use-streams-write 操作，以便第三方应用程序能够将记录推送到流中。

  4. 现在，选择前往第三方应用程序，在您的 Foundry 平台设置中打开第三方应用程序管理页面。

  5. 在屏幕右上角选择注册新应用程序。

  6. 选择一个名称，并将客户端类型设置为服务端应用程序。

  7. 选择创建，系统将显示您的客户端 ID 和密钥。

    离开此页面后，客户端密钥将无法再次访问。请务必将其保存在安全的位置。

  8. 现在，您可以将客户端 ID 和密钥添加到推送工作流页面中。

  9. 滚动至工作流的配置应用程序部分，然后选择管理应用程序，在您的 Foundry 平台设置中打开第三方应用程序管理页面。

  10. 接下来，启用客户端凭证授权设置。

  11. 最后，点击右上角的保存。

第 3 部分：共享流

现在，我们需要将流与我们创建的应用程序共享。

  1. 首先，返回至推送工作流页面。在工作流的使用您的新应用程序部分下，找到您生成的客户端 ID。

  2. 接下来，选择右上角的共享，在屏幕右侧打开流的详情侧边栏的角色选项卡。

  3. 将客户端 ID 复制并粘贴到角色搜索框中，以查找您创建的应用程序。选择+进行搜索，并选择 Editor 角色。

  4. 在侧面板底部选择保存以共享该流。

  5. 返回至推送工作流，并选择下一步。

后续步骤
