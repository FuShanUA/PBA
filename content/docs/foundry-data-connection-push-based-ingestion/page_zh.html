将数据推送到流中

Foundry 中大多数数据摄入工作都集中在从源系统拉取数据并将其同步到数据集或流中。

通过流，Foundry 支持 基于推送的摄入，以支持事件驱动的工作流程。

Foundry 中基于推送的记录摄入遵循与典型 REST 服务相同的原理。通过一系列 REST 端点，我们提供一个可以消耗记录并将其写入流和数据集的基于推送的 API。要向流中推送数据，需要以下高级信息：

  • 流的数据集资源标识符。
  • 分支名称。
  • 用于认证请求的令牌。

如果您已经为您的数据配置了 Foundry 源，您可能希望连接到该源。了解如何 设置流式同步

使用现有源，或学习如何 设置 Kafka 源。

如果您需要从无法正确与 Foundry 身份验证或符合标准流模式的系统接收入站 webhook，请考虑使用 监听器。

下面，我们将讨论将数据推送到流中的步骤：

  1. 设置一个新的流。
  2. 将记录推送到流中。
  3. 共享流。
  4. 测试流。

第一部分：初始设置

首先，开始流创建工作流程。

  1. 登录 Foundry。
  2. 导航到一个 项目（Project）。
  3. 在右上角选择 + 新建。
  4. 向下滚动并选择 流（Stream）。

接下来，我们必须为流定义模式、吞吐量和键。

  • 模式 定义了你的流数据的结构和类型。
  • 吞吐量 表示数据处理速率，并将影响你的流中使用的分区数量。了解更多请参阅我们的 吞吐量 和 分区 文档。
  • 键 用于在使用多个分区时保证唯一ID的顺序。了解更多请参阅我们的 流键 文档。

在本教程中，我们将创建一个简单的单分区流。

  • 将模式设置为 sensor_id: String, temperature: Double, 和 created_at: Timestamp。
  • 将吞吐量设置为 Normal。

然后，配置你的流。

  • 通过选择 Generate from JSON sample... 按钮并粘贴你已有的 JSON blob 来自动从现有的 JSON blob 生成模式。
  • 如果你正在从关系型数据库实时流式传输更新，请设置 变更数据捕获。
  • 考虑你的流的并行性需求，以确定是否需要更高的吞吐量设置。有关更多信息，请查看我们的 分区 文档。
  • 在使用多个分区时，建议为排序保证设置一个键。

在选择创建流之前，必须解决所有验证错误。有关错误的更多详细信息，请将鼠标悬停在页面底部的工具提示上。

在右下角选择创建流按钮以导航到连接页面。在此处，您可以指定如何连接到流数据。

第二部分：将记录推送到流中

我们现在准备好连接我们的流了。

  1. 对于基于推送的数据摄入，选择通过API连接部分下的其中一个选项。

    任何能够发起HTTP请求的语言或技术都可以用来推送记录。我们提供了cURL、Python、通过Node的JavaScript和Java的示例。如果您希望通过Foundry同步而非直接将数据推送到Foundry来摄取数据，请查看如何< a href="/docs/foundry/data-connection/set-up-streaming-sync/">设置流式同步。

    对于这个示例，我们将选择cURL。

    您将进入推送数据工作流程页面。

  2. 接下来，选择一种认证机制。我们支持两种向请求进行认证的方式：

    • 使用第三方应用程序进行推送（推荐）：此方法使用OAuth2流程生成一个安全令牌，可用于将记录推送到您的流中。
    • 使用个人令牌进行推送：此方法仅用于测试目的，使用用户生成的令牌。

  3. 在本教程中，选择使用第三方应用程序进行推送 方法。按照屏幕上的步骤设置第三方应用程序并创建您的 客户端密钥。在配置应用程序的操作限制时，确保包含 api:use-streams-write 操作，以便第三方应用程序可以将记录推送到流中。

  4. 现在，选择转到第三方应用程序，以在您的Foundry平台设置中打开第三方应用程序管理页面。

  5. 在屏幕的右上角选择注册新应用。

  6. 选择一个名称，并将客户端类型设置为服务器应用。

  7. 选择创建，您将看到您的客户端ID和密钥。

    客户端密钥一旦离开此页面将无法访问。请确保将其存储在安全的位置。

  8. 现在，您可以将客户端ID和密钥添加到推送工作流页面。

  9. 向下滚动到工作流的配置应用程序部分，然后选择管理应用程序以在您的Foundry平台设置中打开第三方应用程序管理页面。

  10. 接下来，启用客户端凭证授予设置。

  11. 最后，点击右上角的保存。

第三部分：共享流

现在，我们需要将流与我们创建的应用程序进行共享。

  1. 首先，返回到推送工作流页面。在工作流的使用您的新应用程序部分下，找到您生成的客户端ID。

  2. 接下来，在右上角选择共享，以打开右侧屏幕上的角色选项卡的详情侧边栏。

  3. 将客户端ID复制并粘贴到角色搜索字段中，以查找您创建的应用程序。选择+来搜索并选择Editor角色。

  4. 在侧边栏底部选择保存以共享该流。

  5. 返回到推送工作流并选择下一步。
