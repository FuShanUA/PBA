提交支持工单

如果您在查看 可用资源 后仍未找到合适的解决方案，那么现在是时候提交一个支持工单了。

提交工单可以确保您的问题或请求被跟踪、适当优先处理，并在必要时进行升级。

为了使支持流程更高效，请按照以下步骤收集有价值的信息以提交支持工单：

  • 1. 创建示例
  • 2. 分享相关资产
  • 3. 报告问题
前提条件

请确保您已经使用 获取帮助页面 上提供的步骤诊断了您的问题，查阅了 使用 Chrome™ DevTools 调试 和 HTTP 错误代码 指南后，再继续按照以下说明操作。

1. 创建示例

当您遇到问题并提出问题时，提供清晰的步骤以便我们能够重现该问题，将确保快速解决。这些步骤应包括：

  • 最小化：尽可能少的步骤，同时仍然能产生问题。
  • 完整：提供重现问题所需的所有部分。
  • 可验证：测试您即将提供的流程，以确保它能重现问题。

一个最小、完整且可验证的示例有助于我们在您的环境之外重现问题，反过来也使我们更容易找到您问题的根本原因，并更快地解决问题。

在 Palantir 的上下文中，这样的示例可以是一系列重现行为的步骤，或者是一个最小的资源来重现问题。通常，将预期的行为与观察到的行为进行比较作为示例的一部分是有用的。

提示

以下是一些创建最小、完整和可验证示例的提示：

  • 首先，确定重现此问题所需的步骤。这些步骤越具体、越详细，其他人就越容易重现该问题。
  • 确保你的示例是简洁的，即只包含重现问题所需的最基本信息。这将有助于缩小问题的潜在原因，并使解决问题变得更加容易。
  • 包括你提供的步骤的预期结果以及实际结果。这将帮助其他人理解问题并确定他们是否遇到相同的问题。
  • 确保你的示例是完整的，即包含重现问题所需的所有信息。如果你遗漏了任何重要细节，其他人将很难重现该问题并帮助你解决问题。
  • 最后，确保你的示例是可验证的，即其他人可以使用你提供的信息轻松地重现该问题。如果你的示例不可验证，其他人将很难帮助你解决问题。

示例

以下是一个历史示例，展示了已解决的最小、完整且可验证的问题。

当时重现问题的步骤：

  1. 打开代码仓库应用程序。
  2. 使用侧边栏添加 Graphframes 库。
  3. 对仓库运行检查。

预期行为：库应无错误地添加到代码仓库中。

实际出现的问题行为：显示错误消息，内容为 "o257.loadClass.: java.lang.ClassNotFoundException:"

附加信息：

  • 我资料中的其他包包括 Python 3.12.* 等。
  • 我已经尝试过 x、y 和 z。
  • 除了添加这个包之外，此代码仓库没有其他最近的更改。
  • 这似乎影响了该包的用户。

此示例仅包含重现该问题所需的最基本步骤，不需要任何专有数据。由于该示例包括所有必要的信息，例如预期行为和实际行为，以及已经进行的故障排除等额外上下文，因此被认为是完整的。它是可验证的，因为可以使用相同的步骤重现错误消息。通过提供这些信息，支持团队能够快速重现问题并着手解决。

如果您对该问题提供的修复方法感兴趣，请参阅 需要同时具有 Conda 包和 jar 的包。

3. 报告问题

在指定的应用程序中报告问题时，请导航至 支持 > 报告问题（或 支持 > 联系支持），并 按照向导操作。

作为另一种方法，您也可以在 Palantir 开发者社区 中请求其他用户的支持。

在公共论坛（如 Palantir 开发者社区）上发布内容时，请记得在发布前删除所有敏感数据。
