SAP ERP

The SAP ERP connector allows you to connect Foundry to SAP's on-premise ERP Central Component (ECC) and S/4 HANA (on-premise and Cloud Private Edition ↗). The SAP ERP connector enables Foundry to interact with various types of SAP data, including:

  • SAP Application Tables
  • SAP CDS Views
  • SAP HANA Information Views
  • SAP BW Content Extractors (remote connections only)
  • SAP Transaction Codes and ABAP Reports
  • SAP Function Modules
  • SAP Media (DMS documents)

Using the SAP ERP source requires the installation of the Palantir Foundry Connector 2.0 for SAP Applications add-on on the target SAP application layer.

Setup

  1. Open the Data Connection application and select + New Source in the upper right corner of the screen.
  2. Select SAP ERP from the available connector types.
  3. Follow the additional configuration prompts to continue the setup of your connector using the information in the sections below.

Learn more about setting up a connector in Foundry.

Connection type

The SAP ERP connector supports two connection types:

Connection typeDescription
DirectConnects directly to the SAP system. This is the default.
Remote (via Gateway)Connects to a remote SAP system through a gateway. Enable the Connect via Gateway toggle and provide a Context value to identify the remote system.

The connection type determines which SAP object types are available. Learn more about remote connections and remote agent configuration.

Authentication

The SAP ERP connector supports the following authentication methods:

Authentication methodDescription
Basic AuthProvide the username and password of the technical user created when installing the Connector.
Authentication tokenProvide a token to authenticate.
Custom authentication headerProvide a custom authentication header.
No authenticationUse this option if authentication is set up on the agent machine via certificates.

Networking and connectivity

Make sure to properly configure egress policies to allow Foundry to reach the SAP system. For on-premises SAP environments, agent proxy policies are typically required to route traffic correctly.

Many SAP systems use custom-signed certificates, which can cause SSL handshake exceptions when configuring the connection for the first time. Make sure you have the correct custom certificates from your system and add them to the source.

SAP Data Accelerator

To reach this SAP system through SAP Data Accelerator (SAP's managed connectivity service for systems exposed through an SAP Cloud Connector), set the source's host and port to your Cloud Connector virtual host and port. Then, assign the client certificate provided by Palantir, allow egress to the SAP Data Accelerator control plane and data plane, and add the SAP Data Accelerator property with your control plane URL. This path requires using a Foundry worker.

Create a sync

  1. 在源概览页面，选择批量同步旁边的 + 新建。
  2. 配置标准设置：名称、目标数据集和调度计划。
  3. 将 Transaction type 设置为以下选项之一：

  4. 从下拉菜单中选择 SAP object type（参见下文的 SAP 对象类型）。
  5. 输入 Object name。在输入过程中，该字段会根据所选的对象类型自动提示匹配的对象。
  6. （可选）在 Extras 标签页中配置附加参数。

增量同步

增量同步支持从同一表中进行追加式事务。要启用增量同步，请将 Transaction type 设置为 APPEND。

可用的增量模式取决于对象类型：

增量模式ERP 表CDS 视图HANA 视图BW 内容抽取器
多字段✓✓✓
拼接字段✓✓✓
变更凭证表✓
孪生表✓
SAP 内置增量✓
  • 多字段：导入指定字段中任意一个大于或等于已导入最大值的行。字段间使用逗号分隔。
  • 拼接字段：与多字段类似，但会将字段值拼接在一起，而不是通过 OR 逻辑组合。
  • 变更凭证表：根据 SAP 变更凭证表中的更新情况导入数据行。提供两种子模式：仅 CDPOS 或 CDHDR 和 CDPOS。
  • 孪生表：当独立的“孪生”表中的某个字段满足增量条件时，导入目标表中的数据行。
  • SAP 内置增量：使用抽取器原生的变更数据捕获机制。

增量字段最好为单调递增的值。系统采用“大于等于”比较方式，以避免在同步运行至某一日期中途时发生数据遗漏。Foundry 数据集中可能会出现重复值，应在转换管道的第一步予以清除。

事务码和函数模块对象类型不支持增量同步；它们始终以快照方式运行。

对于表类型的增量同步，每次同步最大行数 设置项用于限制每次运行返回的大致行数。如果间歇性问题（如网络故障）干扰了长时间运行的同步，可利用此设置将大型表的初始同步拆分为一系列更小、更具弹性的运行任务。

孪生表配置

当使用 孪生表 作为增量模式时，增量孪生表 设置项用于指定孪生表名称，而 增量孪生映射 设置项用于定义主表与孪生表之间的连接条件。映射条目采用以下格式：

{PRIMARY_TABLE_NAME}-{FIELD_NAME}={TWIN_TABLE_NAME}-{FIELD_NAME}

多个连接条件之间使用分号连接（即 AND 运算符）。

重置增量状态

如需强制进行全量重新加载并重新初始化增量摄取，请启用同步中的 重置增量状态 开关。系统将从 SAP 执行数据的全量快照，并替换数据集中的所有文件。同步完成后，请关闭该开关，以便后续同步恢复增量追加模式。

ERP 表、CDS 视图、HANA 视图和 BW 内容抽取器对象类型支持重置增量状态。

媒体集

SAP ERP 连接器支持 媒体集，用于摄取存储在 SAP 文档管理系统 (DMS) 中的媒体文件和文档。

要创建新的媒体摄取任务：

  1. 导航至源概述页面。
  2. 在 媒体集同步 旁选择 + 新建。
  3. 定义您的媒体集格式。提供特定格式后，您将能够在下游数据转换中使用特定于该格式的转换步骤。阅读有关 媒体集格式 的更多信息。
  4. 使用源探索工具（按文档类型组织文件）选择您要摄取的文件，或者直接在 过滤器 部分定义您自己的过滤器。

探索

当您选择 子文件夹或媒体文件 时，可以选择特定的文档类型（如 CPD）以摄取所有关联的文档，也可以选择单个文档以仅摄取该特定文件。

Webhook

SAP webhook 允许您通过从 Foundry 调用 BAPI（业务 API）将数据写回 SAP。请参阅 Webhook 以了解如何设置 webhook 的概述。

SAP webhook 支持在 Foundry 管理的云计算中使用 Foundry worker 连接。我们建议为 SAP webhook 使用此运行时，而不是通过 数据连接智能体 来执行它们。在 Foundry worker 上配置的现有 SAP 源会自动受益于云计算，无需迁移；在数据连接智能体上运行的 SAP 源应 切换为 Foundry worker，以利用云端执行的 webhook。

SAP webhook 唯一可用的任务类型是 sap-run-function-webhook-task-v0。以下示例调用 BAPI_SALESORDER_CHANGE 来修改给定销售文档的采购日期：

示例：通过 BAPI 回写至 SAP（转换）


以下《外部转换》会在输入数据集的每一行调用一次 BAPI_SALESORDER_CHANGE，以更新每个销售凭证上的购买日期，并将 SAP 的 RETURN 表记录到输出数据集中以供审查。




Copied!
1from transforms.api import Input, Output, transform
2from transforms.external.systems import external_systems, Source, ResolvedSource
3
4
5@external_systems(
6    sap_source=Source("<source_rid>"),
7)
8@transform(
9    sales_order_updates=Input("<input_dataset_rid>"),  # columns: sales_doc_id, purchase_date (YYYYMMDD)
10    writeback_results=Output("<output_dataset_rid>"),
11)
12def update_purchase_dates(ctx, sap_source: ResolvedSource, sales_order_updates, writeback_results):
13    base_url = sap_source.get_https_connection().url
14    client = sap_source.get_https_connection().get_client()
15
16    results = []
17    for row in sales_order_updates.dataframe().collect():
18        response = client.post(
19            f"{base_url}/v2/function/BAPI_SALESORDER_CHANGE/writeback",
20            json={
21                "commit": True,
22                "input": {
23                    "SALESDOCUMENT": row.sales_doc_id,
24                    "ORDER_HEADER_IN": {"PURCH_DATE": row.purchase_date},
25                    "ORDER_HEADER_INX": {"UPDATEFLAG": "U", "PURCH_DATE": "X"},
26                },
27                "output": ["RETURN"],
28            },
29        )
30        response.raise_for_status()
31        results.append({
32            "sales_doc_id": row.sales_doc_id,
33            "purchase_date": row.purchase_date,
34            "status_code": response.status_code,
35            "sap_return": str(response.json().get("RETURN", [])),
36        })
37
38    writeback_results.write_dataframe(ctx.spark_session.createDataFrame(results))



对于需要 commit 语句才能持久化更改的标准 BAPI，请设置 "commit": true。在调用函数之前，请使用 metadata 端点（GET /v2/function/{functionName}/metadata）来探查该函数预期的输入与输出结构。


示例：通过 BAPI 向 SAP 写回数据（函数）


若要从 Workshop 操作、Ontology 操作后端或任何其他 Foundry 函数调用方同步调用 BAPI，请将此写回操作暴露为带源的 Python 函数。该函数接收带类型的输入，并将 SAP 的 RETURN 结构返回给调用方。




Copied!
1from functions.api import function
2from functions.sources import get_source
3
4
5@function(sources=["sap_source"])
6def update_sales_order_purchase_date(sales_doc_id: str, purchase_date: str) -> str:
7    """Updates the purchase date on a sales order via BAPI_SALESORDER_CHANGE.
8
9    Args:
10        sales_doc_id: SAP sales document number.
11        purchase_date: New purchase date in YYYYMMDD format.
12
13    Returns:
14        A string representation of the SAP RETURN table, useful for surfacing
15        any error or warning messages back to the caller.
16    """
17    source = get_source("sap_source")
18    base_url = source.get_https_connection().url
19    client = source.get_https_connection().get_client()
20
21    response = client.post(
22        f"{base_url}/v2/function/BAPI_SALESORDER_CHANGE/writeback",
23        json={
24            "commit": True,
25            "input": {
26                "SALESDOCUMENT": sales_doc_id,
27                "ORDER_HEADER_IN": {"PURCH_DATE": purchase_date},
28                "ORDER_HEADER_INX": {"UPDATEFLAG": "U", "PURCH_DATE": "X"},
29            },
30            "output": ["RETURN"],
31        },
32        timeout=30,
33    )
34    response.raise_for_status()
35    return str(response.json().get("RETURN", []))



该源必须启用代码导入和导出，函数才能使用它。将 SAP 中严重级别非零的 RETURN 消息作为面向用户的错误呈现，以便调用方能够对其进行响应。


示例：并行回写（转换）

9    sales_order_updates=Input("<input_dataset_rid>"),
10    writeback_results=Output("<output_dataset_rid>"),
11)
12def update_purchase_dates_parallel(
13    ctx, sap_source: ResolvedSource, sales_order_updates, writeback_results
14):
15    base_url = sap_source.get_https_connection().url
16    client = sap_source.get_https_connection().get_client()
17
18    bulkinput = [
19        {
20            "SALESDOCUMENT": row.sales_doc_id,
21            "ORDER_HEADER_IN": {"PURCH_DATE": row.purchase_date},
22            "ORDER_HEADER_INX": {"UPDATEFLAG": "U", "PURCH_DATE": "X"},
23        }
24        for row in sales_order_updates.dataframe().collect()
25    ]
26
27    response = client.post(
28        f"{base_url}/v2/function/BAPI_SALESORDER_CHANGE/parallel_writeback",
29        json={
30            "commit": True,
31            "output": ["RETURN"],
32            "bulkinput": bulkinput,
33            "parallel": True,
34            "paralleljobs": 5,
35            "functioncallperjob": 10,
36        },
37    )
38    response.raise_for_status()
39
40    results = [
41        {"index": i, "result": str(r)} for i, r in enumerate(response.json())
42    ]
43    writeback_results.write_dataframe(ctx.spark_session.createDataFrame(results))



根据工作负载和 SAP 系统的可用容量，调整 paralleljobs（在 SAP 中生成的后台作业）和 functioncallperjob（每个作业处理的调用数）。将 "parallel": false 设置为回退到顺序执行，同时仍使用单个批量请求。


示例：获取 DMS 文档至媒体集（转换）


以下外部转换结合了 DMS 列表和下载端点，用于从 SAP 的文档管理系统获取文档，并将其写入媒体集。当内置的媒体集同步无法提供足够的控制能力时，可以使用此模式，例如，当文档选择依赖于 Foundry 端的数据，或者在下载前需要组合多个过滤器时。



import io
from transforms.api import transform
from transforms.external.systems import external_systems, Source, ResolvedSource
from transforms.mediasets import MediaSetOutput


@external_systems(
    sap_source=Source(""),
)
@transform(
    documents=MediaSetOutput(""),
)
def fetch_dms_documents(ctx, sap_source: ResolvedSource, documents):
    base_url = sap_source.get_https_connection().url
    client = sap_source.get_https_connection().get_client()

    # 列出 DMS 中的文档，按文档类型和文件格式进行过滤。
    list_response = client.get(
        f"{base_url}/v2/get_documents",
        params={"DOCUMENTTYPE": "CPT", "FILEFORMAT": "pdf"},
    )
    list_response.raise_for_status()

    for doc in list_response.json():
        document_id = doc["DOCUMENT_ID"]
        document_name = doc["DOCUMENT_NAME"]

        download_response = client.get(
            f"{base_url}/v2/get_document",
            params={"DOCUMENT_ID": document_id},
        )
        download_response.raise_for_status()

        documents.put_media_item(io.BytesIO(download_response.content), document_name)


例如，为了驱动从 Foundry 输入数据集的数据接入，仅获取由一组 EBELN（采购订单）值引用的文档，可以使用 OBJECTLINK 过滤器调用 /v2/get_documents，或者直接将输入行中的每个 DOCUMENT_ID 传递给 /v2/get_document。
