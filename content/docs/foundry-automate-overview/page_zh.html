自动化

自动化是一款完全向后兼容的产品，它取代了对象监控，成为平台内所有业务自动化的唯一入口。

自动化是一款用于业务自动化的应用程序。通过自动化，你可以定义持续或按计划检查的条件，并设定在满足指定条件时自动执行的操作。

条件可以是基于时间的条件（“每周一上午 9 点触发”），也可以是基于 Foundry 本体构建的对象数据条件（“当添加优先级为 高 的新 告警 对象时触发”），或者是基于时间和对象数据条件的组合。

可用的操作：

  • 提交 Foundry 操作
  • 触发 AIP Logic 函数
  • 执行 Foundry 函数
  • 发送带有附件的平台通知和电子邮件通知

用例

自动化可用于各种不同的自动化工作流，包括：

  • 定时报告发送与摘要：指定时间并向预定义的收件人列表发送每周报告。从 Notepad 或 Notepad 模板生成的 PDF 可以自动附加到电子邮件中。
  • 数据告警：定义并监控对象集，当满足特定数据条件时向用户发出警报；例如，当未决问题对象的总数超过阈值时。
  • 工作流自动化：自动化可用于对满足指定条件的对象数据自动执行操作。可以自动执行的任务包括：
    • 检查数据异常，并自动将这些对象传递给带有修复逻辑的操作。
    • 监控建议或潜在的操作，并在满足预配置的事件和时间条件时自动应用。此类操作可能包括通过 Webhooks 调用外部系统的 API，从而直接在外部系统中应用变更。
  • 监控搜索：配置自动化功能，以便在保存的对象探索有新结果或搜索的所有结果满足聚合条件时发出通知；例如，所有传感器对象中的最高温度超过阈值。

访问自动化

要访问自动化，请在 Foundry 导航侧边栏中选择自动化应用程序图标。请按照我们文档中关于自动化入门的步骤开始操作。

自动化专为业务自动化而设计。如果你需要针对数据连接和流水线构建的健康监控，请参阅健康检查文档。
