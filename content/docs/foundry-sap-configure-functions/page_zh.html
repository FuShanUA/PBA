SAP ERP

The SAP ERP connector allows you to connect Foundry to SAP's on-premise ERP Central Component (ECC) and S/4 HANA (on-premise and Cloud Private Edition ↗). The SAP ERP connector enables Foundry to interact with various types of SAP data, including:

  • SAP Application Tables
  • SAP CDS Views
  • SAP HANA Information Views
  • SAP BW Content Extractors (remote connections only)
  • SAP Transaction Codes and ABAP Reports
  • SAP Function Modules
  • SAP Media (DMS documents)

Using the SAP ERP source requires the installation of the Palantir Foundry Connector 2.0 for SAP Applications add-on on the target SAP application layer.

Setup

  1. Open the Data Connection application and select + New Source in the upper right corner of the screen.
  2. Select SAP ERP from the available connector types.
  3. Follow the additional configuration prompts to continue the setup of your connector using the information in the sections below.

Learn more about setting up a connector in Foundry.

Connection type

The SAP ERP connector supports two connection types:

Connection typeDescription
DirectConnects directly to the SAP system. This is the default.
Remote (via Gateway)Connects to a remote SAP system through a gateway. Enable the Connect via Gateway toggle and provide a Context value to identify the remote system.

The connection type determines which SAP object types are available. Learn more about remote connections and remote agent configuration.

Authentication

The SAP ERP connector supports the following authentication methods:

Authentication methodDescription
Basic AuthProvide the username and password of the technical user created when installing the Connector.
Authentication tokenProvide a token to authenticate.
Custom authentication headerProvide a custom authentication header.
No authenticationUse this option if authentication is set up on the agent machine via certificates.

Networking and connectivity

Make sure to properly configure egress policies to allow Foundry to reach the SAP system. For on-premises SAP environments, agent proxy policies are typically required to route traffic correctly.

Many SAP systems use custom-signed certificates, which can cause SSL handshake exceptions when configuring the connection for the first time. Make sure you have the correct custom certificates from your system and add them to the source.

SAP Data Accelerator

To reach this SAP system through SAP Data Accelerator (SAP's managed connectivity service for systems exposed through an SAP Cloud Connector), set the source's host and port to your Cloud Connector virtual host and port. Then, assign the client certificate provided by Palantir, allow egress to the SAP Data Accelerator control plane and data plane, and add the SAP Data Accelerator property with your control plane URL. This path requires using a Foundry worker.

增量同步

增量同步支持从同一表中进行追加式事务。要启用增量同步，请将事务类型设置为APPEND。

可用的增量模式取决于对象类型：

增量模式ERP表CDS视图HANA视图BW内容提取器
多字段✓✓✓
拼接字段✓✓✓
更改凭证表✓
孪生表✓
SAP内置增量✓
  • 多字段：导入指定字段中任意一个字段值大于或等于已导入最大值的数据行。字段之间用逗号分隔。
  • 拼接字段：与多字段类似，但会将字段值拼接在一起，而不是通过逻辑“或”进行组合。
  • 更改凭证表：根据SAP更改凭证表中的更新情况导入数据行。提供两种子模式：仅CDPOS或CDHDR和CDPOS。
  • 孪生表：当单独的“孪生”表中的某个字段满足增量条件时，从目标表中导入数据行。
  • SAP内置增量：使用提取器原生的变更数据捕获机制。

理想情况下，增量字段应为单调递增的值。系统采用“大于或等于”比较方式，以防同步在某个日期中途执行时出现数据遗漏。Foundry数据集中可能会出现重复值，应在转换管道的第一步将其移除。

事务代码和函数模块对象类型不支持增量同步；它们始终以快照方式运行。

对于表类型的增量同步，每次同步最大行数设置可限制每次运行返回的数据行数近似值。当间歇性问题（如网络故障）中断长时间运行的同步时，可利用此设置将大型表的初始同步拆分为一系列更小、更具弹性的运行任务。

孪生表配置

使用孪生表作为增量模式时，增量孪生表设置用于指定孪生表名称，增量孪生映射设置用于定义主表与孪生表之间的连接条件。映射条目采用以下格式：

{主表名称}-{字段名称}={孪生表名称}-{字段名称}

多个连接条件之间用分号连接（逻辑“与”操作）。

重置增量状态

要强制执行全量重新加载并重新初始化增量接入，请启用同步中的重置增量状态开关。此操作会从SAP执行数据全量快照，替换数据集中的所有文件。同步完成后，请关闭该开关，以便后续同步恢复增量追加。

ERP表、CDS视图、HANA视图和BW内容提取器对象类型支持重置增量状态。

媒体集

SAP ERP 连接器支持 媒体集，用于提取存储在 SAP 文档管理系统 (DMS) 中的媒体文件和文档。

要创建新的媒体提取任务：

  1. 导航至源概述页面。
  2. 在 媒体集同步 旁边选择 + 新建。
  3. 定义你的媒体集格式。提供特定的格式将使你能够在下游数据转换中使用特定于该格式的转换步骤。阅读更多关于 媒体集格式 的信息。
  4. 使用源探索工具（按文档类型组织文件）选择你要提取的文件，或者直接在 过滤器 部分定义你自己的过滤器。

探索

当你选择 子文件夹或媒体文件 时，可以选择特定的文档类型（如 CPD）来提取所有关联的文档，也可以选择单个文档以仅提取该特定文件。

Webhooks

SAP webhooks 允许你通过从 Foundry 调用 BAPI（业务 API）将数据写回 SAP。有关如何设置 webhook 的概述，请参阅 Webhooks。

SAP webhooks 支持在 Foundry 托管的云计算中建立 Foundry worker 连接。对于 SAP webhooks，我们推荐使用此运行时，而不是通过 数据连接智能体 来执行它们。在 Foundry worker 上配置的现有 SAP 源会自动受益于云端执行，无需迁移；而在数据连接智能体上运行的 SAP 源应切换至 Foundry worker，以便利用云端执行的 webhooks。

SAP webhooks 可用的唯一任务类型是 sap-run-function-webhook-task-v0。以下示例调用 BAPI_SALESORDER_CHANGE 来修改给定销售文档的购买日期：

在代码仓库中使用

可以将 SAP ERP 数据源导入到 Python 的《外部转换》或《函数》中，让代码直接访问 SAP 插件的 REST API。当批量同步、媒体集同步或 Webhook 无法提供足够的控制力时，即可采用此方法。例如，若要从 Workshop 操作中同步调用 BAPI、循环遍历大型输入数据集并逐行调用 BAPI、基于 Foundry 端的数据获取特定的 DMS 文档，或者在单次构建中链式调用多个 SAP 接口。

该插件暴露的端点定义在 foundry-sap-connector ↩ 仓库中。以下示例使用了以下端点，均相对于源 URL（https://<host>:<port>/sap/palantir）：

端点用途
POST /v2/function/{functionName}/writeback同步调用单个功能模块（BAPI）。
POST /v2/function/{functionName}/parallel_writeback通过服务端并行化，同步调用带有批量输入的功能模块。
GET /v2/get_documents列出 DMS 文档，可选择按文档类型、文件格式、文件名或对象链接进行过滤。
GET /v2/get_document根据 ID 下载单个 DMS 文档。

在使用这些示例之前，请将 SAP ERP 数据源导入到您的仓库中：

  • 对于转换操作，请遵循《设置外部转换》并使用 external_systems 装饰器。
  • 对于函数操作，请遵循《从函数发起 API 调用》并在 @function(sources=[...]) 装饰器中声明该数据源。

示例：通过 BAPI 向 SAP 写回数据（转换）

以下《外部转换》会对输入数据集的每一行调用一次 BAPI_SALESORDER_CHANGE，以更新每个销售凭证上的购买日期，并将 SAP 的 RETURN 表记录到输出数据集中以供审查。

1from transforms.api import Input, Output, transform
2from transforms.external.systems import external_systems, Source, ResolvedSource
3
4
5@external_systems(
6    sap_source=Source("<source_rid>"),
7)
8@transform(
9    sales_order_updates=Input("<input_dataset_rid>"),  # columns: sales_doc_id, purchase_date (YYYYMMDD)
10    writeback_results=Output("<output_dataset_rid>"),
11)
12def update_purchase_dates(ctx, sap_source: ResolvedSource, sales_order_updates, writeback_results):
13    base_url = sap_source.get_https_connection().url
14    client = sap_source.get_https_connection().get_client()
15
16    results = []
17    for row in sales_order_updates.dataframe().collect():
18        response = client.post(
19            f"{base_url}/v2/function/BAPI_SALESORDER_CHANGE/writeback",
20            json={
21                "commit": True,
22                "input": {
23                    "SALESDOCUMENT": row.sales_doc_id,
24                    "ORDER_HEADER_IN": {"PURCH_DATE": row.purchase_date},
25                    "ORDER_HEADER_INX": {"UPDATEFLAG": "U", "PURCH_DATE": "X"},
26                },
27                "output": ["RETURN"],
28            },
29        )
30        response.raise_for_status()
31        results.append({
32            "sales_doc_id": row.sales_doc_id,
33            "purchase_date": row.purchase_date,
34            "status_code": response.status_code,
35            "sap_return": str(response.json().get("RETURN", [])),
36        })
37
38    writeback_results.write_dataframe(ctx.spark_session.createDataFrame(results))
对于需要 commit 语句来持久化更改的标准 BAPI，请将 `commit` 设为 `true`。在调用函数前，请使用 `元数据` 端点（`GET /v2/function/{functionName}/metadata`）来探查该函数预期的输入与输出结构。
