入门

请注意，本教程仅作为示例。由于Foundry本体论（ontology）是根据您的需求和数据定制的，您可能无法访问相同的本体论资源来完成本教程。

本教程面向新的Workshop构建者，并将介绍一个简单Workshop模块的配置。以下概念将包括创建新模块、配置对象表格（Object Table）、筛选列表（Filter List）和对象视图（Object View）等小部件，以及通过操作支持的按钮组（Action-backed Button Group widget）启用写回功能。

在这个示例场景中，我们的目标是构建一个“航班警报收件箱”，用于显示潜在的航班延误和取消信息，并向用户展示易于理解并采取行动的选项。我们的目标用户是一名需要实时跟踪这些警报的航空公司主管，需要优先处理哪些警报需要响应，快速解决警报。

第一部分：初始设置

  1. 要创建一个新模块，请从左侧工作区导航面板中打开 项目与文件，然后找到您想要的项目或文件夹。到达后，在右上角选择 新建 > 工作区模块，在当前项目或文件夹内创建一个新模块。新模块将继承其所在项目或文件夹的权限。
  2. 模块资源名称将显示在左上角；默认情况下，名称的格式为“New module (2022-04-08 15:16:23)”。我们可以选择此默认值以输入更具描述性和易于记忆的资源名称。在这种情况下，我们将模块命名为“Flight Inbox 教程”。在此更改后，我们的屏幕应如下所示：

第二部分：配置您的初始视图

让我们添加并配置一个对象表小部件用于Flight Alert对象：

  1. 在模块布局的右侧，选择添加小部件，然后从出现的小部件选择器中选择对象表小部件。
  2. 在对象表 1小部件的配置面板中，使用对象集下拉菜单来定义将在该小部件中显示的数据。在下拉菜单中，选择 + 新对象集变量，在出现的变量定义弹出窗口中，选择起始对象集下拉菜单并选择[示例数据] 航班警报。如果您找不到[示例数据] 航班警报，请联系Palantir支持部门。
  3. 返回对象表1小部件的主要配置面板，并查找列配置部分以选择添加所有属性。这将在对象表小部件中为[示例数据] 航班警报对象类型的每个属性添加一列。如果需要，您也可以使用+ 添加列按钮来配置特定的列。
  4. 在列下，调整列的顺序并删除任何不需要的属性。例如，将航班日期列拖动到列表顶部，使其在对象表中显示为对象标题后的第一列。
  5. 在配置面板中向下滚动至默认排序部分，并选择航班日期属性进行排序。

第三部分：配置过滤

要配置过滤功能，请先添加并配置一个用于Flight Alert对象的过滤列表小部件：

  1. 在模块布局的左侧，选择 添加小部件，然后从出现的选择器中选择 过滤列表 小部件。

  2. 在 过滤列表 1 小部件的配置面板中，选择 对象集 下拉菜单以定义将要显示的数据。再次选择 + 新建对象集变量，然后在弹出的变量定义窗口中，选择 起始对象集 下拉菜单，然后选择 [示例数据] 航班警报 . 您必须定义一个新的对象集变量来支持筛选列表小部件，以便筛选不会限制支持该小部件的基本对象集。

  3. 返回到筛选列表 1小部件的主要配置面板，并查找筛选配置部分，使用+ 添加筛选按钮为以下属性添加筛选条件：状态、优先级和航班日期。

  4. 在筛选列表配置中，将布局配置设置为垂直以在堆叠列表中显示筛选条件。为了获得更紧凑的体验，您也可以选择药丸，这会将筛选条件显示为可交互的药丸，打开弹出窗口进行配置。

  5. 在此配置面板的底部，打开切换开关以启用允许用户添加和删除筛选条件。

接下来，修改对象表小部件以接受我们筛选列表小部件输出的筛选条件：

  1. 选择“对象表”小部件，以在屏幕右侧显示其配置面板。
  2. 在配置面板中，选择对象集。在出现的变量定义弹出窗口中，选择过滤器... — 使用变量选项，并选择过滤列表1 过滤输出变量。
  3. 通过在面板中切换一个过滤器来测试过滤功能（例如，通过过滤为优先级：中等的警报类型）。您应该注意到对您的对象表应用了过滤器。

过滤列表1 过滤输出是一个对象集过滤变量。您可以设置默认过滤器，并且如果需要的话，可以启用过滤值提取，以便在模块的其他地方重用所选的值。有关更多信息，请参阅关于对象集过滤变量的文档。

第四部分：为对象视图配置一个可折叠面板

让我们为对象表小部件中的活动对象添加并配置一个对象视图小部件。

  1. 选择屏幕右侧的对象标题周围的区域，并注意出现在模块顶部的部分工具栏。选择添加—右侧选项，在对象表小部件的右侧添加一个新部分。

  2. 选择出现的新部分。在右侧的部分配置面板中，打开部分标题和可折叠选项。此外，将列宽设置为绝对像素大小500。

  3. 接下来，在此部分中选择+ 添加小部件，然后从出现的选取器中选择对象视图小部件。

  4. 在配置面板中，打开要显示的对象 选项并选择 Object Table 1 Active Object 变量，该变量是 Object Table 小部件的输出。

  5. 在对象视图 显示与格式化 设置中，将 表单因子 设置为 完整。由于该部分已经配置了标题，您可以选择切换 隐藏标题 以避免同时显示部分标题和小部件标题。如果您选择 面板 表单因子，则设置 面板行为 为 对象实例，以便面板显示来自对象表的活动对象。

第五部分：通过操作启用写回功能

以下部分假设 [示例数据] 降级飞行警报 操作类型已经创建完成。有关如何创建新操作类型的详细信息，请参阅 关于在 Workshop 中使用操作的文档。

现在，我们添加并配置一个“解决警报”按钮，以便用户在解决一个问题时可以将信息写回到航班警报数据中。

  1. 选择新配置的“对象视图”标题上方的章节标题。在右侧选择加号（+），然后从出现的选择器中选择按钮组小部件。
  2. 接下来，配置按钮的标题和样式。在按钮配置标题下，选择默认名为按钮 1的第一个按钮。在按钮配置中，将文本更改为Resolve Alert，将意图更改为Success，并将左侧图标更改为Small tick。
  3. 然后，我们配置当用户选择该按钮时要触发的操作类型。在相同的项块中，将点击时设置更改为操作，并使用选择一个操作... 下拉菜单以选择 [Example Data] De-escalate Flight Alert 操作。在出现的 参数默认值 部分，我们可以配置此操作的默认输入参数。对于 [示例数据] 航班警报 参数，选择我们用于支撑对象视图小部件的 对象表 1 操作对象 变量。不要为 注释 变量设置默认值，以便用户可以手动输入他们如何解决此航班警报。
  4. 在 [示例数据] 航班警报 参数配置下，将 表单中的可见性 值设为 隐藏。这将防止 Alert 对象在用户与 Resolve Alert 按钮交互后显示的表单中出现。

第六部分：整理您的收件箱并添加标题

我们可以现在配置模块的总体标题。

  1. 选择我们模块顶部的当前未配置的标题。在右侧出现的配置面板中，将图标更改为颜色为红色3的飞机。在同一个配置面板中，将标题输入字段设置为Flight Alert Inbox。
  2. 最后，配置部分标题。要配置我们的筛选器、对象表和对象视图部分的标题，请选择每个标题。然后，使用右侧的配置面板来配置部分下的标题（例如“筛选器”），并为每个部分标题选择一个蓝图图标（例如“筛选器”）。你的最终结果应该看起来像下面和本教程顶部所示的收件箱。
