管理受限制的视图

使用受限制的视图来支持对象类型

为了在对象资源管理器中提供特定对象类型的访问权限，请在 本体论管理器 中将受限制的视图设置为后备数据集。要成功配置并保存由受限制的视图支持的对象类型，您必须拥有对受限制视图输入数据集的 View 访问权限。

受限制的视图文件权限

要创建和编辑受限制的视图，您必须满足在角色界面中的细粒度权限管理流程中授予的以下条件。要访问角色界面，请导航至“设置”并选择 平台设置侧边栏部分中的角色。访问角色接口需要特殊的平台权限；如果您需要此级别的访问权限，请联系您的 Palantir 代表。

角色描述
创建受限制视图资源在文件夹/项目中需要。
为数据集创建受限制视图在受限制视图的上游数据集中需要。
编辑资源粒度策略在资源上编辑粒度策略。
读取资源粒度策略在资源上读取粒度策略。
编辑受限制视图资源在受限制视图上进行编辑（策略，假设标记）。
查看受限制视图资源查看受限制视图的属性（策略，假设标记）。
查看受限制视图事务查看历史事务元数据（策略，假设标记）。

要构建一个受限制视图，您必须对输入数据集具有查看访问权限，并且对输出的受限制视图具有编辑权限。

要在Contour分析中使用受限视图，您需要读取受限视图权限。

要配置并保存由受限视图支持的对象类型，您必须拥有：

  • 对受限视图输入数据集的查看访问权限。
  • 对受限视图的编辑访问权限（以查看/设置/更改策略）。
  • 是本体论管理员组的成员（以访问本体论管理器）。

要在对象资源管理器中对基于数据集的对象使用细粒度策略，您必须在数据集上拥有查看本体论数据源权限，才能看到此类对象。

受限视图策略管理

受限视图策略使用细粒度策略来确定用户可以访问哪些行。了解有关设计细粒度策略、用户属性、策略比较和策略限制的更多信息。

受限制视图的限制

受限制视图类似于数据集，但有一些关键区别。其内容结合了两个动态因素：策略定义以及特定时间点的用户属性和组成员身份。策略定义的历史记录保存在事务历史中。然而，事务历史无法完整记录所有用户属性和组成员身份的历史。

受限制视图旨在简化单个用户的分析消费，不能作为数据转换的输入。在Foundry中构建的管道应具有可重复性，并且与运行它们的具体用户无关。这种期望与受限制视图不兼容，因为受限制视图提供了依赖于用户属性的行级权限。

  • 用户尝试在具有受限视图的管道上协作时，可能无法访问策略语句，且可能不具备相同的用户属性和组成员身份。因此，每位用户在受限视图中看到的行和聚合数据可能不同；用户不应假设基于细粒度权限数据的工作流会与基于常规数据集资源的工作流在Foundry中的行为相同。

  • 对于下游转换，没有强制措施来确保后续的下游转换保留策略列。相比之下，受限视图是只读的，以保护模式和列免受可能造成受限数据暴露的更改。

下表总结了受限视图当前的限制：

操作受限视图是否支持？说明
读取是可通过对象或在Contour中读取受限视图。
实时计算是 具有受限视图，可以通过对象（例如在 Quiver 或 Functions 中）或数据集（使用 Contour 等工具）从可访问行中进行计算。
写回是基于受限视图的对象可以定义写回操作。
导出是可以从受限视图中通过 Quiver、Contour 和其他应用程序导出数据。
批量处理否由于不同用户看到的数据子集不同，受限视图不支持批量处理。
将输出保存为 Foundry 数据集否不支持基于对受限视图执行的转换的输出保存；由于 Spark 不原生支持行级权限，因此无法保证后续事务维持限制的保证。
同步到 Postgres否不支持将受限视图同步到 Postgres，因为依赖于用户属性的行级权限将无法保持。
