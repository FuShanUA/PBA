用于对 Foundry 数据集执行底层写入操作的接口。

write.spark.df(df, partition_cols=NULL, bucket_cols=NULL, bucket_count=NULL, sort_by=NULL)

  • 将指定的 DataFrame ↗ 写入输出数据集。

    参数
    • df (pyspark.sql.DataFrame) – 待写入的 PySpark DataFrame。
    • partition_cols (List[str], 可选) - 写入数据时使用的列分区。
    • bucket_cols (List[str], 可选) - 用于对数据进行分桶的列。如果指定了 bucket_count，则必须设置此参数。
    • bucket_count (int, 可选) – 分桶数量。如果指定了 bucket_cols，则必须设置此参数。
    • sort_by (List[str], 可选) - 用于对分桶后数据进行排序的列。

write.data.frame(rdf)

  • 将指定的 R data.frame ↗ 写入输出数据集。

  • 返回一个 FileSystem 对象，用于直接访问 FoundryFS。

R FileSystem 对象

ls(glob=NULL, regex='.*', show_hidden=FALSE)

  • 列出与指定模式（glob 或 regex）匹配的所有文件，匹配基准为数据集的根目录。

    参数
    • glob (str, 可选) – Unix 文件匹配模式。同时支持 globstar。
    • regex (str, 可选) – 用于匹配文件名的正则表达式模式。
    • show_hidden (bool, 可选) – 是否包含隐藏文件，即以 “.” 或 “_” 为前缀的文件。
    返回由 FileStatus 命名元组 (path, size, modified) 组成的 R 数组 - 包含逻辑路径、文件大小（字节）、修改时间戳（自 1970 年 1 月 1 日 UTC 以来的毫秒数）

open(path, open='r', disk_optimal=FALSE, encoding=default)

  • 以指定模式打开 FoundryFS 文件。

    参数
    • path (str) – 文件在数据集中的逻辑路径。（远程路径）
    • open (str) - 描述用于打开连接的模式。
    • disk_optimal (bool, 可选) – 控制 FoundryFileSystem 处理文件 I/O 的方式。
    • encoding (str, 可选) - 默认值为 R 语言的默认编码 (UTF-8)。
    返回R 连接对象

get_path(path, open='r', disk_optimal=FALSE, encoding=default)

  • 针对给定的 FoundryFS（远程）路径，返回本地临时路径。

    参数
    • path (str) – 文件在数据集中的逻辑路径。（远程路径）
    • open (str) - 描述用于打开连接的模式。
    • disk_optimal (bool, 可选) – 控制 FoundryFileSystem 处理文件 I/O 的方式。
    • encoding (str, 可选) - 默认值为 R 语言的默认编码 (UTF-8)。
    返回str

upload(local_path, remote_path)

  • 将文件从本地路径上传至远程路径。仅支持写入操作。

    参数
    • local_path (str) – 待上传文件的本地路径。
    • remote_path (str) - 文件在数据集中的逻辑路径。
    返回None

进阶主题：disk_optimal 设置

在 FileSystem 的 open() 和 get_path() 方法中，disk_optimal 参数用于控制文件输入与输出 (I/O) 的处理方式。

默认情况下，open() 和 get_path() 中的 disk_optimal 均设为 FALSE。在此模式下，系统会确保在访问文件之前完成下载。

若将 disk_optimal 设为 TRUE，文件下载将与代码执行同步进行。必须通过 fifo() 打开本地临时路径才能正确读取。请注意，并非所有库都支持读取此类文件。

当读取的文件非常大时，可以选择将 disk_optimal 设为 TRUE。

例如，假设我们有一个非常大的 txt 文件，但只想读取前 10 行。可以使用以下代码仅打印前 10 行，而无需读取整个文件。

1disk_optimal_example<- function(large_txt_file) {
2    fs <- large_txt_file$fileSystem()
3
4    ## 使用 fifo() 打开连接
5    ## 文本文件名为 large_txt_file.txt
6    conn <- fs$open("large_txt_file.txt", "r", disk_optimal = TRUE)
7
8    A <- readLines(conn, n = 10)
9    print(A)
10    return(NULL)    
11}

若要使用 R TransformOutput 写入文件后再进行读取，必须将 disk_optimal 设为 false。
