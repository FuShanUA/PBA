使用 Pipeline Builder 创建数据集批量管道

在本教程中，您将使用 Pipeline Builder 创建一个简单的管道，其输出为一个包含航班警报信息的单一数据集。然后，您可以使用 Contour 或 Code Workbook 等工具分析此输出数据集，以回答诸如哪些航班路径最容易受到干扰等问题。

以下使用的数据集可在数据集导入步骤中通过名称搜索；您可以在 Foundry 文件系统中找到它们。

第一部分：初始设置

首先，创建一个新的管道。

  1. 登录到Foundry后，从左侧导航栏中选择应用程序，以搜索并打开Pipeline Builder



  2. 接下来，在Pipeline Builder首页上，通过选择新建管道来创建一个新的管道，然后选择批处理管道。在选择批处理计算下，选择标准。



在某些Foundry环境中，无法创建流式管道。如果您的使用案例需要此功能，请联系Palantir支持团队以获取更多信息。

  1. 选择保存管道的位置。请注意，管道不能保存在个人文件夹中。



  2. 选择创建管道。

第2部分：添加数据集

现在，您可以将数据集添加到您的管道工作流程中。使用示例数据集或开源数据。

从Pipeline Builder页面中，从Foundry中选择添加数据集。

或者，您可以从计算机中拖放文件以将其用作数据集。

在本教程示例中，您可以添加 passengers_preprocessed、flight_alerts_raw 和 status_mapping_raw 数据集。要添加一组数据集，请选择数据集。然后，选择内联 + 图标，或选择 添加到选择。

选择所有需要的数据集后，选择 添加数据集。

第3部分：清理数据

在添加原始数据集之后，您可以执行一些基本的清洗转换以继续定义您的管道。您将对其中三个原始数据集进行转换。

数据集 1

第一步是清理passengers_preprocessed数据集。首先设置一个转换（cast transform）来将dob列名更改为dob_date，同时将值转换为MM/dd/yy格式。

转换

  1. 在您的图中选择passengers_preprocessed节点。

  2. 选择转换。



  3. 在下拉菜单中搜索并选择cast转换以打开转换配置板。



  4. 从表达式字段中选择dob；对于类型，选择Date。

  5. 输入MM/dd/yy作为格式类型。确保使用大写的MM以确保转换成功。将输出列名更改为dob_date。

    您的转换板应如下所示：



  6. 选择应用，将转换添加到您的管道中。

首字母大写转换

现在，您可以将flyer_status列的值格式化为以大写字母开头。

  1. 在转换搜索字段中，搜索并选择首字母大写转换，以打开首字母大写配置板。

  2. 在表达式字段中，从下拉菜单中选择flyer_status列。

    您的首字母大写板应如下所示：



  3. 选择应用，将转换添加到您的管道中。

  4. 在转换配置窗口的左上角，将转换重命名为Passengers_Clean。



  5. 在屏幕右上角选择返回图表，以返回到您的管道图表。



数据集 2

现在，清理 flight_alerts_raw 数据集。第一步是设置另一个转换（cast transform）来将 flight-date 列的值转换为 MM/dd/yy 格式。

转换（Cast transform）

  1. 在您的图表中选择 flight_alerts_raw 数据集节点。

  2. 选择转换（Transform）。



  3. 从下拉菜单中搜索并选择转换，以打开转换配置板。您可以在选择框右侧查看函数定义，了解更多关于该函数的信息。



  4. 在表达式字段中，从下拉菜单中选择flight_date列。

  5. 从类型字段的下拉菜单中选择Date。

  6. 为格式类型输入MM/dd/yy。确保使用大写的MM以确保转换成功。

    您的转换板应该如下所示：



  7. 选择应用以将转换添加到您的管道中。

清理字符串转换

现在，添加一个清理字符串 将移除category列值中的空白字符的转换。例如，该转换会将delay···字符串值转换为delay。

  1. 从下拉菜单中搜索并选择“清理字符串”转换，以打开清理字符串配置面板。

  2. 在表达式字段中，从下拉菜单中选择category列。

  3. 勾选所有三个清理操作选项的复选框：

    • 将空字符串转换为null
    • 将多个空格字符序列减少为一个空格
    • 修剪字符串开头和结尾的空格

    您的清理字符串面板应如下所示：



  4. 选择应用，将转换添加到您的管道中。

  5. 在转换配置窗口的左上角，将转换重命名为Flight Alerts - Clean。

  6. 在右上角选择返回图表，返回到您的管道图表。



数据集 3

最后，清理 status_mapping_raw 数据集。

清理字符串转换

你将只对这个数据集应用一个 清理字符串 转换。

  1. 在你的图中选择 status_mapping_raw 数据集节点。

  2. 选择 转换。



  3. 在 搜索转换和列... 字段中，从下拉菜单中选择 mapped_value 列。



  4. 在同一字段中，从下拉菜单中搜索并选择干净字符串转换。

  5. 勾选所有三个清理操作选项：

  • 将空字符串转换为null

  • 将多个空格字符序列减少为一个空格

  • 修剪字符串开头和结尾的空格

    您的干净字符串板应该看起来像这样：



  1. 选择应用以将转换添加到您的管道中。

  2. 在转换配置窗口的左上角，将转换重命名为Status Mapping - Clean。

  3. 在右上角选择返回图表以返回到您的管道图表。

    您可以看到刚刚添加的转换与所应用的数据集之间的连接关系。



第4部分：关联数据集

现在，你可以使用连接将一些清理后的数据集进行合并。连接允许你将至少有一个匹配列的数据集进行合并。你将在你的管道工作流中添加两个连接。

连接1

你的第一个连接将合并两个清理后的数据集。

  1. 选择Flight Alerts - Clean转换节点。这将是连接的左侧。

  2. 选择连接。



  3. 选择Status Mapping - Clean节点，将其作为连接的右侧。

  4. 选择开始以打开连接配置板。



  5. Verify that the Join type is set to Left join.

  6. Set the Match condition columns to status is equal to value.

  7. Select Show advanced to view additional configuration options.

  8. Set the Prefix of the right Status Mapping - Clean dataset to status.

    Your join configuration board should look like this:



  9. Select Apply to add the join to your pipeline.

  10. View a preview of the join output table in the Preview pane at the bottom of the configuration window.



  11. In the top left corner of the join configuration window, rename the join Join Status.

  12. Select Back to graph at the top right to return to your pipeline graph.



  13. To make the graph easier to read, select the Layout icon to automatically arrange the datasets, or manually drag the two connected datasets next to each other.



Join 2

For your second join, combine your first join output table with another raw dataset.

  1. Add the priority_mapping_raw dataset to the graph by selecting Add datasets.

  2. Select the Join Status node you just added to the graph. This will be the left side of the join.

  3. Select Join.

  4. Select the priority_mapping_raw dataset node to add it as the right side of your join.

  5. Select Start to open the configuration board.



  6. Verify that the Join type is set to Left join.

  7. Set the Match condition columns to priority is equal to value.

  8. Select Show advanced to view additional configuration options.

  9. Set the Prefix of the right priority_mapping_raw dataset to priority.

    Your join configuration board should look like this:



  10. Select Apply to add the join to your pipeline.

  11. View a preview of the join output table in the Preview pane at the bottom of the configuration window.



  12. In the upper left corner of the join configuration window, rename the join Join (2).

  13. Select Back to graph at the top right to return to your pipeline graph.

You can now see the connection between the joins you just added and the datasets to which you applied them.

Part 5: Add an output

Now that you have finished transforming and structuring your data, you can add an output. For this tutorial, you will add a dataset output.

  1. In the Pipeline outputs sidebar to the right of the Pipeline Builder graph, name the output Flight Alerts data. Then select Add dataset output.
  2. Link Join (2) to the output by selecting the white circle to the right of the join node and connecting it to the Flight Alerts datadataset.
  3. Select Use input schema to use existing schema.
  4. From here, select the columns of data to keep. In this case, keep all the data together.



Part 6: Build the pipeline

To build your pipeline, select Deploy in the top right of your graph. Then, choose Save and deploy.

You should see a small alert indicating the deploy was successful. Select View in the alert box to open the Build progress page.

From this page, you can monitor the progress of your build until the dataset output is ready.

You can now access your dataset by selecting Actions > Open.

With this last step, you generated your pipeline output. This output is a dataset that can be further explored in other applications in Foundry, such as Contour or Code Workbook.
