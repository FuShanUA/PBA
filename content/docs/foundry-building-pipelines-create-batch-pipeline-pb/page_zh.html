# 使用 Pipeline Builder 创建数据集批处理管道 在本教程中，您将使用 Pipeline Builder 创建一个简单的管道，输出包含航班警报信息的单个数据集。随后，您可以使用 Contour 或 Code Workbook 等工具分析此输出数据集，以解答诸如哪些航线面临最大中断风险等问题。 以下使用的数据集可在数据集导入步骤中按名称搜索；您可以在 Foundry 文件系统中找到它们。 本教程结束时，您将得到一个如下所示的管道： 该管道将生成一个新的 `Flight Alerts Data` 数据集输出，您可以在平台中对其进行进一步探索。
Palantir 学习门户

您可以在 learn.palantir.com ↗ 上找到关于构建首个管道的深入课程。

## 第 1 部分：初始设置 首先，创建一个新管道。 1. 登录 Foundry 后，从左侧导航栏中选择 **Applications**（应用程序），搜索并打开 **Pipeline Builder**



2. 接着，在 Pipeline Builder 着陆页上，选择 **New pipeline**（新建管道）创建新管道，然后选择 **Batch pipeline**（批处理管道）。在 **Select batch compute**（选择批处理计算资源）下，选择 **Standard**（标准）。



并非所有 Foundry 环境都支持创建流式管道。如果您的用例有此需求，请联系 Palantir Support 获取更多信息。

3. 选择保存管道的位置。请注意，管道不能保存在个人文件夹中。



4. 选择 **Create pipeline**（创建管道）。 ## 第 2 部分：添加数据集 现在，您可以向管道工作流中添加数据集了。请使用包含虚拟或开源数据的示例数据集。 在 Pipeline Builder 页面中，从 Foundry 选择 **Add datasets**（添加数据集）。 或者，您也可以从计算机中拖放文件作为数据集。 在此示例演示中，您可以添加 `passengers_preprocessed`、`flight_alerts_raw` 和 `status_mapping_raw` 数据集。要选择数据集，请选中该数据集。然后，选择行内的 **+** 图标，或选择 **Add to Selection**（添加到所选）。 选择所有需要的数据集后，选择 **Add datasets**（添加数据集）。 ## 第 3 部分：清洗数据 添加原始数据集后，您可以执行一些基本的清洗转换操作，继续定义管道。您将对三个原始数据集进行转换。 ### 数据集 1 第一步是清洗 `passengers_preprocessed` 数据集。首先设置类型转换，将 `dob` 列名更改为 `dob_date`，同时将值转换为 `MM/dd/yy` 格式。 #### 类型转换 (Cast transform) 1. 在图表中选择 `passengers_preprocessed` 节点。 2. 选择 **Transform**（转换）。



3. 从下拉菜单中搜索并选择 **cast**（类型转换）转换，以打开类型转换配置面板。



4. 在 **Expression**（表达式）字段中，选择 `dob`；在 **Type**（类型）中，选择 `Date`（日期）。 5. 在 格式 类型中输入 MM/dd/yy。请务必使用大写的 MM，以确保类型转换成功执行。将输出列名更改为 dob_date。

    数据集 2

    接下来，清洗 flight_alerts_raw 数据集。第一步是设置另一个类型转换，将 flight-date 列的值转换为 MM/dd/yy 格式。

    1. 在图表中选择 flight_alerts_raw 数据集节点。

    2. 选择 转换。

    4. 在 表达式 字段中，从下拉菜单选择 flight_date 列。

    5. 从 类型 字段的下拉菜单中选择 Date。

    6. 在 格式 类型中输入 MM/dd/yy。请务必使用大写的 MM，以确保类型转换成功执行。

      你的转换配置面板应如下所示：

    字符串清洗转换

    1. 从下拉菜单中搜索并选择字符串清洗转换，打开配置面板。

      • 将多个连续的空白字符缩减为单个空白字符
      • 去除字符串首尾的空白字符

    6. 选择右上角的 返回图表，回到流水线图表。

      第四部分：关联数据集

      现在，你可以通过 joins 将部分清理后的数据集合并起来。关联操作允许你通过至少一个匹配列来合并数据集。你将在流水线工作流中添加两个关联。

      关联 1

      你的第一个关联将合并两个已清理的数据集。 选择 Flight Alerts - Clean 转换节点。这将作为关联的左侧。 选择 Join。



      选择 Status Mapping - Clean 节点，将其添加为关联的右侧。 选择 Start 打开关联配置面板。



      确认 Join type 设置为 Left join。 将 Match condition 列设置为 status 等于 value。 选择 Show advanced 查看其他配置选项。 将右侧 Status Mapping - Clean 数据集的 Prefix 设置为 status。 关联配置面板应如下所示：



      选择 Apply，将关联添加到流水线中。 在配置窗口底部的 Preview 窗格中查看关联输出表的预览。



      在关联配置窗口的左上角，将该关联重命名为 Join Status。 选择右上角的 Back to graph 返回流水线图谱。



      为使图谱更易于阅读，选择 Layout 图标自动排列数据集，或手动拖动两个相连的数据集使其相互靠近。



      关联 2

      对于第二个关联，需将第一个关联的输出表与另一个原始数据集合并。 选择 Add datasets，将 priority_mapping_raw 数据集添加到图谱中。 选择刚才添加到图谱中的 Join Status 节点。这将作为关联的左侧。 选择 Join。 选择 priority_mapping_raw 数据集节点，将其添加为关联的右侧。 选择开始打开配置面板。



    2. 确认连接类型已设为左连接。

    3. 将匹配条件列设置为priority等于value。

    4. 选择显示高级选项以查看更多配置选项。

    5. 将右侧priority_mapping_raw数据集的前缀设置为priority。

    第 5 部分：添加输出

    完成数据的转换和结构化后，即可添加输出。在本教程中，你将添加一个数据集输出。

    1. 在 Pipeline Builder 图表右侧的流水线输出侧边栏中，将输出命名为Flight Alerts data。然后选择添加数据集输出。
    2. 选择连接节点右侧的白色圆圈并将其连接到Flight Alerts data数据集，从而将Join (2)链接至该输出。
    3. 选择使用输入架构以套用现有架构。
    4. 在此处选择需保留的数据列。在此示例中，保留全部数据。



    第 6 部分：构建流水线

    要构建流水线，请选择图表右上角的部署。然后，选择保存并部署。

    此时应会出现一条提示，指明部署已成功。在提示框中选择查看以打开构建进度页面。

    在该页面上，你可以监控构建进度，直到数据集输出准备就绪。

    现在，你可以选择操作 > 打开来访问数据集。

    完成最后这一步后，流水线输出即生成完毕。此输出为一个数据集，可在 Foundry 的其他应用中进一步探索，例如《Contour》或《代码工作簿》。
