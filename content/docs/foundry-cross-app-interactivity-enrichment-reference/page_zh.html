Foundry 与 Gotham 之间的拖拽操作

在某些情况下，同一个现实世界的对象可能同时存在 Foundry 和 Gotham 的表现形式，这使得这些对象表现形式互为同义词。同义词是指能够准确表示现实中同一对象的不同资源。通过类型映射，可以将这些同义词相互关联，从而确立它们作为同义表现形式的地位。

为了提升 Foundry 与 Gotham 之间的兼容性，可以通过实施数据扩充来识别现有的同义词。拖拽数据扩充功能能够发现 Foundry 和 Gotham 之间已映射的同义表现形式，从而改变工作流程，增强应用程序与 Palantir 平台之间的互操作性。

要使用数据扩充功能，您的订阅必须同时包含 Gotham 和 Foundry，并启用类型映射。如果您的应用程序仅在 Gotham 或仅在 Foundry 内部执行拖拽操作，则无需实施数据扩充。

在继续阅读下方的教程之前，请确保您已经查阅了《概述》以及《创建拖拽集成点》教程，以掌握拖拽概念和实现的基础知识。

通过数据扩充提升跨应用程序交互性

举一个示例用例，假设您的应用程序能够识别 Gotham 对象 ID，并且您希望创建一个工作流程，将 Gotham 对象 ID 数据从您的应用程序拖拽到 Foundry 应用程序中。您可能首先会尝试使用下方的 dragstart 处理程序将 Gotham 对象 ID 写入 DataTransfer。请注意自定义的 Palantir 媒体类型 application/x-vnd.palantir.rid.gotham.object：

1function handleDragStart(event) {
2    event.dataTransfer.setData(
3        "application/x-vnd.palantir.rid.gotham.object",
4        JSON.stringify(["ri.gotham.XXXXXXXX", "ri.gotham.YYYYYYYY"...])
5    );
6    event.dataTransfer.effectAllowed = "move";
7}

你可能会注意到，当你尝试将数据拖动到 Foundry 放置区时，它们并没有“亮起”，这表明该数据对这些放置区来说不是有效的媒体类型。Foundry 放置区很可能不会接受此数据，因为 DataTransfer 包含特定于 Gotham 的媒体类型。你可以通过实现数据扩充来解决这个问题，以便识别 Gotham 对象的 Foundry 同义词并将其添加到 DataTransfer 中。这允许 Foundry 接受你拖放的数据，因为现在 DataTransfer 中将包含具有可接受媒体类型的同义表示形式。请注意，只有当你的注册配置中启用了类型映射，并且针对你计划扩充的对象也启用了该功能时，此方法才有效。请参阅类型映射文档以了解更多信息。

上下文与指南

以下章节提供了关于数据扩充各个方面的上下文。这包括使数据扩充成为可能的服务概述，以及实现扩充的时机和位置。

添加数据扩充的位置

数据扩充可以添加到拖动区或放置区。在拖放工作流中，没有必要在拖动区和放置区同时添加扩充。确定应用程序之间的数据流向，以判断数据从哪里来以及去往哪里。

如果你的应用程序包含放置区，且你希望用户将 Gotham 和 Foundry 数据拖放至此，我们建议在拖动负载放置后在放置区进行数据扩充。这样一来，你的应用程序就无需同时识别 Gotham 和 Foundry 的概念。相反，你的应用程序可以接受 Gotham 或 Foundry 媒体类型，并在原始拖动负载不属于可接受的媒体类型时，使用来自数据扩充的同义数据。

另外，如果你的应用程序中有拖动区，且你希望用户能够将其拖动至 Foundry 和 Gotham 的放置区，我们建议在将数据添加到 DataTransfer 之前对其进行扩充。这样，你的拖动负载将包含 Foundry 和 Gotham 的媒体类型，从而允许 Foundry 和 Gotham 放置区接受该拖动负载。

请求扩充后的数据

在实现数据扩充时，你必须向 Data Bank Service 发出请求以获取同义数据。Data Bank Service 会识别并返回跨 Foundry 和 Gotham 的同义对象标识符，从而实现跨平台的拖放操作。

向 Data Bank Service 发起请求会带来轻微的性能损耗，因为这需要额外的调用。我们建议你要清楚了解应用程序中添加了多少个扩充的拖动区或放置区，以及由此带来的性能损耗增加情况。请联系你的 Palantir 代表以获取有关此服务的更多信息。

如果你的工作流涉及在 Gotham 和 Foundry 之间拖放数据，请按照扩充教程来发挥 Palantir 媒体类型的强大功能。
