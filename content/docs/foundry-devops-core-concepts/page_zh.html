核心概念

本页面介绍了支撑 Foundry DevOps 和 Foundry Marketplace 的核心概念。我们建议您在继续阅读之前先了解本页内容，因为 Foundry DevOps 和 Marketplace 使用了专门的术语（例如“产品”）。

产品

在 Foundry DevOps 中，“产品”是产品构建者提供出来供安装的 Foundry 资源集合。构建者可以在 Foundry DevOps 中创建新产品。

产品类型示例包括：

  • 本体： Foundry 本体的一个实例，可以是全面的行业标准本体，也可以是本体片段。
  • 用例： 用例可以简单到只是一个单一的 Workshop 应用程序，也可以是多个 Workshop 应用程序、函数和 Carbon 工作区的组合。
  • 管道： 管道的范围可以从单一的数据转换，一直延伸到按顺序排列的众多转换和数据连接的集合。
  • 建模： 封装了任何功能逻辑的容器化可执行模型，包括机器学习、预测、优化、物理模型和业务规则。

每个产品都有在安装时产生的输出，即内容。产品可能需要安装者映射输入才能生成输出内容。

产品是用户可以通过 Marketplace 商店浏览和安装的原子单元。

产品版本

产品构建者每次想要调整现有产品内容或添加新产品内容时，都会创建并发布产品的新版本。安装时，我们通常建议选择最新产品版本。当有新产品版本可用时，安装者可以手动升级或选择自动接收新版本。

商店

商店是产品的集合；这些产品通常具有共同的目的。产品构建者可以将新产品发布到商店。在您的 Foundry 实例上出现的商店可以是本地的（例如，由组织内在您的 Foundry 实例上工作的构建者生产），也可以是远程的（例如，对所有 Foundry 用户开放并由 Palantir 维护的 Foundry Store）。

安装

当安装者满足产品所需的输入（如果有）时，就会创建一个安装。每个产品可以安装多次。在以下情况通常会创建多个安装：

  • 不同的用户群体需要同一个产品。例如，您可以在多个支持团队中安装一个工单管理产品，并将其工单数据作为输入。
  • 您需要为不同的开发环境创建安装。例如，您可以将工单管理产品安装到预生产空间和本体中，然后再安装到生产空间和本体中，每个环境都有各自的输入数据和特定于环境的设置，例如发布通道和升级窗口。
