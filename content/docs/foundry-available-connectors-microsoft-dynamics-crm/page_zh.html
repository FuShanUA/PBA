Microsoft Dynamics CRM

Microsoft Dynamics CRM 连接器是 Palantir 提供的一款用于 Microsoft Dynamics CRM 的驱动程序。

要创建新的 Microsoft Dynamics CRM 数据源，请遵循 Palantir 提供的驱动程序的标准设置流程，然后参考以下章节进行 Microsoft Dynamics CRM 的特定配置与网络设置。有关完整的属性参考，请查阅《Microsoft Dynamics CRM 官方驱动程序文档》 ↗。

配置

以下属性为必填或建议填写项。

属性是否必填？描述默认值
CRMVersion ↗必填要连接的 Dynamics CRM 服务器类型。可接受的取值为 CRM2011+、CRMOnline。CRM2011+
URL ↗必填组织的根 URL。例如，CRM 4.0 或 CRM 2011 的 URL 类似于 http://MySite/MyOrganization。对于 CRM Online，URL 类似于 https://myOrg.crm.dynamics.com/。https://myOrg.crm.dynamics.com/, http://mySite/myOrganization
AuthScheme ↗建议所使用的身份验证方案。可接受的取值为 OAuth、NTLM、Kerberos、AzureServicePrincipal、AzureServicePrincipalCert。Auto
InitiateOAuth ↗建议指定获取或刷新 OAuth 访问令牌的流程，以便在经过身份验证和授权的用户工作时维持其访问权限。REFRESH
InternetFacingDeployment ↗建议是否连接到面向 Internet 的 CRM 部署 (IFD)。FALSE
LanguageCode ↗建议指示语言的代码。1033
OAuthClientId ↗建议指定创建自定义 OAuth 应用程序时分配的客户端 ID。（也称消费者密钥）。此 ID 用于向 OAuth 授权服务器注册自定义应用程序。—
OAuthClientSecret ↗建议指定创建自定义 OAuth 应用程序时分配的客户端密钥。（也称消费者密钥）。此密钥用于向 OAuth 授权服务器注册自定义应用程序。—
OrganizationName ↗建议组织名称。在未使用 IFD 的 Dynamics CRM 4.0 中，组织在 URL 中指定；例如 http://website/organizationname。在使用 IFD 的 Dynamics CRM 4.0 中，必须设置此属性。在其他版本的 CRM 中，此属性为可选。—
Password ↗建议用于对用户进行身份验证的密码。—
ServerVersion ↗建议DynamicsCRM 的服务器 SDK 版本。—
User ↗建议用于身份验证的 SharePoint 用户账户。—
