Db2

将Foundry连接到Db2，以便在Db2数据库和Foundry之间读取和同步数据。

支持的功能

功能状态
探索🟢 通常可用
批量同步🟢 通常可用
增量🟢 通常可用
变更数据捕获同步🟢 通常可用
表导出🟢 通常可用

设置

  1. 打开 数据连接 应用并选择屏幕右上角的+ 新源。
  2. 从可用的连接器类型中选择Db2。
  3. 按照后续配置提示继续使用以下部分中的信息设置您的连接器。

了解更多关于在Foundry中< a href="/docs/foundry/data-connection/set-up-source/">设置连接器的信息。

JDBC 属性

您可以将 属性 ↗ 添加到 JDBC 连接中以配置行为。某些属性对于 Db2 驱动程序是强制性的。这些强制性属性默认已填充，并且在保存您的源之前必须设置。您也可以通过选择 + 添加属性 并查看 推荐 部分来查看可以添加的推荐属性。

将鼠标悬停在 Required 或 Recommended 属性名称上，以访问 Db2 驱动程序的官方文档页面。

配置 Db2 同步

SQL 查询

每次同步只能执行一个 SQL 查询。此查询应产生一个数据表作为输出，并且不应执行像调用存储过程这样的操作。查询结果将保存到 Foundry 的输出数据集。

表导出

Db2 源类型支持 表导出。对于在 z/OS 或 LUW（Linux, Unix, Windows）上运行的 Db2 数据库，不需要额外配置。但是，对于在 IBM iSeries 或 AS/400 上运行的 Db2 数据库，日志记录 ↗ 必须为任何目标表启用。

变更数据捕获

Db2 连接器支持 变更数据捕获（CDC）同步。Db2 CDC 同步是通过 Debezium Db2 连接器 ↗ 实现的。要为您的 Db2 实例启用 CDC，请按照 Debezium 文档 ↗ 中概述的说明进行操作。

一旦为您希望同步到 Foundry 的表启用了变更数据捕获，您可以导航到 概览 页面并选择 + 创建 CDC 同步 来开始创建新的变更数据捕获同步。

为了创建变更数据捕获同步，探索运行时必须正常工作。如果运行时仍在初始化，您可能需要等待几秒钟并刷新页面以继续创建变更数据捕获同步。
