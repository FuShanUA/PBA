从快照创建历史数据集

警告

如果条件允许，将历史数据集作为 APPEND 事务进行摄取 是最佳实践。有关更多详细信息，请参阅下方的警告。

工作流概述

有时，你可能会遇到这样的原始数据集：每天、每周或每小时进行一次新的 SNAPSHOT 导入，用数据集的当前数据替换前一次的视图。然而，保留之前的数据以便确定与前一次视图相比发生了哪些变化，往往非常有用。如上所述，最佳实践是在数据摄取阶段通过使用 APPEND 事务并添加包含导入日期的列来处理此问题。但在无法做到这一点的情况下，你可以在 Python 转换中使用 incremental 装饰器，将这些定期快照追加到该数据集的历史版本中。有关此方法的注意事项及潜在脆弱性，请参阅下方的警告。

示例代码

以下是使用 incremental 装饰器创建历史数据集的示例。

此方案为何有效

增量装饰器在输入和输出的读/写模式上应用了额外的逻辑。在上面的示例中，我们对输入和输出使用了默认的读/写模式。

读取模式

使用 SNAPSHOT 输入时，默认的读取模式为 current，这意味着它会获取整个输入 DataFrame，而不仅仅是自上次事务以来新增的行。如果输入数据集是由 APPEND 事务创建的，我们可以使用 incremental 装饰器，通过 added 读取模式来仅访问自上次构建以来新增的行。转换会从 current 输出中获取模式信息，因此无需像读取 DataFrame 的 previous 版本（例如 dataframe('previous', schema=input.schema)）那样传递模式信息。

写入模式

当我们说转换以增量方式运行时，意味着输出的默认写入模式设置为 modify。此模式会使用构建期间写入的数据来修改现有输出。例如，当输出处于 modify 模式时调用 write_dataframe()，会将写入的 DataFrame 追加到现有输出中。这正是当前场景所发生的情况。

警告

由于此转换使用 SNAPSHOT 数据集作为输入，因此无法恢复由于构建失败或其他原因而可能遗漏的快照。如果这是一个隐患，请不要使用此方法。相反，请联系输入数据集的数据主人，看看是否可以将其转换为 APPEND 数据集，以便您可以访问该数据集的上一个事务。这才是增量计算的设计初衷。

在以下情况下，此方法将失败：

  • 向输入数据集中添加了列
  • 现有表的列数与输入数据模式不匹配
  • 输入数据集的列数据类型发生变化，例如从 integer 变为 decimal
  • 你更改了输入数据集，即使该数据集具有相同的架构。在这种情况下，它会用输入完全替换输出，而不是追加。

资源消耗增加

采用这种模式可能会导致历史数据集中堆积大量小文件。文件堆积并非期望的结果，它会导致构建时间延长，并增加使用该历史数据集的下游转换或分析中的资源消耗。批处理和交互式计算时间可能会增加，因为读取每个文件都会产生额外开销。磁盘使用量也可能会增加，因为压缩是基于单个文件进行的，而不是跨数据集内的多个文件进行。可以通过构建逻辑来定期对数据进行重新快照，从而避免出现这种情况。

通过检查输出文件的数量，我们可以确定最佳的增量写入模式。这种模式允许我们将先前事务的输出作为 DataFrame 读取，将其与传入的数据合并，并将数据文件合并在一起。这样就能将许多小文件整合为一个较大的文件。

检查输出数据集文件系统中的文件数量，并使用 if 语句来设置 write_mode，如下例所示：

1import polars as pl
2from transforms.api import transform, Input, Output, incremental
3
4FILE_COUNT_LIMIT = 100
5
6# Be sure to insert your desired output schema here
7schema = {
8    'Value': pl.Float64,
9    'Time': pl.Datetime,
10    'DataGroup': pl.Utf8
11}
12
13def compute_logic(df):
14    """
15    This is your transforms logic
16    """
17    return df.filter(pl.lit(True))
18
19@incremental(semantic_version=1)
20@transform.using(
21    output=Output("/Org/Project/Folder1/Output_dataset_incremental"),
22    input_df=Input("/Org/Project/Folder1/Input_dataset_incremental"),
23)
24def compute(input_df, output):
25    df = input_df.polars('added')
26    df = compute_logic(df)
27
28    # Fetches a list of the files that are in the dataset
29    files = list(output.filesystem(mode='previous').ls())
30
31    if (len(files) > FILE_COUNT_LIMIT):
32        # Incremental merge and replace
33        previous_df = output.polars('previous', schema)
34        df = pl.concat([df, previous_df])
35        mode = 'replace'
36    else:
37        # Standard incremental mode
38        mode = 'modify'
39
40    output.set_mode(mode)
41    output.write_table(df)
