Vega 图表

Vega 图表微件用于在 Workshop 中，借助 Vega 和 Vega-Lite ↗ 语法创建完全可定制且支持交互的可视化效果。

什么是 Vega？

Vega 允许你以简洁的 JSON 规范形式创建、保存和分享交互式可视化设计，该规范用于描述可视化的外观与行为。 Vega 是一种基于 D3 ↗ 构建的高级可视化规范语言，而 Vega-Lite 则是基于 Vega 构建的更高级语言，为编写常见可视化提供了更为简洁便利的方式。

相比标准的 XY 图表微件，Vega 图表微件提供了更强的可定制性，并支持如下方图表所示的可视化效果，这些示例均由官方 Vega-Lite 示例库 ↗ 提供。

规约


规约 ↗是定义可视化的 JSON 内容；支持以内联方式或使用字符串变量进行指定。




Vega 与 Vega-Lite


该控件支持指定在规约中使用 Vega-Lite 或 Vega。






如果要绘制常规图表，我们推荐使用 Vega-Lite，因为它的语法更加简洁，并且支持选择参数。对于更复杂的可视化，你可以使用 Vega。请参阅 Vega-Lite ↗ 和 Vega ↗ 示例，了解两者各自支持的功能。


主题配置


为了方便复用，你可以配置一个主题并将其注入到配置规范中。默认主题与 XY 图表小部件的 Blueprint 样式保持一致，但你也可以从字符串变量中指定自定义主题，或者完全关闭主题。




构建自定义主题时，请参阅 vega-themes 仓库 ↗ 中的示例。


如果你想修改我们默认的 Blueprint 主题，可以在内联编辑器的 预览选项卡 中找到插入的配置，并将其连同你的修改直接粘贴到配置规范中。


内联编辑器


你可以使用 AIP 来创建 Vega 图表的初始版本。根据给定的输入数据，你可以向 AIP 提供如下提示词：


Make me a bar chart of average points per game, bucketed by the number of seasons played.




这会生成一个图表，你可以通过后续的 AIP 提示词对其进行迭代修改：


Remove the gridlines from this chart, and only show number of seasons greater than 4.




如果 AIP 提示词未能生成预期结果，你可以直接修改配置规范。请参阅现有的 Vega-Lite 示例 ↗ 和其他 Vega 资源以获取修改指导：例如，你可以基于一个示例起步，然后将示例数据替换为对你的输入数据的引用。


内联编辑器：预览


预览 选项卡会展示包含输入数据和主题数据的完整配置规范。你可以截断数据以提高可读性，并将完整的配置规范复制到剪贴板。这便于你验证数据是否符合预期格式。


在线 Vega 编辑器 ↗ 是调试配置规范的有力工具。你可以通过在内联编辑器的 预览 选项卡中将 JSON 复制到剪贴板，从而将配置规范迁移到该编辑器中。你还可以选择截断数据以限制规模，从而更轻松地替换任何敏感数据。


选择参数（仅限 Vega-Lite）


Workshop 推荐使用 Vega-Lite 模式的主要原因是 Vega-Lite 支持 选择参数 ↗。在 Vega 图表小部件中，你可以配置多个选择参数，每个参数都有一个名称和一个输出变量。输出变量可以是对象集过滤器或字符串。




由于选择操作可以在配置规范中的多个不同位置指定，因此此配置不会自动注入到规范中。作为构建者，你需要负责为每个已配置的参数名称在规范中包含对应的选择参数。


以下是一个选择参数示例，它将输出 X 轴上的范围选择：

1...
2"params": [{
3  "name": "intervalSelection",
4  "select": {
5    "type": "interval",
6    "encodings": [ "x" ]
7  }
8}]
9...


并非所有类型的选择都能输出到对象集过滤器中。如果数据输入是带有指定属性的对象集，或者是带有 group by 的聚合，这应该很容易转换；否则，你可以使用字符串输出变量，即以 Vega-Lite 使用的 JSON 格式返回的选择结果。你可以通过函数或变量变换来管理此输出。
