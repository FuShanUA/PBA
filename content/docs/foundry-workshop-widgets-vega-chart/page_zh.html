Vega 图表

Vega 图表 小部件用于在 Workshop 中使用 Vega 和 Vega-Lite ↗ 语法创建完全可自定义的交互式可视化效果。

什么是 Vega？

Vega 允许您以简洁的 JSON 规范形式创建、保存和共享交互式可视化设计，该规范描述了可视化外观和行为。 Vega 是一种基于 D3 的更高层次的可视化规范语言 ↗，而 Vega-Lite 是一种构建在 Vega 之上的更高层次的语言，它提供了更简洁和便捷的方式来编写常见的可视化效果。

Vega 图表小部件比标准的 XY 图表小部件 提供了更多的自定义选项，支持如下的可视化效果，由官方提供 Vega-Lite 示例图库 ↗。

数据配置

Vega 图表小部件有三种不同的配置选项，允许你灵活地将对象数据从你的本体论（Ontology）转换为预期的 Vega 和 Vega-Lite 格式，然后将其注入到你的 JSON 规范中。

  • 对象集： 指定应包含在数据中的对象集及其属性。



  • 聚合：指定对象集，按带有分桶策略的属性进行分组，并进行聚合。每个数据点将包含每个分组属性的值，以及以指定的聚合名称为键的聚合值。



  • 函数：指定一个返回结构体列表的函数，该列表将直接用作数据。

    Copied!
    1interface CompanyData {
2    name: string;
3    marketCap: Long;
4    logoUrl: string;
5}
6
7@Function()
8public companyVegaData(): CompanyData[] {...}





您可以拥有多个数据输入，可以通过其配置的名称在规范中引用。数据将被自动注入到 JSON 规范中，您可以在编辑器的 预览标签 中看到。请注意，您也可以通过匹配 上述数据格式 将数据内联到 JSON 规范中。 。

规范

规范 ↗ 是定义可视化效果的 JSON；规范可以内联指定，也可以通过字符串变量来指定。

Vega 与 Vega-Lite

该小部件允许您指定是使用 Vega-Lite 还是 Vega 来进行您的规范。

如果您要制作常见的图表，我们建议使用 Vega-Lite，因为它更简洁的语法以及对 选择参数 的支持。对于更复杂的可视化效果，您可以使用 Vega。查看 Vega-Lite ↗ 和 Vega ↗ 示例，以了解两者都能支持的内容。

主题配置

为了便于重用，您可以配置一个主题，该主题将被注入到规范中。默认主题与XY图表小部件的Blueprint样式相匹配，但您也可以从字符串变量中指定自己的自定义主题，或者完全关闭主题。

有关构建自定义主题的示例，请参阅 vega-themes 仓库 ↗。

如果您想修改我们的默认 Blueprint 主题，可以在 内联编辑器 的 预览选项卡 中找到插入的配置，并直接将其粘贴到您的规范中并进行更改。

内联编辑器

您可以使用AIP创建Vega图表的初始版本。根据您提供的数据输入，您可以向AIP提供一个提示，例如：

Make me a bar chart of average points per game, bucketed by the number of seasons played.

这将创建一个图表，您可以使用额外的AIP提示进行迭代：

Remove the gridlines from this chart, and only show number of seasons greater than 4.

如果您的AIP提示没有提供所需的结果，您可以直接修改规范。参考现有的 Vega-Lite示例 ↗ 和其他Vega资源，以了解如何修改规范：例如，您可以从一个示例开始，并用对您输入数据的引用替换示例数据。

内联编辑器：预览

预览 选项卡显示了完整的规范，其中包含了您的数据输入和主题数据。您可以截断数据以提高可读性，并将此完整规范复制到剪贴板。这使您能够验证数据是否符合预期的格式。

在线 Vega 编辑器 ↗ 是调试您的规范的宝贵工具。您可以通过将 JSON 复制到内联编辑器的 预览 选项卡中的剪贴板中，将规范移动到编辑器中。还可以选择截断您的数据以限制规模并更容易地替换任何敏感数据。

选择参数（仅限 Vega-Lite）

Workshop 推荐使用 Vega-Lite 模式的首要原因是 Vega-Lite 支持 选择参数 ↗ . 在 Vega 图表小部件中，您可以配置多个选择参数，每个参数都有一个名称和一个输出变量。输出变量可以是对象集过滤器或字符串。

由于选择可以在规范的多个不同位置进行指定，因此此配置不会自动注入到规范中。作为构建者，您需要负责为每个配置的参数名称在规范中包含一个选择参数。

这是一个将输出 X 轴范围选择的选择参数示例：

Copied!
1...
2"params": [{
3  "name": "intervalSelection",
4  "select": {
5    "type": "interval",
6    "encodings": [ "x" ]
7  }
8}]
9...

并非所有类型的选择都可以输出到 对象集过滤器（Object Set Filter）。如果您的数据输入是一个具有指定属性的对象集，或者是一个带有 group by 的聚合，则这应该很容易转换，否则您可以使用字符串输出变量，即以 Vega-Lite 使用的 JSON 格式返回的选择。您可以使用函数或变量转换来管理此输出。
