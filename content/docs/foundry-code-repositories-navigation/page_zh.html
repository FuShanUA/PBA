导航

在代码仓库（Code Repositories）界面顶部，你可以选择以下五个不同的标签页：

  1. 代码（Code）标签页
  2. 分支（Branches）标签页
  3. 拉取请求（Pull requests）标签页
  4. 检查（Checks）标签页
  5. 设置（Settings）标签页

代码（Code）标签页

  1. 应用内帮助（In-App Help）
  2. 分支选项（Branch Options）
  3. 代码编辑器选项（Code Editor Options）
  4. 文件编辑器（File Editor）
  5. 辅助面板（Helper Panels）
  6. 状态栏（Status bar）

应用内帮助（In-App help）

在代码（Code）标签页中，你可以点击 按钮启动分步引导，带你了解代码仓库中可用的核心功能。目前，应用内帮助仅在代码（Code）视图中可用。

若要通过命令面板显示快捷键，请在 Windows 系统下按 F1 键，或在 macOS 系统下按 Fn+F1：

分支选项（Branch options）

使用分支下拉菜单可选择一个现有的沙箱分支进行工作。或者，点击 图标创建一个新的沙箱分支，该分支将包含现有分支上的代码副本。要编辑仓库中的代码，你必须在沙箱分支中进行操作——受保护分支无法直接编辑。

你可以在代码仓库分支（Code Repositories branch）和全局分支（Global branch）（仅限支持的仓库类型）之间进行选择。全局分支在 Python Transforms 和 TypeScript v1 仓库中受支持，可以包含跨多个 Palantir 应用的更改。了解有关全局分支的更多信息。

全局分支必须基于仓库的主分支，而代码仓库分支可以基于任何分支。

代码编辑器选项（Code editor options）

在仓库中编写和编辑代码时，你可以执行以下操作：

  • 点击 按钮对输入数据集样本运行转换（Transform）。这是一种在真实数据上预览代码更改的快捷方式。
  • 点击 按钮运行当前文件中定义的所有单元测试。有关如何向仓库添加单元测试的信息，请参阅《Python 测试》或《Java 测试》。
  • 点击 按钮提交沙箱分支中的所有更改。提交更改后，系统将自动对你的代码运行检查。
  • 如果选择了数据集源文件（即定义转换的文件，例如参见《Python Transforms》），你可以点击 按钮，在对代码运行自动检查后构建输出数据集的新版本。点击该按钮将触发对当前文件的所有输出数据集的构建；如果当前文件未生成任何数据集，则不会触发构建。
  • 点击 按钮创建包含你所做更改的新拉取请求。这允许其他人在将其合并到主代码之前对你的代码进行审查和评论。
  • 点击 ... 按钮可访问其他几个操作：
    • 合并（Merge）：将另一个分支合并到当前分支。
    • 重置（Reset）：重置所有文件的内容，使其与远程分支上的最新提交保持一致。这将清除分支上尚未提交的所有更改。
    • 升级（Upgrade）：将分支升级到最新的语言版本。

文件编辑器（File editor）

点击 图标可创建新文件、文件夹或子项目。若要在代码仓库中添加针对特定语言的子项目，请选择“新建子项目”选项。

辅助面板

Foundry Explorer 辅助

Foundry Explorer 辅助是一个文件导航界面，可快速浏览所有文件和文件夹。选中特定数据集后，点击“打开”即可查看完整数据集。

问题辅助

问题 辅助会提示代码中检测到的任何问题。点击此处列出的具体问题，可直接跳转到出错的代码段。

调试器辅助

调试器 用于在转换运行时检查其行为（参见《调试转换》）。

预览辅助

预览 辅助支持在输入数据集的有限样本上运行代码，无需提交更改即可快速预览代码效果（参见《预览转换》）。

测试辅助

当仓库中包含单元测试时（参见《Python 测试》和《Java 测试》），测试辅助 可运行这些测试并展示结果。

文件更改辅助

文件更改 辅助可用于查看当前文件中所有未提交的更改，以及对比文件的历史版本。

构建辅助

构建 辅助支持触发数据集构建并查看构建进度。选中数据集源文件后，点击构建按钮即可构建新版本的输出数据集，同时对代码运行自动检查。随后，可以在 构建 辅助中查看正在运行的任务进度。

在代码仓库界面右上角点击 构建 按钮，与在 构建 辅助中触发构建的效果完全等同。

文档辅助

文档 辅助包含当前可用编程语言的参考文档。若需查阅产品内置文档中未提供的更详尽的特定语言文档，请参考《支持的语言》。

SQL 临时查询板

SQL 辅助支持快速测试 SQL 查询。编写 SQL 查询并点击 即可预览查询结果。也可以选中仓库中已编写好的对应 .sql 文件，点击 进行测试。

若要查看已标记为收藏的查询，请前往 选项卡。若要查看在 SQL 辅助中执行过的查询历史记录，请前往 选项卡。

要在 SQL 临时查询板中访问输入数据集的特定分支，需将分支名称置于查询前缀中：例如 SELECT * FROM `branch_A`.`/path/to/dataset`。若未指定分支，默认使用 master。

状态栏

状态栏提供环境状态及检查结果信息。状态栏中包含以下信息：

  • 代码辅助状态 - 代码辅助功能对于检测代码问题及运行预览至关重要。脱离代码辅助虽也能编写代码，但要完成整体工作仍需其正常运行。将鼠标悬停在代码辅助状态上，可获取初始化进度详情。若代码辅助初始化失败，请提交支持请求。
  • 问题 - 检测到代码问题时，状态栏左侧会出现提示标识。点击该标识可打开问题辅助。
  • 检查状态 - 检查状态显示在状态栏右侧。有关检查的更多详情，请参阅《检查选项卡》。
  • 文件保存 - 发生任何更改后，文件保存状态会显示自动保存的进度。

分支选项卡

本节概述了如何在 分支 选项卡中创建新分支和发起新的拉取请求。这些功能在《代码选项卡》中同样可用。

在 分支 选项卡中，您可以查看代码仓库中现有的分支列表——这包括您自己的分支以及其他用户的分支。点击 按钮可创建一个新的沙箱分支，该分支将包含特定分支上的代码副本。

列表中的每个分支都包含一个摘要，提供以下功能：

  • “检查”列指示某个分支是否通过了自动代码检查。
  • “Pull request”列会显示分支中现有的 Pull request，并允许您创建新的 Pull request。 若要创建一个包含某分支更改的新 Pull request，请点击“提交更改”按钮。默认情况下，这将创建一个新的 Pull request，将您的更改合并到 master 分支中。如果您想将更改合并到 master 以外的分支，请从下拉菜单中选择其他分支。 如果没有看到创建新 Pull request 的按钮，说明该分支已存在 Pull request。点击“开放” / “已关闭” / “已合并”按钮即可打开完整的 Pull request。
  • 点击分支名称旁边的“查看代码”，可查看该分支上的代码。
  • 要删除分支，请点击 图标。请勿删除非您本人创建的分支。这可能导致他人工作丢失。

标签

分支选项卡还允许您访问 标签 列表，标签类似于不可变的分支。标签可以通过分配版本号或名称，用于标记代码的重要版本以供将来参考。要创建新标签，请导航至分支选项卡的标签部分，并点击“新建标签”按钮。您可以基于分支的当前版本或任意提交来创建标签。

为了强制所有标签名称遵循特定的命名约定，您可以在仓库根目录下名为 repoSettings.json 的文件中添加 tagNameValidation 配置块，例如："tagNameValidation": { "regex": "^(0|[1-9]\\d*)\\.(0|[1-9]\\d*)\\.(0|[1-9]\\d*)(-rc\\d+)?$", "errorMessage": "Tag name must have the format x.x.x or x.x.x-rcx." }

有关推荐的 Git 开发工作流的更多信息，请参阅《开发者最佳实践》。

Pull requests 选项卡

本节概述了如何在 Pull requests 选项卡中创建新的 Pull request。此功能在《代码选项卡》中也可使用。

在 Pull requests 选项卡中，您可以找到有关代码仓库中 Pull request 的信息。Pull request 允许用户在合并更改之前，查看分支上的更改历史并逐行审查代码。任何时候，只要您想将分支上的更改合并到主代码中，都应该创建一个新的 Pull request。

点击 按钮可创建新的 Pull request。默认情况下，新创建的 Pull request 会将您的更改合并到仓库的主分支（通常是 master 分支）中。请选择您要作为新 Pull request 基础的分支。

筛选 Pull requests

您可以通过点击 Pull requests 列表顶部的“开放” / “已关闭”按钮，在开放和已关闭的 Pull request 列表之间切换，并使用搜索栏根据标题或作者进一步筛选列表。

审查 Pull requests

点击其中一个 Pull request 可逐行审查提议的代码更改并添加评论。根据仓库设置，每个 Pull request 在合并前可能至少需要一次批准的审查。

在审查 Transforms 代码的变更时，您还可以查看这些变更如何影响您的数据集。

检查选项卡

在检查选项卡中，您可以查看每个分支上正在运行和已完成检查的摘要。使用分支下拉菜单可选择其他分支。点击特定的检查即可查看更详细的信息。

检查选项卡还将包含为您的代码库定义的所有单元测试的输出。您可以为 Python 和 Java 定义单元测试。

如果您的技术栈上启用了 AIP，错误增强组件将作为失败检查详情视图的补充，帮助您更好地理解并解决出现的问题。

设置选项卡

在设置选项卡中，代码作者可以配置个人的编辑器偏好设置，而仓库管理员则可以控制代码仓库的行为与策略。有关设置选项卡的更多信息，请参阅《代码仓库管理》。

设置选项卡中的大多数选项均面向管理员，仅对具有相应权限的用户（默认为代码仓库数据主人）可用。
