Azure DevOps

Azure DevOps 连接器是用于 Azure DevOps 的 Palantir 提供的驱动程序。

要创建新的 Azure DevOps 源，请按照 Palantir 提供的驱动程序的标准设置流程操作，然后使用以下章节进行 Azure DevOps 特定的配置和网络设置。有关完整的属性参考，请参阅 官方 Azure DevOps 驱动程序文档 ↗。

配置

以下属性为必填或推荐项。

属性是否必填？描述默认值
AzureDevOpsEdition ↗必填正在使用的 AzureDevOps 版本。设置为 [AzureDevOps Online] 或 [AzureDevOps OnPremise]。AzureDevOps Online
Organization ↗必填组织或集合的名称，具体取决于 AzureDevOpsEdition 的值。—
AuthScheme ↗推荐连接到 Azure DevOps 时使用的身份验证类型。Azure DevOps OnPremise 连接仅支持基本身份验证。Basic
Catalog ↗推荐指定此属性以连接到特定的数据目录。—
IncludeCustomFields ↗推荐一个布尔值，指示是否在列列表中包含自定义字段。TRUE
InitiateOAuth ↗推荐指定获取或刷新 OAuth 访问令牌的流程，以便在经过身份验证和授权的用户工作期间保持其访问权限。REFRESH
OAuthClientId ↗推荐指定创建自定义 OAuth 应用程序时分配的客户端 ID。（也称为使用者密钥）。此 ID 将自定义应用程序注册到 OAuth 授权服务器。—
OAuthClientSecret ↗推荐指定创建自定义 OAuth 应用程序时分配的客户端密钥。（也称为使用者密钥）。此密钥将自定义应用程序注册到 OAuth 授权服务器。—
PersonalAccessToken ↗推荐用于访问组织内数据的个人访问令牌。—
Schema ↗推荐指定此属性以连接到特定的架构。REST

网络

下表列出了源为成功运行而必须能够访问的域。

对于每个域，添加相应的 出站策略。如果源托管在本地且无法从 Foundry 直接访问，请改用 智能体代理出站策略；智能体主机本身也必须能够访问所列出的域。有关详细信息，请参阅 使用智能体作为代理。

域是否需要
dev.azure.com如果 Schema=REST（默认）且 AzureDevOpsEdition='AzureDevOps Online'（默认）
analytics.dev.azure.com如果 Schema=Analytics 且 AzureDevOpsEdition='AzureDevOps Online'（默认）
<URL>如果 AzureDevOpsEdition='AzureDevOps OnPremise'
login.microsoftonline.com如果 AuthScheme=AzureAD（默认）且 AzureEnvironment=GLOBAL（默认）
login.chinacloudapi.cn如果 AuthScheme=AzureAD（默认）且 AzureEnvironment=CHINA
login.microsoftonline.us如果 AuthScheme=AzureAD（默认）且 AzureEnvironment=USGOVT 或 USGOVTDOD
