入门

本页将引导您创建第一个使用Pilot的应用程序。在本指南结束时，您将拥有一个运行中的应用程序，其中包括生成的本体论实体、设计规范、前端代码和种子数据，并在实时预览中运行。

打开 Pilot

要打开 Pilot，请从 Foundry 导航侧边栏中的应用程序菜单中选择 Pilot。如果您在应用程序菜单中没有看到 Pilot，请确认您的注册是否启用了 Pilot，并且您是否有访问具有本体论写入权限的项目的权限。

Pilot 欢迎屏幕显示了一个提示输入框、一个 保存位置 选择器以及最近应用程序的列表。

创建您的第一个应用程序

  1. 在提示输入中描述您要构建的应用程序。例如：Build a task management application for tracking team projects with priorities and deadlines。
  2. （可选）选择添加以附加额外的上下文，比如现有的对象类型、操作类型或参考文档。更多详情，请参见提供上下文和附件。
  3. 确认保存位置，这决定了Pilot将使用现有实体或创建新实体的本体论（ontology）。
  4. 选择创建应用程序。

Pilot会创建一个隔离的容器，在此容器中所有代码和本体论（ontology）的修改都会安全进行，直到您显式地部署该应用程序。

等待您的容器

选择创建应用程序后 ，Pilot 为您的应用程序提供了一个容器。此过程可能需要几分钟。您可以通过工作区右上角的状态指示器来监控进度。有关状态监控的更多详细信息，请参阅 The Pilot workspace。

探索工作区

当容器准备就绪后，Pilot 工作区将打开。工作区有两个主要区域：

  • 聊天面板（左）：一个聊天界面，您可以与 Pilot 互动以优化您的应用程序。聊天面板包括提示输入、消息历史记录和上下文附件。
  • 应用程序视图（中心）：四个选项卡，用于查看您应用程序的不同方面：
    • 预览：您生成的应用程序的实时预览。
    • 本体论（Ontology）：一个交互式图表，显示由 Pilot 创建的对象类型、操作类型和关系。
    • 代码（Code）：嵌入式编辑器中的完整应用程序源代码。
    • Pilot 日志（Pilot Logs）：用于调试的容器日志。

有关每个工作区区域的详细说明，请参阅 Pilot 工作区。

通过后续提示进行优化

在 Pilot 完成初始构建后，您可以在聊天面板中输入后续提示继续迭代。例如：

  • 添加新功能: Add a dashboard that shows project completion statistics
  • 更改设计: Use a dark color scheme with blue accents
  • 修改本体论（Ontology）: Add a priority field to the task object type with values High, Medium, and Low

Pilot 会处理您的请求，并更新应用程序的相关部分。当代理完成时，更改将立即出现在实时预览中。
