入门

本页将引导您创建第一个使用Pilot的应用程序。在本指南结束时，您将拥有一个具有生成的本体论实体、设计规范、前端代码和种子数据的运行中的应用程序，并且可以实时预览。

打开 Pilot

要打开 Pilot，请从 Foundry 导航侧边栏中的应用程序菜单中选择 Pilot。如果您在应用程序菜单中看不到 Pilot，请确认您的注册是否已启用 Pilot，并且您是否有访问具有本体论写入权限的项目的权限。

Pilot 欢迎屏幕显示一个提示输入框、一个 保存位置 选择器以及最近应用程序的列表。

创建您的第一个应用程序

  1. 在提示输入中，描述您想要构建的应用程序。例如： 构建一个用于跟踪团队项目的任务管理应用程序，包含优先级和截止日期。
  2. （可选）选择 添加 以附加额外的上下文，例如现有的对象类型、操作类型或参考文档。更多详情，请参阅 提供上下文和附件。
  3. 确认 保存位置，这将决定 Pilot 将使用现有实体或创建新实体的本体论。
  4. 选择 创建应用程序。

Pilot 会创建一个隔离的容器，所有代码和本体论编辑都会安全地进行，直到您明确部署该应用程序。

等待您的容器

当您选择 创建应用程序 后，Pilot 会为您的应用程序分配一个容器。此过程可能需要几分钟时间。您可以使用工作区右上角的状态指示器来监控进度。有关状态监控的更多细节，请参阅 Pilot 工作区。

探索工作区

容器就绪后，Pilot 工作区将打开。工作区有两个主要区域：

  • 聊天面板（左）：一个聊天界面，您可以在此与 Pilot 交互以完善您的应用。聊天面板包括提示输入、消息历史记录和上下文附件。
  • 应用视图（中）：四个选项卡用于查看您应用的不同方面：
    • 预览：您生成的应用程序的实时预览。
    • 本体论：一个交互式图表，显示由 Pilot 创建的对象类型、操作类型和关系。
    • 代码：嵌入式编辑器中的完整应用程序源代码。
    • Pilot 日志：用于调试的容器日志。

有关每个工作区区域的详细说明，请参阅 Pilot 工作区。

通过后续提示进行优化

在 Pilot 完成初始构建后，您可以通过在聊天面板中输入后续提示继续迭代。例如：

  • 添加新功能：添加一个显示项目完成统计信息的仪表板
  • 更改设计： 使用深色配色方案，搭配蓝色强调色
  • 修改本体论： 在任务对象类型中添加优先级字段，值为高、中和低

Pilot 处理您的请求并更新应用程序的相关部分。当代理完成时，更改会立即出现在实时预览中。
