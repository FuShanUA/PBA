单元测试

本页描述的 Python 仓库单元测试仅适用于批量管道（batch pipeline），不支持流式管道（streaming pipeline）。

Python 仓库可以将测试作为检查的一部分运行。这些测试使用流行的 Python 测试框架 pytest ↗ 运行。

CI 任务：condaPackRun

所有 CI 检查都包含 condaPackRun，以及其他任务。

condaPackRun 任务负责安装环境。每个工件都从适当的通道获取，conda 使用这些工件来构建环境。

此任务包含以下三个阶段：

  1. 下载并提取已解决环境中的所有包。
  2. 验证包内容。根据配置，conda 将使用校验和或验证文件大小是否正确。
  3. 将包链接到环境中。

环境规范存储在下面列出的隐藏文件中，作为后续构建的缓存。

  • conda-version-run.linux-64.lock
  • conda-version-test.linux-64.lock

缓存保存7天。如果< a href="/docs/foundry/transforms-python/use-python-libraries/#the-metayaml-file">meta.yaml 文件发生任何更改，会重新缓存。

此任务在很大程度上依赖于添加到仓库中的包数量。添加的包越多，任务运行越慢。

启用样式检查

可以通过在 Python 项目中的 com.palantir.conda.pep8 和 com.palantir.conda.pylint Gradle 插件中应用，启用 PEP8 和 Pylint 样式检查。对于转换仓库，这可以在 Python 子项目中找到。对于库仓库，这可以在根文件夹中找到。

下面是转换 build.gradle 文件的示例：

Copied!
1apply plugin: 'com.palantir.transforms.lang.python'
2apply plugin: 'com.palantir.transforms.lang.python-defaults'
3
4// Apply the pep8 linting plugin
5apply plugin: 'com.palantir.conda.pep8'
6apply plugin: 'com.palantir.conda.pylint'

Pylint 可以在 Python 项目中的 src/.pylintrc 文件中进行配置。例如，可以按以下方式禁用特定消息：

[MESSAGES CONTROL]
disable =
    missing-module-docstring,
    missing-function-docstring
Pylint 限制

并非所有 Pylint 的配置都能保证在 Foundry 中正常工作。如果 src/.pylintrc 中的某个功能在检查中未显示，则表明该功能不受支持。

编写测试

Pytest 会识别以 test_ 开头或以 _test.py 结尾的 Python 文件中的测试。我们建议将所有测试放在项目 src 目录下的 test 包中。测试是带有 test_ 前缀并使用 Python 的 assert 语句进行断言的 Python 函数。Pytest 还会运行使用 Python 内置 unittest ↗ 模块编写的测试。有关编写测试的详细信息，请参阅 pytest 文档 ↗。

Copied!
1test:

在 /transforms-python/src/pytest.ini 处创建一个 pytest.ini 文件，内容如下：

Copied!

并行化测试

默认情况下，pytest 会按顺序运行测试。为了加快测试运行速度，可以将测试发送到多个 CPU。

编辑 transforms build.gradle 文件，并将 numProcesses 设置为应使用的进程数。

Copied!
1apply plugin: 'com.palantir.transforms.lang.python'
2apply plugin: 'com.palantir.transforms.lang.python-defaults'
3
4// Apply the testing plugin
5apply plugin: 'com.palantir.transforms.lang.pytest-defaults'
6
7tasks.pytest {
8    numProcesses "3"
9}

测试并行化使用 pytest-xdist ↗ 测试插件进行。

并行化测试涉及将待测试项发送到任何可用的工作程序，不保证顺序。任何需要全局/共享状态且预期其他前置测试会做出更改的测试应相应调整。
