JupyterLab®

Code Workspaces 允许您在 Foundry 中使用 JupyterLab® ↗。Code Workspaces 中的 JupyterLab® 支持以下操作：

  • 读取和写入表格数据集、Iceberg 表 ↗ 以及虚拟表 ↗。
  • 从非结构化数据集下载或上传文件。
  • 从 conda 或 PyPI 导入 Python 库。
  • 预览 Dash、Streamlit 或 TensorBoard™ 应用程序，以加快迭代速度。
  • 发布 Dash 或 Streamlit 应用程序。
  • 与本体进行交互。

目前，Code Workspaces 的 Python 应用程序支持 Dash ↗ 和 Streamlit ↗。用户可以直接在 Code Workspaces 中创建应用程序，且内置了 Foundry 的版本控制 ↗、分支 ↗ 以及数据治理 ↗ 等功能。

Code Workspace 应用程序支持分支管理。如果您创建了新的工作区分支 ↗，发布新应用程序或同步更改将会在该分支上发布该应用程序的新版本。这样您就可以在将应用程序暴露给用户之前先进行预览。若要在 master 分支上发布，只需将您的分支合并到 master 即可。

Dash 应用程序

已发布的 Dash 应用程序需遵守 30 秒超时限制，这意味着 Dash 服务器必须在执行 Dash 应用程序 Python 文件后的 30 秒内启动。否则，您的 Dash 应用程序将无法启动。

若要创建 Dash ↗ 应用程序，请按照以下说明操作：

  1. 在 JupyterLab® 工作区中打开应用程序选项卡。
  2. 选择发布应用程序，在您的文件和项目中为新应用程序选择一个位置，并提供模块名称和变量名称。系统会在与所提供模块名称对应的路径下自动创建一个文件。
  3. 可选：配置高级设置。默认情况下，应用程序设置将与 Jupyter® 工作区的设置保持一致。
  4. 选择发布并同步以注册新的 Dash 应用程序，并将代码同步到后端代码仓库。待 CI 检查和发布完成后，您可以在应用程序面板中选择链接，查看已发布的应用程序。

Dash 应用程序的模块名称应输入为相对于仓库根目录的点分路径（例如 app，或者如果您的模块位于某个文件夹内，则为 folder.app）。变量名称是模块文件中为 Dash 应用服务器声明的 Python 变量，通常命名为 server。

若要在本地针对现有 Dash 应用程序进行开发：

  1. 打开终端窗口并执行您新创建的文件，例如 python app.py。
  2. 在应用程序选项卡的 Dash 部分选择预览，以预览该应用程序。

您可以利用 Jupyter® Notebooks 中的托管 conda/PyPI 环境 ↗，确保 JupyterLab® 工作区中的 conda/PyPI 环境在已发布的 Dash 应用程序中自动还原。

Streamlit 应用程序

已发布的 Streamlit 应用程序需遵守 30 秒超时限制，这意味着 Streamlit 服务器必须在执行 Streamlit 应用程序 Python 文件后的 30 秒内启动。否则，您发布的 Streamlit 应用程序将无法启动。

若要创建 Streamlit ↗ 应用程序，请按照以下说明操作：

  1. 选择发布应用程序，在您的文件和项目中为新应用程序选择一个位置，并输入相对于仓库根目录的 Python 文件名称，例如 app.py。
  2. 可选：配置高级设置。默认情况下，应用程序设置将与 JupyterLab® 工作区的设置保持一致。
  3. 选择发布并同步以注册新的 Streamlit 应用程序，并将代码同步到后端代码仓库。待 CI 检查和发布完成后，您可以在应用程序面板中选择链接，查看已发布的应用程序。

若要在本地针对现有 Streamlit 应用程序进行开发：

  1. 打开终端窗口，使用 streamlit run app-name.py 通过 Streamlit 运行代码。
  2. 在 JupyterLab® 代码工作区中打开应用程序选项卡，并在 Streamlit 部分选择预览，以预览您在本地开发的应用程序。

借助《Jupyter® Notebooks 中的托管 conda/PyPI 环境》，您可以确保 JupyterLab® 工作区中的 conda/PyPI 环境在已发布的 Streamlit 应用中自动恢复。

TensorBoard™

您可以直接在代码工作区中预览《TensorBoard™ ↗》。操作方法是：在终端运行 tensorboard --logdir=<$PATH_TO_LOGS_DIRECTORY>，填入包含 TensorFlow 日志的目录路径。然后，在 Jupyter® 工作区打开“应用程序”选项卡，并在 TensorBoard™ 部分选择预览。

Jupyter® Notebook 数据转换

Jupyter Notebook 数据转换目前不支持对外部服务发起 API 调用，即使代码工作区中已添加相关网络策略也不例外。若要运行包含外部系统 API 调用的可重现管道，您应改用《外部数据转换》。

代码工作区能够将 Jupyter® notebooks 发布为数据转换管道，并为这些 notebook 生成的输出数据集注册转换 JobSpecs。

请按照以下步骤为输出数据集注册数据转换：

  1. 将您的 Jupyter® notebook 保存为工作区中的文件。

  2. 在数据面板中，选择数据集旁边的蓝色构建图标。

  1. 按照界面提示配置数据转换。代码工作区会自动推断所使用的 Jupyter® notebook 以及为生成输出数据集而添加的输入项。如果推断有误，您可以手动编辑输入与输出。
  1. 选择同步并运行，以保存数据转换配置，并针对输出数据集触发 CI 检查，从而发布数据转换和 JobSpecs。

CI 检查完成后，您的数据转换即可进行构建。您可以使用 Foundry 的其他《数据集成》工具来管理数据转换及其所属的《数据管道》。

数据转换不会自动使用代码工作区的环境。您需要在脚本文件的顶部安装运行时所需的所有库。安装库的代码片段可在库面板中找到。 例如，若要在数据转换中使用 scikit-learn 和 scikit-image，请将此代码片段添加到文件顶部：

Jupyter® Notebooks 中的托管 conda/PyPI 环境

借助由 Maestro 支持的托管环境功能，可以跨 Jupyter® 工作区、应用程序、数据转换和模型输出一致地恢复 Jupyter® conda/PyPI 环境。Maestro 是一款 Foundry 库管理工具，可用于为 Jupyter 工作区安装和管理库。它与 conda 和 pip 集成，简化了库的安装过程，并有助于维持环境的一致性。要将命令转换为 Maestro 格式，请参阅下方关于《conda》或《PyPI》的说明。

托管环境在 Code Workspace 工作流中保持一致，无需在每次重启工作区、启动应用程序或运行 Jupyter® Notebook 转换时重复执行相同的 mamba 和 pip 命令。借助托管环境，工作环境中已安装的库会通过 /home/user/repo 文件夹中的文件进行追踪，从而在各工作流之间将环境恢复至原始状态。

以下两个文件用于追踪您的托管环境：

  1. /home/user/repo/.envs/maestro/meta.yaml：即环境清单，其中包含您的环境所请求的库及版本集合。了解有关 meta.yaml 文件的更多信息。
  2. /home/user/repo/.envs/maestro/hawk.lock：即环境锁定文件，其中包含根据清单中指定的约束条件求解得出的、环境已解析的库及版本集合。hawk.lock 文件取代了已弃用的 conda.lock。了解有关 hawk.lock 文件的更多信息。

不建议修改 hawk.lock 锁定文件，而应依赖安装命令代为更新这些文件。除非发生 git 冲突，否则您也无需修改 meta.yaml 文件；在交互式安装库时，它会自动更新。

您的托管环境安装在 /home/user/envs/default 文件夹中。此环境将在以下阶段之前进行恢复：

  • 启动您的 Jupyter® 工作区
  • 启动由您的工作区生成的应用程序
  • 运行由您的工作区生成的 Jupyter® Notebook 转换
  • 运行模型

随后，转至 .envs/maestro 文件夹以查看 meta.yaml 和 hawk.lock 文件。

5. 根据 hawk.lock 文件的更改同步托管环境

检出新分支或拉取新的 git 更改后，hawk.lock 文件的内容可能会发生变化。

若要同步您的托管环境，请在工作区终端中执行以下命令：



已复制！
1maestro env install

maestro 是用于更新受管环境的命令行界面。上述命令会根据清单文件重新解析锁定文件，并将该环境安装至 /home/user/envs/default 文件夹。

6. 解决环境文件的 Git 冲突

要解决受管环境中的 Git 冲突，首先请删除 hawk.lock 文件。接着前往 meta.yaml 文件解决各项冲突，随后合并你的分支。

合并完成后，在终端中运行 maestro env install 命令，以重新生成 hawk.lock 文件并重新安装受管环境。

7. 升级受管环境

要更新受管环境，请先移除 hawk.lock 文件。随后删除你的 /home/user/envs/default 文件夹（rm -rf /home/user/envs/default）。最后在终端运行 maestro env install 命令以重新生成 hawk.lock 文件。环境将会以升级后的库版本完成重新安装。
