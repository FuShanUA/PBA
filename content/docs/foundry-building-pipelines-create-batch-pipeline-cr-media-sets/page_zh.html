使用代码仓库创建媒体集批处理流水线

本指南将引导您使用代码仓库应用完成一个简单的转换示例。您将学习如何针对媒体集编写和编辑 Python 代码。

1. 创建代码仓库

首先创建一个新仓库。为此，请导航至 Foundry 中的项目，在右上角选择+ 新建，然后选择代码仓库。

在本指南中，我们将编写一个 Python 转换。为您的仓库命名，并在语言模板下的下拉菜单中选择Python。然后，选择初始化仓库。

2. 导入您的数据

如果您已经导入了要使用的原始媒体集，可以直接进入下一步。否则，请参阅有关如何将媒体导入 Foundry的媒体集指南。

3. 导入 transforms-media 库

要使用媒体集转换，必须导入 transforms-media 库。

从左侧边栏导航至库选项，然后搜索并选择 transforms-media 库。接着，选择添加并安装库。

安装完成后，状态应显示为库已安装。

4. 编辑代码

创建新文件

创建 Python 转换仓库时，系统会自动生成一个 examples.py 文件，您可以将其作为基础使用。您可以通过选择省略号图标、选择重命名来为文件重命名。

如果想创建新文件，请选择省略号图标，并在数据集目录下选择新建文件。为确保本工作流的顺利进行，请确保您创建的是 Python 文件。

将正确的库导入到您的文件中

在处理媒体集时，必须使用 @transform 装饰器。媒体集的输入和输出必须使用 transforms.mediasets.MediaSetInput 和 transforms.mediasets.MediaSetOutput 规范进行传递：



已复制！
1from transforms.api import transform
2from transforms.mediasets import MediaSetInput, MediaSetOutput
3@transform(
4    images=MediaSetInput('/examples/images'),
5    ...
6)

确保输出文件存在

如果要创建媒体集输出文件，请务必在转换装饰器中引用它之前先创建该文件。

您可以导航至您的项目文件夹（即您最初创建 Python 转换代码仓库的位置），并从下拉菜单中选择 新建 > 媒体集 来创建此文件。

使用 get_media_item()、get_media_item_by_path() 或 list_media_items_by_path_with_media_reference()

要对媒体集中的每个媒体项进行转换，您必须在转换过程中使用 get_media_item() 或 get_media_item_by_path() 来引用这些媒体项。

请注意，调用 get_media_item() 和 get_media_item_by_path() 会返回一个 Python 类文件流对象。系统支持 io.open() ↗ 接受的所有选项。各项将以流的形式读取，这意味着不支持随机访问。

您可以按如下方式引用单个媒体项：



已复制！
1from transforms.api import transform
2from transforms.mediasets import MediaSetInput, MediaSetOutput
3@transform(
4    images=MediaSetInput('/examples/images'),
5    output_images=MediaSetOutput('/examples/output_images')
6)
7def translate_images(images, output_images):
8    image1 = images.get_media_item_by_path("image1") # 使用路径
9    image2 = images.get_media_item("ri.mio.main.media-item.123") # 使用 RID

不过，您可能希望将媒体集中的所有媒体项加载到 DataFrame 中进行列举。此时必须使用 list_media_items_by_path_with_media_reference() 来列出这些媒体项：



已复制！
1from transforms.api import transform, Output
2from transforms.mediasets import MediaSetInput
3@transform(
4    images=MediaSetInput('/examples/images'),
5    listing_output=Output('/examples/listed_images')
6)
7def translate_images(ctx, images, listing_output):
8    media_items_listing = images.list_media_items_by_path_with_media_reference(ctx)
9
10    # You can perform any PySpark function with the listed output

该列表将具有以下架构：

+--------------------------+-----------+-------------------+
|        mediaItemRid      |    path   |  mediaReference  |
+--------------------------+-----------+-------------------+
| ri.mio.main.media-item.1 | item1.jpg |  {{reference1}}   |
| ri.mio.main.media-item.2 | item2.jpg |  {{reference2}}   |
| ri.mio.main.media-item.3 | item3.jpg |  {{reference3}}   |
+--------------------------+-----------+-------------------+

示例：将 PDF 媒体集转换为新的 PNG 媒体集

在此示例中，我们将遍历 PDF 媒体集中的所有媒体项，并将它们保存为 PNG 图像。

输入示例：

请注意，list_media_items_by_path_with_media_reference() 返回一个数据框（dataframe），这允许您对输出执行 PySpark 函数。在此示例中，我们将收集 mediaItemRid 列的值，并将其作为数组进行遍历：



已复制！
1from transforms.api import transform
2from transforms.mediasets import MediaSetInput, MediaSetOutput
3
4
5@transform(
6    pdfs=MediaSetInput('INPUT_MEDIA_SET_PATH'),
7    output_pngs=MediaSetOutput('OUTPUT_MEDIA_SET_PATH')
8)
9def upload_images(ctx, pdfs, output_images):
10    media_items_listing = pdfs.list_media_items_by_path_with_media_reference(ctx)
11    rid_list = media_items_listing.rdd.map(lambda x: x.mediaItemRid).collect()
12
13    for rid in rid_list:
14        output_png = pdfs.transform_document_to_png(rid, 0)
15        output_images.put_media_item(output_png, "PNG_"+rid)

上述操作将生成一个包含 PNG 文件的输出媒体集（如元数据属性所示）：

4. 测试您的更改

在 Foundry 中，媒体集可以像代码仓库一样进行分支。分支对于测试多步骤数据管道的设计非常有用。例如，您可以独立测试对数据管道某些部分的更改，而不会破坏不依赖您分支的用户的下游依赖项。

编写完数据转换代码后，您应该测试所做的更改。测试可确保您的代码在合并更改前按预期工作。

5. 提交您的更改

现在，您可以在代码仓库中提交更改，为您所做的工作打上标签。在提交之前，您的工作默认会自动保存。当您到达一个停止点时，提交会专门为您这组更改打上标签。

选择提交按钮可提交您的更改并对代码运行自动检查。选择构建按钮也会提交更改、运行自动代码检查，并开始构建您的输出数据集。如果要在不构建数据集的情况下快速测试更改以确保代码通过代码检查，请选择提交。否则，您可以直接跳至合并并构建您的更改。

6. 合并并构建您的更改

对新媒体集输出感到满意后，请与所有协作者合作，合并、构建并查看更改。
