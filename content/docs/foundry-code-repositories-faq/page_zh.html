代码仓库常见问题解答

以下是关于代码仓库的一些常见问题。

有关一般信息，请查阅《代码仓库文档》。

  • 是否可以在不要求打标签的情况下，从分支的最新提交发布 Python 包？

  • 代码在代码工作簿中运行，但在代码仓库中无法运行
  • 在先前能正常使用该 Python 库的代码仓库中，检查因缺少该库而失败

是否可以在不要求打标签的情况下，从分支的最新提交发布 Python 包？

可以。您可以通过修改包根目录下的 build.gradle 文件来发布分支，从而发布 Python 包的最新提交。例如，要发布 master 分支上的最新提交，请按如下方式修改 build.gradle 文件：

Copied!
1condaLibraryPublish.onlyIf { versionDetails().branchName == "master" }

如何恢复代码仓库中之前删除的代码？

如果这些转换已构建到数据集中，您可以使用生成数据集预览的比较功能来查看当时的代码。然后，您可以复制粘贴相关的转换。或者，您可以导航到代码仓库的分支，打开特定分支，并在那里查看完整的更改历史记录。

能否复制我的代码仓库？

平台没有内置复制代码仓库的功能。但是，您可以将仓库克隆到本地计算机，然后将该代码推送到新的代码仓库。如果这样做，请记住将所有输入作为引用添加到新项目中。《了解如何克隆仓库》。

如何判断是否需要升级我的分支？

您可以通过选择代码仓库右上角的 ... 并确认是否出现升级选项，来验证代码仓库是否为最新状态。如果没有升级选项，则说明该仓库已是最新版本。

每次构建开始时，转换能否动态选择输入和输出？

不支持此操作。每当在代码仓库中添加新提交时，持续集成 (CI) 检查会定义输入和输出集合。

在代码预览中代码运行正常，但构建失败

代码预览成功，但构建失败。代码预览在部分数据子集上运行，这通常意味着完整构建运行时，子集中未包含的数据值会导致代码报错。

要进行排查，请执行以下步骤：

  1. 检查失败构建的错误信息。查找以“Caused by”开头的行并仔细阅读。有时它们会明确指出转换文件中的具体代码行。
  2. 尝试升级代码仓库。为此，在仓库右上角选择 ... > Upgrade。这将创建一个用于升级分支的 PR。请确保合并该升级 PR，以便分支切实完成升级。你可以升级任何分支，但将升级提交合并到受保护分支需要经过审查和批准。

代码在代码工作簿中运行正常，但在代码仓库中无法运行

有时，将代码从代码工作簿移植到代码仓库时，如果不修改代码以使其适应代码仓库环境，将无法正常运行。

要进行排查，请执行以下步骤：

  1. 是否已验证转换装饰器正确无误？即 @transform 还是 @transform_df？
  2. 是否已验证所有输入均已声明并作为参数传递给计算函数？
  3. 数据框名称是否与工作簿中的一致？代码仓库是否切实返回了该数据框？
  4. 检查代码工作簿中使用的库在代码仓库中是否也可用。
  5. 验证代码工作簿单元格的输入和分支与代码仓库中使用的数据集是否完全一致。

查阅我们关于构建与检查错误的常见问题解答以获取更多详细信息。

因缺失 Python 库导致检查失败，而该库此前在代码仓库中运行正常

有时，原本正常运行的仓库会开始出现问题：仓库检查报错，提示无法获取 Conda 包。这可能是 PackageNotFoundError，也可能是由于 Conda 缓存损坏导致的 MD5MismatchError。

要进行排查，请执行以下步骤：

  1. 大多数情况下，只需在代码仓库中创建新的提交即可清除 Conda 缓存。打开仓库并进行任意更改（即使是新增一个空行），然后点击 Commit。
  2. 如果执行空行提交后仍有相同症状，可能是 Conda 缓存导致了上层缓存损坏。以下步骤将清理代码仓库的 Gradle 缓存。
  3. 在代码仓库中点击设置（齿轮）图标，启用 Hidden files。
  4. 在 Python 子项目中找到 conda-versions.run.linux-64.lock，将其删除，然后点击 Commit。
  5. 如果清理这两项缓存后仍有相同症状，请联系 Palantir 支持团队。
