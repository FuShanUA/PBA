故障排查参考

本页面介绍了常见的调度问题及其排查步骤。

计划构建问题

调度指标页面

排查调度问题的最佳方法之一是查看调度指标页面。该指标页面可以告诉你故障的根源。常见的故障包括：

  • 计划构建失败。你会在“运行历史”选项卡中看到构建失败的证据。选择这些构建将跳转至《构建》应用中的构建报告以查看完整日志。
  • 计划构建被忽略。对于任何已触发但未构建的构建，“运行历史”选项卡的状态列将显示 Ignored。
  • 调度在预期的时间或节奏下未被触发。在这种情况下，当预期应触发构建时，“运行历史”选项卡可能不会显示构建已被触发。

另一个有用的选项卡是“版本”选项卡，它显示了过去的调度版本和编辑记录。如果调度突然开始出现与预期不同的行为，请检查调度版本中是否有与此变化相符的改动，并考虑将调度恢复到之前正常工作的状态。

计划构建失败

你可以通过查看调度指标页面上的“运行历史”选项卡，来验证调度是否在预期的时间被触发。

如果调度已触发，但构建随后失败，你可以在《构建》应用中像调试任何其他构建一样调试此构建。

如果未设置适当的权限，调度也将无法构建。调度的权限取决于调度所处的令牌模式。了解更多关于项目范围调度的信息。

计划构建被忽略

你可以通过查看调度指标页面上的“运行历史”选项卡，来验证调度是否在预期的时间被触发。此信息通常会给出调度被忽略的原因。

所有数据集均为最新

如果所有目标数据集均为最新（即它们在上次构建后输入未更新），调度运行将被忽略。如果属于这种情况，你会在“运行历史”选项卡中看到此原因。在调度编辑器中，导航至调度列表。随后，你可以选择按 Out-of-date 对数据血缘关系图进行着色，这将让你概览哪些作业规格被视为过期。

在特殊情况下，可以使用高级设置中的“强制构建”选项覆盖此行为，但在这些情况之外使用此选项会造成计算资源的浪费。如果任何目标数据集由 Object Storage V1 (Phonograph) 同步、带有 API 调用的转换或数据连接同步构建，它们可能不会显示为过期，并且可能需要启用“强制构建”选项才能运行调度。

调度构建部分作业规格

如果调度仅触发了部分作业规格，你会在调度指标页面的“运行历史”选项卡中看到相关证据。

导致此行为的原因之一是仅有过期的作业规格。调度将仅构建过期的作业规格，而最新的那些将在构建期间被忽略。有关更多排查详情，请参阅所有数据集均为最新。如果所有这些作业规格均为最新，则会出现构建状态为 Ignored 的情况。

另一个原因可能是作业规范未包含在构建的作业规范图中。在调度编辑器中选中特定调度时，待构建的作业规范会在数据血缘关系图中高亮显示。作业规范的选择取决于构建类型。如果使用连接构建，请务必确认对于在多个分支上使用相同数据集的调度，是否已配置相应的连接作业规范。

调度未触发

可以通过查看调度指标页面上的运行历史选项卡，来核实调度是否在预期时间触发。常见的排查步骤如下：

  • 检查调度是否已暂停。已暂停的调度在恢复前不会触发。
  • 检查调度触发器配置。如果该调度此前曾成功触发，请查看调度历史记录，确认触发器近期是否有变更。
  • 如果调度使用事件触发器，请确认预期事件是否确实发生。
    • 例如，如果构建应在输入更新时触发，请检查输入端的上一次构建是否成功运行，以及该构建的事务是否已在数据集预览历史视图中成功提交。
    • 如果构建应在多个输入更新后触发，请检查所有输入的构建及时间情况。例如，假设某调度包含输入触发器 A1 和 A2，并且开启了“等待所有这些数据集更新”选项。假设该调度上次在 T1 时间运行。若要使其在 T2 时间再次运行，A1 和 A2 必须在 (T1, T2) 时间段内都发生更新。



调度重试与配置不符

并非所有类型的失败都可以重试。调度运行时的重试次数上限为您所在组织的管理员所配置的最大值。有关高级调度设置的更多信息，请查阅相应文档。

可重试的错误代码包括：

  • INTERNAL
  • TIMEOUT
  • CUSTOM_SERVER
  • FAILED_PRECONDITION

调度失败并提示 JobSpecInputsTrashed 或 JobSpecOutputsTrashed，或数据血缘关系警告部分数据集已被废弃

此错误表明调度包含或读取了已废弃的资源。您可以采取以下任一措施予以解决：

  • 从回收站中还原已删除的数据集。
  • 将该已删除的数据集从调度中排除。如果该数据集在调度中作为另一个下游数据集的输入，您还需要执行以下操作之一：
    • 将该下游数据集连同已废弃的数据集一并排除。
    • 修改下游数据集的逻辑，使其不再将该已废弃的数据集作为输入。

调度编辑器问题

调度权限

要在项目级模式下编辑调度，您必须对目标数据集拥有编辑者权限，对触发器数据集拥有查看者权限，并且对该调度所属的项目拥有编辑者权限。如果您失去了某个数据集的权限，请在保存更改前将该数据集从调度中移除。

要编辑、删除或暂停调度，您需要对目标数据集拥有编辑者权限，并对该调度所属的项目拥有编辑者权限。要查看调度，您需要对目标数据集拥有查看者权限。

调度计算信息不可用

查看调度时，可能会因以下任一原因出现“并非所有计算信息均可用。实际计算使用量可能高于所示数值”的提示信息：

  • 一个或多个计划构建仍在进行中，总计算使用量尚未最终确定。
  • 至少有一个计划内构建所使用的资源信息缺失。

显示此消息时，无需用户进行任何操作。该消息旨在提示，在特定时间点所显示的总计算用量可能不准确。
