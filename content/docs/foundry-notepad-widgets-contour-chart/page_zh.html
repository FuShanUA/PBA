Contour 图表

来自 Contour 分析的图表和表格可以通过 Contour chart 部分进行嵌入。你可以通过点击 + Add Widget 或在段落字段中输入 / 来打开控件插入菜单，或者直接通过 Contour 分析本身上的 Copy for Notepad 按钮来添加控件。

需要注意的是，当你打开包含 Contour board 的 Notepad 文档时，Notepad 会更新源数据。

Contour 控件的交互式锁定

Contour 控件上的锁定数据功能与其他控件的运作方式不同。应用该功能后，它会同时锁定 Contour 控件的可视化和数据，同时仍允许用户交互。相比之下，锁定其他控件会生成一个不可交互的控件快照。如果支持的事务被手动删除或被保留策略清除，交互式锁定的控件将不会加载锁定的可视化内容。如果 Contour 控件配置了 Notepad 模板输入参数，则不会被锁定。

有关更多信息，请参阅关于锁定数据功能的文档。

控件属性

  • Path： 从选定的 Contour 分析中指定一个路径。
  • Board： 定义从选定路径中应显示哪个 board。
  • Parameters：（视情况而定）只有在源 Contour 分析定义并使用了参数时，才会显示参数配置。该配置允许单独覆盖这些参数变量值。如果未指定值，则将使用 Contour 分析中的默认参数值。

模板配置

Parameter overrides：（可选）你可以指定 Notepad 模板输入参数来覆盖 Contour 参数。这些值将传递给嵌入的 Contour 图表，允许你自定义图表的外部参数。要在 Notepad 中启用此配置选项，请确保 Contour 图表已设置为使用 Contour 参数。如果图表未使用 Contour 参数，则此配置选项将不可用。

在 Notepad 模板中，有两种编辑模式可使用模板输入参数覆盖 Contour 参数：Use form 和 Use JSON。

Use form

这是覆盖 Contour 参数的默认模式。你可以将单个 Contour 参数映射到 Notepad 参数。有两种方法可以输入覆盖值：

  • 静态输入值： 直接在字段中输入值。这些值对于每个生成的 Notepad 文档保持静态。
  • Notepad 模板输入参数： 使用与 Contour 参数类型相匹配的 Notepad 模板输入参数。

注意： Contour 参数可能配置为允许多个值。由于 Notepad 目前不支持数组参数，你必须使用 Use JSON 编辑模式来覆盖这些参数。在 Use form 模式下，这些值可以被单个值覆盖，但无法配置模板输入参数。

Use JSON

在此模式下，你可以为 Contour 参数及其关联的覆盖值指定一个 JSON 字符串。要切换到此模式，请选择 Use JSON。覆盖值将传递给嵌入的 Contour 图表以覆盖其外部参数。

以下是提供所需 JSON 字符串的示例：

Contour 输入类型Contour 参数名称示例JSON 字符串示例
Stringcarrier_code{"carrier_code": "OO"}
Array of Stringcarrier_codes{"carrier_codes": ["OO", "MQ"]}
Numberbooking_class{"booking_class": 2}
Array of Numberbooking_classes{"booking_classes": [1,2]}
Datedeparture_date{"departure_date": "2022-09-01T00:00:00+02:00"}

你可以将多个值合并为一个 JSON 字符串，如下所示：

{"carrier_code": "OO", "booking_class": 2}
