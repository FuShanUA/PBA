VS Code 工作区

VS Code 工作区及 Palantir 的 Visual Studio Code 扩展与 Microsoft 无任何隶属关系，也未获其认可。

只要在启用了 Code Workspaces 的组织中，VS Code 工作区即可默认使用。

VS Code 工作区将微软 VS Code ↗ IDE 的强大功能直接引入 Palantir 平台。VS Code 工作区是部署在 Palantir 基础设施上的开发环境，利用 Code Workspaces 提供集成开发环境（IDE），用于编写生产级代码。无论您是构建 Python 转换、开发 OSDK React 应用程序，还是处理计算模块（在此处查看支持的工作流完整列表），VS Code 工作区都能提供熟悉且强大的环境，并具备自动环境设置以及与 Foundry 资源的无缝集成。

开始使用 VS Code 工作区

访问工作区

  1. 在 VS Code 中打开现有的或创建新的受支持代码库。
  2. 在 Code Repositories 中，您可以点击右上角的在 VS Code 中打开按钮（仅在受支持的代码库中显示）。
  3. 创建新代码库时，选择 VS Code 将自动为您进入 VS Code 工作区。
  4. 您的环境已准备就绪，可以开始编写代码了。

将 VS Code 设为默认编辑器

您可以将 VS Code 设为所有受支持代码库类型的默认代码编辑器，方法是导航至任意代码库的设置选项卡，然后选择默认在 VS Code 工作区中打开代码库。

请注意，部分代码库无论默认设置如何，都会在 VS Code 中打开，因为这些代码库类型已针对 VS Code 工作区进行了优化。

支持的工作流

VS Code 工作区支持以下各节所述的工作流。

Python 转换

Palantir 扩展会在启动时自动设置您的 Python 环境，并提供以下功能：

  • 预览集成，支持无样本和代码定义的过滤选项
  • 调试支持，支持变量检查
  • 从编辑器内部触发构建
  • 通过资源标识符显示资源路径，便于导航
  • 自动设置代码环境
  • 提供常见模式的代码片段

了解有关 Python 转换的更多信息。

OSDK React 应用程序

VS Code 与开发者控制台集成，让您能够快速构建 React 应用程序。您可以在开发者控制台的代码仓库部分创建 VS Code 工作区。

主要功能包括：

  • 通过 OSDK 实现完全的本体集成，以与您特有的本体资源进行交互
  • Git 代码库配置，预配置 OAuth
  • CI 设置，在每次发布时部署您的网站
  • 自动设置 node/npm 环境
  • 具备实时重载功能的开发服务器，可提供即时反馈

了解有关 OSDK React 应用程序的更多信息。

其他支持的工作流

  • 计算模块： 构建具备运行和调试功能的 Python 计算模块。
  • Python 库： 开发并调试可复用的 Python 库。

参考：VS Code 工作区与代码仓库对比

VS Code 和代码仓库的主要用途有所不同：

  • 代码仓库： Palantir 构建的集成开发环境（IDE），专注于满足所有代码管理需求，包括编辑、版本控制、变更管理和持续集成。这是进行拉取请求审查和仓库管理的首选平台工具。

  • VS Code： 部署在 Palantir 基础设施上并可从 Palantir 平台访问的 VS Code 环境。它提供熟悉的 VS Code 编辑体验，支持自动环境配置，并通过《Visual Studio Code 的 Palantir 扩展》与 Foundry 资源集成。

  • Visual Studio Code 的 Palantir 扩展（本地）： 安装在本地 VS Code 应用程序中以直接与 Foundry 集成的扩展。支持在本地计算机上工作时，对 Python 转换进行预览、调试和构建。
