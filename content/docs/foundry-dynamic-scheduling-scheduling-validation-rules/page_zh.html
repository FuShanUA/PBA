验证规则

约束是任何调度工作流的固有部分。从简单的时限限制到频繁变化的规则矩阵，规则的复杂度各不相同。验证规则允许您将这些约束编码，使最终用户能够了解定义您组织运营的限制和限制，从而构建和修改计划。

每个验证规则都由一个 TypeScript 函数支持，该函数评估当前的计划对象状态是否符合函数逻辑中定义的某个条件。

在 Workshop 的调度甘特图小部件的前端，用户可以看到验证规则的结果。在初始加载时，所有规则都会根据本体论的当前状态进行评估。每次对计划进行修改时，规则都会重新评估。这个过程使用户能够了解他们的决策如何符合特定的约束和限制。

实现验证规则

规则直接在调度甘特图小部件中配置，使您能够根据每个使用案例应用规则。要将规则添加到您的调度甘特图小部件：

  1. 导航到相关对象类型的调度数据层。
  2. 向下滚动到规则部分并选择新增条目。
  3. 在规则名称文本框中，提供将在调度甘特图中显示给最终用户的规则描述。
  4. 从下拉菜单中选择一个约束类型。“HARD”将以带有感叹号的红色圆圈表示。“SOFT”将以带有感叹号的橙色三角形表示。
  5. 从下拉菜单中选择规则函数。
  6. 您的函数应具有scheduleObjectPrimaryKeys输入参数。您可以留空此参数，因为该参数将在运行时由小部件自动填充。

函数接口

以下类型表示编写验证规则所需的信息，包括每个对象的规则状态。

4   Rule result is interpreted as follows:
5   true - Rule validated as passing
6   false - Rule validated as not passing

22
23   Additionally, you can provide a set of related puckIds to point users
24   towards why a rule is passing or failing
25*/
26
27interface IRuleResultDetails {
28    description: string;
29    relatedPuckIds: string[];
30}

可以创建更复杂的验证规则。例如，规则可以检查以下内容：

  • 时间表是否按顺序排列（无间隔）
  • 时间表是否有重叠
  • 分配给时间表的人是否具有匹配的技能或认证
