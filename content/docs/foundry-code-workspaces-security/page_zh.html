安全

Code Workspaces 确保连接到 Foundry 的第三方 IDE 遵循 Foundry 的安全与权限模型。在 Code Workspaces 提供的第三方应用之上叠加这层安全机制，可带来诸多益处：

  • Foundry 会全程跟踪加载到 Code Workspaces 中的数据。除受 Foundry 数据治理与访问控制管控的方法外，Code Workspaces 中的数据下载和上传操作均受到限制。
  • 发往 Code Workspaces 的每个请求都会交由 Foundry 的治理框架进行校验。这意味着，一旦撤销用户访问某 Code Workspace 的权限，或撤销其访问工作空间内所导入数据安全标记的权限，该用户将立即失去对该应用的访问权。
  • Foundry 会全程跟踪 Code Workspaces 产出的数据。因此，一旦用户失去对某些基础数据的访问权限，其对这些基础数据所衍生出的输出数据的访问权也将一并受限。
  • 用户之间完全隔离。每位打开特定 Jupyter® 或 RStudio® Code Workspace 的用户都将获得独立的隔离环境。
  • R 和 Python 包仅能从为代码仓库提供支撑的 Foundry Artifacts 通道中加载，借此可对特定 Code Workspace 内可使用的 Conda、PyPI 或 CRAN 包实施管控。
  • 外部 API 调用仅能指向已配置为网络策略并已添加到 Code Workspace 的 URL。

受限输出模式

在处理敏感数据时，受限输出模式可提供更为强化的防护措施。该模式作为一种“只读”模式，会限制写入操作与外部源连接，确保数据无法从代码工作空间中外泄。具体而言，它允许将受限视图载入工作空间。

启用后，受限输出模式会：

  • 阻止写入操作面向所有 Foundry 输出，涵盖数据集、模型与数据表。
  • 停用遥测收集与日志记录，并禁止上传数据检查点，同时保留代码检查点。
  • 禁用已添加至工作空间的网络策略与数据源。这些资源仍留存于工作空间内，但在受限输出模式下将处于非激活状态。

正如代码检查点相关文档所述，您需自行负责确保在受限输出模式下所编写的代码文件不含数据。

如需启用或停用受限输出模式：

  1. 在代码工作空间中打开设置侧边栏。
  2. 找到受限输出模式开关。
  3. 按需开启或关闭该设置。
  4. 保存更改，并在收到提示时重启工作空间。

启用受限输出模式后，工作空间会提示您重启。该设置在后续工作空间重启期间将持续生效，直至您将其明确关闭。
