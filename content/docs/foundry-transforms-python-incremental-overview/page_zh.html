增量转换

增量计算是一种对输入数据集执行转换以生成输出数据集的高效方法。通过利用转换的构建历史，增量计算避免了每次运行转换时都需要重新计算整个输出数据集的需求。请注意，输入数据集和输出数据集不能相同；输入和输出数据集必须不同，以避免产生循环（环形）依赖。

有关如何创建和管理增量流水线的端到端指南，请参阅《构建流水线》部分。

示例：非增量与增量转换

在本节中，我们将通过先分析一个未使用增量转换的代码示例，来探讨增量转换的优势：

如果向 /examples/students_hair_eye_color 输入数据集中添加了任何新数据，filter() 会对整个输入执行过滤操作，而不仅仅是针对新增的数据。这种做法既浪费计算资源，又浪费时间。

如果转换逻辑能够感知其自身的构建历史，就能在计算输出时更加智能。具体而言，它可以利用输入数据的变更来修改输出数据集。这种在重新物化表时使用已有物化数据的过程，被称为增量计算。如果没有增量计算，输出数据集将始终被最新的转换输出结果所替换。

回到上面展示的转换示例。该转换对 students 数据集执行 filter()，从而筛选出棕色头发的学生。引入增量计算后，如果向 students 中追加了关于两名新学生的数据，该转换就可以利用其构建历史信息，仅将新增的棕色头发学生追加到输出中：

+---+-----+-----+------+                  +---+-----+-----+------+
| id| hair|  eye|   sex|                  | id| hair|  eye|   sex|
+---+-----+-----+------+     Build 1      +---+-----+-----+------+
| 17|Black|Green|Female|    --------->    | 18|Brown|Green|Female|
| 18|Brown|Green|Female|                  +---+-----+-----+------+
| 19|  Red|Black|Female|
+---+-----+-----+------+
        ...                                         ...
+---+-----+-----+------+     Build 2      +---+-----+-----+------+
| 20|Brown|Amber|Female|    --------->    | 20|Brown|Amber|Female|
| 21|Black|Blue |Male  |                  +---+-----+-----+------+
+---+-----+-----+------+

因此，可以使用增量逻辑重写上述转换示例，语法如下：

2from transforms.api import transform, incremental, Input, Output
3
4@incremental()
5@transform.using(

7    processed=Output('/examples/hair_eye_color_processed')
8)
9def filter_hair_color(students, processed):
10    students_df = students.polars('added')
11    processed.write_table(students_df.filter(pl.col('hair') == 'Brown'))

流式数据集增量转换中的空输入

对流式数据集执行增量轻量级转换时，增量窗口内可能不包含任何需要处理的新文件。此时，对空输入调用 .polars(lazy=True) 会触发 BinderException 异常，因为底层集成期望接收一个非空的文件列表。

为避免此错误，请在读取前增加对空输入的检查。可以利用上下文中的 is_incremental 属性，并在调用 .polars(lazy=True) 前检查输入是否包含数据：

Copied!
1from transforms.api import transform, incremental, Input, Output
2
3
4@incremental()
5@transform.using(
6    my_output=Output('/path/to/output'),
7    my_input=Input('/path/to/input'),
8)
9def compute(ctx, my_output, my_input):
10    if ctx.is_incremental and my_input.polars().is_empty():
11        return
12    df = my_input.polars(lazy=True)
13    my_output.write_table(df)

或者，你也可以确保在执行转换前，增量窗口内始终至少包含一个文件。

当参与连接的一个数据集是会被全量读取的参考表，而另一个是增量读取的增量数据集时，执行连接计算是安全的。然而，如果对参与连接的两个数据集都进行增量读取，则需要特殊处理。请参阅利用增量转换连接大型数据集的示例以获取更多信息。
