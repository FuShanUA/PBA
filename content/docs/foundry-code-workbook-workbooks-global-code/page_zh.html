全局代码

全局代码面板位于工作簿界面的右侧，允许您定义变量和函数，这些变量和函数将在整个工作簿中所有该语言的代码转换中可用。例如，您可以使用全局代码来定义在多个转换中使用的常量或定义您希望反复使用的辅助函数。

由于用户定义的函数（UDF），特别是带有循环的函数，通常效率不高，因此此代码可能需要一些时间才能运行。使用全局定义的函数并不总是最佳实践。对于这个例子，pyspark.functions 提供了一种更简单的方法：when((condition), result).otherwise(result)。

让我们尝试不使用 UDF 得到与上面相同的结果：

Copied!
1def passengers_by_age_bracket(titanic_dataset):
2  from pyspark.sql import functions as F
3
4  input_df = titanic_dataset
5  output_df = input_df.withColumn("age_bracket", F.when(input_df.Age.isNull(), 'Not specified')\
6                                                  .when( input_df.Age <= 12, '12 and under')\
7                                                  .when(( (input_df.Age >= 13) & (input_df.Age < 19)), 'Between 13 and 19')\
8                                                  .when(( (input_df.Age >= 19) & (input_df.Age < 65)), 'Between 19 and 65')\
9                                                  .when(input_df.Age >= 65, '65 and over').otherwise('N/A'))
10  output_df = output_df.select('Name','age_bracket')
11
12  return output_df

在大多数情况下，上述转换应该在几秒内完成，而使用 UDF 则需要几分钟。

关于可重复性的说明

请注意，为了确保结果可重复，全局代码中修改的变量和函数不会传播到其他转换中。例如，在 Python 中，如果你在全局代码中定义一个列表如下所示：

Copied!
1my_list = [1,2,3,4]

然后在你的转换中更新该列表：

Copied!
1def my_transform(input_df):
2    my_list.append(5)
3    print(my_list)

运行 my_transform 将会打印 [1,2,3,4,5]，但其他转换仍将接收到 [1,2,3,4] 的值。
