接口

一个 接口（interface） 是一种本体论（ontology）类型，用于描述对象类型（object）的结构及其功能。接口允许对具有相同结构的对象类型进行一致的建模和交互。例如，一个 Facility 接口可能包含 Facility Name 和 Location 属性。Facility 可以由诸如 Airport、Manufacturing Plant 或 Maintenance Hangar 的对象类型实现，每个对象类型都可以包含其他特定于类型的属性。

通过使用 Facility 接口，工作流可以与 Airport、Manufacturing Plant 和 Maintenance Hangar 对象类型进行交互，无论是在聚合情况下还是独立情况下，而无需知道这些对象类型的特定细节。此外，如果引入了实现 Facility 接口的新对象类型，工作流将立即与新对象类型兼容，而无需进一步重构。

查看 当前支持级别，了解有关在平台上何处可以使用接口的更多信息。

接口特性

接口由接口属性（Interface properties）、关联类型约束（link type constraints）、操作类型约束（action type constraints）和关于接口的元数据（metadata）组成。接口属性可以在接口上本地定义（推荐）或使用共享属性（shared properties）来定义。一个接口可以被多个对象类型实现。

与编程语言中的接口类似，您可以 扩展接口（extend an interface） 来创建一个子接口，该子接口继承原始接口的属性，然后向子接口添加新的、更具体的属性。随后，对象类型可以 实现接口（implement the interface），以表明它们符合接口定义。对象类型可以实现多个接口，用于不同的工作流程。接口也可以扩展多个其他接口，包括那些自身又扩展了其他接口的接口，从而通过多层接口继承属性。

接口与对象类型的区别

在本体论中，接口和对象类型之间既有功能上的差异，也有风格上的差异。

对象类型是具体的；它们通过共享或本地属性定义模式，由包含属性值的数据集支持，并且可以实例化为对象。

相比之下，接口是抽象的；它们通过接口属性定义模式，不依赖于数据集，并且不能直接实例化，而必须作为特定的对象类型来实例化。

在风格上，平台通过在图标周围使用虚线来将接口与对象类型区分开来。

当前支持级别

随着对接口本体论类型的支持扩展，不同Palantir平台上的可用性将有所不同。

接口目前在以下应用和服务中受到支持：

  • 本体论管理器: 定义、编辑和实现接口。
  • 市场: 打包和安装接口。
  • 函数: TypeScript v2 函数。

以下应用和服务中部分支持接口：

  • 操作: 定义创建、修改、删除或关联实现接口的对象的操作。接口操作类型约束处于测试阶段，可用于定义实现接口的不同对象类型的预期操作能力。
  • 对象集服务: 按接口搜索和排序对象。按接口聚合的支持正在开发中。接口链接类型的支持正在开发中。
  • 本体论 SDK: 使用接口作为与实现对象类型交互的API层。支持因语言而异；目前支持TypeScript，Java和Python的支持正在开发中。

接口正在积极开发中，但尚未在以下内容中得到支持：

  • Workshop
  • Functions: TypeScript v1 和 Python 函数

开始使用接口

要将接口添加到您的本体论中，您可以 创建 新接口或 扩展 现有接口。一旦您有了一个接口，就可以将其与适当形状的对象类型 实现，或者 编辑 它以更好地适应您的组织，随着本体论的发展。
