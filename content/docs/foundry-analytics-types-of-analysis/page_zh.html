分析类型

Foundry 分析套件支持点击式分析（无代码或低代码）和基于代码的数据分析，提供了针对不同分析和数据源类型优化的工具，以及用于仪表板和报告的工具。

工具和支持的工作流汇总如下：

对象数据（本体）Insight, Quiver
时间序列数据QuiverCode Workbook
地图（地理空间）数据Map, Contour, Quiver, Insight

点击式分析

Foundry 提供了几种用于点击式分析的工具：Contour、Insight 和 Quiver。

  • Contour 能够对大规模表格数据进行分析。
  • Insight 专为业务运营人员设计，用于分析本体并创建对象集。
  • Quiver 支持高级本体分析、仪表板创建以及时间序列数据处理。

此外，Object Explorer 是针对本体层的搜索与发现工具，支持跨本体搜索并查找单个对象。

通过 Contour、Insight 和 Quiver，您可以：

  • 无需编写代码即可实现数据的可视化、过滤和转换。
  • 利用表达式语言执行更高级的数据转换和聚合操作。
  • 组织复杂的分析流程。
  • 创建交互式仪表板，使他人能够以引导式、结构化的方式探索和调查数据。
  • 与同事共享分析结果。

请在下方详细了解这些工具适用的具体场景。

Contour

Contour 适用于以下分析场景：

  • 您要使用的部分或全部数据未映射至本体： 通常情况下，我们建议尽可能使用本体层，但在某些情况下这可能并不合适，例如无需清洗或不会重复使用的单次上传数据。并非所有数据都需要进入本体层。例如，Contour 可用于查看数据管道中的中间数据集。
  • 您需要对超大规模数据集进行操作： Contour 旨在支持对超大规模数据集的分析操作，而 Quiver 支持的对象聚合行数上限为 50,000 行。
  • 您希望将分析结果作为新数据集共享，供其他 Foundry 工具使用。 了解有关将结果保存为数据集的更多信息。
  • 您希望构建参数化分析，以便在不同数据视图和结果之间轻松切换。

了解如何开始使用 Contour。

Insight

Insight 适用于以下分析场景：

  • 您的数据已映射至本体： Insight 利用本体中的数据，例如对象类型、接口、链接类型和操作。
  • 您需要进行无代码的即席分析： 可视化分析路径使得逐步构建复杂查询变得轻松，同时支持简单和复杂的下钻工作流。
  • 您希望探索对象之间的关系： 链接 步骤允许您顺着链接发现本体中相互关联的对象，并根据链接属性过滤数据。
  • 您希望将数据写回本体： Insight 支持根据分析结果创建、更新和删除对象。

了解如何开始使用 Insight。

Quiver

Quiver 适用于以下分析场景：

  • 您的数据已映射至本体： 对象之间的链接在本体中进行了原生表示，因此 Quiver 用户无需执行连接操作，也不必费心识别主键或外键。以链接形式表示的连接，可以通过对对象或对象集执行“搜索周围”来检索任何关联的对象。
  • 您正在处理时间序列数据： Quiver 已针对时间序列数据进行了优化。它包含一个特定的时间序列库，具备传感器和信号处理功能，并可由专有的时间序列数据库提供支持，该数据库已针对高频信号上的特定操作进行了优化。有关更多信息，请参阅“时间序列分析”。
  • 您希望将分析结果嵌入其他具备对象感知能力的应用中：Quiver 仪表板可以嵌入到 Workshop 等运营应用中。
  • 您希望将数据写回到本体中：可以使用操作类型将 Quiver 的分析决策写回到本体中。这能让您立即记录分析结论，供他人使用。

了解如何开始使用 Quiver。

对象浏览器

用户可以在对象浏览器中进行简单的基于对象的分析工作流。对象浏览器是一款搜索和分析工具，用于解答有关本体层中任何内容的问题。用户可以在对象浏览器中以可视化方式构建搜索查询，从简单的筛选到周边搜索，以查找感兴趣的对象。对象浏览器适用于以下分析工作流：

  • 您希望在整个本体中搜索术语：对象浏览器提供了一个搜索入口，用于查找整个本体中的对象或对象类型。搜索结果页面支持查看对象实例和对象类型结果。
  • 您希望探索对象类型及其属性：对象浏览器是查找相关对象数据和查看对象视图的绝佳起点。
  • 您的最终目标是创建相关对象的列表（对象集）：对象浏览器支持创建对象探索和对象列表，以便与他人分享或在 Quiver 或 Insight 等其他应用中用于进一步分析。了解有关探索和对象列表的更多信息。

了解如何开始使用对象浏览器。

基于代码的分析

代码工作簿

代码工作簿是一款允许用户使用直观的图形界面来分析和转换数据的应用。

代码工作簿的设计遵循以下原则：

  • 迭代速度：用户可以快速测试和优化转换与可视化逻辑，以得出有用的结果。
  • 低使用门槛：代码工作簿的图形界面和对使用预置逻辑的支持，旨在让不同技术水平的用户都能轻松上手。
  • 协同合作：不同的用户群体可以在代码工作簿中共享逻辑，并围绕同一项分析展开协作。
  • 模型创建：用户可以在代码工作簿中迭代 Foundry 模型，并轻松查看模型阶段和指标。随后，这些模型可以提交到建模目标中投入运营使用。
  • 平台互操作性：用户可以通过将可视化内容添加到记事本，以及将可投入生产的管道提升至代码仓库，在全平台实现分析成果的产品化。

代码工作簿的核心功能包括：

  • 交互式控制台，用于快速迭代转换逻辑和进行临时数据探索。
  • 可视化支持，可使用常用软件包创建详细且交互式的图像。
  • 模板，可通过简单的界面复用复杂且特定领域的逻辑。
  • 支持多种语言（Python、SQL、R）编写代码和模板，允许用户为分析选择最合适的语言，并在单次分析中结合使用多种语言。
  • 分支管理，通过将各用户的更改相互隔离来促进协作。
  • 直观的用户界面，可轻松自定义 Spark 环境、设置转换的输入类型，以及查看节点之间的关系。

了解如何开始使用代码工作簿。

构建管道应该使用哪种工具？

代码工作簿并未针对构建生产管道进行优化。如果您正在构建或维护生产管道，请使用代码仓库应用，该应用包含版本历史记录、分支和拉取请求，以及其他对构建稳健管道至关重要的功能。有关 Foundry 编写基于代码转换的工具的对比，详情请参阅此比较文档。

时间序列分析

Foundry 为时间序列的存储、监控、转换、分析和写回提供了高级的端到端工具。从历史表现分析到趋势和相关性分析，再到预测，各种工作流都可以利用 Foundry 的时间序列功能。

Quiver 和代码工作簿分别支持无代码和基于代码的时间序列分析。了解如何开始使用时间序列。

基于地图（地理空间）的分析

对于希望将分析与实际业务运营相结合的 Foundry 用户而言，地理空间数据往往是关键的输入要素。Foundry 简化了地理空间数据的转换与分析流程，能够支持基于地图的工作流。

《地图应用》是一款功能强大的地理空间分析工具。此外，Contour 和 Quiver 也提供了地图小部件，可直接用于涉及地理空间数据的分析工作。
