Markdown

Markdown 小部件支持使用 Markdown 格式渲染文本。小部件内的高级功能可用于启用附加工作流，例如对文本进行归属、引用和用户标记。

配置 Markdown 小部件的模块构建者可以使用以下功能：

  • 基本的 Markdown 文本格式，如粗体、斜体、删除线和高亮显示
  • 更高级的 Markdown 格式，如标题、表格、块样式、代码样式和 URL

  • 从特定高亮的锚点文本到本体论（Ontology）对象的嵌入式引用
  • 从特定高亮的锚点文本触发 Workshop 操作事件

以下示例显示了一个配置好的 Markdown 小部件，它显示带有以下用户交互的注释对象，这些交互已配置在嵌入的对象上：

  • 在文本的特定部分显示注释对象
  • 从显示的注释对象触发点击操作和事件
  • 在文本的特定部分创建新的注释对象

配置选项

以下截图显示了未配置的 Markdown 小部件及其初始配置面板的初始状态。

对于 Markdown 小部件，核心配置选项如下：

  • 输入数据：文本/变量

    • 文本：如果选择文本选项，构建者可以直接在配置面板中输入他们想要显示的输入 Markdown 文本。
    • 变量： 如果选择变量选项，可以选择一个字符串变量作为要显示的输入Markdown文本。

  • 显示选项：

    • 标签类型配置

      • 标准：默认此选项已启用，并使用标准Markdown格式显示输入内容。
      • 内联引用：如果启用，允许在文本中嵌入对本体论对象和点击事件的工作区（Workshop）事件。这是Markdown语法的自定义扩展；有关更多信息，请参阅下面的内联引用部分。
      • 注释：如果启用，允许显示嵌入文本中的注释对象、对现有注释对象的用户交互以及使用小部件创建新的注释对象。有关更多信息，请参阅下面的注释部分。

    • 启用等宽字体：切换以启用/禁用在小部件中以等宽字体显示文本。

    • 启用滚动：切换以启用/禁用小部件的滚动功能。

    • 允许长单词换行： 切换以启用/禁用对超出控件宽度的长连续字符串（如长URL）进行换行。当启用时，这是新控件的默认设置，此类字符串会换行到下一行，而不是溢出控件。

    • 按换行符换行： 切换以控制Markdown源中的单行换行如何呈现。当启用时，这是新控件的默认设置，源中的单个换行符会在呈现的输出中开始新的一行。当禁用时，单个换行符会被折叠为空格，遵循标准的Markdown呈现方式。

    • 文本对齐： 设置控件内呈现的Markdown内容的对齐方式。选择左对齐（默认）、居中或右对齐。

      在Markdown表格语法中显式定义的每列对齐方式（例如，| :---: |）将优先于控件级别的文本对齐设置。代码块始终左对齐且全宽，与所选对齐方式无关。

  • 用户文本选择：

    • 输出用户选择的原始文本： 以原始 Markdown 字符串的形式输出用户选择的文本。
      • 选中文本： 包含用户选择的文本作为原始 Markdown 的字符串变量。
    • 输出用户选择的索引： 以数值变量的形式输出用户选择文本的起始和结束索引。
      • 选择起始索引： 包含用户选择文本起始索引的数值输出变量。
      • 选择结束索引： 包含用户选择文本结束索引的数值输出变量。

Markdown 小部件中的对象引用

内联引用

作为一项高级功能，Markdown 小部件允许构建者对 Markdown 文本的子集（“锚点”）进行标记，然后使用这些锚点链接到特定的 Ontology 对象并触发 Workshop 点击事件。

创建这些锚点之一的格式如下：

:objectreference[$text_to_display]{objectType="$object_type_id" primaryKey="$object_primary_key"}

让我们通过一个例子来了解如何在一句话中引用两个 Flight Alerts 对象。首先，我们看一下希望在屏幕上显示给用户的最终状态。注意：下面提到的每个 Flight Alert 对象都可以被用户单独选择，之后将成为 Markdown 小部件的输出选定对象集合。

为了实现上述效果，支撑的 Markdown 输入是以下内容：

Newark airport has __*rarely*__ seen flight issues in May, but there were two high-priority delays: :objectreference[Alert A00150]{objectType="flight-alert" primaryKey="A00150"} and :objectreference[Alert A00182]{objectType="flight-alert" primaryKey="A00182"}

除了上述Markdown输入的语法外，构建者还可以为对象引用配置以下选项：

  • 已选对象集：使用对象引用时必需。这是Markdown小部件的输出对象集。当用户在Markdown小部件中选择一个对象引用时，该对象将输出到此对象集变量中。
  • 对象类型：使用对象引用时必需。构建者应选择将在Markdown小部件中引用的所有对象类型。一旦添加了对象类型，构建者可以在内部配置面板中进一步配置条件格式规则。如果在Markdown小部件中引用了某个对象类型但未在此列表中配置，则对象引用不会出现在Markdown小部件中。
    • 对象类型：指定要配置进一步颜色和事件选项的对象类型。
    • 高亮颜色：选择静态颜色、从具有本体论格式的属性继承颜色，或定义自定义规则来确定颜色。
    • 选择时覆盖事件：为指定的对象类型配置Workshop事件以触发。这些将覆盖任何其他选择事件。
  • 选择行为： 控制Markdown小部件内的选择行为。
    • 如果选择不高亮，在Markdown小部件内选择对象引用不会导致选中状态。
    • 如果选择高亮最近选中的，在Markdown小部件内选择对象引用会导致最近选中的锚文本被高亮。
    • 如果选择高亮选中的引用，在Markdown小部件内的高亮基于所选对象集的内容。当Markdown中的对象引用与另一个小部件的对象1:1对应，并且两个小部件的所选对象集相同时，此选项效果最佳。
  • 选择时的事件： 此选项允许模块构建者配置Workshop事件，以便在Markdown小部件中选择对象引用时触发（例如，使包含更详细对象视图的抽屉出现）。

    • Markdown中的对象引用也可以应用标准的Markdown格式。下面的截图包含了许多Markdown格式的示例，如嵌入对象的标题和表格。

    注释

    注释（Annotation）选项可以用于通过Markdown小部件在文本上显示注释对象。此选项还允许用户通过在文本选定部分创建新的注释对象，或对显示的注释对象执行操作和事件来与小部件中的文本进行交互。

    注释对象使用从零开始的数字索引捕获选定文本，包含起始索引和不包含结束索引，以表示文本选择的起始和结束位置。当前Markdown小部件的注释不支持负索引。

    选择注释（Annotation）选项后，将提供以下配置：

    • 显示现有注释 配置单独的注释对象层，以在小部件中显示各种注释。
      • 注释输入
        • 名称： 在小部件配置中设置标注图层的名称。
        • 标注对象集：指定包含要显示的标注对象类型的对象集。
        • 起始索引：设置表示标注的包含起始索引的数值对象属性。
        • 结束索引：设置表示标注的排除结束索引的数值对象属性。
      • 现有标注交互
        • 选定的标注：包含当前选定的标注对象的对象集。
        • 选择事件：设置在选择标注对象时触发的事件。
        • 在提示框中显示的属性：指定在标注对象悬停提示框中显示的对象属性。
        • 悬停交互：配置悬停交互，例如操作和事件，这些将在标注对象的悬停提示框中显示。
          • 图标：为悬停交互设置显示的图标。
          • 标题：为悬停交互设置显示的标题。当鼠标悬停在交互图标上时会显示该标题。
          • 交互： 在注释对象上配置可触发的操作或事件。
            • 悬停对象：一个特殊的操作参数值，可用于引用悬停的注释对象。
      • 注释显示选项
        • 高亮颜色：设置用于显示注释的颜色。可以静态定义自定义颜色，也可以设置条件格式规则。
        • 注释格式：设置注释的显示方式。选项包括彩色高亮、使用实线或虚线的下划线，或对注释同时进行高亮和下划线。
    • 通过操作或事件创建注释： 配置操作或事件，当在小部件中选中文本时创建新的注释对象。
      • 图标：设置文本高亮时显示的图标。
      • 标题：设置文本高亮时显示的标题。标题在鼠标悬停于图标上时显示。
      • 交互：配置可在高亮文本上触发的操作或事件。
        • 高亮文本： 一个特殊的操作参数值，可用于引用小部件中突出显示的文本。

    限制

    Markdown 小部件不支持渲染 HTML。提供给 Markdown 小部件的 HTML 将作为文本进行渲染。
