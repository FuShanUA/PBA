高级配置

本节介绍如何在 Java 转换中使用高级配置选项。

最大构建时长

为了保障数据时效性或控制成本，可能需要限制作业的运行时长。例如，当作业与外部服务交互且失去响应时，对其运行时长设限就十分必要，否则该作业可能无法正常结束。 在 Code Repositories 中，可以通过 MaxAllowedDuration 和 Compute 装饰器来限制作业时长，如下所示：


```java
package myproject.datasets;

import com.palantir.transforms.lang.java.api.*;
import java.time.Duration;
import org.apache.spark.sql.Dataset;
import org.apache.spark.sql.Row;

public final class FilterTransform {

   @MaxAllowedDuration(value = "PT2H")
   @Compute
   public void myComputeFunction(
           @Input("/examples/students_hair_eye_color") FoundryInput myInput,
           @Output("/examples/students_hair_eye_color_filtered") FoundryOutput myOutput) {
       Dataset inputDf = myInput.asDataFrame().read();
       myOutput.getDataFrameWriter(inputDf.filter("eye = 'Brown'")).write();
   }
}
```


需要注意的是，尽管 MaxAllowedDuration 接收的是 Duration 值，但系统每 5 分钟轮询一次任务。因此，若取值为 PT3M（ISO 8601 格式），任务将在 5 分钟时取消；若取值为 PT7M，则会在 10 分钟时取消，依此类推。
