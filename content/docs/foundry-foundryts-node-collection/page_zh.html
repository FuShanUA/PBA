foundryts.NodeCollection

class foundryts.NodeCollection(*nodes, **kwargs)

FunctionNode 或 SummarizerNode 的集合。

NodeCollection 是一个可迭代对象，可传递给需要多个时间序列的函数，也可用于将集合中的每个节点映射到某个函数。

对于原始序列和点集，NodeCollection 的数据框包含一个名为 series 的额外列，用于表示各个数据点所属的序列。series 的值即为序列 ID，该 ID 在时间序列同步过程中设定，或通过 foundryts.functions.points() 的别名指定。

注意

请勿将 NodeCollection 直接传递给仅需单个输入时间序列的函数，这会导致操作报错。若需对节点中的所有元素应用相同函数，请使用 NodeCollection.map()，该方法会将转换后的点或摘要映射到最终数据框中的对应序列上。请查阅 FoundryTS 函数文档以确认各函数所需的输入参数数量。

map_intervals(intervals, interval_name=None)


使用这些时间间隔为集合中的所有时间序列创建时间范围。




每个区间用于在输入时间序列上创建 `foundryts.functions.time_range()`，以便进行进一步的转换与分析。此方法最适合配合创建 `Interval` 使用，无论是手动创建，还是将 `foundryts.functions.time_range()` 的结果转换为 `Interval`。

生成的数据框包含 `interval.start` 和 `interval.end` 附加列。

- **参数：**
  - **intervals** (`Interval` | `List` [`Interval` ]) – 为集合中所有时间序列创建时间范围的一个或多个区间。
  - **interval_name** (`str` , `optional`) – 数据框中间序列的可选别名。
- **返回值：**
  更新后的 NodeCollection，其中每个项均映射到应用 `func` 后的相应输出。
- **返回类型：**
  Iterable[NodeCollection]


另请参阅
`foundryts.functions.time_series_search()`

to_dataframe(numPartitions=16)

计算集合中的所有时间序列，并将结果拼接为 pyspark.sql.DataFrame。

PySpark DataFrame 支持分布式数据处理与并行转换。处理包含大量行的数据帧时非常实用，例如加载原始序列中的所有数据点，或是获取 FunctionNode 的结果，亦或是对多个 SummarizerNode 或 FunctionNode 的结果进行联合计算。

附加的 series 列会通过序列 ID 或 foundryts.functions.points() 别名，标记各结果所属的具体序列。

  • 参数： numPartitions (int , 可选) – 指定在 Spark 执行器间分发集合内时间序列数据的分区数，用于优化并行数据处理。执行器数量较多时，调大该值有助于提升性能；执行器较少时，调小该值可能更高效。请根据时间序列集合的规模以及您的 Spark 配置（例如执行器数量和执行器内存）进行调整（默认值为 16）。
  • 返回： 集合中所有时间序列及其操作的计算结果，输出为 PySpark 数据帧。
  • 返回类型： pyspark.sql.DataFrame
See Also

NodeCollection.to_pandas()

Note

对于大型集合，请将 numPartitions 设置为合理的值。

to_pandas(parallel=4, mode='thread')

评估此集合中的时间序列查询，并将结果拼接为pandas.DataFrame。

有关 dataframe 形状的详细信息，请参阅 FunctionNode.to_pandas() 和 SummarizerNode.to_pandas()。

额外的 series 列将使用序列 ID 或 foundryts.functions.points() 别名来标注结果所属的序列。

  • 参数：
    • parallel (int , 可选) – 用于评估时间序列查询的并行线程或进程数。如果设置为 1，则不进行多进程处理（默认值为 4）。
    • mode (str , 可选) – 有效选项为 process 或 thread，分别用于控制多进程线程池的类型。
  • 返回： 集合中时间序列查询的输出结果，已评估为 Pandas dataframe。
  • 返回类型： pd.DataFrame
See Also

NodeCollection.to_dataframe()

Note

集合中每个时间序列查询的结果都会在本地进行存储并拼接到最终的 dataframe 中，因此结果必须能够放入内存。

并行执行会增加内存使用量和资源消耗。使用 process 模式可能会引入额外开销。过度的并行化会降低性能。
