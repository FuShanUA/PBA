foundryts.NodeCollection

类 foundryts.NodeCollection(*nodes, **kwargs)

FunctionNode 或 SummarizerNode 的集合。

NodeCollection 是一个可迭代对象，可以传递给期望多个时间序列的函数，或用于将集合中的每个节点映射到一个函数。

对于原始序列和点集，NodeCollection 的 dataframe 包含一个额外的列称为 series，表示这些点所属的序列。series 值将是时间序列同步中设置的序列 ID，或者通过 foundryts.functions.points() 的别名设置。

注意

确保不要将NodeCollection直接传递给期望单个输入时间序列的函数，这会导致操作出错。要对节点的所有元素应用相同的函数，请使用 NodeCollection.map()，这会将转换后的点或汇总映射到最终数据框中的序列。请参阅FoundryTS函数文档以查看函数的输入数量。

columns()

返回一个字符串元组，表示通过将此集合评估为数据框所产生的 pandas.DataFrame 的列名。

注意

嵌套对象的键将被展平为一个元组，嵌套键用 . 连接。

非统一集合将包含集合中每个元素的所有列的并集。

  • 返回值： 包含该集合评估后所得数据框中的列名的元组。
  • 返回类型： Tuple[str]

map(func)

将集合中的每个时间序列映射到输入函数的输出。

这在对一组时间序列应用相同的一组查询时非常有用。

  • 参数：func ( ()-> FunctionNode|SummarizerNode) – 应用于集合中每个时间序列的FoundryTS支持的函数。
  • 返回值： 更新后的 NodeCollection，其中每个项目都映射到应用 func 的相应输出。
  • 返回类型： Iterable[NodeCollection]

map_intervals（intervals, interval_name=None）

使用 intervals 为集合中的所有时间序列创建一个时间范围。

每个 interval 用于在输入的时间序列上创建一个 foundryts.functions.time_range()， 这可用于进一步的转换和分析。最好与创建 Interval 一起使用， 无论是手动创建还是将 foundryts.functions.time_range() 的结果转换为 Interval。

生成的数据框增加了 interval.start 和 interval.end 的附加列。

  • 参数：
    • intervals (Interval|List[Interval]) – 一个或多个区间，用于为集合中的所有时间序列创建时间范围。
    • interval_name (str,可选) – 数据框中区间列的可选别名。
  • 返回值： 更新后的 NodeCollection，其中每个项目都映射到应用 func 的相应输出。
  • 返回类型： Iterable[NodeCollection]
另请参见

foundryts.functions.time_series_search()

to_dataframe(numPartitions=16)

对集合中的所有时间序列进行评估，并将结果连接到 pyspark.sql.DataFrame。

PySpark DataFrame 支持分布式数据处理和并行转换。当处理具有大量行的数据框时，例如加载原始系列中的所有点或 FunctionNode 的结果，或者同时评估多个 SummarizerNode 或 FunctionNode 的结果时，它们会很有用。

还将添加一个 series 列，使用系列 ID 或 foundryts.functions.points() 别名来注明结果所属的系列。

  • 参数：numPartitions（int，可选）– 指定用于在 Spark 执行器之间分布集合中的时间序列数据的分区数，以优化并行数据处理。当执行器数量较多时，较高的值可以提高性能，而较少的执行器可能使用较低的值更高效。请根据时间序列集合的大小和您的 Spark 配置（例如执行器数量和执行器内存）进行调整（默认为 16）。
  • 返回： 此集合中所有时间序列和操作的输出被评估为一个 PySpark DataFrame。
  • 返回类型：pyspark.sql.DataFrame
另请参见

NodeCollection.to_pandas()

注意

to_pandas(parallel=4, mode='thread')

在该集合中评估时间序列查询，并将结果拼接成一个 pandas.DataFrame。

有关数据框结构的详细信息，请参考 FunctionNode.to_pandas() 和 SummarizerNode.to_pandas()。

一个额外的 series 列将使用系列 ID 或 foundryts.functions.points() 别名来注明结果属于哪个系列。

  • 参数：
    • parallel （int，可选）– 用于评估时间序列查询的并行线程或进程数量。如果设置为 1，则不进行多进程处理（默认是 4）。
    • mode （str，可选）– 有效选项是 process 或 thread，每个控制多进程线程池的类型。
  • 返回： 对集合中的时间序列查询的输出结果转换为 Pandas 数据框。
  • 返回类型： pd.DataFrame
另请参见

NodeCollection.to_dataframe()

注意

该集合中每次时间序列查询的结果都会被存储并连接到最终的DataFrame中，因此结果必须适合内存。

并行执行会增加内存使用和资源消耗。使用process模式可能会引入开销。过度的并行性会降低性能。

to_rdd(numPartitions=16)

弃用

自版本1.0.0起已弃用。

通过评估此集合中的节点返回一个（key, object)的RDD。如果此 NodeCollection 中的节点是通过分组或窗口操作得到的，相应的key将包含给定的元数据键或区间。否则，它将是节点的系列标识符。

  • 参数:numPartitions (int) – 用于 Spark 执行的分区数
  • 返回类型:pyspark.RDD
