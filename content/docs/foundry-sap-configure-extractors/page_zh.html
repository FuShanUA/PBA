提取器

面向 SAP 应用的 Palantir Foundry Connector 2.0（以下简称“Connector”）支持通过多种被称为 SAP 提取器的提取接口进行数据接入。这些提取器最初的设计目的是从 SAP 操作系统提取数据至 SAP Business Warehouse (BW)。

提取器分为以下 3 种类型：

  • 业务内容提取器：由 SAP 预先交付的标准提取器，适用于 FI、CO、LO Cockpit 等多种应用组件。在源系统中，它们默认未激活，使用前需通过 RSA5 事务码进行激活。
  • 客户生成的提取器：通常基于客户的实施与配置设置生成。
  • 通用提取器：具备高度可定制性。客户可基于数据库表/视图、信息集和功能模块生成自身的提取逻辑。此类提取器高度可定制，且提取逻辑由客户掌控。

Connector 仅支持使用启用了 ODP 的提取器。根据 SAP Note 1931427 - 《ODP 数据复制 API 2.0》，适用组件（PI_BASIS、SAP_BW 或 DW4CORE）需安装以下支持包 (SP) 才能使用完整的“ODP 数据复制 API 2.0”。请确保满足以下前提条件：

  • PI_BASIS 730 SP 14（属于 SAP NetWeaver 7.30 SP 14）+
  • PI_BASIS 731 SP 16（属于 SAP NetWeaver 7.03 SP 16 和 7.31 SP 16）+
  • PI_BASIS 740 SP 11（属于 SAP NetWeaver 7.40 SP 11）+
  • SAP_BW 750 SP 0（包含原 PI_BASIS 包）+
  • DW4CORE 100 SP 0（包含原 PI_BASIS 包）+
  • DW4CORE 200 SP 0（包含原 PI_BASIS 包）+

关于启用 ODP 提取器的源系统中的授权，请参阅 SAP Note 2855052 - 《ODP 数据复制 API 2.0 所需的授权》。 此外，Connector 针对提取器设有独立角色：请将 /PALANTIR/CONTENT_EXT_ALL 授权角色分配给 Foundry 技术用户。

提取器的配置

来自单一源系统（Connector 版本低于 v2.20.0 (< SP20)）

  • 创建从 Connector 到存放提取器的 SAP 源系统的 RFC 连接。
  • 使用 /n/palantir/param 事务维护以下参数：
    • RFC 参数
      • 参数 ID： EXTRACTOR
      • 参数名称： RFC_CONFIGURATION
      • 参数值： <RFC 连接名称>
    • 上下文配置
      • 参数 ID： EXTRACTOR
      • 参数名称： CONTEXT_CONFIGURATION
      • 参数值： SAPI

来自多个源系统（Connector 版本 v2.20.0 或更高 (>= SP20)）

自 Connector SP20 起，系统支持从多个源系统提取数据，这些源系统也被称为“上下文”。

要配置多个上下文，请按以下步骤操作：

  • 运行 /PALANTIR/PARAM_E1 事务并定义各个上下文：
    • 提取器 ID： <源系统 ID>
    • 描述： <源系统描述>
  • 运行 /PALANTIR/PARAM_E2 事务并定义各上下文参数：
    • 提取器 ID： <上下文 ID>
    • 参数 ID： <Connector 参数 ID>
    • 参数名称： <Connector 参数名称>
    • 参数值： <参数值>

RFC 和 SOURCE 参数专用于多上下文场景；其他提取器参数为标准 Connector 参数。

  • 可将其中一个上下文设为默认。在此情况下，若 Foundry 未指定上下文信息，Connector 将使用该默认上下文。要定义默认上下文，请运行 /PALANTIR/PARAM 事务并使用以下参数值：
    • 参数 ID： EXTRACTOR
    • 参数名称： DEFAULT_CONFIGURATION
    • 参数值： <EXTRACTOR ID>

使用提取器

  • 使用 RODPS_OS_EXPOSE 程序在 SAP 源系统中公开 SAP 提取器。
  • 重启 Foundry Data Connection 智能体以刷新缓存。
  • 创建新的 BW Content Extractor 同步。
