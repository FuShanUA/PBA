提取器

Palantir Foundry 2.0 for SAP Applications（"Connector"）支持通过多个称为SAP 提取器的提取接口进行数据摄取。它们最初设计用于从 SAP 操作系统中提取数据到 SAP Business Warehouse（BW）。

有三种类型的提取器：

  • 业务内容提取器是标准的，并由 SAP 针对各种应用组件（如 FI、CO、LO Cockpit 等）预先提供。它们在源系统中默认未激活，使用前需要通过RSA5事务代码激活。
  • 客户生成的提取器通常是基于客户的实施和配置设置生成的。
  • 通用提取器高度可定制。每个客户都可以根据数据库表/视图、信息集和函数模块生成自己的提取逻辑。这些提取器高度可定制，提取逻辑由客户控制。

连接器只能使用启用了ODP的提取器。完整的“ODP数据复制API 2.0”可通过以下支持包（SP）获得，适用于相关组件（PI_BASIS、SAP_BW或DW4CORE），参考SAP注释 1931427 - ODP数据复制API 2.0。确保满足以下先决条件：

  • PI_BASIS 730 SP 14（SAP NetWeaver 7.30 SP 14的一部分）+
  • PI_BASIS 731 SP 16（SAP NetWeaver 7.03 SP 16和7.31 SP 16的一部分）+
  • PI_BASIS 740 SP 11（SAP NetWeaver 7.40 SP 11的一部分）+
  • SAP_BW 750 SP 0（包括以前的PI_BASIS包）+
  • DW4CORE 100 SP 0（包括以前的PI_BASIS包）+
  • DW4CORE 200 SP 0（包括以前的PI_BASIS包）+

对于包含启用了ODP的提取器的源系统中的授权，请参考SAP注释 2855052 - ODP数据复制API 2.0所需的授权。 连接器还为提取器提供了一个单独的角色：将 /PALANTIR/CONTENT_EXT_ALL 授权角色分配给Foundry技术用户。

提取器配置

从单个源系统（Connector版本早于v2.20.0 (< SP20)）

  • 从Connector创建一个RFC连接到包含提取器的SAP源系统。
  • 使用/n/palantir/param事务来维护以下参数：
    • RFC参数
      • 参数ID： EXTRACTOR
      • 参数名称： RFC_CONFIGURATION
      • 参数值： <RFC connection name>
    • 上下文配置
      • 参数ID： EXTRACTOR
      • 参数名称： CONTEXT_CONFIGURATION
      • 参数值： SAPI

从多个源系统（Connector 版本 v2.20.0 或更新版本（>= SP20））

自 Connector 的 SP20 版本起，可以从多个源系统提取数据，也称为“上下文”。

要配置多个上下文，请按照以下步骤操作：

  • 运行 /PALANTIR/PARAM_E1 事务并定义每个上下文：
    • 提取器 ID（Extractor ID）: <Source System ID>
    • 描述（Description）: <Description of Source System>
  • 运行 /PALANTIR/PARAM_E2 事务并定义每个上下文参数：
    • 提取器 ID（Extractor ID）: <Context ID>
    • 参数 ID（Param ID）: <Connector parameter ID>
    • 参数名称（Param Name）: <Connector parameter name>
    • 参数值（Param Value）: <Parameter Value>

RFC 和 SOURCE 参数是多上下文专用的；其他提取器参数是标准 Connector 参数。

  • 其中一个上下文可以设置为默认。在这种情况下，如果未从 Foundry 提供上下文信息，Connector 将使用默认上下文。要定义默认上下文，请使用以下参数值运行 /PALANTIR/PARAM 事务：
    • 参数 ID（Param ID）: EXTRACTOR
    • 参数名称： DEFAULT_CONFIGURATION
    • 参数值： <EXTRACTOR ID>

使用提取器

  • 使用程序 RODPS_OS_EXPOSE 在 SAP 源系统中暴露 SAP 提取器。
  • 重启 Foundry 数据连接代理 以刷新缓存。
  • 创建一个新的 BW 内容提取器 同步。
