对比：代码仓库、代码工作空间与代码工作簿

Foundry 提供了三款用于编写基于代码的数据转换工具：代码工作簿、代码工作空间和代码仓库。虽然这些产品在功能上有所重叠，但每款产品都针对特定的工作流和用户类型进行了优化。以下指南旨在帮助您确定哪款工具最适合您的需求。

对于构建稳健的生产管道并支持需要额外治理面和审查的工作流，推荐使用代码仓库。借助代码仓库，数据工程师可以批量创建高效的数据管道。适合使用代码仓库的典型工作流包括：

  • 数据规模庞大且需要增量计算的日常管道。
  • 具有严格数据治理要求的高可见性管道，要求能够回退至历史代码的先前版本，或者在单元测试通过后对代码变更进行门控。

对于使用 JupyterLab® 和 RStudio® Workbench 进行快速高效的探索性分析，推荐使用代码工作空间，它将熟悉的 IDE 与 Foundry 平台的优势相结合，例如数据安全、分支管理、构建调度和资源管理。适合使用代码工作空间的典型工作流包括：

  • 逐单元格运行数据分析，并将其内容导出为可共享的报告。
  • 为数据转换管道或机器学习模型制作原型。

对于在超大规模数据上执行基于代码的分析（这类分析不适合使用代码工作空间），推荐使用代码工作簿。这些分析可以是一次性使用，也可以生成定期更新的产出物。代码工作簿也可用于构建管道原型，随后将其提升至仓库。适合使用代码工作簿的典型工作流包括：

  • 通过测试不同的 p 值来研究临床试验结果。
  • 创建交互式可视化以与他人共享。

除上述基于代码的产品外，管道构建器是 Palantir 平台的主要无代码应用，用于构建和维护生产数据管道。管道构建器采用图形和表单界面，用户无需编写代码即可集成数据并创建业务逻辑转换。如果您正在评估是否使用管道构建器或代码仓库来构建管道，请参阅《注意事项：管道构建器与代码仓库》。

表格摘要
代码仓库功能
  • 代码仓库具备高级流水线功能，支持在持久化数据管道中实现复杂工作流，在性能优化和代码生成方面具有高度灵活性。
  • 代码仓库支持的编程语言包括 Python、SQL、Java 和 Mesa。
  • 代码仓库支持增量计算、转换生成、多输出转换以及文件系统访问。
  • 代码仓库不支持可视化功能。
代码工作区特性
  • 代码工作区支持快速高效的探索性工作流，并在 Foundry 中内嵌了对 JupyterLab® 和 RStudio® Workbench 的支持。
  • 代码工作区支持的编程语言包括 Python 和 R。
  • 代码工作区支持文件系统访问，并为基于 Notebook 的分析提供充分的灵活性。
  • 代码工作区不支持分布式 Spark，因此更适合处理能够容纳在工作区计算限制内的数据。
代码工作簿特性
  • 代码工作簿支持高级分析工作流，兼容常见的分析语言和可视化库。
  • 代码工作簿支持的编程语言包括 Python、R 和 SQL。
  • 代码工作簿支持文件系统访问和可视化。
  • 代码工作簿不支持增量计算、转换生成或多输出转换。
代码仓库迭代周期
  • 代码仓库旨在辅助迭代代码逻辑。构建完成后，即可在 Foundry 中进行数据分析。
  • 代码仓库支持数据样本预览，以验证转换逻辑，并支持对输入样本进行预过滤。
  • 代码仓库支持运行时调试。
  • 在代码仓库中，Spark 模块在作业级别启动。
代码工作区迭代周期
  • 代码工作区旨在辅助探索和分析数据。结果可以共享、发布到仪表板、转化为可复用的转换，或导出至生产就绪的管道工具（如代码仓库或管道构建器）。
  • 代码工作区提供 JupyterLab® 和 RStudio® Workbench IDE 的完全灵活性，包括完整的代码和数据预览。
  • 代码工作区支持逐单元格迭代，可即时反馈代码执行情况。
  • 在代码工作区中，无需 Spark 模块，并提供完全可定制的内核，以便对环境进行临时调整。
代码工作簿迭代周期
  • 代码工作簿旨在辅助从数据中生成洞察。所有转换均在完整输入数据上运行，且 Spark 执行模型针对快速迭代进行了优化。
  • 代码工作簿支持完整数据预览。
  • 代码工作簿提供控制台支持，用于对转换进行临时分析。
  • 在代码工作簿中，Spark 模块会保持热状态以实现即时交互，并在工作簿级别启动。
代码仓库操作
  • 代码仓库支持 Foundry 数据管理库和自定义 Python 库。
  • 代码仓库支持数据期望、发布自定义库以及调用自定义库。
  • 代码仓库不支持点选式代码模板。
代码工作区操作
  • 代码工作区可以调用 pip、CRAN 和 conda 库（包括从代码仓库发布的库），并且可以快速修改环境。
  • 代码工作区不支持数据期望或发布自定义库。
  • 代码工作区不支持点选式代码模板。
代码工作簿操作
  • 代码工作簿可以调用从代码仓库发布的自定义库，用户可以将逻辑片段保存为代码模板，以便其他用户进行点选式分析。
  • 代码工作簿不支持数据期望或发布自定义库。
  • 代码工作簿确实会为某些 Spark 环境调用自定义库。
  • 代码工作簿支持点选式模板。
代码仓库变更管理
  • 代码仓库优先考虑变更可追溯性和数据治理，以确保关键管道保持安全与稳健。
  • 代码仓库提供完整的变更日志。
  • 代码仓库提供完整的 Git 工作流、安全标记管理与移除、影响分析视图、高级代码审查工作流以及单元测试。
  • 代码仓库不支持在合并后复制数据。
代码工作空间变更管理
  • 代码工作空间优先支持快速灵活的迭代，提供完整的分支支持与自动 Git 版本控制。
  • 代码工作空间完全由代码仓库提供底层支持，并可得益于其完整的 Git 工作流。
  • 代码工作空间不支持在合并后复制数据。
  • 代码工作空间会将其笔记本内容的安全检查点保留 30 天，允许用户安全地保留和检索任何给定状态，同时也支持在 Git 仓库中永久存储代码备份。
代码工作簿变更管理
  • 代码工作簿通过轻量级的分支工作流，优先支持快速迭代与协同工作。代码工作簿不需要 CI 检查或单元测试。
  • 代码工作簿支持在合并后复制数据。
  • 代码工作簿不提供完整的 Git 工作流、安全标记管理与移除、影响分析视图、高级代码审查工作流或单元测试功能。
