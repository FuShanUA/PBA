全局分支

全局分支（以前称为“Foundry 分支”）使您可以在 Palantir 平台上开发和测试端到端的工作流，而这些工作流通常会破坏生产环境的运行。

全局分支提供了一个统一的体验，让您在单个分支上对多个应用程序进行修改，在不干扰生产环境的情况下对这些更改进行端到端测试，并通过一次点击合并这些更改。

全局分支示例工作流程

您可以在一个分支上对 支持的资源类型 进行更改。您还可以在不将更改写入 main 分支的情况下对分支执行操作。

Global Branching还支持审查流程；开发人员可以根据每个资源的审批策略为不同的资源添加审查者。一旦更改最终确定并获得批准，就可以合并到 main 分支中，从而以安全且受控的方式更新和改进您的工作流程。

Global Branching vs. release management

Foundry -> Foundry Apollo -> Apollo Gotham -> Gotham AIP -> AIP Ontology -> 本体论 ontology -> 本体论 Pipeline -> 管道 pipeline -> 管道 Transform -> 转换 transform -> 转换 Workshop -> Workshop workshop -> Workshop Code Repositories -> 代码仓库 Code Workspaces -> 代码工作区 Data Integration -> 数据集成 Model Integration -> 模型集成 Object -> 对象 object -> 对象 Dataset -> 数据集 dataset -> 数据集 Tenant -> 租户 tenant -> 租户 Marking -> 标记 marking -> 标记 Permission -> 权限 permission -> 权限 Role -> 角色 role -> 角色 Function -> 函数 function -> 函数 Action -> 操作 action -> 操作 Property -> 属性 property -> 属性 Interface -> 接口 interface -> 接口 内容： 全局分支功能允许在单独的分支上进行工作流更改的开发和测试。这在管理开发环境中的更改时非常理想，因为它允许开发人员独立工作，并且仅在功能完成时才将更改合并到 main 分支中。 发布管理 是在服务于不同目的的不同环境中管理资源多个版本的过程。

发布管理 和全局分支可以和谐地协同工作。它们不应被视为解决同一问题的替代方案，而是解决不同问题的补充方案。

例如，当开发需要添加到工作流中的大型功能时，可以同时使用发布管理和全局分支。较大的功能可能需要几周时间进行开发，并需要对数据集和对象类型模式进行基础性更改。您可以在全局分支上开发此功能，并在完成后将更改合并到开发环境中。然后，您可以使用发布管理将这些更改部署到您的测试和生产环境。
