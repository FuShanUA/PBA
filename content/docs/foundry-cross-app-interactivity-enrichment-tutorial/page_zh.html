在 Foundry 和 Gotham 之间实现拖放

要使用数据扩充功能，您的注册实例必须同时部署 Gotham 和 Foundry，并启用类型映射。如果您的应用仅在单独的 Gotham 或单独的 Foundry 环境中执行拖放操作，则无需实现数据扩充。

以下教程概述了实现数据扩充所需的步骤。请注意，虽然文中包含了向拖拽区和放置区添加扩充功能的章节，但根据您的工作流，扩充操作只需在拖拽区或放置区其中之一执行即可。更多信息请参阅《扩充指南》。

拖放扩充教程

具体步骤如下：

  1. 验证 Data Bank Service 是否返回同义数据。
  2. 创建扩充实用程序函数。
  3. 向拖拽区添加扩充或向放置区添加扩充。

建议安装我们平台的最新版本，以增强您的应用与 Palantir 平台之间的拖放互操作性。

验证 Data Bank Service 是否返回同义数据

要实现数据扩充，首先必须验证 Data Bank Service 是否按预期返回同义数据。

支持的媒体类型

目前，Data Bank Service 支持扩充以下列表中的媒体类型，并且支持使用 Foundry 对象的媒体类型扩充 Gotham 对象 RID 媒体类型的数据，反之亦然。

"application/x-vnd.palantir.rid.phonograph2-objects.object",
"application/x-vnd.palantir.rid.gotham.object",

创建扩充实用工具函数

  1. 创建一个向 Data Bank Service 发起请求的函数。以下示例展示了如何使用 Fetch API ↗ 为 Data Bank Service 请求创建请求选项。

2function getRequestOptions(
3    bearerToken
4    // serializedData 是一个映射，将媒体类型映射到其对应的序列化数据版本
5    serializedData
6){
7    return {
8        method: "PUT",
9        headers: new Headers({
10            "Authorization": bearerToken,
11            "Content-Type": "application/json"
12        }),
13        body: JSON.stringify({
14            "dataTransfers": [ serializedData ]
15        }),
16        redirect: "follow"
17    };
18}

21// 将 <BEARER TOKEN> 替换为你的 Bearer Token
22const requestOptions = getRequestOptions(

2. 调用数据银行服务的数据丰富接口，获取可用于丰富拖拽区的同名媒体类型及该媒体类型的附加数据：

1// 将 <HOSTNAME> 替换为您注册的 HOSTNAME
2const batchEnrichDataTransferURL =
3    "https://<HOSTNAME>/data-bank-service/api/data-transfer/batchEnrichDataTransfer";

6// serializedData 的类型为 {[mediaType: string]: string}
7function getEnrichedData(
8    bearerToken,
9    serializedData
10){
11    // 调用步骤 1 中定义的函数
12    const requestOptions = getRequestOptions(bearerToken, serializedData);
13

15        await fetch(batchEnrichDataTransferURL, requestOptions)
16            .then((response) => response.json())
17            .catch((error) => console.error(error));

向拖拽区添加数据增强

请注意，数据增强只需添加到拖拽区或放置区即可。请查阅数据增强指南，以确定本节教程是否与您的工作流相关。

  1. 为了向拖拽区添加数据增强，请调用上一节第二步中的 getEnrichedData 函数。数据增强操作应在页面或组件挂载时执行，而不是在 dragstart 处理程序中进行。这是因为您无法在 dragstart 处理程序中执行耗时的阻塞调用。返回的增强数据将包含请求中的原始数据以及额外的同义数据。

  2. 在 dragstart 处理程序中，将获取的增强数据添加到拖拽事件的 DataTransfer 中。将此数据添加到拖拽事件的 DataTransfer 后，如果数据增强成功，用户便可将此载荷拖拽到 Foundry 和 Gotham 的放置区中。以下代码展示了如何使用 Gotham 对象 ID 请求增强数据，并使用 JSON.stringify ↗ 将数据转换为字符串格式。



Copied!
1const gothamObjectIds = ["ri.gotham.XXXXXXXX", "ri.gotham.YYYYYYYY", "ri.gotham.ZZZZZZZZZZ"];
2
3async function getEnrichedFoundryData(){
4    const enrichedData = await getEnrichedData(
5        "<BEARER TOKEN>",
6        {"application/x-vnd.palantir.rid.gotham.object": JSON.stringify(gothamObjectIds)}
7    );
8
9    const enrichedFoundryIdData = enrichedData?.["application/x-vnd.palantir.rid.phonograph2-objects.object"]
10
11    if (enrichedFoundryIdData != null){
12        try {
13            return JSON.parse(enrichedFoundryIdData);
14        } catch (error) {
15            // we were unable to parse the data
16            console.error(error);
17            return undefined;
18        }
19    }
20}
21
22let enrichedFoundryIdData = null;
23
24// here we asynchronously update enriched data
25// this is necessary because we cannot implement enrichment in the dragstart handler, as that would be blocking
26getEnrichedFoundryData().then((data) => {
27    enrichedFoundryIdData = data;
28});
29
30async function handleDragStart(event) {
31    event.dataTransfer.setData("application/x-vnd.palantir.rid.gotham.object", JSON.stringify(gothamObjectIds));
32
33    if(enrichedFoundryIdData != null){
34        event.dataTransfer.setData("application/x-vnd.palantir.rid.phonograph2-objects.object", enrichedFoundryIdData)
35    }
36
37    // event.dataTransfer now contains the original Gotham object media type data
38    // as well as synonymous foundry data
39    event.dataTransfer.effectAllowed = "move";
40}

使用上述代码，如果你拥有带有 Foundry 同义对象的 Gotham 对象 ID，你可以将同义数据添加到 DataTransfer 中，这样你的拖拽载荷就能同时被 Gotham 和 Foundry 的放置区所接受。

完成此步骤后，你应该能够将你的拖拽区拖放到 Gotham 和 Foundry 的放置区中。请参阅《Palantir 媒体类型文档》，以查找可以接受你的拖拽区的放置区。

向放置区添加数据扩充

请注意，只需向拖拽区或放置区添加数据扩充即可。请查阅《数据扩充指南》，以确定本教程的这一部分是否与你的工作流相关。

要向你的放置区添加数据扩充，你需要对来自事件 DataTransfer 的数据进行扩充。在下面的代码片段中，代码期望获取 Gotham 对象 ID，但如果 DataTransfer 上没有 Gotham 对象媒体类型，则可以从 DataTransfer 中扩充数据。为简便起见，假设下面的放置处理程序首先尝试直接从 DataTransfer 访问 Gotham 对象数据，失败后再转向下面代码块中的扩充逻辑。


```typescript
function handleDrop(event) {
    // 阻止默认的打开链接行为
    event.preventDefault();

    // 执行一些样式清理操作
    removeStylingFromDropZone("valid-small-hover");
    removeStylingFromDropZone("valid-small");
    dropHoveringOverDropZoneCount = 0;
    dragInsideApplicationCount = 0;

    // <尝试从 DataTransfer 访问 Gotham 对象数据，若成功则提前返回，否则继续执行后续逻辑>

    // Foundry 对象 RID 媒体类型数据可以富集为 Gotham 对象媒体类型数据
    const foundryData = event.dataTransfer.getData(
        "application/x-vnd.palantir.rid.phonograph2-objects.object"
    );

    try {
        if(foundryData != null && foundryData !== ""){
            // 尝试解析返回的数据并发送富集请求，
            // 如果无法解析数据，则说明格式有误
            const foundryDataParsed = JSON.parse(foundryData);

            const serializedData = {
                ["application/x-vnd.palantir.rid.phonograph2-objects.object"]: foundryDataParsed
            }

            const enrichedData = await getEnrichedData(
                "",
                serializedData
            );

            if(enrichedData?.["application/x-vnd.palantir.rid.gotham.object"] != null){
                doCoolThingWithGothamObjectIds(
                    enrichedData["application/x-vnd.palantir.rid.gotham.object"]
                );
            }
        }
    } catch (error) {
        console.error("无法解析数据", error)
    }
}
```

此时，您的放置区应能同时接收 Gotham 和 Foundry 的数据。请查阅 《Palantir 媒体类型文档》，寻找可拖拽至您放置区的拖拽源。


结论


借助上述代码片段，您可以向 Data Bank Service 端点发送请求以丰富数据，使其包含在 Gotham 和 Foundry 之间通用的同义词。这样一来，可拖拽数据即可同时被 Gotham 和 Foundry 的放置区接收，放置区也能够同时接受 Foundry 与 Gotham 的数据。
