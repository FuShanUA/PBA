在Foundry和Gotham之间实现拖放功能

要访问数据增强功能，您的注册必须同时包含Gotham和Foundry，并且启用了类型映射。如果您的应用程序仅使用Gotham或仅使用Foundry进行拖放操作，则不需要实现数据增强。

本教程将概述实现数据增强所需的步骤。请注意，有添加增强功能到拖放区域的部分，但根据您的工作流程，只需在拖放区域 或 放置区域之一进行增强即可。有关更多信息，请参阅 增强指南。

拖放增强教程

步骤如下：

  1. 验证 Data Bank 服务返回同义数据。
  2. 创建增强工具函数。
  3. 将增强添加到拖拽区域或将增强添加到放置区域。

我们建议安装我们平台的最新版本，以提高您的应用程序与 Palantir 平台之间的拖放互操作性。

验证 Data Bank 服务返回同义数据

要实现数据增强，您首先必须验证 Data Bank 服务是否如预期返回同义数据。

支持的媒体类型

目前，Data Bank Service 支持丰富以下列表中的媒体类型，并且目前可以将 Foundry 对象 RID（对象资源标识符） 媒体类型的数据与 Gotham 对象（对象标识符） 媒体类型的数据进行丰富，反之亦然。

"application/x-vnd.palantir.rid.phonograph2-objects.object",
"application/x-vnd.palantir.rid.gotham.object",

所需的数据结构

以下是 Data Bank Service 请求所需的示例数据结构。请注意，JSON 对象必须首先被序列化用于 Data Bank Service 请求；我们在接下来的步骤中概述了如何使用 JSON.stringify ↗ 进行此操作。请求格式如下：

Copied!
1{
2    "dataTransfers": [serializedData1, serializedData2, ...]
3}

serializedData 在上述代码片段中是指在序列化后发送到 Data Bank Service 以进行丰富处理的数据。上面的 dataTransfers 对象不应与 DataTransfer 对象（数据传输对象） 混淆。 ，它们名称相似但用途不同。上面的对象包含我们想要增强的数据，增强操作可能在将数据写入DataTransfer对象之前发生，或者在DataTransfer对象被放置在放置区域之后发生。

每个 serializedData 对象的结构如下：

Copied!
1{<media type>: <serialized data>}

例如，如果您有Gotham对象标识符，并且希望用同义的Foundry数据来增强这些数据，您的序列化数据表示如下：

Copied!
1{
2    "dataTransfers": [
3        {
4            "[\"application/x-vnd.palantir.rid.gotham.object\"]: [\"ri.gotham.XXXXXXXX\", \"ri.gotham.YYYYYYYY\", \"ri.gotham.ZZZZZZZZZZ\"]"
5        }
6    ]
7}

在上面的代码片段中，媒体类型是 application/x-vnd.palantir.rid.gotham.object，后面跟着该媒体类型的数据数组。数据银行服务将返回映射的同义媒体类型以及该媒体类型的数据。请注意，引号（"）必须使用 \ 进行转义，以正确格式化序列化的数组。

在下面的curl请求中使用此请求格式，将 application/x-vnd.palantir.rid.gotham.object 媒体类型替换为您的媒体类型，并添加您计划增强的数据。将 <BEARER TOKEN> 替换为您的令牌，并将 <HOSTNAME> 替换为您的注册主机名。请注意，媒体类型的数据是字符串格式，并使用 \ 字符来转义内部引号。

Copied!
1curl --location --request PUT "https://<HOSTNAME>/data-bank-service/api/data-transfer/batchEnrichDataTransfer" \
2--header 'Content-Type: application/json' \
3--header 'Authorization: Bearer <BEARER TOKEN>' \
4--data '{
5    "dataTransfers": [
6        {
7           "application/x-vnd.palantir.rid.gotham.object": "[\"ri.gotham.XXXXXXXX\", \"ri.gotham.YYYYYYYY\", \"ri.gotham.ZZZZZZZZZZ\"]"
8        }
9    ]
10}'

如果您的数据有同义词，您应该会得到类似以下的响应：

Copied!
1{
2    "dataTransfers": [
3        {
4            "dataTransfer": {
5                "application/x-vnd.palantir.rid.gotham.object": "[\"ri.gotham.XXXXXXXX\", \"ri.gotham.YYYYYYYY\", \"ri.gotham.ZZZZZZZZZZ\"]",
6                "application/x-vnd.palantir.rid.phonograph2-objects.object": "[\"ri.phonograph2-objects.main.object.XXXXXXXX\", \"ri.phonograph2-objects.main.object.YYYYYYYY\", \"ri.phonograph2-objects.main.object.ZZZZZZZZZZ\"]",
7            },
8            "errors": []
9        }
10    ]
11}

请注意上面提到的 dataTransfers 对象，Gotham 媒体类型 application/x-vnd.palantir.rid.gotham.object 后面跟着该媒体类型的数据，以及 Foundry 媒体类型 application/x-vnd.palantir.rid.phonograph2-objects.object 后面跟着该媒体类型的数据。

如果您没有为您的对象获得同义词，请确认您的注册和本体论已启用类型映射。如果您的本体论确实启用了类型映射，请按照 类型映射文档 确认是否已为您的对象类型启用了类型映射。我们还建议验证您发送的数据是否映射到预期的同义媒体类型，并且该数据受到数据银行服务的支持。

在本教程的其余部分中，我们将分解使用此请求的步骤，并向您展示如何使用响应来丰富您的拖放工作流。

既然您已经确认数据银行服务为您的数据提供了同义词，就可以继续本教程，为您的应用程序添加增强功能。

创建增强工具函数

  1. 创建一个向数据银行服务发起请求的函数。以下是使用 Fetch API ↗ 创建数据银行服务请求选项的示例。
Copied!
1// create a function for creating your request options object using your serialized data
2function getRequestOptions(
3    bearerToken
4    // serializedData is a mapping of media types to serialized versions of their data
5    serializedData
6){
7    return {
8        method: "PUT",
9        headers: new Headers({
10            "Authorization": bearerToken,
11            "Content-Type": "application/json"
12        }),
13        body: JSON.stringify({
14            "dataTransfers": [ serializedData ]
15        }),
16        redirect: "follow"
17    };
18}
19
20// use this function as follows
21// replace <BEARER TOKEN> with your Bearer Token
22const requestOptions = getRequestOptions(
23    "<BEARER TOKEN>",
24    {
25        ["application/x-vnd.palantir.rid.gotham.object"]:
26        "[\"ri.gotham.XXXXXXXX\", \"ri.gotham.YYYYYYYY\", \"ri.gotham.ZZZZZZZZZZ\"]"
27    }
28);
  1. 调用数据银行服务的增强端点，以获取可用于丰富拖放区域的额外同义媒体类型和该媒体类型的其他数据：
Copied!
1// replace <HOSTNAME> with your enrollment HOSTNAME
2const batchEnrichDataTransferURL =
3    "https://<HOSTNAME>/data-bank-service/api/data-transfer/batchEnrichDataTransfer";
4
5// bearerToken is of type string
6// serializedData is of type {[mediaType: string]: string}
7function getEnrichedData(
8    bearerToken,
9    serializedData
10){
11    // call the function defined in Step 1
12    const requestOptions = getRequestOptions(bearerToken, serializedData);
13
14    const enrichedData =
15        await fetch(batchEnrichDataTransferURL, requestOptions)
16            .then((response) => response.json())
17            .catch((error) => console.error(error));
18
19    // since we are only enriching one data transfer, grab the first entry
20    const enrichedFirstDataTransfer = enrichedData.dataTransfers?.[0]?.dataTransfer;
21    return enrichedFirstDataTransfer;
22}

此端点返回一个序列化的对象列表，每个对象都包含解析后的增强数据以及与该增强相关的潜在错误列表。此端点返回的数据格式如下：

Copied!
1    {
2        "dataTransfers":
3            [
4                {
5                    "dataTransfer": {"<media type>": "<serialized data>"},
6                    "errors": "[<error1>, <error2>...]"
7                },
8                {
9                    "dataTransfer": {"<media type>": "<serialized data>"},
10                    "errors": "[<error1>, <error2>...]"
11                },
12            ...
13            ]
14    }

需要注意的一个潜在错误是请求的媒体类型不被数据银行服务支持。

向拖拽区域添加增强信息

请注意，只需将增强信息添加到拖拽或释放区域。请查阅增强信息指南，以确定本教程的此部分是否与您的工作流程相关。

  1. 要向您的拖拽区域添加增强信息，请从上一节的第二步中调用getEnrichedData函数。增强信息应在您的页面或组件挂载时发生，而不是在dragstart处理程序中。这是因为您不能在dragstart处理程序中进行昂贵且阻塞的调用。返回的增强数据将包含您请求中的原始数据以及额外的同义数据。

  2. 在dragstart处理程序中，将获得的增强数据添加到拖拽事件的DataTransfer中。当此数据被添加到拖拽事件的DataTransfer中时，如果增强成功，用户可以将此负载拖放到Foundry和Gotham释放区域。下面的代码展示了如何使用Gotham对象ID请求增强数据，并使用JSON.stringify ↗将数据转换为字符串格式。

Copied!
1const gothamObjectIds = ["ri.gotham.XXXXXXXX", "ri.gotham.YYYYYYYY", "ri.gotham.ZZZZZZZZZZ"];
2
3async function getEnrichedFoundryData(){
4    const enrichedData = await getEnrichedData(
5        "<BEARER TOKEN>",
6        {"application/x-vnd.palantir.rid.gotham.object": JSON.stringify(gothamObjectIds)}
7    );
8
9    const enrichedFoundryIdData = enrichedData?.["application/x-vnd.palantir.rid.phonograph2-objects.object"]
10
11    if (enrichedFoundryIdData != null){
12        try {
13            return JSON.parse(enrichedFoundryIdData);
14        } catch (error) {
15            // we were unable to parse the data
16            console.error(error);
17            return undefined;
18        }
19    }
20}
21
22let enrichedFoundryIdData = null;
23
24// here we asynchronously update enriched data
25// this is necessary because we cannot implement enrichment in the dragstart handler, as that would be blocking
26getEnrichedFoundryData().then((data) => {
27    enrichedFoundryIdData = data;
28});
29
30async function handleDragStart(event) {
31    event.dataTransfer.setData("application/x-vnd.palantir.rid.gotham.object", JSON.stringify(gothamObjectIds));
32
33    if(enrichedFoundryIdData != null){
34        event.dataTransfer.setData("application/x-vnd.palantir.rid.phonograph2-objects.object", enrichedFoundryIdData)
35    }
36
37    // event.dataTransfer now contains the original Gotham object media type data
38    // as well as synonymous foundry data
39    event.dataTransfer.effectAllowed = "move";
40}

使用上面的代码，如果你有Foundry同义词的Gotham对象ID，你可以将同义数据添加到你的DataTransfer中，这样你的拖拽负载就可以被Gotham和Foundry的放置区域接受。

完成此步骤后，你应该能够将你的拖拽区域拖放到Gotham和Foundry的放置区域中。参考 Palantir媒体类型文档 来找到可以接受你拖拽区域的放置区域。

为放置区域添加增强信息

请注意，只需在拖拽 或 放置区域中添加增强信息即可。查阅增强信息 指南 来确定本教程的这一部分是否适用于您的工作流程。

要向您的放置区域添加增强功能，您需要从事件的DataTransfer中丰富数据。在下面的代码片段中，代码期望Gotham对象ID，但如果DataTransfer上没有Gotham对象媒体类型，则可以使用DataTransfer中的数据进行丰富。为简单起见，假设下面的放置处理程序首先尝试直接从DataTransfer访问Gotham对象数据，如果失败，则继续执行下面代码块中的丰富操作。

Copied!
1function handleDrop(event) {
2    // prevent default open-link behavior
3    event.preventDefault();
4
5    // perform some styling cleanup
6    removeStylingFromDropZone("valid-small-hover");
7    removeStylingFromDropZone("valid-small");
8    dropHoveringOverDropZoneCount = 0;
9    dragInsideApplicationCount = 0;
10
11    // <Attempt to access Gotham object data from the DataTransfer, which returns early on success and continues otherwise>
12
13    // The Foundry object RIDs media type data can be enriched to Gotham object media type data
14    const foundryData = event.dataTransfer.getData(
15        "application/x-vnd.palantir.rid.phonograph2-objects.object"
16    );
17
18    try {
19        if(foundryData != null && foundryData !== ""){
20            // try to parse the returned data & send enrichment request,
21            // if we can't parse the data it must be mis-formatted
22            const foundryDataParsed = JSON.parse(foundryData);
23
24            const serializedData = {
25                ["application/x-vnd.palantir.rid.phonograph2-objects.object"]: foundryDataParsed
26            }
27
28            const enrichedData = await getEnrichedData(
29                "<BEARER TOKEN>",
30                serializedData
31            );
32
33            if(enrichedData?.["application/x-vnd.palantir.rid.gotham.object"] != null){
34                doCoolThingWithGothamObjectIds(
35                    enrichedData["application/x-vnd.palantir.rid.gotham.object"]
36                );
37            }
38        }
39    } catch (error) {
40        console.error("Unable to parse data", error)
41    }
42}

此时，您的放置区域应该能够接受Gotham和Foundry数据。参考 Palantir媒体类型文档，找到可以拖放到您的放置区域上的拖放区域。

结论

使用上面的代码片段，您可以向数据银行服务端点发送请求，以丰富您的数据，使其包含在Gotham和Foundry之间兼容的同义词。这使得可拖动的数据可以被Gotham和Foundry的投放区域接受，并且可以使投放区域接受Foundry和Gotham的数据。
