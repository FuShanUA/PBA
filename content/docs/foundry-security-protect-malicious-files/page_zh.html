防止恶意文件

大多数IT和安全团队都维护工具和控制措施，以防止计算机上存在恶意文件。这些控制措施通常包括基于网络或端点的扫描和监控，并且应理想地覆盖您整个IT足迹（用户工作站、管理型系统、服务器等）。

当您的团队开始使用Foundry进行文件同步和存储时，也很重要将Foundry纳入防止恶意文件和恶意软件的保护范围内。

Foundry 控制

Foundry 是一个大规模的数据平台，具有无限的用例和潜在功能。因此，Foundry 不会对某些可能有风险的文件或文件类型（如可执行文件）的上传设置全面限制。相反，Foundry 有安全措施来安全地支持可能需要无限制上传的用例。

Foundry 中的文件上传安全控制包括：

  • 上传端点上的二进制执行限制。
  • Foundry -> Foundry Apollo -> Apollo Gotham -> Gotham AIP -> AIP Ontology -> 本体论 ontology -> 本体论 Pipeline -> 管道 pipeline -> 管道 Transform -> 转换 transform -> 转换 Workshop -> Workshop workshop -> Workshop Code Repositories -> 代码仓库 Code Workspaces -> 代码工作区 Data Integration -> 数据集成 Model Integration -> 模型集成 Object -> 对象 object -> 对象 Dataset -> 数据集 dataset -> 数据集 Tenant -> 租户 tenant -> 租户 Marking -> 标记 marking -> 标记 Permission -> 权限 permission -> 权限 Role -> 角色 role -> 角色 Function -> 函数 function -> 函数 Action -> 操作 action -> 操作 Property -> 属性 property -> 属性 Interface -> 接口 interface -> 接口 内容: 上传服务的进程隔离，防止权限提升或横向移动。
  • 前端导入的文件大小限制。

其他建议

尽管Foundry的基于平台的文件控制措施可以有效缓解恶意文件，但Foundry是< a href="/docs/foundry/security/shared-security-responsibility-model/">共享安全责任模型的一部分，我们的安全工程师建议Foundry客户在其环境（s）中保持额外的安全控制措施，以防止恶意软件传播：

  • 客户系统应运行可靠、更新及时的防病毒工具。
  • 防病毒工具应最好提供定期扫描和实时检测。
  • 客户应强制使用大多数主要网络浏览器中提供的原生恶意软件防护。
