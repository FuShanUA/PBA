AIP Assist 应用集成

AIP Assist 在 Palantir 平台上与其他应用程序提供了多种集成点，我们正在持续评估这些集成以改进并扩大 AIP Assist 的影响力。这些集成简化了常见的工作流程，并降低了从 AIP Assist 获取帮助的摩擦成本。如果您有任何关于附加、高价值集成的想法，请通过在我们的开发者社区论坛的 参与讨论 ↗ 部分发帖告知我们。AIP Assist 应用集成设计为在平台内高度可见，但您也可以查阅下面的参考文档以探索可用的集成。

AIP Assist 在代码仓库中

向 AIP Assist 提问

在 Code Repositories 中查看代码文件时，您将看到 Ask AIP Assist 助手，它提供了一系列预配置的、与代码相关的操作，AIP Assist 可以执行这些操作来帮助开发。

代码仓库附件

在 Code Repositories 中，AIP Assist 侧边栏显示了配置选项，允许您包含附件以向 AIP Assist 提供额外上下文。您可以选择包括多种不同的组件，包括整个仓库、一个或多个文件，或者一段突出显示的代码。

通过此功能，您可以要求 AIP Assist 执行以下操作：

  • 解释不同文件之间的关系。
  • 优化代码片段。
  • 在仓库中搜索特定代码。
  • 总结或解释代码片段、文件或整个仓库。

AIP Assist还将能够访问附件中引用的数据集和对象的元数据，使用户可以针对本体论（Ontology）对象和输入或输出数据集提出具体问题。

AIP Assist在Carbon工作区

您可以在Carbon工作区中选择启用AIP Assist，以允许用户与专门支持特定用户组的AIP Assist聊天机器人（以前称为AIP Assist代理）进行交互。这需要在AIP聊天机器人工作室中配置专用的AIP Assist聊天机器人，以便用户进行交互。

有关更多信息，请参阅 在Carbon工作区中启用AIP Assist。

AIP Assist 在轮廓中

在使用轮廓中的表达式板时，你可以让 AIP Assist 执行与编写表达式相关的几种不同操作，例如解释代码、查找错误以及将代码翻译成表达式。

AIP Assist 在工作区

发送至 AIP Assist

工作区按钮小部件允许开发者添加一个 Send to AIP Assist 事件，这将打开 AIP Assist 并向其提出一个预配置的提示，该提示基于静态文本或动态变量值。

此功能在与AIP Assist Agents（AIP助手代理）配合使用时非常强大，这些代理是针对有关Workshop应用程序的自定义文档进行训练的，其中“发送到AIP Assist”按钮小部件被使用。在配置Workshop中的Send to AIP Assist事件时，可以设置默认的AIP Assist Agent，在响应配置的提示时将使用该代理。

有关更多信息，请参阅Send to AIP Assist事件 文档。

AIP Assist in Slate

向AIP Assist提问

您可以使用slate.askAIPAssist操作打开 AIP Assist，并指定一个可选的prompt字符串参数，该参数将作为用户的消息发送给AIP Assist。prompt可以从当前用户的应用状态中推导出来，以实现对AIP Assist的针对性提问。

有关更多信息，请参阅 Slate Ask AIP Assist 文档。

AIP Assist 在问题中

AIP Assist 在问题提交表单中

为了便于平台内支持，AIP Assist 已集成到 Issues 应用中的问题提交流程中。用户可以选择 打开 AIP Assist 并立即获得帮助，在提交问题之前咨询 AIP Assist。

AIP Assist 在问题中

如果您已经创建了一个问题，可以使用集成的AIP Assist工具获得即时支持。AIP Assist可以根据问题信息和用户的问题提供答案，或者对冗长的问题进行总结以优化问题解决。

AIP Assist在本体论管理器中

AIP Assist已与本体论管理器集成，以帮助在本体论更新期间解决错误。在更新本体论时，请导航到错误选项卡，并在某个错误上选择用AIP Assist解释，以便AIP Assist提供错误解决建议和错误说明。
