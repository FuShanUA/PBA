授权角色

摘要

Palantir Foundry Connector 2.0 for SAP（以下简称“连接器”）使用四种类型的角色。

  • 服务角色：连接器运行所必需的角色。请勿修改这些角色。
    • /PALANTIR/SERVICE_SLT：SLT 的 Palantir 服务角色
    • /PALANTIR/SERVICE_SLT_740：SAP_BASIS 7.4 的 Palantir 服务角色（技术组件版本，适用于 SAP NetWeaver 和 SAP S/4HANA 系统）
    • /PALANTIR/SERVICE_USER：Palantir 服务角色
  • 内容角色：从 SAP 系统提取数据所必需的角色。可根据业务需求复制和修改这些角色。
    • /PALANTIR/CONTENT_BEX_ALL：BEx 访问角色
    • /PALANTIR/CONTENT_DM_ALL：数据模型访问角色
    • /PALANTIR/CONTENT_EXT_ALL：提取器内容访问
    • /PALANTIR/CONTENT_FUNCTION_ALL：函数访问角色
    • /PALANTIR/CONTENT_INFOPROV_ALL：InfoProvider 内容访问
    • /PALANTIR/CONTENT_SLT_ALL：SLT 访问角色
    • /PALANTIR/CONTENT_TABLE_ALL：表内容访问
    • /PALANTIR/CONTENT_TCODE_ALL：事务代码内容访问
    • /PALANTIR/CONTENT_HANAVIEW_ALL：HANA 信息视图内容访问
    • /PALANTIR/CONTENT_CDS_ALL：ABAP CDS 视图内容访问
  • 回写角色：如果启用从 Foundry 向 SAP 系统回写数据，则需要这些角色。
    • /PALANTIR/OAUTH_CLIENT：Palantir Foundry OAuth 2.0 客户端角色
  • 监控与调试角色：需要这些角色将 SAP 系统信息暴露给 Foundry，以实现远程监控。
    • /PALANTIR/MONITORING：监控
    • /PALANTIR/DEBUG_USER：调试用户访问

服务角色

  • /PALANTIR/SERVICE_USER：用户运行连接器服务的基础角色。对连接器的每次请求都会检查此角色。此角色中的权限对象如下：
    • /PALAU/SRV：Palantir 服务权限对象，包含 03（显示）和 16（执行）活动。
    • S_BTCH_JOB：（后台处理：后台作业操作）连接器运行后台作业（例如分页或清理操作），因此需要此权限对象。
    • S_TCODE：（事务启动时的事务代码检查）仅用于 SU53 和 ST22 事务（SAP 中的权限检查工具和运行时错误），这是调试和正常日志记录机制所必需的。
    • S_RFC_ADM：（RFC 目标管理）此对象用于 36（检查）和 39（扩展检查）活动，以确定 RFC 连接是否处于活动状态并检查权限测试。
    • S_RFC：（RFC 访问权限检查）此对象用于在 SLT 和远程代理场景中执行远程函数调用，包含 16（执行）活动。
    • /SDF/E2E：（端到端诊断权限）此对象用于从 Foundry 对提取过程运行跟踪，包含 03（显示）活动。
    • S_ADMI_FCD：（系统权限）此对象用于运行检查和跟踪，包含 PADM（使用事务 SM04、SM50 进行进程管理）、ST0R（分析跟踪）、ST22（跨客户端转储分析）。
    • S_BTCH_JOB：（后台处理：后台作业操作）此对象用于将连接器提取作为后台作业运行，包含除 MODI（修改其他用户的作业）之外的所有活动。
    • S_DATASET：（文件访问权限）此对象用于在 SAP 中使用 SAT 生成和访问跟踪文件，包含 33（读取）、A6（带过滤读取）、A7（带过滤写入）活动。
    • S_TABU_NAM：（通过通用标准工具访问表）此对象用于访问 /PAL*、DMC*、IUUC* 表，包含 03（显示）活动。
    • S_TABU_DIS：（表维护（通过 SM30 等标准工具））此对象用于访问表，包含 03（显示）活动。

仅当连接到 SAP Landscape Transformation Replication Server 时，才需要以下角色：

  • /PALANTIR/SERVICE_SLT：运行 SLT API 需要此角色。此角色中的权限对象如下：
    • S_DMIS：（SAP SLO 数据迁移服务器的权限对象）此对象限于 03（显示）活动，用于检查 API 端点并调用它们。
  • /PALANTIR/SERVICE_SLT_740：如果 SAP SLT 运行在 SAP_BASIS 7.4+ 上，则需要额外的权限对象。这些对象如下：
    • S_DMC_S_R：（MWB：发送方/接收方中的读/写权限）访问 Foundry 的 SLT 队列需要此对象。
    • S_BTCH_ADM：（后台处理：后台管理员）此对象代表 SAP SLT 管理连接器的后台作业，例如复制对象定义、复制过程对象，以及启动向 SLT 队列的复制。
    • S_DMIS：（SAP SLO 数据迁移服务器的权限对象）此对象限于 03（显示）活动，用于检查 API 端点并调用它们。
    • S_DMIS_MOM：（MWB / 迁移对象建模器的权限）

内容角色

这些角色作为示例提供。请通过复制这些角色并限制对系统中所需对象的访问权限，来调整授权配置文件的内容。

连接器角色

  • /PALANTIR/CONTENT_TABLE_ALL: 此角色用于从数据库表和视图中提取数据。
    • /PALAU/TAB: Palantir 表授权对象：默认通过 * 通配符允许访问所有表。
  • /PALANTIR/CONTENT_DM_ALL: 此角色用于从数据库表中提取数据模型。
    • /PALAU/DMO: Palantir 数据模型授权对象：默认通过 * 通配符允许访问所有表。
  • /PALANTIR/CONTENT_FUNCTION_ALL: 此角色用于运行 SAP 系统的 RFC 函数。根据所使用的业务功能，可能需要额外的授权。
    • /PALAU/FUN: Palantir 函数授权对象：默认通过 * 通配符允许访问所有函数。
  • /PALANTIR/CONTENT_TCODE_ALL: 此角色用于运行 SAP 系统的事务代码（ALV SE38 报表）。根据所使用的业务功能，可能需要额外的授权。
    • /PALAU/TCO: Palantir 事务代码授权对象：默认通过 * 通配符允许访问所有事务代码。
  • /PALANTIR/CONTENT_HANANVIEW_ALL: 此角色用于从 HANA 信息视图中提取数据。
    • /PALAU/HAN: Palantir HANA 授权对象：默认通过 * 通配符允许访问所有 HANA 信息视图。
  • /PALANTIR/CONTENT_CDS_ALL: 此角色用于从 ABAP CDS 视图中提取数据。
    • /PALAU/CDS: Palantir CDS 授权对象：默认通过 * 通配符允许访问所有 CDS 视图。

以下角色仅在连接到 SAP Business Warehouse (BW) 服务器时需要：

  • /PALANTIR/CONTENT_BEX_ALL: 此角色用于从 SAP BW (Business Warehouse) BEx 查询中提取数据。
    • /PALAU/BEX: (Palantir BEx 授权对象)：默认通过 * 通配符允许访问所有 BEx 查询。
  • /PALANTIR/CONTENT_EXT_ALL : 此角色用于从 SAP BW (Business Warehouse) 提取器（启用 ODP）中提取数据。
    • /PALAU/EXT: Palantir 提取器授权对象：默认通过 * 通配符允许访问所有启用 ODP 的提取器。
  • /PALANTIR/CONTENT_INFOPROV_ALL: 此角色用于从 SAP BW (Business Warehouse) InfoProvider 中提取数据。
    • /PALAU/INF: Palantir InfoProvider 授权对象：默认通过 * 通配符允许访问所有 InfoProvider。
    • S_RS_AUTH: 角色中的 BI 分析授权：这是 BW 系统中的所有分析授权。请根据需要进行调整。
    • S_RS_COMP: Business Explorer - 组件：这是 BW 系统中的所有 BEx 对象授权。请根据需要进行调整。
    • S_RS_COMP1: Business Explorer - 组件：数据主人增强功能：这是 BW 系统中的所有 BEx 对象授权。请根据需要进行调整。

以下角色仅在连接到 SAP Landscape Transformation (SLT) 复制服务器时需要：

  • /PALANTIR/CONTENT_SLT_ALL 此角色用于从 SLT 队列中提取数据，这些队列将表从连接的系统复制到 SAP SLT。
    • /PALAU/SLT: (Palantir SLT 授权对象)：默认通过 * 通配符允许访问所有表。

回写角色

  • /PALANTIR/OAUTH_CLIENT: 从 Foundry 向 SAP 系统启用回写功能时需要此角色。它提供了 OAuth 2.0 配置所需的授权对象。
    • S_SERVICE: 在外部服务启动时进行检查。这些服务受 /PALANTIR/* 服务限制。
    • S_OA2C_ADM: OAuth 2.0 客户端配置
    • S_OA2C_USE: OAuth 2.0 客户端使用
    • S_SCOPE: OAuth 2.0 作用域。此角色仅限于 /PALANTIR/SRV_0001 作用域，用于使用 SAP 函数进行 Palantir Foundry 回写。

监控与调试角色

  • /PALANTIR/MONITORING: 需要此角色才能通过连接器对 SAP 系统进行远程监控。它提供了丰富的系统信息，如运行时错误分析（ST22）、SAP 系统日志（SM21）、SAP 后台作业监控（SM37）、授权分析（SU53）、Internet 通信管理器（ICM）、系统资源监控（ST02、ST06）、SAP SLT 控制台（LTRC）、SAP SLT 操作增量队列监控（ODQMON）。
  • /PALANTIR/DEBUG_USER: 发生故障时 Palantir 支持团队需要此角色；开发人员可能需要在 SAP 系统中对问题进行调试。此角色汇集了连接器开发团队所需的全部授权对象。

远程智能体角色

远程智能体角色与主连接器的相应角色一致。这些角色可以在安装了连接器远程智能体的远程系统中维护。

SAP_BASIS 7.0 及以上版本的角色

  • /PALANTIR/CONTENT_RBEX_ALL: 远程 BEx 内容访问
  • /PALANTIR/CONTENT_RFUNCT_ALL: 远程函数内容访问
  • /PALANTIR/CONTENT_RTCODE_ALL: 远程事务码内容访问
  • /PALANTIR/CONTENT_RINFOPRV_ALL: 远程 InfoProvider 内容访问
  • /PALANTIR/CONTENT_RTABLE_ALL: 远程表内容访问
  • /PALANTIR/SERVICE_USER: Palantir 服务角色

Basis 版本 46C、620 或 640 的角色

  • /PALAGT47/CONTENT_RTABLE_ALL: 远程表内容访问
  • /PALAGT47/SERVICE_USER: Palantir 服务角色
