受限视图

标记和角色提供了强大的访问控制能力。然而，某些场景需要更细粒度的权限控制。例如，授予对某类所有对象的访问权限可能不够或并不恰当。某些对象类型需要向不同用户展示不同的对象，就像公司限制销售代表只能查看其所属分支机构的客户一样。受限视图可以提供这种额外层级的访问控制。

用户在 Foundry 中与受限视图资源交互，受限视图由细粒度权限驱动。受限视图将数据集访问限制为仅允许用户查看其有权限访问的行。受限视图构建在底层数据集之上，不能用作转换的输入。受限视图的策略决定了用户可以查看的具体行。通常由具备数据主人角色的用户在创建受限视图时进行定义。创建完成后，受限视图可作为本体中对象类型的底层数据源。例如，如果一行代表一个对象，则受限视图将基于其底层的对象类型来控制用户可以查看哪些对象。

受限视图策略

策略是受限视图的核心。受限视图策略使用细粒度策略来确定用户可以查看哪些行。细粒度策略是一组规则和逻辑运算符，用于比较用户属性、列和值。

  • 用户属性：查看数据的用户的属性。
  • 列名：受限视图底层数据集中的一列。
  • 特定值：字符串、布尔值、数字或数组。

即使不是全部，大多数策略也至少会包含一个与用户属性进行比较的项。基于用户的权限控制至少需要一个这样的项。

在引用用户、用户组或组织时，策略在策略列和策略定义中都需要唯一标识符 (UUID)。为防止与重命名相关的问题，不支持指定名称而必须指定 ID。

创建受限视图

受限视图创建对话框包含以下步骤：

  1. 另存为：选择受限视图在文件系统中的名称和保存位置。
  2. 组合细粒度策略：定义策略语句，以确定用户在访问受限视图或对象时能够查看哪些行或对象。
  3. 审查访问要求：审查上游数据集和下游受限视图中现有的文件级和事务级标记。在具备适当权限的情况下，您可以取消标记（移除）受限视图上的标记。
  4. 汇总：在创建并初始构建受限视图之前，审查您的选择。

另存为

为您的受限视图命名并选择保存位置。通常，您会希望将受限视图保存在与输入数据集不同的项目中。这确保使用受限视图的用户能够在下游项目中拥有查看权限。或者，您也可以将受限视图保存在与输入数据集相同的项目中。使用标记来保护输入数据集。

编写细粒度策略

你可以使用用户属性、列名和特定值来创建基于规则的策略。更多信息请参阅《受限视图策略》。

审查访问要求

对于应仅通过受限视图访问敏感数据的用户，不应授予其对上游数据集的访问权限。在此步骤中，你可以审查数据集与正在创建的受限视图两者的访问要求。如果你拥有相应的标记权限，可以从受限视图中移除继承的标记，及/或对上游数据集应用标记。

摘要

摘要展示了数据集与受限视图最终建议的访问控制。如果你对摘要结果满意，请选择创建以启动受限视图的初始构建。

创建受限视图后，系统会在后台自动创建构建计划，以便在输入数据集更新时随时重新构建。

请查阅《管理文档》，了解如何使用受限视图为对象类型提供支持。

你可以基于包含标记列的数据集创建受限视图。每一行仅对具备必要标记访问权限的用户可见。例如，在下面的受限视图中，用户需要同时拥有 A1 和 A2 才能查看第一行，需要拥有 B1 才能查看第二行。

DataMarkings
Row 1[A1, A2]
Row 2[B1]

请按照以下步骤创建基于标记的受限视图：

  1. 准备一个包含一列或多列标记的数据集，将其作为受限视图进行安全保护。每个单元格必须包含一个由标记 ID 组成的字符串数组 (STRING ARRAY)。详细了解《上游数据集的预期格式》。
  2. 前往数据集预览界面的“列”(COLUMNS) 选项卡，选中该列，点击“添加类型类”，然后输入 marking_type.mandatory，以此对每个标记列进行标注。此步骤并非细粒度权限生效的必要条件，但 Foundry 中的某些界面会据此提示来更恰当地渲染该列。

  1. 基于该数据集创建受限视图。策略规则的左侧应为“用户的标记”。对于右侧，请选择“列”并选中该标记列。如果有多个列，请为每个列创建一条规则，并根据需要使用 AND 或 OR 规则进行组合。

上游数据集的预期格式

创建受限视图所依据的数据集必须包含一列标记 ID。

  • 这些 ID 是通用唯一标识符 (UUID)。
  • 该列必须为字符串数组 (STRING ARRAY) 类型（并包含一系列标记 ID）。
  • 你可以拥有多个标记 ID 列。
  • 你可以在同一列中混合使用标记和组织。

将受限视图添加到 Marketplace 产品中

使用 Foundry DevOps 将受限视图纳入 Marketplace 产品，供其他用户安装和复用。了解如何创建您的第一个产品。

支持的功能

目前仅支持 string 和 boolean 常量。常量只能与字段（列）或“用户组”用户属性进行比较。Marketplace 暂不支持针对同一字段使用多个“字段-常量”比较条件。

向产品添加受限视图

要向产品添加受限视图，请先 创建产品，然后 添加输出。选择 添加文件 选项，从 Compass 文件系统中找到该受限视图并将其添加到您的产品中。

将受限视图添加到产品时，打包的是该受限视图的 策略，而不是数据本身。

向分支添加并合并受限视图 [实验性]

在分支版本控制中对受限视图的支持尚处于 实验性 开发阶段，您的租户环境可能尚未提供此功能。详细了解分支机制。

在分支上添加受限视图

要向分支添加受限视图，请在下拉菜单中选择该分支。接着，使用 添加资源 选项将受限视图添加到分支中。这允许您将受限视图作为已修改资源纳入分支，从而构建或修改其策略。请注意，底层数据集上必须已存在该分支，否则您可能会遇到报错。

此步骤必不可少，因为默认情况下，访问分支上的受限视图会展示主分支的内容。然而，如果您已切换至某分支并编辑了受限视图的策略或标记，系统会自动将这些编辑内容连同受限视图一并添加到该分支。另外需要注意的是，受限视图被添加到分支时会自动触发构建。

默认情况下，数据主人和编辑者有权向分支添加受限视图。数据主人可以编辑策略和标记以添加分支受限视图，而编辑者只能构建受限视图。这使得编辑者能够将受限视图添加到其分支工作流并进行测试，而无需具备该受限视图的数据权属。

查看分支受限视图

分支受限视图的权限与父级受限视图保持一致。如果您有权查看原始受限视图，即可查看其对应的分支受限视图。需要注意的是，如果分支受限视图对应的上游数据集采用了不同的标记，查看者可能会看到此前受限制的数据。

向下游传递受限视图

如果分支上的受限视图发生更新，依托于该受限视图的对象类型需要在分支上进行索引以反映这些变更。在 Ontology Manager 中，使用顶部横幅上的 索引对象类型 选项在分支上对该对象进行索引。完成索引后，引用这些对象的 Workshop 模块及 Ontology Manager 将同步呈现分支上受限视图的变更。

审批受限视图变更

目前正将受限视图与“审批”应用进行集成。

数据主人或编辑者对分支受限视图所作的变更会自动获得审批。再次提醒，编辑者仅能在分支上构建受限视图，而数据主人则既能构建受限视图，又能在分支上修改其策略。

合并分支受限视图

合并分支上受限视图的权限与其他应用中的分支集成逻辑相同：仅分支创建者有权合并受限视图。
