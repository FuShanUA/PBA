配置SLT（SAP Landscape Transformation Replication Server）

SAP Landscape Transformation Replication Server（SLT）是一个数据复制工具，它使用数据库触发器执行变更数据捕获（CDC），从而实现从源SAP系统到目标系统的高效数据复制。

SLT和Palantir Foundry Connector 2.0 for SAP Applications（“Connector”）可以配置为一起工作，以实现从SAP到Foundry的CDC复制。

在此设置中，将创建一个具有专用上下文（队列别名）的SLT配置。

要将SAP表复制到Foundry，每个对象都会配置一个Foundry数据集同步。一旦配置了数据集同步，SLT将在相关SAP对象上配置触发器，SAP中该对象的更改随后将被捕获并存储在SLT服务器的队列中，用于ODP（操作数据供应）场景。Foundry将按照预设的时间表轮询此队列，以将这些更改同步到Foundry数据集。

从版本2.34（SP34）及更高版本开始，连接器使用基于ODP的数据提取通过OData ↗与ODP进行交互。

当首次为SAP对象配置Foundry数据集同步时，SLT将执行完整加载。后续的触发将仅获取该对象的更改。SLT队列的清除由SLT管理。

您也可以配置Foundry数据集同步，让SLT对特定SAP对象执行完整加载（快照），而不是以CDC模式（增量）运行。

本指南假设连接器已在实例上安装。您可以查看连接器的安装说明。

前提条件

直接使用ODP（连接器版本2.33 (SP33)及以下）

  • SAP_BASIS（技术组件版本，适用于SAP NetWeaver和SAP S/4HANA系统）7.4 SP14+ 用于连接器安装
  • DMIS 2011_1_730 组件 SP15+ 用于 SAP SLT 和源系统
  • SAP SLT 配置允许所有批量传输ID（MTIDs）的1:N复制

确保在SLT系统上实施以下SAP注释：

  • SAP OSS 注释 1660374 – 获取不等待足够长时间；超时
  • SAP OSS 注释 2215583 – 从ODQ提取数据时由于数据内容移动导致的错误

确保已阅读以下SAP注释，并在适用的情况下遵循相关步骤：

  • SAP OSS 注释 2697016 – 无法在CNV_IUUC_DB_CONN131中复制表 - 其他配置具有不同的触发设置 - SAP Landscape Transformation Replication Server.
  • SAP OSS 注释 2787584 - 尽管用户拥有SAP_ALL，但在LTRC中仍存在授权问题 - SLT
  • SAP OSS 注释 3032108 - 是否支持1:N复制？ - SLT

在旧版 DMIS 版本中，SAP SLT 场景下，一个表可以有最多4个消费者（1:4注册）。从 DMIS 2018 SP4 开始，SAP 新的 CDC 功能被启用，不再限制允许的消费者数量。

通过 OData 使用 ODP（连接器版本 2.34 (SP34) 及以上）

连接器版本 2.34 引入了 PALODATA 组件，允许您使用 OData API 配置同步 如 SAP 所推荐 ↗。

由于 OData 的近期性质，该组件对 SAP_BASIS 版本 7.50 SP09+ 有更高的要求。 同时，请确保已阅读并根据适用情况实施以下 SAP 注释：

  • SAP OSS 注释 2854759 - 通过 OData V2 提取时的日期字段处理
  • SAP OSS 注释 2878969 - ODP提取 ODATA 函数导入名称正在被截断
  • SAP OSS 注释 3062232 - ODP ODATA V2 - 指针的最后一个 skiptoken 未提供 deltatoken
  • SAP OSS 注释 3023446 - ODP 使用 oData 版本2: odata.maxpagesize 在 delta 请求中被忽略
  • SAP OSS 注释 2888122 - 如果源系统中最新的 delta 请求被删除，则无法进行 ODATA ODP delta

资源 sizing

使用 SLT 的 Connector 可能会大量占用 SAP 系统资源。因此，在开始复制之前，应查阅官方 SAP sizing 文档中的 SLT 部分。在从 SAP 环境开始数据提取之前，请确保 SLT 服务器上有空闲的后台和对话进程（如果 SLT 和 Connector 安装在不同的服务器上，还需确保安装 Connector 的 SAP 应用程序服务器也有空闲进程）。

  • 前往事务 SM50 查看是否有空闲的后台和对话进程。如果没有空闲进程，将数据传输到 Foundry 将会失败。
  • 有关系统配置的更多指导，请参考官方 SAP SLT 配置指南 ↗。

SLT 配置

步骤 2a 和 2b 中只需执行其中一个。我们建议按照步骤 2a 并使用安装后向导。步骤 3 和 4 仅在远程 SLT 场景中需要。

1. 创建到源系统的 RFC 目标连接

要创建一个远程函数调用（RFC）连接，SLT 将使用该连接从 SAP 源系统复制数据，请参考 创建 RFC 连接。

2a. 使用安装后向导为源系统创建 SLT 配置

要使用连接器安装后向导创建 SLT 配置，请使用 /n/PALANTIR/POST_INST 事务代码，并且仅启用步骤 7 创建 SLT 配置。SLT 配置的参数如下：

参数描述
Context Name唯一的配置名称
Context Description将在 SAP 事务 LTRC 和 Foundry 中显示给用户的上下文描述
Data Transfer Jobs数据传输作业数量
Initial Load Jobs初始加载作业数量
Calculation Jobs初始加载范围计算的计算作业数量
Authorization Group默认无授权组
Source RFC Destination逻辑目标 - 源系统的 RFC 目标名称
Read from Single Client如果数据将从单个客户端读取，请勾选此选项
Allow Multiple Usage如果允许多重使用，请勾选此选项

2b. 使用 SLT 工具为源系统创建一个 SLT 配置

只有步骤2a和2b中的一个步骤是必需的。我们建议使用步骤2a中详细说明的安装后向导。

  1. 要创建SLT配置，请使用LTRC事务代码。

  2. 创建一个新的配置。

  3. 在通用数据部分，填写配置名称和描述。

  4. 在源系统部分：

    • 选择RFC连接。
    • 选择我们之前创建的RFC目标连接（例如，我们的示例中的SAP_SOURCE）。

    当您配置源时，不要被“RFC目标”的名称所混淆——在此设置中，您应选择源系统的连接。

    • 启用允许多次使用。

    此设置允许多个订阅者从源系统检索复制的数据，并且在配置创建后无法更改。如 允许多次使用 如果仅使用一个订阅者，则不会产生影响，我们建议在首次配置时启用此功能。以后再启用多次使用需要您删除配置并创建一个新的配置。

    • 在生产环境环境中，启用从单个客户端读取。所使用的客户端编号将是在 RFC 连接中定义的编号。此设置将确保只有生产数据被复制到 Foundry。在现有的 SLT 环境中，所有配置都应具有相同的设置。因此，请查看 SAP OSS 注释 2372636 - Cannot replicate data when different trigger options exist - SLT。

  5. 在目标系统部分：

    • 选择RFC 连接。
    • 对于场景，选择操作数据供应（ODP）。对于 SAP_BASIS 的更高支持包级别，此场景将列在其他下，而不是在RFC 连接下。
    • 将RFC 目标设置为 NONE。
    • 将任何别名名称设置为队列别名。建议将队列命名为显示源系统SAP系统ID（SID）的名称。此队列别名名称在稍后定义Foundry数据集同步时将用作上下文。

  6. 在数据传输设置部分，设置在SAP SLT服务器上运行的数据传输作业数、初始加载作业数和数据计算作业数。有关复杂场景所需的作业数量，请参阅SLT 规格指南 ↗。

  7. 检查并创建连接。

  8. 回到主SLT屏幕，对于事务代码LTRC，确保在SLT系统中对于ODP场景的BAdI实现已激活。要检查该设置，请选择配置名称打开配置，然后点击专家功能选项卡。确保BAdI实现已激活，否则ODP将无法正常工作。

  9. 从主菜单进入SA38事务。如SAP OSS备注1660374 - Fetch does not wait long enough; timeout所述，运行程序SAP_RSADMIN_MAINTAIN以设置参数RODPS_FETCH_TIMEOUT。指定以秒为单位的值。

3. 创建到连接器（远程SLT场景）的RFC目标连接

连接器可以部署在一个满足要求的单独SAP实例上。在这种情况下，连接器通过RFC连接与SLT服务器通信。因此，需要一个RFC连接以启用通信通道。

请参考 RFC连接指南 获取设置说明。

4. 配置连接器以使用SAP SLT（远程SLT场景）

如果SLT和连接器在不同的实例上，请按照以下附加配置步骤操作：

  • 打开事务 /n/PALANTIR/PARAM
  • 输入以下参数值：
    • 参数ID: SLT
    • 参数名称: RFC_CONFIGURATION
    • 参数值: RFC configuration name created in the beginning of this section (eg, SLT_SERVER_RFC)

如果这些参数未在配置表中设置，它们将默认为以下值：

  • timestamp: 关闭
  • fetchOption: XML
  • pageSize: 50000

大型表同步的性能考虑

连接器使用SAP的ODP框架将数据复制到Foundry。ODP将数据从源系统复制到ODP队列（表名ODQDATA_F）。数据暂存于ODQ，一旦数据被传输到ODP队列，连接器便开始分页请求的数据到/PALANTIR/PAG_01和/PALANTIR/PAG_02表。在所有页面成功传输到Foundry后，Foundry会发送一个关闭请求以清理表，此关闭请求会删除/PALANTIR/PAG*表中的条目。

在运行初始同步（例如历史或批量）时，请仔细检查数据库大小并考虑空间需求。空间不足可能导致SLT复制作业失败以及传输到Foundry，因此请根据要复制的表首先进行数据库大小准备。

Parameters used to address known SAP issues

There are two other parameters which are used as a solution to known issues in SAP.

Issue: The SAP SLT server does not provide the exact field length of date/time fields

Problem: The SAP SLT server does not provide the exact field length of date/time fields. This has been fixed with an SAP note. However, if you do encounter any issues regarding data type lengths, the following parameter can be set until the note is implemented:

  • Param ID = SLT
  • Param Name = SLT_DATA_XXX (XXX refers to technical name of the field)
  • Param Value = Length of the data type

In an earlier support package of SAP_BASIS (SP 14), DATS fields were set as 16 characters by SAP. However, DATS fields should have 8 characters (YYYYMMDD). TIMS fields were also set as 16 characters by SAP; TIMS data has 6 characters (HHMMSS).

To address this problem, set the following parameters:

Param IdParam NameParam Value
SLTSLT_DATA_DATS8
SLTSLT_DATA_TIMS6

Issue: SAP QUAN data types that do not have 3 decimal places

Problem: SAP has specific data types to be used for quantity and amount. Quantity data type QUAN contains 3 decimal places by default. When the Connector tries to fetch data which contain QUAN data types, it automatically creates 3 decimal places for these QUAN data types. However, some of the SAP tables which contain QUAN data types have a different number of decimal places but are sent to Foundry as if they have 3 decimal places.

To address this problem, use the following parameter:

  • Param ID = SLT
  • Param Name = SLT_DTYPE_XXX (XXX refers to data type of the field)
  • Param Value = Data Type Name which will be taken Into account

Set the parameter as follows to change QUAN data types to decimal data types:

Param IdParam NameParam Value
SLTSLT_DTYPE_QUANDEC
