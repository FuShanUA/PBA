配置 SLT（SAP 景观转换复制服务器）

SAP 景观转换复制服务器（SLT）是一款数据复制工具，它利用数据库触发器来执行变更数据捕获（CDC），从而实现从源 SAP 系统到目标系统的高效数据复制。

可以对 SLT 和面向 SAP 应用的 Palantir Foundry 连接器 2.0（“连接器”）进行配置以协同工作，从而实现从 SAP 到 Foundry 的 CDC 复制。

在此设置中，将创建一个带有专用上下文（队列别名）的 SLT 配置。

若要将 SAP 表复制到 Foundry，需要为每个对象配置一个 Foundry 数据集同步。配置好数据集同步后，SLT 将在相关 SAP 对象上配置触发器，随后 SAP 中对该对象的变更将被捕获并存储在 SLT 服务器的队列中，以用于 ODP（操作数据分发）场景。Foundry 将按照预设的计划轮询此队列，以便将这些变更同步到 Foundry 数据集中。

从 2.34 (SP34) 及以上版本开始，该连接器使用《基于 OData 的 ODP 数据提取 ↗》来与 ODP 进行交互。

当首次为 SAP 对象配置 Foundry 数据集同步时，SLT 将执行全量加载。后续的触发器将仅获取该对象的变更。SLT 队列的清除工作由 SLT 负责管理。

您还可以配置 Foundry 数据集同步，让 SLT 对特定 SAP 对象执行全量加载（快照），而不是在 CDC 模式（增量）下运行。

本指南假定该连接器已安装在实例上。您可以查阅《连接器安装说明》。

先决条件

直接使用 ODP（连接器版本 2.33 (SP33) 及以下）

  • SAP_BASIS（技术组件版本，适用于 SAP NetWeaver 和 SAP S/4HANA 系统）7.4 SP14+ 用于安装连接器
  • DMIS 2011_1_730 组件 SP15+ 用于 SAP SLT 和源系统
  • 允许所有海量传输 ID (MTID) 进行 1:N 复制的 SAP SLT 配置

请确保在 SLT 系统上实施了以下 SAP 说明：

  • SAP OSS 说明 1660374 – 获取等待时间不够长；超时
  • SAP OSS 说明 2215583 – 从 ODQ 提取期间因数据内容移动而导致的错误

请确保已阅读以下 SAP 说明，并在适用情况下执行相关步骤：

  • SAP OSS 说明 2697016 – 无法在 CNV_IUUC_DB_CONN131 中复制表 - 其他配置具有不同的触发器设置 - SAP 景观转换复制服务器。
  • SAP OSS 说明 2787584 - 尽管用户拥有 SAP_ALL，LTRC 中仍出现授权问题 - SLT
  • SAP OSS 说明 3032108 - 是否支持 1:N 复制？ - SLT

在 SAP SLT 场景中，旧版 DMIS 中的一张表最多可以有 1:4 的注册（最多 4 个消费者）。从 DMIS 2018 SP4 开始，SAP 启用了新的 CDC 功能，消费者数量不再受限。

通过 OData 使用 ODP（连接器版本 2.34 (SP34) 及以上）

连接器版本 2.34 引入了 PALODATA 组件，允许您使用 OData API 配置同步，这也是《SAP 推荐的做法 ↗》。

鉴于 OData 的较新特性，该组件对 SAP_BASIS 版本有更高的先决条件要求，需 7.50 SP09+。 此外，请确保已阅读以下 SAP 说明，并在适用情况下予以实施：

  • SAP OSS 说明 2854759 - 通过 OData V2 提取期间对日期字段的处理
  • SAP OSS 说明 2878969 - ODP 提取 ODATA 函数导入名称被截断
  • SAP OSS 说明 3062232 - ODP ODATA V2 - 指针的最后一个 skiptoken 未提供 deltatoken
  • SAP OSS 说明 3023446 - ODP 与 oData version2：odata.maxpagesize 在增量请求中被忽略
  • SAP OSS 说明 2888122 - 如果源系统中最新的增量请求被删除，则无法进行 ODATA ODP 增量

资源容量规划

在 SAP 系统中，将 Connector 与 SLT 结合使用可能会占用大量资源。因此，在启动数据复制之前，应查阅 SAP 官方关于 SLT 的系统规模评估文档。从 SAP 环境中提取数据之前，请确保 SLT 服务器上（如果 SLT 与 Connector 位于不同服务器，则还需确保安装 Connector 的 SAP 应用服务器上）有可用的后台进程和对话进程。

  • 转到事务 SM50 查看是否有空闲的后台和对话进程。如果没有可用进程，向 Foundry 传输数据将会失败。
  • 有关系统规模评估的更多指导，请参阅 SAP 官方的《SLT 规模评估指南 ↗》。

1. 创建指向源系统的 RFC 目标连接

要创建供 SLT 用于从 SAP 源系统复制数据的远程函数调用（RFC）连接，请参阅《创建 RFC 连接》。

2a. 使用安装后向导为源系统创建 SLT 配置

要使用 Connector 安装后向导创建 SLT 配置，请使用事务代码 /n/PALANTIR/POST_INST，并且仅启用第 7 步 创建 SLT 配置。SLT 配置的参数如下：

参数描述
Context Name唯一的配置名称
Context Description将在 SAP 事务 LTRC 和 Foundry 中向用户展示的上下文描述
Data Transfer Jobs数据传输作业数
Initial Load Jobs初始加载作业数
Calculation Jobs用于初始加载范围计算的计算作业数
Authorization Group默认无授权组
Source RFC Destination逻辑目标 - 源系统的 RFC 目标名称
Read from Single Client如果从单一客户端读取数据，请勾选此选项
Allow Multiple Usage如果允许重复使用，请勾选此选项

2b. 使用 SLT 工具为源系统创建 SLT 配置

步骤 2a 和 2b 仅需执行其一。建议使用步骤 2a 中详述的安装后向导。

  1. 要创建 SLT 配置，请使用事务代码 LTRC。

  2. 创建新配置。

  3. 在 General Data 部分，填写 Configuration Name 和 Description。

  4. 在源系统部分：

    • 选择 RFC Connection。
    • 选择我们先前创建的 RFC Destination 连接（例如，本示例中的 SAP_SOURCE）。

    配置源系统时，不要被“RFC Destination”这个名称所迷惑——在此设置中，您应选择指向源系统的连接。

    • 启用 Allow Multiple Usage。

    此设置允许多个订阅者从源系统获取复制数据，且在配置创建后无法更改。由于仅使用单个订阅者时 Allow Multiple Usage 不会产生影响，我们建议在首次配置时即启用此项。若要后续再启用多重使用，必须删除当前配置并重新创建新配置。

    • 在生产环境中，请启用 Read from Single Client。所使用的客户端编号将在 RFC 连接中定义。此设置将确保仅生产数据被复制到 Foundry。在现有的 SLT 环境中，所有配置应具有相同的设置。因此，请查阅 SAP OSS 说明 《2372636 - 当存在不同的触发选项时无法复制数据 - SLT》。

  5. 在目标系统部分：

    • 选择 RFC Connection。
    • 为场景选择操作数据提供 (ODP)。对于较高支持包级别的 SAP_BASIS，该场景会列在其他项下，而非RFC 连接。
    • 将RFC 目标设置为 NONE。
    • 将任意别名设置为队列别名。建议队列名称能够体现源系统的 SAP 系统 ID (SID)。后续在定义 Foundry 数据集同步时，将使用此队列别名作为上下文。

  6. 在数据传输设置部分，设置要在 SAP SLT 服务器上运行的数据传输作业数、初始加载作业数和数据计算作业数。关于复杂场景所需的作业数，请参阅《SLT 规模调整指南 ↗》。

  7. 检查并创建连接。

  8. 返回 SLT 主界面，针对事务代码 LTRC，确保 SLT 系统上已激活适用于 ODP 场景的 BAdI 实现。如需检查此设置，请选择配置名称打开配置，然后点击专家功能选项卡。务必确保 BAdI 实现已激活，否则 ODP 将无法正常运行。

  9. 从主菜单进入 SA38 事务。按照 SAP OSS 说明 1660374 - Fetch does not wait long enough; timeout 所述，运行程序 SAP_RSADMIN_MAINTAIN 以设置参数 RODPS_FETCH_TIMEOUT。请以秒为单位指定值。

3. 创建与连接器的 RFC 目标连接（远程 SLT 场景）

连接器可部署在满足要求的独立 SAP 实例上。在此情况下，连接器通过 RFC 连接与 SLT 服务器通信。因此，需要建立 RFC 连接以打通通信渠道。

有关设置说明，请参阅《RFC 连接指南》。

4. 配置连接器以使用 SAP SLT（远程 SLT 场景）

如果 SLT 与连接器位于不同实例，请按以下额外步骤进行配置：

  • 打开事务 /n/PALANTIR/PARAM
  • 输入以下参数值：
    • Param ID：SLT
    • Param Name：RFC_CONFIGURATION
    • Param Value：本节开头创建的 RFC 配置名称（例如：SLT_SERVER_RFC）

如果未在配置表中设置这些参数，它们将默认采用以下值：

  • timestamp：OFF
  • fetchOption：XML
  • pageSize：50000

大型表同步的性能考量

连接器使用 SAP 的 ODP 框架将数据复制到 Foundry。ODP 会将数据从源系统复制到 ODP 队列（表名 ODQDATA_F）。数据暂存在 ODQ 中，一旦数据传输至 ODP 队列，连接器便开始将请求的数据分页存入 /PALANTIR/PAG_01 和 /PALANTIR/PAG_02 表。所有分页成功传输至 Foundry 后，Foundry 会发出关闭请求以清理这些表，该请求将删除 /PALANTIR/PAG* 表中的条目。

在执行初始同步（例如历史数据或批量数据同步）时，请仔细审查数据库容量规划并充分考虑空间需求。空间不足可能导致 SLT 复制作业失败以及向 Foundry 的传输中断，因此请务必根据需复制的表提前做好数据库容量规划工作。

用于解决 SAP 已知问题的参数

另外还有两个参数可用于解决 SAP 的已知问题。

问题：SAP SLT 服务器未提供日期/时间字段的确切长度

问题现象：SAP SLT 服务器未提供日期/时间字段的确切长度。该问题已通过 SAP 说明修复。但是，如果在数据类型长度方面仍遇到任何问题，在应用该说明之前可设置以下参数：

  • Param ID = SLT
  • Param Name = SLT_DATA_XXX（XXX 指代字段的技术名称）
  • Param Value = 数据类型的长度

在早期版本的 SAP_BASIS 支持包 (SP 14) 中，SAP 将 DATS 字段设置为 16 个字符。然而，DATS 字段应为 8 个字符 (YYYYMMDD)。SAP 也将 TIMS 字段设置为 16 个字符；而 TIMS 数据应为 6 个字符 (HHMMSS)。

要解决此问题，请设置以下参数：

参数 ID参数名称参数值
SLTSLT_DATA_DATS8
SLTSLT_DATA_TIMS6

问题：SAP 中小数位数不等于 3 的 QUAN 数据类型

问题描述：SAP 具有专门用于数量和金额的数据类型。数量数据类型 QUAN 默认包含 3 位小数。连接器尝试抓取包含 QUAN 数据类型的数据时，会自动为这些 QUAN 数据类型生成 3 位小数。然而，部分包含 QUAN 数据类型的 SAP 表具有不同的小数位数，但发送至 Foundry 时仍被视为具有 3 位小数。

要解决此问题，请使用以下参数：

  • 参数 ID = SLT
  • 参数名称 = SLT_DTYPE_XXX（XXX 代表该字段的数据类型）
  • 参数值 = 将采用的数据类型名称

按如下方式设置参数，将 QUAN 数据类型更改为 decimal 数据类型：

参数 ID参数名称参数值
SLTSLT_DTYPE_QUANDEC
