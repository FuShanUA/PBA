apollo-cli 产品推广追加 [实验性]

该命令为 实验性。要 启用此命令，请运行 apollo-cli configure 命令并启用 V2 experimental commands。

从模板中追加推广管道阶段

从 YAML 模板文件中将推广管道阶段追加到产品的现有管道中。

该命令会与现有管道合并模板阶段，并检测冲突：

  • 阶段通过其源和目标发布渠道进行标识
  • 冲突的阶段（相同源->目标）会提示覆盖确认
  • 不冲突的阶段会被追加到现有管道中

模式： 单个产品： apollo-cli product promotion append --file stages.yaml 批量（环境）： apollo-cli product promotion append --environment --file stages.yaml

冲突解决： --dry-run 预览更改但不应用（显示“将覆盖”以供冲突） --override 自动覆盖所有冲突而无需提示 （默认） 对每个冲突进行交互式提示

示例模板文件（stages.yaml）： stages: - sourceReleaseChannel: BETA targetReleaseChannel: RELEASE stage: type: zeroCanary zeroCanary: soakTime: "PT1440M" productLabelConditions: - labelId: "security.palantir.build/approved" onlyPossibleLabelValues: ["true"]

示例：

将阶段添加到单个产品的管道中

apollo-cli product promotion append com.example --file stages.yaml

预览更改而不应用（干运行）

apollo-cli product promotion append com.example --file stages.yaml --dry-run

将阶段添加到环境中所有产品

apollo-cli product promotion append --environment my-env-id --file stages.yaml

自动覆盖冲突而不提示

apollo-cli 产品推广追加 --environment my-env-id --file stages.yaml --override

标志

标志用途
--dry-run打印将执行的操作
-e, --environment环境
-f, --file单个输入文件
-h, --help追加的帮助
--override无提示地覆盖冲突
--space-id空间ID

从父命令继承的标志

标志用途
--apollo-client-id 用于生成Bearer Token的客户端ID
--apollo-client-secret用于生成Bearer Token的客户端密钥
--apollo-token用于认证端点的Bearer Token
--apollo-token-provider指定如何提供用于认证Apollo端点调用的Bearer Token。有效值为 "auto"、"static"、"service-user" 或 "sso"（默认为 "auto"）。如果指定 "auto"，则从配置中选择模式：如果设置了 "apollo-token"，则为 "static"；如果同时设置了 "apollo-client-id" 和 "apollo-client-secret"，则为 "service-user"；否则为 "sso"（交互式浏览器登录）。如果同时设置了Bearer Token和（客户端ID或客户端密钥），则会报错。如果指定 "static"，则使用 "apollo-token" 提供的令牌。如果指定 "service-user"，则使用 "apollo-client-id" 和 "apollo-client-secret" 从Multipass生成令牌。如果指定 "sso"，则使用针对Multipass的交互式基于浏览器的SSO登录（支持硬件密钥如YubiKey）；生成的令牌将按配置文件缓存并静默刷新。当第一次需要令牌且没有有效的缓存令牌时，会自动触发登录流程。
--apollo-url Apollo 的基础 URL，用于推导 API 端点
--debug启用调试级别日志
--http-timeout所有 apollo 请求的超时时间（分钟）
-k, --insecure-skip-verify跳过服务器证书验证
-o, --output输出格式（json,yaml,pretty）
--profile从配置文件中使用特定的 profile
--quiet不将日志输出到 stderr
