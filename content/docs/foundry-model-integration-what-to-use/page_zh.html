选择合适的建模工具

Palantir 的建模产品套件支持用户开发、管理和部署模型。本页将对不同产品进行比较，帮助您根据自身需求选择合适的工具。

如需建模任务的向导式协助，AI FDE 提供了专属的 机器学习 模式，可帮助您完成模型的训练、评估、部署和调优。该智能体能够引导您走完从特征工程到模型训练与部署的完整工作流，既可使用 Model Studio，也可使用专业代码仓库。要开始使用，请从 AI FDE 模式选择器中选择机器学习模式，或者直接描述您的建模任务，让智能体自动为您选定模式。

批量推理

模型可用于跨数据集运行大规模批量推理管道。

产品详情注意事项
Pipeline Builder使用已训练模型节点直接在管道画布上进行无代码模型推理。模型作为独立 sidecar 与 Spark 执行器并行运行，并自动使用最新模型版本。仅支持具有单个表格输入和输出的模型。暂不支持流式和轻量级执行模式。
Python 转换可直接在 Python 转换中运行批量推理。支持固定特定模型版本。建议使用 @lightweight 装饰器和模型 sidecar。
建模目标批量部署建模目标提供更广泛的模型管理功能，例如模型发布管理和评估。不支持多输出和外部模型、模型 sidecar，或通过 Marketplace 进行部署，详情请见此处。
Jupyter® 笔记本用户可以直接从 Code Workspaces 创建定时的训练和/或推理作业。仅支持运行从同一笔记本创建的推理模型；若要编排在其他地方创建的模型，请使用 Python 转换。

模型部署

模型可以在 Foundry 中通过 REST API 进行部署；部署模型即意味着将模型投入运作，供 Foundry 内外部使用。

产品详情
模型直接部署自动升级的模型部署；最适合快速迭代和部署。
建模目标实时部署生产级建模项目管理；建模目标为模型发布管理和评估提供工具。不支持通过 Marketplace 进行部署，详情请见此处。

了解有关直接部署与通过建模目标进行部署之间差异的更多信息。

函数集成

将模型发布为函数，可方便地在下游 Foundry 应用（包括 Workshop、Slate、操作及更多应用）中使用模型进行实时推理。

| 产品 | 最适用场景 | | --- | --- | | [直接函数发布](/docs/foundry/model-integration/functions-on-models/) | 在带有实时部署的模型上无需编写代码即可创建函数，并支持与本体集成。模型与建模目标应用中也提供了相同功能。 | | [将模型函数导入函数仓库](/docs/foundry/functions/functions-on-models/#write-a-model-backed-typescript-v1-function) | 将模型函数导入 TypeScript v1、v2 或 Python 函数中，以进一步处理预测结果（例如进行本体编辑），并提供模型 API 类型检查支持。 |
