模型适配器概述

模型适配器是一个通用框架，使 Foundry 能够与任意模型进行互操作。模型适配器是构成模型的两大组件之一：

  • 模型构件： 用于保存已训练模型的模型文件、参数、权重、容器或凭据。
  • 模型适配器： Foundry 与模型构件交互所需的逻辑和环境依赖，用于加载、初始化模型并执行推理。

模型适配器可以使 Foundry 与以下对象进行互操作：

  1. 在 Foundry 中训练的模型
  2. 在 Foundry 外部训练的模型文件
  3. 在 Foundry 外部容器化并推送到 Foundry Docker 注册表的模型
  4. 在 Foundry 外部训练和托管的模型

适配器组件

Palantir 通过与特定模型版本对应的模型适配器类进行交互，以一致的方式处理所有模型。

初始化

由于模型只需创建一次即可在平台内的多处使用，适配器需要了解如何从权重或底层容器初始化模型实例。

对于在平台内训练的权重，用户应使用 @auto_serialize 注解来利用内置序列化器，该序列化器适用于大多数模型类型。在必须显式指定序列化/反序列化逻辑的高级场景中，请参阅：load() 和 _save() 方法。

对于容器模型和外部模型，适配器会使用 init_container() 或 init_external() 方法进行初始化，以便用于推理。对于这两种类型的模型，在初始化时也会调用 load() 方法，但该方法随后会在后台调用 init_container() 或 init_external()，从而仅将与这些模型类型相关的上下文（例如 ContainerizedApplicationContext 或 ExternalModelContext）提供给模型实例。除了使用 init_container() 或 init_external() 之外，用户还可以重写 load() 方法来定义这些模型的初始化方式。

要在转换中使用模型，环境必须包含模型适配器的依赖项。为防止模型适配器与转换环境之间发生依赖冲突，请在 ModelInput 中设置 use_sidecar = True，以在隔离的容器中运行模型。predict() 请求将自动路由到 sidecar，无需额外修改代码。请查阅 ModelInput 类参考以获取更多详细信息。

API

每个适配器都必须声明 API 描述。平台依赖该描述（包含预期的输入、输出、列名和类型），以便在各种模型调用场景下实现与其他应用的集成。

有关支持的类型和 API 定义示例，请查阅 API 参考。

推理

初始化完成后，适配器即可用于批处理或交互式工作负载的推理。推理逻辑必须作为 predict() 方法的一部分进行定义。

平台使用提供的 API 定义，以规定的名称和类型调用 predict() 方法，从而执行推理。

有关创建模型适配器的更多信息，请参阅创建模型适配器的文档。您还可以在模型适配器参考页面上查阅完整的 Python API。
