Zoho Projects

Zoho Projects 连接器是用于 Zoho Projects 的 Palantir 提供的驱动程序。

要创建新的 Zoho Projects 源，请遵循 Palantir 提供的驱动程序的标准设置流程，然后使用以下章节进行 Zoho Projects 特定的配置和网络设置。有关完整的属性参考，请参阅 《Zoho Projects 官方驱动程序文档》 ↗。

配置

以下属性为必填或推荐属性。

属性是否必填？描述默认值
InitiateOAuth ↗推荐指定获取或刷新 OAuth 访问令牌的过程，在已认证和已授权用户工作期间维持用户访问权限。REFRESH
OAuthClientId ↗推荐指定创建自定义 OAuth 应用程序时分配的客户端 ID。（也称为使用者密钥）。此 ID 将自定义应用程序注册到 OAuth 授权服务器。—
OAuthClientSecret ↗推荐指定创建自定义 OAuth 应用程序时分配的客户端密钥。（也称为使用者机密）。此密钥将自定义应用程序注册到 OAuth 授权服务器。—
PortalId ↗推荐根据门户 ID 限制公开的实体。如果留空，将公开所有可用的门户。—
ProjectId ↗推荐根据项目 ID 限制公开的实体。如果留空，将公开所有可用的门户。—
Region ↗推荐服务器 URL 中的顶级域 (TLD)。US

网络

下表列出了源为成功运行而需要能够访问的域。

对于每个域，添加相应的 出站策略。如果源托管在本地并且无法从 Foundry 直接访问，请改用 智能体代理出站策略；智能体主机本身也必须能够访问所列出的域。有关详细信息，请参阅 《使用智能体作为代理》。

域是否必填
projectsapi.zoho.<Region>始终必填。区域连接属性映射到 TLD（默认为 Region=US --> .com）
<APIDomain> - 默认值：zohoapis.<Region>始终必填。通过 OAuth 流程自动获取；手动提供 OAuthAccessToken 时在 APIDomain 连接属性中设置
<AccountsServer> - 默认值：accounts.zoho.<Region>始终必填。通过 OAuth 流程自动获取；手动提供 OAuthAccessToken 时在 AccountsServer 连接属性中设置
