创建计划

在《数据血缘关系》计划编辑器中，您可以创建新计划并编辑现有计划。您还可以查看现有计划的指标，并了解该计划将如何构建您的数据集。

进入计划编辑器

以下步骤将引导您进入《数据血缘关系》计划编辑器，您可以在其中创建新计划：

  1. 导航至您想要计划构建的数据集的《数据集预览》页面，打开操作菜单，然后选择管理计划。



  2. 这将带您进入《数据血缘关系》图，右侧的计划面板会列出影响该数据集的所有当前计划。

    要创建计划，请点击创建计划按钮。



  3. 这将带您进入计划编辑器。



定义计划

要定义计划，您需要完成以下部分。

分支选择

在数据血缘关系中创建计划时，计划将应用于图中配置的分支（包括回退分支）。

目标数据集

目标数据集指定了构建结束时的数据集。默认情况下，仅会构建这些数据集。在其他配置中，这些数据集可能是正在构建的一系列数据集的最后一项。

可以通过将目标数据集添加到图中并将其标记为目标来选择它们。

数据集将在数据血缘关系窗口右上角设置的分支上进行构建。图中的数据集将根据它们如何影响计划或如何受计划影响来进行着色。

排除的数据集

排除的数据集指定了在确定要构建哪些数据集时，在图遍历过程中将被忽略的数据集。被忽略数据集上游的所有数据集都不会被构建。

构建时间

触发器指定了构建应运行的时间。当触发器中定义的条件得到满足时，就会运行构建。

有两种基本类型的触发器：

  • 时间：在指定的时刻满足时间触发器。

    《了解有关时间触发器的更多信息。》

  • 当数据集更新时：一旦平台上发生指定的数据集更新，即满足该触发器。

    《了解有关事件触发器的更多信息。》

触发器可以组合起来以创建更复杂的复合触发器。可以使用触发器编辑器中的菜单选择组合触发器的方法：

  • 满足以下任一触发器：当任一组件触发器得到满足时，该触发器即被满足。

  • 满足以下所有触发器：当所有组件触发器都得到满足时，该触发器即被满足。

  • 高级配置：显式配置如何组合组件触发器。

    选择高级配置时，通过在两个触发器之间插入关键字 AND 和 OR 来组合组件触发器。可以通过将复合触发器放在括号 ( 和 ) 内来进一步组合复合触发器。

    • AND 表示当两个组件触发器都得到满足时，复合触发器才会被满足。

    • OR 表示当满足任一组件触发器时，复合触发器即被满足。

    《了解有关复合触发器的更多信息。》

构建类型

构建类型指定了如何构建所选数据集：

  • 单次构建：仅构建目标数据集（不会构建其他数据集）。

  • 全量构建（包含上游）：构建所有目标数据集及该目标的所有上游数据集，排除的数据集除外。若上游依赖发生变化，实际构建的数据集也可能随之改变。

  • 连通构建：构建输入数据集（不含）与目标数据集（含）之间的所有数据集。如果某个数据集位于输入数据集和目标数据集之间，但被显式指定为输入，调度任务仍会对其进行构建。如果某个数据集被选为目标数据集，但在输入数据集与该目标数据集之间不存在构建路径，则不会构建该目标数据集。

对于连通构建，连接目标数据集与上游数据集的所有数据集，必须通过配置调度的分支上的作业规格路径连通起来。

下面通过一个血缘关系示例来说明上述内容，示例包含数据集 D1, ..., D6，其中数据集 D2 的两个不同分支分别作为 D3 和 D4 的输入。在 master 分支上配置一个以 D1 为输入、D5 和 D6 为目标的连通构建调度，系统将尝试仅构建 D2、D4 和 D6。

出现这种情况的原因在于，在 master 分支上从输入 D1 到目标 D6 的连通构建选择会选中数据集 D2、D4 和 D6，因为数据集 D1 和 D6 之间存在作业规格路径。然而，从输入 D1 到目标 D5 之间不存在连通构建选择，因为沿着 master 分支的作业规格路径已断开。

D1 (master) --> D2 (master) --> D4 (master) --> D6 (master)
                 |
                 | D2 (develop)
                 |
                  --> D3 (master) --> D5 (master)

在此示例中，若要将数据集 D3 纳入调度构建，应将构建方式更改为全量构建，这样会同时包含 D3 及作为输入的 D1，因为 D1 是 D5 的上游。或者，若要保持连通构建方式，可以编辑转换逻辑，将 D2 (develop) 作为输入、D3 作为输出，并在 master 分支上增加一个虚拟输入 D7，如下所示。此时，在 master 分支上以 D1 为输入、D5 和 D6 为目标的连通调度构建将构建 D2、D3、D4、D5 和 D6。

D1 (master) --> D2 (master) --> D4 (master) --> D6 (master)
                 |
                 | D2 (develop)
                 |
D7 (master) -------> D3 (master) --> D5 (master)
连通构建分支选择

对于连通构建调度，如果使用的数据集来自不同于调度所在分支的其他分支，那么在输入和目标构建之间必须存在同一分支上的连通作业规格路径，这些数据集才会被纳入构建范围。

构建范围

考虑到数据管道的动态特性，每次触发调度时都会评估纳入构建范围的数据集集合。构建范围定义了调度构建的边界，即使管道内容发生变化，这些边界也将保持固定。目前提供两种范围界定选项：

  1. 按项目：调度仅允许构建所选项目内的合格数据集。
  2. 按用户账户：调度仅允许构建调度创建者/编辑者有权构建的数据集。

项目范围界定

项目范围界定确保调度仅在所选项目内的数据集上运行。这使得调度能够免受用户权限变化的影响。当构建内容不太可能发生大到无法在所选项目内运行的改变时（如下文所述），此选项最为可靠。

在以下情况下，构建无法按项目界定范围：

  • 您没有在项目上创建调度的权限。
  • 您没有构建数据集的充分权限。
  • 调度设置涵盖的数据集会移除标记（当图形按权限着色时，您可以在《数据血缘》中看到这一点）。
  • 调度设置涵盖的数据集必须使用用户账户进行构建。
  • 计划内容无法解析为一组明确的项目。

用户范围设定

采用用户范围设定时，系统将以最后编辑（或创建）该计划的用户身份来触发构建。因此，构建过程只会包含该用户有权构建的数据集。当计划中的流水线可能发生变更并导致项目范围设定失效时（例如数据集被移至其他项目、新数据集需要以用户账户身份运行等），此选项最为可靠。

警告

如果该用户被停用，或者失去了对构建所必需的数据集的访问权限，计划构建将无法启动。在按用户账户进行范围设定时，请确保该账户具备可靠的权限，并记住在用户即将被停用时转移数据权属。

高级设置

高级设置用于指定额外的构建选项：

  • 失败时中止构建： 如果构建中的任何作业失败，立即取消所有其他作业并结束构建。

  • 自定义失败作业的重试次数： 失败作业的运行尝试次数。重试的作业将作为同一构建的一部分运行。在所有重试都已尝试完毕，或出现表明无法重试的错误之前，作业不会被视为失败。将此值设置为 1 可阻止重试，因为作业只会尝试运行一次。请注意，并非所有类型的失败都可以重试。计划运行时的重试次数上限将由您的管理员配置决定。

  • 强制构建： 运行构建时忽略过期状态信息。无论数据集是否过期，都将对其进行构建。几乎在任何情况下都不需要此选项。必须使用此选项的罕见情况包括：

    1. 数据连接同步： 数据连接同步始终被标记为最新状态，因为它们的输入源来自外部；Foundry 无法得知外部数据是否发生了更新。我们建议将数据连接同步与构建的其他部分分开计划，以确保仅对同步操作进行强制构建。
    2. 对象存储 V1 (Phonograph) 同步
外部转换计划

进行 API 调用的转换与数据连接同步的行为不同。这些转换始终被视为过期状态，任何构建计划都会触发它们，无需启用强制构建选项。

  • 构建成功后重新触发： 重复触发计划以持续构建，直到所有输入均已处理完毕且计划的目标不再处于过期状态。若要使此设置生效，计划必须：
    1. 包含至少一个目标输出，该输出需对数据集使用增量事务限制，或对媒体集使用增量批次限制。如果计划中不存在此类输出，将显示内容为 No resources in this schedule require re-triggering 的警告。
    2. 禁用强制构建设置。

保存计划

完成计划定义后，点击保存计划按钮。

系统将返回至计划页面，您将在此处看到已保存的计划。
