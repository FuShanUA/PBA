从预训练文件发布模型

Palantir 支持创建包装在平台外部生成的权重的模型。这些文件可以包括开源模型权重、在本地开发环境中训练的模型、在《代码工作区》应用程序中训练的模型，以及来自旧系统的模型权重。

创建 Palantir 模型后，Palantir 会提供以下功能：

  • 与批处理管道集成以及实时模型托管。
  • 完整的版本控制、细粒度权限控制以及受治理的模型血缘关系。
  • 通过《建模目标》进行模型管理和实时部署。
  • 与本体绑定，允许通过《模型上的函数》进行业务化运营和假设情景分析。

从模型文件创建模型

要从模型文件创建模型，您需要准备以下内容：

  • 可上传至 Palantir 的模型文件
  • 一个模型适配器，用于告知 Palantir 如何加载模型并执行推理

1. 将模型文件上传至非结构化数据集

首先，将您的模型文件上传至 Palantir 平台中的非结构化数据集。在项目中选择 + 新建 > 数据集 以创建新数据集。

接着，选择 导入新数据，并从您的计算机中选择要上传至模型的文件。

如有需要，您可以将多个不同的文件上传至同一数据集。该数据集将是非结构化的，这意味着它没有表格化的模式。

2. 创建模型训练代码库以定义模型适配器逻辑

创建一个新的代码仓库，用于管理从非结构化数据集中读取模型文件的逻辑。该逻辑会将这些文件包装在模型适配器中，并将其作为模型发布。在《代码仓库》应用程序中，选择使用 模型训练 语言模板初始化 模型集成 仓库。

请查阅《模型训练模板》和《模型适配器 API》的完整文档以供参考。

3. 将权重发布至模型

要将非结构化数据集中的模型文件发布为 Palantir 模型，您必须编写一个转换，完成以下操作：

  1. 从非结构化数据集中将已保存的模型文件加载至合适的 Python 对象中
  2. 使用该已加载的 Python 对象实例化模型适配器
  3. 将模型适配器作为模型资源发布

您可以将加载和发布模型的逻辑放置在仓库中的 model_training 文件夹内。

如需了解更多信息，我们建议您查阅以下文档：

  • 完整的《模型适配器 API 定义》
  • 如何《从非结构化数据集中读取文件》
  • 如何《使用模型训练模板创建并发布模型适配器》
  • 《本地训练模型包装示例》

定义好模型训练逻辑后，选择 构建 以执行该逻辑，从而读取模型文件并发布模型。

```python from transforms.api import transform, Input from palantir_models.transforms import ModelOutput, copy_model_to_driver import palantir_models as pm import os import pickle @transform( model_files=Input(""), model_output=ModelOutput("") ) def compute(model_files, model_output): # 将数据集中的所有文件复制到驱动程序 model_files_path = copy_model_to_driver(model_files.filesystem()) # 例如，如果您有一个保存为 pickle 文件的模型，您可以使用 pickle 加载它 with open(os.path.join(model_files_path, "model.pkl"), 'rb') as file: model = pickle.load(file) wrapped_model = ExampleModelAdapter(model) model_output.publish( model_adapter=wrapped_model ) class ExampleModelAdapter(pm.ModelAdapter): @pm.auto_serialize def __init__(self, model): self.model = model @classmethod def api(cls): pass # 实现此模型的 API def predict(self, df_in): pass # 实现推理逻辑 ```

同样的文件加载逻辑也适用于大多数其他场景，各类库（如 PyTorch 或 Tensorflow）通常会提供相应的方法，将序列化文件读取为 Python 对象。
