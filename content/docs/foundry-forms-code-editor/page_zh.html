代码编辑器

Foundry Forms 不再是 Foundry 上进行数据录入或数据回写工作流的推荐方法。请改用 Foundry Ontology 构建用户输入工作流，将相关数据结构表示为对象类型，并通过 Actions 配置回写交互。详情请参阅《Forms 概览》文档。

代码编辑器允许用户查看和编辑表单的 YAML 表示形式。虽然大多数配置都可以通过可视化编辑器完成，但代码编辑器提供了完整的功能，包括添加四种代码专属字段的能力。

要查看代码编辑器的实际运行情况，请选择表单左下方的黑色 </> 按钮将其打开。然后，使用可视化编辑器添加一个字段。观察在配置字段时代码如何自动填充和更新。

选择代码编辑器右上方的书籍图标，即可直接在代码编辑器窗口中打开示例和文档。

代码结构

表单的通用结构如下：


name: 新表单
fields:
- uri: name
  name: 全名
  type: Text
  # 及其他通用选项
  options:
    placeholder: 'Last, First'
    # 及其他特定选项
sheets:
- name: 新工作表
  fields:
  - uri: date_of_birth
    name: 出生日期
    type: DatePicker
    options:
      precision: day
  - uri: weight_kg
    name: 体重
    type: Numeric
    options:
      unit: kg

在最顶层，包含 `name`（表单标题）、`fields`（字段列表）和 `sheets`（工作表列表）。随后，每个 `sheet` 本身也具有这些相同选项。字段拥有通用选项，例如 `uri`、`name` 和 `type`，以及特定选项（位于 `options` 下），例如 `placeholder`、`precision` 和 `unit`。


通用选项


每个字段均可配置以下选项：


    uri: string
    type: string
    name: string
    urlParam: string
    tag: string
    helperText: string
    infoText: string
    columnSpan: integer
    disabled: Boolean
    hidden: Boolean
    isBlock: Boolean
    noLabel: Boolean
    validators: list<Validators>
    transforms: list<Transforms>



    

  • uri: 字段的唯一标识符。若需将字段回写至 Fusion 或本体，则必须分别与列名或属性 ID 匹配。若该字段无需回写，则必须以 display. 前缀开头。
    • 

  • type: 字段类型（例如 Text、Numeric、DatePicker）。
    • 

  • name: 字段标签。
    • 

  • urlParam: 与字段关联的 URL 参数。允许用户在表单嵌入 iframe 时预填充值。
    • 

  • tag: 字段的另一个唯一标识符，但无限制条件。当某个字段引用另一个字段时（例如 Calculation、Template），即使用此标识符，但附件除外。
    • 

  • helperText: 字段下方的说明文字。
    • 

  • infoText: 显示在字段标签旁边的信息提示内容。
    • 

  • columnSpan: 字段占据的列数。
    • 

  • disabled: 允许用户将字段设为只读。
    • 

  • hidden: 允许用户隐藏字段，同时记录其值。
    • 

  • isBlock: 允许用户将标签显示在字段上方，而非左侧。
    • 

  • noLabel: 允许用户隐藏字段标签。此选项仅能通过代码编辑器使用。
    • 

  • validators: 字段上的验证器。
    • 

  • transforms: 字段上的转换器。
    •

隐藏字段


hidden 字段允许用户在不显示字段的情况下记录值。其配置如下：


- uri: Hidden
  name: Hidden
  type: Hidden
  options:
    val: string



列表


当字段本身不支持存储多个值时，List 字段允许用户进行存储。其配置如下：


- uri: names
  name: Full names
  type: List
  options:
    # allowZeroItems: true
    item:
      type: Text
      # 以及其他支持的通用选项 (helperText, infoText, validators)
      options:
        placeholder: 'Last, First'
        # 以及其他特定选项



allowZeroItems 为可选项，默认值为 false。


复合


composite 字段允许用户将多个字段的值存储为一个字符串化 JSON。其配置如下：


- uri: info
  name: Composite
  type: Composite
  options:
    fields:
      - uri: date_of_birth
        name: Date of birth
        type: DatePicker
        # 以及其他支持的通用选项 (helperText, infoText, disabled, isBlock, noLabel, validators)
        options:
          precision: day
          # 以及其他特定选项
      - uri: weight_kg
        name: Weight
        type: Numeric
        options:
          unit: kg



复合字段中不允许包含隐藏字段以及那些允许存储多个值的字段。


列表聚合


list aggregate 字段类似于计算字段和模板字段，但作用于存储多个值的字段。其配置如下：



- uri: single_value
  tag: single_value
  name: 单个值
  type: RadioButtons
  options:
    options:
      - value: '10'
      - value: '20'
      - value: '30'
- uri: multi_values
  name: 多个值
  tag: multi_values
  type: Checkboxes
  options:
    options:
      - value: '40'
      - value: '50'
      - value: '60'
- uri: avg_value
  name: 平均值
  type: ListAggregate
  options:
    listOperation:
      operation: mean
      listTag: [single_value, multi_values]
    # errorValue: 未选择任何值



或者，与 复合 字段配合使用时：


- uri: grades
  tag: grades
  name: 成绩
  type: List
  options:
    item:
      type: Composite
      options:
        fields:
          - uri: name
            name: 姓名
            type: Text
            options:
              placeholder: '姓, 名'
          - uri: grade
            name: 分数
            type: Numeric
            options:
              unit: '%'
- uri: all_grades
  name: 所有成绩
  type: ListAggregate
  options:
    listOperation:
      operation: concatenate
      joinWith: ', '
      listTag: grades
      displayItem: grade
    # errorValue: 未添加任何成绩



配置 options 字段时：


    

  • errorText 为可选参数，默认值为空字符串。
    • 

  • 在 listOperation 中：

      

    • operation 可以是 concatenate、sum、min、max、mean 或 count。
      • 

    • listTag 表示待聚合字段的标签（或标签数组）。
      • 

    • 仅当操作为 concatenate 时，才需要设置 joinWith。
      • 

    • displayItem 仅在聚合字段为 复合 字段时才需要设置，其值为嵌套字段的 URI。
