代码编辑器

Foundry 表单不再是 Foundry 上数据录入或写回工作流的推荐方法。相反，使用 Foundry 本体论构建用户输入工作流，将相关数据结构表示为对象类型，并配置与操作的写回交互。在 表单概述 文档中了解更多信息。

代码编辑器 允许用户查看和编辑表单的 YAML 表示。虽然大多数配置可以通过视觉编辑器完成，但代码编辑器提供了完整的功能，包括添加四个 仅代码字段 的能力。

要查看代码编辑器的实际效果，请通过选择表单左下角的黑色 </> 按钮打开它。然后，使用视觉编辑器添加一个字段。注意随着字段的配置，代码如何自动填充和更新。

在代码编辑器右上角选择书籍图标，可在代码编辑器窗口内直接打开示例和文档。

代码结构

表单的一般结构如下：

Copied!
1name: New form
2fields:
3  - uri: name
4    name: Full name
5    type: Text
6    # and other generic options
7    options:
8      placeholder: 'Last, First'
9      # and other specific options
10sheets:
11  - name: New sheet
12    fields:
13      - uri: date_of_birth
14        name: Date of birth
15        type: DatePicker
16        options:
17          precision: day
18      - uri: weight_kg
19        name: Weight
20        type: Numeric
21        options:
22          unit: kg

最顶层有 name（表单的标题）、fields（字段列表）和 sheets（工作表列表）。然后，每个 sheet 本身也有这些选项。字段具有 通用选项，如 uri、name 和 type，以及特定选项（位于 options 下），如 placeholder、precision 和 unit。

通用选项

以下选项可以为每个字段进行配置：

    uri: string
    type: string
    name: string
    urlParam: string
    tag: string
    helperText: string
    infoText: string
    columnSpan: integer
    disabled: Boolean
    hidden: Boolean
    isBlock: Boolean
    noLabel: Boolean
    validators: list<Validators>
    transforms: list<Transforms>
  • uri： 字段的唯一标识符。如果该字段要写回 Fusion 或本体论，它需要分别与列名或属性 ID 匹配。如果该字段不被写回，它需要以前缀 display. 开头。
  • type： 字段的类型（例如，Text，Numeric，DatePicker）。
  • name： 字段的标签。
  • urlParam： 与字段关联的 URL 参数。这允许用户在表单嵌入到 iframe 中时预填值。
  • tag： 字段的另一个唯一标识符，但没有限制。当一个字段引用另一个字段时（例如，Calculation，Template），使用此标识符，但不包括 附件。
  • helperText： 字段下方的说明文字。
  • infoText： 显示在字段标签旁边的提示信息内容。
  • columnSpan： 字段占用的列数。
  • disabled： 允许用户将字段设为只读。
  • hidden: 允许用户隐藏字段但记录该值。
  • isBlock: 允许用户将标签显示在字段上方而不是左侧。
  • noLabel: 允许用户隐藏字段的标签。这只能通过代码编辑器使用。
  • validators:验证器 on the field.
  • transforms:转换 on the field.

仅代码字段

四个字段只能通过代码编辑器使用：

  • 隐藏
  • 列表
  • 复合
  • 列表聚合

隐藏

hidden 字段允许用户记录一个值而不显示字段。其配置如下：

- uri: Hidden
  name: Hidden
  type: Hidden
  options:
    val: string

List

List 字段允许用户在字段本身不支持的情况下存储多个值。其配置如下：

- uri: names
  name: Full names
  type: List
  options:
    # allowZeroItems: true
    item:
      type: Text
      # and other supported generic options (helperText, infoText, validators)
      options:
        placeholder: 'Last, First'
        # and other specific options

allowZeroItems 是可选的，默认为 false。

Composite

composite 字段允许用户将多个字段的值存储为一个字符串化的 JSON。其配置如下：

- uri: info
  name: Composite
  type: Composite
  options:
    fields:
      - uri: date_of_birth
        name: Date of birth
        type: DatePicker
        # and other supported generic options (helperText, infoText, disabled, isBlock, noLabel, validators)
        options:
          precision: day
          # and other specific options
      - uri: weight_kg
        name: Weight
        type: Numeric
        options:
          unit: kg

Hidden（隐藏） 字段和那些 允许存储多个值 不能在复合字段中使用。

列表聚合

列表聚合字段类似于 计算 和 模板 字段，但可以用于 存储多个值 的字段。其配置如下：

- uri: single_value
  tag: single_value
  name: Single Value
  type: RadioButtons
  options:
    options:
      - value: '10'
      - value: '20'
      - value: '30'
- uri: multi_values
  name: Multiple Values
  tag: multi_values
  type: Checkboxes
  options:
    options:
      - value: '40'
      - value: '50'
      - value: '60'
- uri: avg_value
  name: Average Value
  type: ListAggregate
  options:
    listOperation:
      operation: mean
      listTag: [single_value, multi_values]
    # errorValue: No values selected

或者当与 复合 字段一起使用时：

- uri: grades
  tag: grades
  name: Grades
  type: List
  options:
    item:
      type: Composite
      options:
        fields:
          - uri: name
            name: Name
            type: Text
            options:
              placeholder: 'Last, First'
          - uri: grade
            name: Grade
            type: Numeric
            options:
              unit: '%'
- uri: all_grades
  name: All grades
  type: ListAggregate
  options:
    listOperation:
      operation: concatenate
      joinWith: ', '
      listTag: grades
      displayItem: grade
    # errorValue: No grades added

当配置 options 字段时：

  • errorText 是可选的，默认为空字符串。
  • 在 listOperation 内部：
    • operation 可以是 concatenate、sum、min、max、mean 或 count。
    • listTag 是被聚合的字段的标签（或标签数组）。
    • joinWith 仅在操作为 concatenate 时需要。
    • displayItem 仅在聚合的字段是 复合 字段时需要，并且是嵌套字段的 URI。
