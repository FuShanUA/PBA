PySpark 转换

PySpark 是一种封装语言，允许你与 Apache Spark 后端进行交互以快速处理数据。Spark 可以在分布式服务器网络上处理非常大的数据集，这在正确使用时能带来显著的性能和可靠性优势。然而，它也有一些限制，尤其是如果你更习惯于关系型数据库系统如 SQL 的话。例如，Spark 无法确切知道某一行存在于哪台服务器上，因此无法直接选择特定行进行更新或删除。如果你习惯于这样思考你的数据库，那么你必须调整你的概念模型，将数据集整体看待，并基于列而不是行来处理数据。

  • DataFrame（数据框）: 是在 命名列（命名列） 下的一组行
    • 结构上类似于 SQL 数据库，但不是关系型的
    • 不可变（Immutable）: DataFrame 不能被修改
      • 更改后它被创建，但可以转换成一个新的DataFrame（导致两个DataFrame：原始的和转换后的）。数据集可以被覆盖，但Foundry会跟踪版本历史，以便您随时探索并返回到旧的构建版本。
    • 延迟计算：一系列转换任务被评估为一个（组合）操作，然后在触发构建时执行。
  • 弹性分布式数据集：（RDD）是DataFrame的基础数据结构。通过将DataFrame分成多个不相交的子集，可以在集群（计算机网络）中的多台计算机（节点）上并行评估转换。这一切都在后台发生，但在使用PySpark编写代码时需要注意。
  • 共享变量：默认情况下，Spark会将每个在转换任务中使用的变量的副本分别发送到每台并行计算机（节点）—这是为了效率。如果您必须在任务之间共享一个变量，Spark支持两种类型的共享变量：
    1. 广播变量： 缓存（保存）一个值到内存（RAM）中，该值会广播到集群中的所有计算机（节点）
    2. 累加器：可以进行添加或聚合的变量，包括（但不限于）计数器和求和。这一概念与GroupedData相关，对于统计计算很有用。
  • 为什么使用DataFrame？：Spark DataFrame专为处理大量结构化数据（超过太字节）而设计和优化。
  • 我为什么要编写PySpark代码？：PySpark使您能够以比单独使用Contour或Blacksmith更复杂和灵活的方式，在代码仓库和代码工作簿中自定义您想要转换数据集的方式。
  • PySpark不适用于什么？：PySpark用于转换数据集，而不是访问单个值。您可能能够计算总和和平均值，但不应该直接引用数据。

与SQL不同，SQL查询的结果是“视图”（虚拟表结果集），而使用PySpark处理数据集会生成全新的数据集。这使你不仅可以基于派生的数据集构建新的数据集，组织中的其他成员也可以将中间数据集用于他们自己的数据处理任务。在Palantir Foundry（一种数据操作系统）中，数据集通过父子（或源-结果）有向树关系自动关联。这使得任何人都可以轻松追踪Spark转换的数据血缘关系。换句话说，你可以探索你的数据集的依赖关系是如何构建的，以及这些数据集来自何处。你还可以发现组织中的其他成员是如何使用数据集的，这样你可以从示例中学习或有效地减少重复工作。

理解PySpark代码

初始代码基础

在代码工作簿中，你的函数可能看起来像这样：

Copied!
1def new_frame(old_frame):
2  df = old_frame
3  # df = transformations on df
4  return df
  • old_frame: 引用了代表存储在Foundry中的DataFrame（数据框）。old_frame是不可变的，这意味着它不能在此new_frame函数中被修改。从某种意义上说，所有转换的中间步骤都会生成一个新的、不可变的数据框，我们可能希望再次转换它或直接返回。这并不完全正确，但作为一种认知模型，它将帮助你更好地组织你的代码。
  • new_frame: 在这个函数中，你可以定义一系列想要应用于old_frame的转换。你的return语句应该返回一个DataFrame（在本例中我们将其称为df）。在内部，你对这个DataFrame所应用的每一个转换都会被组合并优化，然后再应用于输入数据集。一旦你用你的代码触发构建，结果就会保存到Foundry中的新数据集文件中，构建完成后你可以探索这些结果。

DataFrame 中的数据 不能直接引用，因为它既不是 Array 也不是 Dictionary。实际上，由于内部发生的大量分区和洗牌操作，无论如何都很难确定数据在任何时刻的位置。除非你正在对数据集进行筛选或聚合，否则你编写的代码应该相对 与数据集内容无关。排序通常成本高且速度慢，所以一般规则是假设每一行都是随机排列的，将你的工具集限制在列、过滤器、聚合和你自己的创造性解决问题方法上。

警告

非常重要的是，你要跟踪进入的列的模式，因为 PySpark 不是类型安全的，并且会尝试评估所有转换操作，并在运行时任何操作失败时中断。

不要对字符串或日期执行数学函数，或对数字执行字符串操作，或对整数执行日期操作，因为冲突类型的行為很难预测。

在对它们进行操作之前，请确保将值转换为正确的类型。

命名列

DataFrame 的每一列都有名称（并且可以重命名）。列名是唯一的，区分大小写。请遵循以下指南以用于 Foundry 数据集：

  • 始终使用小写字母或数字。
  • 用_（下划线）分隔单词，而不是空格（因为不允许使用空格）。
  • 根据惯例避免camelCasedColumnNames。
  • 永远不要使用特殊字符，例如(、) 或 &。
  • 聚合函数有时会自动使用特殊字符为列命名。在将数据框返回到您的转换中之前，您需要提供别名或重命名该列。

链接转换

当您进入现有的代码时，您会注意到并没有硬性规定如何命名引用DataFrame的变量。在本速查表中，DataFrame将被引用为df，但在其他示例中，它可能是raw、out、input、table、something_specific。只要能完成任务，任何名称都可以。 您还会注意到这个模式： 

Copied!
1df = df.select("firstName", "age")
2df = df.withColumn("age", df.age.cast("integer"))
3df = df.filter(df.age > 21)
4df = df.withColumnRenamed("firstName", "first_name")
5return df
或者（同样的内容，但写法不同）： 
Copied!
1return df.select("firstName", "age") \
2       .withColumn("age", df.age.cast("integer")) \
3       .filter(df.age > 21) \
4       .withColumnRenamed("firstName", "first_name")
如果您不熟悉编程：在=左侧的df是存储对右侧df应用的转换结果的地方，在继续执行下一行代码之前。在这个例子中，我们将结果存储在一个同名的变量中，本质上是在每一步之后覆盖df的内容。您可以使用不同的名称来保存DataFrame转换的结果，但在大多数情况下，覆盖变量名并继续下去是可以的。在每个转换函数的最后，我们必须返回新的dataframe作为变量（在第一个例子中）或作为最后一个转换的结果（在第二个例子中）。 两种示例完成相同的事情：

  1. 仅选择df的两列，这些列是我们想要包含在转换后的数据集中的
  2. 将age列进行类型转换，以确保它是整数而不是字符串。
  3. 筛选数据集的行，只包括age > 21的条目
  4. 将列firstName重命名为first_name

结果数据集将只包含两列first_name, age，并且21岁及以下的人被排除在外。这就是df最终包含的内容，您可以return它或对其应用更多转换。我们将在接下来的章节中更详细地探讨这些转换。

在Foundry中编写PySpark

在Foundry中编写PySpark有两种工具： 代码仓库（Code Repositories） 和 代码工作区（Code Workspaces） 。

在 代码仓库（Code Repositories） 中，为了使用大多数函数，您必须在.py文档的顶部声明以下导入语句：

Copied!
1from pyspark.sql import functions as F

在 代码笔记本，这是一个全局导入，已经包含在内，因此您可以无需额外配置即可使用大部分功能。
