认证和授权

本体论 MCP（OMCP）是基于 开发者控制台应用程序 的 OAuth 2.0 配置构建的，该配置将其暴露出来。当一个 MCP 客户端连接到您的本体论 MCP 服务器时，它使用相同的 OAuth 2.0 客户端和授权服务器进行身份验证，该服务器处理您应用程序的所有其他请求；MCP 没有需要配置的单独的身份验证系统。

您已经为开发者控制台应用程序定义的应用程序限制、权限和 OAuth 客户端设置也适用于通过本体论 MCP 发出的请求。颁发给 MCP 访问的令牌将限定于您授予应用程序的操作和资源。

支持的授权类型

本体论 MCP 支持 Foundry 为第三方应用程序提供的两种 OAuth 2.0 授权类型。选择与您的 MCP 客户端代表用户执行操作方式相匹配的授权类型：

  • 授权码授予：当MCP客户端应代表最终用户执行操作时，使用此授予类型。每个用户都会明确同意请求的作用域，生成的访问令牌将限定在Foundry中该用户的权限范围内。这是交互式代理和编辑器的适当选择，其中个别用户登录以访问本体论资源。
  • 客户端凭证授予：对于非交互式、服务到服务的工作流，当MCP客户端作为服务用户而不是特定最终用户代表执行操作时，使用此授予类型。客户端使用客户端ID和客户端密钥进行身份验证，因此必须是能够安全存储密钥的保密客户端。这是后端服务和集成（如Microsoft Copilot Studio）的适当选择。

您可以根据打算支持的MCP客户端，在同一开发者控制台应用程序中启用一种或两种授予类型。从开发者控制台中您的应用程序的OAuth & 权限页面为每种授予类型配置重定向URL、作用域和客户端密钥。

对于Foundry支持的OAuth 2.0流程、端点和参数的完整描述，请参阅 为Foundry编写OAuth2客户端。

作用域和限制

授予MCP客户端的访问令牌受到在开发者控制台应用程序上配置的 作用域 的限制。请确保应用程序被授予通过MCP公开的本体论资源所需的权限，并且请求用户或服务用户对底层对象、操作和查询具有必要的 权限。
