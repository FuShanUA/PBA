构件设置

如果您希望导入并使用 Python 库，请参阅《共享 Python 库》部分。

库选项卡包含可在代码仓库中引用的仓库列表，我们将其称为底层仓库。该列表包含了 Foundry 环境中所有的共享代码仓库，以及外部或公共库。您可以通过库选项卡来发现并添加底层仓库。

查看构件设置需要 artifacts:view-repository 权限，管理构件设置需要 artifacts:manage-repository 权限。

向代码仓库添加新构件

要添加新构件，请选择 添加，并从以下两种仓库类型中选择其一：

  1. 本地仓库：这些是 Foundry 环境中配置为共享库的其他代码仓库。
  2. 外部仓库：存储于 Foundry 环境外部的构件仓库。这些可以是外部的 Foundry 仓库，或者是您环境中可用的公共构件仓库。

如果添加的构件仓库包含对其他仓库的引用，这些仓库也会被一并添加。添加仓库的所有依赖项均需具备相同的访问权限。

从其他项目添加本地仓库时，系统会添加指向该仓库的项目引用。这要求您自己的代码仓库具备 compass:view-project-imports 和 compass:import-resource-to 权限，并且被引用的共享仓库具备 compass:import-resource-from 权限。

虽然可以对底层仓库进行重新排序和删除操作，但这可能会破坏使用了这些仓库中包的转换构建。请在权衡可能产生的影响后再执行此类操作。

Beta OSDK 生成器版本开关

启用 启用 Beta OSDK 版本 选项后，用户可以生成 Beta（候选发布）版本的本体生成器。该开关默认处于关闭状态。

要启用 Beta OSDK 版本：

  1. 导航至 分支 选项卡，确保您当前处于默认分支。
  2. 在仓库的 functions.json 文件中，将 "allowBetaGeneratorVersions": true 添加至该文件。若要查看此文件，可能需要通过选择文件标题顶部附近的齿轮图标来开启 显示隐藏文件 开关。

  1. 务必在 源代码控制 选项卡中提交此更改；否则，该更改将不会在设置中生效。
  2. 导航至 库设置，开启 Beta OSDK 版本 开关。
  3. 现在，您可以在 函数导入 选项卡中创建新版本时，通过 生成器 下拉菜单发现 Beta 生成器版本。
